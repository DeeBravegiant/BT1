# Q1230: ECC validation public key extraction mismatch in base_ec_core.h

## Question
Can an unprivileged attacker enter through `coinbase::api::schnorr_mp::sign_ac` with ac key blob, access_structure, digest, receiver, and peer messages after refresh but before public-key export, reach `include-internal/cbmpc/internal/crypto/base_ec_core.h` `base_ec_core module`, and use key_blob whose public-key extraction path and signing path parse different fields to bypass the requirement that exported public key is derived from the same validated state used by signing, causing caller authorizes one public key while protocol signs with another and producing an in-scope cb-mpc bounty impact?
