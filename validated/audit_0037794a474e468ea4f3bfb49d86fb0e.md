### Title
Hardcoded Test Private Key Used in Production Consensus Precommit Signing - (`File: crates/apollo_signature_manager/src/lib.rs`)

### Summary
The production `create_signature_manager()` function unconditionally instantiates `LocalKeyStoreSignatureManager` using `LocalKeyStore::new_for_testing()`, which embeds a well-known hardcoded ECDSA private key (`0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133`) in the binary. This key is used to sign consensus precommit votes and peer identity challenges. Because the key is publicly committed to the repository and is not gated by `#[cfg(test)]`, any party with repository access can forge valid precommit vote signatures for any block hash on behalf of any node running this code.

### Finding Description

`create_signature_manager()` is the sole production factory for the `SignatureManager` component: [1](#0-0) 

It calls `SignatureManager::new()`, which resolves to `LocalKeyStoreSignatureManager::new()`: [2](#0-1) 

That constructor calls `LocalKeyStore::new_for_testing()`, which is **not** behind `#[cfg(test)]`: [3](#0-2) 

The hardcoded private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` and its corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` are committed in plain text in the public repository.

The production node startup in `crates/apollo_node/src/components.rs` calls `create_signature_manager()` for both `LocalExecutionWithRemoteDisabled` and `LocalExecutionWithRemoteEnabled` modes — the latter being the mode used in the distributed deployment layout (`signature_manager.json`): [4](#0-3) 

The `SignatureManager` exposes two signing operations over this key:

- `sign_precommit_vote(block_hash)` — signs consensus precommit votes that finalize blocks
- `sign_identification(peer_id, challenge)` — signs peer identity challenges for P2P authentication [5](#0-4) 

The TODO comment in `lib.rs` explicitly acknowledges this is unresolved for production: [1](#0-0) 

### Impact Explanation

**High.** The `sign_precommit_vote` path is the consensus-critical signing operation. Because the private key is publicly known, any attacker can:

1. Compute a valid ECDSA signature over any `BlockHash` using the known private key.
2. Broadcast forged precommit votes attributed to any validator node running this code, since all such nodes share the same signing identity.
3. Forge peer identity challenge responses, allowing impersonation of any node using this key in P2P authentication.

If the validator set registers the corresponding public key (`0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a`) for one or more validators, an attacker can cast forged precommit votes for arbitrary block hashes on behalf of those validators, directly threatening consensus integrity. This matches the allowed impact: **"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

### Likelihood Explanation

The private key is committed in plain text in a public repository. No privilege is required to read it. The only precondition is that the production deployment registers the corresponding public key in the validator set, which is the natural outcome of deploying nodes that use this code path without substituting a real key.

### Recommendation

1. Remove `new_for_testing()` from non-test compilation paths. Gate it with `#[cfg(any(test, feature = "testing"))]`.
2. Implement a production `KeyStore` that loads the private key from an external secret (environment variable, mounted Kubernetes secret, or HSM), consistent with the existing `SECRET_CONFIGURATION.md` infrastructure.
3. Resolve the open TODO at `lib.rs:39` before any production deployment.
4. Rotate the exposed key pair; treat `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` as permanently compromised.

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);

    MessageDigest(blake2s_to_felt(&message))
}
