### Title
Single-Provider RPC Configuration Enables Compromised Oracle to Forge Foreign-Chain Verification Signatures — (File: `crates/contract/src/foreign_chain_rpc.rs`)

---

### Summary

The `ChainEntry` validation in the MPC contract permits a foreign chain to be configured with a single RPC provider and `quorum = 1`. Because each MPC node independently queries only its own configured providers, a compromised single provider feeds identical false data to every node. All nodes then sign the same fabricated payload hash, threshold signing succeeds, and the contract accepts the resulting signature — producing a valid MPC attestation for a non-existent or invalid foreign-chain transaction. This enables invalid bridge executions (e.g., minting NEAR-side tokens for a deposit that never occurred on the foreign chain).

---

### Finding Description

**Root cause — no minimum provider count enforced at vote time**

`TryFrom<dtos::ChainEntry> for ChainEntry` in `crates/contract/src/foreign_chain_rpc.rs` validates only that `quorum >= 1` and `quorum <= providers_len`. It places no lower bound on `providers_len` itself: [1](#0-0) 

A `ChainEntry` with exactly one provider and `quorum = 1` passes all checks and can be voted into the on-chain whitelist by threshold-many participants.

**Node-side fan-out provides no cross-validation with a single provider**

`FanOut::extract` in `crates/foreign-chain-inspector/src/lib.rs` queries every configured inspector in parallel and requires all successful responses to agree: [2](#0-1) 

When only one inspector is present (single provider), the "all agree" check is trivially satisfied by whatever that provider returns — including fabricated data.

**Contract cannot validate extracted values**

`respond_verify_foreign_tx` in `crates/contract/src/lib.rs` verifies only that the submitted signature is valid against the root public key and the caller-supplied `payload_hash`. It performs no validation of the `extracted_values` embedded in that hash: [3](#0-2) 

The contract has no way to verify what foreign-chain state the nodes actually observed; correctness is entirely delegated to the nodes' independent RPC queries.

**Each node queries its own providers independently — no cross-node oracle comparison**

The design explicitly states that each node queries its own locally configured providers independently: [4](#0-3) 

If every node's single provider is the same compromised endpoint, every node independently arrives at the same false extracted values, signs the same false `payload_hash`, and the threshold signing protocol succeeds normally.

**Attacker-controlled entry path**

1. Threshold-many participants vote in a `ChainEntry` for a chain (e.g., Ethereum) with one provider (`quorum = 1`, `providers = [alchemy]`).
2. Attacker compromises that provider's RPC endpoint (e.g., via a supply-chain attack or BGP hijack).
3. Attacker causes the provider to return a fabricated receipt for a non-existent transaction ID.
4. A user (or the attacker themselves) submits `verify_foreign_transaction` for that transaction ID.
5. Every MPC node queries the compromised provider, receives identical false data, and builds the same `ForeignTxSignPayload` from the fabricated `ExtractedValue`s.
6. Threshold signing succeeds; the leader submits `respond_verify_foreign_tx`.
7. The contract accepts the signature (valid against the root key) and resolves the yield with the false payload hash.
8. The attacker uses the returned signature to trigger an invalid bridge execution (e.g., Omnibridge mints tokens for a deposit that never happened).

No MPC participant collusion is required. The attacker only needs to compromise an external RPC provider that the network has legitimately whitelisted.

---

### Impact Explanation

A compromised single RPC provider causes the entire MPC network to produce a valid threshold signature attesting to false foreign-chain state. The resulting signature is indistinguishable from an honest one and can be used to trigger invalid bridge executions — direct theft of funds from the bridge. This maps to the allowed impact: **forged foreign-chain verification causing invalid bridge execution**.

---

### Likelihood Explanation

The attack requires two conditions:
1. Participants vote in a single-provider `ChainEntry` (no contract-level guard prevents this).
2. That provider is later compromised.

Major RPC providers (Alchemy, Infura, QuickNode) are high-value targets. A single-provider configuration is a natural bootstrap choice for a newly whitelisted chain. The combination is realistic, especially during early deployment when operator diversity is low.

---

### Recommendation

1. **Enforce a minimum provider count** in `TryFrom<dtos::ChainEntry> for ChainEntry`: reject any `ChainEntry` where `providers_len < 2` (or a configurable minimum). This mirrors the original report's recommendation to aggregate prices from multiple oracles.

2. **Enforce a meaningful quorum**: require `quorum >= 2` so that at least two independent providers must agree before a node accepts a result.

3. **Consider cross-node oracle diversity**: document (or enforce via governance) that different nodes should use different providers for the same chain, so a single compromised provider cannot simultaneously corrupt all nodes' views.

impl TryFrom<dtos::ChainEntry> for ChainEntry {
    type Error = ChainEntryValidationError;

    fn try_from(entry: dtos::ChainEntry) -> Result<Self, Self::Error> {
        let dtos::ChainEntry { providers, quorum } = entry;
        if quorum == 0 {
            return Err(ChainEntryValidationError::ZeroQuorum);
        }
        let providers_len = u64::try_from(providers.len()).map_err(|e| {
            ChainEntryValidationError::ProvidersLenOverflow {
                len: providers.len(),
                reason: e.to_string(),
            }
        })?;
        if quorum > providers_len {
            return Err(ChainEntryValidationError::QuorumExceedsProviders {
                quorum,
                providers_len,
            });
        }
        for (id, config) in providers.iter() {
            if let ChainRouting::PathSegment { segment } = &config.chain_routing
                && segment.contains('/')
            {
                return Err(ChainEntryValidationError::PathSegmentContainsSlash {
                    provider_id: id.0.clone(),
                });
            }
            if let (
                ChainRouting::QueryParam {
                    name: routing_name, ..
                },
                dtos::AuthScheme::Query { name: auth_name },
            ) = (&config.chain_routing, &config.auth_scheme)
                && routing_name == auth_name
            {
                return Err(ChainEntryValidationError::QueryParamCollidesWithAuth {
                    provider_id: id.0.clone(),
                    name: auth_name.clone(),
                });
            }
        }
        Ok(ChainEntry { providers, quorum })
    }

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
