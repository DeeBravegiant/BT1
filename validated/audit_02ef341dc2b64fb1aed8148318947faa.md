Audit Report

## Title
Unconditional `metricOmmSwapCallback` invocation with zero deltas causes `MetricOmmSimpleRouter` to revert on no-op swaps - (`metric-core/contracts/MetricOmmPool.sol`)

## Summary

`MetricOmmPool.swap()` unconditionally invokes `metricOmmSwapCallback` on `msg.sender` after every swap, including when `_executeSwap` returns `(0, 0, 0)`. `MetricOmmSimpleRouter.metricOmmSwapCallback()` reverts with `InvalidSwapDeltas` whenever `amount0Delta <= 0 && amount1Delta <= 0`, which is satisfied by `(0, 0)`. Any swap through the router that legitimately produces zero deltas — a documented, reachable state — permanently reverts.

## Finding Description

**Unconditional callback in `MetricOmmPool.swap()`:**

Both branches of the `zeroForOne` conditional fire the callback with no guard on whether either delta is non-zero: [1](#0-0) [2](#0-1) 

**Zero-delta path is reachable:** `_executeSwap` internal helpers explicitly return `(0, 0, 0)` when the price limit is immediately satisfied (4 such return sites confirmed in `MetricOmmPool.sol`): [3](#0-2) 

**Revert in router:** When the pool calls the callback with `(0, 0, callbackData)`, the condition `0 <= 0 && 0 <= 0` is `true` and `InvalidSwapDeltas` is thrown: [4](#0-3) 

This revert propagates back through `MetricOmmPool.swap()`, reverting the entire transaction.

**Contrast with liquidity path:** `LiquidityLib.addLiquidity` correctly guards its callback with `if (amount0Added > 0 || amount1Added > 0)`, demonstrating the intended pattern is known but not applied to the swap callback: [5](#0-4) 

**Execution path:**
1. User calls `MetricOmmSimpleRouter.exactInputSingle()` with a `priceLimitX64` at or beyond the current oracle price.
2. Pool calls `_executeSwap` → internal helper returns `(0, 0, 0)`.
3. Pool unconditionally calls `IMetricOmmSwapCallback(msg.sender).metricOmmSwapCallback(0, 0, "")`.
4. Router guard fires: `revert InvalidSwapDeltas()`.
5. Entire transaction reverts. [6](#0-5) 

## Impact Explanation

The swap path through `MetricOmmSimpleRouter` — the primary user-facing router — is broken for any input that produces zero deltas. This includes `exactInputSingle`, `exactOutputSingle`, and multi-hop `exactInput`/`exactOutput` paths where an intermediate hop produces zero deltas. This constitutes broken core pool/swap functionality, a directly allowed impact under the contest rules.

## Likelihood Explanation

The zero-delta case is reachable by any unprivileged caller via a standard `exactInputSingle` call. It occurs whenever the caller supplies a `priceLimitX64` that is at or beyond the current oracle mid-price, or when oracle price moves between transaction submission and execution, pushing the limit into the immediately-satisfied range. Both are normal live-trading conditions. No special privileges or malicious setup are required.

## Recommendation

**Option A (preferred — fix in pool):** Skip the callback entirely when both deltas are zero, mirroring the `LiquidityLib` pattern:

```solidity
// In MetricOmmPool.swap(), before firing the callback:
if (amount0Delta != 0 || amount1Delta != 0) {
    IMetricOmmSwapCallback(msg.sender).metricOmmSwapCallback(amount0Delta, amount1Delta, callbackData);
}
```

**Option B (fix in router):** Change the router guard to distinguish the zero-delta no-op case from a genuinely invalid callback:

```solidity
if (amount0Delta < 0 && amount1Delta < 0) revert InvalidSwapDeltas();
if (amount0Delta == 0 && amount1Delta == 0) return; // no-op swap, nothing to pay
```

## Proof of Concept

```solidity
// Deploy pool with token0/token1, add liquidity, deploy MetricOmmSimpleRouter.
// For !zeroForOne direction, set priceLimitX64 = 1 (immediately satisfied: 1 < initialPriceX64).

router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: address(this),
        zeroForOne: false,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 1,            // immediately satisfied
        tokenIn: address(token1),
        deadline: block.timestamp + 1,
        extensionData: ""
    })
);
// Expected: succeeds with zero output (no-op swap)
// Actual:   reverts with InvalidSwapDeltas() — pool calls callback with (0,0),
//           MetricOmmSimpleRouter.metricOmmSwapCallback checks 0<=0 && 0<=0 → true → revert
```
