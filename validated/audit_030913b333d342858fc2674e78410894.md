### Title
Unvalidated `builder` Field in `ProposalInit` Allows Fee Redirection to Arbitrary Address - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary
`ProposalInit.builder` is received over the P2P consensus stream and accepted by the validator without any check in `is_proposal_init_valid`. `convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in `BlockInfo`, which is the address that receives all transaction fees. Because both the proposer and the validator derive the `partial_block_hash` from the same proposer-supplied `init.builder`, the `ProposalFin` commitment check passes even when the field is set to an attacker-controlled address, and the block is committed with the attacker as the fee recipient.

### Finding Description

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`. The `builder` field is never checked. [1](#0-0) 

`convert_to_sn_api_block_info` then maps `init.builder` verbatim to `sequencer_address`: [2](#0-1) 

`sequencer_address` is the address credited with all transaction fees via `add_fee_to_sequencer_balance`: [3](#0-2) 

The `partial_block_hash` produced by the batcher includes `sequencer_address`. The proposer pre-computes `proposal_commitment` using the same malicious `builder`, so the final commitment comparison in `validate_proposal` passes: [4](#0-3) 

The proposer's own build path sets `builder_address` from local config with a TODO noting it should eventually come from the committee: [5](#0-4) 

### Impact Explanation

A malicious proposer sets `ProposalInit.builder` to an attacker-controlled address. The validator executes the block with that address as `sequencer_address`, so every fee transfer in the block credits the attacker. The `proposal_commitment` check passes because both sides compute `partial_block_hash` from the same proposer-supplied `init.builder`. The block is finalized and committed with the wrong fee recipient, producing an incorrect balance state. This matches: **Critical — Incorrect fee/balance effect with economic impact**.

### Likelihood Explanation

Requires a malicious proposer in the BFT validator set. In a multi-validator deployment any validator who wins a round can execute this. The TODO comment (`// TODO(Asmaa): Get it from committee once we have it.`) confirms the field is intentionally proposer-supplied and not yet committee-constrained, making the gap production-reachable today.

### Recommendation

Add a `builder` check inside `is_proposal_init_valid`. Until the committee-derived value is available, the validator should reject any `init.builder` that does not equal the locally configured `builder_address` (or a known-valid set). This mirrors how `l2_gas_price_fri` and `l1_da_mode` are already enforced against locally-derived reference values. [1](#0-0) 

### Proof of Concept

1. Malicious proposer constructs `ProposalInit` with `builder = 0xATTACKER`.
2. Validator receives it and calls `is_proposal_init_valid` — `builder` is not checked. [6](#0-5) 
3. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: 0xATTACKER, … }`. [7](#0-6) 
4. Batcher executes all transactions with `sequencer_address = 0xATTACKER`; every fee transfer calls `add_fee_to_sequencer_balance` crediting `0xATTACKER`.
5. Batcher returns `partial_block_hash` that commits to `sequencer_address = 0xATTACKER`.
6. Proposer pre-computed `proposal_commitment` using the same `builder = 0xATTACKER`, so `built_block == received_fin.proposal_commitment` — check passes. [4](#0-3) 
7. Block is committed; all fees from the block are in `0xATTACKER`'s balance; the legitimate sequencer receives nothing.

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
