Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns ETH/USD `updatedAt` While Serving a Frozen `rsETHPrice` During Protocol Pause - (File: `contracts/oracles/RSETHPriceFeed.sol`)

## Summary
`RSETHPriceFeed.latestRoundData()` computes a composite rsETH/USD price by multiplying the stored `rsETHPrice` from `LRTOracle` by the live ETH/USD Chainlink price, but returns `updatedAt` sourced solely from the ETH/USD feed. When `LRTOracle` auto-pauses due to a price drop beyond `pricePercentageLimit`, `rsETHPrice` is frozen while `updatedAt` continues to advance with the ETH/USD heartbeat, causing the contract to return a fresh-looking timestamp paired with a stale rsETH component.

## Finding Description
`latestRoundData()` forwards all five return values from `ETH_TO_USD.latestRoundData()` and then overwrites only `answer`:

```solidity
// contracts/oracles/RSETHPriceFeed.sol L63-70
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`RS_ETH_ORACLE.rsETHPrice()` reads the storage variable `uint256 public override rsETHPrice` in `LRTOracle`, which is only written at line 313 of `_updateRsETHPrice()`. That write is never reached when `_updateRsETHPrice()` detects a downside breach:

```solidity
// contracts/LRTOracle.sol L277-281
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated; function exits here
}
```

After `_pause()` sets `paused = true`, the public `updateRSETHPrice()` reverts on `whenNotPaused`, so `rsETHPrice` is permanently frozen until a manager calls `updateRSETHPriceAsManager()`. During this window — which has no bounded duration — every call to `latestRoundData()` returns:
- `updatedAt` = ETH/USD feed timestamp (advances every ~1 hour heartbeat, passes any standard staleness check)
- `answer` = frozen `rsETHPrice` × current ETH/USD (stale rsETH component)

The same defect exists in `getRoundData()`, which additionally applies the current `rsETHPrice` to a historical ETH/USD round, producing a nonsensical composite for any round other than the latest.

## Impact Explanation
**Low — Contract fails to deliver promised returns.** `RSETHPriceFeed` is deployed as a Chainlink `AggregatorV3Interface`-compatible oracle. Its core promise is that `updatedAt` reflects when the composite answer was last valid. During any auto-pause, this promise is broken: the contract returns a fresh `updatedAt` for a price whose rsETH component is arbitrarily stale. No funds are directly lost within this repository, but the contract does not deliver the staleness metadata it is designed to provide.

## Likelihood Explanation
The pause path is triggered automatically by the public `updateRSETHPrice()` function whenever on-chain price computation yields a drop exceeding `pricePercentageLimit` from `highestRsethPrice`. Any unprivileged caller can invoke `updateRSETHPrice()`. If market conditions cause the computed price to breach the threshold, the next public call to `updateRSETHPrice()` triggers the pause and immediately freezes `rsETHPrice`. The ETH/USD feed then continues updating normally. The condition is not exotic — it is the protocol's own downside-protection mechanism — and the staleness window persists until a manager manually intervenes.

## Recommendation
Introduce a `rsETHPriceUpdatedAt` storage variable in `LRTOracle` that is written alongside `rsETHPrice = newRsETHPrice` at line 313. Expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()`, return `min(ethUpdatedAt, rsETHPriceUpdatedAt)` as `updatedAt`. This ensures any consumer performing a standard Chainlink staleness check correctly detects when either component is stale.

## Proof of Concept
1. Deploy `LRTOracle` with `pricePercentageLimit` set (e.g., 1%).
2. Call `updateRSETHPrice()` once to establish `highestRsethPrice`.
3. Simulate a price drop beyond 1% (e.g., by manipulating underlying asset prices in a fork test).
4. Call `updateRSETHPrice()` again — `_updateRsETHPrice()` detects `isPriceDecreaseOffLimit = true`, calls `_pause()`, and returns without updating `rsETHPrice`.
5. Advance time by several ETH/USD heartbeat intervals (e.g., `vm.warp(block.timestamp + 3 hours)`).
6. Call `RSETHPriceFeed.latestRoundData()`. Assert:
   - `updatedAt` is within the last heartbeat (e.g., `block.timestamp - updatedAt < 3600`) — staleness check passes.
   - `answer` still encodes the pre-pause `rsETHPrice` — the rsETH component is 3+ hours stale.
7. Confirm `updateRSETHPrice()` reverts with `ContractPaused` throughout the window.
