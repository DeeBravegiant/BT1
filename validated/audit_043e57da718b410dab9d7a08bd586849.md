Audit Report

## Title
Unbounded Nested-Loop Gas Consumption in `updateRSETHPrice()` Due to Missing Length Cap on `supportedAssetList` — (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle._getTotalEthInProtocol()` iterates over every entry in `supportedAssetList` with no upper-bound guard, and for each asset calls `LRTDepositPool.getAssetDistributionData()`, which in turn calls `NodeDelegator.getAssetUnstaking()` once per NDC. Because `_addNewSupportedAsset()` imposes no maximum on `supportedAssetList.length`, the total number of external calls to `getQueuedWithdrawals` grows as O(assets × NDCs), and the inner withdrawal iteration adds further depth. When the combined gas cost exceeds the block gas limit, every call to the public `updateRSETHPrice()` reverts, permanently staling the rsETH price and disabling the auto-pause safety mechanism.

## Finding Description

**Confirmed call chain:**

`updateRSETHPrice()` (public, `whenNotPaused` only) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → outer loop over `supportedAssets.length` → `getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)` → middle loop over `nodeDelegatorQueue.length` → `getAssetUnstaking(asset)` → inner loop over `queuedWithdrawals.length`.

**Outer loop — no cap on `supportedAssetList`:**

`LRTConfig._addNewSupportedAsset()` pushes to `supportedAssetList` with no maximum-length guard:

```solidity
// LRTConfig.sol lines 114-115
isSupportedAsset[asset] = true;
supportedAssetList.push(asset);   // ← no length cap
```

`_getTotalEthInProtocol()` iterates the full list unconditionally:

```solidity
// LRTOracle.sol lines 336-348
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    ...
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    ...
}
```

**Middle loop — NDC queue (bounded by `maxNodeDelegatorLimit`, default 10):**

```solidity
// LRTDepositPool.sol lines 446-456
uint256 ndcsCount = nodeDelegatorQueue.length;
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset);
    ...
}
```

**Inner loop — EigenLayer queued withdrawals:**

```solidity
// NodeDelegator.sol lines 406-427
(IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
    _getDelegationManager().getQueuedWithdrawals(address(this));

for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
    ...
    for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
        ...
    }
}
```

**Why existing mitigations are insufficient:**

`LRTUnstakingVault.maxUncompletedWithdrawalCount` bounds the *total* number of queued withdrawals across all NDCs, which partially limits the inner loop. However, it does not address the dominant gas cost: `getAssetUnstaking()` makes one external call to `getQueuedWithdrawals()` per NDC *per asset*. With N assets and K NDCs, this produces N×K external calls to EigenLayer on every `updateRSETHPrice()` invocation, regardless of withdrawal count. Since N (assets) has no cap while K (NDCs) defaults to 10 and is admin-adjustable, the O(N×K) external call overhead grows unboundedly as the protocol onboards more LST assets — which is an explicitly designed and expected protocol behavior.

`maxNodeDelegatorLimit` bounds K but not N. No analogous `maxSupportedAssetLimit` exists.

## Impact Explanation

**Medium — Unbounded gas consumption / permanent freezing of unclaimed yield.**

Once the combined gas cost of N×K external calls (plus inner withdrawal iterations) exceeds the block gas limit, every invocation of the public `updateRSETHPrice()` reverts. Consequences:
1. `rsETHPrice` becomes permanently stale — new depositors receive an incorrect rsETH mint amount.
2. The downside-protection auto-pause in `_updateRsETHPrice()` can never trigger, removing a critical safety rail.
3. The fee-minting path (`protocolFeeInETH` → `IRSETH.mint()`) is permanently blocked, freezing unclaimed protocol yield — a concrete allowed impact.

## Likelihood Explanation

The protocol is explicitly designed to onboard additional LST assets over time via `addNewSupportedAsset()` (gated by `TIME_LOCK_ROLE`, a normal governance action). Each new asset multiplies the per-call external-call count by the number of active NDCs. With a realistic asset count of 10–15, the default NDC limit of 10, and any non-trivial pending withdrawal backlog, the block gas limit becomes reachable through ordinary protocol growth without any adversarial action. The trigger (`updateRSETHPrice()`) is callable by any unprivileged external account.

## Recommendation

1. **Add a maximum-length guard in `_addNewSupportedAsset()`** analogous to `maxNodeDelegatorLimit`:
   ```solidity
   if (supportedAssetList.length >= maxSupportedAssetLimit) revert MaxSupportedAssetLimitReached();
   ```
2. **Reduce the per-price-update call count for `getAssetUnstaking()`** — maintain an incremental accounting variable updated when withdrawals are queued and completed, rather than fetching and iterating all queued withdrawals on every price update.
3. **Enforce a combined cap** such that `maxSupportedAssetLimit × maxNodeDelegatorLimit` keeps `updateRSETHPrice()` well within the block gas limit under worst-case withdrawal backlog.

## Proof of Concept

1. Governance adds N supported assets via `LRTConfig.addNewSupportedAsset()` (no length cap enforced — confirmed at `LRTConfig.sol` line 115).
2. Admin adds K NDCs up to `maxNodeDelegatorLimit` via `LRTDepositPool.addNodeDelegatorContractToQueue()`.
3. Operators queue M EigenLayer withdrawals across the NDCs via `NodeDelegator.initiateUnstaking()` (bounded by `maxUncompletedWithdrawalCount` in total, but each NDC's share is fetched separately per asset).
4. Any external caller invokes `LRTOracle.updateRSETHPrice()`.
5. Execution enters `_getTotalEthInProtocol()` → N iterations → each calls `getTotalAssetDeposits()` → K NDC iterations → each calls `getAssetUnstaking()` (one `getQueuedWithdrawals` external call per NDC per asset = N×K external calls) → inner withdrawal iteration.
6. With N=15, K=10, and even a modest withdrawal backlog, the cumulative gas of N×K=150 external EigenLayer calls plus SLOAD-heavy inner loops exceeds 30M gas, causing an out-of-gas revert on every `updateRSETHPrice()` call.
7. `rsETHPrice` is never updated; the auto-pause safety mechanism is permanently disabled; protocol fee minting is frozen.

- Fork mainnet/Holesky with the deployed contracts.
- Call `addNewSupportedAsset()` 15 times via the timelock.
- Add 10 NDCs and queue several withdrawals per NDC.
- Call `updateRSETHPrice()` and measure gas via `vm.expectRevert` + `gasleft()` assertions to confirm out-of-gas reversion.
