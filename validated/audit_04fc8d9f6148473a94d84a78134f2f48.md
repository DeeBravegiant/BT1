Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` Due to Uncapped `supportedAssetList` Driving O(N×D×W) External Calls - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is publicly callable and internally executes a call chain whose gas cost scales as O(supportedAssets × NDCs × queuedWithdrawals). While NDCs and queued withdrawals are explicitly capped, `supportedAssetList` has no analogous cap and grows monotonically through normal governance. As the protocol onboards additional LSTs, the function's gas cost grows proportionally until it exceeds the block gas limit, permanently preventing price updates and rendering the rsETH exchange rate stale.

## Finding Description
`updateRSETHPrice()` is `public` and `whenNotPaused`, callable by any address. [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over every entry in `supportedAssets`: [2](#0-1) 

For each asset, it calls `ILRTDepositPool.getTotalAssetDeposits(asset)`: [3](#0-2) 

`getTotalAssetDeposits` calls `getAssetDistributionData`, which loops over every NDC and calls `getAssetUnstaking` per NDC: [4](#0-3) 

`getAssetUnstaking` calls EigenLayer's `getQueuedWithdrawals(address(this))` — one external call per (asset, NDC) pair — and then iterates over all returned withdrawals and their strategies, calling `strategy.sharesToUnderlyingView()` for each non-ETH strategy: [5](#0-4) 

The total gas cost is therefore **O(N × D × W)** where N = supported assets, D = NDCs, W = queued withdrawals per NDC.

NDCs are bounded by `maxNodeDelegatorLimit` (default 10): [6](#0-5) 

Queued withdrawals are bounded by `maxUncompletedWithdrawalCount` (max 80). However, `addNewSupportedAsset` imposes **no cap** on `supportedAssetList`: [7](#0-6) 

The list only grows via `supportedAssetList.push(asset)` with no upper-bound check: [8](#0-7) 

At N=50, D=10, W=80: 500 external calls to `getQueuedWithdrawals` plus up to 40,000 iterations of `sharesToUnderlyingView` — well beyond the Ethereum block gas limit (~30M gas).

## Impact Explanation
`rsETHPrice` is the sole on-chain exchange rate for rsETH. If `updateRSETHPrice()` becomes uncallable due to gas exhaustion, the price becomes permanently stale. Downstream consumers `LRTDepositPool.getRsETHAmountToMint()` and `LRTWithdrawalManager.getExpectedAssetAmount()` continue to use the stale price, mispricing all deposits and withdrawals. Additionally, the price-deviation circuit breaker in `_updateRsETHPrice()` can auto-pause the protocol if the price drifts too far from the stale value, causing a temporary freeze of deposits and withdrawals. **Impact: Medium — Unbounded gas consumption / Temporary freezing of funds.** [9](#0-8) 

## Likelihood Explanation
No attacker action is required. The degradation occurs through routine governance operation: each new LST onboarded via `addNewSupportedAsset` (TIME_LOCK_ROLE) linearly increases the gas cost of every `updateRSETHPrice()` call. The protocol is explicitly designed to onboard multiple LSTs over time. The condition is reached through expected protocol growth, not adversarial action. Likelihood is **Medium**.

## Recommendation
1. Introduce an explicit cap on `supportedAssetList` in `_addNewSupportedAsset`, analogous to `maxNodeDelegatorLimit` for NDCs.
2. Refactor `getAssetDistributionData` to avoid calling `getAssetUnstaking` (which calls `getQueuedWithdrawals`) inside the per-asset loop. Since `getQueuedWithdrawals` returns all withdrawals for an NDC regardless of asset, pre-aggregate unstaking amounts across all assets in a single pass per NDC rather than one pass per (asset, NDC) pair.
3. Consider separating TVL aggregation from the price-update path so that gas-heavy accounting does not block the critical price-update function.

## Proof of Concept
Call chain for any external caller:

```
LRTOracle.updateRSETHPrice()                          // public, whenNotPaused
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each asset in supportedAssetList:   // NO CAP — grows with protocol
                 └─ LRTDepositPool.getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ for each NDC in nodeDelegatorQueue:  // bounded by maxNodeDelegatorLimit (10)
                                └─ NodeDelegator.getAssetUnstaking(asset)
                                     └─ DelegationManager.getQueuedWithdrawals(ndc)  // 1 external call per (asset, NDC)
                                          └─ for each queued withdrawal:  // bounded by maxUncompletedWithdrawalCount (80)
                                               └─ strategy.sharesToUnderlyingView(...)  // external call
```

**Foundry fork test plan:**
1. Deploy the protocol on a mainnet fork.
2. Use the TIME_LOCK_ROLE to call `addNewSupportedAsset` in a loop, adding N mock LST tokens (each with a valid price oracle).
3. Add D=10 NDCs and queue W=80 withdrawals per NDC via `initiateUnstaking`.
4. Call `updateRSETHPrice()` with increasing N and measure gas via `vm.expectRevert` or `gasleft()` snapshots.
5. Observe gas consumption growing as O(N × D × W) and eventually reverting out-of-gas at the block gas limit.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }

    /// @dev private function to add a new supported asset
    /// @param asset Asset address
    /// @param depositLimit Deposit limit for the asset
    function _addNewSupportedAsset(address asset, uint256 depositLimit) private {
        UtilLib.checkNonZeroAddress(asset);
        if (depositLimit == 0) {
            revert InvalidDepositLimit();
        }
        if (isSupportedAsset[asset]) {
            revert AssetAlreadySupported();
        }
        isSupportedAsset[asset] = true;
        supportedAssetList.push(asset);
        depositLimitByAsset[asset] = depositLimit;
        emit AddedNewSupportedAsset(asset, depositLimit);
    }
