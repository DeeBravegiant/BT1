### Title
Unauthenticated Vote Acceptance — Any Network Peer Can Forge Committee Votes to Force Arbitrary Consensus Decision — (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`handle_vote` in `single_height_consensus.rs` checks that `vote.voter` is a known committee member address but **never verifies the cryptographic signature** on the vote. An unprivileged network peer can forge prevotes and precommits claiming to be from any committee member, accumulate a forged quorum, and force the consensus engine to decide on an attacker-chosen `ProposalCommitment`, causing the sequencer to commit a wrong block with wrong state, receipts, events, and L1 messages.

---

### Finding Description

In `handle_vote`, the only authorization guard is an address-membership check:

```rust
// TODO(Asmaa): verify the signature
if !self.committee.members().iter().any(|s| s.address == vote.voter) {
    debug!("Ignoring vote from non validator: vote={:?}", vote);
    return VecDeque::new();
}
``` [1](#0-0) 

This is the exact structural analog of the Phantasia bug: the code checks that a supplied identity (`vote.voter`) matches a stored value (committee member address), but never checks that the message was actually **signed** by the private key corresponding to that identity.

The `Vote` protobuf message carries a `signature` field:

```proto
Hashes signature = 7;
``` [2](#0-1) 

The verification function `verify_precommit_vote_signature` already exists in `signature_manager.rs`: [3](#0-2) 

But it is never called in the vote-handling path. The companion `make_self_vote` in the state machine confirms the signing side is also unimplemented — self-votes are emitted with an empty default signature:

```rust
// TODO(Asmaa): sign the vote
signature: RawSignature::default(),
``` [4](#0-3) 

Once a forged vote passes the address check it is fed directly into the state machine as a real vote:

```rust
let sm_vote = match vote.vote_type {
    VoteType::Prevote  => StateMachineEvent::Prevote(vote),
    VoteType::Precommit => StateMachineEvent::Precommit(vote),
};
self.state_machine.handle_event(sm_vote)
``` [5](#0-4) 

The state machine's `upon_decision` fires as soon as a 2/3 precommit quorum is accumulated for any `ProposalCommitment`: [6](#0-5) 

---

### Impact Explanation

**Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result.**

Once `SMRequest::DecisionReached(decision)` is emitted with an attacker-chosen `ProposalCommitment`, the consensus manager calls `decision_reached` on the context, which calls `batcher.decision_reached` and commits the block. The committed block's state diff, transaction receipts, events, and L1 messages all derive from the attacker-chosen commitment. Every downstream consumer (RPC, L1 bridge, provers) receives the corrupted view.

---

### Likelihood Explanation

**High.** The attack requires only:
1. P2P network connectivity to the target node (no privileged position).
2. Knowledge of the current committee (public, derived from staking state).
3. The ability to send crafted `Vote` messages with any `voter` address and any `signature` bytes.

No cryptographic material belonging to any committee member is needed. The attacker can forge votes from every committee member simultaneously, trivially exceeding the 2/3 quorum threshold in a single burst.

---

### Recommendation

Implement signature verification inside `handle_vote` before the vote is forwarded to the state machine. The committee entry should carry the validator's public key alongside its address and weight. Each incoming vote must be verified:

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // ... height check ...
    let Some(member) = self.committee.members().iter().find(|s| s.address == vote.voter) else {
        return VecDeque::new();
    };
    // NEW: verify the vote signature before accepting
    if !verify_precommit_vote_signature(
        vote.proposal_commitment_as_block_hash(),
        vote.signature.clone(),
        member.public_key,
    ).unwrap_or(false) {
        warn!("Invalid signature on vote from {:?}", vote.voter);
        return VecDeque::new();
    }
    // ... rest of handling ...
}
```

The signing side (`make_self_vote`) must also be completed so that legitimate votes carry valid signatures.

### Proof of Concept

    let forged_vote = Vote {
