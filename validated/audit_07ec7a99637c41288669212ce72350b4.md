### Title
Missing Lower-Bound Timestamp Guard in `is_proposal_init_valid` Allows Proposer-Controlled Stale Timestamps to Anchor Stale L1 Gas Prices into Committed Blocks — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` enforces a tight **upper** bound on `ProposalInit.timestamp` (at most `block_timestamp_window_seconds` = 1 s into the future) but has **no lower bound relative to `now`**. The only lower bound is `timestamp >= last_block_timestamp`. Because L1 gas prices are then looked up at `init_proposed.timestamp` — not at the validator's current clock — a malicious proposer can anchor the block to an arbitrarily old timestamp (as far back as the previous block), causing the committed block to carry stale L1 gas prices that the validator will accept.

---

### Finding Description

In `is_proposal_init_valid` the timestamp window is asymmetric:

```
lower bound: init_proposed.timestamp >= last_block_timestamp   // can be minutes in the past
upper bound: init_proposed.timestamp <= now + block_timestamp_window_seconds  // 1 s
``` [1](#0-0) 

After passing that check, the validator resolves L1 gas prices **at the proposer-supplied timestamp**, not at `now`:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,   // ← proposer-controlled stale value
    proposal_init_validation.previous_proposal_init.as_ref(),
    gas_price_params,
).await;
``` [2](#0-1) 

The validator then checks that the proposed L1 prices are within `l1_gas_price_margin_percent` of the prices it just fetched **at that same stale timestamp**: [3](#0-2) 

Because both sides of the comparison use the same stale timestamp, a proposer who deliberately sets `timestamp = last_block_timestamp + 1` and populates the price fields with the L1 prices that were current at that old time will always pass the margin check, regardless of how much L1 gas prices have moved since then.

The same stale `timestamp` is forwarded verbatim into `convert_to_sn_api_block_info` and ultimately into the `BlockInfo` handed to the blockifier: [4](#0-3) 

The `ProposalInit` struct carries the timestamp as a plain `u64` with no freshness constraint beyond the lower bound: [5](#0-4) 

**Queued-proposal amplification.** When a `ProposalInit` arrives for a future round it is stored in `queued_proposals` with its original `timeout` and `init` — no timestamp validation occurs at queueing time: [6](#0-5) 

When the round eventually arrives (potentially many round-timeouts later), `set_height_and_round` dequeues the proposal and calls `validate_current_round_proposal` with the original `init` and `timeout`: [7](#0-6) 

At dequeue time `now` has advanced, but `init.timestamp` (set when the proposal was created) is still `>= last_block_timestamp` and `<= now + 1 s`, so both timestamp checks pass. The L1 gas prices used for the block are those from the creation time, not the execution time.

---

### Impact Explanation

The committed block header carries:
1. A stale `block_timestamp` (potentially many seconds/minutes behind wall-clock time).
2. Stale `l1_gas_price` / `l1_data_gas_price` values (both FRI and WEI) derived from that old timestamp.

These values flow directly into every transaction's fee calculation inside the blockifier. If L1 gas prices have risen since the stale timestamp, users pay less than the current market rate — a direct, quantifiable revenue loss for the protocol. The wrong prices also propagate into the block hash commitment via `PartialBlockHashComponents`, producing a block hash that does not reflect current L1 conditions.

This matches the allowed impact: **"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

A single malicious or compromised proposer can trigger this in any round where it holds the proposer slot. No special network position is required beyond being the elected proposer. The attack is especially easy to mount via the queued-proposal path: send a future-round proposal with `timestamp = last_block_timestamp + 1` before the round begins; it will be queued without any timestamp check and validated later with stale prices.

---

### Recommendation

Add a symmetric lower-bound check in `is_proposal_init_valid` that mirrors the existing upper-bound check:

```rust
// Proposed addition after the existing lower-bound check:
if now > init_proposed.timestamp
    && now - init_proposed.timestamp > proposal_init_validation.block_timestamp_window_seconds
{
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "Timestamp is too far in the past: now={}, proposed={}, max_age={}",
            now,
            init_proposed.timestamp,
            proposal_init_validation.block_timestamp_window_seconds
        ),
    ));
}
```

This ensures the proposer-supplied timestamp is within `[now - window, now + window]`, preventing stale timestamps from anchoring stale L1 gas prices. The `block_timestamp_window_seconds` parameter (already serialised in config) can be tuned to accommodate legitimate clock skew without opening a large staleness window. [8](#0-7) 

---

### Proof of Concept

**Setup**: blocks produced every ~10 s; L1 gas price doubles between block N and block N+1.

1. Block N is committed at `T_prev`; `last_block_timestamp = T_prev`.
2. Current time is `T_now = T_prev + 60` (six round-timeouts have elapsed).
3. Malicious proposer constructs `ProposalInit` with:
   - `timestamp = T_prev + 1`
   - `l1_gas_price_fri = P_old` (L1 price at `T_prev + 1`, half of current)
   - `l1_data_gas_price_fri = D_old` (similarly stale)
4. Validator runs `is_proposal_init_valid`:
   - `T_prev + 1 >= T_prev` → **passes** lower bound.
   - `T_prev + 1 <= T_now + 1 = T_prev + 61` → **passes** upper bound.
   - `get_l1_prices_in_fri_and_wei(provider, T_prev + 1, ...)` returns `P_old`.
   - `within_margin(P_old, P_old, margin)` → **passes** price check.
5. Block N+1 is committed with `block_timestamp = T_prev + 1` and `l1_gas_price = P_old`.
6. Every transaction in block N+1 is charged at half the current L1 gas price; the protocol absorbs the difference. [9](#0-8) [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;

pub(crate) async fn get_l1_prices_in_fri_and_wei_and_conversion_rate(
    l1_gas_price_provider_client: Arc<dyn L1GasPriceProviderClient>,
    timestamp: u64,
    previous_proposal_init: Option<&PreviousProposalInitInfo>,
    gas_price_params: &GasPriceParams,
) -> (L1PricesInFri, L1PricesInWei, u128) {
    // One of these paths should fill the return values:
    // 1. Both L1 gas price and eth/strk rate are Ok, use those.
    // 2. Otherwise, use previous block info.
    // 3. If that isn't available either, use min gas prices and default eth/strk rate.

    // Get the eth to fri rate from the oracle, and the L1 gas price (in wei) from the provider.
    let (eth_to_fri_rate, price_info) = tokio::join!(
        l1_gas_price_provider_client.get_rate(timestamp),
        l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp))
    );
    if price_info.is_err() {
        warn!("Failed to get l1 gas price from provider: {:?}", price_info);
        CONSENSUS_L1_GAS_PRICE_PROVIDER_ERROR.increment(1);
    }
    if eth_to_fri_rate.is_err() {
        warn!("Failed to get eth to fri rate from oracle: {:?}", eth_to_fri_rate);
    }
    if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
        // Both L1 prices and rate are Ok, so we can use them.
        info!(
            "raw eth_to_fri_rate (from oracle): {eth_to_fri_rate}, raw l1 gas price wei (from \
             provider): {price_info:?}"
        );
        apply_fee_transformations(&mut price_info, gas_price_params);
        let prices_in_wei = L1PricesInWei {
            l1_gas_price: price_info.base_fee_per_gas,
            l1_data_gas_price: price_info.blob_fee,
        };
        // Apply the eth/strk rate to get prices in fri.
        let l1_gas_prices_fri_result =
            L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
        // If conversion fails, leave return_value=None to try backup methods.
        if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
            return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
        } else {
            warn!(
                "Failed to convert L1 gas prices to FRI: {:?}",
                l1_gas_prices_fri_result.clone().err()
            );
        }
    }
