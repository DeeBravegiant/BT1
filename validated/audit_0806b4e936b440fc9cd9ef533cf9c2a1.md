### Title
`estimate_minimal_gas_vector` Omits Archival Data (Calldata Storage) Cost, Allowing Fee Underpayment for Large-Calldata Transactions — (`File: crates/blockifier/src/fee/gas_usage.rs`)

### Summary

The `estimate_minimal_gas_vector` function, used as the lower-bound gate in `check_fee_bounds`, omits the archival data gas cost (calldata + signature storage, `gas_per_data_felt × calldata_length`). A user can craft a transaction with maximum-length calldata and set `l2_gas.max_amount` (or `l1_gas.max_amount`) to just above the incomplete minimal estimate. The transaction passes pre-validation, executes, then fails the post-execution bounds check. The fee charged on revert is capped at the (too-low) resource bounds, leaving the archival data cost entirely uncompensated.

### Finding Description

`estimate_minimal_gas_vector` computes only two components:

1. OS VM steps scaled by `extended_calldata_length` (via `os_resources_for_tx_type`) → converted to L2 gas (or L1 gas in `NoL2Gas` mode)
2. DA gas cost from state changes → L1 gas / L1 data gas [1](#0-0) 

It does **not** include the archival data cost charged by `ArchivalDataResources::get_calldata_and_signature_gas_cost`:

```
gas_amount = gas_per_data_felt × (calldata_length + signature_length)
``` [2](#0-1) 

With `gas_per_data_felt = 5120` (all current versioned constants) and `max_calldata_length = 5000`, the omitted archival data cost is up to **25,600,000 L2 gas units** per transaction.

`check_fee_bounds` uses this incomplete estimate as the sole lower-bound gate:

```rust
if minimal_gas_amount > &resource_bounds.max_amount { ... reject ... }
``` [3](#0-2) 

Because the archival data cost is absent from `minimal_gas_amount`, a user can set `l2_gas.max_amount = minimal_estimate + 1`, which passes `check_fee_bounds`. The actual gas charged by `TransactionResources::to_gas_vector` (which **does** include archival data) then exceeds the resource bounds. [4](#0-3) 

When post-execution check detects the overdraft, the transaction is reverted and the fee is set to `post_execution_report.recommended_fee()` — which is capped at the resource bounds, not the actual gas: [5](#0-4) 

The archival data cost (up to 25,600,000 L2 gas × `l2_gas_price`) is never collected.

The gateway's stateless `validate_resource_bounds` performs no minimal-gas check at all — it only verifies non-zero bounds, a price floor, and an upper cap: [6](#0-5) 

### Impact Explanation

**Critical — Incorrect fee/resource accounting with direct economic impact.**

The sequencer stores and processes up to 5,000 felt-words of calldata (archival data cost ≈ 25.6 M L2 gas) but collects a fee covering only the OS-step cost. The attacker pays a fraction of the true cost per transaction. Repeated at scale this drains sequencer revenue proportional to `(archival_data_cost − minimal_estimate) × l2_gas_price × tx_rate`.

### Likelihood Explanation

Any unprivileged user can submit a V3 (`AllResources`) or V1 (`L1Gas`) transaction. The attack requires only:
1. Setting `l2_gas.max_amount` (or `l1_gas.max_amount`) to the value returned by `starknet_estimateFee` with an empty calldata transaction (the minimal estimate), then
2. Attaching maximum-length calldata.

No special account, privileged key, or network position is required. The gateway's `max_calldata_length = 5000` is a public, documented limit.

### Recommendation

Include the archival data cost in `estimate_minimal_gas_vector`. The calldata and signature lengths are already available on the `AccountTransaction` object:

```rust
// In estimate_minimal_gas_vector, after computing vm_resources_cost:
let archival_data_resources = ArchivalDataResources {
    extended_calldata_length: tx.extended_calldata_length(),
    signature_length: tx.signature_length(),
    ..Default::default()
};
let archival_gas_cost = archival_data_resources
    .to_gas_vector(versioned_constants, gas_usage_vector_computation_mode);

da_gas_cost
    .checked_add(vm_resources_cost)
    .and_then(|v| v.checked_add(archival_gas_cost))
    .unwrap_or_else(|| panic!(...))
```

This mirrors the fix recommended in the Optimism report: include the data-length-dependent intrinsic cost in the lower bound.

### Proof of Concept

1. Obtain the minimal L2 gas estimate for a zero-calldata invoke transaction (e.g., via `starknet_estimateFee`). Call it `M`.
2. Construct a V3 invoke transaction with:
   - `calldata` = 5,000 arbitrary felt values
   - `resource_bounds.l2_gas.max_amount = M + 1`
   - `resource_bounds.l2_gas.max_price_per_unit` = current L2 gas price
3. Submit to the gateway. `validate_resource_bounds` passes (non-zero, price ≥ floor, amount ≤ cap).
4. Batcher picks up the transaction. `check_fee_bounds` passes because `minimal_gas_amount_vector.l2_gas ≤ M < M+1`.
5. Transaction executes. Actual L2 gas = `M + 5120×5000 = M + 25,600,000`.
6. `PostExecutionReport` detects `MaxGasAmountExceeded` (actual > M+1). Transaction reverts.
7. Fee charged = `(M+1) × l2_gas_price`. Archival data cost of `25,600,000 × l2_gas_price` is uncompensated. [7](#0-6) [8](#0-7) [9](#0-8) [6](#0-5)

/// Returns an estimated lower bound for the gas required by the given account transaction.
pub fn estimate_minimal_gas_vector(
    block_context: &BlockContext,
    tx: &AccountTransaction,
    gas_usage_vector_computation_mode: &GasVectorComputationMode,
) -> GasVector {
    // TODO(Dori, 1/8/2023): Give names to the constant VM step estimates and regression-test them.
    let BlockContext { block_info, versioned_constants, .. } = block_context;
    let state_changes_by_account_tx = match &tx.tx {
        // We consider the following state changes: sender balance update (storage update) + nonce
        // increment (contract modification) (we exclude the sequencer balance update and the ERC20
        // contract modification since it occurs for every tx).
        Transaction::Declare(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 0,
            // TODO(Yoni): BLOCKIFIER-RESET: should be 1.
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
        Transaction::Invoke(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 0,
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
        // DeployAccount also updates the address -> class hash mapping.
        Transaction::DeployAccount(_) => StateChangesCount {
            n_storage_updates: 1,
            n_class_hash_updates: 1,
            n_compiled_class_hash_updates: 0,
            n_modified_contracts: 1,
        },
    };

    // TODO(Yoni): BLOCKIFIER-RESET: reuse TransactionReceipt code.
    let data_segment_length = get_onchain_data_segment_length(&state_changes_by_account_tx);
    let os_steps_for_type = versioned_constants
        .os_resources_for_tx_type(&tx.tx_type(), tx.extended_calldata_length())
        .n_steps
        + versioned_constants.os_kzg_da_resources(data_segment_length).n_steps;

    let resources = ExtendedExecutionResources {
        vm_resources: ExecutionResources { n_steps: os_steps_for_type, ..Default::default() },
        ..Default::default()
    };
    let da_gas_cost = get_da_gas_cost(&state_changes_by_account_tx, block_info.use_kzg_da);
    let vm_resources_cost = get_extended_vm_resources_cost(
        versioned_constants,
        &resources,
        0,
        gas_usage_vector_computation_mode,
    );
    da_gas_cost.checked_add(vm_resources_cost).unwrap_or_else(|| {
        panic!(
            "Overflow in minimal gas estimation; attempted to add {da_gas_cost:?} to \
             {vm_resources_cost:?}"
        )
    })
}

    /// Returns the cost for transaction calldata and transaction signature. Each felt costs a
    /// fixed and configurable amount of gas. This cost represents the cost of storing the
    /// calldata and the signature on L2.
    fn get_calldata_and_signature_gas_cost(
        &self,
        versioned_constants: &VersionedConstants,
        mode: &GasVectorComputationMode,
    ) -> GasVector {
        let archival_gas_costs = versioned_constants.get_archival_data_gas_costs(mode);

        // TODO(Avi, 20/2/2024): Calculate the number of bytes instead of the number of felts.
        let total_data_size = u64_from_usize(self.extended_calldata_length + self.signature_length);
        let gas_amount =
            (archival_gas_costs.gas_per_data_felt * total_data_size).to_integer().into();

        match mode {
            GasVectorComputationMode::All => GasVector::from_l2_gas(gas_amount),
            GasVectorComputationMode::NoL2Gas => GasVector::from_l1_gas(gas_amount),
        }
    }

        &self,

        let fee_type = &tx_info.fee_type();
        match tx_info {
            TransactionInfo::Current(context) => {
                let resources_amount_tuple = match &context.resource_bounds {
                    ValidResourceBounds::L1Gas(l1_gas_resource_bounds) => vec![(
                        L1Gas,

                            tx_context.get_gas_prices(),
                            &tx_context.block_context.versioned_constants,
                        ),
                        block_info.gas_prices.l1_gas_price(fee_type),
                    )],
                    ValidResourceBounds::AllResources(AllResourceBounds {
                        l1_gas: l1_gas_resource_bounds,
                        l2_gas: l2_gas_resource_bounds,
                        l1_data_gas: l1_data_gas_resource_bounds,
                    }) => {
                        let GasPriceVector { l1_gas_price, l1_data_gas_price, l2_gas_price } =
                            block_info.gas_prices.gas_price_vector(fee_type);
                        vec![
                            (
                                L1Gas,
                                l1_gas_resource_bounds,
                                minimal_gas_amount_vector.l1_gas,
                                *l1_gas_price,
                            ),
                            (
                                L1DataGas,
                                l1_data_gas_resource_bounds,
                                minimal_gas_amount_vector.l1_data_gas,
                                *l1_data_gas_price,
                            ),
                            (
                                L2Gas,
                                l2_gas_resource_bounds,
                                minimal_gas_amount_vector.l2_gas,
                                *l2_gas_price,
                            ),
                        ]
                    }
                };
                let insufficiencies = resources_amount_tuple
                    .iter()
                    .flat_map(
                        |(resource, resource_bounds, minimal_gas_amount, actual_gas_price)| {
                            let mut insufficiencies_resource = vec![];
                            if minimal_gas_amount > &resource_bounds.max_amount {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasAmountTooLow {
                                        resource: *resource,
                                        max_gas_amount: resource_bounds.max_amount,
                                        minimal_gas_amount: *minimal_gas_amount,
                                    },
                                );
                            }
                            if resource_bounds.max_price_per_unit < actual_gas_price.get() {
                                insufficiencies_resource.push(
                                    ResourceBoundsError::MaxGasPriceTooLow {
                                        resource: *resource,
                                        max_gas_price: resource_bounds.max_price_per_unit,
                                        actual_gas_price: (*actual_gas_price).into(),
                                    },
                                );
                            }
                            insufficiencies_resource
                        },
                    )
                    .collect::<Vec<_>>();
                if !insufficiencies.is_empty() {
                    return Err(Box::new(TransactionFeeError::InsufficientResourceBounds {
                        errors: insufficiencies,
                    }))?;
                }
            }
            TransactionInfo::Deprecated(context) => {
                let max_fee = context.max_fee;
                let min_fee = get_fee_by_gas_vector(
                    block_info,
                    minimal_gas_amount_vector,
                    fee_type,
                    tx_context.effective_tip(),
                );
                if max_fee < min_fee {
                    return Err(TransactionPreValidationError::TransactionFeeError(Box::new(
                        TransactionFeeError::MaxFeeTooLow { min_fee, max_fee },
                    )));
                }
            }
        };
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)

                min_gas_price: self.config.min_gas_price,

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {

                gas_amount: resource_bounds.l2_gas.max_amount,
