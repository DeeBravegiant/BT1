Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns ETH/USD `updatedAt` for a Potentially Stale rsETH Price — (`contracts/oracles/RSETHPriceFeed.sol`)

## Summary

`RSETHPriceFeed.latestRoundData()` computes `answer` from `LRTOracle.rsETHPrice` (a stored, manually-updated value) but returns `updatedAt` verbatim from the live ETH/USD Chainlink feed. Because `updatedAt` always reflects the ETH/USD heartbeat, downstream consumers that check `updatedAt` for staleness will never detect that `rsETHPrice` itself is stale. An attacker can use block stuffing to delay the permissionless `updateRSETHPrice()` call after a slashing event, keeping the stored price inflated while `updatedAt` appears fresh.

## Finding Description

In `RSETHPriceFeed.latestRoundData()` (lines 63–70), `updatedAt` is taken directly from `ETH_TO_USD.latestRoundData()` while `answer` is computed from `RS_ETH_ORACLE.rsETHPrice()`:

```solidity
// contracts/oracles/RSETHPriceFeed.sol lines 68-69
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`RS_ETH_ORACLE.rsETHPrice()` resolves to `LRTOracle.rsETHPrice` (line 28 of `LRTOracle.sol`), a storage variable written only when `_updateRsETHPrice()` is called. No timestamp is stored alongside `rsETHPrice` in `LRTOracle`, and `RSETHPriceFeed` never checks one.

`updateRSETHPrice()` is public and permissionless (line 87 of `LRTOracle.sol`), meaning any address can call it, but block stuffing can prevent any transaction from landing for a window of N blocks.

**Attack sequence:**
1. A slashing event reduces the true rsETH/ETH rate by an amount within `pricePercentageLimit` (so the auto-pause at lines 270–282 of `LRTOracle.sol` does not trigger).
2. Attacker fills blocks with high-gas transactions, preventing `updateRSETHPrice()` from being included.
3. During this window, `latestRoundData()` returns `answer` = stale (pre-slash, inflated) `rsETHPrice` × current ETH/USD, and `updatedAt` = ETH/USD Chainlink heartbeat timestamp (fresh).
4. Any downstream consumer checking `updatedAt` for staleness sees a fresh timestamp and accepts the inflated rsETH/USD price.

The downside-protection pause (lines 270–282) only triggers if the price drop exceeds `pricePercentageLimit`. For small slashing events (e.g., 1–2%) within the limit, the protocol does not pause, and the stale price persists silently.

## Impact Explanation

The oracle fails to deliver the promised current rsETH/USD price: `updatedAt` does not reflect when `rsETHPrice` was last updated, so staleness detection by downstream consumers is silently bypassed. This maps to **Low. Block stuffing** — the attacker uses block stuffing to delay the price update, causing the oracle to misrepresent the freshness of the rsETH price. The `updatedAt` mismatch is also a structural flaw that exists independently of block stuffing (keeper downtime or network congestion also triggers it).

## Likelihood Explanation

Block stuffing on Ethereum mainnet costs approximately `block_gas_limit × base_fee` per block (~0.6 ETH/block at 20 gwei). For a 10-block window (~2 minutes), cost ≈ 6 ETH. Slashing events on EigenLayer restaked assets are a known, non-hypothetical risk. The `updatedAt` mismatch is a structural design flaw that is always present; block stuffing is one concrete mechanism to exploit it, but keeper downtime or network congestion also creates the same condition without any attacker cost.

## Recommendation

1. **Track rsETH price update time in `LRTOracle`:** Add a `rsETHPriceUpdatedAt` timestamp written alongside `rsETHPrice` in `_updateRsETHPrice()` (line 313 of `LRTOracle.sol`).
2. **Expose it via `IRSETHOracle`:** Add `rsETHPriceUpdatedAt()` to the `IRSETHOracle` interface used by `RSETHPriceFeed`.
3. **Return the correct `updatedAt` from `RSETHPriceFeed`:** Use `Math.min(ethUpdatedAt, RS_ETH_ORACLE.rsETHPriceUpdatedAt())` as `updatedAt` in `latestRoundData()`.
4. **Add a staleness guard inside `RSETHPriceFeed`:** Revert if `rsETHPriceUpdatedAt` is older than a configurable heartbeat (e.g., 24 hours), so downstream consumers receive a revert rather than a silently stale price.
