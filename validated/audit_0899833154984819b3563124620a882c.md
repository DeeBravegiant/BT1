Audit Report

## Title
Missing Time-Based Staleness Check Allows Stale Chainlink Prices to Be Accepted as Valid - (`contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol`)

## Summary

`ChainlinkOracleForRSETHPoolCollateral.getRate()` fetches `updatedAt` from Chainlink's `latestRoundData()` but never compares it against `block.timestamp`, leaving an effectively infinite staleness window. The only staleness guard present — `answeredInRound < roundID` — is a deprecated Chainlink pattern that Chainlink no longer guarantees to be meaningful. A stale collateral price passes all validation and is returned as current, causing rsETH to be minted at an incorrect rate for any depositor who calls during a feed outage.

## Finding Description

In `getRate()`, the oracle calls `latestRoundData()` and receives `updatedAt` aliased as `timestamp`. The three checks performed are:

```solidity
if (answeredInRound < roundID) revert StalePrice();   // deprecated, unreliable
if (timestamp == 0) revert IncompleteRound();          // only catches missing timestamp
if (ethPrice <= 0) revert InvalidPrice();              // price sign check only
``` [1](#0-0) 

There is no check of the form `block.timestamp - timestamp > MAX_STALENESS`. The `timestamp` value is fetched and then silently discarded. A price last updated hours or days ago passes all three guards and is returned as the current collateral rate. [2](#0-1) 

This oracle is consumed by `RSETHPoolV3` via `supportedTokenOracle[token]`, where `viewSwapRsETHAmountAndFee` calls `IOracle(supportedTokenOracle[token]).getRate()` to price collateral tokens and compute the rsETH mint amount: [3](#0-2) 

The same pattern exists in `RSETHPoolV3ExternalBridge` and `RSETHPoolV3WithNativeChainBridge`: [4](#0-3) [5](#0-4) 

The public `deposit(address token, uint256 amount, string referralId)` function is callable by any unprivileged user and directly triggers this code path: [6](#0-5) 

## Impact Explanation

**Concrete allowed impact: Protocol insolvency (Critical) / Contract fails to deliver promised returns (Low)**

When a Chainlink collateral feed goes stale and the true market price has fallen below the last reported price:
- The oracle returns the stale (inflated) price
- `viewSwapRsETHAmountAndFee` computes `rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate` using the inflated `tokenToETHRate`
- Users receive more rsETH than the deposited collateral is worth
- Repeated deposits during the stale window drain rsETH value from existing holders, creating protocol insolvency

If the true price has risen above the stale price, depositors receive fewer rsETH tokens than entitled — matching "Low: Contract fails to deliver promised returns."

## Likelihood Explanation

Chainlink feeds go stale during network congestion and, critically, during L2 sequencer outages — a well-documented occurrence on chains where these pool contracts are deployed. No privileged access is required; any external user can call `deposit()` with a supported collateral token. The window of exploitability lasts as long as the feed remains stale and the contract is unpaused. The `answeredInRound < roundID` guard provides no protection because Chainlink's data provider infrastructure no longer guarantees this relationship holds for freshness.

## Recommendation

Add a configurable `maxStaleness` parameter set per oracle instance (different feeds have different heartbeat intervals) and enforce it in `getRate()`:

```solidity
uint256 public immutable maxStaleness;

constructor(address _oracle, uint256 _maxStaleness) {
    oracle = _oracle;
    maxStaleness = _maxStaleness;
}

function getRate() public view returns (uint256) {
    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (block.timestamp - timestamp > maxStaleness) revert StalePrice(); // add this
    if (ethPrice <= 0) revert InvalidPrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

## Proof of Concept

1. Deploy a mock Chainlink aggregator that returns a fixed price with `updatedAt = block.timestamp - 4 hours` and `answeredInRound == roundID`.
2. Deploy `ChainlinkOracleForRSETHPoolCollateral` pointing to the mock.
3. Call `getRate()` — it returns the stale price without reverting (all three guards pass).
4. Register this oracle as `supportedTokenOracle[token]` in `RSETHPoolV3`.
5. Call `deposit(token, amount, "")` as an unprivileged user.
6. Observe that `wrsETH.mint` is called with an rsETH amount computed from the 4-hour-old price.
7. Repeat with a mock price 20% below the stale price to demonstrate over-minting and protocol insolvency.

Foundry fork test: fork an L2 mainnet at a block immediately after a historical sequencer outage, confirm the relevant Chainlink feed's `updatedAt` lags `block.timestamp` by more than the feed's heartbeat, and call `deposit()` to show the stale price is accepted. [1](#0-0)
