Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` via Nested External-Call Loop — (`contracts/LRTOracle.sol`)

## Summary
`updateRSETHPrice()` is publicly callable with only a `whenNotPaused` guard. Its internal call chain executes an `O(supportedAssets × nodeDelegatorQueue)` nested loop of cold external calls with no gas ceiling. As the protocol scales, this function can exceed the block gas limit, making on-chain price updates permanently unexecutable without admin intervention to shrink the queue.

## Finding Description

**Unrestricted entrypoint:**
`updateRSETHPrice()` carries only `whenNotPaused`; any EOA or contract may call it. [1](#0-0) 

**Outer loop — all supported assets:**
`_getTotalEthInProtocol()` iterates over every entry in `lrtConfig.getSupportedAssetList()` and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [2](#0-1) 

**Inner loop — LST assets, 3 cold calls per NDC:**
`getAssetDistributionData` loops over the full `nodeDelegatorQueue` and fires `IERC20.balanceOf`, `INodeDelegator.getAssetBalance`, and `INodeDelegator.getAssetUnstaking` per NDC per asset. [3](#0-2) 

**Inner loop — ETH asset, 2 cold calls per NDC:**
`getETHDistributionData` runs a separate loop over `nodeDelegatorQueue` calling `getEffectivePodShares` and `getAssetUnstaking` per NDC. [4](#0-3) 

**No hard ceiling on queue length:**
`maxNodeDelegatorLimit` defaults to 10 at initialization and can be raised by admin to any value; the only lower-bound check prevents shrinking below the current queue length, not growing above a safe gas threshold. [5](#0-4) 

**State write deferred to the end:**
`rsETHPrice` is committed only on the final line of `_updateRsETHPrice()`. An OOG revert at any earlier point leaves the stored value unchanged. [6](#0-5) 

**Deposit path reads stored price with no freshness guard:**
`getRsETHAmountToMint` divides by `lrtOracle.rsETHPrice()` — the raw storage slot — with no timestamp or staleness check. If `updateRSETHPrice()` becomes permanently unexecutable, every subsequent deposit mints at the last successfully stored price regardless of how stale it is. [7](#0-6) 

**Why existing checks are insufficient:**
The `whenNotPaused` modifier does not bound gas. `maxNodeDelegatorLimit` has no protocol-enforced upper ceiling tied to block gas. There is no gas-remaining check inside either loop, no partial-update mechanism, and no staleness guard in the deposit path.

## Impact Explanation

**Medium — Unbounded gas consumption.** The total external call count per `updateRSETHPrice()` invocation is `O(supportedAssets × nodeDelegatorQueue)`. Each iteration issues multiple cold CALL opcodes (~2,100 gas each for ERC-20 `balanceOf` on first access, plus the EigenLayer strategy calls). With the current default of 10 NDCs and several LST assets, the function already consumes significant gas; as the queue grows toward and beyond the admin-adjustable limit, the transaction will eventually exceed the 30M block gas limit, making the price update permanently unexecutable on-chain without admin queue reduction. This matches the allowed impact "Medium. Unbounded gas consumption."

## Likelihood Explanation

The loop grows with normal protocol operation — adding NDCs and LST assets is expected behavior. `updateRSETHPrice()` is called by off-chain keepers; if it begins reverting due to OOG, the price silently goes stale with no on-chain alarm. No attacker action is required to trigger the gas exhaustion; it emerges organically as the protocol scales. The absence of any hard gas-safe ceiling in the contract makes this a latent but structurally guaranteed failure mode.

## Recommendation

1. **Enforce a hard, gas-safe ceiling.** Add a constant `MAX_NDC_COUNT` and `MAX_ASSET_COUNT` derived from empirical gas measurements, and enforce them in `addNodeDelegatorContractToQueue` and the asset registration path.
2. **Add a gas-remaining guard inside the loop.** Check `gasleft()` at the top of each iteration and revert with a descriptive error if insufficient gas remains to complete the loop safely.
3. **Record a freshness timestamp.** Write `rsETHPriceLastUpdated = block.timestamp` alongside every successful `rsETHPrice` commit. In `getRsETHAmountToMint`, revert if `block.timestamp - rsETHPriceLastUpdated > MAX_STALENESS`.
4. **Consider off-chain TVL aggregation.** Compute the TVL sum off-chain and submit a signed result on-chain, eliminating the on-chain loop entirely and making gas cost constant.

import "forge-std/Test.sol";
import "../contracts/LRTOracle.sol";
import "../contracts/LRTDepositPool.sol";

contract UnboundedGasPoC is Test {
    LRTOracle  oracle;
    LRTDepositPool pool;

    function setUp() public {
        // fork + deploy / attach to existing proxy addresses
    }

    function testUnboundedGas() public {
        // 1. Measure gas at current queue size
        uint256 gasBefore = gasleft();
        oracle.updateRSETHPrice();
        uint256 gasUsed = gasBefore - gasleft();
        emit log_named_uint("gas used (baseline NDC count)", gasUsed);

        // 2. Admin adds NDCs up to maxNodeDelegatorLimit
        //    (or vm.store to inflate nodeDelegatorQueue.length)
        // ...

        // 3. Re-measure — gas grows linearly with queue length
        gasBefore = gasleft();
        oracle.updateRSETHPrice();
        uint256 gasUsedLarge = gasBefore - gasleft();
        emit log_named_uint("gas used (large NDC count)", gasUsedLarge);

        // 4. At sufficient queue size, call with block gas limit
        (bool ok,) = address(oracle).call{gas: 30_000_000}(
            abi.encodeWithSelector(oracle.updateRSETHPrice.selector)
        );
        assertFalse(ok, "expected OOG revert at block gas limit");

        // 5. rsETHPrice is unchanged (stale)
        // assert oracle.rsETHPrice() == stalePriceBefore
    }
}
