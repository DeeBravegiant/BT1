Audit Report

## Title
`instantWithdrawal()` Pays Out at Stale Pre-Slash `rsETHPrice`, Enabling Front-Running of EigenLayer Slashing Events — (File: contracts/LRTWithdrawalManager.sol)

## Summary

`instantWithdrawal()` computes the asset payout using the stored `rsETHPrice` state variable in `LRTOracle`, which is only updated on explicit calls to `updateRSETHPrice()`. When an EigenLayer slashing event reduces the NodeDelegator's withdrawable shares on-chain, a window exists between the slashing and the next oracle update during which any rsETH holder can call `instantWithdrawal()` at the inflated pre-slash rate, extracting more assets than their rsETH is worth and transferring the slashing loss entirely to remaining holders.

## Finding Description

`instantWithdrawal()` computes the payout via `getExpectedAssetAmount()`:

```solidity
uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
```

`getExpectedAssetAmount()` reads the stored `rsETHPrice` state variable directly:

```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```

`rsETHPrice` is a stored state variable set only inside `_updateRsETHPrice()`, which is only reachable via explicit calls to `updateRSETHPrice()` or `updateRSETHPriceAsManager()`. There is no forced price refresh inside `instantWithdrawal()`.

When EigenLayer slashing occurs, the NodeDelegator's withdrawable shares decrease immediately on-chain. `NodeDelegatorHelper.getAssetBalance()` calls `getDelegationManager().getWithdrawableShares()` — a live read — so `getTotalAssetDeposits()` and `_getTotalEthInProtocol()` would reflect the post-slash value if called. However, `rsETHPrice` remains at the pre-slash value until `updateRSETHPrice()` is explicitly invoked.

The oracle's downside-protection auto-pause is gated entirely inside `_updateRsETHPrice()`:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

Until that call happens, the protocol is not paused and `instantWithdrawal()` remains open at the stale price.

By contrast, the queued withdrawal path (`unlockQueue` → `_unlockWithdrawalRequests` → `_calculatePayoutAmount`) applies a minimum-of-locked-vs-current protection:

```solidity
uint256 currentReturn = (request.rsETHUnstaked * rsETHPrice) / assetPrice;
return (request.expectedAssetAmount < currentReturn) ? request.expectedAssetAmount : currentReturn;
```

`instantWithdrawal()` has no equivalent protection.

The `LRTUnstakingVault` holds physical assets already withdrawn from EigenLayer; its balance is unaffected by the slashing event itself. The check `assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)` only verifies physical availability, not fairness of the payout relative to the post-slash rsETH value. The vault therefore has sufficient assets to service the inflated withdrawal.

## Impact Explanation

An rsETH holder who observes an EigenLayer slashing event on-chain before `updateRSETHPrice()` is called can call `instantWithdrawal()` and receive assets valued at the pre-slash rsETH price. The slashing loss is then borne entirely by remaining rsETH holders when the oracle is eventually updated and the price drops. This constitutes a direct transfer of value from remaining rsETH holders to the attacker.

**Impact: High — Theft of unclaimed yield / value from remaining rsETH holders.**

## Likelihood Explanation

- EigenLayer slashing is an explicitly anticipated event; `SlashingLib` is imported and used in both `LRTUnstakingVault` and `NodeDelegatorHelper`.
- `updateRSETHPrice()` is not called atomically with slashing; there is always an observable on-chain delay.
- `rsETHPrice` is a public state variable; any observer can detect staleness by comparing it against a fresh `_getTotalEthInProtocol()` computation.
- `isInstantWithdrawalEnabled[asset]` must be `true`, but this is an operational configuration expected to be enabled for normal protocol operation.
- No special privileges are required beyond holding rsETH.
- The attack is repeatable across multiple slashing events and multiple attackers.

## Recommendation

1. In `instantWithdrawal()`, call `updateRSETHPrice()` (or an equivalent fresh price computation) before computing `assetAmountUnlocked`, so the payout always reflects the current post-slash state of the protocol.
2. Alternatively, apply the same `_calculatePayoutAmount()` minimum-of-locked-vs-current logic used in the queued withdrawal path: compute the payout at both the current oracle price and a freshly computed price, and use the lesser.
3. At minimum, add a single-block delay to `instantWithdrawal()` to prevent same-block front-running of oracle updates.

## Proof of Concept

1. EigenLayer slashing event is finalized on-chain; the NodeDelegator's withdrawable shares decrease, reducing the true value of rsETH. `NodeDelegatorHelper.getAssetBalance()` now returns a lower value via the live `getWithdrawableShares()` call.
2. `rsETHPrice` in `LRTOracle` is still the pre-slash value because `updateRSETHPrice()` has not yet been called.
3. Attacker (rsETH holder) calls `instantWithdrawal(asset, rsETHAmount, "")` before any call to `updateRSETHPrice()`.
4. `getExpectedAssetAmount()` computes `rsETHAmount * rsETHPrice_stale / assetPrice`, returning a larger asset amount than the post-slash fair value.
5. The check `assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)` passes because the vault holds physical assets unaffected by the slashing.
6. `unstakingVault.redeem(asset, assetAmountUnlocked)` transfers the inflated amount to `LRTWithdrawalManager`, which forwards it to the attacker.
7. `updateRSETHPrice()` is eventually called; `rsETHPrice` drops to reflect the slash. Remaining rsETH holders now hold rsETH worth less than before the slash, having absorbed the full slashing loss that the attacker escaped.

**Foundry fork test outline:**
- Fork mainnet at a block where the protocol is deployed and `isInstantWithdrawalEnabled` is true for an asset.
- Simulate a slashing event by directly reducing the NodeDelegator's EigenLayer shares (via `vm.store` on the strategy contract or by calling `burnOperatorShares` as the AllocationManager).
- Verify that `getExpectedAssetAmount()` still returns the pre-slash amount while `_getTotalEthInProtocol()` returns the post-slash amount.
- Call `instantWithdrawal()` as an unprivileged rsETH holder and confirm the inflated payout.
- Call `updateRSETHPrice()` and confirm the price drop, demonstrating the loss absorbed by remaining holders.
