### Title
Pool Wallet Joins Pool Without Checking Unclaimed Self-Pooling Rewards, Causing Permanent Reward Diversion — (`File: chia/pools/pool_wallet.py`)

### Summary

`PoolWallet.join_pool()` allows a farmer to transition from `SELF_POOLING` to `FARMING_TO_POOL` without first checking whether unclaimed coinbase rewards (unspent `p2_singleton` coins) exist. The code's own docstring explicitly warns this must be prevented, but the guard is absent. Any unclaimed rewards accumulated during self-pooling are subsequently absorbed to the pool's `target_puzzle_hash` instead of the farmer's address, causing permanent, irrecoverable loss of XCH.

### Finding Description

The `PoolWallet` class docstring explicitly states the risk:

> "If this wallet is in SELF_POOLING state, the coin ID associated with the current pool wallet contains the rewards gained while self-farming, so **care must be taken to disallow joining a new pool while we still have money on the pooling singleton UTXO**." [1](#0-0) 

Despite this warning, `join_pool()` performs no such check. It only guards against: a pending `target_state`, an unconfirmed transaction, and a lock-height constraint when leaving a pool. There is no guard that checks whether the wallet holds unspent coinbase `p2_singleton` coins before allowing the `SELF_POOLING → FARMING_TO_POOL` transition: [2](#0-1) 

The `get_confirmed_balance()` method confirms that coinbase rewards are tracked only while in `SELF_POOLING` state: [3](#0-2) 

Once the singleton transitions to `FARMING_TO_POOL`, `create_absorb_spend()` constructs the absorb spend using the **current** pool state's inner puzzle, which routes absorbed rewards to the pool's `target_puzzle_hash`: [4](#0-3) 

The absorb spend is permissionless — anyone (including the pool operator) can submit it. After the farmer joins the pool, the pool can absorb all previously unclaimed `p2_singleton` coins, and the XCH flows to the pool's address, not the farmer's.

### Impact Explanation

**High — Unauthorized diversion of XCH pool rewards.**

A farmer who has accumulated self-pooling rewards and then joins a pool without first claiming them will have those rewards permanently redirected to the pool's `target_puzzle_hash`. The pool operator (or any third party) can trigger the absorb spend permissionlessly. The farmer has no recourse once the singleton has transitioned states, because the `p2_singleton` puzzle enforces that only the current singleton (with its current inner puzzle) can claim the reward coins.

This matches the allowed High impact: *"Bypass of wallet, daemon, keychain, RPC, pool, or Data Layer authorization that enables unauthorized signing, key use, coin control, payout redirection, singleton mutation, or protected state transitions."*

### Likelihood Explanation

A malicious pool operator can advertise attractive terms to encourage farmers to join quickly. The farmer, unaware of unclaimed rewards (especially since `get_confirmed_balance()` returns 0 once the state changes), calls `join_pool()`. The pool then absorbs the pre-existing `p2_singleton` coins. This requires no special privileges — only the ability to construct a valid absorb spend bundle, which is fully documented in the pool protocol.

### Recommendation

Add a check in `join_pool()` that rejects the transition when the wallet is currently in `SELF_POOLING` state and has unspent coinbase coin records:

```python
async def join_pool(self, target_state: PoolState, fee: uint64, action_scope: WalletActionScope) -> uint64:
    ...
    current_state: PoolWalletInfo = await self.get_current_state()

    # ADD: Guard against losing unclaimed self-pooling rewards
    if current_state.current.state == SELF_POOLING.value:
        unspent = await self.wallet_state_manager.coin_store.get_unspent_coins_for_wallet(self.wallet_id)
        unclaimed = [r for r in unspent if r.coinbase]
        if unclaimed:
            raise ValueError(
                "Cannot join a pool while self-pooling rewards remain unclaimed. "
                "Please claim rewards first using pw_absorb_rewards."
            )
    ...
``` [2](#0-1) 

### Proof of Concept

1. Farmer creates a `PoolWallet` in `SELF_POOLING` state.
2. Farmer farms several blocks; `p2_singleton` reward coins accumulate at `p2_singleton_puzzle_hash`.
3. Farmer calls `pw_join_pool` (→ `join_pool()`) to transition to `FARMING_TO_POOL`. The call succeeds with no error despite unclaimed rewards.
4. Pool operator (or any node) calls `create_absorb_spend()` with the singleton now in `FARMING_TO_POOL` state. The absorb spend routes the previously accumulated `p2_singleton` coins to the pool's `target_puzzle_hash`.
5. Farmer's self-pooling rewards are permanently lost to the pool.

The `get_confirmed_balance()` method returning `0` once the state changes means the farmer's wallet UI shows no balance, masking the loss: [3](#0-2)

    async def join_pool(self, target_state: PoolState, fee: uint64, action_scope: WalletActionScope) -> uint64:
        if target_state.state != FARMING_TO_POOL.value:
            raise ValueError(f"join_pool must be called with target_state={FARMING_TO_POOL} (FARMING_TO_POOL)")
        if self.target_state is not None:
            raise ValueError(f"Cannot join a pool while waiting for target state: {self.target_state}")
        if await self.have_unconfirmed_transaction():
            raise ValueError(
                "Cannot join pool due to unconfirmed transaction. If this is stuck, delete the unconfirmed transaction."
            )

        current_state: PoolWalletInfo = await self.get_current_state()

        total_fee = fee
        if current_state.current == target_state:
            self.target_state = None
            msg = f"Asked to change to current state. Target = {target_state}"
            self.log.info(msg)
            raise ValueError(msg)
        elif current_state.current.state in {SELF_POOLING.value, LEAVING_POOL.value}:
            total_fee = fee
        elif current_state.current.state == FARMING_TO_POOL.value:
            total_fee = uint64(fee * 2)

        if self.target_state is not None:
            raise ValueError(
                f"Cannot change to state {target_state} when already having target state: {self.target_state}"
            )
        PoolWallet._verify_initial_target_state(target_state)
        if current_state.current.state == LEAVING_POOL.value:
            history: list[tuple[uint32, CoinSpend]] = await self.get_spend_history()
            last_height: uint32 = history[-1][0]
            if (
                await self.wallet_state_manager.blockchain.get_finished_sync_up_to()
                <= last_height + current_state.current.relative_lock_height
            ):
                raise ValueError(
                    f"Cannot join a pool until height {last_height + current_state.current.relative_lock_height}"
                )

        self.target_state = target_state
        self.next_transaction_fee = fee
        self.next_tx_config = action_scope.config.tx_config
        await self.generate_travel_transactions(fee, action_scope)
        return total_fee

    async def get_confirmed_balance(self, record_list: object | None = None) -> uint128:
        amount: uint128 = uint128(0)
        if (await self.get_current_state()).current.state == SELF_POOLING.value:
            unspent_coin_records: list[WalletCoinRecord] = list(
                await self.wallet_state_manager.coin_store.get_unspent_coins_for_wallet(self.wallet_id)
            )
            for record in unspent_coin_records:
                if record.coinbase:
                    amount = uint128(amount + record.coin.amount)
        return amount
