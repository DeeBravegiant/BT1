Audit Report

## Title
Permissionless `verifyStaleBalance` + `verifyCheckpointProofs` + `updateRSETHPrice` Sequence Triggers Auto-Pause, Temporarily Freezing Unlocked Withdrawals ā€” (`contracts/LRTOracle.sol`, `contracts/LRTWithdrawalManager.sol`)

## Summary

An unprivileged attacker can call the permissionless EigenPod functions `verifyStaleBalance` and `verifyCheckpointProofs` to finalize a checkpoint reflecting a slashed validator's reduced balance, then immediately call the public `LRTOracle.updateRSETHPrice()` to trigger the downside auto-pause. This pauses `LRTWithdrawalManager`, blocking `completeWithdrawal` for users who already have unlocked requests. The freeze is temporary ā€” an admin can unpause at any time ā€” but users with ready-to-claim withdrawals are blocked until admin intervention.

## Finding Description

**Permissionless EigenPod entry points confirmed in code:**

`IEigenPod.verifyStaleBalance` NatSpec explicitly states: *"this method allows anyone to start a checkpoint as soon as a slashing occurs on the beacon chain."* [1](#0-0) 

`IEigenPod.verifyCheckpointProofs` NatSpec states: *"Anyone can call this method to submit proofs towards the current checkpoint."* [2](#0-1) 

**Share reduction propagates to rsETH price:**

After checkpoint finalization, EigenLayer reduces the NDC's withdrawable shares. `NodeDelegator.getEffectivePodShares()` reads those shares directly via `NodeDelegatorHelper.getWithdrawableShare()`, which calls `DelegationManager.getWithdrawableShares()`. [3](#0-2) 

`LRTDepositPool.getETHDistributionData()` sums `getEffectivePodShares()` across all NDCs, feeding into `getTotalAssetDeposits` which `LRTOracle._getTotalEthInProtocol()` uses to compute `newRsETHPrice`. [4](#0-3) 

**Auto-pause on price drop:**

`LRTOracle._updateRsETHPrice()` compares `newRsETHPrice` against `highestRsethPrice`. If the drop exceeds `pricePercentageLimit`, it pauses `LRTDepositPool`, `LRTWithdrawalManager`, and itself in a single call. [5](#0-4) 

`updateRSETHPrice()` is public and callable by anyone when not paused. [6](#0-5) 

**Withdrawal completion blocked:**

`completeWithdrawal` carries `whenNotPaused`, so users with already-unlocked requests (whose `expectedAssetAmount` was fixed at `unlockQueue` time) cannot complete them while the pause is active. [7](#0-6) 

**Unpause requires admin:**

`LRTWithdrawalManager.unpause()` is restricted to `onlyLRTAdmin`, meaning the freeze persists until admin action. [8](#0-7) 

## Impact Explanation

**Medium ā€” Temporary freezing of funds.** Users with already-unlocked withdrawal requests (where `expectedAssetAmount` is fixed) cannot call `completeWithdrawal` while the pause is active. No funds are permanently lost ā€” the admin unpause path is always available and requires no price condition. Once unpaused, users receive the amount they were promised. The slashing loss itself is real and would eventually be reflected regardless; the attacker only accelerates the price update timing.

## Likelihood Explanation

Requires a real beacon-chain slashing event on one of the NDC's validators (not attacker-controlled) and `pricePercentageLimit` set to a value the slashing drop exceeds ā€” both plausible in production. The three attacker calls (`verifyStaleBalance`, `verifyCheckpointProofs`, `updateRSETHPrice`) are all permissionless and can be batched atomically. The attacker bears no cost beyond gas. The attack is repeatable after each admin unpause if the slashed validator's balance remains unresolved.

## Recommendation

1. **Separate withdrawal completion from oracle pause:** Allow `completeWithdrawal` for already-unlocked requests (where `expectedAssetAmount` is already fixed at unlock time) even when the withdrawal manager is paused, since those requests carry no price risk. A separate `unlockedWithdrawalsOnly` modifier or a bypass flag for requests with `nonce < nextLockedNonce` would suffice.
2. **Delay price-drop auto-pause:** Require the price to remain below the threshold for N consecutive `updateRSETHPrice` calls or a time window before pausing, preventing a single atomic transaction from triggering the pause.
3. **Rate-limit public `updateRSETHPrice`:** Add a minimum time interval between successive public calls (e.g., 1 hour), while preserving the manager's ability to call `updateRSETHPriceAsManager` at any time.

// Foundry fork test (mainnet fork, post-slashing block)
function testAttackerTriggersAutoPause() external {
    // Preconditions:
    // - NDC has an ACTIVE slashed validator (beacon chain slashing occurred)
    // - No active checkpoint on the EigenPod
    // - pricePercentageLimit is set (e.g., 1e16 = 1%)
    // - LRTWithdrawalManager has at least one unlocked withdrawal request

    address attacker = makeAddr("attacker");
    IEigenPod pod = NodeDelegator(ndc).eigenPod();

    vm.startPrank(attacker);
    // Step 1: Start checkpoint via staleness proof (permissionless)
    pod.verifyStaleBalance(beaconTimestamp, stateRootProof, validatorProof);
    // Step 2: Finalize checkpoint with slashed balance (permissionless)
    pod.verifyCheckpointProofs(balanceContainerProof, balanceProofs);
    // Step 3: Trigger auto-pause via public oracle update
    LRTOracle(oracle).updateRSETHPrice();
    vm.stopPrank();

    assertTrue(LRTWithdrawalManager(withdrawalManager).paused());

    // Step 4: User with unlocked request cannot complete withdrawal
    vm.prank(user);
    vm.expectRevert("Pausable: paused");
    LRTWithdrawalManager(withdrawalManager).completeWithdrawal(asset, "");
}
