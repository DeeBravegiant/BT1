### Title
Unvalidated `ProposalInit.builder` Lets Any Proposer Inject Arbitrary `sequencer_address` into Block Execution and Block Hash — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address). Validators never check this field in `is_proposal_init_valid`. Because `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, any legitimate proposer can set `builder` to an arbitrary address. That address flows into block execution (affecting `get_sequencer_address` syscall results and fee collection) and into `PartialBlockHashComponents.sequencer`, corrupting the block hash that is committed to storage and L1.

---

### Finding Description

`ProposalInit` contains two distinct address fields:

- `proposer` — the consensus identity, validated against the committee in `handle_proposal` in `manager.rs`
- `builder` — the sequencer/block-builder address, **never validated** [1](#0-0) 

`is_proposal_init_valid` checks `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 gas prices, and `fee_proposal_fri`. It does not check `builder` at all. [2](#0-1) 

`convert_to_sn_api_block_info` then maps the proposer-supplied `init.builder` directly to `sequencer_address` in the `BlockInfo` that is handed to the batcher: [3](#0-2) 

That `BlockInfo` is passed to the batcher via `initiate_validation` → `ValidateBlockInput.block_info`: [4](#0-3) 

Inside the batcher, `BlockExecutionArtifacts::new` calls `PartialBlockHashComponents::new(&block_info, header_commitments)`, which stores `block_info.sequencer_address` as the `sequencer` field: [5](#0-4) 

`sequencer` is then hashed directly into the block hash: [6](#0-5) 

Because both the proposer and every validator derive `sequencer_address` from the same `init.builder` value, the `ProposalCommitment` comparison at the end of `validate_proposal` passes — the mismatch is invisible to the protocol: [7](#0-6) 

---

### Impact Explanation

**Wrong execution result (Critical):** Every transaction in the block that calls the `get_sequencer_address` syscall receives the attacker-chosen address instead of the real sequencer address. Contracts that gate logic on the sequencer address (e.g., fee-token contracts, operator-gated contracts) produce wrong results that are committed as canonical state.

**Wrong block hash / state commitment (Critical):** `sequencer_address` is a direct input to `calculate_block_hash`. A forged `builder` value produces a different `PartialBlockHash`, a different `ProposalCommitment`, and ultimately a different on-chain block hash. All downstream commitments (L1 anchoring, proof verification, state sync) are built on a corrupted value.

**Fee misdirection (Critical):** Sequencer fees collected during block execution are credited to the attacker-supplied address, not the legitimate sequencer.

---

### Likelihood Explanation

Any node that is legitimately selected as proposer for a round can trigger this. Proposer selection is weighted-random from the validator committee, so any committee member can exploit this without any special privilege beyond being selected. No external setup is required — the attacker simply sets `builder` to an arbitrary address in the `ProposalInit` they broadcast.

---

### Recommendation

Add a check for `builder` inside `is_proposal_init_valid`. The validator node knows its own expected `builder_address` from `ContextStaticConfig`. Reject any `ProposalInit` whose `builder` does not equal the locally configured `builder_address`:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder_address,
            init_proposed.builder
        ),
    ));
}
```

Add `expected_builder_address: ContractAddress` to `ProposalInitValidation` and populate it from `self.config.static_config.builder_address` in `validate_proposal` inside `sequencer_consensus_context.rs`. [8](#0-7) [9](#0-8) 

---

### Proof of Concept

1. Attacker node is selected as proposer for height H, round R.
2. In `initiate_build`, the attacker overrides `builder_address` in their local `ProposalBuildArguments` (or patches the binary) to set `init.builder = ContractAddress(0xdeadbeef)`.
3. The attacker broadcasts `ProposalPart::Init(init)` with `builder = 0xdeadbeef` to all validators.
4. Each validator calls `is_proposal_init_valid` — no check on `builder`, so the proposal passes.
5. Each validator calls `initiate_validation` → `convert_to_sn_api_block_info(init)` → `sequencer_address = 0xdeadbeef`.
6. The batcher executes the block with `sequencer_address = 0xdeadbeef`; all `get_sequencer_address` syscalls return `0xdeadbeef`.
7. `PartialBlockHashComponents::new` stores `sequencer = SequencerContractAddress(0xdeadbeef)`.
8. `PartialBlockHash::from_partial_block_hash_components` produces a hash that includes `0xdeadbeef`.
9. The proposer's `ProposalFin.proposal_commitment` was computed with the same `builder`, so `built_block == received_fin.proposal_commitment` — the mismatch check passes.
10. All validators vote to commit the block. The canonical block hash, all receipts, and all `get_sequencer_address` results are permanently wrong. [10](#0-9) [11](#0-10) [12](#0-11)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
