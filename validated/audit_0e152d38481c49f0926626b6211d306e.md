### Title
Missing Vote Signature Verification Allows Any Peer to Forge Validator Votes and Manipulate Consensus Decisions - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`handle_vote` in `SingleHeightConsensus` accepts votes from any network peer claiming to be any committee member without verifying the cryptographic signature on the vote. The `Vote` struct carries a `signature: RawSignature` field and a dedicated `verify_precommit_vote_signature` function exists in `apollo_signature_manager`, but neither is called during vote processing. Simultaneously, the state machine emits its own votes with `RawSignature::default()` (all-zero bytes). Any peer reachable over the p2p gossip channel can forge 2/3+ of committee precommits for an arbitrary `ProposalCommitment`, causing `DecisionReached` to fire for a block the honest validators never agreed on.

### Impact Explanation

An attacker who can inject messages into the p2p gossip broadcast channel (any connected peer) can:

1. Enumerate the committee member addresses from the public validator set.
2. Craft `Vote` messages with `voter` set to each committee member's `ContractAddress` and `signature` set to `RawSignature::default()` (or any bytes — they are never checked).
3. Broadcast forged prevotes and precommits for an arbitrary `ProposalCommitment` value.
4. Because `handle_vote` only checks membership by address, all forged votes are accepted and counted toward quorum.
5. Once 2/3+ weighted precommits accumulate for the chosen commitment, `upon_decision` fires `SMRequest::DecisionReached`, committing a block the honest validators never validated. [6](#0-5) 

The decided `ProposalCommitment` propagates to `decision_reached` in the orchestrator, which calls `batcher.decision_reached` and `state_sync_client.add_new_block`, writing the attacker-chosen block hash into the canonical chain. This produces wrong state roots, wrong transaction/event/receipt commitments, and wrong L1 anchoring data — matching the **Critical: Wrong state, receipt, event, L1 message, class hash, storage value** impact category.

### Likelihood Explanation

The p2p vote broadcast channel is accessible to any peer that can connect to the sequencer's libp2p endpoint. The `voter` field is a Starknet `ContractAddress`, not a libp2p peer ID; there is no binding between the transport-layer peer identity and the claimed validator address. Committee addresses are public (derived from the staking contract). The attack requires no private key material and no privileged access — only network reachability and knowledge of committee addresses.

### Recommendation

1. **Implement signing**: Replace `RawSignature::default()` in `make_self_vote` with a call to `SignatureManager::sign_precommit_vote(block_hash)` before broadcasting.
2. **Implement verification**: In `handle_vote`, after confirming the voter is a committee member, look up the member's registered public key and call `verify_precommit_vote_signature(block_hash, vote.signature, public_key)`. Reject votes that fail verification.
3. **Extend to prevotes**: The current `verify_precommit_vote_signature` only covers precommits. A parallel signing/verification scheme is needed for prevotes.
4. **Remove the TODOs** only after both sides are wired end-to-end and covered by integration tests that inject forged signatures and assert rejection.

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
