### Title
Hardcoded Test Private Key Used as Production Consensus Signing Key - (File: `crates/apollo_signature_manager/src/lib.rs`)

### Summary
The production `SignatureManager` component, which signs consensus precommit votes (block hash commitments) and peer identity challenges, is instantiated using a hardcoded well-known test private key. Because the key is embedded in public source code, any party can forge valid precommit vote signatures for arbitrary block hashes, directly compromising consensus integrity.

### Finding Description

`create_signature_manager()` is the production factory function called by the node's component initialization path. It constructs a `SignatureManager` backed by `LocalKeyStore::new_for_testing()`: [1](#0-0) 

`LocalKeyStore::new_for_testing()` hardcodes a fixed, publicly visible STARK private key: [2](#0-1) 

The private key `0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133` and its corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` are committed in the repository and publicly known.

This `create_signature_manager()` is called unconditionally in the production node's component setup: [3](#0-2) 

The `SignatureManager` uses this key to sign two critical consensus messages:

1. **Precommit votes** — `sign_precommit_vote(block_hash)` signs the block hash that a validator commits to in the final Tendermint round: [4](#0-3) 

2. **Peer identity challenges** — `sign_identification(peer_id, challenge)` authenticates the validator's network identity: [5](#0-4) 

The developers themselves acknowledge this is unresolved with a TODO: [1](#0-0) 

### Impact Explanation

Because the private key is publicly known, any party can:
- Compute valid ECDSA signatures over arbitrary block hashes using the known key.
- Forge precommit votes attributed to the validator node running this code.
- Forge peer identity responses, allowing impersonation of the validator at the network layer.

This directly affects consensus decisions: a forged precommit vote for a wrong block hash, if accepted by the consensus engine, corrupts the committed block hash and all downstream state commitments derived from it.

**Impact category:** High — "Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."

### Likelihood Explanation

The private key is embedded in a public repository. Any observer of the source code can extract it immediately. No privileged access, network position, or special capability is required. The trigger is unprivileged: anyone can call `ecdsa_sign` with the known key against any block hash.

### Recommendation

1. Remove `LocalKeyStore::new_for_testing()` from the production code path entirely. The `_new` constructor already exists for production use but is currently unused.
2. Load the validator signing key from a secret management system (e.g., Kubernetes Secret / ExternalSecret, HSM, or an encrypted file) at startup, consistent with how `NetworkConfig.secret_key` is handled (marked `Private`, loaded via config file injection).
3. Gate `new_for_testing()` behind the `#[cfg(test)]` or `testing` feature flag so it cannot be called from production code.
4. Add a startup assertion that the loaded key is not equal to the known test key value.

### Proof of Concept

```rust
use starknet_core::crypto::ecdsa_sign;
use starknet_types_core::felt::Felt;

// Publicly known key from source code
let private_key = Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133"
);

// Forge a precommit vote for any block hash
let target_block_hash = Felt::from_hex_unchecked("0xdeadbeef...");
// Domain: PRECOMMIT_VOTE || block_hash.to_bytes_be(), then blake2s -> Felt
let message_digest = compute_precommit_digest(target_block_hash);
let forged_signature = ecdsa_sign(&private_key, &message_digest).unwrap();

// forged_signature is a valid precommit vote for target_block_hash,
// indistinguishable from one produced by the real validator node.
```

The forged signature will pass `verify_precommit_vote_signature` against the known public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a`, which is the key the consensus engine uses to verify this validator's votes. [6](#0-5)
