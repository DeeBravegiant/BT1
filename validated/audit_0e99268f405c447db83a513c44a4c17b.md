Audit Report

## Title
`instantWithdrawal` uses stale `rsETHPrice` to compute payout, allowing users to exit at pre-slash rate and shift losses to remaining rsETH holders - (File: `contracts/LRTWithdrawalManager.sol`)

## Summary
`LRTOracle.rsETHPrice` is a stored state variable updated only via an explicit `updateRSETHPrice()` call. `instantWithdrawal()` computes the asset payout using this stored value without refreshing it, creating a window after an EigenLayer slash where a user can exit at the inflated pre-slash price. The excess assets are drawn from `LRTUnstakingVault`, and the unabsorbed slash loss is redistributed to all remaining rsETH holders when the price is eventually updated.

## Finding Description
`LRTOracle.rsETHPrice` is declared as a plain storage variable at `LRTOracle.sol:28`:

```solidity
uint256 public override rsETHPrice;
```

It is only written inside `_updateRsETHPrice()`, which is triggered exclusively by an explicit call to the public `updateRSETHPrice()` (`LRTOracle.sol:87-89`) or the manager-gated `updateRSETHPriceAsManager()`. There is no automatic, atomic refresh tied to EigenLayer state changes.

`instantWithdrawal()` at `LRTWithdrawalManager.sol:228` calls:

```solidity
uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
```

`getExpectedAssetAmount()` at `LRTWithdrawalManager.sol:593` resolves to:

```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```

using the stored `rsETHPrice` with no prior refresh. An EigenLayer slash reduces `getEffectivePodShares()` on one or more `NodeDelegator` contracts, which lowers `_getTotalEthInProtocol()` and thus the true rsETH price — but this is only reflected in `rsETHPrice` after an explicit `updateRSETHPrice()` call.

The queued-withdrawal path is partially protected: `_calculatePayoutAmount()` at `LRTWithdrawalManager.sol:833-834` takes the minimum of the locked `expectedAssetAmount` and the current return at unlock time, so a price drop after initiation reduces the payout. `instantWithdrawal` has no such minimum-of-two protection.

The downside-protection mechanism in `_updateRsETHPrice()` (`LRTOracle.sol:270-282`) pauses the protocol if the price drop exceeds `pricePercentageLimit`. However: (a) if `pricePercentageLimit == 0` the check is skipped entirely (`pricePercentageLimit > 0 && ...`); (b) for slashes below the threshold, no pause is triggered and the stale-price window remains fully exploitable; (c) the pause is only triggered when `updateRSETHPrice()` is called, not atomically with the slash itself.

## Impact Explanation
A user who calls `instantWithdrawal()` while `rsETHPrice` is stale burns rsETH at an inflated rate and receives more underlying assets from `LRTUnstakingVault` than their rsETH is currently worth. When `updateRSETHPrice()` is eventually called, the price drops to reflect the slash and all remaining rsETH holders absorb the full loss the exiting user avoided. This constitutes **High — Theft of unclaimed yield** from remaining rsETH holders.

## Likelihood Explanation
EigenLayer operator slashing is an explicitly anticipated risk for restaking protocols. The slash event is observable on-chain via EigenLayer events. The window between a slash and the next `updateRSETHPrice()` call is non-zero and unpredictable — price updates are not atomic with slashes. Any user monitoring EigenLayer slash events can call `instantWithdrawal()` within this window without needing to front-run any specific pending transaction. The `instantWithdrawal` feature is manager-enabled per asset and is a live production path. For slashes below `pricePercentageLimit` (or when `pricePercentageLimit == 0`), no automatic pause is triggered, leaving the window fully open.

## Recommendation
Call `updateRSETHPrice()` (or an equivalent internal price refresh) atomically at the start of `instantWithdrawal()` before computing `getExpectedAssetAmount`. This ensures the price reflects any pending losses before allowing a user to exit:

```solidity
function instantWithdrawal(address asset, uint256 rsETHUnstaked, string calldata referralId)
    external nonReentrant whenNotPaused onlySupportedAsset(asset) onlySupportedStrategy(asset) onlyInstantWithdrawalAllowed(asset)
{
    ILRTOracle(lrtConfig.getContract(LRTConstants.LRT_ORACLE)).updateRSETHPrice();
    uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
    ...
}
```

Alternatively, adopt the same minimum-of-two protection used in `_calculatePayoutAmount` for the queued withdrawal path.

## Proof of Concept
1. EigenLayer slashes an operator, reducing `getEffectivePodShares()` across one or more `NodeDelegator` contracts. The true rsETH price is now lower than `LRTOracle.rsETHPrice`.
2. `updateRSETHPrice()` has not yet been called; `rsETHPrice` remains at the pre-slash value. The price drop is below `pricePercentageLimit` (or `pricePercentageLimit == 0`), so no automatic pause is triggered.
3. Attacker holds `R` rsETH. They call `instantWithdrawal(asset, R, "")`.
4. `getExpectedAssetAmount` at `LRTWithdrawalManager.sol:593` computes `payout = R × rsETHPrice_stale / assetPrice`, which is larger than `R × rsETHPrice_true / assetPrice`.
5. Attacker receives the inflated payout from `LRTUnstakingVault`; their rsETH is burned.
6. `updateRSETHPrice()` is called; `rsETHPrice` drops to reflect the slash. All remaining rsETH holders now hold tokens worth less — they absorbed the attacker's share of the slash loss.

**Foundry fork test outline:**
- Fork mainnet/testnet at a block after a known EigenLayer slash event.
- Snapshot `lrtOracle.rsETHPrice()` before calling `updateRSETHPrice()`.
- Call `instantWithdrawal` as an attacker with `R` rsETH; record `assetAmountUnlocked`.
- Call `updateRSETHPrice()`; record `newRsETHPrice`.
- Assert `assetAmountUnlocked > R * newRsETHPrice / assetPrice`, confirming the attacker received more than the post-slash fair value.
