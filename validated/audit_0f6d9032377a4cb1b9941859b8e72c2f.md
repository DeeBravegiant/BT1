### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Inject Arbitrary Sequencer Address into Block Hash and Execution Context - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many fields of a received `ProposalInit` (height, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, `starknet_version`, `version_constant_commitment`, `fee_proposal_fri`) but never checks `init_proposed.builder` against the locally-configured `builder_address`. Because `builder` is passed verbatim to `convert_to_sn_api_block_info` and becomes the `sequencer_address` in `BlockInfo`, a malicious proposer can inject any address into the block's execution context and block hash without being rejected.

### Finding Description

`ProposalInit` carries a `builder` field described as "Address of the one who builds/sequences the block." [1](#0-0) 

The local node stores the expected value in `ContextStaticConfig.builder_address`: [2](#0-1) 

When building a proposal the node correctly uses its own configured address: [3](#0-2) 

When **validating** a received proposal, `is_proposal_init_valid` is called with a `ProposalInitValidation` struct that has no `builder_address` field: [4](#0-3) 

The function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`, but `init_proposed.builder` is never compared to any locally-trusted reference: [5](#0-4) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` immediately converts the unchecked `init` (including the attacker-controlled `builder`) into a `BlockInfo` and sends it to the batcher: [6](#0-5) 

`PartialBlockHashComponents::new` maps `block_info.sequencer_address` (derived from `init.builder`) directly into the block hash: [7](#0-6) 

`calculate_block_hash` chains the sequencer field into the Poseidon hash: [8](#0-7) 

### Impact Explanation

A malicious proposer sets `ProposalInit.builder` to an arbitrary `ContractAddress`. The validator node accepts the proposal (no check exists), executes all transactions with the attacker-chosen sequencer address as `BlockInfo.sequencer_address`, and commits a block whose `PartialBlockHash` encodes the wrong sequencer. Two concrete harms follow:

1. **Wrong block hash committed.** The `PartialBlockHash` (and therefore the full block hash) is computed over the attacker-supplied sequencer address instead of the legitimate one. Every downstream consumer that verifies the block hash (L1 bridge, provers, sync peers) receives a hash that does not correspond to the canonical chain.

2. **Wrong `get_execution_info` result.** Any contract that calls the `get_execution_info` / `get_block_info` syscall during the block receives the attacker-chosen sequencer address. Contracts that gate logic on the sequencer address (fee collection, access control, MEV-protection) will behave incorrectly.

Both effects match the allowed impact: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input* (Critical).

### Likelihood Explanation

Any consensus validator that is legitimately selected as the round proposer can trigger this. No special privilege beyond being in the validator set is required. The attacker simply sets `init.builder` to any address before broadcasting the `ProposalInit`. All honest validators will accept the proposal because `is_proposal_init_valid` never inspects `builder`. Likelihood is **Medium** (requires being selected as proposer, which rotates among validators).

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` and populate it from `ContextStaticConfig.builder_address` at the call site in `SequencerConsensusContext::validate_proposal`. Inside `is_proposal_init_valid`, add an equality check:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

This mirrors the existing pattern used for `l1_da_mode`, `l2_gas_price_fri`, and `starknet_version`.

### Proof of Concept

1. Attacker is selected as proposer for height H, round R.
2. Attacker constructs `ProposalInit { builder: ContractAddress::from(0xdeadbeef), … }` with all other fields valid.
3. Attacker broadcasts the proposal stream to honest validators.
4. Each honest validator calls `validate_proposal` → `is_proposal_init_valid`. The function checks every field **except** `builder` and returns `Ok(())`.
5. `initiate_validation` calls `convert_to_sn_api_block_info(&init)`, producing a `BlockInfo` with `sequencer_address = 0xdeadbeef`.
6. The batcher executes all transactions in the block with `sequencer_address = 0xdeadbeef`. Any contract calling `get_execution_info` receives `0xdeadbeef` as the sequencer.
7. `PartialBlockHashComponents::new` sets `sequencer = SequencerContractAddress(0xdeadbeef)`.
8. `calculate_block_hash` chains `0xdeadbeef` into the Poseidon hash, producing a block hash that encodes the wrong sequencer.
9. Consensus reaches quorum; the block with the corrupted sequencer address and wrong block hash is committed. [9](#0-8) [10](#0-9) [11](#0-10)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}
