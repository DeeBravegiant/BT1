Audit Report

## Title
`highestRsethPrice` One-Way Ratchet With No Reset Enables Indefinite Auto-Pause Loop After Price Drop - (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle` stores `highestRsethPrice` as a monotonically increasing all-time-high watermark with no admin reset path. After any price drop exceeding `pricePercentageLimit` (e.g., from an EigenLayer slashing event), the downside-protection logic auto-pauses `LRTOracle`, `LRTDepositPool`, and `LRTWithdrawalManager`. Because `highestRsethPrice` never decreases, every subsequent call to the public `updateRSETHPrice()` re-triggers the auto-pause immediately after each admin unpause, keeping user deposits and withdrawals frozen until the admin disables the protection entirely.

## Finding Description

`highestRsethPrice` is updated only upward:

```solidity
// update highest price if new price exceeds it
if (newRsETHPrice > highestRsethPrice) {
    highestRsethPrice = newRsETHPrice;
}
``` [1](#0-0) 

There is no corresponding path to decrease or reset it. The downside-protection block compares the current price against this permanent peak:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
``` [2](#0-1) 

The public entry point is callable by any EOA when the contract is not paused:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [3](#0-2) 

The privileged `updateRSETHPriceAsManager()` (which lacks `whenNotPaused`) does not provide an escape: when called while paused, `_updateRsETHPrice()` still hits the `isPriceDecreaseOffLimit` branch, calls `_pause()` (which returns immediately via `if (paused) return`), and exits early without updating `rsETHPrice` or `highestRsethPrice`. [4](#0-3) [5](#0-4) 

The only admin escape is `setPricePercentageLimit(0)`, which disables the downside protection entirely: [6](#0-5) 

**Exploit loop:**
1. Price drops significantly from `highestRsethPrice` (e.g., slashing).
2. Any EOA calls `updateRSETHPrice()` → all three contracts auto-pause.
3. Admin calls `unpause()` on `LRTOracle`, `LRTDepositPool`, `LRTWithdrawalManager`.
4. Any EOA immediately calls `updateRSETHPrice()` again → same condition fires, all three auto-pause again.
5. Steps 3–4 repeat indefinitely.

## Impact Explanation

Every iteration of the loop freezes `LRTDepositPool` and `LRTWithdrawalManager`, preventing users from depositing ETH/LSTs and completing pending withdrawals. This constitutes **temporary (potentially indefinite) freezing of funds**, matching the allowed **Medium: Temporary freezing of funds** impact tier. The freeze persists until the admin disables the price-drop guard entirely, which removes a meaningful security control.

## Likelihood Explanation

EigenLayer slashing is a documented, realistic risk for any restaking protocol. A meaningful slash can drop rsETH price by several percent from its all-time high. Once that threshold is crossed, the griefing loop requires no special privilege — any EOA can call the public `updateRSETHPrice()` to re-trigger the pause after each admin unpause. The precondition (a slashing event) is realistic, and the exploit itself is trivially repeatable. Likelihood is **Medium**.

## Recommendation

Add an admin-restricted function to reset `highestRsethPrice` to the current `rsETHPrice`, allowing the protocol to re-anchor its downside-protection baseline after a verified slashing event:

```solidity
function resetHighestRsethPrice() external onlyLRTAdmin {
    highestRsethPrice = rsETHPrice;
    emit HighestRsethPriceReset(rsETHPrice);
}
```

This allows the admin to acknowledge the new price floor after a slashing event without disabling the protection entirely.

## Proof of Concept

1. `highestRsethPrice` is set to `1.10e18` after a period of rewards accrual.
2. An EigenLayer slashing event reduces total ETH; `_getTotalEthInProtocol()` now yields a price of `1.04e18`.
3. `pricePercentageLimit` is `5e16` (5%).
4. `diff = 1.10e18 - 1.04e18 = 0.06e18`; `pricePercentageLimit.mulWad(highestRsethPrice) = 0.055e18`; `0.06e18 > 0.055e18` → `isPriceDecreaseOffLimit = true`.
5. Any EOA calls `updateRSETHPrice()` → `LRTOracle`, `LRTDepositPool`, `LRTWithdrawalManager` all pause.
6. Admin calls `unpause()` on all three contracts.
7. Any EOA calls `updateRSETHPrice()` again. Price is still `1.04e18`; `highestRsethPrice` is still `1.10e18` (never decreased). Same condition fires → all three pause again.
8. Steps 6–7 repeat indefinitely. Users cannot deposit or withdraw.

**Foundry test plan:** Deploy `LRTOracle` with a mock `LRTDepositPool` and `LRTWithdrawalManager`. Set `highestRsethPrice = 1.10e18`, mock `_getTotalEthInProtocol()` to return a value yielding `1.04e18`, set `pricePercentageLimit = 5e16`. Call `updateRSETHPrice()` as an unprivileged EOA, assert all three contracts are paused. Call `unpause()` as admin on all three. Call `updateRSETHPrice()` again as the same EOA, assert all three are paused again. Assert `highestRsethPrice` is still `1.10e18` throughout.
