Audit Report

## Title
Unbounded O(n × m) External-Call Loop in `_getTotalEthInProtocol()` Permanently Stales `rsETHPrice` — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and, for each, makes three external calls per node delegator via `LRTDepositPool.getAssetDistributionData()`. Because `updateRSETHPrice()` is unrestricted and both arrays grow through normal governance operations with no on-chain gas-safe cap, the transaction will eventually exceed the block gas limit, permanently freezing `rsETHPrice` and causing all subsequent mints to use a stale price.

## Finding Description

`_getTotalEthInProtocol()` loops over `supportedAssets` and calls `getTotalAssetDeposits` for each:

```solidity
// contracts/LRTOracle.sol L331–349
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    uint256 assetER = getAssetPrice(asset);                                          // 1 external oracle call
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset); // nested loop
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getTotalAssetDeposits` → `getAssetDistributionData` performs **3 external calls per NDC per asset**:

```solidity
// contracts/LRTDepositPool.sol L446–456
for (uint256 i; i < ndcsCount;) {
    assetLyingInNDCs += IERC20(asset).balanceOf(nodeDelegatorQueue[i]);
    assetStakedInEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset);
    assetUnstakingFromEigenLayer += INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset);
    unchecked { ++i; }
}
```

The entry point has no access control:

```solidity
// contracts/LRTOracle.sol L87–89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`maxNodeDelegatorLimit` is freely updatable by admin with no on-chain upper bound that accounts for nested gas cost:

```solidity
// contracts/LRTDepositPool.sol L290–296
function updateMaxNodeDelegatorLimit(uint256 maxNodeDelegatorLimit_) external onlyLRTAdmin {
    if (maxNodeDelegatorLimit_ < nodeDelegatorQueue.length) {
        revert InvalidMaximumNodeDelegatorLimit();
    }
    maxNodeDelegatorLimit = maxNodeDelegatorLimit_;
```

The total external calls per `updateRSETHPrice()` invocation is `supportedAssets.length × nodeDelegatorQueue.length × 3` plus one oracle call per asset. Both arrays grow through normal governance operations (`addNewSupportedAsset` via `TIME_LOCK_ROLE`, `addNodeDelegatorContractToQueue` via admin). No existing guard bounds the combined product.

When the transaction reverts OOG, `rsETHPrice` is never updated. All subsequent mints divide by the stale stored value:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

Note that `updateRSETHPriceAsManager()` (L94) also calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, so the manager path fails identically once the gas threshold is crossed.

## Impact Explanation

**Medium — Unbounded Gas Consumption.** Once the block gas limit is exceeded, `rsETHPrice` is permanently frozen at its last stored value. A stale price causes every subsequent `depositAsset`/`depositETH` call to mint an incorrect amount of rsETH (over- or under-minting depending on drift direction), breaking the core accounting invariant for all depositors. This matches the allowed impact class "Unbounded gas consumption" and secondarily "Contract fails to deliver promised returns."

## Likelihood Explanation

The protocol is explicitly designed to support multiple LSTs and scale EigenLayer delegation across many node delegators. `maxNodeDelegatorLimit` initializes at 10 and has no on-chain ceiling. At 10 assets × 20 NDCs the loop executes 600 external calls; each EigenLayer strategy call (e.g., `getAssetBalance` reading strategy shares) costs well above the minimum 2,500 gas estimate when accounting for cold storage reads inside EigenLayer contracts. The gas cost grows quadratically with normal protocol expansion, making this a realistic operational failure rather than a theoretical edge case. Any unprivileged address can trigger the failure by calling the public `updateRSETHPrice()`.

## Recommendation

1. Enforce a hard on-chain cap on `maxNodeDelegatorLimit × supportedAssets.length` that is derived from a conservative gas budget (e.g., ≤ 5M gas for the loop), reverting `updateMaxNodeDelegatorLimit` and `addNewSupportedAsset` if the product would exceed it.
2. Refactor `_getTotalEthInProtocol()` to maintain a running per-asset TVL accumulator updated incrementally on deposit/withdrawal/rebalance events, eliminating the full recomputation on every price update.
3. Alternatively, introduce a paginated TVL aggregation pattern where per-asset totals are stored lazily and `updateRSETHPrice()` reads cached values rather than recomputing from scratch.

## Proof of Concept

1. Governance adds 10 supported LST assets via `LRTConfig.addNewSupportedAsset` (TIME_LOCK_ROLE).
2. Admin calls `LRTDepositPool.updateMaxNodeDelegatorLimit(20)` then `addNodeDelegatorContractToQueue` with 20 NDC addresses.
3. Any unprivileged EOA calls `LRTOracle.updateRSETHPrice()`.
4. `_getTotalEthInProtocol()` executes 10 × 20 × 3 = 600 external calls to EigenLayer strategy contracts plus 10 oracle calls in a single transaction.
5. At realistic per-call costs (5,000–15,000 gas for EigenLayer strategy reads with cold storage), the loop alone consumes 3M–9M gas; with arithmetic, storage writes in `_updateRsETHPrice`, and EVM overhead the transaction approaches or exceeds the 30M block gas limit as counts grow further.
6. `updateRSETHPrice()` reverts OOG; `rsETHPrice` is frozen; all subsequent `depositAsset`/`depositETH` calls compute `rsethAmountToMint` using the stale price, breaking deposit accounting for all users.

**Foundry fork test plan:** Deploy against a mainnet fork, register 10 mock LST assets with price oracles, deploy 20 mock NDCs implementing `getAssetBalance`/`getAssetUnstaking` with realistic EigenLayer storage reads, call `updateRSETHPrice{gas: 30_000_000}()`, and assert the transaction reverts with OOG or that `gasleft()` after the call is near zero.
