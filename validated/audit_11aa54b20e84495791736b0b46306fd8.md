### Title
Unconstrained `fee_proposal_fri` During Startup Window Enables Permanent Fee Market Corruption ‚ÄĒ (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

During the first `fee_proposal_window_size` (10) blocks, the `fee_proposal_fri` field in `ProposalInit` is accepted by validators without any bounds check. A malicious proposer can set this field to an extreme value (e.g., `u128::MAX`), permanently corrupting the `fee_proposals_window` and forcing the `l2_gas_price` to an astronomically high value for all subsequent blocks.

### Finding Description

In `is_proposal_init_valid`, the bounds check on `fee_proposal_fri` is gated on `fee_actual` being `Some`: [1](#0-0) 

`fee_actual` is `None` whenever `compute_fee_actual` cannot fill a complete `window_size`-entry window: [2](#0-1) 

`window_size` is 10 blocks: [3](#0-2) 

So for the first 10 blocks of a fresh network (or after a restart from genesis), **any** `fee_proposal_fri` value passes `is_proposal_init_valid` unchallenged. The validator constructs `ProposalInitValidation` with `fee_actual: compute_fee_actual(...)` which returns `None` during this window: [4](#0-3) 

After consensus decides the block, `finalize_decision` unconditionally records the accepted value: [5](#0-4) 

Once 10 blocks with `fee_proposal_fri = u128::MAX` are committed, `compute_fee_actual` returns `u128::MAX` as the median. From that point forward, `compute_proposer_fee_proposal` clamps every future `fee_proposal` to the band `[u128::MAX/(1+margin), u128::MAX]`: [6](#0-5) 

Validators enforce the same band, so honest proposers are also forced into this range. `calculate_next_l2_gas_price` then passes this corrupted `fee_actual` to `calculate_next_l2_gas_price_for_fin` as a floor: [7](#0-6) 

The `fee_proposal_fri` value is also bound into the `ProposalCommitment` via `Poseidon(partial_block_hash, fee_proposal_fri)`, so the corrupted value is cryptographically committed and propagated to the cende blob: [8](#0-7) 

### Impact Explanation

After 10 blocks of manipulation, `fee_actual = u128::MAX`. Every subsequent proposer is forced to publish `fee_proposal_fri ‚Čą u128::MAX`, which validators enforce. `l2_gas_price` is driven to `u128::MAX`, making every transaction's `max_price_per_unit` insufficient. The network is permanently halted for ordinary users. The corruption is self-reinforcing: once the window is filled with extreme values, honest proposers cannot escape the band without being rejected by validators. This matches the **Critical** impact category: "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."

### Likelihood Explanation

The attacker must be a consensus proposer during the first 10 blocks of a fresh network or genesis restart. In a permissioned validator set this requires being a committee member; in a permissionless one any node qualifies. The 10-block window is always present at genesis and is never guarded by any rate-limit or timelock ‚ÄĒ the direct analog to the missing `up_deploy_code` timelock.

### Recommendation

When `fee_actual` is `None`, enforce a hard cap on `fee_proposal_fri` using the current `l2_gas_price` as a reference (e.g., reject any `fee_proposal_fri` that exceeds `l2_gas_price * some_multiplier`). Alternatively, apply the same `fee_proposal_bounds` logic using `l2_gas_price` as a stand-in for `fee_actual` during the startup window, so the invariant is never fully relaxed.

### Proof of Concept

1. Start a fresh network (genesis, `fee_proposals_window` empty).
2. Become the proposer for blocks 0‚Äď9 (or collude with proposers for those blocks).
3. For each block, emit `ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), ... }`.
4. `is_proposal_init_valid` skips the bounds check because `fee_actual` is `None`: [9](#0-8) 
5. Each block is decided; `record_fee_proposal(height, Some(u128::MAX))` fills the window.
6. At block 10, `compute_fee_actual` returns `Some(u128::MAX)`.
7. `compute_proposer_fee_proposal` clamps every future `fee_proposal` to `‚Čą u128::MAX`.
8. `calculate_next_l2_gas_price_for_fin` sets `l2_gas_price ‚Čą u128::MAX`.
9. All user transactions are rejected for insufficient fee; the network is effectively halted.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub fn compute_fee_proposal(
    fee_target: Option<GasPrice>,
    fee_actual: GasPrice,
    margin_ppt: u128,
) -> GasPrice {
    let Some(fee_target) = fee_target else {
        return fee_actual;
    };
    let (lower, upper) = fee_proposal_bounds(fee_actual, margin_ppt);
    GasPrice(fee_target.0.clamp(lower, upper))
}
