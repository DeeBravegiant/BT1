Audit Report

## Title
Pool Admin Can Reduce Timelock to Zero and Instantly Disable Stop-Loss Protection — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension` enforces a per-pool `timelock` intended to give LPs a reaction window before critical parameters change. No minimum floor is enforced on the timelock value, so a pool admin can propose `newTimelock = 0`, wait the current timelock period, execute the change, and then immediately propose and execute `drawdownE6 = 0` in the same block. With `drawdownE6 == 0`, `_afterSwapOracleStopLoss` returns unconditionally on every swap, permanently disabling the stop-loss for all LPs.

## Finding Description

`initialize` decodes and stores the timelock with no lower-bound validation: [1](#0-0) 

`proposeOracleStopLossTimelock` similarly accepts any `newTimelock` including `0`: [2](#0-1) 

`_afterTimelock` computes `executeAfter = block.timestamp + timelock`. When `timelock == 0`, `executeAfter == block.timestamp`: [3](#0-2) 

`_requireElapsed` checks `block.timestamp < executeAfter`. When `executeAfter == block.timestamp`, this condition is `false`, so the check passes immediately in the same block as the proposal: [4](#0-3) 

`_validateDrawdown` only rejects values strictly above `1e6`; `drawdownE6 == 0` passes: [5](#0-4) 

Once `drawdownE6` is committed as `0`, `_afterSwapOracleStopLoss` returns early on every swap, bypassing all watermark checks: [6](#0-5) 

The NatDoc explicitly states the timelock exists "so LPs can react," but no contract-level minimum enforces this guarantee. [7](#0-6) 

## Impact Explanation

The stop-loss extension is the sole on-chain mechanism blocking swaps when per-bin LP value per share falls below the drawdown floor. Disabling it removes the only guard against value-drain attacks (e.g., oracle manipulation causing the pool to trade at a bad price). LPs who joined relying on the stop-loss have no on-chain recourse once `drawdownE6 == 0` is committed; all subsequent swaps proceed regardless of how far per-share value has fallen. This constitutes broken core pool functionality causing potential loss of LP principal — a qualifying impact under the allowed gate.

## Likelihood Explanation

The post-deployment path requires only that the pool admin wait one timelock cycle (whatever the current non-zero timelock is), then execute `newTimelock = 0`. After that, all future parameter changes — including `drawdownE6 = 0` — are executable in the same block as the proposal. No unprivileged actor is needed; the pool admin alone can execute this. Once `timelock == 0` is committed, the state is irreversible without another timelock cycle (which is now also instant). The attack is repeatable and requires no special market conditions.

## Recommendation

1. Enforce a non-zero minimum timelock constant in both `initialize` and `executeOracleStopLossTimelock`:

```solidity
uint32 private constant MIN_TIMELOCK = 1 days;

function _validateTimelock(uint32 timelock) private pure {
    if (timelock < MIN_TIMELOCK) revert OracleStopLossTimelockTooShort(timelock);
}
```

Apply `_validateTimelock` in `initialize` (line 58 area) and in `executeOracleStopLossTimelock` before committing the new value (line 91 area).

2. Reject `drawdownE6 == 0` in `_validateDrawdown` (or gate it behind a separate explicit "disable" function with its own access control), so the stop-loss cannot be silently neutered via a normal parameter change.

## Proof of Concept

```solidity
// Step 1: Pool deployed with any non-zero timelock, e.g. 1 hour
extension.initialize(pool, abi.encode(uint32(100_000), uint32(58), uint32(3600)));

// Step 2: Admin proposes timelock = 0 (passes _validateTimelock — no such check exists)
vm.prank(admin);
extension.proposeOracleStopLossTimelock(pool, 0);
// executeAfter = block.timestamp + 3600

// Step 3: Wait 1 hour, then execute
vm.warp(block.timestamp + 3600);
vm.prank(admin);
extension.executeOracleStopLossTimelock(pool);
// oracleStopLossConfig[pool].timelock == 0

// Step 4: In the same block, propose drawdownE6 = 0
vm.prank(admin);
extension.proposeOracleStopLossDrawdown(pool, 0);
// executeAfter = block.timestamp + 0 = block.timestamp → _requireElapsed passes immediately

// Step 5: Execute in the same block
vm.prank(admin);
extension.executeOracleStopLossDrawdown(pool);
// drawdownE6 == 0 → _afterSwapOracleStopLoss returns at line 217 for every swap
// Stop-loss permanently disabled; no swap will ever revert for value drain
```
