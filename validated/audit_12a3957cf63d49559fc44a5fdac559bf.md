### Title
`FlatStorageChunkView::iter_range` Returns Stale Flat-Storage State Without Applying Pending Deltas — (File: `core/store/src/flat/chunk_view.rs`)

---

### Summary

`FlatStorageChunkView::iter_range()` reads directly from the committed `DBCol::FlatState` column (i.e., the state at the flat storage head), completely ignoring the in-memory delta chain that covers blocks between the flat head and `self.block_hash`. The sibling methods `get_value()` and `contains_key()` correctly walk the delta chain for the target block, but `iter_range()` does not. Any runtime path that iterates over trie state — including smart-contract host functions such as `storage_iter_prefix` — will therefore observe stale key-value pairs, producing incorrect state transitions and an incorrect state root.

---

### Finding Description

`FlatStorageChunkView` is the interface through which the runtime reads flat storage for a specific block (`self.block_hash`). The flat storage head normally lags one or more blocks behind the block being applied, because `update_flat_head` is called only after all chunks for a block are processed.

`get_value()` and `contains_key()` handle this correctly by delegating to `FlatStorage::get_value` / `FlatStorage::contains_key`, which walk the in-memory delta map from `self.block_hash` back to the flat head, overlaying the most-recent change for each key:

```rust
// core/store/src/flat/chunk_view.rs  line 41-47
pub fn get_value(&self, key: &[u8]) -> Result<Option<FlatStateValue>, crate::StorageError> {
    self.flat_storage.get_value(&self.block_hash, key)   // applies deltas
}
pub fn contains_key(&self, key: &[u8]) -> Result<bool, crate::StorageError> {
    self.flat_storage.contains_key(&self.block_hash, key) // applies deltas
}
```

`iter_range()`, however, bypasses the delta chain entirely and reads only the committed DB state at the flat head:

```rust
// core/store/src/flat/chunk_view.rs  line 49-53

// like in get_value() and contains_key(), because otherwise we're iterating over
// old state that might have been updated by `self.block_hash`
pub fn iter_range(&self, from: Option<&[u8]>, to: Option<&[u8]>) -> FlatStateIterator<'_> {
    self.store.iter_range(self.flat_storage.shard_uid(), from, to)
}
``` [1](#0-0) 

The `FlatStoreAdapter::iter_range` it calls reads only `DBCol::FlatState`, which represents the committed state at the flat head, not at `self.block_hash`:

```rust
// core/store/src/adapter/flat_store.rs  line 103-141
/// It reads data only from `FlatState` column which represents the state at
/// flat storage head. Reads only committed changes.
pub fn iter_range<'a>(...) -> FlatStateIterator<'a> { ... }
``` [2](#0-1) 

The flat head is kept behind the current block by design (the non-strict update mode keeps at least `BLOCKS_WITH_CHANGES_FLAT_HEAD_GAP` blocks of deltas in memory): [3](#0-2) 

So during every chunk application, `self.block_hash` is ahead of the flat head, and `iter_range` will return state that is missing keys added, retaining keys deleted, and showing old values for keys updated in the intervening blocks.

---

### Impact Explanation

When a smart contract calls a host function that iterates over storage (e.g., `storage_iter_prefix`, `storage_iter_range`), the runtime calls `Trie::iter()`, which uses `FlatStorageChunkView::iter_range()`. The contract receives a stale view of state:

- **Keys added** in blocks between the flat head and the current block are **invisible**.
- **Keys deleted** in those blocks are **still visible**.
- **Updated values** appear with their old content.

A contract that makes decisions based on this iteration (e.g., conditional transfers, access-control checks, aggregation over stored data) will produce incorrect state changes. Because the state root is derived from the trie changes produced by chunk application, an incorrect iteration result propagates to an **incorrect state root** committed to the chain.

The corrupted value is: the `state_root` field of `ChunkExtra` / the block's `chunk_header.prev_state_root`, which is the canonical output of chunk application. [4](#0-3) 

---

### Likelihood Explanation

Any unprivileged user can submit a transaction calling a deployed smart contract that uses storage iteration. The flat head is structurally always behind the block being applied during normal operation, so the stale-read condition is present on every block. No special privileges, timing, or network position are required.

---

### Recommendation

`iter_range()` must apply the in-memory delta chain for blocks between the flat head and `self.block_hash` before returning results, exactly as `get_value()` and `contains_key()` do. The corrected implementation should:

1. Collect the ordered list of blocks from `self.block_hash` back to the flat head (via `FlatStorage::get_blocks_to_head`).
2. Read the base range from `DBCol::FlatState`.
3. Overlay the deltas for each block in ascending order, inserting new keys, removing deleted keys, and replacing updated values.

The existing `flat_storage_iterator` helper in `flat_storage_resharder.rs` demonstrates this pattern and can serve as a reference. [5](#0-4)

    /// Get sequence of blocks `target_block_hash` (inclusive) to flat head (exclusive)
    /// in backwards chain order. Returns an error if there is no path between them.
    fn get_blocks_to_head(
        &self,
        target_block_hash: &CryptoHash,
    ) -> Result<Vec<CryptoHash>, FlatStorageError> {
        let flat_head = &self.flat_head;
        let mut block_hash = *target_block_hash;
        let mut blocks = vec![];
        let mut first_height = None;
        while block_hash != flat_head.hash {
            let metadata = self
                .deltas
                .get(&block_hash)
                .ok_or_else(|| self.create_block_not_supported_error(target_block_hash))?
                .metadata;
            if first_height.is_none() {
                // Keep track of the height of the initial block.
                first_height = Some(metadata.block.height);
            }

            // Maybe skip to the previous block with changes.
            block_hash = match metadata.prev_block_with_changes {
                None => {
                    blocks.push(block_hash);
                    metadata.block.prev_hash
                }
                Some(prev_block_with_changes) => {
                    // Don't include blocks with no changes in the result,
                    // unless it's the target block.
                    if &block_hash == target_block_hash {
                        blocks.push(block_hash);
                    }
                    if prev_block_with_changes.height > flat_head.height {
                        prev_block_with_changes.hash
                    } else {
                        flat_head.hash
                    }
                }
            };
        }
        self.metrics.set_distance_to_head(
            blocks.len(),
            first_height.map(|height| height - flat_head.height),
        );
        if blocks.len() >= Self::HOPS_LIMIT {
            tracing::warn!(
                target: "chain",
                shard_id = %self.shard_uid.shard_id(),
                flat_head_height = flat_head.height,
                cached_deltas = self.deltas.len(),
                num_hops = blocks.len(),
                "flat storage needs too many hops to access a block");
        }

        Ok(blocks)

    fn update_flat_head_impl(
        &self,
        block_hash: &CryptoHash,
        strict: bool,
    ) -> Result<(), FlatStorageError> {
        let mut guard = self.0.write();
        if guard.move_head_hold_count > 0 {
            return Ok(());
        }

        let new_head = guard.get_new_flat_head(*block_hash, strict)?;
        if new_head == guard.flat_head.hash {
            tracing::debug!(target: "store", shard_id = %guard.shard_uid.shard_id(), height = %guard.flat_head.height, "update_flat_head, flat head already at block");
            return Ok(());
        }

        let shard_uid = guard.shard_uid;
        let shard_id = shard_uid.shard_id();

        tracing::debug!(target: "store", flat_head = ?guard.flat_head.hash, ?new_head, %shard_id, "moving flat head");
        let blocks = guard.get_blocks_to_head(&new_head)?;

        for block_hash in blocks.into_iter().rev() {
            let mut store_update = guard.store.store_update();
            // Delta must exist because flat storage is locked, and we could retrieve
            // path from old to new head. Otherwise, we return internal error.
            let changes = guard
                .store
                .get_delta(shard_uid, block_hash)
                .ok_or_else(|| missing_delta_error(&block_hash))?;
            changes.apply_to_flat_state(&mut store_update, guard.shard_uid);
            let metadata = guard
                .deltas
                .get(&block_hash)
                .ok_or_else(|| missing_delta_error(&block_hash))?
                .metadata;
            let block = metadata.block;
            let block_height = block.height;
            store_update.set_flat_storage_status(
                shard_uid,
                FlatStorageStatus::Ready(FlatStorageReadyStatus { flat_head: block }),
            );

            guard.metrics.set_flat_head_height(block.height);
            guard.flat_head = block;

            // Remove old deltas from disk and memory.
            // Do it for each head update separately to ensure that old data is removed properly if node was
            // interrupted in the middle.
            // TODO (#7327): in case of long forks it can take a while and delay processing of some chunk.
            // Consider avoid iterating over all blocks and make removals lazy.
            guard
                .deltas
                .extract_if(|_, delta| delta.metadata.block.height <= metadata.block.height)
                .for_each(|(hash, _)| store_update.remove_delta(shard_uid, hash));

            store_update.commit();
            tracing::debug!(target: "store", %shard_id, %block_hash, %block_height, "moved flat storage head");
        }
        guard.update_delta_metrics();

        Ok(())
    }

    /// Returns an iterator over a shard's flat storage at the given block hash. This
    /// iterator contains both flat storage values and deltas.
    fn flat_storage_iterator<'a>(
        &self,
        flat_store: &'a FlatStoreAdapter,
        shard_uid: &ShardUId,
        block_hash: &CryptoHash,
    ) -> Result<Box<FlatStorageAndDeltaIter<'a>>, Error> {
        let mut iter: Box<FlatStorageAndDeltaIter<'a>> = Box::new(
            flat_store
                .iter(*shard_uid)
                // Get the flat storage iter and wrap the value in Optional::Some to
                // match the delta iterator so that they can be chained.
                .map(|(key, value)| FlatStorageAndDeltaIterItem::Entry((key, Some(value)))),
        );

        // Get all the blocks from flat head to the wanted block hash.
        let flat_storage = self
            .runtime
            .get_flat_storage_manager()
            .get_flat_storage_for_shard(*shard_uid)
            .expect("the flat storage undergoing resharding must exist!");
        // Must reverse the result because we want ascending block heights.
        let mut blocks_to_head = flat_storage.get_blocks_to_head(block_hash).map_err(|err| {
            StorageError::StorageInconsistentState(format!(
                "failed to find path from block {block_hash} to flat storage head ({err})"
            ))
        })?;
        blocks_to_head.reverse();
        tracing::debug!(
            target = "resharding",
            flat_store_blocks_to_head_len = blocks_to_head.len(),
        );

        // Get all the delta iterators and wrap the items in Result to match the flat
        // storage iter so that they can be chained.
        for block in blocks_to_head {
            let Some(deltas) = flat_store.get_delta(*shard_uid, block) else {
                continue;
            };
            // Chain the iterators effectively adding a block worth of deltas.
            // Before doing so insert a commit point to separate changes to the same key in different transactions.
            iter = Box::new(iter.chain(iter::once(FlatStorageAndDeltaIterItem::CommitPoint)));
            let deltas_iter = deltas.0.into_iter();
            let deltas_iter = deltas_iter.map(|item| FlatStorageAndDeltaIterItem::Entry(item));
            iter = Box::new(iter.chain(deltas_iter));
        }

        Ok(iter)
    }
