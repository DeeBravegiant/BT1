Audit Report

## Title
L2 Price Providers Accept Stale Pre-Outage Oracle Prices After Sequencer Restart - (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

## Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are explicitly designed for L2 deployment (both carry `FUTURE_TOLERANCE` to handle sequencer clock skew) but neither checks a Chainlink sequencer uptime feed before quoting bid/ask prices. When the L2 sequencer goes offline, oracle keepers cannot push new prices, freezing the stored `refTime`. Upon sequencer restart, if the frozen price is younger than `MAX_TIME_DELTA` (up to 7 days), `_isStale` returns `false` and the pool quotes the pre-outage price — enabling an attacker to swap at a stale rate before any keeper pushes a fresh update, causing direct LP principal loss.

## Finding Description

Both contracts implement `_isStale` as a pure age check against `block.timestamp`:

```solidity
// PriceProviderL2.sol L135-150
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

`MAX_TIME_DELTA` is bounded only at `> 0` and `<= 7 days`. [1](#0-0) 

`_getBidAndAskPrice()` in `PriceProviderL2` reads the stored oracle price via `IPricedOracle.price()` and immediately applies only the staleness check — no sequencer liveness gate exists anywhere in the call path: [2](#0-1) 

`ProtectedPriceProviderL2._computeBidAsk()` has the identical gap — staleness check only, no sequencer uptime gate: [3](#0-2) 

Neither constructor accepts a `_sequencerUptimeFeed` parameter: [4](#0-3) [5](#0-4) 

The grep search across all `.sol` files confirms zero occurrences of `sequencerUptimeFeed` or `GRACE_PERIOD` in any Solidity source — the pattern exists only in `registry.json` (for the deployed `ChainlinkVerifierL2`), confirming the check is known and used in the same system but absent from both L2 price providers.

The oracle layer (`PythOracle`, `ChainlinkOracle`) is push-based: keepers write prices on-chain. When the sequencer is down, keepers cannot push, so `refTime` freezes at the last pre-outage value. Upon restart, the stored price is returned verbatim by `IPricedOracle.price()` with its old `refTime`. If that `refTime` is within `MAX_TIME_DELTA`, `_isStale` returns `false` and the stale price reaches the pool swap. [6](#0-5) 

## Impact Explanation

Direct LP principal loss via bad-price execution. An attacker swaps against a stale pre-outage bid/ask price while the true market price has moved during the outage. The pool receives token input valued at the stale rate and pays out token output at the same stale rate; the difference between stale and current market price is extracted by the attacker. LP balances no longer cover fair-value LP claims at current market prices, satisfying the "bad-price execution" and "pool insolvency" impact gates. Severity is High given the direct, quantifiable LP loss proportional to price movement during the outage. [7](#0-6) 

## Likelihood Explanation

- Both contracts are deployed on Arbitrum, Base, and other L2s per the registry, all of which have documented sequencer outage history.
- The attack requires no special privilege — any EOA can call `swap()` immediately after sequencer restart.
- The vulnerability window is bounded only by how quickly oracle keepers push a fresh price post-restart, which can be minutes.
- `MAX_TIME_DELTA` up to 7 days means even a multi-hour outage leaves the price well within the staleness window.
- The attack is repeatable on every sequencer restart event. [1](#0-0) 

## Recommendation

Add a Chainlink sequencer uptime feed check with a grace period to both `PriceProviderL2._getBidAndAskPrice()` and `ProtectedPriceProviderL2._computeBidAsk()`, mirroring the pattern already present in `ChainlinkVerifierL2`:

```solidity
// Add immutable to constructor:
AggregatorV3Interface public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600; // 1 hour

// Add before the _isStale check in _getBidAndAskPrice() / _computeBidAsk():
if (address(sequencerUptimeFeed) != address(0)) {
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    // answer == 0: sequencer up; answer == 1: sequencer down
    if (answer != 0 || block.timestamp - startedAt < GRACE_PERIOD) {
        return (0, type(uint128).max); // fail closed
    }
}
```

This blocks swaps during sequencer downtime and for `GRACE_PERIOD` after restart, giving keepers time to push a fresh price. [2](#0-1) [8](#0-7) 

## Proof of Concept

```
Setup (Foundry fork test on Arbitrum):
  - Deploy PriceProviderL2 with MAX_TIME_DELTA = 1 days, FUTURE_TOLERANCE = 30s
  - Push oracle price = 100 at T=0 (refTime = T=0)

Attack:
  T=0:      Simulate sequencer outage (warp time, no new oracle pushes possible)
  T=4h:     True market price moves to 110 (+10%)
  T=4h:     Sequencer restarts (vm.warp(T+4h))
  T=4h+1s:  Attacker calls pool.swap(zeroForOne=true, ...)
             → PriceProviderL2._isStale(T=0, T+4h, 1 day) → false (4h < 1 day)
             → Pool quotes bid/ask around 100 (stale)
             → Attacker buys token1 at price 100, sells on external DEX at 110
             → 10% profit extracted; LPs absorb the loss

Assert: LP token balances no longer cover fair-value redemption at market price 110.
``` [6](#0-5) [9](#0-8)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
