### Title
Zero `fee_proposal_fri` accepted during startup window permanently locks SNIP-35 fee market at zero — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `window_size` blocks (the startup window), `is_proposal_init_valid` skips the `fee_proposal_fri` bounds check because `fee_actual` is `None`. A malicious proposer can set `fee_proposal_fri = GasPrice(0)`, which every validator accepts. Once the startup window is filled with zero fee proposals, `compute_fee_actual` returns `GasPrice(0)`, `fee_proposal_bounds(0, margin)` collapses to `(0, 0)`, and the only valid `fee_proposal_fri` forever is zero — permanently locking the SNIP-35 fee market with no recovery path. This is the direct sequencer analog of the AMM `createPool(amount=0)` bug: a one-time initialization window that can be poisoned with a zero value, after which the invariant can never be restored.

---

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` validates `fee_proposal_fri` in two sequential steps.

**Step 1 — presence check** (lines 371–394): for `starknet_version >= V0_14_3`, `fee_proposal_fri` must be `Some(...)`. `Some(GasPrice(0))` satisfies this check and falls through. [1](#0-0) 

**Step 2 — bounds check** (lines 396–416): `fee_proposal_fri` must lie within `[lower_bound, upper_bound]` derived from `fee_actual`. This check is guarded by `if let (Some(fee_actual), Some(fee_proposal)) = (proposal_init_validation.fee_actual, ...)`. The comment is explicit: *"During initiation (fee_actual is None, <window_size blocks), bounds are not enforced."* [2](#0-1) 

`fee_actual` is `None` whenever `compute_fee_actual` returns `None`, which happens for any height below `window_size` or when any entry in the window is missing/`None`: [3](#0-2) 

During the startup window, `proposal_init_validation.fee_actual` is always `None`: [4](#0-3) 

So `Some(GasPrice(0))` passes both checks and the block is decided. `record_fee_proposal` stores `GasPrice(0)` in `fee_proposals_window`: [5](#0-4) 

Once `window_size` consecutive blocks carry `fee_proposal_fri = 0`, `compute_fee_actual` returns `Some(GasPrice(0))`. `fee_proposal_bounds` then computes:

```
upper = 0 * (1000 + margin) / 1000 = 0
lower = 0 * 1000 / (1000 + margin) = 0
``` [6](#0-5) 

The bounds are `(0, 0)`. Any `fee_proposal_fri > 0` now fails `fee_proposal.0 > upper_bound` and is rejected. The only valid value is `GasPrice(0)` forever. There is no recovery path: the window stays filled with zeros because only zero passes validation, and `prune_fee_proposals_window` only removes old entries — it cannot introduce non-zero values. [7](#0-6) 

The corrupted `fee_proposal_fri = 0` is then hashed into every future `ProposalCommitment`:

```
ProposalCommitment = Poseidon(partial_block_hash, 0)
```

instead of `Poseidon(partial_block_hash, real_fee)`: [8](#0-7) 

And forwarded to the cende blob as `FeeProposalInfo { fee_proposal_fri: Some(GasPrice(0)) }`: [9](#0-8) 

---

### Impact Explanation

**Corrupted value**: `ProposalCommitment` is permanently `Poseidon(partial_block_hash, 0)` instead of `Poseidon(partial_block_hash, real_fee_proposal)`. Every block's consensus commitment is wrong relative to the intended SNIP-35 design.

**Economic impact**: The SNIP-35 fee market (the oracle-driven L2 gas price adjustment mechanism) is permanently disabled. `fee_actual` stays at zero, `effective_min` in `calculate_next_l2_gas_price_for_fin` falls back to `config_min` only, and the oracle-derived fee target can never be incorporated. Fee estimation via RPC returns values derived from a broken fee market. [10](#0-9) 

This matches: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact** and **High — RPC execution, fee estimation returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

The attack window is the first `window_size` blocks of the chain. A proposer who controls even a fraction of the initial validator set can propose multiple blocks with `fee_proposal_fri = 0` during this window. All validators accept them because the bounds check is unconditionally skipped when `fee_actual` is `None`. Once the window is saturated, the state is irreversible without a chain upgrade. The startup window is a one-time event with no second chance.

---

### Recommendation

Add a minimum-value guard on `fee_proposal_fri` that applies regardless of whether `fee_actual` is available. Specifically, when `fee_proposal_fri` is `Some(price)`, require `price >= VersionedConstants::latest_constants().min_gas_price`. This mirrors the honest proposer's own fallback behavior (`compute_proposer_fee_proposal` returns `self.l2_gas_price` — never zero — when `fee_actual` is `None`): [11](#0-10) 

The fix should be inserted immediately after the presence check and before the bounds check in `is_proposal_init_valid`.

---

### Proof of Concept

1. Chain launches. `fee_proposals_window` is empty. `fee_actual = None` for all heights `< window_size`.
2. Malicious proposer builds `ProposalInit` with `fee_proposal_fri = Some(GasPrice(0))` for each of the first `window_size` blocks.
3. Each validator calls `is_proposal_init_valid`. Step 1 passes (`Some(GasPrice(0))` is `Some(_)`). Step 2 is skipped (`fee_actual = None`). Block is accepted.
4. `decision_reached` calls `record_fee_proposal(height, Some(GasPrice(0)))` for each block.
5. After `window_size` blocks, `compute_fee_actual` returns `Some(GasPrice(0))` (median of all-zero window).
6. Next proposal arrives with `fee_proposal_fri = Some(GasPrice(X))` where `X > 0`. Validator computes `fee_proposal_bounds(GasPrice(0), margin) = (0, 0)`. Check: `X > 0` → rejected as `InvalidProposalInit`.
7. Only `fee_proposal_fri = Some(GasPrice(0))` passes. All future `ProposalCommitment`s are `Poseidon(partial_block_hash, 0)`. SNIP-35 fee market is permanently locked at zero.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub(crate) fn fee_proposal_bounds(fee_actual: GasPrice, margin_ppt: u128) -> (u128, u128) {
    let denom = U256::from(PPT_DENOMINATOR);
    let scaled = denom + U256::from(margin_ppt);
    let fee_actual_u256 = U256::from(fee_actual.0);
    let upper = u128::try_from(fee_actual_u256 * scaled / denom).unwrap_or(u128::MAX);
    let lower = u128::try_from(fee_actual_u256 * denom / scaled).unwrap_or(0);
    (lower, upper)
}

pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
