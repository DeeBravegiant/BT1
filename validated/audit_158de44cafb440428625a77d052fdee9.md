Audit Report

## Title
Permanent Freezing of Unclaimed Yield When `totalKernelStaked` Drops to Zero Mid-Period — (`contracts/KERNEL/KernelDepositPool.sol`)

## Summary

`KernelDepositPool` uses a Synthetix-style accumulator that short-circuits to zero when `totalKernelStaked == 0`. Any staker can call the public `initiateWithdrawal` function at any time, reducing `totalKernelStaked` to zero during an active reward window. Once that happens, `rewardRate * elapsed_time` worth of reward tokens accumulate in the contract balance with no on-chain mechanism to recover them. The contract's own NatSpec explicitly acknowledges this behavior at lines 18–22.

## Finding Description

`rewardPerToken()` (L408–414) returns the frozen `rewardPerTokenStored` whenever `totalKernelStaked == 0`, skipping all reward accrual for that interval:

```solidity
function rewardPerToken() public view returns (uint256) {
    if (totalKernelStaked == 0) {
        return rewardPerTokenStored;  // accumulator frozen
    }
    ...
}
```

The guard in `notifyRewardAmount` (L570) only prevents starting a new period with zero stakers; it does not prevent stakers from exiting mid-period via `initiateWithdrawal` (L320–326), which unconditionally decrements `totalKernelStaked`:

```solidity
balanceOf[msg.sender] -= _amount;
totalKernelStaked -= _amount;
```

There is no `withdrawalDelay` floor tied to the reward period, no minimum-stake invariant enforced during an active window, and no admin rescue path. The four admin functions (`setRewardsDuration`, `notifyRewardAmount`, `setWithdrawalDelay`, `setMaxNumberOfWithdrawalsPerUser`) at L544–621 contain no `recoverERC20` or equivalent. Reward tokens corresponding to the zero-staked interval remain in the contract balance permanently.

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield.** Reward tokens transferred into the contract via `notifyRewardAmount` that correspond to any zero-staked interval are irrecoverably locked. They cannot be redistributed to future stakers, returned to the protocol treasury, or recovered by any on-chain call. This directly and permanently deprives stakers of yield they are entitled to.

## Likelihood Explanation

Any staker can call `initiateWithdrawal` at any time with no restriction tied to the active reward window. A single large staker (or a coordinated exit) can drive `totalKernelStaked` to zero. The reward period can span weeks (`duration` is admin-configurable), giving ample time for a full unstake. The protocol's own mitigation is an off-chain operational guarantee (NatSpec L18–22), not a code-level invariant, making this scenario realistically reachable.

## Recommendation

Add an admin-only function to recover unallocated rewards after a period ends:

```solidity
function recoverUnallocatedRewards(address recipient) external onlyRole(DEFAULT_ADMIN_ROLE) {
    if (block.timestamp < finishAt) revert RewardDurationNotFinished();
    uint256 totalPending = /* sum of all earned() values or a tracked accumulator */;
    uint256 unallocated = rewardsToken.balanceOf(address(this)) - totalPending;
    rewardsToken.safeTransfer(recipient, unallocated);
}
```

Alternatively, enforce a minimum stake invariant during active reward windows by reverting `initiateWithdrawal` if the withdrawal would reduce `totalKernelStaked` to zero while `block.timestamp < finishAt`.

## Proof of Concept

1. Admin calls `notifyRewardAmount(1_000e18)` with `duration = 30 days`; `rewardRate = 1_000e18 / 30 days`. `totalKernelStaked > 0` at this point, so the call succeeds.
2. Alice is the sole staker: `balanceOf[Alice] = 1e18`, `totalKernelStaked = 1e18`.
3. After 1 day, Alice calls `initiateWithdrawal(1e18)`. `totalKernelStaked` drops to `0`. The `updateReward` modifier snapshots her earned rewards correctly for day 1.
4. After `withdrawalDelay`, Alice calls `claimWithdrawal` and receives her KERNEL tokens.
5. For the remaining 29 days, every call to `rewardPerToken()` returns the frozen `rewardPerTokenStored`. No rewards accumulate for any address.
6. At period end: `rewardsToken.balanceOf(KernelDepositPool) ≈ rewardRate * 29 days` tokens remain locked.
7. No admin function can recover these tokens. Confirmed by exhaustive review of L544–621.

**Foundry test sketch:**
```solidity
function test_rewardsLockedOnZeroStake() public {
    vm.prank(admin);
    pool.notifyRewardAmount(1_000e18); // Alice staked 1e18 beforehand

    vm.warp(block.timestamp + 1 days);
    vm.prank(alice);
    pool.initiateWithdrawal(1e18); // totalKernelStaked -> 0

    vm.warp(block.timestamp + withdrawalDelay);
    vm.prank(alice);
    pool.claimWithdrawal(1);

    vm.warp(pool.finishAt());
    uint256 locked = rewardsToken.balanceOf(address(pool)) - pool.rewards(alice);
    assertGt(locked, 0); // rewards for 29 days are permanently locked
}
```
