### Title
Unauthenticated Invoke Transaction Bypasses Signature Verification via `skip_stateful_validations`, Enabling Mempool Eviction of Legitimate Transactions — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`skip_stateful_validations` unconditionally skips the account's `__validate__` entry point (and therefore signature verification) for any invoke transaction with `nonce == 1` when the target account has any transaction in the mempool and the on-chain nonce is still 0. An unprivileged attacker can exploit this to inject an unsigned invoke for a victim's not-yet-deployed account, and — by paying a marginally higher fee — evict the victim's legitimate invoke from the mempool via fee escalation before sequencing.

---

### Finding Description

The gateway's stateful validation path in `extract_state_nonce_and_run_validations` calls `run_pre_validation_checks`, which in turn calls `skip_stateful_validations`:

```
extract_state_nonce_and_run_validations
  └─ run_pre_validation_checks
       ├─ validate_state_preconditions   (nonce range, resource bounds)
       ├─ validate_by_mempool            (read-only nonce + fee-escalation check)
       └─ skip_stateful_validations      ← returns true → __validate__ skipped
``` [1](#0-0) 

`skip_stateful_validations` returns `true` when:
1. The transaction is an `Invoke`.
2. `tx.nonce() == Nonce(Felt::ONE)`.
3. `account_nonce == Nonce(Felt::ZERO)` (account not yet deployed on-chain).
4. `account_tx_in_pool_or_recent_block(sender_address)` returns `true`. [2](#0-1) 

When `skip_validate = true`, `run_validate_entry_point` sets `validate: false` in `ExecutionFlags`, which causes the blockifier's `StatefulValidator` to skip the account's `__validate__` entry point entirely — meaning **no signature verification is performed at the gateway**: [3](#0-2) 

The gateway then constructs `AddTransactionArgs` with the on-chain nonce and forwards the transaction to the mempool: [4](#0-3) 

The mempool's `add_tx` and `validate_tx` perform only nonce and fee-escalation checks — no signature verification: [5](#0-4) 

If the attacker's fake invoke carries a higher fee than the victim's legitimate invoke at the same nonce, `validate_fee_escalation` passes and the victim's transaction is removed from the pool: [6](#0-5) 

---

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions and evicts valid transactions before sequencing.**

An attacker can permanently evict a victim's legitimate invoke from the mempool without possessing the victim's private key. The attacker's fake invoke will fail `__validate__` at block-building time (the blockifier always runs `__validate__` with `validate: true` during sequencing), but by then the victim's transaction is already gone. The victim must detect the loss and resubmit, potentially missing time-sensitive operations (e.g., DeFi interactions, NFT mints).

---

### Likelihood Explanation

**Medium.** The preconditions are:
- The victim's account must be undeployed on-chain (nonce = 0).
- The victim must have submitted a deploy_account (or any transaction) that is visible in the mempool.
- The attacker must observe the victim's pending invoke and outbid its fee.

All three conditions are routinely satisfied during the common UX pattern of "deploy_account + invoke in the same batch." The mempool is observable via P2P gossip, making fee observation trivial.

---

### Recommendation

1. **Remove the signature-skip for third-party invokes.** The skip is only safe when the *same user* submits both the deploy_account and the invoke atomically. Add a check that the invoke's transaction hash is signed by the expected key, or defer the invoke to after the deploy_account is confirmed.

2. **Alternatively, restrict the skip to transactions whose sender matches the deploy_account sender already in the mempool**, and verify that the deploy_account and invoke share the same sender address and were submitted in the same request context.

3. **At minimum**, document that `skip_stateful_validations` intentionally admits unsigned invokes and add a mempool-level guard that prevents fee-escalation replacement of transactions in the `nonce == 1, account_nonce == 0` window.

---

### Proof of Concept

**Setup:** Alice deploys a new account and submits an invoke in the same batch.

```
Alice → Gateway: deploy_account(class_hash=C, salt=S, nonce=0, sig=alice_sig)
Alice → Gateway: invoke(sender=alice_addr, nonce=1, calldata=[transfer 100 STRK to alice_wallet], tip=10, sig=alice_sig)
```

Both are admitted. `account_tx_in_pool_or_recent_block(alice_addr)` now returns `true`.

**Attack:**

```
Attacker → Gateway: invoke(sender=alice_addr, nonce=1, calldata=[transfer 100 STRK to attacker_addr], tip=11, sig=GARBAGE)
```

**Gateway processing:**

1. Stateless validation passes (well-formed transaction).
2. `get_nonce_from_state(alice_addr)` → `0` (not deployed yet).
3. `validate_nonce`: `0 ≤ 1 ≤ max_allowed` → passes.
4. `validate_by_mempool`: fee-escalation check — attacker's tip (11) > Alice's tip (10) → passes.
5. `skip_stateful_validations`: `nonce==1 && account_nonce==0 && alice_in_mempool==true` → returns `true`.
6. `run_validate_entry_point(validate=false)` → **`__validate__` skipped, GARBAGE signature accepted**.
7. `mempool_client.add_tx(attacker_invoke)` → fee escalation removes Alice's invoke, inserts attacker's invoke.

**Block building:**

- Alice's deploy_account executes (nonce 0→1).
- Attacker's invoke executes: `__validate__` runs with `validate=true`, fails (GARBAGE signature) → reverted.
- Alice's legitimate invoke is **permanently lost** from the mempool. [7](#0-6) [8](#0-7) [9](#0-8)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}

    fn add_tx_validations(
        &mut self,
        tx_reference: TransactionReference,
        tx: &InternalRpcTransaction,
        account_nonce: Nonce,
    ) -> MempoolResult<()> {
        self.validate_incoming_tx(tx_reference, account_nonce)?;
        let replaced_tx_reference = self.validate_fee_escalation(tx_reference)?;

        // The replaced transaction is still pooled, so its bytes still count toward
        // `size_in_bytes()`. Credit what its removal will free: a same-size bump nets to zero (no
        // overflow handling), and a larger replacement only needs room for the delta, consistent
        // with how a fresh next-nonce transaction is treated. The removal happens only after
        // capacity is confirmed below, so a rejected incoming transaction never strands the
        // account.
        let freed_bytes = replaced_tx_reference.map_or(0, |reference| {
            self.tx_pool
                .get_by_tx_hash(reference.tx_hash)
                .expect("Replacement target from pool must exist.")
                .total_bytes()
        });

        if self.exceeds_capacity(tx, freed_bytes) {
            self.handle_capacity_overflow(tx, account_nonce, freed_bytes)?;
        }

        // Capacity is confirmed: this is the final, infallible mutation before the incoming
        // transaction is inserted by the caller.
        if let Some(existing_tx_reference) = replaced_tx_reference {
            self.remove_replaced_tx(existing_tx_reference);
        }

        Ok(())
    }
