### Title
Unauthenticated Consensus Vote Acceptance — Any Network Peer Can Forge Votes from Any Validator (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`handle_vote` in `SingleHeightConsensus` accepts prevotes and precommits from the P2P network without verifying the cryptographic signature carried in the `Vote` struct. The only guard is a membership check on the `voter` address field, which is attacker-controlled wire data. Any connected peer can therefore inject votes attributed to any committee member, accumulate a spurious quorum, and drive the local node to a `DecisionReached` for a block the real validators never approved.

---

### Finding Description

`handle_vote` in `single_height_consensus.rs` performs two checks before accepting a vote:

1. Height equality (`vote.height != height`)
2. Committee membership (`vote.voter` is in `committee.members()`)

The signature field is explicitly skipped with a `// TODO(Asmaa): verify the signature` comment:

### Impact Explanation

The Tendermint state machine in `state_machine.rs` reaches `DecisionReached` once it accumulates a quorum of precommits for the same `proposal_commitment`. Because `handle_vote` accepts any vote whose `voter` field names a committee member — regardless of whether the real key-holder signed it — an attacker who knows the committee composition (public information) can:

1. Forge `2f+1` prevotes for an arbitrary `proposal_commitment` → triggers `upon_prevote_quorum`, causing the local node to broadcast its own precommit.
2. Forge `2f+1` precommits for the same commitment → triggers `DecisionReached`.

The local node then calls `decision_reached` on the `SequencerConsensusContext`, which commits the block to the batcher and state-sync, producing a wrong committed state root, wrong receipts, and wrong event commitments — all derived from a block the honest validators never agreed to.

**Matching impact**: *High — signature/hash logic binds the wrong signer; and potentially Critical — wrong state/receipt/event committed from accepted input.*

---

### Likelihood Explanation

The P2P network is permissionless at the connection level; any node can join and broadcast `Vote` protobuf messages. The committee addresses are deterministically derived from public staking data (`get_proposer`, `get_actual_proposer`), so the attacker needs no privileged information. The `RawSignature::default()` value (all-zero bytes) is accepted without complaint because verification is entirely absent. No rate-limiting or stake-weighted admission exists at the vote-acceptance layer.

---

### Recommendation

1. **Verify incoming vote signatures** inside `handle_vote` before the vote is fed to the state machine. Look up the committee member's `public_key` by `vote.voter`, then call `verify_precommit_vote_signature` (already present in `apollo_signature_manager`) and drop the vote if verification fails.

2. **Sign outgoing votes** in `make_self_vote` using the node's local key store (the `SignatureManager::sign_precommit_vote` method already exists).

3. **Reject before propagation**: move the signature check before the `continue_propagation` call in `manager.rs::handle_vote` so forged votes are never re-broadcast.

---

### Proof of Concept

```
Attacker node (any peer on the P2P network):

1. Observe committee for height H, round R:
   validators = [V1, V2, V3, V4]   (public staking data)
   quorum_threshold = 3

2. Craft 3 forged Prevote messages:
   Vote { vote_type: Prevote, height: H, round: R,
          proposal_commitment: Some(ATTACKER_BLOCK_HASH),
          voter: V1,   signature: RawSignature::default() }
   Vote { ..., voter: V2, signature: RawSignature::default() }
   Vote { ..., voter: V3, signature: RawSignature::default() }

3. Broadcast all three to the victim node.
   → handle_vote passes membership check for V1/V2/V3.
   → Signature check is skipped (TODO comment).
   → state_machine accumulates 3 prevotes → upon_prevote_quorum fires.
   → Victim node broadcasts its own Precommit for ATTACKER_BLOCK_HASH.

4. Craft 3 forged Precommit messages (same pattern) and broadcast.
   → state_machine accumulates quorum of precommits.
   → DecisionReached(ATTACKER_BLOCK_HASH) is returned.
   → SequencerConsensusContext::decision_reached commits the attacker-chosen
      block to the batcher and state-sync.

Result: victim node commits wrong state root / receipts / events for height H.
```
