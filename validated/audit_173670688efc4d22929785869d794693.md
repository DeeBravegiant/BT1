Audit Report

## Title
Permissionless `updateRSETHPrice()` With No Cooldown Enables Oracle-Dip Minting Arbitrage - (File: contracts/LRTOracle.sol)

## Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function with no minimum-elapsed-time guard. An unprivileged caller can invoke it at the exact moment a supported LST oracle price is temporarily depressed (but within the `pricePercentageLimit` band), writing a lower `rsETHPrice` to storage. Because `LRTDepositPool.getRsETHAmountToMint()` divides by the stored `rsETHPrice`, a subsequent ETH deposit in the same transaction mints more rsETH than the fair price warrants, diluting existing holders.

## Finding Description

`updateRSETHPrice()` is declared `public whenNotPaused` with no cooldown, no `lastUpdated` check, and no per-block rate-limit: [1](#0-0) 

Internally, `_updateRsETHPrice()` computes `newRsETHPrice` as `(totalETHInProtocol - protocolFeeInETH) / rsethSupply`, where `totalETHInProtocol` is the live sum of all supported assets valued at their **current** oracle prices: [2](#0-1) 

When a supported LST oracle price (e.g., stETH/ETH) dips within the `pricePercentageLimit` band, `newRsETHPrice` falls below `highestRsethPrice` but the downside-protection branch does **not** pause — it only pauses when the decrease **exceeds** the limit: [3](#0-2) 

For a sub-limit dip, execution falls through and `rsETHPrice` is unconditionally written to the depressed value: [4](#0-3) 

`LRTDepositPool.getRsETHAmountToMint()` then uses this stored value as the denominator: [5](#0-4) 

For an ETH deposit, `lrtOracle.getAssetPrice(ETH_TOKEN)` returns `1e18` regardless of the LST dip, so the numerator is unaffected while the denominator is depressed. The attacker can execute both calls atomically in a single transaction: (1) call `updateRSETHPrice()` to lock in the depressed price, then (2) call `depositETH()` to mint excess rsETH at that price. The `minRSETHAmountExpected` slippage guard is set by the attacker themselves and provides no protection.

## Impact Explanation

**High — Theft of unclaimed yield.**

Existing rsETH holders accumulate yield as the protocol's TVL grows relative to rsETH supply. When the attacker mints excess rsETH at a depressed `rsETHPrice`, the new rsETH is under-collateralised at the moment of minting. Once the LST oracle recovers, `rsETHPrice` rises again, but the attacker's excess rsETH participates in that recovery. The yield that should have accrued to existing holders is instead captured by the attacker — a direct, proportional theft of yield.

## Likelihood Explanation

**Medium.** stETH/ETH and similar LST oracle prices fluctuate naturally by fractions of a percent daily. The attacker requires no special role, no flash loan, and no oracle manipulation — only a monitoring bot and a contract that calls `updateRSETHPrice()` followed by `depositETH()` atomically. The attack is bounded by `pricePercentageLimit` but even a 0.5–1% dip (well within normal market noise) is sufficient for meaningful profit at scale. The attack is repeatable every time a qualifying dip occurs.

## Recommendation

Add a minimum-elapsed-time guard to `updateRSETHPrice()`:

```solidity
uint256 public minUpdateInterval; // e.g. 1 hours
uint256 public lastPriceUpdateTime;

function updateRSETHPrice() public whenNotPaused {
    require(
        block.timestamp >= lastPriceUpdateTime + minUpdateInterval,
        "UpdateTooFrequent"
    );
    lastPriceUpdateTime = block.timestamp;
    _updateRsETHPrice();
}
```

The privileged `updateRSETHPriceAsManager()` path should bypass this check so the protocol can respond to emergencies. The interval should reflect the minimum meaningful yield accrual period (e.g., 1–8 hours).

## Proof of Concept

```
Block N:
  stETH/ETH Chainlink price: 0.990 (natural 1% dip, within pricePercentageLimit of 2%)
  Current rsETHPrice (stored): 1.050 ETH  (highestRsethPrice = 1.050)

Attacker contract (single transaction):
  1. Call LRTOracle.updateRSETHPrice()
       totalETHInProtocol drops ~1% due to stETH devaluation
       newRsETHPrice ≈ 1.0395 ETH  (< highestRsethPrice, within limit → no pause)
       rsETHPrice storage ← 1.0395

  2. Call LRTDepositPool.depositETH{value: 1000 ETH}(0, "")
       rsethAmountToMint = 1000e18 * 1e18 / 1.0395e18 ≈ 962.0 rsETH
       Fair amount at 1.050:                              952.4 rsETH
       Excess minted:                                     ≈ 9.6 rsETH

Block N+k (stETH recovers):
  LRTOracle.updateRSETHPrice() called by anyone
       rsETHPrice recovers to ~1.050
  Attacker's 9.6 excess rsETH × 1.050 ≈ 10.1 ETH profit
  Extracted from existing holders via dilution.
```

Foundry fork test plan: fork mainnet, mock a Chainlink stETH/ETH feed returning `0.99e18`, deploy an attacker contract that calls `updateRSETHPrice()` then `depositETH{value: 1000 ether}(0, "")` atomically, assert that `rsethAmountToMint > getRsETHAmountToMint(ETH, 1000 ether)` computed at the pre-dip price, and assert that existing holders' rsETH redemption value decreases after the attack.
