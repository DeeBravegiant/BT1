Audit Report

## Title
Missing Staleness Check in `ChainlinkPriceOracle.getAssetPrice()` Allows Stale Price to Deflate `rsETHPrice` and Over-Mint rsETH - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards `updatedAt` and `answeredInRound`, accepting any price regardless of age. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, an attacker can wait for a Chainlink feed to go stale at a below-market price, trigger a price update that deflates the stored `rsETHPrice`, and deposit to mint rsETH at a discount — stealing yield from existing rsETH holders.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches Chainlink round data but reads only the `price` field, discarding `updatedAt` and `answeredInRound`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
```

The protocol's own `ChainlinkOracleForRSETHPoolCollateral` (used for L2 pool collateral) explicitly validates both staleness indicators before accepting a price:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

`ChainlinkPriceOracle` is the oracle registered in `LRTOracle.assetPriceOracle` for supported LSTs. Its output flows into two critical paths:

**Path 1 — `rsETHPrice` update (public, permissionless):**
`LRTOracle.updateRSETHPrice()` (public, `whenNotPaused` only) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` for every supported LST → stale price accepted → `totalETHInProtocol` under-counted → `rsETHPrice` deflated and written to storage.

**Path 2 — rsETH minting:**
`LRTDepositPool.getRsETHAmountToMint()` divides by the stored `rsETHPrice`:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

A deflated `rsETHPrice` in the denominator directly inflates the rsETH minted per unit deposited.

**Partial mitigation — `pricePercentageLimit`:**
`_updateRsETHPrice()` contains a downside-protection check: if `newRsETHPrice` drops more than `pricePercentageLimit` below `highestRsethPrice`, the protocol auto-pauses. However, this guard is bypassed entirely when `pricePercentageLimit == 0` (its default unset value), and it only triggers on drops exceeding the threshold — a stale price causing a sub-threshold drop (e.g., within 1%) still results in a deflated `rsETHPrice` being stored and deposits proceeding normally.

## Impact Explanation

When a Chainlink feed for a supported LST goes stale at a price lower than the true market price:

1. `_getTotalEthInProtocol()` under-counts the protocol's ETH value.
2. `rsETHPrice = totalETHInProtocol / rsethSupply` is deflated.
3. A depositor calling `depositETH()` receives `ETH_deposited * 1e18 / deflatedRsETHPrice` rsETH — more than the deposited ETH is worth.
4. When the oracle price corrects and `rsETHPrice` is updated upward, the attacker's rsETH is redeemable for more ETH than deposited.

This constitutes **theft of unclaimed yield from existing rsETH holders** via dilution: the attacker's over-minted rsETH claims a larger share of the protocol's total ETH, reducing the redemption value for all other holders.

**Impact: High — Theft of unclaimed yield.**

## Likelihood Explanation

Chainlink price feeds operate on a deviation threshold (typically 0.5%) plus a heartbeat (typically 24 hours). During low-volatility periods, a feed can remain at its last reported price for up to 24 hours without triggering an update. Network congestion or oracle node failures can also cause feeds to go stale. The attack requires no special privileges: `updateRSETHPrice()` is public and `depositETH()` / `depositAsset()` are open to any user. The attack is most viable when `pricePercentageLimit` is 0 or when the stale price deviation is within the configured limit.

**Likelihood: Low-Medium** — requires a Chainlink feed to go stale at a lower price, but this is a realistic on-chain condition.

## Recommendation

Add staleness validation to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the checks already present in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optionally: if (block.timestamp - updatedAt > MAX_STALENESS) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

1. Assume stETH/ETH Chainlink feed last reported `1.05e18` (true value) but has gone stale; it still returns `1.02e18` (3% below actual). Assume `pricePercentageLimit` is 0 or the drop is within the configured limit.
2. Protocol holds 1000 stETH. True `totalETHInProtocol` = 1050 ETH. Stale `totalETHInProtocol` = 1020 ETH.
3. With `rsethSupply = 1000`, true `rsETHPrice = 1.05e18`. Stale `rsETHPrice = 1.02e18`.
4. Attacker calls `LRTOracle.updateRSETHPrice()` — stored `rsETHPrice` drops to `1.02e18`.
5. Attacker calls `LRTDepositPool.depositETH{value: 10 ether}()`:
   - `rsethAmountToMint = 10e18 * 1e18 / 1.02e18 ≈ 9.804 rsETH`
   - Fair amount at true price: `10e18 / 1.05e18 ≈ 9.524 rsETH`
   - Excess: `≈ 0.28 rsETH` (~2.9% over-mint)
6. When the Chainlink feed updates and `rsETHPrice` corrects to `1.05e18`, the attacker's `9.804 rsETH` is redeemable for `≈ 10.29 ETH` — a profit of `≈ 0.29 ETH` at the expense of existing holders.

**Foundry fork test plan:**
- Fork mainnet with a stETH/ETH Chainlink feed.
- Warp `block.timestamp` past the feed's heartbeat without triggering a feed update (simulating staleness by mocking `latestRoundData` to return an old `updatedAt`).
- Call `updateRSETHPrice()` and record `rsETHPrice`.
- Call `depositETH` and record rsETH minted.
- Restore the oracle to the true price, call `updateRSETHPrice()` again.
- Assert that the attacker's rsETH balance is redeemable for more ETH than deposited, at the expense of other holders' share.
