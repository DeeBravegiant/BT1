### Title
`ProposalInit.builder` Not Validated in `is_proposal_init_valid`, Allowing a Malicious Proposer to Commit an Arbitrary Sequencer Address and Redirect Fees — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`builder_address` is set once from `ContextStaticConfig` at node startup and is never refreshed through the dynamic-config update path. More critically, the validator's `is_proposal_init_valid` function checks every other security-sensitive field of `ProposalInit` (`height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, `fee_proposal_fri`) but **never checks `init_proposed.builder`**. A malicious proposer can therefore broadcast a `ProposalInit` with an arbitrary `builder` address; every honest validator will accept the proposal, execute the block with that address as the sequencer, and commit a block header whose `sequencer` field (and therefore block hash) is wrong, while all transaction fees are transferred to the attacker-controlled address.

---

### Finding Description

**Static-config-only `builder_address` (the direct analog to the Midas bug)**

`builder_address` lives in `ContextStaticConfig`:

```rust
pub struct ContextStaticConfig {
    ...
    pub builder_address: ContractAddress,
    ...
}
``` [1](#0-0) 

`update_dynamic_config()` only fetches and applies `ContextDynamicConfig`; `ContextStaticConfig` (and therefore `builder_address`) is never refreshed at runtime. The TODO comment in the proposer path acknowledges this is a placeholder:

```rust
// TODO(Asmaa): Get it from committee once we have it.
builder_address: self.config.static_config.builder_address,
``` [2](#0-1) 

**Missing validation of `builder` in `is_proposal_init_valid`**

When a validator receives a proposal, `is_proposal_init_valid` enforces:

```rust
if !(init_proposed.height == proposal_init_validation.height
    && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
    && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
{
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
``` [3](#0-2) 

`init_proposed.builder` is never compared against any locally-trusted reference. The full list of checked fields is: `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, four L1 gas prices, and `fee_proposal_fri`. `builder` is absent from every check.

**How `builder` flows into the committed block**

After validation passes, `init.builder` is used directly as the sequencer address in the block header sent to state sync:

```rust
let sequencer = SequencerContractAddress(init.builder);
let block_header_without_hash = BlockHeaderWithoutHash {
    ...
    sequencer,
    l1_da_mode: init.l1_da_mode,
    ...
};
``` [4](#0-3) 

It is also passed to the batcher via `convert_to_sn_api_block_info(&init)`, which sets `block_info.sequencer_address`. The blockifier uses `block_context.block_info.sequencer_address` as the fee-transfer recipient for every transaction in the block. [5](#0-4) 

**Why the `ProposalFin` commitment check does not catch this**

Both the proposer's batcher and the validator's batcher derive `sequencer_address` from the same `init.builder` value. They therefore produce identical execution results and identical `ProposalCommitment` values. The final check `built_block != received_fin.proposal_commitment` passes, and the wrong block is committed unanimously. [6](#0-5) 

---

### Impact Explanation

A malicious validator that wins a proposal round sets `ProposalInit.builder` to an attacker-controlled address. All honest validators accept the proposal (no check on `builder`), execute the block with that address as sequencer, and commit:

1. **Wrong block header** — `sequencer` field in `BlockHeaderWithoutHash` is the attacker's address; this propagates into the block hash, corrupting the canonical chain state.
2. **Fee theft** — every transaction fee in the block is transferred to the attacker's address instead of the legitimate sequencer, constituting a direct economic loss.

Both effects are permanent once the block is committed to storage and propagated via state sync.

---

### Likelihood Explanation

Any validator that is legitimately selected as the round proposer can trigger this. Validator selection is a normal, non-privileged operation in BFT consensus. No special access beyond being a committee member is required. The attack is silent: honest validators see a valid `ProposalFin` commitment and vote to commit the block.

---

### Recommendation

Add a check inside `is_proposal_init_valid` that rejects proposals whose `builder` field does not match the locally-configured (or committee-derived) expected sequencer address:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

`expected_builder` should be added to `ProposalInitValidation` and populated from the committee/staking manager (resolving the existing TODO), or at minimum from `self.config.static_config.builder_address` until the committee path is implemented. Additionally, `builder_address` should be moved to `ContextDynamicConfig` so it can be updated at runtime without a node restart, completing the analog fix to the Midas `tokensReceiver` setter gap.

---

### Proof of Concept

1. Malicious validator `M` is selected as proposer for height `H`.
2. `M` constructs `ProposalInit { builder: attacker_address, height: H, l1_da_mode: <correct>, l2_gas_price_fri: <correct>, ... }` with all other fields matching what honest validators expect.
3. `M` streams the proposal to the network.
4. Each honest validator calls `is_proposal_init_valid`: all checked fields pass; `builder` is never checked.
5. `initiate_validation` is called with `block_info` derived from `init`, setting `sequencer_address = attacker_address` in the batcher's block context.
6. The batcher executes all transactions, transferring fees to `attacker_address`.
7. The batcher returns a `ProposalCommitment` that matches `M`'s `ProposalFin` (both used the same `init.builder`).
8. All validators precommit; the block is decided and committed to storage with `sequencer = attacker_address` and all fees credited to the attacker. [7](#0-6) [8](#0-7) [9](#0-8)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

```rust
async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}
