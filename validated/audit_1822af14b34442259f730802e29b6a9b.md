Audit Report

## Title
Missing `latestRoundData()` Output Validation Allows Stale Price to Corrupt rsETH Exchange Rate - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `price`, performing no staleness, completeness, or sign checks. A stale (artificially low) price propagates into `LRTOracle._updateRsETHPrice()`, deflating `rsETHPrice` and allowing any caller to mint excess rsETH at the expense of existing holders. The same three-check pattern is already correctly applied in the sibling contract `ChainlinkOracleForRSETHPoolCollateral.getRate()`, confirming the team is aware of the requirement.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` fetches the price as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

All five return values are available (`roundId`, `price`, `startedAt`, `updatedAt`, `answeredInRound`) but only `price` is used. Three checks are absent:

1. **No staleness check** — `answeredInRound >= roundId` is never verified.
2. **No incomplete-round check** — `updatedAt != 0` is never verified.
3. **No sign check** — `price > 0` is never verified.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` applies all three:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The unvalidated price flows through the following call chain:

- `LRTOracle.getAssetPrice()` → `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` [3](#0-2) 
- `LRTOracle._getTotalEthInProtocol()` accumulates `totalETHInProtocol` using the stale price [4](#0-3) 
- `LRTOracle._updateRsETHPrice()` computes and writes `rsETHPrice = newRsETHPrice` [5](#0-4) 
- `LRTDepositPool.getRsETHAmountToMint()` divides by the deflated `rsETHPrice` to mint rsETH [6](#0-5) 

`updateRSETHPrice()` is public and requires no role: [7](#0-6) 

**Existing guard analysis — `pricePercentageLimit`:**

`_updateRsETHPrice()` contains a downside-protection check:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
if (isPriceDecreaseOffLimit) { ... _pause(); return; }
``` [8](#0-7) 

This guard is **insufficient** for two reasons:
1. It is disabled entirely when `pricePercentageLimit == 0` (the default unset value), which is a realistic deployment state.
2. Even when set, stale prices that cause a decrease *within* the configured limit (e.g., a 0.4% stale deviation against a 1% limit) pass through silently and are written to `rsETHPrice`, enabling the theft scenario at a smaller but still real scale.

## Impact Explanation

**High — Theft of unclaimed yield.**

When a Chainlink LST/ETH feed goes stale (last reported price below the true current price), any caller can invoke `updateRSETHPrice()` to write the deflated price to `rsETHPrice`. Subsequent depositors calling `depositAsset()` or `depositETH()` receive more rsETH than their assets are worth (division by a smaller denominator), directly diluting the proportional claim of all existing rsETH holders on the protocol's ETH TVL. This constitutes theft of accrued yield belonging to existing holders.

**Secondary — Negative price DoS (Medium).**

If a feed returns `price <= 0`, `uint256(price)` wraps to a near-`2^256` value in Solidity 0.8.x; the subsequent `* 1e18` multiplication overflows and reverts, causing every call to `updateRSETHPrice()` to revert until the feed recovers. This maps to temporary freezing of the price-update mechanism.

## Likelihood Explanation

Chainlink feeds have documented histories of returning stale data during network congestion, sequencer downtime (on L2s), or heartbeat misses. `updateRSETHPrice()` is public and permissionless — any actor, including one who benefits from a deflated `rsETHPrice`, can call it at the worst possible moment. No special access, governance capture, or key compromise is required. The attack is repeatable as long as the feed remains stale.

## Recommendation

Apply the same three-check pattern already used in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Chainlink: stale price");
    require(updatedAt != 0,            "Chainlink: incomplete round");
    require(price > 0,                 "Chainlink: invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

1. A Chainlink LST/ETH feed (e.g., stETH/ETH) enters a stale state: `answeredInRound < roundId`, last reported price is 0.5% below the true current price, and `pricePercentageLimit` is either 0 or set to ≥ 0.5%.
2. Attacker calls `LRTOracle.updateRSETHPrice()` (public, no access control). [7](#0-6) 
3. `_getTotalEthInProtocol()` calls `ChainlinkPriceOracle.getAssetPrice(stETH)`, which returns the stale low price with no rejection. [1](#0-0) 
4. `totalETHInProtocol` is underestimated; `newRsETHPrice` is set below the true value. The `pricePercentageLimit` guard either does not fire (limit is 0 or deviation is within limit) and `rsETHPrice` is written with the deflated value. [5](#0-4) 
5. Attacker immediately calls `LRTDepositPool.depositAsset()`. `getRsETHAmountToMint()` divides by the deflated `rsETHPrice`, minting more rsETH than the deposited assets are worth. [6](#0-5) 
6. Existing rsETH holders are diluted; the attacker has extracted yield that belonged to them.

**Foundry fork test plan:** Fork mainnet, mock a Chainlink aggregator returning `answeredInRound < roundId` with a price 0.5% below spot, call `updateRSETHPrice()`, assert `rsETHPrice` is deflated, then call `depositAsset()` and assert the minted rsETH exceeds the fair-value amount computed using the true price.
