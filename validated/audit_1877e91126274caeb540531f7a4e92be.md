Audit Report

## Title
`RSETHPriceFeed` Returns ETH/USD `updatedAt` Timestamp, Masking Stale rsETH/ETH Rate - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed` implements `AggregatorV3Interface` and computes rsETH/USD by multiplying the rsETH/ETH rate from `LRTOracle` with the ETH/USD Chainlink price. Both `latestRoundData()` and `getRoundData()` return `updatedAt` sourced exclusively from the ETH/USD feed, not from the rsETH oracle. Because `LRTOracle.rsETHPrice` is operator-updated and exposes no timestamp, any external integrator performing a standard Chainlink staleness check will silently accept a stale rsETH price as fresh.

## Finding Description
In `contracts/oracles/RSETHPriceFeed.sol`, `latestRoundData()` (L63–70) and `getRoundData()` (L53–61) both destructure all five return values — including `updatedAt` — from `ETH_TO_USD`, then override only `answer` with `int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18`. The `updatedAt` field is returned unchanged from the ETH/USD feed.

The `IRSETHOracle` interface (L22–24) exposes only `rsETHPrice()` with no timestamp. `LRTOracle` stores `rsETHPrice` as a plain `uint256` state variable (L28) with no `lastUpdated` field. `updateRSETHPrice()` is gated by `whenNotPaused` (L87), meaning during any protocol pause the rsETH price is frozen while the ETH/USD Chainlink feed continues updating normally. No existing check in `RSETHPriceFeed` compensates for this: there is no guard, no minimum-timestamp logic, and no documentation warning.

The result is that `updatedAt` reflects only the ETH/USD component's freshness, while the rsETH/ETH component embedded in `answer` may be arbitrarily stale.

## Impact Explanation
`RSETHPriceFeed` is deployed as a Chainlink-compatible oracle intended for external protocol integration (e.g., lending markets, DEXes, vaults — as evidenced by its `AggregatorV3Interface` implementation and its listing in the README as `RSETHPriceFeed (Morph)`). The `updatedAt` field is the canonical staleness signal that downstream Chainlink integrators rely on. By returning only the ETH/USD timestamp, the contract fails to deliver its core promise: accurate, trustworthy price freshness metadata for rsETH/USD. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value within the LRT-rsETH protocol itself.**

## Likelihood Explanation
No attacker action is required. The condition triggers whenever `LRTOracle` is not updated while the ETH/USD Chainlink feed continues updating — a realistic, non-adversarial scenario. `LRTOracle` has an explicit `paused` state (L38) that blocks `updateRSETHPrice()` via `whenNotPaused` (L87). Any protocol pause, operator delay, or emergency freeze produces this condition. The `RSETHPriceFeed` is already deployed on Morph mainnet, making this a live exposure.

## Recommendation
Return the minimum of the ETH/USD `updatedAt` and the rsETH oracle's last-update timestamp. Add a `lastUpdatedAt` storage variable to `LRTOracle` that is written in `_updateRsETHPrice()` alongside `rsETHPrice`, expose it via `ILRTOracle`, and update `IRSETHOracle` in `RSETHPriceFeed` to include a `rsETHPriceUpdatedAt()` getter. In both `latestRoundData()` and `getRoundData()`, set `updatedAt = min(ethToUsdUpdatedAt, rsETHOracleUpdatedAt)` before returning.

## Proof of Concept
1. Deploy `LRTOracle` and call `updateRSETHPrice()` at `T=0`; `rsETHPrice` is set (e.g., 1.05e18).
2. Call `LRTOracle.pause()` (or trigger the auto-pause via price deviation); `updateRSETHPrice()` now reverts.
3. At `T=3600`, the ETH/USD Chainlink feed updates normally; its `updatedAt` becomes `3600`.
4. At `T=7200`, call `RSETHPriceFeed.latestRoundData()`.
5. Observe: `updatedAt = 3600` (from ETH/USD), `answer = 1.05 * ethUsdPrice` (rsETH component is 7200s stale).
6. An external lending protocol checks `block.timestamp - updatedAt = 3600s` against a 1-hour threshold — passes.
7. The protocol prices rsETH using a 2-hour-stale rate with no indication of staleness.

Foundry fork test: fork mainnet, deploy `RSETHPriceFeed` pointing at the live `LRTOracle` and ETH/USD feed, call `LRTOracle.pause()` as the manager, warp `block.timestamp` forward by 2 hours, call `latestRoundData()`, and assert that `block.timestamp - updatedAt < 3600` while `rsETHPrice` has not changed since before the warp.
