Audit Report

## Title
Stale Cross-Chain Rate in `CrossChainRateReceiver` Allows Depositors to Mint Excess agETH, Stealing Unclaimed Yield — (`contracts/agETH/AGETHRateReceiver.sol` / `contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver` stores `lastUpdated` on every `lzReceive` call but `getRate()` returns the stored `rate` unconditionally with no staleness enforcement. `AGETHPoolV3.deposit()` consumes this rate directly to compute agETH minted. When the L2 rate lags the true L1 rate, a depositor receives more agETH than their ETH contribution warrants, extracting yield that belongs to existing holders.

## Finding Description
`CrossChainRateReceiver.lzReceive()` records `lastUpdated = block.timestamp` at line 97 of `contracts/cross-chain/CrossChainRateReceiver.sol`, but `getRate()` at lines 103–105 returns `rate` without any reference to `lastUpdated`. There is no `MAX_STALENESS` constant, no `require`, and no revert path for an aged rate anywhere in the contract.

`AGETHPoolV3.getRate()` (line 104–106 of `contracts/agETH/AGETHPoolV3.sol`) delegates directly to `IOracle(agETHOracle).getRate()`. `deposit()` (lines 115–128) calls `viewSwapAgETHAmountAndFee` (lines 160–169), which computes:

```
agETHAmount = amountAfterFee * 1e18 / agETHToETHrate
```

where `agETHToETHrate` is the unchecked stale value. Because the agETH/ETH rate monotonically increases as staking rewards accrue on L1, any period without a LayerZero update leaves the L2 rate below the true rate. A depositor who calls `deposit()` during this window receives `amountAfterFee * 1e18 / staleRate` agETH, which is strictly greater than the fair amount `amountAfterFee * 1e18 / trueRate`. The surplus agETH represents a claim on ETH that already belongs to existing holders via accrued yield.

`AGETHRateReceiver` (`contracts/agETH/AGETHRateReceiver.sol`) is a thin subclass of `CrossChainRateReceiver` that adds no additional validation, confirming no staleness guard exists anywhere in the inheritance chain.

## Impact Explanation
**High — Theft of unclaimed yield.** Existing agETH holders have accrued staking yield reflected in the rising L1 rate. A depositor minting at a stale (lower) rate receives a disproportionately large share of total agETH supply. On redemption at the true rate, they recover more ETH than deposited; the difference is extracted directly from yield that had already accrued to prior holders. The magnitude scales linearly with staleness duration and deposit size.

## Likelihood Explanation
`deposit()` is fully permissionless. `lastUpdated` is a public storage variable, so an attacker can trivially monitor on-chain when the rate was last refreshed and time a large deposit to coincide with maximum staleness. LayerZero message delivery is not guaranteed on any fixed schedule; keeper inactivity, network congestion, or normal inter-update windows all create exploitable windows. No privileged access, flash loan, or governance action is required.

## Recommendation
Add a staleness check in `CrossChainRateReceiver.getRate()`:

```solidity
uint256 public constant MAX_RATE_AGE = 24 hours; // configurable by owner

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= MAX_RATE_AGE, "Rate is stale");
    return rate;
}
```

This causes `deposit()` to revert when the oracle is stale. Alternatively, enforce the check inside `AGETHPoolV3.viewSwapAgETHAmountAndFee` directly, or expose `lastUpdated` through the `IOracle` interface so callers can enforce their own freshness policy.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

// Fork test on L2 deployment
// 1. Deploy AGETHPoolV3 pointing to a MockAGETHRateReceiver
// 2. MockAGETHRateReceiver.getRate() returns STALE_RATE (e.g. 1.04e18)
//    while the true current L1 rate is TRUE_RATE (e.g. 1.05e18)
// 3. Attacker calls pool.deposit{value: 100 ether}("")
// 4. agETHMinted = 100e18 * 1e18 / 1.04e18 ≈ 96.15 agETH
// 5. At true rate: 96.15 * 1.05e18 / 1e18 ≈ 100.96 ETH redeemable on L1
// 6. assert(redeemableETH > deposit) — attacker profits ~0.96 ETH per 100 ETH
//    extracted from existing holders' unclaimed yield

contract StaleRatePoC {
    uint256 constant TRUE_RATE  = 1.05e18;
    uint256 constant STALE_RATE = 1.04e18; // ~1 day of staleness

    function testStaleRateExploit(AGETHPoolV3 pool) external payable {
        uint256 deposit = 100 ether;
        // MockAGETHRateReceiver is pre-configured to return STALE_RATE
        uint256 agETHMinted = pool.deposit{value: deposit}("");
        uint256 redeemableETH = agETHMinted * TRUE_RATE / 1e18;
        assert(redeemableETH > deposit); // surplus extracted from prior holders
    }
}
```
