Audit Report

## Title
Stale `rsETHPrice` Masked as Fresh via ETH/USD `updatedAt` Under Block Stuffing — (`contracts/oracles/RSETHPriceFeed.sol`)

## Summary
`RSETHPriceFeed.latestRoundData()` computes rsETH/USD by multiplying the Chainlink ETH/USD answer by `LRTOracle.rsETHPrice`, but returns `updatedAt` exclusively from the ETH/USD feed. Because no timestamp is stored for when `rsETHPrice` was last written, an attacker who uses block stuffing to prevent `updateRSETHPrice()` from being mined can cause the feed to serve a below-true rsETH/USD price while advertising a fresh `updatedAt`, defeating any downstream staleness guard.

## Finding Description
`RSETHPriceFeed.latestRoundData()` assembles its return values as:

```solidity
// contracts/oracles/RSETHPriceFeed.sol L68-69
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`updatedAt` is the ETH/USD Chainlink heartbeat timestamp. `rsETHPrice` is a separate storage variable in `LRTOracle` updated only when `_updateRsETHPrice()` executes successfully (line 313). No `rsETHPriceUpdatedAt` timestamp is stored, and `RSETHPriceFeed` performs no staleness check on `rsETHPrice`.

`updateRSETHPrice()` is `public` with only a `whenNotPaused` guard:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

**Exploit path:**
1. Attacker fills consecutive blocks to their gas limit, preventing any `updateRSETHPrice()` call from being included.
2. During this window, EigenLayer rewards accrue, increasing the true rsETH/ETH exchange rate.
3. `LRTOracle.rsETHPrice` remains frozen at its pre-accrual value.
4. Every call to `RSETHPriceFeed.latestRoundData()` returns an `answer` below the true rsETH/USD price, yet returns an `updatedAt` at most ~1 hour old (ETH/USD heartbeat), making the feed appear fresh to any consumer relying on `updatedAt` for staleness detection.

No existing check reconciles the freshness of the two components of the returned `answer`.

## Impact Explanation
**Low — Block stuffing.** The oracle serves a stale (below-true) rsETH/USD price for the duration of the stuffed window. The `updatedAt` field reflects only ETH/USD freshness, not `rsETHPrice` freshness, so downstream staleness checks are bypassed. This directly matches the allowed impact "Low — Block stuffing."

## Likelihood Explanation
Block stuffing on Ethereum mainnet requires paying for the full block gas limit per block, making it expensive. The attack becomes economically rational if the attacker holds a short position against rsETH collateral in a lending protocol and the profit from triggered liquidations exceeds the stuffing cost. No code-level mitigation exists within `RSETHPriceFeed` or `LRTOracle` to prevent or detect this condition.

## Recommendation
1. **Track `rsETHPrice` update time.** Add `uint256 public rsETHPriceUpdatedAt` in `LRTOracle` and set it alongside `rsETHPrice` at line 313:
   ```solidity
   rsETHPrice = newRsETHPrice;
   rsETHPriceUpdatedAt = block.timestamp;
   ```
2. **Return the correct `updatedAt` from `RSETHPriceFeed`.** Override `updatedAt` with the minimum of the two component timestamps:
   ```solidity
   uint256 rsETHPriceUpdatedAt = RS_ETH_ORACLE.rsETHPriceUpdatedAt();
   updatedAt = updatedAt < rsETHPriceUpdatedAt ? updatedAt : rsETHPriceUpdatedAt;
   ```
3. **Add a maximum staleness check inside `latestRoundData()`** that reverts if `rsETHPriceUpdatedAt` is older than a configured heartbeat.

## Proof of Concept
```solidity
function test_blockStuffingStalePrice() external {
    // 1. Record current rsETHPrice
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // 2. Simulate reward accrual: increase LST balance in deposit pool
    deal(address(stETH), depositPool, stETHBalance * 110 / 100);

    // 3. Simulate block stuffing: advance time without calling updateRSETHPrice()
    vm.warp(block.timestamp + 24 hours);

    // 4. Query the feed
    (, int256 answer,, uint256 updatedAt,) = rsETHPriceFeed.latestRoundData();

    // 5. answer is stale (below true price) because rsETHPrice was not updated
    assertEq(uint256(answer) / ethUsdPrice, priceBefore); // frozen at pre-accrual value

    // 6. updatedAt is recent (ETH/USD heartbeat), masking the staleness
    assertGt(updatedAt, block.timestamp - 1 hours);
}
```
