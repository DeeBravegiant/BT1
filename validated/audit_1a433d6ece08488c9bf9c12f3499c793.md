### Title
Detached `burn.and(mint)` Promise in `ft_on_transfer` `SwapMigratedToken` Branch Permanently Locks User Tokens on Failure - (File: `near/omni-bridge/src/lib.rs`)

### Summary

`ft_on_transfer` is the NEAR analog of `onERC721Received`: it is a public callback invoked by any NEP-141 token contract when tokens are pushed to the bridge via `ft_transfer_call`. The `SwapMigratedToken` branch immediately returns `U128(0)` (telling the token contract to keep all tokens at the bridge) and then `.detach()`s the `burn.and(mint)` promise with no failure callback. If the detached promise fails for any reason, the transferred tokens are permanently locked inside the bridge contract with no recovery path.

### Finding Description

In `ft_on_transfer` at line 279–283:

```rust
BridgeOnTransferMsg::SwapMigratedToken => {
    self.swap_migrated_token(sender_id, token_id, amount)
        .detach();
    PromiseOrPromiseIndexOrValue::Value(U128(0))
}
``` [1](#0-0) 

`U128(0)` is returned synchronously to the NEP-141 token contract, which interprets it as "refund zero tokens" — the full `amount` is permanently credited to the bridge. The actual work is done by the detached promise returned from `swap_migrated_token`:

```rust
fn swap_migrated_token(...) -> Promise {
    let new_token = self.migrated_tokens.get(&old_token)
        .near_expect(BridgeError::TokenNotMigrated);
    let burn = ext_token::ext(old_token).burn(amount);
    let mint = ext_token::ext(new_token).mint(sender_id, amount, None);
    burn.and(mint)
}
``` [2](#0-1) 

`.detach()` means NEAR's runtime discards the promise result entirely. There is no `#[callback_result]` handler, no refund branch, and no re-entrancy guard. If either the `burn` or the `mint` cross-contract call fails — for any reason — the state changes from those sub-calls are rolled back, but the `U128(0)` return value that already committed the tokens to the bridge is **not** rolled back. The tokens are irrecoverably stuck.

Contrast this with every other branch in `ft_on_transfer` (`InitTransfer`, `FastFinTransfer`, `UtxoFinTransfer`), which all propagate their promise results back through `as_return()`, allowing the NEP-141 standard's refund mechanism to fire on failure. [3](#0-2) 

### Impact Explanation

**Critical — Irreversible fund lock.** Any user holding old migrated tokens who calls `ft_transfer_call` targeting the bridge with `msg = "SwapMigratedToken"` will have their tokens permanently locked inside the bridge contract if the detached `burn.and(mint)` promise fails. There is no admin escape hatch, no retry function, and no refund mechanism in the contract for this code path.

### Likelihood Explanation

**Medium.** The failure condition is reachable through ordinary use:

1. **Gas exhaustion.** `ft_transfer_call` is a multi-hop cross-contract call. The gas budget is split across: the token contract's transfer logic, `ft_on_transfer` itself, and the detached `burn.and(mint)` pair. A user who attaches the NEAR-recommended default gas (e.g., 30 TGas) may leave insufficient gas for both the `burn` and `mint` sub-calls. NEAR does not enforce a minimum gas check before `.detach()`.

2. **Parallel promise semantics.** `burn.and(mint)` schedules two independent cross-contract calls. If `burn` succeeds but `mint` fails (e.g., the new token contract's `mint` reverts due to a storage or access-control issue), the old tokens are burned but the user receives nothing — a net destruction of value. If `burn` fails but `mint` succeeds, new tokens are minted without the old supply being reduced, creating unbacked supply.

3. **No privileged role required.** Any holder of an old migrated token can trigger this path by calling `ft_transfer_call` on the old token contract with the correct `msg`.

### Recommendation

Replace the `.detach()` pattern with a proper callback that refunds the user on failure:

```rust
BridgeOnTransferMsg::SwapMigratedToken => {
    self.swap_migrated_token(sender_id.clone(), token_id, amount)
        .then(
            Self::ext(env::current_account_id())
                .with_static_gas(SWAP_CALLBACK_GAS)
                .swap_migrated_token_callback(sender_id, amount),
        )
        // Return the promise so ft_transfer_call can refund on panic
}
```

The `swap_migrated_token_callback` should inspect `#[callback_result]` and, on failure, call `ft_transfer_call` back to the old token contract to return `amount` to `sender_id`. This mirrors the pattern used by `init_transfer` and `fast_fin_transfer`.

### Proof of Concept

1. DAO calls `migrate_deployed_token(ChainKind::Eth, old_token, new_token)` — legitimate migration.
2. User calls `old_token.ft_transfer_call(bridge, amount, "SwapMigratedToken")` with 30 TGas.
3. Bridge's `ft_on_transfer` fires: `migrated_tokens.get(&old_token)` succeeds, `burn.and(mint)` is detached, `U128(0)` is returned → bridge now holds `amount` of `old_token`.
4. The detached promise runs with remaining gas. If gas is exhausted (or `mint` reverts), both sub-calls fail and their state is rolled back.
5. The bridge still holds `amount` of `old_token`. The user has neither old nor new tokens. No recovery function exists. [1](#0-0) [2](#0-1)
