Audit Report

## Title
`stakedButUnverifiedNativeETH` Permanently Non-Decrementable After Full EigenLayer Slash Traps NodeDelegator in Queue - (`contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

## Summary
When ETH is staked via `stake32Eth()` but `verifyWithdrawalCredentials()` has not yet been called, `stakedButUnverifiedNativeETH` is incremented. If the delegated EigenLayer operator is subsequently slashed to `maxMagnitude = 0`, `verifyWithdrawalCredentials()` becomes permanently uncallable because EigenLayer's `increaseDelegatedShares()` explicitly reverts at `maxMagnitude = 0`. This makes `stakedButUnverifiedNativeETH` permanently non-zero, causing `_checkResidueEthBalance()` to always revert with `NodeDelegatorHasETH`, permanently trapping the NodeDelegator in the queue with no admin escape hatch.

## Finding Description

**Root cause — `getEffectivePodShares()` sums two independent quantities:**

`getEffectivePodShares()` returns `stakedButUnverifiedNativeETH + withdrawableShare`. [1](#0-0)  After a full slash, `withdrawableShare` (derived from EigenLayer's `DelegationManager.getWithdrawableShares()`) correctly drops to 0. However, `stakedButUnverifiedNativeETH` is a local storage variable that EigenLayer's slashing mechanism never touches.

**`stakedButUnverifiedNativeETH` has exactly one write path for decrement:**

`stake32Eth()` increments it by 32 ether per validator. [2](#0-1)  The only decrement is in `verifyWithdrawalCredentials()`, which subtracts `validatorFields.length * 32 ether` and then calls `eigenPod.verifyWithdrawalCredentials()`. [3](#0-2)  Internally, `eigenPod.verifyWithdrawalCredentials()` calls `EigenPodManager.recordBeaconChainETHBalanceUpdate()` → `DelegationManager.increaseDelegatedShares()`. The `IDelegationManager` interface explicitly documents: *"If the operator was slashed 100% for the strategy (the operator's maxMagnitude = 0), then increasing delegated shares is blocked and will revert."* [4](#0-3)  After a full slash, this entire call chain reverts, making `verifyWithdrawalCredentials()` permanently uncallable and `stakedButUnverifiedNativeETH` permanently non-zero.

**`_checkResidueEthBalance()` gates NDC removal with a strict `!= 0` check:**

`_removeNodeDelegatorContractFromQueue()` calls `_checkResidueEthBalance()`, which reverts with `NodeDelegatorHasETH` if `getEffectivePodShares() != 0`. [5](#0-4)  Since `getEffectivePodShares()` permanently returns `stakedButUnverifiedNativeETH > 0`, this check always reverts. [6](#0-5) 

**No admin escape hatch exists.** A full grep of `NodeDelegator.sol` confirms `stakedButUnverifiedNativeETH` is only ever written in `stake32Eth()` (increment) and `verifyWithdrawalCredentials()` (decrement). There is no setter, no admin override, and no bypass path. [7](#0-6) 

**Exploit path:**
1. LRT Operator calls `stake32Eth()` — normal operation, `stakedButUnverifiedNativeETH += 32 ether`.
2. Before `verifyWithdrawalCredentials()` is called (a window of hours to days), an AVS slashes the delegated operator to `maxMagnitude = 0` via EigenLayer's `AllocationManager.slashOperator()`.
3. `verifyWithdrawalCredentials()` now permanently reverts; `stakedButUnverifiedNativeETH` is stuck.
4. `getEffectivePodShares()` permanently returns `>= 32 ether`.
5. `removeNodeDelegatorContractFromQueue()` permanently reverts with `NodeDelegatorHasETH`.
6. If the queue is at `maxNodeDelegatorLimit`, `addNodeDelegatorContractToQueue()` reverts with `MaximumNodeDelegatorLimitReached`, blocking protocol migration. [8](#0-7) 

## Impact Explanation

**Critical — Permanent freezing of funds.** The 32 ETH per unverified validator is in the beacon chain. After a full slash, the validator can exit and ETH lands in the EigenPod, but `startCheckpoint()` → `completeCheckpoint()` attempts to award shares via `increaseDelegatedShares()`, which reverts at `maxMagnitude = 0`. The ETH is irrecoverable through any EigenLayer mechanism. Additionally, the NodeDelegator is permanently trapped in the queue with no admin escape hatch, and if the queue is full, protocol migration to a new operator is permanently blocked.

## Likelihood Explanation

**Low-Medium.** Two preconditions must coincide: (1) ETH staked but credentials not yet verified — a normal operational window lasting hours to days; (2) the delegated operator is registered with an AVS that slashes to `maxMagnitude = 0`. EigenLayer's slashing mechanism is functioning as designed; this is not external protocol compromise. An operator choosing to register with a malicious or misbehaving AVS is a realistic risk in the EigenLayer ecosystem. The conditions are independent and can occur without any action by LRT users.

## Recommendation

1. **Add an admin function to reset `stakedButUnverifiedNativeETH`** (e.g., `onlyLRTAdmin`) for post-slash recovery, allowing the admin to zero out the variable when EigenLayer confirms the operator's `maxMagnitude = 0`.
2. **Alternatively, modify `_checkResidueEthBalance()`** to skip the `getEffectivePodShares()` check when the operator's `maxMagnitude == 0`, treating `stakedButUnverifiedNativeETH` as irrecoverably burned.
3. **Add a `forceRemoveNodeDelegator()` function** callable by admin that bypasses the ETH balance check when the operator has been fully slashed, to allow protocol migration.

## Proof of Concept

```solidity
function test_slashTrapsNDC() public {
    // 1. Stake 32 ETH, do NOT verify credentials
    nodeDelegator.stake32Eth(pubkey, sig, root);
    assertEq(nodeDelegator.stakedButUnverifiedNativeETH(), 32 ether);

    // 2. AVS slashes operator to maxMagnitude = 0 via EigenLayer AllocationManager
    IAllocationManager.SlashingParams memory params = IAllocationManager.SlashingParams({
        operator: operator,
        operatorSetId: 1,
        strategies: beaconStrategies,
        wadsToSlash: [1e18], // 100%
        description: "slash"
    });
    vm.prank(avs);
    allocationManager.slashOperator(avs, params);

    // 3. getEffectivePodShares() returns stakedButUnverifiedNativeETH (withdrawableShare = 0)
    assertEq(nodeDelegator.getEffectivePodShares(), 32 ether);

    // 4. removeNodeDelegatorContractFromQueue reverts permanently
    vm.prank(admin);
    vm.expectRevert(ILRTDepositPool.NodeDelegatorHasETH.selector);
    lrtDepositPool.removeNodeDelegatorContractFromQueue(address(nodeDelegator));

    // 5. verifyWithdrawalCredentials also reverts (maxMagnitude = 0 blocks increaseDelegatedShares)
    vm.prank(operator);
    vm.expectRevert();
    nodeDelegator.verifyWithdrawalCredentials(ts, stateRootProof, indices, proofs, fields);
}
```

        // 2. revert if node delegator contract has any asset balances.
