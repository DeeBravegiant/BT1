### Title
Harvester Can Unconditionally Redirect Farmer Block Reward to Arbitrary Address Without Enforcement - (`File: chia/farmer/farmer_api.py`)

### Summary
A connected harvester peer can set `farmer_reward_address_override` in both `NewProofOfSpace` and `RespondSignatures` protocol messages to redirect the farmer's 1/8 XCH block reward to any arbitrary puzzle hash. The CHIP-22 fee quality check in `notify_farmer_reward_taken_by_harvester_as_fee` is advisory-only (logs a warning), and `_process_respond_signatures` accepts the override unconditionally with zero enforcement. Any harvester — including a malicious third-party harvester — can divert the farmer's entire block reward to itself on every won block.

### Finding Description

The `NewProofOfSpace` streamable message includes an optional `farmer_reward_address_override` field: [1](#0-0) 

When the farmer receives a `NewProofOfSpace` with this field set, it calls `notify_farmer_reward_taken_by_harvester_as_fee`: [2](#0-1) 

That function performs a fee quality check but **only logs warnings** — it never blocks the flow or rejects the override: [3](#0-2) 

The farmer then proceeds to request signatures from the harvester. The harvester's `RespondSignatures` response also carries `farmer_reward_address_override`: [4](#0-3) 

In `_process_respond_signatures`, the farmer **unconditionally replaces** its own `farmer_target` with whatever address the harvester supplied, with no fee quality re-check: [5](#0-4) 

This overridden address is then embedded in `DeclareProofOfSpace` and broadcast to the full node as the canonical farmer reward address: [6](#0-5) 

### Impact Explanation

The farmer's 1/8 XCH block reward (`calculate_base_farmer_reward`) is permanently redirected to the harvester-controlled address on every block the harvester helps win. The farmer receives nothing. This is a direct, on-chain, irreversible reward diversion affecting XCH. It matches the Critical/High impact category: **unauthorized reward diversion affecting XCH**. [7](#0-6) 

### Likelihood Explanation

Any harvester that connects to a farmer — including commercial "third-party harvester" services explicitly contemplated by CHIP-22 — can exploit this. The attacker only needs a valid proof of space (which they legitimately produce as part of normal harvesting). No key compromise, no admin access, and no cryptographic break is required. The farmer has no way to detect or prevent the diversion at runtime because the only defense is a log warning.

### Recommendation

1. In `new_proof_of_space`, after calling `notify_farmer_reward_taken_by_harvester_as_fee`, **enforce** the fee quality check: if `fee_info is None` or `fee_quality > fee_threshold`, drop the proof and return without proceeding.
2. In `_process_respond_signatures`, re-validate that the `farmer_reward_address_override` in `RespondSignatures` matches the one originally declared in `NewProofOfSpace` and that the fee quality threshold was legitimately met before accepting the override.
3. Store the validated override decision (pass/fail) from `new_proof_of_space` and gate `_process_respond_signatures` on that stored result.

### Proof of Concept

1. Malicious harvester connects to a farmer's server (normal operation).
2. Harvester finds a valid proof of space for a signage point.
3. Harvester sends `NewProofOfSpace` with `farmer_reward_address_override = attacker_puzzle_hash` and `fee_info = None` (or any `applied_fee_threshold`).
4. Farmer calls `notify_farmer_reward_taken_by_harvester_as_fee` → logs a warning → **continues**.
5. Farmer sends `RequestSignatures` to the harvester.
6. Harvester responds with `RespondSignatures` also containing `farmer_reward_address_override = attacker_puzzle_hash`.
7. `_process_respond_signatures` sets `farmer_reward_address = attacker_puzzle_hash` unconditionally.
8. Farmer broadcasts `DeclareProofOfSpace` with `farmer_reward_address = attacker_puzzle_hash` to the full node.
9. Full node creates the block; the 1/8 XCH farmer reward coin is created at `attacker_puzzle_hash`.
10. Farmer receives 0 XCH for the block they won. [8](#0-7)
