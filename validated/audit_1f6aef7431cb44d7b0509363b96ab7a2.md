### Title
Silent `ft_transfer_call` Failure in `fin_transfer_send_tokens_callback` Permanently Finalizes Transfer Without Delivering Tokens — (`near/omni-bridge/src/lib.rs`)

---

### Summary

When a cross-chain transfer to NEAR includes a `msg` field (triggering `ft_transfer_call` / `mint`-with-message), a failure of that call is silently swallowed by `is_refund_required`. The transfer is already recorded in `finalised_transfers` before the token delivery attempt, and because `is_refund_required` returns `false` on `Err`, the callback takes the success branch: fees are paid, `FinTransferEvent` is emitted, and the transfer ID is never removed from `finalised_transfers`. The recipient receives zero tokens and the transfer can never be retried.

---

### Finding Description

**Step 1 — Transfer is permanently finalized before token delivery.**

In `process_fin_transfer_to_near`, `add_fin_transfer` is called first, inserting the transfer ID into `finalised_transfers`: [1](#0-0) 

Only after that does the function call `send_tokens` and chain `fin_transfer_send_tokens_callback`.

**Step 2 — `send_tokens` uses `ft_transfer_call` (or `mint`-with-message) when `msg` is non-empty.** [2](#0-1) 

Both paths (`mint` with `Some(msg)` for deployed tokens, `ft_transfer_call` for locked tokens) are async cross-contract calls that can fail.

**Step 3 — `is_refund_required` returns `false` on promise failure.** [3](#0-2) 

When `env::promise_result_checked` returns `Err` (the `ft_transfer_call` panicked), the function returns `false` — "don't refund" — with only a comment "Unexpected case."

**Step 4 — The callback takes the success branch.** [4](#0-3) 

Because `is_refund_required` returned `false`, the `else` branch executes: fees are minted/transferred to the fee recipient and `FinTransferEvent` is emitted. `remove_fin_transfer` is never called, so the transfer ID stays permanently in `finalised_transfers`.

The same silent-failure path applies when `msg` is empty (`is_ft_transfer_call = false`): `is_refund_required` unconditionally returns `false`, so a failed `ft_transfer` or `mint` (no-message) is also silently treated as success.

---

### Impact Explanation

The transfer ID is permanently recorded in `finalised_transfers`. Any attempt to re-submit the proof for the same transfer will be rejected with `NonceAlreadyUsed`-equivalent logic. The recipient receives zero tokens. For deployed (bridge-minted) tokens, no tokens are ever created. For locked tokens, the funds remain trapped in the bridge contract. The loss is irreversible.

This matches the allowed impact: **Critical — Irreversible fund lock / permanently unclaimable user value in bridge flows.**

---

### Likelihood Explanation

Any cross-chain transfer that includes a `msg` field (e.g., a DeFi swap instruction) is at risk. The recipient contract's `ft_on_transfer` can panic for legitimate reasons: slippage exceeded, pool drained, contract paused, or insufficient gas forwarded. This is a normal operational scenario, not an edge case. No privileged access is required; the `msg` field is a standard public protocol input set by the original sender.

---

### Recommendation

In `is_refund_required`, treat `Err(_)` as a delivery failure and return `true` (trigger the refund/revert path):

```rust
Err(_) => true,  // ft_transfer_call panicked → delivery failed, revert finalization
```

Additionally, for the `is_ft_transfer_call = false` case, the callback should inspect the promise result and revert finalization on failure rather than unconditionally treating it as success.

The `remove_fin_transfer` path already exists and correctly burns minted tokens, reverts lock actions, and emits `FailedFinTransferEvent` — it just needs to be reached on promise failure.

---

### Proof of Concept

1. Alice initiates a transfer on EVM of 1000 USDC to NEAR, with `msg = '{"action":"swap","min_out":"999"}'` targeting a NEAR DEX contract.
2. A relayer calls `fin_transfer` on NEAR. `fin_transfer_callback` → `process_fin_transfer_to_near` runs.
3. `add_fin_transfer` records the transfer ID in `finalised_transfers`.
4. `send_tokens` calls `mint(dex_contract, 1000, Some(msg))` on the bridge token.
5. The DEX's `ft_on_transfer` panics (slippage exceeded). The `mint`/`ft_transfer_call` promise result is `Err`.
6. `fin_transfer_send_tokens_callback` is called. `is_refund_required(true)` hits `Err(_) => false`.
7. The `else` branch executes: relayer fee is paid, `FinTransferEvent` is emitted.
8. Alice's 1000 USDC equivalent is permanently lost. The transfer ID is in `finalised_transfers` forever. No retry is possible. [3](#0-2) [5](#0-4) [6](#0-5)

    pub fn fin_transfer_send_tokens_callback(
        &mut self,
        #[serializer(borsh)] transfer_message: TransferMessage,
        #[serializer(borsh)] fee_recipient: &AccountId,
        #[serializer(borsh)] is_ft_transfer_call: bool,
        #[serializer(borsh)] storage_owner: &AccountId,
        #[serializer(borsh)] lock_actions: Vec<LockAction>,
    ) {
        let token = self.get_token_id(&transfer_message.token);

        if Self::is_refund_required(is_ft_transfer_call) {
            self.burn_tokens_if_needed(
                token.clone(),
                U128(
                    transfer_message
                        .amount_without_fee()
                        .near_expect(BridgeError::InvalidFee),
                ),
            );

            self.revert_lock_actions(&lock_actions);

            self.remove_fin_transfer(&transfer_message.get_transfer_id(), storage_owner);

            env::log_str(
                &OmniBridgeEvent::FailedFinTransferEvent { transfer_message }.to_log_string(),
            );
        } else {
            // Send fee to the fee recipient
            if transfer_message.fee.fee.0 > 0 {
                if self.is_deployed_token(&token) {
                    ext_token::ext(token)
                        .with_static_gas(MINT_TOKEN_GAS)
                        .mint(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                } else {
                    ext_token::ext(token)
                        .with_attached_deposit(ONE_YOCTO)
                        .with_static_gas(FT_TRANSFER_GAS)
                        .ft_transfer(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                }
            }

            if transfer_message.fee.native_fee.0 > 0 {
                let native_token_id = self.get_native_token_id(transfer_message.get_origin_chain());

                ext_token::ext(native_token_id)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }

            env::log_str(&OmniBridgeEvent::FinTransferEvent { transfer_message }.to_log_string());
        }
    }

    fn is_refund_required(is_ft_transfer_call: bool) -> bool {
        if is_ft_transfer_call {
            match env::promise_result_checked(0, MAX_FT_TRANSFER_CALL_RESULT) {
                Ok(value) => {
                    if let Ok(amount) = near_sdk::serde_json::from_slice::<U128>(&value) {
                        // Normal case: refund if the used token amount is zero
                        // The amount can be zero if the `ft_on_transfer` in the receiver contract returns an amount instead of `0`, or if it panics.
                        amount.0 == 0
                    } else {
                        // Unexpected case: don't refund
                        false
                    }
                }
                // Unexpected case: don't refund
                Err(_) => false,
            }
        } else {
            // Not ft_transfer_call: don't refund
            false
        }
    }

    fn process_fin_transfer_to_near(
        &mut self,
        recipient: AccountId,
        predecessor_account_id: &AccountId,
        transfer_message: TransferMessage,
        storage_deposit_actions: &Vec<StorageDepositAction>,
    ) -> Promise {
        let mut required_balance = self.add_fin_transfer(&transfer_message.get_transfer_id());

        let token = self.get_token_id(&transfer_message.token);
        let fast_transfer = FastTransfer::from_transfer(transfer_message.clone(), token.clone());
        let fast_transfer_status = self.get_fast_transfer_status(&fast_transfer.id());

        let lock_actions = vec![self.unlock_tokens_if_needed(
            transfer_message.get_origin_chain(),
            &token,
            transfer_message.amount.0,
        )];

        // If fast transfer happened, change recipient and fee recipient to the relayer that executed fast transfer
        let (recipient, msg, fee_recipient) = match fast_transfer_status {
            Some(status) => {
                require!(
                    !status.finalised,
                    BridgeError::FastTransferAlreadyFinalised.as_ref()
                );
                self.remove_fast_transfer(&fast_transfer.id());
                (status.relayer.clone(), String::new(), status.relayer)
            }
            None => (
                recipient,
                transfer_message.msg.clone(),
                predecessor_account_id.clone(),
            ),
        };

        let mut storage_deposit_action_index: usize = 0;
        require!(
            Self::check_storage_balance_result(
                (storage_deposit_action_index + 1)
                    .try_into()
                    .near_expect(BridgeError::Cast)
            ) && storage_deposit_actions[storage_deposit_action_index].account_id == recipient
                && storage_deposit_actions[storage_deposit_action_index].token_id == token,
            BridgeError::StorageRecipientOmitted.as_ref()
        );
        storage_deposit_action_index += 1;

        // One yoctoNear is required to send tokens to the recipient
        required_balance = required_balance.saturating_add(ONE_YOCTO);

        if transfer_message.fee.fee.0 > 0 {
            require!(
                Self::check_storage_balance_result(
                    (storage_deposit_action_index + 1)
                        .try_into()
                        .near_expect(BridgeError::Cast)
                ) && storage_deposit_actions[storage_deposit_action_index].account_id
                    == fee_recipient
                    && storage_deposit_actions[storage_deposit_action_index].token_id == token,
                BridgeError::StorageFeeRecipientOmitted.as_ref()
            );
            storage_deposit_action_index += 1;

            required_balance = required_balance.saturating_add(ONE_YOCTO);
        }

        if transfer_message.fee.native_fee.0 > 0 {
            let native_token_id = self.get_native_token_id(transfer_message.get_origin_chain());

            require!(
                Self::check_storage_balance_result(
                    (storage_deposit_action_index + 1)
                        .try_into()
                        .near_expect(BridgeError::Cast)
                ) && storage_deposit_actions[storage_deposit_action_index].account_id
                    == fee_recipient
                    && storage_deposit_actions[storage_deposit_action_index].token_id
                        == native_token_id,
                BridgeError::StorageNativeFeeRecipientOmitted.as_ref()
            );
        }

        self.update_storage_balance(
            predecessor_account_id.clone(),
            required_balance,
            env::attached_deposit(),
        );

        self.send_tokens(
            token.clone(),
            recipient,
            U128(
                transfer_message
                    .amount_without_fee()
                    .near_expect(BridgeError::InvalidFee),
            ),
            &msg,
        )
        .then(
            Self::ext(env::current_account_id())
                .with_static_gas(SEND_TOKENS_CALLBACK_GAS)
                .fin_transfer_send_tokens_callback(
                    transfer_message,
                    &fee_recipient,
                    !msg.is_empty(),
                    predecessor_account_id,
                    lock_actions,
                ),
        )
    }
