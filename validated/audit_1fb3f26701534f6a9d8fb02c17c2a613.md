### Title
Single-Participant `verify_tee()` Instantaneous Attestation Snapshot Persistently Enables Expired-TEE Nodes to Submit Signatures - (File: `crates/contract/src/lib.rs`)

---

### Summary

`verify_tee()` is callable by any single participant and sets the global `accept_requests` flag based on an instantaneous snapshot of attestation validity. Because `assert_caller_is_attested_participant_and_protocol_active()` (called inside `respond()`) only checks that a stored attestation *exists* for the caller — not that it is still cryptographically valid — a participant can time a `verify_tee()` call to lock `accept_requests = true` at a moment when attestations are temporarily valid, then let their own attestation expire. The flag persists until the next `verify_tee()` call, allowing a node running a no-longer-approved image to continue submitting threshold signatures.

---

### Finding Description

**Root cause — `verify_tee()` reads instantaneous state and writes persistent flag:** [1](#0-0) 

`verify_tee()` requires only `voter_or_panic()` — a single participant — not a threshold quorum. It calls `reverify_and_cleanup_participants()` which evaluates attestation validity at the current block timestamp: [2](#0-1) 

When the result is `TeeValidationResult::Full`, it unconditionally sets `self.accept_requests = true`: [3](#0-2) 

This flag persists in contract storage until the next `verify_tee()` call.

**Root cause — `respond()` checks attestation existence, not validity:** [4](#0-3) 

The `assert_caller_is_attested_participant_and_protocol_active()` call delegates to `is_caller_an_attested_participant()`, which only verifies that a stored attestation exists and that the account ID and public key match: [5](#0-4) 

It does **not** re-verify the attestation's expiry timestamp, image hash, or OS measurement against the current allowed lists. `reverify_and_cleanup_participants()` filters participants but does not remove stale entries from `stored_attestations`: [6](#0-5) 

So a stored-but-expired attestation still passes the per-call check in `respond()`.

**Attack sequence (analog to RAACMinter instantaneous-snapshot manipulation):**

| Step | RAACMinter analog | NEAR MPC |
|---|---|---|
| 1 | Temporarily inflate `totalBorrowed` | Submit a fresh valid attestation via `submit_participant_info()` |
| 2 | Call `updateEmissionRate()` | Call `verify_tee()` — snapshot shows all attestations valid |
| 3 | Persistent state set | `accept_requests = true` written to contract storage |
| 4 | Repay loan | Let attestation expire (or image hash rotate out) |
| 5 | Inflated rate persists | `accept_requests = true` persists; expired-attestation node passes `respond()` checks |

---

### Impact Explanation

A node whose TEE attestation has since expired (e.g., because the image hash it was attested against was rotated out of the allowed list) can continue to call `respond()` and deliver threshold signatures. This breaks the TEE security invariant: the system is designed to guarantee that only nodes running an approved image inside a genuine TDX enclave can contribute to signing. A node that has drifted off the approved image list — potentially running modified or unauthorized code — can still participate in signing rounds as long as `accept_requests = true` and its stale attestation remains in `stored_attestations`.

This matches the **Medium** allowed impact: *participant-state manipulation that breaks production safety/accounting invariants* — specifically the TEE attestation liveness invariant — without requiring network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

- `verify_tee()` is callable by any single current participant with no rate limit beyond NEAR transaction costs.
- `submit_participant_info()` is open to any account with a deposit; a participant can refresh their own attestation at will.
- The window between `verify_tee()` calls is documented as ~2 days, giving a long persistence window.
- The attack requires only one participant to time two sequential transactions, which is trivially achievable on NEAR.

---

### Recommendation

1. **Re-verify attestation validity inside `respond()`**: `assert_caller_is_attested_participant_and_protocol_active()` should call `reverify_participants()` for the caller's node ID, not just check stored-attestation existence.

2. **Threshold-gate `verify_tee()`**: Require at least `threshold` participants to call `verify_tee()` (or aggregate their calls) before `accept_requests` is set to `true`, analogous to how `vote_new_parameters` requires threshold agreement before a state transition.

3. **Time-weighted / multi-block attestation check**: Rather than a single instantaneous snapshot, require that attestations have been continuously valid for a minimum observation window before `accept_requests` is set to `true`.

4. **Remove stale attestations on reverification failure**: `reverify_and_cleanup_participants()` should evict entries from `stored_attestations` when they fail re-verification, so that the existence check in `is_caller_an_attested_participant()` cannot be satisfied by a stale entry.

### Proof of Concept

```
Setup:
  - 3 participants: A, B, C; threshold = 2
  - Image hash H1 is currently allowed; H2 is voted in; H1 enters grace period
  - All three nodes submitted attestations under H1

Step 1 (during grace period):
  - Node A calls submit_participant_info() with a fresh attestation under H1
    (valid because H1 is still in grace period)

Step 2 (during grace period):
  - Node A calls verify_tee()
  - reverify_and_cleanup_participants() sees all 3 attestations valid (H1 still in grace)
  - accept_requests = true is written to contract storage

Step 3 (grace period expires):
  - H1 is no longer in the allowed list
  - Nodes B and C have not yet submitted attestations under H2
  - Node A has not submitted under H2 either
  - All three attestations are now cryptographically invalid

Step 4 (post-expiry):
  - Node A calls respond(request, signature)
  - assert_caller_is_attested_participant_and_protocol_active():
      stored_attestations.get(A.tls_public_key) → returns stale H1 attestation ✓
      account_id matches ✓, account_public_key matches ✓ → Ok(())
  - accept_requests == true ✓ (set in Step 2, never reset)
  - respond() succeeds — Node A (running H1, no longer approved) delivers a signature

Result:
  A node running a no-longer-approved image contributes to threshold signatures,
  bypassing the TEE security invariant. The persistent accept_requests = true flag
  was set via a single-participant instantaneous attestation snapshot, exactly
  mirroring the RAACMinter emission-rate manipulation pattern.
```

    pub fn verify_tee(&mut self) -> Result<bool, Error> {
        log!("verify_tee: signer={}", env::signer_account_id());
        // Caller must be a participant (node or operator).
        self.voter_or_panic();
        let ProtocolContractState::Running(running_state) = &mut self.protocol_state else {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        };
        let current_params = running_state.parameters.clone();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        match self.tee_state.reverify_and_cleanup_participants(
            current_params.participants(),
            tee_upgrade_deadline_duration,
        ) {
            TeeValidationResult::Full => {
                self.accept_requests = true;
                log!("All participants have an accepted Tee status");
                Ok(true)
            }

    pub fn reverify_and_cleanup_participants(
        &mut self,
        participants: &Participants,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeValidationResult {
        self.allowed_docker_image_hashes
            .cleanup_expired_hashes(tee_upgrade_deadline_duration);

        let participants_with_valid_attestation: Vec<_> = participants
            .participants()
            .iter()
            .filter(|(_, _, participant_info)| {
                // Use the stored NodeId (keyed by TLS public key) so the real
                // `account_public_key` participates in re-verification. If
                // there is no stored attestation for this TLS key, the
                // participant is invalid.
                let Some(node_id) = self.find_node_id_by_tls_key(&participant_info.tls_public_key)
                else {
                    return false;
                };

                let tee_status =
                    self.reverify_participants(&node_id, tee_upgrade_deadline_duration);

                matches!(tee_status, TeeQuoteStatus::Valid)
            })
            .cloned()
            .collect();

        if participants_with_valid_attestation.len() != participants.len() {
            let participants_with_valid_attestation =
                Participants::init(participants.next_id(), participants_with_valid_attestation);

            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            }
        } else {
            TeeValidationResult::Full
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
