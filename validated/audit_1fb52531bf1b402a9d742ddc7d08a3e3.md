Audit Report

## Title
Unprotected `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Pause - (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no role-based access control, allowing any unprivileged external caller to invoke `_updateRsETHPrice()`. When the computed rsETH price has dropped more than `pricePercentageLimit` below `highestRsethPrice`, this function pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself — freezing all user deposits and withdrawals. The external `pause()` function requires `PAUSER_ROLE`, but the internal `_pause()` path reachable through `updateRSETHPrice()` has no such guard, creating a bypass.

## Finding Description
In `contracts/LRTOracle.sol`, `updateRSETHPrice()` (L87–89) is callable by any address. It delegates to `_updateRsETHPrice()` (L214), which at L270–281 checks whether the new price has dropped more than `pricePercentageLimit` below `highestRsethPrice`. If so, it calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` unconditionally — with no check on `msg.sender`.

The asymmetry with the price-increase path is critical: at L260–265, if the price has risen above the threshold, non-manager callers receive `PriceAboveDailyThreshold`. No equivalent guard exists for the price-decrease/pause path. The privileged `updateRSETHPriceAsManager()` (L94–96) exists but does not close the unprotected path — both call the same `_updateRsETHPrice()` internal function.

The external `pause()` function at L138 requires `onlyRole(LRTConstants.PAUSER_ROLE)`, but `_pause()` at L319–323 is an internal function that sets `paused = true` directly. Any caller reaching `_updateRsETHPrice()` through the public `updateRSETHPrice()` entry point can trigger `_pause()` without holding `PAUSER_ROLE`, bypassing the intended access control on the external pause path.

Exploit path:
1. Attacker monitors on-chain state until `(highestRsethPrice - newRsETHPrice) > pricePercentageLimit * highestRsethPrice` (e.g., after an EigenLayer slashing event or LST depeg reduces total ETH in protocol).
2. Attacker calls `lrtOracle.updateRSETHPrice()` — no role required.
3. `_updateRsETHPrice()` computes `newRsETHPrice` below threshold, reaches L277–281, and calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()`.
4. All deposits (`depositETH`, `depositAsset`) and withdrawals are frozen until an admin with `onlyLRTAdmin` calls `unpause()` on each contract.

## Impact Explanation
**Medium — Temporary freezing of funds.** All user deposits into `LRTDepositPool` and all withdrawals via `LRTWithdrawalManager` are blocked for the duration of the pause. Users who need to exit or enter positions during the freeze period are unable to do so. The attacker controls the exact timing of the freeze, enabling them to trigger it at maximally disruptive moments (e.g., during high withdrawal demand or market stress). Funds are not permanently lost, but access is temporarily denied — matching the "Temporary freezing of funds" impact class.

## Likelihood Explanation
**Medium.** The precondition — rsETH price dropping more than `pricePercentageLimit` below `highestRsethPrice` — is realistic in production given EigenLayer slashing risk or LST depegs reducing `totalETHInProtocol`. No special privileges, flash loans, or oracle manipulation are required. Once the precondition is met, any address can trigger the pause with a single external call. The condition can persist for extended periods during market stress, giving an attacker ample opportunity to choose the most disruptive timing.

## Recommendation
1. Add a role check to the price-decrease pause path in `_updateRsETHPrice()`, mirroring the existing check at L260–265 for price increases: require `MANAGER` role (or `PAUSER_ROLE`) before executing the pause branch.
2. Alternatively, remove the public `updateRSETHPrice()` entry point entirely and require all price updates to go through `updateRSETHPriceAsManager()`.
3. If a permissionless keeper path is intentionally desired, split the logic: allow unprivileged callers to update the price only when it is within safe bounds, and require a privileged role to trigger the pause path.
