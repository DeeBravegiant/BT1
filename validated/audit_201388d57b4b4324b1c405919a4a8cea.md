Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` via Nested Loops Over `nodeDelegatorQueue` and `supportedAssetList` - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a permissionless public function that computes total protocol TVL by iterating over every supported asset and, for each asset, making 3 external calls per NodeDelegator contract. With no hard cap on `supportedAssetList` length and no hard ceiling on `maxNodeDelegatorLimit`, the gas cost grows as O((A+1) × N × 3) external calls. As the protocol adds more assets and NDCs through normal operations, this function will eventually exceed the Ethereum block gas limit, permanently freezing the rsETH price oracle and disabling the price-decrease circuit breaker.

## Finding Description
`updateRSETHPrice()` at `LRTOracle.sol:87` is public with only a `whenNotPaused` guard — no role restriction. It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()` at `LRTOracle.sol:331`, which iterates over every entry in `supportedAssetList` (fetched from `LRTConfig`). For each asset, it calls `ILRTDepositPool.getTotalAssetDeposits(asset)` → `getAssetDistributionData(asset)` at `LRTDepositPool.sol:426`, which loops over every NDC in `nodeDelegatorQueue` making 3 external calls per NDC: `IERC20(asset).balanceOf(ndc[i])`, `INodeDelegator(ndc[i]).getAssetBalance(asset)`, and `INodeDelegator(ndc[i]).getAssetUnstaking(asset)`. Additionally, the ETH path via `getETHDistributionData()` at `LRTDepositPool.sol:482` independently loops over all NDCs with 3 more external calls each. Total external calls = (A + 1) × N × 3.

`_addNewSupportedAsset` at `LRTConfig.sol:106` has no length cap — it simply pushes to `supportedAssetList`. `updateMaxNodeDelegatorLimit` at `LRTDepositPool.sol:290` only enforces that the new limit is ≥ the current queue length, with no upper ceiling. Both arrays can grow without bound through normal admin operations (`TIME_LOCK_ROLE` for assets, `onlyLRTAdmin` for NDCs).

## Impact Explanation
This is **Medium — Unbounded Gas Consumption**, with a concrete secondary path to **Medium — Temporary Freezing of Funds**. When `updateRSETHPrice()` reverts out-of-gas, the rsETH price becomes permanently stale. More critically, the price-decrease circuit breaker at `LRTOracle.sol:270–281` — which pauses `LRTDepositPool` and `WithdrawalManager` on a large price drop — becomes unreachable, disabling a key safety mechanism. The function is the sole on-chain price update path; no fallback exists.

## Likelihood Explanation
The protocol is explicitly designed to support multiple LST assets and multiple NDCs. Adding assets and NDCs are routine operational actions, not attacks. Any public caller (including keeper bots) will hit the gas limit once arrays are large enough. At 10 NDCs and 10 assets, the function already executes 330 external calls (~1.65M gas for calls alone plus loop overhead and storage reads). Scaling to 30+ assets or 50+ NDCs — plausible for a growing multi-asset restaking protocol — pushes the function past the 30M block gas limit. No attacker action is required; normal protocol growth triggers the condition.

## Recommendation
1. Enforce a hard cap (e.g., ≤ 20) on `supportedAssetList` length inside `_addNewSupportedAsset` in `LRTConfig.sol`.
2. Enforce a hard ceiling on `maxNodeDelegatorLimit` in `updateMaxNodeDelegatorLimit` in `LRTDepositPool.sol` (e.g., ≤ 20) that cannot be raised beyond a safe bound.
3. Consider a push-based oracle pattern where TVL per asset is updated incrementally rather than recomputed in full on every call.
4. Alternatively, paginate `_getTotalEthInProtocol` so no single transaction must iterate all assets × all NDCs.

## Proof of Concept
Call sequence with A assets and N NDCs:
```
updateRSETHPrice()                                    // LRTOracle.sol:87 — public, no role check
  └─ _updateRsETHPrice()                              // LRTOracle.sol:214
       └─ _getTotalEthInProtocol()                    // LRTOracle.sol:331
            ├─ for each asset (A iterations):         // LRTOracle.sol:336
            │    └─ getTotalAssetDeposits(asset)      // LRTDepositPool.sol:385
            │         └─ getAssetDistributionData()   // LRTDepositPool.sol:426
            │              └─ for each NDC (N iters): // LRTDepositPool.sol:447
            │                   ├─ IERC20.balanceOf(ndc[i])
            │                   ├─ INodeDelegator(ndc[i]).getAssetBalance(asset)
            │                   └─ INodeDelegator(ndc[i]).getAssetUnstaking(asset)
            └─ ETH path: getETHDistributionData()     // LRTDepositPool.sol:482
                 └─ for each NDC (N iters):
                      ├─ ndc[i].balance
                      ├─ INodeDelegator(ndc[i]).getEffectivePodShares()
                      └─ INodeDelegator(ndc[i]).getAssetUnstaking(ETH_TOKEN)
```
Total external calls = (A + 1) × N × 3.

**Foundry fork test plan:**
1. Fork mainnet; deploy or reuse the live LRTDepositPool and LRTOracle.
2. Using the admin/timelock, call `addNewSupportedAsset` repeatedly to register 20+ mock LST assets (each with a mock price oracle returning `1e18`).
3. Call `updateMaxNodeDelegatorLimit(50)` and `addNodeDelegatorContractToQueue` to register 50 mock NDCs (each returning fixed values for `getAssetBalance`/`getAssetUnstaking`).
4. Call `updateRSETHPrice{gas: 30_000_000}()` and assert it reverts with out-of-gas.
5. Confirm that `rsETHPrice` remains at its pre-call stale value and that subsequent calls also revert, demonstrating permanent oracle freeze.
