### Title
Gateway `validate_resource_bounds` Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Admission Control — (`crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs`)

---

### Summary

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client()` populates the `BlockInfo` it returns with `block_header.l2_gas_price` — the gas price that was used for transactions **inside the previous block**. The gateway's `StatefulTransactionValidator::validate_resource_bounds()` then uses that value as the reference threshold for admission control. The correct reference is `block_header.next_l2_gas_price`, which is the gas price that will govern the **next** block (the one the incoming transaction will actually be executed in). The developers acknowledge this with an explicit TODO comment at the call site.

---

### Finding Description

`SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block()` creates a `GatewayFixedBlockSyncStateClient` pinned to the latest committed block number. [1](#0-0) 

When `get_block_info()` is called on that client it lazily fetches and caches the block header via `get_block_info_from_sync_client()`. That function builds a `BlockInfo` using `block_header.l2_gas_price` for the STRK L2 gas price field: [2](#0-1) 

`validate_resource_bounds()` then reads that cached value and uses it as the threshold against which the incoming transaction's `max_price_per_unit` is compared: [3](#0-2) 

The TODO comment at that exact line confirms the wrong field is being used:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [4](#0-3) 

`BlockHeaderWithoutHash` carries a separate `next_l2_gas_price: GasPrice` field — the EIP-1559-derived price for the **next** block — which is what the consensus orchestrator reads when it sets `context.l2_gas_price` for the block being built: [5](#0-4) 

`get_block_info_from_sync_client()` never reads `next_l2_gas_price`; it only reads `l2_gas_price`. The `BlockInfo` struct has no field for `next_l2_gas_price`, so the stale value is silently cached and reused for every admission check on that transaction. [6](#0-5)

### Impact Explanation

**Case 1 — Gas price rising (`next_l2_gas_price > l2_gas_price`):**
The gateway threshold is computed from the lower, stale `l2_gas_price`. A transaction whose `max_price_per_unit` falls in the range `[threshold_stale, threshold_correct)` passes gateway admission and enters the mempool, but will fail execution in the next block because the actual block gas price exceeds the transaction's declared maximum. The gateway **admits invalid transactions**.

**Case 2 — Gas price falling (`next_l2_gas_price < l2_gas_price`):**
The gateway threshold is computed from the higher, stale `l2_gas_price`. A transaction whose `max_price_per_unit` falls in the range `[threshold_correct, threshold_stale)` is rejected by the gateway even though it would succeed in the next block. The gateway **rejects valid transactions**.

Both cases match the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The L2 gas price is updated every block via an EIP-1559-style formula. Any block with non-zero L2 gas consumption produces a `next_l2_gas_price` that differs from `l2_gas_price`. This condition holds for virtually every block under normal network load, making the divergence between the two values a routine occurrence rather than an edge case.

---

### Recommendation

In `get_block_info_from_sync_client()`, replace `block_header.l2_gas_price` with `block_header.next_l2_gas_price` when populating the `l2_gas_price` field of the returned `BlockInfo`. Because `next_l2_gas_price` is a single `GasPrice` (FRI-denominated), the wei equivalent should be derived using the same ETH-to-FRI conversion rate already available in the block header, consistent with how the consensus orchestrator converts it.

```rust
// Before (stale):
l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,

// After (correct):
l2_gas_price: block_header.next_l2_gas_price.try_into()?,
``` [8](#0-7)

### Proof of Concept

1. Block N is committed with `l2_gas_price = 1000 FRI` and `next_l2_gas_price = 1100 FRI` (10 % increase due to high gas usage).
2. A user submits an invoke transaction with `max_price_per_unit = 1050 FRI` (above the stale threshold of `1000 * min_gas_price_percentage / 100`, e.g. `1000` at 100 %).
3. `validate_resource_bounds()` computes `threshold = 1000 * 100 / 100 = 1000`. Since `1050 >= 1000`, the check passes and the transaction is admitted to the mempool.
4. The batcher pulls the transaction and attempts to execute it in block N+1, which uses `next_l2_gas_price = 1100 FRI` as the block gas price.
5. The blockifier's pre-validation rejects the transaction with `MaxGasPriceTooLow` because `1050 < 1100`.
6. The transaction was admitted by the gateway but fails execution — an invalid transaction was accepted through the admission path.

Relevant code path: `validate_resource_bounds` → `get_block_info()` → `get_block_info_from_sync_client()` → stale `l2_gas_price` used instead of `next_l2_gas_price`. [3](#0-2) [2](#0-1)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
