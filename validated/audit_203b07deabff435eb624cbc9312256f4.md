Audit Report

## Title
Stale Chainlink Price Accepted Without Staleness Validation, Enabling Forced Protocol Pause - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all safety return values, accepting stale prices silently. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any external caller can trigger an rsETH price update using a stale feed value. If the stale price is sufficiently below `highestRsethPrice`, the protocol's own downside-protection circuit-breaker fires, pausing deposits, withdrawals, and the oracle until an admin manually intervenes.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` (line 52) fetches only the raw `price` from `latestRoundData()`, discarding `roundId`, `startedAt`, `updatedAt`, and `answeredInRound`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
```

No staleness check (`block.timestamp - updatedAt <= MAX_STALENESS`), no round-completeness check (`updatedAt != 0`), and no `answeredInRound >= roundId` check are performed. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same repository applies all three guards (lines 30–32).

The stale price flows through `LRTOracle.getAssetPrice()` → `_getTotalEthInProtocol()` (line 339) → `_updateRsETHPrice()` (line 231), which computes `newRsETHPrice`. At lines 270–281, if `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit * highestRsethPrice`, the function calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on the oracle, then returns without updating state.

`updateRSETHPrice()` (line 87) is `public` with only a `whenNotPaused` modifier — no role restriction — so any EOA can trigger this path.

## Impact Explanation
**Medium — Temporary freezing of funds.** All user-facing deposit (`depositAsset`, `depositETH`) and withdrawal (`initiateWithdrawal`, `instantWithdrawal`, `completeWithdrawal`) functions revert with `Paused` until an admin calls `unpause()` on each contract. Funds are not lost, but access is frozen for an indeterminate period. This matches the allowed impact "Temporary freezing of funds."

## Likelihood Explanation
Chainlink feeds have documented heartbeat intervals (e.g., 24 hours for some LST/ETH feeds). During network congestion, L2 sequencer downtime, or feed deprecation, `updatedAt` can lag by the full heartbeat window while `latestRoundData()` continues to return the last price without reverting. An attacker monitoring mempool or feed timestamps can call `updateRSETHPrice()` at the optimal moment — requiring zero capital, zero permissions, and a single transaction. The attack is repeatable each time the feed goes stale and the price delta exceeds the configured threshold.

## Recommendation
Add staleness and round-completeness checks in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();
require(answeredInRound >= roundId, "Stale price");
require(updatedAt != 0, "Incomplete round");
require(price > 0, "Invalid price");
require(block.timestamp - updatedAt <= MAX_STALENESS, "Price too old");
```

`MAX_STALENESS` should be set per feed based on its documented heartbeat interval.

## Proof of Concept
1. Deploy a mock Chainlink aggregator that returns a fixed price with `updatedAt` set to `block.timestamp - 25 hours` (past a 24-hour heartbeat).
2. Configure `ChainlinkPriceOracle` to use this mock feed for a supported LST asset.
3. Set `pricePercentageLimit` to `1e16` (1%) and `highestRsethPrice` to a value such that the stale price produces a `newRsETHPrice` more than 1% below it.
4. Call `LRTOracle.updateRSETHPrice()` from any EOA.
5. Observe: `lrtDepositPool.paused() == true`, `withdrawalManager.paused() == true`, `LRTOracle.paused == true`.
6. Confirm all deposit and withdrawal calls revert with `Paused` until admin calls `unpause()` on each contract.
