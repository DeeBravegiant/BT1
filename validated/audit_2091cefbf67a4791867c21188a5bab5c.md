### Title
Mempool Capacity Check Includes Non-Evictable `delayed_declares` Bytes, Allowing Declare-Flood Griefing That Permanently Blocks Legitimate Transaction Admission - (`crates/apollo_mempool/src/mempool.rs`)

---

### Summary

The mempool's `size_in_bytes()` function aggregates both `tx_pool` bytes and `delayed_declares` bytes for the capacity gate, but the eviction relief path (`try_make_space`) only removes transactions from `tx_pool`. When an attacker fills `delayed_declares` to the configured `capacity_in_bytes` limit, the eviction mechanism cannot reclaim any space, and every subsequent legitimate transaction is rejected with `MempoolError::MempoolFull` — even though the sequenceable pool (`tx_pool`) is empty.

---

### Finding Description

**Invariant broken:** "When the mempool is at capacity, the eviction mechanism can always free space for a non-gap-creating transaction."

**Root cause — wrong counter in the capacity gate:**

`size_in_bytes()` is the total of two disjoint stores:

```rust
// crates/apollo_mempool/src/mempool.rs  line 918-920
fn size_in_bytes(&self) -> u64 {
    self.tx_pool.size_in_bytes() + self.delayed_declares.size_in_bytes()
}
```

This total is used verbatim in the admission check:

```rust
// crates/apollo_mempool/src/mempool.rs  line 940-945
fn exceeds_capacity(&self, tx: &InternalRpcTransaction, freed_bytes: u64) -> bool {
    (self.size_in_bytes() + tx.total_bytes()).saturating_sub(freed_bytes)
        > self.config.static_config.capacity_in_bytes
}
```

When `exceeds_capacity` fires, the only relief path is `try_make_space`, which exclusively evicts from `tx_pool`:


`delayed_declares` is never touched by `try_make_space`. There is no eviction path for it.

**Attack path:**

1. Attacker submits `N` Declare transactions from `N` distinct accounts, each with `tx_nonce == account_nonce` (no gap). In Starknet (non-FIFO) mode, every Declare is routed to `delayed_declares`:

```rust
// crates/apollo_mempool/src/mempool.rs  line 502-509
let should_delay_declare =
    matches!(&args.tx.tx, InternalRpcTransactionWithoutTxHash::Declare(_))
        && !self.is_fifo();
if should_delay_declare {
    self.delayed_declares.push_back(self.clock.now(), args);
```

2. `delayed_declares.size_in_bytes()` grows until `size_in_bytes()` ≥ `capacity_in_bytes`.

3. A legitimate Invoke transaction arrives from a fresh account at `nonce == account_nonce`:
   - `exceeds_capacity` → `true`
   - `handle_capacity_overflow` is called; `creating_gap = false` (nonce closes the gap)
   - `try_make_space` is called; `tx_pool` has **zero gap accounts** (all attacker declares are in `delayed_declares` or queued without gaps)
   - `try_make_space` returns `false`
   - `MempoolError::MempoolFull` is returned

```rust
// crates/apollo_mempool/src/mempool.rs  line 1031-1052
fn handle_capacity_overflow(...) -> Result<(), MempoolError> {
    ...
    let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;
    let required_space = tx.total_bytes().saturating_sub(freed_bytes);
    if !creating_gap && self.try_make_space(required_space) {
        return Ok(());
    }
    Err(MempoolError::MempoolFull)
}
```

4. After `declare_delay` (default 1 s), `add_ready_declares()` moves the attacker's declares into `tx_pool`. `size_in_bytes()` is unchanged (bytes merely shift stores). The declares are now queued (no gap), so they are still **not evictable** by `try_make_space`. Legitimate transactions remain blocked until the attacker's declares are committed in a block.

**Existing test confirms the accounting:**

```rust
// crates/apollo_mempool/src/fee_mempool_test.rs  line 642-685

    // "Prepare also declare transactions, to make sure delayed declares are counted."
    ...

}
```

The test explicitly documents that `delayed_declares` bytes count toward the cap, but no test verifies that `try_make_space` can reclaim them — because it cannot.

---

### Impact Explanation

Every valid non-Declare transaction submitted while `delayed_declares` holds the capacity is rejected at the mempool admission boundary with `MempoolError::MempoolFull`. The sequenceable `tx_pool` may be completely empty, yet the node refuses all new transactions. This directly matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The block builder receives zero transactions from the mempool during the attack window, producing empty blocks.

---

### Likelihood Explanation

- Any unprivileged user can submit Declare transactions; no special privilege is required.
- The default `capacity_in_bytes` is 1 GB, requiring ~2 M transactions at ~500 B each — impractical at default settings.
- In deployments with a reduced capacity (e.g., 100 MB), ~200 K transactions suffice.
- The attack window lasts from submission until the attacker's declares are committed (declare_delay + one block time, typically a few seconds).
- The attacker must control many accounts with valid nonces and fee balances.

Likelihood is **Low** at default capacity, **Medium** at reduced capacity.

---

### Recommendation

The capacity gate and the eviction mechanism must operate on the same set. Two options:

**Option A (preferred):** Exclude `delayed_declares` from the capacity gate; charge them against a separate, smaller quota:

```rust
fn exceeds_capacity(&self, tx: &InternalRpcTransaction, freed_bytes: u64) -> bool {
    (self.tx_pool.size_in_bytes() + tx.total_bytes()).saturating_sub(freed_bytes)
        > self.config.static_config.capacity_in_bytes
}
```

**Option B:** Extend `try_make_space` to also evict from `delayed_declares` (drop the oldest delayed declare when no gap account exists in `tx_pool`).

---

```rust
                break;
```

```rust
    fn handle_capacity_overflow(
        &mut self,
        tx: &InternalRpcTransaction,
        account_nonce: Nonce,
        freed_bytes: u64,
    ) -> Result<(), MempoolError> {
        let address = tx.contract_address();

        let account_has_gap = self.accounts_with_gap.contains(&address);
        let account_has_txs = self.tx_pool.contains_account(address);
        let closing_gap = tx.nonce() == account_nonce;
        let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;

        // Only the net growth must be evicted: an accompanying replacement removal frees
        // `freed_bytes` (0 when there is no replacement).
        let required_space = tx.total_bytes().saturating_sub(freed_bytes);
        if !creating_gap && self.try_make_space(required_space) {
            return Ok(());
        }

        Err(MempoolError::MempoolFull)
    }
```
