Audit Report

## Title
`RSETH.initialize()` Does Not Set `maxMintAmountPerDay`, Blocking All rsETH Minting Until Manually Corrected - (File: contracts/RSETH.sol)

## Summary
`RSETH.initialize()` and `RSETH.reinitialize()` both leave `maxMintAmountPerDay` at its Solidity default of zero. The `checkDailyMintLimit` modifier unconditionally reverts for any non-zero `amount` when `maxMintAmountPerDay == 0`, meaning every call to `RSETH.mint()` — and therefore every user deposit through `LRTDepositPool` — reverts until a manager separately calls `setMaxMintAmountPerDay()`.

## Finding Description
`maxMintAmountPerDay` is declared as a plain state variable with no initializer: [1](#0-0) 

The `checkDailyMintLimit` modifier checks `currentPeriodMintedAmount + amount > maxMintAmountPerDay`. When `maxMintAmountPerDay` is zero, this evaluates to `0 + amount > 0`, which is `true` for any non-zero `amount`, causing an unconditional revert: [2](#0-1) 

Neither `initialize()` nor `reinitialize()` assigns `maxMintAmountPerDay`: [3](#0-2) [4](#0-3) 

The only setter is a separate manager-gated call: [5](#0-4) 

`mint()` applies the modifier unconditionally: [6](#0-5) 

Notably, `remainingDailyMintLimit()` explicitly short-circuits on `maxMintAmountPerDay == 0`, confirming the developers are aware this state can exist — yet the modifier has no equivalent guard: [7](#0-6) 

Both `depositETH` and `depositAsset` in `LRTDepositPool` call `_mintRsETH` → `RSETH.mint()`, so all user deposits revert: [8](#0-7) [9](#0-8) 

## Impact Explanation
**Medium — Temporary freezing of funds.** The entire deposit mechanism is non-functional from deployment until `setMaxMintAmountPerDay` is explicitly called by a manager. Every `depositETH` and `depositAsset` call reverts. Individual transactions revert cleanly (no ETH or LSTs are permanently lost), but the protocol cannot deliver its core service — issuing rsETH in exchange for deposited assets — for the entire window between `initialize()` and the manual `setMaxMintAmountPerDay()` call.

## Likelihood Explanation
**Medium.** The `reinitialize()` function was added specifically to set `periodStartTime` and `custodyAddress` as a second initialization step, establishing a multi-step initialization pattern. A deployer following this pattern may reasonably assume `maxMintAmountPerDay` is handled elsewhere and omit the `setMaxMintAmountPerDay` call. The bug is silent — no event or revert signals the uninitialized state at deployment time — and only manifests when the first deposit is attempted.

## Recommendation
Add a `_maxMintAmountPerDay` parameter to `initialize()` (or `reinitialize()`) and assign it there with a non-zero check:

```solidity
function initialize(address admin, address lrtConfigAddr, uint256 _maxMintAmountPerDay) external initializer {
    UtilLib.checkNonZeroAddress(admin);
    UtilLib.checkNonZeroAddress(lrtConfigAddr);
    if (_maxMintAmountPerDay == 0) revert InvalidMaxMintAmount();
    __ERC20_init("rsETH", "rsETH");
    __Pausable_init();
    lrtConfig = ILRTConfig(lrtConfigAddr);
    maxMintAmountPerDay = _maxMintAmountPerDay;
    emit UpdatedLRTConfig(lrtConfigAddr);
    emit MaxMintAmountPerDayUpdated(_maxMintAmountPerDay);
}
```

If the contract is already deployed, `setMaxMintAmountPerDay()` must be called immediately after deployment before any deposits are expected.

## Proof of Concept
1. Deploy `RSETH` proxy and call `initialize(admin, lrtConfigAddr)`. `maxMintAmountPerDay` is now `0`.
2. Call `reinitialize(periodStartTime, custodyAddress)`. `maxMintAmountPerDay` is still `0`.
3. Any address calls `LRTDepositPool.depositETH{value: 1 ether}(0, "")`.
4. `LRTDepositPool._mintRsETH(rsethAmountToMint)` calls `RSETH.mint(msg.sender, rsethAmountToMint)`.
5. `checkDailyMintLimit(rsethAmountToMint)` evaluates `0 + rsethAmountToMint > 0` → `true` → reverts with `DailyMintLimitExceeded(rsethAmountToMint, 0)`.
6. All deposits are blocked until a manager calls `setMaxMintAmountPerDay(nonZeroValue)`.

**Foundry test sketch:**
```solidity
function test_depositBlockedWhenMaxMintNotSet() public {
    // Deploy and initialize without calling setMaxMintAmountPerDay
    rsETH = new RSETH();
    rsETH.initialize(admin, address(lrtConfig));
    rsETH.reinitialize(block.timestamp, custodyAddress);
    // maxMintAmountPerDay == 0 at this point

    vm.deal(user, 1 ether);
    vm.prank(user);
    vm.expectRevert(
        abi.encodeWithSelector(RSETH.DailyMintLimitExceeded.selector, 1 ether, 0)
    );
    lrtDepositPool.depositETH{value: 1 ether}(0, "");
}
```
