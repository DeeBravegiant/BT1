### Title
Blue-box `normalized_to_identity` bypass skips intermediate block VDF validation in weight proof segments — (`chia/full_node/weight_proof.py`)

### Summary
In `_validate_sub_slot_data`, when a non-end-of-slot (block) entry is being validated and the enclosing slot-end entry has `cc_slot_end.normalized_to_identity == True`, the function returns `(True, [])` immediately, adding zero VDF tuples to `to_validate`. This means the `cc_infusion_point` and `cc_sp_vdf_info` proofs for every intermediate block in that slot are never verified. An attacker who can supply a `SubEpochChallengeSegment` with a legitimately blue-boxed slot-end (or who can set `normalized_to_identity=True` on a crafted slot-end proof that passes `validate_vdf`) can include arbitrarily forged challenge-chain infusion-point VDFs for all intermediate blocks in that slot, and the weight proof verifier will accept them.

### Finding Description

In `_validate_segment` (lines 1031–1051), for each sampled segment entry after the challenge block, `_validate_sub_slot_data` is called and its returned `vdf_list` is appended to `to_validate`: [1](#0-0) 

Inside `_validate_sub_slot_data`, the `else` branch (non-end-of-slot entries) scans forward for the enclosing slot-end and, if `normalized_to_identity is True`, returns immediately with an empty list: [2](#0-1) 

At the point of that `return`, `to_validate` is `[]` — no intermediate block VDFs (neither `cc_infusion_point` nor `cc_signage_point`) have been appended. The slot-end entry itself is validated separately when its own index is processed: [3](#0-2) 

A valid `normalized_to_identity` slot-end proof proves the VDF ran from the identity element to the slot-end output over `sub_slot_iters` iterations. It says nothing about the correctness of intermediate infusion-point VDFs, which are computed over sub-intervals from the previous block's output.

### Impact Explanation
A syncing node that receives a forged `WeightProof` containing a sampled `SubEpochChallengeSegment` with a blue-boxed slot-end will accept the segment as valid even though the challenge-chain infusion-point VDFs for intermediate blocks are fabricated. This allows an attacker to forge weight proof trust: the syncing node believes the attacker's chain has sufficient weight and switches to it, potentially accepting a chain that never had valid blocks at those heights. This is a **Critical** impact: forged weight proof trust via a sync protocol message from an unprivileged remote peer.

### Likelihood Explanation
The attack requires the attacker to either:
- Obtain a legitimate weight proof that already contains a blue-boxed slot-end (common on mainnet where timelords produce `normalized_to_identity` proofs), then replace the intermediate block VDF proofs with forged ones; or
- Compute a valid `normalized_to_identity` slot-end VDF proof themselves (requires a timelord, but no privileged access).

The first path requires no VDF computation at all — the attacker just mutates an existing weight proof. This makes the attack straightforward for any peer that can send a `RespondProofOfWeight` message.

### Recommendation
Remove the early-return optimization entirely, or — if the optimization is intentional for performance — verify that a valid `normalized_to_identity` slot-end proof is cryptographically sufficient to cover all intermediate block VDFs (it is not, by construction). The correct fix is to always append intermediate block VDF tuples to `to_validate` regardless of the slot-end's `normalized_to_identity` flag, and let the VDF batch verifier check them.

### Proof of Concept
1. Obtain any legitimate `WeightProof` from a synced node via `RequestProofOfWeight` / `RespondProofOfWeight`.
2. Find a sampled `SubEpochChallengeSegment` where a slot-end `SubSlotData` has `cc_slot_end.normalized_to_identity == True` (common on mainnet).
3. For every non-end-of-slot `SubSlotData` in that slot that comes after the challenge block, replace `cc_infusion_point` and `cc_ip_vdf_info` with arbitrary bytes / a zeroed-out `VDFProof` and `VDFInfo`.
4. Send the mutated `WeightProof` to a syncing node.
5. Observe that `_validate_sub_slot_data` returns `(True, [])` for each of those entries (the blue-box path at line 1128–1130 fires), so the forged VDFs are never passed to the VDF verifier.
6. The syncing node accepts the weight proof and may switch to the attacker's chain. [4](#0-3)

def _validate_sub_slot_data(
    constants: ConsensusConstants,
    sub_slot_idx: int,
    sub_slots: list[SubSlotData],
    ssi: uint64,
) -> tuple[bool, list[tuple[VDFProof, ClassgroupElement, VDFInfo]]]:
    sub_slot_data = sub_slots[sub_slot_idx]
    assert sub_slot_idx > 0
    prev_ssd = sub_slots[sub_slot_idx - 1]
    to_validate = []
    if sub_slot_data.is_end_of_slot():
        if sub_slot_data.icc_slot_end is not None:
            input = ClassgroupElement.get_default_element()
            if not sub_slot_data.icc_slot_end.normalized_to_identity and prev_ssd.icc_ip_vdf_info is not None:
                assert prev_ssd.icc_ip_vdf_info
                input = prev_ssd.icc_ip_vdf_info.output
            assert sub_slot_data.icc_slot_end_info
            to_validate.append((sub_slot_data.icc_slot_end, input, sub_slot_data.icc_slot_end_info))
        assert sub_slot_data.cc_slot_end_info
        assert sub_slot_data.cc_slot_end
        input = ClassgroupElement.get_default_element()
        if (not prev_ssd.is_end_of_slot()) and (not sub_slot_data.cc_slot_end.normalized_to_identity):
            assert prev_ssd.cc_ip_vdf_info
            input = prev_ssd.cc_ip_vdf_info.output
        if not validate_vdf(sub_slot_data.cc_slot_end, constants, input, sub_slot_data.cc_slot_end_info):
            log.error(f"failed cc slot end validation  {sub_slot_data.cc_slot_end_info}")
            return False, []
    else:
        # find end of slot
        idx = sub_slot_idx
        while idx < len(sub_slots) - 1:
            curr_slot = sub_slots[idx]
            if curr_slot.is_end_of_slot():
                # dont validate intermediate vdfs if slot is blue boxed
                assert curr_slot.cc_slot_end
                if curr_slot.cc_slot_end.normalized_to_identity is True:
                    log.debug(f"skip intermediate vdfs slot {sub_slot_idx}")
                    return True, to_validate
                else:
                    break
            idx += 1
        if sub_slot_data.icc_infusion_point is not None and sub_slot_data.icc_ip_vdf_info is not None:
            input = ClassgroupElement.get_default_element()
            if not prev_ssd.is_challenge() and prev_ssd.icc_ip_vdf_info is not None:
                input = prev_ssd.icc_ip_vdf_info.output
            to_validate.append((sub_slot_data.icc_infusion_point, input, sub_slot_data.icc_ip_vdf_info))
        assert sub_slot_data.signage_point_index is not None
        if sub_slot_data.cc_signage_point:
            assert sub_slot_data.cc_sp_vdf_info
            input = ClassgroupElement.get_default_element()
            if not sub_slot_data.cc_signage_point.normalized_to_identity:
                is_overflow = is_overflow_block(constants, sub_slot_data.signage_point_index)
                input = sub_slot_data_vdf_input(
                    constants, sub_slot_data, sub_slot_idx, sub_slots, is_overflow, prev_ssd.is_end_of_slot(), ssi
                )
            to_validate.append((sub_slot_data.cc_signage_point, input, sub_slot_data.cc_sp_vdf_info))

        input = ClassgroupElement.get_default_element()
        assert sub_slot_data.cc_ip_vdf_info
        assert sub_slot_data.cc_infusion_point
        cc_ip_vdf_info = sub_slot_data.cc_ip_vdf_info
        if not sub_slot_data.cc_infusion_point.normalized_to_identity and prev_ssd.cc_slot_end is None:
            assert prev_ssd.cc_ip_vdf_info
            input = prev_ssd.cc_ip_vdf_info.output
            assert sub_slot_data.total_iters
            assert prev_ssd.total_iters
            ip_vdf_iters = uint64(sub_slot_data.total_iters - prev_ssd.total_iters)
            cc_ip_vdf_info = VDFInfo(
                sub_slot_data.cc_ip_vdf_info.challenge, ip_vdf_iters, sub_slot_data.cc_ip_vdf_info.output
            )
        to_validate.append((sub_slot_data.cc_infusion_point, input, cc_ip_vdf_info))

    return True, to_validate
