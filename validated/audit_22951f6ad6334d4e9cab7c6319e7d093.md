Audit Report

## Title
Stale Chainlink Price Accepted Without Staleness Validation Enables Permissionless `rsETHPrice` Manipulation - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, accepting stale prices without any staleness or round-completeness check. Because `LRTOracle.updateRSETHPrice()` is a public, permissionless function, any unprivileged caller can trigger a `rsETHPrice` update during a Chainlink staleness window, locking a depressed exchange rate into storage. An attacker can then deposit a non-stale asset (e.g., ETH) at the artificially low `rsETHPrice` to receive more rsETH than the deposited assets are worth, diluting existing rsETH holders — a theft of unclaimed yield. A secondary path allows a forced auto-pause if the stale price drop exceeds `pricePercentageLimit`.

## Finding Description

**Root cause — missing staleness guard in `ChainlinkPriceOracle`:**

`contracts/oracles/ChainlinkPriceOracle.sol` line 52 captures only `price` from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
```

`updatedAt` (age of the price) and `answeredInRound` (round completeness) are silently discarded. The same codebase's `ChainlinkOracleForRSETHPoolCollateral` (used for pool collateral) explicitly performs all three checks:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

This inconsistency confirms the protocol is aware of the requirement but failed to apply it to the L1 core oracle.

**Permissionless trigger — `updateRSETHPrice()` is public:**

`contracts/LRTOracle.sol` line 87:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

No role check. Any EOA or contract can call this at any time. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which iterates over all supported LST assets and calls `getAssetPrice(asset)` — routed through the staleness-blind `ChainlinkPriceOracle` — to compute `totalETHInProtocol`. The result is divided by `rsethSupply` to produce `newRsETHPrice`, which is written to storage as `rsETHPrice` (line 313).

**Exploit path — cross-asset deposit dilution:**

The attack exploits the fact that `rsETHPrice` is stored and used as the denominator in minting, while the numerator uses a live oracle call per asset:

```solidity
// LRTDepositPool.sol line 520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

If the Chainlink stETH/ETH feed is stale at a depressed price, the attacker:
1. Calls `updateRSETHPrice()` → `totalETHInProtocol` is computed lower (stETH undervalued) → `rsETHPrice` is stored lower than true value.
2. Deposits ETH (whose price is always 1e18, unaffected by the stale stETH feed). The minting formula divides by the now-depressed `rsETHPrice`, issuing more rsETH than the deposited ETH is worth.
3. When `rsETHPrice` is next updated with a fresh Chainlink price, the attacker's rsETH is worth more than deposited — at the expense of existing holders (dilution of the pool).

**Existing guards are insufficient:**

- `pricePercentageLimit` only triggers a pause if the price drop exceeds the configured threshold; within the threshold, the stale price is written without restriction.
- The upside `PriceAboveDailyThreshold` check only applies when `newRsETHPrice > highestRsethPrice`, not for downward manipulation.
- `unlockQueue` in `LRTWithdrawalManager` has operator-supplied price bounds (`minimumRsEthPrice`, `maximumRsEthPrice`) that partially mitigate the withdrawal path, but the deposit path has no equivalent guard.

**Cross-chain propagation:**

`RSETHMultiChainRateProvider.updateRate()` (inherited from `MultiChainRateProvider`, line 108) is also permissionless (`external payable nonReentrant`). It reads `ILRTOracle(rsETHPriceOracle).rsETHPrice()` and broadcasts the stale rate to all L2 pools, where it governs wrsETH minting.

## Impact Explanation

**Primary — Theft of unclaimed yield (High):** An attacker deposits ETH or a non-stale LST while `rsETHPrice` is artificially depressed by a stale Chainlink feed. They receive more rsETH than the deposited assets are worth. When the oracle recovers, the attacker's rsETH is worth more than deposited, at the direct expense of existing rsETH holders whose proportional share of the pool is diluted. This matches the allowed impact: **Theft of unclaimed yield**.

**Secondary — Temporary freezing of funds (Medium):** If the stale price drop exceeds `pricePercentageLimit`, `_updateRsETHPrice()` calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on the oracle itself, freezing all deposits and withdrawals until an admin manually unpauses. An attacker can deliberately trigger this at a moment of Chainlink latency. This matches the allowed impact: **Temporary freezing of funds**.

## Likelihood Explanation

Chainlink LST/ETH feeds (e.g., stETH/ETH, cbETH/ETH) have heartbeat intervals of 24 hours and deviation thresholds of 0.5%. During periods of high network congestion or rapid LST price movements, feeds can lag behind the true price. The attacker requires no capital at risk for the trigger itself — only observation of a staleness window and a gas cost to call `updateRSETHPrice()`. The deposit step requires capital but is fully recoverable (the attacker receives rsETH). Likelihood: **Low-Medium** (requires a Chainlink staleness window, which occurs periodically in practice).

## Recommendation

Apply the same staleness and round-completeness checks already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
uint256 public constant MAX_STALENESS = 25 hours; // configurable per feed

function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();
    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (block.timestamp - updatedAt > MAX_STALENESS) revert PriceStale();
    if (price <= 0) revert InvalidPrice();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, consider adding an access-control modifier (e.g., `onlyLRTManager`) or a minimum time-between-updates guard to `updateRSETHPrice()` to prevent permissionless triggering of price snapshots.

## Proof of Concept

**Setup:** Protocol holds stETH and ETH. stETH Chainlink feed heartbeat is 24h. Attacker observes the feed has not updated for 20+ hours and the last reported price is 0.5% below the true current stETH/ETH rate.

**Steps:**
1. Attacker calls `LRTOracle.updateRSETHPrice()` from any EOA. The function is `public whenNotPaused` with no role check.
2. Inside `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `ChainlinkPriceOracle.getAssetPrice(stETH)` returns the stale low price without reverting.
3. `totalETHInProtocol` is computed lower than the true value; `newRsETHPrice` is set lower than the true rate (within `pricePercentageLimit` so no pause is triggered) and written to `rsETHPrice`.
4. Attacker calls `LRTDepositPool.depositETH{value: largeAmount}(0, "")`. The minting calculation: `rsethAmountToMint = (largeAmount * 1e18) / depressedRsETHPrice` issues more rsETH than the deposited ETH is worth at the true rate.
5. When `updateRSETHPrice()` is next called with a fresh Chainlink price, `rsETHPrice` recovers to its true value. The attacker's rsETH is now worth more than deposited, at the expense of existing holders.

**Foundry fork test plan:**
- Fork Ethereum mainnet at a block where a Chainlink LST feed is near its heartbeat boundary.
- Mock `latestRoundData()` to return a price with `updatedAt = block.timestamp - 23 hours` (stale but within heartbeat).
- Call `updateRSETHPrice()` from an unprivileged address; assert `rsETHPrice` is written lower than the pre-call value.
- Deposit ETH; assert `rsethAmountToMint` exceeds the fair-value amount computed at the true oracle price.
