Audit Report

## Title
Missing Sequencer Uptime Feed Check in `ProtectedPriceProviderL2` and `PriceProviderL2` Enables Stale-Price Swaps During L2 Sequencer Downtime — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`, `smart-contracts-poc/contracts/PriceProviderL2.sol`)

## Summary

Both L2 price providers rely solely on `_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)` as their L2 safety gate. Neither contract queries a Chainlink sequencer uptime feed nor enforces a post-restart grace period. When the L2 sequencer goes down, oracle `refTime` freezes at the pre-downtime value; `_isStale` returns `false` for up to `MAX_TIME_DELTA` (up to 7 days), and `getBidAndAskPrice()` returns the frozen stale bid/ask to the pool swap path, enabling direct LP fund extraction at pre-downtime prices.

## Finding Description

**Root cause — sole staleness gate:**

`ProtectedPriceProviderL2._computeBidAsk` and `PriceProviderL2._getBidAndAskPrice` both gate on `_isStale` only:

```solidity
// ProtectedPriceProviderL2.sol L206-209
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
    return (0, type(uint128).max);
}
```

`_isStale` is a pure time-delta check with no sequencer awareness:

```solidity
// ProtectedPriceProviderL2.sol L138-153
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Neither constructor accepts a `_sequencerUptimeFeed` parameter, and no `GRACE_PERIOD` constant exists in either source file. `PriceProviderFactoryL2.createPriceProvider` (L41-79) also passes no sequencer feed when deploying `PriceProviderL2`.

**Registry ABI confirms the intended interface was missing from source:**

The deployed registry ABI for `PriceProviderL2` (`registry.json` L597-653) includes constructor parameter `_sequencerUptimeFeed` (address), and the ABI exposes `GRACE_PERIOD()` (L720-732) and `sequencerUptimeFeed()` (L1234). The `ChainlinkVerifierL2` ABI (L5685-5790) also exposes `GRACE_PERIOD()` and `sequencerUptimeFeed()`. The source code contracts implement none of these.

**Exploit path:**

1. Pool is deployed on Arbitrum/Base/Optimism using `ProtectedPriceProviderL2` or `PriceProviderL2`.
2. Oracle publishes price `P` at time `T`; `refTime = T`.
3. L2 sequencer goes down at `T+1`. Oracle stops updating; `refTime` stays at `T`.
4. Real market price moves significantly (e.g., drops 50%).
5. Attacker submits swap via L1 rollup inbox (bypassing sequencer).
6. `_isStale(T, T+1800, 3600, ...)` → `1800 ≤ 3600` → `false` (not stale).
7. `getBidAndAskPrice()` returns bid/ask derived from stale price `P`.
8. Pool executes swap at stale price; attacker extracts value from LPs.
9. On sequencer restart, oracle data is still within `MAX_TIME_DELTA`; attacker front-runs the first oracle update and repeats.

## Impact Explanation

This is a **bad-price execution** impact: a stale bid/ask quote derived from a frozen pre-downtime oracle price reaches the pool's swap path (`swap` → `_getBidAndAskPriceX64` → `SwapMath.midAndSpreadFeeX64FromBidAsk` → `_executeSwap`). The pool is a pure oracle market maker — its solvency guarantee rests entirely on the oracle price being current. Every swap executed at a stale price directly transfers value from LPs to the attacker. The exploitation window is up to `MAX_TIME_DELTA` (up to 7 days by constructor bound). This constitutes direct loss of LP principal, meeting the Critical/High threshold.

## Likelihood Explanation

- Arbitrum and Optimism sequencers have documented historical downtime events.
- The protocol explicitly targets L2 deployments: Arbitrum, Base, Optimism, Avalanche, BSC, Polygon, Berachain, Monad, MegaETH (registry deployments L5800-5836).
- No privileged action is required; any public trader can submit a swap via the L1 rollup inbox during sequencer downtime.
- `MAX_TIME_DELTA` up to 7 days creates an extremely wide exploitation window.
- The attack is repeatable on every sequencer downtime event.

## Recommendation

Add a sequencer uptime feed check to both `ProtectedPriceProviderL2` and `PriceProviderL2`, consistent with the `GRACE_PERIOD`/`sequencerUptimeFeed` interface already present in the registry ABI:

```solidity
AggregatorV3Interface public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600; // 1 hour

function _isSequencerUp() internal view returns (bool) {
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    if (answer != 0) return false;
    if (block.timestamp - startedAt < GRACE_PERIOD) return false;
    return true;
}
```

Call `_isSequencerUp()` at the top of `_computeBidAsk` / `_getBidAndAskPrice` and return `(0, type(uint128).max)` if it returns `false`. Pass `_sequencerUptimeFeed` through `PriceProviderFactoryL2.createPriceProvider` to the deployed `PriceProviderL2`.

## Proof of Concept

1. Deploy `ProtectedPriceProviderL2` on Arbitrum fork with `MAX_TIME_DELTA = 3600`.
2. Set oracle price `P = 2000e8` at `T = 0`; `refTime = T`.
3. `vm.warp(T + 1)` — simulate sequencer going down; oracle stops updating.
4. Real price drops to `P' = 1000e8`.
5. At `T + 1800`, call `pool.swap(...)` (attacker submits via L1 inbox).
6. Assert `_isStale(T, T+1800, 3600, ...)` returns `false`.
7. Assert `getBidAndAskPrice()` returns bid/ask derived from `P = 2000e8` (stale).
8. Assert attacker receives token output priced at 2000 USDC/ETH while market is 1000 USDC/ETH — LP loses 1000 USDC per ETH traded.
9. `vm.warp(T + 3001)` — sequencer restarts; repeat swap before oracle updates to confirm post-restart window.
