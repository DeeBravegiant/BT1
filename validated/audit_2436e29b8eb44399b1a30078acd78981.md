### Title
Unprivileged `logMetadata` Accepts Arbitrary Token Address, Enabling Fake Token Registration Across Chains — (`File: evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.logMetadata` is `external payable` with zero access control and accepts any caller-supplied `tokenAddress`. It reads `name`, `symbol`, and `decimals` from that address via external calls and emits a `BridgeTypes.LogMetadata` event. In the Wormhole variant it also publishes a Wormhole message. The NEAR bridge's `deploy_token_callback` trusts any `LogMetadata` event whose emitter is a registered factory — which the OmniBridge contract is — and deploys a NEAR-side token using the metadata verbatim. Because the EVM contract imposes no constraint on which token address may be logged, an unprivileged attacker can inject arbitrary name/symbol/decimals into the cross-chain token registry.

---

### Finding Description

`OmniBridge.logMetadata` (line 224) is declared `external payable` with no role check, no allowlist, and no validation of `tokenAddress`:

```solidity
function logMetadata(address tokenAddress) external payable {
    string memory name  = IERC20Metadata(tokenAddress).name();
    string memory symbol = IERC20Metadata(tokenAddress).symbol();
    uint8  decimals     = IERC20Metadata(tokenAddress).decimals();

    logMetadataExtension(tokenAddress, name, symbol, decimals);   // Wormhole publish
    emit BridgeTypes.LogMetadata(tokenAddress, name, symbol, decimals);
}
``` [1](#0-0) 

`logMetadataExtension` is overridden in `OmniBridgeWormhole` to publish a Wormhole message carrying the same payload: [2](#0-1) 

On the NEAR side, `deploy_token_callback` accepts the resulting proof and deploys a NEAR token. Its only gate is that the emitter address must equal the registered factory — which the OmniBridge contract always satisfies. The `token_address`, `name`, `symbol`, and `decimals` fields are consumed without further validation: [3](#0-2) 

The Starknet bridge's `log_metadata` (line 144) has the same structure — no caller restriction, any `ContractAddress` accepted: [4](#0-3) 

---

### Impact Explanation

An attacker deploys a malicious ERC-20 whose `name()`, `symbol()`, and `decimals()` return arbitrary values (e.g., `"USD Coin"`, `"USDC"`, `6`). Calling `logMetadata(maliciousToken)` on the deployed OmniBridge:

1. Emits `LogMetadata(maliciousToken, "USD Coin", "USDC", 6)` from the legitimate, factory-registered bridge contract address.
2. In the Wormhole deployment, simultaneously publishes a Wormhole VAA carrying the same fake metadata.
3. A relayer submits the proof to NEAR's `deploy_token`, which passes the factory check and calls `deploy_token_internal`, permanently registering a NEAR-side token whose `name`/`symbol` are `"USD Coin"`/`"USDC"` but whose backing address is the attacker's worthless contract.

This is asset-identity divergence: a token with a misleading identity is permanently inscribed in the cross-chain registry. Users who inspect the NEAR token by name/symbol rather than by its derived account ID (`0x<malicious>.eth.bridge.near`) may believe they are receiving or holding a legitimate USDC-backed asset. The Wormhole path propagates the same fake registration to every chain that processes the VAA.

The `deploy_token_internal` guard `require!(self.deployed_tokens.insert(&token_id), ...)` prevents re-registration of the same EVM address, but does not prevent an attacker from registering many distinct malicious addresses, each with the same misleading human-readable metadata. [5](#0-4) 

---

### Likelihood Explanation

The attack requires no privilege, no leaked key, and no colluding party. The attacker only needs to:
- Deploy a standard ERC-20 contract with crafted `name`/`symbol`/`decimals` return values (trivial, ~$1 in gas).
- Call `logMetadata` on the live OmniBridge (one transaction, no ETH value required beyond gas).

The NEAR relayer infrastructure is designed to pick up and process every `LogMetadata` event from registered factories automatically, so the fake registration propagates without any further attacker action.

---

### Recommendation

1. **Restrict the caller** — add an access-control role (e.g., `DEFAULT_ADMIN_ROLE` or a dedicated `TOKEN_LISTER_ROLE`) to `logMetadata` and `logMetadata1155`, mirroring the pattern used by `addCustomToken` and `setMetadata`.
2. **Alternatively, validate the token address** — require that `tokenAddress` is either already in `isBridgeToken` or in a curated allowlist before emitting the event.
3. Apply the same fix to the Starknet `log_metadata` entry point.

// Malicious ERC-20 that returns fake metadata
contract FakeUSDC {
    function name()     external pure returns (string memory) { return "USD Coin"; }
    function symbol()   external pure returns (string memory) { return "USDC"; }
    function decimals() external pure returns (uint8)         { return 6; }

// Attack
contract Attack {
    function run(address omniBridge) external payable {
        FakeUSDC fake = new FakeUSDC();
        // No role required — call succeeds unconditionally
        IOmniBridge(omniBridge).logMetadata(address(fake));
        // LogMetadata(fake, "USD Coin", "USDC", 6) is now emitted from the
        // legitimate, factory-registered bridge contract.
        // Relayer submits proof → NEAR deploys "USD Coin / USDC" token
        // backed by the worthless FakeUSDC address.
    }
}

        fn log_metadata(ref self: ContractState, token: ContractAddress) {
            // There are two possible metadata standards in use.
            // 1. Old style: name and symbol are felt252 values.
            // 2. New style: name and symbol are ByteArray values (ERC20 ABI).
            // We are using low-level contract calls to determine the type.

            let call_data: Array<felt252> = array![];
            let mut res = syscalls::call_contract_syscall(
                token, selector!("name"), call_data.span(),
            )
                .unwrap_syscall();

            let name = if res.len() == 1 {
                // Old standard (felt252)
                let name = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize name',
                );
                utils::felt252_to_string(name)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize name',
                )
            };

            let mut res = syscalls::call_contract_syscall(
                token, selector!("symbol"), call_data.span(),
            )
                .unwrap_syscall();

            let symbol = if res.len() == 1 {
                // Old standard (felt252)
                let symbol = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize symbol',
                );
                utils::felt252_to_string(symbol)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize symbol',
                )
            };

            let decimals = {
                let mut res = syscalls::call_contract_syscall(
                    token, selector!("decimals"), call_data.span(),
                )
                    .unwrap_syscall();

                let decimals = OptionTrait::expect(
                    Serde::<u8>::deserialize(ref res), 'Could not deserialize decimals',
                );
                decimals
            };

            self.emit(Event::LogMetadata(LogMetadata { address: token, name, symbol, decimals }))
        }
