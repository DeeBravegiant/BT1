### Title
Stale Pending Data Fallback Omits `l1_data_gas_price`/`l2_gas_price`, Corrupting `starknet_estimateFee` and `starknet_simulateTransactions` Results — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When `pending_data.block.parent_block_hash()` does not match `latest_header.block_hash`, `read_pending_data` constructs a synthetic `DeprecatedPendingBlock` using `..Default::default()`. This omits `l1_data_gas_price` and `l2_gas_price` from the latest header. The `Deprecated` variant of `PendingBlockOrDeprecated` always returns `GasPricePerToken::default()` (zero) for both fields. Those zeros flow into `create_block_context`, which silently replaces them with `NonzeroGasPrice::MIN` (= `GasPrice(1)`) via `unwrap_or`. Any call to `starknet_estimateFee` or `starknet_simulateTransactions` with `block_id=Tag::Pending` during this window receives a fee estimate computed with `l1_data_gas_price=1` and `l2_gas_price=1` instead of the real pending prices.

### Finding Description

**Step 1 — Entry point (public RPC):**
`estimate_fee` in `api_impl.rs` is called with `block_id=Tag::Pending`. [1](#0-0) 

**Step 2 — Stale-data branch in `read_pending_data`:**
When `pending_data.block.parent_block_hash() != latest_header.block_hash`, a synthetic `DeprecatedPendingBlock` is constructed. Only `eth_l1_gas_price` and `strk_l1_gas_price` are copied from the latest header; `l1_data_gas_price` and `l2_gas_price` are left to `..Default::default()`. [2](#0-1) 

**Step 3 — `Deprecated` variant always returns zero for the missing fields:**
`l1_data_gas_price()` returns `GasPricePerToken::default()` and `l2_gas_price()` returns `GasPricePerToken::default()` for any `Deprecated` block, by explicit design for backward compatibility. [3](#0-2) 

**Step 4 — Conversion preserves the zeros:**
`client_pending_data_to_execution_pending_data` calls `block.l1_data_gas_price()` and `block.l2_gas_price()`, forwarding the zeros into `ExecutionPendingData`. [4](#0-3) 

**Step 5 — Silent replacement with `NonzeroGasPrice::MIN` in `create_block_context`:**
`NonzeroGasPrice::new(0)` fails, so `unwrap_or(NonzeroGasPrice::MIN)` substitutes `GasPrice(1)` for both `l1_data_gas_price` and `l2_gas_price` in the `BlockContext` used for execution. [5](#0-4) 

### Impact Explanation

Any `starknet_estimateFee` or `starknet_simulateTransactions` call with `block_id=Tag::Pending` during the transient window where pending data is stale returns fee estimates computed with `l1_data_gas_price=1` and `l2_gas_price=1` instead of the real pending prices. For modern transactions (v3) that consume L1 data gas or L2 gas, the returned `overall_fee`, `l1_data_gas_price`, and `l2_gas_price` fields in `FeeEstimation` are wrong by orders of magnitude. This is an authoritative-looking wrong value from a public RPC endpoint.

### Likelihood Explanation

The stale-pending-data window occurs naturally at every block transition: the node finalizes a new block before the pending sync loop fetches fresh pending data. Any client polling `starknet_estimateFee` with `Tag::Pending` at block boundaries will hit this path. No attacker privilege is required; the public RPC endpoint is sufficient.

### Recommendation

In the stale-data branch of `read_pending_data`, copy `l1_data_gas_price` and `l2_gas_price` from `latest_header.block_header_without_hash` into the synthetic block, exactly as `l1_gas_price` is already copied. Alternatively, use a `PendingBlock` (the `Current` variant) for the fallback so that all price fields are populated from the header and the `Deprecated` zero-return paths are not exercised.

### Proof of Concept

1. Write a block header with non-trivial `l1_data_gas_price` and `l2_gas_price` into storage.
2. Set `pending_data.block.parent_block_hash` to a hash that does **not** match `latest_header.block_hash`.
3. Call `starknet_estimateFee` with `block_id=Tag::Pending` on a transaction that consumes L1 data gas or L2 gas.
4. Assert that the returned `l1_data_gas_price` and `l2_gas_price` in `FeeEstimation` equal `GasPrice(1)` (i.e., `NonzeroGasPrice::MIN`) rather than the values stored in the header.
