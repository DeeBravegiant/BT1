Audit Report

## Title
Stale `rsETHPrice` After EigenPod Checkpoint Finalization Enables Yield Theft via Deposit-Before-Price-Update — (`contracts/LRTOracle.sol`, `contracts/LRTDepositPool.sol`)

## Summary
`rsETHPrice` is a stored value updated only by explicit calls to `updateRSETHPrice()`. When an EigenPod checkpoint finalizes, the protocol's live TVL (read via `getEffectivePodShares()`) increases immediately, but the stored price does not. An attacker can deposit ETH at the stale (pre-reward) price, then call the permissionless `updateRSETHPrice()`, capturing a portion of the consensus rewards that should have accrued proportionally to all existing rsETH holders.

## Finding Description
`LRTOracle._getTotalEthInProtocol()` reads live EigenLayer share balances on every call via `ILRTDepositPool.getTotalAssetDeposits()` → `getETHDistributionData()` → `INodeDelegator.getEffectivePodShares()`, which returns `stakedButUnverifiedNativeETH + withdrawableShare` fetched live from EigenLayer's `DelegationManager`. Meanwhile, `rsETHPrice` is a cached storage variable updated only when `_updateRsETHPrice()` is called.

After a checkpoint finalizes and `podOwnerDepositShares` increases in EigenLayer, there is a window — potentially spanning multiple blocks — during which `getTotalAssetDeposits(ETH)` returns a higher value than what `rsETHPrice` reflects. `getRsETHAmountToMint` uses `rsETHPrice` as the denominator: `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`, so any deposit during this window mints excess rsETH at the stale (lower) price.

The attacker then calls `updateRSETHPrice()` (public, no role required) to crystallize the higher price. The `pricePercentageLimit` guard is bypassable: the attacker's own deposit D dilutes the price increase to `R/(T+D)`, which can be made arbitrarily small by choosing D large enough. If `pricePercentageLimit == 0`, there is no guard at all. Even if the threshold is hit, the manager will eventually call `updateRSETHPriceAsManager()`, and the attacker still profits from having deposited at the stale price.

**Attack sequence (no front-running required):**
1. Observe `CheckpointFinalized` event — live TVL increases by R, `rsETHPrice` remains stale at `T/S`.
2. Call `depositETH(D)` — mints `D*S/T` rsETH at the stale price.
3. Call `updateRSETHPrice()` — price rises to `(T+R+D)*T / (S*(T+D))`.
4. Initiate withdrawal — attacker's rsETH is now worth `D*(T+R+D)/(T+D)` ETH, a profit of `R*D/(T+D)` ETH.

Existing holders receive `T*R/(T+D)` instead of `R` in yield — a loss of `R*D/(T+D)` ETH.

## Impact Explanation
This is **theft of unclaimed yield** (High). Existing rsETH holders receive less yield than they are entitled to. The attacker extracts `R·D/(T+D)` ETH of yield, where R is the checkpoint reward, D is the attacker's deposit, and T is the pre-reward TVL. For large D relative to T, this approaches the full reward R. The loss is permanent and borne by all existing holders proportionally.

## Likelihood Explanation
Low-to-Medium. The attack is fully permissionless — no privileged access, no front-running, no victim mistakes required. The attacker executes all steps themselves. The only constraints are: (1) capital proportional to TVL to be economically meaningful, and (2) capital must be locked for the EigenLayer withdrawal delay (~7 days). EigenPod checkpoints are finalized periodically (at least daily in active protocols), so the window recurs regularly. Any actor with idle capital whose opportunity cost is below the protocol yield rate is incentivized to execute this.

## Recommendation
1. **Atomic price update on deposit**: Call `_updateRsETHPrice()` at the start of `depositETH()` / `depositAsset()` to ensure the price is always fresh before minting.
2. **Alternatively**: Compute rsETH to mint using live TVL rather than the stored `rsETHPrice`, so the mint ratio always reflects current state: `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset) * rsethSupply) / _getTotalEthInProtocol()`.
3. **Minimum**: Revert deposits when the live TVL diverges from `rsethSupply * rsETHPrice` by more than a configurable threshold.

## Proof of Concept
```solidity
// Foundry fork test (mainnet fork)
function testYieldTheft() public {
    // State: T = 100_000 ETH TVL, S = rsETH supply, rsETHPrice = T/S (stale)
    uint256 T = 100_000 ether;
    uint256 R = 10 ether; // checkpoint reward

    // 1. Simulate checkpoint finalization: podOwnerDepositShares increases by R
    //    Live TVL is now T+R; rsETHPrice is still T/S
    vm.prank(address(eigenPod));
    eigenPodManager.recordBeaconChainETHBalanceUpdate(address(nodeDelegator), T, int256(R));

    // 2. Attacker deposits D ETH at stale price — mints D*S/T rsETH
    uint256 D = 1_000_000 ether;
    vm.deal(attacker, D);
    vm.prank(attacker);
    lrtDepositPool.depositETH{value: D}(0, "");
    uint256 attackerRsETH = rsETH.balanceOf(attacker);

    // 3. Attacker calls updateRSETHPrice — price rises to reflect T+R+D TVL
    uint256 priceBefore = lrtOracle.rsETHPrice();
    lrtOracle.updateRSETHPrice();
    uint256 priceAfter = lrtOracle.rsETHPrice();
    assertGt(priceAfter, priceBefore);

    // 4. Attacker's rsETH is worth more than D ETH — profit = R*D/(T+D)
    uint256 attackerETHValue = attackerRsETH * priceAfter / 1e18;
    assertGt(attackerETHValue, D);
    // Existing holders received T*R/(T+D) instead of R — loss confirmed
}
```
