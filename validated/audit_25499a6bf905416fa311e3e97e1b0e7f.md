Audit Report

## Title
Nested Cross-Contract Loops in `updateRSETHPrice()` Cause Multiplicative Gas Consumption - (File: contracts/LRTOracle.sol)

## Summary
The public `updateRSETHPrice()` function triggers a three-level nested call chain: for each supported asset it calls `getTotalAssetDeposits()`, which calls `getAssetDistributionData()`, which loops over all NDCs and calls `getAssetUnstaking()` on each, which in turn fetches and iterates all queued EigenLayer withdrawals and their strategies. The total number of `getQueuedWithdrawals` external calls is `N_assets × M_NDCs`, each returning full withdrawal arrays that are then iterated, making gas cost multiplicative and potentially exceeding the block gas limit as the protocol scales.

## Finding Description
The call chain is confirmed exactly as described:

**Layer 1** — `updateRSETHPrice()` at `LRTOracle.sol:87` is `public whenNotPaused` with no further access control. It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which loops over all supported assets and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [1](#0-0) [2](#0-1) 

**Layer 2** — `getTotalAssetDeposits()` calls `getAssetDistributionData()`. For non-ETH assets, `getAssetDistributionData()` loops over all NDCs and calls `INodeDelegator.getAssetUnstaking(asset)` on each. For ETH, `getETHDistributionData()` performs the same loop calling `getAssetUnstaking(LRTConstants.ETH_TOKEN)` per NDC. [3](#0-2) [4](#0-3) 

**Layer 3** — `getAssetUnstaking()` calls `_getDelegationManager().getQueuedWithdrawals(address(this))`, which returns all queued withdrawals for that NDC, then iterates over every withdrawal and every strategy within each withdrawal — a nested W × S loop with external calls per strategy. [5](#0-4) 

**Critical amplification**: `getQueuedWithdrawals` is called once per `(asset, NDC)` pair, not once per NDC. With N assets and M NDCs, this is `N × M` full EigenLayer array fetches, even though the result for a given NDC is identical regardless of which asset is being queried. This is the primary inefficiency beyond the nested loop structure itself.

**Existing bounds**: `maxNodeDelegatorLimit` is initialized to 10 but is admin-adjustable upward. `maxUncompletedWithdrawalCount` in `LRTUnstakingVault` is a global counter across all NDCs, not per-NDC, and is also admin-configurable. Neither provides a hard on-chain cap that prevents the gas issue at realistic production scale. [6](#0-5) [7](#0-6) 

## Impact Explanation
The impact is **Medium — Unbounded gas consumption**, which is an explicitly allowed impact class. As the protocol adds supported assets, deploys more NDCs, and accumulates pending EigenLayer withdrawals through normal operation, the gas cost of `updateRSETHPrice()` grows multiplicatively. If the function exceeds the block gas limit, `rsETHPrice` becomes permanently stale, breaking all deposit and withdrawal pricing that depends on it, and halting protocol fee accrual. [8](#0-7) [9](#0-8) 

## Likelihood Explanation
Any unprivileged external caller can invoke `updateRSETHPrice()` — the only guard is `whenNotPaused`. No attacker action is required; the condition is triggered by organic protocol growth (more assets, more NDCs, more EigenLayer queued withdrawals). The `initiateUnstaking` and `undelegate` functions are called by operators in normal protocol operation, continuously increasing the queued withdrawal count. [1](#0-0) [10](#0-9) 

## Recommendation
1. **Eliminate per-asset redundant `getQueuedWithdrawals` calls**: Cache the result of `getQueuedWithdrawals(ndc)` once per NDC and compute all asset amounts from the single cached result, rather than calling it once per `(asset, NDC)` pair.
2. **Cache `getAssetUnstaking` results in storage**: Maintain a `assetUnstaking[ndc][asset]` mapping updated only in `initiateUnstaking` and `completeUnstaking`, and read the cached value in `getAssetDistributionData()` instead of re-querying EigenLayer on every price update.
3. **Decouple the oracle from live NDC enumeration**: Have `updateRSETHPrice()` read from a pre-aggregated TVL snapshot updated incrementally, rather than iterating all NDCs and all EigenLayer queued withdrawals in a single transaction.

## Proof of Concept
1. Deploy with 5 supported assets, 10 NDCs (`maxNodeDelegatorLimit = 10`), each NDC having 10 queued EigenLayer withdrawals with 3 strategies each.
2. Call `updateRSETHPrice()` from any EOA.
3. Observe: `getQueuedWithdrawals` is called `5 × 10 = 50` times (once per asset per NDC). Each call returns 10 withdrawals × 3 strategies = 30 strategy entries to iterate, for 1,500 inner iterations plus 50 external array-returning calls.
4. Measure gas: each `getQueuedWithdrawals` STATICCALL copies large memory arrays; at ~5,000 gas per EigenLayer storage read plus memory overhead, the call approaches or exceeds Ethereum's 30M block gas limit.
5. A Foundry fork test against a mainnet/testnet fork with mocked EigenLayer state (or a local mock `DelegationManager` returning arrays of the described sizes) can reproduce and measure the gas consumption directly.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
