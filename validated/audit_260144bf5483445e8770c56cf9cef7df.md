Audit Report

## Title
High Watermarks Not Reset on Full Bin Liquidity Withdrawal, Causing False Stop-Loss Triggers - (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension` maintains per-bin high watermarks (`highWatermarks[pool][binIdx]`) that are updated exclusively in `afterSwap`. When all shares are removed from a bin, the stale watermark persists. When new liquidity is subsequently added to the empty bin at the protocol's initial per-share rate, the first swap touching that bin computes a metric far below the frozen watermark, triggering a false `OracleStopLossTriggered` revert that blocks swaps through that bin ŌĆö permanently if `decayPerSecondE8 == 0`.

## Finding Description

`OracleValueStopLossExtension` only overrides `afterSwap` and `initialize` from `BaseMetricExtension`. The `afterRemoveLiquidity` hook is not registered; its base implementation reverts with `ExtensionNotImplemented`. [1](#0-0) 

The sole watermark update path is `afterSwap` ŌåÆ `_afterSwapOracleStopLoss` ŌåÆ `_checkAndUpdateWatermarks`. Inside `_afterSwapOracleStopLoss`, bins with `totalShares == 0` are explicitly skipped: [2](#0-1) 

This means the watermark is never cleared while the bin is empty. When an LP removes all shares, `LiquidityLib.removeLiquidity` sets `binTotalShares[binIdx]` to zero but leaves `highWatermarks[pool][binIdx]` intact at its last high value. [3](#0-2) 

When new liquidity is subsequently added to the empty bin, `LiquidityLib.addLiquidity` initialises token balances using the pool's fixed `initialScaledToken0PerShareE18` / `initialScaledToken1PerShareE18` constants: [4](#0-3) 

The resulting per-share metric is the protocol's baseline initial value, which is structurally lower than a watermark that was ratcheted up by swap-driven value accumulation. On the next swap touching that bin, `_checkAndUpdateWatermarks` computes: [5](#0-4) 

`_applyWatermark` detects `metric < (hwm * floorMultiplier) / E6` and sets `breached = true`: [6](#0-5) 

The swap reverts with `OracleStopLossTriggered`. The pool admin can manually reset watermarks via `proposeOracleStopLossHighWatermarks` + `executeOracleStopLossHighWatermarks`, but this path is timelocked and requires privileged action ŌĆö the pool remains broken until the admin intervenes.

## Impact Explanation

Any swap crossing or touching the affected bin reverts. If the affected bin is the active `curBinIdx`, all swaps in the pool are blocked. With `decayPerSecondE8 == 0` the block is permanent without admin intervention. Even with decay enabled, recovery requires waiting until the watermark decays below `metric / floorMultiplier`, which can span days to weeks. This constitutes broken core pool functionality (unusable swap flow) meeting the allowed impact gate.

## Likelihood Explanation

The trigger requires three routine, unprivileged actions: (1) an LP removes all shares from a bin ŌĆö explicitly permitted since `newUserShares == 0` passes the `minimalMintableLiquidity` check; (2) any LP adds liquidity to the now-empty bin; (3) any swap touches that bin. Bins draining to zero when price moves out of range and refilling when price returns is the standard LP lifecycle. No adversarial intent is required; the scenario arises from normal usage. [7](#0-6) 

## Recommendation

Implement `afterRemoveLiquidity` in `OracleValueStopLossExtension`. For each bin in the removal delta whose post-removal `binTotalShares` equals zero, `delete highWatermarks[pool_][binIdx]`. Alternatively, in `_afterSwapOracleStopLoss`, when `totalShares == 0` is detected for a bin, explicitly `delete highWatermarks[pool_][binIdx]` rather than `continue`-ing past it.

## Proof of Concept

```
Setup: pool with OracleValueStopLossExtension
  drawdownE6 = 500_000 (50% drawdown floor)
  decayPerSecondE8 = 0 (no decay)

1. LP1 adds 10_000 shares to bin 0 (current bin).
   ŌåÆ bin0: token0=1000, token1=1000, totalShares=10_000

2. Swaps occur through bin 0.
   ŌåÆ afterSwap fires; watermark ratchets up:
     highWatermarks[pool][0].token0 = 2000

3. LP1 removes all 10_000 shares from bin 0.
   ŌåÆ binTotalShares[0] = 0
   ŌåÆ highWatermarks[pool][0].token0 = 2000  ŌåÉ NOT cleared

4. LP2 adds 10_000 shares to bin 0 (empty bin).
   ŌåÆ LiquidityLib initialises at initialScaledToken0PerShareE18
   ŌåÆ bin0: token0 = 500 (initial rate), totalShares = 10_000

5. Any swap touching bin 0:
   ŌåÆ _afterSwapOracleStopLoss: totalShares=10_000 (non-zero), enters check
   ŌåÆ metricT0 Ōēł 500*1e6/10_000 = 50
   ŌåÆ decayed hwm0 = 2000 (no decay)
   ŌåÆ threshold = 2000 * 500_000 / 1_000_000 = 1000
   ŌåÆ 50 < 1000 ŌåÆ OracleStopLossTriggered revert
   ŌåÆ ALL swaps through bin 0 permanently blocked
```

  function _checkAndUpdateWatermarks(
    address pool_,
    int8 binIdx,
    uint256 metricT0,
    uint256 metricT1,
    uint256 floorMultiplier,
    uint256 decayRate,
    bool zeroForOne
  ) private {
    BinHighWatermarks storage hwmS = highWatermarks[pool_][binIdx];
    uint256 dt = block.timestamp - hwmS.lastDecayTs;

    (uint256 hwm0, bool breach0) = _applyWatermark(metricT0, _decayed(hwmS.token0, decayRate, dt), floorMultiplier);
    if (breach0 && zeroForOne) {
      revert OracleStopLossTriggered(binIdx, true, metricT0, (hwm0 * floorMultiplier) / E6);
    }

    (uint256 hwm1, bool breach1) = _applyWatermark(metricT1, _decayed(hwmS.token1, decayRate, dt), floorMultiplier);
    if (breach1 && !zeroForOne) {
      revert OracleStopLossTriggered(binIdx, false, metricT1, (hwm1 * floorMultiplier) / E6);
    }

    // forge-lint: disable-next-line(unsafe-typecast)
    hwmS.token0 = uint104(hwm0);
    // forge-lint: disable-next-line(unsafe-typecast)
    hwmS.token1 = uint104(hwm1);
    hwmS.lastDecayTs = uint32(block.timestamp);
  }
