### Title
Unvalidated `fee_proposal_fri` During Bootstrap Window Allows Arbitrary L2 Gas Price Manipulation — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

During the bootstrap window (the first `fee_proposal_window_size` blocks), `is_proposal_init_valid` explicitly skips all bounds enforcement on the proposer-supplied `fee_proposal_fri` field. A malicious proposer can inject an arbitrary value (e.g., `u128::MAX` or `0`) that is accepted by every validator, stored permanently in the `fee_proposals_window`, and then used as the median `fee_actual` that floors the L2 gas price for all subsequent blocks.

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` validates `fee_proposal_fri` only when `proposal_init_validation.fee_actual` is `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // bounds check here — SKIPPED when fee_actual is None
}
``` [1](#0-0) 

`fee_actual` is `None` whenever `compute_fee_actual` cannot fill a complete sliding window of `fee_proposal_window_size` entries:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

After consensus decides on the block, `finalize_decision` unconditionally records the proposer's stated value into the window:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [3](#0-2) 

`record_fee_proposal` inserts the value directly into `fee_proposals_window` with no further validation:

```rust
fn record_fee_proposal(&mut self, height: BlockNumber, fee_proposal_fri: Option<GasPrice>) {
    self.fee_proposals_window.insert(height, fee_proposal_fri);
}
``` [4](#0-3) 

Once the window fills, `compute_fee_actual` returns the median of those stored values. That median is then used as `effective_min` in `calculate_next_l2_gas_price_for_fin`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [5](#0-4) 

The injected `fee_proposal_fri` is also hashed into the `ProposalCommitment` that consensus votes on:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [6](#0-5) 

The validator uses the proposer's stated `fee_proposal_fri` (not an independently computed value) when building the commitment:

```rust
let batcher_block_commitment = proposal_commitment_from(
    finished_info.proposal_commitment.partial_block_hash,
    fee_proposal,  // = args.init.fee_proposal_fri
);
``` [7](#0-6) 

The value is also persisted to `BlockHeaderWithoutHash.fee_proposal_fri` via `update_state_sync_with_new_block`, so it survives restarts and is re-loaded into the window by `initialize_fee_proposals_window`. [8](#0-7) 

### Impact Explanation

A malicious proposer who controls any block during the bootstrap window can set `fee_proposal_fri = u128::MAX`. Every validator accepts it (bounds check is skipped), the value is stored in the window, and once the window fills, `fee_actual = u128::MAX`. `effective_min` becomes `u128::MAX`, so `calculate_next_base_gas_price` returns `u128::MAX` as the minimum L2 gas price. Every subsequent transaction must declare `max_price_per_unit >= u128::MAX`, which is impossible in practice — the network is permanently DoS'd at the fee level. Conversely, setting `fee_proposal_fri = 0` drives the gas price floor to zero, enabling zero-fee transactions.

This matches the impact category: **Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact (Critical)**.

### Likelihood Explanation

The bootstrap window is `fee_proposal_window_size` blocks long (a protocol constant). Any validator node that wins the proposer lottery during those blocks can trigger this. On a fresh network or after a restart that clears the window, the attack surface is open for the entire window duration. No special privilege beyond being a consensus participant is required.

### Recommendation

**Short term**: When `fee_actual` is `None`, validate `fee_proposal_fri` against a hard absolute range (e.g., `[min_gas_price, some_configured_max]`) rather than skipping all validation. The proposer's fallback is `l2_gas_price`; the validator can enforce that the proposed value is within a fixed multiplicative band of `l2_gas_price` even during bootstrap.

**Long term**: Decouple the "no historical median yet" case from "no validation at all." Introduce a bootstrap-specific bound anchored to the locally-known `l2_gas_price` so that the proposer-supplied value cannot deviate arbitrarily from the validator's own reference point.

### Proof of Concept

1. Network starts at height 0 (`fee_proposal_window_size = N`). `fee_actual` is `None` for heights `0..N`.
2. Attacker wins the proposer slot at height 0 and sends `ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), starknet_version: V0_14_3, ... }`.
3. `is_proposal_init_valid` reaches the fee-proposal check:
   - `fee_proposal_required = true` (version ≥ V0_14_3) → presence check passes.
   - `proposal_init_validation.fee_actual = None` → the `if let (Some(fee_actual), Some(fee_proposal))` arm is **not entered**; no bounds check runs.
4. Proposal is accepted; `finalize_decision` calls `record_fee_proposal(BlockNumber(0), Some(GasPrice(u128::MAX)))`.
5. Attacker repeats for heights 1 through N−1 (or enough to dominate the median).
6. At height N, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))`.
7. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX`; the next L2 gas price is `u128::MAX`.
8. All user transactions are rejected at pre-validation (`max_price_per_unit < actual_gas_price`), permanently halting the network. [1](#0-0) [9](#0-8) [10](#0-9)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
