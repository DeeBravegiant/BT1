Audit Report

## Title
`LineaMessenger.sendETHToL1ViaBridge` Delivers `value - minimumFee` to L1 Target Instead of Full `value`, Breaking `IL2Messenger` Interface Invariant — (`contracts/bridges/LineaMessenger.sol`)

## Summary

`LineaMessenger` implements `IL2Messenger` but silently delivers less ETH to the L1 target than the `value` parameter specifies. The Linea bridge's `sendMessage` deducts `_fee` from `msg.value` before crediting the recipient, so the L1 vault receives `value - minimumFee`. Pool contracts emit `BridgedETHToL1ViaNativeBridge` recording the full `ethBalanceMinusFees` as bridged, creating a persistent cross-chain accounting discrepancy that grows with every bridge call.

## Finding Description

`LineaMessenger.sendETHToL1ViaBridge` at [1](#0-0)  calls:

```solidity
ILineaMessageService(l2bridge).sendMessage{ value: value }(target, minimumFee, bytes(""));
```

The Linea `sendMessage(address _to, uint256 _fee, bytes _calldata)` signature (defined in [2](#0-1) ) deducts `_fee` from `msg.value` before crediting `_to`. The L1 target therefore receives `value - minimumFee`, not `value`.

Every other `IL2Messenger` implementation forwards the full `value`:
- `ArbitrumMessenger`: `withdrawEth{ value: value }(target)` [3](#0-2) 
- `OptimismMessenger`: `bridgeETHTo{ value: value }(target, ...)` [4](#0-3) 

All pool contracts (`RSETHPool`, `RSETHPoolNoWrapper`, `RSETHPoolV2ExternalBridge`, `RSETHPoolV3ExternalBridge`) call through the `IL2Messenger` abstraction and emit events recording the full `ethBalanceMinusFees` as bridged: [5](#0-4) [6](#0-5) 

The `minimumFee` check at [7](#0-6)  only guards against `value <= minimumFee`; it does not correct the delivery shortfall.

## Impact Explanation

**Low — Contract fails to deliver promised returns, but doesn't lose value.** The L1 vault (`l1VaultETHForL2Chain`) receives `ethBalanceMinusFees - minimumFee` ETH while on-chain events record `ethBalanceMinusFees`. The `minimumFee` is paid to the Linea relayer and is not lost from the broader system, but it is permanently diverted away from the intended L1 recipient. The cumulative shortfall grows with every bridge call.

## Likelihood Explanation

This fires deterministically on every call to `bridgeAssetsViaNativeBridge` on a Linea-deployed pool instance. No special conditions are required. The `minimumFee` on Linea is non-zero by protocol design. The `BRIDGER_ROLE` calls this function as part of normal protocol operations.

## Recommendation

In `LineaMessenger.sendETHToL1ViaBridge`, send only `value - minimumFee` as `msg.value` to the bridge while keeping `minimumFee` as the fee argument:

```solidity
ILineaMessageService(l2bridge).sendMessage{ value: value - minimumFee }(target, minimumFee, bytes(""));
```

Alternatively, pool callers should pre-compute the net delivered amount (querying `minimumFeeInWei()` from the bridge) and pass `value - minimumFee` as both the ETH sent and the `value` argument to `sendETHToL1ViaBridge`, then emit that net amount in `BridgedETHToL1ViaNativeBridge`.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";
import {LineaMessenger} from "contracts/bridges/LineaMessenger.sol";

contract MockLineaBridge {
    uint256 public constant MIN_FEE = 0.001 ether;
    address public lastTo;
    uint256 public lastValueDelivered;

    function minimumFeeInWei() external pure returns (uint256) { return MIN_FEE; }

    // Linea sendMessage: delivers msg.value - _fee to _to
    function sendMessage(address _to, uint256 _fee, bytes calldata) external payable {
        lastTo = _to;
        lastValueDelivered = msg.value - _fee;
        payable(_to).transfer(lastValueDelivered);
    }
    receive() external payable {}
}

contract MockL1Vault {
    uint256 public received;
    receive() external payable { received += msg.value; }
}

contract LineaMessengerTest is Test {
    LineaMessenger messenger;
    MockLineaBridge bridge;
    MockL1Vault l1Vault;

    function setUp() public {
        messenger = new LineaMessenger(address(this));
        bridge = new MockLineaBridge();
        l1Vault = new MockL1Vault();
    }

    function test_lineaDeliversLessThanValue() public {
        uint256 value = 1 ether;
        messenger.sendETHToL1ViaBridge{value: value}(address(bridge), address(l1Vault), value);
        // L1 vault receives value - minimumFee, not value
        assertEq(l1Vault.received(), value - bridge.MIN_FEE());
        // Accounting discrepancy = minimumFee
        assertGt(value - l1Vault.received(), 0);
    }
}
```

The assertion `l1Vault.received() == value - MIN_FEE` passes, confirming the L1 vault receives less than the `value` recorded in the `BridgedETHToL1ViaNativeBridge` event. [8](#0-7)
