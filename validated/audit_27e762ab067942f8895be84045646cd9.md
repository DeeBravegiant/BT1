Audit Report

## Title
Permissionless `claim` Enables Forced Fee Extraction from Any Reward Claimant - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim` accepts a caller-supplied `account` parameter with no `msg.sender` restriction. Any external caller holding a valid Merkle proof — which the protocol publishes off-chain — can trigger a claim on behalf of any user. When `feeInBPS > 0`, this permanently reduces the victim's claimable yield by up to 10% without their consent, with the fee irrecoverably transferred to `protocolTreasury`.

## Finding Description
`MerkleDistributor.claim` performs no check that `account == msg.sender`: [1](#0-0) 

After verifying the Merkle proof, the contract computes and deducts a fee from the victim's claimable amount: [2](#0-1) 

The `feeInBPS` is owner-configurable up to `MAX_FEE_IN_BPS = 1000` (10%): [3](#0-2) 

By contrast, `KernelMerkleDistributor._processClaim` explicitly enforces the caller restriction: [4](#0-3) 

The exploit path is: (1) protocol publishes Merkle tree with Alice's leaf `(index, Alice, cumulativeAmount)`; (2) owner sets `feeInBPS > 0`; (3) Alice defers claiming, intending to wait for a fee reduction; (4) attacker calls `MerkleDistributor.claim(index, Alice, cumulativeAmount, aliceProof)`; (5) contract deducts fee and transfers it to `protocolTreasury`; (6) Alice's `userClaims[Alice].cumulativeAmount` is updated to `cumulativeAmount`, permanently preventing recovery of the extracted fee. [5](#0-4) 

## Impact Explanation
**High — Theft of unclaimed yield.** A victim user loses up to 10% of their entitled yield permanently. The fee is transferred to `protocolTreasury` and cannot be reclaimed. The victim's cumulative claim state is updated to the full `cumulativeAmount`, closing off any future claim for the extracted portion. This matches the allowed impact class "Theft of unclaimed yield."

## Likelihood Explanation
**Medium.** All inputs required — `index`, `account`, `cumulativeAmount`, and `merkleProof` — are published by the protocol's off-chain distribution service and become visible on-chain once any legitimate user submits a transaction. No privileged access, key compromise, or brute force is required. The attack is repeatable across all eligible accounts in the Merkle tree and can be executed by any unprivileged external caller.

## Recommendation
Add an `account == msg.sender` guard to `MerkleDistributor.claim`, mirroring the pattern already used in `KernelMerkleDistributor._processClaim`:

```solidity
if (account != msg.sender) revert Unauthorized();
``` [4](#0-3) 

## Proof of Concept
1. Deploy `MerkleDistributor` with `feeInBPS = 500` (5%).
2. Owner calls `setMerkleRoot` with a tree containing leaf `(index=1, account=Alice, cumulativeAmount=1000e18)`.
3. Attacker obtains Alice's proof from the protocol's off-chain API.
4. Attacker calls `MerkleDistributor.claim(1, Alice, 1000e18, aliceProof)`.
5. Contract computes `fee = 50e18`, transfers `950e18` to Alice and `50e18` to `protocolTreasury`.
6. `userClaims[Alice].cumulativeAmount == 1000e18` — Alice can never reclaim the 50e18.

**Foundry test plan:** Fork mainnet (or use a local deployment), initialize `MerkleDistributor` with `feeInBPS = 500`, construct a two-leaf Merkle tree, call `claim` from `address(attacker)` with Alice's `(index, account, cumulativeAmount, proof)`, assert Alice received `950e18` and `protocolTreasury` received `50e18`, and assert `isClaimed(1, Alice) == true`. [6](#0-5)

    /// @inheritdoc IMerkleDistributor
    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
```
