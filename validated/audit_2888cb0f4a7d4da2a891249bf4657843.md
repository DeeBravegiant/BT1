Audit Report

## Title
ETH Donation via Unrestricted `receive()` Triggers Illegitimate Protocol Fee Minting, Diluting rsETH Holders — (`contracts/LRTDepositPool.sol` / `contracts/LRTOracle.sol`)

## Summary
`LRTDepositPool` exposes a bare, permissionless `receive()` fallback that accepts ETH from any caller without accounting. Because `getETHDistributionData()` measures `address(this).balance` directly, any donated ETH is immediately reflected in protocol TVL. When the public `updateRSETHPrice()` is subsequently called, the oracle treats the balance increase as staking yield, computes a fee, and mints rsETH to the treasury — diverting to the treasury a portion of value that should have accrued entirely to rsETH holders.

## Finding Description

**Step 1 — Unrestricted ETH ingress.**
`LRTDepositPool` at [1](#0-0)  accepts ETH from any caller with no access control, no accounting update, and no event. This is distinct from the named ingress functions (`receiveFromRewardReceiver`, `receiveFromNodeDelegator`, `receiveFromLRTConverter`) which are also payable but semantically scoped.

**Step 2 — Raw balance used as TVL.**
`getETHDistributionData()` reads `address(this).balance` directly: [2](#0-1)  There is no distinction between ETH that arrived via `depositETH()` and ETH that arrived via the bare `receive()`.

**Step 3 — Donated ETH flows into `totalETHInProtocol`.**
`_getTotalEthInProtocol()` iterates all supported assets and for ETH delegates to `getETHDistributionData()` via `getTotalAssetDeposits`: [3](#0-2) 

**Step 4 — Public price update treats balance increase as yield.**
`updateRSETHPrice()` is callable by any unprivileged address: [4](#0-3)  Inside `_updateRsETHPrice()`, the entire TVL increase since the last price update is treated as yield and a fee is computed: [5](#0-4) 

**Step 5 — Fee is minted to treasury, diluting holders.**
The fee is converted to rsETH at the new price and minted to the treasury: [6](#0-5) 

**Why existing guards are insufficient:**

- `pricePercentageLimit`: Only reverts if the price increase exceeds the configured threshold *and* `pricePercentageLimit > 0`. Small donations (or a zero limit) bypass it entirely. [7](#0-6) 
- `_checkAndUpdateDailyFeeMintLimit`: Caps per-day damage but does not prevent the attack; it can be repeated across days. If `maxFeeMintAmountPerDay == 0` the revert at L205 would block the attack, but a zero limit means fee minting is disabled entirely — the normal operating state has a non-zero limit. [8](#0-7) 

## Impact Explanation

Let `D` = donated ETH, `F` = `protocolFeeInBPS / 10_000`, `S` = rsETH total supply, `T` = previous TVL.

- Without donation: rsETH price = `T / S`.
- With donation + fee: rsETH price = `(T + D − D·F) / S`.
- Treasury receives `D·F / newPrice` rsETH.
- Each existing holder's pro-rata claim is reduced by exactly `D·F` worth of ETH value.

The fee portion `D·F` of the donated ETH is diverted from rsETH holders to the treasury. This is a direct, quantifiable loss of yield for all rsETH holders. **Impact: High — Theft of unclaimed yield.** The yield that should accrue entirely to rsETH holders (the donated ETH increasing backing per share) is partially captured by the treasury via an illegitimate fee trigger.

## Likelihood Explanation

No special role or permission is required. Any EOA or contract can call `receive()` and then `updateRSETHPrice()`. The `pricePercentageLimit` guard is bypassable with small donations. The `maxFeeMintAmountPerDay` cap limits per-day damage but does not prevent the attack; it can be repeated across days. Motivation exists for any party that is short rsETH or wishes to harm the protocol's holders at a predictable cost (attacker spends `D`, holders lose `D·F`).

## Recommendation

1. **Track legitimate ETH separately.** Maintain an internal accounting variable (e.g., `ethReceivedLegitimately`) incremented only by `depositETH`, `receiveFromRewardReceiver`, `receiveFromNodeDelegator`, and `receiveFromLRTConverter`. Use this variable instead of `address(this).balance` in `getETHDistributionData()`.

2. **Remove or restrict the bare `receive()` fallback.** If ETH must be accepted from arbitrary sources, emit an event and add the amount to the accounting variable so it is treated as a deposit (rsETH is minted to the sender), not as free yield.

3. **Alternatively**, in `_updateRsETHPrice()`, cap `rewardAmount` to a value derived from known yield sources (e.g., ETH forwarded through `receiveFromRewardReceiver`) rather than the raw TVL delta.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

interface IDepositPool {
    function depositETH(uint256 minRSETH, string calldata ref) external payable;
}
interface IOracle {
    function updateRSETHPrice() external;
    function rsETHPrice() external view returns (uint256);
}
interface IRSETH {
    function balanceOf(address) external view returns (uint256);
}

contract DonationFeeExploit is Test {
    address depositPool; // deployed LRTDepositPool proxy
    address oracle;      // deployed LRTOracle proxy
    address rsETH;       // rsETH token
    address treasury;    // PROTOCOL_TREASURY

    function testDonationTriggersFee() public {
        uint256 treasuryBefore = IRSETH(rsETH).balanceOf(treasury);
        uint256 priceBefore    = IOracle(oracle).rsETHPrice();

        // Step 1: attacker donates 1 ETH directly to LRTDepositPool via receive()
        vm.deal(address(this), 1 ether);
        (bool ok,) = depositPool.call{value: 1 ether}("");
        require(ok, "ETH send failed");

        // Step 2: anyone calls updateRSETHPrice (public, no role required)
        IOracle(oracle).updateRSETHPrice();

        // Step 3: treasury received rsETH as fee funded by the donation
        uint256 treasuryAfter = IRSETH(rsETH).balanceOf(treasury);
        assertGt(treasuryAfter, treasuryBefore, "treasury should have gained rsETH fee");

        // Step 4: price increased but by less than the full donation
        //         (fee portion was diverted to treasury, not to holders)
        uint256 priceAfter = IOracle(oracle).rsETHPrice();
        assertGt(priceAfter, priceBefore, "price should have increased");

        emit log_named_uint("rsETH minted to treasury", treasuryAfter - treasuryBefore);
        emit log_named_uint("price increase (wei)",     priceAfter - priceBefore);
    }
}
```
