### Title
Stale `parameters_votes` Carried Back After Resharing Cancellation Breaks Re-vote Invariant - (`crates/contract/src/state/resharing.rs`)

### Summary

When `vote_cancel_resharing` reverts the contract to `previous_running_state`, it clones and returns that state without resetting `parameters_votes`. During resharing, the re-proposal path (`vote_new_parameters` on the resharing state) mutates `previous_running_state.parameters_votes` in-place. Any partial re-proposal votes accumulated before cancellation are silently carried into the restored `RunningContractState`, reducing the number of explicit votes required to trigger the next resharing.

### Finding Description

When the contract transitions from Running → Resharing, `transition_to_resharing_no_checks` stores a fresh `previous_running_state` via `RunningContractState::new()`, which initialises `parameters_votes` to `ThresholdParametersVotes::default()` (empty). [1](#0-0) 

During resharing, participants may call `vote_new_parameters` to propose a different participant set (re-proposal). This call delegates directly into `self.previous_running_state.process_new_parameters_proposal(proposal)`, which mutates `previous_running_state.parameters_votes` in-place: [2](#0-1) [3](#0-2) 

When resharing is subsequently cancelled, `vote_cancel_resharing` clones `self.previous_running_state` and sets only `previously_cancelled_resharing_epoch_id`. It does **not** reset `parameters_votes`: [4](#0-3) 

The restored `RunningContractState` therefore contains whatever re-proposal votes were accumulated before cancellation. Because `ThresholdParametersVotes` stores votes keyed by `(AuthenticatedAccountId → ProposedThresholdParameters)` with no epoch binding, those stale votes are immediately valid in the new Running epoch and count toward the next `vote_new_parameters` call for the same proposal content. [5](#0-4) 

### Impact Explanation

The production invariant is that triggering resharing from Running state requires **all** `new_num_participants` proposed participants to cast explicit votes in that epoch: [6](#0-5) 

Stale votes from a cancelled re-proposal reduce the number of new explicit votes required. If `k` participants voted for proposal `P2` during resharing and resharing is then cancelled, only `|P2| − k` additional votes are needed in the restored Running state to trigger resharing with `P2`. Participants who voted during resharing cannot withdraw those votes after cancellation; their intent is locked in without any mechanism to retract it. This breaks the accounting invariant that every resharing epoch requires a fresh, complete quorum of explicit votes.

This matches the **Medium** allowed impact: *participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants*.

### Likelihood Explanation

The scenario is realistic in adversarial conditions:

1. Resharing is initiated (all current participants vote for proposal `P1`).
2. During resharing, a subset of participants vote for re-proposal `P2` (e.g., a set that excludes certain nodes).
3. The resharing is cancelled by threshold votes from previous participants.
4. The restored Running state silently carries `k` votes for `P2`.
5. Only `|P2| − k` more votes are needed to trigger resharing with `P2` — without the `k` participants needing to re-vote.

Resharing cancellation is an explicitly supported and tested operational flow, making this path reachable in production.

### Recommendation

In `vote_cancel_resharing`, reset `parameters_votes` to `ThresholdParametersVotes::default()` before returning the restored running state:

```rust
let running_state = if threshold_cancellation_votes_reached {
    let mut previous_running_state = self.previous_running_state.clone();
    previous_running_state.previously_cancelled_resharing_epoch_id =
        Some(prospective_epoch_id);
    // Reset stale re-proposal votes so the next resharing requires a
    // fresh, complete quorum of explicit votes.
    previous_running_state.parameters_votes = ThresholdParametersVotes::default();
    Some(previous_running_state)
} else {
    None
};
```

This mirrors the behaviour of `RunningContractState::new()`, which already resets `parameters_votes` to default on every fresh construction. [7](#0-6) 

### Proof of Concept

1. Start with a Running state: participants `{A, B, C}`, threshold 2.
2. All three vote for resharing to `{A, B, C, D}` → contract enters Resharing state. `previous_running_state.parameters_votes` is empty.
3. During resharing, `A` calls `vote_new_parameters` with re-proposal `P2 = {A, B}`, threshold 2. `previous_running_state.parameters_votes` now contains `A → P2`.
4. `A` and `B` call `vote_cancel_resharing` (threshold = 2). Contract reverts to Running state. The restored `RunningContractState.parameters_votes` still contains `A → P2`.
5. In the restored Running state, `B` calls `vote_new_parameters` with `P2` and the correct `prospective_epoch_id`. `process_new_parameters_proposal` counts `A`'s stale vote plus `B`'s new vote → `n_votes = 2 = |P2|` → resharing to `{A, B}` is triggered immediately, without `A` having to re-vote explicitly in the Running epoch. [4](#0-3) [3](#0-2)

    pub(super) fn process_new_parameters_proposal(
        &mut self,
        proposal: &ProposedThresholdParameters,
    ) -> Result<bool, Error> {
        // ensure the proposal is valid against the current parameters
        self.parameters
            .validate_incoming_proposal(proposal.parameters())?;

        // Validate effective per-domain thresholds (updates override, absent
        // domains keep theirs) against the proposed participant count.
        let new_num_participants =
            u64::try_from(proposal.participants().len()).expect("participant count fits in u64");
        let threshold_updates = proposal.per_domain_thresholds();
        // Reject unknown domain IDs: the loop below iterates existing domains, so
        // an unknown ID would otherwise be silently ignored here (it's caught at
        // the resharing transition, but we fail fast at vote acceptance).
        for id in threshold_updates.keys() {
            if self.domains.get_domain_by_domain_id(*id).is_none() {
                return Err(DomainError::UnknownDomainInProposal { domain_id: *id }.into());
            }
        }
        let effective_domains: Vec<DomainConfig> = self
            .domains
            .domains()
            .iter()
            .map(|domain| {
                let effective_threshold = threshold_updates
                    .get(&domain.id)
                    .copied()
                    .unwrap_or(domain.reconstruction_threshold);
                DomainConfig {
                    reconstruction_threshold: effective_threshold,
                    ..domain.clone()
                }
            })
            .collect();
        for domain in &effective_domains {
            validate_domain_threshold(domain, new_num_participants)?;
        }

        // The GovernanceThreshold must dominate every domain's effective ReconstructionThreshold;
        // enforced here so the state transition is self-contained (single source of truth).
        ThresholdParameters::validate_governance_against_reconstruction(
            new_num_participants,
            proposal.threshold(),
            max_reconstruction_threshold(&effective_domains),
        )?;

        // ensure the signer is a proposed participant
        let candidate = AuthenticatedAccountId::new(proposal.participants())?;

        // If the signer is not a participant of the current epoch, they can only vote after
        // `threshold` participant of the current epoch have casted their vote to admit them.
        if AuthenticatedAccountId::new(self.parameters.participants()).is_err() {
            let n_votes = self
                .parameters_votes
                .n_votes(proposal, self.parameters.participants());
            if n_votes < self.parameters.threshold().value() {
                return Err(VoteError::VoterPending.into());
            }
        }

        // finally, vote.
        let n_votes = self.parameters_votes.vote(proposal, candidate);
        Ok(new_num_participants == n_votes)

    pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
        let previous_running_participants = self.previous_running_state.parameters.participants();
        let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
        self.cancellation_requests.insert(authenticated_candidate);

        let cancellation_votes_count = self.cancellation_requests.len() as u64;
        let previous_running_threshold = self.previous_running_state.parameters.threshold();

        let threshold_cancellation_votes_reached: bool =
            cancellation_votes_count >= previous_running_threshold.value();

        let running_state = if threshold_cancellation_votes_reached {
            let mut previous_running_state = self.previous_running_state.clone();
            let prospective_epoch_id = self.prospective_epoch_id();
            previous_running_state.previously_cancelled_resharing_epoch_id =
                Some(prospective_epoch_id);

            Some(previous_running_state)
        } else {
            None
        };

        Ok(running_state)
