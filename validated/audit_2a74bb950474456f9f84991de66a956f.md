Audit Report

## Title
Zero rsETH Minted for Dust Deposits Causes Permanent Loss of Depositor Funds - (File: contracts/pools/RSETHPoolV3.sol)

## Summary
All L2 pool deposit functions guard only against `amount == 0` but not against a post-calculation `rsETHAmount == 0`. Because `viewSwapRsETHAmountAndFee` uses integer division (`amountAfterFee * 1e18 / rsETHToETHrate`), a 1 wei ETH deposit truncates to `rsETHAmount = 0`. The depositor's ETH is retained by the pool while they receive no rsETH or wrsETH in return.

## Finding Description
In `RSETHPoolV3.deposit()` (L246–265), `RSETHPoolV3ExternalBridge.deposit()` (L366–384), and `RSETHPoolNoWrapper.deposit()` (L231–244), the only zero-value guard is:

```solidity
if (amount == 0) revert InvalidAmount();
```

`viewSwapRsETHAmountAndFee` computes:

```solidity
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

With `rsETHToETHrate ≈ 1.05e18`, a deposit of 1 wei yields `1 * 1e18 / 1.05e18 = 0` due to integer truncation. The `limitDailyMint` modifier (L96–125) does not block this: `dailyMintAmount + 0 > dailyMintLimit` evaluates false, so execution proceeds. `wrsETH.mint(msg.sender, 0)` is a no-op; in `RSETHPoolNoWrapper`, `rsETH.safeTransfer(msg.sender, 0)` is also a no-op. The deposited ETH is absorbed into the pool balance with no rsETH issued.

## Impact Explanation
A depositor who sends 1 wei receives 0 rsETH/wrsETH. Their ETH is permanently retained by the pool. This matches **Low** — *"Contract fails to deliver promised returns, but doesn't lose value"*: the protocol retains the deposited ETH while the depositor is silently shortchanged with no recourse.

## Likelihood Explanation
Any unprivileged external caller can trigger this with `deposit{value: 1}("")` — no preconditions, no special role, no victim cooperation required. The path is fully reachable on all affected pool variants. While accidental 1 wei deposits are rare in practice, the path is unconditionally open.

## Recommendation
Add an explicit post-computation guard in each `deposit()` function immediately after computing `rsETHAmount`:

```solidity
if (rsETHAmount == 0) revert InvalidAmount();
```

Alternatively, enforce a protocol-level minimum deposit (analogous to `LRTDepositPool.minAmountToDeposit`) in all L2 pool contracts.

## Proof of Concept
1. `RSETHPoolV3.getRate()` returns `1.05e18`.
2. Caller invokes `RSETHPoolV3.deposit{value: 1}("")`.
3. `limitDailyMint(1, ETH_IDENTIFIER)` calls `viewSwapRsETHAmountAndFee(1)`:
   - `fee = 1 * feeBps / 10_000 = 0`
   - `amountAfterFee = 1`
   - `rsETHAmount = 1 * 1e18 / 1.05e18 = 0`
4. Modifier check: `0 + 0 > dailyMintLimit` → false, execution continues.
5. `wrsETH.mint(msg.sender, 0)` — no tokens issued.
6. Pool retains 1 wei ETH; depositor receives nothing.

Identical path exists in `RSETHPoolV3ExternalBridge.deposit()` (L366–384) and `RSETHPoolNoWrapper.deposit()` (L231–244).
