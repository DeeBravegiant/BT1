Audit Report

## Title
Chainlink Price Not Validated for Zero/Negative Value in `getAssetPrice` - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` fetches `int256 price` from Chainlink's `latestRoundData()` and casts it directly to `uint256` without any non-positive guard. A zero price causes depositors to receive 0 rsETH for deposited assets and causes all withdrawals for that asset to revert via division by zero. A negative price produces a near-`type(uint256).max` value, massively inflating protocol TVL and enabling protocol insolvency. The same repository's `ChainlinkOracleForRSETHPoolCollateral` already applies the correct `if (ethPrice <= 0) revert InvalidPrice()` guard, confirming protocol awareness of the risk.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` performs no validation on the returned price:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L49-55
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
``` [1](#0-0) 

Contrast with `ChainlinkOracleForRSETHPoolCollateral.getRate()`, which correctly guards:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L32
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

`LRTOracle.getAssetPrice()` delegates directly to this unguarded oracle: [3](#0-2) 

**Zero-price path 1 — deposit minting:**
`LRTDepositPool.getRsETHAmountToMint()` computes `(amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. With `getAssetPrice == 0`, this returns 0, so the depositor's assets are transferred in but 0 rsETH is minted. [4](#0-3) 

**Zero-price path 2 — withdrawal payout:**
`_calculatePayoutAmount` computes `(request.rsETHUnstaked * rsETHPrice) / assetPrice`. With `assetPrice == 0`, this is a division by zero, reverting every withdrawal for that asset. [5](#0-4) 

**Negative-price path — TVL inflation:**
`int256(-1)` cast to `uint256` yields `type(uint256).max`. `_getTotalEthInProtocol` accumulates `totalAssetAmt.mulWad(assetER)` with this enormous value, causing `newRsETHPrice` to spike and triggering `PriceAboveDailyThreshold` for non-manager callers, or minting unbounded protocol fee rsETH to treasury. [6](#0-5) 

## Impact Explanation

- **Critical — Direct theft of user funds:** A depositor calling `depositAsset` during a zero-price event transfers real assets into the protocol but receives 0 rsETH. The assets are held by the protocol with no corresponding claim token issued to the depositor.
- **Medium — Temporary freezing of funds:** All pending withdrawal requests for the affected asset revert with division by zero until the oracle recovers, freezing user withdrawals.
- **Critical — Protocol insolvency:** A negative Chainlink price produces a near-`type(uint256).max` exchange rate, inflating `totalETHInProtocol` and `newRsETHPrice`, enabling excessive protocol fee rsETH minting to treasury and corrupting the rsETH/ETH peg.

## Likelihood Explanation
Chainlink feeds can return 0 or negative values during circuit-breaker events, feed deprecation, or oracle malfunction. No attacker action is required — any depositor or withdrawer calling the affected functions during such an event triggers the impact. The protocol already applies the correct guard in `ChainlinkOracleForRSETHPoolCollateral`, confirming awareness of the risk and making the omission in `ChainlinkPriceOracle` a straightforward oversight.

## Recommendation
Add a non-positive price guard in `ChainlinkPriceOracle.getAssetPrice()`:

```diff
 (, int256 price,,,) = priceFeed.latestRoundData();
+if (price <= 0) revert InvalidPrice();
 return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [7](#0-6) 

## Proof of Concept

1. Deploy a mock Chainlink feed for a supported LST asset (e.g., stETH/ETH) that returns `answer = 0` from `latestRoundData()`.
2. Register it via `ChainlinkPriceOracle.updatePriceFeedFor(stETH, mockFeed)`.
3. Call `LRTDepositPool.depositAsset(stETH, 1e18, 0, referral)`:
   - `getRsETHAmountToMint(stETH, 1e18)` → `(1e18 * 0) / rsETHPrice = 0`
   - 1e18 stETH is transferred in; 0 rsETH is minted — depositor loses funds.
4. Call `LRTWithdrawalManager.unlockQueue(stETH, ...)` with a pending request:
   - `_createUnlockParams` fetches `assetPrice = 0`
   - `_calculatePayoutAmount` executes `(rsETHUnstaked * rsETHPrice) / 0` → division by zero → revert
   - All stETH withdrawals are frozen.
5. For the negative-price scenario: set mock feed to return `answer = -1`, call `updateRSETHPrice()` — `uint256(-1)` inflates `totalETHInProtocol` to near-max, spiking rsETH price and triggering insolvency-level accounting corruption.
