Audit Report

## Title
Nested Unbounded Loop in `updateRSETHPrice()` Causes Permanent Gas DoS on the Price Oracle - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function that triggers a nested loop: once over every supported asset, and once per asset over every node delegator in `nodeDelegatorQueue`, with three external calls per NDC per asset. As the protocol adds assets and node delegators through normal governance, the cumulative gas cost grows as O(N×M) and can exceed the Ethereum block gas limit, permanently preventing oracle price updates.

## Finding Description
`updateRSETHPrice()` is declared `public` with only a `whenNotPaused` guard, callable by any address. [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over every entry in `lrtConfig.getSupportedAssetList()` (no hard cap enforced) and for each asset calls `ILRTDepositPool.getTotalAssetDeposits(asset)`. [2](#0-1) 

`getTotalAssetDeposits` delegates to `getAssetDistributionData`, which contains an inner loop over the full `nodeDelegatorQueue` array, making **three external calls per NDC** (`IERC20.balanceOf`, `INodeDelegator.getAssetBalance`, `INodeDelegator.getAssetUnstaking`). [3](#0-2) 

`maxNodeDelegatorLimit` starts at 10 but has no ceiling — the admin can raise it freely, and the only constraint is that it cannot be set below the current queue length. [4](#0-3) 

The ETH asset path (`getETHDistributionData`) also loops over `nodeDelegatorQueue` with external calls per NDC, compounding the cost further. [5](#0-4) 

No existing guard limits the total number of external calls made within a single `updateRSETHPrice()` invocation. Once the cumulative gas cost exceeds ~30M gas, every call to `updateRSETHPrice()` reverts out-of-gas, and because removing NDCs requires zero balances (enforced by `_checkResidueEthBalance` / `_checkResidueLSTBalance`), the condition cannot be easily reversed while funds remain deployed. [6](#0-5) 

## Impact Explanation
**Medium — Unbounded gas consumption.** When `updateRSETHPrice()` becomes permanently uncallable, `rsETHPrice` is never updated, causing all subsequent `depositETH`/`depositAsset` calls to use a stale exchange rate (incorrect rsETH minting for all depositors). The fee-minting path inside `_updateRsETHPrice` is also permanently disabled, constituting **Medium — Permanent freezing of unclaimed yield** as a secondary impact. The downside-protection circuit breaker (auto-pause on price drop) can never trigger.

## Likelihood Explanation
No attacker action is required. Normal protocol governance — adding supported LST assets and raising `maxNodeDelegatorLimit` to accommodate more node delegators — is sufficient. With N=10 assets and M=10 NDCs the call count is already ~300 external calls; at N=20, M=20 it reaches ~1,200+, pushing well past the 30M gas block limit when combined with cold SLOADs and EigenLayer strategy reads inside each NDC. The condition is irreversible while NDCs hold balances.

## Recommendation
1. Introduce a strict, non-raisable hard cap on `maxNodeDelegatorLimit` (e.g., ≤ 10) in `updateMaxNodeDelegatorLimit`.
2. Enforce a hard cap on `supportedAssetList` length inside `LRTConfig.addNewSupportedAsset`.
3. Alternatively, refactor `_getTotalEthInProtocol()` to consume an incrementally maintained TVL cache updated on each deposit/withdrawal, eliminating the full recomputation on every oracle update.

## Proof of Concept
Call chain demonstrating the nested unbounded loop:

```
updateRSETHPrice()                          // public, no access control
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for assetIdx in supportedAssets (N, no hard cap):
                 └─ getTotalAssetDeposits(asset)
                      └─ getAssetDistributionData(asset)
                           └─ for i in nodeDelegatorQueue (M, admin-raisable):
                                ├─ IERC20(asset).balanceOf(ndc[i])
                                ├─ INodeDelegator(ndc[i]).getAssetBalance(asset)
                                └─ INodeDelegator(ndc[i]).getAssetUnstaking(asset)
```

**Foundry fork test plan:**
1. Deploy or fork the protocol with `maxNodeDelegatorLimit` raised to 20.
2. Register 20 NDC contracts and 10 supported assets.
3. Call `updateRSETHPrice()` with a gas limit of 30,000,000.
4. Assert the transaction reverts with out-of-gas.
5. Confirm `rsETHPrice` is unchanged and subsequent deposits mint at the stale rate.

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

        for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)
            uint256 assetER = getAssetPrice(asset);
            // totalAssetAmt is in 1e18 precision (standard token decimals)
            uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }
