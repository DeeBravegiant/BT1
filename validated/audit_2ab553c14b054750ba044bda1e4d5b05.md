Audit Report

## Title
Missing `timelock` Validation in `initialize()` and `proposeOracleStopLossTimelock()` Allows Pool Admin to Instantly Bypass LP Protection Timelock - (File: metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol)

## Summary
`OracleValueStopLossExtension.initialize()` validates `drawdownE6` and `decayPerSecondE8` but accepts any `uint32` value for `timelock`, including zero. `proposeOracleStopLossTimelock()` similarly imposes no lower-bound on `newTimelock`. A pool initialized with `timelock = 0`, or one whose admin reduces the timelock to zero via the proposal flow, allows the pool admin to propose and execute drawdown/decay/watermark changes atomically in the same block, collapsing the sole LP-protection delay to zero.

## Finding Description

The NatSpec at L15-16 explicitly states: *"Drawdown and decay changes are timelocked so LPs can react."* The timelock is the only mechanism bounding pool-admin trust.

**Root cause 1 — `initialize()` skips timelock validation:** [1](#0-0) 

`_validateDrawdown` and `_validateDecay` are called, but `timelock` is stored verbatim with no minimum check. A pool creator passes `timelock = 0` and the config is accepted.

**Root cause 2 — `proposeOracleStopLossTimelock()` accepts `newTimelock = 0`:** [2](#0-1) 

Even if a pool starts with a non-zero timelock, the admin can propose `newTimelock = 0`. After the current timelock elapses (a one-time wait), `executeOracleStopLossTimelock` sets `oracleStopLossConfig[pool_].timelock = 0`.

**Why `timelock = 0` collapses the protection:**

`_afterTimelock` computes `executeAfter = block.timestamp + 0 = block.timestamp`. [3](#0-2) 

`_requireElapsed` checks `block.timestamp < executeAfter`, which evaluates to `block.timestamp < block.timestamp` — always false — so it never reverts. [4](#0-3) 

**Exploit flow:**

1. Admin creates pool with `timelock = 0` (or reduces it to 0 via the proposal flow after one wait).
2. Admin calls `proposeOracleStopLossDrawdown(pool, 1_000_000)` (drawdown = 100 %, effectively disabling the stop-loss) — `executeAfter = block.timestamp`.
3. In the same transaction/block, admin calls `executeOracleStopLossDrawdown(pool)` — `_requireElapsed` passes immediately.
4. `oracleStopLossConfig[pool_].drawdownE6` is now `1_000_000` (E6), meaning `floorMultiplier = E6 - drawdown = 0`. The stop-loss floor is zero; `_afterSwapOracleStopLoss` will never revert regardless of value loss. [5](#0-4) 

Alternatively, the admin sets `drawdownE6 = 0` to disable the check entirely via the early-return at L217, or maximises `decayPerSecondE8` to `E8` (100 %/s), causing watermarks to decay to zero instantly and permanently disabling breach detection.

## Impact Explanation

The pool admin exceeds the intended admin boundary by bypassing the timelock — an explicitly listed allowed impact ("Admin-boundary break: pool admin exceeds caps, bypasses timelocks"). LPs who deposited under the assumption that drawdown/decay changes are delayed have no time to exit before the stop-loss is silently disabled. Once the stop-loss is off, swaps that drain LP value are no longer blocked by `afterSwap`, leading to direct loss of LP principal.

## Likelihood Explanation

Any pool admin (a semi-trusted role, not a fully trusted factory owner) can trigger this. Path 1 (initialize with `timelock = 0`) requires no waiting and is available at pool creation. Path 2 (reduce to 0 via proposal) requires one wait equal to the current timelock, after which the admin has permanent zero-delay control. No special privileges beyond pool admin are needed; the call is fully on-chain and repeatable.

## Recommendation

Add a minimum timelock constant (e.g., `MIN_TIMELOCK = 1 days`) and enforce it in both places:

```solidity
uint32 private constant MIN_TIMELOCK = 1 days;

function _validateTimelock(uint32 timelock) private pure {
    if (timelock < MIN_TIMELOCK) revert OracleStopLossTimelockTooShort(timelock);
}
```

Call `_validateTimelock(timelock)` in `initialize()` alongside the existing drawdown/decay checks, and call `_validateTimelock(newTimelock)` at the top of `proposeOracleStopLossTimelock()`.

## Proof of Concept

```solidity
// Foundry unit test sketch
function test_zeroTimelockBypassesLPProtection() public {
    // 1. Initialize pool with timelock = 0
    bytes memory data = abi.encode(uint32(500_000), uint32(58), uint32(0));
    vm.prank(address(factory));
    ext.initialize(pool, data);

    // 2. Propose drawdown = 1_000_000 (100%, disables stop-loss)
    vm.prank(poolAdmin);
    ext.proposeOracleStopLossDrawdown(pool, 1_000_000);

    // 3. Execute in the same block — no revert because executeAfter == block.timestamp
    vm.prank(poolAdmin);
    ext.executeOracleStopLossDrawdown(pool);

    // 4. Confirm stop-loss is disabled
    assertEq(ext.oracleStopLossConfig(pool).drawdownE6, 1_000_000);
    // afterSwap now returns early at drawdown==E6 → floorMultiplier==0, no breach possible
}
```
