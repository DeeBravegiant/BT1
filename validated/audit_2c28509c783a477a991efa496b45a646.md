The critical finding is in how `coinbase::convert()` handles trailing bytes. Let me confirm the exact path:

### Title
Cross-Protocol Blob Confusion: ECDSA-2PC Key Blob Accepted by Schnorr x-Only Key Extraction API — (`src/cbmpc/c_api/schnorr2pc.cpp`, `src/cbmpc/api/schnorr2pc.cpp`)

---

### Summary

`cbmpc_schnorr_2p_extract_public_key_xonly` accepts a valid ECDSA-2PC key blob and returns `CBMPC_SUCCESS` with a 32-byte x-only public key. No C-layer or API-layer guard rejects the wrong protocol type. The root cause is that `deserialize_key_blob` in `schnorr2pc.cpp` uses `coinbase::convert()`, which silently ignores trailing bytes, while the ECDSA blob's extra fields (`c_key`, `paillier`) are exactly those trailing bytes.

---

### Finding Description

**Call chain:**

```
cbmpc_schnorr_2p_extract_public_key_xonly(key_blob, out_pub_key)
  → validate_cmem(key_blob)                          // only checks size > 0 && data != null
  → coinbase::api::schnorr_2p::extract_public_key_xonly(view_cmem(key_blob), pk)
      → deserialize_key_blob(key_blob, key)
          → coinbase::convert(blob, in)              // ← NO trailing-bytes check
          → blob.version == 1 ?                      // passes: ECDSA also uses version=1
          → blob_to_key(blob, key)                   // validates role, curve, x_share, Q
      → pub_key_xonly = key.Q.get_x().to_bin(32)
```

**Blob format mismatch:**

The Schnorr-2PC `key_blob_v1_t` serializes: `version | role | curve | Q_compressed | x_share`. [1](#0-0) 

The ECDSA-2PC `key_blob_v1_t` serializes: `version | role | curve | Q_compressed | x_share | c_key | paillier`. [2](#0-1) 

Both use `version = 1` and the same field order for the first five fields.

**The missing trailing-bytes check:**

`coinbase::convert<T>(T& dst, mem_t src)` does **not** enforce that all input bytes are consumed: [3](#0-2) 

The strict `coinbase::deser()` overload does enforce this: [4](#0-3) 

`deserialize_key_blob` in `schnorr2pc.cpp` uses `coinbase::convert`, not `coinbase::deser`: [5](#0-4) 

When an ECDSA blob is fed in, the converter reads `version`, `role`, `curve`, `Q_compressed`, `x_share` and stops. The remaining `c_key` and `paillier` bytes are silently ignored. No error is set.

**Validation in `blob_to_key` passes for a fresh ECDSA secp256k1 blob:**

- `blob.role ≤ 1` — always true for a valid ECDSA blob
- `blob.curve == secp256k1` — true if the ECDSA DKG used secp256k1
- `q.is_in_range(blob.x_share)` — true for a freshly generated ECDSA key (the ECDSA comment explicitly notes `x_share` may be unreduced only after refresh; a fresh key has `x_share ∈ [0, q)`)
- `key.Q.from_bin(curve, blob.Q_compressed)` — succeeds because `Q_compressed` is a valid secp256k1 point in both blob types [6](#0-5) 

The function then returns:
```cpp
pub_key_xonly = key.Q.get_x().to_bin(/*size=*/32);
``` [7](#0-6) 

This is the x-coordinate of the **ECDSA** key's public point, returned as a BIP-340 x-only key with `CBMPC_SUCCESS`.

The C-layer wrapper `cbmpc_schnorr_2p_extract_public_key_xonly` performs only `validate_cmem`, which checks only that `size > 0` and `data != null`: [8](#0-7) [9](#0-8) 

---

### Impact Explanation

An unprivileged API caller who holds (or can supply) an ECDSA-2PC key blob for secp256k1 can call `cbmpc_schnorr_2p_extract_public_key_xonly` and receive `CBMPC_SUCCESS` plus a 32-byte value that the caller and any downstream consumer will treat as a valid BIP-340 x-only public key. The returned bytes are the x-coordinate of the ECDSA key's public point — a structurally valid secp256k1 x-coordinate — so no downstream format check will reject it. Any application that then uses this value for BIP-340 key registration, address derivation, or signature verification is operating on a key extracted from the wrong protocol context.

---

### Likelihood Explanation

The precondition is straightforward: the attacker needs a valid ECDSA-2PC secp256k1 key blob (obtainable via `cbmpc_ecdsa_2p_dkg`) and the ability to call `cbmpc_schnorr_2p_extract_public_key_xonly`. Both are public API entry points. No threshold collusion, no memory access, and no cryptographic break is required. The only constraint is that the ECDSA blob must be fresh (not refreshed), so that `x_share ∈ [0, q)`.

---

### Recommendation

Replace `coinbase::convert(blob, in)` with `coinbase::deser(in, blob)` in every `deserialize_key_blob` function that parses opaque key blobs from untrusted callers. `coinbase::deser` enforces strict deserialization and rejects trailing bytes, which would cause the ECDSA blob (which has extra `c_key` and `paillier` fields) to fail parsing when interpreted as a Schnorr blob. [5](#0-4)

static error_t blob_to_key(const key_blob_v1_t& blob, coinbase::mpc::schnorr2p::key_t& key) {
  if (blob.role > 1) return coinbase::error(E_FORMAT, "invalid key blob role");
  if (static_cast<curve_id>(blob.curve) != curve_id::secp256k1)
    return coinbase::error(E_FORMAT, "invalid key blob curve");

  key.role = static_cast<coinbase::mpc::party_t>(static_cast<int32_t>(blob.role));
  key.curve = coinbase::crypto::curve_secp256k1;
  const auto& q = key.curve.order();
  if (!q.is_in_range(blob.x_share)) return coinbase::error(E_FORMAT, "invalid key blob");
  key.x_share = blob.x_share;

  return key.Q.from_bin(key.curve, blob.Q_compressed);
}

template <typename... ARGS>
error_t deser(mem_t bin, ARGS&... args) {
  converter_t converter(bin);
  converter.convert(args...);
  error_t rv = converter.get_rv();
  if (rv != SUCCESS) return rv;

  // Strict deserialization: reject trailing bytes
  if (converter.get_offset() != converter.get_size()) {
    return coinbase::error(E_BADARG);
  }

  return SUCCESS;
}

cbmpc_error_t cbmpc_schnorr_2p_extract_public_key_xonly(cmem_t key_blob, cmem_t* out_pub_key) {
  try {
    if (!out_pub_key) return E_BADARG;
    *out_pub_key = cmem_t{nullptr, 0};
    const auto vkb = validate_cmem(key_blob);
    if (vkb) return vkb;

    coinbase::buf_t pk;
    const coinbase::error_t rv = coinbase::api::schnorr_2p::extract_public_key_xonly(view_cmem(key_blob), pk);
    if (rv) return rv;

    return alloc_cmem_from_buf(pk, out_pub_key);
  } catch (const std::bad_alloc&) {
    if (out_pub_key) *out_pub_key = cmem_t{nullptr, 0};
    return E_INSUFFICIENT;
  } catch (...) {
    if (out_pub_key) *out_pub_key = cmem_t{nullptr, 0};
    return E_GENERAL;
  }
}
