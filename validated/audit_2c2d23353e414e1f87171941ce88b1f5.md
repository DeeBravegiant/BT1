### Title
Batch collateral check uses only top-level tx fee, leaving sub-transaction script execution costs uncovered — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs`)

---

### Summary

`validateBatchCollateral` correctly gates on `hasAnyRedeemers` (which includes sub-tx redeemers), but then delegates to `Babbage.validateTotalCollateral` using only the **top-level tx body and its collateral inputs**. The collateral sufficiency check (Part 5) compares the collateral balance against `txfee topTxBody * collateralPercent / 100`. Because the minimum-fee rule for the top-level tx (`getConwayMinFeeTx` → `alonzoMinFeeTx` → `totExUnits`) only sums execution units from the **top-level tx's witnesses**, sub-tx execution units are never priced into the required fee or the required collateral. An attacker can therefore submit a batch whose top-level tx carries a minimal fee and minimal collateral while embedding expensive, deliberately-failing Plutus scripts in sub-transactions.

### Finding Description

**`validateBatchCollateral` (lines 283–302)**

```haskell
validateBatchCollateral pp tx (UTxO utxo) =
  when (hasAnyRedeemers tx) $
    Babbage.validateTotalCollateral pp (tx ^. bodyTxL) utxoCollateral
  where
    utxoCollateral = Map.restrictKeys utxo (tx ^. bodyTxL . collateralInputsTxBodyL)
    hasAnyRedeemers t =
      hasRedeemers t || any hasRedeemers (t ^. bodyTxL . subTransactionsTxBodyL)
    hasRedeemers = not . null . (^. witsTxL . rdmrsTxWitsL . unRedeemersL)
``` [1](#0-0) 

`hasAnyRedeemers` is `True` when any sub-tx has redeemers. `validateTotalCollateral` is then called with `tx ^. bodyTxL` (top-level body) and `utxoCollateral` restricted to the **top-level tx's** `collateralInputsTxBodyL`. Part 5 of `validateTotalCollateral` checks:

```
balance ≥ ⌈txfee topTxBody × collateralPercent / 100⌉
``` [2](#0-1) 

`txfee topTxBody` is the top-level tx's fee field only.

**Fee calculation does not include sub-tx execution units**

The minimum fee is validated at line 373:

```haskell
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
``` [3](#0-2) 

For Dijkstra, `getMinFeeTxUtxo = getConwayMinFeeTxUtxo`:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [4](#0-3) 

`getMinFeeTx = getConwayMinFeeTx` → `alonzoMinFeeTx` → `totExUnits tx`:

```haskell
totExUnits tx = foldMap snd $ tx ^. witsTxL . rdmrsTxWitsL . unRedeemersL
``` [5](#0-4) 

`tx ^. witsTxL` is the **top-level tx's witnesses only**. Sub-tx redeemers live in each sub-tx's own `witsTxL`, which is never consulted here. The Dijkstra `EraTx` instance confirms no override:

```haskell
getMinFeeTx = getConwayMinFeeTx
``` [6](#0-5)

``` [7](#0-6) 

`validateExUnitsTooBigUTxO` also calls `totExUnits tx`, so sub-tx execution units are unchecked against the per-tx cap.
