Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Auto-Pause via `updateRSETHPrice()` During Any LST Price Dip — (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is a `public` function with no role check. Its internal implementation `_updateRsETHPrice()` contains an asymmetric access-control design: price increases above the threshold revert for non-managers, but price decreases above the threshold unconditionally pause `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself — with no caller check. Any unprivileged address can call `updateRSETHPrice()` at a moment when a transient LST price dip satisfies the threshold, causing a protocol-wide pause that freezes all deposits and withdrawals until an admin manually unpauses.

## Finding Description

**Entry point — no access control:**
`updateRSETHPrice()` at [1](#0-0)  carries only the `whenNotPaused` modifier. Any EOA or contract can call it at any time.

**Asymmetric caller check — the root cause:**
For price *increases* above the threshold, `_updateRsETHPrice()` checks `msg.sender` for the `MANAGER` role and reverts for non-managers: [2](#0-1) 

For price *decreases* above the threshold, there is no equivalent caller check — the pause executes unconditionally for any caller: [3](#0-2) 

**`highestRsethPrice` is the all-time high:**
`highestRsethPrice` is only ever updated upward at [4](#0-3)  meaning any transient dip below the historical peak by more than `pricePercentageLimit` satisfies the condition.

**Price is read from external oracles:**
`_getTotalEthInProtocol()` reads live asset prices from `IPriceFetcher` oracles. [5](#0-4)  A transient LST price dip that moves the computed rsETH price below the historical peak by the threshold is sufficient — no oracle manipulation required.

**Exploit path:**
1. Attacker monitors on-chain oracle prices for supported LSTs.
2. When a transient price dip causes `newRsETHPrice` to fall below `highestRsethPrice` by more than `pricePercentageLimit`, attacker calls `updateRSETHPrice()` from any address.
3. `_updateRsETHPrice()` computes the dip, enters the `isPriceDecreaseOffLimit` branch, and calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()`.
4. All deposits and withdrawals are frozen. Only an admin can unpause.
5. After admin unpauses, the attacker can repeat the attack during the next transient dip.

## Impact Explanation

All user deposits (`LRTDepositPool`) and all withdrawals (`LRTWithdrawalManager`) are frozen until an admin calls `unpause()`. This is a concrete **Medium — Temporary freezing of funds**. The freeze duration is unbounded by the contract and depends solely on admin response time. The attack is repeatable with zero cost to the attacker.

## Likelihood Explanation

- The attacker needs no capital, no special role, and no oracle manipulation — only the ability to observe oracle prices and submit a transaction at the right block.
- LST price dips are historically common (stETH depegged ~6% during the May 2022 LUNA crash; even minor Chainlink price feed updates can produce transient dips).
- With `pricePercentageLimit = 1e16` (1%), even minor transient dips qualify.
- The attack is repeatable: after each admin unpause, the attacker can trigger it again during the next qualifying dip.
- The SECURITY.md exclusion for "depegging of an external stablecoin" does not apply: stETH is a liquid staking token, not a stablecoin, and the vulnerability is in the access-control design, not in oracle behavior.

## Recommendation

Add a role check to `updateRSETHPrice()` (restrict to `MANAGER` or `OPERATOR_ROLE`), mirroring the existing check in the price-increase branch. Alternatively, apply the same `msg.sender` manager check inside the `isPriceDecreaseOffLimit` branch before executing the pause, so that only privileged callers can trigger the auto-pause. A separate privileged `triggerEmergencyPause()` function could be exposed for the auto-pause use case.

## Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity 0.8.27;

// Foundry fork test (local fork, no public mainnet)
// Setup: deploy LRTOracle, LRTDepositPool, LRTWithdrawalManager with mocks.
// Set pricePercentageLimit = 1e16 (1%).
// Set highestRsethPrice = 1.05e18 (simulate prior ATH).
// Mock oracle returns assetPrice producing newRsETHPrice = 1.03e18
// (~1.9% dip from ATH, exceeds 1% threshold).

function test_unprivilegedCallerTriggersAutoPause() public {
    // Arrange: price dip satisfies threshold
    mockOracle.setPrice(stETH, 1.03e18); // dip from ATH of 1.05e18 => ~1.9% > 1%

    // Act: call as unprivileged address
    vm.prank(address(1)); // no role
    lrtOracle.updateRSETHPrice();

    // Assert: protocol is paused
    assertTrue(lrtDepositPool.paused(),    "DepositPool should be paused");
    assertTrue(withdrawalManager.paused(), "WithdrawalManager should be paused");
    assertTrue(lrtOracle.paused(),         "LRTOracle should be paused");

    // Assert: attack is repeatable after admin unpause
    vm.prank(admin);
    lrtOracle.unpause();
    vm.prank(address(1));
    lrtOracle.updateRSETHPrice(); // triggers pause again
    assertTrue(lrtOracle.paused(), "LRTOracle paused again");
}
```
