### Title
Hardcoded Consensus Signing Private Key Used in Production Node — (`File: crates/apollo_signature_manager/src/lib.rs`)

### Summary

`create_signature_manager()` — the function called in the production node component setup — unconditionally instantiates `LocalKeyStore::new_for_testing()`, which contains a hardcoded, publicly-visible ECDSA private key. Every sequencer node therefore signs precommit votes and peer-identity challenges with the same well-known key, allowing any party who reads the source code to forge valid consensus signatures.

### Finding Description

`LocalKeyStore::new_for_testing()` hard-codes the private key:

```
0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133
``` [1](#0-0) 

`create_signature_manager()` in `lib.rs` calls `SignatureManager::new()`, which in turn calls `LocalKeyStore::new_for_testing()`: [2](#0-1) 

A TODO comment on line 39 acknowledges the production key store has not been implemented:

> `// TODO(Elin): understand how key store would look in production and better define the way the signature manager is created.` [3](#0-2) 

Despite this, `create_signature_manager()` is called unconditionally in the production node component builder: [4](#0-3) 

The `SignatureManager` is a live production component (`LocalExecutionWithRemoteEnabled` in the distributed deployment config) that handles two signing operations:

- `sign_precommit_vote(block_hash)` — signs the block hash for Tendermint precommit votes
- `sign_identification(peer_id, challenge)` — signs the peer-identity challenge for the `StarkAuthentication` protocol [5](#0-4) 

Both operations use the same hardcoded private key, whose corresponding public key is:

```
0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a
``` [6](#0-5) 

### Impact Explanation

Because the private key is embedded in the public repository and every node uses it:

1. **Forged precommit votes**: Any observer can compute `sign_precommit_vote(arbitrary_block_hash)` for any block hash and produce a signature that passes `verify_precommit_vote_signature`. This maps directly to the allowed impact: *"signature/hash logic binds the wrong signer"* — the signature does not authenticate the actual validator; it authenticates whoever holds the publicly-known key.

2. **Forged peer-identity challenges**: The `StarkAuthentication` handshake (`ChallengeAndIdentity` / `Signature`) can be answered by any party, undermining the authentication layer of the consensus P2P network. [7](#0-6) 

### Likelihood Explanation

The private key is committed in plaintext in the public repository. No special access, privilege, or network position is required. Any reader of the source code can extract it immediately. The production node binary calls `create_signature_manager()` without any configuration-driven key injection, so the exposure is unconditional.

### Recommendation

**Short term:** Gate `LocalKeyStore::new_for_testing()` behind the `testing` feature flag (already defined in `Cargo.toml`) and make `create_signature_manager()` require an explicit key path or secret-store reference. Panic or refuse to start if no production key is supplied.

**Long term:** Implement a proper `KeyStore` backend (HSM, KMS, or file-backed with OS-level permissions) that loads the private key from a secret mount (analogous to the existing `NetworkConfig.secret_key` pattern using `Sensitive<Vec<u8>>`), never from a compile-time constant. [8](#0-7) 

### Proof of Concept

```rust
// Attacker extracts the hardcoded key from the public source:
const KNOWN_PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
    "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
));

// Forge a precommit vote for an arbitrary block hash:
let target_block_hash = BlockHash(felt!("0xdeadbeef..."));
let forged_sig = ecdsa_sign(&KNOWN_PRIVATE_KEY, &build_precommit_vote_message_digest(target_block_hash));

// verify_precommit_vote_signature passes because every production node
// registered the same public key 0x125d56b1...
assert!(verify_precommit_vote_signature(target_block_hash, forged_sig, KNOWN_PUBLIC_KEY).unwrap());
```

The forged signature is indistinguishable from a legitimate validator's precommit vote, satisfying the consensus signature check and allowing an attacker to inject votes for arbitrary block hashes into the consensus protocol. [9](#0-8) [10](#0-9)

fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);

    MessageDigest(blake2s_to_felt(&message))
