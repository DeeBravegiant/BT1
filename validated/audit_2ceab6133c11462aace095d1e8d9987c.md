Audit Report

## Title
Unbounded Nested-Loop Gas Consumption in `updateRSETHPrice()` and `depositAsset()` via Unbound EigenLayer Queued Withdrawals — (`contracts/LRTOracle.sol`, `contracts/NodeDelegator.sol`)

## Summary
`updateRSETHPrice()` is a permissionless public function whose call chain traverses four nested loops: supported assets → node delegators → EigenLayer queued withdrawals → strategies per withdrawal. The protocol's `maxUncompletedWithdrawalCount` cap (hard-limited to 80) governs only protocol-initiated withdrawals and does not bound the actual count returned by EigenLayer's `getQueuedWithdrawals`. EigenLayer operator-forced undelegations can push the real count above the safe threshold, causing both `updateRSETHPrice()` and every `depositAsset`/`depositETH` call to revert out-of-gas.

## Finding Description

**Entry point:** `updateRSETHPrice()` is declared `public` with only a `whenNotPaused` guard, callable by any address. [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`:

**Loop 1** iterates over every entry in `supportedAssetList` and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [2](#0-1) 

`getTotalAssetDeposits` calls `getAssetDistributionData`:

**Loop 2** iterates over every entry in `nodeDelegatorQueue` and calls `INodeDelegator.getAssetUnstaking(asset)` for each NDC. [3](#0-2) 

`getAssetUnstaking` in `NodeDelegator.sol` fetches **all** queued withdrawals from EigenLayer via `getQueuedWithdrawals(address(this))`:

**Loop 3** iterates over every `Withdrawal` returned, and **Loop 4** iterates over every strategy inside each withdrawal. [4](#0-3) 

The combined iteration count is `supportedAssets.length × nodeDelegatorQueue.length × queuedWithdrawals.length × withdrawal.strategies.length`. Critically, `getQueuedWithdrawals` is called `N × M` times (once per asset per NDC), each returning the full withdrawal list.

**Why the existing cap is insufficient:** `maxUncompletedWithdrawalCount` is hard-capped at 80 and governs only protocol-initiated `initiateUnstaking` and `undelegate` calls. It does not bound what EigenLayer's `getQueuedWithdrawals` actually returns. EigenLayer operators can force-undelegate at any time, creating new queued withdrawals outside the protocol's counter. The protocol's own comment explicitly acknowledges this gap: [5](#0-4) 

The comment states "120 is the max number of uncompleted withdrawals that allows us to still perform update rsETH price" and notes a "buffer for theoretical operator forced undelegations." This confirms the protocol is aware that forced undelegations can push the real count above 80 and toward the 120 danger threshold.

**Deposit path also affected:** Every `depositAsset`/`depositETH` call triggers `_beforeDeposit` → `_checkIfDepositAmountExceedesCurrentLimit` → `getTotalAssetDeposits` → `getAssetDistributionData` → `getAssetUnstaking`, traversing the same nested loops. [6](#0-5) 

## Impact Explanation

**Medium — Unbounded gas consumption:** The multiplicative loop structure has no on-chain hard cap that covers all withdrawal sources. With N=10 assets, M=10 NDCs, K=15+ withdrawals per NDC (forced undelegations), and S=3 strategies each, gas cost approaches or exceeds the 30M block gas limit.

**Medium — Temporary freezing of funds:** Once the gas cost exceeds the block limit, `updateRSETHPrice()` becomes uncallable (stale price) and every `depositAsset`/`depositETH` reverts out-of-gas. The freeze persists until the queued withdrawal count is reduced below the safe threshold by completing withdrawals.

## Likelihood Explanation

EigenLayer operators can force-undelegate at any time without protocol consent, creating queued withdrawals that bypass `maxUncompletedWithdrawalCount`. With 10 NDCs each delegated to a different operator, a coordinated or independent set of forced undelegations (each NDC holding multiple strategies) can push the total above 120 without any malicious action by the protocol's own operators. No attacker capability is required — normal protocol scaling combined with a single forced-undelegation event is sufficient. The condition is repeatable until withdrawals are completed.

## Recommendation

1. **Cache `getAssetUnstaking` results** off-chain or in a separate keeper call rather than recomputing them inside `updateRSETHPrice` and `depositAsset`. Store a `totalUnstaking[asset]` value updated by privileged keepers.
2. **Decouple the deposit-limit check from the full TVL computation.** Maintain a running `totalDeposited[asset]` counter incremented/decremented on deposit and withdrawal, eliminating the need to iterate over NDCs on every deposit.
3. **Add a hard cap on the number of queued withdrawals processed** inside `getAssetUnstaking`, reverting or skipping beyond the cap, so a single call can never exceed a known gas budget.
4. **Bound `supportedAssetList` length** with an explicit maximum analogous to `maxNodeDelegatorLimit`.

## Proof of Concept

// withdrawals (from forced undelegations) with 3 strategies each.
//
// Gas cost ≈ 10 × 10 × 15 × (getQueuedWithdrawals overhead + 3 × strategy calls)
//          ≈ 10 × 10 × 15 × ~17,000 gas ≈ 25.5M gas → exceeds block limit

// Steps:
// 1. EigenLayer operators force-undelegate 10 NDCs, each producing 15 queued withdrawals.

// 3. Any user attempts a deposit:
//      lrtDepositPool.depositAsset(stETH, 1 ether, 0, "")  // reverts: out of gas

        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

            address asset = supportedAssets[assetIdx];
            // assetER is in 1e18 precision (1.0 = 1e18)

            // totalAssetAmt is in 1e18 precision (standard token decimals)

            totalETHInProtocol += totalAssetAmt.mulWad(assetER);

            unchecked {
                ++assetIdx;
            }
        }
    }

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
