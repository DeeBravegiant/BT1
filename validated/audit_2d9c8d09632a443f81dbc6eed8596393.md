Audit Report

## Title
Pusher Delegation Signature Replay in `allowPushers` Enables Forced Re-Delegation After Revocation, Causing Oracle Stale-Price DoS on Pool Swaps — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

## Summary

`CompressedOracleV1.allowPushers` constructs its EIP-191 digest over `(chainid, address(this), deadline, pusher, msg.sender)` with no nonce or used-signature bitmap. A creator who retains the original calldata can replay the identical signature after a pusher calls `revokePusher()`, re-establishing `namespaceRemapping[pusher] = creator` within the same deadline window. This permanently redirects the pusher's price updates to the creator's namespace, starving the pusher's own namespace of fresh prices and causing every `MetricOmmPool.swap()` that depends on that feed to revert with `BidIsZero`.

## Finding Description

`allowPushers` (L204–209) hashes only `(chainid, oracle, deadline, pusher, msg.sender)` — no nonce, no used-signature set:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
```

The function's own NatSpec (L186–191) acknowledges the replay risk but incorrectly treats the deadline as a complete mitigation. The deadline only blocks replay *after* it expires; within the window the same `(deadline, pusher, sig)` tuple is unconditionally accepted on every call.

`revokePusher` (L238–243) sets `namespaceRemapping[msg.sender] = address(0)`, but the creator immediately re-calls `allowPushers` with the identical arguments, restoring `namespaceRemapping[pusher] = creator`. This cycle repeats until `block.timestamp > deadline`.

The `fallback` push path (L315–316) reads `namespaceRemapping[msg.sender]` on every price push:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

While the delegation is active, every push the pusher makes is written into the creator's namespace, not the pusher's own namespace, leaving the pusher's namespace with a stale `refTime`.

## Impact Explanation

The price chain is `MetricOmmPool.swap()` → `_getBidAndAskPriceX64()` → `PriceProvider.getBidAndAskPrice()` → `CompressedOracleV1.price(feedId, pool)`, which reads the pusher's namespace. When that namespace stops receiving updates, `PriceProvider._getBidAndAskPrice()` (L198–199) returns `(0, type(uint128).max)` on the staleness check. `MetricOmmPool._getBidAndAskPriceX64()` (L806–808) then reverts with `BidIsZero` on every swap call for the entire remaining deadline window. This is broken core pool functionality / unusable swap flow, meeting the allowed impact gate.

## Likelihood Explanation

Any pusher who signed a delegation with a future deadline is at risk. The creator needs only to retain the original calldata and resubmit it after each `revokePusher()` call. `allowPushers` is permissionless — no privileged role is required. Business relationships that end before the deadline expires are a realistic and foreseeable trigger. The attack is cheap (a single transaction per revocation) and repeatable indefinitely within the deadline window.

## Recommendation

Add a per-pusher nonce included in the signed digest and increment it on each successful `allowPushers` call, invalidating any prior signature:

```solidity
mapping(address => uint256) public pusherNonces;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonces[pusher]++
    ))
);
```

This is the standard EIP-712 pattern and ensures each signature is consumed exactly once.

## Proof of Concept

1. Pusher A signs `keccak256(abi.encode(chainid, oracle, deadline=T, pusherA, creatorB))` and provides the signature to Creator B.
2. Creator B calls `allowPushers(T, [pusherA], [sig])` → `namespaceRemapping[pusherA] = creatorB`. Pusher A's pushes land in Creator B's namespace.
3. Pusher A calls `revokePusher()` → `namespaceRemapping[pusherA] = address(0)`. Pushes return to Pusher A's namespace.
4. Creator B calls `allowPushers(T, [pusherA], [sig])` again with the **identical** arguments (before `block.timestamp > T`) → `namespaceRemapping[pusherA] = creatorB` is restored.
5. Steps 3–4 repeat indefinitely until deadline T expires.
6. Pusher A's namespace receives no updates; `PriceProvider._getBidAndAskPrice()` returns `(0, type(uint128).max)`.
7. `MetricOmmPool.swap()` reverts with `BidIsZero` on every call for the remaining deadline window.
