### Title
Single Attested Participant Can Deliver Arbitrary CKD Response for `AppPublicKey` Variant, Bypassing Threshold Requirement — (`File: crates/contract/src/lib.rs`)

### Summary

`respond_ckd()` performs no cryptographic verification of the `CKDResponse` content when the request uses the `AppPublicKey` (legacy, non-publicly-verifiable) variant. A single Byzantine attested participant — strictly below the signing threshold — can call `respond_ckd()` with an arbitrary `big_y`/`big_c` pair and the contract will accept it, resolve all queued yields, and deliver the fabricated output to every waiting caller.

### Finding Description

In `respond_ckd()` the contract branches on the variant of `request.app_public_key`:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← no check at all
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

For the `AppPublicKeyPV` variant the contract enforces the pairing equation `e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)`, which cryptographically binds the response to the network's threshold key and the user's ephemeral public key. [2](#0-1) 

For the `AppPublicKey` (legacy) variant the arm is an empty block — **no pairing check, no signature check, no binding to the network key**. The contract proceeds directly to `resolve_yields_for`, serialising whatever `CKDResponse` the caller supplied and resuming every queued yield with it. [3](#0-2) 

The only gate before this branch is `assert_caller_is_attested_participant_and_protocol_active()`, which requires the caller to be a single attested participant — not a threshold quorum. [4](#0-3) 

The `AppPublicKey` variant is still accepted by `request_app_private_key` and is documented as the supported legacy path. [5](#0-4) 

### Impact Explanation

A single Byzantine participant (strictly below threshold) can:

1. Observe any pending `CKDRequest` with `AppPublicKey` variant in `pending_ckd_requests`.
2. Call `respond_ckd(request, CKDResponse { big_y: <arbitrary>, big_c: <arbitrary> })`.
3. The contract accepts the call, drains the entire fan-out queue, and delivers the fabricated `(big_y, big_c)` to every caller who submitted that request.

Because the `AppPublicKey` variant is privately verifiable only (the user needs their own `app_sk` to decrypt `big_c`), the user has no on-chain means to detect that the response is fake. The threshold safety invariant — that a valid CKD output requires cooperation of at least `t` participants — is completely bypassed for this variant. This breaks the production safety/accounting invariant of the request lifecycle: a CKD response is accepted and finalised without the required threshold authorisation.

**Allowed impact matched:** *Medium — request-lifecycle manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration.*

### Likelihood Explanation

- The `AppPublicKey` variant is the documented legacy path and is actively used by existing integrations.
- Any single attested participant (one compromised or malicious node) can execute this attack with a direct on-chain call; no collusion, no key leakage, and no network-level capability is required.
- The attack is silent: the user receives a well-formed `CKDResponse` struct and has no way to distinguish it from a legitimate threshold output without their own `app_sk` and off-chain verification.

### Recommendation

Apply the same `ckd_output_check` pairing verification to the `AppPublicKey` variant, or reject `AppPublicKey` requests in `respond_ckd` and require all new requests to use `AppPublicKeyPV`. If the legacy variant must be kept for backwards compatibility, at minimum document that its responses carry no on-chain integrity guarantee and warn callers to verify off-chain.

### Proof of Concept

```rust
// Attacker is a single attested participant.
// Victim submitted: request_app_private_key({ app_public_key: AppPublicKey(pk), ... })

let fake_response = CKDResponse {
    big_y: Bls12381G1PublicKey([0xde; 48]),  // arbitrary garbage
    big_c: Bls12381G1PublicKey([0xad; 48]),  // arbitrary garbage
};

// Single participant calls respond_ckd — no threshold quorum needed.
// Contract branch: AppPublicKey(_) => {} — no check performed.
// resolve_yields_for delivers fake_response to all waiting callers.
contract.respond_ckd(ckd_request, fake_response).expect("accepted");
```

  - **Privately verifiable** (legacy): a single G1 point, e.g. `"bls12381g1:<base58>"` or `{"AppPublicKey": "bls12381g1:<base58>"}`.
