Audit Report

## Title
Stale `rsETHPrice` During `PriceAboveDailyThreshold` Window Enables Over-Minting and Protocol Insolvency - (File: contracts/LRTOracle.sol)

## Summary
When the true rsETH price rises beyond `pricePercentageLimit`, non-manager callers of `updateRSETHPrice()` receive a `PriceAboveDailyThreshold` revert, leaving the stale, lower `rsETHPrice` in storage. Because `LRTDepositPool.getRsETHAmountToMint()` divides by this stale price, depositors receive more rsETH than the backing warrants. After a manager corrects the price, those depositors can initiate a withdrawal at the higher price and extract more ETH than they deposited, draining value from honest depositors and leaving the protocol insolvent.

## Finding Description
`LRTOracle._updateRsETHPrice()` computes `newRsETHPrice` and then, if the increase exceeds `pricePercentageLimit`, reverts for any caller without the `MANAGER` role:

```solidity
// contracts/LRTOracle.sol lines 260-266
if (isPriceIncreaseOffLimit) {
    if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
        revert PriceAboveDailyThreshold();   // entire tx reverts
    }
}
``` [1](#0-0) 

The revert unwinds the entire call, so the write at line 313 never executes and `rsETHPrice` remains at its stale, lower value:

```solidity
// contracts/LRTOracle.sol line 313
rsETHPrice = newRsETHPrice;
``` [2](#0-1) 

Contrast this with the downside path, which actively pauses the deposit pool, withdrawal manager, and oracle when the price drops too far:

```solidity
// contracts/LRTOracle.sol lines 277-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [3](#0-2) 

No equivalent pause exists for the upside case. Deposits continue unimpeded while `rsETHPrice` is stale.

`LRTDepositPool.getRsETHAmountToMint()` uses the stale value directly as the denominator with no freshness guard:

```solidity
// contracts/LRTDepositPool.sol line 520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [4](#0-3) 

A lower denominator produces a larger rsETH mint. Once the manager calls `updateRSETHPriceAsManager()` and corrects the price, the attacker initiates a withdrawal. `getExpectedAssetAmount()` now uses the corrected, higher `rsETHPrice`:

```solidity
// contracts/LRTWithdrawalManager.sol line 593
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
``` [5](#0-4) 

The `_calculatePayoutAmount` min-guard does not mitigate this: when the attacker initiates withdrawal after the price update, `expectedAssetAmount` (stored at initiation) and `currentReturn` (computed at unlock) are both calculated at the same corrected price and are therefore equal, so the minimum equals the full inflated amount:

```solidity
// contracts/LRTWithdrawalManager.sol lines 833-834
uint256 currentReturn = (request.rsETHUnstaked * rsETHPrice) / assetPrice;
return (request.expectedAssetAmount < currentReturn) ? request.expectedAssetAmount : currentReturn;
``` [6](#0-5) 

## Impact Explanation
**Critical — Protocol insolvency.**

An attacker depositing `D` ETH while `rsETHPrice = P_stale` receives `rsETH = D / P_stale`. After the manager updates to `P_actual > P_stale`, the attacker withdraws `rsETH * P_actual = D * (P_actual / P_stale) > D`. The surplus `D * (P_actual / P_stale − 1)` is drawn from other depositors' backing. With a 5 % reward event against a 1 % daily limit, a 1 000 ETH deposit yields ~50 ETH of pure profit extracted from the shared pool. Repeated or large-scale exploitation leaves the protocol with more rsETH obligations than backing ETH.

## Likelihood Explanation
**Medium.** EigenLayer restaking rewards and LST rebases can produce single-period TVL increases that exceed a conservatively set `pricePercentageLimit`. `updateRSETHPrice()` is a public, permissionless function; any on-chain observer can detect the moment it starts reverting. The manager response window (minutes to hours) is sufficient for a monitoring attacker to deposit and lock in the stale-price rsETH before the correction. No special privileges, flash loans, or oracle manipulation are required — only a standard `depositETH()` call.

## Recommendation
1. **Mirror the downside protection on the upside:** when `isPriceIncreaseOffLimit` is true, pause `LRTDepositPool` (and optionally `LRTWithdrawalManager`) until a manager resolves the price, exactly as is done for excessive price decreases at lines 277–282 of `LRTOracle.sol`.
2. **Alternatively**, compute the mint amount from live TVL rather than the cached `rsETHPrice`, so the denominator always reflects current backing even when the stored price is stale.
3. **At minimum**, emit a prominent event when `PriceAboveDailyThreshold` is hit so off-chain monitoring can trigger an immediate manager response.

## Proof of Concept
1. Deploy with `pricePercentageLimit = 1e16` (1 %) and `rsETHPrice = 1.00e18`.
2. A large EigenLayer reward batch is processed; actual TVL rises 5 %. True price is now `1.05e18`.
3. Attacker calls `updateRSETHPrice()` → reverts `PriceAboveDailyThreshold`. `rsETHPrice` stays at `1.00e18`.
4. Attacker calls `depositETH{value: 1000 ether}(0, "")`.
   - `getRsETHAmountToMint` returns `1000e18 * 1e18 / 1.00e18 = 1000e18` rsETH.
   - Fair amount at true price: `1000e18 / 1.05e18 ≈ 952.38e18` rsETH.
   - Attacker receives **~47.6 rsETH excess**.
5. Manager calls `updateRSETHPriceAsManager()`. `rsETHPrice` is updated to `1.05e18`.
6. Attacker calls `initiateWithdrawal(ETH, 1000e18, "")`.
   - `getExpectedAssetAmount` returns `1000e18 * 1.05e18 / 1e18 = 1050 ETH`.
   - `expectedAssetAmount = 1050 ETH` is stored in the withdrawal request.
7. Operator calls `unlockQueue`; `_calculatePayoutAmount` computes `currentReturn = 1000e18 * 1.05e18 / 1e18 = 1050 ETH`; `min(1050, 1050) = 1050 ETH`.
8. After the delay, attacker calls `completeWithdrawal` and receives **1050 ETH** having deposited **1000 ETH** — a 50 ETH profit drawn from the shared pool.

**Foundry fork test plan:** fork mainnet, set `pricePercentageLimit = 1e16`, manipulate oracle to return a 5 % higher TVL, call `updateRSETHPrice()` as a non-manager (expect revert), call `depositETH` with 1 000 ETH, call `updateRSETHPriceAsManager()`, call `initiateWithdrawal`, advance blocks past `withdrawalDelayBlocks`, call `unlockQueue`, call `completeWithdrawal`, assert attacker ETH balance increased by ~50 ETH and total protocol backing decreased by the same amount.
