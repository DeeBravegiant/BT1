### Title
Missing Enforcement of Key Lockout After `E_ECDSA_2P_BIT_LEAK` Enables Iterative Private Key Recovery via Parallel Sessions - (File: `src/cbmpc/protocol/ecdsa_2p.cpp`)

---

### Summary

The internal API function `sign_with_global_abort` omits the ZK proof of correctness for efficiency. When a malicious P2 sends a crafted Paillier ciphertext, P1 detects the resulting invalid signature and returns `E_ECDSA_2P_BIT_LEAK`. The library explicitly delegates the entire key-lockout and session-abort responsibility to the caller application, but provides no atomic mechanism to halt all in-flight sessions. A malicious P2 can open many parallel sessions and trigger bit-leaks across all of them before the application can respond, enabling iterative recovery of P1's private key share.

---

### Finding Description

`sign_with_global_abort` (and its batch variant) is exposed in the internal API at `include-internal/cbmpc/internal/protocol/ecdsa_2p.h` and is explicitly discussed in `SECURE_USAGE.md`. It is the only protocol in the library with this property.

In `sign_batch_impl`, when `global_abort_mode` is set, the ZK proof of correctness (`zk_ecdsa_sign_2pc_integer_commit_t`) is entirely skipped:

```cpp
if (!global_abort_mode) {
    if (rv = zk_ecdsa[i].prove(...)) return rv;
}
```

P2 sends only the raw Paillier ciphertext `c` without any proof. P1 decrypts it, reconstructs the signature, and verifies it against the public key. If verification fails, P1 returns `E_ECDSA_2P_BIT_LEAK`:

```cpp
if (rv = ecc_verification_key.verify(msgs[i], sigs[i]))
    if (global_abort_mode)
        return coinbase::error(E_ECDSA_2P_BIT_LEAK, "signature verification failed");
```

The library's documented security contract (`SECURE_USAGE.md`) states:

> *"The application using the library is responsible for ensuring that appropriate action is taken (locking the key, moving to `sign()`, etc.) if the `E_ECDSA_2P_BIT_LEAK` error is received. **This is not taken care of by the low-level library**."*

> *"It isn't secure to open hundreds of signing sessions in parallel, if it isn't possible to abort them all in case cheating is detected in any one of them."*

The library provides only per-session `mpc_abort` — there is no global, atomic abort mechanism across all sessions sharing a key. The race window is:

1. P2 opens N parallel `sign_with_global_abort` sessions against P1.
2. In each session, P2 sends a crafted ciphertext encoding a bit-guess about P1's private key share `x1`.
3. P1 returns `E_ECDSA_2P_BIT_LEAK` in each session (success/failure of verification leaks 1 bit per session).
4. Before P1's application layer can detect the first error, call `mpc_abort` on all sessions, and lock the key, the remaining sessions have already completed their final round.
5. P2 accumulates bits across sessions. After ~256 sessions (the bit-length of `x1`), P2 reconstructs P1's full private key share.

The first bad trust transition is P1 accepting and decrypting P2's unauthenticated Paillier ciphertext in `global_abort_mode` without any ZK proof binding it to P2's committed key share. The accepted bad output is the oracle response (success vs. `E_ECDSA_2P_BIT_LEAK`) that leaks a bit of `x1` per session.

---

### Impact Explanation

P2 recovers P1's private key share `x1`. Since P2 already holds `x2`, P2 can reconstruct the full private scalar `x = x1 + x2 mod q` and produce valid ECDSA signatures unilaterally, without P1's participation. This satisfies the Critical impact criterion: a single malicious peer obtains the private scalar and can forge signatures.

---

### Likelihood Explanation

`sign_with_global_abort` is accessible via the internal API (`include-internal/cbmpc/internal/protocol/ecdsa_2p.h`), which is a documented, supported API path explicitly covered in `SECURE_USAGE.md`. Any application that uses the internal API for performance reasons (the stated motivation for the function's existence) is exposed. The attack requires only that P2 open parallel sessions — a normal operational pattern for high-throughput signing services — and send crafted ciphertexts. No threshold collusion, no leaked keys, and no special hardware are required.

---

### Recommendation

1. **Provide an atomic global-abort primitive**: Add a shared cancellation token or key-level lock object that all sessions referencing the same key can observe. When any session returns `E_ECDSA_2P_BIT_LEAK`, the token is set and all other sessions referencing that key abort before completing their final round.
2. **Enforce a session-count limit per key in global-abort mode**: Reject new `sign_with_global_abort` calls once a configurable parallel-session ceiling is reached, reducing the number of bits P2 can extract in a single burst.
3. **Consider removing `sign_with_global_abort` from the internal API** and replacing it with a variant that accepts an externally-provided cancellation handle, making the lockout contract impossible to accidentally omit.

### Proof of Concept

```
// P2-controlled pseudocode
key_p2 = load_key_share_p2()
bits_learned = []

for bit_position in range(256):
    // Open a fresh session
    session = open_sign_with_global_abort_session(p1_endpoint)
    
    // Craft a Paillier ciphertext that encodes a guess for bit `bit_position` of x1
    crafted_c = craft_bit_probe_ciphertext(key_p2, bit_position, guess=0)
    session.send_round4(crafted_c)   // skip ZK proof — global_abort_mode accepts it
    
    result = session.wait_for_p1_response()
    if result == SUCCESS:
        bits_learned.append((bit_position, 0))   // guess was correct
    elif result == E_ECDSA_2P_BIT_LEAK:
        bits_learned.append((bit_position, 1))   // guess was wrong

// After ~256 sessions (before P1 can lock the key across all parallel sessions):
x1_recovered = reconstruct_from_bits(bits_learned)
full_private_key = (x1_recovered + key_p2.x2) mod q
forge_signature(full_private_key, arbitrary_message)
```

The race is won because the library's `sign_batch_impl` completes all rounds and returns `E_ECDSA_2P_BIT_LEAK` only after P1 has already decrypted and verified — the oracle response is already committed before P1's application layer can abort sibling sessions.
