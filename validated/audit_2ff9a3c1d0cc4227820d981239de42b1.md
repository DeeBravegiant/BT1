### Title
Peer-Advertised Zero Window Size Disables v3 Rate Limiting for Request-Type Protocol Messages - (File: `chia/server/rate_limits_v3.py`)

### Summary
In `rl_settings_v3_from_configure_message`, a `window_size` value of `0` in a peer's `ConfigureWindowSizes` handshake message is unconditionally interpreted as "unlimited" (`None`). There is no guard preventing an unprivileged peer from sending `window_size=0` for request-type messages (e.g., `request_blocks`) that the local node has configured with a finite window. This allows any connecting peer to disable the v3 in-flight rate limiter for those message types, enabling unlimited flooding of the victim node.

### Finding Description

In `rl_settings_v3_from_configure_message`, the sentinel value `0` is decoded as `None` (unlimited):

```python
result[message_type] = RLSettingsV3(window_size=None if window_size == 0 else window_size)
```

The only guard present rejects peers that try to assign a **non-zero** window to a message type that is locally configured as unlimited:

```python
if message_type in rate_limits_v3 and rate_limits_v3[message_type].window_size is None and window_size != 0:
    raise ProtocolError(Err.INVALID_HANDSHAKE)
```

This guard does **not** reject a peer that sends `window_size=0` for a request-type message that locally has a finite window (e.g., `request_blocks` with `window_size=2`). The parsed `None` is then stored in `peer_rl_settings_v3`.

During handshake, `rate_limit_windows` is only initialized for message types where the peer advertised a non-`None` window:

```python
for msg_type, setting in self.peer_rl_settings_v3.items():
    if setting.window_size is None:
        continue
    if msg_type not in self.rate_limit_windows:
        self.rate_limit_windows[msg_type] = RateLimitsV3(receive_window=0, in_flight=0)
```

In `_api_call`, the receive window enforcement is gated on `rl_window` being present:

```python
rl_window = self.rate_limit_windows.get(msg_type)
if rl_window is not None:
    window_size = self.peer_rl_settings_v3.get(msg_type, RLSettingsV3(window_size=None)).window_size
    if window_size is not None and rl_window.receive_window >= window_size:
        ...  # ban peer
```

If the peer advertised `window_size=0` → parsed as `None` → `rate_limit_windows` entry never created → `rl_window` is `None` → the entire receive window check is skipped. The same pattern applies in `_read_one_message`'s pre-queue check.

Additionally, v3-tracked messages (sent with a nonce via `call_api`) bypass the v2 rate limiter entirely, as confirmed by the test `test_unlimited_messages_bypass_v2_rl` and `test_v3_messages_wrt_v2_rate_limiter`. So both rate limiters are bypassed simultaneously.

### Impact Explanation

An unprivileged peer connecting to any full node can advertise `window_size=0` for `request_blocks`, `request_block`, `request_block_headers`, `request_puzzle_state`, `request_coin_state`, and other request-type messages during the v3 handshake. The victim node will accept this, disable its receive window for those types, and process an unlimited number of concurrent requests from that peer. This enables a single attacker to exhaust the victim node's CPU and I/O resources, causing a long-lived inability to process valid blocks, respond to sync requests, or serve other honest peers — matching the High impact criterion.

### Likelihood Explanation

Any peer that negotiates `RATE_LIMITS_V3` capability can exploit this. The `ConfigureWindowSizes` message is sent during the normal handshake and is fully attacker-controlled. No special privileges, keys, or prior state are required. The only prerequisite is establishing a TCP connection to the target node.

### Recommendation

In `rl_settings_v3_from_configure_message`, reject peers that send `window_size=0` for message types that the local node has configured with a finite (non-`None`) window:

```python
if message_type in rate_limits_v3:
    local_window = rate_limits_v3[message_type].window_size
    if local_window is None and window_size != 0:
        raise ProtocolError(Err.INVALID_HANDSHAKE)
    if local_window is not None and window_size == 0:
        raise ProtocolError(Err.INVALID_HANDSHAKE)  # NEW: prevent unlimited bypass
result[message_type] = RLSettingsV3(window_size=None if window_size == 0 else window_size)
```

### Proof of Concept

1. Attacker connects to a full node that advertises `RATE_LIMITS_V3`.
2. During handshake, attacker sends a `ConfigureWindowSizes` message with `request_blocks: uint16(0)` (and valid non-zero values for all other types to pass the length check).
3. Victim calls `rl_settings_v3_from_configure_message` → stores `RLSettingsV3(window_size=None)` for `request_blocks` → skips `rate_limit_windows` initialization for that type.
4. Attacker sends unlimited tracked `request_blocks` messages (with nonce, bypassing v2 rate limiter).
5. Victim's `_api_call` finds `rl_window = self.rate_limit_windows.get(request_blocks)` → `None` → skips all receive window enforcement.
6. Victim processes unlimited concurrent block requests, exhausting resources. [1](#0-0) [2](#0-1) [3](#0-2)

```python
def rl_settings_v3_from_configure_message(msg: ConfigureWindowSizes) -> dict[ProtocolMessageTypes, RLSettingsV3]:
    """
    Parse a `ConfigureWindowSizes` message rate limit settings. The peer
    controls what is included so only the entries present in the message are
    considered. Unknown entries are silently skipped while invalid ones result
    in a `ProtocolError` exception.
    """
    result: dict[ProtocolMessageTypes, RLSettingsV3] = {}
    for msg_type_val, window_size in msg.settings:
        try:
            message_type = ProtocolMessageTypes(msg_type_val)
        except ValueError:
            continue
        # Don't allow peers to alter our unlimited (typically response type)
        # protocol messages windows.
        if message_type in rate_limits_v3 and rate_limits_v3[message_type].window_size is None and window_size != 0:
            raise ProtocolError(Err.INVALID_HANDSHAKE)
        result[message_type] = RLSettingsV3(window_size=None if window_size == 0 else window_size)
    return result
```
