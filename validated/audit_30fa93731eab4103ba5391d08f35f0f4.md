### Title
Zip-Truncation in `updateUTxOTxWitness` Allows Spending Unowned UTxO Inputs Without Witnesses — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

---

### Summary

The Byron UTxO witness validation in `updateUTxOTxWitness` uses Haskell's `zip` to pair input addresses with witnesses. Because `zip` silently truncates to the shorter list, a transaction carrying fewer witnesses than inputs causes tail inputs to receive **no witness validation at all**. An unprivileged attacker can spend UTxO outputs they do not own by submitting a transaction with N inputs but only M < N witnesses, where the M witnesses are valid for M inputs they legitimately control.

---

### Finding Description

In `updateUTxOTxWitness`:

```haskell
addresses <-
  mapM (`UTxO.lookupAddress` utxo) (NE.toList $ txInputs tx)
    `wrapError` UTxOValidationUTxOError

mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))   -- line 389
  `wrapError` UTxOValidationTxValidationError
``` [1](#0-0) 

`addresses` has length equal to the number of `TxIn`s (all inputs). `V.toList witness` has length equal to the number of witnesses supplied by the submitter. Haskell's `zip` truncates to the shorter list — if `length witness < length (txInputs tx)`, the excess inputs are silently dropped from the validation loop.

`TxWitness` is defined as `Vector TxInWitness` with no length constraint: [2](#0-1) 

Neither `validateTx` nor `validateTxAux` checks that `length witness == length (txInputs tx)`:

- `validateTx` (lines 241–260): checks inputs exist in UTxO, attributes, network magic. No witness count check. [3](#0-2) 
- `validateTxAux` (lines 183–234): checks fee, size, balance. No witness count check. [4](#0-3) 

There is no guard anywhere in the production code that enforces the invariant `length witness == length (txInputs tx)`. A grep for `length.*txInputs` or `V.length.*witness` in production code returns zero results. [5](#0-4) 

---

### Impact Explanation

**Critical.** An attacker can steal ADA from any Byron-era UTxO output:

1. Construct a transaction with 2 inputs: one the attacker owns (`inputA`) and one the victim owns (`inputB`).
2. Provide only 1 witness — a valid `VKWitness` for `inputA`.
3. `zip [addrA, addrB] [witA]` produces `[(addrA, witA)]`.
4. `mapM_ (uncurry $ validateWitness …)` validates only `(addrA, witA)` — passes.
5. `addrB` is never passed to `validateWitness`; the victim's input is accepted without any authorization check.
6. `updateUTxOTx` removes both inputs from the UTxO and adds the attacker's outputs.

This is a direct, unauthorized ledger state transition spending funds the attacker does not own. All honest nodes running this code would accept the transaction identically, causing no divergence but a permanent invalid ledger state (stolen funds). Recovery would require a hard fork.

---

### Likelihood Explanation

The exploit requires only:
- Knowledge of any Byron UTxO output (public on-chain).
- The ability to submit a transaction (fully unprivileged).
- Constructing a CBOR-encoded `ATxAux` with a witness vector shorter than the input list — trivially achievable with any CBOR library.

No key compromise, governance majority, or privileged access is required.

---

### Recommendation

Insert an explicit length-equality guard at the start of the witness validation block in `updateUTxOTxWitness`, before the `zip`:

```haskell
let nInputs   = length (NE.toList $ txInputs tx)
    nWitnesses = V.length witness
(nInputs == nWitnesses)
  `orThrowError` UTxOValidationTxValidationError
      (TxValidationWitnessCountMismatch nInputs nWitnesses)
```

Add a corresponding `TxValidationWitnessCountMismatch Natural Natural` constructor to `TxValidationError`. This makes the invariant explicit and locally testable.

---

-- Pseudocode unit test

    witness = V.fromList [validWitnessFor input1]  -- only 1 witness for 2 inputs

result <- runExceptT $ runReaderT
            (updateUTxOTxWitness env utxo txAux)
            TxValidation
-- EXPECTED: Left (some witness error)
-- ACTUAL (current code): Right updatedUTxO  -- input2 spent without authorization
assert (isLeft result)
```

The `zip` at line 389 produces only one pair, `mapM_` validates only that pair, and the function returns `Right` — silently accepting the theft of `input2`. [6](#0-5)

validateTxAux env utxo (ATxAux (Annotated tx _) _ txBytes) = do
  -- Check that the size of the transaction is less than the maximum
  txSize
    <= maxTxSize
    `orThrowError` TxValidationTxTooLarge txSize maxTxSize

  -- Calculate the minimum fee from the 'TxFeePolicy'
  minFee <-
    if isRedeemUTxO inputUTxO
      then pure $ mkKnownLovelace @0
      else calculateMinimumFee feePolicy

  -- Calculate the balance of the output 'UTxO'
  balanceOut <-
    balance (txOutputUTxO tx)
      `wrapError` TxValidationLovelaceError "Output Balance"

  -- Calculate the balance of the restricted input 'UTxO'
  balanceIn <-
    balance inputUTxO
      `wrapError` TxValidationLovelaceError "Input Balance"

  -- Calculate the 'fee' as the difference of the balances
  fee <-
    subLovelace balanceIn balanceOut
      `wrapError` TxValidationLovelaceError "Fee"

  -- Check that the fee is greater than the minimum
  (minFee <= fee) `orThrowError` TxValidationFeeTooSmall tx minFee fee
  where
    Environment {protocolParameters} = env

    maxTxSize = ppMaxTxSize protocolParameters
    feePolicy = ppTxFeePolicy protocolParameters

    txSize :: Natural

    inputUTxO = S.fromList (NE.toList (txInputs tx)) <| utxo

    calculateMinimumFee ::
      MonadError TxValidationError m => TxFeePolicy -> m Lovelace
    calculateMinimumFee = \case
      TxFeePolicyTxSizeLinear txSizeLinear ->
        calculateTxSizeLinear txSizeLinear txSize

updateUTxOTxWitness env utxo ta = do
  whenTxValidation $ do
    -- Get the signing addresses for each transaction input from the 'UTxO'
    addresses <-
      mapM (`UTxO.lookupAddress` utxo) (NE.toList $ txInputs tx)
        `wrapError` UTxOValidationUTxOError

    -- Validate witnesses and their signing addresses
    mapM_
      (uncurry $ validateWitness pmi sigData)
      (zip addresses (V.toList witness))
      `wrapError` UTxOValidationTxValidationError

    -- Validate the tx including witnesses
    validateTxAux env utxo ta
      `wrapError` UTxOValidationTxValidationError

  -- Update 'UTxO' ignoring witnesses
  updateUTxOTx env utxo aTx
  where
    Environment {protocolMagic} = env
    pmi = getAProtocolMagicId protocolMagic

    aTx@(Annotated tx _) = aTaTx ta
    witness = taWitness ta
    sigData = recoverSigData aTx
