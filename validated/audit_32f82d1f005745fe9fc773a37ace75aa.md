Audit Report

## Title
Stale `ethValueInWithdrawal` in `LRTConverter` causes `rsETHPrice` overstatement when LST price falls during unstaking, shortchanging subsequent depositors — (`contracts/LRTConverter.sol` / `contracts/LRTDepositPool.sol`)

## Summary
When an LST is transferred to `LRTConverter` via `transferAssetFromDepositPool`, its ETH value is snapshotted into `ethValueInWithdrawal` at the transfer-time oracle price. If the LST price falls before unstaking completes, `ethValueInWithdrawal` overstates the converter's contribution to protocol TVL. This inflates `rsETHPrice`, causing subsequent depositors to receive fewer rsETH than their collateral's fair ETH value. No re-pricing mechanism exists during the unstaking window.

## Finding Description
`transferAssetFromDepositPool` records `ethValueInWithdrawal += (_amount * lrtOracle.getAssetPrice(_asset)) / 1e18` at transfer time (LRTConverter.sol L140). For LST assets, `getAssetDistributionData` returns `assetLyingInConverter = 0` (LRTDepositPool.sol L460), delegating the converter's value entirely to `getETHDistributionData()`, which returns the stored (stale) `ethValueInWithdrawal` as `ethLyingInConverter` (LRTDepositPool.sol L498-499). `_getTotalEthInProtocol()` sums `getTotalAssetDeposits(asset) * getAssetPrice(asset)` for all assets (LRTOracle.sol L336-343). For `ETH_TOKEN`, `getAssetPrice = 1e18`, so `ethValueInWithdrawal` enters the sum at face value. For the LST, the converter portion contributes 0. The result is:

```
totalETH = (LST_not_in_converter * P_LST_current) + ETH + ethValueInWithdrawal
```

where `ethValueInWithdrawal = LST_in_converter * P_LST_at_transfer / 1e18`. If `P_LST_at_transfer > P_LST_current`, the overstatement is `LST_in_converter * (P_LST_at_transfer - P_LST_current) / 1e18`. This inflates `rsETHPrice = totalETH / rsethSupply` (LRTOracle.sol L250), and `getRsETHAmountToMint = amount * assetPrice / rsETHPrice` (LRTDepositPool.sol L520) yields fewer rsETH than fair value. `ethValueInWithdrawal` is only reduced when ETH is physically returned via `_sendEthToDepositPool` (LRTConverter.sol L255-258); there is no mechanism to re-price it against the live oracle during the unstaking window. As a secondary effect, the inflated `rsETHPrice` prevents the downside-protection pause (LRTOracle.sol L270-281) from triggering even when the LST has genuinely lost value.

## Impact Explanation
New depositors of any supported asset receive fewer rsETH than the fair ETH value of their deposit during the window when an LST is in the converter and its price has declined. Existing rsETH holders benefit at the expense of new depositors. The protocol does not lose funds overall, but the contract fails to deliver the promised exchange rate to new depositors. This matches the allowed Low impact: "Contract fails to deliver promised returns, but doesn't lose value."

## Likelihood Explanation
The operator regularly transfers LSTs to `LRTConverter` for unstaking as part of the intended operational flow — this is not an attack. Lido stETH unstaking takes multiple days. LST prices fluctuate in normal market conditions. `updateRSETHPrice()` is public and permissionless, callable by anyone at any time. The combination requires no adversarial action: normal protocol operation + market price movement + public function call + new deposit is sufficient to trigger the impact. The scenario is routine and repeatable.

## Recommendation
Instead of storing the ETH value at transfer time, store the raw LST token amount in the converter and compute its current ETH value dynamically in `getETHDistributionData()` using the live oracle price. Alternatively, add a re-pricing step in `_updateRsETHPrice()` that recomputes `ethValueInWithdrawal` from the current oracle prices of all LSTs held in the converter before computing `totalETHInProtocol`.
