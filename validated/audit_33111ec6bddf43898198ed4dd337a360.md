1. P1 (to-be-removed) calls `vote_remove_launcher_hash` → stored in `launcher_votes.vote_by_account` as P1's vote. `count_votes` = 1.
2. Resharing completes removing P1. `threshold_parameters.participants()` now = {P2, P3, P4}, `total_participants` = 3.
3. P1 can no longer vote (blocked by `AuthenticatedParticipantId::new(threshold_parameters.participants())`).
4. Before `CLEAN_TEE_STATUS` runs: P2 and P3 call `vote_remove_launcher_hash`.
5. `count_votes(Remove(launcher_hash))` = 3 (P1 stale + P2 + P3).
6. `votes (3) >= total_participants (3)` → **TRUE** → launcher hash removed.

P4's vote was never cast. The unanimity requirement is bypassed with only 2 of 3 current participants.

### Title
Stale Pre-Resharing Vote Bypasses Unanimity in `vote_remove_launcher_hash` — (`crates/contract/src/lib.rs`, `crates/contract/src/tee/proposal.rs`)

### Summary
A participant scheduled for removal via resharing can pre-cast a `Remove(launcher_hash)` vote before resharing completes. After resharing, the stale vote persists in `launcher_votes.vote_by_account` until `CLEAN_TEE_STATUS` executes. During this window, N-2 of the N-1 remaining participants can vote Remove, causing `count_votes` to return N-1 (stale + fresh votes) while `total_participants` is also N-1, satisfying the unanimity check without the final current participant's consent.

### Finding Description
`LauncherHashVotes::count_votes` iterates all entries in `vote_by_account` without filtering by current participants: [2](#0-1) 

`vote_remove_launcher_hash` compares this raw count against `threshold_parameters.participants().len()`, which reflects the post-resharing participant set: [4](#0-3) 

`clean_non_participant_votes` (which would strip the stale vote) is only called via a deferred promise after resharing, not atomically. The `get_remaining_votes` method exists and correctly filters, but it is not invoked at vote-counting time: [3](#0-2) 

### Impact Explanation
A launcher image hash is removed without true unanimity of current participants. This invalidates TEE attestations for all nodes running that launcher, preventing them from being recognized as valid participants. If enough nodes are affected, the MPC network's signing set drops below threshold, freezing threshold signature production. This breaks the unanimity invariant that protects against unilateral launcher invalidation.

### Likelihood Explanation
Requires: (1) controlling one participant that is being removed via resharing, and (2) N-2 of the remaining N-1 participants also voting Remove in the window before `CLEAN_TEE_STATUS` executes. The window is bounded by NEAR block time but is non-zero. The removed participant's pre-vote is a deliberate action; the remaining votes could come from honest participants who genuinely want removal, making the attacker's contribution purely the timing exploit of their stale vote.

### Recommendation
In `vote_remove_launcher_hash`, filter the vote count to only include votes from current participants before comparing against `total_participants`. Specifically, call `get_remaining_votes(threshold_parameters.participants())` on `self.tee_state.launcher_votes` before invoking `count_votes`, or inline the filter inside `count_votes` by passing the current participant set. This eliminates the dependency on the deferred cleanup promise for correctness of the unanimity check.

### Proof of Concept
1. Initialize contract with N=4 participants {P1, P2, P3, P4}.
2. P1 calls `vote_remove_launcher_hash(H)` → 1 vote stored.
3. Resharing removes P1; new set = {P2, P3, P4}, `total_participants` = 3.
4. Before `CLEAN_TEE_STATUS` promise executes: P2 and P3 call `vote_remove_launcher_hash(H)`.
5. `count_votes` = 3 (P1 stale + P2 + P3); `total_participants` = 3; check passes.
6. Assert launcher hash H is removed — P4 never voted.
7. After `CLEAN_TEE_STATUS` executes and stale votes are cleaned: repeat steps 4–5 with only P2 and P3 voting → `count_votes` = 2 < 3 → hash not removed, confirming the cleanup fixes the issue.

    /// Returns a new `LauncherHashVotes` containing only votes from current participants.
    pub fn get_remaining_votes(&self, participants: &Participants) -> Self {

            .vote_by_account
            .iter()
            .filter(|(participant_id, _)| {
                participants.is_participant_given_participant_id(&participant_id.get())

            .map(|(participant_id, vote)| (participant_id.clone(), vote.clone()))
