Haskell's `zip` truncates to the shorter list. `addresses` has length N (one per `txInputs`), `V.toList witness` has length M. ### Title
Witness-count truncation via `zip` allows spending unwitnessed UTxO inputs — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

### Summary
`updateUTxOTxWitness` uses Haskell's `zip` to pair UTxO input addresses with transaction witnesses. Because `zip` silently truncates to the shorter list, a transaction with N inputs but only M < N witnesses passes witness validation for only the first M inputs. The remaining N-M inputs are consumed without any cryptographic authorization check.

### Finding Description
In `updateUTxOTxWitness` [1](#0-0) , the witness validation loop is:

```haskell
mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))
```

`addresses` is built by mapping `UTxO.lookupAddress` over all `txInputs` — it has exactly N elements. [2](#0-1) 

`V.toList witness` is the raw witness vector from `aTaWitness`, which is attacker-controlled and can have any length M. [3](#0-2) 

Haskell's `zip` produces `min(N, M)` pairs. When M < N, inputs at indices M..N-1 are never passed to `validateWitness` and are consumed without any signature or key check. No length-equality guard exists before or after the zip in this function, in `validateTxAux`, or in `validateTx`. [4](#0-3) 

### Impact Explanation
An unprivileged attacker who controls one UTxO entry can craft an `ATxAux` that also spends arbitrary victim UTxO entries by:
1. Listing the victim's `TxIn`s alongside their own in `txInputs`.
2. Providing only 1 witness (valid for their own input).
3. Submitting the transaction — `zip` drops all victim inputs from witness validation, and the transaction is accepted.

This constitutes **direct, unauthorized destruction/transfer of ADA** from victim addresses — a Critical impact. Because all honest nodes run the same code, they all accept the transaction deterministically; there is no divergence, but the ledger state is permanently invalid (funds stolen without authorization).

### Likelihood Explanation
The exploit requires only the ability to submit a transaction (fully unprivileged). The attacker needs to know a victim's `TxIn` reference (observable on-chain) and their own valid key for one input. No special access, governance role, or key compromise is needed. The path is direct and locally testable.

### Recommendation
Before the `zip`, assert that the witness count equals the input count:

```haskell
let inputCount   = length addresses
    witnessCount = V.length witness
when (witnessCount /= inputCount) $
  throwError $ UTxOValidationTxValidationError
             $ TxValidationWitnessWrongNumber inputCount witnessCount

mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))
```

A new `TxValidationWitnessWrongNumber` constructor (or reuse of an existing error) should be added to `TxValidationError`. [5](#0-4) 

### Proof of Concept
Construct an `ATxAux` where `txInputs` contains 3 entries (one owned by the attacker, two owned by victims) and `aTaWitness` is a vector of length 1 (valid witness for the attacker's input only). Call `updateUTxOTxWitness` with a UTxO containing all three inputs. The function returns `Right utxo'` — all three inputs are consumed, the two victim inputs are spent without any witness validation, and the attacker receives the combined value.

validateTxAux ::
  MonadError TxValidationError m =>
  Environment ->
  UTxO ->
  ATxAux ByteString ->
  m ()
validateTxAux env utxo (ATxAux (Annotated tx _) _ txBytes) = do
  -- Check that the size of the transaction is less than the maximum
  txSize
    <= maxTxSize
    `orThrowError` TxValidationTxTooLarge txSize maxTxSize

  -- Calculate the minimum fee from the 'TxFeePolicy'
  minFee <-
    if isRedeemUTxO inputUTxO
      then pure $ mkKnownLovelace @0
      else calculateMinimumFee feePolicy

  -- Calculate the balance of the output 'UTxO'
  balanceOut <-
    balance (txOutputUTxO tx)
      `wrapError` TxValidationLovelaceError "Output Balance"

  -- Calculate the balance of the restricted input 'UTxO'
  balanceIn <-
    balance inputUTxO
      `wrapError` TxValidationLovelaceError "Input Balance"

  -- Calculate the 'fee' as the difference of the balances
  fee <-
    subLovelace balanceIn balanceOut
      `wrapError` TxValidationLovelaceError "Fee"

  -- Check that the fee is greater than the minimum
  (minFee <= fee) `orThrowError` TxValidationFeeTooSmall tx minFee fee
  where
    Environment {protocolParameters} = env

    maxTxSize = ppMaxTxSize protocolParameters
    feePolicy = ppTxFeePolicy protocolParameters

    txSize :: Natural
    txSize = fromIntegral $ BS.length txBytes

    inputUTxO = S.fromList (NE.toList (txInputs tx)) <| utxo

    calculateMinimumFee ::
      MonadError TxValidationError m => TxFeePolicy -> m Lovelace
    calculateMinimumFee = \case
      TxFeePolicyTxSizeLinear txSizeLinear ->
        calculateTxSizeLinear txSizeLinear txSize
          `wrapError` TxValidationLovelaceError "Minimum Fee"
