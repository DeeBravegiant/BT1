### Title
Validator Accepts Proposer-Supplied `builder` Address Without Checking Against Locally-Configured `builder_address`, Corrupting Sequencer Address in Block Hash and Fee Collection - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates many fields of an incoming `ProposalInit` against locally-configured values, but never checks `init.builder` against `self.config.static_config.builder_address`. The proposer-supplied `builder` is passed directly to `convert_to_sn_api_block_info`, which uses it as `sequencer_address` in `BlockInfo`. This address is then used as the fee recipient during transaction execution and is committed into the block hash via `PartialBlockHashComponents`. A malicious proposer can set `builder` to any address; the validator will accept the proposal because it uses the same proposer-supplied value for its own commitment computation, so the commitments match.

### Finding Description

`ContextStaticConfig` holds a fixed `builder_address` field, set at node initialization: [1](#0-0) 

When **building** a proposal, the node correctly uses this configured value: [2](#0-1) [3](#0-2) 

When **validating** a received proposal, `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, L1 gas prices, and `fee_proposal_fri` — but **never** checks `init.builder`: [4](#0-3) [5](#0-4) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [6](#0-5) 

This `BlockInfo` (with the proposer-supplied `builder` as `sequencer_address`) is sent to the batcher for block execution. The batcher then builds `PartialBlockHashComponents` from this `BlockInfo`: [7](#0-6) 

The `sequencer` field is included in the final block hash: [8](#0-7) 

Because the validator uses the proposer-supplied `builder` for its own commitment computation, the `ProposalFin` comparison passes regardless of what address the proposer chose: [9](#0-8) 

The wrong `builder` is also persisted to the block header via `update_state_sync_with_new_block`: [10](#0-9) 

### Impact Explanation

**Fee redirection (Critical — incorrect fee/balance effect with economic impact):** In Starknet, `BlockInfo.sequencer_address` is the fee recipient. A malicious proposer sets `init.builder` to their own address; the validator executes the block with that address as the fee recipient, redirecting all transaction fees from the legitimate sequencer to the attacker.

**Wrong block hash commitment (High — wrong state/block commitment):** The `sequencer_address` is hashed into `PartialBlockHashComponents` and thus into the `ProposalCommitment`. The committed block hash encodes the wrong sequencer address, producing an incorrect authoritative block hash that is stored on-chain and propagated to state sync.

**Wrong block header stored (High — wrong authoritative RPC view):** The `sequencer` field in `BlockHeaderWithoutHash` is set from `init.builder`, so `starknet_getBlockWithTxHashes` and related RPC calls return the attacker-controlled address as the block's sequencer.

### Likelihood Explanation

Requires a malicious consensus proposer — a validator-set participant who wins the proposer slot for a given round. In the current production deployment with `assume_no_malicious_validators: true`, this is treated as low probability. However, the invariant is structurally broken: the validator has a locally-configured `builder_address` that it never enforces on incoming proposals, directly mirroring the `Pool.buyNFT()` pattern where a fixed initialization-time value is bypassed by an unchecked parameter.

### Recommendation

Add a check in `is_proposal_init_valid` (or in `ProposalInitValidation`) that rejects any `ProposalInit` whose `builder` field does not equal the locally-configured `builder_address`:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

Add `builder_address: ContractAddress` to `ProposalInitValidation` and populate it from `self.config.static_config.builder_address` at the call sites in `sequencer_consensus_context.rs`.

### Proof of Concept

1. Malicious proposer wins round `R` at height `H`.
2. Proposer constructs `ProposalInit` with `builder = ATTACKER_ADDRESS` (instead of the legitimate `0x1176a1bd84444c89232ec27754698e5d2e7e1a7f1539f12027f28b23ec9f3d8`).
3. Validator receives the `ProposalInit`. `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, L1 prices, `fee_proposal_fri` — all pass. `builder` is never checked.
4. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `BlockInfo { sequencer_address: ATTACKER_ADDRESS, ... }` is sent to the batcher.
5. Batcher executes all transactions with `ATTACKER_ADDRESS` as fee recipient; all fees go to the attacker.
6. Batcher computes `PartialBlockHashComponents { sequencer: ATTACKER_ADDRESS, ... }` → `ProposalCommitment` encodes the wrong sequencer.
7. Validator's `built_block == received_fin.proposal_commitment` check passes (both sides used `ATTACKER_ADDRESS`).
8. Block is committed with wrong sequencer address in block hash and block header.

```rust
#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
```
