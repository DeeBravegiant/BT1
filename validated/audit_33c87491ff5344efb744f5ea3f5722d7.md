### Title
Unconstrained `fee_proposal_fri` During Startup/Transition Poisons `fee_proposals_window` and Corrupts Future `next_l2_gas_price` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

When `fee_actual` is `None` (startup: first `window_size` blocks, or during the V0_14_3 protocol transition when any block in the sliding window carries `fee_proposal_fri = None`), `is_proposal_init_valid` skips all bounds enforcement on `fee_proposal_fri`. The proposer's own code documents the expected fallback ("freeze at `l2_gas_price`; the validator derives the same fallback so both sides agree"), but the validator never enforces this. A proposer can inject an arbitrary `fee_proposal_fri` value that is committed into the block, seeds `fee_proposals_window`, and — once the window fills — corrupts `fee_actual` and every subsequent `next_l2_gas_price`.

---

### Finding Description

**Invariant stated in code but not enforced:**

`compute_proposer_fee_proposal` documents:

> "When `fee_actual` is `None` (window incomplete), freeze at `l2_gas_price`; the validator derives the same fallback so both sides agree." [1](#0-0) 

The proposer obeys this: when `fee_actual` is `None` it returns `self.l2_gas_price`. But `is_proposal_init_valid` uses an `if let` pattern that silently skips the check when `fee_actual` is `None`:

```rust
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // bounds check — never reached when fee_actual is None
}
``` [2](#0-1) 

`fee_actual` is `None` whenever `compute_fee_actual` returns `None`: [3](#0-2) 

This happens in two production-reachable conditions:
1. `height < window_size` — the first `window_size` blocks after genesis or restart.
2. Any block in `[height - window_size, height)` has `fee_proposal_fri = None` — the entire V0_14_3 transition window.

**Attack chain:**

1. Malicious proposer sends `ProposalInit` with `fee_proposal_fri = u128::MAX` during the startup window.
2. `is_proposal_init_valid` reaches the `if let` guard; `fee_actual` is `None`; the pattern does not match; no error is returned.
3. `initiate_validation` proceeds; the batcher executes the block normally.
4. `validate_proposal` computes `proposal_commitment_from(batcher_partial_block_hash, init.fee_proposal_fri)` = `Poseidon(partial, u128::MAX)` and compares it to the proposer's `ProposalFin.proposal_commitment` — they match, so the proposal is accepted. [4](#0-3) 

5. `finalize_decision` calls `record_fee_proposal(height, Some(u128::MAX))`, inserting the poisoned value into `fee_proposals_window`. [5](#0-4) 

6. After `window_size` blocks, `compute_fee_actual` returns the median of the window — dominated by the injected extreme values.
7. `calculate_next_l2_gas_price_for_fin` uses `fee_actual` as a floor: `effective_min = max(config_min, fee_actual)`. [6](#0-5) 

8. `self.l2_gas_price` is updated to the inflated value; the next `ProposalInit` must carry this inflated `l2_gas_price_fri` (enforced by the exact-equality check in `is_proposal_init_valid`). [7](#0-6) 

9. All subsequent blocks carry the inflated `l2_gas_price_fri`, causing every transaction's fee to be computed against the wrong base price.

---

### Impact Explanation

**Impact: Critical — Incorrect fee, gas, bouncer, resource accounting with economic impact.**

`l2_gas_price_fri` is the base fee used by the blockifier for all L2 transaction fee calculations. An artificially inflated value makes every transaction prohibitively expensive (or, with a near-zero injection, artificially cheap). The corruption is persistent: once the window is seeded with extreme values, `fee_actual` remains wrong for `window_size` blocks, and `l2_gas_price` propagates the error forward through the EIP-1559-style adjustment loop. The `parent_proposal_commitment` in the cende blob is also corrupted because it is computed as `proposal_commitment_from(partial_block_hash, parent_fee_proposal)` where `parent_fee_proposal` is read from the poisoned window. [8](#0-7) 

---

### Likelihood Explanation

The startup window is a **recurring, predictable condition**: it occurs at genesis, after every restart, and during the V0_14_3 protocol upgrade transition. Any validator that is selected as proposer during these `window_size` blocks can trigger the issue. In a BFT system where honest validators do not enforce the fallback value, a single malicious proposer round during startup is sufficient to inject a poisoned entry. The V0_14_3 transition creates an additional window of `window_size` blocks where the condition is reachable on a live network.

---

### Recommendation

In `is_proposal_init_valid`, when `fee_actual` is `None`, enforce that `init_proposed.fee_proposal_fri == proposal_init_validation.l2_gas_price_fri` (the expected fallback). Replace the silent skip with an explicit equality check:

```rust
match (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri) {
    (None, Some(fee_proposal)) => {
        // Window not yet full: proposer must freeze at l2_gas_price.
        if fee_proposal != proposal_init_validation.l2_gas_price_fri {
            return Err(InvalidProposalInit(..., "fee_proposal must equal l2_gas_price during startup"));
        }
    }
    (Some(fee_actual), Some(fee_proposal)) => {
        // Normal bounds check.
        let (lower, upper) = fee_proposal_bounds(fee_actual, margin_ppt);
        if fee_proposal.0 < lower || fee_proposal.0 > upper { ... }
    }
    _ => {}
}
```

This closes the gap between the documented invariant and the enforced invariant.

### Proof of Concept

```
Precondition: sequencer at height 0 (startup), window_size = 10.

1. Malicious proposer sends ProposalInit {
       height: 0,
       fee_proposal_fri: Some(GasPrice(u128::MAX)),
       l2_gas_price_fri: <honest value>,
       ...
   }

2. is_proposal_init_valid:
   - fee_actual = compute_fee_actual(&window, height=0, window_size=10)
     → None (height 0 < window_size 10)
   - if let (Some(fee_actual), Some(fee_proposal)) = (None, Some(u128::MAX))
     → pattern does not match → no error

3. Proposal accepted. record_fee_proposal(0, Some(u128::MAX)) called.

4. Repeat for heights 1..9 (proposer controls or colludes for majority of startup window).

5. At height 10: compute_fee_actual returns median ≈ u128::MAX.

6. calculate_next_l2_gas_price_for_fin:
   effective_min = max(config_min, u128::MAX) = u128::MAX
   → next_l2_gas_price = u128::MAX

7. All blocks from height 10 onward carry l2_gas_price_fri = u128::MAX.
   Every transaction fee calculation uses u128::MAX as the base price.
```

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
