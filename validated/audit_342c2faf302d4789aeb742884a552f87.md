### Title
Shared-list aliasing in `_generate_unsigned_transaction` corrupts clawback memos for all outputs beyond the first — (`chia/wallet/wallet.py`)

### Summary
When `memos=None` and `len(amounts) > 1` with a CLAWBACK decorator active, `[[]] * len(amounts)` creates aliased references to a single list. `ClawbackPuzzleDecorator.decorate_memos` mutates that list in-place via `insert(0, target_puzzle_hash)`, so every subsequent output's memo list already contains prior outputs' puzzle hashes. All `CreateCoin` conditions end up with the same corrupted memo list, breaking wallet sync's ability to identify clawback recipients.

### Finding Description
**Root cause:** `chia/wallet/wallet.py` line 296:
```python
memos = [[]] * len(amounts)   # all elements alias the same list object
```
**Mutation site:** `chia/wallet/puzzles/clawback/puzzle_decorator.py` line 67:
```python
memos.insert(0, target_puzzle_hash)   # mutates the shared object
```
For N outputs, after the list comprehension all N entries of `decorated_memos` reference the same list containing all N puzzle hashes in reverse insertion order. The REMARK metadata path is also skipped for multi-output spends (line 48 of `clawback/puzzle_decorator.py`), leaving the corrupted memo as the sole sync signal.

### Impact Explanation
The sender loses the practical ability to claw back output 0 (and potentially others) because the wallet never surfaces the clawback UI for those coins — they are attributed to the wrong recipient via the corrupted hint. Recipient 0 may also fail to see the coin. This is a direct financial security impact on clawback-enabled wallets sending to multiple recipients.

### Likelihood Explanation
Any user with a CLAWBACK decorator configured (set via `puzzle_decorator_override` or the wallet's decorator manager config) who sends to two or more recipients in a single transaction without explicitly supplying `memos` triggers this. This is a normal, documented use case.

### Recommendation
Replace the aliasing construction with a list comprehension that creates independent lists:
```python
memos = [[] for _ in range(len(amounts))]
```
Additionally, `ClawbackPuzzleDecorator.decorate_memos` should avoid mutating the caller's list in-place; it should return a new list:
```python
return self.decorate(inner_puzzle), [target_puzzle_hash] + memos
```
