### Title
Unconstrained `fee_proposal_fri` During Startup Window Poisons `fee_actual` and Suppresses L2 Gas Price Floor — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

During the first `fee_proposal_window_size` blocks after genesis (or any fresh start), the `fee_proposal_fri` field in `ProposalInit` is **proposer-controlled and entirely unchecked**. The bounds guard in `is_proposal_init_valid` is explicitly skipped when `fee_actual` is `None` (i.e., the sliding window is not yet full). A malicious proposer can set `fee_proposal_fri` to an arbitrarily low value, have it accepted by all validators, and have it recorded into `fee_proposals_window`. Once the window fills, `compute_fee_actual` returns the median of these poisoned values, which then becomes the floor for the EIP-1559 L2 gas price calculation in all subsequent blocks — suppressing the SNIP-35 USD-denominated fee floor and allowing the network to charge users far less than the intended cost.

---

### Finding Description

**Step 1 — Bounds check is explicitly skipped during startup.**

In `validate_proposal.rs`, `is_proposal_init_valid` only enforces the `fee_proposal_fri` margin when `fee_actual` is `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(...);
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

`fee_actual` is `None` whenever `compute_fee_actual` cannot find all `window_size` entries in `fee_proposals_window`: [2](#0-1) 

**Step 2 — The unchecked value is unconditionally recorded.**

`finalize_decision` calls `record_fee_proposal` with the proposer-supplied `init.fee_proposal_fri` regardless of whether bounds were enforced: [3](#0-2) 

`record_fee_proposal` is a simple insert with no validation: [4](#0-3) 

**Step 3 — Poisoned window drives down `fee_actual`.**

After `window_size` blocks, `compute_fee_actual` returns the median of the recorded values. If a malicious proposer controlled enough slots during startup to set the median to a low value (e.g., just above `config_min`), `fee_actual` is permanently suppressed for the next window.

**Step 4 — Suppressed `fee_actual` lowers the EIP-1559 gas price floor.**

`calculate_next_l2_gas_price_for_fin` uses `fee_actual` as the minimum floor for the EIP-1559 base fee:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [5](#0-4) 

Without manipulation, `fee_actual` would be the median of honest SNIP-35 proposals — potentially orders of magnitude above `config_min`. With manipulation, `effective_min` collapses to `config_min`, removing the SNIP-35 USD-cost floor entirely.

**Step 5 — Future `fee_proposal` bounds are anchored to the poisoned `fee_actual`.**

Once `fee_actual` is established at a low value, `fee_proposal_bounds` constrains all future proposals to within `margin_ppt` of that low value: [6](#0-5) 

This creates a self-reinforcing feedback loop: the gas price stays near `config_min` for the entire duration of the poisoned window, and each subsequent window inherits the suppressed median.

**Step 6 — `fee_proposal_fri` is bound into the `ProposalCommitment` that validators sign.**

Validators sign over the poisoned value without any recourse: [7](#0-6) 

---

### Impact Explanation

The L2 gas price used for transaction execution (`l2_gas_price_fri` in `ProposalInit`) is computed by each validator using `calculate_next_l2_gas_price_for_fin` with the poisoned `fee_actual` as the floor. All validators independently arrive at the same suppressed price (since they all use the same recorded window), so the block is committed with a gas price far below the honest SNIP-35 target. Users pay less than the intended USD-denominated cost of computation, and the protocol loses fee revenue. This matches the **Critical** impact category: "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."

---

### Likelihood Explanation

The attack window is the first `fee_proposal_window_size` blocks. In a decentralized Tendermint-style consensus, the proposer rotates deterministically. An attacker who controls even a majority of proposer slots during this window (e.g., by controlling a majority of stake at genesis, or by being the sole initial validator) can set the median `fee_proposal_fri` to any value above `config_min`. Even a minority of controlled slots shifts the median downward. The attack requires no special privileges beyond being a legitimate consensus participant during the startup window — a condition that is trivially satisfied by any validator.

---

### Recommendation

Apply a minimum bound on `fee_proposal_fri` even when `fee_actual` is `None`. The simplest fix is to require `fee_proposal_fri >= config_min_gas_price` unconditionally during startup. A stronger fix is to derive a startup floor from the oracle (`compute_fee_target`) and reject proposals below it, mirroring the honest proposer logic in `compute_proposer_fee_proposal`. This ensures the window is never seeded with values that undermine the SNIP-35 floor. [1](#0-0) 

---

### Proof of Concept

1. Network launches at genesis with `fee_proposal_window_size = W`.
2. Attacker controls the proposer role for blocks `0` through `W-1` (or a majority of them).
3. For each block, attacker sets `fee_proposal_fri = config_min + 1` in `ProposalInit`.
4. `is_proposal_init_valid` is called; `fee_actual` is `None` (window not full); the bounds check at lines 398–416 of `validate_proposal.rs` is skipped; the proposal is accepted.
5. `finalize_decision` calls `record_fee_proposal(height, Some(GasPrice(config_min + 1)))` for each block.
6. At block `W`, `compute_fee_actual` returns `Some(GasPrice(config_min + 1))` (median of all poisoned entries).
7. `calculate_next_l2_gas_price_for_fin` computes `effective_min = max(config_min, config_min + 1) = config_min + 1`.
8. The EIP-1559 price can now decrease to `config_min + 1` whenever gas usage is below target — far below the honest SNIP-35 target (e.g., 1000× higher).
9. All subsequent blocks execute transactions at the suppressed gas price; users pay a fraction of the intended fee.
10. The poisoned window persists for `W` more blocks, and each new window inherits the suppressed median unless honest proposals can outweigh the attacker's entries.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub(crate) fn fee_proposal_bounds(fee_actual: GasPrice, margin_ppt: u128) -> (u128, u128) {
    let denom = U256::from(PPT_DENOMINATOR);
    let scaled = denom + U256::from(margin_ppt);
    let fee_actual_u256 = U256::from(fee_actual.0);
    let upper = u128::try_from(fee_actual_u256 * scaled / denom).unwrap_or(u128::MAX);
    let lower = u128::try_from(fee_actual_u256 * denom / scaled).unwrap_or(0);
    (lower, upper)
}
