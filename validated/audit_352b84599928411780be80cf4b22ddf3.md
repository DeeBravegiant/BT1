Audit Report

## Title
Incomplete Pause: `_transfer` Missing `whenNotPaused` Allows Token Movement While Paused — (File: contracts/RSETH.sol)

## Summary
`RSETH._transfer` does not apply `whenNotPaused`, while `mint` and `burnFrom` do. Any rsETH holder can call the standard ERC20 `transfer` or `transferFrom` while the contract is paused, bypassing the intended operational freeze. No funds are directly lost, but the contract fails to deliver its promised invariant that pausing halts all token movement.

## Finding Description
`RSETH` inherits `PausableUpgradeable` and gates both `mint` (line 235) and `burnFrom` (line 245) with `whenNotPaused`. However, the `_transfer` override at line 287–291 only enforces the per-address block mechanism and then calls `super._transfer` with no pause check:

```solidity
// contracts/RSETH.sol L287-291
function _transfer(address from, address to, uint256 amount) internal override {
    _enforceNotBlocked(from);
    _enforceNotBlocked(to);
    super._transfer(from, to, amount);   // no whenNotPaused
}
```

Because `ERC20Upgradeable.transfer` and `transferFrom` both route through this override, any holder can move tokens freely while the contract is paused. The `recoverFrozenFunds` admin function at line 218 calls `super._transfer` directly (bypassing the override entirely), so adding `whenNotPaused` to `_transfer` would not interfere with fund recovery.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

When the contract is paused in response to an incident, the pause does not freeze token movement. Holders can transfer rsETH to new addresses before an admin applies `blockUserTransfers` to the original address, undermining the per-address freeze and fund-recovery mechanism. No direct theft or fund loss occurs, but the security invariant of the pause is broken.

## Likelihood Explanation
Any rsETH holder can exploit this with a standard `transfer` call the moment the contract is paused. No privileged access, special knowledge, or additional preconditions are required beyond holding rsETH. The `PAUSER_ROLE` pausing the contract is a realistic operational action during incidents, making the window immediately exploitable.

## Recommendation
Add `whenNotPaused` to the `_transfer` override:

```solidity
function _transfer(address from, address to, uint256 amount)
    internal
    override
    whenNotPaused
{
    _enforceNotBlocked(from);
    _enforceNotBlocked(to);
    super._transfer(from, to, amount);
}
```

This does not affect `recoverFrozenFunds`, which calls `super._transfer` directly and bypasses the override.

## Proof of Concept
1. Deploy `RSETH` and mint tokens to `alice`.
2. Call `pause()` from an address holding `PAUSER_ROLE`.
3. From `alice`, call `rsETH.transfer(bob, amount)`.
4. Observe the call succeeds — `_transfer` runs `_enforceNotBlocked` (passes) then `super._transfer` (no pause check), completing the transfer.
5. Confirm `mint(alice, 1)` reverts with `Pausable: paused` while the transfer in step 3 does not.

Foundry test sketch:
```solidity
function test_transferBypassesPause() public {
    vm.prank(pauser);
    rsETH.pause();
    assertTrue(rsETH.paused());

    vm.prank(alice);
    rsETH.transfer(bob, 1 ether); // should revert if pause is enforced, but succeeds
    assertEq(rsETH.balanceOf(bob), 1 ether);
}
```
