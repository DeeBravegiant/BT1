Audit Report

## Title
Missing Chainlink Staleness Check Allows Stale stETH/ETH Price to Inflate rsETH Minting Rate — (`contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` discards `updatedAt` and `answeredInRound` from `latestRoundData()`, accepting any price regardless of age. A stale-high stETH/ETH price inflates `rsETHPrice` via the permissionless `updateRSETHPrice()`, and because the same stale price appears in the numerator of `getRsETHAmountToMint()` while the inflated `rsETHPrice` appears in the denominator, the inflation does not cancel when other assets are present in the pool. An attacker mints rsETH at a rate exceeding the true stETH/ETH exchange rate, extracting value from existing rsETH holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` reads only `price` from `latestRoundData()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
```

No check on `updatedAt` against a heartbeat, and no check that `answeredInRound >= roundId`. The stETH/ETH Chainlink feed has a 24-hour heartbeat and a 0.5% deviation threshold; during low-volatility periods the feed can go stale for up to 24 hours while the last recorded price remains above the current market price.

The public `updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` path uses this unchecked price to compute `totalETHInProtocol`, which is divided by `rsethSupply` to set `rsETHPrice`. A subsequent `depositAsset(stETH, ...)` call uses the same stale price in the numerator and the inflated stored `rsETHPrice` in the denominator. Because the denominator is only partially inflated (other assets dilute the effect), the attacker receives more rsETH than the deposited stETH is worth.

The contrast with the pool-side oracle in the same repository is direct: `ChainlinkOracleForRSETHPoolCollateral.getRate()` explicitly checks `answeredInRound < roundID`, `timestamp == 0`, and `ethPrice <= 0`, while `ChainlinkPriceOracle.getAssetPrice()` performs none of these checks.

The `pricePercentageLimit` guard is bypassed entirely when `pricePercentageLimit == 0`, which is the default state (no value is set in `initialize()`). Even when configured, a 0.5% stale deviation falls within a typical 1% limit. The downside-protection auto-pause does not apply because the stale-high scenario causes a price *increase*. The `minRSETHAmountExpected` slippage parameter protects the depositor from receiving *fewer* rsETH than expected, but does not prevent receiving *more*.

## Impact Explanation
Existing rsETH holders are diluted: the attacker's rsETH represents a claim on protocol assets exceeding the value deposited. For a protocol with $100M TVL (50% stETH, 50% other assets), a 0.5% stale deviation, and a $10M deposit, the attacker extracts approximately $25,000 per transaction. The attack is repeatable and scales with deposit size. This constitutes direct theft of user funds (existing rsETH holders' claims on protocol assets are permanently reduced), matching the Critical impact class.

## Likelihood Explanation
The stETH/ETH Chainlink feed (24h heartbeat, 0.5% deviation threshold) can remain stale for up to 24 hours during low-volatility periods. `updateRSETHPrice()` is permissionless — any external caller can trigger it. The `pricePercentageLimit` guard is bypassed when set to 0 (the default) or when the stale deviation falls within the configured limit. The multi-asset pool state (ETH + multiple LSTs) is the normal production configuration. No privileged access is required.

## Recommendation
Add staleness and round-completeness checks to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();
require(answeredInRound >= roundId, "Stale price: incomplete round");
require(block.timestamp - updatedAt <= HEARTBEAT, "Stale price: heartbeat exceeded");
require(price > 0, "Invalid price");
```

Store a per-feed heartbeat value (e.g., 3600s for 1h feeds, 86400s for 24h feeds) and enforce it. This mirrors the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`.
