### Title
TDH2 Partial Decryption ZK Proof Does Not Bind to Label — (File: `src/cbmpc/crypto/tdh2.cpp`)

### Summary

`private_share_t::decrypt` receives a `label` parameter and correctly forwards it to `ciphertext.verify`, but the inner ZK proof hash — `ei = ro::hash_number(Xi, Yi, Zi).mod(q)` — omits the `label` entirely. The corresponding verifier `check_partial_decryption_helper` also omits the label. A partial decryption `(Xi, ei, fi)` is therefore valid for **any** ciphertext sharing the same `R1`, regardless of label. A malicious encryptor who reuses the same scalar `r` across two ciphertexts with different labels can present partial decryptions obtained for label `L1` as valid proofs for a ciphertext under label `L2`, bypassing label-based access control.

### Finding Description

In `private_share_t::decrypt` (`src/cbmpc/crypto/tdh2.cpp`, lines 68–92):

```cpp

``` [1](#0-0) 

The ZK proof challenge `ei` is computed over `(Xi, Yi, Zi)` only. The `label` (and the rest of the ciphertext: `c`, `R2`, `e`, `f`, `iv`) is absent from the hash.

The verifier `check_partial_decryption_helper` (`src/cbmpc/crypto/tdh2.cpp`, lines 104–125) mirrors this omission:

```cpp
bn_t ei_test = ro::hash_number(Xi, Yi, Zi).mod(q);   // label NOT checked
if (ei != ei_test) return coinbase::error(E_CRYPTO);
``` [2](#0-1) 

The only ciphertext field used in verification is `R1` (to reconstruct `Yi = fi * R1 - ei * Xi`). Therefore a tuple `(Xi, ei, fi)` produced for ciphertext `CT1` with label `L1` is cryptographically indistinguishable from a valid partial decryption for any ciphertext `CT2` that shares the same `R1`, including one with label `L2 ≠ L1`.

Both `combine_additive` and `combine` (the access-structure variant) call `check_partial_decryption_helper` without passing the label:

```cpp
if (rv = partial_decryption.check_partial_decryption_helper(Qi[rid - 1], ciphertext, curve)) return rv;
``` [3](#0-2) 

The public API entry points `coinbase::api::tdh2::partial_decrypt` and `coinbase::api::tdh2::combine_additive` / `combine_ac` flow directly into this code path. [4](#0-3) [5](#0-4) 

### Impact Explanation

A malicious encryptor who controls the randomness `r` can construct two valid ciphertexts:

- `CT1 = encrypt(P1, L1, r)` — label `L1`, `R1 = r·G`
- `CT2 = encrypt(P2, L2, r)` — label `L2`, same `R1 = r·G`

Both pass `ciphertext.verify` for their respective labels. The attacker presents `CT1` to the threshold parties; each party calls `private_share_t::decrypt(CT1, L1, partial)`, verifies the ciphertext, and emits `(Xi, ei, fi)`. Because the ZK proof is bound only to `R1` (not to `L1`), the same tuple is a valid proof for `CT2`.

The attacker then calls `combine_additive(pk, Qi, L2, partials_from_CT1, CT2, plaintext)`:

1. `ciphertext.verify(pk, L2)` passes — `CT2` is a well-formed ciphertext for `L2`
2. `check_partial_decryption_helper(Qi, CT2, curve)` passes — uses `CT2.R1 = CT1.R1`, ZK proof verifies
3. `ciphertext.decrypt(V, plain, L2)` succeeds — `V = Σ xi·R1` is correct

The threshold parties believe they authorised decryption under label `L1`; the combiner obtains a valid decryption under label `L2`. This breaks the label-binding guarantee of TDH2: partial decryptions are accepted under the wrong label.

### Likelihood Explanation

The attacker must control the encryption randomness `r` (i.e., be the encryptor of `CT1`). This is a realistic position for any party that initiates a TDH2 encryption workflow. The attack requires no threshold collusion, no key leakage, and no break of any hardness assumption — only the ability to reuse `r` across two encryptions, which is trivially achievable by the encryptor.

### Recommendation

Include `label` (and ideally the full ciphertext transcript: `c`, `R1`, `R2`, `iv`) in the ZK proof challenge hash inside `private_share_t::decrypt`:

```cpp
// Before (vulnerable):
ei = ro::hash_number(Xi, Yi, Zi).mod(q);

// After (fixed):
ei = ro::hash_number(Xi, Yi, Zi, ciphertext.R1, ciphertext.R2, ciphertext.c, ciphertext.iv, label).mod(q);
```

The same inputs must be reproduced in `check_partial_decryption_helper`, which must therefore also receive the `label` parameter. Both `combine_additive` and `combine` must forward `label` to `check_partial_decryption_helper`.

### Proof of Concept

```
// Setup: n=2 additive TDH2 shares (x0, x1), public key Q = (x0+x1)·G
// Attacker controls encryption randomness r.

buf_t r_bytes = fixed_random_r();          // attacker-chosen r
bn_t r = bn_t::from_bin(r_bytes);

// CT1: label "authorized", plaintext P1
ciphertext_t CT1 = pk.encrypt(P1, "authorized", r, s1, iv1);

// CT2: label "unauthorized", plaintext P2, SAME r → same R1
ciphertext_t CT2 = pk.encrypt(P2, "unauthorized", r, s2, iv2);

// Parties produce partial decryptions for CT1 / "authorized"
partial_decryption_t pd0, pd1;
share0.decrypt(CT1, "authorized", pd0);   // ei = H(Xi, Yi, Zi) — no label
share1.decrypt(CT1, "authorized", pd1);

// Attacker combines CT2 / "unauthorized" using pd0, pd1
buf_t recovered;
combine_additive(pk, {Q0, Q1}, "unauthorized", {pd0, pd1}, CT2, recovered);
// → recovered == P2, despite parties never consenting to "unauthorized"

error_t partial_decryption_t::check_partial_decryption_helper(const ecc_point_t& Qi, const ciphertext_t& ciphertext,
                                                              ecurve_t curve) const {
  error_t rv = UNINITIALIZED_ERROR;

  if (rv = curve.check(Qi))
    return coinbase::error(rv, "partial_decryption_t::check_partial_decryption_helper: check Qi failed");
  if (rv = curve.check(Xi))
    return coinbase::error(rv, "partial_decryption_t::check_partial_decryption_helper: check Xi failed");

  const auto& G = curve.generator();
  const mod_t& q = curve.order();
  if (!q.is_in_range(ei) || !q.is_in_range(fi)) return coinbase::error(E_CRYPTO);

  const ecc_point_t& R1 = ciphertext.R1;
  ecc_point_t Yi = fi * R1 - ei * Xi;
  ecc_point_t Zi = fi * G - ei * Qi;

  bn_t ei_test = ro::hash_number(Xi, Yi, Zi).mod(q);
  if (ei != ei_test) return coinbase::error(E_CRYPTO);

  return SUCCESS;
}

```cpp
error_t combine_additive(const public_key_t& pub_key, const pub_shares_t& Qi, mem_t label,
                         const partial_decryptions_t& partial_decryptions, const ciphertext_t& ciphertext,
                         buf_t& plain) {
  error_t rv = UNINITIALIZED_ERROR;
  const auto& curve = pub_key.Q.get_curve();
  int n = int(Qi.size());
  for (const auto& _Qi : Qi) {
    if (rv = curve.check(_Qi)) return coinbase::error(rv, "combine_additive: check Qi failed");
  }
  if ((int)partial_decryptions.size() != n) return coinbase::error(E_CRYPTO);

  if (rv = ciphertext.verify(pub_key, label)) return rv;

  ecc_point_t V = curve.infinity();
  for (int i = 0; i < n; i++) {
    const partial_decryption_t& partial_decryption = partial_decryptions[i];

    const int rid = partial_decryption.rid;
    if (rid < 1 || rid > n) return coinbase::error(E_CRYPTO);
    if (rv = partial_decryption.check_partial_decryption_helper(Qi[rid - 1], ciphertext, curve)) return rv;

    V += partial_decryption.Xi;
  }

  if (rv = ciphertext.decrypt(V, plain, label)) return rv;
  return SUCCESS;
}

error_t combine_additive(mem_t public_key, const std::vector<mem_t>& public_shares, mem_t label,
                         const std::vector<mem_t>& partial_decryptions, mem_t ciphertext, buf_t& plaintext) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(public_key, "public_key",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_arg(label, "label")) return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(
          ciphertext, "ciphertext", coinbase::api::detail::MAX_CIPHERTEXT_BLOB_SIZE))
    return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_vec_arg_max_size(
          public_shares, "public_shares", coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_vec_arg_max_size(
          partial_decryptions, "partial_decryptions", coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;

  if (public_shares.size() != partial_decryptions.size())
    return coinbase::error(E_BADARG, "public_shares and partial_decryptions size mismatch");

  coinbase::crypto::tdh2::public_key_t pk;
  error_t rv = pk.from_bin(public_key);
  if (rv) return rv;
  if (rv = validate_public_key(pk)) return rv;

  coinbase::crypto::tdh2::ciphertext_t ct;
  rv = coinbase::convert(ct, ciphertext);
  if (rv) return rv;

  auto curve = pk.Q.get_curve();
  if (!curve.valid()) return coinbase::error(E_FORMAT, "public key missing curve");

  coinbase::crypto::tdh2::pub_shares_t Qi;
  Qi.reserve(public_shares.size());
  for (const auto& m : public_shares) {
    coinbase::crypto::ecc_point_t P;
    rv = P.from_bin(curve, m);
    if (rv) return rv;
    Qi.emplace_back(std::move(P));
  }

  coinbase::crypto::tdh2::partial_decryptions_t partials;
  partials.resize(partial_decryptions.size());
  for (size_t i = 0; i < partial_decryptions.size(); i++) {
    rv = coinbase::convert(partials[i], partial_decryptions[i]);
    if (rv) return rv;
  }

  return coinbase::crypto::tdh2::combine_additive(pk, Qi, label, partials, ct, plaintext);
}
