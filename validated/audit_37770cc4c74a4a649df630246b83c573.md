Audit Report

## Title
Missing `recipient != address(this)` Validation Allows Self-Transfer to Silently Drain LP Reserves Into Protocol-Fee Surplus — (`metric-core/contracts/MetricOmmPool.sol`)

## Summary

The `swap` function in `MetricOmmPool` accepts an arbitrary `recipient` address with no check that `recipient != address(this)`. When a caller sets `recipient = address(pool)`, the output-token transfer is an ERC-20 self-transfer (balance unchanged), but `binTotals.scaledToken1` (or `scaledToken0`) has already been decremented by `_executeSwap`. The resulting accounting gap is permanently classified as spread-fee surplus and extracted on the next `collectFees` call, irrecoverably reducing LP claims on that token.

## Finding Description

**Root cause — no recipient validation:**

`swap` accepts `recipient` without any guard:

```solidity
// MetricOmmPool.sol line 217-225
function swap(
    address recipient,   // ← no require(recipient != address(this))
    bool zeroForOne,
    ...
) external whenNotPaused nonReentrant(PoolActions.SWAP) returns (int128, int128) {
    require(amountSpecified != 0, InvalidAmount());
``` [1](#0-0) 

**Step 1 — `_executeSwap` decrements `binTotals.scaledToken1` unconditionally:**

```solidity
// line 739
binTotals.scaledToken1 = uint128(uint256(binTotals.scaledToken1) - uint256(-amount1DeltaScaled));
``` [2](#0-1) 

**Step 2 — output token is transferred to `recipient` via `safeTransfer`:**

```solidity
// lines 251-254
if (amount1Delta < 0) {
    transferToken1(recipient, uint256(-amount1Delta));   // self-transfer if recipient == pool
}
``` [3](#0-2) 

`transferToken1` calls `IERC20(TOKEN1).safeTransfer(to, amount)`: [4](#0-3) 

When `to == address(this)`, a standard ERC-20 self-transfer leaves `balanceOf(address(this))` unchanged. `balance1()` reads `IERC20(TOKEN1).balanceOf(address(this))`, so it is unaffected: [5](#0-4) 

**Step 3 — `IncorrectDelta` guard only checks the INPUT token:**

```solidity
// lines 261-263
if (amount0Delta > 0 && balance0Before + uint256(amount0Delta) > balance0()) {
    revert IncorrectDelta();
}
``` [6](#0-5) 

This verifies only that the pool received the correct token0 input. It says nothing about where the output token1 went. The self-recipient case passes silently.

**Step 4 — `collectFees` computes surplus as the gap between real balance and bin totals:**

```solidity
// lines 387-388
uint256 surplus1Scaled =
    balance1() * TOKEN_1_SCALE_MULTIPLIER
    - uint256(binTotals.scaledToken1)      // ← artificially low after attack
    - notionalFee1AmountScaled;
``` [7](#0-6) 

Because `balance1()` is unchanged but `binTotals.scaledToken1` was decremented, `surplus1Scaled` is inflated by exactly the output amount of the attack swap. The next `collectFees` call distributes this inflated surplus to admin and protocol, permanently removing it from LP-accessible reserves. [8](#0-7) 

The grep search confirmed no `recipient != address(this)` check exists anywhere in the codebase.

## Impact Explanation

Every LP's proportional claim on token1 is computed from `binTotals.scaledToken1`. After the attack, that value is understated relative to `balance1()`. When LPs call `removeLiquidity`, they receive amounts proportional to the understated bin total. The missing token1 (now classified as surplus) is extracted by `collectFees` and is not accessible to LPs via any path. The attack is repeatable: each swap with `recipient = address(pool)` drains another slice of LP reserves. This constitutes a direct, irrecoverable loss of LP principal — a High/Critical impact under the allowed impact gate (direct loss of user principal, pool insolvency: balances fail to cover LP claims).

## Likelihood Explanation

- **Trigger**: Any address that can call `swap` (no role restriction) can set `recipient = address(pool)`. The pool is not paused by default.
- **Cost to attacker**: The attacker must pay the input token (token0) and receives nothing in return. This is a griefing/competitive attack; the attacker's loss equals the LP's loss at oracle price.
- **Motivation**: A competitor, a malicious factory operator who later calls `collectFees`, or any actor willing to spend funds to harm LPs.
- **Detection**: Silent — no event distinguishes a self-recipient swap from a normal one; `Swap` is emitted with the pool address as `recipient`.
- **Repeatability**: Unlimited; each call drains another slice.

Likelihood: **Medium** (unprivileged, zero-code-change trigger; attacker must spend capital equal to the damage inflicted).

## Recommendation

Add a recipient validation at the top of `swap`:

```solidity
require(recipient != address(this), InvalidRecipient());
```

Alternatively, validate inside each output-transfer branch before the transfer executes:

```solidity
if (amount1Delta < 0) {
    require(recipient != address(this), InvalidRecipient());
    transferToken1(recipient, uint256(-amount1Delta));
}
```

The same guard should be applied to the `!zeroForOne` branch for `transferToken0`. This mirrors the standard fix for self-referential address bugs: reject the degenerate case before any state mutation occurs.

## Proof of Concept

```solidity
contract DrainLPReserves is IMetricOmmSwapCallback {
    IMetricOmmPoolActions pool;
    IERC20 token0;

    constructor(address _pool, address _token0) {
        pool = IMetricOmmPoolActions(_pool);
        token0 = IERC20(_token0);
    }

    function attack(int128 amountIn) external {
        token0.approve(address(pool), type(uint256).max);
        // recipient = address(pool): output token1 self-transfers back to pool
        pool.swap(
            address(pool),   // ← recipient is the pool itself
            true,            // zeroForOne: pay token0, "receive" token1
            amountIn,
            0,
            "",
            ""
        );
        // Post-conditions:
        // pool.balance1() unchanged (self-transfer)
        // pool.binTotals.scaledToken1 decreased by output amount
        // surplus1Scaled inflated by output amount
        // next collectFees() extracts LP token1 as protocol/admin fees
    }

    function metricOmmSwapCallback(int256 amount0Delta, int256, bytes calldata) external {
        if (amount0Delta > 0) {
            token0.transfer(msg.sender, uint256(amount0Delta));
        }
    }
}
```

**Foundry invariant check (post-attack):**

```solidity
assertGt(
    pool.balance1() * TOKEN_1_SCALE_MULTIPLIER,
    uint256(binTotals.scaledToken1) + notionalFeeToken1Scaled,
    "surplus1Scaled inflated by attack output amount"
);
```

The gap equals the output amount of the attack swap. On the next `collectFees`, this gap is distributed as admin/protocol fees, permanently reducing LP claims.
