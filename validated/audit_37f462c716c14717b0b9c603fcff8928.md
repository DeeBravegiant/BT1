Audit Report

## Title
Stale Chainlink Price Used Without Validation in `ChainlinkPriceOracle` Leads to Incorrect rsETH Minting - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` and discards every return value except `price`, performing no staleness, incomplete-round, or non-positive price checks. This unguarded price feeds into `LRTOracle._updateRsETHPrice()`, which commits a global `rsETHPrice`. Because `updateRSETHPrice()` is public and permissionless, an attacker can commit a stale-derived (understated) `rsETHPrice` and then deposit after the feed recovers, receiving excess rsETH at the expense of existing holders.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads only the `price` field from `latestRoundData()`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No check is made on `answeredInRound`, `updatedAt`, or whether `price > 0`. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` explicitly guards all three failure modes:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unguarded oracle is the one registered in `LRTOracle.assetPriceOracle` for mainnet LST assets. `LRTOracle._getTotalEthInProtocol()` iterates all supported assets and calls `getAssetPrice(asset)` on it, accumulating TVL:

```solidity
// contracts/LRTOracle.sol L339-343
uint256 assetER = getAssetPrice(asset);
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);
```

The result is used to compute and store `rsETHPrice`:

```solidity
// contracts/LRTOracle.sol L250, L313
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
...
rsETHPrice = newRsETHPrice;
```

`updateRSETHPrice()` is public and permissionless (only `whenNotPaused`):

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`LRTDepositPool.getRsETHAmountToMint()` uses the stored `rsETHPrice` as the denominator:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

**Exploit path:**
1. A Chainlink feed for a supported LST (e.g., cbETH) becomes stale at `P_stale < P_actual`.
2. Attacker calls `updateRSETHPrice()`. `_getTotalEthInProtocol()` reads `P_stale`, understating TVL. `rsETHPrice` is committed below its true value.
3. The Chainlink feed updates to `P_actual`.
4. Attacker calls `depositAsset(cbETH, amount, ...)`. The numerator in `getRsETHAmountToMint` reads the now-live `getAssetPrice(cbETH) = P_actual`, while the denominator is the already-committed stale `rsETHPrice`. The attacker receives excess rsETH.

The `pricePercentageLimit` guard in `_updateRsETHPrice()` provides partial mitigation: if the stale-induced price drop exceeds the configured limit, the protocol pauses. However, this guard fails to prevent the attack when (a) `pricePercentageLimit == 0` (unset), or (b) the staleness-induced deviation falls within the configured limit.

## Impact Explanation

**Critical — Protocol insolvency / direct dilution of existing rsETH holders.**

Excess rsETH is minted to the attacker without a corresponding increase in TVL. After the feed recovers and `rsETHPrice` is updated to its true value, the protocol has issued more rsETH than the TVL supports. Existing holders' redemption value is permanently diluted. Repeated exploitation accelerates insolvency.

## Likelihood Explanation

Chainlink feeds can become stale during L1 network congestion, feed deprecation, or sequencer downtime on L2. The `updateRSETHPrice()` function is public and permissionless, so any actor monitoring feed staleness on-chain can trigger the price commit at the worst moment. The inconsistency with `ChainlinkOracleForRSETHPoolCollateral` — which performs all three checks — confirms the omission is unintentional. Likelihood is **medium**: requires a feed to go stale (observable on-chain condition) and a subsequent feed recovery before the next honest `updateRSETHPrice()` call.

## Recommendation

Apply the same three guards used in `ChainlinkOracleForRSETHPoolCollateral` inside `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, add a per-feed heartbeat check: `if (block.timestamp - updatedAt > MAX_DELAY) revert StalePrice();`.

## Proof of Concept

1. Assume cbETH/ETH Chainlink feed is configured in `ChainlinkPriceOracle` and the feed's last update was at price `1.05e18` (stale; actual price is now `1.10e18`).
2. Attacker calls `LRTOracle.updateRSETHPrice()`. `_getTotalEthInProtocol()` reads `1.05e18` for cbETH. With 1000 cbETH in the protocol and rsETH supply of 1050: `rsETHPrice = (1000 * 1.05e18) / 1050 = 1.0e18`.
3. The Chainlink feed updates to `1.10e18`.
4. Attacker deposits 100 cbETH via `LRTDepositPool.depositAsset()`. `getRsETHAmountToMint` computes: `(100 * 1.10e18) / 1.0e18 = 110 rsETH`.
5. True fair value: `(100 * 1.10e18) / ((1000 * 1.10e18) / 1050) ≈ 100 rsETH`.
6. Attacker receives 110 rsETH instead of 100 — 10 rsETH of excess value extracted from existing holders.

**Foundry fork test plan:**
- Fork mainnet with a cbETH/ETH Chainlink feed.
- Mock `latestRoundData()` to return a stale `answeredInRound < roundId` with a depressed price.
- Call `LRTOracle.updateRSETHPrice()` as the attacker; assert `rsETHPrice` is set below true value.
- Restore the mock to return the current price.
- Call `LRTDepositPool.depositAsset()` and assert the minted rsETH exceeds the fair-value amount.
