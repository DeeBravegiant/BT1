### Title
Attacker-Controlled `hidden_puzzle_hash` Injected into R-CAT Wallet via Crafted DEFAULT_CATS Spend — (`chia/wallet/wallet_state_manager.py`)

### Summary

An unprivileged attacker who holds any DEFAULT_CATS token (Spacebucks, Marmot, etc.) can broadcast a valid on-chain spend whose puzzle reveal wraps the inner puzzle in a revocation layer with an attacker-chosen `hidden_puzzle_hash`. The victim wallet's `handle_cat` blindly extracts that hash from the parent coin's puzzle reveal and passes it to `RCATWallet.get_or_create_wallet_for_cat`, creating a wallet whose `puzzle_for_pk` permanently embeds the attacker's revocation key. Every future R-CAT coin tracked by that wallet is spendable by the attacker via the hidden puzzle path.

---

### Finding Description

**Entry point — `determine_coin_type`** [1](#0-0) 

The parent coin's `coin_spend` is fetched from the peer and passed verbatim to `handle_cat`.

**Step 1 — hint check passes trivially** [2](#0-1) 

The hint is a memo the attacker freely sets in the `CREATE_COIN` condition. The attacker sets it to any of the victim's known (public) puzzle hashes. `derivation_record` is found, so execution continues.

**Step 2 — standard CAT puzzle hash mismatch triggers special-type branch** [3](#0-2) 

Because the new coin's puzzle hash is `CAT(tail, revocation_layer(attacker_hph, victim_inner))`, it does not equal the standard CAT hash. The code then calls `match_revocation_layer` on the **parent coin's** inner puzzle (`uncurried_puzzle_reveal.args.at("rrf")`). Since the attacker controls the parent coin, this returns `(attacker_hidden_puzzle_hash, victim_inner_puzzle_hash)` and `wallet_type` is set to `RCATWallet`.

**Step 3 — `match_revocation_layer` returns attacker-controlled value without validation** [4](#0-3) 

The function simply reads the first curried argument of the revocation layer. No signature, ownership, or key-authorization check is performed.

**Step 4 — DEFAULT_CATS membership check passes, wallet created with attacker's hash** [5](#0-4) 

`parent_data.tail_program_hash.hex() in self.default_cats` is `True` (attacker used a real DEFAULT_CATS tail). `RCATWallet.get_or_create_wallet_for_cat` is called with `hidden_puzzle_hash=revocation_layer_match[0]` — the attacker's value.

**Step 5 — wallet persists attacker's hash and uses it for all future puzzle derivation** [6](#0-5) 

`self.cat_info = RCATInfo(limitations_program_hash, None, hidden_puzzle_hash)` is stored to the database. [7](#0-6) 

Every call to `puzzle_for_pk` wraps the victim's inner puzzle in `create_revocation_layer(self.cat_info.hidden_puzzle_hash, ...)`, embedding the attacker's hash permanently.

---

### Impact Explanation

The revocation layer is a "p2-puzzle-or-hidden-puzzle" construct. Whoever knows the preimage of `hidden_puzzle_hash` can spend any coin whose puzzle hash was derived from it via the hidden path — without the victim's signature. Because the attacker chose `hidden_puzzle_hash`, they know the preimage. All coins the victim receives into this R-CAT wallet (including the triggering coin itself) are immediately spendable by the attacker. This is unauthorized coin control over CAT assets — a Critical/High impact.

---

### Likelihood Explanation

- Attacker precondition: own any amount of a DEFAULT_CATS token (freely purchasable on DEXes).
- Victim precondition: no existing R-CAT wallet for that asset_id (first-time encounter).
- No privileged access, no leaked keys, no broken cryptography required.
- The attack is a single valid on-chain spend; it is fully automatable.

---

### Recommendation

In `handle_cat`, after identifying `wallet_type = RCATWallet`, verify that the **new coin's** puzzle hash actually equals the expected R-CAT puzzle hash derived from the victim's key and the extracted `hidden_puzzle_hash` before creating or converting any wallet:

```python
expected_rcat_hash = construct_cat_puzzle(
    CAT_MOD, asset_id,
    create_revocation_layer(revocation_layer_match[0], our_inner_puzzle.get_tree_hash())
).get_tree_hash()
if expected_rcat_hash != coin_state.coin.puzzle_hash:
    return None
```

This ensures the wallet is only created when the received coin actually encodes the victim's own key under the revocation layer, preventing injection of an arbitrary `hidden_puzzle_hash`.

### Proof of Concept

1. Attacker acquires 1 mojo of Spacebucks (asset_id `a628c1c2...`).
2. Attacker crafts a spend that creates a new coin: `CAT(CAT_MOD_HASH, SPACEBUCKS_TAIL, revocation_layer(ATTACKER_HPH, victim_p2_puzzle_hash))` with memo/hint = `victim_p2_puzzle_hash`.
3. Attacker broadcasts the spend bundle; full node confirms it.

10. Assert: `wallet.cat_info.hidden_puzzle_hash == ATTACKER_HPH` ✓
11. Attacker spends the coin using the hidden puzzle path (no victim signature needed). [8](#0-7) [9](#0-8) [10](#0-9) [11](#0-10)

def match_revocation_layer(uncurried_puzzle: UncurriedPuzzle) -> tuple[bytes32, bytes32] | None:
    if uncurried_puzzle.mod == REVOCATION_LAYER:
        return bytes32(uncurried_puzzle.args.at("rf").as_atom()), bytes32(uncurried_puzzle.args.at("rrf").as_atom())

        return None  # pragma: no cover

        hidden_puzzle_hash: bytes32,
        name: str | None = None,
    ) -> Self:

        self.standard_wallet = wallet
        self.log = logging.getLogger(__name__)

        for id, w in wallet_state_manager.wallets.items():

                assert isinstance(w, cls)
                if w.get_asset_id() == limitations_program_hash:

                    return w

        self.wallet_state_manager = wallet_state_manager

            cat_info = DEFAULT_CATS[limitations_program_hash.hex()]
            name = cat_info["name"]

        # Inherited from duplicating parent
        await self.wallet_state_manager.add_new_wallet(self)

        delete: bool = False
