Audit Report

## Title
Missing L2 Sequencer Uptime and Heartbeat Staleness Check Enables Stale Price Exploitation After Sequencer Recovery - (File: contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol)

## Summary
`ChainlinkOracleForRSETHPoolCollateral.getRate()` performs no L2 sequencer uptime feed check and no time-based heartbeat staleness check. After an Arbitrum (or other L2) sequencer outage, the pre-outage collateral token price is accepted as valid, allowing any depositor to mint wrsETH at an inflated rate immediately after sequencer recovery, diluting existing rsETH/wrsETH holders.

## Finding Description
`ChainlinkOracleForRSETHPoolCollateral.getRate()` (lines 26–37) calls `AggregatorV3Interface(oracle).latestRoundData()` and applies only two guards:

- `answeredInRound < roundID` — detects an incomplete round
- `timestamp == 0` — detects a missing timestamp

Neither check prevents acceptance of a price that was last updated hours before a sequencer outage. There is no comparison of `block.timestamp` against `timestamp + HEARTBEAT`, and no query to a Chainlink L2 sequencer uptime feed.

This oracle is wired directly into the deposit path of `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge` via `viewSwapRsETHAmountAndFee`:

```solidity
uint256 tokenToETHRate = IOracle(supportedTokenOracle[token]).getRate();
rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate;
```

When the sequencer recovers, `latestRoundData()` may still return the pre-outage price (the round has not yet been updated), passing both existing checks (`answeredInRound == roundID`, `timestamp != 0`). The stale price is used to compute `rsETHAmount`, resulting in over-minting.

## Impact Explanation
**High — Theft of unclaimed yield.**

If a collateral token's price fell during the outage (e.g., wstETH drops from 1.2 ETH to 1.1 ETH), a depositor calling `deposit(token, amount, referralId)` immediately after recovery receives wrsETH computed at the stale 1.2 ETH rate. The excess wrsETH minted (~9% in this example) represents value extracted from the pool that belongs to existing rsETH/wrsETH holders as accrued yield. This is a concrete, repeatable extraction of value from existing holders, not merely a theoretical dilution.

## Likelihood Explanation
Arbitrum sequencer outages are documented historical events. The protocol explicitly targets Arbitrum and other L2s. No special permissions, governance access, or attacker setup are required — any external caller can invoke `deposit()` immediately after sequencer recovery. The window exists for as long as Chainlink has not yet pushed the updated price round to the L2, which can be minutes to tens of minutes. The attack is repeatable across any sequencer downtime event during which collateral prices move.

## Recommendation
Add a Chainlink L2 sequencer uptime feed check and a time-based heartbeat staleness check to `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
address public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600;
uint256 public constant HEARTBEAT = 3600; // configure per feed

function getRate() public view returns (uint256) {
    if (sequencerUptimeFeed != address(0)) {
        (, int256 seqAnswer, uint256 startedAt,,) =
            AggregatorV3Interface(sequencerUptimeFeed).latestRoundData();
        if (seqAnswer != 0) revert SequencerDown();
        if (block.timestamp - startedAt <= GRACE_PERIOD) revert GracePeriodNotOver();
    }

    (uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
        AggregatorV3Interface(oracle).latestRoundData();

    if (answeredInRound < roundID) revert StalePrice();
    if (timestamp == 0) revert IncompleteRound();
    if (ethPrice <= 0) revert InvalidPrice();
    if (block.timestamp - timestamp > HEARTBEAT) revert StalePrice();

    return uint256(ethPrice) * 1e18 / 10 ** uint256(AggregatorV3Interface(oracle).decimals());
}
```

## Proof of Concept
**Foundry fork test plan (Arbitrum mainnet fork):**

1. Fork Arbitrum at a block just before a known sequencer outage (or simulate by manipulating `block.timestamp` and mocking `latestRoundData` to return a timestamp `> HEARTBEAT` seconds in the past).
2. Deploy `ChainlinkOracleForRSETHPoolCollateral` with the wstETH/ETH Chainlink feed.
3. Mock `latestRoundData` to return: `roundID = answeredInRound = 100`, `timestamp = block.timestamp - 7200` (2 hours stale), `ethPrice = 1.2e18`.
4. Call `RSETHPoolV3.deposit(wstETH, 100e18, "")` as an unprivileged address.
5. Assert that `wrsETH.balanceOf(attacker)` reflects the 1.2 ETH/wstETH rate rather than the current 1.1 ETH/wstETH rate.
6. Confirm the stale price passes both existing checks (`answeredInRound == roundID`, `timestamp != 0`) and no revert occurs.
7. Quantify the excess wrsETH minted relative to a deposit at the correct current price, demonstrating extraction of value from existing holders.
