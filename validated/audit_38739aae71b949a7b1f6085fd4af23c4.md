Audit Report

## Title
Missing `msg.sender == account` Validation Allows Forced Fee Extraction from Any Claimant - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary

`MerkleDistributor.claim` accepts an arbitrary `account` parameter and processes the full claim lifecycle without verifying that `msg.sender == account`. Any external caller holding a valid Merkle proof for a victim can trigger the victim's claim, causing the protocol fee (up to 10%) to be deducted from the victim's entitled tokens and routed to `protocolTreasury`. The victim's claim state is permanently consumed, and they receive fewer tokens than their full entitlement with no recourse.

## Finding Description

`MerkleDistributor.claim` at `contracts/utils/MerkleDistributor/MerkleDistributor.sol` L97–147 is `external` and accepts `account` as a caller-supplied parameter. The function performs Merkle proof verification, fee deduction, and token transfer without any check that `msg.sender == account`:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    // ... no msg.sender == account check ...
    uint256 fee = (claimableAmount * feeInBPS) / 10_000;
    uint256 amountToSend = claimableAmount - fee;
    IERC20(token).safeTransfer(account, amountToSend);
    IERC20(token).safeTransfer(protocolTreasury, fee);
``` [1](#0-0) 

The sibling contract `KernelMerkleDistributor._processClaim` explicitly guards against this exact scenario:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
``` [2](#0-1) 

`MerkleDistributor.sol` has no equivalent guard. Additionally, unlike `KernelMerkleDistributor` which conditionally transfers the fee only `if (fee > 0)`, `MerkleDistributor` unconditionally calls `safeTransfer(protocolTreasury, fee)` even when `fee == 0`, though the primary impact arises when `feeInBPS > 0`. [3](#0-2) 

Merkle proofs for all eligible accounts are published off-chain by design (users need them to claim), making every eligible account's proof publicly available. The attacker supplies a victim's `(index, account, cumulativeAmount, merkleProof)` tuple — all public data — and the contract processes the claim as legitimate.

## Impact Explanation

**High — Theft of unclaimed yield.**

When `feeInBPS > 0` (up to `MAX_FEE_IN_BPS = 1000`, i.e., 10%), an attacker who forces a victim's claim causes the victim to permanently lose up to 10% of their entitled distribution. The fee is transferred to `protocolTreasury` rather than to the victim. The victim's `lastClaimedIndex` and `cumulativeAmount` are updated, marking the claim as consumed. The victim cannot re-claim the lost amount. This constitutes direct, permanent theft of unclaimed yield from an arbitrary number of eligible accounts. [4](#0-3) [5](#0-4) 

## Likelihood Explanation

The attack requires only:
1. A valid Merkle proof for the target account — publicly available by design.
2. A single unprivileged transaction with no special roles or capital.

The attacker bears only gas cost and gains nothing directly, but can systematically force fee extraction from every eligible account in the published Merkle tree in one scripted sweep. The only precondition that limits impact is `feeInBPS > 0`; if the owner has set a non-zero fee (which is the purpose of the fee mechanism), the attack is immediately exploitable. The attack is trivially repeatable across all leaves.

## Recommendation

Add the same caller restriction present in `KernelMerkleDistributor._processClaim` at the top of the `claim` function in `MerkleDistributor.sol`, before Merkle proof verification:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

Also add the `Unauthorized` error to the `IMerkleDistributor` interface in `MerkleDistributor.sol`, and conditionally transfer the fee only when `fee > 0` to match the pattern in `KernelMerkleDistributor`. [2](#0-1) 

## Proof of Concept

**Setup:**
- Owner deploys `MerkleDistributor`, sets `feeInBPS = 1000` (10%), publishes a Merkle root.
- Alice is entitled to 1000 tokens; her leaf is `keccak256(abi.encodePacked(1, alice, 1000))` and her proof is public.

**Attack sequence:**
1. Attacker calls `MerkleDistributor.claim(1, alice, 1000, aliceProof)` from any EOA.
2. Contract verifies the proof (valid — proof is for Alice's leaf, not the caller).
3. `claimableAmount = 1000`, `fee = 100`, `amountToSend = 900`.
4. 900 tokens transferred to Alice; 100 tokens transferred to `protocolTreasury`.
5. `userClaims[alice].lastClaimedIndex = 1` — Alice's claim is permanently consumed.
6. Alice loses 100 tokens she would have received in full had she claimed herself.

**Foundry test plan:**
```solidity
function test_forcedClaimExtractsFee() public {
    // Setup: deploy MerkleDistributor, set feeInBPS=1000, fund contract, set merkle root
    // Build leaf: keccak256(abi.encodePacked(index, alice, amount))
    // Generate valid proof for alice
    uint256 aliceBalanceBefore = token.balanceOf(alice);
    uint256 treasuryBalanceBefore = token.balanceOf(treasury);
    vm.prank(attacker); // attacker, not alice
    distributor.claim(index, alice, amount, aliceProof);
    assertEq(token.balanceOf(alice), aliceBalanceBefore + amount * 9000 / 10_000);
    assertEq(token.balanceOf(treasury), treasuryBalanceBefore + amount * 1000 / 10_000);
    // Alice's claim is consumed; she cannot reclaim the lost 10%
    assertTrue(distributor.isClaimed(index, alice));
}

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableAmount = cumulativeAmount - userClaims[account].cumulativeAmount;

        // Ensure there is something to claim
        if (claimableAmount == 0) {
            revert NoTokensToClaim();
        }

        // Update user claim info, and send the token.
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeAmount = cumulativeAmount;

        // Send the claimable amount to the user - deducting the fee
        uint256 fee = (claimableAmount * feeInBPS) / 10_000;
        uint256 amountToSend = claimableAmount - fee;

        IERC20(token).safeTransfer(account, amountToSend);

        // Send the fee to the protocol treasury
        IERC20(token).safeTransfer(protocolTreasury, fee);

        emit Claimed(index, account, claimableAmount);
    }
