Audit Report

## Title
Stale `pendingStateGuard` Not Cleared in `purgeStateGuardRole` Allows Unauthorized Oracle Role Takeover — (File: `smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

## Summary
`purgeStateGuardRole` deletes `stateGuard[feedId]` but leaves `pendingStateGuard[feedId]` intact. A previously nominated address can call the permissionless `acceptStateGuardRole` after the guard has purged itself, installing itself as the new `stateGuard` without ADMIN consent. This unauthorized guard can then call `setPriceGuard` to set arbitrary price bounds, causing bad-price execution in any Metric OMM pool consuming that feed.

## Finding Description
`OracleBase` implements a two-step guard-transfer pattern: `setStateGuardRole` writes `pendingStateGuard[feedId]`, and `acceptStateGuardRole` promotes the nominee and clears `pendingStateGuard`. A separate function `purgeStateGuardRole` is intended to remove the active guard entirely:

```solidity
// L120-124
function purgeStateGuardRole(bytes32 feedId) external checkRole(feedId) {
    delete stateGuard[feedId];
    emit StateGuardDeleted(feedId);
}
```

It deletes only `stateGuard[feedId]`. [1](#0-0) 

`acceptStateGuardRole` has no access control beyond the `pendingStateGuard` check: [2](#0-1) 

After `purgeStateGuardRole`, `checkRole` falls back to `ADMIN_ROLE` since `stateGuard[feedId]` is zero. [3](#0-2)  ADMIN can call `purgePendingStateGuardRole` to clean up, but this is not atomic with the purge — B can front-run it. [4](#0-3) 

Exploit path:
1. Guard A calls `setStateGuardRole(feedId, B)` → `pendingStateGuard[feedId] = B`.
2. Guard A calls `purgeStateGuardRole(feedId)` → `stateGuard[feedId] = address(0)`, `pendingStateGuard[feedId]` remains B.
3. B calls `acceptStateGuardRole(feedId)` → passes the only check (`pendingStateGuard[feedId] == msg.sender`), installs itself as `stateGuard`.
4. B calls `setPriceGuard(feedId, 1, type(uint128).max)` → disables effective price clamping.

## Impact Explanation
The unauthorized `stateGuard` can set `priceGuard[feedId]` to `{min: 1, max: type(uint128).max}`, effectively disabling price bound enforcement. [5](#0-4)  Any stale, inverted, or manipulated price pushed to the feed will pass the guard and be returned via the `price(feedId, pool)` path to Metric OMM pools during swaps, constituting bad-price execution and direct loss of trader principal. This meets the Critical/High impact threshold under the allowed impact gate (bad-price execution, admin-boundary break).

## Likelihood Explanation
The trigger sequence is realistic and requires no special privilege beyond being a previously nominated address. Guard A calling `purgeStateGuardRole` is the natural "undo" action when a nomination needs to be cancelled (e.g., B is compromised). The race between B's `acceptStateGuardRole` and ADMIN's `purgePendingStateGuardRole` is non-atomic and may not be known to ADMIN if the nomination was not publicly announced. B can monitor the mempool and front-run ADMIN's cleanup. The condition is repeatable for any feed with a stale pending nomination.

## Recommendation
In `purgeStateGuardRole`, also delete `pendingStateGuard[feedId]`:

```diff
function purgeStateGuardRole(bytes32 feedId) external checkRole(feedId) {
    delete stateGuard[feedId];
+   delete pendingStateGuard[feedId];
    emit StateGuardDeleted(feedId);
}
```

This atomically cancels any in-flight nomination when the active guard is removed, eliminating the race window entirely.

## Proof of Concept
```solidity
// 1. Guard A nominates B
vm.prank(guardA);
oracle.setStateGuardRole(feedId, address(B));
assertEq(oracle.pendingStateGuard(feedId), address(B));

// 2. Guard A purges itself (intending to cancel the nomination)
vm.prank(guardA);
oracle.purgeStateGuardRole(feedId);
assertEq(oracle.stateGuard(feedId), address(0));
// pendingStateGuard is NOT cleared:
assertEq(oracle.pendingStateGuard(feedId), address(B)); // still set

// 3. B accepts — no ADMIN consent required, front-runs purgePendingStateGuardRole
vm.prank(address(B));
oracle.acceptStateGuardRole(feedId);
assertEq(oracle.stateGuard(feedId), address(B)); // B is now unauthorized guard

// 4. B disables the price guard
vm.prank(address(B));
oracle.setPriceGuard(feedId, 1, type(uint128).max);
// Any price, including stale/manipulated, now passes the guard
// and will be consumed by Metric OMM pools on the next swap via price(feedId, pool)
```
