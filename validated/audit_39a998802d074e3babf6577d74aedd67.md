Audit Report

## Title
Validator Exit Mid-Checkpoint Creates ETH Accounting Gap That Triggers Protocol-Wide Pause — (`contracts/NodeDelegator.sol`, `contracts/LRTOracle.sol`)

## Summary

`getETHDistributionData` accounts for ETH staked via EigenLayer exclusively through `getEffectivePodShares`, which sums `stakedButUnverifiedNativeETH` and EigenLayer's live `withdrawableShare`. When a validator exits after a checkpoint starts, calling the permissionless `verifyCheckpointProofs` with a 0-balance proof immediately reduces `withdrawableShare` by 32 ETH, while the swept ETH sitting in the EigenPod is invisible to all accounting. Any caller can then invoke the public `updateRSETHPrice`, which computes a depressed `newRsETHPrice`; if the drop exceeds `pricePercentageLimit`, `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` are all paused, freezing user deposits and withdrawals until an admin manually unpauses.

## Finding Description

**ETH accounting path.** `_getTotalEthInProtocol` calls `getTotalAssetDeposits(ETH)`, which delegates to `getETHDistributionData`. [1](#0-0) 

That function counts only `nodeDelegatorQueue[i].balance` (the NDC contract) and `getEffectivePodShares()`. The EigenPod is a separate contract; its balance is never added. [2](#0-1) 

**`getEffectivePodShares` internals.** The function returns `stakedButUnverifiedNativeETH + withdrawableShare`, where `withdrawableShare` is read live from `DelegationManager.getWithdrawableShares`. [3](#0-2) 

`stakedButUnverifiedNativeETH` is incremented on `stake32Eth` and decremented on `verifyWithdrawalCredentials`. For a validator whose credentials have already been verified (i.e., any active validator), this field is 0 for that validator's 32 ETH. [4](#0-3) [5](#0-4) 

**The accounting gap.** EigenLayer's checkpoint records the pod's ETH balance at `startCheckpoint` time. If a validator exits *after* the checkpoint starts, the 32 ETH is swept to the EigenPod after `podBalanceGwei` was snapshotted. When `verifyCheckpointProofs` is called with the 0-balance proof, EigenLayer immediately applies a −32 ETH share delta to `withdrawableShare`. The swept 32 ETH in the EigenPod is not part of `podBalanceGwei` and will not be credited until the *next* checkpoint. During this window, `getEffectivePodShares` understates protocol ETH by 32 ETH per exited validator.

**`verifyCheckpointProofs` is permissionless.** The EigenPod interface explicitly states: *"Anyone can call this method to submit proofs towards the current checkpoint."* [6](#0-5) 

**Pause trigger.** `updateRSETHPrice` is `public whenNotPaused`, callable by any EOA. [7](#0-6) 

`_updateRsETHPrice` computes `newRsETHPrice = (totalETHInProtocol - fee) / rsethSupply`. If `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit × highestRsethPrice`, all three contracts are paused and the function returns without updating `rsETHPrice`. [8](#0-7) 

**Complete permissionless exploit sequence:**
1. Operator starts a checkpoint (`startCheckpoint` — routine operational event).
2. A validator exits; 32 ETH is swept to the EigenPod (observable on-chain).
3. Attacker calls `eigenPod.verifyCheckpointProofs(...)` with the 0-balance proof — permissionless.
4. Attacker calls `LRTOracle.updateRSETHPrice()` — public.
5. `newRsETHPrice` is understated; if the drop exceeds `pricePercentageLimit`, `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` are all paused.

No existing guard prevents this: there is no minimum delay between proof submission and price update, no check for an active checkpoint, and no inclusion of EigenPod balance in the accounting.

## Impact Explanation

**Medium — Temporary freezing of funds.** All user deposits and withdrawals are halted. The pause is not self-healing: it persists until `onlyLRTAdmin` manually calls `unpause()` on each of the three contracts. The ETH is not lost — it is recovered when the next checkpoint is finalized — but users are locked out for the entire duration. [9](#0-8) [10](#0-9) 

## Likelihood Explanation

- Validator exits are routine operational events, observable on-chain by anyone.
- `verifyCheckpointProofs` is permissionless; any EOA can submit the 0-balance proof immediately after observing the exit.
- `updateRSETHPrice` is public; any EOA can call it in the same block.
- Exploitability scales inversely with validator count: with `pricePercentageLimit = 1%` (1e16), a protocol with fewer than 100 active validators is vulnerable to a single exit triggering the pause. Multiple simultaneous exits amplify the effect at any scale.
- No front-running, no privileged access, no governance capture required.

## Recommendation

1. **Include EigenPod ETH in accounting.** In `getEffectivePodShares` (or `getETHDistributionData`), add `address(eigenPod).balance` so swept-but-not-yet-checkpointed ETH is not invisible to the price oracle.
2. **Alternatively**, track the EigenPod balance at checkpoint start and subtract it from the share delta to avoid double-counting, mirroring EigenLayer's own internal logic.
3. **Tighten the pause trigger.** Require the price to remain below `highestRsethPrice - pricePercentageLimit` for more than one oracle update before pausing, to avoid single-block transient drops triggering a freeze.

## Proof of Concept

```solidity
// Foundry fork test outline
function test_validatorExitMidCheckpointPausesProtocol() external {
    // 1. Fork mainnet; NDC has N < 100 active validators, rsETH supply > 0,
    //    pricePercentageLimit = 1e16 (1%)

    // 2. Operator starts checkpoint
    vm.prank(operator);
    nodeDelegator.startCheckpoint(false);

    // 3. Simulate validator exit: advance beacon state so one validator has
    //    0 effective balance; 32 ETH is swept to eigenPod address

    // 4. Attacker submits 0-balance checkpoint proof (permissionless)
    vm.prank(attacker);
    eigenPod.verifyCheckpointProofs(balanceContainerProof, zeroBalanceProofs);
    // withdrawableShare is now reduced by 32 ETH;
    // eigenPod.balance holds the 32 ETH but is not counted

    // 5. Attacker calls updateRSETHPrice
    vm.prank(attacker);
    lrtOracle.updateRSETHPrice();

    // 6. Assert all three contracts are paused
    assertTrue(lrtDepositPool.paused());
    assertTrue(lrtWithdrawalManager.paused());
    assertTrue(lrtOracle.paused());
}
```
