Audit Report

## Title
Nested Unbounded Loop in `NodeDelegator.getAssetUnstaking()` Called Per-Asset Per-NDC Causes Gas Exhaustion in Deposits and Price Updates - (File: contracts/NodeDelegator.sol)

## Summary

`NodeDelegator.getAssetUnstaking()` performs a nested loop over all EigenLayer-queued withdrawals and their strategies, making multiple external calls per iteration. This function is invoked for every NDC for every supported asset inside the publicly reachable `updateRSETHPrice()` and on every user deposit via `depositETH()`/`depositAsset()`. The protocol's own cap on uncompleted withdrawals can be bypassed by EigenLayer forced undelegations, and the gas cost scales multiplicatively with assets × NDCs × queued withdrawals × strategies, creating a realistic path to block gas limit exhaustion.

## Finding Description

`NodeDelegator.getAssetUnstaking()` fetches all queued withdrawals from EigenLayer's `DelegationManager` and iterates over them in a nested loop, calling `strategy.underlyingToken()` and `strategy.sharesToUnderlyingView()` per strategy per withdrawal: [1](#0-0) 

This function is called for every NDC inside `getAssetDistributionData()`: [2](#0-1) 

And for every NDC inside `getETHDistributionData()`: [3](#0-2) 

`getTotalAssetDeposits()` calls one of these two functions per asset: [4](#0-3) 

`LRTOracle._getTotalEthInProtocol()` calls `getTotalAssetDeposits()` for every supported asset in a loop: [5](#0-4) 

`updateRSETHPrice()` is publicly callable with no access restriction beyond `whenNotPaused`: [6](#0-5) 

Every user deposit triggers the same chain through `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()`: [7](#0-6) [8](#0-7) 

The protocol attempts to mitigate this by capping `maxUncompletedWithdrawalCount` at 80, and the comment explicitly acknowledges the gas ceiling concern: [9](#0-8) 

However, the comment itself notes "Need buffer for theoretical operator forced undelegations (ndc count * asset count = 15)" — EigenLayer forced undelegations bypass the protocol's own counter, pushing the real queued withdrawal count above 80. Additionally, the threshold was computed for a fixed number of NDCs and assets; adding more supported assets or raising `maxNodeDelegatorLimit` reduces the safe threshold without updating the cap.

## Impact Explanation

The gas cost scales as O(assets × NDCs × queued_withdrawals × strategies). When this exceeds the block gas limit, `depositETH()` and `depositAsset()` revert for all users, and `updateRSETHPrice()` becomes uncallable. This constitutes **Medium: Unbounded gas consumption** and **Medium: Temporary freezing of funds** — users cannot deposit, and the rsETH exchange rate becomes stale, breaking fee accrual and withdrawal pricing.

## Likelihood Explanation

The protocol already operates with multiple supported assets (stETH, ETHx, sfrxETH, ETH) and up to 10 NDCs. With 6 assets, 10 NDCs, and 80 queued withdrawals (8 per NDC, 2 strategies each), a single `updateRSETHPrice()` call executes approximately 6 × 10 × 17 = 1,020 external calls (~2.1M gas cold for external calls alone, plus loop overhead and ABI decoding). EigenLayer forced undelegations — which the protocol explicitly acknowledges as possible — can push the count above 80 without triggering the protocol's counter, crossing the gas limit. No attacker action is required; normal protocol growth or an EigenLayer operator forced undelegation is sufficient.

## Recommendation

1. Cache the result of `getQueuedWithdrawals()` per NDC once per transaction and reuse it across all asset queries, rather than re-fetching and re-iterating EigenLayer's queue once per asset per NDC.
2. Store a running `assetUnstaking` tally updated incrementally on each `initiateUnstaking` / `completeUnstaking` event, eliminating the need to iterate EigenLayer's queue on every read path.
3. If on-chain iteration is retained, introduce a `maxLoops` / pagination parameter to `updateRSETHPrice()` and the distribution data functions.

## Proof of Concept

**Setup:** 6 supported assets, 10 NDCs, 80 total queued EigenLayer withdrawals (8 per NDC, 2 strategies each). EigenLayer forces undelegation on all 10 NDCs, adding 10 more withdrawals beyond the protocol counter (total: 90).

**Call sequence:**
1. Any external account calls `updateRSETHPrice()` (no access restriction).
2. `_getTotalEthInProtocol()` loops over 6 assets.
3. For each asset, `getTotalAssetDeposits()` → `getAssetDistributionData()` loops over 10 NDCs.
4. For each NDC, `getAssetUnstaking()` calls `DelegationManager.getQueuedWithdrawals()` (1 external call) then loops over 9 withdrawals × 2 strategies = 18 external calls to strategy contracts.
5. Total external calls: 6 × 10 × (1 + 18) = **1,140 external calls**.
6. At 2,100 gas cold: ~2.4M gas for external calls alone, plus loop overhead, storage reads, and ABI decoding of withdrawal arrays.

**Foundry fork test plan:** Deploy a fork of mainnet, register 6 assets and 10 NDCs, simulate 80 queued withdrawals via `vm.mockCall` on `DelegationManager.getQueuedWithdrawals`, then call `updateRSETHPrice()` and `depositETH()` with `gas: 30_000_000` and assert they revert with out-of-gas. Increase withdrawal count incrementally to find the exact threshold.

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
