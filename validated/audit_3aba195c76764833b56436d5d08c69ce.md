### Title
Gateway Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Threshold Check — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful admission check for V3 transaction L2 gas price uses the **current block's** `l2_gas_price` as the reference price, but the transaction will actually be executed at the **next block's** `next_l2_gas_price` (computed by the EIP-1559 mechanism). When the EIP-1559 price is rising (gas usage above target), the gateway's threshold is lower than the real execution price, so transactions whose `max_price_per_unit` falls between the two values are admitted into the mempool but will be rejected by the blockifier at execution time. The inverse holds when the price is falling: valid transactions are rejected at the gateway. A developer TODO in the code explicitly acknowledges this discrepancy.

---

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the L2 gas price reference from the latest committed block's `BlockInfo`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  ~L229-240

    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;          // ← price used for block N, not block N+1
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The threshold is then computed as:

```rust
// ~L367-372
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...) }
``` [2](#0-1) 

The **correct** reference is `next_l2_gas_price`, which is stored in the block header and computed by `calculate_next_l2_gas_price_for_fin` at the end of each block:

```rust
// crates/apollo_consensus_orchestrator/src/build_proposal.rs ~L326-333
let next_l2_gas_price = calculate_next_l2_gas_price_for_fin(
    args.l2_gas_price,
    args.build_param.height,
    info.l2_gas_used,
    args.override_l2_gas_price_fri,
    &args.min_l2_gas_price_per_height,
    args.fee_actual,
);
``` [3](#0-2) 

The EIP-1559 formula can move the price by up to `price / gas_price_max_change_denominator` (denominator = 48, ≈ 2%) per block:

```rust
// crates/apollo_consensus_orchestrator/src/fee_market/mod.rs ~L124-129
let denominator =
    gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
let price_change = (price_u256 * gas_delta) / denominator;
``` [4](#0-3) 

The `SyncStateReaderFactory` always supplies the latest committed block number to `GatewayFixedBlockSyncStateClient`, so `get_block_info()` returns block N's `l2_gas_price`, not block N's `next_l2_gas_price`:

```rust
// crates/apollo_gateway/src/sync_state_reader.rs ~L545-548
let gateway_fixed_block_sync_state_client = GatewayFixedBlockSyncStateClient::new(
    self.shared_state_sync_client.clone(),
    latest_block_number,   // block N
);
``` [5](#0-4) 

The `BlockInfo` mapping in the test confirms `l2_gas_price.price_in_fri` (block N's execution price) is what is returned, not `next_l2_gas_price`: [6](#0-5) 

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions or rejects valid transactions before sequencing.**

**Case 1 — Price rising (gas usage above target):**
`next_l2_gas_price > l2_gas_price`. The gateway threshold is anchored to the lower, stale price. Any V3 transaction whose `max_price_per_unit` satisfies:

```
(min_gas_price_percentage% × l2_gas_price_N) ≤ tx_price < (min_gas_price_percentage% × next_l2_gas_price_N)
```

passes gateway admission and enters the mempool, but will be rejected by the blockifier at execution time because `tx_price < next_l2_gas_price_N`. This is the direct analog of the original report: the gateway uses an optimistic (lower) price for a threshold that should use the conservative (higher) price, admitting transactions that should be rejected.

**Case 2 — Price falling (gas usage below target):**
`next_l2_gas_price < l2_gas_price`. The gateway threshold is too high, causing valid transactions (whose price is above the real next-block price but below the stale threshold) to be incorrectly rejected.

---

### Likelihood Explanation

**Likelihood: Medium.**

The EIP-1559 mechanism is always active once the chain is past genesis. Whenever sustained gas usage is above or below the target (`gas_target = 1,040,000,000` from versioned constants), the price drifts and the stale-price window opens. The maximum per-block drift is ≈2% (denominator 48), so the exploitable band is narrow but persistent across every block where usage deviates from target. No privileged access is required; any user can submit a V3 transaction. [7](#0-6) 

---

### Recommendation

Replace the `l2_gas_price` read with `next_l2_gas_price` from the latest committed block header, exactly as the TODO comment instructs:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
// Fix: read block_header_without_hash.next_l2_gas_price instead of
//      block_info.gas_prices.strk_gas_prices.l2_gas_price
```

`GatewayFixedBlockStateReader::get_block_info` (or a new dedicated method) should expose `next_l2_gas_price` from `BlockHeaderWithoutHash`, and `validate_resource_bounds` should pass that value to `validate_tx_l2_gas_price_within_threshold`.

---

### Proof of Concept

1. Observe the latest committed block N with `l2_gas_price = P` and `next_l2_gas_price = P' = P × (1 + δ)` where `δ > 0` (gas usage above target).
2. Construct a V3 `invoke` transaction with:
   - `max_price_per_unit = floor(min_gas_price_percentage% × P)` — exactly at the gateway threshold.
3. Submit to the gateway. `validate_resource_bounds` computes `threshold = min_gas_price_percentage% × P` and passes the check (`tx_price >= threshold`). The transaction is admitted.
4. The batcher builds block N+1 with `l2_gas_price = P'`. The blockifier rejects the transaction because `tx_price < P'`.
5. The transaction entered the mempool and consumed sequencer resources despite being unexecutable in the next block. [8](#0-7) [9](#0-8)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
