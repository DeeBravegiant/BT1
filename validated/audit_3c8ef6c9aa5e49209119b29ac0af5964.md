### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect All Block Fees to Arbitrary Address - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the address of the block builder / sequencer). This field is proposer-supplied over the P2P wire, but `is_proposal_init_valid` never checks it against any locally-trusted reference. `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, which is the recipient of every fee transfer executed in the block. A malicious proposer can set `builder` to an attacker-controlled address; all validators will accept the proposal, execute transactions, and commit a block whose fee token balances and `sequencer_address` are wrong.

---

### Finding Description

`ProposalInit` is defined with a `builder: ContractAddress` field: [1](#0-0) 

When a validator node receives a proposal, `is_proposal_init_valid` checks `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri`. The `builder` field is **never checked**: [2](#0-1) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info`, which maps the unchecked `init.builder` directly to `sequencer_address`: [3](#0-2) 

The resulting `BlockInfo` is forwarded to the batcher via `ValidateBlockInput`: [4](#0-3) 

Inside the blockifier, every fee transfer uses `block_context.block_info.sequencer_address` as the recipient: [5](#0-4) 

The proposer side sets `builder` from its own `builder_address` argument: [6](#0-5) 

There is no corresponding check on the validator side that `init.builder` equals the locally-configured builder address. The `ProposalInitValidation` struct does not even carry a `builder` field to compare against: [7](#0-6) 

The Tendermint layer does verify that `init.proposer` matches the expected committee proposer for the round: [8](#0-7) 

But `builder` is a separate, unchecked field.

---

### Impact Explanation

**Critical — Incorrect fee/balance with economic impact; wrong state/storage value committed.**

Every transaction fee in the block is transferred to `sequencer_address = init.builder`. If a malicious proposer sets `builder` to an attacker-controlled address, all fee token balances in the committed state are wrong: the attacker's address is credited, the legitimate sequencer receives nothing. The committed block header also records the wrong `sequencer_address`. Because all validators independently re-execute with the same attacker-supplied `builder`, they all compute the same (corrupted) block hash and reach consensus on the corrupted state. The corruption is permanent and irreversible once the block is finalized.

---

### Likelihood Explanation

**High.** Any validator that wins the proposer role for a round — a normal, expected event in round-robin or weighted selection — can exploit this. No external attacker capability is required beyond being a current-round proposer. The attack is silent: the proposal passes all existing validation checks and produces a valid `ProposalFin` commitment that matches what validators compute (because they all use the attacker-supplied `builder`).

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`, analogous to how `l2_gas_price_fri` is validated against the locally-computed value:

```rust
// In ProposalInitValidation:
pub builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

The validator node must populate `proposal_init_validation.builder` from its own locally-configured `builder_address` (the same source used in `ProposalBuildArguments.builder_address`), not from the incoming `ProposalInit`.

---

### Proof of Concept

1. Validator A is the current-round proposer.
2. A sets `init.builder = attacker_address` (any address it controls) instead of the legitimate sequencer address.
3. A broadcasts the `ProposalInit` with this modified `builder`.
4. Each validator node calls `is_proposal_init_valid` — passes, because `builder` is not checked.
5. Each validator calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: attacker_address, ... }`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to `attacker_address`.
7. The batcher returns a `partial_block_hash` computed over the state that includes fees credited to `attacker_address`.
8. All validators compute the same `partial_block_hash` and send matching `ProposalFin` commitments.
9. Consensus reaches decision; the block is committed with wrong fee balances and wrong `sequencer_address` in the block header.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
