Audit Report

## Title
Stale `rsETHPrice` Enables Yield Theft via Deposit Before Public `updateRSETHPrice()` Call - (File: contracts/LRTOracle.sol)

## Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function that updates the stored `rsETHPrice` state variable. Between calls, `rsETHPrice` grows stale while underlying LST assets continue accruing yield and their live Chainlink prices rise. Because `LRTDepositPool.getRsETHAmountToMint()` divides the live asset price by the stale `rsETHPrice`, an attacker can deposit at the artificially low denominator, receive more rsETH than fair value, then call `updateRSETHPrice()` themselves to realize the accumulated yield — diluting all existing rsETH holders.

## Finding Description

`getRsETHAmountToMint()` computes the rsETH mint amount as:

```solidity
// LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

`lrtOracle.getAssetPrice(asset)` reads a **live** Chainlink price, while `lrtOracle.rsETHPrice()` returns the **stale** cached value last written by `_updateRsETHPrice()`. [1](#0-0) 

`updateRSETHPrice()` is unconditionally public:

```solidity
// LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [2](#0-1) 

The only guard inside `_updateRsETHPrice()` that could block a public caller is the `pricePercentageLimit` check: [3](#0-2) 

This check only fires when `pricePercentageLimit > 0` **and** the price increase exceeds `pricePercentageLimit × highestRsethPrice`. For normal daily LST yield (~0.011%/day at 4% APY), the price increase is far below any operationally reasonable limit (e.g., 1% = 1e16), so the guard does not trigger and the public caller succeeds.

**Exploit path:**
1. Attacker observes on-chain that live asset prices have risen since the last `updateRSETHPrice()` call, meaning `rsETHPrice` is stale-low.
2. Attacker calls `LRTDepositPool.depositETH{value: X}(0, "")`. The mint formula uses the stale (lower) `rsETHPrice` denominator, minting more rsETH than the deposited ETH is worth at the true current price.
3. Attacker calls `LRTOracle.updateRSETHPrice()`. `_updateRsETHPrice()` recomputes the price from live TVL divided by the now-larger rsETH supply (inflated by the attacker's over-mint), setting a new `rsETHPrice` that is slightly lower than the true fair price — the difference is borne by all pre-existing holders.
4. Attacker's rsETH balance is worth more ETH per token than they deposited. Existing holders' share of TVL is diluted by the attacker's excess tokens.

The `previousTVL` calculation inside `_updateRsETHPrice()` uses `rsethSupply.mulWad(rsETHPrice)` with the stale price: [4](#0-3) 

After the attacker's deposit, `rsethSupply` is inflated, so `previousTVL` is overstated, which reduces the computed `rewardAmount` and thus the protocol fee — a secondary benefit to the attacker.

## Impact Explanation

**High — Theft of unclaimed yield.**

Each update cycle, the yield accrued by existing rsETH holders (reflected in rising LST/ETH Chainlink prices) is partially captured by the attacker through over-minted rsETH shares. The dilution is permanent: once the attacker's excess rsETH exists, existing holders' proportional claim on TVL is reduced. The attack is repeatable every update interval with no privileged access required.

## Likelihood Explanation

**Medium.** The attacker requires no special role, no leaked key, and no oracle manipulation. All necessary information (live asset prices, current `rsETHPrice`, `highestRsethPrice`, `pricePercentageLimit`) is readable on-chain. The only precondition is that some yield has accrued since the last price update, which is true continuously. The attack is bounded per cycle by the yield accumulated in the interval but is fully repeatable. The `pricePercentageLimit` guard does not block it for normal daily yield magnitudes.

## Recommendation

1. **Atomic price refresh on deposit:** Call `_updateRsETHPrice()` (or an equivalent freshness check) at the start of `depositETH` / `depositAsset`, before computing `rsethAmountToMint`. This ensures the mint denominator always reflects current TVL.

2. **Compute mint amount from live TVL directly:** Replace `lrtOracle.rsETHPrice()` in `getRsETHAmountToMint()` with an inline computation using `_getTotalEthInProtocol()` and `rsETH.totalSupply()`, bypassing the stale cache entirely.

3. **Staleness window enforcement:** Revert deposits if `rsETHPrice` was last updated more than a configurable staleness threshold ago, forcing a price refresh before any deposit can proceed.

## Proof of Concept

**State:** `rsETHPrice = 1.010e18` (last updated 24 h ago). stETH/ETH Chainlink rate has since risen so true rsETH price is `1.0101e18` (≈0.01% daily yield on a large TVL).

**Steps (single EOA, no flash loan needed):**

```
1. attacker.depositETH{value: 1000 ether}(0, "")
   - getAssetPrice(ETH_TOKEN) = 1e18  (live)
   - rsETHPrice()             = 1.010e18  (stale)
   - minted = 1000e18 * 1e18 / 1.010e18 = 990.099 rsETH
   - fair mint at true price  = 1000e18 * 1e18 / 1.0101e18 = 990.001 rsETH
   - excess minted            ≈ 0.098 rsETH

2. attacker.updateRSETHPrice()
   - newRsETHPrice = totalETHInProtocol / (originalSupply + 990.099)
   - price increase < pricePercentageLimit → no revert
   - rsETHPrice updated; attacker's 990.099 rsETH now priced at new rate

3. Attacker holds rsETH worth ~0.098 rsETH × newRsETHPrice more than deposited,
   at the expense of all pre-existing holders whose per-token TVL claim is diluted.
```

**Foundry fork test outline:**
```solidity
function testStaleYieldTheft() public fork {
    uint256 snapshot = vm.snapshot();
    // warp forward 1 day to let yield accrue without updating rsETHPrice
    vm.warp(block.timestamp + 1 days);
    uint256 staleMint = depositPool.getRsETHAmountToMint(ETH_TOKEN, 1000 ether);
    // now update price first, then compute fair mint
    oracle.updateRSETHPrice();
    uint256 fairMint  = depositPool.getRsETHAmountToMint(ETH_TOKEN, 1000 ether);
    assertGt(staleMint, fairMint, "attacker receives excess rsETH");
}
```
