### Title
Missing Vote Signature Verification Allows Forged Consensus Quorum — (`crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` accepts prevote and precommit messages from any peer that claims a committee-member address, without ever verifying the cryptographic signature carried in the `Vote` struct. An attacker who knows the committee composition can forge votes for any `ProposalCommitment`, manufacture a false quorum, and force the node to reach a consensus decision on an arbitrary block.

---

### Finding Description

`handle_vote` in `crates/apollo_consensus/src/single_height_consensus.rs` performs three checks before accepting a vote:

1. Height matches the current height.
2. The `vote.voter` address is a member of the committee.
3. The vote is not a duplicate or equivocation.

It explicitly skips the fourth, critical check — cryptographic signature verification — with a `TODO` comment:

```rust
// TODO(Asmaa): verify the signature
``` [1](#0-0) 

The `Vote` struct carries a `signature: RawSignature` field: [2](#0-1) 

The `apollo_signature_manager` crate already provides `verify_precommit_vote_signature` and `sign_precommit_vote`, which are tested and functional: [3](#0-2) 

Neither function is called anywhere in the vote-processing path. The manager's `handle_vote` only checks for parse errors and height before forwarding to `shc.handle_vote`: [4](#0-3) 

The committee-membership check (`vote.voter` is in the committee) is the only identity guard, but it only checks the *claimed* address — it does not verify that the sender actually controls the private key for that address.

---

### Impact Explanation

Once `handle_vote` accepts a forged vote, it is fed directly into the Tendermint state machine: [5](#0-4) 

When the state machine accumulates a quorum of forged precommits for an attacker-chosen `ProposalCommitment`, it emits `SMRequest::DecisionReached`: [6](#0-5) 

`DecisionReached` propagates to `context.decision_reached`, which calls `batcher.decision_reached()` and commits the block. A forged quorum therefore causes the node to commit a wrong block, producing wrong state roots, wrong receipts, wrong events, and wrong L1 messages — matching the **Critical: Wrong state, receipt, event, L1 message** impact class.

---

### Likelihood Explanation

- Committee addresses are public (published by the committee provider at each height).
- The attacker only needs to send gossip-layer `Vote` messages with `voter` set to any committee address and `signature` set to any bytes (including `RawSignature::default()`).
- No privileged access is required; any peer reachable by the gossip network can send these messages.
- The `// TODO` comment confirms the gap is known and intentional for now, meaning it is present in the current production code path.

---

### Recommendation

Before the `VoteStatus::New` branch in `handle_vote`, look up the committee member's public key by `vote.voter` and call `verify_precommit_vote_signature` (for precommits) or the equivalent prevote verifier. Reject the vote if verification fails and report the peer. The `apollo_signature_manager` crate already provides the necessary primitives: [7](#0-6) 

---

### Proof of Concept

1. Observe the target node's current consensus height `H` and round `R` (available from metrics or gossip).
2. Obtain the committee for height `H` (public via the committee provider).
3. Choose any `ProposalCommitment` value `C` (e.g., `Felt::ONE`).
4. For each committee member address `V_i`, craft:
   ```
   Vote {
       vote_type: VoteType::Precommit,
       height: H,
       round: R,
       proposal_commitment: Some(ProposalCommitment(C)),
       voter: V_i,
       signature: RawSignature::default(),   // any bytes; never checked
   }
   ```
5. Broadcast these votes to the target node via the p2p gossip channel.
6. `handle_vote` passes the committee-membership check (address is valid) and the `// TODO` skips signature verification; each vote is accepted as `VoteStatus::New`.
7. Once the quorum threshold is met, `upon_decision` fires and emits `DecisionReached` with commitment `C`.
8. The batcher commits the block corresponding to `C`, corrupting the chain state. [8](#0-7)

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }

    // shc - None if the height was just completed and we should drop the message.
    async fn handle_vote(
        &mut self,
        height: BlockNumber,
        shc: Option<&mut SingleHeightConsensus>,
        vote: (Result<Vote, ProtobufConversionError>, BroadcastedMessageMetadata),
        broadcast_channels: &mut BroadcastVoteChannel,
    ) -> Result<Requests, ConsensusError> {
        let (message, metadata) = match vote {
            (Ok(message), metadata) => {
                // TODO(matan): Hold onto report_sender for use in later errors by SHC.
                if broadcast_channels
                    .broadcast_topic_client
                    .continue_propagation(&metadata)
                    .now_or_never()
                    .is_none()
                {
                    error!("Unable to send continue_propagation. {:?}", metadata);
                }
                (message, metadata)
            }
            (Err(e), metadata) => {
                // Failed to parse consensus message. Report the peer and drop the vote.
                self.report_peer(broadcast_channels, &metadata);
                warn!(
                    "Failed to parse incoming consensus vote, dropping vote. Error: {e}. Vote \
                     metadata: {metadata:?}"
                );
                return Ok(VecDeque::new());
            }
        };

        // TODO(matan): We need to figure out an actual caching strategy under 2 constraints:
        // 1. Malicious - must be capped so a malicious peer can't DoS us.
        // 2. Parallel proposals - we may send/receive a proposal for (H+1, 0).
        match message.height.cmp(&height) {
            std::cmp::Ordering::Greater => {
                if self.cache.should_cache_vote(&height, 0, &message) {
                    trace!("Cache message for a future height. {:?}", message);
                    let duplicate_report = self.cache.cache_future_vote(message);
                    if let Err(duplicate_report) = duplicate_report {
                        warn!("Duplicate vote found: {:?}", duplicate_report);
                        self.report_peer(broadcast_channels, &metadata);
                    }
                }
                Ok(VecDeque::new())
            }
            std::cmp::Ordering::Less => {
                trace!("Drop message from past height. {:?}", message);
                Ok(VecDeque::new())
            }
            std::cmp::Ordering::Equal => match shc {
                Some(shc) => {
                    if self.cache.should_cache_vote(&height, shc.current_round(), &message) {
                        Ok(shc.handle_vote(message))
                    } else {
                        Ok(VecDeque::new())
                    }
                }
                None => {
                    trace!("Drop message from just completed height. {:?}", message);
                    Ok(VecDeque::new())
                }
            },
        }
    }
