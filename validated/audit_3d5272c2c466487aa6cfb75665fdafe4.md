### Title
Missing `ppuWellFormed` Bounds Checks for Dijkstra-Era Reference-Script Fee Parameters Allow Zero-Valued Updates That Corrupt Fee Calculation - (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs`)

---

### Summary

The `ppuWellFormed` implementation for `DijkstraEra` omits validity checks for the four new Dijkstra-era protocol parameters: `maxRefScriptSizePerTx`, `maxRefScriptSizePerBlock`, `refScriptCostStride`, and `refScriptCostMultiplier`. An unprivileged governance participant can submit a `ParameterChange` proposal that sets `maxRefScriptSizePerTx = 0` or `maxRefScriptSizePerBlock = 0`, which — once ratified — permanently disables all reference-script usage by making every transaction exceed the zero-byte limit. This is a direct analog of the external report's "fee/limit setter uses wrong bounds, rendering the mechanism ineffective."

---

### Finding Description

The `ConwayEraPParams` type class requires each era to implement `ppuWellFormed`, which is called by `actionWellFormed` in the GOV rule to reject malformed `ParameterChange` proposals before they enter the governance queue.

In `ConwayEra`, `ppuWellFormed` validates that critical size and fee parameters cannot be set to zero:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/PParams.hs:934-953
instance ConwayEraPParams ConwayEra where
  ppuWellFormed pv ppu =
    and
      [ isValid (/= 0) ppuMaxBBSizeL
      , isValid (/= 0) ppuMaxTxSizeL
      , isValid (/= 0) ppuMaxBHSizeL
      , isValid (/= 0) ppuMaxValSizeL
      , isValid (/= 0) ppuCollateralPercentageL
      ...
      ]
``` [1](#0-0) 

In `DijkstraEra`, the `ppuWellFormed` implementation copies the same checks but **does not add any checks for the four new Dijkstra parameters** (`maxRefScriptSizePerTx`, `maxRefScriptSizePerBlock`, `refScriptCostStride`, `refScriptCostMultiplier`):

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/PParams.hs:539-557
instance ConwayEraPParams DijkstraEra where

      , isValid (/= 0) ppuMaxValSizeL
      , isValid (/= 0) ppuCollateralPercentageL
      , isValid (/= EpochInterval 0) ppuCommitteeMaxTermLengthL
      , isValid (/= EpochInterval 0) ppuGovActionLifetimeL

      , ppu /= emptyPParamsUpdate
      , isValid (/= 0) ppuNOptL

There are no checks for `ppuMaxRefScriptSizePerTxL`, `ppuMaxRefScriptSizePerBlockL`, `ppuRefScriptCostStrideL`, or `ppuRefScriptCostMultiplierL`.

The `actionWellFormed` function in the GOV rule is the sole gate that calls `ppuWellFormed`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Gov.hs:393-399

  where
    isWellFormed = case ga of
      ParameterChange _ ppd _ -> ppuWellFormed pv ppd
      _ -> True
``` [3](#0-2)

If `maxRefScriptSizePerTx` or `maxRefScriptSizePerBlock` is set to `0` via a ratified governance action, every transaction that includes any reference script (even a single byte) will fail the `totalRefScriptSize <= 0` check, permanently freezing all DApp activity that relies on reference scripts.

Additionally, `tierRefScriptFee` contains a runtime `error` guard:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Tx.hs:127-128
tierRefScriptFee multiplier sizeIncrement
  | multiplier <= 0 || sizeIncrement <= 0 = error "Size increment and multiplier must be positive"
``` [6](#0-5) 

`refScriptCostStride` is typed as `NonZero Word32` (preventing zero), but `refScriptCostMultiplier` is typed as `PositiveInterval` (preventing ≤ 0). However, neither is validated in `ppuWellFormed`, meaning a governance proposal setting `maxRefScriptSizePerTx = 0` or `maxRefScriptSizePerBlock = 0` (both plain `Word32`) passes the well-formedness gate and can be ratified.

---

### Impact Explanation

**High — Permanent freezing of funds/withdrawals where recovery requires a hard fork.**

Once a `ParameterChange` proposal setting `maxRefScriptSizePerTx = 0` is ratified and enacted, every subsequent transaction that touches any UTxO containing a reference script will be rejected by `validateRefScriptSize`. Since the vast majority of Cardano DApps use reference scripts (introduced in Babbage), this effectively freezes all DApp-locked funds. Funds locked in Plutus script addresses that require reference-script witnesses cannot be spent or withdrawn. Recovery would require a new governance action to raise the limit back — but if the governance system itself relies on DApp infrastructure (e.g., DRep tooling using reference scripts), even that path may be impaired. In the worst case, a hard fork is needed to override the parameter.

---

### Likelihood Explanation

The attack path is fully reachable by any governance participant (DRep, SPO, or CC member) who can assemble the required voting threshold for a `ParameterChange` proposal. No privileged key or supermajority is required beyond the normal governance thresholds. The proposal itself is syntactically valid (zero is a legal `Word32`) and passes the current `ppuWellFormed` check because no check for these parameters exists. The attacker need only submit a `ParameterChange` with `ppuMaxRefScriptSizePerTxL .~ SJust 0` and obtain the required votes.

---

### Recommendation

Add non-zero guards for the four new Dijkstra parameters inside `ppuWellFormed` for `DijkstraEra`:

```haskell
instance ConwayEraPParams DijkstraEra where
  ppuWellFormed _pv ppu =
    and
      [ -- existing checks ...
        isValid (/= 0) ppuMaxBBSizeL
      , ...
        -- Add these:
      , isValid (/= 0) ppuMaxRefScriptSizePerTxL
      , isValid (/= 0) ppuMaxRefScriptSizePerBlockL
      ]
``` [2](#0-1) 

The `refScriptCostStride` (`NonZero Word32`) and `refScriptCostMultiplier` (`PositiveInterval`) types already enforce non-zero/positive at the type level, so no additional `ppuWellFormed` check is strictly needed for those two, but explicit checks would be consistent with the pattern used for other parameters.

---

### Proof of Concept

1. In `DijkstraEra`, submit a `ParameterChange` governance action:
   ```haskell
   let ppu = emptyPParamsUpdate & ppuMaxRefScriptSizePerTxL .~ SJust 0
   ga <- mkParameterChangeGovAction SNothing ppu
   proposal <- mkProposal ga
   submitProposal proposal  -- succeeds: ppuWellFormed has no check for this field
   ```
2. Obtain the required DRep/SPO/CC votes and pass two epochs.
3. After enactment, attempt any transaction that uses a reference script:
   ```haskell
   -- totalRefScriptSize = scriptSize > 0 = maxRefScriptSizePerTx
   -- => ConwayTxRefScriptsSizeTooBig is raised for every such tx
   submitTx tx  -- fails with ConwayTxRefScriptsSizeTooBig
   ```
4. All funds locked in Plutus script addresses requiring reference-script witnesses are now permanently frozen. [7](#0-6) [8](#0-7) [3](#0-2)
