### Title
Missing `max_l2_gas_amount` Validation for Declare Transactions in Gateway Admission — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on every incoming RPC transaction **except** `Declare`. An explicit empty branch silently skips the check for Declare transactions, allowing any user to submit a Declare with an arbitrarily large `l2_gas.max_amount` that bypasses the gateway's admission control.

### Finding Description

In `validate_resource_bounds`, after checking that the L2 gas price is above the minimum, the function checks whether `l2_gas.max_amount` exceeds the configured `max_l2_gas_amount` limit. However, the check is guarded by an explicit empty `if let RpcTransaction::Declare(_) = tx { }` block that causes Declare transactions to skip the check entirely:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { ... });
}
``` [1](#0-0) 

Invoke and DeployAccount transactions are correctly rejected when `max_amount > max_l2_gas_amount`. Declare transactions are not. The TODO comment confirms this is a known, unaddressed gap.

### Impact Explanation

A Declare transaction with `l2_gas.max_amount = u128::MAX` (or any value exceeding `max_l2_gas_amount`) passes stateless validation and is admitted to the mempool. The gateway's admission control invariant — that no transaction with an out-of-range gas bound enters the mempool — is broken for the Declare type. This matches the required impact:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

Downstream effects: the mempool can be filled with Declare transactions carrying unrealistic gas bounds, and fee estimation for those transactions will be computed against an unconstrained `max_amount`, potentially returning misleading authoritative-looking values to callers of `estimate_fee`.

### Likelihood Explanation

The trigger is fully unprivileged: any user with a valid Sierra class can craft a `RpcDeclareTransactionV3` with `resource_bounds.l2_gas.max_amount` set to an arbitrarily large value and submit it via the standard HTTP gateway endpoint. No special role, key, or peer relationship is required. The code path is exercised on every Declare submission.

### Recommendation

Remove the empty `if let RpcTransaction::Declare(_) = tx { }` guard and apply the same `max_l2_gas_amount` ceiling to Declare transactions as is applied to Invoke and DeployAccount transactions:

```rust
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If a different (higher) ceiling is intentionally desired for Declare transactions, introduce a separate `max_l2_gas_amount_declare` config field and enforce it explicitly rather than leaving the check absent.

### Proof of Concept

1. Obtain the gateway's configured `max_l2_gas_amount` (e.g., from the node config schema).
2. Construct a valid `RpcDeclareTransactionV3` with:
   ```
   resource_bounds.l2_gas.max_amount = u128::MAX
   resource_bounds.l2_gas.max_price_per_unit >= config.min_gas_price
   ```
3. Submit via `POST /add_declare_transaction`.
4. Observe that the gateway returns a transaction hash (admission success) instead of `MaxGasAmountTooHigh`.
5. Confirm the transaction appears in the mempool despite violating the `max_l2_gas_amount` limit.

The relevant admission path is:

`StatelessTransactionValidator::validate` → `validate_resource_bounds` → empty Declare branch → no `MaxGasAmountTooHigh` error returned. [2](#0-1) [3](#0-2)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
