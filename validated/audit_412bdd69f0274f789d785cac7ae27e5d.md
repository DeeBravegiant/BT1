Looking at the external report's vulnerability class — dynamically created entities becoming operational before proper authorization, with admin-change complexity and censorship risk — I need to find an analog in Cardano Ledger's governance system.

The closest analog is in the Conway era's Constitutional Committee (CC) lifecycle: an `UpdateCommittee` governance proposal acts as a "factory" that creates new committee members. The GOVCERT rule explicitly allows "potential future members" (cold credentials appearing in any pending `UpdateCommittee` proposal) to register hot keys immediately, before the proposal is ratified or enacted. Before protocol version 11, the GOV rule (which runs during block application) does **not** reject votes from these unelected members — only the MEMPOOL rule does. A block producer can bypass the mempool entirely.
