### Title
Single Byzantine Participant Can Permanently Prevent DKG/Resharing Completion via Unbounded Key Event Instance Abort — (File: `crates/contract/src/state/key_event.rs`)

---

### Summary

A single authenticated MPC participant, acting as a Byzantine node strictly below the signing threshold, can repeatedly abort active key event instances during Distributed Key Generation (`Initializing` state) or resharing (`Resharing` state) at negligible cost. Because there is no cap on abort attempts, no cooldown between aborts, and no bond requirement, the attacker can prevent DKG from ever completing, permanently freezing the network's ability to produce threshold signatures.

---

### Finding Description

`KeyEvent::vote_abort` in `crates/contract/src/state/key_event.rs` is the core implementation called by `MpcContract::vote_abort_key_event_instance` in `crates/contract/src/lib.rs` for both `Initializing` and `Resharing` protocol states. [1](#0-0) 

The function:
1. Verifies the caller is an authenticated participant via `verify_vote`.
2. Checks only that the caller has not already voted **success** in the current instance.
3. Immediately sets `self.instance = None`, destroying the active attempt. [2](#0-1) 

There is no:
- Threshold requirement (a single participant suffices).
- Rate limit or cooldown between successive abort calls.
- Bond or cost beyond NEAR gas fees.
- Cap on the total number of abort attempts (`next_attempt_id` increments unboundedly). [3](#0-2) 

After each abort, the leader must call `start_keygen_instance` (or `start_reshare_instance`) with the next `attempt_id`. Because NEAR contract state is public, the Byzantine participant can read the new `attempt_id` and abort the new instance within the 30-block timeout window (`key_event_timeout_blocks = 30`, i.e., ~30 seconds). [4](#0-3) 

The dispatch path in `state.rs` confirms the same `vote_abort` logic applies to both `Initializing` and `Resharing` states: [5](#0-4) 

The public contract entry point requires only that the caller is an attested participant: [6](#0-5) 

**Structural analog to the Taiko report**: In Taiko, a single user could contest an SGX-proven block (resetting its cooldown timer) with no valid proof, at a fixed bond cost, indefinitely. Here, a single participant can abort a key event instance (resetting the attempt counter) with no valid computation, at only gas cost, indefinitely. Both attacks exploit a lifecycle-reset primitive that requires no threshold agreement and has no economic deterrent.

---

### Impact Explanation

**During `Initializing` state (DKG):** DKG never completes → the contract never transitions to `Running` → no threshold signatures can ever be produced → all funds and cross-chain operations controlled by the MPC network are permanently frozen. This is a **Critical** impact: permanent freezing of funds controlled by the MPC network.

**During `Resharing` state:** Resharing never completes → the participant set cannot be updated → a compromised or malicious participant cannot be removed from the network. Signing with the old key continues, but the network is stuck and cannot recover from a Byzantine node by rotating it out.

---

### Likelihood Explanation

Any single participant in the MPC network can execute this attack. The cost is only NEAR gas fees (fractions of a cent per abort transaction). The attacker needs only to monitor the chain for new key event instances (trivially automated) and submit an abort transaction within the 30-block (~30-second) window. There is no economic deterrent, no rate limit, and no governance mechanism that can stop the attack without a full contract upgrade. The attack is easy to automate and sustain indefinitely.

---

### Recommendation

1. **Require threshold votes to abort**: Change `vote_abort` to collect abort votes from multiple participants (at least a threshold) before clearing the instance, mirroring the threshold requirement for `vote_success`.
2. **Implement a per-participant abort cooldown**: Prevent the same participant from aborting consecutive instances without a waiting period.
3. **Introduce an abort bond**: Require participants to stake a bond when voting abort, slashed if the abort is deemed malicious (e.g., if the instance would have succeeded without the abort).
4. **Cap abort attempts per participant per epoch**: Track how many times each participant has voted abort and reject further abort votes beyond a small limit.

---

### Proof of Concept

1. Network enters `Initializing` state (DKG in progress for domain 0, epoch 0).
2. Leader (lowest participant ID) calls `start_keygen_instance` with `key_event_id = { epoch_id: 0, domain_id: 0, attempt_id: 0 }`. A `KeyEventInstance` is created with `expires_on = current_block + 31`.
3. Byzantine participant (any single participant) reads the current `attempt_id` from public chain state and calls `vote_abort_key_event_instance({ epoch_id: 0, domain_id: 0, attempt_id: 0 })` within 30 blocks.
4. `KeyEvent::vote_abort` passes all checks (participant is authenticated, has not voted success) and sets `self.instance = None`. [7](#0-6) 
5. Leader calls `start_keygen_instance` with `attempt_id: 1`. `next_attempt_id` increments. [8](#0-7) 
6. Byzantine participant reads `attempt_id: 1` from chain state and aborts again.
7. Steps 5–6 repeat indefinitely. DKG never completes. The contract never transitions to `Running`. No threshold signatures are ever produced.

    /// Start a new key event instance as the leader, if one isn't already active.
    /// The leader is always the participant with the lowest participant ID.
    pub fn start(&mut self, key_event_id: KeyEventId, timeout_blocks: u64) -> Result<(), Error> {
        self.cleanup_if_timed_out();
        if self.instance.is_some() {
            return Err(KeyEventError::ActiveKeyEvent.into());
        }
        let expected_key_event_id =
            KeyEventId::new(self.epoch_id, self.domain.id, self.next_attempt_id);
        if key_event_id != expected_key_event_id {
            return Err(KeyEventError::KeyEventIdMismatch.into());
        }
        self.verify_leader()?;
        self.instance = Some(KeyEventInstance::new(self.next_attempt_id, timeout_blocks));
        self.next_attempt_id = self.next_attempt_id.next();
        Ok(())
    }

    /// Casts a vote to abort the current keygen instance.
    /// A new instance needs to be started later to start a new keygen attempt.
    pub fn vote_abort(&mut self, key_event_id: KeyEventId) -> Result<(), Error> {
        let candidate = self.verify_vote(&key_event_id)?;
        if self
            .instance
            .as_ref()
            .unwrap()
            .completed
            .contains(&candidate)
        {
            return Err(VoteError::VoteAlreadySubmitted.into());
        }
        self.instance = None;
        Ok(())
    }
