### Title
`MockAttestation::Valid` Unconditionally Bypasses TEE Attestation Verification — (File: `crates/mpc-attestation/src/attestation.rs`)

### Summary
The `MockAttestation::Valid` variant, which is part of the production-facing `Attestation` public interface, always returns `Ok(())` without performing any real TEE verification. Any existing participant can submit `Attestation::Mock(MockAttestation::Valid)` to register as a TEE-verified node without running inside a TDX enclave, bypassing the security boundary that protects key shares.

### Finding Description
The `Attestation` enum exposed in the public contract interface (`near_mpc_contract_interface`) includes a `Mock(MockAttestation)` variant. `MockAttestation::Valid` is defined to unconditionally pass all verification:

```rust
// crates/mpc-attestation/src/attestation.rs
match self {
    MockAttestation::Valid => Ok(()),          // ← always succeeds, no checks
    MockAttestation::Invalid => Err(...),
    MockAttestation::WithConstraints { .. } => { /* real checks */ }
}
``` [1](#0-0) 

This is called from `MockAttestation::verify` and `VerifiedAttestation::re_verify`, both of which are invoked by `TeeState::add_participant` and `TeeState::reverify_participants` in the contract: [2](#0-1) 

`TeeState::add_participant` accepts any `Attestation` variant with no guard against the `Mock` path: [3](#0-2) 

The `Attestation` and `MockAttestation` types are fully serializable and part of the public contract ABI: [4](#0-3) [5](#0-4) 

The DTO-to-internal mapping in `dto_mapping.rs` faithfully converts `dtos::MockAttestation::Valid` → `MockAttestation::Valid` with no rejection: [6](#0-5) 

Additionally, dev-build measurements are compiled into production binaries with an unresolved security TODO: [7](#0-6) 

### Impact Explanation
TEE attestation is the sole mechanism that proves an MPC node is running inside a TDX enclave with the correct software stack, protecting key shares from extraction. A participant who submits `MockAttestation::Valid` passes `is_caller_an_attested_participant` and all re-verification checks without ever running in a TEE. This allows a Byzantine participant to hold a key share and participate in threshold signing rounds while operating outside the trusted boundary — materially enabling key-share exposure and unauthorized signing capability. This matches: **Bypass of threshold-signature requirements or unauthorized access to MPC key shares, signing capability, or secret material.** [8](#0-7) 

### Likelihood Explanation
The `Attestation::Mock(MockAttestation::Valid)` variant is reachable by any existing participant (a role that requires threshold voting to obtain, but not privileged operator access). Once a participant is admitted to the set, they can call `submit_participant_info` with a mock attestation at any time. No special tooling is required — the type is part of the public JSON/Borsh ABI and can be constructed trivially.

### Recommendation
1. Reject `Attestation::Mock(_)` at the contract's public entry point (`submit_participant_info`) in production builds, or gate the `Mock` variant behind a compile-time feature flag that is excluded from production WASM builds.
2. Remove `include_measurements!("assets/tcb_info_dev.json")` from production builds (resolve `TODO(#1433)`).
3. Add an explicit allowlist check so that only `Attestation::Dstack(_)` is accepted when the contract is in `Running` or `Resharing` state.

### Proof of Concept
1. Attacker is an existing participant (admitted via normal threshold vote).
2. Attacker calls `submit_participant_info` with:
   ```json
   { "attestation": { "Mock": "Valid" } }
   ```
3. `TeeState::add_participant` calls `attestation.verify_locally(...)`, which dispatches to `MockAttestation::verify`, which calls `verify_constraints`, which returns `Ok(())` unconditionally for `MockAttestation::Valid`.
4. A `VerifiedAttestation::Mock(MockAttestation::Valid)` entry is stored for the attacker's TLS key.
5. All subsequent calls to `is_caller_an_attested_participant` and `reverify_participants` return `Valid` for the attacker indefinitely, with no expiry and no TEE requirement.
6. The attacker participates in threshold signing rounds and holds a key share entirely outside a TDX enclave. [9](#0-8) [10](#0-9)

    pub(crate) fn reverify_participants(
        &self,
        node_id: &NodeId,
        tee_upgrade_deadline_duration: Duration,
    ) -> TeeQuoteStatus {
        let allowed_mpc_docker_image_hashes =
            self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline_duration);
        let allowed_launcher_compose_hashes = self.get_allowed_launcher_compose_hashes();
        let allowed_measurements = self.get_accepted_measurements();

        let participant_attestation = self.stored_attestations.get(&node_id.tls_public_key);
        let Some(participant_attestation) = participant_attestation else {
            return TeeQuoteStatus::Invalid("participant has no attestation".to_string());
        };

        // Verify the attestation quote
        let time_stamp_seconds = Self::current_time_seconds();
        match participant_attestation.verified_attestation.re_verify(
            time_stamp_seconds,
            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

```rust
    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

impl IntoContractType<MockAttestation> for dtos::MockAttestation {
    fn into_contract_type(self) -> MockAttestation {
        match self {
            dtos::MockAttestation::Valid => MockAttestation::Valid,
            dtos::MockAttestation::Invalid => MockAttestation::Invalid,
            dtos::MockAttestation::WithConstraints {

                mpc_docker_image_hash,

                        rtmr2: m.rtmr2.into(),
                    },
                    key_provider_event_digest: m.key_provider_event_digest.into(),
