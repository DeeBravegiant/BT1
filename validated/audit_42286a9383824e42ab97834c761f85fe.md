Audit Report

## Title
`SfrxETHPriceOracle.getAssetPrice()` Returns sfrxETH/frxETH Instead of sfrxETH/ETH, Causing Persistent rsETH Mispricing - (`contracts/oracles/SfrxETHPriceOracle.sol`)

## Summary
`SfrxETHPriceOracle.getAssetPrice()` calls `pricePerShare()` on the sfrxETH ERC-4626 vault and returns the result directly as the sfrxETH/ETH exchange rate. Because sfrxETH's underlying asset is frxETH (not ETH), `pricePerShare()` returns the sfrxETH/frxETH rate. The missing frxETH/ETH conversion factor means `rsETHPrice` is structurally incorrect on every price update, causing the protocol to fail to deliver its promised returns.

## Finding Description
`SfrxETHPriceOracle` at [1](#0-0)  returns `ISfrxETH(sfrxETHContractAddress).pricePerShare()` directly. The interface comment at [2](#0-1)  falsely states "Price is in ETH, not USD," but sfrxETH is an ERC-4626 vault whose underlying asset is frxETH. `pricePerShare()` returns the amount of frxETH redeemable per sfrxETH share — the sfrxETH/frxETH rate — not sfrxETH/ETH.

This value flows directly into `_getTotalEthInProtocol()` at [3](#0-2) , where `assetER` (denominated in frxETH, not ETH) is multiplied by the sfrxETH deposit balance to compute `totalETHInProtocol`. The resulting `rsETHPrice` computed at [4](#0-3)  is therefore always denominated incorrectly.

The correct computation requires:
```
sfrxETH/ETH = pricePerShare() × (frxETH/ETH)
```
The frxETH/ETH multiplier is always absent. frxETH is a distinct token that trades at a persistent, non-zero spread to ETH (historically 0.1–0.5% under normal conditions, several percent during stress). The error is structural and continuous — not conditional on any attacker action.

`updateRSETHPrice()` at [5](#0-4)  is a public, permissionless function, so the incorrect price is propagated on every call. Downstream consumers include `RSETHPriceFeed.latestRoundData()` at [6](#0-5)  and `RSETHPool.viewSwapRsETHAmountAndFee` at [7](#0-6) .

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The protocol promises that `rsETHPrice` reflects the ETH value of the backing assets. Because the sfrxETH/frxETH rate is used in place of sfrxETH/ETH, the TVL calculation is always off by the frxETH/ETH spread. Under normal market conditions this is a persistent ~0.1–0.5% mispricing that is always active. Users depositing or withdrawing sfrxETH receive rsETH amounts that do not accurately reflect the ETH-denominated value of their deposit. The depeg-amplified scenario (several percent error) is excluded by SECURITY.md's exclusion of impacts relying on external stablecoin depegging not caused by the attacker; the baseline structural error under normal conditions is the valid in-scope impact.

## Likelihood Explanation
The bug is always active whenever sfrxETH is a supported asset. No attacker action is required — any call to the public `updateRSETHPrice()` propagates the incorrect price. The precondition (sfrxETH listed as a supported asset) is a normal protocol operating state. frxETH has never traded exactly at 1:1 with ETH, so the error is never zero.

## Recommendation
Replace the direct `pricePerShare()` return with a two-step computation:

```solidity
function getAssetPrice(address asset) external view returns (uint256) {
    if (asset != sfrxETHContractAddress) revert InvalidAsset();

    // sfrxETH/frxETH rate (ERC-4626 pricePerShare)
    uint256 sfrxEthToFrxEth = ISfrxETH(sfrxETHContractAddress).pricePerShare();

    // frxETH/ETH rate from a Chainlink feed or Curve TWAP
    uint256 frxEthToEth = IFrxEthOracle(frxEthOracle).getRate();

    // sfrxETH/ETH = (sfrxETH/frxETH) * (frxETH/ETH) / 1e18
    return sfrxEthToFrxEth * frxEthToEth / 1e18;
}
```

A Chainlink frxETH/ETH feed or a Curve TWAP oracle for the frxETH/ETH pool can supply the missing multiplier. The `frxEthOracle` address should be set at initialization and restricted to admin updates.

## Proof of Concept
1. Deploy a fork with sfrxETH as a supported asset and `SfrxETHPriceOracle` registered.
2. Observe that `pricePerShare()` on the live sfrxETH contract returns ~1.08e18 (sfrxETH/frxETH).
3. Observe that the live frxETH/ETH rate is ~0.999 (frxETH trades at a ~0.1% discount to ETH).
4. Call `LRTOracle.updateRSETHPrice()` (public, no privileges required).
5. Assert that `rsETHPrice` reflects `1.08e18` as the sfrxETH contribution rather than the correct `1.08 × 0.999 = 1.0791e18`.
6. The ~0.1% persistent overvaluation of sfrxETH TVL means rsETH holders backed by sfrxETH receive a slightly inflated rsETHPrice, while ETH depositors are diluted by the same factor on every price update.
