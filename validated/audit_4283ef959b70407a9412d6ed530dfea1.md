### Title
STRK Native Fees Permanently Locked in Starknet OmniBridge — No Distribution Path Exists (`starknet/src/omni_bridge.cairo`)

### Summary
The Starknet `OmniBridge` contract collects STRK tokens from users as `native_fee` during `init_transfer`, transferring them into the contract itself. However, the contract exposes no function to distribute, withdraw, or forward those STRK tokens to any relayer or fee recipient. Every STRK native fee paid by any user is permanently locked in the contract with no recovery path.

### Finding Description

In `init_transfer`, when `native_fee > 0`, the contract pulls STRK from the caller directly into `get_contract_address()`: [1](#0-0) 

The full public interface of `IOmniBridge` is: [2](#0-1) 

None of these functions transfer STRK out of the contract. `fin_transfer` only mints or transfers the bridged ERC20 token to the recipient — it does not touch `strk_token_address` at all: [3](#0-2) 

The `strk_token_address` storage slot is written once in the constructor and read only inside `init_transfer` to pull fees in. There is no `claim_fee`, `withdraw_fee`, `rescue`, or any outbound STRK transfer anywhere in the contract: [4](#0-3) 

By contrast, the NEAR side correctly distributes native fees to relayers inside `send_fee_internal`: [5](#0-4) 

The Starknet contract has no equivalent outbound path.

### Impact Explanation

Every STRK token paid as `native_fee` by any user calling `init_transfer` on Starknet is irreversibly locked in the `OmniBridge` contract. There is no admin rescue, no relayer claim, and no upgrade path that could recover these funds without a full contract replacement. This satisfies the **Critical — irreversible fund lock** impact criterion.

### Likelihood Explanation

The `native_fee` parameter is a normal, user-controlled input to the public `init_transfer` entrypoint. No privileged role is required. Any user who pays a non-zero STRK native fee — which is the intended incentive mechanism for relayers — triggers the lock. The Starknet chain has no `NativeFeeRestricted` role guard (unlike NEAR), so the path is fully open to all callers.

### Recommendation

Add an outbound STRK fee distribution function. The simplest correct fix mirrors the NEAR `send_fee_internal` pattern: when `fin_transfer` is called with a `fee_recipient` set in the payload, transfer the accumulated STRK native fee to that recipient. Alternatively, add an explicit `claim_native_fee` function that the relayer can call after proving a finalized transfer, sending the stored STRK amount to the designated `fee_recipient` from the `FinTransfer` payload.

### Proof of Concept

1. User calls `init_transfer(token_address, 1000, 0, 500, "near:recipient.near", "")` on the Starknet `OmniBridge`, where `500` is the STRK `native_fee`.
2. The contract executes `IERC20Dispatcher { contract_address: strk_token }.transfer_from(caller, get_contract_address(), 500)` — 500 STRK enters the contract.
3. The relayer observes the `InitTransfer` event, signs and submits the transfer on NEAR, and calls `fin_transfer` on Starknet.
4. `fin_transfer` mints/transfers the bridged token to the recipient. No STRK is sent anywhere.
5. The 500 STRK remains in the `OmniBridge` contract balance indefinitely. No function exists to retrieve it. Repeating across all users accumulates an unbounded locked STRK balance.

        fn fin_transfer(
            ref self: ContractState, signature: Signature, payload: TransferMessagePayload,
        ) {
            assert(!_is_paused(@self, PAUSE_FIN_TRANSFER), 'ERR_FIN_TRANSFER_PAUSED');

            assert(
                !self.is_transfer_finalised(payload.destination_nonce), 'ERR_NONCE_ALREADY_USED',
            );
            _set_transfer_finalised(ref self, payload.destination_nonce);

            _verify_borsh_signature(
                ref self, @payload.to_borsh(self.omni_bridge_chain_id.read()), signature,
            );

            if self.is_bridge_token(payload.token_address) {
                IBridgeTokenDispatcher { contract_address: payload.token_address }
                    .mint(payload.recipient, payload.amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: payload.token_address }
                    .transfer(payload.recipient, payload.amount.into());
                assert(success, 'ERR_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::FinTransfer(
                        FinTransfer {
                            origin_chain: payload.origin_chain,
                            origin_nonce: payload.origin_nonce,
                            token_address: payload.token_address,
                            amount: payload.amount,
                            recipient: payload.recipient,
                            fee_recipient: payload.fee_recipient,
                            message: payload.message,
                        },
                    ),
                )
        }
