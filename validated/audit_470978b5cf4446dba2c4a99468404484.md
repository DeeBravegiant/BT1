Audit Report

## Title
Missing Staleness Check in `ChainlinkPriceOracle.getAssetPrice()` Causes Users to Receive Fewer Assets in `instantWithdrawal` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but silently discards the `updatedAt` timestamp and `answeredInRound` fields, accepting any price regardless of age. When `LRTWithdrawalManager.instantWithdrawal()` consumes a stale, inflated asset price, the division `rsETHPrice / assetPrice` yields a lower `assetAmountUnlocked`, and the user's rsETH is burned irreversibly with no `minAmountOut` guard to reject the unfavorable rate.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` discards all staleness indicators:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

The `updatedAt` timestamp and `answeredInRound` are never validated. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` (used for pool-level pricing) performs both checks:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

`LRTWithdrawalManager.instantWithdrawal()` calls `getExpectedAssetAmount()` at line 228, which computes:

```solidity
// contracts/LRTWithdrawalManager.sol L593
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```

The rsETH burn is executed immediately after on line 229, before any slippage check, and `instantWithdrawal` accepts no `minAssetAmountOut` parameter. If `getAssetPrice(asset)` returns a stale, inflated value, the quotient is smaller, and the user receives fewer assets than the fair value of their burned rsETH with no recourse.

## Impact Explanation

**Low. Contract fails to deliver promised returns, but doesn't lose value.**

When the Chainlink feed for a supported LST is stale and its last reported price is higher than the current market price, `getAssetPrice(asset)` returns an inflated value. The formula `rsETHPrice / inflatedAssetPrice` produces a lower `assetAmountUnlocked`. The user burns rsETH worth X ETH but receives assets worth less than X ETH. The rsETH burn is irreversible. The shortfall accrues to the unstaking vault, benefiting remaining participants at the expense of the instant withdrawer. No protocol funds are lost, but the user does not receive the promised fair-value return.

## Likelihood Explanation

Chainlink LST/ETH feeds (e.g., stETH/ETH) have 24-hour heartbeats. During network congestion, rapid LST price movement (e.g., a slashing event), or a temporary oracle outage, the on-chain price can lag the real price by a meaningful margin within that window. `instantWithdrawal` is a public, permissionless function callable by any rsETH holder whenever the feature is enabled for an asset. No special role or precondition beyond holding rsETH is required. The condition is a known, recurring property of Chainlink feeds, not a hypothetical edge case.

## Recommendation

1. Add staleness and completeness checks to `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern in `ChainlinkOracleForRSETHPoolCollateral`:
   ```solidity
   (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
       priceFeed.latestRoundData();
   if (answeredInRound < roundId) revert StalePrice();
   if (updatedAt == 0) revert IncompleteRound();
   if (block.timestamp - updatedAt > HEARTBEAT) revert StalePrice();
   return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
   ```
2. Add a `minAssetAmountOut` parameter to `instantWithdrawal()` so users can specify a floor and revert if the oracle-derived amount falls below it, analogous to `minRSETHAmountExpected` in `LRTDepositPool.depositETH()`.

## Proof of Concept

1. stETH/ETH Chainlink feed last updated 20 hours ago at price `1.05e18`; current real price is `1.00e18` (5% drop due to a slashing event).
2. User calls `instantWithdrawal(stETH, 1e18 rsETH, "")`.
3. `getExpectedAssetAmount` computes `1e18 * rsETHPrice / 1.05e18` → user receives ~4.76% fewer stETH than fair value.
4. `burnFrom` executes at line 229; the burn is irreversible.
5. No revert occurs because `ChainlinkPriceOracle.getAssetPrice()` never checks `updatedAt` and `instantWithdrawal` has no `minAmountOut` guard.
