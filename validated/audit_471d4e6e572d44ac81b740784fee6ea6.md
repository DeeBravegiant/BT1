### Title
Gateway L2 Gas Price Admission Uses Stale Previous-Block Price While Batcher Enforces Next-Block Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `validate_resource_bounds` checks a transaction's `max_price_per_unit` for L2 gas against the **previous block's** L2 gas price. The batcher, however, executes transactions against the **next block's** L2 gas price, which is computed dynamically via SNIP-35 (EIP-1559-style adjustment). When the next block's price is higher than the previous block's price — the normal outcome under high load — transactions that passed gateway admission will fail the batcher's `check_fee_bounds` pre-validation with `MaxGasPriceTooLow`, and be silently dropped from sequencing.

### Finding Description

In `StatefulTransactionValidator::validate_resource_bounds`, the threshold is computed from the **previous** block's L2 gas price:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

The admission threshold is:

```
tx_l2_gas_price >= (min_gas_price_percentage / 100) * previous_block_l2_gas_price
```

With the default `min_gas_price_percentage = 100`, this becomes `tx_l2_gas_price >= previous_block_l2_gas_price`. [2](#0-1) 

The batcher, however, builds its `BlockContext` with the **next block's** L2 gas price, computed by the consensus orchestrator via `calculate_next_l2_gas_price` (SNIP-35 EIP-1559 adjustment): [3](#0-2) 

During batcher execution, `perform_pre_validation_stage` calls `check_fee_bounds`, which enforces `max_price_per_unit >= block_l2_gas_price` (the next block's price). If the next block's price exceeds the previous block's price, the transaction fails with `ResourceBoundsError::MaxGasPriceTooLow`: [4](#0-3) 

The same stale price is also used in `run_validate_entry_point` when building the `BlockContext` for the `__validate__` entry point simulation at the gateway: [5](#0-4) 

The TODO comment in the source explicitly acknowledges the discrepancy: `// TODO(Arni): getnext_l2_gas_price from the block header.`

### Impact Explanation

A transaction with `tx_l2_gas_price = previous_block_l2_gas_price` passes gateway admission and enters the mempool. When the batcher attempts to sequence it, `check_fee_bounds` rejects it with `MaxGasPriceTooLow` because `next_block_l2_gas_price > previous_block_l2_gas_price`. The transaction is never included in a block. From the user's perspective, the gateway accepted the transaction (returned a transaction hash), but it is silently dropped from sequencing. This matches the **High** impact: *Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.*

### Likelihood Explanation

SNIP-35 increases the L2 gas price whenever gas usage exceeds the target. Under any sustained load, the next block's price will exceed the previous block's price. The discrepancy is bounded by the per-block clamping margin, but it is non-zero and predictable. Any user submitting a transaction with `max_price_per_unit` exactly at the previous block's price will trigger this path.

### Recommendation

Replace the stale `previous_block_l2_gas_price` in `validate_resource_bounds` with the **next block's** L2 gas price, which is already computed by the consensus orchestrator and stored in the block header as `next_l2_gas_price`. The TODO comment in the code already identifies this fix. The gateway's `GatewayFixedBlockStateReader` should expose `next_l2_gas_price` from the block header, and `validate_resource_bounds` should use that value as the threshold reference.

### Proof of Concept

1. Observe the current L2 gas price from the latest block header: `P_prev`.
2. Submit a V3 transaction with `l2_gas.max_price_per_unit = P_prev` (exactly at the threshold with `min_gas_price_percentage = 100`).
3. Gateway calls `validate_resource_bounds`: `P_prev >= 100% * P_prev` → **passes**.
4. Transaction enters the mempool.
5. The consensus orchestrator computes `P_next` via SNIP-35 for the next block. Under any non-zero gas usage, `P_next > P_prev`.
6. Batcher builds `BlockContext` with `l2_gas_price = P_next`.
7. Batcher calls `perform_pre_validation_stage` → `check_fee_bounds`: `P_prev < P_next` → **fails** with `MaxGasPriceTooLow`.
8. Transaction is dropped from sequencing. The user received a valid-looking transaction hash from the gateway but the transaction is never sequenced. [6](#0-5)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    // Performs static checks before executing validation entry point.
    // Note that nonce is incremented during these checks.
    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }
