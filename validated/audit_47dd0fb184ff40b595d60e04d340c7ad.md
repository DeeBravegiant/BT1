### Title
Unguarded `decode('utf8')` in `ProofsChecker.from_program` Causes Unhandled `UnicodeDecodeError` During Wallet Sync ŌĆö (`chia/wallet/vc_wallet/cr_cat_drivers.py`)

### Summary

`ProofsChecker.from_program` unconditionally calls `.decode("utf8")` on flag atoms extracted from an on-chain puzzle. An unprivileged attacker can craft a CR-CAT whose curried `proofs_checker` contains flag atoms with arbitrary (non-UTF-8) bytes. When the victim's wallet syncs and `handle_cat` calls `ProofsChecker.from_program`, a `UnicodeDecodeError` is raised outside any exception guard, propagating up through the wallet sync path.

### Finding Description

The vulnerable line is: [1](#0-0) 

```python
return cls([flag.at("f").as_atom().decode("utf8") for flag in uncurried_puzzle.args.at("f").as_iter()])
```

CLVM atoms are arbitrary byte sequences ŌĆö consensus imposes no UTF-8 constraint on curried puzzle arguments. An attacker can freely curry a `proofs_checker` whose flag atoms contain bytes such as `b'\xff\xfe'`.

`ProofsChecker.from_program` is called in two places inside `handle_cat`:

**Line 1309** ŌĆö when converting an existing CAT wallet to CR: [2](#0-1) 

**Line 1334** ŌĆö when auto-creating a new CRCAT wallet: [3](#0-2) 

The only `try/except` in `handle_cat` covers `CRCAT.get_next_from_coin_spend` and catches only `ValueError`: [4](#0-3) 

`UnicodeDecodeError` is not a subclass of `ValueError`, so it escapes this guard entirely. Both call sites at lines 1309 and 1334 are outside any exception handler.

`handle_cat` itself is called from `determine_coin_type` with no surrounding try/except: [5](#0-4) 

### Impact Explanation

- **Line 1309 path**: Triggered when the victim already has a plain CAT wallet for the same tail hash. The attacker sends a CR-CAT with that tail hash and non-UTF-8 flag atoms to the victim. Every sync attempt for that coin raises `UnicodeDecodeError`, permanently preventing the wallet from converting or processing that CRCAT.
- **Line 1334 path**: Triggered when `automatically_add_unknown_cats: true` is set in the wallet config (or the tail is in `default_cats`). Any attacker can send a crafted CRCAT to any such wallet and break sync for that tail hash.

The impact matches the **High** scope: permanent inability for an honest wallet to process valid on-chain coin states.

### Likelihood Explanation

- No signature or key material is required from the attacker.
- CLVM/CAT consensus validation does not inspect or constrain the byte content of curried `proofs_checker` flag atoms.
- The attacker only needs to submit a spend bundle that creates a CR-CAT with a malformed `proofs_checker` and send the output coin to the victim.
- The condition at line 1292ŌĆō1295 (early return if a CRCAT wallet already exists for that tail) means the bug is only reachable on first encounter of a new tail hash, but that is sufficient to permanently block wallet creation for that tail.

### Recommendation

Wrap the `.decode("utf8")` call in a `try/except (UnicodeDecodeError, ValueError)` and raise a `ValueError` with a descriptive message, so the existing guard at line 1267ŌĆō1271 can catch it:

```python
def _decode_flag(atom: bytes) -> str:
    try:
        return atom.decode("utf-8")
    except UnicodeDecodeError:
        raise ValueError(f"Flag atom is not valid UTF-8: {atom!r}")

return cls([_decode_flag(flag.at("f").as_atom()) for flag in uncurried_puzzle.args.at("f").as_iter()])
```

Additionally, the call sites at lines 1309 and 1334 should be wrapped in their own `try/except ValueError` with a `return None` fallback, consistent with the pattern used for `get_next_from_coin_spend`.

### Proof of Concept

```python
from chia.wallet.vc_wallet.cr_cat_drivers import ProofsChecker, PROOF_FLAGS_CHECKER
from chia.wallet.uncurried_puzzle import uncurry_puzzle
from chia.types.blockchain_format.program import Program

# Craft a proofs_checker with a non-UTF-8 flag atom
bad_flag = b'\xff\xfe'
malicious_checker = PROOF_FLAGS_CHECKER.curry(
    [Program.to((bad_flag, "1"))]
)
uncurried = uncurry_puzzle(malicious_checker)

# This raises UnicodeDecodeError instead of returning safely
try:
    result = ProofsChecker.from_program(uncurried)
    print("FAIL: should have raised")
except UnicodeDecodeError as e:
    print(f"CONFIRMED: UnicodeDecodeError raised: {e}")
except ValueError:
    print("OK: ValueError raised (safe)")
```
