### Title
Block-Level Reference Script Size Limit Bypass via Sub-Transaction Reference Scripts — (`File: eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Bbody.hs`)

---

### Summary

In the Dijkstra era, the block-level reference script size check (`validateBodyRefScriptsSizeTooBig`) uses `totalRefScriptSizeInBlock`, which calls `txNonDistinctRefScriptsSize` and only inspects the **top-level transaction's** inputs. It is blind to reference scripts consumed by **sub-transactions** embedded within a Dijkstra-era `TopTx`. The per-transaction check (`validateAllRefScriptSize`) correctly uses `batchNonDistinctRefScriptsSize`, which aggregates reference script sizes across the top-level transaction and all its sub-transactions. This asymmetry allows an attacker to exceed the intended block-level reference script size limit (`ppMaxRefScriptSizePerBlockG`) by concentrating all reference script usage inside sub-transactions.

---

### Finding Description

The Dijkstra era introduces nested transactions (`TopTx` / `SubTx`). Each `TopTx` may embed an ordered map of `SubTx` entries in `subTransactionsTxBodyL`.

**Per-transaction check (correct):**

`dijkstraLedgerTransition` calls `validateAllRefScriptSize`, which delegates to `batchNonDistinctRefScriptsSize`:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
```

This correctly sums reference script sizes for the top-level transaction **and** every sub-transaction. [1](#0-0) [2](#0-1) 

**Block-level check (incomplete):**

`validateBodyRefScriptsSizeTooBig` calls `totalRefScriptSizeInBlock`, which iterates over the block's `Tx TopTx era` sequence and calls `txNonDistinctRefScriptsSize` for each:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
```

`txNonDistinctRefScriptsSize` only inspects `referenceInputsTxBodyL` and `inputsTxBodyL` of the **top-level** transaction body. The `subTransactionsTxBodyL` field — which is a Dijkstra-specific extension — is never visited. Sub-transaction inputs and their associated reference scripts are entirely invisible to this function. ---

### Impact Explanation

An attacker can craft a block containing multiple Dijkstra-era transactions where each transaction places all reference script usage inside sub-transactions and keeps the top-level transaction's own inputs free of reference scripts. Each individual transaction passes the per-transaction limit (`ppMaxRefScriptSizePerTxG`, currently 200 KB) because `batchNonDistinctRefScriptsSize` counts sub-transaction reference scripts. However, the block-level check sees 0 bytes of reference scripts per transaction (since `totalRefScriptSizeInBlock` ignores sub-transactions), so the block-level limit (`ppMaxRefScriptSizePerBlockG`, currently 1 MB) is never triggered regardless of how many such transactions are included.

Concretely: 10 transactions each with 200 KB of reference scripts entirely in sub-transactions produce 2 MB of actual reference script load on validators, while the block-level check reports 0 KB ≤ 1 MB and accepts the block. This allows an attacker to exceed the intended block-level resource limit, forcing all validating nodes to load and process more reference script data than the protocol parameters permit.

This matches the allowed Medium impact: **attacker-controlled transactions exceed intended validation limits**.

---

### Likelihood Explanation

The Dijkstra era is active at protocol version 12. Any transaction sender who can construct a valid Dijkstra-era `TopTx` with sub-transactions can exploit this. No privileged access, governance majority, or key compromise is required. The attacker only needs to:

1. Hold UTxO entries whose outputs carry large reference scripts (achievable by any user who has previously submitted transactions with `referenceScriptTxOutL` set).
2. Construct sub-transactions that reference those inputs.
3. Submit a block (or collude with a block producer) containing enough such transactions to exceed `ppMaxRefScriptSizePerBlockG`.

A block producer acting in self-interest (e.g., to collect fees) would include valid transactions, making this reachable without a malicious block producer if the attacker simply submits the transactions to the mempool.

---

### Recommendation

`totalRefScriptSizeInBlock` (or the Dijkstra-era override of `validateBodyRefScriptsSizeTooBig`) must be updated to use `batchNonDistinctRefScriptsSize` (or an equivalent that folds over `subTransactionsTxBodyL`) instead of `txNonDistinctRefScriptsSize` when processing Dijkstra-era transactions. The block-level accumulator should sum reference script sizes across the top-level transaction **and** all embedded sub-transactions, mirroring the logic already present in `validateAllRefScriptSize` at the per-transaction level.

---

### Proof of Concept

1. Produce N UTxO entries, each carrying a reference script of size S bytes (e.g., S = `ppMaxRefScriptSizePerTxG / N`).
2. For each of M top-level Dijkstra transactions, construct a `TopTx` whose `subTransactionsTxBodyL` contains sub-transactions that collectively reference all N UTxO entries. The top-level transaction body itself references **no** inputs with reference scripts.
3. Each transaction satisfies `batchNonDistinctRefScriptsSize ≤ ppMaxRefScractSizePerTxG` (per-transaction check passes).
4. Submit all M transactions in a single block. `totalRefScriptSizeInBlock` reports 0 for each transaction (sub-transaction inputs are invisible), so the block-level check passes even though the true total is `M × N × S`, which can be made arbitrarily large relative to `ppMaxRefScriptSizePerBlockG`. -- | Validate that total reference script size does not exceed block limit.
validateBodyRefScriptsSizeTooBig ::
  forall era.
  ( AlonzoEraTx era
  , BabbageEraTxBody era
  , InjectRuleFailure "BBODY" ConwayBbodyPredFailure era
  , EraBlockBody era
  , ConwayEraPParams era
  ) =>
  PParams era ->
  BlockBody era ->
  UTxO era ->
  Rule (EraRule "BBODY" era) 'Transition ()
validateBodyRefScriptsSizeTooBig pp blockBody utxo =
  let protVer = pp ^. ppProtocolVersionL
      txs = blockBody ^. txSeqBlockBodyL
      totalSize = totalRefScriptSizeInBlock protVer txs utxo
      maxSize = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerBlockG
   in totalSize
        <= maxSize
          ?! injectFailure
            ( BodyRefScriptsSizeTooBig $
                Mismatch
                  { mismatchSupplied = totalSize
                  , mismatchExpected = maxSize
                  }
            )

totalRefScriptSizeInBlock ::
  (AlonzoEraTx era, BabbageEraTxBody era) => ProtVer -> StrictSeq (Tx TopTx era) -> UTxO era -> Int
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | pvMajor protVer <= natVersion @10 =
      getSum $ foldMap' (Monoid.Sum . txNonDistinctRefScriptsSize (UTxO utxo)) txs
  | otherwise =
      snd $ F.foldl' accum (utxo, 0) txs
  where
    accum (!accUtxo, !accSum) tx =
      let updatedUtxo = accUtxo `Map.union` unUTxO toAdd
          toAdd
            | IsValid True <- tx ^. isValidTxL = txouts $ tx ^. bodyTxL
            | otherwise = collOuts $ tx ^. bodyTxL
       in (updatedUtxo, accSum + txNonDistinctRefScriptsSize (UTxO accUtxo) tx)
