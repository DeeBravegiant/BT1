### Title
Silent `zip` Truncation in `updateUTxOTxWitness` Allows Spending Unwitnessed Inputs — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

---

### Summary

`updateUTxOTxWitness` uses Haskell's `zip` to pair UTxO addresses with witnesses before calling `validateWitness`. Because `zip` silently truncates to the shorter list, submitting a `TxAux` with fewer witnesses than inputs causes the trailing inputs to be consumed without any witness check. No length-equality guard exists anywhere in the validation path.

---

### Finding Description

In `updateUTxOTxWitness`, the witness-validation loop is:

```haskell
mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))   -- line 389
``` [1](#0-0) 

`addresses` has exactly `length (txInputs tx)` elements (built by `mapM lookupAddress` over all inputs at line 383). `witness` is the raw `Vector TxInWitness` from the submitted `TxAux`. Haskell's `zip` stops at the shorter list — if `V.length witness < length (txInputs tx)`, the surplus inputs are silently dropped from the validation loop and their addresses are never passed to `validateWitness`. [2](#0-1) 

No other function in the call chain compensates:

- `validateTx` checks only that every `TxIn` is present in the UTxO, attribute sizes, and network magic — no witness count. [3](#0-2) 
- `validateTxAux` checks only transaction size, minimum fee, and balance conservation — no witness count. [4](#0-3) 
- `TxValidationError` has no `WitnessCountMismatch` constructor, confirming no such check was ever added. [5](#0-4) 
- After the `whenTxValidation` block, `updateUTxOTx` unconditionally removes all inputs from the UTxO and adds all outputs. [6](#0-5) 

---

### Impact Explanation

An unprivileged attacker who knows the `TxIn` identifier of any UTxO entry (all UTxO entries are public on-chain) can:

1. Construct a transaction with two inputs: one they own (`U_attacker`) and one they do not (`U_victim`).
2. Provide exactly one witness — a valid signature for `U_attacker`.
3. `zip [addr_attacker, addr_victim] [witness_attacker]` produces `[(addr_attacker, witness_attacker)]`.
4. `validateWitness` is called once and passes; `addr_victim` is never checked.
5. `updateUTxOTx` removes both `U_attacker` and `U_victim` from the UTxO and credits the attacker's output address.

This is direct, unauthorized transfer of ADA — **Critical** impact (invalid ledger state transition resulting in loss of ADA).

---

### Likelihood Explanation

The precondition is trivially satisfiable: any node on the network can submit a `TxAux` with an arbitrary-length witness vector. CBOR encoding of `TxAux` does not enforce `len(witnesses) == len(inputs)`. The attack requires no privileged access, no key leakage, and no governance majority.

---

### Recommendation

Add an explicit length-equality check before the `zip`, e.g.:

```haskell
let nInputs   = length addresses
    nWitnesses = V.length witness
when (nInputs /= nWitnesses) $
  throwError $ TxValidationWitnessCountMismatch nInputs nWitnesses
```

Add `TxValidationWitnessCountMismatch Int Int` to `TxValidationError` and cover it with CBOR round-trip and unit tests. [7](#0-6) 

---

validateTxAux ::
  MonadError TxValidationError m =>
  Environment ->
  UTxO ->
  ATxAux ByteString ->

validateTxAux env utxo (ATxAux (Annotated tx _) _ txBytes) = do
  -- Check that the size of the transaction is less than the maximum
  txSize
    <= maxTxSize
    `orThrowError` TxValidationTxTooLarge txSize maxTxSize

  -- Calculate the minimum fee from the 'TxFeePolicy'
  minFee <-
    if isRedeemUTxO inputUTxO
      then pure $ mkKnownLovelace @0
      else calculateMinimumFee feePolicy

  -- Calculate the balance of the output 'UTxO'
  balanceOut <-
    balance (txOutputUTxO tx)
      `wrapError` TxValidationLovelaceError "Output Balance"

  -- Calculate the balance of the restricted input 'UTxO'
  balanceIn <-
    balance inputUTxO
      `wrapError` TxValidationLovelaceError "Input Balance"

  -- Calculate the 'fee' as the difference of the balances
  fee <-
    subLovelace balanceIn balanceOut
      `wrapError` TxValidationLovelaceError "Fee"

  -- Check that the fee is greater than the minimum
  (minFee <= fee) `orThrowError` TxValidationFeeTooSmall tx minFee fee
  where
    Environment {protocolParameters} = env

    maxTxSize = ppMaxTxSize protocolParameters
    feePolicy = ppTxFeePolicy protocolParameters

    txSize :: Natural
    txSize = fromIntegral $ BS.length txBytes

    inputUTxO = S.fromList (NE.toList (txInputs tx)) <| utxo

    calculateMinimumFee ::
      MonadError TxValidationError m => TxFeePolicy -> m Lovelace
    calculateMinimumFee = \case
      TxFeePolicyTxSizeLinear txSizeLinear ->
        calculateTxSizeLinear txSizeLinear txSize
          `wrapError` TxValidationLovelaceError "Minimum Fee"
