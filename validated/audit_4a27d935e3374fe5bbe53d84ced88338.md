### Title
FIFO `rewind_txs` Silently Drops Co-Staged Nonce-N+1 Transaction When Nonce-N Is Rejected With No Committed Txs — (`crates/apollo_mempool/src/fifo_transaction_queue.rs`)

### Summary

When two transactions from the same address (nonce 0 and nonce 1) are both staged in the same FIFO batch and `commit_block` arrives with `committed_nonces={}` and `rejected_tx_hashes={hash(nonce_0)}`, the `collect_txs_to_rewind` logic marks the address as "do not rewind," then `staged_txs.clear()` silently drops nonce_1 from the staging buffer. Nonce_1 remains in `tx_pool` but is never re-inserted into the FIFO queue, making it permanently unsequenceable. The user cannot resubmit it because the pool still holds it (duplicate rejection). Only TTL expiry eventually removes it.

### Finding Description

**`collect_txs_to_rewind` — the `else` branch (no committed txs for address):** [1](#0-0) 

When `committed_nonces` has no entry for address A, the code finds the minimum-nonce staged tx and checks whether it is rejected. If it is, the address is excluded from `addresses_to_rewind`. This is correct for nonce_0 itself, but it also silently abandons every higher-nonce tx that was co-staged (nonce_1, nonce_2, …).

**`rewind_txs` — unconditional `staged_txs.clear()`:** [2](#0-1) 

After computing `txs_to_rewind` (empty for address A), `staged_txs.clear()` is called unconditionally. This drops nonce_1 from the staging buffer even though it was never re-enqueued.

**`remove_rejected_txs` — `remove_by_address` is a no-op for nonce_1:** [3](#0-2) 

`remove_by_address` scans `self.queue` and `self.staged_txs`. By the time it runs, `staged_txs` is already empty (cleared above) and nonce_1 was never re-inserted into `self.queue`. So nonce_1 is not removed from `tx_pool` either — it is permanently orphaned.

**FIFO `add_tx_inner` does not re-enqueue existing pool txs:** [4](#0-3) 

In FIFO mode, only newly submitted transactions are inserted into the queue. There is no path that scans the pool for orphaned txs and re-enqueues them.

**The existing test covers only the non-co-staged case:** [5](#0-4) 

In `test_rejected_tx_removes_same_address_from_fifo_queue`, nonce_1 is assigned a *different* timestamp (`1001` vs `1000`), so only nonce_0 is staged; nonce_1 stays in the queue and is correctly removed by `remove_by_address`. The co-staged scenario (both at the same timestamp/block) is not tested.

### Impact Explanation

A valid nonce_1 transaction is silently dropped from sequencing. It cannot be resubmitted (the pool still holds it, so `add_tx` returns `DuplicateTransaction`). It will not appear in any future block until TTL expiry removes it from the pool. This is a permanent, silent loss of a valid user transaction from the sequencing pipeline — matching the **High** impact: "Mempool/gateway/RPC admission rejects valid transactions before sequencing."

### Likelihood Explanation

The trigger is straightforward for any unprivileged sender:
1. Submit nonce_0 as a transaction that will fail execution (contract revert, insufficient balance, etc.) — this is a normal Starknet scenario.
2. Submit nonce_1 as a valid transaction.
3. Both must be assigned the same FIFO timestamp/block_number so that a single `get_txs` call stages them together. In Echonet/FIFO mode all txs from the same source block share the same metadata, making co-staging the common case.
4. The block commits with nonce_0 in `rejected_tx_hashes` and no committed nonce for address A.

No privileged access is required. The attacker controls only their own transaction content and signing keys.

### Recommendation

In the `else` branch of `collect_txs_to_rewind`, when the minimum-nonce staged tx is rejected, only exclude that specific tx from rewind. All higher-nonce co-staged txs for the same address should still be re-enqueued (pushed to the front of the FIFO queue), because their nonce gap is caused by the rejection of nonce_0, not by any fault of their own. Alternatively, when `staged_txs.clear()` is called, any staged tx that was neither rewound nor explicitly rejected should be re-inserted into the queue before clearing.

fn test_rejected_tx_removes_same_address_from_fifo_queue(mut mempool: Mempool) {
    let rejected_tx = add_tx_input!(tx_hash: 1, address: "0x1", tx_nonce: 0, account_nonce: 0);
    let future_same_address_tx =
        add_tx_input!(tx_hash: 2, address: "0x1", tx_nonce: 1, account_nonce: 0);
    let other_address_tx = add_tx_input!(tx_hash: 3, address: "0x2", tx_nonce: 0, account_nonce: 0);

    mempool.update_tx_block_metadata(tx_hash!(1), tx_metadata(1000, 100));
    mempool.update_tx_block_metadata(tx_hash!(2), tx_metadata(1001, 101));
    mempool.update_tx_block_metadata(tx_hash!(3), tx_metadata(1001, 101));

    for input in [&rejected_tx, &future_same_address_tx, &other_address_tx] {
        add_tx(&mut mempool, input);
    }

    // Stage only the first tx (timestamp 1000).
    assert_eq!(mempool.resolve_batch_timestamp(), 1000);
    get_txs_and_assert_expected(&mut mempool, 10, &[rejected_tx.tx]);

    // Reject tx. In FIFO, this removes same-address queued txs.
    commit_block(&mut mempool, [], [tx_hash!(1)]);

    // Next timestamp batch should include only the other address tx.
    assert_eq!(mempool.resolve_batch_timestamp(), 1001);
    get_txs_and_assert_expected(&mut mempool, 10, &[other_address_tx.tx]);
}
