Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice()` via Deeply Nested External Calls Across Assets, NDCs, and Queued Withdrawals - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function that triggers a deeply nested call chain multiplying external calls across supported assets × NDCs × queued EigenLayer withdrawals × strategies per withdrawal. Under normal protocol operations, as NDCs accumulate queued withdrawals up to the configured maximums (`maxNodeDelegatorLimit` = 10, `maxUncompletedWithdrawalCount` = 80), the cumulative gas cost can approach or exceed the Ethereum block gas limit, permanently bricking the rsETH price oracle.

## Finding Description
The call chain is fully confirmed by the code:

`updateRSETHPrice()` (public, `whenNotPaused` only) calls `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()`. [1](#0-0) 

`_getTotalEthInProtocol()` iterates over every supported asset and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [2](#0-1) 

`getTotalAssetDeposits()` delegates to `getAssetDistributionData()`, which loops over every NDC and makes three external calls per NDC: `IERC20.balanceOf()`, `INodeDelegator.getAssetBalance()`, and `INodeDelegator.getAssetUnstaking()`. [3](#0-2) 

The same pattern applies to ETH via `getETHDistributionData()`, which also calls `getAssetUnstaking(ETH_TOKEN)` per NDC. [4](#0-3) 

`getAssetUnstaking()` is the most expensive call. It calls `DelegationManager.getQueuedWithdrawals(address(this))` — a single call that reads all queued withdrawal structs from EigenLayer storage — then iterates over every withdrawal × every strategy, calling `strategy.sharesToUnderlyingView()` for each non-ETH strategy. [5](#0-4) 

`maxNodeDelegatorLimit` is initialized to 10 and `maxUncompletedWithdrawalCount` is capped at 80. With 3 supported assets, 10 NDCs, and 80 queued withdrawals per NDC, the conservative call count is:

`1 + 3 × (1 + 10 × (1 + 1 + 1 + 80)) ≈ 2,494 external calls`

At ~2,100 gas per cold external call, this is ~5.2M gas in CALL overhead alone, before accounting for EigenLayer's internal storage reads within `getQueuedWithdrawals()` (which reads all 80 withdrawal structs from cold storage per call) and `sharesToUnderlyingView()` computation. The total can realistically approach or exceed the 30M gas block limit.

No attacker action is required. Operators routinely call `initiateUnstaking()` and `undelegate()` as normal protocol operations, which accumulate queued withdrawals up to the configured maximum. [6](#0-5) 

## Impact Explanation
**Medium. Unbounded gas consumption.** The gas cost of `updateRSETHPrice()` grows proportionally with the product of (supported assets) × (NDCs) × (queued withdrawals per NDC) × (strategies per withdrawal). Under the protocol's own configured maximums, this product can produce a transaction that exceeds the Ethereum block gas limit, making `updateRSETHPrice()` permanently uncallable. Secondary consequences include a permanently stale `rsETHPrice`, blocked fee minting to treasury, disabled downside price-protection pausing, and mispriced rsETH minting for depositors — but the primary in-scope impact is unbounded gas consumption causing a DoS on the public oracle update function.

## Likelihood Explanation
`updateRSETHPrice()` has no access control beyond `whenNotPaused`. The gas cost increases automatically as operators perform routine unstaking operations (`initiateUnstaking`, `undelegate`). No adversarial action is needed; normal protocol scaling toward the configured limits (`maxNodeDelegatorLimit` = 10, `maxUncompletedWithdrawalCount` = 80) is sufficient to trigger the condition. The function is callable by any external account, and the degradation is gradual and predictable.

## Recommendation
1. **Decouple `getAssetUnstaking` from the price update path**: Maintain an on-chain accounting variable (e.g., `totalUnstakingByAsset[asset]`) updated incrementally on `initiateUnstaking` and `completeUnstaking`, eliminating the need to iterate over all queued withdrawals on every price update.
2. **Cache the `INodeDelegator` reference** outside the inner loop and avoid repeated `lrtConfig` lookups per NDC per asset.
3. **Cap the number of supported assets** with an explicit maximum analogous to `maxNodeDelegatorLimit`.
4. **Consider a keeper/off-chain aggregation pattern** where the total ETH value is submitted by a trusted off-chain actor and validated on-chain, rather than computed fully on-chain in a single transaction.

## Proof of Concept
Deploy a fork test against mainnet or a local EigenLayer mock. Configure: 3 supported assets, 10 NDCs, each NDC with 80 queued withdrawals (via `initiateUnstaking` calls up to `maxUncompletedWithdrawalCount`). Call `updateRSETHPrice()` and measure gas via `vm.expectCall` counts or `gasleft()` snapshots. Expected result: gas consumption exceeds 15–30M gas, causing the transaction to revert with out-of-gas on mainnet (block gas limit ~30M). A Foundry fork test using `vm.deal`, `vm.prank`, and EigenLayer's `queueWithdrawals` can reproduce this with realistic state.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
