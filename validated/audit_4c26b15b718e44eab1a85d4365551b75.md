### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect All Block Fees to an Arbitrary Address - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

`ProposalInit.builder` is the on-wire field that becomes `BlockInfo.sequencer_address` — the address that receives every transaction fee in the block. The consensus validator never checks this field against any expected value. A malicious proposer can set `builder` to any arbitrary address; the validator will execute the block with that address as fee recipient, compute a commitment that is internally consistent with the spoofed value, and accept the proposal. All fees for the block are permanently redirected to the attacker-controlled address.

### Finding Description

`ProposalInit` carries a `builder` field documented as "Address of the one who builds/sequences the block." [1](#0-0) 

`convert_to_sn_api_block_info` maps this field directly to `BlockInfo.sequencer_address`, which is the fee-recipient address used by the blockifier for every transaction in the block: [2](#0-1) 

`initiate_validation` passes the resulting `block_info` (containing the proposer-supplied `builder`) straight to the batcher without any prior check: [3](#0-2) 

`is_proposal_init_valid` — the sole gate that validates the semantic content of `ProposalInit` — checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas-price fields, `timestamp`, and `fee_proposal_fri`. It does **not** check `builder`: [4](#0-3) 

`ProposalInitValidation` — the struct that carries the validator's reference values — has no `builder` field at all, so there is no mechanism to enforce a correct value even if the code wanted to: [5](#0-4) 

Because `builder` feeds into `sequencer_address`, which is part of the block header, it is included in `partial_block_hash`. The `ProposalCommitment` is `Poseidon(partial_block_hash, fee_proposal_fri)`: [6](#0-5) 

The validator recomputes the commitment from the batcher's output using the same `init.fee_proposal_fri` it received from the proposer. Because the batcher also used the spoofed `builder` when executing the block, the `partial_block_hash` it returns already encodes the attacker's address. The validator's locally-computed commitment therefore matches the proposer's `ProposalFin.proposal_commitment`, and the proposal is accepted: [7](#0-6) 

### Impact Explanation

Every transaction fee in the block is transferred to `BlockInfo.sequencer_address`. By spoofing `builder`, a malicious proposer permanently redirects the entire fee revenue of one block to an address of their choosing. The committed block hash encodes the attacker's address, so the state is irrevocably wrong after `decision_reached`. This satisfies: **"Critical. Incorrect fee … balance … with economic impact."**

### Likelihood Explanation

The attacker must be the elected proposer for the current round — a role that rotates among the validator committee. Any Byzantine validator who wins a proposer slot can trigger this with zero additional preconditions: no chained exploit, no special network position, and no knowledge beyond the normal protocol messages. The attack is silent (no error is logged for a missing `builder` check) and leaves no on-chain trace distinguishing it from a legitimate block.

### Recommendation

1. Add a `builder: ContractAddress` field to `ProposalInitValidation`, populated from the node's own configured builder address (the same value used in `initiate_build`).
2. In `is_proposal_init_valid`, assert `init_proposed.builder == proposal_init_validation.builder` and return `InvalidProposalInit` on mismatch, mirroring the existing `l1_da_mode` / `l2_gas_price_fri` checks.
3. Add a regression test analogous to `invalid_proposal_init` that mutates `init.builder` and asserts rejection.

### Proof of Concept

```
1. Malicious validator M is elected proposer for height H, round R.

2. M constructs ProposalInit with:
     builder = 0xATTACKER_ADDRESS   // arbitrary; not the node's configured address
     (all other fields valid)

3. M sends ProposalPart::Init(init) to peers.

4. Each validator calls validate_proposal(init, …):
     a. is_proposal_init_valid() — passes; builder is never checked.
     b. initiate_validation() calls convert_to_sn_api_block_info(init):
          sequencer_address = 0xATTACKER_ADDRESS
        Passes this BlockInfo to batcher.validate_block().

5. Batcher executes all transactions with sequencer_address = 0xATTACKER_ADDRESS.
   Every fee transfer goes to 0xATTACKER_ADDRESS.

6. Batcher returns partial_block_hash that encodes 0xATTACKER_ADDRESS.

7. Validator computes:
     batcher_commitment = Poseidon(partial_block_hash, init.fee_proposal_fri)
   This matches ProposalFin.proposal_commitment sent by M (M computed the same hash).
   → ProposalFinMismatch check passes.

8. Consensus reaches decision; decision_reached commits the block.
   All fees for block H are permanently held by 0xATTACKER_ADDRESS.
   The legitimate sequencer receives nothing.
```

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
