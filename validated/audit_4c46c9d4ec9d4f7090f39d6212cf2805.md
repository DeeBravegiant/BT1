### Title
Unvalidated `ProposalInit.builder` Field Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Header State - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit.builder` is a proposer-supplied field that flows directly into `sequencer_address` inside `BlockInfo` used for block execution. The `is_proposal_init_valid` function validates many `ProposalInit` fields but never checks `init.builder` against any locally-trusted reference. A malicious proposer can set `builder` to an arbitrary address; validators accept the proposal, execute the block with the wrong `sequencer_address`, and commit a corrupted block header to state — redirecting all sequencer fees and poisoning every `get_sequencer_address` syscall result for that block.

### Finding Description

`ProposalInit` carries a `builder` field (the address of the block sequencer/builder): [1](#0-0) 

During proposal building, the honest proposer sets this from its own node configuration: [2](#0-1) 

When a validator receives a proposal, `is_proposal_init_valid` is called. It checks `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas price fields, and `fee_proposal_fri`. It does **not** check `init.builder`: [3](#0-2) 

The `ProposalInitValidation` struct — the sole source of locally-trusted reference values passed into `is_proposal_init_valid` — contains no expected `builder` field at all: [4](#0-3) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`: [5](#0-4) 

`convert_to_sn_api_block_info` places `init.builder` verbatim into `sequencer_address`: [6](#0-5) 

The same `init.builder` is also written into the committed `BlockHeaderWithoutHash` that is forwarded to state sync: [7](#0-6) 

Because both the proposer and every validator derive `sequencer_address` from the same (unvalidated) `init.builder`, the batcher on both sides computes the same `PartialBlockHash`, the `ProposalFin` commitment comparison passes, and the corrupted block is committed without any alarm: [8](#0-7) 

### Impact Explanation

**Wrong execution result / wrong state (Critical):** Every `get_sequencer_address` syscall executed inside the block returns the attacker-controlled address instead of the legitimate sequencer address. Contracts that branch on the sequencer address (e.g., fee-market or governance contracts) execute incorrect logic, producing wrong storage writes, events, and receipts that are permanently committed to state.

**Incorrect fee accounting (Critical):** In Starknet, transaction fee transfers are directed to `sequencer_address`. With `sequencer_address` pointing to the attacker's address, all sequencer fees collected in the block are transferred to the attacker rather than the legitimate sequencer, constituting direct economic theft proportional to the block's total fee revenue.

**Corrupted block header in state:** The `BlockHeaderWithoutHash.sequencer` field stored by state sync carries the wrong address, poisoning any downstream consumer (provers, L1 verifiers, RPC `starknet_getBlockWithTxHashes`) that reads the sequencer field from storage.

### Likelihood Explanation

Any registered validator that wins a proposal round can exploit this. In a BFT network tolerating `f` Byzantine validators, up to `f` nodes can mount this attack every time they are elected proposer. No special knowledge or external resource is required beyond being a scheduled proposer. The attack is silent: no existing guard (proposer-identity check, `ProposalFin` commitment comparison, gas-price margin check) detects a modified `builder` field.

### Recommendation

1. Add an `expected_builder: ContractAddress` field to `ProposalInitValidation`, populated from the validator's own configuration or from a proposer→builder registry.
2. In `is_proposal_init_valid`, assert `init_proposed.builder == proposal_init_validation.expected_builder` and return `Err(InvalidProposalInit(...))` on mismatch.
3. If a per-proposer builder registry is not yet available, enforce `init.builder == init.proposer` as a conservative interim invariant, since the proposer identity is already authenticated by the consensus layer.

### Proof of Concept

```
1. Malicious validator M is elected proposer for height H, round R.

2. M modifies initiate_build so that:
       init.builder = ContractAddress::from(M_ATTACKER_WALLET)
   instead of the node's configured builder_address.

3. M broadcasts ProposalPart::Init(init) with the modified builder field.

4. Each honest validator calls:
       is_proposal_init_valid(&proposal_init_validation, &init, ...)
   The function checks height, l1_da_mode, l2_gas_price_fri, all L1 gas
   prices, starknet_version, version_constant_commitment, fee_proposal_fri
   — but has no branch that reads init.builder. Returns Ok(()).

5. initiate_validation calls convert_to_sn_api_block_info(&init), which
   produces BlockInfo { sequencer_address: M_ATTACKER_WALLET, ... }.
   This BlockInfo is sent to the batcher via validate_block().

6. The batcher executes all transactions with sequencer_address =
   M_ATTACKER_WALLET. Every fee transfer in the block sends funds to
   M_ATTACKER_WALLET. Every get_sequencer_address syscall returns
   M_ATTACKER_WALLET.

7. The batcher returns PartialBlockHash computed over the (corrupted)
   block header. M's proposer side computed the same hash. The
   ProposalFin commitment comparison passes:
       built_block == received_fin.proposal_commitment  ✓

8. decision_reached commits the block. BlockHeaderWithoutHash.sequencer
   = M_ATTACKER_WALLET is written to state sync storage.

Result: M has stolen all sequencer fees for block H and permanently
corrupted the on-chain sequencer address for that block.
``` [3](#0-2) [9](#0-8) [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
