Audit Report

## Title
`checkDailyMintLimit` Blocks All Minting When `maxMintAmountPerDay` Is Zero — (`contracts/RSETH.sol`)

## Summary
`RSETH.initialize()` and `reinitialize()` never set `maxMintAmountPerDay`, leaving it at the Solidity default of `0`. The `checkDailyMintLimit` modifier treats `0` as a hard cap of zero rather than a sentinel for "unlimited," causing every call to `mint()` — and therefore every `depositETH` and `depositAsset` call — to revert with `DailyMintLimitExceeded` until a manager explicitly calls `setMaxMintAmountPerDay` with a non-zero value.

## Finding Description
`RSETH.initialize()` sets up ERC20, Pausable, and `lrtConfig` but does not assign `maxMintAmountPerDay`. `reinitialize()` sets `periodStartTime` and `custodyAddress` but also leaves `maxMintAmountPerDay` untouched. The only setter, `setMaxMintAmountPerDay`, accepts `0` without validation.

The enforcement modifier:
```solidity
// contracts/RSETH.sol L50-51
if (currentPeriodMintedAmount + amount > maxMintAmountPerDay) {
    revert DailyMintLimitExceeded(currentPeriodMintedAmount + amount, maxMintAmountPerDay);
}
```
When `maxMintAmountPerDay == 0`, this reduces to `amount > 0`, which is always true for any real deposit. `mint()` applies this modifier unconditionally. `LRTDepositPool._mintRsETH` calls `IRSETH(rsethToken).mint(msg.sender, rsethAmountToMint)`, so both `depositETH` and `depositAsset` revert in this state. `remainingDailyMintLimit()` returns `0` when `maxMintAmountPerDay == 0`, confirming the code treats `0` as "no minting allowed" rather than "unlimited."

Two paths reach this state:
1. **Default post-deployment**: Between `reinitialize()` and the first call to `setMaxMintAmountPerDay`, the contract is live but all deposits revert.
2. **Manager resets to zero**: `setMaxMintAmountPerDay(0)` is accepted silently, re-entering the broken state.

## Impact Explanation
When `maxMintAmountPerDay == 0`, every call to `LRTDepositPool.depositETH` and `LRTDepositPool.depositAsset` reverts. No rsETH is minted and no assets are accepted. No funds already held are lost, but the contract fails to deliver its core promised return — liquid restaking tokens in exchange for deposited ETH/LSTs.

**Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
Path 1 is reachable by any depositor during the window between `reinitialize()` and the first `setMaxMintAmountPerDay` call — no attacker capability required, just a normal deposit attempt. Path 2 requires a manager action but is accepted without revert. Neither path requires an unprivileged attacker to do anything beyond a standard deposit call.

**Likelihood: Low.**

## Recommendation
Add a sentinel bypass in `checkDailyMintLimit` so that `maxMintAmountPerDay == 0` means "no limit":
```solidity
modifier checkDailyMintLimit(uint256 amount) {
    if (maxMintAmountPerDay != 0) {
        if (block.timestamp >= periodStartTime + 1 days) {
            currentPeriodMintedAmount = 0;
            periodStartTime = getCurrentPeriodStartTime();
        }
        if (currentPeriodMintedAmount + amount > maxMintAmountPerDay) {
            revert DailyMintLimitExceeded(currentPeriodMintedAmount + amount, maxMintAmountPerDay);
        }
        currentPeriodMintedAmount += amount;
    }
    _;
}
```
Also update `remainingDailyMintLimit()` to return `type(uint256).max` when `maxMintAmountPerDay == 0`. Alternatively, set a non-zero default in `initialize()` and add `require(_maxMintAmountPerDay > 0)` in `setMaxMintAmountPerDay`.

## Proof of Concept
1. Deploy `RSETH` and call `initialize(admin, lrtConfig)`. `maxMintAmountPerDay` is `0`.
2. Call `reinitialize(periodStartTime, custodyAddress)`. `maxMintAmountPerDay` remains `0`.
3. A depositor calls `LRTDepositPool.depositETH{value: 1 ether}(0, "")`.
4. `_beforeDeposit` passes (amount > 0, within deposit limit, rsethAmountToMint > 0).
5. `_mintRsETH` calls `IRSETH(rsethToken).mint(depositor, rsethAmount)`.
6. `checkDailyMintLimit` evaluates `0 + rsethAmount > 0` → `true` → reverts with `DailyMintLimitExceeded(rsethAmount, 0)`.
7. All deposits revert until `setMaxMintAmountPerDay` is called with a non-zero value.
