### Title
Missing Paillier Ciphertext Validation in `blob_to_keyset` Allows Tampered HD Keyset Blob to Produce Accepted-Invalid Derived ECDSA-2PC Key Blobs — (`src/cbmpc/api/hd_keyset_ecdsa_2p.cpp`)

### Summary

`blob_to_keyset` in `src/cbmpc/api/hd_keyset_ecdsa_2p.cpp` omits the `paillier.verify_cipher(c_key)` check that its counterpart `blob_to_key` in `src/cbmpc/api/ecdsa2pc.cpp` explicitly performs. An attacker who controls the serialized `keyset_blob_v1_t` (e.g., via a tampered persisted blob) can supply a structurally valid blob whose `c_key` is not a valid Paillier ciphertext. The entire `derive_ecdsa_2p_keys` call chain accepts this blob, returns `SUCCESS`, and emits derived ECDSA-2PC key blobs that carry the malformed `c_key`. Those blobs are then rejected only when the caller attempts to use them for signing via `coinbase::api::ecdsa_2p::sign`, which calls `blob_to_key` and does perform `verify_cipher`.

### Impact Explanation

An attacker who can supply a tampered `keyset_blob_v1_t` (e.g., a blob read from an externally stored/transmitted opaque buffer) can cause `derive_ecdsa_2p_keys` to return `SUCCESS` and emit derived ECDSA-2PC key blobs that are structurally well-formed but cryptographically invalid. The derive stage accepts and propagates the invalid state without any error signal. The invalidity surfaces only at sign time, where `blob_to_key` rejects the blob. This constitutes accepted invalid cryptographic output from a public API: the derive API vouches for blobs it should have rejected.

Additionally, the same missing checks apply to the `refresh` path, which also calls `deserialize_keyset_blob → blob_to_keyset` and then passes the resulting `ecdsa2pc::key_t` (with the unvalidated `c_key`) directly into `ecdsa2pc::refresh`. [6](#0-5) 

### Likelihood Explanation

The keyset blob is an opaque, caller-persisted byte string. Any storage or transport layer that an attacker can write to (database, file system, network) is a viable injection point. Crafting a structurally valid blob with an invalid `c_key` (e.g., setting `c_key = N`, which is in `[0, N²)` but not coprime with `N`) requires only knowledge of the serialization format, which is fully described by `keyset_blob_v1_t::convert`.

### Recommendation

Add the same defensive checks to `blob_to_keyset` that `blob_to_key` already performs:

1. Check `(blob.role == 0) == blob.paillier.has_private_key()`.
2. Check `N.value().get_bits_count() >= paillier_t::bit_size`.
3. Call `blob.paillier.verify_cipher(blob.c_key)` and return an error on failure.
4. If P1 (private key present), decrypt `c_key` and assert the plaintext equals `N.mod(blob.x_share)`.

These checks are already proven correct in `blob_to_key` and should be factored into a shared helper to prevent future divergence.

  //
  // In our ECDSA-2PC protocol, party P1 owns the Paillier private key, and `c_key` is an encryption of P1's share under
  // that key. Reject malformed / tampered blobs early.
  const bool paillier_has_private = blob.paillier.has_private_key();
  if ((blob.role == 0) != paillier_has_private) return coinbase::error(E_FORMAT, "invalid key blob");

  const auto& N = blob.paillier.get_N();
  if (N.value().get_bits_count() < coinbase::crypto::paillier_t::bit_size) {
    return coinbase::error(E_FORMAT, "invalid key blob");
  }

  // Intentionally do not enforce `x_share in [0, q)` here:
  // in ECDSA-2PC this share is maintained as a Paillier-compatible integer representative and can be
  // unreduced after refresh, so rejecting non-reduced values would break valid refreshed key blobs.
  //
  // However, `x_share` must remain in Z_N so that Paillier-related operations are well-defined and to avoid
  // attacker-controlled bignum blowups.
  if (!N.is_in_range(blob.x_share)) return coinbase::error(E_FORMAT, "invalid key blob");

  // Ensure `c_key` is a well-formed Paillier ciphertext under this key.
  {
    coinbase::crypto::vartime_scope_t vartime_scope;
    if (blob.paillier.verify_cipher(blob.c_key)) return coinbase::error(E_FORMAT, "invalid key blob");
  }

  // If we have the private key (P1), bind the share to its Paillier encryption.
  if (paillier_has_private) {
    const coinbase::crypto::bn_t plain = blob.paillier.decrypt(blob.c_key);
    if (plain != N.mod(blob.x_share)) return coinbase::error(E_FORMAT, "invalid key blob");
  }

static error_t serialize_ecdsa2pc_key_blob(const coinbase::mpc::ecdsa2pc::key_t& key, buf_t& out) {
  curve_id cid;
  if (!from_internal_curve(key.curve, cid)) return coinbase::error(E_BADARG, "unsupported curve");
  if (cid == curve_id::ed25519) return coinbase::error(E_BADARG, "unsupported curve");

  ecdsa2pc_key_blob_v1_t blob;
  blob.role = static_cast<uint32_t>(key.role);
  blob.curve = static_cast<uint32_t>(cid);
  blob.Q_compressed = key.Q.to_compressed_bin();
  blob.x_share = key.x_share;
  blob.c_key = key.c_key;
  blob.paillier = key.paillier;
  out = coinbase::convert(blob);
  return SUCCESS;
}
