Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Pause via Natural Price Dip — (`contracts/LRTOracle.sol`)

## Summary
`updateRSETHPrice()` is a permissionless `public` function with no role guard. When the computed `newRsETHPrice` falls below `highestRsethPrice` by more than `pricePercentageLimit`, the downside-protection branch at lines 277–281 unconditionally pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself for any caller. Any unprivileged address can freeze the entire protocol by calling this function during a routine market dip, with no oracle manipulation required.

## Finding Description
`updateRSETHPrice()` at line 87 carries only `whenNotPaused` — no role check: [1](#0-0) 

Inside `_updateRsETHPrice()`, the upside branch at lines 260–266 correctly gates privileged action behind a `MANAGER` role check, reverting for unprivileged callers: [2](#0-1) 

The downside branch at lines 270–282 has no equivalent caller-identity check. When `isPriceDecreaseOffLimit` is true, it unconditionally calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` for any `msg.sender`: [3](#0-2) 

`highestRsethPrice` is a monotonically non-decreasing watermark, updated only when `newRsETHPrice > highestRsethPrice`: [4](#0-3) 

This means any prior all-time high permanently raises the bar, making the protocol perpetually vulnerable to the next dip below that peak. `_pause()` is idempotent but irreversible without admin action: [5](#0-4) 

Unpausing requires `onlyLRTAdmin`: [6](#0-5) 

## Impact Explanation
**Medium — Temporary freezing of funds.** All three pause calls execute atomically in a single transaction from an unprivileged address. After the call, `LRTDepositPool.paused() == true` (no deposits), `LRTWithdrawalManager.paused() == true` (no withdrawals or claims), and `LRTOracle.paused == true` (price updates blocked). User funds are inaccessible until an admin manually calls `unpause()` on each contract. The freeze is temporary (admin-recoverable) but requires no attacker privilege and can be repeated after each unpause.

## Likelihood Explanation
- `pricePercentageLimit` is admin-configurable (e.g., 1% = `1e16`). A 1% intraday dip in stETH, ETHx, or any supported LST is routine market behavior.
- `highestRsethPrice` is the all-time peak, so even a single prior price update at a higher value permanently lowers the threshold for triggering the pause.
- No MEV, no front-running, no oracle manipulation, and no privileged access are required — only a public call at the right block.
- The attacker pays only gas. The attack is repeatable after each admin unpause.

## Recommendation
Mirror the upside branch pattern in the downside branch. Two options:

1. **Role-gate the pause branch**: inside `_updateRsETHPrice()`, only execute the `lrtDepositPool.pause()` / `withdrawalManager.pause()` / `_pause()` calls when `msg.sender` holds `PAUSER_ROLE` or `MANAGER`. For unprivileged callers, revert (e.g., `revert PriceBelowDailyThreshold()`) instead of pausing.

2. **Separate concerns**: remove the auto-pause side-effect from `_updateRsETHPrice()` entirely. Emit an event on price-drop-beyond-threshold and let an off-chain keeper with `PAUSER_ROLE` react via the existing `pause()` function at line 138.

## Proof of Concept
Minimal Foundry fork test sequence:

```solidity
// 1. Deploy protocol; set pricePercentageLimit = 1e16 (1%)
// 2. Call updateRSETHPrice() at normal price → highestRsethPrice is set to P
// 3. Drop mock oracle price so that newRsETHPrice < P * 0.99 (> 1% below peak)
// 4. Call lrtOracle.updateRSETHPrice() as address(1) (unprivileged)
// 5. Assert:
assertEq(lrtDepositPool.paused(), true);
assertEq(withdrawalManager.paused(), true);
assertEq(lrtOracle.paused(), true);
```

Step 4 succeeds with no revert because `updateRSETHPrice()` has no role check and the downside branch at lines 277–281 executes unconditionally. The condition `diff > pricePercentageLimit.mulWad(highestRsethPrice)` is satisfied by the mock price drop, and all three pause calls complete atomically. [7](#0-6)
