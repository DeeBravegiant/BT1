Audit Report

## Title
`rsETHPrice` Baseline Advanced Unconditionally While Fee Is Skipped During Partial Pause — (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle._updateRsETHPrice()` skips protocol-fee collection when `protocolPaused` is `true` (line 244), but unconditionally writes `rsETHPrice = newRsETHPrice` at line 313 regardless. Because `previousTVL` on the next call is derived from the already-advanced `rsETHPrice`, any staking rewards that accrued during the pause window are permanently excluded from fee collection. The oracle's own `paused` flag is independent of `lrtDepositPool.paused()` and `withdrawalManager.paused()`, so `updateRSETHPrice()` remains publicly callable during routine operational pauses of either downstream contract.

## Finding Description

`_updateRsETHPrice()` computes the pause state as:

```solidity
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;
``` [1](#0-0) 

When `protocolPaused` is `true`, `protocolFeeInETH` is forced to zero:

```solidity
uint256 protocolFeeInETH = 0;
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [2](#0-1) 

`newRsETHPrice` is then computed from the full `totalETHInProtocol` (no fee deducted), and the price is committed unconditionally:

```solidity
rsETHPrice = newRsETHPrice;
``` [3](#0-2) 

The public entry point is gated only by the oracle's own `paused` flag:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [4](#0-3) 

The oracle's `paused` is a separate `bool` from `lrtDepositPool.paused()` and `withdrawalManager.paused()`. [5](#0-4)  The downside-protection auto-pause only fires when the oracle itself detects an out-of-limit price drop and calls `_pause()`. [6](#0-5)  During any ordinary operational pause of the deposit pool or withdrawal manager, the oracle remains unpaused and `updateRSETHPrice()` is open to any caller.

**Exploit path:**
1. `lrtDepositPool` is paused for an upgrade; oracle is not paused.
2. EigenLayer rewards accrue; `totalETHInProtocol` grows by `Δ`.
3. Any rsETH holder calls `updateRSETHPrice()`. `protocolPaused = true`, so `protocolFeeInETH = 0`, but `rsETHPrice` is advanced to `(previousTVL + Δ) / rsethSupply`.
4. Deposit pool is unpaused.
5. Next `updateRSETHPrice()` call: `previousTVL = rsethSupply × newRsETHPrice` already incorporates `Δ`. No fee is ever collected on `Δ`.

The caller is economically incentivised: advancing the price baseline without fee dilution increases per-share value for all rsETH holders.

## Impact Explanation

**High — Theft of unclaimed yield.** The protocol treasury is the intended recipient of `protocolFeeInBPS` of all staking rewards. Every call to `updateRSETHPrice()` during a partial pause permanently excludes the reward delta for that period from fee collection. The loss is irreversible once `rsETHPrice` has been advanced, scales with reward size and fee rate, and can be repeated across every operational pause.

## Likelihood Explanation

**Medium.** Deposit pool and withdrawal manager pauses are routine operational events (upgrades, EigenLayer queue management, security incidents). The oracle is not co-paused in these scenarios. Any rsETH holder — an unprivileged external actor — can call the public `updateRSETHPrice()` function at any time the oracle itself is not paused. No special permissions, flash loans, or governance capture are required.

## Recommendation

Do not advance `rsETHPrice` when the fee calculation was skipped. Two equivalent fixes:

1. **Guard the price write:** only commit `rsETHPrice = newRsETHPrice` when `!protocolPaused` (or equivalently, when `protocolFeeInETH` was actually computed).
2. **Defer the baseline:** when `protocolPaused`, compute `newRsETHPrice` for display/event purposes but skip the `rsETHPrice = newRsETHPrice` assignment and the `highestRsethPrice` ratchet update. This also prevents the downside-protection threshold from advancing during a pause.

Option 2 is preferable because it also prevents `highestRsethPrice` from being ratcheted upward during a pause, which would otherwise raise the threshold against which future price drops are measured.

## Proof of Concept

**Minimal call sequence (local fork):**

```
State:  rsETHPrice = 1.05e18, rsethSupply = 1_000_000e18, protocolFeeInBPS = 1000 (10%)
        lrtDepositPool.paused() = false, oracle.paused = false

Step 1: admin calls lrtDepositPool.pause()
        → lrtDepositPool.paused() = true, oracle.paused = false

Step 2: EigenLayer rewards accrue; totalETHInProtocol increases by 1_000e18

Step 3: attacker (any EOA) calls lrtOracle.updateRSETHPrice()
        → protocolPaused = true
        → protocolFeeInETH = 0
        → newRsETHPrice = (1_050_000e18 + 1_000e18) / 1_000_000e18 = 1.051e18
        → rsETHPrice = 1.051e18  ← baseline advanced, no fee minted

Step 4: admin calls lrtDepositPool.unpause()

Step 5: anyone calls lrtOracle.updateRSETHPrice()
        → previousTVL = 1_000_000e18 × 1.051e18 = 1_051_000e18
        → totalETHInProtocol ≈ 1_051_000e18 (no new rewards)
        → rewardAmount = 0, protocolFeeInETH = 0
        → The 1_000e18 reward from Step 2 was never subject to the fee

Treasury loss: 1_000e18 × 10% = 100e18 ETH worth of rsETH fee, permanently unrecoverable.
```

**Foundry invariant test sketch:**

```solidity
// Invariant: sum of all protocol fees minted must equal
// sum over all updateRSETHPrice() calls of
//   (totalETHInProtocol - previousTVL) * protocolFeeInBPS / 10_000
//   when totalETHInProtocol > previousTVL AND !protocolPaused at call time.
// Fork the state, pause depositPool, advance rewards, call updateRSETHPrice(),
// unpause, call again — assert treasury balance matches expected cumulative fee.
```
