### Title
Ed25519 `from_bin` Accepts Non-Canonical y-Coordinate Encodings, Causing Key Blob Malleability — (`src/cbmpc/crypto/ec25519_core.cpp`, `src/cbmpc/api/eddsa2pc.cpp`)

### Summary
`ec25519_core::from_bin` does not reject y-coordinate values in the range [p, 2^255−1] (19 non-canonical values). These are accepted, decoded to the same point as their canonical counterpart (y mod p), and pass `curve.check`. Re-encoding via `to_compressed_bin` always produces the canonical form. This creates blob malleability: two distinct 32-byte `Q_compressed` encodings produce identical signing keys and identical `get_public_key_compressed` output.

### Finding Description

The Ed25519 field prime is p = 2^255 − 19. The y-coordinate in the compressed encoding is a 255-bit value (high bit is the x-sign bit). Values y ∈ [p, 2^255−1] are non-canonical — they are not the unique representative of their field element.

In `ec25519_core::from_bin`: [1](#0-0) 

`fe_t::to_fe` stores the raw `uint256_t` without reduction: [2](#0-1) 

All subsequent field arithmetic (`y*y`, etc.) operates on the lazy representation; `fe_freeze` is only called inside `from_fe()` / `is_zero()` / `is_odd()`, so the arithmetic result is identical to using y mod p. No error is returned for y ≥ p.

`blob_to_key` calls `curve.check` after `from_bin`: [3](#0-2) 

`ecurve_t::check` only verifies subgroup membership and non-infinity — it does not verify canonical encoding: [4](#0-3) 

Re-encoding via `to_bin` always produces the canonical form because `y.from_fe()` calls `fe_freeze`: [5](#0-4) 

`serialize_key_blob` uses `to_compressed_bin()` which goes through this same canonical path: [6](#0-5) 

### Impact Explanation

An API caller can supply a `key_blob_v1_t` with `Q_compressed` encoding y = p + k (for k ∈ [0, 18]) instead of the canonical y = k. Both blobs:
- Pass `from_bin` without error
- Pass `curve.check` (subgroup check operates on the decoded point, not the encoding)
- Produce identical `key.Q` internal state
- Return identical output from `get_public_key_compressed`

This is **blob malleability**: for any valid key blob, up to 19 distinct byte sequences are accepted as equivalent. The impact is a parsing/serialization invariant break — the blob boundary does not enforce a unique canonical representation of Q. This is a **Medium** impact finding. It does **not** enable signature forgery, key recovery, or bypass of the honest-participant requirement; the decoded key is identical in all cases.

### Likelihood Explanation

Trivially constructable: take any valid blob, add p to the y-coordinate bytes of `Q_compressed` (with the high bit still clear), and the modified blob is accepted. No cryptographic knowledge is required beyond knowing p = 2^255 − 19.

### Recommendation

Add an explicit canonical-encoding check in `ec25519_core::from_bin` after extracting the y-coordinate: reject any input where the 255-bit y value is ≥ p. This is a single comparison against the field prime after `buf[0] &= 0x7f`. RFC 8032 §5.1.3 requires this check ("if the resulting value is ≥ p, decoding fails"). [7](#0-6)

static error_t from_bin(point_t& R, mem_t bin) {
  if (bin.size != 32) return coinbase::error(E_FORMAT);

  buf_t buf = bin.rev();
  uint8_t neg = buf[0] >> 7;
  buf[0] &= 0x7f;
  fe_t y = fe_t::to_fe(uint256_t::from_bin(buf));

  // x² = (y² - 1) / (dy² + 1)

  fe_t u, v, w, vxx, check;

  u = y * y;
  v = u * formula_t::get_d();
  u -= fe_t::one();       // u = y^2-1
  v += fe_t::one();       // v = dy^2+1
  w = u * v;              // w = u*v
  fe_t x = w.pow22523();  // x = w^((q-5)/8)
  x *= u;                 // x = u * w^((q-5)/8)

  vxx = x * x;
  vxx *= v;
  check = vxx - u;  // vx^2-u
  if (!check.is_zero()) {
    check = vxx + u;  // vx^2+u
    if (!check.is_zero()) {
      return coinbase::error(E_CRYPTO);
    }
    static const fe_t sqrtm1 =
        fe_t::from_bn(bn_t::from_hex("2b8324804fc1df0b2b4d00993dfbd7a72f431806ad2fe478c4ee1b274a0ea0b0"));
    x *= sqrtm1;
  }

  uint256_t x_val = x.from_fe();
  if (neg != (x_val.w0 & 1)) x = -x;

  R.x = x;
  R.y = y;
  R.z = fe_t::one();
  return 0;
}
