### Title
Dijkstra Era `getMinFeeTxUtxo` Wired to Conway's `txNonDistinctRefScriptsSize`, Silently Excluding Subtransaction Reference Scripts from Fee Calculation — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces nested transactions (subtransactions) and defines `batchNonDistinctRefScriptsSize` to compute the total reference-script byte count across both the top-level transaction and all its subtransactions. However, the `EraUTxO DijkstraEra` instance wires `getMinFeeTxUtxo` directly to `getConwayMinFeeTxUtxo`, which internally calls only `txNonDistinctRefScriptsSize` (top-level only). The Dijkstra-specific `batchNonDistinctRefScriptsSize` is never invoked during fee validation, so reference scripts carried by subtransactions are invisible to the minimum-fee check.

---

### Finding Description

The Dijkstra era adds a new transaction structure that allows a top-level `Tx TopTx DijkstraEra` to embed a list of subtransactions via `subTransactionsTxBodyL`. Each subtransaction can reference UTxO outputs that carry reference scripts, contributing to the total deserialization cost that the tiered reference-script fee (`tierRefScriptFee`) is designed to price.

To handle this, `batchNonDistinctRefScriptsSize` is defined in `Dijkstra/UTxO.hs`:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
```

This function correctly aggregates reference-script sizes from the top-level transaction **and** every subtransaction.

However, the `EraUTxO DijkstraEra` instance sets:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
```

`getConwayMinFeeTxUtxo` is defined in `Conway/UTxO.hs` as:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
```

It passes only `txNonDistinctRefScriptsSize utxo tx` — the top-level count — to `getMinFeeTx`. Subtransaction reference scripts are never counted. The Dijkstra `EraTx` instance also delegates `getMinFeeTx = getConwayMinFeeTx`, so the entire fee pipeline ignores subtransaction reference scripts.

The correct Dijkstra implementation should be:

```haskell
getMinFeeTxUtxo pp tx utxo =
  getMinFeeTx pp tx $ batchNonDistinctRefScriptsSize utxo tx
```

This is structurally identical to the reported Mantra bug: a new function (`batchNonDistinctRefScriptsSize` / `ConvertToDenom`) is implemented for new protocol functionality but is never wired into the validation path that enforces it.

---

### Impact Explanation

The tiered reference-script fee (`tierRefScriptFee`) was introduced specifically to deter DDoS attacks via large reference scripts (see `docs/adr/2024-08-14_009-refscripts-fee-change.md`). By embedding large reference scripts exclusively in subtransactions, an attacker can submit a Dijkstra transaction whose minimum-fee check passes with a fee computed only over the top-level transaction's reference scripts. The subtransaction reference scripts — which nodes must still deserialize and validate — are processed at zero marginal cost to the submitter. This allows an attacker to exceed the intended per-transaction reference-script fee budget, modifying the effective fee paid outside the design parameters of the protocol.

This matches the allowed impact: **Medium — attacker-controlled transactions modify fees outside design parameters.**

---

### Likelihood Explanation

Any unprivileged transaction sender can craft a `DijkstraEra` transaction with subtransactions that reference large scripts. No special privilege, key, or governance action is required. The attacker only needs to construct a valid Dijkstra transaction with subtransactions that use reference inputs pointing to UTxO outputs containing large Plutus scripts. The ledger will accept the transaction as long as the top-level fee satisfies `getMinFeeTxUtxo`, which ignores subtransaction reference scripts entirely.

---

### Recommendation

Replace the `getMinFeeTxUtxo` binding in the `EraUTxO DijkstraEra` instance with a Dijkstra-specific implementation that uses `batchNonDistinctRefScriptsSize`:

```haskell
-- In eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs

instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo   -- replace getConwayMinFeeTxUtxo

getDijkstraMinFeeTxUtxo ::
  ( EraTx era
  , DijkstraEraTxBody era
  ) =>
  PParams era -> Tx TopTx era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx
```
