### Title
Block-Level Reference Script Size Limit Bypassed via Sub-Transaction Reference Inputs in Dijkstra Era — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Bbody.hs`)

---

### Summary

`dijkstraBbodyTransition` delegates the block-level reference-script-size check to `Conway.validateBodyRefScriptsSizeTooBig`, which internally calls `txNonDistinctRefScriptsSize` — a Conway function that only inspects the **top-level** transaction body. Sub-transactions are a Dijkstra-only concept; `txNonDistinctRefScriptsSize` has no knowledge of them. The per-transaction check in `dijkstraLedgerTransition` correctly uses `batchNonDistinctRefScriptsSize`, which sums both top-level and sub-transaction reference scripts. The block-level check does not. An attacker can therefore craft a block whose top-level transactions carry zero reference inputs while their sub-transactions carry reference inputs pointing to large scripts, causing the block-level limit `ppMaxRefScriptSizePerBlock` to be silently bypassed.

---

### Finding Description

**`txNonDistinctRefScriptsSize`** (Conway/UTxO.hs:183–187) only reads `tx ^. bodyTxL . referenceInputsTxBodyL` and `tx ^. bodyTxL . inputsTxBodyL` from the single top-level body: [1](#0-0) 

**`batchNonDistinctRefScriptsSize`** (Dijkstra/UTxO.hs:264–277) was introduced precisely to fix this gap at the per-tx level — it adds `txNonDistinctRefScriptsSize` for every sub-transaction: [2](#0-1) 

The **per-transaction** check in `dijkstraLedgerTransition` (Ledger.hs:392) correctly uses `batchNonDistinctRefScriptsSize`, bounded by `ppMaxRefScriptSizePerTx`: [3](#0-2) 

The **block-level** check in `dijkstraBbodyTransition` (Bbody.hs:363) calls `Conway.validateBodyRefScriptsSizeTooBig`, which calls `totalRefScriptSizeInBlock`, which calls `txNonDistinctRefScriptsSize` — **not** `batchNonDistinctRefScriptsSize`: [4](#0-3) [5](#0-4) 

Sub-transaction reference scripts are therefore **invisible** to the block-level guard.

---

### Impact Explanation

`ppMaxRefScriptSizePerBlock` is a resource-protection parameter. Its purpose is to cap the total script-loading cost imposed on validating nodes per block. Because the block-level check only sums top-level reference inputs, an attacker can construct a block where every top-level transaction has zero top-level reference inputs but each carries sub-transactions whose reference inputs resolve to large scripts. The block passes `validateBodyRefScriptsSizeTooBig` with a reported size of 0 (or near 0), while the actual script-loading cost is up to `N × ppMaxRefScriptSizePerTx` — potentially many multiples of `ppMaxRefScriptSizePerBlock`. This exceeds the intended validation limit, fitting the **Medium** impact category.

---

### Likelihood Explanation

The Dijkstra era is the only era with sub-transactions. Any unprivileged transaction author can craft such a block; no special keys, governance majority, or privileged access is required. The exploit is deterministic and locally testable against the STS rules.

---

### Recommendation

Replace the call to `Conway.validateBodyRefScriptsSizeTooBig` in `dijkstraBbodyTransition` with a Dijkstra-specific block-level check that uses `batchNonDistinctRefScriptsSize` (or an equivalent accumulating fold over top-level txs and their sub-transactions) instead of `txNonDistinctRefScriptsSize`. The per-tx analogue (`validateAllRefScriptSize` using `batchNonDistinctRefScriptsSize`) already exists in `Ledger.hs` and demonstrates the correct pattern.

totalRefScriptSizeInBlock ::
  (AlonzoEraTx era, BabbageEraTxBody era) => ProtVer -> StrictSeq (Tx TopTx era) -> UTxO era -> Int
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | pvMajor protVer <= natVersion @10 =
      getSum $ foldMap' (Monoid.Sum . txNonDistinctRefScriptsSize (UTxO utxo)) txs
  | otherwise =
      snd $ F.foldl' accum (utxo, 0) txs
  where
    accum (!accUtxo, !accSum) tx =
      let updatedUtxo = accUtxo `Map.union` unUTxO toAdd
          toAdd
            | IsValid True <- tx ^. isValidTxL = txouts $ tx ^. bodyTxL
            | otherwise = collOuts $ tx ^. bodyTxL
       in (updatedUtxo, accSum + txNonDistinctRefScriptsSize (UTxO accUtxo) tx)
