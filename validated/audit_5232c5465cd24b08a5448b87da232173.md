### Title
Unvalidated `builder` Field in `ProposalInit` Allows Malicious Proposer to Redirect Sequencer Fees and Corrupt Block Hash — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` validates most fields of `ProposalInit` that flow into the block hash, but silently omits `builder`. Because `builder` is mapped directly to `sequencer_address` in `BlockInfo` — which is both the fee-collection target in the blockifier and an input to `calculate_block_hash` — a malicious proposer can set it to any arbitrary address. All validators accept the proposal, execute the block with the attacker-controlled sequencer address, and commit a block whose hash encodes the wrong sequencer. This is the direct Sequencer analog of the external "escapeUnsafeCharacters misses single-quote" bug: a validation function that is supposed to sanitize every security-relevant field of a network-supplied message silently passes one field through unchecked, allowing it to "escape" into a sensitive downstream context.

### Finding Description

`ProposalInit` carries a `builder` field — the address of the entity that sequences the block. [1](#0-0) 

On the proposer side, `builder` is set from a static config value: [2](#0-1) 

On the validator side, `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (FRI and WEI), `timestamp`, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri`. It does **not** check `builder`: [3](#0-2) 

The `ProposalInitValidation` struct — the validator's reference values — has no `builder` field at all: [4](#0-3) 

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [5](#0-4) 

That `sequencer_address` is then embedded in `PartialBlockHashComponents`: [6](#0-5) 

And is hashed into the final block hash: [7](#0-6) 

`builder` is also written into `BlockHeaderWithoutHash.sequencer` in `update_state_sync_with_new_block`: [8](#0-7) 

The `ProposalFin` commitment check (`built_block != received_fin.proposal_commitment`) does not catch this because both the proposer and the validator derive the partial block hash from the same (attacker-controlled) `builder` value, so they agree on the commitment: [9](#0-8) 

### Impact Explanation

In the Starknet blockifier, `sequencer_address` is the fee-collection target: transaction fees are transferred to this address during execution. A malicious proposer that sets `builder` to an arbitrary address causes:

1. **Fee theft**: All transaction fees for the block are paid to the attacker-controlled address instead of the legitimate sequencer.
2. **Wrong block hash committed**: The block hash encodes the wrong `sequencer_address`, producing a permanently incorrect on-chain record. Every downstream system (state sync, L1 anchoring, RPC) will see and propagate this wrong hash.
3. **Wrong block header stored**: `BlockHeaderWithoutHash.sequencer` is persisted with the attacker's address, corrupting the authoritative storage view.

This matches the allowed impact: *"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact"* and *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result."*

### Likelihood Explanation

Any consensus participant that is selected as the round proposer (a normal, rotating role in Tendermint) can trigger this with a single-field change to their `ProposalInit` message. No special privilege beyond being the current-round proposer is required. The attack is silent — no error is logged, no metric is incremented, and the proposal passes all existing validation checks.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`, analogously to how `version_constant_commitment` is handled — the validator compares the proposer-supplied value against its own locally-configured `builder_address`:

```rust
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

`ProposalInitValidation` should be populated with `builder: self.config.static_config.builder_address` at the call sites in `sequencer_consensus_context.rs`.

### Proof of Concept

1. A node is selected as the round proposer.
2. In `initiate_build`, instead of using `args.builder_address`, the node sets `builder: attacker_address` in the constructed `ProposalInit`.
3. The modified `ProposalInit` is broadcast to all validators.
4. Each validator calls `is_proposal_init_valid` — it checks timestamp, starknet_version, version_constant_commitment, height, l1_da_mode, l2_gas_price_fri, all four L1 gas prices, and fee_proposal_fri. `builder` is not checked. The function returns `Ok(())`.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing a `BlockInfo` with `sequencer_address = attacker_address`.
6. The batcher executes the block: all transaction fees are transferred to `attacker_address`.
7. `PartialBlockHashComponents::new` encodes `attacker_address` as `sequencer`.
8. `calculate_block_hash` hashes `attacker_address` into the block hash.
9. The proposer's `ProposalFin.proposal_commitment` matches the validators' computed commitment (both used `attacker_address`), so the fin check passes.
10. Consensus reaches decision; the block is committed with the wrong sequencer address and wrong block hash.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
