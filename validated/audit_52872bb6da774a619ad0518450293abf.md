### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Commit Arbitrary Sequencer Address into Block Hash — (`File: crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/builder address). In `convert_to_sn_api_block_info`, this field is mapped directly to `BlockInfo.sequencer_address`, which is then committed into `PartialBlockHashComponents.sequencer` and ultimately into the block hash. The validator's `is_proposal_init_valid` function never checks `builder` against any locally-trusted reference value. A malicious proposer can set `builder` to an arbitrary address; all validators will accept the proposal and commit a block hash containing the wrong sequencer address, misdirecting execution-time fee collection and producing a wrong on-chain block hash commitment.

---

### Finding Description

`ProposalInit` has two identity fields:

- `proposer` — the consensus participant who proposed the block. This **is** checked in `handle_proposal` against the committee's expected proposer for the round.
- `builder` — "Address of the one who builds/sequences the block." This is **never** validated anywhere in the validator path.

The critical data flow is:

```
ProposalInit.builder
  → convert_to_sn_api_block_info()  [utils.rs:332]
      sequencer_address: init.builder
  → BlockInfo.sequencer_address
  → PartialBlockHashComponents::new()  [block_hash_calculator.rs:231]
      sequencer: SequencerContractAddress(block_info.sequencer_address)
  → calculate_block_hash()  [block_hash_calculator.rs:258]
      .chain(&partial_block_hash_components.sequencer.0)
  → PartialBlockHash / ProposalCommitment
```

In `is_proposal_init_valid`, the following fields are validated: `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, L1 gas prices (within margin), and `fee_proposal_fri`. The `builder` field is absent from both `ProposalInitValidation` and from every check inside `is_proposal_init_valid`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

The proposer sets `builder` in `initiate_build` from its own local `args.builder_address`. No validator has a reference value for what the proposer's `builder_address` should be, so no cross-check is possible without a protocol-level fix. [5](#0-4) 

---

### Impact Explanation

**Wrong block hash commitment (Critical):** `sequencer_address` is a direct input to `calculate_block_hash`. A spoofed `builder` value produces a `PartialBlockHash` — and therefore a `ProposalCommitment` — that encodes the wrong sequencer address. Every validator independently re-executes the block using the same `init.builder` value, so all nodes converge on the same wrong hash. The committed block hash is permanently wrong.

**Fee misdirection (Critical):** During blockifier execution, `sequencer_address` is the address that receives sequencer fees. A malicious proposer who sets `builder` to their own address redirects all sequencer fees for that block to themselves, away from the legitimate builder. [6](#0-5) 

---

### Likelihood Explanation

Any validator that wins a proposal round can exploit this. In a BFT protocol, every validator is a legitimate proposer in some round — this requires no special privilege beyond normal consensus participation. The attack is silent: validators accept the proposal normally (the `proposer` identity check passes), and the wrong `builder` value is never flagged. The only observable effect is the wrong sequencer address in the committed block hash and misdirected fees. [7](#0-6) 

---

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. The validator should compare `init_proposed.builder` against a locally-configured expected builder address (analogous to how `l2_gas_price_fri` is compared against the locally-computed value). If the system supports a dynamic builder address (e.g., from a builder registry), the validator should fetch the expected builder for the current height and reject proposals that deviate.

```rust

    return Err(ValidateProposalError::InvalidProposalInit(

        proposal_init_validation.clone(),

``` [8](#0-7) 

---

### Proof of Concept

1. Attacker controls a validator node that wins round `R` at height `H`.
2. In `initiate_build`, attacker sets `builder_address` to `ContractAddress::from(ATTACKER_ADDRESS)` instead of the legitimate sequencer address.
3. The resulting `ProposalInit` has `builder = ATTACKER_ADDRESS`.
4. The `ProposalInit` is streamed to all peer validators as `ProposalPart::Init`.
5. Each validator calls `validate_proposal` → `is_proposal_init_valid`. The `builder` field is never checked. Validation passes.
6. Each validator calls `convert_to_sn_api_block_info(&init)`, which sets `sequencer_address = ATTACKER_ADDRESS`.
7. `PartialBlockHashComponents::new(&block_info, ...)` stores `sequencer = ATTACKER_ADDRESS`.
8. `PartialBlockHash::from_partial_block_hash_components(...)` hashes `ATTACKER_ADDRESS` into the commitment.
9. All validators return the same `ProposalCommitment` (containing the wrong sequencer). Consensus decides on this commitment.
10. The block is committed to storage with `sequencer_address = ATTACKER_ADDRESS` in the block hash. All sequencer fees for block `H` are paid to `ATTACKER_ADDRESS`. [9](#0-8) [10](#0-9) [11](#0-10)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
