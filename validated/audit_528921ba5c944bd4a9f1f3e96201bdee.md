---

### Summary

In the Dijkstra era, a top-level transaction can embed sub-transactions (`subTransactionsTxBodyL`), each carrying its own Plutus redeemers with associated `ExUnits` budgets. The `dijkstraUtxoTransition` rule validates the per-transaction ExUnits limit and the minimum fee using `totExUnits tx`, which only sums the **top-level** transaction's redeemers. Sub-transaction ExUnits are silently excluded from both checks. An unprivileged sender can therefore craft a batch transaction whose sub-transactions contain arbitrarily expensive Plutus scripts, paying only the minimum fee for the top-level scripts and bypassing the `maxTxExUnits` per-transaction cap.

---

### Finding Description

The Dijkstra era introduces nested sub-transactions. Each sub-transaction carries its own `TxWits` (including redeemers with `ExUnits`), but sub-transactions have **no fee field** â€” the single top-level `txfee` is intended to cover the entire batch.

### Impact Explanation

**Fee underpayment (Medium):** An attacker submits a batch transaction whose top-level body carries zero or minimal redeemers (paying a small `txscriptfee`) while embedding sub-transactions with large Plutus ExUnits budgets. The ledger accepts the transaction because `minfee` only accounts for top-level ExUnits. Scripts in sub-transactions execute at the expense of block producers without the sender paying the corresponding fee. This modifies fees outside design parameters.

**ExUnits limit bypass (Medium):** By distributing ExUnits across sub-transactions, an attacker can cause a single batch to consume far more than `maxTxExUnits` of script execution resources per transaction, violating the intended per-transaction resource cap and potentially degrading block validation performance.

Both impacts fall under: *"Attacker-controlled transactionsâ€¦ exceed intended validation limits or modify feesâ€¦ outside design parameters."*

---

### Likelihood Explanation

Any unprivileged transaction sender on the Dijkstra era can construct such a batch. No special role, key, or governance access is required. The attacker only needs to:
1. Craft a top-level transaction with minimal or zero top-level redeemers.
2. Embed one or more sub-transactions with expensive Plutus scripts and large ExUnits budgets.
3. Set `txfee` to satisfy `minfee` computed from top-level ExUnits only.

The transaction passes all phase-1 checks and is accepted into a block.

---

### Recommendation

1. **Batch-aware `totExUnits`:** Define a Dijkstra-specific `totBatchExUnits` that sums ExUnits across the top-level transaction and all sub-transactions:

   ```haskell
   totBatchExUnits tx =
     totExUnits tx
       <> foldMap totExUnits (tx ^. bodyTxL . subTransactionsTxBodyL)
   ```

   Use this in both `validateExUnitsTooBigUTxO` and the minimum fee calculation for Dijkstra-era transactions.

2. **Batch-aware `minfee`:** Override `getMinFeeTx` for `DijkstraEra` to include `txscriptfee prices (totBatchExUnits tx)` instead of `txscriptfee prices (totExUnits tx)`.

3. **Block-level ExUnits accounting:** Verify that `validateExUnits` in the BBODY rule also aggregates sub-transaction ExUnits when computing the block total, to prevent the same bypass at the block level.

### Proof of Concept

```
1. Construct a Dijkstra top-level transaction T with:
   - One regular input (to satisfy InputSetEmptyUTxO)
   - txfee = minfee(pp, T, utxo)  -- computed with totExUnits(T) = 0
   - One sub-transaction S containing:
       * A Plutus script spending a UTxO
       * Redeemer with ExUnits = maxTxExUnits (or any large value)

2. Submit T. The ledger evaluates:
   - validateFeeTooSmallUTxO: minfee uses totExUnits(T) = 0, so txscriptfee = 0.
     T's fee covers only size-based cost. PASSES.
   - validateExUnitsTooBigUTxO: totalExUnits = totExUnits(T) = 0 â‰¤ maxTxExUnits. PASSES.
   - validateBatchCollateral: detects S has redeemers, checks collateral. PASSES (collateral provided).

3. T is accepted. S's Plutus script executes consuming up to maxTxExUnits resources
   without the sender paying the corresponding txscriptfee.
```
