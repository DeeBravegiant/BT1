### Title
Unvalidated `ProposalInit.builder` Used as `sequencer_address` (Fee Recipient) in Block Execution — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

### Summary

`ProposalInit.builder` is a proposer-supplied field that is passed verbatim as `sequencer_address` (the fee recipient) into `BlockInfo` for both the proposer and validator execution paths. The validator's `is_proposal_init_valid` never checks `builder` against any locally-trusted reference. A malicious proposer can set `builder` to an arbitrary address, redirecting all transaction fees in the block to that address and corrupting the block hash, state diff, and fee-token storage root — all of which validators will accept and commit.

---

### Finding Description

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address`:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs
Ok(starknet_api::block::BlockInfo {
    block_number: init.height,
    block_timestamp: BlockTimestamp(init.timestamp),
    sequencer_address: init.builder,   // ← proposer-supplied, never validated
    ...
})
``` [1](#0-0) 

This `BlockInfo` is forwarded to the batcher in both the build path (`ProposeBlockInput`) and the validate path (`ValidateBlockInput`): [2](#0-1) [3](#0-2) 

`sequencer_address` is the fee recipient for every transaction in the block (the OS charges fees by transferring to `block_context.block_info_for_execute.sequencer_address`): [4](#0-3) 

It is also hashed into `PartialBlockHashComponents` and therefore into the final block hash:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs
sequencer: SequencerContractAddress(block_info.sequencer_address),
``` [5](#0-4) [6](#0-5) 

The validator's `ProposalInitValidation` struct contains no `builder` field, and `is_proposal_init_valid` never checks it:

```rust
pub(crate) struct ProposalInitValidation {
    pub height: BlockNumber,
    pub block_timestamp_window_seconds: u64,
    pub previous_proposal_init: Option<PreviousProposalInitInfo>,
    pub l1_da_mode: L1DataAvailabilityMode,
    pub l2_gas_price_fri: GasPrice,
    pub starknet_version: StarknetVersion,
    pub fee_actual: Option<GasPrice>,
    // ← no builder / sequencer_address field
}
``` [7](#0-6) 

The full `is_proposal_init_valid` function validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, all four L1 gas prices, and `fee_proposal_fri` — but never `builder`: [8](#0-7) 

---

### Impact Explanation

**Impact: Critical — Incorrect fee accounting, wrong state diff, wrong block hash with economic impact.**

A malicious proposer sets `ProposalInit.builder` to an attacker-controlled L2 address. Every validator:

1. Passes `is_proposal_init_valid` (no check on `builder`).
2. Calls `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_address`.
3. Executes the block with `attacker_address` as fee recipient; all transaction fees are credited to the attacker.
4. Computes `PartialBlockHashComponents { sequencer: attacker_address, … }` and derives the same `partial_block_hash` as the proposer.
5. Votes for and commits the block.

The committed state diff contains wrong fee-token storage updates (attacker's balance inflated, legitimate sequencer's balance unchanged). The block hash encodes the attacker's address as `sequencer_address`. Both are permanent on-chain.

---

### Likelihood Explanation

**Likelihood: Medium.**

The attacker must be selected as the block proposer by the consensus protocol. In a rotating-proposer BFT scheme this occurs in normal operation for every validator. No special privilege beyond being a registered validator is required. The attack is silent — the block commitment matches between proposer and all validators because both sides derive `sequencer_address` from the same unvalidated `init.builder` field.

---

### Recommendation

Add `builder` (or equivalently `sequencer_address`) to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`. Each validator node should compare `init.builder` against its own locally-configured `builder_address` (the same value the proposer sets in `initiate_build`):

```rust
// In ProposalInitValidation:
pub builder_address: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(InvalidProposalInit(..., "builder mismatch"));
}
``` [9](#0-8) 

---

### Proof of Concept

1. Attacker runs a validator node and is selected as proposer for height `H`.
2. In `initiate_build`, instead of using `args.builder_address` (the node's own address), the attacker patches the code to set `builder = attacker_controlled_l2_address`.
3. The `ProposalInit` is broadcast with `builder = attacker_controlled_l2_address`.
4. Every honest validator calls `is_proposal_init_valid` — passes, because `builder` is not checked.
5. Every honest validator calls `convert_to_sn_api_block_info(init)` → `sequencer_address = attacker_controlled_l2_address`.
6. The batcher executes all transactions; fee transfers go to `attacker_controlled_l2_address`.
7. `PartialBlockHashComponents::new(block_info, …)` stores `sequencer = attacker_controlled_l2_address`.
8. `PartialBlockHash::from_partial_block_hash_components` produces the same hash on proposer and all validators.
9. `ProposalFin.proposal_commitment` matches; consensus reaches decision; block is committed.
10. The attacker has stolen all transaction fees for block `H`, and the block hash permanently encodes the wrong sequencer address. [10](#0-9) [11](#0-10) [12](#0-11)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
