### Title
Sub-Transaction Reference Script Fees Excluded from Minimum Fee Validation in Dijkstra Era - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces batch (nested) transactions where a top-level transaction embeds sub-transactions. The minimum fee validation for a Dijkstra batch uses `getConwayMinFeeTxUtxo`, which computes reference-script size only from the top-level transaction's inputs. A helper function `batchNonDistinctRefScriptsSize` that correctly aggregates reference-script sizes across all sub-transactions exists in the same module but is never wired into the fee check. An unprivileged transaction author can therefore include arbitrarily large reference scripts exclusively inside sub-transactions and pay a fee that is lower than the protocol-mandated minimum, modifying fees outside design parameters.

---

### Finding Description

**Vulnerability class:** Fee/accounting bug — a required fee component is not forwarded to the validation function (direct analog of the Wormhole "missing `{value:msg.value}`" pattern).

**Root cause — three code locations:**

1. `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`, line 141 — the Dijkstra `EraUTxO` instance delegates `getMinFeeTxUtxo` to the Conway implementation:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

2. `eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs`, lines 174–175 — `getConwayMinFeeTxUtxo` passes only the top-level transaction's reference-script size to `getMinFeeTx`:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

3. `eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs`, lines 183–187 — `txNonDistinctRefScriptsSize` only unions the top-level transaction's `referenceInputsTxBodyL` and `inputsTxBodyL`; it never touches sub-transactions:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

**The correct function already exists but is unused.** `batchNonDistinctRefScriptsSize` (lines 264–277 of the same Dijkstra UTxO file) sums reference-script sizes across the top-level transaction *and* all sub-transactions:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

It is exported from the module but never called from the fee-validation path.

**Fee validation entry point** — `dijkstraUtxoTransition` (line 373) calls `Shelley.validateFeeTooSmallUTxO pp tx originalUtxo`, which internally calls `getMinFeeTxUtxo` — the function that is incorrectly set to `getConwayMinFeeTxUtxo`: [5](#0-4) 

Sub-transactions have no independent fee field (`DijkstraSubTxBodyRaw` has no `fee` constructor argument), so the single top-level fee is the only fee paid for the entire batch. [6](#0-5) 

---

### Impact Explanation

The Conway reference-script fee was introduced precisely because "overhead of using reference scripts was not properly accounted for" in Babbage (ADR 009). The tiered pricing in `getConwayMinFeeTx` / `tierRefScriptFee` is the mechanism that enforces this cost. [7](#0-6) 

By placing all large reference scripts exclusively in sub-transactions, an attacker pays zero reference-script fee for those scripts. The top-level fee covers only the top-level transaction's byte size and Plutus execution units; the sub-transaction reference-script tier cost is silently omitted. This directly modifies the fee collected by the protocol below the design-mandated minimum — matching the **Medium** allowed impact: *"attacker-controlled transactions … modify fees … outside design parameters."*

---

### Likelihood Explanation

The attack requires no special privilege. Any transaction author can construct a `DijkstraTxBody` with an empty `dtbrReferenceInputs` / `dtbrSpendInputs` at the top level and populate `dtbrSubTransactions` with sub-transactions that reference large scripts. The value-conservation check (`validateValueNotConservedUTxO`) is satisfied as long as inputs cover outputs + fee + deposits; the fee itself is only checked against the (under-counted) minimum. The exploit is deterministic and reproducible on every node running the Dijkstra era rules.

---

### Recommendation

Override `getMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance to use `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
-- in eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
getDijkstraMinFeeTxUtxo :: (EraTx era, DijkstraEraTxBody era) => PParams era -> Tx TopTx era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx

instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo  -- was: getConwayMinFeeTxUtxo
```

This mirrors the pattern already used for Conway, where `getConwayMinFeeTxUtxo` replaced the Babbage version to add reference-script accounting.

---

### Proof of Concept

1. Deploy a UTxO output containing a large Plutus reference script (e.g., 50 KiB) on the Dijkstra testnet.
2. Construct a `DijkstraTxBody` where:
   - `dtbrSpendInputs` = one small UTxO (to satisfy `InputSetEmptyUTxO`)
   - `dtbrReferenceInputs` = empty
   - `dtbrSubTransactions` = one `DijkstraSubTxBody` whose `dstbrReferenceInputs` contains the 50 KiB script UTxO
   - `dtbrFee` = `getConwayMinFeeTxUtxo pp tx utxo` (the under-counted minimum)
3. Submit the transaction. The `validateFeeTooSmallUTxO` check passes because it calls `getConwayMinFeeTxUtxo`, which computes `txNonDistinctRefScriptsSize` over the top-level body only (0 bytes of reference scripts). The 50 KiB sub-transaction reference script contributes 0 to the fee.
4. Compare with the correct fee: `batchNonDistinctRefScriptsSize` would add `tierRefScriptFee multiplier stride costPerByte 51200` ≈ 768,000 lovelace (at default parameters) to the minimum fee. The attacker saves this entire amount per batch submission. [8](#0-7) [9](#0-8)
