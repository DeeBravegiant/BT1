### Title
Governance Threshold Upper Bound Permits 100% Quorum, Enabling Permanent Signing Freeze via TEE Expiry — (`crates/contract/src/primitives/thresholds.rs`)

---

### Summary

The `governance_threshold_upper_relative_bound` function allows the governance threshold to be set equal to the total participant count (`n`). The code itself acknowledges this is dangerous. When threshold = n, a single participant going offline or having an expired TEE attestation makes governance quorum unreachable. A subsequent call to `verify_tee` then permanently sets `accept_requests = false`, freezing all signing operations with no recovery path.

---

### Finding Description

`governance_threshold_upper_relative_bound` in `crates/contract/src/primitives/thresholds.rs` returns `n` (100% of participants):

```rust
/// Upper bound on the GovernanceThreshold for `n` participants:
/// Currently set to 100% of participants but would be a discussion subject
/// to drop this upper bound down not to have problems with smart contract
/// being locked if t = n and if an operator stops voting
pub(crate) fn governance_threshold_upper_relative_bound(n: u64) -> u64 {
    n
}
``` [1](#0-0) 

This means `validate_threshold` accepts threshold = n as a valid value: [2](#0-1) 

When threshold = n, the `verify_tee` function creates an irrecoverable freeze:

1. `verify_tee` is callable by any participant and checks whether kicking out expired-attestation participants would break the threshold relation.
2. It calls `validate_governance_against_reconstruction(remaining, current_threshold, max_reconstruction_threshold)`.
3. If `remaining < current_threshold` (i.e., one participant expired and threshold = n), `validate_threshold` returns `MaxRequirementFailed` because `k.value() > n_shares`.
4. `verify_tee` then sets `self.accept_requests = false` and returns `Ok(false)`. [3](#0-2) 

With `accept_requests = false`, all new `sign`, `request_app_private_key`, and `verify_foreign_transaction` calls are blocked. Recovery requires `vote_new_parameters` to lower the threshold, but `validate_incoming_proposal` enforces:

```rust
if n_old < self.threshold().value() {
    return Err(InvalidCandidateSet::InsufficientOldParticipants.into());
}
``` [4](#0-3) 

With threshold = n and one participant offline, no valid resharing proposal can be submitted — the network is permanently locked.

The code comment on `governance_threshold_upper_relative_bound` explicitly names this exact failure mode, making this a self-documented design gap rather than a speculative concern.

---

### Impact Explanation

**Critical — Permanent freezing of all signing operations controlled by the MPC network.**

Once `accept_requests = false` is set and the governance quorum is unreachable (threshold = n, one participant offline), every pending and future `sign` request times out. No foreign-chain transaction can be signed. The MPC network's core function — threshold signature issuance — is permanently unavailable with no on-chain recovery path.

---

### Likelihood Explanation

The governance threshold is set through `vote_new_parameters`, which requires threshold-of-n votes. A network that votes threshold = n (e.g., for maximum security) enters this fragile state. TEE attestations expire automatically on a fixed schedule; no adversarial action is needed to trigger expiry. Any participant (including a Byzantine one below the signing threshold) can then call `verify_tee` to trigger the freeze. The code comment confirms the developers are aware this is a reachable failure mode.

---

### Recommendation

Lower `governance_threshold_upper_relative_bound` to a value strictly less than `n`, for example 80% (`4 * n / 5`), so that even at the maximum allowed threshold, at least one participant can go offline without making governance quorum unreachable:

```rust
pub(crate) fn governance_threshold_upper_relative_bound(n: u64) -> u64 {
    // Cap at 80% to ensure governance remains reachable even if one participant
    // goes offline or has an expired TEE attestation.
    (4_u64.saturating_mul(n)).div_ceil(5).min(n)
}
```

This mirrors the recommendation in the referenced Olympus report: restrict the valid range of the threshold parameter to a tight window that avoids the dangerous boundary condition.

---

### Proof of Concept

**Setup**: 5 participants, governance threshold = 5 (100%), one CaitSith domain with reconstruction threshold = 5.

1. Participant 5's TEE attestation expires (automatic, time-based).
2. Any participant (including a Byzantine one) calls `verify_tee`.
3. `reverify_and_cleanup_participants` returns `TeeValidationResult::Partial { participants_with_valid_attestation: [1,2,3,4] }` (4 remaining).
4. `validate_governance_against_reconstruction(4, Threshold(5), Some(ReconstructionThreshold(5)))` is called.
5. `validate_threshold(4, Threshold(5))` fails: `5 > 4` → `MaxRequirementFailed`.
6. `verify_tee` logs the error and sets `self.accept_requests = false`.
7. All `sign` calls now fail. `vote_new_parameters` with any proposal including only 4 old participants fails `InsufficientOldParticipants` (needs 5 old participants, only 4 available).
8. The contract is permanently frozen with no on-chain recovery path. [5](#0-4) [2](#0-1)

    fn validate_threshold(n_shares: u64, k: Threshold) -> Result<(), Error> {
        if k.value() > n_shares {
            return Err(InvalidThreshold::MaxRequirementFailed {
                max: n_shares,
                found: k.value(),
            }
            .into());
        }
        if k.value() < MIN_THRESHOLD_ABSOLUTE {
            return Err(InvalidThreshold::MinAbsRequirementFailed.into());
        }
        let lower_relative_bound = governance_threshold_lower_relative_bound(n_shares);
        if k.value() < lower_relative_bound {
            return Err(InvalidThreshold::MinRelRequirementFailed {
                required: lower_relative_bound,
                found: k.value(),
            }
            .into());
        }
        let upper_relative_bound = governance_threshold_upper_relative_bound(n_shares);
        if k.value() > upper_relative_bound {
            return Err(InvalidThreshold::MaxRelRequirementFailed {
                max: upper_relative_bound,
                found: k.value(),
            }
            .into());
        }
        Ok(())
    }
