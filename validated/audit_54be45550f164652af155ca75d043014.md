### Title
Gateway L2 Gas Price Admission Check Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Incorrect Transaction Admission - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` compares a transaction's `max_price_per_unit` against the **current committed block's** `l2_gas_price`. However, the transaction will be executed in the **next** block, which uses `next_l2_gas_price` from the block header. This stale reference causes the gateway to accept transactions that will fail blockifier pre-validation during sequencing, and to reject transactions that would be valid for the next block. A TODO comment in the production code explicitly acknowledges the wrong field is being read.

---

### Finding Description

In `validate_resource_bounds`, the gateway fetches the previous block's `BlockInfo` and reads `strk_gas_prices.l2_gas_price` as the reference price:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs
async fn validate_resource_bounds(...) {
    if self.config.validate_resource_bounds {
        // TODO(Arni): getnext_l2_gas_price from the block header.
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info()
            .await?
            .gas_prices
            .strk_gas_prices
            .l2_gas_price;
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
}
``` [1](#0-0) 

The threshold is then computed as `min_gas_price_percentage * previous_block_l2_gas_price / 100` (default `min_gas_price_percentage = 100`, so threshold equals the previous block's price exactly): [2](#0-1) 

The block header, however, carries a distinct `next_l2_gas_price` field — the price that will govern the **next** block: [3](#0-2) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` reads from `block_header_without_hash` but maps only `l2_gas_price` into `BlockInfo`; `next_l2_gas_price` is silently discarded: [4](#0-3) 

The `next_l2_gas_price` is computed by the consensus orchestrator's EIP-1559-style adjustment (`calculate_next_l2_gas_price_for_fin`) and can diverge from the current block's price whenever gas usage is above or below the target: [5](#0-4) 

---

### Impact Explanation

**Scenario A — price rising (`next_l2_gas_price > l2_gas_price`):**

A user queries the current price, sets `max_price_per_unit = l2_gas_price`, and submits. The gateway threshold equals `l2_gas_price`, so the transaction passes admission and enters the mempool. When the batcher builds the next block at `next_l2_gas_price`, the blockifier's pre-validation fires:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    // MaxGasPriceTooLow
}
``` [6](#0-5) 

The transaction is rejected during sequencing after having been admitted by the gateway. The gateway has accepted an invalid transaction.

**Scenario B — price falling (`next_l2_gas_price < l2_gas_price`):**

A user sets `max_price_per_unit = next_l2_gas_price` (sufficient for the next block). The gateway threshold equals `l2_gas_price > next_l2_gas_price`, so the transaction is rejected with `GAS_PRICE_TOO_LOW` even though it would succeed in the next block. The gateway has rejected a valid transaction.

Both scenarios match the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The production default `min_gas_price_percentage = 100` is confirmed in the config schema: [7](#0-6) 

---

### Likelihood Explanation

The L2 gas price adjusts every block via the EIP-1559-style mechanism. Any block with gas usage above or below the target produces a `next_l2_gas_price` that differs from `l2_gas_price`. This is the normal operating condition of the network, not an edge case. Every transaction submitted at exactly the current price during a period of rising prices will be admitted by the gateway and then rejected by the batcher — mirroring the "concurrent buyers" scenario in the external report.

---

### Recommendation

Replace the read of `l2_gas_price` with `next_l2_gas_price` from the block header. This requires:

1. Extending `GatewayFixedBlockStateReader::get_block_info` (or adding a new method) to surface `next_l2_gas_price` from `BlockHeaderWithoutHash`.
2. In `validate_resource_bounds`, use `next_l2_gas_price` as the reference price passed to `validate_tx_l2_gas_price_within_threshold`.

The existing TODO comment already identifies the correct fix: `// TODO(Arni): getnext_l2_gas_price from the block header.` [8](#0-7) 

---

### Proof of Concept

1. Read the latest committed block's `l2_gas_price` (call it `P_cur`) and `next_l2_gas_price` (call it `P_next`) from the block header. Confirm `P_next > P_cur` (rising-price block).

2. Construct a V3 `AllResources` invoke transaction with `l2_gas.max_price_per_unit = P_cur`.

3. Submit to the gateway. With `min_gas_price_percentage = 100`, the threshold is `P_cur`, so `max_price_per_unit >= threshold` passes.

4. Observe the transaction enters the mempool.

5. When the batcher builds the next block at `P_next`, the blockifier pre-validation rejects the transaction with `MaxGasPriceTooLow { max_gas_price: P_cur, actual_gas_price: P_next }`.

The gateway has admitted a transaction that the sequencer cannot include — the direct sequencer analog of the "exact ETH value with no slippage" revert in the external report.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
