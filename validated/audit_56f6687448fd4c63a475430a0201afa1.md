### Title
Unvalidated `ProposalInit.builder` Address Accepted as `sequencer_address`, Enabling Fee Redirection to Arbitrary Address - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit.builder` is a proposer-supplied field that is mapped directly to `sequencer_address` in the execution `BlockInfo` without any validation in `is_proposal_init_valid`. A malicious proposer can set `builder` to an arbitrary address; every validator will execute the block with that address as the sequencer, crediting all transaction fees to the attacker. The `ProposalFin` commitment check does not catch this because both sides derive the commitment from the same proposer-supplied `builder`.

---

### Finding Description

`convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address`: [1](#0-0) 

This `BlockInfo` is passed to the batcher in both the build path (`propose_block`) and the validate path (`validate_block`): [2](#0-1) 

`is_proposal_init_valid` checks timestamp, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri`. It does **not** check `builder`: [3](#0-2) 

The proposer sets `builder` from a static local config field, with a TODO acknowledging it should eventually come from the committee: [4](#0-3) 

`ProposalInitValidation` — the struct that carries all locally-derived reference values used for validation — has no `builder` field at all: [5](#0-4) 

The `ProposalFin` commitment comparison (`built_block != received_fin.proposal_commitment`) does not catch a tampered `builder` because the validator executes the block using the proposer's `builder`, so both sides produce the same `PartialBlockHash` and the same `proposal_commitment_from(partial, fee_proposal_fri)`: [6](#0-5) 

---

### Impact Explanation

`sequencer_address` is the fee-recipient address in the Starknet execution model. All transaction fees charged during block execution are credited to `block_context.block_info.sequencer_address`. A malicious proposer who sets `builder = attacker_address` causes every validator to execute the block with `attacker_address` as the sequencer, so the entire block's fee revenue is credited to the attacker. The corrupted value is `BlockInfo.sequencer_address` for every transaction in the block, and the corrupted state is the fee-token balance of the attacker's address after block commitment. This matches the allowed impact: *"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."* [7](#0-6) 

---

### Likelihood Explanation

The attacker must be the legitimate proposer for the current consensus round (a validator in the committee). The `proposer` identity field is validated against the committee: [8](#0-7) 

However, `builder` is a separate field from `proposer` and is not subject to any committee check. In a network where validators are not all trusted (the normal Byzantine fault-tolerant assumption), any validator who becomes proposer can execute this attack for their round. The attack requires no external tooling beyond modifying the proposer's own `builder_address` config or patching the proposer binary.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and validate it in `is_proposal_init_valid`. The reference value should be derived from the committee (as the existing TODO acknowledges) or, as an interim measure, from the validator's own locally-configured `builder_address`. Concretely:

1. Add `expected_builder: ContractAddress` to `ProposalInitValidation`.
2. Populate it from `self.config.static_config.builder_address` (or from the committee once available) at the call sites in `validate_proposal` and `set_height_and_round`.
3. In `is_proposal_init_valid`, reject any `init_proposed.builder != proposal_init_validation.expected_builder`. [9](#0-8) 

---

### Proof of Concept

1. A validator is selected as proposer for round R at height H.
2. The validator modifies its `builder_address` config (or patches `initiate_build`) to set `builder = ATTACKER_ADDRESS`.
3. The proposer broadcasts `ProposalInit { builder: ATTACKER_ADDRESS, ... }` followed by transactions and `ProposalFin`.
4. Each validating node calls `convert_to_sn_api_block_info(&init)` → `sequencer_address = ATTACKER_ADDRESS` and passes this to `batcher.validate_block(...)`.
5. The batcher executes all transactions with `ATTACKER_ADDRESS` as the fee recipient; the resulting `PartialBlockHash` encodes `ATTACKER_ADDRESS` as the sequencer.
6. `proposal_commitment_from(partial, init.fee_proposal_fri)` on the validator side equals the proposer's `ProposalFin.proposal_commitment` → the `ProposalFinMismatch` check passes.
7. Consensus reaches decision; `finalize_decision` calls `convert_to_sn_api_block_info(init)` again and writes the block header with `sequencer_address = ATTACKER_ADDRESS` to state sync.
8. All fee-token balances credited to `ATTACKER_ADDRESS` are permanently committed. [10](#0-9) [11](#0-10)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
