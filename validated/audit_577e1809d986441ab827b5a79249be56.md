Audit Report

## Title
Pusher Delegation Signature Replay in `allowPushers` Enables Forced Re-Delegation After Revocation, Causing Oracle Stale-Price DoS on Pool Swaps — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

## Summary

`CompressedOracleV1.allowPushers` constructs its EIP-191 signed digest from `(chainid, address(this), deadline, pusher, msg.sender)` with no nonce and no used-signature tracking. A creator who retains a pusher's original calldata can replay the identical signature after the pusher calls `revokePusher()`, re-establishing `namespaceRemapping[pusher] = creator` within the same deadline window. This redirects all of the pusher's price updates away from their own namespace, starving any dependent `PriceProvider` of fresh prices and causing every `MetricOmmPool.swap()` call to revert with `BidIsZero`.

## Finding Description

`allowPushers` at lines 204–209 hashes only `(block.chainid, address(this), deadline, pusher, msg.sender)` — no nonce, no per-use invalidation:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
```

A grep for `pusherNonces`, `usedSignatures`, or `signatureUsed` in the file returns zero matches — there is no replay guard.

`revokePusher()` (lines 238–243) sets `namespaceRemapping[msg.sender] = address(0)`, but the original `(deadline, pusher, sig)` tuple remains valid until `block.timestamp > deadline`. The creator immediately resubmits the identical calldata to `allowPushers`, restoring `namespaceRemapping[pusher] = creator`. This cycle is repeatable indefinitely within the deadline window.

The `fallback` push path (lines 315–316) reads `namespaceRemapping[msg.sender]` on every call:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

While the delegation is active, every push the pusher makes lands in the creator's namespace, not the pusher's own namespace, so the pusher's namespace `refTime` goes stale.

`PriceProvider._getBidAndAskPrice()` (lines 197–199) returns `(0, type(uint128).max)` on staleness:

```solidity
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
    return (0, type(uint128).max);
}
```

`MetricOmmPool._getBidAndAskPriceX64()` (lines 806–808) then reverts:

```solidity
if (bid == 0) revert BidIsZero();
```

## Impact Explanation

Every `swap()` call on any pool whose `PriceProvider` points to the victimized pusher's namespace reverts with `BidIsZero` for the entire remaining deadline window. This is broken core pool functionality — the swap flow is completely unusable — which falls squarely within the allowed impact gate ("Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows").

## Likelihood Explanation

- `allowPushers` is permissionless; no privileged role is required to replay.
- The attacker only needs to retain the original transaction calldata (publicly available on-chain from the first submission).
- Any pusher who signed a delegation with a future deadline and later calls `revokePusher()` is at risk.
- Business relationships ending before the deadline expires are a realistic, non-contrived trigger.
- The replay can be executed in the same block as `revokePusher()`, giving the pusher no effective recourse.

## Recommendation

Add a per-pusher nonce included in the signed digest and incremented on each use, invalidating any prior signature:

```solidity
mapping(address => uint256) public pusherNonces;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonces[pusher]++
    ))
);
```

This ensures each signature is single-use and a revoked pusher can never be re-delegated without producing a fresh signature.

## Proof of Concept

1. Pusher A signs `keccak256(abi.encode(chainid, oracle, deadline=T, pusherA, creatorB))` and provides the signature to Creator B.
2. Creator B calls `allowPushers(T, [pusherA], [sig])` → `namespaceRemapping[pusherA] = creatorB`.
3. Pusher A calls `revokePusher()` → `namespaceRemapping[pusherA] = address(0)`.
4. Creator B immediately calls `allowPushers(T, [pusherA], [sig])` with the **identical** signature (before `block.timestamp > T`) → `namespaceRemapping[pusherA] = creatorB` again.
5. Steps 3–4 repeat indefinitely; Pusher A's namespace receives no price updates.
6. `PriceProvider._getBidAndAskPrice()` returns `(0, type(uint128).max)` due to staleness.
7. `MetricOmmPool.swap()` reverts with `BidIsZero` on every call until deadline T expires.
