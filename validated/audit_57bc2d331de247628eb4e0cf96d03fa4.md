Audit Report

## Title
ETH Sent Alongside Non-WETH Token Swaps Is Silently Accepted and Permanently Exposed to Theft — (`metric-periphery/contracts/MetricOmmSimpleRouter.sol`, `metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
All four swap entry points in `MetricOmmSimpleRouter` (`exactInputSingle`, `exactOutputSingle`, `exactInput`, `exactOutput`) are declared `external payable` with no guard rejecting `msg.value > 0` when `tokenIn != WETH`. The `pay()` helper in `PeripheryPayments` only consumes the router's native ETH balance in the `token == WETH` branch; for any other token it falls through to a plain `safeTransferFrom`, leaving the attached ETH stranded in the router. Any caller can then drain that ETH via `refundETH()`, which unconditionally transfers the full contract balance to `msg.sender`.

## Finding Description
All four swap entry points are `external payable` with no `msg.value` validation: [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

Inside `pay()`, the `token == WETH` branch is the only one that consumes native ETH. When `token != WETH`, the `else` branch executes `safeTransferFrom` and the router's ETH balance is untouched: [5](#0-4) 

The `receive()` guard only blocks bare ETH transfers (no calldata) from non-WETH senders. It does not fire when ETH is attached to a function call such as `exactInputSingle{value: 1 ether}(...)`: [6](#0-5) 

`refundETH()` sends the entire contract ETH balance to `msg.sender`, not to the original depositor, enabling any third party to front-run and drain the stranded ETH: [7](#0-6) 

## Impact Explanation
Direct loss of user principal (native ETH). A user who attaches ETH to a non-WETH swap loses that ETH with no revert, no warning, and no automatic refund. The ETH accumulates in the router and is claimable by any address that calls `refundETH()` before the victim does. This satisfies the "Critical/High/Medium direct loss of user principal" allowed impact gate.

## Likelihood Explanation
Medium. Users familiar with Uniswap-style routers commonly send native ETH for swaps involving the native token. A user who passes the wrong `tokenIn` address (e.g., the raw token address instead of WETH), or who uses a script that attaches ETH unconditionally, will silently lose funds. The `payable` modifier on all four entry points provides no friction against this mistake, and the exploit requires no special privileges — any unprivileged trader can trigger it and any unprivileged caller can drain the stranded balance.

## Recommendation
Add a guard in each swap entry point that reverts when ETH is sent but the input token is not WETH:

```solidity
if (msg.value > 0 && params.tokenIn != WETH) revert NativeTokenNotAccepted();
```

Alternatively, at the end of each swap function, automatically refund any unused ETH to `msg.sender`:

```solidity
if (address(this).balance > 0) _transferETH(msg.sender, address(this).balance);
```

## Proof of Concept
1. Pool has `token0 = WETH`, `token1 = USDC`.
2. Alice calls `exactInputSingle{value: 1 ether}(params)` where `params.tokenIn = address(USDC)`.
3. The swap executes normally — USDC is pulled from Alice via `safeTransferFrom`; the 1 ETH is accepted by the `payable` function and ignored by `pay()` (falls into the `else` branch at line 85–87 of `PeripheryPayments.sol`).
4. Alice's 1 ETH sits in the router (`address(this).balance == 1 ether`).
5. Bob (front-runner) calls `refundETH()` in the next block and receives Alice's 1 ETH via `_transferETH(msg.sender, balance)`.
6. Alice has lost 1 ETH with no revert or on-chain indication of the error.

    // If the payer is contract it means we are in the middle of a path. In the middle of a path we operate on ERC20 only.
    if (payer == address(this)) {
      IERC20(token).safeTransfer(recipient, value);
    } else if (token == WETH) {
      uint256 nativeBalance = address(this).balance;
      if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
      } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
      } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
      }
    } else {
      IERC20(token).safeTransferFrom(payer, recipient, value);
    }
  }
