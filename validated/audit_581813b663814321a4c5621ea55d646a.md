### Title
Internal VM Debug Messages Leaked Through Public RPC `query` Endpoint via `WasmUnknownError`/`LinkError` - (File: `runtime/runtime/src/function_call.rs`)

### Summary
An unprivileged user can trigger a `CallFunction` view query against a deployed contract and receive an RPC response that contains raw internal wasmtime debug strings — including trap type names, nondeterministic trap labels, and raw wasmtime instantiation error text — that are explicitly marked "for diagnostics only" in the source code but are propagated verbatim into the public JSON-RPC response.

### Finding Description

**Step 1 — Raw debug message captured from wasmtime**

In `runtime/near-vm-runner/src/wasmtime_runner/mod.rs`, when wasmtime produces a trap that cannot be classified deterministically, the raw debug representation of the trap is captured into a `VMRunnerError::WasmUnknownError`:

```rust
C::Unknown { trap } => {
    return Err(VMRunnerError::WasmUnknownError {
        debug_message: format!("unhandled trap type: {trap:?}"),
    });
}
// ...
return Err(VMRunnerError::WasmUnknownError {
    debug_message: format!("nondeterministic trap: {}", nondeterministic_message),
});
```

For non-trap instantiation failures that are not `UnknownImportError`, the raw `cause.to_string()` from wasmtime is placed into `FunctionCallError::LinkError { msg: description }`. [1](#0-0) 

**Step 2 — Debug message preserved verbatim into the function-call outcome**

In `runtime/runtime/src/function_call.rs`, `VMRunnerError::WasmUnknownError { debug_message }` is soft-failed into a `FunctionCallError::WasmUnknownError { msg: debug_message }` — the raw debug string is copied without sanitization:

```rust
Err(VMRunnerError::WasmUnknownError { debug_message }) => {
    tracing::error!(target: "runtime", "wasm unknown error: {}", debug_message);
    debug_assert!(false, "wasm unknown error: {}", debug_message);
    return Ok(VMOutcome::nop_outcome(FunctionCallError::WasmUnknownError {
        msg: debug_message,   // ← raw wasmtime debug string
    }));
}
``` [2](#0-1) 

The code comment explicitly states the message is "for diagnostics only." [3](#0-2) 

**Step 3 — Debug message becomes the `ExecutionError` string in the protocol error type**

In `runtime/runtime/src/conversions.rs`, `FunctionCallError::WasmUnknownError { msg }` is converted to `near_primitives::errors::FunctionCallError::ExecutionError(msg)` — the raw debug string becomes the `ExecutionError` payload:

```rust
From::WasmUnknownError { msg } => Self::ExecutionError(msg),
From::LinkError { msg } => Self::ExecutionError(format!("Link Error: {}", msg)),
From::LoadingError { msg } => Self::ExecutionError(format!("Loading Error: {}", msg)),
``` [4](#0-3) 

**Step 4 — The string is placed into `RpcQueryError::ContractExecutionError.vm_error`**

`chain/client/src/view_client_actor.rs` maps `near_chain_primitives::error::QueryError::ContractExecutionError { error_message, error, ... }` to `client_primitives::QueryError::ContractExecutionError { vm_error: error_message, error, ... }`. The `error_message` field is the `Display` of the `FunctionCallError`, which for `WasmUnknownError` is `"Unknown error during contract execution: {msg}"` — the raw debug string. [5](#0-4) 

**Step 5 — The string is returned verbatim in the public JSON-RPC response**

`process_query_response` in `chain/jsonrpc/src/lib.rs` serializes `ContractExecutionError` as a successful JSON-RPC result with the `vm_error` string placed directly into the `"error"` field:

```rust
RpcQueryError::ContractExecutionError {
    vm_error,
    error: _,
    block_height,
    block_hash,
} => Ok(json!({
    "error": vm_error,   // ← raw debug string returned to caller
    "logs": json!([]),
    "block_height": block_height,
    "block_hash": block_hash,
})),
``` [6](#0-5) 

Additionally, the structured `EXPERIMENTAL_call_function` endpoint serializes `RpcCallFunctionError::ContractExecutionError { vm_error: near_primitives::errors::FunctionCallError, ... }` — the full structured `FunctionCallError` (including the `ExecutionError(String)` payload) — into the JSON error body. [7](#0-6) 

### Impact Explanation

Any unprivileged caller who can submit a `query` RPC request with `CallFunction` receives internal wasmtime diagnostic strings in the JSON response body. Concretely:

- `"error": "Unknown error during contract execution: unhandled trap type: ..."` — exposes wasmtime's internal trap enum debug representation.
- `"error": "Unknown error during contract execution: nondeterministic trap: interrupt"` — exposes the nondeterministic trap label.
- `"error": "Link Error: <raw wasmtime instantiation error>"` — exposes raw wasmtime instantiation error text for non-`UnknownImportError` link failures.

This is an information disclosure of internal VM implementation details through the public RPC result, directly analogous to the external report's stack-trace/directory leak. The corrupted/affected value is the **RPC result** returned to the caller.

### Likelihood Explanation

Triggering `WasmUnknownError` requires a contract execution that produces a nondeterministic or unclassified wasmtime trap (e.g., OS interrupt, heap misalignment, or a future wasmtime trap variant). This is not trivially achievable with normal WASM opcodes but becomes reachable if a wasmtime version introduces a new trap kind or if a platform-specific signal fires during execution. The `LinkError` path with `cause.to_string()` is reachable with a crafted contract that produces a non-`UnknownImportError` instantiation failure. Likelihood is low-moderate.

### Recommendation

1. In `runtime/runtime/src/function_call.rs`, replace the raw `debug_message` in the `WasmUnknownError` outcome with a fixed opaque string (e.g., `"internal VM error"`) and log the `debug_message` only to the node's internal tracing/logging system.
2. In `runtime/near-vm-runner/src/wasmtime_runner/mod.rs`, sanitize the `LinkError` message for the non-`UnknownImportError` branch to a fixed string rather than `cause.to_string()`.
3. In `chain/jsonrpc/src/lib.rs` `process_query_response`, consider whether the full `vm_error` string should be returned verbatim or replaced with a structured, bounded error code.

### Proof of Concept

```
POST / HTTP/1.1
Content-Type: application/json

{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "query",
  "params": {
    "request_type": "call_function",
    "finality": "final",
    "account_id": "<account with contract triggering WasmUnknownError>",
    "method_name": "trigger_nondeterministic_trap",
    "args_base64": ""
  }
}

Response:
{
  "jsonrpc": "2.0",
  "id": "1",
  "result": {
    "error": "Unknown error during contract execution: nondeterministic trap: interrupt",
    "logs": [],
    "block_height": 12345,
    "block_hash": "..."
  }
}
```

The `"error"` field contains the raw internal wasmtime diagnostic string, exposing VM implementation internals to any unprivileged RPC caller.
