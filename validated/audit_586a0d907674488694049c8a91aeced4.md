Audit Report

## Title
Unrestricted `receive()` on `LRTDepositPool` Allows Donation to Trigger Illegitimate Fee Minting to Treasury — (`contracts/LRTDepositPool.sol`, `contracts/LRTOracle.sol`)

## Summary

`LRTDepositPool` accepts ETH from any sender via a bare, permissionless `receive()` fallback. Because `getETHDistributionData()` uses `address(this).balance` as `ethLyingInDepositPool`, any donated ETH is immediately counted in `totalETHInProtocol`. When `updateRSETHPrice()` is subsequently called, the oracle treats the TVL increase as staking yield, computes a `protocolFeeInETH` on it, and mints rsETH to the treasury — yield that should accrue entirely to rsETH holders is partially redirected to the treasury via illegitimate fee minting.

## Finding Description

**Root cause — unrestricted ETH entry point:**

`LRTDepositPool` exposes a bare fallback with no access control: [1](#0-0) 

Any address can send ETH directly to the deposit pool. The named receive helpers (`receiveFromRewardReceiver`, `receiveFromNodeDelegator`, `receiveFromLRTConverter`) are also unrestricted, but the bare `receive()` is the simplest entry point.

**Raw balance included in TVL:**

`getETHDistributionData()` sets `ethLyingInDepositPool` directly from the contract's native balance: [2](#0-1) 

`getTotalAssetDeposits(ETH_TOKEN)` sums all distribution components including this value: [3](#0-2) 

**Oracle counts donation as yield:**

`_getTotalEthInProtocol()` calls `getTotalAssetDeposits` for every supported asset, including ETH: [4](#0-3) 

In `_updateRsETHPrice()`, `previousTVL` is computed as `rsethSupply × rsETHPrice` (the last recorded price). Any ETH donated since the last price update inflates `totalETHInProtocol` above `previousTVL`, causing the fee logic to treat the donation as staking yield: [5](#0-4) 

**Fee rsETH minted to treasury:**

The computed fee is minted as rsETH to the treasury address: [6](#0-5) 

**Existing guards are insufficient:**

- `pricePercentageLimit` only reverts if the price increase exceeds the configured threshold; small donations (or a zero limit) pass through freely.
- `maxFeeMintAmountPerDay` caps per-day damage but does not prevent the attack.
- `updateRSETHPrice()` is fully public with only a `whenNotPaused` modifier: [7](#0-6) 

## Impact Explanation

**High — Theft of unclaimed yield.**

For a donation of `D` ETH with protocol fee `F` BPS:
- rsETH holders receive `D × (1 − F/10000)` in value accrual instead of the full `D`.
- The treasury receives `D × F/10000` in newly minted rsETH that it did not earn through legitimate staking activity.

The fee mechanism is designed to take a cut of *staking rewards*, not arbitrary ETH transfers. The treasury receives rsETH backed by the donated ETH, diluting all existing rsETH holders by the fee portion of the donation. This is a concrete, on-chain transfer of value from rsETH holders to the treasury triggered by an unprivileged external action.

## Likelihood Explanation

The attack path is fully permissionless: `receive()` has no role check, no minimum, and no guard. `updateRSETHPrice()` is public. The only cost to the attacker is the donated ETH itself, so direct financial motivation is absent unless the attacker controls the treasury or holds a position that benefits from diluting rsETH holders. However, the path is unconditionally reachable by any external account on unmodified code, and the impact is repeatable up to the `maxFeeMintAmountPerDay` cap per day.

## Recommendation

1. **Restrict `receive()`** to only accept ETH from known, trusted senders (e.g., `NodeDelegator`, `RewardReceiver`, `LRTConverter`). Revert on arbitrary ETH transfers.
2. Alternatively, **replace `address(this).balance` in `getETHDistributionData()`** with an explicit storage variable that is only incremented through controlled entry points (`depositETH`, `receiveFromRewardReceiver`, etc.), so donated ETH is never counted in TVL.

## Proof of Concept

```solidity
// Foundry fork test (pseudocode)
function testDonationInflatesFee() public {
    // Precondition: rsETH supply > 0, protocolFeeInBPS > 0, protocol not paused
    uint256 treasuryBefore = rsETH.balanceOf(treasury);

    // Step 1: attacker donates 1 ETH directly to LRTDepositPool
    vm.deal(attacker, 1 ether);
    vm.prank(attacker);
    (bool ok,) = address(lrtDepositPool).call{value: 1 ether}("");
    require(ok);

    // Step 2: anyone calls updateRSETHPrice (public, no role required)
    lrtOracle.updateRSETHPrice();

    // Step 3: treasury received fee rsETH minted on the donated ETH
    assertGt(rsETH.balanceOf(treasury), treasuryBefore);
}
```

The assertion confirms that fee rsETH was minted on donated ETH, diluting all existing rsETH holders by the fee portion of the donation. The test is reproducible on a local fork with no privileged access required.
