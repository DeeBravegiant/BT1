Audit Report

## Title
Unbounded Gas Consumption in `getAssetUnstaking()` via Uncapped `getQueuedWithdrawals` Iteration Blocks `updateRSETHPrice()` — (`contracts/NodeDelegator.sol`)

## Summary
`updateRSETHPrice()` is publicly callable and routes through `_getTotalEthInProtocol()` → `getTotalAssetDeposits(ETH_TOKEN)` → `getETHDistributionData()`, which calls `getAssetUnstaking(ETH_TOKEN)` on every registered NDC. Each call fetches and iterates the entire `Withdrawal[]` array from EigenLayer's `DelegationManager` with no bound. As queued beacon-chain validator withdrawals accumulate across NDCs through normal protocol operation, the transaction's gas cost grows without ceiling until it permanently exceeds the block gas limit, making `updateRSETHPrice()` and `updateRSETHPriceAsManager()` uncallable.

## Finding Description
The full call chain is confirmed in the repository:

1. `updateRSETHPrice()` at `LRTOracle.sol:87` is `public whenNotPaused` — no access control beyond pause state.
2. It calls `_updateRsETHPrice()`, which calls `_getTotalEthInProtocol()` at `LRTOracle.sol:231`.
3. `_getTotalEthInProtocol()` at `LRTOracle.sol:336–348` iterates all supported assets and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each, including `ETH_TOKEN`.
4. For `ETH_TOKEN`, this reaches `getETHDistributionData()` in `LRTDepositPool.sol:484–493`, which loops over every NDC in `nodeDelegatorQueue` and calls `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(LRTConstants.ETH_TOKEN)`.
5. `getAssetUnstaking()` at `NodeDelegator.sol:405–427` calls `_getDelegationManager().getQueuedWithdrawals(address(this))`, ABI-decodes the **entire** returned `Withdrawal[]` array into memory, then iterates every `Withdrawal` struct and every `strategies[]` entry within it — with no cap.

EigenLayer imposes no per-staker limit on the number of queued withdrawals. Each beacon-chain validator exit is a separate `queueWithdrawals` call, so a protocol operating many validators per NDC will organically accumulate a large queue. Gas complexity is `O(NDCs × queued_withdrawals_per_NDC × strategies_per_withdrawal)`. The ABI-decode of large `Withdrawal[]` arrays from an external call is itself expensive. There is no administrative escape hatch: `updateRSETHPriceAsManager()` at `LRTOracle.sol:94–96` calls the identical `_updateRsETHPrice()` internal function and is equally affected.

## Impact Explanation
Once the cumulative gas cost of `updateRSETHPrice()` exceeds the 30M block gas limit, the rsETH price (`rsETHPrice`) can no longer be updated. Protocol fee accrual — rsETH minted to the treasury inside `_updateRsETHPrice()` — stops permanently. This matches two allowed impact classes: **Medium — Unbounded gas consumption** (primary) and **Medium — Permanent freezing of unclaimed yield** (secondary, as protocol fees can never be minted again once the threshold is crossed).

## Likelihood Explanation
No attacker action is required. The condition arises from normal protocol operation: each validator exit queues a withdrawal in EigenLayer. With `maxNodeDelegatorLimit = 10` NDCs and a realistic 80+ queued validator exits per NDC, the 800+ Solidity loop iterations plus repeated large ABI-decode operations are sufficient to breach the block gas limit. Any public caller — including automated price-update keepers — will encounter the OOG revert once the threshold is crossed, and the state cannot be unwound without a contract upgrade.

## Recommendation
1. **Replace on-chain iteration with an accounting variable.** Maintain a storage variable (e.g., `pendingUnstakingETH`) that is incremented when a withdrawal is queued and decremented when it is completed, eliminating the need to iterate EigenLayer's queue at price-update time.
2. **If iteration must be retained**, add a hard cap on the number of withdrawals processed per NDC per call and document the operational limit, so the function degrades gracefully rather than OOG-reverting.
3. **Decouple price update from full TVL recomputation** by caching intermediate TVL components and updating them incrementally on state-changing operations (queue, complete) rather than recomputing the entire state in a single transaction.

## Proof of Concept
Deploy a mock `DelegationManager` that returns 80 `Withdrawal` structs (each with one `beaconChainETHStrategy` entry and `32 ether` shares) for any `getQueuedWithdrawals(address)` call. Wire 10 NDCs to this mock. Call `updateRSETHPrice()` and measure gas consumed. The gas will exceed 30,000,000 (Ethereum block gas limit), causing the call to revert with out-of-gas. The same result is observed for `updateRSETHPriceAsManager()`. A Foundry fork test can reproduce this by:
```solidity
function test_updateRSETHPrice_OOG() public {
    // Deploy MockDelegationManager returning 80 withdrawals per NDC
    // Register 10 NDCs each backed by MockDelegationManager
    uint256 gasBefore = gasleft();
    vm.expectRevert(); // OOG
    lrtOracle.updateRSETHPrice();
    // Or: assert(gasBefore - gasleft() > 30_000_000);
}
```
