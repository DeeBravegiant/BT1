### Title
Unauthenticated P2P Peer Can Forge `fee_proposal_fri` in `SignedBlockHeader`, Corrupting `fee_proposals_window` and Causing Validator to Reject All Honest Proposals — (`crates/apollo_p2p_sync/src/client/header.rs`)

### Summary

The `fee_proposal_fri` field (proto field 22) in `SignedBlockHeader` is explicitly excluded from the block hash and from the block signature. The p2p sync client's `parse_data_for_block` performs no cryptographic verification and no range check on this field. Any unauthenticated peer can send a header with an arbitrary `fee_proposal_fri` value; it is stored verbatim into `StorageBlockHeader`. When `initialize_fee_proposals_window` bootstraps the consensus context from local storage, it reads these forged values into `fee_proposals_window`. `compute_fee_actual` then returns an extreme median, and `is_proposal_init_valid`'s `fee_proposal_bounds` check rejects every honest proposer's `fee_proposal_fri`.

### Finding Description

**Step 1 — Unauthenticated ingestion with no field validation.**

`parse_data_for_block` in the p2p sync client checks only two things: block number ordering and `signatures.len() == 1`. It performs no cryptographic verification of the block signature and no validation of `fee_proposal_fri`. [1](#0-0) 

**Step 2 — Forged value stored verbatim.**

`write_to_storage` calls `append_header`, which copies `fee_proposal_fri` directly from the wire message into `StorageBlockHeader` with no bounds check. [2](#0-1) [3](#0-2) 

**Step 3 — Block signature does not cover `fee_proposal_fri`.**

`verify_block_signature` signs `Poseidon(block_hash, state_diff_commitment)`. The proto schema and the generated Rust struct both carry the explicit warning: *"WARNING: this field is currently not part of the block hash, so the value must be trusted."* Even if the p2p sync verified the signature, a forged `fee_proposal_fri` would pass undetected. [4](#0-3) [5](#0-4) 

**Step 4 — `initialize_fee_proposals_window` reads the forged value.**

Before joining consensus, the node calls `initialize_fee_proposals_window`, which reads `fee_proposal_fri` from `state_sync_client.get_block()` (backed by the same local storage populated by p2p sync) and inserts it into `fee_proposals_window` without any sanity check. [6](#0-5) 

**Step 5 — `compute_fee_actual` returns the forged extreme.**

If the attacker fills the entire window (size 10 by default) with `u128::MAX`, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))` as the median. [7](#0-6) 

**Step 6 — Validator rejects all honest proposals.**

`is_proposal_init_valid` computes `fee_proposal_bounds(u128::MAX, margin_ppt=2)`:
- `upper = u128::MAX` (saturated)
- `lower = u128::MAX × 1000 / 1002 ≈ u128::MAX − 3.4×10³³`

Any honest proposer's `fee_proposal_fri` (e.g., 10 gwei = 10¹⁰) is astronomically below `lower`. The validator returns `InvalidProposalInit("Fee proposal out of bounds")` for every proposal, stalling consensus. [8](#0-7) [9](#0-8) 

### Impact Explanation

A corrupted validator permanently rejects every honest `ProposalInit` at the current height. If the attacker targets enough validators (≥ 1/3 of the committee), consensus halts. Additionally, if the proposer's own window is corrupted, `calculate_next_l2_gas_price` derives an extreme `next_l2_gas_price`, which propagates into block fees for all subsequent transactions — a direct economic impact.

### Likelihood Explanation

The attack requires only that the victim node syncs at least `window_size` (default: 10) consecutive block headers from the malicious peer before calling `initialize_fee_proposals_window`. In a permissionless p2p network this is straightforward: the attacker runs a node, the victim connects to it during catch-up, and the attacker serves correctly-numbered headers with forged `fee_proposal_fri`. No cryptographic material is needed because the field is outside the block hash.

### Recommendation

1. **Clamp on read**: In `initialize_fee_proposals_window`, reject or clamp any `fee_proposal_fri` value that falls outside a plausible range (e.g., `[min_gas_price, some_sane_upper_bound]`) before inserting into `fee_proposals_window`.
2. **Median robustness**: The existing median is already resistant to a single outlier, but not to a full-window attack. Consider requiring a supermajority of `Some` values within a sane range before computing `fee_actual`.
3. **Long-term**: Include `fee_proposal_fri` in the block hash (or a separate authenticated commitment) so that p2p peers cannot forge it without invalidating the block signature.

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub(crate) fn fee_proposal_bounds(fee_actual: GasPrice, margin_ppt: u128) -> (u128, u128) {
    let denom = U256::from(PPT_DENOMINATOR);
    let scaled = denom + U256::from(margin_ppt);
    let fee_actual_u256 = U256::from(fee_actual.0);
    let upper = u128::try_from(fee_actual_u256 * scaled / denom).unwrap_or(u128::MAX);
    let lower = u128::try_from(fee_actual_u256 * denom / scaled).unwrap_or(0);
    (lower, upper)
}
