Audit Report

## Title
Permanent Freezing of Unclaimed Yield When `totalKernelStaked` Drops to Zero Mid-Period - (File: `contracts/KERNEL/KernelDepositPool.sol`)

## Summary
`KernelDepositPool` implements a Synthetix-style staking rewards model where `rewardPerToken()` freezes accumulation whenever `totalKernelStaked == 0`. If the last staker calls `initiateWithdrawal()` during an active reward period, all remaining reward tokens allocated for that period become permanently unclaimable. The contract has no on-chain recovery mechanism for `rewardsToken` and explicitly acknowledges this risk in its NatSpec while relying solely on operational controls.

## Finding Description
`rewardPerToken()` (lines 408–414) returns `rewardPerTokenStored` unchanged when `totalKernelStaked == 0`, halting all reward accumulation. The `updateReward` modifier (lines 232–242) uses this value to checkpoint rewards, so any elapsed time with zero stakers produces no reward accumulation — those tokens are silently stranded.

`notifyRewardAmount()` (line 570) guards against starting a period with zero stakers via `if (totalKernelStaked == 0) revert NoStakedTokens()`, but this check applies only at call time. After the period starts, `initiateWithdrawal()` (lines 325–326) freely decrements `balanceOf[msg.sender]` and `totalKernelStaked` with no check on whether a reward period is active (`block.timestamp < finishAt`). If the last staker withdraws, `totalKernelStaked` hits zero and `rewardRate * remainingSeconds` worth of reward tokens are permanently locked.

`KernelDepositPool` does not inherit `Recoverable` (line 24) and contains no admin sweep function for `rewardsToken`. The contract's own NatSpec (lines 18–22) acknowledges this exact scenario and states the mitigation is purely operational: "ensuring there are always some tokens staked." There is no on-chain enforcement of this invariant.

## Impact Explanation
Reward tokens transferred into the contract via `notifyRewardAmount` become permanently unclaimable whenever `totalKernelStaked` drops to zero mid-period. No function exists to recover or redistribute them. This is **permanent freezing of unclaimed yield** — a Medium-severity impact in the allowed scope.

## Likelihood Explanation
Any staker holding the last staked tokens can trigger this by calling `initiateWithdrawal()`, a standard unprivileged user action. No special role or privilege is required. Stakers may withdraw for legitimate reasons (market conditions, better opportunities) without awareness of the protocol-level consequence. The scenario is realistic and repeatable across every reward period.

## Recommendation
Add an on-chain guard in `initiateWithdrawal()` that prevents `totalKernelStaked` from reaching zero while a reward period is active:

```solidity
if (totalKernelStaked - _amount == 0 && block.timestamp < finishAt) revert ActiveRewardPeriod();
```

Alternatively, add an admin-callable recovery function (mirroring the `Recoverable.recoverTokens` pattern in `contracts/utils/Recoverable.sol` lines 41–57) that can sweep stranded `rewardsToken` balance after a reward period ends with zero stakers, bounded to only the unallocated surplus.

## Proof of Concept
1. Admin calls `notifyRewardAmount(1_000e18)` while Alice has 100 KERNEL staked. `rewardRate` is set, `finishAt = block.timestamp + duration`. Reward tokens are transferred into the contract.
2. Alice calls `initiateWithdrawal(100)`. `updateReward(Alice)` checkpoints her earned rewards up to this point. `totalKernelStaked` becomes 0.
3. Time advances. `rewardPerToken()` returns `rewardPerTokenStored` unchanged for the entire remaining period because `totalKernelStaked == 0`.
4. `rewardRate * (finishAt - step2_timestamp)` worth of reward tokens are never distributed.
5. No function in `KernelDepositPool` can recover these tokens. They remain locked permanently.

Foundry test sketch:
```solidity
function test_rewardsLockedWhenTotalStakedZero() public {
    vm.prank(alice);
    pool.stake(100e18);
    vm.prank(admin);
    pool.notifyRewardAmount(1_000e18);
    vm.prank(alice);
    pool.initiateWithdrawal(100e18); // totalKernelStaked = 0
    vm.warp(block.timestamp + duration);
    // rewardsToken balance of pool == 1_000e18 minus alice's earned share
    // remaining tokens are permanently locked, no recovery path exists
    assertGt(rewardsToken.balanceOf(address(pool)), 0);
}
```
