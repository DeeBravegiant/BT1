Audit Report

## Title
`AnchoredPriceProvider._isStale` Unconditionally Rejects Future `refTime`, Blocking All Swaps on L2 — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary

`AnchoredPriceProvider`, explicitly designated "the one standard provider for public pools," contains an L1-only `_isStale` that returns `true` for any oracle `refTime > block.timestamp`. On L2 chains (Base, HyperEVM), sequencer clock skew routinely causes oracle timestamps to lead `block.timestamp` by a few seconds. When this occurs, every swap through any pool using `AnchoredPriceProvider` reverts with `PriceProviderFailed`. The protocol already acknowledged this class of bug by shipping `FUTURE_TOLERANCE` in `PriceProviderL2` and `ProtectedPriceProviderL2`, but no equivalent L2 variant exists for `AnchoredPriceProvider`.

## Finding Description

`AnchoredPriceProvider._isStale` (lines 222–230) unconditionally treats any `refTime > nowTs` as stale:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← no tolerance for clock skew
    return (nowTs - refTime) > maxDelta;
}
```

`_readLeg` (line 283) calls this with `block.timestamp`:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS))
    return (mid, spreadBps, refTime, false);
```

When `ok = false`, `_getBidAndAskPrice` (line 260) returns `(0, type(uint128).max)`, causing `getBidAndAskPrice` (line 216) to revert `FeedStalled()`. `MetricOmmPool._getBidAndAskPriceX64` (lines 806–812) catches this and reverts `PriceProviderFailed(reason)`, blocking every `swap` call.

By contrast, `PriceProviderL2._isStale` (lines 135–150) tolerates clock skew via `FUTURE_TOLERANCE`:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;   // ← tolerate within futureTol
}
```

`PriceProviderFactoryL2` passes `_futureTolerance` at construction. `AnchoredProviderFactory` deploys only `AnchoredPriceProvider` with no `futureTolerance` parameter and no L2 variant. No `AnchoredPriceProviderL2.sol` exists in the repository.

## Impact Explanation

All swaps through any public pool whose price provider is an `AnchoredPriceProvider` instance become non-functional on L2 whenever the oracle's `refTime` exceeds `block.timestamp`. Since `AnchoredPriceProvider` is the designated standard provider for public pools, this renders the primary trading path for the entire protocol unusable on L2 during affected windows. This matches the "broken core pool functionality causing unusable swap flows" impact gate.

## Likelihood Explanation

On Base and HyperEVM, sequencer block timestamps and Pyth/Chainlink publication timestamps are not perfectly synchronized. Oracle timestamps leading `block.timestamp` by 1–5 seconds is a routine occurrence, not an edge case. The protocol's own decision to build `FUTURE_TOLERANCE` into `PriceProviderL2` and `ProtectedPriceProviderL2` is direct acknowledgment that this condition occurs in production. Any unprivileged trader calling `swap` on an affected pool triggers the revert with no special setup required.

## Recommendation

Add an `AnchoredPriceProviderL2` variant (or add an immutable `FUTURE_TOLERANCE` to `AnchoredPriceProvider`) and update `_isStale` to mirror the L2-aware logic already present in `PriceProviderL2`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Pass `FUTURE_TOLERANCE` (bounded to `<= 1 hours` as in `PriceProviderL2`) into `_readLeg` and `_isStale` calls. Update `AnchoredProviderFactory` (or add an `AnchoredProviderFactoryL2`) to accept and forward `futureTolerance` at deployment.

## Proof of Concept

1. Deploy `AnchoredPriceProvider` on Base pointing to a Pyth feed via `AnchoredProviderFactory.createAnchoredProvider`.
2. Create a public pool using this provider as its price provider.
3. Pyth publishes a price update with `publishTime = block.timestamp + 2` (sequencer clock is 2 s behind oracle).
4. Any trader calls `pool.swap(...)`.
5. `_getBidAndAskPriceX64` → `getBidAndAskPrice` → `_getBidAndAskPrice` → `_readLeg(baseFeedId)`.
6. `_isStale(block.timestamp + 2, block.timestamp, MAX_REF_STALENESS)` → `refTime > nowTs` → returns `true`.
7. `_readLeg` returns `ok = false` → `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
8. `getBidAndAskPrice` reverts `FeedStalled()`.
9. `_getBidAndAskPriceX64` catches and reverts `PriceProviderFailed(...)`.
10. All swaps revert until the sequencer's clock catches up.

Foundry fork test: fork Base mainnet, mock a Pyth oracle returning `publishTime = block.timestamp + 2`, deploy `AnchoredPriceProvider`, call `getBidAndAskPrice`, assert revert `FeedStalled()`.
