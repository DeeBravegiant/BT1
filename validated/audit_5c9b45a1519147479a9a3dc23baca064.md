Audit Report

## Title
`ScrollMessenger.sendETHToL1ViaBridge` Hardcodes `gasLimit=0`, Causing L2→L1 Relay to Fail When Target Is a Contract - (`contracts/bridges/ScrollMessenger.sol`)

## Summary
`ScrollMessenger.sendETHToL1ViaBridge` passes a hardcoded `gasLimit=0` to `IScrollMessenger.sendMessage`, with a developer comment claiming this uses a "default gas limit." Scroll's protocol treats `gasLimit` as the literal gas forwarded to the target on L1 relay. Since `l1VaultETHForL2Chain` is the `L1Vault` contract (not an EOA), every relay attempt will fail out-of-gas on L1. ETH is not permanently lost (Scroll's `replayMessage` allows recovery), but the protocol systematically fails to credit ETH to `L1Vault`.

## Finding Description
The call chain is:

1. `RSETHPool.bridgeAssetsViaNativeBridge()` calls `IL2Messenger(messenger).sendETHToL1ViaBridge{value: ethBalanceMinusFees}(l2Bridge, l1VaultETHForL2Chain, ethBalanceMinusFees)`. [1](#0-0) 

2. `ScrollMessenger.sendETHToL1ViaBridge` forwards to `IScrollMessenger(l2bridge).sendMessage{value: value}(target, value, "", 0, msg.sender)` with `gasLimit=0`. The developer comment is factually wrong — `gasLimit=0` is not a "use default" flag in Scroll's protocol. [2](#0-1) 

3. `IScrollMessenger` explicitly documents `gasLimit` as "Gas limit required to complete the message relay on corresponding chain" — a literal gas cap, not a sentinel value. [3](#0-2) 

4. `L1Vault` is a full upgradeable Solidity contract (`Initializable`, `ReentrancyGuardUpgradeable`, `AccessControlUpgradeable`) with a `receive()` function and non-trivial storage. It requires well above 0 gas to accept ETH. [4](#0-3) [5](#0-4) 

In Scroll's L1ScrollMessenger, the relay executes `target.call{gas: gasLimit, value: value}(message)`. With `gasLimit=0`, the call to `L1Vault` immediately reverts out-of-gas. The L2 transaction succeeds (ETH leaves the pool, `BridgedETHToL1ViaNativeBridge` is emitted), but the L1 relay fails and ETH is held in the Scroll L1 bridge escrow.

Compare with `OptimismMessenger`, which correctly sets `DEFAULT_GAS_LIMIT = 200_000`: [6](#0-5) 

`ScrollMessenger` has no equivalent minimum gas constant, and the comment is factually incorrect about Scroll's behavior.

## Impact Explanation
Every invocation of `bridgeAssetsViaNativeBridge()` when the Scroll messenger is configured will succeed on L2 but fail on L1 relay. ETH is not credited to `l1VaultETHForL2Chain`, so `L1Vault` cannot call `depositETHForL1VaultETH()` → `lrtDepositPool.depositETH()` → mint rsETH → bridge back to L2 users. The protocol fails to deliver its promised return (ETH deposited into `L1Vault` → rsETH minted → sent back to L2 users). ETH is not permanently lost because Scroll's `replayMessage` mechanism allows replaying the failed message with a corrected gas limit, but this requires manual intervention outside the protocol's normal flow.

**Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
This is a systematic failure, not a probabilistic one. Every single call to `bridgeAssetsViaNativeBridge()` with the Scroll messenger configured will fail on L1 relay. The bug is in the messenger contract itself — the `BRIDGER_ROLE` operator is performing their normal duties correctly, but the hardcoded `gasLimit=0` makes every relay attempt fail. No attacker action is required; the failure is inherent to the code.

## Recommendation
Set a non-zero minimum gas limit in `ScrollMessenger`, analogous to `OptimismMessenger.DEFAULT_GAS_LIMIT`:

```solidity
uint256 public constant DEFAULT_GAS_LIMIT = 50_000;

function sendETHToL1ViaBridge(address l2bridge, address target, uint256 value) external payable nonReentrant {
    if (msg.value != value) revert MismatchedMsgValue();
    IScrollMessenger(l2bridge).sendMessage{ value: value }(target, value, "", DEFAULT_GAS_LIMIT, msg.sender);
}
```

A value of at least `50_000` is appropriate for a plain ETH receive to a contract with `ReentrancyGuardUpgradeable` and `AccessControlUpgradeable` storage reads. Ideally make it configurable via an admin setter.
