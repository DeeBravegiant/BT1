### Title
Attached NEAR deposit silently frozen when existing participant calls `submit_participant_info` - (File: crates/contract/src/lib.rs)

---

### Summary

`submit_participant_info` is `#[payable]` and contains a conditional refund path. When an **existing participant** updates their TEE attestation (not a new insertion), the branch that refunds excess deposit is never entered, so any NEAR tokens attached to the call are permanently frozen in the contract with no recovery path.

---

### Finding Description

`submit_participant_info` (line 758) is marked `#[payable]`, meaning the NEAR runtime accepts any attached deposit without automatically reverting. The function computes a boolean guard:

```rust
let caller_is_not_participant = self.voter_account().is_err();
let is_new_attestation = matches!(
    attestation_insertion_result,
    ParticipantInsertion::NewlyInsertedParticipant
);
let attestation_storage_must_be_paid_by_caller =
    is_new_attestation || caller_is_not_participant;
``` [1](#0-0) 

The refund logic is gated entirely on this flag:

```rust
if attestation_storage_must_be_paid_by_caller {
    // ... check attached >= cost, then refund diff ...
    if let Some(diff) = attached.checked_sub(cost)
        && diff > NearToken::from_yoctonear(0)
    {
        Promise::new(account_id).transfer(diff).detach();
    }
}
``` [2](#0-1) 

When an **existing participant** re-submits their attestation (e.g., after a TEE software upgrade), both conditions are false:

- `is_new_attestation` = `false` (the participant already exists in `tee_state`)
- `caller_is_not_participant` = `false` (they are a current participant)

Therefore `attestation_storage_must_be_paid_by_caller` = `false`, the `if` block is skipped entirely, and `env::attached_deposit()` is never read or refunded. Any NEAR tokens attached to the call are silently absorbed by the contract.

The contract has no general withdrawal function; the only recovery path would be a full contract upgrade voted through `propose_update` / `vote_update`, which requires threshold consensus. [3](#0-2) 

---

### Impact Explanation

Attached NEAR tokens are permanently frozen in the MPC contract balance. There is no per-caller accounting and no withdrawal endpoint, so the funds cannot be recovered without a threshold-voted contract upgrade. This breaks the production accounting invariant that `#[payable]` functions must either consume or refund every yoctoNEAR received.

This maps to the allowed **Medium** impact: *"Balance … manipulation that breaks production safety/accounting invariants."*

---

### Likelihood Explanation

Participants call `submit_participant_info` whenever they rotate their TEE attestation (e.g., after a node image upgrade). The function is `#[payable]`, so wallets and scripts that habitually attach a small deposit (e.g., 1 yoctoNEAR for full-access-key enforcement, mirroring the pattern used by `sign`) will silently lose those tokens on every re-attestation. The probability is low but non-zero and grows with the frequency of TEE upgrades across the participant set.

---

### Recommendation

Add an unconditional refund of any excess deposit at the end of `submit_participant_info`, mirroring the pattern used by `require_deposit`:

```rust
// After the conditional storage-payment block:
let attached = env::attached_deposit();
if attached > NearToken::from_yoctonear(0) {
    Promise::new(account_id).transfer(attached).detach();
}
```

Alternatively, restructure the function so the refund path is always executed regardless of `attestation_storage_must_be_paid_by_caller`.

---

### Proof of Concept

1. Participant `alice.near` is already registered in `tee_state` (existing participant).
2. After a TEE image upgrade, `alice.near` calls `submit_participant_info` with a new attestation and attaches `1_000_000_000_000_000_000_000_000` yoctoNEAR (1 NEAR) — a common pattern when callers mirror the deposit requirement of `sign`.
3. `voter_account()` succeeds → `caller_is_not_participant = false`.
4. `add_participant` returns `ParticipantInsertion::UpdatedExistingParticipant` → `is_new_attestation = false`.
5. `attestation_storage_must_be_paid_by_caller = false` → the entire refund block is skipped.
6. The 1 NEAR is absorbed by the contract. No refund promise is created.
7. `alice.near` has permanently lost 1 NEAR with no on-chain recovery path short of a threshold-voted contract upgrade. [4](#0-3)

```rust
    #[payable]
    #[handle_result]
    pub fn submit_participant_info(
        &mut self,
        proposed_participant_attestation: dtos::Attestation,
        tls_public_key: dtos::Ed25519PublicKey,
    ) -> Result<(), Error> {
        let proposed_participant_attestation =
            proposed_participant_attestation.try_into_contract_type()?;

        let account_key = env::signer_account_pk();
        let account_id = Self::assert_caller_is_signer();

        log!(
            "submit_participant_info: signer={}, proposed_participant_attestation={:?}, account_key={:?}",
            account_id,
            proposed_participant_attestation,
            account_key
        );

        // Save the initial storage usage to know how much to charge the proposer for the storage
        // used
        let initial_storage = env::storage_usage();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // The node always signs submissions with an Ed25519 key
        // (`near_signer_key`), so the signer key here is Ed25519 in practice.
        // Reject non-Ed25519 signer keys rather than silently storing a value
        // we could never match against in `is_caller_an_attested_participant`.
        let account_public_key = dtos::Ed25519PublicKey::try_from(&account_key).map_err(|_| {
            InvalidParameters::InvalidTeeRemoteAttestation {
                reason: "signer account key must be Ed25519".to_string(),
            }
        })?;

        // Add the participant information to the contract state
        let attestation_insertion_result = self
            .tee_state
            .add_participant(
                NodeId {
                    account_id: account_id.clone(),
                    tls_public_key,
                    account_public_key,
                },
                proposed_participant_attestation,
                tee_upgrade_deadline_duration,
            )
            .map_err(|err| {
                let reason = match &err {
                    AttestationSubmissionError::InvalidAttestation(_) => {
                        format!("TeeQuoteStatus is invalid: {err}")
                    }
                    AttestationSubmissionError::TlsKeyOwnedByOtherAccount => err.to_string(),
                };
                InvalidParameters::InvalidTeeRemoteAttestation { reason }
            })?;

        let caller_is_not_participant = self.voter_account().is_err();
        let is_new_attestation = matches!(
            attestation_insertion_result,
            ParticipantInsertion::NewlyInsertedParticipant
        );

        let attestation_storage_must_be_paid_by_caller =
            is_new_attestation || caller_is_not_participant;

        if attestation_storage_must_be_paid_by_caller {
            // `saturating_sub`: if a re-submission shrinks the entry, charge nothing
            // rather than underflow. Intentional asymmetry: we do not refund freed bytes
            // either — the caller already paid for the larger entry, and we'd rather
            // accept that asymmetry than open a refund path for payload-shrinking games.
            let storage_used = env::storage_usage().saturating_sub(initial_storage);
            let cost = env::storage_byte_cost().saturating_mul(storage_used as u128);
            let attached = env::attached_deposit();

            if attached < cost {
                return Err(InvalidParameters::InsufficientDeposit {
                    attached: attached.as_yoctonear(),
                    required: cost.as_yoctonear(),
                }
                .into());
            }

            // Refund the difference if the proposer attached more than required
            if let Some(diff) = attached.checked_sub(cost)
                && diff > NearToken::from_yoctonear(0)
            {
                Promise::new(account_id).transfer(diff).detach();
            }
        }

        Ok(())
    }
