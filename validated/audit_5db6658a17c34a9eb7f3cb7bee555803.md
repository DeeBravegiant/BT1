Audit Report

## Title
Stale Cross-Chain Rate Enables Over-Issuance of rsETH Up to the Full Daily Mint Limit — (`contracts/cross-chain/CrossChainRateReceiver.sol`, `contracts/pools/RSETHPoolV3.sol`)

## Summary

`CrossChainRateReceiver.getRate()` returns the stored `rate` unconditionally with no staleness check, despite `lastUpdated` being recorded on every `lzReceive` call. `RSETHPoolV3.deposit()` uses this potentially stale rate to compute rsETH minted per ETH deposited. Because `dailyMintLimit` is denominated in rsETH, a stale (lower-than-true) rate lets an attacker consume the entire daily limit while depositing less ETH than the minted rsETH is actually worth, extracting the accrued yield difference.

## Finding Description

`CrossChainRateReceiver` stores `rate` and `lastUpdated` as public state variables. [1](#0-0) 

`lastUpdated` is written on every `lzReceive` call but is **never read** by `getRate()` or any consumer. `getRate()` returns `rate` unconditionally: [2](#0-1) 

`RSETHPoolV3.getRate()` delegates directly to this oracle: [3](#0-2) 

`viewSwapRsETHAmountAndFee` computes rsETH output as `amountAfterFee * 1e18 / rsETHToETHrate`. A stale low rate inflates `rsETHAmount`: [4](#0-3) 

The `limitDailyMint` modifier computes `rsETHAmount` from the same stale rate and compares it against `dailyMintLimit`, which is denominated in rsETH: [5](#0-4) 

**Exploit flow:**
1. rsETH accrues staking yield on L1; the true rate rises from `stale_rate` to `true_rate`.
2. A LayerZero update has not yet arrived; `CrossChainRateReceiver.rate` remains at `stale_rate < true_rate`.
3. A new day begins, resetting `dailyMintAmount = 0`.
4. Attacker calls `deposit{value: dailyMintLimit * stale_rate / 1e18}()`.
5. `viewSwapRsETHAmountAndFee` returns `rsETHAmount = dailyMintLimit` (exactly hits the cap).
6. `limitDailyMint` passes; `wrsETH.mint(attacker, dailyMintLimit)` executes.
7. The minted rsETH is redeemable on L1 at `true_rate`, worth `dailyMintLimit * true_rate / 1e18` ETH — more than the attacker deposited.

No existing guard prevents this: `limitDailyMint` only checks `dailyMintAmount + rsETHAmount > dailyMintLimit`, which passes exactly at the limit. [6](#0-5) 

## Impact Explanation

**High — Theft of unclaimed yield.**

The attacker receives rsETH redeemable for more ETH than deposited. The shortfall is borne by existing rsETH holders / the protocol treasury, representing the yield that accrued between the last rate update and the deposit. Exact excess per day:

```
excess_ETH ≈ dailyMintLimit × (true_rate − stale_rate) / 1e18
```

This repeats every day the rate remains stale. The impact is concrete and bounded by `dailyMintLimit` and the rate drift, both of which are non-zero under normal operating conditions.

## Likelihood Explanation

LayerZero message delivery is not instantaneous and is subject to network congestion and relayer downtime. rsETH accrues staking yield continuously, so `true_rate` predictably drifts above `stale_rate` over time. `deposit()` is a public, permissionless function requiring no special role. An attacker can monitor `lastUpdated` on-chain and act the moment a new day begins with a stale rate. No victim mistake or privileged access is required.

## Recommendation

1. **Add a staleness check in `getRate()`** (or in `RSETHPoolV3.getRate()`): revert or return a sentinel value if `block.timestamp − lastUpdated > MAX_RATE_AGE` (e.g., 24 hours).
2. **Alternatively, denominate `dailyMintLimit` in ETH** rather than rsETH, so the cap bounds deposited value regardless of rate accuracy.
3. Consider emitting an alert or pausing deposits automatically when the rate has not been updated within the expected window.

```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```
