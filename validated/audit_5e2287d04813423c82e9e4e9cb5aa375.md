Audit Report

## Title
Missing Chainlink `latestRoundData` Safety Checks Allow Zero Price to Permanently Freeze Depositor Funds - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all return values except `answer`, performing no validation. If Chainlink returns `price = 0` (e.g., incomplete round), the zero propagates through `LRTDepositPool.getRsETHAmountToMint()`, causing a depositor who passes `minRSETHAmountExpected = 0` to have their full LST amount transferred into the protocol while receiving 0 rsETH тАФ with no on-chain recovery path.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` captures only `price` from `latestRoundData()`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol#52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No checks on `price > 0`, `updatedAt > 0`, or `answeredInRound >= roundId` are present. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same codebase applies all three guards:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol#30-32
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

**Zero-price exploit path:**

1. Chainlink returns `answer = 0` (incomplete round / circuit-breaker edge case).
2. `ChainlinkPriceOracle.getAssetPrice(asset)` returns `uint256(0) * 1e18 / 1e18 = 0`.
3. `LRTDepositPool.getRsETHAmountToMint(asset, amount)` computes `(amount * 0) / rsETHPrice = 0`.
4. `_beforeDeposit` checks `rsethAmountToMint < minRSETHAmountExpected` тЖТ `0 < 0` is false тЖТ passes.
5. `IERC20(asset).safeTransferFrom(msg.sender, address(this), depositAmount)` тАФ user's LST is taken.
6. `_mintRsETH(0)` тЖТ `RSETH.mint(msg.sender, 0)` тЖТ `checkDailyMintLimit(0)` passes (`0 + 0 > maxMintAmountPerDay` is false) тЖТ `_mint(msg.sender, 0)` is a no-op.
7. User holds 0 rsETH and has no withdrawal claim. Deposited LST is permanently locked.

The `_beforeDeposit` slippage guard is the only protection, but it is trivially bypassed when `minRSETHAmountExpected = 0`, which is the default/common caller pattern.

## Impact Explanation

**Critical тАФ Permanent freezing of funds.** Any depositor who calls `depositAsset()` or `depositETH()` while the relevant Chainlink feed returns `price = 0` will lose their full deposit with no on-chain recovery mechanism. The user receives 0 rsETH and therefore cannot initiate any withdrawal, as all withdrawal paths require holding rsETH. There is no admin function in `LRTDepositPool` that returns assets to a depositor based on their original deposit record.

## Likelihood Explanation

Chainlink feeds are documented to return `answer = 0` for incomplete rounds (`updatedAt == 0`). This is a known edge case, not a theoretical one тАФ the same protocol team already guards against it in `ChainlinkOracleForRSETHPoolCollateral`, confirming awareness. `ChainlinkPriceOracle` is the primary oracle for all LST collateral assets (stETH, rETH, etc.) in the core deposit path, making every depositor active during such an event a potential victim. No attacker capability is required; an ordinary user calling `depositAsset` with the default `minRSETHAmountExpected = 0` is sufficient.

## Recommendation

Apply the same validation pattern already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt > 0, "Incomplete round");
    require(price > 0, "Invalid price");

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

**Foundry fork test outline:**

1. Deploy or fork with a mock `AggregatorV3Interface` for the stETH/ETH feed that returns `(1, 0, 0, 0, 1)` from `latestRoundData()` (simulating an incomplete round).
2. Register the mock feed via `ChainlinkPriceOracle.updatePriceFeedFor(stETH, mockFeed)`.
3. Call `LRTDepositPool.depositAsset(stETH, 10 ether, 0, "")` from a user who has approved 10 stETH.
4. Assert: `IERC20(stETH).balanceOf(depositPool)` increased by 10 ether.
5. Assert: `IRSETH(rsETH).balanceOf(user) == 0`.
6. Confirm no function in `LRTDepositPool` allows the user to recover their 10 stETH without holding rsETH.
