Audit Report

## Title
Queued Withdrawal rsETH Irrecoverable During Pause — (File: `contracts/LRTWithdrawalManager.sol`)

## Summary
`LRTWithdrawalManager.initiateWithdrawal` transfers rsETH from the caller into the contract and enqueues a `WithdrawalRequest`. Every function capable of returning those funds or processing the request — `completeWithdrawal`, `completeWithdrawalForUser`, and `unlockQueue` — is gated by `whenNotPaused`. No `cancelWithdrawal` or emergency-return function exists anywhere in the contract, so any rsETH deposited before a pause event is frozen for the entire duration of the pause with no recovery path for the user.

## Finding Description
`initiateWithdrawal` (L166–175) performs an immediate, irrevocable `safeTransferFrom` of the caller's rsETH into the contract and calls `_addUserWithdrawalRequest` (L744–758), which stores the request and increments `nextUnusedNonce` with no inverse operation. The three downstream paths that could return value to the user are all blocked while paused:

- `completeWithdrawal` — `whenNotPaused` (L183)
- `completeWithdrawalForUser` — `whenNotPaused` (L199)
- `unlockQueue` — `whenNotPaused` (L279)

A grep over the entire contract confirms zero occurrences of `whenPaused`, `cancelWithdrawal`, or any emergency-return logic. `LRTConfig.pauseAll()` (L262–285) pauses `LRTWithdrawalManager` atomically alongside all other protocol contracts in a single privileged call. After that call, a user whose rsETH is already inside the contract has no on-chain action available to recover it.

## Impact Explanation
Any rsETH transferred into `LRTWithdrawalManager` via `initiateWithdrawal` before a pause is frozen for the full duration of the pause. Because `pauseAll` is a single-transaction, protocol-wide action, the freeze is immediate and affects all in-flight withdrawal requests simultaneously. The pause duration is unbounded by the contract. This is a concrete, demonstrable **temporary freezing of user funds**, matching the allowed Medium impact class.

## Likelihood Explanation
`PAUSER_ROLE` can invoke `pauseAll` at any time in response to a real or perceived security incident — a routine incident-response action. Users who have already called `initiateWithdrawal` cannot anticipate or avoid the freeze; their rsETH is already inside the contract at the moment of the pause. The window between a user queuing a withdrawal and the pause being lifted can be arbitrarily long. No attacker capability is required; the scenario is triggered by normal protocol operation.

## Recommendation
Add a `cancelWithdrawal(address asset)` function that:
1. Carries no pause guard (or uses `whenPaused`) so it is callable during a pause.
2. Pops the caller's oldest pending (still-locked) nonce from `userAssociatedNonces[asset][msg.sender]`.
3. Deletes the corresponding `withdrawalRequests` entry.
4. Decreases `assetsCommitted[asset]` by `request.expectedAssetAmount`.
5. Returns `request.rsETHUnstaked` rsETH to the caller via `safeTransfer`.

Alternatively, add an admin-only emergency function (callable only when paused) that removes a specific queued request and returns the rsETH to the original requester.

## Proof of Concept
1. Alice calls `initiateWithdrawal(stETH, 1e18, "")`.
   - `1e18` rsETH is transferred from Alice to `LRTWithdrawalManager` (L166).
   - `assetsCommitted[stETH] += expectedAmount` (L173).
   - Request stored at nonce `N` (L751–756).
2. `PAUSER_ROLE` calls `LRTConfig.pauseAll()` (L262–285), pausing `LRTWithdrawalManager`.
3. Alice calls `completeWithdrawal(stETH, "")` → reverts: `Pausable: paused` (L183).
4. Operator calls `unlockQueue(stETH, ...)` → reverts: `Pausable: paused` (L279).
5. No `cancelWithdrawal` exists (confirmed by grep). Alice's `1e18` rsETH remains locked in the contract for the entire duration of the pause.

**Foundry test sketch:**
```solidity
function test_rsETH_frozen_during_pause() public {
    vm.prank(alice);
    withdrawalManager.initiateWithdrawal(stETH, 1e18, "");

    vm.prank(pauser);
    lrtConfig.pauseAll();

    vm.prank(alice);
    vm.expectRevert("Pausable: paused");
    withdrawalManager.completeWithdrawal(stETH, "");

    // Alice has no other on-chain recovery path.
    assertEq(rsETH.balanceOf(address(withdrawalManager)), 1e18);
}
```
