### Title
Single Attested Participant Can Forge CKD Response for `AppPublicKey` Requests Without Cryptographic Verification - (File: crates/contract/src/lib.rs)

### Summary

`respond_ckd` in `MpcContract` applies a BLS12-381 pairing-based output check (`ckd_output_check`) only for the `AppPublicKeyPV` variant of CKD requests. For the `AppPublicKey` (legacy, privately-verifiable) variant, the contract performs **no cryptographic verification** of the response before resolving all queued yields. A single Byzantine attested participant — strictly below the signing threshold — can call `respond_ckd` with an arbitrary forged `CKDResponse` for any pending `AppPublicKey` request, and the contract will accept and deliver it to the user.

### Finding Description

In `respond_ckd`, after confirming the caller is an attested participant, the contract branches on the request's `app_public_key` variant:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}   // ← no check at all
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}

pending_requests::resolve_yields_for(
    &mut self.pending_ckd_requests,
    &request,
    serde_json::to_vec(&response).unwrap(),
)
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` verifies the pairing equation `e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)` on-chain, making forgery cryptographically infeasible. [2](#0-1) 

For `AppPublicKey`, no analogous check exists. The contract immediately calls `resolve_yields_for`, which drains the entire fan-out queue and resumes every parked yield with the attacker-supplied bytes. [3](#0-2) 

The `AppPublicKey` variant is the legacy default and is the format used in all non-PV e2e tests and production integrations. [4](#0-3) 

The authorization check `assert_caller_is_attested_participant_and_protocol_active` only confirms the caller holds a valid TEE attestation and is in the current participant set — it does not require threshold agreement or any proof that the response was computed correctly. [5](#0-4) 

**Exploit path:**
1. User calls `request_app_private_key` with `AppPublicKey` variant; a yield is parked in `pending_ckd_requests`.
2. A single malicious attested participant (Byzantine, below threshold) calls `respond_ckd` with an arbitrary `CKDResponse{big_y: ..., big_c: ...}` before honest nodes respond.
3. The contract accepts the forged response with no verification, calls `resolve_yields_for`, and delivers the forged bytes to the user's parked yield.
4. Honest nodes' subsequent `respond_ckd` calls return `RequestNotFound` — the entry is already gone. [6](#0-5) 

### Impact Explanation

The user receives forged `(big_y, big_c)` values. When they unmask using their private key `a`, they derive a wrong key: `CKDOutput::unmask(a)` on attacker-chosen points yields garbage, not `msk · H(app_id)`. Any secret material or wallet key derived from this output is wrong and attacker-controlled. This is **unauthorized confidential key derivation output without the required participant authorization** — a Critical impact under the allowed scope. The threshold-signature requirement (that `t` of `n` participants must cooperate) is bypassed entirely for `AppPublicKey` CKD requests.

### Likelihood Explanation

Any single attested participant who turns Byzantine can execute this attack. The attacker needs only a valid TEE attestation (already held as a participant) and knowledge of the pending request key (observable on-chain from the `pending_ckd_requests` map or from the `request_app_private_key` transaction). No collusion, no leaked keys, no privileged operator access is required. The `AppPublicKey` variant is the primary production format.

### Recommendation

Apply the same pairing-based output verification to `AppPublicKey` responses. Since `AppPublicKey` provides only a G1 point `pk1 = a·G1` (no `pk2`), the current `ckd_output_check` cannot be applied directly. Two options:

1. **Deprecate `AppPublicKey` in favor of `AppPublicKeyPV`**, which carries both `pk1` and `pk2` and enables the on-chain pairing check. Reject new `AppPublicKey` requests at the contract level.
2. **Require threshold-quorum acknowledgment** before resolving `AppPublicKey` CKD yields: collect `respond_ckd` calls from at least `t` distinct attested participants for the same `(request, response)` pair before calling `resolve_yields_for`. This mirrors how threshold signing works in practice.

    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

/// Check that `e(big_c, g2) = e(big_y, app_pk2) . e(hash_point, public_key)`.
///
/// Point validation is fully delegated to the host, as in
/// [`app_public_key_check`].
pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}

```rust
/// Resume every yield queued for `request` with `response_bytes`, draining the
/// fan-out map in one pass. Returns `Err(RequestNotFound)` if the map held no entry.
///
/// Resuming a yield that has already timed out is a no-op at the SDK level.
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
