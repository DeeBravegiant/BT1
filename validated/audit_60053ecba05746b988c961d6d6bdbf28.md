Audit Report

## Title
Zero rsETH Minted to Depositor Due to Inflatable rsETHPrice With No Zero-Shares Guard - (File: `contracts/LRTDepositPool.sol`)

## Summary
`LRTDepositPool._beforeDeposit` computes `rsethAmountToMint` and checks it against `minRSETHAmountExpected`, but never asserts `rsethAmountToMint > 0`. Because `LRTDepositPool.receive()` accepts arbitrary ETH that is immediately counted in `totalETHInProtocol`, and `LRTOracle.updateRSETHPrice()` is permissionless with `pricePercentageLimit` defaulting to 0, an attacker who is the sole rsETH holder can inflate the stored price and cause a victim's deposit to mint zero rsETH ‚Äî permanently losing the victim's ETH to the pool.

## Finding Description

**Root cause 1 ‚Äî No zero-shares guard in `_beforeDeposit`:**

`_beforeDeposit` at `contracts/LRTDepositPool.sol:648-670` computes `rsethAmountToMint` and only checks `rsethAmountToMint < minRSETHAmountExpected`. When `minRSETHAmountExpected == 0` (the default for many callers) and `rsethAmountToMint` rounds to 0, the check `0 < 0` is `false`, so no revert occurs. `_mintRsETH(0)` is called, minting nothing to the depositor while their ETH is retained by the contract. [1](#0-0) 

**Root cause 2 ‚Äî `receive()` accepts arbitrary ETH, immediately reflected in TVL:**

`contracts/LRTDepositPool.sol:58` has an open `receive() external payable { }`. Any ETH sent directly to the contract is immediately reflected in `address(this).balance`, which is returned as `ethLyingInDepositPool` in `getETHDistributionData()`. [2](#0-1) [3](#0-2) 

**Root cause 3 ‚Äî `updateRSETHPrice()` is permissionless:**

`contracts/LRTOracle.sol:87-89` exposes `updateRSETHPrice()` as a public function gated only by `whenNotPaused`. Any unprivileged caller can commit an inflated price to storage. [4](#0-3) 

**Root cause 4 ‚Äî `pricePercentageLimit` defaults to 0, disabling the price-increase guard:**

`LRTOracle.initialize()` at `contracts/LRTOracle.sol:64-68` never sets `pricePercentageLimit`. The guard at `contracts/LRTOracle.sol:256-257` is `pricePercentageLimit > 0 && priceDifference > ...`, so when `pricePercentageLimit == 0` the entire guard short-circuits to `false`, accepting any magnitude of price increase from an unprivileged caller. [5](#0-4) [6](#0-5) 

**Root cause 5 ‚Äî Mint formula truncates to zero via integer division:**

`contracts/LRTDepositPool.sol:520` computes `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. If `rsETHPrice` is inflated beyond `amount * assetPrice`, integer division yields 0. [7](#0-6) 

**Combined exploit path:**

With `rsethSupply = 1 wei` (attacker is sole holder after seeding) and a direct ETH donation `D` to the deposit pool, `_updateRsETHPrice()` computes `newRsETHPrice = (seed + D) * 1e18 / seed`. With `seed = 1 wei` and `D = 1e18 wei`, `newRsETHPrice ‚âà 1e36 + 1e18`. For a victim depositing `V = 1e18` ETH: `rsethAmountToMint = (1e18 * 1e18) / (1e36 + 1e18) = 1e36 / (1e36 + 1e18) = 0` (integer truncation, since denominator exceeds numerator). The check `0 < 0` does not revert. The victim's 1e18 ETH is absorbed into the pool with 0 rsETH minted.

The attacker then redeems their 1 wei rsETH at the stored inflated price, recovering `‚âà 1e18` ETH (their donation). The victim's ETH remains in the pool. In a second round, the attacker re-seeds the now-empty pool (rsethSupply = 0 resets price to 1e18), calls `updateRSETHPrice()` to reflect the victim's stranded ETH in the price, and redeems again ‚Äî recovering the victim's full deposit. Net attacker profit ‚âà victim's deposit amount.

## Impact Explanation

**Critical ‚Äî Direct theft of user funds.** A victim who calls `depositETH(0, "")` after the attacker inflates the price transfers their ETH to `LRTDepositPool`, receives 0 rsETH, and has no recourse. The attacker recovers the victim's ETH through a two-round redemption sequence. This matches the allowed impact: *Direct theft of any user funds, whether at-rest or in-motion*.

## Likelihood Explanation

**Medium.** The attack requires: (1) the attacker to be the first (or only) rsETH holder ‚Äî realistic at protocol launch or after a full withdrawal cycle; (2) `pricePercentageLimit == 0` ‚Äî confirmed as the default state, persisting until explicit admin action; (3) the victim to call `depositETH` with `minRSETHAmountExpected == 0` ‚Äî common when frontends omit slippage protection or integrating contracts pass 0. The ETH cost of the donation equals the victim's deposit size (break-even on round 1, profitable on round 2). The attack is repeatable as long as the attacker can re-seed the pool before other depositors.

## Recommendation

1. **Add a zero-shares guard in `_beforeDeposit`:**
   ```solidity
   if (rsethAmountToMint == 0) revert ZeroSharesMinted();
   ```

2. **Set a non-zero `pricePercentageLimit` during `initialize`** (e.g., 1% = `1e16`) so that large single-block price jumps are rejected for non-manager callers.

3. **Reject direct ETH donations** by reverting in `receive()` unless the sender is a known NodeDelegator, RewardReceiver, or LRTConverter, preventing the donation inflation vector entirely.

## Proof of Concept

```
Precondition: rsETHPrice = 1e18, rsethSupply = 0, pricePercentageLimit = 0

Step 1 ‚Äî Attacker seeds the pool:
  attacker.depositETH{value: 1}(0, "")
  ‚Üí rsethAmountToMint = (1 * 1e18) / 1e18 = 1
  ‚Üí attacker holds 1 wei rsETH; pool ETH balance = 1 wei

Step 2 ‚Äî Attacker donates ETH directly:
  attacker sends 1e18 wei ETH to LRTDepositPool via receive()
  ‚Üí pool ETH balance = 1 + 1e18 wei

Step 3 ‚Äî Attacker inflates the stored price:
  attacker calls LRTOracle.updateRSETHPrice()
  ‚Üí totalETHInProtocol = 1 + 1e18
  ‚Üí rsethSupply = 1
  ‚Üí newRsETHPrice = (1 + 1e18) * 1e18 / 1 = 1e18 + 1e36
  ‚Üí pricePercentageLimit == 0 ‚Üí isPriceIncreaseOffLimit = false ‚Üí no revert
  ‚Üí rsETHPrice = 1e18 + 1e36 stored

Step 4 ‚Äî Victim deposits:
  victim.depositETH{value: 1e18}(0, "")
  ‚Üí rsethAmountToMint = (1e18 * 1e18) / (1e18 + 1e36)
                      = 1e36 / (1e36 + 1e18) = 0  [integer truncation]
  ‚Üí 0 < 0 is false ‚Üí no revert
  ‚Üí _mintRsETH(0) ‚Üí victim gets 0 rsETH
  ‚Üí pool ETH balance = 1 + 1e18 + 1e18 = 2e18 + 1 wei

Step 5 ‚Äî Attacker redeems (round 1):
  attacker redeems 1 wei rsETH at stored price ‚âà 1e36
  ‚Üí recovers ‚âà 1e18 wei = 1 ETH (their donation)
  ‚Üí rsethSupply = 0; pool has ‚âà 1e18 wei (victim's ETH)

Step 6 ‚Äî Attacker re-seeds and redeems (round 2):
  attacker.depositETH{value: 1}(0, "")  ‚Üí gets 1 wei rsETH (price reset to 1e18)
  attacker calls updateRSETHPrice()
  ‚Üí totalETHInProtocol ‚âà 1e18 + 1; rsethSupply = 1
  ‚Üí newRsETHPrice ‚âà 1e36
  attacker redeems 1 wei rsETH ‚Üí recovers ‚âà 1e18 wei = victim's 1 ETH

Net: attacker steals victim's 1e18 ETH; victim holds 0 rsETH.
```
