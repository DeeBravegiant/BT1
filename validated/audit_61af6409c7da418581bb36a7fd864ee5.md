Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Pause via `updateRSETHPrice()` During Price Drops, Temporarily Freezing User Withdrawals — (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is a public, permissionless function. When the computed rsETH price has dropped more than `pricePercentageLimit` below `highestRsethPrice`, any caller can invoke it to atomically pause `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. This temporarily blocks all user-facing withdrawal paths until an admin manually unpauses each contract. The asymmetry with the upside-protection branch — which explicitly reverts for non-managers — confirms the downside auto-pause path was not intended to be publicly triggerable.

## Finding Description

`updateRSETHPrice()` carries only a `whenNotPaused` guard and no role check:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the upside-protection branch (L252–266) explicitly gates execution behind a manager-role check when the price increase exceeds the threshold — non-managers receive `PriceAboveDailyThreshold`. The downside-protection branch (L269–282) has no equivalent gate:

```solidity
// contracts/LRTOracle.sol L269-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

`LRTOracle` holds `PAUSER_ROLE` on both `LRTDepositPool` and `LRTWithdrawalManager`, so these `pause()` calls succeed. Once `LRTWithdrawalManager` is paused, all three user-facing withdrawal functions revert:

- `initiateWithdrawal()` — `whenNotPaused` at L158
- `completeWithdrawal()` — `whenNotPaused` at L183
- `instantWithdrawal()` — `whenNotPaused` at L219

Recovery requires `onlyLRTAdmin` to call `unpause()` on each of the three contracts separately (L143 in `LRTOracle`, L352 in `LRTWithdrawalManager`).

**Exploit path:**
1. `pricePercentageLimit` is set (e.g., `1e16` = 1%).
2. An EigenLayer slashing event or LST depeg causes the live computed rsETH price to fall more than 1% below `highestRsethPrice`.
3. Any EOA calls `LRTOracle.updateRSETHPrice()`.
4. `isPriceDecreaseOffLimit` evaluates to `true`; all three contracts are paused atomically.
5. All users attempting `completeWithdrawal()`, `initiateWithdrawal()`, or `instantWithdrawal()` receive `Pausable: paused` reverts until admin intervention.

## Impact Explanation

**Medium — Temporary freezing of funds.** Users with pending unlocked withdrawal requests cannot call `completeWithdrawal()` to retrieve their assets. New withdrawals via `initiateWithdrawal()` and `instantWithdrawal()` are also blocked. The freeze is temporary (admin can unpause) but occurs precisely during market stress — when users most urgently need to exit — and requires no capital or special role from the attacker.

## Likelihood Explanation

The trigger condition (rsETH price dropping beyond `pricePercentageLimit`) is realistic during EigenLayer slashing events, LST depeg events, or large coordinated withdrawals that reduce TVL faster than `highestRsethPrice` is updated. Once the on-chain price condition is met, any monitoring bot or griefing actor can call the public function in a single transaction with no capital requirement. The attack is repeatable: after admin unpauses, if the price condition still holds, the same caller can immediately re-trigger the pause.

## Recommendation

Apply the same manager-role gate used in the upside-protection branch to the downside auto-pause path. Either:

1. **Restrict `updateRSETHPrice()`** to `onlyLRTOperator` or `onlyLRTManager`, relying on `updateRSETHPriceAsManager()` as the sole public-facing entry point for privileged callers; or
2. **Separate the auto-pause trigger**: in `_updateRsETHPrice()`, when `isPriceDecreaseOffLimit` is true, emit an event and set a flag rather than pausing immediately, and require a privileged actor to call a separate `triggerEmergencyPause()` function.

The upside-protection branch already demonstrates the correct pattern — add `if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) { revert ...; }` before the downside pause block.
