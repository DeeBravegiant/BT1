### Title
Stale `parameters_votes` Not Cleared on Resharing Cancellation Allows Governance Threshold Bypass - (File: `crates/contract/src/state/resharing.rs`)

### Summary

When `vote_cancel_resharing()` succeeds and the contract reverts to `Running` state, the `parameters_votes` field inside `previous_running_state` is not cleared. This field may have accumulated re-proposal votes cast via `vote_new_parameters()` during the resharing phase. Those stale votes are silently carried into the next `Running` epoch, allowing a subsequent resharing to be triggered with fewer fresh participant votes than the protocol requires.

### Finding Description

`ResharingContractState` stores a `previous_running_state: RunningContractState` that is created with `RunningContractState::new()` at the moment of the Running→Resharing transition. `RunningContractState::new()` explicitly initialises `parameters_votes` to `ThresholdParametersVotes::default()`. [1](#0-0) 

During resharing, any participant may call `vote_new_parameters` (re-proposal). This call routes to `ResharingContractState::vote_new_parameters`, which delegates to `self.previous_running_state.process_new_parameters_proposal(proposal)`, mutating `previous_running_state.parameters_votes` in place. [2](#0-1) 

If the re-proposal never reaches full consensus (fewer than `n` votes), the partial votes remain in `previous_running_state.parameters_votes`. When `vote_cancel_resharing()` subsequently reaches threshold and returns the running state, it clones `previous_running_state` verbatim and only sets `previously_cancelled_resharing_epoch_id`—it never resets `parameters_votes`. [3](#0-2) 

The returned `RunningContractState` therefore enters the next epoch carrying stale votes. When participants later call `vote_new_parameters` for the next resharing attempt, `ThresholdParametersVotes::vote` counts all stored entries whose value equals the submitted proposal—including the stale ones. [4](#0-3) 

The consensus check is `new_num_participants == n_votes`, so if `k` stale votes already exist for proposal `B`, only `n − k` fresh votes are needed to trigger resharing—not the full `n`. [5](#0-4) 

### Impact Explanation

The governance invariant requires every proposed participant to cast a fresh, explicit vote for a resharing proposal in the current epoch. Stale votes from a cancelled resharing epoch violate this invariant: participants who voted for re-proposal `B` during the cancelled resharing are counted as having voted for `B` in the next epoch without re-affirming their consent. In the extreme case where `n − 1` participants voted for `B` during resharing, a single additional vote in the next epoch suffices to trigger resharing—a near-complete bypass of the all-participants governance requirement. A successful resharing changes the active participant set and key shares, which directly controls who can co-sign threshold signatures and CKD outputs for all supported foreign chains.

### Likelihood Explanation

The preconditions are operationally realistic: resharing is a routine governance operation; re-proposals (`vote_new_parameters` during resharing) are explicitly supported and documented; and cancellation (`vote_cancel_resharing`) is the designed recovery path when resharing stalls. Any epoch in which resharing is initiated, partially re-proposed, and then cancelled leaves the contract in the vulnerable state. No privileged access or key material is required—any subset of legitimate participants below the signing threshold can trigger the condition.

### Recommendation

In `ResharingContractState::vote_cancel_resharing()`, reset `parameters_votes` to `ThresholdParametersVotes::default()` before returning `previous_running_state`, mirroring the invariant enforced by `RunningContractState::new()`:

```rust
pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
    ...
    if threshold_cancellation_votes_reached {
        let mut previous_running_state = self.previous_running_state.clone();
        previous_running_state.previously_cancelled_resharing_epoch_id =
            Some(prospective_epoch_id);
        // FIX: clear stale re-proposal votes so the next resharing
        // requires fresh consent from all participants.
        previous_running_state.parameters_votes = ThresholdParametersVotes::default();
        Some(previous_running_state)
    }
    ...
}

### Proof of Concept

1. **Setup**: Running state with participants `{P1, P2, P3}`, governance threshold `t = 2`, all-participants vote required for resharing (`n = 3`).
2. **Trigger resharing**: All three vote for proposal `A` → contract enters `Resharing` state (epoch `E+1`). `previous_running_state.parameters_votes` is empty.
3. **Accumulate stale votes**: `P1` and `P2` call `vote_new_parameters` with re-proposal `B` (epoch `E+2`). `previous_running_state.parameters_votes` now holds `{P1→B, P2→B}`. Consensus not reached (2 < 3).
4. **Cancel resharing**: `P1` and `P2` call `vote_cancel_resharing` (threshold `t = 2` met). Contract returns `previous_running_state` with `previously_cancelled_resharing_epoch_id = E+1` and **`parameters_votes = {P1→B, P2→B}` intact**.
5. **Exploit in next epoch**: `P3` calls `vote_new_parameters` with proposal `B` (epoch `E+2`). `ThresholdParametersVotes::vote(B, P3)` counts `{P1→B, P2→B, P3→B}` = 3 = `n`. Resharing with `B` is triggered immediately—with only **one fresh vote** from `P3`, and without fresh consent from `P1` or `P2`. [7](#0-6) [3](#0-2)

    pub fn vote_cancel_resharing(&mut self) -> Result<Option<RunningContractState>, Error> {
        let previous_running_participants = self.previous_running_state.parameters.participants();
        let authenticated_candidate = AuthenticatedAccountId::new(previous_running_participants)?;
        self.cancellation_requests.insert(authenticated_candidate);

        let cancellation_votes_count = self.cancellation_requests.len() as u64;
        let previous_running_threshold = self.previous_running_state.parameters.threshold();

        let threshold_cancellation_votes_reached: bool =
            cancellation_votes_count >= previous_running_threshold.value();

        let running_state = if threshold_cancellation_votes_reached {
            let mut previous_running_state = self.previous_running_state.clone();
            let prospective_epoch_id = self.prospective_epoch_id();
            previous_running_state.previously_cancelled_resharing_epoch_id =
                Some(prospective_epoch_id);

            Some(previous_running_state)
        } else {
            None
        };

        Ok(running_state)
    }
