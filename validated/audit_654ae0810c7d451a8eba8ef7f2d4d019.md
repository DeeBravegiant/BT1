Audit Report

## Title
Pause Bypass in `RSETHMultiChainRateProvider` Broadcasts Stale Pre-Drop Rate to All L2 Receivers — (`contracts/cross-chain/RSETHMultiChainRateProvider.sol`)

## Summary
`LRTOracle` auto-pauses and freezes `rsETHPrice` at its pre-drop value when a price decrease exceeds `pricePercentageLimit`. Because `RSETHMultiChainRateProvider.getLatestRate()` reads `rsETHPrice` directly without checking `paused`, and `MultiChainRateProvider.updateRate()` is permissionless, any caller can broadcast the frozen inflated rate to every L2 receiver while the oracle is paused. This defeats the protocol's own downside-protection mechanism and exposes L2 liquidity pools to redemptions at an inflated rate.

## Finding Description
`_updateRsETHPrice()` computes `newRsETHPrice` but, when the price drop exceeds `pricePercentageLimit`, calls `_pause()` and returns **before** reaching `rsETHPrice = newRsETHPrice` at line 313:

```solidity
// LRTOracle.sol L277-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated here
}
// ...
rsETHPrice = newRsETHPrice;  // L313 — never reached on auto-pause
```

`rsETHPrice` is therefore frozen at the last pre-drop value. The public `updateRSETHPrice()` is blocked by `whenNotPaused`, so no normal path can correct it while paused.

`RSETHMultiChainRateProvider.getLatestRate()` reads this frozen value directly:

```solidity
// RSETHMultiChainRateProvider.sol L26-28
function getLatestRate() public view override returns (uint256) {
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

`MultiChainRateProvider.updateRate()` has no access control and no pause check — only `nonReentrant`:

```solidity
// MultiChainRateProvider.sol L108
function updateRate() external payable nonReentrant {
```

An attacker calls `updateRate()` with sufficient ETH for LayerZero fees. The inflated frozen rate is encoded and sent to every registered L2 `CrossChainRateReceiver`, which stores it unconditionally:

```solidity
// CrossChainRateReceiver.sol L93-95
uint256 _rate = abi.decode(_payload, (uint256));
rate = _rate;
```

L2 pools that consume this rate for rsETH redemptions will price rsETH above its true post-drop value, allowing redemptions that drain ETH from the pool at the expense of liquidity providers.

## Impact Explanation
**Critical — Direct theft of user funds.** The auto-pause fires precisely because the true rsETH value has dropped materially (e.g., validator slashing). Broadcasting the pre-drop inflated rate to L2 pools enables any user to redeem rsETH at the inflated rate, extracting more ETH than the underlying assets are worth and directly draining ETH from L2 liquidity providers. The pause mechanism's sole purpose is to prevent exactly this propagation; the bypass nullifies it entirely.

## Likelihood Explanation
- The auto-pause requires no admin action; it is triggered automatically by `_updateRsETHPrice()` whenever a price drop exceeds `pricePercentageLimit`.
- `updateRate()` is callable by any EOA or contract; the only cost is gas and LayerZero fees (a few dollars to tens of dollars).
- The window between auto-pause and manual `unpause()` (restricted to `onlyLRTAdmin`) can be hours to days, providing ample time for repeated exploitation.
- No victim mistake is required; the attacker acts unilaterally.

## Recommendation
Add a pause check inside `RSETHMultiChainRateProvider.getLatestRate()`:

```solidity
function getLatestRate() public view override returns (uint256) {
    require(!ILRTOracle(rsETHPriceOracle).paused(), "Oracle paused");
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

Alternatively, add the same guard inside `MultiChainRateProvider.updateRate()` at the base contract level so all future rate providers inherit the protection. A complementary hardening is to expose a `pauseAwarePrice()` view on `LRTOracle` that reverts when paused and call that instead of reading `rsETHPrice` directly.

## Proof of Concept
```solidity
// Foundry mainnet fork test
function testPauseBypassBroadcastsStaleRate() public {
    // 1. Reference deployed LRTOracle and RSETHMultiChainRateProvider
    // 2. Mock asset price oracle to return a value that causes >pricePercentageLimit drop
    // 3. Call LRTOracle.updateRSETHPrice()
    //    → triggers auto-pause; rsETHPrice frozen at pre-drop value
    uint256 stalePre = lrtOracle.rsETHPrice();
    assertTrue(lrtOracle.paused());

    // 4. Attacker calls updateRate with LayerZero fee
    uint256 fee = provider.estimateTotalFee();
    vm.deal(attacker, fee);
    vm.prank(attacker);
    provider.updateRate{value: fee}();

    // 5. Provider rate equals stale pre-drop value
    assertEq(provider.rate(), stalePre);

    // 6. L2 receiver (or mock lzReceive call) stores the same stale rate
    assertEq(l2Receiver.rate(), stalePre);
    // True protocol price is significantly lower — rate diverges from reality
}
```
