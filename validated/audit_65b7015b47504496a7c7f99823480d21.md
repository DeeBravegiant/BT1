### Title
Missing Minimum Price Bound Enforcement for `l1_gas` and `l1_data_gas` Resource Bounds at Gateway Admission — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The `StatelessTransactionValidator::validate_resource_bounds` function enforces a minimum price floor only on `l2_gas.max_price_per_unit`, leaving `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` unchecked. The `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` has the same gap, acknowledged by a developer TODO. A transaction with zero L1/L1-data gas price bounds passes both gateway validators and is admitted to the mempool, but is deterministically rejected by the blockifier at pre-validation once it reaches execution.

### Finding Description

In `StatelessTransactionValidator::validate_resource_bounds`:

```rust
// Line 60-88 in crates/apollo_gateway/src/stateless_transaction_validator.rs
if !self.config.validate_resource_bounds {
    return Ok(());  // entire check bypassed
}
let resource_bounds = *tx.resource_bounds();
// Only checks total fee is non-zero (passes if l2_gas alone is non-zero)
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(...ZeroResourceBounds...);
}
// Only enforces minimum on l2_gas price — l1_gas and l1_data_gas have NO floor
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(...MaxGasPriceTooLow...);
}
``` [1](#0-0) 

The `max_possible_fee` check at line 66 passes as long as any single resource has non-zero bounds. A transaction with `l1_gas.max_price_per_unit = 0`, `l1_data_gas.max_price_per_unit = 0`, and `l2_gas.max_price_per_unit >= min_gas_price` satisfies both checks and is admitted.

The stateful validator repeats the same pattern — only `l2_gas` is checked, with an explicit developer TODO acknowledging the gap:

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...)
``` [2](#0-1) 

At execution time, the blockifier's `check_resource_bounds` in `account_transaction.rs` checks **all three** resources:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(ResourceBoundsError::MaxGasPriceTooLow { ... });
}
``` [3](#0-2) 

Since `min_l1_data_gas_price_wei = 1` (always non-zero after clamping in `apply_fee_transformations`), any transaction with `l1_data_gas.max_price_per_unit = 0` will **always** fail this check at execution. [4](#0-3) 

The production config schema confirms `min_l1_data_gas_price_wei = 1`: [5](#0-4) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

Any unprivileged user can craft a V3 transaction with:
- `l1_gas.max_price_per_unit = 0`
- `l1_data_gas.max_price_per_unit = 0`
- `l2_gas.max_price_per_unit >= 8_000_000_000` (the configured `min_gas_price`)
- `l2_gas.max_amount > 0`

This transaction passes both the stateless and stateful gateway validators, is admitted to the mempool, forwarded to the batcher, and only rejected at blockifier pre-validation. The gateway's stated invariant — "at least one resource bound is greater than zero" — is satisfied, but the per-resource price floor that the L2 gas check establishes is absent for L1 and L1-data gas. The blockifier's `TransactionFeeError::InsufficientResourceBounds` is the only guard, and it fires too late (post-admission).

### Likelihood Explanation

**Medium.** The attack requires no privilege. Any user with RPC access can submit such transactions. The `validate_resource_bounds` flag is `true` in production (the bypass path is only for bootstrap), so the partial check is always active. The gap is structural and reproducible on every block where L1 gas price > 0 (i.e., always, given `min_l1_data_gas_price_wei = 1`).

### Recommendation

Extend `validate_resource_bounds` in `StatelessTransactionValidator` to enforce a minimum price floor on all three resources, mirroring the existing L2 check:

```rust
// After the existing l2_gas check:
if resource_bounds.l1_gas.max_price_per_unit.0 < self.config.min_l1_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
if resource_bounds.l1_data_gas.max_price_per_unit.0 < self.config.min_l1_data_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Resolve the TODO in `StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` to also validate L1 and L1-data gas prices against the previous block's prices. Add `min_l1_gas_price` and `min_l1_data_gas_price` fields to `StatelessTransactionValidatorConfig` (analogous to the existing `min_gas_price`). [6](#0-5) 

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with:
   - `resource_bounds.l1_gas = ResourceBounds { max_amount: 1, max_price_per_unit: 0 }`
   - `resource_bounds.l1_data_gas = ResourceBounds { max_amount: 1, max_price_per_unit: 0 }`
   - `resource_bounds.l2_gas = ResourceBounds { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 }` (meets `min_gas_price`)
2. Submit via the gateway RPC endpoint.
3. Observe: `StatelessTransactionValidator::validate_resource_bounds` returns `Ok(())` — `max_possible_fee > 0` (L2 gas alone satisfies it), and `l2_gas.max_price_per_unit >= min_gas_price`.
4. Observe: `StatefulTransactionValidator::validate_resource_bounds` returns `Ok(())` — only L2 gas price is checked against the previous block threshold.
5. Transaction is admitted to the mempool.
6. When the batcher executes it, `blockifier::account_transaction::check_resource_bounds` fires `ResourceBoundsError::MaxGasPriceTooLow { resource: L1DataGas, max_gas_price: 0, actual_gas_price: 1 }` and the transaction is rejected at pre-validation — after admission. [7](#0-6) [8](#0-7) [9](#0-8)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
