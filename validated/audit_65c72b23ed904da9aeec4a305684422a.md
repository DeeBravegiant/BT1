Audit Report

## Title
Protocol Fee Incorrectly Charged on Rewards Accrued During Pause Period - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle._updateRsETHPrice()` skips fee extraction when `protocolPaused` is `true`, but the public entry point `updateRSETHPrice()` is gated by `whenNotPaused`, so `rsETHPrice` is never snapshotted at pause time. When the protocol is unpaused, the first call to `updateRSETHPrice()` charges the protocol fee on the full TVL delta since the last pre-pause price update, including all staking rewards that accrued during the pause period, contrary to the stated design intent.

## Finding Description
`_updateRsETHPrice()` computes `previousTVL` from the stored `rsETHPrice`:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);  // L234
```

It then conditionally charges a fee only when the protocol is not paused:

```solidity
bool protocolPaused = lrtDepositPool.paused() || withdrawalManager.paused() || paused;  // L240
if (!protocolPaused && totalETHInProtocol > previousTVL) {                               // L244
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```

The public entry point carries `whenNotPaused`, blocking any call during a pause:

```solidity
function updateRSETHPrice() public whenNotPaused {  // L87
    _updateRsETHPrice();
}
```

The automatic downside-protection path inside `_updateRsETHPrice()` pauses all three contracts and **returns early without updating `rsETHPrice`**:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();   // L278
    if (!withdrawalManager.paused()) withdrawalManager.pause(); // L279
    _pause();                                                // L280
    return;                                                  // L281
}
```

`LRTDepositPool.pause()` and `LRTOracle._pause()` do not call `_updateRsETHPrice()` before setting the paused flag:

```solidity
// LRTDepositPool.sol L349-351
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
    _pause();
}

// LRTOracle.sol L319-323
function _pause() internal {
    if (paused) return;
    paused = true;
    emit Paused(msg.sender);
}
```

Result: `rsETHPrice` is frozen at its pre-pause value. EigenLayer staking rewards continue to accrue during the pause, growing `totalETHInProtocol`. On the first `updateRSETHPrice()` call after unpausing, `protocolPaused = false`, and `rewardAmount = totalETHInProtocol − previousTVL` spans the entire interval from the last pre-pause price update through the pause period and beyond. The fee is charged on `R_pause + R_post` when it should only be charged on `R_post`.

Note: `updateRSETHPriceAsManager()` (L94) bypasses `whenNotPaused`, but if the price is still below the `pricePercentageLimit` threshold while paused, `_updateRsETHPrice()` will again hit the `isPriceDecreaseOffLimit` branch and `return` early without updating `rsETHPrice`, making a manual snapshot impossible in the most common pause scenario.

## Impact Explanation
The protocol fee is minted as rsETH to the treasury, diluting all existing rsETH holders. By charging the fee on `R_pause` — rewards that accrued during a period the code explicitly intends to be fee-free — rsETH holders lose yield they are entitled to. This is **theft of unclaimed yield** (High impact).

## Likelihood Explanation
Pauses are a routine protocol event: the automatic downside-protection logic in `_updateRsETHPrice()` triggers a pause whenever the rsETH price drops beyond `pricePercentageLimit`, and the `PAUSER_ROLE` can also pause manually. EigenLayer staking rewards accrue continuously regardless of pause state. Every pause-then-unpause cycle silently mis-charges fees on the paused-period rewards. No privileged attacker is required; the mis-accounting is automatic. Likelihood: **Medium**.

## Recommendation
Snapshot `rsETHPrice` before setting the paused flag in each pause entry point:

1. In `LRTOracle._pause()` (called by the automatic price-drop path), invoke `_updateRsETHPrice()` before setting `paused = true`. The existing `if (paused) return` guard prevents re-entrancy. However, since the automatic pause is triggered from within `_updateRsETHPrice()` after `newRsETHPrice` is already computed, the simplest fix is to update `rsETHPrice = newRsETHPrice` **before** the early `return` in the `isPriceDecreaseOffLimit` branch.

2. In `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()`, call `ILRTOracle(lrtOracleAddress).updateRSETHPriceAsManager()` before `_pause()` so that `rsETHPrice` reflects the TVL at the moment of the manual pause.

```diff
// LRTOracle.sol — automatic pause path
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
+   rsETHPrice = newRsETHPrice;   // snapshot before pausing
    _pause();
    return;
}
```

## Proof of Concept
1. Protocol is live; `updateRSETHPrice()` is called at block B0, setting `rsETHPrice = P0`.
2. At block B1, the rsETH price drops beyond `pricePercentageLimit`. `_updateRsETHPrice()` pauses all three contracts and returns **without** updating `rsETHPrice`. `rsETHPrice` remains `P0`.
3. During blocks B1–B2 (pause period), EigenLayer staking rewards accrue. `totalETHInProtocol` grows by `R_pause`.
4. At block B2, admin unpauses all three contracts.
5. At block B3, anyone calls `updateRSETHPrice()`:
   - `protocolPaused = false`
   - `previousTVL = rsethSupply × P0`
   - `totalETHInProtocol = previousTVL + R_pause + R_post`
   - `rewardAmount = R_pause + R_post`
   - `protocolFeeInETH = (R_pause + R_post) × feeBPS / 10_000`
6. The fee on `R_pause` is charged even though the protocol was paused during that entire period. rsETH holders are diluted by `R_pause × feeBPS / 10_000` worth of rsETH that should not have been minted to the treasury.

A Foundry fork test can confirm this by: (a) recording `rsETHPrice` and `totalETHInProtocol` before pause, (b) advancing time to simulate reward accrual, (c) unpausing, (d) calling `updateRSETHPrice()`, and (e) asserting that the rsETH minted to the treasury equals `(R_pause + R_post) × feeBPS / 10_000` rather than the expected `R_post × feeBPS / 10_000`.
