### Title
Timelord hardcodes `prev_transaction_block_height=0` in `_check_for_new_ip`, causing all v2-proof blocks to be silently dropped after scheduling — (`chia/timelord/timelord.py`)

### Summary
`_can_infuse_unfinished_block` validates a v2 proof using the correct `prev_transaction_block_height` from `get_last_tx_height()`, schedules the block, and adds it to `self.unfinished_blocks`. When the VDF completes, `_check_for_new_ip` re-calls `iters_from_block` with `uint32(0)` hardcoded. `verify_and_get_quality_string` unconditionally returns `None` for any v2 proof when `prev_transaction_block_height < HARD_FORK2_HEIGHT`, causing `assert required_iters is not None` to raise. The exception is swallowed, the block is never matched, and `NewInfusionPointVDF` is never broadcast.

### Finding Description
In `_check_for_new_ip` (timelord.py line 594), `iters_from_block` is called with `uint32(0)` as `prev_transaction_block_height`. In `verify_and_get_quality_string` (proof_of_space.py lines 139–141):

```python
if plot_param.strength_v2 is not None and prev_transaction_block_height < constants.HARD_FORK2_HEIGHT:
    log.info("v2 proof support has not yet activated")
    return None
```

Since `0 < HARD_FORK2_HEIGHT` always, every v2 proof returns `None` here. Back in `iters_from_block`, `assert required_iters is not None` raises `AssertionError`. In `_check_for_new_ip` this is caught by `except Exception`, the block is skipped, `ip_iters` stays `None`, and `assert ip_iters is not None` fires. The outer `_manage_chains` loop catches that too. No infusion point is ever sent.

### Impact Explanation
Every legitimate farmer using a v2 plot whose block reaches the timelord post-HARD_FORK2 loses their block reward permanently. The timelord silently discards the infusion point. This is a **High** impact: permanent inability for honest farmers to have valid blocks infused under normal network conditions.

### Likelihood Explanation
This triggers for 100% of v2-proof blocks once `HARD_FORK2_HEIGHT` is active. No special attacker action is required — any farmer with a v2 plot triggers it automatically.

### Recommendation
In `_check_for_new_ip`, replace `uint32(0)` with `self.last_state.get_last_tx_height()`, matching the call in `_can_infuse_unfinished_block`:

```python
_, ip_iters = iters_from_block(
    self.constants,
    unfinished_block.reward_chain_block,
    self.last_state.get_sub_slot_iters(),
    self.last_state.get_difficulty(),
    self.get_height(),
    self.last_state.get_last_tx_height(),  # was uint32(0)
)
```

### Proof of Concept
Unit test outline:
1. Set `HARD_FORK2_HEIGHT = 100`, `last_tx_height = 200`.
2. Construct a `NewUnfinishedBlockTimelord` with a valid v2 proof.
3. Call `_can_infuse_unfinished_block` → returns non-None (block scheduled).
4. Call `_check_for_new_ip` with the matching iteration.
5. Observe: `iters_from_block(..., uint32(0))` raises `AssertionError` (caught), `ip_iters` is `None`, second assert fires, no `NewInfusionPointVDF` is sent.
6. Fix: pass `last_tx_height=200` → proof validates, block is matched, infusion point is broadcast.
