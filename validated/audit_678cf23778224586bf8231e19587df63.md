Audit Report

## Title
Shared `dailyMintAmount` Counter Across ETH and LST Deposit Paths Causes Temporary Deposit Denial - (File: contracts/pools/RSETHPoolV3.sol, contracts/pools/RSETHPoolV3ExternalBridge.sol, contracts/pools/RSETHPoolV3WithNativeChainBridge.sol)

## Summary
In `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge`, a single `dailyMintAmount` counter is shared across both ETH and LST token deposit paths via the `limitDailyMint` modifier. When ETH deposits exhaust `dailyMintLimit`, LST token deposits revert with `DailyMintLimitExceeded` for the remainder of the 24-hour window, and vice versa. The impact is correctly classified as Low — the protocol fails to deliver its promised deposit service, but depositors' funds remain in their own wallets and are not frozen.

## Finding Description
In `RSETHPoolV3.sol`, the `limitDailyMint` modifier accumulates rsETH minted from all deposit types into a single `uint256 public dailyMintAmount` state variable:

```solidity
// RSETHPoolV3.sol L96–125
modifier limitDailyMint(uint256 amount, address token) {
    ...
    if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
        revert DailyMintLimitExceeded();
    }
    dailyMintAmount += rsETHAmount;
    _;
}
```

Both public deposit functions apply this modifier:
- `deposit(string referralId)` at L251 — ETH path — passes `limitDailyMint(msg.value, ETH_IDENTIFIER)`
- `deposit(address token, uint256 amount, string referralId)` at L280 — LST path — passes `limitDailyMint(amount, token)`

Because the modifier runs before the function body, a revert in `limitDailyMint` prevents the `safeTransferFrom` from ever executing. The user's LST tokens are never transferred to the contract; they remain in the user's wallet. The same pattern is confirmed in `RSETHPoolV3ExternalBridge.sol` (L130–159, L371, L399) and `RSETHPoolV3WithNativeChainBridge.sol` (L108–137, L287, L316). `RSETHPoolV2ExternalBridge.sol` is not affected because it only supports ETH deposits and its `limitDailyMint` modifier takes no token parameter.

The existing `dailyMintLimit` check is not insufficient — it functions as designed. The root cause is that the limit is applied globally across independent asset types rather than per-asset, so one asset's deposit volume consumes capacity that belongs to another.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

When the shared counter is exhausted by one deposit path, users attempting to deposit via the other path receive a revert. Their funds are never transferred to the contract and remain fully accessible in their own wallets. No funds are frozen, stolen, or lost. The protocol simply fails to accept deposits for up to 24 hours, denying users the yield accrual that would have begun at deposit time. This matches the Low impact category: the protocol fails to deliver its promised service (deposit acceptance and wrsETH minting), but depositors do not lose value. The submitted claim of Medium ("Temporary freezing of funds") is not supported because the depositors' assets are never in the contract and are freely usable outside the protocol during the blocked window.

## Likelihood Explanation
Medium likelihood. Any unprivileged depositor making a sufficiently large ETH deposit can exhaust the shared `dailyMintLimit` and block LST depositors for the remainder of the 24-hour window. No special privilege is required. The pools are deployed on multiple L2 chains, increasing the probability of organic limit exhaustion. The block is self-resolving after 24 hours when `currentDay > lastMintDay` resets `dailyMintAmount` to zero.

## Recommendation
Replace the single shared `dailyMintAmount` / `dailyMintLimit` pair with per-asset mappings:

```solidity
mapping(address token => uint256) public dailyMintAmount;
mapping(address token => uint256) public dailyMintLimit;
mapping(address token => uint256) public lastMintDay;
```

Update `limitDailyMint` to key all reads and writes on the `token` parameter. Provide a setter `setDailyMintLimit(address token, uint256 limit)` and migrate existing limits during reinitializer. Apply the same change to `RSETHPoolV3ExternalBridge` and `RSETHPoolV3WithNativeChainBridge`.

## Proof of Concept
1. Deploy `RSETHPoolV3` with `dailyMintLimit = L` (e.g., 1000 wrsETH) and both ETH and wstETH deposit paths enabled.
2. Alice calls `deposit("")` with enough ETH to mint exactly `L` wrsETH. `dailyMintAmount` is now `L`.
3. Bob calls `deposit(wstETH, amount, "")`. The `limitDailyMint` modifier computes `rsETHAmount > 0` for Bob's wstETH and checks `L + rsETHAmount > L` → true → reverts with `DailyMintLimitExceeded`.
4. Bob's wstETH is never transferred (modifier runs before `safeTransferFrom` at L284); it remains in Bob's wallet.
5. Bob cannot deposit until `getCurrentDay() > lastMintDay`, i.e., up to 24 hours later.
6. Foundry test: fork the target L2, set `dailyMintLimit` to a small value, call `deposit("")` with ETH to fill it, then call `deposit(wstETH, ...)` and assert it reverts with `DailyMintLimitExceeded`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

### Citations

**File:** contracts/pools/RSETHPoolV3.sol (L96-125)
```text
    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }
```

**File:** contracts/pools/RSETHPoolV3.sol (L246-265)
```text
    function deposit(string memory referralId)
        external
        payable
        nonReentrant
        whenNotPaused
        limitDailyMint(msg.value, ETH_IDENTIFIER)
    {
        if (!isEthDepositEnabled) revert EthDepositDisabled();
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }
```
