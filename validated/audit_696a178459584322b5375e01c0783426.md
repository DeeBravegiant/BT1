### Title
Consensus Vote Signature Not Verified — Any Peer Can Impersonate Any Validator to Force a False Decision - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`handle_vote` in `SingleHeightConsensus` checks that the `vote.voter` address belongs to the committee but never verifies the cryptographic signature on the vote. A single malicious peer can therefore send precommit votes that claim to originate from any set of real validators, accumulate a fake 2/3+ quorum, and force a node to reach a `DecisionReached` event without any real validator having actually cast those votes.

---

### Finding Description

`Vote` carries a `signature: RawSignature` field and the codebase already contains `verify_precommit_vote_signature` in `apollo_signature_manager`. However, `handle_vote` in `single_height_consensus.rs` contains an explicit `// TODO(Asmaa): verify the signature` comment and performs no cryptographic check:

### Impact Explanation

An attacker controlling one network peer can:

1. Observe a valid `proposal_commitment` being validated by the network (the commitment is broadcast in `ProposalFin` and in prevotes).
2. Craft `N` precommit `Vote` messages, each with a different real validator's `ContractAddress` as `voter`, all pointing to the same `proposal_commitment`, all with `signature: RawSignature::default()`.
3. Broadcast these to a target node. Each vote passes the membership check and the duplicate check (different `voter` keys), and is fed directly into the Tendermint state machine.
4. The state machine accumulates the fake weight and fires `SMRequest::DecisionReached` — committing the block — without any real validator having cast a precommit.

The committed block itself may be valid (the local node did execute and validate it), but the decision was reached without a real 2/3 quorum. Different nodes may reach different decisions in the same round, breaking Tendermint's safety guarantee and producing a chain fork. The resulting divergent state roots, transaction receipts, and event commitments constitute wrong state from the perspective of the network.

This maps to: **Critical — Wrong state/receipt/event commitment** because a fork means different nodes store different execution outputs for the same height.

---

### Likelihood Explanation

Any peer on the gossipsub network can send vote messages. No privileged position is required. The attacker does not need to be a validator. The attack requires only knowledge of the current committee (public information) and the `proposal_commitment` being voted on (broadcast in `ProposalFin`). The `RawSignature::default()` (all-zero bytes) is accepted without error because no verification is performed.

---

### Recommendation

Call `verify_precommit_vote_signature` (already implemented in `apollo_signature_manager`) inside `handle_vote` before accepting a vote. The validator's public key must be looked up from the committee by `vote.voter`. Votes with invalid or missing signatures must be dropped and the originating peer reported.

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // Resolve public key for the claimed voter.
    let Some(member) = self.committee.members().iter().find(|s| s.address == vote.voter) else {
        debug!("Ignoring vote from non validator: vote={:?}", vote);
        return VecDeque::new();
    };
    // Verify the signature before trusting the voter identity.
    match verify_precommit_vote_signature(
        block_hash_from_commitment(vote.proposal_commitment),
        vote.signature,
        member.public_key,
    ) {
        Ok(true) => {}
        _ => {
            warn!("Invalid vote signature, dropping: {:?}", vote);
            return VecDeque::new();
        }
    }
    ...
}
```

Additionally, map `originated_peer_id` → validator identity at the network layer so that a peer cannot even claim a `voter` address that does not correspond to its authenticated network identity.

### Proof of Concept

```rust
// Attacker node: after observing proposal_commitment C for height H round R,
// send fake precommit votes for every committee member.
for member in committee.members() {
    let fake_vote = Vote {
        vote_type: VoteType::Precommit,
        height: H,
        round: R,
        proposal_commitment: Some(C),
        voter: member.address,          // real validator address
        signature: RawSignature::default(), // empty — never checked
    };
    broadcast_vote(fake_vote);
}
// Target node accumulates all votes, reaches 2/3+ weight,
// fires DecisionReached without any real validator precommit.
```

        let determine_status = |old: &Vote, new: &Vote| {
            if old.proposal_commitment == new.proposal_commitment {
                VoteStatus::Duplicate
            } else {
                VoteStatus::Conflict(old.clone(), new.clone())
            }
        };

        // Check Map
        let key = (vote.round, vote.voter);
        let map_entry = match vote.vote_type {
            VoteType::Prevote => self.prevotes.get(&key),
            VoteType::Precommit => self.precommits.get(&key),
        };

        if let Some((old_vote, _)) = map_entry {
            return determine_status(old_vote, vote);
        }

        // Check Queue
        for event in &self.events_queue {

                (StateMachineEvent::Prevote(v), VoteType::Prevote) => v,
                (StateMachineEvent::Precommit(v), VoteType::Precommit) => v,
                _ => continue,
            };

            if queued_vote.round == vote.round && queued_vote.voter == vote.voter {
                return determine_status(queued_vote, vote);
            }
        }
