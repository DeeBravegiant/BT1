### Title
Unprotected `init` Function Allows Any Caller to Seize MPC Network Control Before Legitimate Initialization — (`crates/contract/src/lib.rs`)

### Summary
The `init` function in the MPC contract carries no caller restriction (`#[private]` or equivalent), meaning any NEAR account can call it during the window between contract deployment and legitimate initialization. An attacker who wins this race sets an arbitrary participant set and threshold, gaining full governance and signing authority over the MPC network.

### Finding Description
The `init` function is decorated with `#[init]` (NEAR SDK: callable exactly once, before contract state is set) but has **no** `#[private]` attribute and no in-body caller check:

```rust
// crates/contract/src/lib.rs:1924-1973
#[handle_result]
#[init]                          // ← no #[private]
pub fn init(
    parameters: dtos::ThresholdParameters,
    init_config: Option<dtos::InitConfig>,
) -> Result<Self, Error> {
    ...
    Ok(Self { ... })
}
```

By contrast, `init_running` — the migration-path initializer — carries **both** `#[init]` and `#[private]`:

```rust
// crates/contract/src/lib.rs:1976-1979
#[private]
#[init]
#[handle_result]
pub fn init_running(...) -> Result<Self, Error> {
```

The deployment guides (localnet, testnet, TEE) all show `init` called as a **separate transaction** after the WASM is uploaded:

```bash
# crates/contract/tests/sandbox/upgrade_to_current_contract.rs:438-474
# docs/localnet/localnet.md:265-268
near contract call-function as-transaction mpc-contract.test.near init \
  file-args /tmp/init_args.json ... sign-as mpc-contract.test.near ...
```

This creates a race window. Any NEAR account that submits a call to `init` before the legitimate deployer can supply attacker-controlled `ThresholdParameters` — including an attacker-owned participant set and a threshold of 1 — and the contract will accept it, because the only validation performed is structural (`parameters.validate()`), not caller identity.

The analog to the original report is direct: just as `initialize()` functions in the EVM contracts were callable by anyone before the owner was set (because `owner == address(0)` bypassed `onlyOwner`), `init` here is callable by anyone before the contract state is set, because NEAR's `#[init]` guard only prevents *re-initialization*, not *unauthorized first initialization*.

### Impact Explanation
An attacker who calls `init` first:
- Installs themselves (or colluding accounts) as the sole participants with threshold = 1.
- Immediately controls all governance votes (`vote_add_domains`, `vote_new_parameters`, `propose_update`, `vote_update`, etc.).
- Can trigger key generation for all domains under their control, producing MPC key shares they fully control.
- Can issue threshold signatures for arbitrary foreign-chain transactions without any legitimate participant authorization.

This satisfies: **Critical — Unauthorized transaction execution, threshold signature issuance, or confidential key derivation output without the required participant authorization.**

### Likelihood Explanation
The deployment guides for localnet, testnet, and TEE-based production all show `init` called as a separate CLI step after WASM deployment. Any observer of the NEAR mempool or block explorer who sees the deploy transaction can immediately submit a competing `init` call. The attacker needs no privileged access — only a funded NEAR account and knowledge of the contract address (public on-chain).

### Recommendation
Add `#[private]` to `init`, mirroring `init_running`:

```rust
#[private]
#[init]
#[handle_result]
pub fn init(
    parameters: dtos::ThresholdParameters,
    init_config: Option<dtos::InitConfig>,
) -> Result<Self, Error> { ... }
```

This restricts `init` to calls where `predecessor_account_id == current_account_id`, i.e., only the contract account itself (the deployer) can initialize it. Deployment tooling already supports atomic deploy+init via `--initFunction`/`--initArgs`, so this change does not break any legitimate workflow.

### Proof of Concept

1. Attacker watches NEAR for a transaction deploying the MPC contract WASM to `mpc-contract.near`.
2. Before the deployer's `init` transaction is finalized, attacker submits:
   ```bash
   near contract call-function as-transaction mpc-contract.near init \
     json-args '{
       "parameters": {
         "threshold": 1,
         "participants": { "next_id": 1, "participants": [
           ["attacker.near", 0, {"tls_public_key": "ed25519:<attacker_key>", "url": "https://attacker.example"}]
         ]}
       }
     }' prepaid-gas '300.0 Tgas' attached-deposit '0 NEAR' \
     sign-as attacker.near network-config mainnet sign-with-keychain send
   ```
3. If attacker's transaction lands first, `init` succeeds and the contract state is set with `attacker.near` as the sole participant at threshold 1.
4. Attacker calls `vote_add_domains` to add signing domains, then participates in key generation as the only node.
5. Attacker now holds the only key share and can produce valid threshold signatures for any foreign-chain transaction, draining any funds controlled by the MPC network.

async fn init_running_rejects_external_callers_pre_initialization() {
    let (worker, contract) = init().await;
    let number_of_participants = 2;
    let (accounts, participants) = gen_accounts(&worker, number_of_participants).await;

    let threshold_parameters = ThresholdParameters::new(
        participants.clone(),
        Threshold::new(number_of_participants as u64),
    )
    .unwrap();

    let init_running_args = serde_json::json!({
            "domains": [],
            "next_domain_id": 0,
            "keyset": Keyset::new(EpochId::new(2), vec![]),
            "parameters": threshold_parameters,
    });

    let execution_error = accounts[0]
        .call(contract.id(), method_names::INIT_RUNNING)
        .max_gas()
        .args_json(init_running_args)
        .transact()
        .await
        .unwrap()
        .into_result()
        .expect_err("method is private and not callable from participant account.");

    let error_message = format!("{:?}", execution_error);

    let expected_error_message = "Smart contract panicked: Method init_running is private";

    assert!(
        error_message.contains(expected_error_message),
        "init_running call was accepted by external caller. expected method to be private. {:?}",
        error_message
    )
