Audit Report

## Title
Stale `rsETHPrice` Propagated to L2 Pools After Auto-Pause Causes Depositors to Receive Under-Backed rsETH — (`contracts/LRTOracle.sol`, `contracts/cross-chain/RSETHRateProvider.sol`, `contracts/cross-chain/CrossChainRateProvider.sol`, `contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPoolNoWrapper.sol`)

## Summary

When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it pauses the protocol and returns early, leaving `rsETHPrice` frozen at the pre-drop peak value P. Because `RSETHRateProvider.updateRate()` is permissionless and reads the stale `rsETHPrice` with no pause check, anyone can push the inflated rate to L2 pools. L2 pools (`RSETHPoolV3`, `RSETHPoolNoWrapper`) then compute rsETH issuance using the stale rate, causing depositors to receive rsETH tokens whose actual backing is only a fraction of what they deposited.

## Finding Description

**Step 1 — Auto-pause freezes `rsETHPrice` at peak P.**

In `LRTOracle._updateRsETHPrice()`, when `newRsETHPrice` has dropped more than `pricePercentageLimit` below `highestRsethPrice`, the function pauses `lrtDepositPool`, `withdrawalManager`, and the oracle itself, then returns early: [1](#0-0) 

The assignment `rsETHPrice = newRsETHPrice` at line 313 is never reached. `rsETHPrice` remains at the old peak P while actual backing per rsETH is now P × (1 − drop%). [2](#0-1) 

**Step 2 — `RSETHRateProvider.getLatestRate()` reads the stale value with no pause check.**

`rsETHPrice` is a plain public state variable on `LRTOracle`. The `paused` flag only blocks `updateRSETHPrice()` via `whenNotPaused`; reads are unrestricted. [3](#0-2) 

**Step 3 — `CrossChainRateProvider.updateRate()` is permissionless.**

No role check exists. Any EOA can call `updateRate()`, which reads `getLatestRate()` (returning stale P) and sends it via LayerZero to the registered `RSETHRateReceiver`. [4](#0-3) 

**Step 4 — `CrossChainRateReceiver.lzReceive()` stores the pushed rate unconditionally.**

No staleness check, no circuit breaker, no minimum-rate sanity check. [5](#0-4) 

**Step 5 — L2 pools compute rsETH issuance from the stale rate.**

Both `RSETHPoolV3.viewSwapRsETHAmountAndFee()` and `RSETHPoolNoWrapper.viewSwapRsETHAmountAndFee()` divide `amountAfterFee` by the stale rate P returned by `getRate()`: [6](#0-5) [7](#0-6) 

Since P > actual backing (e.g., 0.9P after a 10% slash), users receive fewer rsETH tokens than the current backing justifies. On redemption they recover only 0.9 ETH per ETH deposited.

**Step 6 — L2 pools are not automatically paused by the L1 auto-pause.**

The auto-pause only calls `lrtDepositPool.pause()` and `withdrawalManager.pause()` — both L1 contracts. `RSETHPoolV3` and `RSETHPoolNoWrapper` on L2 have independent `paused` state variables and are not touched. [8](#0-7) [9](#0-8) 

## Impact Explanation

Every ETH deposited into an L2 pool during the window between the L1 auto-pause and admin recovery is exchanged at the stale inflated rate P. A user depositing 1 ETH receives `1e18 / P` rsETH tokens, each backed by only 0.9P ETH, so their position is worth 0.9 ETH — a direct, permanent 10% loss of principal. This is **Critical — direct theft of user funds**.

## Likelihood Explanation

- The auto-pause is a production feature that activates on any slashing event or oracle anomaly exceeding `pricePercentageLimit`.
- `updateRate()` is permissionless; routine keepers or a griever will call it.
- L2 pools have no staleness guard, no cross-chain pause relay, and no minimum-rate sanity check.
- The recovery window (admin unpause + manual rate correction) can span hours to days.

## Recommendation

1. **Gate `updateRate()` on oracle health**: In `RSETHRateProvider` (or `CrossChainRateProvider`), revert if `ILRTOracle(rsETHPriceOracle).paused() == true`.
2. **Propagate the pause to L2**: When the L1 auto-pause triggers, emit a cross-chain LayerZero message to pause all registered L2 pools.
3. **Add a rate-drop circuit breaker in `CrossChainRateReceiver`**: Reject incoming rates that are more than X% below the current stored rate.
4. **Add a staleness check in L2 pools**: Revert in `viewSwapRsETHAmountAndFee` if `lastUpdated` on the oracle is older than a configurable threshold.

## Proof of Concept

```solidity
// Foundry fork test — simplified
function test_staleRateAfterAutoPause() public {
    // 1. Simulate slash: totalETH drops 10%, triggering auto-pause
    vm.prank(oracle_updater);
    lrtOracle.updateRSETHPrice();
    // rsETHPrice frozen at P; actual backing is 0.9P

    assertTrue(lrtOracle.paused());
    assertEq(lrtOracle.rsETHPrice(), P); // stale

    // 2. Anyone pushes stale rate to L2 receiver (permissionless)
    rsETHRateProvider.updateRate{value: lzFee}();

    // 3. L2 receiver holds stale rate P
    assertEq(rsETHRateReceiver.getRate(), P);

    // 4. L2 user deposits 1 ETH — pool is NOT paused
    uint256 balanceBefore = wrsETH.balanceOf(user);
    vm.prank(user);
    rsETHPoolV3.deposit{value: 1 ether}("ref");
    uint256 rsETHReceived = wrsETH.balanceOf(user) - balanceBefore;

    // 5. rsETH received computed at stale rate P: rsETHReceived = 1e18 / P
    //    Actual redemption value: rsETHReceived * 0.9P / 1e18 = 0.9 ETH
    uint256 redemptionValue = rsETHReceived * actualBacking / 1e18;
    assertLt(redemptionValue, 1 ether); // ~10% permanent loss
}
```
