Audit Report

## Title
Missing Staleness Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables rsETH Price Manipulation and Yield Theft — (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` discards all validation fields from `latestRoundData()`, accepting stale prices without any freshness check. A permissionless call to `LRTOracle.updateRSETHPrice()` propagates the stale price into the stored `rsETHPrice`. An attacker who then deposits ETH at the deflated `rsETHPrice` receives excess rsETH, extracting value from existing holders when the oracle recovers.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` reads only the `answer` field from `latestRoundData()`:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`updatedAt`, `answeredInRound`, and `roundId` are all silently discarded. There is no heartbeat check, no `updatedAt != 0` guard, and no `answeredInRound >= roundId` check.

The same codebase already implements the correct pattern in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
// contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol L27-32
(uint80 roundID, int256 ethPrice,, uint256 timestamp, uint80 answeredInRound) =
    AggregatorV3Interface(oracle).latestRoundData();
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The stale price propagates as follows:

1. `LRTOracle.updateRSETHPrice()` (public, no access control) → `_updateRsETHPrice()`
2. `_updateRsETHPrice()` calls `_getTotalEthInProtocol()`, which calls `getAssetPrice(asset)` for each supported LST
3. `getAssetPrice()` delegates to `ChainlinkPriceOracle.getAssetPrice()` — returns stale-low price
4. `totalETHInProtocol` is undervalued → `newRsETHPrice = totalETHInProtocol / rsethSupply` is deflated
5. `rsETHPrice` is stored at the deflated value

The deposit minting formula in `LRTDepositPool.getRsETHAmountToMint()` is:

```solidity
// contracts/LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

For ETH deposits, `getAssetPrice(ETH_TOKEN)` returns a fixed 1e18 (1:1 with ETH), while `rsETHPrice()` returns the deflated stored value. The attacker therefore receives `ETH_amount * 1e18 / deflated_rsETHPrice` rsETH — more than the fair share.

## Impact Explanation

**High — Theft of unclaimed yield from existing rsETH holders.**

When a Chainlink LST/ETH feed (e.g., stETH/ETH, 24 h heartbeat on mainnet) goes stale at a price below the true market rate, the attacker deflates `rsETHPrice` via the permissionless `updateRSETHPrice()`, then deposits ETH to mint excess rsETH at the deflated price. When the oracle recovers and `rsETHPrice` rises back to its true level, the attacker's rsETH redeems for more ETH than was deposited. The surplus is taken from existing holders' proportional share of the pool — a direct theft of accrued yield.

**Medium — Temporary freezing of funds** (secondary): if the stale price drop exceeds `pricePercentageLimit`, `_updateRsETHPrice()` pauses both `LRTDepositPool` and `LRTWithdrawalManager`, freezing all user funds until an admin manually unpauses.

## Likelihood Explanation

Chainlink feeds have documented heartbeat intervals (24 h for stETH/ETH on mainnet) and can lag during periods of low L1 activity or sequencer downtime. The attacker requires:

1. Off-chain monitoring of the feed's `updatedAt` timestamp.
2. Observation that the last reported price is stale and below the true market rate.
3. A single permissionless call to `updateRSETHPrice()`.
4. An immediate ETH deposit via `depositETH()`.

No privileged access, no governance capture, and no brute-force is required. The attack is repeatable on every staleness event.

## Recommendation

Apply the same staleness guards already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();
    // Optional: configurable per-asset heartbeat
    // if (block.timestamp - updatedAt > maxStaleness[asset]) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, consider a configurable per-asset maximum staleness window so feeds that have not updated within their expected heartbeat interval are rejected.

## Proof of Concept

**Setup (Foundry fork test on mainnet):**

1. Fork mainnet at a block where the stETH/ETH Chainlink feed's `updatedAt` is near its 24 h heartbeat limit.
2. Warp `block.timestamp` forward by 24 h + 1 s without advancing the feed (simulating a stale feed).
3. Call `LRTOracle.updateRSETHPrice()` — no revert, `rsETHPrice` is set to a deflated value using the stale stETH price.
4. Record `rsETHPrice_before` and `rsETHPrice_deflated`.
5. Call `LRTDepositPool.depositETH{value: 10 ether}(0, "")` — attacker receives `10e18 * 1e18 / rsETHPrice_deflated` rsETH, more than the fair `10e18 * 1e18 / rsETHPrice_before`.
6. Warp forward to allow the Chainlink feed to update (or mock the feed returning the true price).
7. Call `updateRSETHPrice()` again — `rsETHPrice` recovers to its true value.
8. Assert that the attacker's rsETH balance redeems for more ETH than was deposited; the difference is sourced from existing holders' share.

**Key assertions:**
- `rsETHPrice` after step 3 < `rsETHPrice` before step 3 (deflation confirmed).
- Attacker rsETH minted > fair share (excess confirmed).
- Existing holder's redemption value decreases by the attacker's excess (yield theft confirmed).
