Audit Report

## Title
stETH Priced via Chainlink Market Rate Feed While All Other LSTs Use Protocol Exchange Rates, Enabling Deposit/Withdrawal Arbitrage During Depegs - (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` is the only oracle adapter used for stETH, and it reads whatever Chainlink aggregator is configured — which for stETH is the secondary-market rate feed. Every other supported LST (rETH, swETH, sfrxETH, EthX) has a dedicated adapter that calls a protocol-internal exchange rate function immune to secondary-market sentiment. When stETH trades at a discount, the protocol's TVL is undervalued, rsETH price is depressed, and an unprivileged attacker can deposit ETH at the artificially low rsETH price and withdraw stETH at the same depressed rate, extracting value from existing rsETH holders once the market rate recovers.

## Finding Description

**Root cause — no dedicated stETH oracle adapter:**

`contracts/oracles/ChainlinkPriceOracle.sol` L49-55 calls `latestRoundData()` on whatever aggregator is registered per asset: [1](#0-0) 

The `contracts/oracles/` directory contains dedicated adapters for rETH, swETH, sfrxETH, and EthX — each calling a protocol-internal exchange rate — but no `StETHPriceOracle`: [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

stETH therefore falls through to `ChainlinkPriceOracle` with the Chainlink stETH/ETH market-rate feed.

**TVL undervaluation path:**

`LRTOracle._getTotalEthInProtocol()` multiplies each asset's balance by its oracle price: [6](#0-5) 

When the market rate is depressed, `totalETHInProtocol` is undervalued and `_updateRsETHPrice()` computes a lower `rsETHPrice`.

**Partial mitigation — automatic pause on large depegs:**

`_updateRsETHPrice()` pauses the deposit pool and withdrawal manager if the price drop exceeds `pricePercentageLimit`: [7](#0-6) 

This guard is effective only when `pricePercentageLimit > 0` **and** the depeg exceeds the configured threshold. If `pricePercentageLimit` is zero (unset) or the depeg is smaller than the threshold (e.g., a 0.5% depeg with a 1% limit), the price update proceeds and the protocol remains open.

**Deposit minting at depressed price:**

`LRTDepositPool.getRsETHAmountToMint()` uses the stored (now depressed) `rsETHPrice`: [8](#0-7) 

**Withdrawal payout at depressed price:**

`LRTWithdrawalManager.getExpectedAssetAmount()` divides by the depressed asset price: [9](#0-8) 

**`_calculatePayoutAmount` — incomplete protection:**

At `unlockQueue` time the payout is capped at the lesser of the locked-in expected amount and the current return: [10](#0-9) 

If stETH has recovered by unlock time, `currentReturn` equals `expectedAssetAmount` (both computed with recovered prices), so the attacker still receives the inflated stETH amount locked in at initiation.

**Instant withdrawal — no delay:**

When `isInstantWithdrawalEnabled[stETH]` is true, `instantWithdrawal()` bypasses the 8-day delay entirely: [11](#0-10) 

## Impact Explanation

**Impact: High — Theft of unclaimed yield from existing rsETH holders.**

When stETH depegs within the configured `pricePercentageLimit` (or when the limit is unset), an attacker deposits ETH at a depressed rsETH price, receiving more rsETH than fair value. Upon withdrawal they receive stETH at the same depressed rate. After market recovery, the stETH they hold is worth more than the ETH they deposited. The excess rsETH minted to the attacker permanently dilutes all existing rsETH holders, constituting theft of their accrued yield.

Concrete example (0.5% depeg, within a 1% `pricePercentageLimit`):
- Protocol: 100 stETH × 0.995 = 99.5 ETH TVL; 100 rsETH supply; rsETH price = 0.995 ETH.
- Attacker deposits 1 ETH → mints `1 / 0.995 ≈ 1.00503 rsETH`.
- Attacker withdraws stETH: `expectedAssetAmount = 1.00503 × 0.995 / 0.995 = 1.00503 stETH`.
- After recovery (stETH = 1.005 ETH): attacker holds 1.00503 stETH ≈ 1.010 ETH — ~1% profit extracted from existing holders.

## Likelihood Explanation

stETH secondary-market depegs are documented real-world events (e.g., April 2024, ~1% depeg). The attack requires no privileged access: `updateRSETHPrice()` is public, `depositETH()` and `depositAsset()` are open to any user, and `initiateWithdrawal()` / `instantWithdrawal()` are open to any rsETH holder. The automatic pause guard only fires when `pricePercentageLimit > 0` and the depeg exceeds it; depegs smaller than the threshold leave the protocol fully open. If instant withdrawal is enabled for stETH, the attack requires no waiting period. **Likelihood: Medium.**

## Recommendation

Replace the Chainlink stETH/ETH market-rate feed with a protocol-internal exchange rate source. Create a dedicated `StETHPriceOracle` adapter (analogous to `RETHPriceOracle`, `SwETHPriceOracle`, etc.) that calls `stETH.getPooledEthByShares(1e18)` (Lido's protocol-guaranteed exchange rate). Register this adapter in `LRTOracle` for the stETH asset, ensuring consistency with all other LST oracle adapters and eliminating secondary-market price exposure.

## Proof of Concept

**Prerequisites:** `pricePercentageLimit` is 0 or the stETH depeg is smaller than the configured threshold; `isInstantWithdrawalEnabled[stETH]` is true for the instant path.

**Standard withdrawal path (8-day delay):**
1. stETH market rate drops to 0.995 ETH (Lido internal rate = 1.005 ETH); depeg is within `pricePercentageLimit`.
2. Attacker calls `LRTOracle.updateRSETHPrice()` — rsETH price drops to ~0.995 ETH; protocol remains unpaused.
3. Attacker calls `LRTDepositPool.depositETH{value: 1 ether}(0, "")` → receives `1e18 / 0.995e18 ≈ 1.00503e18` rsETH.
4. Attacker calls `LRTWithdrawalManager.initiateWithdrawal(stETH, 1.00503e18, "")` → `expectedAssetAmount = 1.00503 stETH` locked in.
5. After 8 days, operator calls `unlockQueue(stETH, ...)`. If stETH has recovered, `_calculatePayoutAmount` returns `min(1.00503, currentReturn)` — both converge to 1.00503 stETH.
6. Attacker calls `completeWithdrawal(stETH, "")` → receives 1.00503 stETH worth ~1.010 ETH at exchange rate.

**Instant withdrawal path (no delay):**
Steps 1–3 same as above. Then:
4. Attacker calls `LRTWithdrawalManager.instantWithdrawal(stETH, 1.00503e18, "")` → immediately receives `1.00503 stETH × (1 - fee)`.
5. stETH market rate recovers; attacker holds stETH worth more than the 1 ETH deposited — profit extracted with no waiting period.

            }
        }
    }
