Audit Report

## Title
Unrestricted `receive()` With No ETH Rescue Path Causes Temporary (Potentially Permanent) Fund Freezing and Accounting Distortion - (File: contracts/LRTConverter.sol)

## Summary
`LRTConverter` accepts ETH from any sender via an unrestricted `receive()` function but provides no standalone ETH withdrawal or rescue mechanism. ETH deposited outside of a Lido/Swell claim cycle is frozen in the contract until an operator calls `claimStEth()` or `claimSwEth()`. When that sweep eventually occurs, `_sendEthToDepositPool(address(this).balance)` over-decrements `ethValueInWithdrawal` by the stray ETH amount, permanently distorting the TVL reported by `getETHDistributionData()`.

## Finding Description
`LRTConverter` declares an open, unrestricted `receive()` function: [1](#0-0) 

The only two code paths that move ETH out of the contract are `claimStEth()` and `claimSwEth()`, both gated by `onlyLRTOperator` and both calling `_sendEthToDepositPool(address(this).balance)`: [2](#0-1) [3](#0-2) 

`_sendEthToDepositPool` forwards the **entire native balance** and simultaneously decrements `ethValueInWithdrawal`: [4](#0-3) 

`ethValueInWithdrawal` is only incremented when LSTs are transferred in via `transferAssetFromDepositPool` (proportional to the oracle-priced LST value). If stray ETH is present when a claim is swept, the decrement exceeds the legitimate claim amount, permanently understating `ethValueInWithdrawal`. `getETHDistributionData()` reads this value directly: [5](#0-4) 

`LRTConverter` does not inherit `Recoverable` and has no `rescueETH`, `transferETH`, or equivalent function. The `Recoverable` utility exists in the codebase and provides exactly this pattern: [6](#0-5) 

## Impact Explanation
**Medium — Temporary (potentially permanent) freezing of funds.** ETH sent directly to `LRTConverter` is inaccessible to its sender and cannot be recovered without operator intervention. If no further Lido/Swell claims are ever initiated, the ETH is permanently irrecoverable. Additionally, when the stray ETH is eventually swept, `ethValueInWithdrawal` is over-decremented, causing `getETHDistributionData()` to permanently underreport the ETH value in the converter, distorting protocol TVL and rsETH pricing.

## Likelihood Explanation
Any external account can trigger this with a single ETH transfer to the contract address — no privileges, no special conditions, no protocol state required. Mistaken transfers, protocol refunds, and bridge deliveries are all realistic triggers. The `receive()` function imposes zero restrictions.

## Recommendation
Add a `rescueETH()` function restricted to `onlyLRTAdmin`, following the existing `Recoverable` pattern already present in the codebase:

```solidity
function rescueETH(address recipient, uint256 amount) external onlyLRTAdmin {
    UtilLib.checkNonZeroAddress(recipient);
    (bool success,) = payable(recipient).call{value: amount}("");
    if (!success) revert EthTransferFailed();
    emit EthRescued(recipient, amount);
}
```

Alternatively, restrict `receive()` to only accept ETH from the Lido withdrawal queue and the Swell exit contract, reverting for all other senders.

## Proof of Concept
1. Deploy the protocol with `LRTConverter` configured.
2. An external user calls `address(lrtConverter).call{value: 1 ether}("")` — `receive()` accepts it silently.
3. No `claimStEth()` or `claimSwEth()` is pending; the 1 ETH is frozen with no callable exit.
4. `ethValueInWithdrawal` is unchanged, so `getETHDistributionData()` does not account for the stranded ETH.
5. When an operator later calls `claimStEth()` for a legitimate 10 ETH withdrawal, `_sendEthToDepositPool(11 ether)` is invoked. `ethValueInWithdrawal` is decremented by 11 ETH instead of 10 ETH, permanently understating the converter's tracked withdrawal value by 1 ETH.
6. All subsequent calls to `getETHDistributionData()` return a `ethLyingInConverter` that is 1 ETH lower than the true outstanding withdrawal value, distorting rsETH pricing.

        address lrtDepositPoolAddress = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);

        if (ethValueInWithdrawal > _amount) {
            ethValueInWithdrawal -= _amount;
        } else {
            ethValueInWithdrawal = 0;
        }
        // Send eth to deposit pool
        ILRTDepositPool(lrtDepositPoolAddress).receiveFromLRTConverter{ value: _amount }();
        emit EthTransferred(lrtDepositPoolAddress, _amount);
    }

    function recoverETH(address recipient, uint256 amount) external onlyRole(DEFAULT_ADMIN_ROLE) {
        UtilLib.checkNonZeroAddress(recipient);
        if (amount == 0) revert ZeroAmount();
        if (address(this).balance < amount) revert InsufficientBalance();

        (bool success,) = payable(recipient).call{ value: amount }("");
        if (!success) revert TransferFailed();

        emit ETHRecovered(recipient, amount);
    }
