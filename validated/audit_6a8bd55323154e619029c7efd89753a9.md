Audit Report

## Title
Missing Default Fee Initialization in `addSupportedToken` Allows Fee-Free Deposits - (File: contracts/pools/RSETHPool.sol)

## Summary
`RSETHPool.addSupportedToken` registers a new ERC-20 token but never initializes `tokenFeeBps[token]`, leaving it at the Solidity default of `0`. Any unprivileged depositor can immediately call `deposit(token, amount, referralId)` and receive rsETH at the full oracle rate with zero fee deducted, until a separate admin call to `setTokenFeeBps` is made. The protocol loses all swap-fee revenue on that token for the entire window between registration and fee configuration.

## Finding Description
`addSupportedToken` (lines 637–655) sets `supportedTokenOracle[token]`, `tokenBridge[token]`, and pushes to `supportedTokenList`, but never writes to `tokenFeeBps[token]`:

```solidity
// RSETHPool.sol L651-653
supportedTokenList.push(token);
supportedTokenOracle[token] = oracle;
tokenBridge[token] = bridge;
// tokenFeeBps[token] is never set → Solidity default: 0
```

The public `deposit` path (lines 284–305) calls `viewSwapRsETHAmountAndFee(amount, token)` (lines 326–347), which reads `tokenFeeBps[token]` directly:

```solidity
// RSETHPool.sol L335-336
uint256 feeBpsForToken = tokenFeeBps[token]; // 0 for any new token
fee = amount * feeBpsForToken / 10_000;      // fee == 0
```

There is no per-token deposit-enable guard, no check that `tokenFeeBps[token]` has been explicitly set, and no minimum-fee requirement. The only guards on `deposit` are `nonReentrant`, `whenNotPaused`, and `onlySupportedToken` — all of which pass immediately after `addSupportedToken` completes. `setTokenFeeBps` (lines 583–594) is a completely separate `DEFAULT_ADMIN_ROLE` call that must be made after the fact.

## Impact Explanation
Every deposit made between `addSupportedToken` and the subsequent `setTokenFeeBps` call accrues zero fee: `feeEarnedInToken[token]` remains `0` and the BRIDGER_ROLE collects nothing. Depositors receive rsETH at the full oracle-rate conversion, extracting value the protocol is designed to retain as fee yield. This constitutes **theft of unclaimed yield (High)** under the allowed impact scope.

## Likelihood Explanation
The condition is triggered every time a new token is added — a `TIMELOCK_ROLE` action that emits a public `AddSupportedToken` event. Any user monitoring on-chain events or the mempool can immediately deposit at 0 fee with no special privileges, no oracle manipulation, and no governance capture. The window persists until a separate admin transaction is confirmed, which may span multiple blocks or longer.

## Recommendation
Add a `_feeBps` parameter to `addSupportedToken` and set `tokenFeeBps[token] = _feeBps` atomically during registration, applying the same `_feeBps > 10_000 → revert InvalidFeeAmount()` guard already present in `setTokenFeeBps`. This eliminates the zero-fee window entirely:

```solidity
function addSupportedToken(address token, address oracle, address bridge, uint256 _feeBps)
    external onlyRole(TIMELOCK_ROLE)
{
    // ... existing checks ...
    if (_feeBps > 10_000) revert InvalidFeeAmount();
    supportedTokenList.push(token);
    supportedTokenOracle[token] = oracle;
    tokenBridge[token] = bridge;
    tokenFeeBps[token] = _feeBps;
    emit AddSupportedToken(token, oracle, bridge);
}
```

## Proof of Concept
1. Admin calls `addSupportedToken(tokenX, oracleX, bridgeX)` — `tokenFeeBps[tokenX]` is `0`.
2. Attacker calls `deposit(tokenX, 1_000e18, "ref")` in the same or next block.
3. `viewSwapRsETHAmountAndFee(1_000e18, tokenX)` computes `fee = 1_000e18 * 0 / 10_000 = 0`.
4. Attacker receives `rsETHAmount = 1_000e18 * tokenToETHRate / rsETHToETHrate` — full oracle-rate amount, no fee deducted.
5. `feeEarnedInToken[tokenX]` remains `0`; protocol collects nothing.
6. Admin later calls `setTokenFeeBps(tokenX, 30)` — all prior deposits were fee-free.

**Foundry test sketch:**
```solidity
function test_zeroFeeWindowOnNewToken() public {
    vm.prank(timelockAdmin);
    pool.addSupportedToken(address(tokenX), address(oracleX), address(bridgeX));

    // tokenFeeBps[tokenX] == 0 at this point
    assertEq(pool.tokenFeeBps(address(tokenX)), 0);

    uint256 depositAmount = 1_000e18;
    tokenX.mint(attacker, depositAmount);
    vm.startPrank(attacker);
    tokenX.approve(address(pool), depositAmount);
    pool.deposit(address(tokenX), depositAmount, "ref");
    vm.stopPrank();

    // Fee earned is zero — protocol received no fee revenue
    assertEq(pool.feeEarnedInToken(address(tokenX)), 0);
}
```
