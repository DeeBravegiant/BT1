### Title
Unvalidated `ProposalInit.builder` Allows Malicious Proposer to Inject Arbitrary `sequencer_address` into Block Hash and `get_block_info` Syscall — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries two distinct authority fields: `proposer` (the consensus identity) and `builder` (the block-builder/sequencer address). `is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri` — but never checks `builder`. Because `builder` flows directly into `sequencer_address` inside `BlockInfo`, which is (a) returned by the `get_block_info` syscall to every executing contract and (b) hashed into `PartialBlockHashComponents` → `PartialBlockHash` → `ProposalCommitment`, a malicious proposer can set `builder` to any felt value. All validators will execute with that injected address, compute the same (wrong) commitment, and commit a block whose `sequencer_address` and block hash are both corrupted.

---

### Finding Description

**`ProposalInit.builder` is proposer-supplied and never validated.**

`ProposalInit` is defined with both `proposer` and `builder`: [1](#0-0) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, L1 gas prices, and `fee_proposal_fri`. The `builder` field is absent from every check: [2](#0-1) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [3](#0-2) 

That `BlockInfo` is forwarded verbatim to `batcher.validate_block(ValidateBlockInput { block_info, … })`: [4](#0-3) 

Inside the batcher, `BlockContext::new(block_metadata.block_info, …)` is constructed from this `block_info`, so every transaction executes with the injected `sequencer_address`: [5](#0-4) 

After execution, `PartialBlockHashComponents::new(&block_info, header_commitments)` captures `sequencer_address` as `sequencer`: [6](#0-5) 

`sequencer` is then hashed into the block hash: [7](#0-6) 

Because both the proposer and every validator derive `sequencer_address` from the same unvalidated `init.builder`, the `ProposalFinMismatch` guard does **not** fire — both sides compute the same (wrong) commitment and the block is committed.

The `get_block_info` syscall exposes `sequencer_address` to contracts at execution time: [8](#0-7) 

The Cairo OS block-hash function also hashes `block_info.sequencer_address` directly: [9](#0-8) 

---

### Impact Explanation

1. **Wrong `get_block_info` syscall result (Critical):** Every contract that calls `get_block_info()` during the block receives the attacker-chosen `sequencer_address`. Any access-control pattern of the form `assert caller == get_block_info().sequencer_address` is bypassable or can be forced to fail.

2. **Wrong block hash committed (Critical):** `sequencer_address` is a direct input to the Poseidon block-hash chain. The committed `PartialBlockHash`, `ProposalCommitment`, and final `BlockHash` all embed the injected value. The on-chain state root and all downstream proofs are built over a block whose identity field is attacker-controlled.

3. **Wrong authoritative RPC view (High):** `starknet_getBlockWithTxHashes` / `starknet_getBlockWithTxs` return the stored `sequencer_address` from the block header, which will reflect the injected value to all clients.

---

### Likelihood Explanation

In a decentralized Tendermint-style network any validator can become the proposer for a given round. No special privilege beyond holding a validator key is required. The attack is a single-field substitution in `ProposalInit` before broadcasting — no mempool manipulation, no race condition, no external oracle dependency.

---

### Recommendation

Add a `builder` check inside `is_proposal_init_valid`. The validator knows its own `builder_address` (the same value used in `initiate_build`). Reject any `ProposalInit` whose `builder` does not match the locally configured builder address:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

Add `expected_builder: ContractAddress` to `ProposalInitValidation` and populate it from `SequencerConsensusContext::builder_address` when constructing the validation struct, mirroring how `l1_da_mode` and `l2_gas_price_fri` are already propagated. [10](#0-9)

### Proof of Concept

```
1. Malicious validator becomes proposer for height H, round R.

2. It constructs ProposalInit {
       height: H,
       round:  R,
       proposer: <own address>,
       builder: ContractAddress::from(0xdeadbeef),  // ← injected
       l2_gas_price_fri: <valid>,
       l1_da_mode: <valid>,
       timestamp: <valid>,
       starknet_version: LATEST,
       version_constant_commitment: expected_version_constant_commitment(),
       l1_gas_price_fri / wei / data_gas: <within margin>,
       fee_proposal_fri: <within margin>,
   }

3. Broadcasts ProposalPart::Init(init) followed by valid transactions and Fin.

4. Each validator calls is_proposal_init_valid → passes (builder not checked).

5. initiate_validation → convert_to_sn_api_block_info(init)
       → BlockInfo { sequencer_address: 0xdeadbeef, … }

6. Batcher executes all transactions under sequencer_address = 0xdeadbeef.
   - get_block_info() syscall returns sequencer_address = 0xdeadbeef to every contract.

7. PartialBlockHashComponents { sequencer: 0xdeadbeef, … } is hashed.
   - Both proposer and all validators compute the same PartialBlockHash.
   - ProposalFinMismatch guard does NOT fire.

8. decision_reached commits the block.
   - Block header: sequencer_address = 0xdeadbeef  (wrong, persisted).
   - Block hash: Poseidon(…, 0xdeadbeef, …)        (wrong, persisted).
   - RPC starknet_getBlock* returns sequencer_address = 0xdeadbeef.
```

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
