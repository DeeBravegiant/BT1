Audit Report

## Title
Missing Caller Identity Check Allows Anyone to Force Fee-Deducted Claims on Behalf of Any User - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary
`MerkleDistributor.claim()` accepts a caller-supplied `account` parameter but never verifies that `msg.sender == account`. Any external caller can reconstruct a victim's valid merkle proof from public on-chain data and trigger a claim on their behalf, causing the protocol fee (up to 10%) to be permanently deducted from the victim's claimable allocation. The victim's claim state is then marked consumed, preventing any recovery.

## Finding Description
In `contracts/utils/MerkleDistributor/MerkleDistributor.sol`, the `claim` function at lines 97–147 processes the full claim lifecycle — merkle proof verification, `userClaims[account]` state update, and token transfer — without any check that `msg.sender == account`. [1](#0-0) 

The fee deduction at lines 138–144 permanently redirects up to `MAX_FEE_IN_BPS = 1000` BPS (10%) of the victim's claimable amount to `protocolTreasury`: [2](#0-1) 

After the forced claim, `userClaims[account].lastClaimedIndex` and `userClaims[account].cumulativeAmount` are updated at lines 134–135, so any subsequent call by the victim reverts with `AlreadyClaimed`. [3](#0-2) 

All inputs required to execute this attack (`index`, `account`, `cumulativeAmount`, `merkleProof`) are derivable from the public `MerkleRootSet` event and the published merkle tree. No privileged access is required.

By contrast, `KernelMerkleDistributor._processClaim()` explicitly guards this path: [4](#0-3) 

`MerkleDistributor` has no equivalent guard.

## Impact Explanation
**High — Theft of unclaimed yield.** A victim receives `claimableAmount - fee` tokens instead of their full `claimableAmount`. With `MAX_FEE_IN_BPS = 1000`, up to 10% of every user's entitled distribution can be permanently redirected to `protocolTreasury` by any external caller. The victim's claim state is consumed and irrecoverable. [5](#0-4) 

## Likelihood Explanation
**High.** The attack requires no privileged access, no private keys, no flash loans, and no oracle manipulation. The merkle tree is public; all proof inputs are derivable from on-chain events. Any external caller can execute this against any unclaimed account at any time while the contract is unpaused. The attack is repeatable across every new merkle root epoch.

## Recommendation
Add a caller identity check at the top of `claim()`, mirroring the pattern already used in `KernelMerkleDistributor._processClaim()`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

The `Unauthorized` error should also be added to the `IMerkleDistributor` interface in `MerkleDistributor.sol`, as it currently only exists in the `KernelMerkleDistributor` interface.

## Proof of Concept
1. Protocol owner calls `setMerkleRoot(root)`. The merkle tree is public; Alice's leaf is `keccak256(abi.encodePacked(index, alice, cumulativeAmount))`.
2. Attacker reconstructs Alice's valid `merkleProof` from the public tree.
3. Attacker calls `MerkleDistributor.claim(index, alice, cumulativeAmount, merkleProof)` before Alice does.
4. Contract verifies the proof (passes), computes `fee = claimableAmount * feeInBPS / 10_000`, transfers `claimableAmount - fee` to Alice, and `fee` to `protocolTreasury`.
5. `userClaims[alice].lastClaimedIndex` and `userClaims[alice].cumulativeAmount` are updated.
6. Alice calls `claim` herself — reverts with `AlreadyClaimed`.
7. Alice has permanently lost up to 10% of her entitled token distribution with no recourse.

**Foundry test sketch:**
```solidity
function testForcedClaimDeductsFee() public {
    // Setup: deploy MerkleDistributor with feeInBPS = 1000, set merkle root
    // Derive Alice's valid proof from the merkle tree
    vm.prank(attacker);
    distributor.claim(index, alice, cumulativeAmount, aliceProof);
    // Assert: alice received cumulativeAmount * 9000 / 10000
    // Assert: protocolTreasury received cumulativeAmount * 1000 / 10000
    // Assert: alice's subsequent claim reverts with AlreadyClaimed
}
```
