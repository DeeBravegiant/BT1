### Title
Injected `attrRemain` Bytes in Byron Address CBOR Permanently Freeze Funds — (`eras/byron/ledger/impl/src/Cardano/Chain/Common/Address.hs`)

---

### Summary

An unprivileged attacker can permanently freeze any ADA sent to a Byron address by crafting a transaction output whose address CBOR contains injected unknown-attribute bytes (`attrRemain`). Because `checkAddrSpendingData` re-hashes `addrAttributes` including those bytes, but `addrRoot` was committed without them, the hash check always fails and the funds can never be spent. Recovery requires a hard fork.

---

### Finding Description

**Root cause — `checkAddrSpendingData` hashes the full `addrAttributes` including `attrRemain`**

`checkAddrSpendingData` reconstructs `Address'` from the UTxO-stored address and hashes it:

```haskell
checkAddrSpendingData asd addr =
  addrRoot addr == addressHash address'
  && addrType addr == addrSpendingDataToType asd
  where
    address' = Address' (addrType addr, asd, addrAttributes addr)
``` [1](#0-0) 

`addressHash` serializes via `EncCBOR` using `serialize byronProtVer`: [2](#0-1) 

`EncCBOR (Attributes AddrAttributes)` calls `encCBORAttributes`, which **starts with `attrRemain` as the base map** and merges known fields into it:

```haskell
encCBORAttributes encs attr =
  encCBOR $ foldr go (fromUnparsedFields $ attrRemain attr) encs
``` [3](#0-2) 

So any non-empty `attrRemain` bytes are included in the serialization and therefore in the hash.

**`addrRoot` is committed without `attrRemain`**

`makeAddress` uses `mkAttributes`, which always sets `attrRemain` to the empty map:

```haskell
mkAttributes dat = Attributes dat (UnparsedFields M.empty)
``` [4](#0-3) 

So every legitimately created address has `addrRoot = hash(Address'(type, spendingData, Attributes{..., attrRemain=∅}))`. [5](#0-4) 

**`DecCBOR Address` preserves `attrRemain` from the wire**

When an address is decoded from a transaction output, unknown attribute keys (anything other than `1` and `2`) are preserved in `attrRemain`:

```haskell
go :: Word8 -> LByteString -> AddrAttributes -> Decoder s (Maybe AddrAttributes)
go n v acc = case n of
  1 -> ...
  2 -> ...
  _ -> pure Nothing   -- key stays in attrRemain
``` [6](#0-5) 

**`validateTxOutNM` explicitly permits up to 127 bytes of `attrRemain`**

The only guard on address attributes in transaction outputs is a size cap, not a zero-`attrRemain` requirement:

```haskell
unknownAttributesLength (addrAttributes (txOutAddress txOut))
  < 128
  `orThrowError` TxValidationUnknownAddressAttributes
``` [7](#0-6) 

**CRC32 is not a barrier**

`decodeCrcProtected` checks CRC32 of the body, but the attacker controls the entire CBOR blob and can compute the correct CRC32 for any body they construct: [8](#0-7) 

---

### Impact Explanation

The invariant `∀ addr ∈ UTxO, ∃ witness satisfying checkAddrSpendingData` is broken. Any ADA sent to the crafted address is permanently unspendable. `validateWitness` calls `checkVerKeyAddress vk addr` → `checkAddrSpendingData (VerKeyASD vk) addr`, which always returns `False` because the hash includes the injected `attrRemain` bytes that were absent when `addrRoot` was computed. [9](#0-8) 

---

### Likelihood Explanation

The attacker needs only to:
1. Observe any target Byron address (public information on-chain).
2. Decode its CBOR, keep `addrRoot` and `addrType` unchanged, inject 1–127 bytes under an unknown attribute key (e.g., key `0x05`) into `addrAttributes`.
3. Re-compute CRC32 over the modified body (trivial arithmetic).
4. Submit a transaction with this crafted address as an output.

No privileged access, no key material, no governance majority is required. The attack is fully local and testable.

---

### Recommendation

In `validateTxOutNM`, reject any transaction output whose address has non-empty `attrRemain`:

```haskell
attributesAreKnown (addrAttributes (txOutAddress txOut))
  `orThrowError` TxValidationUnknownAddressAttributes
```

`attributesAreKnown` already exists and checks `M.null . fromUnparsedFields . attrRemain`. [10](#0-9) 

Alternatively, `checkAddrSpendingData` could strip `attrRemain` before hashing, but that would change the hash function for all existing addresses and is more invasive.

---

### Proof of Concept

```haskell
-- Construct a legitimate address
let vk   = someVerificationKey
    addr = makeVerKeyAddress NetworkMainOrStage vk
-- addr has addrRoot = hash(Address'(ATVerKey, VerKeyASD vk, Attributes{..., attrRemain=∅}))

-- Craft a poisoned address: same addrRoot, same addrType, but inject attrRemain
let poisonedAttrs = (addrAttributes addr)
      { attrRemain = UnparsedFields (M.singleton 0x05 "AAAA") }
    poisonedAddr  = addr { addrAttributes = poisonedAttrs }
-- Re-encode with correct CRC32 (encodeCrcProtected handles this automatically)

-- Fund poisonedAddr in a transaction output → passes validateTxOutNM (4 bytes < 128)

-- Spend attempt:
assert $ checkVerKeyAddress vk poisonedAddr == False
-- addressHash(Address'(ATVerKey, VerKeyASD vk, Attributes{..., attrRemain={"0x05":"AAAA"}}))
-- ≠ addrRoot (which was computed with attrRemain=∅)
-- → TxValidationWitnessWrongKey → funds permanently frozen
```

encCBORAttributes ::
  forall t. [(Word8, t -> LBS.ByteString)] -> Attributes t -> Encoding
encCBORAttributes encs attr =
  encCBOR
    $ foldr go (fromUnparsedFields $ attrRemain attr) encs
  where
    go ::
      (Word8, t -> LBS.ByteString) ->
      Map Word8 LBS.ByteString ->
      Map Word8 LBS.ByteString
    go (k, f) = M.alter (insertCheck $ f (attrData attr)) k
      where
        insertCheck :: a -> Maybe LByteString -> Maybe a
        insertCheck v Nothing = Just v
        insertCheck _ (Just v') =
          panic
            $ "encCBORAttributes: impossible: field no. "
            <> show k
            <> " is already encoded as unparsed field: "
            <> show v'
```
