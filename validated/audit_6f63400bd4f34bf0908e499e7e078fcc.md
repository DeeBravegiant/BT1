Audit Report

## Title
Fee Double-Counting on Price Recovery Due to Lowered `rsETHPrice` Baseline - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle._updateRsETHPrice()` computes the fee baseline using `rsETHPrice`, which is unconditionally lowered on every price-decrease update. When the price subsequently recovers, the protocol charges fees against the lowered baseline, treating principal recovery as new yield. This causes rsETH to be incorrectly minted to the treasury, diluting all existing rsETH holders.

## Finding Description
The fee baseline is computed at line 234 using the last stored `rsETHPrice`:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);   // L234
``` [1](#0-0) 

If `totalETHInProtocol > previousTVL`, a fee is charged on the difference:

```solidity
uint256 rewardAmount = totalETHInProtocol - previousTVL;
protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
``` [2](#0-1) 

At the end of every call, `rsETHPrice` is unconditionally set to the new price — including when it is lower:

```solidity
rsETHPrice = newRsETHPrice;   // L313
``` [3](#0-2) 

The contract maintains `highestRsethPrice` as a true high-water mark, but it is only updated upward and is only used for the pause guard — never for the fee baseline:

```solidity
if (newRsETHPrice > highestRsethPrice) {
    highestRsethPrice = newRsETHPrice;
}
``` [4](#0-3) 

The downside-protection pause only fires when `pricePercentageLimit > 0` and the drop exceeds the configured threshold. If `pricePercentageLimit == 0` (disabled) or the drop is within the limit, execution continues, `rsETHPrice` is lowered, and the double-fee scenario is fully reachable:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
if (isPriceDecreaseOffLimit) { ... _pause(); return; }
``` [5](#0-4) 

**Exploit path:**
1. 1000 rsETH minted; `rsETHPrice = 1.0`, `highestRsethPrice = 1.0`.
2. Yield accrues; `totalETHInProtocol = 1200`. `previousTVL = 1000`. Fee taken on 200 ETH. `rsETHPrice = 1.2`, `highestRsethPrice = 1.2`. ✅ Correct.
3. LST dips; `totalETHInProtocol = 900 < previousTVL = 1200`. No fee. `rsETHPrice` lowered to `0.9`. `highestRsethPrice` stays at `1.2`.
4. LST recovers; `totalETHInProtocol = 1200`. `previousTVL = supply * 0.9 = 900`. `rewardAmount = 300`. Fee charged on 300 ETH of "yield" — but this is pure recovery of the step-3 loss. rsETH is minted to treasury, diluting all holders. ❌

`updateRSETHPrice()` is public and permissionless, so no privileged actor is required: [6](#0-5) 

## Impact Explanation
**High — Theft of unclaimed yield.** The protocol mints rsETH to the treasury on recovery of losses. This dilutes all rsETH holders proportionally: their share of the underlying ETH is reduced by the incorrectly minted fee tokens. The effect compounds every time the price dips and recovers within the configured threshold, and the magnitude scales with the size of the dip and the total rsETH supply.

## Likelihood Explanation
**Medium.** LST prices (stETH, cbETH, rETH, etc.) fluctuate relative to ETH due to secondary-market discounts, slashing events, and oracle lag. A dip followed by recovery is a routine market event. `updateRSETHPrice()` is a public, permissionless function callable by anyone, so no privileged actor is required to trigger the mispricing. The scenario is fully reachable whenever `pricePercentageLimit` is zero or the dip is within the configured limit.

## Recommendation
Replace the fee baseline with `highestRsethPrice` (the existing high-water mark) instead of `rsETHPrice`:

```solidity
// Use highestRsethPrice as the fee baseline so recovery of losses is never charged
uint256 previousTVL = rsethSupply.mulWad(highestRsethPrice);
```

This ensures fees are only charged on TVL that genuinely exceeds the previous all-time high, exactly mirroring the intent of `highestRsethPrice`.

## Proof of Concept
Foundry fork/unit test plan:
1. Deploy `LRTOracle` with `pricePercentageLimit = 0` (or any value where the dip is within limit).
2. Deposit 1000 stETH; assert `rsETHPrice = 1e18`, `highestRsethPrice = 1e18`.
3. Mock oracle to return `1.2e18`; call `updateRSETHPrice()`. Assert fee minted on 200 ETH of gain. Assert `rsETHPrice = 1.2e18`, `highestRsethPrice = 1.2e18`.
4. Mock oracle to return `0.9e18`; call `updateRSETHPrice()`. Assert no fee minted. Assert `rsETHPrice = 0.9e18`, `highestRsethPrice = 1.2e18`.
5. Mock oracle to return `1.2e18`; call `updateRSETHPrice()`. Assert fee is minted on `supply * (1.2 - 0.9) = supply * 0.3` ETH — demonstrating fee charged on principal recovery, not new yield.
6. Verify treasury rsETH balance increased and existing holder ETH-per-rsETH ratio decreased relative to the correct (no-fee) baseline.
