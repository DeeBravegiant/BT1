### Title
Unverified `MockAttestation::Valid` Accepted in Production Contract, Enabling TEE Attestation Bypass for Existing Participants — (File: `crates/mpc-attestation/src/attestation.rs`, `crates/contract/src/tee/tee_state.rs`, `crates/contract/src/lib.rs`)

---

### Summary

The production MPC contract accepts `Attestation::Mock(MockAttestation::Valid)` in `submit_participant_info` without any cryptographic TEE proof. This is the direct analog to the external report's unverified header: a self-asserted "I am valid" claim is accepted as authenticated TEE identity. Any existing participant can submit this variant to permanently bypass the TEE attestation requirement, and the re-verification path (`reverify_and_cleanup_participants`) will never evict them.

---

### Finding Description

The `MockAttestation::Valid` variant in `crates/mpc-attestation/src/attestation.rs` unconditionally returns `Ok(())` with zero cryptographic verification:

```rust
MockAttestation::Valid => Ok(()),
``` [1](#0-0) 

The `add_participant` function in `TeeState` calls `attestation.verify_locally(...)`, which for the `Mock` variant dispatches directly to `mock_attestation.verify(...)` — no DCAP quote, no report-data binding, no hardware proof required: [2](#0-1) 

`submit_participant_info` in the production contract accepts this path without any feature flag or environment guard: [3](#0-2) 

The result is stored in `tee_state.stored_attestations` as a `VerifiedAttestation::Mock(MockAttestation::Valid)`: [4](#0-3) 

The re-verification path `reverify_and_cleanup_participants`, called during `vote_new_parameters`, also passes `MockAttestation::Valid` unconditionally — so a participant holding this attestation is **never evicted**: [5](#0-4) 

The `is_caller_an_attested_participant` check used by `respond`, `respond_ckd`, and `respond_verify_foreign_tx` only verifies that a stored attestation exists and that the stored `account_public_key` matches the signer — it does **not** verify that the attestation is a real Dstack/TEE quote: [6](#0-5) 

There is no production guard anywhere in the contract that rejects `MockAttestation` variants. The sandbox test explicitly confirms this works in a production-equivalent environment: [7](#0-6) 

---

### Impact Explanation

An existing participant (already in the active participant set) can call `submit_participant_info(Attestation::Mock(MockAttestation::Valid), their_tls_key)` with a small deposit. This overwrites their stored `VerifiedAttestation` with `MockAttestation::Valid`. From that point:

1. They pass `is_caller_an_attested_participant` on every `respond`, `respond_ckd`, and `respond_verify_foreign_tx` call — without running in a TEE.
2. `reverify_and_cleanup_participants` never evicts them, so their participant status is permanent regardless of TEE hardware state.
3. The production safety invariant — that all active participants must be running in a cryptographically verified TEE — is broken for that participant.

A participant operating outside a TEE has their key share unprotected by hardware isolation, weakening the MPC network's security model. This breaks the participant-state and attestation-authorization invariant that the entire TEE-gated signing flow depends on.

**Classified as: Medium** — participant-state manipulation that breaks a production safety invariant (TEE attestation requirement) without requiring network-level DoS or operator misconfiguration.

---

### Likelihood Explanation

- **Reachable entry path**: Any existing participant can call `submit_participant_info` directly — it is a public, payable contract method.
- **No collusion required**: A single participant below the signing threshold can exploit this unilaterally.
- **No special tooling**: The `MockAttestation::Valid` variant is a standard Borsh-serializable enum value exposed in the public ABI.
- **Persistent**: Once submitted, the mock attestation survives all re-verification sweeps indefinitely.

---

### Recommendation

1. **Reject `MockAttestation` variants in production**: Add a guard in `add_participant` (or in `submit_participant_info`) that returns an error if the attestation is a `Mock` variant when the contract is not in a test/devnet mode. A compile-time feature flag (`#[cfg(feature = "mock-attestation")]`) or a runtime config flag would both work.
2. **Remove `MockAttestation` from the production ABI**: The `MockAttestation` enum is currently part of the public contract interface (`near-mpc-contract-interface`). It should be gated behind a non-default feature so it cannot be submitted on mainnet.
3. **Add a re-verification type check**: `reverify_and_cleanup_participants` should treat `VerifiedAttestation::Mock` as invalid in production, evicting any participant whose stored attestation is a mock.

---

    pub fn submit_participant_info(
        &mut self,
        proposed_participant_attestation: dtos::Attestation,
        tls_public_key: dtos::Ed25519PublicKey,
    ) -> Result<(), Error> {
        let proposed_participant_attestation =
            proposed_participant_attestation.try_into_contract_type()?;

        let account_key = env::signer_account_pk();
        let account_id = Self::assert_caller_is_signer();

        log!(
            "submit_participant_info: signer={}, proposed_participant_attestation={:?}, account_key={:?}",
            account_id,
            proposed_participant_attestation,
            account_key
        );

        // Save the initial storage usage to know how much to charge the proposer for the storage
        // used
        let initial_storage = env::storage_usage();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // The node always signs submissions with an Ed25519 key
        // (`near_signer_key`), so the signer key here is Ed25519 in practice.
        // Reject non-Ed25519 signer keys rather than silently storing a value
        // we could never match against in `is_caller_an_attested_participant`.
        let account_public_key = dtos::Ed25519PublicKey::try_from(&account_key).map_err(|_| {
            InvalidParameters::InvalidTeeRemoteAttestation {
                reason: "signer account key must be Ed25519".to_string(),
            }
        })?;

        // Add the participant information to the contract state
        let attestation_insertion_result = self
            .tee_state
            .add_participant(
                NodeId {
                    account_id: account_id.clone(),
                    tls_public_key,
                    account_public_key,
                },
                proposed_participant_attestation,
                tee_upgrade_deadline_duration,
            )
            .map_err(|err| {
                let reason = match &err {
                    AttestationSubmissionError::InvalidAttestation(_) => {
                        format!("TeeQuoteStatus is invalid: {err}")
                    }
                    AttestationSubmissionError::TlsKeyOwnedByOtherAccount => err.to_string(),
                };
                InvalidParameters::InvalidTeeRemoteAttestation { reason }
            })?;

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
