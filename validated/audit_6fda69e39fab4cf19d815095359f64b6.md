Audit Report

## Title
`isUpdater` Grants Persist Across `transferProviderOwnership`, Enabling Stale Updater to Manipulate Oracle Spread — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`, `PriceProviderFactoryL2.sol`, `AnchoredProviderFactory.sol`)

## Summary

All three factory contracts store `mapping(address provider => mapping(address updater => bool)) public isUpdater` that is never cleared in `transferProviderOwnership`. A previous provider owner can pre-grant updater rights to a controlled address, transfer ownership, and then use those retained rights to call `setConfidence` post-transfer — setting `confidenceParam` to `0` (DoS via stalled-feed sentinel) or `CONFIDENCE_MAX` (100× spread widening causing bad-price execution). The new owner has no enumeration path to discover or bulk-revoke stale grants.

## Finding Description

**Root cause — `isUpdater` not cleared on transfer:**

`PriceProviderFactory.transferProviderOwnership` (L92–102), `PriceProviderFactoryL2.transferProviderOwnership` (L95–105), and `AnchoredProviderFactory.transferProviderOwnership` (L230–240) all update `providerOwner` and `_providersByCreator` but never touch `isUpdater`:

```solidity
providerOwner[provider] = newOwner;
_providersByCreator[previousOwner].remove(provider);
_providersByCreator[newOwner].add(provider);
emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
```

`_requireUpdater` passes for any address with `isUpdater[provider][addr] == true`, regardless of whether that address is still affiliated with the current owner:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
```

**Exploit path — `setConfidence` → `setConfidenceParam` → corrupted `confidenceParam`:**

`setConfidence` (PriceProviderFactory L130–142) calls `_requireUpdater` then `PriceProvider.setConfidenceParam`. A stale updater passes `_requireUpdater` and sets `confidenceParam` to any value in `[0, CONFIDENCE_MAX]`.

**DoS path (`confidenceParam = 0`, `PriceProvider` with `marginStep < 0`):**

`_getBidAndAskPrice` computes:
```solidity
uint256 adjustedSpread = spread * confidenceParam;   // = 0
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread); // bid == ask == mid
```
With `marginStep < 0`, `stepBidFactor > stepAskFactor`, so after step adjustment `bidOut >= askOut`, triggering:
```solidity
if (bidOut >= askOut) return (0, type(uint128).max);
```
`getBidAndAskPrice()` then reverts with `FeedStalled`:
```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
```
Every pool swap against this provider reverts.

**Bad-price path (`confidenceParam = CONFIDENCE_MAX = 1_000_000`):**

`adjustedSpread = spread * 1_000_000` — the spread fed to `_getBidAskFrom` is 100× the oracle spread, producing a bid/ask pair far outside the fair market range. Swappers execute at severely distorted prices.

**`AnchoredPriceProvider` partial mitigation:** The band clamp in `_computeBidAsk` (`bidOut = Math.min(refBid, cBid)`, `askOut = Math.max(refAsk, cAsk)`) prevents the `confidenceParam = 0` DoS by falling back to the reference band. However, `confidenceParam = CONFIDENCE_MAX` still widens the shaped quote beyond the reference band, producing a wider-than-intended spread that reaches the pool.

**Non-enumerability of `isUpdater`:** Because `isUpdater` is a plain nested mapping (not an `EnumerableSet`), the new owner cannot iterate granted addresses. They can only revoke addresses they already know about. On-chain event logs (`UpdaterGranted`) are the only discovery path, requiring the new owner to be aware of this risk and actively query.

## Impact Explanation

- **Pool DoS (broken core functionality):** Stale updater sets `confidenceParam = 0` on a `PriceProvider` with `marginStep < 0`. Every subsequent `pool.swap()` call invokes `getBidAndAskPrice()`, which reverts with `FeedStalled`. Swap and liquidity flows are completely unusable until the new owner discovers and revokes the stale updater — a non-trivial remediation with no enumeration path.
- **Bad-price execution (direct principal loss):** Stale updater sets `confidenceParam = CONFIDENCE_MAX`. The spread becomes 100× the oracle spread. Swappers receive prices far outside the fair market range, suffering direct principal loss on every swap routed through this provider.

Both outcomes fall within the allowed impact gate: broken core pool functionality and bad-price execution.

## Likelihood Explanation

The trigger requires a previous owner who deliberately pre-grants updater rights before transferring. Provider creation is permissionless — any address can call `createPriceProvider`. The previous owner is not a trusted factory owner or oracle admin; after transfer they are an unprivileged actor who should have no remaining privileges. The `CONFIDENCE_COOLDOWN` of 1 minute limits update frequency but permits sustained manipulation (60 updates/hour). The new owner has no enumeration path, making discovery and remediation non-trivial.

## Recommendation

Introduce an `updaterEpoch` counter per provider and include it in the authorization check, so incrementing the epoch on transfer atomically invalidates all prior grants without requiring enumeration:

```solidity
mapping(address provider => uint256) public updaterEpoch;
mapping(address provider => mapping(uint256 epoch => mapping(address updater => bool))) public isUpdater;

function transferProviderOwnership(address provider, address newOwner) external ... {
    ...
    updaterEpoch[provider]++;   // invalidates all prior grants atomically
    providerOwner[provider] = newOwner;
    ...
}
```

Alternatively, store updaters in an `EnumerableSet.AddressSet` per provider so the transfer function can iterate and clear them.

## Proof of Concept

1. Alice calls `factory.createPriceProvider(...)` with `marginStep < 0` — Alice is `providerOwner[provider]`.
2. Alice calls `factory.grantUpdater(provider, alice_bot)` — `isUpdater[provider][alice_bot] = true`.
3. Alice calls `factory.transferProviderOwnership(provider, bob)` — `providerOwner[provider] = bob`, but `isUpdater[provider][alice_bot]` remains `true`.
4. Bob is unaware of `alice_bot`.
5. `alice_bot` calls `factory.setConfidence([provider], [0])`. `_requireUpdater` passes because `isUpdater[provider][alice_bot]` is still `true`. `PriceProvider.setConfidenceParam(0)` executes.
6. `confidenceParam` is now `0`. The next `pool.swap(...)` invokes `priceProvider.getBidAndAskPrice()`, which computes `adjustedSpread = 0`, applies step factors with `marginStep < 0` producing `bidOut >= askOut`, returns `(0, type(uint128).max)`, and reverts with `FeedStalled` — the pool is DoS'd.
7. Alternatively in step 5, `alice_bot` sets confidence to `CONFIDENCE_MAX = 1_000_000`, making `adjustedSpread` 100× the oracle spread. Swappers execute at severely distorted prices, losing principal on every trade.
