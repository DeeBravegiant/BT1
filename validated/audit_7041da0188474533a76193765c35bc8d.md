### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Commit Arbitrary Sequencer Address to Block Header and Execution Context — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address). `is_proposal_init_valid` validates height, L1 DA mode, L2 gas price, timestamp, starknet version, version constant commitment, L1 gas prices, and fee proposal — but **never validates `builder`**. Because `convert_to_sn_api_block_info` maps `init.builder` directly to `sequencer_address` in the `BlockInfo` passed to the batcher, a malicious proposer can set `builder` to any arbitrary address. Validators accept the proposal, execute all transactions with the wrong sequencer address, and commit a block header containing the attacker-controlled address. The wrong address is also hashed into the `PartialBlockHash` / `ProposalCommitment` that the entire committee votes on.

---

### Finding Description

**`ProposalInit.builder` is proposer-supplied and never validated.**

`ProposalInit` has two identity fields:

```
pub proposer: ContractAddress,   // who proposed in consensus
pub builder:  ContractAddress,   // who builds/sequences the block
``` [1](#0-0) 

When a legitimate node builds a proposal, `builder` is set from `self.config.static_config.builder_address`: [2](#0-1) 

`convert_to_sn_api_block_info` maps `init.builder` → `sequencer_address` in the `BlockInfo` that is forwarded to the batcher for both build and validate paths: [3](#0-2) 

`is_proposal_init_valid` checks height, L1 DA mode, L2 gas price, timestamp, starknet version, version constant commitment, L1 gas prices, and fee proposal. It never checks `builder`: [4](#0-3) 

The `sequencer_address` derived from `builder` is included in `PartialBlockHashComponents`, which feeds the `PartialBlockHash` / `ProposalCommitment` that validators vote on: [5](#0-4) [6](#0-5) 

Because both the proposer and every validator derive the `PartialBlockHash` from the same proposer-supplied `builder`, the `ProposalFinMismatch` check passes even when `builder` is attacker-controlled: [7](#0-6) 

After consensus, `finalize_decision` calls `update_state_sync_with_new_block`, which commits `init.builder` as the `sequencer` field of the block header: [8](#0-7) 

---

### Impact Explanation

1. **Wrong block header committed to storage.** The `sequencer_address` in the finalized `BlockHeaderWithoutHash` is the attacker-controlled address. Every RPC call to `starknet_getBlockWithTxHashes` / `starknet_getBlockWithTxs` returns the wrong `sequencer_address` for that block — an authoritative-looking wrong value.

2. **Wrong syscall result during execution.** All transactions in the block execute with `BlockInfo.sequencer_address = attacker_address`. Contracts calling `get_sequencer_address` (deprecated) or `get_block_info` (v2) receive the wrong address. Any contract logic that branches on the sequencer address (e.g., fee-bypass checks, sequencer-only gates) is subverted.

3. **Wrong block hash committed on-chain.** `sequencer_address` is a direct input to `calculate_block_hash`. The block hash committed to L1 and used as `previous_block_hash` for all subsequent blocks is computed with the wrong sequencer address, permanently corrupting the chain's hash chain from that block onward.

4. **Fee misdirection.** Transaction fees are credited to `sequencer_address`. With an attacker-controlled `builder`, all fees in the block are credited to the attacker's address rather than the legitimate sequencer.

---

### Likelihood Explanation

Any validator that wins a proposal slot (round-robin or weighted selection) can exploit this. No external access or special privilege beyond being a committee member is required. The attack is a single-field modification in the outbound `ProposalInit` message. No existing guard in `is_proposal_init_valid`, the consensus manager's `handle_proposal`, or the batcher catches it. [9](#0-8) 

The consensus manager checks `proposer` against the committee-derived expected proposer, but `builder` is never checked anywhere.

---

### Recommendation

**Short term:** Add a validation rule in `is_proposal_init_valid` that rejects any `ProposalInit` whose `builder` field does not match the locally-configured `builder_address` (or a committee-derived expected builder address). This mirrors the existing `proposer` check in `handle_proposal`.

**Long term:** Include `builder` in the `ProposalInitValidation` struct so the expected value is derived from authoritative local state (committee registry or node config) rather than trusting the proposer-supplied wire value. Add a test analogous to `invalid_proposal_init` that verifies a mismatched `builder` is rejected.

---

### Proof of Concept

1. Attacker is a legitimate validator and wins a proposal slot for height H.
2. Attacker constructs `ProposalInit` with `builder = attacker_wallet_address` (any address they control).
3. Attacker streams the proposal to the network. All other validators call `validate_proposal` → `is_proposal_init_valid`. The function checks height, L1 DA mode, L2 gas price, timestamp, starknet version, version constant commitment, L1 gas prices, and fee proposal. It returns `Ok(())` without inspecting `builder`.
4. `initiate_validation` calls `convert_to_sn_api_block_info(&init)`, producing a `BlockInfo` with `sequencer_address = attacker_wallet_address`. This is forwarded to the batcher via `validate_block`.
5. The batcher executes all transactions with `sequencer_address = attacker_wallet_address`. All fees are credited to the attacker's address.
6. `PartialBlockHashComponents::new(&block_info, ...)` sets `sequencer = attacker_wallet_address`. The resulting `PartialBlockHash` matches the proposer's claimed `ProposalCommitment` (both sides used the same `builder`). `ProposalFinMismatch` check passes.
7. Consensus reaches decision. `finalize_decision` commits `init.builder = attacker_wallet_address` as the `sequencer` field of the block header.
8. The block hash is computed with `attacker_wallet_address` as sequencer and committed to L1.
9. RPC returns `sequencer_address = attacker_wallet_address` for block H. Contracts that called `get_sequencer_address` during block H received `attacker_wallet_address`.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
}
