### Title
Unprivileged `TransferToGasKey` Action Can Permanently Block Gas Key and Account Deletion via `GasKeyBalanceTooHigh` Guard - (`runtime/runtime/src/access_keys.rs`)

### Summary

An unprivileged external account can send a `TransferToGasKey` action targeting any victim account's gas key, pushing the gas key balance above `GasKeyInfo::MAX_BALANCE_TO_BURN` (1 NEAR). Once above this threshold, both `DeleteKey` and `DeleteAccount` on that account fail with `GasKeyBalanceTooHigh`. Because `TransferToGasKey` carries no actor-permission check, any account can fund any gas key on any receiver account. The attacker can continuously re-fund the gas key to frontrun the victim's withdrawal+delete attempts, permanently blocking gas key removal and account deletion.

### Finding Description

`action_transfer_to_gas_key` in `runtime/runtime/src/access_keys.rs` increments the balance of a gas key on the receipt's `receiver_id` account without any ownership check:

```rust

    state_update: &mut TrieUpdate,
    result: &mut ActionResult,
    account_id: &AccountId,   // = receipt receiver_id
    action: &TransferToGasKeyAction,
) -> Result<(), RuntimeError> {
    ...
    gas_key_info.balance = gas_key_info.balance.checked_add(action.deposit)...;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);
    Ok(())

``` [1](#0-0) 

`check_actor_permissions` explicitly exempts `TransferToGasKey` from any actor-identity check:

```rust
Action::TransferToGasKey(_) => (),
``` [2](#0-1) 

This means any account can send a transaction with `receiver_id = victim.near` and `Action::TransferToGasKey { public_key: victim_gas_key_pk, deposit: X }` to inflate the victim's gas key balance.

`delete_gas_key` (called by `action_delete_key`) blocks deletion when the gas key balance exceeds `MAX_BALANCE_TO_BURN`:

```rust
if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
    result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh { ... }.into());
    return Ok(());
}
``` [3](#0-2) 

`action_delete_account` applies the same guard over the aggregate of all gas key balances:

```rust
let gas_key_balance_to_burn = compute_gas_key_balance_sum(state_update, account_id)?;
if gas_key_balance_to_burn > GasKeyInfo::MAX_BALANCE_TO_BURN {
    result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh { ... }.into());
    return Ok(());
}
``` [4](#0-3) 

`MAX_BALANCE_TO_BURN` is exactly 1 NEAR:

```rust
pub const MAX_BALANCE_TO_BURN: Balance = Balance::from_near(1);
``` [5](#0-4) 

The victim's only recourse is to call `WithdrawFromGasKey` (which does require `actor_id == account_id`) to drain the balance below 1 NEAR before deleting. However, because `WithdrawFromGasKey` and `DeleteKey` must be submitted as separate transactions or as a batch where the withdrawal amount must be specified exactly, an attacker can frontrun the victim's batch by adding a small deposit between the victim's signing and execution, causing the victim's withdrawal to leave a residual balance that still exceeds 1 NEAR, and the subsequent `DeleteKey` to fail again.

### Impact Explanation

The corrupted protocol value is the victim account's runtime state transition: `DeleteKey` and `DeleteAccount` actions that should succeed (the account owner has the right to delete their own keys and account) are permanently blocked at the runtime execution layer. The victim cannot remove a gas key from their account and cannot delete their account as long as the attacker keeps the gas key balance above 1 NEAR. This is a denial-of-service against account lifecycle management, directly analogous to M-03 where locking dust tokens blocked strategy migration. [6](#0-5) 

### Likelihood Explanation

The attack is reachable by any unprivileged account via a standard public RPC transaction. The attacker must spend NEAR (the deposited amount goes into the victim's gas key balance, which the victim can withdraw), making it economically costly to sustain. However, the cost to initiate the block is just over 1 NEAR (or less if the victim's gas key already has a balance close to the threshold). The attacker can repeatedly re-fund after each victim withdrawal to maintain the block. The attack requires no special privileges, no validator access, and no node-level capabilities. [1](#0-0) 

### Recommendation

Add an actor-permission check to `TransferToGasKey` in `check_actor_permissions` so that only the account owner (or an authorized actor) can fund their own gas keys, mirroring the restriction already applied to `WithdrawFromGasKey`. Alternatively, remove the `GasKeyBalanceTooHigh` guard on `DeleteKey`/`DeleteAccount` and instead burn the full balance unconditionally (accepting the economic loss), or provide a separate "force-drain and delete" path that is not gated on the balance threshold. [7](#0-6) 

### Proof of Concept

1. Victim `alice.near` has a gas key `gas_key_pk` with balance 0.5 NEAR.
2. Attacker `eve.near` submits a transaction: `signer_id=eve.near, receiver_id=alice.near, actions=[TransferToGasKey { public_key: gas_key_pk, deposit: 0.6 NEAR }]`.
3. The transaction is accepted (no permission check on `TransferToGasKey`). Alice's gas key balance is now 1.1 NEAR > `MAX_BALANCE_TO_BURN`.
4. Alice submits `DeleteKey { public_key: gas_key_pk }`. Runtime calls `delete_gas_key`, which checks `gas_key_info.balance (1.1 NEAR) > MAX_BALANCE_TO_BURN (1 NEAR)` → returns `GasKeyBalanceTooHigh`. Key deletion fails.
5. Alice submits `WithdrawFromGasKey { public_key: gas_key_pk, amount: 0.2 NEAR }` followed by `DeleteKey` in the same transaction batch. Eve frontruns by adding 0.2 NEAR + 1 yoctoNEAR. Alice's withdrawal succeeds (balance goes from 1.3 NEAR to 1.1 NEAR), but `DeleteKey` still fails because 1.1 NEAR > 1 NEAR.
6. Alice's `DeleteAccount` also fails via `compute_gas_key_balance_sum` returning > 1 NEAR. [3](#0-2) [4](#0-3)

fn delete_gas_key(
    config: &RuntimeConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
) -> Result<(), RuntimeError> {
    if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
        result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh {
            account_id: account_id.clone(),
            public_key: Some(Box::new(public_key.clone())),
            balance: gas_key_info.balance,
        }
        .into());
        return Ok(());
    }
    result.tokens_burnt =
        result.tokens_burnt.checked_add(gas_key_info.balance).ok_or(IntegerOverflowError)?;
    let num_nonces = gas_key_info.num_nonces as usize;
    for i in 0..gas_key_info.num_nonces {
        remove_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i);
    }
    let nonce_key_len = gas_key_nonce_key_len(account_id, &public_key.into());
    let nonce_remove_compute = storage_removes_compute(
        &config.wasm_config.ext_costs,
        num_nonces,
        nonce_key_len * num_nonces,
        AccessKey::NONCE_VALUE_LEN * num_nonces,
    );
    result.compute_usage = safe_add_compute(result.compute_usage, nonce_remove_compute)?;
    remove_access_key(state_update, account_id.clone(), public_key.clone());
    account.set_storage_usage(account.storage_usage().saturating_sub(gas_key_storage_cost(
        &config.fees,
        public_key,
        access_key,
        gas_key_info.num_nonces,
    )));
    Ok(())
}

pub(crate) fn action_transfer_to_gas_key(
    state_update: &mut TrieUpdate,
    result: &mut ActionResult,
    account_id: &AccountId,
    action: &TransferToGasKeyAction,
) -> Result<(), RuntimeError> {
    let Some(mut access_key) = get_access_key(state_update, account_id, &action.public_key)? else {
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };
    let Some(gas_key_info) = access_key.gas_key_info_mut() else {
        // Key exists but is not a gas key
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };

    gas_key_info.balance = gas_key_info.balance.checked_add(action.deposit).ok_or_else(|| {
        RuntimeError::StorageError(StorageError::StorageInconsistentState(
            "gas key balance integer overflow".to_string(),
        ))
    })?;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);
    Ok(())
}

pub(crate) fn check_actor_permissions(
    action: &Action,
    account: &Option<Account>,
    actor_id: &AccountId,
    account_id: &AccountId,
) -> Result<(), ActionError> {
    match action {
        Action::DeployContract(_)
        | Action::Stake(_)
        | Action::AddKey(_)
        | Action::DeleteKey(_)
        | Action::DeployGlobalContract(_)
        | Action::UseGlobalContract(_)
        | Action::WithdrawFromGasKey(_) => {
            if actor_id != account_id {
                return Err(ActionErrorKind::ActorNoPermission {
                    account_id: account_id.clone(),
                    actor_id: actor_id.clone(),
                }
                .into());
            }
        }
        Action::DeleteAccount(_) => {
            if actor_id != account_id {
                return Err(ActionErrorKind::ActorNoPermission {
                    account_id: account_id.clone(),
                    actor_id: actor_id.clone(),
                }
                .into());
            }
            let account = account.as_ref().unwrap();
            if !account.locked().is_zero() {
                return Err(ActionErrorKind::DeleteAccountStaking {
                    account_id: account_id.clone(),
                }
                .into());
            }
        }
        Action::CreateAccount(_)
        | Action::FunctionCall(_)
        | Action::Transfer(_)
        | Action::TransferToGasKey(_) => (),
        Action::Delegate(_) | Action::DelegateV2(_) => (),
        Action::DeterministicStateInit(_) => (),
    };
    Ok(())
