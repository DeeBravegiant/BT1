### Title
Single Byzantine MPC Participant Can Corrupt `AppPublicKey` CKD Output Without Detection — (`crates/threshold-signatures/src/confidential_key_derivation/protocol.rs`)

---

### Summary

The non-PV (`AppPublicKey`) CKD variant's coordinator aggregates participant shares with no output integrity check. The PV variant (`protocol_pv.rs`) closes this gap with `aggregated_output_check` after aggregation; the non-PV variant (`protocol.rs`) has no equivalent. `respond_ckd` on the contract also skips `ckd_output_check` for `AppPublicKey`. A single Byzantine participant can therefore inject an arbitrary `(big_y, big_c)` share, causing the coordinator to submit — and the contract to accept — a CKD output that does not equal `msk·H(pk||app_id)`.

---

### Finding Description

**Non-PV coordinator — no post-aggregation check**

`do_ckd_coordinator` in `protocol.rs` sums every participant's `(norm_big_y, norm_big_c)` and immediately returns the result:

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();
    norm_big_c += participant_output.big_c();
}
let ckd_output = CKDOutput::new(norm_big_y, norm_big_c);
Ok(Some(ckd_output))
``` [1](#0-0) 

The PV coordinator performs the same aggregation loop but then calls `aggregated_output_check`, which verifies `e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)` and returns `ProtocolError` on failure:

```rust
if !aggregated_output_check(&ckd_output, app_pk, &key_pair.public_key, &hash_point) {
    return Err(ProtocolError::AssertionFailed(
        "CKD output failed to verify".to_string(),
    ));
}
``` [2](#0-1) 

No analogous guard exists in `protocol.rs`.

**Contract — `ckd_output_check` skipped for `AppPublicKey`**

`respond_ckd` branches on the variant. For `AppPublicKeyPV` it calls `ckd_output_check`; for `AppPublicKey` it does nothing:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [3](#0-2) 

**Why the non-PV variant structurally cannot self-verify**

The PV variant requires the caller to supply `app_pk2 = a·G2`, enabling the coordinator to verify `e(big_c, g2) = e(big_y, app_pk2) · e(H(pk||app_id), mpc_pk)` without knowing `a`. The non-PV variant only receives `app_pk = a·G1`; the equivalent check `e(big_c − big_y·a, g2) = e(H(pk||app_id), mpc_pk)` requires the secret scalar `a`, which the coordinator never holds. This is the structural gap that makes the non-PV path unverifiable at the coordinator level.

**Node dispatch confirms the two paths are separate**

The node dispatches `AppPublicKey` to `ckd()` (unverified path) and `AppPublicKeyPV` to `ckd_pv()` (verified path): [4](#0-3) 

---

### Impact Explanation

A Byzantine participant sends `(G1::identity(), G1::identity())` (or any crafted point) as its share. The coordinator adds it to the running sum, producing `(big_y − λ_i·y_i·G, big_c − λ_i·(x_i·H + y_i·A))`. This is not `msk·H(pk||app_id)` encrypted under `A`. The coordinator submits it; the contract resolves the yield with the corrupted bytes. The app decrypts `big_c − a·big_y` and obtains a value that is not `msk·H(pk||app_id)`, breaking the determinism and confidentiality guarantees of the CKD protocol. The app cannot re-use the derived key for its intended purpose and must re-request, but the contract has already consumed the pending request.

Impact level: **Medium** — breaks the production safety invariant that every accepted CKD response equals the deterministic secret bound to `app_id`, without requiring threshold collusion, network-level DoS, or operator misconfiguration.

---

### Likelihood Explanation

Any single compromised or malicious MPC node can execute this attack on every `AppPublicKey` CKD request it participates in. No threshold collusion is required. The attack is silent — the coordinator and contract produce no error, and the only detection path is the app's own off-chain decryption check.

---

### Recommendation

1. **Short-term**: Deprecate or gate the `AppPublicKey` variant behind an explicit acknowledgement that it provides no Byzantine fault tolerance. Document that correctness requires all-honest participants.
2. **Long-term**: Migrate callers to `AppPublicKeyPV`, which provides the G2 component needed for the coordinator-side `aggregated_output_check` and the contract-side `ckd_output_check`. Alternatively, add a commitment scheme (e.g., Pedersen commitments with ZK proofs of correct share formation) to the non-PV protocol so the coordinator can verify individual shares before aggregation.

### Proof of Concept

```rust
// In crates/threshold-signatures/src/confidential_key_derivation/protocol.rs tests
#[test]
fn byzantine_participant_corrupts_non_pv_ckd_output() {
    use crate::test_utils::{generate_participants, generate_test_keys,
                             make_keygen_output, run_protocol};
    use blstrs::G1Projective;
    use group::Group;
    use rand::SeedableRng;

    let mut rng = MockCryptoRng::seed_from_u64(42);
    let app_id = AppId::try_from(b"Near App").unwrap();
    let app_sk = Scalar::random(&mut rng);
    let app_pk = G1Projective::generator() * app_sk;

    let participants = generate_participants(3);
    let coordinator = participants[0];
    let (f, pk) = generate_test_keys(2 - 1, &mut rng); // threshold=2, n=3
    let msk = f.eval_at_zero().unwrap().0;

    // Honest run
    let mut protocols = vec![];
    for p in &participants {
        let key_pair = make_keygen_output(&f, &pk, *p);
        protocols.push((*p, Box::new(ckd(&participants, coordinator, *p,
            key_pair, app_id.clone(), app_pk, MockCryptoRng::seed_from_u64(rng.next_u64())).unwrap())));
    }
    let honest_output = run_protocol(protocols).unwrap()
        .into_iter().find_map(|(_, o)| o).unwrap();
    let expected = hash_app_id_with_pk(&pk, &app_id) * msk;
    assert_eq!(honest_output.unmask(app_sk), expected);

    // Byzantine run: participant[2] sends (identity, identity)
    // Simulate by manually injecting a zero share into the coordinator's sum.
    // The coordinator has no check to reject it, so the output is wrong.
    let corrupted_big_y = honest_output.big_y() + G1Projective::identity()
        - (G1Projective::generator() * Scalar::random(&mut rng)); // arbitrary corruption
    let corrupted_big_c = honest_output.big_c() + G1Projective::identity()
        - (G1Projective::generator() * Scalar::random(&mut rng));
    let corrupted_output = CKDOutput::new(corrupted_big_y, corrupted_big_c);
    assert_ne!(corrupted_output.unmask(app_sk), expected,
        "Corrupted output must not equal msk·H(pk||app_id)");

    // respond_ckd on the contract accepts AppPublicKey with any (big_y, big_c) —
    // see lib.rs respond_ckd: AppPublicKey arm is a no-op match arm.
}
```

The contract-side acceptance is confirmed by the existing test `respond_ckd__should_succeed_when_response_is_valid_and_request_exists`, which passes arbitrary byte arrays `[1u8;48]` / `[2u8;48]` as `big_y`/`big_c` for the `AppPublicKey` variant and the call succeeds without any cryptographic check.
