### Title
Stale TEE Attestation Accepted for `respond*` After Image Hash Revocation — (File: `crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` checks only for the **existence** of a stored attestation in `stored_attestations`, not its **current validity** against the on-chain allowed image hash list. After a new MPC image hash is voted in and the old one is removed from the allowed list by `verify_tee`, a node running the old (potentially compromised) image can continue to call `respond`, `respond_ckd`, and `respond_verify_foreign_tx` — and have those calls accepted — until `clean_invalid_attestations` physically removes the stale attestation entry. This is a direct analog to the "deactivated passport still accepted offline" class: the contract validates the credential's existence in local storage rather than checking it against the authoritative revocation source (the current allowed-hash list).

---

### Finding Description

**Root cause — `is_caller_an_attested_participant` never calls `re_verify`**

`is_caller_an_attested_participant` (`crates/contract/src/tee/tee_state.rs`, lines 469–498) performs three checks:

1. Is the signer in the current participant set?
2. Does `stored_attestations` contain an entry keyed by the participant's TLS public key?
3. Do the `account_id` and `account_public_key` fields of that entry match the signer?

It does **not** call `VerifiedAttestation::re_verify` (`crates/mpc-attestation/src/attestation.rs`, lines 215–255), which would check whether the stored `mpc_image_hash` is still present in the current `allowed_mpc_docker_image_hashes` list.

**The revocation path is lazy and asynchronous**

When `verify_tee` is called, it removes the expired image hash from the allowed list and spawns `clean_invalid_attestations` to prune stale `stored_attestations` entries. Separately, after every successful `vote_reshared`, `clean_invalid_attestations` is spawned as a **detached** promise (`crates/contract/src/lib.rs`, lines 1195–1205). A detached promise that runs out of gas or panics is silently dropped — the host `vote_reshared` transaction still succeeds.

```
// vote_reshared spawns cleanup as fire-and-forget
Promise::new(env::current_account_id())
    .function_call(
        method_names::CLEAN_INVALID_ATTESTATIONS.to_string(),
        ...
    )
    .detach();   // ← failure here is silent
```

**The window**

Between the moment `verify_tee` removes an image hash from the allowed list and the moment `clean_invalid_attestations` physically deletes the corresponding `stored_attestations` entry, the node's attestation still exists in storage. Because `is_caller_an_attested_participant` only checks existence, the node passes the gate and can call `respond*` with a cryptographically valid signature produced by its (now-revoked) key share.

**Concrete call path**

```
respond() / respond_ckd() / respond_verify_foreign_tx()
  └─ assert_caller_is_attested_participant_and_protocol_active()
       └─ tee_state.is_caller_an_attested_participant(participants)
            ├─ participants.info(&signer_id)          ✓ still a participant
            ├─ stored_attestations.get(&tls_pk)       ✓ entry still present
            ├─ account_id match                       ✓
            └─ account_public_key match               ✓
            // re_verify() is NEVER called here
```

---

### Impact Explanation

**High — participant/attestation authorization bypass.**

A node whose TEE image hash has been revoked (e.g., because the image was found to be compromised and governance voted in a replacement) can continue to:

- Participate in the off-chain threshold-signing protocol using its existing key share.
- Submit `respond*` calls that the contract accepts, because the stale attestation passes `is_caller_an_attested_participant`.

During this window the node is operating under a credential the governance system has explicitly invalidated. If the image was rotated because it was compromised, the node may be running adversarial code that leaks its signing share to an attacker. A single leaked share below the reconstruction threshold does not immediately break security, but it reduces the effective security margin of the threshold scheme and, combined with other shares obtained through the same window, could enable key recovery or unauthorized signature production.

---

### Likelihood Explanation

**Low–Medium.**

The preconditions are:

1. Governance reaches threshold to vote in a new image hash (`vote_code_hash`).
2. The 7-day upgrade deadline elapses without the node resubmitting a fresh attestation.
3. `verify_tee` removes the old hash from the allowed list.
4. `clean_invalid_attestations` has not yet run (detached promise failed or `max_scan` was too small to reach this entry), leaving the stale attestation in `stored_attestations`.
5. The node continues to call `respond*` during this window.

Steps 1–3 are normal operational events (image upgrades happen). Step 4 is the exploitable gap: detached promises can fail silently, and `clean_invalid_attestations` is bounded by `max_scan` (`RESHARE_CLEAN_INVALID_ATTESTATIONS_MAX_SCAN`), so a large `stored_attestations` map may not be fully swept in one call. Step 5 requires the node operator to be adversarial or the node to be compromised.

---

### Recommendation

Add an inline `re_verify` call inside `is_caller_an_attested_participant` so that the stored attestation is validated against the current allowed-hash lists at every `respond*` invocation — analogous to checking the "police database" at the point of use rather than relying solely on the cached local record:

```rust
// After confirming the attestation entry exists:
let tee_upgrade_deadline = ...; // from config
attestation.verified_attestation.re_verify(
    Self::current_time_seconds(),
    &self.get_allowed_mpc_docker_image_hashes(tee_upgrade_deadline),
    &self.get_allowed_launcher_compose_hashes(),
    &self.get_accepted_measurements(),
).map_err(|_| AttestationCheckError::AttestationExpiredOrRevoked)?;
```

This eliminates the window between hash revocation and physical attestation removal, ensuring that a node whose image hash has been invalidated is immediately rejected at the contract level without waiting for the next `verify_tee` / `clean_invalid_attestations` sweep.

### Proof of Concept

1. Network has participants `{P1, P2, P3}`, threshold 2, image hash `H1`.
2. Governance votes in new image hash `H2` (threshold reached).
3. `P1` does not upgrade; it does not call `submit_participant_info` with a `H2`-based attestation.
4. After 7 days, `verify_tee` is called: `H1` is removed from `allowed_mpc_docker_image_hashes`. The detached `clean_invalid_attestations` call runs out of gas (or `max_scan` is too small) and silently fails. `P1`'s entry remains in `stored_attestations`.
5. A user submits a `sign` request. `P1` participates in the off-chain Cait-Sith signing protocol using its key share (derived under `H1`-era code).
6. `P1` calls `respond(request, signature)`. The contract executes `assert_caller_is_attested_participant_and_protocol_active()`:
   - `P1` is still in the participant set → passes.
   - `stored_attestations` still contains `P1`'s entry → passes.
   - Account ID and key match → passes.
   - `re_verify` is never called → the revoked `H1` attestation is not detected.
7. The contract accepts the response and resolves the pending yield. `P1` — running potentially compromised code — has successfully participated in and completed a threshold signing operation after its attestation was revoked. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

    /// launcher-hash / measurement whitelists, or whose attestation has expired.
    /// Returns the number of entries removed.
    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())
            }
            Self::Mock(mock_attestation) => mock_attestation.verify_constraints(
                timestamp_seconds,
                allowed_mpc_docker_image_hashes,
                allowed_launcher_docker_compose_hashes,
                allowed_measurements,
            ),
        }
    }

    #[handle_result]
    pub fn respond(
        &mut self,
        request: SignatureRequest,
        response: dtos::SignatureResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!("respond: signer={}, request={:?}", &signer, &request);

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain)?;

        let signature_is_valid = match (&response, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                // generate the expected public key
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");
                let affine = *k256::PublicKey::try_from(&secp_pk)
                    .expect("stored key is always valid")
                    .as_affine();
                let expected_public_key =
                    derive_key_secp256k1(&affine, &request.tweak).map_err(RespondError::from)?;

                let payload_hash = request.payload.as_ecdsa().expect("Payload is not ECDSA");

                // Check the signature is correct
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    payload_hash,
                    &expected_public_key,
                )
                .is_ok()
            }
            (
                dtos::SignatureResponse::Ed25519 { signature },
                PublicKeyExtended::Ed25519 {
                    edwards_point: public_key_edwards_point,
                    ..
                },
            ) => {
                let derived_public_key_edwards_point = derive_public_key_edwards_point_ed25519(
                    &public_key_edwards_point,
                    &request.tweak,
                );
                let derived_public_key_32_bytes =
                    dtos::Ed25519PublicKey::from(derived_public_key_edwards_point.compress());

                let message = request.payload.as_eddsa().expect("Payload is not EdDSA");

                near_mpc_signature_verifier::verify_eddsa_signature(
                    signature,
                    message,
                    &derived_public_key_32_bytes,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_signature_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }
