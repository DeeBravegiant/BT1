Audit Report

## Title
`AnchoredPriceProvider._isStale` Unconditionally Rejects Future `refTime`, Breaking All Swaps on L2 — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary

`AnchoredPriceProvider`, explicitly described as "the one standard provider for public pools," contains an L1-only `_isStale` that treats any oracle `refTime > block.timestamp` as stale. The protocol deploys on L2 chains and already built `FUTURE_TOLERANCE`-aware variants for other providers, but no such variant exists for `AnchoredPriceProvider`. When sequencer clock skew causes oracle `refTime` to exceed `block.timestamp`, every swap through any pool using this provider reverts.

## Finding Description

`AnchoredPriceProvider._isStale` unconditionally returns `true` when `refTime > nowTs`: [1](#0-0) 

`_readLeg` calls `_isStale` and returns `ok = false` on any future `refTime`: [2](#0-1) 

`_getBidAndAskPrice` returns the stall sentinel `(0, type(uint128).max)` when `ok = false`: [3](#0-2) 

`getBidAndAskPrice` reverts `FeedStalled()` on that sentinel: [4](#0-3) 

`MetricOmmPool._getBidAndAskPriceX64` catches and re-reverts as `PriceProviderFailed`, blocking every swap: [5](#0-4) 

The L2-aware `PriceProviderL2._isStale` already handles this with a `futureTol` parameter: [6](#0-5) 

The contracts directory contains `PriceProviderL2.sol`, `ProtectedPriceProviderL2.sol`, and `PriceProviderFactoryL2.sol`, but no `AnchoredPriceProviderL2.sol`:



## Impact Explanation

`AnchoredPriceProvider` is the designated standard provider for public pools (NatSpec: "the one standard provider for public pools"). When oracle `refTime` exceeds `block.timestamp` on L2, all swaps through every pool using this provider revert with `PriceProviderFailed`. This matches the "broken core pool functionality causing unusable swap flows" impact gate. No funds are directly stolen, but the primary trading path for the entire protocol on L2 is non-functional during affected windows.

## Likelihood Explanation

The protocol explicitly targets L2 chains (Base, HyperEVM), evidenced by the existence of `PriceProviderL2.sol`, `ProtectedPriceProviderL2.sol`, and `PriceProviderFactoryL2.sol`. Pyth and Chainlink publish prices with timestamps that routinely lead the sequencer's `block.timestamp` by seconds. The protocol already acknowledged this by building `FUTURE_TOLERANCE` into the other L2 variants. The omission in `AnchoredPriceProvider` is an oversight that triggers on any L2 deployment whenever oracle timestamp leads block timestamp — a routine, unprivileged, externally-caused condition requiring no attacker action.

## Recommendation

Add a `FUTURE_TOLERANCE` immutable to `AnchoredPriceProvider` (bounded to `<= 1 hours` as in `PriceProviderL2`) and replace the three-argument `_isStale` with the L2-aware four-argument variant:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Pass `FUTURE_TOLERANCE` into `_readLeg` and `_isStale` calls. Alternatively, deploy a dedicated `AnchoredPriceProviderL2` variant and a corresponding L2 factory, mirroring the pattern already used for `PriceProviderL2` and `ProtectedPriceProviderL2`.

## Proof of Concept

1. Deploy `AnchoredPriceProvider` on Base pointing to a Pyth feed via `AnchoredProviderFactory.createAnchoredProvider`.
2. Pyth publishes a price update with `publishTime = block.timestamp + 2` (sequencer clock 2 s behind oracle).
3. Trader calls `swap` on a pool using this provider.
4. Pool calls `getBidAndAskPrice()` → `_getBidAndAskPrice()` → `_readLeg(baseFeedId)`.
5. `_readLeg` calls `_isStale(block.timestamp + 2, block.timestamp, MAX_REF_STALENESS)`.
6. `refTime > nowTs` → returns `true` → `ok = false`.
7. `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
8. `getBidAndAskPrice` reverts `FeedStalled()`.
9. `MetricOmmPool._getBidAndAskPriceX64` catches and reverts `PriceProviderFailed(...)`.
10. All swaps are blocked until the sequencer's clock catches up.
