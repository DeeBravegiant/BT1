Audit Report

## Title
`RSETH._transfer` Missing `whenNotPaused` Guard Allows Transfers During Emergency Pause - (File: contracts/RSETH.sol)

## Summary
`RSETH` inherits `PausableUpgradeable` but its `_transfer` override at line 287 carries no `whenNotPaused` modifier, while `mint()` and `burnFrom()` do. Any rsETH holder can call the standard ERC20 `transfer()` or `transferFrom()` freely even when the contract is paused, meaning the pause does not halt token movement as the contract design intends.

## Finding Description
`RSETH` is declared with separate `ERC20Upgradeable` and `PausableUpgradeable` inheritance (line 13), not `ERC20PausableUpgradeable`. The `_transfer` override at lines 287–291 only enforces the address-block check via `_enforceNotBlocked` and then delegates to `super._transfer`. No `whenNotPaused` guard is present. `ERC20Upgradeable._transfer` calls `_beforeTokenTransfer` (line 231 of `ERC20Upgradeable.sol`), but because `_beforeTokenTransfer` is never overridden to check `!paused()`, the paused state is invisible to the transfer path. By contrast, `mint()` (line 235) and `burnFrom()` (line 245) both carry `whenNotPaused`, creating an asymmetry: minting and burning are halted by a pause, but peer-to-peer transfers are not.

Exploit path:
1. PAUSER_ROLE calls `pause()` — `paused()` returns `true`.
2. Any rsETH holder calls `rsETH.transfer(recipient, amount)`.
3. The call reaches `_transfer`, passes `_enforceNotBlocked`, and succeeds via `super._transfer`.
4. Tokens move freely despite the paused state.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The pause mechanism is documented and coded to halt all rsETH operations in an emergency. It succeeds for `mint` and `burnFrom` but silently fails for the primary transfer path. No user funds are lost or permanently frozen; balances are conserved and merely redistributed between addresses. The higher-severity scenario described in the submission (attacker moving tokens obtained via a compromised minter) requires privileged-address compromise, which is explicitly excluded by SECURITY.md. Without that precondition, the standalone impact is that the contract does not deliver its promised emergency-stop behavior for transfers — a Low-severity deviation from specification.

## Likelihood Explanation
Triggerable by any rsETH holder with no special privileges, directly via the standard ERC20 `transfer()` or `transferFrom()` interface, at any time the contract is paused. No setup beyond holding rsETH is required.

## Recommendation
Add `whenNotPaused` to the `_transfer` override:

```diff
-function _transfer(address from, address to, uint256 amount) internal override {
+function _transfer(address from, address to, uint256 amount) internal override whenNotPaused {
```

Alternatively, replace the separate `PausableUpgradeable` inheritance with `ERC20PausableUpgradeable`, which enforces `!paused()` in `_beforeTokenTransfer` for all transfer, mint, and burn paths automatically.

## Proof of Concept
```solidity
// 1. Deploy RSETH, mint tokens to alice.
// 2. PAUSER_ROLE calls rsETH.pause().
// 3. assert(rsETH.paused() == true);
// 4. vm.prank(alice);

// 5. assert(rsETH.balanceOf(bob) == amount);
// 6. vm.prank(minter);
//    rsETH.mint(alice, amount);     // reverts with "Pausable: paused"
// Confirms: transfer path is unguarded; mint/burn paths are correctly guarded.
```
