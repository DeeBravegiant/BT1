### Title
ECDSA-2PC `detach_private_scalar` Leaks Paillier Private Key (p, q) in "Public" Blob — (`File: src/cbmpc/api/ecdsa2pc.cpp`)

### Summary
`coinbase::api::ecdsa_2p::detach_private_scalar` is documented to produce a "scalar-removed" blob suitable for backup workflows. For P1, the blob's `paillier` field is copied verbatim into the output, including the Paillier private factors `p` and `q`. Any party that obtains this blob can reconstruct the Paillier private key and decrypt `c_key` to recover P1's EC key share `x1`, breaking the 2-of-2 threshold.

### Finding Description

The ECDSA-2PC key blob for P1 is defined as:

```cpp

  uint32_t version, role, curve;
  buf_t Q_compressed;
  coinbase::crypto::bn_t x_share;
  coinbase::crypto::bn_t c_key;
  coinbase::crypto::paillier_t paillier;   // ← contains p, q when role==0 (P1)
  void convert(coinbase::converter_t& c) { c.convert(version, role, curve, Q_compressed, x_share, c_key, paillier); }
};

`detach_private_scalar` zeroes out `x_share` with a sentinel value but copies the full `paillier` object — including `p` and `q` — into the output "public" blob:

```cpp

  ...
  key_blob_v1_t pub;
  pub.role      = static_cast<uint32_t>(key.role);
  pub.curve     = static_cast<uint32_t>(cid);
  pub.Q_compressed = key.Q.to_compressed_bin();
  pub.x_share   = key.paillier.get_N().value();  // sentinel: out-of-range
  pub.c_key     = key.c_key;
  pub.paillier  = key.paillier;   // ← p, q still present!
  out_public_key_blob = coinbase::convert(pub);
  return SUCCESS;
}

The `attach_private_scalar` function confirms this is structural: it explicitly checks that P1's "public" blob still carries the Paillier private key and uses it to verify the restored scalar:

The public API header describes the output as "a key blob with its private scalar removed" and does not warn that the Paillier private key remains: [5](#0-4) 

By contrast, the other signing APIs (`schnorr_2p`, `schnorr_mp`, `ecdsa_mp`, `eddsa_2p`, `eddsa_mp`) all explicitly state "The public blob is **safe to persist as public-only material**". The ECDSA-2PC header omits this phrase, but the function name `detach_private_scalar` and the description "private scalar removed" create a strong expectation that the output blob is safe to treat as non-sensitive. [6](#0-5) 

### Impact Explanation

An attacker who obtains `out_public_key_blob` from P1's `detach_private_scalar` call can:

1. Deserialize the blob to extract `p` and `q` (the Paillier private factors).
2. Reconstruct the Paillier private key and call `paillier.decrypt(c_key)` to recover `x1` — P1's EC key share.
3. Combined with their own `x2` (if the attacker is P2, or has compromised P2), compute the full ECDSA private key `x = x1 + x2 mod q` and sign arbitrary messages without P1's participation.

This is a single-party compromise of the 2-of-2 threshold: P2 alone, by obtaining the "public" blob, recovers the full signing key.

### Likelihood Explanation

The `detach_private_scalar` API is the documented path for PVE-based backup workflows. A developer following the backup pattern would:
- Call `detach_private_scalar` to separate the scalar for HSM/PVE storage.
- Store `out_public_key_blob` with reduced protection (e.g., in a database, logged, or sent to a backup coordinator), reasoning that the private scalar has been removed.

The function name and description provide no indication that the blob still contains 2048-bit Paillier primes. The omission of the "safe to persist as public-only material" phrase present in all other `detach_private_scalar` variants is easy to miss.

### Recommendation

1. **Strip the Paillier private key from the output blob.** In `detach_private_scalar`, replace `pub.paillier = key.paillier` with a public-only copy: call `paillier_t::create_pub(key.paillier.get_N().value())` to build a public-only Paillier object and assign that instead.

2. **Update `attach_private_scalar` accordingly.** Since the restored blob will no longer carry `p`/`q`, the Paillier-decrypt consistency check (`paillier.decrypt(c_key) == x_share`) must be removed or replaced with a public-key-only check (e.g., re-encrypt `x_share` and compare ciphertexts, or rely solely on the EC point check `x_mod_q * G == Qi_self`).

3. **Add an explicit documentation warning** to the ECDSA-2PC `detach_private_scalar` header that the current output blob retains the Paillier private key, until the fix is shipped.

error_t detach_private_scalar(mem_t key_blob, buf_t& out_public_key_blob, buf_t& out_private_scalar) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(key_blob, "key_blob",
                                                                          coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))
    return rv;
  coinbase::mpc::ecdsa2pc::key_t key;
  error_t rv = deserialize_key_blob(key_blob, key);
  if (rv) return rv;

  curve_id cid;
  if (!from_internal_curve(key.curve, cid)) return coinbase::error(E_BADARG, "unsupported curve");
  if (cid == curve_id::ed25519) return coinbase::error(E_BADARG, "unsupported curve");

  // Variable-length big-endian encoding (may grow after refresh).
  out_private_scalar = key.x_share.to_bin();

  // Produce a v1-format blob with an invalid (out-of-range) scalar share so it is
  // rejected by sign/refresh APIs.
  key_blob_v1_t pub;
  pub.role = static_cast<uint32_t>(key.role);
  pub.curve = static_cast<uint32_t>(cid);
  pub.Q_compressed = key.Q.to_compressed_bin();
  pub.x_share = key.paillier.get_N().value();  // x_share == N is out of range
  pub.c_key = key.c_key;
  pub.paillier = key.paillier;
  out_public_key_blob = coinbase::convert(pub);
  return SUCCESS;
}
