Audit Report

## Title
Unbounded Nested Gas Consumption in `updateRSETHPrice()` Renders rsETH Price Update Uncallable at Scale - (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function that drives rsETH pricing. Its internal call chain performs nested iteration over supported assets, node delegators, and EigenLayer queued withdrawals. As the protocol scales or EigenLayer operators force undelegations, the cumulative gas cost can exceed the block gas limit, permanently preventing price updates and causing stale pricing across all deposit and withdrawal operations.

## Finding Description
The call chain from `updateRSETHPrice()` creates deeply nested, unbounded iteration:

**Layer 1** — `LRTOracle._getTotalEthInProtocol()` loops over every supported asset and calls `getTotalAssetDeposits(asset)` for each: [1](#0-0) 

**Layer 2** — `LRTDepositPool.getAssetDistributionData()` loops over every NDC in `nodeDelegatorQueue` and calls `getAssetUnstaking(asset)` per NDC: [2](#0-1) 

The ETH path (`getETHDistributionData`) adds a second full NDC traversal calling `getAssetUnstaking(ETH_TOKEN)` per NDC: [3](#0-2) 

**Layer 3** — `NodeDelegator.getAssetUnstaking()` fetches the full EigenLayer queued withdrawal list and iterates over every withdrawal and every strategy within it, making external calls to `sharesToUnderlyingView()`: [4](#0-3) 

The total iteration count is `O(assets × NDCs × queued_withdrawals × strategies_per_withdrawal)`.

The critical weakness is that `maxUncompletedWithdrawalCount` (capped at 80 by `setMaxUncompletedWithdrawalCount`) is a **protocol-side soft counter**. EigenLayer's `getQueuedWithdrawals` returns the actual on-chain queue. When an EigenLayer operator forces an undelegation, new withdrawal roots are created on EigenLayer directly, bypassing the protocol's counter check in `undelegate()`. The protocol's own comment in `LRTUnstakingVault` explicitly acknowledges this: [5](#0-4) 

`updateRSETHPrice()` has no access control — it is callable by any address: [6](#0-5) 

`updateRSETHPriceAsManager()` shares the identical internal call chain and would also revert: [7](#0-6) 

## Impact Explanation
**Medium. Unbounded gas consumption.** If `updateRSETHPrice()` reverts due to gas exhaustion, the stored `rsETHPrice` becomes permanently stale. All deposits via `depositETH`/`depositAsset` mint rsETH at the stale rate, and all withdrawals via `LRTWithdrawalManager.initiateWithdrawal` compute `expectedAssetAmount` using the stale price. Both the public and manager-only price update paths share the same internal call chain and would both be blocked.

## Likelihood Explanation
The protocol is actively growing its NDC count and queued withdrawal count. The comment in `LRTUnstakingVault` explicitly acknowledges that 120 uncompleted withdrawals would break `updateRSETHPrice()`, and the cap is set at 80 to leave a buffer of 15 for forced undelegations (`ndc count × asset count = 15`). EigenLayer operators can force undelegations at any time, pushing the actual on-chain queue above the soft cap without any protocol-side check being able to prevent it. This is a realistic operational scenario explicitly anticipated by the protocol developers.

## Recommendation
1. **Bound `getAssetUnstaking` iteration**: Store the queued withdrawal amount as a running total updated on `initiateUnstaking` / `completeUnstaking`, eliminating the live EigenLayer traversal from the price-update hot path.
2. **Decouple asset accounting from price updates**: Cache per-asset TVL snapshots updated lazily or via separate operator calls, rather than recomputing the full sum on every `updateRSETHPrice()` call.
3. **Single NDC traversal**: Avoid calling `getAssetDistributionData` (which re-traverses all NDCs) once per supported asset; compute the NDC traversal once and accumulate all asset balances in a single pass.

**Foundry fork test plan**: Deploy against a mainnet fork. Configure 10 NDCs and 10 supported assets. Use `vm.mockCall` on EigenLayer's `DelegationManager.getQueuedWithdrawals` to return 80+ withdrawal structs per NDC, each with 3 strategies. Call `updateRSETHPrice()` with a gas limit matching the current Ethereum block gas limit (~30M). Assert the call reverts with out-of-gas. Alternatively, simulate forced undelegations by directly writing to EigenLayer storage to push the actual queue above the protocol's soft cap, then call `updateRSETHPrice()`.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
