### Title
Sub-Transaction Reference Script Fee Component Omitted from Minimum Fee Validation in Dijkstra Era - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era introduces nested ("sub") transactions embedded inside a top-level batch transaction. The minimum fee check for a Dijkstra batch transaction correctly charges the per-byte size fee for the entire serialized batch (including sub-transactions), but the **reference script fee component** is computed using only the top-level transaction's inputs. Reference scripts referenced exclusively through sub-transaction inputs are not charged the `minFeeRefScriptCostPerByte` tiered fee. An attacker can craft a batch transaction whose sub-transactions reference large Plutus scripts, paying only the base size fee while forcing validators to deserialize and process those scripts at no additional cost — the same DDoS vector that was exploited on Cardano mainnet in June 2024.

---

### Finding Description

The Dijkstra era's `EraUTxO` instance sets:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` is defined as:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

`txNonDistinctRefScriptsSize` only inspects the **top-level** transaction's `inputsTxBodyL` and `referenceInputsTxBodyL`:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

The Dijkstra codebase already defines `batchNonDistinctRefScriptsSize`, which correctly aggregates reference script sizes across the top-level transaction **and all sub-transactions**:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum (foldMap' (Sum . txNonDistinctRefScriptsSize utxo)
        (tx ^. bodyTxL . subTransactionsTxBodyL))
``` [4](#0-3) 

This function is exported but **never used** in the fee validation path. The UTXO transition rule calls:

```haskell
{- minfee pp txTop utxo₀ ≤ txfee txb -}
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
``` [5](#0-4) 

`validateFeeTooSmallUTxO` calls `getMinFeeTxUtxo pp tx utxo`, which resolves to `getConwayMinFeeTxUtxo` → `txNonDistinctRefScriptsSize` (top-level only). [6](#0-5) 

The `getConwayMinFeeTx` function (used by both Conway and Dijkstra via `getMinFeeTx`) adds the reference script fee as a separate component on top of the base size fee:

```haskell
getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee
  where
    refScriptsFee = tierRefScriptFee ... refScriptsSize
``` [7](#0-6) 

The base size fee (`alonzoMinFeeTx`) uses `sizeTxF`, which for Dijkstra serializes the entire batch including sub-transactions — so the per-byte component is correct. Only the reference script fee component is missing for sub-transactions. [8](#0-7) 

---

### Impact Explanation

An attacker submits a Dijkstra batch transaction whose sub-transactions include `referenceInputsTxBodyL` pointing to UTxO entries containing large Plutus scripts. The top-level transaction body carries no reference inputs. The fee check passes because `txNonDistinctRefScriptsSize` sees zero reference script bytes at the top level. Validators must nonetheless deserialize and process all reference scripts from sub-transaction inputs during script collection and execution. The attacker pays only the base per-byte fee for the serialized batch, not the tiered `minFeeRefScriptCostPerByte` surcharge that was specifically introduced to price this deserialization work. This reopens the reference-script DDoS vector (the June 2024 mainnet attack) for Dijkstra-era transactions.

**Impact category**: Medium — attacker-controlled transactions modify fees outside design parameters (reference script fee is bypassed for sub-transaction reference inputs).

---

### Likelihood Explanation

The attack requires only the ability to submit a valid Dijkstra batch transaction — no privileged access, no governance majority, no key compromise. The attacker needs to:
1. Create UTxO entries containing large Plutus scripts (standard minting/locking operations).
2. Craft a batch transaction with sub-transactions that reference those UTxOs as reference inputs.
3. Set the top-level fee to satisfy only the base size-based minimum.

This is fully within the capability of any unprivileged transaction sender. The `batchNonDistinctRefScriptsSize` function's existence in the same file confirms the developers are aware that sub-transaction reference scripts must be accounted for — making this an oversight rather than a design choice.

---

### Recommendation

Replace `getMinFeeTxUtxo = getConwayMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance with a Dijkstra-specific implementation that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
getDijkstraMinFeeTxUtxo :: (EraTx era, DijkstraEraTxBody era) => PParams era -> Tx TopTx era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx
```

and set `getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance. [9](#0-8) 

---

### Proof of Concept

1. Mint or lock a UTxO `refOut` containing a large Plutus V3 script (e.g., 50 KiB) as a reference script.
2. Construct a sub-transaction `subTx` with `referenceInputsTxBodyL = {refOut}` and any valid spend input.
3. Construct a top-level Dijkstra batch transaction `topTx` with `subTransactionsTxBodyL = {subTx}` and **no** reference inputs at the top level.
4. Compute the minimum fee using `getMinFeeTxUtxo pp topTx utxo` — it returns `alonzoMinFeeTx pp topTx` (base size fee only, zero reference script surcharge).
5. Set `feeTxBodyL` to this value and submit.
6. The `validateFeeTooSmallUTxO` check passes because `txNonDistinctRefScriptsSize utxo topTx = 0` (no top-level reference inputs).
7. Validators must still deserialize the 50 KiB script from `refOut` when processing `subTx`'s script needs, but the attacker paid no `tierRefScriptFee` for it.

The correct minimum fee should have included `tierRefScriptFee(batchNonDistinctRefScriptsSize(utxo, topTx))`, which for a 50 KiB script at `minFeeRefScriptCostPerByte = 15` would be approximately `tierRefScriptFee(51200) ≈ 1,152,000` lovelace of additional fee — entirely bypassed.
