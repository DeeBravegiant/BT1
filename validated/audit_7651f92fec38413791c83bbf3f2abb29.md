### Title
Missing `chunk.shard_id() == shard_id` Assertion in `set_state_header` Allows Cross-Shard State Substitution — (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`set_state_header` accepts a `ShardStateSyncResponseHeader` for `shard_id = Y` that embeds a `ShardChunk` belonging to shard X. Neither `validate_chunk_proofs` nor the `verify_path` Merkle membership check enforces that the embedded chunk belongs to the requested shard. A malicious peer can exploit this to make a syncing node apply shard X's state root, transactions, and receipts under shard Y's `ShardUId`, producing a `ChunkExtra` state root that diverges from every honest node.

---

### Finding Description

**Entrypoint:** A syncing node requests a state sync header from a peer. The peer's response is deserialized and passed directly to `set_state_header`.

**Validation gap — step 1 (`validate_chunk_proofs`):**

`validate_chunk_proofs` in `chain/chain/src/validate.rs` checks only internal hash consistency of the chunk body: `chunk_hash`, `tx_root`, and `receipts_root`. It never receives `shard_id` as a parameter and never asserts `chunk.shard_id() == shard_id`. [1](#0-0) 

**Validation gap — step 3a (`verify_path`):**

The Merkle membership check at lines 394–403 verifies that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` is somewhere in `sync_prev_block_header.chunk_headers_root()`. The `chunk_headers_root` is a Merkle tree over **all** shards' chunks. `verify_path` only checks that the hash is reachable from the root via the supplied path; it does not check which leaf index (shard position) the path leads to. Shard 0's chunk with shard 0's valid proof will satisfy this check even when `shard_id = 1` is requested. [2](#0-1) 

`verify_path` itself is a pure path-traversal function with no index/position enforcement: [3](#0-2) 

**No assertion anywhere in `set_state_header`:**

There is no line of the form `chunk.shard_id() == shard_id` anywhere in `set_state_header`. The function stores the header under `StateHeaderKey(shard_id, sync_hash)` without ever confirming the embedded chunk belongs to `shard_id`. [4](#0-3) 

**Downstream impact in `set_state_finalize`:**

`set_state_finalize` retrieves the stored header and extracts the chunk. It derives `shard_uid` from the `shard_id` parameter (Y), but uses `chunk_header.prev_state_root()` from the embedded chunk (shard X). The runtime's `apply_chunk` is called with shard Y's `ShardUId` but shard X's state root, transactions, and gas parameters. [5](#0-4) 

The resulting `ChunkExtra` is saved under shard Y's `ShardUId` with a state root derived from shard X's trie, diverging from every honest node's `ChunkExtra` for shard Y. [6](#0-5) 

---

### Impact Explanation

The syncing node's `ChunkExtra.state_root` for shard Y is permanently wrong after `set_state_finalize`. All subsequent block processing for shard Y on this node will fail `validate_chunk_with_chunk_extra_and_receipts_root`'s `prev_state_root` check, or silently diverge if the node is not a validator. The node is effectively split from the honest network on shard Y's state, a direct consensus/state-root divergence. [7](#0-6) 

---

### Likelihood Explanation

State sync headers are downloaded from peers selected by the syncing node. Any peer that can respond to a `StateRequestHeader` message can supply a crafted header. The attacker needs only to be reachable as a peer and to know the `sync_hash` (which is public). No validator key or privileged access is required. The crafted header requires only publicly available on-chain data (shard 0's chunk, its Merkle proof, and shard 1's receipt proofs). [8](#0-7) 

---

### Recommendation

Add an explicit shard identity check immediately after extracting the chunk in `set_state_header`:

```rust
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

This should be placed after line 376 (`let chunk = shard_state_header.cloned_chunk();`) and before `validate_chunk_proofs`. [9](#0-8) 

---

### Proof of Concept

1. Build a two-shard `TestEnv`, produce enough blocks to trigger state sync.
2. From the honest node, obtain `shard_state_header_0 = compute_state_response_header(shard_id=0, sync_hash)`.
3. Craft a `ShardStateSyncResponseHeaderV2` that copies `chunk`, `chunk_proof`, `prev_chunk_header`, `prev_chunk_proof`, and `state_root_node` from `shard_state_header_0`, but uses the valid `incoming_receipts_proofs` and `root_proofs` for shard 1 (obtainable from the honest node's `compute_state_response_header(shard_id=1, sync_hash)`).
4. Call `set_state_header(shard_id=1, sync_hash, crafted_header)` on the victim node — assert it returns `Ok(())`.
5. Download and apply state parts using `chunk_prev_state_root()` from the crafted header (shard 0's state root).
6. Call `set_state_finalize(shard_id=1, sync_hash)`.
7. Compare `get_chunk_extra(sync_prev_block.hash(), shard_uid_1)` on the victim node against the honest node — the state roots will differ.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
