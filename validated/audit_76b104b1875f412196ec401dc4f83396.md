### Title
Missing Zero-Value Validation for `minFeeRefScriptCostPerByte` in `ppuWellFormed` Allows Fee-Bypass Governance Proposal - (File: `eras/conway/impl/src/Cardano/Ledger/Conway/PParams.hs`)

---

### Summary

`ppuWellFormed` in the `ConwayEraPParams ConwayEra` instance validates many protocol parameter update fields against zero, but omits a non-zero check for `ppuMinFeeRefScriptCostPerByteL`. This allows a `ParameterChange` governance proposal setting `minFeeRefScriptCostPerByte = 0` to pass the `MalformedProposal` gate and enter the ratification queue. If enacted, the reference-script fee protection introduced specifically to prevent a DDoS attack vector is silently eliminated.

---

### Finding Description

`ppuWellFormed` is the sole ledger-level well-formedness gate for `ParameterChange` governance actions. It is called from `actionWellFormed` inside `conwayGovTransition`:

```haskell
actionWellFormed pv ga = failureUnless isWellFormed $ MalformedProposal ga
  where
    isWellFormed = case ga of
      ParameterChange _ ppd _ -> ppuWellFormed pv ppd
      _ -> True
``` [1](#0-0) 

The `ConwayEra` instance of `ppuWellFormed` checks eleven fields for non-zero/non-empty values, but `ppuMinFeeRefScriptCostPerByteL` is absent from the list:

```haskell
instance ConwayEraPParams ConwayEra where
  ppuWellFormed pv ppu =
    and
      [ isValid (/= 0) ppuMaxBBSizeL
      , isValid (/= 0) ppuMaxTxSizeL
      ...
      , hardforkConwayBootstrapPhase pv
          || isValid ((/= CompactCoin 0) . unCoinPerByte) ppuCoinsPerUTxOByteL
      , ppu /= emptyPParamsUpdate
      , pvMajor pv < natVersion @11
          || isValid (/= 0) ppuNOptL
      ]   -- ← no check for ppuMinFeeRefScriptCostPerByteL
``` [2](#0-1) 

`minFeeRefScriptCostPerByte` is typed as `NonNegativeInterval`, which admits the value `0 % 1`. The `Dijkstra` era's `ppuWellFormed` also omits this check, inheriting the same gap. [3](#0-2) 

The parameter is registered as updatable via governance (tag 33): [4](#0-3) 

The fee calculation in `getConwayMinFeeTx` reads this parameter directly:

```haskell
refScriptCostPerByte = unboundRational (pp ^. ppMinFeeRefScriptCostPerByteL)
refScriptsFee = tierRefScriptFee ... refScriptCostPerByte refScriptsSize
``` [5](#0-4) 

When `refScriptCostPerByte = 0`, `tierRefScriptFee` returns `Coin 0` for any reference-script size, completely eliminating the surcharge.

---

### Impact Explanation

`minFeeRefScriptCostPerByte` was introduced after a real DDoS attack on June 25 2024 to price large reference-script deserialization work into transaction fees. ADR-009 documents that without a sufficiently high value for this parameter, an attacker can submit transactions containing large Plutus scripts that are cheap to include but expensive for nodes to validate. [6](#0-5) 

Setting it to zero via an enacted `ParameterChange` action restores that attack surface: an attacker can then submit transactions with reference scripts up to the per-tx size limit (200 KiB in Conway, or the `maxRefScriptSizePerTx` parameter in Dijkstra) at zero marginal cost, forcing all validating nodes to perform unbounded deserialization work per transaction. This modifies fees outside design parameters — matching the **Medium** allowed impact: *"Attacker-controlled transactions… exceed intended validation limits or modify fees… outside design parameters."*

---

### Likelihood Explanation

The missing check means any actor can craft a `ParameterChange` proposal with `minFeeRefScriptCostPerByte = SJust 0` and have it accepted into the on-chain proposal queue without triggering `MalformedProposal`. Enactment requires governance ratification (DRep + committee majority in post-bootstrap Conway). This is a meaningful barrier, but the missing validation removes the first line of defense — the ledger-level well-formedness gate — that is explicitly designed to prevent such proposals from ever reaching ratification. The partial fix pattern in the external report (key validated, value not) is mirrored exactly here: many sibling parameters are checked, but `minFeeRefScriptCostPerByte` is not.

---

### Recommendation

Add a non-zero check for `ppuMinFeeRefScriptCostPerByteL` inside `ppuWellFormed` in both `ConwayEraPParams ConwayEra` and `ConwayEraPParams DijkstraEra`, mirroring the pattern already used for `ppuCoinsPerUTxOByteL`:

```haskell
, isValid (/= minBound) ppuMinFeeRefScriptCostPerByteL
```

or equivalently, using `boundRational`:

```haskell
, isValid (> minBound) ppuMinFeeRefScriptCostPerByteL
```

A corresponding test case should be added to `pparamUpdateSpec` in `Test.Cardano.Ledger.Conway.Imp.GovSpec` alongside the existing `testMalformedProposal` cases. [7](#0-6) 

---

### Proof of Concept

1. Construct a `PParamsUpdate` with only `ppuMinFeeRefScriptCostPerByteL` set to `SJust (0 % 1)` (a valid `NonNegativeInterval`).
2. Wrap it in a `ParameterChange SNothing ppu SNothing` governance action.
3. Submit the proposal. `actionWellFormed` calls `ppuWellFormed`, which returns `True` because no check for `ppuMinFeeRefScriptCostPerByteL` exists — the proposal is accepted without `MalformedProposal`.
4. After governance ratification and enactment, `pp ^. ppMinFeeRefScriptCostPerByteL` equals `0`.
5. Any subsequent transaction referencing large Plutus scripts pays zero reference-script fee (`tierRefScriptFee` with `curTierPrice = 0` accumulates nothing), bypassing the DDoS mitigation entirely. [8](#0-7)

```haskell
tierRefScriptFee ::
  HasCallStack =>
  -- | Growth factor or step multiplier
  Rational ->
  -- | Increment size in which price grows linearly according to the price
  Int ->
  -- | Base fee. Currently this is customizable by `ppMinFeeRefScriptCostPerByteL`
  Rational ->
  -- | Total RefScript size in bytes
  Int ->
  Coin
tierRefScriptFee multiplier sizeIncrement
  | multiplier <= 0 || sizeIncrement <= 0 = error "Size increment and multiplier must be positive"
  | otherwise = go 0
  where
    go !acc !curTierPrice !n
      | n < sizeIncrement =
          Coin $ floor (acc + toRational n * curTierPrice)
      | otherwise =
          go (acc + sizeIncrementRational * curTierPrice) (multiplier * curTierPrice) (n - sizeIncrement)
    sizeIncrementRational = toRational sizeIncrement
```

```markdown
It was identified a while ago that there is an overhead associated with deserializing scripts, so much so that it would be possible to create a very large script that was fairly expensive to deserialize, but very cheap to execute. This opened up an attack vector when such a Plutus script would be used as a reference script. This problem was exacerbated by the fact that there was no real limit on the total size of reference scripts that could be used in a transaction, thus being limited only by the size of the transaction itself. Therefore this opened up Cardano to a DDoS attack where an attacker could submit many such transactions that would cost very little, but would be expensive for a `cardano-node` to validate.

In order to prevent such an attack a `"minFeeRefScriptCostPerByte"` protocol parameter was introduced in the Conway era. The idea was fairly simple: we would calculate the total size of reference scripts used by a transaction and multiply it by the value specified by this protocol parameter. Result would be added to the transaction fee. However, in order for this approach to be a definite deterrent of such attacks this parameter would have to be set to a fairly high value. Community was not very keen on having this value set to a high enough value, since that could make reference scripts almost as expensive to use as regular scripts, thus significantly reducing their usability. Taking this fact into consideration, a decision was made to set this parameter to a fairly moderate value to at l ... (truncated)

This attack forced us to make a quick decision on implementing a definite prevention of such attacks, while trying to not have a drastic impact on the common use case that DApp developers rely on so much.
