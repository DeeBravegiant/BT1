### Title
Deposit Not Refunded to Existing Participant on Attestation Re-submission - (File: `crates/contract/src/lib.rs`)

### Summary

In `submit_participant_info`, when an existing participant re-submits their TEE attestation (an update, not a new entry), the function returns `Ok(())` without refunding any attached deposit. The deposit-handling block is gated behind `attestation_storage_must_be_paid_by_caller`, which is `false` for existing-participant re-submissions, so any NEAR attached to the call is silently absorbed by the contract — breaking the accounting invariant that `#[payable]` functions must refund excess deposits.

### Finding Description

`submit_participant_info` is marked `#[payable]`, meaning it accepts NEAR deposits. After calling `add_participant`, the function computes:

```rust
let attestation_storage_must_be_paid_by_caller =
    is_new_attestation || caller_is_not_participant;
``` [1](#0-0) 

When the caller is an **existing participant** re-submitting their attestation (`is_new_attestation == false` and `caller_is_not_participant == false`), this flag is `false`. The entire deposit-handling block — which both checks sufficiency and issues a refund — is skipped:

```rust
if attestation_storage_must_be_paid_by_caller {
    // ... deposit check and refund ...
}
Ok(())   // ← returns here with no deposit handling
``` [2](#0-1) 

Any NEAR attached to the call is permanently retained by the contract. There is no unconditional refund path for the `false` branch.

Compare this to the `require_deposit` helper used by `sign` and `request_app_private_key`, which always refunds excess: [3](#0-2) 

And to `propose_update`, which also always refunds excess: [4](#0-3) 

`submit_participant_info` is the only `#[payable]` function in the contract that has a code path returning `Ok(())` without any deposit handling.

### Impact Explanation

MPC nodes must periodically re-submit their TEE attestations (e.g., after a TEE image upgrade or attestation expiry). Each such call is a normal operational event. If a node attaches any NEAR — whether intentionally (believing storage must be paid) or accidentally — the deposit is permanently locked in the contract with no recovery path. This breaks the production accounting invariant that excess deposits are always refunded, and permanently removes funds from the caller's balance into the contract's balance with no corresponding state benefit.

### Likelihood Explanation

Existing participants call `submit_participant_info` regularly as part of TEE lifecycle management. The function is `#[payable]`, which signals to callers that a deposit may be required. A node operator following the pattern of other contract methods (all of which require at least 1 yoctoNEAR) will naturally attach a small deposit. Any amount above zero is silently lost. The trigger condition — existing participant re-submitting — is the **common case** in steady-state operation.

### Recommendation

Add an unconditional refund for the `else` branch (or after the `if` block) to return any attached deposit when storage charging is not required:

```rust
} else {
    let attached = env::attached_deposit();
    if attached > NearToken::from_yoctonear(0) {
        Promise::new(account_id).transfer(attached).detach();
    }
}
```

Alternatively, restructure the deposit handling to always refund excess regardless of the `attestation_storage_must_be_paid_by_caller` flag, consistent with every other `#[payable]` method in the contract.

### Proof of Concept

1. Contract is in `Running` state with participant `alice.near` already registered.
2. `alice.near` calls `submit_participant_info` with a fresh attestation and attaches `1_000_000_000_000_000_000_000_000` yoctoNEAR (1 NEAR).
3. `add_participant` returns `ParticipantInsertion::UpdatedExistingParticipant` (not `NewlyInsertedParticipant`).
4. `is_new_attestation = false`, `caller_is_not_participant = false` → `attestation_storage_must_be_paid_by_caller = false`.
5. The `if` block at line 826 is skipped entirely.
6. Function returns `Ok(())`.
7. Alice's 1 NEAR is now held by the contract with no refund promise scheduled.
8. The contract balance increased by 1 NEAR; Alice's balance decreased by 1 NEAR + gas fees. [5](#0-4)

    /// (Prospective) Participants can submit their tee participant information through this
    /// endpoint.
    #[payable]
    #[handle_result]
    pub fn submit_participant_info(
        &mut self,
        proposed_participant_attestation: dtos::Attestation,
        tls_public_key: dtos::Ed25519PublicKey,
    ) -> Result<(), Error> {
        let proposed_participant_attestation =
            proposed_participant_attestation.try_into_contract_type()?;

        let account_key = env::signer_account_pk();
        let account_id = Self::assert_caller_is_signer();

        log!(
            "submit_participant_info: signer={}, proposed_participant_attestation={:?}, account_key={:?}",
            account_id,
            proposed_participant_attestation,
            account_key
        );

        // Save the initial storage usage to know how much to charge the proposer for the storage
        // used
        let initial_storage = env::storage_usage();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        // The node always signs submissions with an Ed25519 key
        // (`near_signer_key`), so the signer key here is Ed25519 in practice.
        // Reject non-Ed25519 signer keys rather than silently storing a value
        // we could never match against in `is_caller_an_attested_participant`.
        let account_public_key = dtos::Ed25519PublicKey::try_from(&account_key).map_err(|_| {
            InvalidParameters::InvalidTeeRemoteAttestation {
                reason: "signer account key must be Ed25519".to_string(),
            }
        })?;

        // Add the participant information to the contract state
        let attestation_insertion_result = self
            .tee_state
            .add_participant(
                NodeId {
                    account_id: account_id.clone(),
                    tls_public_key,
                    account_public_key,
                },
                proposed_participant_attestation,
                tee_upgrade_deadline_duration,
            )
            .map_err(|err| {
                let reason = match &err {
                    AttestationSubmissionError::InvalidAttestation(_) => {
                        format!("TeeQuoteStatus is invalid: {err}")
                    }
                    AttestationSubmissionError::TlsKeyOwnedByOtherAccount => err.to_string(),
                };
                InvalidParameters::InvalidTeeRemoteAttestation { reason }
            })?;

        let caller_is_not_participant = self.voter_account().is_err();
        let is_new_attestation = matches!(
            attestation_insertion_result,
            ParticipantInsertion::NewlyInsertedParticipant
        );

        let attestation_storage_must_be_paid_by_caller =
            is_new_attestation || caller_is_not_participant;

        if attestation_storage_must_be_paid_by_caller {
            // `saturating_sub`: if a re-submission shrinks the entry, charge nothing
            // rather than underflow. Intentional asymmetry: we do not refund freed bytes
            // either — the caller already paid for the larger entry, and we'd rather
            // accept that asymmetry than open a refund path for payload-shrinking games.
            let storage_used = env::storage_usage().saturating_sub(initial_storage);
            let cost = env::storage_byte_cost().saturating_mul(storage_used as u128);
            let attached = env::attached_deposit();

            if attached < cost {
                return Err(InvalidParameters::InsufficientDeposit {
                    attached: attached.as_yoctonear(),
                    required: cost.as_yoctonear(),
                }
                .into());
            }

            // Refund the difference if the proposer attached more than required
            if let Some(diff) = attached.checked_sub(cost)
                && diff > NearToken::from_yoctonear(0)
            {
                Promise::new(account_id).transfer(diff).detach();
            }
        }

        Ok(())
    }
