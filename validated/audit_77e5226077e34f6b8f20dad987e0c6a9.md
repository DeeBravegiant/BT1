Audit Report

## Title
Missing Chainlink Price Staleness Check in `ChainlinkPriceOracle` Leads to Incorrect rsETH Price Calculation - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards the `updatedAt` timestamp and `answeredInRound` values, accepting stale prices without any freshness validation. This stale price propagates into `LRTOracle._getTotalEthInProtocol()`, corrupting the rsETH price used for all deposits, withdrawals, and fee minting. The same contract codebase already applies the missing checks in `ChainlinkOracleForRSETHPoolCollateral`, confirming the protocol is aware of the requirement.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` fetches the Chainlink price as:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

All five return values of `latestRoundData()` are available, but only `price` is used. The `updatedAt` timestamp and `answeredInRound` are silently discarded. By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` explicitly validates `answeredInRound < roundID` and `timestamp == 0` before returning a price. [2](#0-1) 

`LRTOracle.updateRSETHPrice()` is a public, permissionless function. [3](#0-2) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over all supported assets and calls `getAssetPrice(asset)` for each, multiplying by total deposits to compute total ETH in the protocol. [4](#0-3) 

The resulting `totalETHInProtocol` is used to compute `newRsETHPrice = (totalETHInProtocol - protocolFeeInETH) / rsethSupply`. [5](#0-4) 

If `newRsETHPrice` drops below `highestRsethPrice` by more than `pricePercentageLimit`, the protocol automatically pauses `LRTDepositPool`, `LRTWithdrawalManager`, and itself. [6](#0-5) 

## Impact Explanation
**Medium — Temporary freezing of funds:** A stale, lower price for any LST asset causes `_getTotalEthInProtocol()` to underestimate TVL. If the resulting `newRsETHPrice` drop exceeds `pricePercentageLimit` relative to `highestRsethPrice`, the protocol automatically pauses all deposits and withdrawals. Funds remain frozen until an admin manually unpauses. This is a concrete, reachable temporary freeze triggered by a permissionless public call.

**Low — Contract fails to deliver promised returns:** A stale, higher price inflates `newRsETHPrice`. Depositors calling `getRsETHAmountToMint()` receive fewer rsETH tokens than entitled, since `rsethAmountToMint = (amount * assetPrice) / rsETHPrice` — a higher denominator reduces the mint amount.

## Likelihood Explanation
`updateRSETHPrice()` is callable by any unprivileged address with no access control. Chainlink feeds go stale during L2 sequencer downtime, network congestion, or feed deprecation. The protocol is deployed on multiple chains, increasing exposure. An attacker can call `updateRSETHPrice()` at any moment a feed is stale to trigger the incorrect price update. No special privileges, victim mistakes, or external protocol compromise are required — only a stale Chainlink feed, which is a known and documented occurrence.

## Recommendation
Apply the same staleness checks already present in `ChainlinkOracleForRSETHPoolCollateral.sol` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) = priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept
1. A Chainlink LST/ETH feed (e.g., stETH/ETH) goes stale — its `updatedAt` timestamp is hours old and `answeredInRound < roundId`.
2. Any unprivileged caller invokes `LRTOracle.updateRSETHPrice()`.
3. `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `ChainlinkPriceOracle` returns the stale (lower) price with no revert.
5. `_getTotalEthInProtocol()` underestimates total ETH.
6. `newRsETHPrice` is computed lower than `highestRsethPrice`.
7. If the difference exceeds `pricePercentageLimit`, `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` are called automatically, and `LRTOracle` itself is paused.
8. All user deposits and withdrawals are frozen until an admin manually unpauses.

**Foundry fork test plan:** Fork a network where a supported LST/ETH Chainlink feed exists. Use `vm.mockCall` to make `latestRoundData()` return a stale round (e.g., `answeredInRound = roundId - 1`, `updatedAt = block.timestamp - 25 hours`). Call `LRTOracle.updateRSETHPrice()` as an unprivileged address. Assert that `LRTDepositPool.paused()` returns `true` and that a subsequent deposit reverts.
