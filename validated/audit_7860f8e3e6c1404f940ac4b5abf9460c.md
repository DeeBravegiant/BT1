### Title
Unvalidated `tokenAddress` in `initTransfer` Combined with Permissionless `logMetadata` Enables Unbacked Wrapped-Token Minting on NEAR - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary
`OmniBridge.initTransfer` accepts any caller-supplied `tokenAddress` without checking it against a registered-token whitelist. Because `logMetadata` is also permissionless, an attacker can register a fake ERC20 (one whose `transferFrom` returns `true` without moving tokens) on NEAR via the normal `deploy_token` proof flow, then call `initTransfer` with that fake token to emit a valid `InitTransfer` event while locking zero real assets. Presenting the proof of that event to NEAR's `fin_transfer` causes the bridge to mint unbacked wrapped tokens for the attacker.

### Finding Description

**Step 1 – Permissionless `logMetadata` accepts any address.**

`logMetadata` on EVM is open to any caller and emits a `LogMetadata` event from the registered OmniBridge factory contract: [1](#0-0) 

A fake ERC20 that implements `name()`, `symbol()`, and `decimals()` passes this call without issue.

**Step 2 – NEAR `deploy_token` accepts a proof of `LogMetadata` from the registered factory.**

NEAR's `deploy_token` calls `verify_proof` and then in `deploy_token_callback` only checks that the emitter equals the registered factory address: [2](#0-1) [3](#0-2) 

Because the `LogMetadata` event was emitted by the real OmniBridge contract (the registered factory), this check passes. The fake token is now registered in `token_decimals` and `deployed_tokens` on NEAR, and a NEAR-side token contract is deployed with the bridge holding mint authority. [4](#0-3) 

**Step 3 – `initTransfer` does not validate `tokenAddress` against any whitelist.**

`initTransfer` branches on `customMinters` and `isBridgeToken`, but for any other address falls through to a plain `safeTransferFrom`: [5](#0-4) 

`SafeERC20.safeTransferFrom` only requires the call to succeed and return `true`. A fake ERC20 that returns `true` from `transferFrom` without moving any tokens satisfies this check. The function then emits `InitTransfer` with the attacker-chosen `tokenAddress` and `amount`: [6](#0-5) 

**Step 4 – NEAR `fin_transfer_callback` mints unbacked tokens.**

`fin_transfer_callback` verifies the emitter is the registered factory and looks up `token_decimals` for the token — both checks pass because the fake token was registered in Step 2: [7](#0-6) 

It then calls `process_fin_transfer_to_near`, which calls `send_tokens` to transfer (mint) the NEAR-side wrapped token to the attacker — with no real EVM-side collateral ever locked.

### Impact Explanation

An unprivileged attacker can mint an arbitrary quantity of NEAR-side wrapped tokens that have no EVM-side backing. This constitutes **unauthorized creation of wrapped bridge assets** and breaks the 1:1 backing guarantee of the bridge. The minted tokens are real NEP-141 tokens that can be transferred, sold, or used in DeFi, draining value from legitimate holders of the same wrapped token.

### Likelihood Explanation

The attack requires only:
1. Deploying a trivial fake ERC20 contract (a few lines of Solidity).
2. Calling two permissionless EVM functions (`logMetadata`, `initTransfer`).
3. Submitting two standard bridge proof transactions on NEAR (`deploy_token`, `fin_transfer`).

No privileged access, leaked keys, or external dependency compromise is needed. Any on-chain actor can execute this.

### Recommendation

In `initTransfer`, require that `tokenAddress` is a known registered token — i.e., that `isBridgeToken[tokenAddress]` is true, or that `ethToNearToken[tokenAddress]` is non-empty, or that `customMinters[tokenAddress]` is set, or that `multiTokens[tokenAddress].tokenAddress != address(0)`. Reject any `tokenAddress` that does not satisfy at least one of these conditions. This mirrors the validation already present in `finTransfer` (which only acts on tokens it recognizes) and closes the gap that allows a fake token to be used as collateral.

### Proof of Concept

```solidity
// 1. Deploy fake ERC20
contract FakeToken {
    function name() external pure returns (string memory) { return "Fake"; }
    function symbol() external pure returns (string memory) { return "FAKE"; }
    function decimals() external pure returns (uint8) { return 18; }
    // Returns true without moving any tokens
    function transferFrom(address, address, uint256) external pure returns (bool) { return true; }
}

// 2. Register fake token on NEAR via logMetadata + deploy_token proof
omniBridge.logMetadata(address(fakeToken));
// → off-chain: submit LogMetadata proof to NEAR deploy_token

// 3. Call initTransfer with fake token — no real tokens locked
omniBridge.initTransfer(
    address(fakeToken),
    1_000_000e18,   // arbitrary large amount
    0,              // fee
    0,              // nativeFee
    "attacker.near",
    ""
);
// safeTransferFrom calls fakeToken.transferFrom → returns true, 0 tokens moved
// InitTransfer event emitted by real OmniBridge

// 4. Submit InitTransfer proof to NEAR fin_transfer
// → NEAR mints 1_000_000e18 wrapped FAKE tokens to attacker.near
//    with zero EVM-side collateral
```
