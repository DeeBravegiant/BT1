Audit Report

## Title
Missing `cancelWithdrawal()` Causes Temporary rsETH Freeze in `LRTWithdrawalManager` - (File: contracts/LRTWithdrawalManager.sol)

## Summary
`initiateWithdrawal()` irrevocably transfers a user's rsETH into `LRTWithdrawalManager` with no user-controlled cancellation path. When the unstaking vault holds zero assets for the requested asset — a routine state during EigenLayer's mandatory withdrawal delay — `unlockQueue()` reverts, `completeWithdrawal()` is blocked by the locked-nonce check, and no `cancelWithdrawal()` function exists. The user's rsETH is frozen in the contract for the duration of the EigenLayer withdrawal cycle.

## Finding Description
`initiateWithdrawal()` pulls rsETH from the caller at [1](#0-0)  and records the request via `_addUserWithdrawalRequest` at [2](#0-1) . From this point, three exit paths exist for the user, all of which are blocked:

1. **`completeWithdrawal()`**: Calls `_processWithdrawalCompletion`, which pops the nonce and checks `usersFirstWithdrawalRequestNonce >= nextLockedNonce[asset]`, reverting with `WithdrawalLocked` until an operator has called `unlockQueue()`. [3](#0-2) 

2. **`unlockQueue()`** (operator-only): Calls `_createUnlockParams`, which reads `unstakingVault.balanceOf(asset)` as `totalAvailableAssets`. [4](#0-3)  If the vault holds zero assets, the function reverts immediately at [5](#0-4)  — before any request is processed. The unstaking vault is only funded after EigenLayer queued withdrawals complete and are claimed by `NodeDelegator`, a process that takes days to weeks.

3. **`instantWithdrawal()`**: Gated by `onlyInstantWithdrawalAllowed(asset)` [6](#0-5)  and additionally requires vault liquidity via `getAssetsAvailableForInstantWithdrawal`. [7](#0-6)  Both conditions fail when the vault is empty.

No `cancelWithdrawal()` function exists anywhere in the contract. The entire contract has been reviewed and confirmed. [8](#0-7) 

## Impact Explanation
**Medium — Temporary freezing of funds.** A user's rsETH is locked inside `LRTWithdrawalManager` with no user-controlled exit for the duration of the EigenLayer withdrawal cycle. The freeze is bounded (it resolves once assets flow into the vault and the operator calls `unlockQueue()`), placing this squarely in the "Temporary freezing of funds" allowed impact class.

## Likelihood Explanation
**Medium.** EigenLayer imposes a mandatory queued-withdrawal delay (currently ~7 days on mainnet). Any user who calls `initiateWithdrawal()` while the unstaking vault holds zero assets for the requested asset — a normal, recurring protocol state between NodeDelegator claims — will have their rsETH frozen for that duration. This is not an edge case; it is the expected state for most of the withdrawal lifecycle.

## Recommendation
Add a `cancelWithdrawal(address asset)` function restricted to the request owner that:
1. Verifies the caller's oldest nonce is still in the locked queue (`usersFirstWithdrawalRequestNonce >= nextLockedNonce[asset]`).
2. Removes the nonce from `userAssociatedNonces[asset][msg.sender]` and deletes the corresponding entry from `withdrawalRequests`.
3. Decrements `assetsCommitted[asset]` by `request.expectedAssetAmount`.
4. Returns `request.rsETHUnstaked` of rsETH to `msg.sender` via `safeTransfer`.

## Proof of Concept
1. User calls `initiateWithdrawal(stETH, 1e18, "")` → 1e18 rsETH transferred to `LRTWithdrawalManager`; `assetsCommitted[stETH]` incremented. [9](#0-8) 
2. Unstaking vault holds 0 stETH (EigenLayer withdrawal pending — normal state).
3. Operator calls `unlockQueue(stETH, ...)` → reverts at `if (params.totalAvailableAssets == 0) revert AmountMustBeGreaterThanZero()`. [5](#0-4) 
4. User calls `completeWithdrawal(stETH, "")` → reverts at `WithdrawalLocked`. [10](#0-9) 
5. User calls `instantWithdrawal(stETH, 1e18, "")` → reverts at `InstantWithdrawalNotEnabled` or `CantInstantWithdrawMoreThanAvailable`. [11](#0-10) 
6. No `cancelWithdrawal()` exists. User's 1e18 rsETH remains frozen until EigenLayer withdrawal completes, assets are claimed into the vault, and the operator successfully calls `unlockQueue()`.

**Foundry test plan**: Deploy `LRTWithdrawalManager` on a mainnet fork. Mock `LRTUnstakingVault.balanceOf(stETH)` to return 0. Call `initiateWithdrawal` as a normal user. Assert all three exit paths revert. Assert the contract holds the user's rsETH. Confirm no `cancelWithdrawal` selector exists on the contract.

    /*//////////////////////////////////////////////////////////////

    /// @notice Initiates a withdrawal request for converting rsETH to a specified LST.
    /// @param asset The LST address the user wants to receive.
    /// @param rsETHUnstaked The amount of rsETH the user wishes to unstake.
    /// @dev This function is only callable by the user and is used to initiate a withdrawal request for a specific
    /// asset. Will be finalised by calling `completeWithdrawal` after the manager unlocked the request and the delay
    /// has past. There is an edge case were the user withdraws last underlying asset and that asset gets slashed.
    function initiateWithdrawal(
        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        override
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }

        IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked);

        uint256 expectedAssetAmount = getExpectedAssetAmount(asset, rsETHUnstaked);

        if (expectedAssetAmount > getAvailableAssetAmount(asset)) revert ExceedAmountToWithdraw();

        // preventing over-withdrawal.
        assetsCommitted[asset] += expectedAssetAmount;

        _addUserWithdrawalRequest(asset, rsETHUnstaked, expectedAssetAmount);

        emit ReferralIdEmitted(referralId);
    }

    /// @notice Completes a user's withdrawal process by transferring the ETH/LST amount corresponding to the rsETH
    /// unstaked.
    /// @param asset The asset address the user wishes to withdraw.
    function completeWithdrawal(address asset, string calldata referralId) external nonReentrant whenNotPaused {
        _processWithdrawalCompletion(asset, msg.sender, referralId);
    }

    /// @notice Allows operators to complete a user's withdrawal process
    /// @param asset The asset address the user wishes to withdraw
    /// @param user The address of the user whose withdrawal to complete
    /// @param referralId The referral identifier for tracking
    /// @dev Not expected to be used for ETH; potential gas grief scenarios are non-impactful for ETH
    function completeWithdrawalForUser(
        address asset,
        address user,
        string calldata referralId
    )
        external
        nonReentrant
        whenNotPaused
        onlyLRTOperator
    {
        _processWithdrawalCompletion(asset, user, referralId);
        emit AssetWithdrawalCompletedBy(msg.sender);
    }

    /// @notice Allows users to instantly withdraw their assets by burning rsETH tokens
    /// @param asset The address of the asset (ETH/LST) to withdraw
    /// @param rsETHUnstaked The amount of rsETH tokens to burn for withdrawal
    /// @param referralId The referral identifier for tracking
    /// @dev Uses the fee set at execution time. Managers can raise it right before this call, making withdrawals cost
    /// more than expected.
    function instantWithdrawal(
        address asset,
        uint256 rsETHUnstaked,
        string calldata referralId
    )
        external
        nonReentrant
        whenNotPaused
        onlySupportedAsset(asset)
        onlySupportedStrategy(asset)
        onlyInstantWithdrawalAllowed(asset)
    {
        if (rsETHUnstaked == 0 || rsETHUnstaked < minRsEthAmountToWithdraw[asset]) {
            revert InvalidAmountToWithdraw();
        }
        if (IERC20(lrtConfig.rsETH()).balanceOf(msg.sender) < rsETHUnstaked) revert NotEnoughRsETH();
        uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked);
        IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked);
        ILRTUnstakingVault unstakingVault = ILRTUnstakingVault(lrtConfig.getContract(LRTConstants.LRT_UNSTAKING_VAULT));
        if (assetAmountUnlocked > unstakingVault.getAssetsAvailableForInstantWithdrawal(asset)) {
            revert CantInstantWithdrawMoreThanAvailable();
        }

        unstakingVault.redeem(asset, assetAmountUnlocked);

        uint256 fee = (assetAmountUnlocked * instantWithdrawalFee) / 10_000;
        uint256 userAmount = assetAmountUnlocked - fee;

        address feeRecipient = instantWithdrawalFeeRecipient;
        if (feeRecipient == address(0)) {
            // Backwards-compatible default: send fees to the protocol treasury
            feeRecipient = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
        }
        if (fee > 0) {
            _transferAsset(asset, feeRecipient, fee);
            emit InstantWithdrawalFeeCollected(msg.sender, asset, fee);
        }

        _transferAsset(asset, msg.sender, userAmount);
        emit ReferralIdEmitted(referralId);
        emit AssetWithdrawalFinalized(msg.sender, asset, rsETHUnstaked, userAmount);
    }
