Audit Report

## Title
Reward Tokens Permanently Frozen When `totalKernelStaked` Drops to Zero Mid-Period - (File: contracts/KERNEL/KernelDepositPool.sol)

## Summary
`KernelDepositPool.notifyRewardAmount` transfers reward tokens into the contract and sets a `rewardRate` for the active period. If all stakers exit via `initiateWithdrawal` + `claimWithdrawal` during that period, `totalKernelStaked` reaches zero, `rewardPerToken()` stops accumulating, and the proportional reward tokens for the zero-staked window are permanently locked with no on-chain recovery path.

## Finding Description
`notifyRewardAmount` (L566–592) pulls reward tokens into the contract and sets `rewardRate`. It guards against being called when `totalKernelStaked == 0` (L570: `if (totalKernelStaked == 0) revert NoStakedTokens()`), but this guard only prevents starting a new period with zero stakers — it does not prevent stakers from withdrawing to zero after the period has begun.

`rewardPerToken()` (L408–414) short-circuits when `totalKernelStaked == 0`:
```solidity
if (totalKernelStaked == 0) {
    return rewardPerTokenStored;  // accumulation halts
}
```
Any `rewardRate * elapsed` accrued during the zero-staked window is silently discarded. The contract's own NatSpec (L14–23) explicitly acknowledges this: *"If `totalKernelStaked` ever hits zero during a reward distribution window, any remaining rewards for that period will stay locked in the contract."*

The only admin write functions are `setRewardsDuration`, `notifyRewardAmount`, `setWithdrawalDelay`, and `setMaxNumberOfWithdrawalsPerUser` — none can retrieve stuck reward tokens. There is no `recoverTokens` or `emergencyWithdraw`.

The stated mitigation in the NatSpec is purely operational ("ensuring there are always some tokens staked"), not enforced on-chain. Any staker can call `initiateWithdrawal` (L320) followed by `claimWithdrawal` (L344) after `withdrawalDelay` — both are public, permissionless functions — reducing `totalKernelStaked` to zero at any time.

## Impact Explanation
Reward tokens transferred into the contract via `notifyRewardAmount` are permanently unrecoverable once `totalKernelStaked` hits zero mid-period. This is **permanent freezing of unclaimed yield** — a valid Medium impact per the allowed scope.

## Likelihood Explanation
No privileged action is required. Any staker can initiate and complete a withdrawal during an active reward window. If the last staker exits — whether voluntarily, in response to market conditions, or as part of a coordinated exit — `totalKernelStaked` reaches zero and the remaining rewards are frozen. The `withdrawalDelay` (capped at 30 days, L35) is the only friction, and it does not prevent the outcome.

## Recommendation
Add an admin-callable `recoverRewards` function that transfers excess `rewardsToken` balance (i.e., `rewardsToken.balanceOf(address(this))` minus the sum of all `rewards[user]` and the currently accruing but unallocated portion) to a configurable treasury address. Alternatively, enforce on-chain that `totalKernelStaked` cannot reach zero while `block.timestamp < finishAt` by reverting in `initiateWithdrawal` if the withdrawal would leave `totalKernelStaked == 0` during an active reward period.

## Proof of Concept
1. Admin calls `notifyRewardAmount(1_000e18)` — 1,000 reward tokens enter the contract; `rewardRate` is set, `finishAt = block.timestamp + duration`.
2. All stakers call `initiateWithdrawal(balance)`, reducing `totalKernelStaked` to 0 (L326: `totalKernelStaked -= _amount`).
3. After `withdrawalDelay`, all stakers call `claimWithdrawal` — their KERNEL is returned.
4. `totalKernelStaked == 0`. `rewardPerToken()` (L409–411) now always returns the frozen `rewardPerTokenStored`.
5. Time passes; `finishAt` is reached. The reward tokens proportional to the zero-staked window remain in the contract.
6. No function exists to retrieve them. Calling `notifyRewardAmount` again would revert at L570 (`NoStakedTokens`) if still zero, or compound the stuck balance if a new staker re-enters.

Foundry test sketch:
```solidity
function test_rewardsFrozenOnZeroStake() public {
    // Setup: stake, notify rewards
    vm.prank(admin); pool.notifyRewardAmount(1_000e18);
    // All stakers withdraw
    vm.prank(staker); pool.initiateWithdrawal(stakerBalance);
    vm.warp(block.timestamp + withdrawalDelay + 1);
    vm.prank(staker); pool.claimWithdrawal(1);
    assertEq(pool.totalKernelStaked(), 0);
    // Warp past finishAt
    vm.warp(pool.finishAt() + 1);
    // Reward tokens remain stuck — no function to recover them
    assertGt(rewardsToken.balanceOf(address(pool)), 0);
}
```
