### Title
Caller Deposit Not Refunded on Early-Return Failure Paths in Wallet Contract Callbacks - (File: runtime/near-wallet-contract/implementation/wallet-contract/src/lib.rs)

### Summary
The ETH-implicit account wallet contract (`WalletContract`) contains two callback functions — `address_check_callback` and `nep_141_storage_balance_callback` — that silently discard the caller's attached NEAR deposit when they return early due to intermediate cross-contract call failures. The deposit is permanently locked in the wallet contract's balance rather than being refunded to the rightful owner.

### Finding Description

When an external caller (relayer or user) calls `rlp_execute` with a non-zero attached deposit, the contract captures it in a `CallerDeposit` struct and passes it through the promise chain. The correct refund path exists only in `rlp_execute_callback`:

```rust
// rlp_execute_callback — CORRECT: refunds on failure
PromiseResult::Failed => {
    if let Some(CallerDeposit { account_id, yocto_near }) = caller_deposit {
        let refund_promise = env::promise_batch_create(&account_id);
        env::promise_batch_action_transfer(refund_promise, NearToken::from_yoctonear(yocto_near.into()));
    }
    ...
}
```

However, `address_check_callback` has **three** early-return paths that drop `caller_deposit` without issuing any refund:

1. **Registrar call fails** (line 142–148): `PromiseResult::Failed` → returns `ExecuteResponse` error, deposit silently discarded.
2. **JSON deserialization fails** (line 151–157): Unexpected registrar response → returns error, deposit silently discarded.
3. **Target is a named account, signer is not the wallet** (line 168–173): Returns an "Invalid target" error, deposit silently discarded.

Similarly, `nep_141_storage_balance_callback` has **three** early-return paths that drop `caller_deposit`:

1. **`storage_balance_of` call fails** (line 204–210): `PromiseResult::Failed` → returns error, deposit silently discarded.
2. **JSON deserialization fails** (line 213–219): Unexpected response → returns error, deposit silently discarded.
3. **Action is not a FunctionCall** (line 245–253): Returns error, deposit silently discarded.

In all six cases, the deposit that was attached to the original `rlp_execute` call remains in the wallet contract's balance with no mechanism to recover it.

### Impact Explanation

Any external caller who attaches a non-zero NEAR deposit to `rlp_execute` and whose transaction routes through either the address-check path (ETH-to-ETH base token transfer) or the ERC-20 emulation path (`storage_balance_of` check) will permanently lose their deposit if any intermediate cross-contract call fails or returns an unexpected result. The corrupted protocol value is the caller's **account balance** — NEAR tokens are irreversibly transferred to the wallet contract and cannot be withdrawn.

### Likelihood Explanation

The address registrar call and `storage_balance_of` call are external cross-contract calls that can fail due to: the registrar contract being temporarily unavailable, a panic in the registrar, gas exhaustion, or a malformed response. These are realistic failure modes in production. Any user performing an ETH-to-ETH base token transfer or an ERC-20 transfer via the wallet contract with an attached deposit is at risk.

### Recommendation

Add deposit refund logic to every early-return path in `address_check_callback` and `nep_141_storage_balance_callback`, mirroring the pattern already used in `rlp_execute_callback`:

```rust
// In address_check_callback, PromiseResult::Failed branch:
PromiseResult::Failed => {
    if let Some(CallerDeposit { account_id, yocto_near }) = caller_deposit {
        let refund_promise = env::promise_batch_create(&account_id);
        env::promise_batch_action_transfer(refund_promise, NearToken::from_yoctonear(yocto_near.into()));
    }
    return PromiseOrValue::Value(ExecuteResponse { ... });
}
```

Apply the same pattern to all other early-return paths in both callbacks.

### Proof of Concept

1. External caller (relayer) calls `rlp_execute` on a wallet contract with `attached_deposit = 5 NEAR`, targeting an ETH-implicit account that triggers the address-check path.
2. `inner_rlp_execute` captures `caller_deposit = Some(CallerDeposit { account_id: relayer, yocto_near: 5e24 })` and schedules `address_registrar.lookup(address).then(address_check_callback(..., caller_deposit))`.
3. The address registrar contract panics or runs out of gas → `PromiseResult::Failed`.
4. `address_check_callback` hits the early return at line 142–148, returning an error `ExecuteResponse` without issuing any transfer back to the relayer.
5. The wallet contract's balance increases by 5 NEAR. The relayer's balance decreases by 5 NEAR permanently.
6. The deposit is irrecoverable: the wallet contract has no withdrawal function, and the `has_in_flight_tx` flag is reset to `false`, allowing future transactions to proceed normally while the deposit remains locked.
