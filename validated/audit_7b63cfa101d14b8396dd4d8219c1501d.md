Audit Report

## Title
Missing Non-Positive Price Guard in `getAssetPrice()` Causes Overflow Revert, Freezing Deposits and Price Updates - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` casts the `int256 price` from Chainlink's `latestRoundData()` directly to `uint256` without a non-positive check. In Solidity 0.8.x, the subsequent `uint256(price) * 1e18` multiplication overflows and reverts when `price` is negative. This propagates through `LRTOracle.getAssetPrice()` into both `LRTDepositPool._beforeDeposit()` (blocking all user deposits) and `LRTOracle.updateRSETHPrice()` (freezing the rsETH price), constituting a temporary freezing of funds.

## Finding Description

In `contracts/oracles/ChainlinkPriceOracle.sol` at L52–54:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

There is no `price <= 0` guard. If `price` is negative (e.g., `-1`), then `uint256(-1)` = `type(uint256).max`. Multiplying by `1e18` overflows under Solidity 0.8.x checked arithmetic and reverts.

The sibling contract `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` applies the correct guard at L32:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
uint256 normalizedPrice = uint256(ethPrice) * 1e18 / ...
``` [2](#0-1) 

This inconsistency confirms the team is aware of the risk but failed to apply the guard in `ChainlinkPriceOracle`.

The revert propagates through two confirmed paths:

**Path 1 — Deposits blocked:**
`LRTDepositPool.depositETH()` / `depositAsset()` → `_beforeDeposit()` (L665) → `getRsETHAmountToMint()` (L520) → `lrtOracle.getAssetPrice(asset)` (L520) → `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)` (L157 in `LRTOracle`) → `ChainlinkPriceOracle.getAssetPrice()` → overflow revert. [3](#0-2) [4](#0-3) 

**Path 2 — Price update blocked:**
`LRTOracle.updateRSETHPrice()` (public, L87) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` (L339) → same overflow revert. [5](#0-4) [6](#0-5) 

No existing check in `ChainlinkPriceOracle` guards against a negative return value.

## Impact Explanation

This matches **Medium — Temporary freezing of funds**. When any supported asset's Chainlink feed returns a negative price, all calls to `depositETH()` and `depositAsset()` revert, blocking user deposits until the feed recovers or the oracle is replaced by an admin. The rsETH price also cannot be updated during this window. Funds already deposited are not lost, making this temporary rather than permanent.

## Likelihood Explanation

Chainlink `latestRoundData()` can return a negative `answer` in edge cases such as extreme market dislocations or feed misconfiguration. While rare for LST/ETH feeds, it is a documented possibility. The protocol's own `ChainlinkOracleForRSETHPoolCollateral` guards against it explicitly, confirming the team treats it as a realistic scenario. No privileged access or attacker action is required — the condition arises from the feed itself, and any user attempting a deposit during that window triggers the revert.

## Recommendation

Add a non-positive price check before casting, consistent with the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [7](#0-6) 

## Proof of Concept

1. Deploy a mock `AggregatorV3Interface` whose `latestRoundData()` returns `price = -1` and `decimals() = 8`.
2. Register it via `ChainlinkPriceOracle.updatePriceFeedFor(asset, mockFeed)`.
3. Call `ChainlinkPriceOracle.getAssetPrice(asset)`:
   - `uint256(-1)` = `type(uint256).max`
   - `type(uint256).max * 1e18` → Solidity 0.8.x overflow → revert.
4. Call `LRTOracle.updateRSETHPrice()` → reverts at `_getTotalEthInProtocol()` → rsETH price frozen.
5. Call `LRTDepositPool.depositAsset(asset, amount, 0, "")` → reverts at `getRsETHAmountToMint()` → deposits blocked.

Foundry test sketch:
```solidity
function test_negativePrice_freezesDeposits() public {
    MockChainlinkFeed feed = new MockChainlinkFeed(-1, 8);
    vm.prank(manager);
    chainlinkOracle.updatePriceFeedFor(asset, address(feed));

    vm.expectRevert(); // arithmetic overflow
    chainlinkOracle.getAssetPrice(asset);

    vm.expectRevert();
    lrtOracle.updateRSETHPrice();

    vm.expectRevert();
    lrtDepositPool.depositAsset(asset, 1 ether, 0, "");
}
```
