### Title
P2P Sync Accepts Unverified Sierra Class Body, Advancing Marker While Leaving Expected Class Hash Unregistered — (`crates/apollo_p2p_sync/src/client/class.rs`)

### Summary

`write_to_storage` calls `add_class(class_body)` without ever comparing the `ClassHashes::class_hash` returned by the class manager (the Poseidon hash computed from the body) against the `class_hash` key that came from the state diff. The TODO at line 39 explicitly acknowledges this gap. An unauthenticated p2p peer can exploit this to permanently corrupt a syncing node's class storage.

### Finding Description

**Step 1 — `parse_data_for_block` (class.rs:87–161):**

The function reads the already-stored state diff and collects `class_hash_to_compiled_class_hash` as the set of expected class hashes. For each `(api_contract_class, class_hash)` tuple received from the peer, it checks only that the peer-supplied `class_hash` is a key in that map (`is_declared` check, line 144). It inserts the peer-supplied class body into `declared_classes_result` keyed by the peer-supplied `class_hash` — **without verifying that `hash(class_body) == class_hash`**. [1](#0-0) 

**Step 2 — `write_to_storage` (class.rs:35–48):**

The loop iterates over `(class_hash, class)` pairs. It calls `class_manager_client.add_class(class.clone()).await`, passing only the class body. The `while let Err(err)` pattern discards the `Ok(ClassHashes { class_hash: computed_hash, ... })` return value entirely. The TODO at line 39 explicitly flags that the returned `class_hash` is never verified against the state-diff key. [2](#0-1) 

**Step 3 — `ClassManager::add_class` (class_manager.rs:71–113):**

The class manager independently computes the Poseidon hash from the class body and stores the class under that computed hash. It returns `ClassHashes { class_hash: computed_hash, ... }`. If `computed_hash ≠ state_diff_class_hash`, the class is stored under the wrong key and the state-diff key remains unregistered. [3](#0-2) 

**Step 4 — Marker advances unconditionally (class.rs:67–70):**

After the loop, `update_class_manager_block_marker` is committed regardless of whether any class was stored under the correct hash. [4](#0-3) 

### Impact Explanation

After the attack:
- `class_hash_A` (from the state diff) has no entry in the class manager.
- `class_hash_B = hash(class_body_X)` has an entry, but nothing in the state diff references it.
- The `class_manager_block_marker` has advanced past the affected block.
- The node will never re-sync the missing class (the marker already passed it).
- Every subsequent `get_executable(class_hash_A)` returns `None`, causing execution failures for any contract declared with that class hash.

This maps to the allowed Critical impact: **"Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution"** — specifically, the class is permanently absent for the expected hash, making the node unable to execute any contract using it.

### Likelihood Explanation

The attacker is an unauthenticated p2p peer. No special privileges are required. The peer only needs to:
1. Know a valid `class_hash_A` that appears in the state diff for a block the victim is syncing.
2. Send any valid Sierra class body whose Poseidon hash is not `class_hash_A`.

The `is_declared` check passes because `class_hash_A` is in the state diff. No other guard exists.

### Recommendation

In `write_to_storage`, after `add_class` succeeds, compare the returned `ClassHashes::class_hash` against the state-diff key. If they differ, treat it as a `BadPeerError` (disconnect/penalize the peer) and do **not** advance the marker. The TODO at line 39 already identifies this exact fix.

    pub async fn add_class(&mut self, class: RawClass) -> ClassManagerResult<ClassHashes> {
        let sierra_class = SierraContractClass::try_from(&class)?;
        let class_hash = sierra_class.calculate_class_hash();
        if let Ok(Some(executable_class_hash_v2)) =
            self.classes.get_executable_class_hash_v2(class_hash)
        {
            // Class already exists.
            return Ok(ClassHashes { class_hash, executable_class_hash_v2 });
        }

        let compilation_start_time = Instant::now();
        let (raw_executable_class, executable_class_hash_v2) =
            self.compiler.compile(class.clone()).await.map_err(|err| match err {
                SierraCompilerClientError::SierraCompilerError(error) => {
                    ClassManagerError::SierraCompiler { class_hash, error }
                }
                SierraCompilerClientError::ClientError(error) => {
                    ClassManagerError::Client(error.to_string())
                }
            })?;
        debug!(
            %class_hash,
            compiled_class_hash = %executable_class_hash_v2,
            compilation_elapsed_ms = compilation_start_time.elapsed().as_millis(),
            class_size_bytes =
                class.size().map_or("Failed to get class size".to_owned(), |size| size.to_string()),
            casm_size_bytes =
                raw_executable_class.size().map_or("Failed to get casm size".to_owned(), |size| size.to_string()),
            "Finished compiling class."
        );

        self.validate_class_length(&raw_executable_class)?;
        Self::validate_class_version(&sierra_class)?;
        self.classes.set_class(
            class_hash,
            class,
            executable_class_hash_v2,
            raw_executable_class,
        )?;

        let class_hashes = ClassHashes { class_hash, executable_class_hash_v2 };
        Ok(class_hashes)
    }
