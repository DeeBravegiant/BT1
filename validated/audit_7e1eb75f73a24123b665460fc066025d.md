### Title
`SyncStateReader::get_compiled_class_hash` Is Unimplemented (`todo!()`), Causing Gateway Stateful Validation to Panic for Declare V2/V3 Transactions — (`File: crates/apollo_gateway/src/sync_state_reader.rs`)

---

### Summary

The `BlockifierStateReader` implementation for `SyncStateReader` — the production state reader used by the gateway for stateful transaction validation — leaves `get_compiled_class_hash` as `todo!()`. This panics at runtime whenever the blockifier's `CachedState` needs to fetch the initial compiled class hash value, which occurs during Declare V2/V3 transaction validation. Every such transaction submitted to the gateway triggers the panic, causing the gateway to reject all Declare V2/V3 transactions before sequencing.

---

### Finding Description

In `crates/apollo_gateway/src/sync_state_reader.rs`, the `BlockifierStateReader` impl for `SyncStateReader` contains:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [1](#0-0) 

All other methods in the same impl block (`get_storage_at`, `get_nonce_at`, `get_compiled_class`, `get_class_hash_at`) are fully implemented by delegating to the `state_sync_client`. Only `get_compiled_class_hash` is left as `todo!()`. [2](#0-1) 

The `SyncStateReader` is the concrete type produced by `SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block`, which is the factory used for all non-genesis gateway stateful validation. [3](#0-2) 

The call chain that reaches the panic is:

1. Gateway receives a Declare V2/V3 transaction and runs stateful validation via the blockifier.
2. The blockifier wraps `SyncStateReader` in `StateReaderAndContractManager`, whose `get_compiled_class_hash` unconditionally delegates to the inner state reader: [4](#0-3) 

3. The blockifier's `CachedState` calls `get_compiled_class_hash` on the underlying state when the cache does not yet hold the initial value for a class hash (e.g., when computing the state diff after `set_compiled_class_hash` is written during Declare execution): [5](#0-4) 

4. This reaches `SyncStateReader::get_compiled_class_hash` → `todo!()` → **panic**.

The `todo!()` macro in Rust panics unconditionally with "not yet implemented". In a tokio async context, the panic propagates as a task failure, causing the gateway to return an error for the request. The gateway process itself survives, but every Declare V2/V3 transaction is rejected.

For comparison, the RPC execution state reader (`apollo_rpc_execution`) has a complete, correct implementation of `get_compiled_class_hash` that queries storage: [6](#0-5) 

The `SyncStateReader` is missing the equivalent implementation.

---

### Impact Explanation

**High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

Any user submitting a Declare V2/V3 transaction to the gateway triggers the panic. The gateway's stateful validation task crashes, and the transaction is rejected with an internal error. No Declare V2/V3 transaction can pass gateway admission, meaning no new Cairo 1 contract classes can be declared through the sequencer's gateway. This is a complete denial of service for the Declare transaction type.

---

### Likelihood Explanation

The trigger requires no special privileges. Any unprivileged user can submit a standard Declare V2/V3 transaction (the normal path for deploying new Cairo 1 contracts). The bug is deterministic: every such transaction hits the `todo!()` path. The only condition is that the node has at least one block (so `SyncStateReader` is used rather than `GenesisStateReader`).

---

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state sync client, analogous to the existing `get_storage_at`, `get_nonce_at`, and `get_class_hash_at` implementations. The `StateSyncClient` trait already exposes the necessary interface. The implementation should follow the same pattern as `apollo_rpc_execution`'s state reader, returning `CompiledClassHash::default()` for undeclared or Cairo 0 classes.

---

### Proof of Concept

1. Start the sequencer node with at least one committed block (so `SyncStateReader` is active).
2. Submit any valid Declare V2/V3 transaction to the gateway endpoint.
3. The gateway's stateful validation calls `blockifier::CachedState::get_compiled_class_hash` for the declared class hash.
4. `CachedState` finds no cached initial value and calls `SyncStateReader::get_compiled_class_hash`.
5. `todo!()` panics: `"not yet implemented"`.
6. The tokio task fails; the gateway returns an internal error to the caller.
7. The transaction is never admitted to the mempool or sequenced.

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        let mut cache = self.cache.borrow_mut();

        if cache.get_compiled_class_hash(class_hash).is_none() {
            let compiled_class_hash = self.state.get_compiled_class_hash(class_hash)?;
            cache.set_compiled_class_hash_initial_value(class_hash, compiled_class_hash);
        }

        let compiled_class_hash = cache
            .get_compiled_class_hash(class_hash)
            .unwrap_or_else(|| panic!("Cannot retrieve '{class_hash:?}' from the cache."));
        Ok(*compiled_class_hash)
    }

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
