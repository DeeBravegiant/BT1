Audit Report

## Title
`ChainlinkPriceOracle.getAssetPrice` Missing Zero-Price Validation Enables Deflated rsETH Price and Protocol Pause - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice` calls `latestRoundData()` and casts the returned `int256 price` directly to `uint256` without checking `price > 0`. A zero return silently propagates through `LRTOracle._getTotalEthInProtocol` into `_updateRsETHPrice`, either triggering an automatic protocol-wide pause (temporary freeze of funds) or committing a deflated `rsETHPrice` to state. The public `updateRSETHPrice()` function allows any caller to exploit a zero-price window immediately.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice` (L49–55) performs no positivity check:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`uint256(0) * 1e18 / 10**decimals == 0` — no revert, no error.

This zero flows into `LRTOracle._getTotalEthInProtocol` (L339–343):

```solidity
uint256 assetER = getAssetPrice(asset);
uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
totalETHInProtocol += totalAssetAmt.mulWad(assetER);  // silently drops asset's TVL
```

`_updateRsETHPrice` (L250) then computes:

```solidity
uint256 newRsETHPrice = (totalETHInProtocol - protocolFeeInETH).divWad(rsethSupply);
```

With one asset's TVL silently zeroed, `newRsETHPrice` is artificially deflated. The downside-protection block (L270–282) then evaluates:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

If the deflation exceeds `pricePercentageLimit`, all three contracts are paused and `rsETHPrice` is **not** updated (early return). If within threshold, `rsETHPrice = newRsETHPrice` (L313) commits the deflated value.

The same codebase already applies the correct guard in `ChainlinkOracleForRSETHPoolCollateral.getRate` (L32): `if (ethPrice <= 0) revert InvalidPrice();` — confirming the developers recognize this check is necessary but omitted it in `ChainlinkPriceOracle`.

No existing guard in `_updateRsETHPrice` or `_getTotalEthInProtocol` catches a zero asset price before it corrupts the computation.

## Impact Explanation
**Primary — Medium: Temporary freezing of funds.** When `pricePercentageLimit` is configured and the affected asset represents sufficient TVL share, the deflated `newRsETHPrice` triggers `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `LRTOracle._pause()`. All user-facing deposits and withdrawals are frozen until an admin manually unpauses each contract. The attacker needs no funds and no privilege.

**Secondary — Low: Contract fails to deliver promised returns.** If the price drop falls within the threshold, `rsETHPrice` is committed at the deflated value. Subsequent depositors receive more rsETH than they are owed; withdrawers receive less ETH than they are owed.

## Likelihood Explanation
Chainlink feeds can return 0 during incomplete rounds, feed deprecation, or aggregator edge cases. `updateRSETHPrice()` (L87) is public with no access control — any EOA or contract can call it at any time. An attacker monitoring on-chain Chainlink round data can call `updateRSETHPrice()` the moment a zero price is observed, with no capital requirement and no privilege. The window may be brief but is fully attacker-exploitable. The same risk pattern is already acknowledged by the developers in the sibling oracle contract.

## Recommendation
Add a positivity check in `ChainlinkPriceOracle.getAssetPrice`, mirroring the pattern in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Optionally, also add stale-round and incomplete-round checks (`answeredInRound < roundId`, `updatedAt == 0`) as already present in the sibling contract.

## Proof of Concept
1. Deploy or fork with a supported asset (e.g., stETH) whose Chainlink feed is controlled or observed to return `price = 0`.
2. Call `LRTOracle.updateRSETHPrice()` (public, no access control).
3. Execution path: `updateRSETHPrice()` → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `LRTOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns `0`.
4. `totalETHInProtocol` is computed without stETH's TVL contribution; `newRsETHPrice` is deflated.
5. **Path A (pause):** If `pricePercentageLimit > 0` and `diff > pricePercentageLimit.mulWad(highestRsethPrice)`: `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()` — all user operations frozen.
6. **Path B (price corruption):** If within threshold: `rsETHPrice = newRsETHPrice` — deflated price committed, mispricing all subsequent deposits and withdrawals.

Foundry fork test plan: mock the Chainlink aggregator for stETH to return `(1, 0, block.timestamp, block.timestamp, 1)`, call `updateRSETHPrice()`, assert either `lrtDepositPool.paused() == true` (Path A) or `lrtOracle.rsETHPrice() < expected` (Path B).
