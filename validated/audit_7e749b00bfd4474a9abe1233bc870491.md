### Title
Sub-transaction `directDeposits` bypass batch value conservation, enabling ADA creation from nothing — (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

### Summary
In the Dijkstra era, sub-transactions can carry a `directDeposits` field that adds ADA directly to account balances. The sub-transaction UTxO rule (`SUBUTXO`) calls `Shelley.updateUTxOStateNoFees`, which has no knowledge of `directDeposits` and therefore does not deduct the deposited amount from the UTxO. No value-conservation check exists for sub-transactions. The top-level `validateValueNotConservedUTxO` check operates only on the top-level transaction body against the *original* UTxO and *original* cert state, so sub-transaction direct deposits are invisible to it. An unprivileged submitter can therefore create ADA from nothing by embedding a sub-transaction that carries `directDeposits` without a matching UTxO deduction.

### Finding Description

**Processing pipeline (Dijkstra LEDGER rule)**

The `dijkstraLedgerTransition` in `Rules/Ledger.hs` processes sub-transactions first via `SUBLEDGERS`, then calls `UTXOW` for the top-level transaction:

```
SUBLEDGERS → (for each sub-tx) SUBLEDGER → SUBENTITIES + SUBUTXOW/SUBUTXO
                                                    ↓
                                        applyDirectDeposits  (adds to accounts)
                                        updateUTxOStateNoFees (no direct-deposit deduction)
```

After all sub-transactions are processed, the top-level UTXO rule runs:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Utxo.hs ~line 381
{- consumed pp utxo₀ txb = produced pp certState txb -}
runTest $ Shelley.validateValueNotConservedUTxO pp originalUtxo certState txBody
```

`originalUtxo` and `certState` are the pre-sub-transaction snapshots; `txBody` is the top-level body only.

**Sub-transaction direct deposits are applied without UTxO deduction**

In `SUBENTITIES` (`Rules/SubEntities.hs` lines 182–187):


`applyDirectDeposits` (in `libs/cardano-ledger-core/src/Cardano/Ledger/State/Account.hs` lines 290–298) simply increments account balances:

```haskell
applyDirectDeposits (DirectDeposits dd) =
  updateAccountBalances
    (\amount account -> addCompactCoin amount (account ^. balanceAccountStateL))
    dd
```

No ADA is removed from the UTxO or any other pot.

**Sub-transaction UTxO update ignores direct deposits**

In `SUBUTXO` (`Rules/SubUtxo.hs` lines 265–278):

```haskell
newState <-
  Shelley.updateUTxOStateNoFees
    pp
    utxoState
    txBody
    certState
    (utxosGovState utxoState)
    (tellEvent . TotalDeposits (hashAnnotated txBody))
    (\a b -> tellEvent $ TxUTxODiff a b)
pure $ newState & utxosDonationL <>~ txBody ^. treasuryDonationTxBodyL
```

`Shelley.updateUTxOStateNoFees` is the Shelley-era function; it removes spent inputs and adds new outputs but has no concept of `directDeposits`. The direct-deposit amount is never subtracted from the UTxO.

**Value conservation is explicitly absent for sub-transactions**

The mapping function `dijkstraUtxoToDijkstraSubUtxoPredFailure` (`Rules/SubUtxo.hs` lines 333, 351) documents this explicitly:

```haskell
ValueNotConservedUTxO _ -> error "Impossible: `ValueNotConservedUTxO` for SUBUTXO"
WithdrawalsExceedAccountBalance _ -> error "Impossible: `WithdrawalsExceedAccountBalance` for SUBUTXO"
```
