### Title
Pool Farmer Permanently Locked in Waiting Room via Repeated Absorb Spends Resetting `ASSERT_HEIGHT_RELATIVE` Timer — (`File: chia/pools/pool_puzzles.py`)

---

### Summary

When a pool singleton is in the `LEAVING_POOL` (waiting room) state, absorb spends (`create_absorb_spend`) are still accepted by the on-chain puzzle. Each absorb spend destroys the current singleton coin and creates a new one. Because `ASSERT_HEIGHT_RELATIVE` is evaluated against the **birth height of the coin being spent**, every absorb resets the `relative_lock_height` countdown to zero. A malicious pool operator can repeatedly submit absorb spends just before the lock expires, permanently preventing the farmer from completing the exit transition.

---

### Finding Description

The pool waiting room inner puzzle (`POOL_WAITING_ROOM_MOD`, curried in `create_waiting_room_inner_puzzle`) enforces exit eligibility via `ASSERT_HEIGHT_RELATIVE` with the `relative_lock_height` parameter. This condition requires the singleton coin to be at least `relative_lock_height` blocks old before the escape spend path can be taken. [1](#0-0) 

The `create_absorb_spend` function supports absorb spends in **both** the `FARMING_TO_POOL` and `LEAVING_POOL` (waiting room) states: [2](#0-1) 

Specifically, for the waiting room case: [3](#0-2) 

This absorb spend spends the current singleton coin and creates a new one. The new singleton coin's birth height is the block at which the absorb is confirmed. Because `ASSERT_HEIGHT_RELATIVE` is relative to the **current coin's birth height**, the `relative_lock_height` countdown restarts from zero every time an absorb occurs.

The test suite explicitly confirms that absorb spends succeed while in the waiting room: [4](#0-3) 

And that `ASSERT_HEIGHT_RELATIVE_FAILED` is the error when the lock has not yet expired: [5](#0-4) 

The pool operator controls when to submit absorb spends. As long as there are unabsorbed `p2_singleton` reward coins at the `p2_singleton_puzzle_hash` (which accumulate while the farmer's plots still point to the pool contract address), the pool can submit an absorb spend at any time — including just before the `relative_lock_height` expires — resetting the timer indefinitely. [6](#0-5) 

---

### Impact Explanation

This is a **High** severity finding. A malicious pool operator can permanently prevent a farmer from leaving the pool by front-running the farmer's exit transaction with an absorb spend each time the `relative_lock_height` is about to expire. The farmer's singleton is perpetually locked in `LEAVING_POOL` state, their XCH rewards continue to flow to the pool's `target_puzzle_hash`, and they cannot transition to `SELF_POOLING` or any other pool. This constitutes:

- **Permanent inability for a farmer to process a valid pool state transition** (exit from pool)
- **Pool membership state corruption** — the farmer's singleton is stuck in a state they cannot escape
- **Reward diversion** — farming rewards continue to be directed to the pool's puzzle hash while the farmer is trapped

---

### Likelihood Explanation

The pool operator has both the **capability** and **economic incentive** to execute this attack:

1. The pool knows the `p2_singleton_puzzle_hash` and can monitor unabsorbed reward coins.
2. Farmers typically take weeks to re-plot after initiating a pool exit, so reward coins continue to accumulate.
3. The pool can front-run the farmer's exit transaction (which is visible in the mempool) with an absorb spend, since absorb spends require no signature from the farmer.
4. The cost to the pool is minimal (just transaction fees), while the benefit is retaining the farmer's hashrate and rewards indefinitely. [7](#0-6) 

---

### Recommendation

The waiting room inner puzzle (`POOL_WAITING_ROOM_MOD`) should track the **block height at which the singleton first entered the waiting room** (i.e., the height of the travel spend that transitioned to `LEAVING_POOL`), rather than relying on `ASSERT_HEIGHT_RELATIVE` against the current coin's birth height. One approach is to use `ASSERT_HEIGHT_ABSOLUTE` with the target exit height embedded in the waiting room state, so that absorb spends cannot reset the countdown. Alternatively, absorb spends should be disallowed when the singleton is in `LEAVING_POOL` state. [8](#0-7) 

---

### Proof of Concept

1. Farmer joins pool with `relative_lock_height = 1440`.
2. Farmer initiates exit (`pw_self_pool`), singleton enters `LEAVING_POOL` at block H. The singleton coin is born at block H.
3. At block H + 1439 (one block before the lock expires), the pool submits an absorb spend for any unabsorbed `p2_singleton` reward coin. This is permissionless — no farmer signature required.
4. The absorb spend is confirmed. A new singleton coin is born at block H + 1439. The `ASSERT_HEIGHT_RELATIVE` timer resets.
5. The farmer's exit transaction fails with `ASSERT_HEIGHT_RELATIVE_FAILED`.
6. The pool repeats step 3 every 1439 blocks indefinitely.
7. The farmer is permanently locked in `LEAVING_POOL` state. [9](#0-8) [10](#0-9)

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(
            [
                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    reward_parent: bytes32 = pool_parent_id(height, genesis_challenge)
    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph
    ).to_serialized()
    reward_coin: Coin = Coin(reward_parent, p2_singleton_puzzle.get_tree_hash(), reward_amount)
    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])
    assert p2_singleton_puzzle.get_tree_hash() == reward_coin.puzzle_hash
    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends

    async def claim_pool_rewards(
        self, fee: uint64, max_spends_in_tx: int | None, action_scope: WalletActionScope
    ) -> None:
        # Search for p2_puzzle_hash coins, and spend them with the singleton
        if await self.have_unconfirmed_transaction():
            raise ValueError(
                "Cannot claim due to unconfirmed transaction. If this is stuck, delete the unconfirmed transaction."
            )

        if max_spends_in_tx is None:
            max_spends_in_tx = self.DEFAULT_MAX_CLAIM_SPENDS
        elif max_spends_in_tx <= 0:
            self.log.info(f"Bad max_spends_in_tx value of {max_spends_in_tx}. Set to {self.DEFAULT_MAX_CLAIM_SPENDS}.")
            max_spends_in_tx = self.DEFAULT_MAX_CLAIM_SPENDS

        unspent_coin_records = await self.wallet_state_manager.coin_store.get_unspent_coins_for_wallet(self.wallet_id)
        if len(unspent_coin_records) == 0:
            raise ValueError("Nothing to claim, no transactions to p2_singleton_puzzle_hash")
        farming_rewards: list[TransactionRecord] = await self.wallet_state_manager.tx_store.get_farming_rewards()
        coin_to_height_farmed: dict[Coin, uint32] = {}
        for tx_record in farming_rewards:
            height_farmed: uint32 | None = tx_record.height_farmed(
                self.wallet_state_manager.constants.GENESIS_CHALLENGE
            )
            assert height_farmed is not None
            coin_to_height_farmed[tx_record.additions[0]] = height_farmed
        history: list[tuple[uint32, CoinSpend]] = await self.get_spend_history()
        assert len(history) > 0
        delayed_seconds, delayed_puzhash = get_delayed_puz_info_from_launcher_spend(history[0][1])
        current_state: PoolWalletInfo = await self.get_current_state()
        last_solution: CoinSpend = history[-1][1]

        all_spends: list[CoinSpend] = []
        total_amount = 0

        # The coins being claimed are gathered into the `SpendBundle`, :absorb_spend:
        # We use an announcement in the fee spend to ensure that the claim spend is spent in the same block as the fee
        # We only need to do this for one of the coins, because each `SpendBundle` can only be spent as a unit

        first_coin_record = None
        for coin_record in unspent_coin_records:
            if coin_record.coin not in coin_to_height_farmed:
                continue
            if first_coin_record is None:
                first_coin_record = coin_record
            if len(all_spends) >= max_spends_in_tx:
                # Limit the total number of spends, so the SpendBundle fits into the block
                self.log.info(f"pool wallet truncating absorb to {max_spends_in_tx} spends to fit into block")
                print(f"pool wallet truncating absorb to {max_spends_in_tx} spends to fit into block")
                break
            absorb_spend: list[CoinSpend] = create_absorb_spend(
                last_solution,
                current_state.current,
                current_state.launcher_coin,
                coin_to_height_farmed[coin_record.coin],
                self.wallet_state_manager.constants.GENESIS_CHALLENGE,
                delayed_seconds,
                delayed_puzhash,
            )
            last_solution = absorb_spend[0]
            all_spends += absorb_spend
            total_amount += coin_record.coin.amount
            self.log.info(
                f"Farmer coin: {coin_record.coin} {coin_record.coin.name()} {coin_to_height_farmed[coin_record.coin]}"
            )
        if len(all_spends) == 0 or first_coin_record is None:
            raise ValueError("Nothing to claim, no unspent coinbase rewards")

        claim_spend = WalletSpendBundle(all_spends, G2Element())
