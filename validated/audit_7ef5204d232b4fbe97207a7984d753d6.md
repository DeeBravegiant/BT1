### Title
Pool URL HTTPS Invariant Bypassed via 301/308 Redirect in `_pool_get_pool_info` — (`chia/farmer/farmer.py`)

### Summary
A malicious pool operator can serve a 301/308 redirect from their HTTPS endpoint to an attacker-controlled HTTP URL. The farmer's `update_pool_state` validates HTTPS only on the *original* configured URL, not on the redirect target. The HTTP redirect target is persisted to disk and subsequently used for all pool communications, including partial submissions, which are then sent in plaintext.

### Finding Description
`_pool_get_pool_info` uses `aiohttp` with default redirect-following behavior. When a 301/308 redirect is detected, the final response URL is extracted and returned as `new_pool_url` with no scheme validation:

```python
# farmer.py lines 374-379
if (
    response_url_str != url
    and len(resp.history) > 0
    and all(r.status in {301, 308} for r in resp.history)
):
    new_pool_url = response_url_str.replace("/pool_info", "")
```

`update_pool_state` then writes this URL directly to the persistent config:

```python
# farmer.py lines 619-623
if pool_info_result is not None and pool_info_result.new_pool_url is not None:
    with PoolingShareState.acquire(...) as editable_pool_config:
        editable_pool_config.pool_url = pool_info_result.new_pool_url  # no HTTPS check
```

The HTTPS guard at lines 600–603 only runs against the URL loaded at the *start* of the iteration, not against the redirect target being written. On the subsequent iteration, the in-memory `pool_state[...]["pool_config"]` is updated with the HTTP URL (line 592) before the HTTPS check fires and skips further pool communication. Partial submissions in `farmer_api.py` read `pool_url` from this in-memory state and POST to it without any independent HTTPS enforcement.

### Impact Explanation
- **Pool membership state corruption**: The persisted `pool_url` is permanently set to an HTTP URL, violating the mainnet HTTPS invariant.
- **Plaintext partial submission**: `PostPartialRequest` payloads — containing authentication tokens, BLS signatures, and proofs of space — are transmitted over unencrypted HTTP to the attacker's endpoint. A network observer or the attacker can intercept these.
- **Payout redirection risk**: The attacker controls the pool endpoint receiving partials and can manipulate point accounting, effectively diverting pool rewards.
- **Farmer pooling disruption**: After the HTTP URL is persisted, the HTTPS check on subsequent iterations causes `continue`, halting all pool_info and farmer updates while partials continue to be sent to the HTTP endpoint.

### Likelihood Explanation
Any pool operator the farmer has joined can trigger this by configuring their HTTPS server to issue a 308 redirect to an HTTP URL. No special access beyond operating the pool server is required. The farmer's HTTPS check is bypassed entirely because it only validates the pre-redirect URL.

### Recommendation
Validate the scheme of `new_pool_url` before persisting it, mirroring the existing check:

```python
if pool_info_result is not None and pool_info_result.new_pool_url is not None:
    new_url = pool_info_result.new_pool_url
    if enforce_https and not new_url.startswith("https://"):
        self.log.error(f"Redirect target must be HTTPS on mainnet: {new_url}")
    else:
        with PoolingShareState.acquire(...) as editable_pool_config:
            editable_pool_config.pool_url = new_url
```

### Proof of Concept
1. Configure a pool server at `https://pool.example.com` to respond to `GET /pool_info` with `HTTP 308 Location: http://evil.com/pool_info`.
2. Configure `http://evil.com/pool_info` to return a valid `GetPoolInfoResponse` JSON body.
3. Start a farmer on mainnet with `pool_url = https://pool.example.com`.
4. After one `update_pool_state` cycle, assert that `pooling_share_state.yaml` contains `pool_url: http://evil.com`.
5. After the second `update_pool_state` cycle, assert that `self.farmer.pool_state[p2sh]["pool_config"].pool_url == "http://evil.com"`.
6. Trigger a partial submission and observe via network capture that `POST http://evil.com/partial` is sent in plaintext, containing the authentication token and BLS signature.
