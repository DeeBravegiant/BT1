Audit Report

## Title
First Depositor Can Inflate rsETH Price via Donation to Freeze Subsequent Depositors' Funds - (File: contracts/LRTOracle.sol / contracts/LRTDepositPool.sol)

## Summary

`updateRSETHPrice()` is publicly callable with no access control, and `pricePercentageLimit` defaults to `0`, which disables the price-increase guard entirely. Because `getTotalAssetDeposits` counts raw `address(this).balance` and `IERC20.balanceOf`, an attacker can donate assets to inflate TVL, commit an arbitrarily high `rsETHPrice`, and cause subsequent depositors who pass `minRSETHAmountExpected = 0` to receive 0 rsETH while permanently losing their deposited assets.

## Finding Description

**Root cause 1 — `updateRSETHPrice()` is unrestricted:**

`updateRSETHPrice()` is `public whenNotPaused` with no role check, so any EOA can call it at any time. [1](#0-0) 

**Root cause 2 — `pricePercentageLimit == 0` disables the price-increase guard:**

The guard short-circuits when `pricePercentageLimit` is `0` (its storage default), meaning any arbitrarily large price increase is accepted unconditionally. [2](#0-1) 

**Root cause 3 — TVL uses raw balances, so donations inflate it:**

For LST assets, `assetLyingInDepositPool = IERC20(asset).balanceOf(address(this))`. [3](#0-2) 

For ETH, `ethLyingInDepositPool = address(this).balance`, and the contract has an open `receive()`. [4](#0-3) [5](#0-4) 

**Root cause 4 — `_beforeDeposit` does not enforce `rsethAmountToMint > 0`:**

The only guard is `rsethAmountToMint < minRSETHAmountExpected`. When a caller passes `minRSETHAmountExpected = 0` and the inflated price causes `rsethAmountToMint` to round down to `0`, the condition `0 < 0` is `false` — no revert occurs, and `_mintRsETH(0)` is called. [6](#0-5) [7](#0-6) 

**Price computation uses the stored (manipulated) value:** [8](#0-7) 

## Impact Explanation

A victim who deposits a non-zero amount of ETH or LST and receives 0 rsETH has permanently lost those assets: they hold no rsETH to redeem, and there is no refund path in the contract. This constitutes **direct theft / permanent freezing of deposited funds** — Critical severity under the allowed impact scope.

## Likelihood Explanation

- `updateRSETHPrice()` requires no privilege — any EOA can call it.
- `pricePercentageLimit` is `0` by default (uninitialized storage); the guard is off unless an admin explicitly sets it before the first deposit.
- The open `receive()` and `balanceOf`-based accounting make the donation step trivially executable.
- Passing `minRSETHAmountExpected = 0` is a natural call pattern for users who do not want slippage protection, and is the default in many frontend integrations.
- The attack is repeatable: after each victim deposit, the attacker can call `updateRSETHPrice()` again to re-inflate the price.

## Recommendation

1. **Enforce `rsethAmountToMint > 0` unconditionally** in `_beforeDeposit`:
   ```solidity
   if (rsethAmountToMint == 0) revert ZeroRsETHMinted();
   ```
2. **Set a non-zero default for `pricePercentageLimit`** in `initialize`, or require it to be set before the first deposit is accepted.
3. **Pre-mint a meaningful amount of rsETH** to the protocol treasury at initialization to make the first-depositor attack economically infeasible.

## Proof of Concept

Assume `pricePercentageLimit == 0` (default), `protocolFeeInBPS == 0`, and ETH is a supported asset.

1. Admin calls `updateRSETHPrice()` once to initialize `rsETHPrice = 1 ether` (supply is 0).
2. **Alice** calls `depositETH{value: 1 wei}(0, "")` → `rsethAmountToMint = (1 * 1e18) / 1e18 = 1` → Alice receives 1 wei rsETH.
3. Alice sends 10,000 ETH directly to `LRTDepositPool` (accepted by `receive()`).
4. Alice calls `LRTOracle.updateRSETHPrice()`:
   - `rsethSupply = 1`
   - `totalETHInProtocol = 10_000e18 + 1`
   - `newRsETHPrice = (10_000e18 + 1) / 1 ≈ 10_001e18`
   - `pricePercentageLimit == 0` → `isPriceIncreaseOffLimit = false` → price committed.
5. **Bob** calls `depositETH{value: 10_000e18}(0, "")`:
   - `getRsETHAmountToMint` → `(10_000e18 * 1e18) / 10_001e18 = 0` (integer truncation)
   - `_beforeDeposit`: `0 < 0` is false → no revert
   - `_mintRsETH(0)` → Bob receives 0 rsETH; 10,000 ETH is permanently lost.
6. Alice calls `updateRSETHPrice()` again; her 1 wei rsETH now backs ~20,001 ETH of TVL. She can initiate withdrawal to recover the full pool.
