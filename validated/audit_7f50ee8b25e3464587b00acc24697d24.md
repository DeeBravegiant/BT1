Looking at the exact dispatch logic and the downstream validation chain to determine if this is exploitable.

### Title
P2P Peer Can Inject `compiled_class_hash=None` in `DeclaredClass` Protobuf to Misclassify Sierra Class as Deprecated, Corrupting `compiled_class_hash` Storage — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

The `DataOrFin<StateDiffChunk> TryFrom<protobuf::StateDiffsResponse>` dispatch uses the presence/absence of `compiled_class_hash` in a `DeclaredClass` protobuf message as the sole discriminant between `StateDiffChunk::DeclaredClass` and `StateDiffChunk::DeprecatedDeclaredClass`. A malicious p2p peer can send a `DeclaredClass` message with `compiled_class_hash=None` for a class_hash that was legitimately declared as a Sierra class. The p2p sync client accepts this without verifying the assembled `ThinStateDiff` against the `state_diff_commitment` stored in the block header, writing the corrupted state diff directly to storage. The result is that the class_hash lands in `deprecated_declared_classes` instead of `class_hash_to_compiled_class_hash`, leaving the `compiled_class_hash` table without an entry for that class, breaking all subsequent execution that depends on it.

---

### Finding Description

**Step 1 — The dispatch discriminant.**

The `TryFrom<protobuf::StateDiffsResponse>` for `DataOrFin<StateDiffChunk>` branches solely on `compiled_class_hash.is_some()`: [1](#0-0) 

When `compiled_class_hash` is `None`, the code calls `declared_class.try_into()` targeting `DeprecatedDeclaredClass`, which only extracts `class_hash` and silently discards the fact that the wire message was a `DeclaredClass`: [2](#0-1) 

**Step 2 — `unite_state_diffs` routes to the wrong storage bucket.**

`StateDiffChunk::DeclaredClass` inserts into `class_hash_to_compiled_class_hash`; `StateDiffChunk::DeprecatedDeclaredClass` pushes to `deprecated_declared_classes`: [3](#0-2) 

Both variants return `len() == 1`: [4](#0-3) 

**Step 3 — The only validation in `parse_data_for_block` is a length count.**

After accumulating chunks, the function checks only that `current_state_diff_len == target_state_diff_len` and that deprecated classes are non-conflicting. There is **no verification of the assembled `ThinStateDiff` against the `state_diff_commitment` stored in the block header**: [5](#0-4) 

**Step 4 — The state diff commitment is cryptographically bound to the `declared_classes` vs `deprecated_declared_classes` distinction.**

`calculate_state_diff_hash` hashes `class_hash_to_compiled_class_hash` and `deprecated_declared_classes` as separate, distinct segments: [6](#0-5) 

Moving a class_hash from one segment to the other produces a completely different commitment. The correct commitment is already stored in the block header (synced from the peer), but it is never compared against the assembled diff.

**Step 5 — The header sync path also performs no block-hash or state-diff-commitment verification.**

`HeaderStreamBuilder::parse_data_for_block` only checks that the block number matches; there is no block hash verification and no signature-over-state-diff-commitment check: [7](#0-6) 

**Step 6 — The corrupted `ThinStateDiff` is written directly to storage.**

`write_to_storage` calls `append_state_diff` unconditionally: [8](#0-7) 

`append_state_diff` writes `class_hash_to_compiled_class_hash` into `compiled_class_hash_table`. Because the corrupted diff has the class_hash in `deprecated_declared_classes` instead, the `compiled_class_hash_table` receives no entry for it: [9](#0-8) 

---

### Impact Explanation

After the attack, the node's storage has the targeted class_hash recorded as a deprecated (Cairo 0) class with no `compiled_class_hash` row. Any transaction that invokes a contract whose class was declared as Sierra in that block will fail at the point where the executor looks up the compiled class hash — the entry simply does not exist. This is a **Critical** impact: wrong (absent) compiled class selected for execution, caused by an unauthenticated p2p peer with no special privileges.

---

### Likelihood Explanation

Any node that participates in p2p sync and connects to at least one malicious peer is vulnerable. The attacker needs only to:
1. Serve a valid-length sequence of `StateDiffsResponse` messages for a block that contains at least one Sierra class declaration.
2. Set `compiled_class_hash = None` on the `DeclaredClass` message for that class.

No keys, no signatures, no operator access required. The length check passes because both variants count as 1. The attack is deterministic and requires a single malformed message per targeted class.

---

### Recommendation

In `parse_data_for_block` (or in `write_to_storage`), after assembling the full `ThinStateDiff`, compute `calculate_state_diff_hash(&result)` and compare it against the `state_diff_commitment` already stored in the block header. Reject the peer with `BadPeerError` on mismatch. This is the same pattern already used in `apollo_committer` (`verify_state_diff_hash`): [10](#0-9) 

---

### Proof of Concept

```rust
// Construct a DeclaredClass protobuf with compiled_class_hash = None
let proto = protobuf::StateDiffsResponse {
    state_diff_message: Some(
        protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(
            protobuf::DeclaredClass {
                class_hash: Some(some_sierra_class_hash.into()),
                compiled_class_hash: None,   // <-- attacker-controlled omission
            }
        )
    )
};

let chunk: DataOrFin<StateDiffChunk> = proto.try_into().unwrap();

// Assert: variant is DeprecatedDeclaredClass, NOT DeclaredClass
assert!(matches!(
    chunk.0.unwrap(),
    StateDiffChunk::DeprecatedDeclaredClass(_)   // wrong bucket
));
// The class_hash will be pushed to deprecated_declared_classes,
// compiled_class_hash_table will have no entry, execution will break.
```

    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);

    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
