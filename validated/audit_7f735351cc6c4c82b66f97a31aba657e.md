Audit Report

## Title
`removeLiquidity` silently skips LP token transfer when floor-rounding produces zero external amount, causing permanent LP principal loss — (File: metric-core/contracts/libraries/LiquidityLib.sol)

## Summary
`LiquidityLib.removeLiquidity` decrements both per-bin scaled balances and pool-level `binTotals` before converting the scaled removal amount to external token units via floor division. When the scaled amount is smaller than `token0ScaleMultiplier` (e.g., `< 10^12` for USDC), the floor division yields zero and the `if (amount0Removed > 0)` guard silently skips the ERC-20 transfer. The LP's shares are permanently burned and the pool's internal accounting is decremented, but no tokens are returned; the stranded balance is later swept as protocol/admin spread fees via `collectFees`.

## Finding Description
In `removeLiquidity`, state mutation precedes the external transfer:

**Step 1 — bin accounting and shares decremented (state mutated first):** [1](#0-0) 

**Step 2 — pool totals decremented:** [2](#0-1) 

**Step 3 — floor-division conversion to external units:** [3](#0-2) 

`_deltasScaledToExternal` with `Math.Rounding.Floor` performs integer division that can produce zero: [4](#0-3) 

**Step 4 — silent skip:** [5](#0-4) 

When `totalToken0ToRemoveScaled < token0ScaleMultiplier`, `amount0Removed = 0` and the transfer branch is never entered. No revert occurs; the `LiquidityRemoved` event fires with `amount0Removed = 0`, which is indistinguishable from a legitimate zero-token-0 removal.

The stranded tokens inflate the pool's ERC-20 balance relative to `binTotals.scaledToken0`. `collectFees` computes spread-fee surplus as `balance0() * TOKEN_0_SCALE_MULTIPLIER - binTotals.scaledToken0 - notionalFees`, so the stranded amount is swept to admin/protocol: [6](#0-5) 

The asymmetry is intentional on the add side (ceil protects the pool) but absent on the remove side: [7](#0-6) 

The scale multiplier for USDC/USDT (6 decimals) is `10^12`: [8](#0-7) 

## Impact Explanation
This is a direct loss of LP principal with no revert and no distinguishable on-chain signal. The LP burns shares and receives zero tokens; the stranded balance is permanently out of reach and is eventually swept as protocol/admin fees. Maximum loss per call is up to `(token0ScaleMultiplier − 1)` scaled units — up to ~1 USDC per `removeLiquidity` call for USDC pools. Calls are repeatable by any LP on any USDC/USDT pool, accumulating losses. USDC and USDT are explicitly in scope per the allowed-impact gate.

## Likelihood Explanation
No special permissions are required. Any LP calling `removeLiquidity` on a USDC/USDT pool triggers this path when the bin's scaled token0 balance at removal time is `< 10^12` scaled units (i.e., `< 1 USDC` in external units). This is a normal post-swap state: swaps consume token0 from bins, and a bin that started with 1 USDC can be traded down to any sub-unit scaled balance. The condition is reachable by any unprivileged LP after ordinary swap activity.

## Recommendation
After `_deltasScaledToExternal` with `Floor`, add a guard that reverts if a non-zero scaled amount rounded to zero:

```solidity
if (totalToken0ToRemoveScaled > 0 && amount0Removed == 0) {
    revert RoundedToZero();
}
if (totalToken1ToRemoveScaled > 0 && amount1Removed == 0) {
    revert RoundedToZero();
}
```

Alternatively, use `Math.Rounding.Ceil` on the remove path (accepting that the pool may transfer 1 extra token unit per call), or accumulate sub-unit remainders in a per-position credit claimable later.

## Proof of Concept
1. Deploy a pool with USDC (6 decimals) as `token0`; `TOKEN_0_SCALE_MULTIPLIER = 10^12`.
2. LP calls `addLiquidity` on a bin above the current bin, paying 1 USDC. The bin receives `10^12` scaled units.
3. Swaps execute, consuming token0 from that bin until `binState.token0BalanceScaled = 5 × 10^11` (0.5 USDC in scaled units).
4. LP calls `removeLiquidity` to burn all shares in that bin.
   - `amount0Scaled = 5 × 10^11 * sharesToRemove / binTotalSharesVal = 5 × 10^11` (100% of shares).
   - `binState.token0BalanceScaled` decremented by `5 × 10^11`. ✓ (state mutated)
   - `binTotals.scaledToken0` decremented by `5 × 10^11`. ✓ (state mutated)
   - `amount0Removed = 5 × 10^11 / 10^12 = 0`. ← floor rounds to zero
   - `if (0 > 0)` → **transfer skipped silently**.
5. LP's USDC balance is unchanged. The 0.5 USDC is permanently stranded in the pool.
6. On next `collectFees`, the 0.5 USDC appears as surplus and is swept to admin/protocol.
7. `LiquidityRemoved` event fires with `amount0Removed = 0` — no on-chain indication of the loss.
