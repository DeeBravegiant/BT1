Audit Report

## Title
Unrestricted `claim()` Allows Any Caller to Force-Claim on Behalf of Any Account, Permanently Denying Fee-Free Yield — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary
`MerkleDistributor.claim()` accepts an arbitrary `account` parameter with no `msg.sender == account` restriction, allowing any external caller to trigger a claim for any account at the current `feeInBPS`. Because `isClaimed` permanently marks the claim as settled once executed, a victim who was waiting for a fee reduction loses the fee delta permanently — the tokens are transferred to `protocolTreasury` and cannot be recovered.

## Finding Description
The `claim` function at L97–106 is `external` with no caller restriction:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
```

The fee is computed at the **current** `feeInBPS` at call time (L138):

```solidity
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
```

After execution, `userClaims[account].lastClaimedIndex` is updated (L134), causing `isClaimed(index, account)` to return `true` (L93), and any subsequent call reverts with `AlreadyClaimed` (L115–117). The owner can reduce `feeInBPS` to 0 at any time via `setFeeInBPS` (L198–206). Merkle proofs are public off-chain data; an attacker needs no privileged access to construct a valid call. The exploit path is:

1. `feeInBPS = 1000` (10%); victim holds unclaimed yield of `X` tokens.
2. Owner announces or submits a transaction to set `feeInBPS = 0`.
3. Attacker observes the pending transaction and front-runs it by calling `claim(index, victim, X, proof)`.
4. Victim receives `0.9X`; `0.1X` is sent to `protocolTreasury`.
5. Owner's `setFeeInBPS(0)` executes — but `isClaimed` is already `true` for the victim.
6. Victim cannot re-claim; the `0.1X` fee delta is permanently lost.

No existing guard prevents this: the only checks are `whenNotPaused`, `InvalidIndex`, `InvalidMerkleProof`, and `AlreadyClaimed` — none of which restrict the caller identity.

## Impact Explanation
**High — Theft of unclaimed yield.** The victim permanently loses up to `MAX_FEE_IN_BPS / 10_000 = 10%` of their total unclaimed yield. The lost amount is diverted to `protocolTreasury`, not returned to the victim. This matches the allowed impact "High. Theft of unclaimed yield" exactly. The loss is bounded and calculable: `claimableAmount * feeInBPS / 10_000`.

## Likelihood Explanation
- Merkle proofs are standard public off-chain data; the attacker requires no privileged access.
- The attacker only needs to monitor the mempool or governance announcements for a pending `setFeeInBPS` call.
- The call costs only gas; the attacker holds no tokens and takes no financial risk.
- Fee reductions are a plausible and intended operational event (the `setFeeInBPS` function exists for this purpose).
- The attack is repeatable across all unclaimed accounts in the same Merkle tree.

## Recommendation
Add a caller restriction to `claim()` so only the account itself (or an explicitly approved delegate) may trigger a claim:

```solidity
function claim(
    uint256 index,
    address account,
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    require(msg.sender == account, "MerkleDistributor: caller is not account");
    // ... rest of logic
}
```

If third-party claiming is a desired feature (e.g., for gas relayers), implement an EIP-712 signed-delegation pattern so the account explicitly authorizes a specific caller and fee rate.

## Proof of Concept
Foundry fork-test outline:

```solidity
function testForceClaimDeniesFeeFreeYield() public {
    // Setup: feeInBPS = 1000, victim has 1000e18 claimable, valid merkle proof available
    uint256 victimInitialBalance = token.balanceOf(victim);

    // Attacker front-runs owner's setFeeInBPS(0) tx
    vm.prank(attacker); // attacker != victim, no special privileges
    distributor.claim(currentIndex, victim, 1000e18, proof);

    // Owner's fee reduction executes after
    vm.prank(owner);
    distributor.setFeeInBPS(0);

    // Victim attempts to re-claim — reverts
    vm.prank(victim);
    vm.expectRevert(IMerkleDistributor.AlreadyClaimed.selector);
    distributor.claim(currentIndex, victim, 1000e18, proof);

    // Assert: victim received 900e18, lost 100e18 to treasury
    assertEq(token.balanceOf(victim) - victimInitialBalance, 900e18);
    assertEq(token.balanceOf(protocolTreasury), 100e18);
    // Had victim self-claimed after fee=0, they would have received 1000e18
}
```

The yield difference equals exactly `feeInBPS * claimableAmount / 10_000 = 100e18`, confirming permanent loss with no recovery path.
