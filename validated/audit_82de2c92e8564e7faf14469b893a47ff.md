### Title
Proposer Can Set Arbitrary `fee_proposal_fri` During Startup Window, Permanently Corrupting `fee_actual` and `l2_gas_price` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid`, the `fee_proposal_fri` bounds check is gated on `proposal_init_validation.fee_actual` being `Some`. During the first `fee_proposal_window_size` (= **10**) blocks, `fee_actual` is always `None`, so the bounds check is entirely skipped. A malicious proposer can place any `fee_proposal_fri` value — including `u128::MAX` — into `ProposalInit` during this window. Every validator accepts it (they all see `fee_actual = None`), the value is committed into `fee_proposals_window`, and once the window fills the corrupted entries become the median (`fee_actual`) that permanently floors `l2_gas_price` for all subsequent blocks.

---

### Finding Description

**Root cause — `validate_proposal.rs` lines 396–416:**

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
```

The `if let` pattern silently skips the entire check when `fee_actual` is `None`. [1](#0-0) 

`fee_actual` is `None` whenever `height < fee_proposal_window_size`. The window size is **10** in every shipped versioned-constants file: [2](#0-1) 

The validator constructs `ProposalInitValidation.fee_actual` by calling `compute_fee_actual` at the same height, so every honest validator independently arrives at `None` for the same 10-block window — they will all accept any `fee_proposal_fri` value the proposer supplies. [3](#0-2) 

At decision time, the committed `fee_proposal_fri` is unconditionally recorded into `fee_proposals_window`:

```rust
self.update_l2_gas_price(height, l2_gas_used);
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [4](#0-3) 

`record_fee_proposal` inserts the value with no further validation: [5](#0-4) 

Once height ≥ 10, `compute_fee_actual` computes the **median** of the 10 stored values. If the attacker controlled ≥ 6 of those 10 slots, the median is the attacker's value. [6](#0-5) 

That median becomes `fee_actual`, which is used as the **floor** for `l2_gas_price` in every subsequent block:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [7](#0-6) 

An honest proposer freezes at `l2_gas_price` when `fee_actual` is `None`, but a malicious proposer is not constrained to do so: [8](#0-7) 

---

### Impact Explanation

If a malicious proposer controls ≥ 6 of the first 10 blocks (possible when the validator set is small at genesis, or when the network restarts after a revert that clears the window), they can set `fee_proposal_fri = u128::MAX` for those blocks. After block 10, `fee_actual` = `u128::MAX`, which becomes the permanent floor for `l2_gas_price`. All user transactions whose `max_price_per_unit` is below `u128::MAX` are rejected by the gateway and mempool, effectively halting the chain economically. Even a partial attack (controlling 3–4 of 10 slots) shifts the median upward, inflating fees for all users by an amount proportional to the attacker's share of the window.

The corrupted `fee_proposal_fri` is also hashed into the `ProposalCommitment` via `Poseidon(partial_block_hash, fee_proposal_fri)`, so the wrong value is permanently committed on-chain and cannot be corrected without a chain revert. [9](#0-8) 

---

### Likelihood Explanation

The window is exactly 10 blocks. At genesis or after any chain revert that resets the window, the first 10 proposers face zero bounds enforcement. In a single-sequencer deployment (the current production topology for Starknet), the sequencer is the only proposer for all 10 blocks and can trivially exploit this. In a multi-validator setup, a coalition controlling > 50 % of the first 10 proposal slots achieves the same result. The `fee_proposal_margin_ppt` of 2 ppt (0.2 %) means that once the window is poisoned, the per-block correction rate is too slow to recover quickly.

---

### Recommendation

1. **Enforce an absolute maximum on `fee_proposal_fri` unconditionally**, regardless of whether `fee_actual` is available. A configurable `max_fee_proposal_fri` (e.g., derived from `max_l2_gas_price`) should be checked in `is_proposal_init_valid` before the `fee_actual`-gated bounds check.

2. **During the startup window, clamp `fee_proposal_fri` to `[min_gas_price, some_max]`** rather than accepting any value. The honest proposer already freezes at `l2_gas_price` when `fee_actual` is `None`; the validator should enforce the same constraint.

3. **Pre-populate the window from committed chain history** before entering consensus (the `initialize_fee_proposals_window` path already exists for restarts — ensure it is always called, including at genesis with synthetic minimum-price entries).

---

### Proof of Concept

1. Network starts at height 0; `fee_proposals_window` is empty; `fee_actual = None` for heights 0–9.
2. Malicious proposer crafts `ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), starknet_version: V0_14_3, ... }` for blocks 0–9.
3. Each validator calls `is_proposal_init_valid`. The `if let (Some(fee_actual), Some(fee_proposal))` pattern evaluates to `(None, Some(...))` — the inner block is **never entered**; the proposal passes. [10](#0-9) 
4. Consensus commits each block; `record_fee_proposal(height, Some(GasPrice(u128::MAX)))` is called 10 times. [5](#0-4) 
5. At height 10, `compute_fee_actual` sorts the 10-entry window `[u128::MAX × 10]`; median = `u128::MAX`. [11](#0-10) 
6. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX`; `l2_gas_price` is pinned at `u128::MAX` for all future blocks. [7](#0-6) 
7. All user transactions fail gateway admission (`max_price_per_unit < l2_gas_price`); the chain is economically halted.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
```
