### Title
Fee-on-Transfer Token Accounting Divergence in `initTransfer` — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary
`initTransfer` in `OmniBridge.sol` accepts any ERC-20 token via `safeTransferFrom` and immediately emits an `InitTransfer` event recording the caller-supplied `amount`. For fee-on-transfer tokens the contract receives strictly less than `amount`, but the event — which drives minting/release on the destination chain — records the full requested value. This creates unbacked wrapped supply.

### Finding Description
In `initTransfer`, the non-bridge, non-custom-minter ERC-20 path is:

```solidity
} else {
    IERC20(tokenAddress).safeTransferFrom(
        msg.sender,
        address(this),
        amount          // ← caller-controlled, not verified against actual receipt
    );
}
``` [1](#0-0) 

Immediately after, the event is emitted with the same unverified `amount`:

```solidity
emit BridgeTypes.InitTransfer(
    msg.sender, tokenAddress, currentOriginNonce,
    amount, fee, nativeFee, recipient, message
);
``` [2](#0-1) 

No pre-balance snapshot is taken before the transfer and no post-balance check is performed to confirm `balanceOf(address(this))` increased by exactly `amount`. For a fee-on-transfer token the contract receives `amount - fee_amount` while the event (and therefore the NEAR-side proof) records `amount`.

### Impact Explanation
The `InitTransfer` event is the authoritative source for the NEAR bridge to mint or release tokens on the destination chain. If the event overstates the locked amount, the destination chain credits more tokens than are actually held in the EVM vault. Repeated bridging of a fee-on-transfer token progressively undercollateralizes the bridge, eventually allowing redemptions that cannot be backed by real locked tokens — a direct asset-backing guarantee violation.

This maps to the allowed impact: **High — asset-identity, balance-accounting divergence that breaks backing guarantees or sends value to the wrong party.**

### Likelihood Explanation
Any unprivileged user can call `initTransfer` with any ERC-20 address. Fee-on-transfer tokens (e.g., USDT on some chains, reflection tokens, tokens with protocol fees) are common in the wild. No special role or leaked key is required; the attacker simply calls the public function with such a token. Likelihood is **Medium** — it requires a fee-on-transfer token to be routed through the bridge, which is a realistic scenario given the bridge accepts arbitrary ERC-20 addresses.

### Recommendation
Capture the contract's token balance before and after the `safeTransferFrom` call and use the delta as the canonical locked amount:

```solidity
uint256 balanceBefore = IERC20(tokenAddress).balanceOf(address(this));
IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
uint256 actualReceived = IERC20(tokenAddress).balanceOf(address(this)) - balanceBefore;
require(actualReceived == amount, "ERR_FEE_ON_TRANSFER_TOKEN");
// or: use actualReceived in place of amount for the event and downstream logic
``` [1](#0-0) 

### Proof of Concept
1. Deploy or identify a fee-on-transfer ERC-20 token (e.g., 1% fee on every transfer) that is not registered as a `isBridgeToken` or `customMinters` entry.
2. Call `OmniBridge.initTransfer(tokenAddress, 1000, 0, 0, "near:alice.near", "")`.
3. The contract receives 990 tokens (after 1% fee) but emits `InitTransfer(..., amount=1000, ...)`.
4. The NEAR bridge processes the proof derived from this event and mints/releases 1000 units to `alice.near`.
5. The EVM vault is now short by 10 tokens per such call. After N calls the vault is undercollateralized by `N * 10` tokens, enabling redemptions that cannot be fulfilled. [3](#0-2)

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

        initTransferExtension(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue
        );

        emit BridgeTypes.InitTransfer(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message
        );
    }
