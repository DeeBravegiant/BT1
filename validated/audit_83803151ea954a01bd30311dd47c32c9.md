Audit Report

## Title
Stale `rsETHPrice` After Downside-Pause Early Return Causes Protocol to Miss Fees on Post-Pause Yield — (`contracts/LRTOracle.sol`)

## Summary
When `_updateRsETHPrice()` triggers the downside-protection pause via `isPriceDecreaseOffLimit`, it returns early at line 281 without writing `newRsETHPrice` to `rsETHPrice`. The stored `rsETHPrice` is left at the pre-drop (inflated) value. On the next call after unpausing, `previousTVL` is computed from this stale price, so genuine yield accrued while the protocol was paused generates zero protocol fee.

## Finding Description
Inside `_updateRsETHPrice()`:

**Step 1** — `previousTVL` is anchored to the stored (stale) price: [1](#0-0) 

**Step 2** — The fee gate compares live TVL against this baseline: [2](#0-1) 

**Step 3** — If the drop exceeds `pricePercentageLimit`, the protocol pauses and returns early: [3](#0-2) 

**Step 4** — The write that never happens when the early return fires: [4](#0-3) 

After unpausing, `protocolPaused` at line 240 becomes `false` once all three contracts are unpaused: [5](#0-4) 

At that point, `previousTVL = rsethSupply × stale_rsETHPrice` (e.g. 1050 ETH) exceeds `totalETHInProtocol` (e.g. 950 ETH after yield accrual on a slashed base of 900 ETH), so `protocolFeeInETH` remains 0 and all yield accrued during the pause is invisible to the fee mechanism.

## Impact Explanation
The protocol treasury fails to collect fees on all yield accrued between the pause event and the point (if ever) where TVL recovers past the stale `previousTVL` baseline. No user funds are lost, but the protocol fails to deliver its promised fee revenue. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

## Likelihood Explanation
The trigger condition (`isPriceDecreaseOffLimit`) is an intended production path activated by any slashing event or significant LST de-peg that moves the price beyond `pricePercentageLimit`. Yield continues to accrue on remaining assets during the pause. The subsequent fee-miss is automatic and requires no attacker action — it fires on the first `updateRSETHPrice()` call after the admin unpauses all three contracts.

## Recommendation
Before the early return, write the computed `newRsETHPrice` to `rsETHPrice` so the baseline is accurate when the protocol resumes:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
+   rsETHPrice = newRsETHPrice;   // anchor baseline to actual drop level
    _pause();
    return;
}
```

## Proof of Concept

```solidity
// 1. Setup: rsETHPrice = 1.05e18, rsethSupply = 1000e18, TVL = 1050e18
// 2. Simulate slashing: underlying asset value drops → totalETHInProtocol = 900e18
// 3. Call updateRSETHPrice() → isPriceDecreaseOffLimit fires, early return
//    → rsETHPrice still == 1.05e18 (stale)
// 4. vm.warp(block.timestamp + 7 days)  // yield accrues during pause
// 5. Simulate yield: totalETHInProtocol = 950e18
// 6. Unpause oracle, lrtDepositPool, and withdrawalManager
// 7. Call updateRSETHPrice()
//    → previousTVL = 1000e18 * 1.05e18 / 1e18 = 1050e18
//    → totalETHInProtocol (950e18) < previousTVL (1050e18)
//    → protocolFeeInETH == 0
// 8. Assert: protocolFeeInETH == 0  ✓ (bug confirmed)
//    Expected: protocolFeeInETH > 0 (50e18 of yield should generate fees)
```
