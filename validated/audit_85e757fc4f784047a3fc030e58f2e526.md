Audit Report

## Title
Token Deposit Fee Defaults to Zero in RSETHPool, Allowing Fee-Free Token Deposits - (File: contracts/pools/RSETHPool.sol)

## Summary

`RSETHPool.sol` maintains a separate `tokenFeeBps` mapping for per-token fee rates. Because Solidity mappings default to zero and `addSupportedToken` never initializes `tokenFeeBps[token]`, any user can call `deposit(token, amount, referralId)` immediately after a token is added and receive rsETH with zero fee deducted. The protocol permanently loses all fee revenue from token deposits until the admin separately calls `setTokenFeeBps`.

## Finding Description

`RSETHPool` declares a dedicated per-token fee mapping at line 88: [1](#0-0) 

The `viewSwapRsETHAmountAndFee(amount, token)` function reads this mapping directly to compute the fee: [2](#0-1) 

The `addSupportedToken` function adds a token to the pool but never writes to `tokenFeeBps[token]`, leaving it at the Solidity default of `0`: [3](#0-2) 

The `deposit(token, amount, referralId)` function calls `viewSwapRsETHAmountAndFee` and accumulates the returned fee, but since `tokenFeeBps[token] == 0`, the fee is always zero and nothing is collected: [4](#0-3) 

The only remedy is a separate admin call to `setTokenFeeBps`, which is gated behind `DEFAULT_ADMIN_ROLE` — a different role from `TIMELOCK_ROLE` used by `addSupportedToken` — with no enforcement or fallback: [5](#0-4) 

By contrast, `RSETHPoolV3` uses the single shared `feeBps` for both ETH and token deposits, so it is not affected: [6](#0-5) 

## Impact Explanation

Any user can deposit supported tokens (e.g., wstETH) into the Arbitrum `RSETHPool` and receive the full rsETH amount with zero fee deducted. The protocol collects zero fee revenue from all token deposits for the entire window between `addSupportedToken` and an explicit `setTokenFeeBps` call. Depositors effectively receive the protocol's fee revenue as extra rsETH — constituting **theft of unclaimed yield (High)**.

## Likelihood Explanation

The zero-fee condition is triggered automatically every time a new token is added via `addSupportedToken`. There is no atomic mechanism to set the fee at token-addition time, and no default fallback to `feeBps`. Any depositor can observe the on-chain `AddSupportedToken` event and immediately begin depositing fee-free. The admin must remember to issue a separate `setTokenFeeBps` call with no on-chain enforcement; the window is open from the moment the token is added until that call is made.

## Recommendation

Initialize `tokenFeeBps[token]` to `feeBps` (or an explicit fee parameter) inside `addSupportedToken`, ensuring newly added tokens inherit the protocol's default fee rate atomically:

```solidity
function addSupportedToken(address token, address oracle, address bridge) external onlyRole(TIMELOCK_ROLE) {
    // ... existing checks ...
    supportedTokenList.push(token);
    supportedTokenOracle[token] = oracle;
    tokenBridge[token] = bridge;
+   tokenFeeBps[token] = feeBps; // initialize to default fee
    emit AddSupportedToken(token, oracle, bridge);
}
```

Alternatively, modify `viewSwapRsETHAmountAndFee(amount, token)` to fall back to `feeBps` when `tokenFeeBps[token]` is `0`.

## Proof of Concept

1. Admin calls `addSupportedToken(wstETH, oracle, bridge)` — `tokenFeeBps[wstETH]` remains `0`.
2. User calls `deposit(wstETH, 1_000e18, "ref")`.
3. `viewSwapRsETHAmountAndFee(1_000e18, wstETH)` computes:
   - `feeBpsForToken = tokenFeeBps[wstETH] = 0`
   - `fee = 1_000e18 * 0 / 10_000 = 0`
   - `amountAfterFee = 1_000e18`
4. `feeEarnedInToken[wstETH] += 0` — no fee collected.
5. User receives the full rsETH amount with zero fee deducted.
6. Protocol loses all fee revenue from wstETH deposits until admin calls `setTokenFeeBps(wstETH, X)`.

**Foundry test plan:**
```solidity
function test_tokenDepositZeroFee() public {
    // Setup: add wstETH with feeBps = 50 already set on pool
    pool.addSupportedToken(address(wstETH), address(oracle), address(bridge));
    // tokenFeeBps[wstETH] is 0 — never initialized
    assertEq(pool.tokenFeeBps(address(wstETH)), 0);

    uint256 amount = 1_000e18;
    deal(address(wstETH), user, amount);
    vm.prank(user);
    wstETH.approve(address(pool), amount);
    vm.prank(user);
    pool.deposit(address(wstETH), amount, "ref");

    // Fee collected should be 0
    assertEq(pool.feeEarnedInToken(address(wstETH)), 0);
}
```
