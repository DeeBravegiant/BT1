### Title
Instantaneous UTxO Stake Snapshot at Epoch Boundary Enables DRep Voting-Power Manipulation for Governance Ratification - (File: eras/conway/impl/src/Cardano/Ledger/Conway/Governance.hs)

---

### Summary

In the Conway era, the `DRepPulser` is initialized at every epoch boundary using the **instantaneous** `InstantStake` (live UTxO stake) at that exact moment. An unprivileged actor who controls a registered DRep can submit a transaction in the last block of an epoch that creates large UTxO outputs staked to their DRep credential, inflating that DRep's computed voting weight for the entire following epoch's ratification run. After the epoch boundary is processed the attacker spends those outputs, recovering the capital. Because DRep stake weight directly determines whether governance actions (treasury withdrawals, protocol-parameter changes, hard-fork initiations, committee updates, no-confidence motions) cross their ratification threshold, this is a direct path to enacting unauthorized governance actions.

---

### Finding Description

`setFreshDRepPulsingState` is called at the end of `epochTransition` (Conway `EPOCH` rule). It reads the live `InstantStake` from `UTxOState` and stores it verbatim as `dpInstantStake` inside the new `DRepPulser`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Governance.hs
instantStake = utxoState ^. instantStakeG          -- line 482
...
dpInstantStake = instantStake  -- used as part of the snapshot  -- line 501
```

`computeDRepDistr` then iterates over every account and looks up each credential's stake in `dpInstantStake` to build the DRep distribution:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Governance/DRepPulser.hs
mInstantStake = Map.lookup stakeCred instantStakeCredentials   -- line 214
stakeAndDeposits = fold $ mInstantStake <> mProposalDeposit    -- line 216
...
updatedDistr = Map.insertWith (<>) dRep (stakeAndDeposits <> balance) distr  -- line 232
```

The resulting `reDRepDistr` is fed directly into `dRepAcceptedRatio` inside the `RATIFY` rule:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Ratify.hs
dRepAcceptedRatio RatifyEnv {reDRepDistr, ...} gasDRepVotes govAction =
  toInteger yesStake %? toInteger totalExcludingAbstainStake   -- line 259
```

There is no time-averaging, no stability-window delay, and no use of an older snapshot (the Shelley "go" snapshot is two epochs old; the DRep distribution uses the freshest possible value). The `dpStakePoolDistr` for SPO voting does use `ssStakeMarkPoolDistr` from the SNAP rule, but that snapshot is itself also taken from the same instantaneous stake at the same epoch boundary, so both distributions share the same manipulation surface.

**Attack steps:**

1. Attacker registers a stake credential and delegates it to their own DRep (or an existing DRep they control) in any earlier transaction.
2. In the **last block of epoch N**, the attacker submits a transaction that creates one or more large UTxO outputs carrying that stake credential (e.g., by temporarily acquiring ADA off-chain or by consolidating existing holdings).
3. The epoch-N/epoch-(N+1) boundary is processed. `setFreshDRepPulsingState` reads `instantStake`, which now includes the attacker's inflated UTxO stake. The `DRepPulser` is initialized with this value.
4. During epoch N+1, `computeDRepDistr` assigns the inflated stake to the attacker's DRep. `dRepAcceptedRatio` returns a ratio that crosses the governance threshold for a pending action the attacker wants ratified (e.g., a `TreasuryWithdrawals` action that already has committee approval).
5. The governance action is enacted by `ratifyTransition` → `ENACT`.
6. In epoch N+1, the attacker spends the inflated UTxO outputs, recovering the capital.

---

### Impact Explanation

`dRepAcceptedRatio` is the sole gating function for DRep ratification of `TreasuryWithdrawals`, `ParameterChange`, `NewConstitution`, `HardForkInitiation`, `UpdateCommittee`, and `NoConfidence` actions. Crossing the threshold causes `acceptedByEveryone` to return `True`, which triggers `trans @(EraRule "ENACT" era)` — permanently enacting the action on-chain. Possible consequences include:

- Unauthorized treasury drain (`TreasuryWithdrawals`)
- Unauthorized protocol-parameter change (`ParameterChange`)
- Unauthorized hard-fork initiation (`HardForkInitiation`)
- Unauthorized committee replacement or no-confidence motion

All of these fall under **Critical: Unauthorized governance, treasury, protocol-parameter, committee, constitution, or hard-fork action is enacted.**

---

### Likelihood Explanation

- **No special privilege required**: any holder of ADA can register a stake credential, delegate to a DRep they control, and create UTxO outputs.
- **Capital cost**: the attacker must hold enough ADA to shift the DRep ratio above the threshold for one epoch (~5 days). For a treasury withdrawal targeting a small treasury or a governance action where the attacker's DRep already holds near-threshold stake, the required capital may be modest relative to the gain.
- **Timing**: the attacker must land their transaction in the last block of an epoch. Block producers are known in advance (VRF-based slot leadership), so the attacker can predict the last slot and submit accordingly.
- **Coordination**: for actions requiring committee approval, the attacker must also have committee votes. For `TreasuryWithdrawals` and `ParameterChange`, committee approval is the only other gate; SPO approval is not required for those action types.
- **Recovery**: the attacker recovers the capital in the next epoch, making the net cost only transaction fees and the opportunity cost of ~5 days of capital lock-up.

---

### Recommendation

Use the existing **"mark" snapshot** (taken at the epoch boundary and aged through the mark→set→go pipeline) for the DRep stake distribution, consistent with how SPO stake is used for leader election. Concretely, `dpInstantStake` in `DRepPulser` should be replaced with the resolved active stake from `ssStakeMark` (or `ssStakeSet`/`ssStakeGo` for stronger delay), so that UTxO outputs created in the last block of an epoch do not immediately inflate DRep voting weight. Alternatively, require that stake delegated to a DRep must have been present for at least one full epoch before it counts toward ratification, mirroring the stability-window protection already applied to SPO leader election.

**Epoch N boundary processing (`epochTransition`):**
- `SNAP` runs → `ssStakeMark` includes attacker's 1B ADA
- `setFreshDRepPulsingState` runs → `dpInstantStake` includes attacker's 1B ADA

- Pending `TreasuryWithdrawals` action with attacker's DRep voting Yes
- `dRepAcceptedRatio` = `(honest_yes + 1B) / (total_active - abstain)` → crosses threshold
- `acceptedByEveryone` = True → `ENACT` fires → treasury drained

  Map.foldlWithKey' go (dRepDistr, poolDistr)

      let instantStakeCredentials = instantStake ^. instantStakeCredentialsL
          mInstantStake = Map.lookup stakeCred instantStakeCredentials
          mProposalDeposit = Map.lookup stakeCred proposalDeposits
          stakeAndDeposits = fold $ mInstantStake <> mProposalDeposit
       in ( addToDRepDistr accountState stakeAndDeposits drepAccum
          , addToPoolDistr accountState mProposalDeposit poolAccum
          )
    addToPoolDistr accountState mProposalDeposit distr = fromMaybe distr $ do
      stakePool <- accountState ^. stakePoolDelegationAccountStateL
      proposalDeposit <- mProposalDeposit
      ips <- Map.lookup stakePool $ distr ^. poolDistrDistrL
      pure $
        distr
          & poolDistrDistrL %~ Map.insert stakePool (ips & individualTotalPoolStakeL <>~ proposalDeposit)
          & poolDistrTotalL %~ \t -> unsafeNonZero (unNonZero t <> fromCompact proposalDeposit)

```haskell
dRepAcceptedRatio ::
  forall era.
  RatifyEnv era ->
  Map (Credential DRepRole) Vote ->
  GovAction era ->
  Rational
dRepAcceptedRatio RatifyEnv {reDRepDistr, reDRepState, reCurrentEpoch} gasDRepVotes govAction =
  toInteger yesStake %? toInteger totalExcludingAbstainStake
  where
    accumStake (!yes, !tot) drep (CompactCoin stake) =
      case drep of
        DRepCredential cred ->
          case Map.lookup cred reDRepState of
            Nothing -> (yes, tot) -- drep is not registered, so we don't consider it
            Just drepState
              | reCurrentEpoch > drepExpiry drepState -> (yes, tot) -- drep is expired, so we don't consider it
              | otherwise ->
                  case Map.lookup cred gasDRepVotes of
                    -- drep hasn't voted for this action, so we don't count
                    -- the vote but we consider it in the denominator:
                    Nothing -> (yes, tot + stake)
                    Just VoteYes -> (yes + stake, tot + stake)
                    Just Abstain -> (yes, tot)
                    Just VoteNo -> (yes, tot + stake)
        DRepAlwaysNoConfidence ->
          case govAction of
            NoConfidence _ -> (yes + stake, tot + stake)
            _ -> (yes, tot + stake)
        DRepAlwaysAbstain -> (yes, tot)
    (yesStake, totalExcludingAbstainStake) = Map.foldlWithKey' accumStake (0, 0) reDRepDistr
```
