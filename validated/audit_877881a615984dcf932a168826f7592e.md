Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Auto-Pause via `updateRSETHPrice()` — (`contracts/LRTOracle.sol`)

## Summary

`updateRSETHPrice()` in `LRTOracle.sol` is declared `public whenNotPaused` with no role guard, allowing any address to invoke it. When the computed rsETH price falls more than `pricePercentageLimit` below `highestRsethPrice`, the internal `_updateRsETHPrice()` unconditionally calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on the oracle itself, freezing all user deposits and withdrawals without any privileged authorization.

## Finding Description

`updateRSETHPrice()` at `contracts/LRTOracle.sol:87` carries only a `whenNotPaused` modifier:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`_updateRsETHPrice()` at lines 270–282 contains the downside-protection branch:

```solidity
if (newRsETHPrice < highestRsethPrice) {
    uint256 diff = highestRsethPrice - newRsETHPrice;
    bool isPriceDecreaseOffLimit =
        pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);

    if (isPriceDecreaseOffLimit) {
        if (!lrtDepositPool.paused()) lrtDepositPool.pause();
        if (!withdrawalManager.paused()) withdrawalManager.pause();
        _pause();
        return;
    }
```

For this auto-pause mechanism to function at all, `LRTOracle` must hold `PAUSER_ROLE` on both `LRTDepositPool` (whose `pause()` at line 349 requires `onlyRole(LRTConstants.PAUSER_ROLE)`) and `LRTWithdrawalManager` (whose `pause()` at line 347 has the same guard). This is a necessary deployment precondition for the feature to work. Given that precondition, any unprivileged caller can reach the three-contract pause path the moment the price condition is satisfied.

The existence of `updateRSETHPriceAsManager()` at lines 94–96 — which is `onlyLRTManager` — confirms the design intent that sensitive price updates should be role-restricted, yet the public entry point bypasses this entirely.

## Impact Explanation

**Medium — Temporary freezing of funds.**

Once the auto-pause fires:
- `LRTDepositPool.depositETH()` and `depositAsset()` revert on `whenNotPaused`.
- `LRTWithdrawalManager.initiateWithdrawal()`, `completeWithdrawal()`, and `instantWithdrawal()` revert on `whenNotPaused`.
- `LRTOracle.updateRSETHPrice()` itself becomes uncallable.

All user funds are frozen until an admin holding `DEFAULT_ADMIN_ROLE` manually calls `unpause()` on each of the three contracts. The attacker spends only gas; no capital is required. This matches the allowed impact "Medium. Temporary freezing of funds."

## Likelihood Explanation

**Medium.** The trigger condition — rsETH price dropping more than `pricePercentageLimit` below `highestRsethPrice` — is realistic under EigenLayer slashing events or LST de-pegs (stETH, ETHx) that reduce `totalETHInProtocol`. If `pricePercentageLimit` is set conservatively (e.g., 1%), even routine reward-cycle fluctuations can satisfy the condition. Once the condition is met, the attack requires a single permissionless transaction. An attacker can monitor on-chain state and call `updateRSETHPrice()` before any operator response.

## Recommendation

Add a role guard to `updateRSETHPrice()` so that the auto-pause path is only reachable by authorized callers:

```solidity
// Option A: restrict the public entry point to operators
function updateRSETHPrice() public whenNotPaused onlyLRTOperator {
    _updateRsETHPrice();
}

// Option B: decouple the auto-pause trigger from price reads entirely
// and expose it as a separate, role-gated function
```

Alternatively, remove the auto-pause logic from `_updateRsETHPrice()` and expose it as a standalone role-gated function so that price reads and emergency pausing are independently controlled.

## Proof of Concept

1. Deploy with `highestRsethPrice = 1.05e18`, `pricePercentageLimit = 1e16` (1%).
2. A slashing event reduces `totalETHInProtocol`; `_getTotalEthInProtocol()` now returns a value yielding `newRsETHPrice = 1.03e18`.
3. `diff = 1.05e18 - 1.03e18 = 0.02e18 > 0.01 × 1.05e18 = 0.0105e18` → `isPriceDecreaseOffLimit = true`.
4. Attacker calls `LRTOracle.updateRSETHPrice()` (no role required, oracle not yet paused).
5. `_updateRsETHPrice()` executes lines 278–280: `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()`.
6. All deposits and withdrawals revert for every user until an admin manually unpauses all three contracts.

**Foundry test sketch:**
```solidity
function test_unprivilegedCallerTriggersAutoPause() public {
    // Setup: price condition satisfied
    vm.mockCall(...); // mock _getTotalEthInProtocol to return low value
    address attacker = makeAddr("attacker");
    vm.prank(attacker);
    lrtOracle.updateRSETHPrice(); // no role required
    assertTrue(lrtDepositPool.paused());
    assertTrue(withdrawalManager.paused());
    assertTrue(lrtOracle.paused());
}
```
