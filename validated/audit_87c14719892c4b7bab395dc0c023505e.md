Audit Report

## Title
Zero-rsETHAmount Deposit Silently Consumes Depositor ETH — (`contracts/pools/RSETHPoolV2NBA.sol`)

## Summary

`RSETHPoolV2NBA.deposit` accepts any non-zero ETH value but does not guard against `rsETHAmount` computing to zero via integer truncation in `viewSwapRsETHAmountAndFee`. When a deposit is small enough that `amountAfterFee * 1e18 < rsETHToETHrate`, `rsETHAmount` evaluates to zero, `wrsETH.mint` succeeds silently minting nothing, and the deposited ETH is retained in the pool. The next `moveAssetsForBridging` call sweeps that ETH to the bridger, permanently removing it from the depositor's reach with no recourse.

## Finding Description

`deposit` enforces only `amount == 0`: [1](#0-0) 

`viewSwapRsETHAmountAndFee` performs two truncating integer divisions: [2](#0-1) 

For a 1-wei deposit with `feeBps = 100` and `rsETHToETHrate = 1.05e18`: `fee = 1 * 100 / 10_000 = 0`, `amountAfterFee = 1`, `rsETHAmount = 1 * 1e18 / 1.05e18 = 0`. The result `(0, 0)` is returned without error.

`wrsETH.mint` delegates directly to OpenZeppelin's `ERC20Upgradeable._mint`: [3](#0-2) 

OpenZeppelin's `_mint` does not revert on `amount = 0`; it emits `Transfer(address(0), to, 0)` and returns normally. The call at line 115 therefore succeeds silently, minting nothing to the depositor.

Because `fee` also rounds to zero, `feeEarnedInETH` is not incremented. The deposited ETH is included in `ethBalanceMinusFees` and swept to the bridger on the next call: [4](#0-3) 

Once swept, the ETH is irrecoverable by the depositor.

## Impact Explanation

A depositor who sends any sub-threshold ETH amount (any value where `amountAfterFee * 1e18 < rsETHToETHrate`) loses their ETH permanently: no wrsETH is issued, no revert occurs, and the ETH is swept to L1 on the next bridging call. This constitutes **Permanent freezing of funds** (Critical) — the depositor's ETH is permanently removed from their control with no on-chain recovery mechanism.

## Likelihood Explanation

No special role or permission is required; `deposit` is a public payable function callable by any EOA or contract. With a typical rate of ~1.05e18, a deposit of exactly 1 wei always triggers the bug. The minimum safe deposit is `ceil(rsETHToETHrate / 1e18)` wei of `amountAfterFee`, which is a non-trivial threshold that is not communicated to users. Automated scripts, bots, or users testing with dust amounts can trigger this without any front-running or external dependency.

## Recommendation

Add a post-computation guard in `deposit` that reverts if `rsETHAmount == 0`:

```solidity
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);
if (rsETHAmount == 0) revert InvalidAmount();
```

Alternatively, enforce a minimum deposit amount at the entry point derived from the current oracle rate.

## Proof of Concept

```solidity
// Preconditions:
//   feeBps = 100 (1%)
//   rsETHToETHrate = 1.05e18 (oracle returns 1.05 ETH per rsETH)

// 1. Depositor calls deposit{value: 1}("")
// 2. amount = 1, passes `amount == 0` check
// 3. viewSwapRsETHAmountAndFee(1):
//      fee = 1 * 100 / 10_000 = 0
//      amountAfterFee = 1 - 0 = 1
//      rsETHAmount = 1 * 1e18 / 1.05e18 = 0  ← truncated
// 4. feeEarnedInETH += 0  (unchanged)
// 5. wrsETH.mint(depositor, 0)  ← no-op, no revert
// 6. 1 wei sits in address(this).balance; depositor holds 0 wrsETH
// 7. moveAssetsForBridging() sweeps 1 wei to bridger
//    ethBalanceMinusFees = balance(1) - feeEarnedInETH(0) = 1
//    depositor has no recourse

// Foundry fuzz invariant:
// for all amount in [1, rsETHToETHrate / 1e18]:
//   deposit{value: amount} MUST either revert OR return rsETHAmount > 0
// Currently NEITHER holds — the call succeeds and rsETHAmount == 0.
```

    function deposit(string memory referralId) external payable nonReentrant whenNotPaused {
        uint256 amount = msg.value;

        if (amount == 0) revert InvalidAmount();

        (uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

        feeEarnedInETH += fee;

        wrsETH.mint(msg.sender, rsETHAmount);

        emit SwapOccurred(msg.sender, rsETHAmount, fee, referralId);
    }

**File:** contracts/pools/RSETHPoolV2NBA.sol (L124-133)
```text
    function viewSwapRsETHAmountAndFee(uint256 amount) public view returns (uint256 rsETHAmount, uint256 fee) {
        fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
    }
```
