Audit Report

## Title
Permissionless `updateRSETHPrice()` Enables Any Caller to Trigger Protocol-Wide Pause - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is declared `public` with only a `whenNotPaused` guard. When the computed rsETH price falls more than `pricePercentageLimit` below `highestRsethPrice`, the function autonomously calls `lrtDepositPool.pause()` and `withdrawalManager.pause()`, freezing all user deposits and withdrawals. Because the function is permissionless, any external account can trigger this protocol-wide pause whenever the price condition is met, without any privileged access.

## Finding Description
`LRTOracle._updateRsETHPrice()` contains the following automatic pause propagation block:

```solidity
// contracts/LRTOracle.sol L277-282
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

This is reached from the public entry point:

```solidity
// contracts/LRTOracle.sol L87-89
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

`LRTDepositPool.pause()` is guarded by `onlyRole(LRTConstants.PAUSER_ROLE)`. For the oracle's internal call to succeed, `LRTOracle` must hold `PAUSER_ROLE` in `LRTConfig` — a prerequisite the protocol must satisfy for the safety mechanism to function at all. Once `LRTOracle` holds that role (as the design requires), any unprivileged caller can invoke `updateRSETHPrice()`, satisfy the price-drop condition, and cause `LRTOracle` to pause both `LRTDepositPool` and `LRTWithdrawalManager` on their behalf.

`LRTDepositPool.depositETH()` and `depositAsset()` both carry `whenNotPaused`, so they revert for all users once the pause is active. The pause can only be lifted by an admin calling `unpause()` — no user action can reverse it.

The `onlyRole` modifier resolves roles against `lrtConfig` using `msg.sender`, so when `LRTOracle` calls `lrtDepositPool.pause()`, the role check is performed against the oracle contract address, not the original external caller.

## Impact Explanation
Any user who calls `updateRSETHPrice()` while the price-drop condition holds triggers an immediate, protocol-wide pause of `LRTDepositPool` and `LRTWithdrawalManager`. All deposits and withdrawals are frozen until an admin manually unpauses. Additionally, `L1Vault.depositETHForL1VaultETH()` — which calls `lrtDepositPool.depositETH()` — also reverts during this window, meaning ETH already bridged from L2 to L1 cannot be converted to rsETH and bridged back, leaving the L2 wrsETH partially unbacked for the duration of the freeze.

**Impact class**: Medium — Temporary freezing of funds.

## Likelihood Explanation
The trigger condition (rsETH price dropping beyond `pricePercentageLimit`) is realistic: EigenLayer slashing events, a supported LST depeg, or a sudden drop in underlying asset oracle prices can all cause the computed rsETH price to fall. Once conditions are met, any unprivileged user can call the public `updateRSETHPrice()` to lock the protocol. No admin collusion, key compromise, or special role is required. The condition is observable on-chain, so a griefing actor can monitor and call the function the moment the threshold is breached.

## Recommendation
- Restrict `updateRSETHPrice()` to authorized callers (e.g., `MANAGER` or `OPERATOR` role), mirroring the existing `updateRSETHPriceAsManager()` pattern, so that the automatic pause cannot be triggered by arbitrary users.
- Alternatively, separate the price-update logic from the pause-propagation logic: emit an event on threshold breach and let a separate, access-controlled function execute the pause, preventing a single public call from atomically freezing the protocol.

## Proof of Concept
1. A slashing event or LST depeg causes on-chain TVL to drop, making `newRsETHPrice` fall more than `pricePercentageLimit` below `highestRsethPrice` (verified at `LRTOracle.sol` L270-274).
2. Any unprivileged user calls `LRTOracle.updateRSETHPrice()` (L87).
3. Inside `_updateRsETHPrice()`, `isPriceDecreaseOffLimit` evaluates to `true` (L273-274).
4. The oracle calls `lrtDepositPool.pause()` (L278) — succeeds because `LRTOracle` holds `PAUSER_ROLE`.
5. The oracle calls `withdrawalManager.pause()` (L279).
6. All subsequent calls to `LRTDepositPool.depositETH()` (L76-93) and `depositAsset()` (L99-118) revert with `Pausable: paused`.
7. `L1Vault.depositETHForL1VaultETH()` also reverts, leaving bridged ETH stranded.
8. The freeze persists until an admin calls `lrtDepositPool.unpause()` (L354) — no user action can lift it.

**Foundry fork test plan**: Deploy a fork, set `pricePercentageLimit` to a non-zero value, manipulate an underlying asset oracle to drop the computed rsETH price beyond the threshold, call `updateRSETHPrice()` from an EOA with no roles, assert `lrtDepositPool.paused() == true`, then assert `depositETH()` reverts.
