Audit Report

## Title
Single Reverting Asset Price Oracle Permanently Blocks All rsETH Price Updates — (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle._getTotalEthInProtocol()` calls `getAssetPrice(asset)` for every supported asset inside an unbounded loop with no `try/catch` or skip logic. If any one asset's price oracle reverts — due to a deprecated Chainlink aggregator, sequencer downtime, or a zeroed oracle address — the entire `updateRSETHPrice()` call reverts. The stored `rsETHPrice` becomes stale, protocol fee minting is blocked, and the price-drop auto-pause cannot fire until an admin manually replaces the broken oracle.

## Finding Description

`_getTotalEthInProtocol()` is a private function called by every price-update path:

```
updateRSETHPrice() (public, whenNotPaused)
updateRSETHPriceAsManager() (onlyLRTManager)
  └─ _updateRsETHPrice()
       └─ _getTotalEthInProtocol()
            └─ for each supportedAsset:
                 getAssetPrice(asset)   ← hard revert, no try/catch
```

The loop at lines 336–348 calls `getAssetPrice(asset)` unconditionally: [1](#0-0) 

`getAssetPrice` is guarded by `onlySupportedOracle`, which reverts when `assetPriceOracle[asset] == address(0)`: [2](#0-1) 

And the underlying `IPriceFetcher.getAssetPrice(asset)` call can itself revert: [3](#0-2) 

There is no `try/catch`, no `continue`, and no fallback. A single failing oracle for any supported asset causes the entire loop — and therefore the entire price update — to revert. The two downstream consequences are:

1. **Fee minting blocked**: `IRSETH.mint(treasury, ...)` inside `_updateRsETHPrice()` is never reached. [4](#0-3) 

2. **Auto-pause blocked**: The downside-protection pause that halts `LRTDepositPool` and `LRTWithdrawalManager` on a large price drop also lives inside `_updateRsETHPrice()` and is therefore also blocked. [5](#0-4) 

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield**: Protocol fees are minted inside `_updateRsETHPrice()`. If the function cannot complete, accrued yield is never minted to the treasury. This persists indefinitely until an admin calls `updatePriceOracleFor` with a working replacement oracle — there is no automatic recovery path in the contract.

**Low — Contract fails to deliver promised returns**: The stored `rsETHPrice` becomes stale. All depositors and withdrawers transact at an incorrect exchange rate for the entire duration of the oracle outage.

## Likelihood Explanation

No attacker action is required. The protocol supports multiple LST assets (stETH, ETHx, etc.), each backed by its own price oracle adapter. Chainlink feeds are documented to revert under real-world conditions: L2 sequencer downtime, deprecated aggregators, and stale-price circuit breakers. With N supported assets, the probability that at least one oracle is temporarily unavailable grows with N. This is not an "incorrect data" scenario (excluded by SECURITY.md) — it is an oracle *revert*, a distinct and well-known failure mode that the protocol's own code fails to handle.

## Recommendation

Wrap the `getAssetPrice` call in a `try/catch` so that a single failing oracle does not block the entire update:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    try this.getAssetPrice(asset) returns (uint256 assetER) {
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    } catch {
        emit AssetPriceOracleFailed(asset); // alert off-chain monitoring
        // optionally revert if the failing asset has non-negligible TVL
    }
    unchecked { ++assetIdx; }
}
```

Alternatively, revert only when the failing asset's TVL contribution exceeds a configurable threshold, and skip it otherwise.

## Proof of Concept

1. Deploy with two supported assets: `stETH` (oracle healthy) and `ETHx` (oracle reverts — e.g., Chainlink aggregator deprecated after L2 sequencer restart).
2. Call `updateRSETHPrice()` from any unprivileged address.
3. `_getTotalEthInProtocol()` processes `stETH` successfully, then calls `getAssetPrice(ETHx)`.
4. The `ETHx` oracle reverts. The entire transaction reverts.
5. `rsETHPrice` is never updated. Protocol fees are never minted. The price-drop auto-pause cannot fire.
6. This state persists until an admin calls `updatePriceOracleFor(ETHx, newOracle)`.

**Foundry fork test plan**:
- Fork mainnet/L2 at a block where a Chainlink aggregator is deprecated or the sequencer feed is down.
- Register the broken feed as the price oracle for one supported asset.
- Call `lrtOracle.updateRSETHPrice()` and assert it reverts.
- Confirm `rsETHPrice` is unchanged and no `FeeMinted` event was emitted.
