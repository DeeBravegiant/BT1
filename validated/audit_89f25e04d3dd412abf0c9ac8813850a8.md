### Title
Pool Waiting Room Absorb Spend Resets `relative_lock_height` Countdown, Enabling Malicious Pool to Permanently Block Farmer Exit — (`chia/pools/pool_puzzles.py`)

---

### Summary

When a pool singleton is in the `LEAVING_POOL` (waiting room) state, the `pool_waitingroom_innerpuz` supports two spend paths: an **exit** path (spend_type=1) that enforces `ASSERT_HEIGHT_RELATIVE relative_lock_height` blocks since the singleton coin's creation, and an **absorb** path (spend_type=0) that claims a farming reward without requiring any signature from the farmer. The absorb path spends the current singleton coin and creates a **new** singleton coin. Because `ASSERT_HEIGHT_RELATIVE` is measured from the **coin being spent**, each absorb resets the countdown to zero. A malicious pool operator can repeatedly submit absorb spends using legitimate farming rewards to reset the countdown indefinitely, permanently preventing the farmer from exiting the pool.

---

### Finding Description

`create_absorb_spend` in `chia/pools/pool_puzzles.py` constructs the waiting-room absorb spend as:

```python
elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
    inner_sol = Program.to([0, reward_amount, height])
``` [1](#0-0) 

This is spend_type=0 (absorb/claim). The spend bundle is submitted with `G2Element()` — an empty/null signature — confirming no farmer authorization is required:

```python
coin_db.update_coin_store_for_spend_bundle(
    SpendBundle(coin_sols, G2Element()), time, DEFAULT_CONSTANTS.MAX_BLOCK_COST_CLVM
)
``` [2](#0-1) 

The absorb spends the current singleton coin and creates a new one. The exit path enforces `ASSERT_HEIGHT_RELATIVE relative_lock_height`, which is measured from the **creation height of the coin being spent** — i.e., the new singleton coin created by the absorb. This is confirmed by the test structure: after absorbing at block 3, the exit is attempted at block 10000 (well past 3 + 5000), and it succeeds. If the pool absorbs again at block 4999, the countdown resets to 4999 + 5000 = 9999. [3](#0-2) 

The waiting room puzzle is curried with `relative_lock_height` as a fixed parameter:

```python
return POOL_WAITING_ROOM_MOD.curry(
    target_puzzle_hash, p2_singleton_puzzle_hash, bytes(owner_pubkey), pool_reward_prefix, relative_lock_height
)
``` [4](#0-3) 

The exit condition (`ASSERT_HEIGHT_RELATIVE`) is enforced only on the exit spend path, not on the absorb path. The absorb path creates a new singleton coin at the current block, silently resetting the countdown.

The pool operator has all the public information needed to construct the absorb spend (`last_coin_spend`, `current_state`, `launcher_coin`, `height`, `genesis_challenge`, `delay_time`, `delay_ph`) — all readable from the blockchain. No private farmer key is needed. [5](#0-4) 

---

### Impact Explanation

A malicious pool operator can:

1. Monitor the blockchain for new `p2_singleton` farming rewards created by the farmer's plots.
2. Submit an absorb spend before the `relative_lock_height` countdown expires, creating a new singleton coin at the current block and resetting the countdown.
3. Repeat indefinitely as long as the farmer's plots continue to farm blocks.

The farmer's singleton remains permanently in `LEAVING_POOL` state. Their farming rewards continue to be directed to the pool's `target_puzzle_hash` against their will. The farmer cannot redirect rewards to themselves or another pool.

This constitutes:
- **Unauthorized payout redirection**: farming rewards continue flowing to the pool after the farmer declared intent to leave.
- **Permanent DOS of a protected singleton state transition**: the farmer's exit from the pool is blocked indefinitely.

This matches the **High** impact category: *Bypass of pool authorization that enables payout redirection and permanent inability for farmers to execute pool actions under normal network assumptions.*

---

### Likelihood Explanation

- The pool operator controls the absorb timing and has the infrastructure to monitor the blockchain.
- No farmer signature or private key is required.
- All inputs to `create_absorb_spend` are public blockchain data.
- As long as the farmer's plots are active, new `p2_singleton` rewards are created, giving the pool material to absorb.
- The attack is most effective for farmers with higher plot counts (more frequent rewards) and pools with higher `relative_lock_height` values (up to the maximum of 1000 blocks, ~14 hours). [6](#0-5) 

---

### Recommendation

The `pool_waitingroom_innerpuz` should anchor the `ASSERT_HEIGHT_RELATIVE` countdown to the **block at which the farmer first entered the waiting room** (the initial travel spend height), not the creation height of the current singleton coin. One approach: record the entry block height in the singleton's state when transitioning to `LEAVING_POOL`, and enforce `ASSERT_HEIGHT_ABSOLUTE (entry_height + relative_lock_height)` in the exit path instead of `ASSERT_HEIGHT_RELATIVE`. This ensures absorb spends cannot reset the countdown.

---

### Proof of Concept

From `chia/_tests/pools/test_pool_puzzles_lifecycle.py`:

1. Farmer enters `LEAVING_POOL` at block 1 (`travel_coinsol`). Singleton coin created at block 1.
2. Immediate exit attempt at block 1 fails: `ASSERT_HEIGHT_RELATIVE_FAILED` (need block 1 + 5000 = 5001).
3. Pool absorbs a reward at block 3 (no signature required, `G2Element()`). New singleton coin created at block 3.
4. Farmer now needs block 3 + 5000 = 5003 to exit.
5. Pool absorbs again at block 5002. New singleton coin created at block 5002.
6. Farmer now needs block 5002 + 5000 = 10002 to exit.
7. Pool repeats indefinitely — farmer never exits. [7](#0-6) [8](#0-7)

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(
            [
                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    reward_parent: bytes32 = pool_parent_id(height, genesis_challenge)
    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph
    ).to_serialized()
    reward_coin: Coin = Coin(reward_parent, p2_singleton_puzzle.get_tree_hash(), reward_amount)
    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])
    assert p2_singleton_puzzle.get_tree_hash() == reward_coin.puzzle_hash
    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends
