Audit Report

## Title
Missing `try/catch` on Chainlink `latestRoundData()` Freezes `updateRSETHPrice()` and Blocks Deposits — (`contracts/LRTOracle.sol`)

## Summary

`_getTotalEthInProtocol()` iterates over all supported assets and calls `getAssetPrice()` with no error handling. A revert from any single asset's Chainlink price feed (e.g., stale data, sequencer downtime, deprecated feed) propagates unconditionally through the entire call stack, causing `updateRSETHPrice()` to revert on every invocation and leaving `rsETHPrice` permanently stale. Deposits of the affected asset are also blocked for the duration of the outage.

## Finding Description

The full call chain contains no `try/catch` at any level:

**`_getTotalEthInProtocol()` loops over all supported assets** (`LRTOracle.sol:336–339`):
```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);   // no error handling
``` [1](#0-0) 

**`getAssetPrice()` delegates directly to the registered `IPriceFetcher`** with no guard:
```solidity
function getAssetPrice(address asset) public view onlySupportedOracle(asset) returns (uint256) {
    return IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset);
}
``` [2](#0-1) 

**`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` with no `try/catch`**:
```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [3](#0-2) 

If `latestRoundData()` reverts (stale feed, paused aggregator, L2 sequencer down), the revert bubbles through `getAssetPrice()` → `_getTotalEthInProtocol()` → `_updateRsETHPrice()` → `updateRSETHPrice()`. The `rsETHPrice` state variable is never updated and becomes permanently stale until the oracle recovers.

**Secondary effect on deposits:** `getRsETHAmountToMint()` calls `lrtOracle.getAssetPrice(asset)` directly: [4](#0-3) 

`_beforeDeposit()` calls `getRsETHAmountToMint()`, so `depositAsset()` / `depositETH()` for the affected asset also reverts: [5](#0-4) 

## Impact Explanation

| Effect | Scope |
|---|---|
| `updateRSETHPrice()` always reverts | All assets — entire price update mechanism frozen |
| `rsETHPrice` becomes stale | Protocol-wide exchange rate incorrect |
| `depositAsset()` / `depositETH()` reverts | Only for the asset whose oracle is reverting |

The price update freeze is protocol-wide: even assets with healthy oracles cannot get their contribution reflected in `rsETHPrice` because the loop aborts on the first failing oracle. Deposits of the affected asset are blocked for the duration of the outage. This constitutes **Medium — Temporary freezing of funds**.

## Likelihood Explanation

Chainlink feeds are documented to revert under several non-adversarial conditions: deprecated/paused aggregators, L2 sequencer downtime (Arbitrum, Optimism), and integrator-side staleness checks. The protocol supports multiple LST assets, each with its own Chainlink feed; any single feed experiencing a routine outage triggers this condition. No privileged action or attacker is required — the condition arises from normal infrastructure behavior.

## Recommendation

Wrap each per-asset oracle call in a `try/catch` inside `_getTotalEthInProtocol()`. On failure, either skip the asset using its last known price as a fallback, or revert with a specific error identifying the failing asset:

```solidity
try IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset) returns (uint256 price) {
    totalETHInProtocol += totalAssetAmt.mulWad(price);
} catch {
    revert OracleFailedForAsset(asset);
}
```

Similarly, `getAssetPrice()` used in `getRsETHAmountToMint()` should propagate a typed error so callers can distinguish oracle failures from other reverts.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;
import "forge-std/Test.sol";

contract RevertingOracle {
    function getAssetPrice(address) external pure returns (uint256) {
        revert("stale");
    }
}

contract OracleFreezeTest is Test {
    function testOracleFreezesPriceUpdate() public {
        // 1. Fork mainnet with LRTOracle deployed
        // 2. As LRTAdmin, replace one asset oracle with RevertingOracle
        vm.prank(lrtAdmin);
        lrtOracle.updatePriceOracleFor(assetA, address(new RevertingOracle()));

        // 3. updateRSETHPrice() now always reverts
        vm.expectRevert();
        lrtOracle.updateRSETHPrice();

        // 4. rsETHPrice is stale (unchanged)
        assertEq(lrtOracle.rsETHPrice(), stalePriceSnapshot);

        // 5. Deposits of assetA are blocked
        vm.expectRevert();
        depositPool.depositAsset(assetA, 1 ether, 0, "");

        // 6. Deposits of assetB succeed but use stale rsETHPrice
        depositPool.depositAsset(assetB, 1 ether, 0, "");
    }
}
```
