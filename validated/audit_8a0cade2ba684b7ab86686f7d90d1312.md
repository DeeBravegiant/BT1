Looking at the ENS M-13 bug class: **unchecked user-supplied field that bypasses the only guard and flows into a privileged operation**. The Sequencer analog is `ProposalInit.builder` — a proposer-supplied address that is never validated in `is_proposal_init_valid` yet flows directly into `sequencer_address` in `BlockInfo`, which drives fee collection and the block hash commitment.

Let me trace the exact path before writing the report.

The `is_proposal_init_valid` function checks: timestamp bounds, `starknet_version`, `version_constant_commitment`, `height`/`l1_da_mode`/`l2_gas_price_fri` (combined), all four L1 gas prices within margin, and `fee_proposal_fri` presence/bounds. The `ProposalInitValidation` struct has no `builder` field — it is never checked.

`convert_to_sn_api_block_info` maps `init.builder` → `sequencer_address` unconditionally. `PartialBlockHashComponents::new` then maps `block_info.sequencer_address` → `sequencer` in the partial block hash. `calculate_block_hash` chains `sequencer` into the Poseidon hash. Both the proposer and every validator use the same `init.builder` value, so they agree on the (wrong) commitment — the `ProposalFin` comparison does not catch it.

### Title
Unchecked `ProposalInit.builder` Lets a Malicious Proposer Redirect All Block Fees and Corrupt the Committed Sequencer Address — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit.builder` is a proposer-supplied field that is never validated by `is_proposal_init_valid`. It flows unchanged into `sequencer_address` inside `BlockInfo`, which is the address that receives transaction fees and is committed into the partial block hash. A validator selected as proposer can set `builder` to any address, causing every validator to execute the block with the wrong sequencer address, redirect all fees to the attacker, and finalize a block hash that permanently commits the wrong sequencer.

### Finding Description

`ProposalInit` carries two address fields:

- `proposer` — the consensus participant who proposed the block
- `builder` — "Address of the one who builds/sequences the block" [1](#0-0) 

The proposer sets `builder` from its own node configuration: [2](#0-1) 

The validator receives `ProposalInit` over the network and calls `is_proposal_init_valid`. The `ProposalInitValidation` struct — the sole source of locally-trusted reference values — contains no `builder` field: [3](#0-2) 

`is_proposal_init_valid` checks height, timestamp, `starknet_version`, `version_constant_commitment`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri`. It never reads `init_proposed.builder`: [4](#0-3) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [5](#0-4) 

That `BlockInfo` is forwarded to the batcher as `ValidateBlockInput.block_info`. The batcher executes every transaction in the block with `sequencer_address = init.builder`. After execution, `PartialBlockHashComponents::new` reads `block_info.sequencer_address` and stores it as `sequencer`: [6](#0-5) 

`calculate_block_hash` chains `sequencer` into the Poseidon hash: [7](#0-6) 

Because both the proposer and every validator derive `sequencer_address` from the same `init.builder`, they compute identical (but wrong) `PartialBlockHash` values. The `ProposalFin` commitment comparison at line 244 of `validate_proposal.rs` passes, and the block is finalized with the attacker's address as sequencer. [8](#0-7) 

### Impact Explanation

**Fee theft (Critical — economic impact):** In Starknet, transaction fees are transferred to `sequencer_address`. A malicious proposer who sets `builder = attacker_address` causes every fee paid in that block to be credited to the attacker rather than the legitimate sequencer. The attacker keeps the full block reward at no extra cost beyond being selected as proposer.

**Permanent block hash corruption (High):** The finalized block hash commits `attacker_address` as the sequencer. This value is stored on-chain and used as `previous_block_hash` for subsequent blocks. The chain permanently records the wrong sequencer for that block.

**Wrong `get_sequencer_address` syscall result (High):** Any contract that calls `get_sequencer_address` during execution of that block receives `attacker_address`, potentially breaking contracts that gate logic on the sequencer identity.

### Likelihood Explanation

The attacker must be a legitimate validator that is selected as proposer for a round. In Tendermint-style consensus this is a deterministic, rotating role — every validator will eventually be proposer. No external capability is required beyond normal validator participation. The attack is silent: no error is raised, no metric fires, and the `ProposalFin` commitment check passes because both sides use the same unchecked `builder` value.

### Recommendation

Add `builder` (and optionally `proposer`) to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`:

```rust
pub(crate) struct ProposalInitValidation {
    pub height: BlockNumber,
    pub builder: ContractAddress,   // add: locally-trusted builder address
    // ... existing fields
}
```

In `is_proposal_init_valid`, reject any proposal whose `init_proposed.builder` differs from `proposal_init_validation.builder`. The validator already knows its own builder address (it is part of node configuration); this value should be populated when constructing `ProposalInitValidation` in `validate_proposal` inside `sequencer_consensus_context.rs`.

Alternatively, enforce `builder == proposer` as a protocol invariant, since in the current implementation the proposer always sets `builder` to its own address.

### Proof of Concept

```
1. Attacker is a legitimate validator and is selected as proposer for height H, round R.

2. Attacker constructs ProposalInit with:
     builder = <attacker_controlled_address>   // any address
     height  = H                               // correct
     l2_gas_price_fri = <expected>             // correct
     l1_gas_price_fri = <within margin>        // correct
     ... all other checked fields correct ...

3. Attacker streams ProposalInit + transaction batches + ProposalFin to validators.

4. Each validator calls is_proposal_init_valid:
     - height ✓, timestamp ✓, starknet_version ✓, l1_da_mode ✓,
       l2_gas_price_fri ✓, L1 prices ✓, fee_proposal_fri ✓
     - builder: NOT CHECKED → passes

5. Each validator calls convert_to_sn_api_block_info(init):
     sequencer_address = init.builder = <attacker_controlled_address>

6. Batcher executes all transactions with sequencer_address = attacker.
   All fees are credited to attacker.

7. PartialBlockHashComponents.sequencer = attacker_address.
   Both proposer and validators compute the same PartialBlockHash.
   ProposalFin comparison passes.

8. Block is finalized. Chain permanently records attacker as sequencer for block H.
   Attacker has collected all fees from block H.
```

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
