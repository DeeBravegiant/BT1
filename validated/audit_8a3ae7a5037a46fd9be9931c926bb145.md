Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns ETH/USD `updatedAt` Instead of Composite rsETH/USD Staleness Timestamp — (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed` implements `AggregatorV3Interface` and computes a composite rsETH/USD `answer` from two independent sources: `ETH/USD` (Chainlink) and `rsETH/ETH` (`LRTOracle.rsETHPrice`). However, the `updatedAt` field returned by both `latestRoundData()` and `getRoundData()` is taken exclusively from the ETH/USD Chainlink feed, with no timestamp tracked for the `rsETH/ETH` component. Any downstream consumer applying a standard Chainlink staleness guard on `updatedAt` will pass the check even when `rsETHPrice` is arbitrarily stale.

## Finding Description
In `contracts/oracles/RSETHPriceFeed.sol` lines 63–70, `latestRoundData()` destructures all five return values from `ETH_TO_USD.latestRoundData()` — including `updatedAt` — and then overwrites only `answer` with the composite price:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

The same pattern appears in `getRoundData()` at lines 53–61. The `updatedAt` value is never adjusted to account for the freshness of `RS_ETH_ORACLE.rsETHPrice()`.

In `contracts/LRTOracle.sol` line 28, `rsETHPrice` is stored as a plain `uint256` with no associated timestamp:

```solidity
uint256 public override rsETHPrice;
```

The internal `_updateRsETHPrice()` function (lines 214–316) writes `rsETHPrice = newRsETHPrice` at line 313 but records no `lastUpdatedAt`. The public entry point `updateRSETHPrice()` (lines 87–89) is permissionless but requires an explicit external call — there is no on-chain mechanism that forces it to be called on any schedule.

As a result, `updatedAt` returned by `RSETHPriceFeed` reflects only when the ETH/USD Chainlink heartbeat last fired, not when the composite rsETH/USD price was last valid. A downstream protocol that checks `block.timestamp - updatedAt < staleness_threshold` will pass the check even if `rsETHPrice` has not been refreshed for days.

## Impact Explanation
**Low — Contract fails to deliver promised returns.**

`RSETHPriceFeed` advertises itself as a Chainlink-compatible `AggregatorV3Interface` feed. The `updatedAt` field in that interface is the canonical signal downstream consumers use to detect stale prices. Returning an `updatedAt` that reflects only the ETH/USD component while the rsETH/ETH component may be arbitrarily stale means the contract fails to deliver the promised semantics of the interface. Downstream lending protocols that use this feed for collateral valuation will operate on a stale composite price without triggering their staleness circuit-breaker. In a price-decrease scenario (e.g., slashing event before `updateRSETHPrice()` is called), this could cause overvaluation of rsETH collateral.

## Likelihood Explanation
`updateRSETHPrice()` is permissionless but not automatically triggered. Any period of network congestion, keeper failure, or simply no caller results in a stale `rsETHPrice`. The ETH/USD Chainlink feed continues updating on its own heartbeat independently, so `updatedAt` from `latestRoundData()` will appear fresh to any consumer. This is a realistic operational condition. No privileged access or attacker action is required — the flaw manifests passively whenever `updateRSETHPrice()` is not called.

## Recommendation
Track the timestamp at which `rsETHPrice` is last written in `LRTOracle`, and expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()`, return `min(ethToUsdUpdatedAt, rsETHPriceUpdatedAt)` as `updatedAt`:

```solidity
// In LRTOracle:
uint256 public rsETHPriceUpdatedAt;
// In _updateRsETHPrice(), after line 313:
rsETHPriceUpdatedAt = block.timestamp;

// In RSETHPriceFeed.latestRoundData():
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
uint256 rsETHUpdatedAt = RS_ETH_ORACLE.rsETHPriceUpdatedAt();
if (rsETHUpdatedAt < updatedAt) updatedAt = rsETHUpdatedAt;
```

Apply the same fix to `getRoundData()`.

## Proof of Concept
1. `updateRSETHPrice()` is called at time `T`. `LRTOracle.rsETHPrice` is set to `1.05e18`.
2. 48 hours pass. No one calls `updateRSETHPrice()`. The ETH/USD Chainlink feed updates normally every heartbeat; its `updatedAt` is `T + 48h`.
3. A lending protocol calls `RSETHPriceFeed.latestRoundData()`.
4. Returned: `updatedAt = T + 48h` (from ETH/USD — appears fresh), `answer = 1.05e18 * ETH_USD_price / 1e18` (stale rsETH/ETH component).
5. The lending protocol's staleness check (`block.timestamp - updatedAt < threshold`) passes.
6. The protocol uses the stale rsETH/USD price for collateral valuation.
