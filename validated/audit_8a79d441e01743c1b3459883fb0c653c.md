Audit Report

## Title
`getExpectedAssetAmount` Returns Gross Pre-Fee Amount, Understating Actual Asset Received from `instantWithdrawal` - (File: contracts/LRTWithdrawalManager.sol)

## Summary

`getExpectedAssetAmount` is documented as returning "the asset amount to receive when trading in rsETH," but it computes only the oracle-derived gross amount with no deduction for `instantWithdrawalFee`. `instantWithdrawal` uses this same gross value internally and then silently deducts the fee before transferring to the caller. Any on-chain integrator or user that queries `getExpectedAssetAmount` as a pre-flight quote for `instantWithdrawal` will consistently receive `instantWithdrawalFee` basis-points less than the quoted value.

## Finding Description

`getExpectedAssetAmount` (L580–594) computes the return purely from oracle prices:

```solidity
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset);
```

No fee is subtracted. The NatDoc comment states "Get asset amount to receive when trading in rsETH," implying this is the net amount the caller will receive.

Inside `instantWithdrawal` (L228–250), the same gross value is used as the basis, and the fee is deducted afterward:

```solidity
uint256 assetAmountUnlocked = getExpectedAssetAmount(asset, rsETHUnstaked); // gross
...
uint256 fee = (assetAmountUnlocked * instantWithdrawalFee) / 10_000;
uint256 userAmount = assetAmountUnlocked - fee;                              // net
...
_transferAsset(asset, msg.sender, userAmount);                               // user gets net
```

`instantWithdrawalFee` is configurable up to 1000 bps (10%) by `onlyLRTManager` (L372–375). Whenever it is non-zero, the structural mismatch is active. An on-chain integrator that enforces `received >= getExpectedAssetAmount(asset, rsETHAmount)` will revert on every `instantWithdrawal` call while the fee is set, permanently blocking that integration path.

## Impact Explanation

**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The view function's documented and named purpose is to return the amount the caller will receive. It returns a value that is systematically higher than what `instantWithdrawal` actually delivers whenever `instantWithdrawalFee > 0`. No funds are lost from the protocol; the fee is redirected to the fee recipient. However, the contract's public interface fails to deliver the promised return to callers relying on the canonical quote function.

## Likelihood Explanation

`instantWithdrawalFee` is currently settable to any value from 0 to 1000 bps. The discrepancy is present and proportional to the fee on every `instantWithdrawal` call whenever the fee is non-zero. No special attacker capability is required — any user or integrator following the natural "query view → execute" pattern is affected. The condition is repeatable and deterministic.

## Recommendation

Add a dedicated net-amount view function for `instantWithdrawal` callers:

```solidity
function getExpectedAssetAmountAfterFee(address asset, uint256 rsETHAmount)
    public view returns (uint256 netAmount)
{
    uint256 gross = getExpectedAssetAmount(asset, rsETHAmount);
    uint256 fee = (gross * instantWithdrawalFee) / 10_000;
    netAmount = gross - fee;
}
```

Alternatively, update the NatDoc on `getExpectedAssetAmount` to explicitly state it returns the gross pre-fee amount and is not suitable as a quote for `instantWithdrawal`, and expose `getExpectedAssetAmountAfterFee` as the canonical function for that flow.

## Proof of Concept

1. `instantWithdrawalFee` is set to 100 bps (1%) via `setInstantWithdrawalFee(100)`.
2. User calls `getExpectedAssetAmount(ETH, 1000e18)` → returns `X` ETH.
3. User calls `instantWithdrawal(ETH, 1000e18, "")` expecting to receive `X` ETH.
4. Contract computes `fee = X * 100 / 10_000 = 0.01 * X`, transfers `userAmount = 0.99 * X` to user.
5. User receives 1% less than quoted. An upstream contract asserting `received >= X` reverts on every call while the fee is non-zero.

Foundry test sketch:
```solidity
uint256 quoted = withdrawalManager.getExpectedAssetAmount(ETH, 1000e18);
uint256 balBefore = address(user).balance;
vm.prank(user);
withdrawalManager.instantWithdrawal(ETH, 1000e18, "");
uint256 received = address(user).balance - balBefore;
assertLt(received, quoted); // passes when instantWithdrawalFee > 0
```
