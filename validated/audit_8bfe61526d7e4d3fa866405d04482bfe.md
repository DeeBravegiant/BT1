### Title
Expired MPC Image Hash Included in Valid Set Due to Off-by-One in `valid_entries` — (File: `crates/contract/src/tee/proposal.rs`)

---

### Summary

The `AllowedDockerImageHashes::valid_entries` function uses `rposition` to locate the most recently expired image hash and then returns every entry **from that index onwards** (inclusive). This off-by-one means the expired entry itself is always included in the "valid" set, silently extending the effective grace period for old MPC images beyond the configured `tee_upgrade_deadline_duration`. The analog to the GoGoPool M-06 class is exact: a time-window boundary is computed correctly but the accounting update is anchored to the wrong index, causing the remainder of the old window to bleed into the new one.

---

### Finding Description

`AllowedDockerImageHashes` stores a sorted list of whitelisted MPC Docker image hashes, each tagged with the block timestamp at which it was voted in (`added: Timestamp`). The function `valid_entries` is the single source of truth for which hashes are currently accepted for attestation verification:

```rust
// crates/contract/src/tee/proposal.rs  lines 170-193
fn valid_entries(&self, tee_upgrade_deadline_duration: Duration) -> Vec<AllowedMpcDockerImage> {
    let current_time = Timestamp::now();
    let cutoff_index = self
        .allowed_tee_proposals
        .iter()
        .rposition(|allowed_docker_image| {
            let Some(grace_period_deadline) = allowed_docker_image
                .added
                .checked_add(tee_upgrade_deadline_duration)
            else { ... return true; };
            // "if the grace period for this docker hash is in the past,
            //  then older hashes are no longer accepted"
            grace_period_deadline < current_time
        })
        .unwrap_or(0);

    self.allowed_tee_proposals
        .get(cutoff_index..)   // ← includes the expired entry at cutoff_index
        .unwrap_or(&[])
        .to_vec()
}
```

`rposition` returns the index of the **rightmost** entry whose grace period has already elapsed. `get(cutoff_index..)` then returns a slice that **starts at that expired entry**, so the expired hash is always present in the returned "valid" list.
