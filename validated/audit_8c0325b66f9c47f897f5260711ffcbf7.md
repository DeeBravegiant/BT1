### Title
Discarded Return Value of `check_request_preconditions()` in `verify_foreign_transaction()` Skips Domain-Config Payload Validation - (File: crates/contract/src/lib.rs)

### Summary

`verify_foreign_transaction()` calls `check_request_preconditions()` but silently discards its return value `(domain_config, predecessor)`. The `sign()` function, by contrast, binds and uses both fields — `domain_config` to validate the payload type against the domain's protocol and to reject `ConfidentialKeyDerivation` domains, and `predecessor` to construct the canonical `SignatureRequest`. Because `verify_foreign_transaction()` drops the return value, none of that per-domain protocol validation is applied to foreign-tx requests, allowing a caller to enqueue a request whose payload is incompatible with the domain's actual signing scheme.

### Finding Description

In `crates/contract/src/lib.rs`, the `sign()` function correctly captures the return value of `check_request_preconditions()`:

```rust
// sign() — return value used
let (domain_config, predecessor) = self.check_request_preconditions(
    request.domain_id,
    DomainPurpose::Sign,
    Gas::from_tgas(self.config.sign_call_gas_attachment_requirement_tera_gas),
    MINIMUM_SIGN_REQUEST_DEPOSIT,
);
// domain_config then drives payload-type validation:
match domain_config.protocol {
    Protocol::CaitSith | Protocol::DamgardEtAl => { /* validate ECDSA scalar */ }
    Protocol::Frost => { /* validate EdDSA bytes */ }
    Protocol::ConfidentialKeyDerivation => { env::panic_str("..."); }
}
```

`verify_foreign_transaction()` makes the identical call but throws away the result:

```rust
// verify_foreign_transaction() — return value discarded
self.check_request_preconditions(
    request.domain_id,
    DomainPurpose::ForeignTx,
    Gas::from_tgas(self.config.sign_call_gas_attachment_requirement_tera_gas),
    MINIMUM_SIGN_REQUEST_DEPOSIT,
);
// No protocol-level payload validation follows.
```

The function then proceeds directly to `args_into_verify_foreign_tx_request(request)` and `enqueue_yield_request(...)` without ever inspecting `domain_config.protocol`. This is a direct structural analog to the ERC20 `transferFrom()` missing-return-value bug: a function whose return value carries security-critical information is called, but the caller ignores it entirely. [1](#0-0) [2](#0-1) 

### Impact Explanation

Because `domain_config` is never inspected in `verify_foreign_transaction()`, the following invariants enforced in `sign()` are absent:

1. **Protocol-payload mismatch is not caught at submission time.** A caller can submit a `verify_foreign_transaction` request against a `Frost` (EdDSA) domain while supplying an ECDSA-shaped payload, or vice versa. The request is enqueued and dispatched to MPC nodes, which will attempt to process it under the wrong signing scheme.
2. **`ConfidentialKeyDerivation` domains are not rejected.** `sign()` explicitly panics when `domain_config.protocol == ConfidentialKeyDerivation`. `verify_foreign_transaction()` performs no such guard, so a CKD domain can be targeted, causing MPC nodes to attempt a signing operation against a key whose material is intended only for confidential derivation.

Both conditions corrupt the request lifecycle: the yield slot is consumed, the deposit is locked, and MPC nodes receive a structurally invalid task. In the CKD-domain case, nodes may expose key-derivation material through an unintended signing path, satisfying the **Medium** impact criterion of *contract execution-flow manipulation that breaks production safety/accounting invariants*. [3](#0-2) [4](#0-3) 

### Likelihood Explanation

The entry point is the public, unprivileged, payable `verify_foreign_transaction()` method. Any NEAR account that attaches the minimum deposit and the required gas can call it. No special role, key, or collusion is required. The caller only needs to supply a `domain_id` that resolves to a non-`ForeignTx`-purpose domain (e.g., a CKD domain) and a well-formed `VerifyForeignTransactionRequestArgs` struct. The contract will accept the call, pass the deposit check inside `check_request_preconditions`, and enqueue the malformed request. [5](#0-4) 

### Recommendation

Bind and use the return value of `check_request_preconditions()` in `verify_foreign_transaction()`, mirroring the pattern in `sign()`:

```rust
let (domain_config, _predecessor) = self.check_request_preconditions(
    request.domain_id,
    DomainPurpose::ForeignTx,
    Gas::from_tgas(self.config.sign_call_gas_attachment_requirement_tera_gas),
    MINIMUM_SIGN_REQUEST_DEPOSIT,
);

// Guard against incompatible protocols, as sign() does:
match domain_config.protocol {
    Protocol::CaitSith | Protocol::DamgardEtAl => { /* valid for foreign-tx */ }
    Protocol::ConfidentialKeyDerivation => {
        env::panic_str("ConfidentialKeyDerivation is not supported for foreign-tx requests");
    }
    Protocol::Frost => {
        env::panic_str("Frost is not supported for foreign-tx requests");
    }
}
```

This ensures that the domain-config information returned by `check_request_preconditions()` is never silently discarded and that the same protocol-level guards present in `sign()` are uniformly applied.

### Proof of Concept

1. Deploy the contract and initialize it with at least one `ConfidentialKeyDerivation` domain (domain ID `D_ckd`).
2. Call `verify_foreign_transaction` with `domain_id = D_ckd`, a valid `ForeignChainRpcRequest`, and the minimum deposit.
3. Observe that `check_request_preconditions` passes (deposit and gas checks succeed), the discarded `domain_config` is never inspected, and the request is enqueued under `pending_verify_foreign_tx_requests`.
4. MPC nodes pick up the request and attempt to produce a signature using the CKD domain key — a key whose material is intended exclusively for confidential derivation, not for signing foreign-chain payloads.
5. The yield slot is consumed and the deposit is locked regardless of whether the nodes can fulfill the request, breaking the request-lifecycle accounting invariant. [6](#0-5) [7](#0-6)

        log!(
            "verify_foreign_transaction: predecessor={:?}, request={:?}",
            env::predecessor_account_id(),
            request
        );

        self.check_request_preconditions(
            request.domain_id,
            DomainPurpose::ForeignTx,
            Gas::from_tgas(self.config.sign_call_gas_attachment_requirement_tera_gas),
            MINIMUM_SIGN_REQUEST_DEPOSIT,
        );

        let requested_chain = request.request.chain();
        let supported_chains = self.get_supported_foreign_chains();
        if !supported_chains.contains(&requested_chain) {
            env::panic_str(
                &InvalidParameters::ForeignChainNotSupported {
                    requested: requested_chain,
                }
                .to_string(),
            );
        }

        let callback_gas = Gas::from_tgas(
            self.config
                .return_signature_and_clean_state_on_success_call_tera_gas,
        );

        let request = args_into_verify_foreign_tx_request(request);
        let callback_args = serde_json::to_vec(&(&request,)).unwrap();

            callback_args,

    pub fn respond(
        &mut self,
        request: SignatureRequest,
        response: dtos::SignatureResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!("respond: signer={}, request={:?}", &signer, &request);

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain)?;

        let signature_is_valid = match (&response, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                // generate the expected public key
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");
                let affine = *k256::PublicKey::try_from(&secp_pk)
                    .expect("stored key is always valid")
                    .as_affine();
                let expected_public_key =
                    derive_key_secp256k1(&affine, &request.tweak).map_err(RespondError::from)?;

                let payload_hash = request.payload.as_ecdsa().expect("Payload is not ECDSA");

                // Check the signature is correct
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    payload_hash,
                    &expected_public_key,
                )
                .is_ok()
            }
            (
                dtos::SignatureResponse::Ed25519 { signature },
                PublicKeyExtended::Ed25519 {
                    edwards_point: public_key_edwards_point,
                    ..
                },
            ) => {
                let derived_public_key_edwards_point = derive_public_key_edwards_point_ed25519(
                    &public_key_edwards_point,
                    &request.tweak,
                );
                let derived_public_key_32_bytes =
                    dtos::Ed25519PublicKey::from(derived_public_key_edwards_point.compress());

                let message = request.payload.as_eddsa().expect("Payload is not EdDSA");

                near_mpc_signature_verifier::verify_eddsa_signature(
                    signature,
                    message,
                    &derived_public_key_32_bytes,
                )
                .is_ok()
            }

                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }

        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

            &mut self.pending_signature_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
