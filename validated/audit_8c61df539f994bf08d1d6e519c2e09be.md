### Title
Rosetta RPC NEP-141 Event Extraction Ignores Receipt Failure Status, Allowing Fake Token Transfer Injection - (File: chain/rosetta-rpc/src/adapters/nep141.rs)

### Summary

The Rosetta RPC adapter's `collect_nep141_events` function processes NEP-141 `EVENT_JSON:` log events from **all** execution outcomes in a block, including those with `ExecutionStatusView::Failure`. An unprivileged user can deploy a contract that emits a crafted `ft_transfer` event log and then panics, causing the receipt to fail. The Rosetta `/block` endpoint will still include the fake fungible token transfer in its response, corrupting the `FungibleTokenEvent` list and the Rosetta operations derived from it.

### Finding Description

In `collect_nep141_events`, every outcome in `receipt_execution_outcomes` is passed unconditionally to `extract_events`:

```rust
pub(crate) fn collect_nep141_events(
    receipt_execution_outcomes: &Vec<ExecutionOutcomeWithIdView>,
    ...
) -> crate::errors::Result<Vec<FungibleTokenEvent>> {
    let mut res = Vec::new();
    for outcome in receipt_execution_outcomes {
        let events = extract_events(outcome);   // ← no status check
        for event in events {
            res.extend(compose_rosetta_nep141_events(...)?);
        }
    }
    Ok(res)
}

The call site in `ExecutionToReceipts::for_block` collects **all** execution outcomes for the block (no failure filter) and feeds them directly to `collect_nep141_events`: [3](#0-2) 

Note that the same `execution_outcomes` vector is later used to build the `statuses` map (which correctly records `Failure`), but that map is only used for Rosetta transaction metadata — it is never consulted to gate event extraction. [4](#0-3) 

### Impact Explanation

The corrupted protocol value is the **Rosetta RPC `/block` response** — specifically the `FungibleTokenEvent` entries and the Rosetta `operations` list derived from them via `convert_fungible_token_balance_change_to_operations`. An attacker can inject arbitrary `ft_transfer` events (any `old_owner_id`, `new_owner_id`, and `amount`) into the Rosetta view of any block, making it appear that token balances changed when they did not. Exchange integrations and indexers relying on the Rosetta `/block` endpoint to track NEP-141 balances would record phantom debits and credits. [5](#0-4) 

### Likelihood Explanation

Any unprivileged NEAR account can deploy a contract and call a method that emits a crafted `EVENT_JSON:` log then panics. The receipt will be included in a block with `ExecutionStatus::Failure`, and the Rosetta RPC will process it on the next `/block` query. No special privileges are required; the cost is only the gas and storage deposit for a contract deployment.

### Recommendation

In `collect_nep141_events` (or in `extract_events`), skip outcomes whose status is `ExecutionStatusView::Failure` or `ExecutionStatusView::Unknown` before extracting events:

```rust
for outcome in receipt_execution_outcomes {
    if matches!(outcome.outcome.status,
        ExecutionStatusView::Failure(_) | ExecutionStatusView::Unknown) {
        continue;
    }
    let events = extract_events(outcome);
    ...
}
```

### Proof of Concept

1. Deploy a contract with a method that executes:
   ```
   env::log_str("EVENT_JSON:{\"standard\":\"nep141\",\"version\":\"1.0.0\",\"event\":\"ft_transfer\",\"data\":[{\"old_owner_id\":\"victim.near\",\"new_owner_id\":\"attacker.near\",\"amount\":\"1000000000000000000000000\"}]}");
   panic!("intentional failure");
   ```
2. Call the method from any account. The receipt lands on-chain with `ExecutionStatus::Failure`.
3. Query the Rosetta `/block` endpoint for that block height.
4. Observe that the response contains a `FungibleTokenEvent` showing a 1 NEAR transfer from `victim.near` to `attacker.near`, even though no token transfer occurred.

pub(crate) fn extract_events(
    execution_outcome: &ExecutionOutcomeWithIdView,
) -> Vec<crate::models::Nep141Event> {
    let prefix = "EVENT_JSON:";
    execution_outcome
        .outcome
        .logs
        .iter()
        .filter_map(|untrimmed_log| {
            let log = untrimmed_log.trim();
            if !log.starts_with(prefix) {
                return None;
            }

            match serde_json::from_str::<'_, crate::models::Nep141Event>(log[prefix.len()..].trim())
            {
                Ok(result) => Some(result),
                Err(_err) => None,
            }
        })
        .collect()
}
