Audit Report

## Title
Unbounded Gas Growth via Nested Loop Over Queued EigenLayer Withdrawals in Public Deposit Path — (File: `contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`)

## Summary
`NodeDelegator.getAssetUnstaking()` fetches all queued EigenLayer withdrawals and iterates over them in a nested loop with external calls per strategy. This function is invoked for every NDC in `nodeDelegatorQueue` on every call to `depositETH()` and `depositAsset()`, via `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()` → `getAssetDistributionData()`. As the protocol accumulates queued withdrawals through normal operation, the gas cost of every deposit grows proportionally and can exceed the block gas limit.

## Finding Description
The full confirmed call chain is:

```
depositETH() / depositAsset()
  → _beforeDeposit()                              [LRTDepositPool.sol L661]
    → _checkIfDepositAmountExceedesCurrentLimit() [LRTDepositPool.sol L677]
      → getTotalAssetDeposits()                   [LRTDepositPool.sol L393]
        → getAssetDistributionData()              [LRTDepositPool.sol L451]
          → [loop over NDCs] getAssetUnstaking()  [NodeDelegator.sol L405]
            → getQueuedWithdrawals()              [external EigenLayer call]
              → [loop over withdrawals]
                → [loop over strategies] sharesToUnderlyingView() [external call]
```

`getAssetUnstaking()` at `NodeDelegator.sol` L405-427 calls `_getDelegationManager().getQueuedWithdrawals(address(this))` and then iterates over every queued withdrawal and every strategy within it, making a cold external call to `strategy.sharesToUnderlyingView()` per non-ETH strategy. [1](#0-0) 

`getAssetDistributionData()` at `LRTDepositPool.sol` L446-456 calls `getAssetUnstaking()` for every entry in `nodeDelegatorQueue` without any gas bound. [2](#0-1) 

The same pattern applies to ETH deposits via `getETHDistributionData()`, which also calls `getAssetUnstaking(LRTConstants.ETH_TOKEN)` per NDC. [3](#0-2) 

`_checkIfDepositAmountExceedesCurrentLimit()` unconditionally calls `getTotalAssetDeposits()` on every deposit, with no caching or short-circuit. [4](#0-3) 

There are no gas-bounding guards, iteration caps, or caching mechanisms anywhere in this call chain. The total work is **O(NDCs × queued\_withdrawals\_per\_NDC × strategies\_per\_withdrawal)** with cold external calls (~2100 gas minimum each) at the innermost level.

## Impact Explanation
As the protocol scales with more NDCs and more EigenLayer withdrawals queued through normal restaking operations, the gas cost of `depositETH()` and `depositAsset()` grows without bound. Once the cumulative gas cost approaches the ~30M block gas limit, these functions become permanently uncallable by any user, matching **Medium — Unbounded gas consumption** and **Medium — Temporary freezing of funds**.

## Likelihood Explanation
No attacker action is required. Normal protocol operation — adding NDCs and queuing EigenLayer withdrawals for validator exits or LST unstaking cycles — is sufficient to trigger the condition. Any unprivileged user calling `depositETH()` or `depositAsset()` at that point will receive an out-of-gas revert. The `maxNodeDelegatorLimit` and `maxUncompletedWithdrawalCount` parameters exist but impose no hard ceiling relative to the block gas limit.

## Recommendation
1. **Decouple the deposit limit check from full distribution recomputation**: maintain a cached running total of `totalAssetDeposits` updated on deposit/withdrawal events, rather than recomputing across all NDCs and all queued withdrawals on every deposit.
2. **Remove `getAssetUnstaking()` from the hot deposit path**: track unstaking amounts via events or a storage variable updated when withdrawals are queued/completed.
3. **Bound `maxUncompletedWithdrawalCount` per NDC** to a value that keeps worst-case gas well below the block gas limit, accounting for NDC count and strategy count.
4. **Introduce a gas-bounded iteration cap** in `getAssetUnstaking()` if on-chain iteration is retained.

## Proof of Concept
1. Protocol has `N = 10` NDCs.
2. Each NDC has `M = 50` queued EigenLayer withdrawals, each with `S = 3` strategies.
3. A user calls `depositETH(minRSETHAmountExpected, referralId)`.
4. The call reaches `getAssetUnstaking()` for each NDC: `10 × 50 × 3 = 1500` inner iterations, each with at least one cold external call (~2100 gas) plus strategy logic.
5. Estimated gas for nested loops alone: `1500 × ~5000 gas = 7,500,000 gas`, plus `getQueuedWithdrawals()` data loading overhead per NDC.
6. As `M` grows, total gas approaches or exceeds the 30M block gas limit, making `depositETH()` uncallable.

A Foundry fork test can reproduce this by deploying mock EigenLayer contracts that return `M` queued withdrawals per NDC, then calling `depositETH()` with `vm.expectRevert()` for out-of-gas, or measuring gas with `gasleft()` snapshots across increasing values of `M`.

### Citations

**File:** contracts/NodeDelegator.sol (L405-427)
```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
