### Title
Caller Deposit Permanently Lost on Intermediate Callback Failure in Wallet Contract - (File: runtime/near-wallet-contract/implementation/wallet-contract/src/lib.rs)

### Summary
The `WalletContract` tracks an external caller's attached NEAR deposit via `CallerDeposit` so it can be refunded if the cross-contract call fails. `rlp_execute_callback` correctly issues this refund. However, two intermediate callbacks — `address_check_callback` and `nep_141_storage_balance_callback` — contain multiple early-return error paths that drop `caller_deposit` without issuing any refund, permanently locking the caller's NEAR inside the wallet contract.

### Finding Description

`CallerDeposit` is constructed in `inner_rlp_execute` to record the external caller's attached deposit: [1](#0-0) 

The design intent is that `rlp_execute_callback` refunds this deposit if the inner promise fails: [2](#0-1) 

However, two intermediate callbacks have early-return paths that silently discard `caller_deposit`:

**`nep_141_storage_balance_callback`** — triggered for every ERC-20 emulated transfer:

1. `PromiseResult::Failed` (lines 204–209): the `storage_balance_of` call to the token contract fails; returns an error value immediately without refunding `caller_deposit`.
2. Deserialization error (lines 213–219): the token contract returns unparseable data; same early return, no refund.
3. Unexpected action type (lines 245–253): the action is not a `FunctionCall`; same early return, no refund. [3](#0-2) 

**`address_check_callback`** — triggered for EOA base-token transfers to eth-implicit accounts:

1. `PromiseResult::Failed` (lines 142–148): the address registrar call fails; early return, no refund.
2. Deserialization error (lines 151–157): registrar returns unparseable data; early return, no refund.
3. Invalid target with non-access-key relayer (lines 168–173): faulty relayer detected; early return, no refund. [4](#0-3) 

In all these cases, the caller's NEAR tokens remain in the wallet contract's balance with no mechanism to retrieve them.

### Impact Explanation

An external caller (relayer or any NEAR account) who attaches NEAR tokens to `rlp_execute` for an ERC-20 transfer or an EOA base-token transfer to an eth-implicit account loses their entire attached deposit if the intermediate promise fails. The tokens are credited to the wallet contract's balance and are unrecoverable. The corrupted value is the caller's `yocto_near` balance recorded in `CallerDeposit`. [5](#0-4) 

### Likelihood Explanation

The `nep_141_storage_balance_callback` path is reachable by any external account calling `rlp_execute` with an ERC-20 transfer and an attached deposit. The `PromiseResult::Failed` branch fires whenever the NEP-141 token contract's `storage_balance_of` call fails — which can happen due to the token contract panicking, running out of gas, or being paused. This is a realistic production scenario. No special privileges are required; any unprivileged NEAR account can trigger it.

### Recommendation

In every early-return error path inside `address_check_callback` and `nep_141_storage_balance_callback`, issue a refund promise before returning, mirroring the pattern already used in `rlp_execute_callback`:

```rust
if let Some(CallerDeposit { account_id, yocto_near }) = caller_deposit {
    let refund_promise = env::promise_batch_create(&account_id);
    env::promise_batch_action_transfer(
        refund_promise,
        NearToken::from_yoctonear(yocto_near.into()),
    );
}
return PromiseOrValue::Value(ExecuteResponse { ... });
```

This should be applied to all five early-return sites identified above.

### Proof of Concept

1. Deploy a NEP-141 token contract that panics on `storage_balance_of`.
2. As an external relayer account, call `rlp_execute` on a victim's eth-implicit wallet contract with `target` = the malicious token contract, an ERC-20 transfer payload, and an attached deposit of e.g. 3 NEAR.
3. `inner_rlp_execute` records `CallerDeposit { account_id: relayer, yocto_near: 3e24 }` and schedules `storage_balance_of` → `nep_141_storage_balance_callback`.
4. The token contract panics; `nep_141_storage_balance_callback` receives `PromiseResult::Failed` and returns early at line 205 without issuing any refund promise.
5. The relayer's 3 NEAR remain in the wallet contract's balance permanently. [6](#0-5) [7](#0-6)

pub struct CallerDeposit {
    pub account_id: AccountId,
    pub yocto_near: NonZeroU128,
}

impl CallerDeposit {
    pub fn new(context: &ExecutionContext) -> Option<Self> {
        // Only track for external (non-self) callers
        if context.current_account_id == context.predecessor_account_id {
            return None;
        }

        NonZeroU128::new(context.attached_deposit.as_yoctonear()).map(|yocto_near| Self {
            account_id: context.predecessor_account_id.clone(),
            yocto_near,
        })
    }
