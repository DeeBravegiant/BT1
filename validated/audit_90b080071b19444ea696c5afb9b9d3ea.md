### Title
Unbounded `fee_proposal_fri` accepted during startup window poisons `fee_actual` and drives `l2_gas_price` toward `u128::MAX` — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` enforces a margin-bounded check on `fee_proposal_fri` only when `fee_actual` is `Some`. During the first `fee_proposal_window_size = 10` blocks (`fee_actual` is `None`), the field is accepted with **no bounds whatsoever**. A malicious proposer can set `fee_proposal_fri = u128::MAX` for a majority of those blocks; the value is stored verbatim in `fee_proposals_window`, poisons the median that becomes `fee_actual`, and drives `effective_min` — and therefore `l2_gas_price` — toward `u128::MAX` for all subsequent blocks.

### Finding Description

In `is_proposal_init_valid`, the bounds check is gated on both `fee_actual` and `fee_proposal` being `Some`:

```rust
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(...);
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

When `fee_actual` is `None` (the first `window_size` blocks), the entire branch is skipped. An honest proposer freezes at `l2_gas_price`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [2](#0-1) 

But the validator never checks that the proposer's `fee_proposal_fri` equals (or is near) `l2_gas_price`. Any value — including `u128::MAX` — passes.

At decision time the accepted value is stored permanently in `fee_proposals_window`:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [3](#0-2) 

`fee_proposal_window_size` is 10 across all deployed versioned-constants files: [4](#0-3) 

`compute_fee_actual` returns the median of the 10-entry window. For an even window the median is the average of the 5th and 6th sorted values: [5](#0-4) 

If 5 of the 10 startup blocks carry `fee_proposal_fri = u128::MAX` and 5 carry the honest `l2_gas_price ≈ 8 × 10⁹`, the sorted window is `[8e9 × 5, u128::MAX × 5]`; the median is `(8e9 + u128::MAX) / 2 ≈ 1.7 × 10³⁸`.

That poisoned `fee_actual` is then used as `effective_min` in `calculate_next_l2_gas_price_for_fin`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
``` [6](#0-5) 

With `effective_min ≈ u128::MAX / 2`, `calculate_next_base_gas_price` enters the "price below minimum" branch and ratchets `l2_gas_price` upward by `price / 333` per block: [7](#0-6) 

Once `fee_actual` becomes available (block 10+), the margin check (`fee_proposal_margin_ppt = 2`, i.e. 0.2 ‰ per block) only allows the window to drift down by 0.2 ‰ per block. Recovering from `u128::MAX / 2` to `8 × 10⁹` requires on the order of `ln(1.7e38 / 8e9) / 0.0002 ≈ 330 000` blocks (~10 days at 2.6 s/block).

Additionally, `fee_proposal_fri` is committed into the block header and forwarded to the centralized cende pipeline: [8](#0-7) [9](#0-8) 

### Impact Explanation

`l2_gas_price` is the gas price charged to every Starknet transaction. Driving it toward `u128::MAX / 2` makes every transaction fail with "insufficient max fee" or forces users to pay astronomically high fees. The effect persists for ~10 days due to the 0.2 ‰/block recovery rate. This is a direct, economically significant, and long-lasting disruption of the fee market — matching the "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" critical impact class.

### Likelihood Explanation

The attack window is the first 10 blocks of the chain (or after a revert to genesis). In a two-validator BFT setup with round-robin proposer selection, the malicious validator proposes exactly 5 of the first 10 blocks — sufficient to poison the median. The proposer role is not privileged beyond being a consensus participant; any Byzantine validator selected as proposer during the startup window can execute this attack without any special access.

### Recommendation

When `fee_actual` is `None`, validate `fee_proposal_fri` against the locally-computed `l2_gas_price_fri` (the same fallback the honest proposer uses) with the same `fee_proposal_margin_ppt` geometric bounds:

```rust
// When fee_actual is None, the honest fallback is l2_gas_price_fri.
// Enforce the same margin to prevent extreme values from poisoning the window.
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    if proposal_init_validation.fee_actual.is_none() {
        let fallback = proposal_init_validation.l2_gas_price_fri;
        let (lower, upper) = fee_proposal_bounds(
            fallback,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower || fee_proposal.0 > upper {
            return Err(ValidateProposalError::InvalidProposalInit(...));
        }
    }
}
```

### Proof of Concept

```
Setup: 2-validator BFT network, round-robin proposer selection, fresh chain start.

Blocks 0, 2, 4, 6, 8 — malicious proposer:
  ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), ... }
  → is_proposal_init_valid: fee_actual = None → bounds check skipped → ACCEPTED
  → record_fee_proposal(height, Some(GasPrice(u128::MAX)))

Blocks 1, 3, 5, 7, 9 — honest proposer:
  ProposalInit { fee_proposal_fri: Some(GasPrice(8_000_000_000)), ... }
  → record_fee_proposal(height, Some(GasPrice(8_000_000_000)))

Block 10:
  fee_proposals_window = [8e9, MAX, 8e9, MAX, 8e9, MAX, 8e9, MAX, 8e9, MAX]
  sorted              = [8e9, 8e9, 8e9, 8e9, 8e9, MAX, MAX, MAX, MAX, MAX]
  fee_actual          = (8e9 + MAX) / 2 ≈ 1.7e38

  calculate_next_l2_gas_price_for_fin:
    effective_min = max(config_min, 1.7e38) = 1.7e38
    l2_gas_price  → ratchets toward 1.7e38 at +0.3%/block

All subsequent transactions: max_fee < l2_gas_price → REJECTED
Recovery: ~330 000 blocks (~10 days)
