### Title
Unconstrained `fee_proposal_fri` During Initiation Period Seeds Manipulated `fee_actual` Baseline â€” (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The SNIP-35 fee-market mechanism uses a sliding window of `fee_proposal_fri` values to compute `fee_actual` (the median of the last `fee_proposal_window_size` blocks). During the first `fee_proposal_window_size` blocks the window is incomplete, `fee_actual` is `None`, and the validator's `is_proposal_init_valid` explicitly skips bounds enforcement. A malicious proposer during this initiation period can publish any `fee_proposal_fri` value â€” including `u128::MAX` â€” with no rejection from any validator. Those extreme values seed the window; once the window fills, `fee_actual` is computed from the poisoned entries and becomes the permanent floor for the EIP-1559 L2 gas price, causing all subsequent transactions to pay wrong fees.

---

### Finding Description

**Root cause â€” `is_proposal_init_valid` in `validate_proposal.rs`:**

```
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...bounds check...
}
```

The `if let` pattern silently passes when `fee_actual` is `None`. No alternative guard exists. [1](#0-0) 

**Why `fee_actual` is `None` for the first `window_size` blocks:**

`compute_fee_actual` returns `None` whenever `height < window_size` (the subtraction underflows and the function warns and returns `None`):

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ...");
    return None;
};
``` [2](#0-1) 

The window size is `10` in production versioned constants: [3](#0-2) 

**How the poisoned window propagates to the L2 gas price:**

`calculate_next_l2_gas_price_for_fin` uses `fee_actual` as the effective minimum for the EIP-1559 base-fee calculation:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [4](#0-3) 

If `fee_actual` is `u128::MAX`, `effective_min` becomes `u128::MAX`, and `calculate_next_base_gas_price` ratchets the gas price upward toward `u128::MAX` over subsequent blocks. [5](#0-4) 

**The `fee_proposal_fri` is also bound into the `ProposalCommitment`:**

```rust

``` [6](#0-5) 

Consensus signs over the commitment, so once a block with an extreme `fee_proposal_fri` is decided, the value is permanently recorded in the `fee_proposals_window` via `record_fee_proposal`: [7](#0-6)

### Impact Explanation

After the 10-block initiation window, `fee_actual` is the median of the poisoned entries. `fee_actual` is then used as the floor for `calculate_next_l2_gas_price_for_fin`, which computes the `l2_gas_price` that is:

1. Broadcast in `ProposalFin.fin_payload.l2_gas_info.next_l2_gas_price_fri` to all validators.
2. Used by the batcher as the `l2_gas_price` for the next block's `BlockInfo`, which is the price charged to every transaction.

A proposer who sets `fee_proposal_fri = u128::MAX` across all 10 initiation blocks causes `fee_actual = u128::MAX`, which drives `l2_gas_price â†’ u128::MAX` over subsequent blocks. Every transaction's fee is computed against this price, resulting in economically incorrect (and practically infinite) fees for all users â€” a permanent, network-wide economic impact.

This maps directly to the allowed impact: **"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

- **Trigger**: Any validator who is selected as proposer during blocks 0â€“9 of the network. In a BFT consensus system this is a normal, non-admin role.
- **No existing guard**: The code comment explicitly documents the absence of enforcement: *"During initiation (fee_actual is None, <window_size blocks), bounds are not enforced."*
- **Persistence**: Once the 10 blocks are committed, the poisoned `fee_actual` is locked in. The margin is only Â±0.2% per block (`fee_proposal_margin_ppt = 2`), so recovery from `u128::MAX` is practically impossible.
- **Analog to ERC4626 first-deposit attack**: The initiation period is structurally identical to the "first depositor" window â€” no baseline exists, so any value is accepted, and that value seeds all future calculations.

---

### Recommendation

1. **Seed the window at genesis / network start**: Pre-populate `fee_proposals_window` with `window_size` entries equal to `min_gas_price` (or a configured bootstrap value) before the first block is proposed. This ensures `fee_actual` is always `Some` and bounds are always enforced.

2. **Enforce an absolute cap during initiation**: When `fee_actual` is `None`, reject any `fee_proposal_fri` outside `[min_gas_price, max_bootstrap_fee_proposal]` where `max_bootstrap_fee_proposal` is a protocol constant.

3. **Mirror the ERC4626 recommendation**: Analogous to "seed the pool during deployment" â€” initialize the fee-proposals window in `initialize_fee_proposals_window` with a safe default for heights below `window_size` rather than leaving them absent.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
