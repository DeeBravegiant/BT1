Audit Report

## Title
Missing Chainlink Staleness Validation Enables Stale-Price-Triggered Auto-Pause, Permanently Freezing Unclaimed Yield — (`contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice` discards all `latestRoundData` fields except `answer`, accepting stale prices silently. When a Chainlink feed is stale, the understated price propagates through `_getTotalEthInProtocol` into `_updateRsETHPrice`, which auto-pauses the deposit pool, withdrawal manager, and oracle itself if `newRsETHPrice` drops more than `pricePercentageLimit` below `highestRsethPrice`. This permanently destroys the unclaimed yield that would have been minted as protocol fees during the freeze, and requires manual admin intervention to recover.

## Finding Description

**Root cause — no staleness check in `ChainlinkPriceOracle.getAssetPrice`:** [1](#0-0) 

`latestRoundData` returns `(roundId, answer, startedAt, updatedAt, answeredInRound)`. Only `answer` is captured; `updatedAt` and `answeredInRound` are silently dropped. There is no `require(updatedAt >= block.timestamp - maxStaleness)` or `require(answeredInRound >= roundId)` guard anywhere in the call chain.

**Stale price propagates to `_getTotalEthInProtocol`:** [2](#0-1) 

Every supported LST asset's price is fetched via `getAssetPrice`. A stale feed returns the last-known (lower) price, understating `totalETHInProtocol`.

**Auto-pause fires in `_updateRsETHPrice`:** [3](#0-2) 

When `newRsETHPrice < highestRsethPrice` by more than `pricePercentageLimit`, the function pauses `lrtDepositPool`, `withdrawalManager`, and `this` via `_pause()`, then returns early — `IRSETH.mint` for protocol fees is never reached.

**`updateRSETHPrice()` is gated by `whenNotPaused`:** [4](#0-3) 

Once paused, the public entry point is blocked. `updateRSETHPriceAsManager()` bypasses the oracle's own pause but cannot un-pause the deposit pool or withdrawal manager. [5](#0-4) 

**Precondition check — `pricePercentageLimit > 0`:** [6](#0-5) 

The auto-pause branch only fires when `pricePercentageLimit > 0`, which requires prior admin configuration via `setPricePercentageLimit`. This is a normal operational setting, not an unrealistic precondition.

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield.**

Protocol fees that should have been minted during the freeze window are permanently lost: `_updateRsETHPrice` returns early at the pause branch before reaching `IRSETH.mint`. Once the oracle is unpaused, future fee minting resumes from the new price baseline, but the yield accrued during the frozen period is never recovered. Additionally, `withdrawalManager` being paused constitutes temporary freezing of funds until admin manually calls `unpause()` on all three contracts.

## Likelihood Explanation

`updateRSETHPrice()` is `public` with no access control beyond `whenNotPaused` — any unprivileged EOA can call it. No oracle operator compromise is required; normal Chainlink liveness failure (network congestion, node outage, or deviation threshold not crossed) is sufficient. For `pricePercentageLimit = 1e16` (1%) and ~5% LST APY, the required staleness is approximately 73 days — unlikely for major feeds under normal conditions, but the threshold drops to zero if the last captured round already reflected a depeg or price anomaly. The trigger is repeatable and requires no victim interaction.

## Recommendation

Add staleness and round-completeness checks in `ChainlinkPriceOracle.getAssetPrice`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(answeredInRound >= roundId, "Stale round");
require(updatedAt >= block.timestamp - maxStaleness, "Stale price");
require(price > 0, "Non-positive price");
```

Store a per-feed `maxStaleness` value in `ChainlinkPriceOracle` alongside `assetPriceFeed`, configurable by `onlyLRTManager`. Typical values: 3600 s for ETH/USD, 86 400 s for slower LST feeds.
