## Title
Unprivileged Caller Can Force EigenPod Checkpoint via `verifyStaleBalance` to Trigger Oracle Downside-Protection Pause — (`contracts/LRTOracle.sol`, `contracts/NodeDelegator.sol`)

## Summary
`NodeDelegator.startCheckpoint` is gated by `onlyLRTOperator`, but EigenLayer's `IEigenPod.verifyStaleBalance` and `verifyCheckpointProofs` are explicitly permissionless. Any external caller can bypass the operator guard, force-complete a checkpoint on a slashed validator, then call the public `LRTOracle.updateRSETHPrice()`. If the resulting rsETH price drop exceeds `pricePercentageLimit`, the oracle's downside-protection logic pauses `LRTDepositPool` and `LRTWithdrawalManager`, temporarily freezing user deposits and withdrawals.

## Finding Description

**Guard bypass:**

`NodeDelegator.startCheckpoint` enforces `onlyLRTOperator`: [1](#0-0) 

However, `IEigenPod.verifyStaleBalance` carries no caller restriction — the interface comment explicitly states it allows **anyone** to start a checkpoint as soon as a slashing occurs: [2](#0-1) [3](#0-2) 

`verifyCheckpointProofs` is similarly permissionless: [4](#0-3) 

**Price impact path:**

After checkpoint finalization with a negative `balanceDeltaWei`, EigenLayer calls `recordBeaconChainETHBalanceUpdate`, reducing the NDC's `beaconChainETHStrategy` shares. `NodeDelegator.getEffectivePodShares()` reads these shares via `NodeDelegatorHelper.getWithdrawableShare()`: [5](#0-4) 

`LRTDepositPool.getETHDistributionData()` aggregates `getEffectivePodShares()` across all NDCs: [6](#0-5) 

`LRTOracle._updateRsETHPrice()` calls `_getTotalEthInProtocol()` → `getTotalAssetDeposits()` → `getETHDistributionData()`, so the reduced shares flow directly into `newRsETHPrice`: [7](#0-6) 

**Pause trigger:**

The downside-protection block in `_updateRsETHPrice()` pauses `LRTDepositPool`, `LRTWithdrawalManager`, and the oracle itself if the price drop exceeds `pricePercentageLimit`: [8](#0-7) 

`updateRSETHPrice()` is `public whenNotPaused`, callable by any address: [9](#0-8) 

The asymmetry is critical: for upside price moves, an unprivileged caller is **reverted** (`PriceAboveDailyThreshold`) if the move exceeds the limit, but for downside moves the same unprivileged caller **successfully triggers a protocol-wide pause**: [10](#0-9) 

## Impact Explanation

**Medium — Temporary freezing of funds.** An unprivileged attacker can pause `LRTDepositPool` and `LRTWithdrawalManager`, preventing all user deposits and withdrawals until a privileged `LRTAdmin` manually unpauses. The price drop is real (the validator was genuinely slashed), but the attacker controls the *timing* — they can force the checkpoint at the worst moment (e.g., immediately after the initial slash, before the operator has decided it is appropriate to checkpoint, and before correlation penalties finalize). This causes a premature price drop that may cross `pricePercentageLimit` and trigger the pause ahead of the operator's intended schedule. No ETH is stolen; the freeze is temporary but requires privileged intervention to resolve.

## Likelihood Explanation

- Requires a validator registered to a `NodeDelegator` eigenPod to be slashed on the beacon chain — an external precondition, but beacon chain slashings occur in production.
- The attacker must construct valid beacon chain Merkle proofs for `verifyStaleBalance` and `verifyCheckpointProofs`. This is technically demanding but well-documented; EigenLayer's own test suite and tooling demonstrate the proof construction.
- No privileged access, no front-running dependency, no collusion required.
- The `eigenPod` address is publicly readable from `NodeDelegator.eigenPod`.
- `pricePercentageLimit` is intended for production use; if set to zero the pause is never triggered, but a non-zero value is the expected configuration.
- The attack is repeatable: after the admin unpauses, the attacker can repeat if another slashing event occurs.

## Recommendation

1. **Restrict `updateRSETHPrice` to privileged callers** (e.g., `onlyLRTOperator` or a keeper role) so that even if a checkpoint is force-started externally, the attacker cannot also force the oracle update that triggers the pause.
2. **Require sustained price deviation before pausing**: instead of pausing on a single `updateRSETHPrice` call, require the price to remain below the threshold for a configurable number of blocks or calls.
3. **Apply a TWAP or smoothing window** to `rsETHPrice` so a single checkpoint-induced drop does not immediately cross the pause threshold.
4. **Document the EigenLayer design**: acknowledge that `verifyStaleBalance` is intentionally permissionless and that checkpoints can be externally triggered, so operators are aware of the timing risk.

function testForceCheckpointPause() external {
    // Preconditions:
    // - nodeDelegator.eigenPod() has an ACTIVE validator that is slashed on beacon chain
    // - lrtOracle.pricePercentageLimit() > 0
    // - validator.lastCheckpointedAt < beaconTimestamp

    IEigenPod pod = nodeDelegator.eigenPod();

    // Step 1: unprivileged caller starts checkpoint via verifyStaleBalance
    // (bypasses NodeDelegator.startCheckpoint's onlyLRTOperator guard entirely)
    vm.prank(attacker);
    pod.verifyStaleBalance(beaconTimestamp, stateRootProof, validatorProof);

    // Step 2: unprivileged caller completes checkpoint
    vm.prank(attacker);
    pod.verifyCheckpointProofs(balanceContainerProof, balanceProofs);
    // EigenLayer now calls recordBeaconChainETHBalanceUpdate with negative balanceDeltaWei,
    // reducing the NDC's beaconChainETHStrategy shares.

    // Step 3: unprivileged caller triggers oracle price update
    vm.prank(attacker);
    lrtOracle.updateRSETHPrice();
    // _updateRsETHPrice reads reduced getEffectivePodShares() → lower newRsETHPrice
    // diff > pricePercentageLimit.mulWad(highestRsethPrice) → pause triggered

    // Assert: protocol is paused
    assertTrue(lrtDepositPool.paused());
    assertTrue(lrtWithdrawalManager.paused());
    assertTrue(lrtOracle.paused());
}
