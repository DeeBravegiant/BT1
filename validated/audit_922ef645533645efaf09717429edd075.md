### Title
Consensus Vote Signatures Are Never Verified — Any Peer Can Forge Quorum-Forming Votes - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` accepts prevote and precommit messages from the network and feeds them directly into the consensus state machine without ever verifying the cryptographic signature carried in the `Vote.signature` field. A `verify_precommit_vote_signature` function exists and is tested in isolation, but it is never called on the production path. Any network peer can therefore forge votes attributed to any committee member, manufacture a quorum, and drive the consensus engine to a `DecisionReached` outcome for an arbitrary `ProposalCommitment`.

---

### Finding Description

`Vote` carries a `signature: RawSignature` field alongside `voter: ContractAddress`. [1](#0-0) 

`handle_vote` in `SingleHeightConsensus` performs two membership checks — height equality and committee membership — but the comment at line 242 explicitly acknowledges that signature verification is absent:

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // TODO(Asmaa): verify the signature
    ...
    if !self.committee.members().iter().any(|s| s.address == vote.voter) {
        debug!("Ignoring vote from non validator: vote={:?}", vote);
        return VecDeque::new();
    }
    ...
    // Vote is accepted and forwarded to the state machine
    self.state_machine.handle_event(sm_vote)
}
``` [2](#0-1) 

The only guard is that `vote.voter` must appear in the committee address list. Because `voter` is a plain `ContractAddress` field supplied by the sender, any peer can set it to any committee member's address and attach a garbage signature. The check passes, the vote is counted with that member's weight, and the state machine advances.

The verification function that *should* be called exists but is only exercised in unit tests: [3](#0-2) 

A parallel unimplemented TODO exists for proposal-fin signature validation: [4](#0-3)

### Impact Explanation

The state machine reaches `DecisionReached` when it accumulates a quorum of precommits for a `ProposalCommitment` value, including the virtual-proposer requirement: [5](#0-4) 

Because signatures are not verified, an attacker who knows the committee composition (public information) can:

1. Send forged prevotes attributed to enough committee members to satisfy the prevote quorum.
2. Send forged precommits attributed to the same members (including the virtual proposer) for any `ProposalCommitment` of their choice.
3. The state machine emits `SMRequest::DecisionReached(decision)` with the attacker-chosen commitment.

The manager then calls `context.decision_reached`, which triggers `add_new_block` on the state-sync client and finalises the block. The committed `ProposalCommitment` (partial block hash) is wrong, producing an incorrect state root, wrong transaction set, wrong receipts, and wrong event commitments — matching the **Critical: wrong state, receipt, event, or block hash** impact category.

---

### Likelihood Explanation

The attack requires only a single network peer that can send messages on the broadcast vote channel. The committee addresses are public (fetched via `get_committee`). No privileged access, no key material, and no special timing are required. The attacker does not need to be a validator. The only prerequisite is the ability to send a `Vote` protobuf message to the target node, which is the normal p2p gossip path. [6](#0-5) 

---

### Recommendation

Call `verify_precommit_vote_signature` (or an equivalent `verify_prevote_signature`) inside `handle_vote` before the vote is forwarded to the state machine. Reject and report the peer on verification failure. The verification infrastructure already exists:

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // Verify the cryptographic signature before any further processing.
    let public_key = self.committee.members().iter()
        .find(|s| s.address == vote.voter)
        .map(|s| s.public_key);
    if let Some(pk) = public_key {
        match verify_vote_signature(&vote, pk) {
            Ok(true) => {}
            _ => {
                warn!("Signature verification failed for vote: {:?}", vote);
                return VecDeque::new(); // caller should also report the peer
            }
        }
    }
    ...
}

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
