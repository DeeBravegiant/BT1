### Title
Expired TEE Attestations Not Checked in `is_caller_an_attested_participant`, Allowing Participants with Lapsed Attestations to Submit Threshold Signatures — (File: `crates/contract/src/tee/tee_state.rs`)

---

### Summary

`is_caller_an_attested_participant` in `crates/contract/src/tee/tee_state.rs` verifies that a stored attestation *exists* for the calling participant but never checks whether that attestation has *expired*. Because `respond()`, `respond_ckd()`, and `respond_verify_foreign_tx()` all gate on this function, a participant whose TEE certificate has lapsed can continue submitting threshold signatures — and having them accepted by the contract — until an operator explicitly calls `clean_invalid_attestations()` or `verify_tee()`. This is the direct analog of the perps finding: just as expired orders were still included in mid-price and impact-price calculations, expired attestations are still accepted as valid authorization for signature submissions.

---

### Finding Description

`is_caller_an_attested_participant` performs four checks:

1. The caller is in the active participants list.
2. A `StoredAttestation` entry exists in `stored_attestations` keyed by the participant's TLS public key.
3. The stored `node_id.account_id` matches the signer.
4. The stored `node_id.account_public_key` matches the signer's Ed25519 key. [1](#0-0) 

It does **not** call `re_verify()` on the stored `VerifiedAttestation`, which is the function that checks `expiry_timestamp_seconds < timestamp_seconds`: [2](#0-1) 

The expiry check only runs inside `reverify_participants()` and `reverify_and_cleanup_participants()`, which are invoked by `verify_tee()` and `clean_invalid_attestations()` — both of which are **explicitly triggered**, not automatic: [3](#0-2) 

`assert_caller_is_attested_participant_and_protocol_active()` — the guard used by all three `respond*` methods and `vote_pk` — delegates entirely to `is_caller_an_attested_participant` with no additional expiry gate: [4](#0-3) 

The three critical signing endpoints that rely on this guard: [5](#0-4) [6](#0-5) [7](#0-6) 

`clean_invalid_attestations` removes expired entries from `stored_attestations`, which would cause `is_caller_an_attested_participant` to return `AttestationNotFound` on the next call. But until that cleanup runs, the expired entry remains in the map and the participant passes the check: [8](#0-7) 

---

### Impact Explanation

The TEE attestation is the contract's proof that a participant is running a trusted, approved MPC image. Once the attestation expires, that proof is void — the node may be running an outdated or compromised image. Despite this, the node can continue to:

- Submit `respond()` calls that resolve pending user signature yields.
- Submit `respond_ckd()` calls that resolve confidential key derivation yields.
- Submit `respond_verify_foreign_tx()` calls that resolve foreign-chain verification yields.
- Cast `vote_pk()` votes during DKG/resharing.

A Byzantine participant below the signing threshold whose attestation has lapsed (e.g., because they deliberately let it expire after a TEE image upgrade mandate) can continue to contribute signing shares. If the node is running a compromised image, it may exfiltrate key-share material during this window. This breaks the production safety invariant that only participants with currently-valid TEE attestations may participate in threshold operations, mapping to the **Medium** allowed impact: *participant-state or contract execution-flow manipulation that breaks production safety/accounting invariants without relying on network-level DoS or operator misconfiguration*.

---

### Likelihood Explanation

The window of exposure is the interval between attestation expiry and the next invocation of `verify_tee()` or `clean_invalid_attestations()`. Neither is called automatically. In a production network where `verify_tee()` is called on a periodic schedule (e.g., daily), a participant can operate with an expired attestation for the entire inter-call interval. A participant who deliberately submits an attestation with a minimal expiry (just enough to pass the insertion check in `add_participant`) and then lets it lapse faces no on-chain enforcement until an operator acts. The attack requires only that the participant be a current member of the participant set — no collusion above threshold is needed.

---

### Recommendation

Add an expiry check inside `is_caller_an_attested_participant` by calling `re_verify()` on the stored `VerifiedAttestation` before returning `Ok(())`:

```rust
pub(crate) fn is_caller_an_attested_participant(
    &self,
    participants: &Participants,
) -> Result<(), AttestationCheckError> {
    // ... existing checks ...

    // NEW: reject if the attestation has expired
    let now_seconds = Self::current_time_seconds();
    attestation
        .verified_attestation
        .re_verify(
            now_seconds,
            &self.get_allowed_mpc_docker_image_hashes(Duration::ZERO),
            &self.get_allowed_launcher_compose_hashes(),
            &self.get_accepted_measurements(),
        )
        .map_err(|_| AttestationCheckError::AttestationExpiredOrInvalid)?;

    Ok(())
}
```

This mirrors the check already performed in `reverify_participants` and ensures that the authorization gate used by all `respond*` and `vote_pk` methods enforces attestation validity at call time, not only when an operator explicitly sweeps for expired entries.

---

### Proof of Concept

1. A participant (P) is in the active participant set with a valid attestation that expires at timestamp `T`.
2. At time `T+1`, P's attestation is expired. No one has called `verify_tee()` or `clean_invalid_attestations()` yet, so P's entry remains in `stored_attestations`.
3. A user submits a `sign()` request; the contract parks a yield.
4. P's node (now running an unverified or compromised image) computes a signature share and calls `respond(request, signature)`.
5. `assert_caller_is_attested_participant_and_protocol_active()` calls `is_caller_an_attested_participant()`, which finds P's entry in `stored_attestations`, matches account ID and public key, and returns `Ok(())` — **without checking expiry**.
6. The contract verifies the cryptographic signature (which is valid because P still holds its key share) and resolves the user's yield with the signature.
7. The user receives a threshold signature produced with participation from a node whose TEE environment is no longer verified. [1](#0-0) [4](#0-3) [9](#0-8)

    pub(crate) fn reverify_participants(

            &allowed_mpc_docker_image_hashes,
            &allowed_launcher_compose_hashes,
            &allowed_measurements,
        ) {
            Ok(()) => TeeQuoteStatus::Valid,
            Err(err) => TeeQuoteStatus::Invalid(err.to_string()),
        }
    }

    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }

    pub fn re_verify(
        &self,
        timestamp_seconds: u64,
        allowed_mpc_docker_image_hashes: &[NodeImageHash],
        allowed_launcher_docker_compose_hashes: &[LauncherDockerComposeHash],
        allowed_measurements: &[ExpectedMeasurements],
    ) -> Result<(), VerificationError> {
        match self {
            Self::Dstack(ValidatedDstackAttestation {
                mpc_image_hash,
                launcher_compose_hash,
                expiry_timestamp_seconds: expiration_timestamp_seconds,
                measurements,
            }) => {
                let attestation_has_expired = *expiration_timestamp_seconds < timestamp_seconds;

                if attestation_has_expired {
                    return Err(VerificationError::Custom(format!(
                        "The attestation expired at t = {:?}, time_now = {:?}",
                        expiration_timestamp_seconds, timestamp_seconds
                    )));
                }

                let () = verify_mpc_hash(mpc_image_hash, allowed_mpc_docker_image_hashes)?;
                let () = verify_launcher_compose_hash(
                    launcher_compose_hash,
                    allowed_launcher_docker_compose_hashes,
                )?;

                verify_measurements(measurements, allowed_measurements)?;

                Ok(())
            }
            Self::Mock(mock_attestation) => mock_attestation.verify_constraints(
                timestamp_seconds,
                allowed_mpc_docker_image_hashes,
                allowed_launcher_docker_compose_hashes,
                allowed_measurements,
            ),
        }
    }

    #[handle_result]
    pub fn respond(
        &mut self,
        request: SignatureRequest,
        response: dtos::SignatureResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!("respond: signer={}, request={:?}", &signer, &request);

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain)?;

        let signature_is_valid = match (&response, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                // generate the expected public key
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");
                let affine = *k256::PublicKey::try_from(&secp_pk)
                    .expect("stored key is always valid")
                    .as_affine();
                let expected_public_key =
                    derive_key_secp256k1(&affine, &request.tweak).map_err(RespondError::from)?;

                let payload_hash = request.payload.as_ecdsa().expect("Payload is not ECDSA");

                // Check the signature is correct
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    payload_hash,
                    &expected_public_key,
                )
                .is_ok()
            }
            (
                dtos::SignatureResponse::Ed25519 { signature },
                PublicKeyExtended::Ed25519 {
                    edwards_point: public_key_edwards_point,
                    ..
                },
            ) => {
                let derived_public_key_edwards_point = derive_public_key_edwards_point_ed25519(
                    &public_key_edwards_point,
                    &request.tweak,
                );
                let derived_public_key_32_bytes =
                    dtos::Ed25519PublicKey::from(derived_public_key_edwards_point.compress());

                let message = request.payload.as_eddsa().expect("Payload is not EdDSA");

                near_mpc_signature_verifier::verify_eddsa_signature(
                    signature,
                    message,
                    &derived_public_key_32_bytes,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_signature_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }
