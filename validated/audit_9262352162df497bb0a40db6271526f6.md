### Title
Missing `state_diff_commitment` Cross-Check in P2P Sync State Diff Assembly — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

`StateDiffStreamBuilder::parse_data_for_block` assembles peer-supplied `StateDiffChunk` messages into a `ThinStateDiff` and writes it to storage, but never verifies that `calculate_state_diff_hash(&assembled_diff)` equals the `state_diff_commitment` stored in the block header. A malicious p2p peer can send chunks that satisfy the length check but hash to a completely different value, permanently corrupting the node's state diff storage and causing `starknet_getStateUpdate` to return state data that does not match the on-chain commitment.

---

### Finding Description

`parse_data_for_block` in `StateDiffStreamBuilder` reads `state_diff_length` from the stored header and accumulates `StateDiffChunk` messages until the total length matches: [1](#0-0) 

It then validates only:
- Total length equals `target_state_diff_len` (line 99–104)
- No empty chunks (line 90–92)
- No conflicting/duplicate parts via `unite_state_diffs` and `validate_deprecated_declared_classes_non_conflicting` [2](#0-1) 

After passing these checks, the assembled diff is returned and written directly to storage: [3](#0-2) 

**There is no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment` anywhere in this path.**

The header itself is also accepted from the peer with only a block-number check and a signature-count check — no block hash verification and no `state_diff_commitment` verification: [4](#0-3) 

The `verify_state_diff_hash` guard that does perform this check exists only in the **committer** component, which is a separate service used by the block-producing sequencer node, not by the p2p sync client: [5](#0-4) 

The committer's check is never invoked during p2p sync.

---

### Impact Explanation

A syncing node that accepts state diff data from a malicious peer will store a `ThinStateDiff` whose Poseidon hash does not match the `state_diff_commitment` field in the corresponding `BlockHeader`. Every subsequent call to `starknet_getStateUpdate` for that block will return the attacker-controlled state diff, which is an authoritative-looking wrong value. Downstream consumers (indexers, bridges, wallets, provers) that rely on this RPC to reconstruct state will silently receive corrupted data.

---

### Likelihood Explanation

Any unauthenticated p2p peer the syncing node connects to can execute this attack. The attacker only needs to:
1. Send a `SignedBlockHeader` with an arbitrary `state_diff_commitment` and a chosen `state_diff_length = N`.
2. Send `N` length-units of `StateDiffChunk` messages with attacker-chosen content.

No validator, operator, or privileged access is required. The attack is silent — the node logs no error and advances its state marker normally.

---

### Recommendation

After assembling the full `ThinStateDiff` in `parse_data_for_block`, compute its hash and compare it against the commitment stored in the header before returning `Ok(Some(...))`:

```rust
let stored_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment;

if let Some(commitment) = stored_commitment {
    let computed = calculate_state_diff_hash(&result);
    if computed != commitment {
        return Err(ParseDataError::BadPeer(BadPeerError::InvalidStateDiffHash {
            block_number,
            expected: commitment,
            got: computed,
        }));
    }
}
```

This mirrors the existing guard in `apollo_committer` and closes the gap for the p2p sync path. [6](#0-5) 

---

### Proof of Concept

1. Start a syncing node with `P2pSyncClient`.
2. Act as a peer; send a `SignedBlockHeader` for block 0 with `state_diff_length = 1` and `state_diff_commitment = H(legit_diff)` where `legit_diff` contains storage key `K → V_legit`.
3. Send a single `StateDiffChunk::ContractDiff` with the same length-1 contribution but storage key `K → V_evil`.
4. The node accepts the chunk (length matches), writes the evil diff to storage, and advances the state marker.
5. Call `starknet_getStateUpdate(block_number=0)` — the response contains `K → V_evil`, not `K → V_legit`, while the header still advertises `state_diff_commitment = H(legit_diff)`.
6. Assert `calculate_state_diff_hash(response.state_diff) != response.block_header.state_diff_commitment` — the invariant is broken.

    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }
