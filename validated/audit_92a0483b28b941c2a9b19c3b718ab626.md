### Title
Unvalidated `fee_proposal_fri` During Startup Window Allows Arbitrary Fee Market Manipulation — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

During the first `fee_proposal_window_size` blocks (the startup window), `is_proposal_init_valid` skips all bounds checking on the proposer-supplied `fee_proposal_fri` field in `ProposalInit`. A malicious proposer can set `fee_proposal_fri` to any value — including `u128::MAX` — and every honest validator will accept the proposal without rejection. The injected value is then stored in `fee_proposals_window`, poisons the `fee_actual` median computation once the window fills, and drives the sequencer's L2 gas price to an attacker-controlled level for all subsequent blocks.

### Finding Description

In `is_proposal_init_valid`, the bounds check on `fee_proposal_fri` is gated on `proposal_init_validation.fee_actual` being `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    ...bounds check...
}
``` [1](#0-0) 

`fee_actual` is computed by `compute_fee_actual`, which returns `None` whenever `height < window_size` (checked_sub underflows) or any entry in the window is missing or `None`:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

`fee_actual` is passed into `ProposalInitValidation` at both call sites in `validate_proposal` and `set_height_and_round`: [3](#0-2) 

The honest proposer falls back to `l2_gas_price` when `fee_actual` is `None`:

```rust
let Some(fee_actual) = fee_actual else {
    warn!("fee_actual unavailable, freezing fee_proposal at l2_gas_price");
    return self.l2_gas_price;
};
``` [4](#0-3) 

But the validator enforces **no corresponding constraint** on the received value. Any `fee_proposal_fri` — including `GasPrice(u128::MAX)` — passes `is_proposal_init_valid` during the startup window.

After consensus decides the block, `decision_reached` calls `record_fee_proposal`, which stores `init.fee_proposal_fri` into `fee_proposals_window`: [5](#0-4) 

Once `window_size` blocks have been committed, `compute_fee_actual` returns the median of the stored values. If a malicious proposer injected `u128::MAX` for even half the startup blocks, the median becomes `u128::MAX`. This value is then used as the floor for the next L2 gas price via `calculate_next_l2_gas_price_for_fin`.

The injected `fee_proposal_fri` is also bound into the `ProposalCommitment` via `proposal_commitment_from`:

```rust
ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
``` [6](#0-5) 

And forwarded to the cende blob pipeline as `FeeProposalInfo { fee_proposal_fri: init.fee_proposal_fri }`, persisting the attacker-controlled value in the centralized recorder. [7](#0-6) 

The `fee_proposal_fri` is also stored in `BlockHeaderWithoutHash.fee_proposal_fri`, so it is read back by `initialize_fee_proposals_window` on node restart, making the poisoning persistent across restarts.

### Impact Explanation

Once the startup window is poisoned, `fee_actual` becomes attacker-controlled. `calculate_next_l2_gas_price_for_fin` uses `fee_actual` as a floor for the next L2 gas price. With `fee_actual = u128::MAX`, the L2 gas price for all subsequent blocks becomes `u128::MAX`. The gateway's `validate_resource_bounds` then rejects every user transaction whose `max_price_per_unit` is below this threshold — effectively all of them — causing a complete denial of service for the fee market. This is an incorrect fee/gas accounting effect with direct economic impact, matching the "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" impact category.

### Likelihood Explanation

The startup window is exactly the first `fee_proposal_window_size` blocks (10 blocks per `VersionedConstants`). At network launch or after a revert to genesis, the proposer rotation is deterministic and known in advance. A Byzantine validator who is scheduled to propose during any of these 10 blocks can inject the extreme value. In a BFT network with `f < n/3` Byzantine nodes, a single Byzantine proposer suffices — honest validators will accept the proposal because the bounds check is unconditionally skipped when `fee_actual` is `None`. No special privilege beyond being a scheduled proposer is required.

### Recommendation

When `fee_actual` is `None` (startup window), the validator should enforce that `fee_proposal_fri` equals the locally-computed fallback value (`l2_gas_price`), mirroring the constraint the honest proposer already applies:

```rust
// When fee_actual is unavailable, the honest proposer freezes at l2_gas_price.
// Enforce the same constraint on the validator side.
match (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri) {
    (None, Some(fee_proposal)) => {
        let expected = proposal_init_validation.l2_gas_price_fri; // the local fallback
        if fee_proposal != expected {
            return Err(ValidateProposalError::InvalidProposalInit(...));
        }
    }
    (Some(fee_actual), Some(fee_proposal)) => {
        // existing bounds check
    }
    ...
}
```

Alternatively, apply a hard absolute cap (e.g., `MAX_REASONABLE_GAS_PRICE`) to `fee_proposal_fri` regardless of whether `fee_actual` is available.

### Proof of Concept

1. Network launches at height 0. `fee_proposal_window_size = 10`.
2. Malicious proposer is scheduled for block 0 (or any block in `[0, 9]`).
3. Proposer sends `ProposalInit { fee_proposal_fri: Some(GasPrice(u128::MAX)), ... }`.
4. Every honest validator calls `is_proposal_init_valid`. `compute_fee_actual` returns `None` (height 0 < window_size 10). The `if let (Some(fee_actual), Some(fee_proposal))` arm does not fire. Validation passes.
5. Consensus decides the block. `decision_reached` calls `record_fee_proposal(height=0, Some(GasPrice(u128::MAX)))`.
6. Malicious proposer repeats for blocks 1–9 (or bribes/controls enough proposers to fill ≥ 5 of the 10 slots with `u128::MAX`).
7. At height 10, `compute_fee_actual` returns `Some(GasPrice(u128::MAX))` (median of the poisoned window).
8. `calculate_next_l2_gas_price_for_fin` sets the next L2 gas price to `u128::MAX`.
9. All subsequent user transactions are rejected by the gateway's `validate_resource_bounds` because `max_price_per_unit < u128::MAX`. [1](#0-0) [8](#0-7) [9](#0-8)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
