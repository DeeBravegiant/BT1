Audit Report

## Title
Unprivileged Depositor Can Exhaust Daily Mint Limit to DoS All Deposits for Up to 24 Hours — (`contracts/pools/RSETHPoolV3.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`, `contracts/pools/RSETHPoolV3WithNativeChainBridge.sol`)

## Summary
The `limitDailyMint` modifier in all three L2 pool contracts tracks cumulative minted rsETH in `dailyMintAmount`, which is only ever incremented on deposit and reset once per calendar day. Any unprivileged depositor with sufficient capital can exhaust the entire daily limit in a single transaction, causing every subsequent deposit within that 24-hour window to revert with `DailyMintLimitExceeded`. The attacker retains wrsETH (a yield-bearing asset) and pays only the protocol fee, making the attack economically viable and indefinitely repeatable.

## Finding Description
The `limitDailyMint` modifier is applied to both `deposit(string)` and `deposit(address,uint256,string)` in all three contracts:

```solidity
modifier limitDailyMint(uint256 amount, address token) {
    // ...
    if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
        revert DailyMintLimitExceeded();
    }
    dailyMintAmount += rsETHAmount;   // only ever increases
    _;
}
```

`dailyMintAmount` is incremented on every successful deposit and is only reset to zero when `getCurrentDay() > lastMintDay` — once per 24-hour period relative to `startTimestamp`. There is no code path that decrements it. The only function that conceptually reverses a deposit, `swapAssetToPremintedRsETH`, is gated behind `onlyRole(OPERATOR_ROLE)` and does not touch `dailyMintAmount` at all.

An attacker deposits an amount large enough to push `dailyMintAmount` to `dailyMintLimit`. After this, the modifier's check `dailyMintAmount + rsETHAmount > dailyMintLimit` will revert for every subsequent caller — regardless of deposit size — until the next day boundary. The attacker receives wrsETH in exchange, which is a yield-bearing L2 wrapper for rsETH and can be held, bridged, or sold. The only cost to the attacker is the protocol fee (`feeBps`), which is capped at 10% in `RSETHPoolV3` and `RSETHPoolV3WithNativeChainBridge`, making the attack economically viable at any reasonable fee level.

## Impact Explanation
**Medium — Temporary freezing of funds.**

All deposit paths are blocked for up to 24 hours across every affected L2 pool. Users who need to enter the protocol during that window are completely denied service. The freeze is self-resolving at the next day boundary but can be repeated every day at minimal cost to the attacker (only the fee on the exhausting deposit).

## Likelihood Explanation
**Medium.**

Any address with sufficient capital to reach `dailyMintLimit` can execute this attack. The attacker retains full economic value in wrsETH (minus the protocol fee), so the only barrier is capital availability, not willingness to lose funds. On chains where the daily limit is set conservatively to limit bridge risk, the capital requirement may be modest. The attack can be repeated every 24 hours indefinitely.

## Recommendation
1. **Decrement `dailyMintAmount` on reverse swaps.** `swapAssetToPremintedRsETH` should reduce `dailyMintAmount` by the corresponding rsETH amount when called.
2. **Rate-limit per depositor** rather than (or in addition to) globally, so a single actor cannot exhaust the shared limit.
3. **Allow the admin to reset `dailyMintAmount`** in an emergency without waiting for the next calendar day.
4. **Consider a per-transaction cap** (maximum single deposit size) to raise the cost of limit exhaustion without removing the global safety rail.

## Proof of Concept
```solidity
// Foundry test sketch (same pattern applies to all three pool variants)

function test_exhaustDailyMintLimit_DoS() public {
    // Setup: dailyMintLimit = L (e.g. 100 ether worth of rsETH)
    // Attacker has L ETH

    // Step 1: Attacker exhausts the daily limit in one deposit
    vm.prank(attacker);
    pool.deposit{value: L_in_ETH}("ref");
    // dailyMintAmount == dailyMintLimit now

    // Step 2: Any subsequent user deposit reverts
    vm.prank(victim);
    vm.expectRevert(RSETHPoolV3.DailyMintLimitExceeded.selector);
    pool.deposit{value: 1 ether}("ref");

    // Step 3: Attacker holds wrsETH worth ~L ETH (minus fee)
    // Step 4: Next day, attacker repeats — dailyMintAmount resets to 0 on first deposit
    vm.warp(block.timestamp + 1 days);
    vm.prank(attacker);
    pool.deposit{value: L_in_ETH}("ref"); // succeeds again, re-exhausts limit
}
```
