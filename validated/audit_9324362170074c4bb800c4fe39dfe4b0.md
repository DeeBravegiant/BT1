Audit Report

## Title
Missing Sign Check Before `int256`→`uint256` Cast Allows Chainlink Negative Price to Freeze Protocol Operations - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` casts the `int256 price` returned by Chainlink's `latestRoundData()` directly to `uint256` without verifying the value is positive. Under Solidity 0.8.27 checked arithmetic, a negative price causes the subsequent `* 1e18` multiplication to overflow and revert. This revert propagates through every protocol path that calls `getAssetPrice()` — deposits, withdrawals, price updates, and conversions — temporarily freezing those operations until an admin replaces the oracle.

## Finding Description
In `ChainlinkPriceOracle.getAssetPrice()` (lines 52–54), the raw `int256 price` from `latestRoundData()` is cast directly to `uint256` with no sign guard:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

For any negative `price` (e.g., `-1`), two's-complement reinterpretation yields `uint256(int256(-1)) = 2^256 − 1`. The immediately following `* 1e18` then overflows under Solidity 0.8.x checked arithmetic, causing an unconditional revert.

The sister oracle `ChainlinkOracleForRSETHPoolCollateral` already applies the correct guard at line 32:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
```

`ChainlinkPriceOracle` has no equivalent check.

The revert propagates through the following confirmed call chains (verified by grep — 6 call-sites in `LRTDepositPool.sol`, 4 in `LRTWithdrawalManager.sol`, 4 in `LRTConverter.sol`, 7 in `LRTOracle.sol`):

- `LRTOracle._getTotalEthInProtocol()` (line 339) → `getAssetPrice()` → revert → `updateRSETHPrice()` frozen.
- `LRTDepositPool.depositAsset()` / `depositETH()` → `getAssetPrice()` → revert → deposits frozen.
- `LRTWithdrawalManager` withdrawal paths → `getAssetPrice()` → revert → withdrawals frozen.
- `LRTConverter` conversion paths → `getAssetPrice()` → revert → conversions frozen.

No existing check in `ChainlinkPriceOracle` guards against this; the `onlySupportedAsset` modifier only validates that a price feed is registered, not that the returned value is valid.

## Impact Explanation
**Medium — Temporary freezing of funds.** A single Chainlink LST/ETH feed returning a negative or zero value causes all deposits, withdrawals, and price updates to revert for every asset routed through `ChainlinkPriceOracle`. The freeze persists until an admin replaces the oracle contract, which may require governance delay. User funds are not stolen but are inaccessible during the freeze window.

## Likelihood Explanation
Chainlink's `latestRoundData()` specification does not guarantee a positive `answer`; deprecated, migrated, or malfunctioning feeds have historically returned zero or negative values. No attacker action is required — the condition arises from a feed anomaly alone. `ChainlinkPriceOracle` is the oracle for all Chainlink-backed LST assets; a single affected feed is sufficient to trigger the freeze across the entire protocol.

## Recommendation
Add a positivity check immediately after reading the price, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Also consider adding staleness checks (`answeredInRound < roundId`, `updatedAt == 0`) consistent with the sister oracle.

## Proof of Concept
1. A Chainlink LST/ETH feed (e.g., stETH/ETH) returns `answer = -1` due to feed deprecation or malfunction.
2. Any caller invokes `LRTDepositPool.depositAsset(stETH, amount, minRSETH, ref)`.
3. Internally: `_beforeDeposit()` → `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)`.
4. `uint256(int256(-1))` = `2^256 − 1`; `(2^256 − 1) * 1e18` overflows under Solidity 0.8.27 checked arithmetic → unconditional revert.
5. All deposits revert. `updateRSETHPrice()` also reverts (same path via `_getTotalEthInProtocol()` → `getAssetPrice()`). Withdrawals and conversions revert at their respective `getAssetPrice()` call-sites.
6. Protocol operations remain frozen until admin replaces the oracle via `LRTOracle.updatePriceOracleFor()`.

**Foundry fork test plan:** Deploy a mock `AggregatorV3Interface` that returns `answer = -1`. Register it as the price feed for stETH in `ChainlinkPriceOracle`. Call `LRTDepositPool.depositAsset(stETH, 1e18, 0, "")` and assert the call reverts with arithmetic overflow. Repeat for `LRTOracle.updateRSETHPrice()`.
