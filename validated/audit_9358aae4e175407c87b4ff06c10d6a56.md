Audit Report

## Title
Unbounded Nested Loop in `updateRSETHPrice` via `getAssetUnstaking` Across Assets, NDCs, and EigenLayer Queued Withdrawals Can Permanently OOG - (File: contracts/LRTOracle.sol)

## Summary

`LRTOracle.updateRSETHPrice()` is a public function that executes a triple-nested loop: over all supported assets, over all node delegators per asset, and over all EigenLayer queued withdrawals per node delegator. As the protocol scales to its configured operational limits through normal growth, this function can permanently exceed the block gas limit, making the rsETH price permanently stale and disabling fee minting and price-based pause protection.

## Finding Description

The public `updateRSETHPrice()` at `LRTOracle.sol:87` calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`. Inside `_getTotalEthInProtocol()` (L331–349), the code iterates over every supported asset and calls `ILRTDepositPool.getTotalAssetDeposits(asset)`: [1](#0-0) 

`getTotalAssetDeposits` calls `getAssetDistributionData`, which iterates over every entry in `nodeDelegatorQueue` and calls `getAssetUnstaking(asset)` on each NDC: [2](#0-1) 

`getAssetUnstaking` in `NodeDelegator` (L405–427) fetches **all** queued withdrawals from EigenLayer's `DelegationManager` and iterates over them with a nested loop over strategies per withdrawal: [3](#0-2) 

The total gas cost scales as:

```
|supportedAssets| × |nodeDelegatorQueue| × |queuedWithdrawals_per_NDC| × |strategies_per_withdrawal|
```

- `supportedAssetList` has no hard cap; assets are added via `TIME_LOCK_ROLE` with no maximum enforced: [4](#0-3) 

- `maxNodeDelegatorLimit` (default 10) is freely raisable by admin with no upper bound: [5](#0-4) 

- `maxUncompletedWithdrawalCount` in `LRTUnstakingVault` is a global counter across all NDCs, also admin-configurable with no hard ceiling: [6](#0-5) 

Each parameter is set independently for legitimate operational reasons. No single parameter appears dangerous in isolation, but their product can easily exceed the 30M gas block limit at normal operational scale (e.g., 10 assets × 10 NDCs × 50 queued withdrawals × 2 strategies = 10,000 external calls, each involving EigenLayer internal logic well above the base STATICCALL cost).

## Impact Explanation

**Medium — Unbounded gas consumption.** If `updateRSETHPrice()` permanently OOGs:

1. The stored `rsETHPrice` becomes permanently stale. Both `getRsETHAmountToMint` (deposit path) and `getExpectedAssetAmount` (withdrawal path) consume this stale value, causing all users to receive incorrect rsETH or asset amounts: [7](#0-6) 

2. Protocol fee minting stops entirely, as fees are only minted inside `_updateRsETHPrice()`: [8](#0-7) 

3. The price-based automatic pause protection stops functioning: [9](#0-8) 

## Likelihood Explanation

No attacker action is required. The protocol is designed to scale: more LSTs are added over time, more NDCs are deployed to handle restaking capacity, and each NDC accumulates queued EigenLayer withdrawals during normal operation. Each admin action (adding an asset, adding an NDC, raising the withdrawal cap) is individually reasonable. The combination of these independently-set parameters naturally reaches the OOG threshold as the protocol grows. `updateRSETHPrice()` is public and callable by any address, so once the gas threshold is crossed, no privileged actor can rescue it through this function (though `updateRSETHPriceAsManager` shares the same internal path and would also OOG). [10](#0-9) 

## Recommendation

1. **Decouple `getAssetUnstaking` from the price update path.** Cache the unstaking amount in storage and update it lazily (e.g., when `initiateUnstaking` / `completeUnstaking` are called) rather than recomputing it via a live EigenLayer query on every price update.

2. **Alternatively**, remove the `getAssetUnstaking` contribution from `_getTotalEthInProtocol` and account for it separately, since assets in the EigenLayer withdrawal queue are already committed and their value does not change the rsETH price until they are received.

3. **Add a hard cap** on `maxNodeDelegatorLimit` and `maxUncompletedWithdrawalCount` that is validated against the block gas limit, and enforce a maximum on `supportedAssetList` length.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
