Audit Report

## Title
`RSETHPriceFeed.getRoundData()` Returns Fabricated Historical Price Using Current rsETH/ETH Rate - (File: contracts/oracles/RSETHPriceFeed.sol)

## Summary
`RSETHPriceFeed.getRoundData()` implements the `AggregatorV3Interface` but computes the returned `answer` by multiplying the historical ETH/USD price for the requested round by the **current** `rsETHPrice()` from `LRTOracle`, rather than the rsETH/ETH rate that existed at that historical round. Since no historical rsETH/ETH rate registry exists on-chain, the function cannot be implemented correctly, yet it silently returns a fabricated price instead of reverting. Any external protocol consuming this feed for historical price lookups receives systematically incorrect data.

## Finding Description
In `contracts/oracles/RSETHPriceFeed.sol` at lines 53–61, `getRoundData` fetches the historical ETH/USD price from the Chainlink aggregator for `_roundId`, then multiplies it by `RS_ETH_ORACLE.rsETHPrice()`:

```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.getRoundData(_roundId);
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```

`RS_ETH_ORACLE.rsETHPrice()` resolves to `LRTOracle.rsETHPrice` (line 28 of `contracts/LRTOracle.sol`), a single storage slot updated only when `_updateRsETHPrice()` is called (line 313). It holds the **current** rsETH/ETH exchange rate, not any historical rate. There is no on-chain mapping of historical rsETH/ETH rates keyed by Chainlink round ID.

The result is: `answer = currentRsETHRate × historicalETHUSDPrice`, which is not the historical rsETH/USD price. The `latestRoundData()` function (lines 63–70) is correct by coincidence — both its ETH/USD price and the rsETH/ETH rate are current — but `getRoundData` for any round other than the most recent one returns a fabricated value.

## Impact Explanation
`RSETHPriceFeed` is deployed as an `AggregatorV3Interface`-compatible contract explicitly designed for external integration. The `getRoundData` function is a standard interface method that callers expect to return accurate historical prices. Instead it returns `currentRsETHRate × historicalETHUSDPrice`. As rsETH accrues yield over time, the current rate is always ≥ any historical rate, so the function systematically overstates historical rsETH/USD prices. This maps to **Low — Contract fails to deliver promised returns, but doesn't lose value**: the price feed contract fails to deliver the accurate historical price data it promises via the `AggregatorV3Interface`, but no funds within LRT-rsETH are directly lost.

## Likelihood Explanation
The function is `external view` and callable by any address without preconditions. Any external protocol that integrates `RSETHPriceFeed` as a Chainlink-compatible feed and calls `getRoundData` for any round other than the latest will receive incorrect data. The Chainlink interface is specifically designed for third-party consumption, making this a realistic integration path.

## Recommendation
Since the protocol does not store historical rsETH/ETH rates on-chain, the correct fix is to revert in `getRoundData` with a `NotSupported()` error to prevent silent mispricing:

```solidity
function getRoundData(uint80 /*_roundId*/) external pure returns (...) {
    revert NotSupported();
}
```

Alternatively, implement a historical rate checkpoint mapping updated whenever `updateRSETHPrice()` is called, keyed by the current Chainlink round ID at update time.

## Proof of Concept
Deploy `RSETHPriceFeed` pointing to a mock `ETH_TO_USD` feed and a mock `LRTOracle`. Configure:
- Round R1 (30 days ago): ETH/USD = $2,000, rsETH/ETH at that time = 1.02 → correct answer = $2,040
- Current rsETH/ETH = 1.05 (stored in `LRTOracle.rsETHPrice`)

Call `getRoundData(R1_roundId)`:
```
answer = int256(1.05e18) * int256(2000e8) / 1e18 = 2100e8  // $2,100 — WRONG
```
Expected: `2040e8`. The function returns a price inflated by current rsETH appreciation. A Foundry fork test can confirm this by recording `rsETHPrice` at two different block timestamps and verifying `getRoundData` returns the same inflated value for both rounds rather than the historically correct prices.
