Audit Report

## Title
Missing Chainlink `latestRoundData()` Return Value Validation Enables rsETH Price Manipulation - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` and casts the returned `int256 price` directly to `uint256` with no checks for non-positive values, stale rounds, or incomplete rounds. When a Chainlink feed returns `price = 0` (circuit-breaker floor), the deflated asset value propagates into `LRTOracle._getTotalEthInProtocol()`, allowing any unprivileged caller to commit a deflated `rsETHPrice` to storage via the public `updateRSETHPrice()`, then deposit a correctly-priced asset and receive excess rsETH, extracting value from existing rsETH holders.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` performs no validation on the Chainlink response:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L49-55
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Three checks are absent: `price <= 0`, `answeredInRound < roundId`, and `updatedAt == 0`. The sister contract `ChainlinkOracleForRSETHPoolCollateral` in the same repo performs all three correctly at L30-32.

**Exploit path (circuit-breaker `price = 0`):**

1. A Chainlink feed for supported LST asset `X` hits its minimum-price circuit breaker and returns `price = 0`.
2. `getAssetPrice(X)` returns `uint256(0) * 1e18 / decimals = 0`.
3. Attacker calls `LRTOracle.updateRSETHPrice()` (public, no access control, L87). Inside `_updateRsETHPrice()`, `_getTotalEthInProtocol()` (L331-349) sums `totalAssetAmt.mulWad(assetER)` for each asset; asset `X` contributes `0`, understating `totalETHInProtocol`.
4. `newRsETHPrice = (totalETHInProtocol - fee) / rsethSupply` is deflated and written to `rsETHPrice` storage (L313).
5. The downside-pause guard at L273-274 only fires when `pricePercentageLimit > 0`; its default is `0`, so it is inactive.
6. Attacker calls `depositAsset(Y, amount, 0, "")` for a correctly-priced asset `Y`. `getRsETHAmountToMint` computes `(amount * getAssetPrice(Y)) / rsETHPrice` (L520) — the deflated denominator yields excess rsETH.
7. When the Chainlink feed recovers, attacker redeems excess rsETH at the true backing ratio, extracting value from all other rsETH holders.

**Negative price scenario:** A negative `int256` cast to `uint256` produces `2^256 - |price|`, inflating `totalETHInProtocol` and `rsETHPrice` astronomically, which would harm depositors by minting near-zero rsETH per deposit.

## Impact Explanation

**High — Theft of unclaimed yield / value from existing rsETH holders.** The attacker mints rsETH at a below-fair-value exchange rate and redeems at the recovered rate, extracting value proportional to the TVL share of the affected LST. No privileged access is required; both `updateRSETHPrice()` and `depositAsset()` are callable by any external user.

## Likelihood Explanation

Chainlink circuit-breaker events (returning `0` or a min/max sentinel) are documented and have occurred on mainnet (LUNA crash, stETH depeg). The attack requires no special permissions: `updateRSETHPrice()` is public and `depositAsset()` is open to any user. An attacker only needs to observe the on-chain Chainlink answer and act within the same block or shortly after. The `pricePercentageLimit` guard is disabled by default (`0`), leaving no on-chain protection until an admin explicitly configures it.

## Recommendation

Add the following checks inside `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

if (price <= 0)                  revert InvalidPrice();
if (updatedAt == 0)              revert IncompleteRound();
if (answeredInRound < roundId)   revert StalePrice();
// Optionally: if (block.timestamp - updatedAt > MAX_STALENESS) revert StalePrice();
```

## Proof of Concept

**Foundry fork test outline:**

1. Fork mainnet at a block where a supported LST Chainlink feed is live.
2. Deploy a mock `AggregatorV3Interface` that returns `price = 0` for asset `X`.
3. Call `ChainlinkPriceOracle.updatePriceFeedFor(X, mockFeed)` (as LRTManager).
4. Record `rsETHPrice` before attack.
5. Call `LRTOracle.updateRSETHPrice()` as an unprivileged EOA — assert `rsETHPrice` is now deflated.
6. Call `LRTDepositPool.depositAsset(Y, amount, 0, "")` as attacker — assert rsETH minted exceeds `amount * getAssetPrice(Y) / trueRsETHPrice`.
7. Restore mock feed to correct price; call `updateRSETHPrice()` again.
8. Initiate withdrawal for attacker's rsETH — assert ETH received exceeds original deposit value, confirming value extraction from other holders.
