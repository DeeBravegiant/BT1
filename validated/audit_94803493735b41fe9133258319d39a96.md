Audit Report

## Title
Permissionless `updateRSETHPrice()` Triggers Auto-Pause of `LRTWithdrawalManager`, Blocking Already-Unlocked Withdrawals — (`contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function that, upon detecting a price drop exceeding `pricePercentageLimit` relative to `highestRsethPrice`, automatically calls `pause()` on `LRTWithdrawalManager`. Because `completeWithdrawal` is gated by `whenNotPaused`, users whose rsETH has already been burned during `unlockQueue` cannot collect their LST/ETH until an admin manually calls `unpause()`. The freeze duration is unbounded from the user's perspective.

## Finding Description

**Permissionless entrypoint:**
`updateRSETHPrice()` carries only `whenNotPaused` — no role guard — making it callable by any EOA or contract at any time. [1](#0-0) 

**Auto-pause on price drop:**
Inside `_updateRsETHPrice()`, when `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit.mulWad(highestRsethPrice)`, the function calls `withdrawalManager.pause()` and returns immediately. [2](#0-1) 

`LRTWithdrawalManager.pause()` requires `PAUSER_ROLE`. For the auto-pause to function as designed, the oracle must hold that role in production — which is the clear intent of the code. [3](#0-2) 

**All user-facing withdrawal paths blocked:**
`completeWithdrawal`, `initiateWithdrawal`, `instantWithdrawal`, and `unlockQueue` all carry `whenNotPaused`. [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

**Critical gap — rsETH already burned before `completeWithdrawal`:**
During `unlockQueue`, rsETH is burned from the contract immediately. [8](#0-7) 

Users whose requests have been unlocked have already surrendered their rsETH. `completeWithdrawal` is their only path to receive LST/ETH, but it is blocked by the pause. [9](#0-8) 

**Price drop precondition:**
`_getTotalEthInProtocol()` sums `getTotalAssetDeposits(asset) * assetPrice` across all supported assets. An EigenLayer slashing event that reduces NDC balances directly lowers `totalETHInProtocol` and thus `newRsETHPrice`. [10](#0-9) 

**Unpause requires admin:** [11](#0-10) 

## Impact Explanation
**Medium — Temporary freezing of funds.** Users who have completed the full withdrawal lifecycle (waited the delay, had their request unlocked by an operator, had their rsETH burned) cannot call `completeWithdrawal` to receive their LST/ETH. Their funds are locked in the contract until an admin unpauses. The freeze duration is entirely dependent on admin response time and is unbounded from the user's perspective.

## Likelihood Explanation
- `updateRSETHPrice()` is public; any EOA, bot, or contract can call it at any time without any privileged access.
- EigenLayer slashing is a documented, non-negligible risk for any restaking protocol and directly reduces `getTotalAssetDeposits`, lowering `newRsETHPrice`.
- A single call by any unprivileged actor at the moment a qualifying price drop exists on-chain is sufficient to trigger the pause.
- No front-running, victim mistakes, or external protocol compromise is required — the attacker simply calls a public function.
- The condition is repeatable: after an admin unpauses, any subsequent qualifying price drop can re-trigger the pause via the same public function.

## Recommendation
1. **Exempt `completeWithdrawal` from the pause**, or introduce a separate `withdrawalCompletionPaused` flag so that already-unlocked requests (where rsETH has already been burned) can always be finalized regardless of oracle-triggered pauses.
2. Alternatively, restrict `updateRSETHPrice()` to a role (e.g., `MANAGER`) so the auto-pause cannot be triggered by an arbitrary caller. The existing `updateRSETHPriceAsManager()` already provides a privileged path; making the public variant role-gated removes the permissionless trigger.
3. Consider separating the "pause new deposits/initiations" concern from the "pause completions" concern at the contract level.

## Proof of Concept
```solidity
// Fork test (Foundry) — unmodified production contracts
function test_publicUpdatePausesCompleteWithdrawal() external {
    // 1. User initiates withdrawal; operator unlocks it (rsETH burned in unlockQueue)
    vm.prank(user);
    withdrawalManager.initiateWithdrawal(asset, rsETHAmount, "");
    vm.roll(block.number + withdrawalManager.withdrawalDelayBlocks() + 1);
    vm.prank(operator);
    withdrawalManager.unlockQueue(asset, type(uint256).max, 0, type(uint256).max, 0, type(uint256).max);
    // rsETH is now burned; user's request is unlocked and ready to complete

    // 2. Simulate EigenLayer slashing: reduce NDC balance so getTotalAssetDeposits drops
    //    enough that newRsETHPrice < highestRsethPrice * (1 - pricePercentageLimit)
    _simulateSlashing(slashAmount);

    // 3. Any unprivileged caller triggers the price update
    vm.prank(address(0xdead));
    lrtOracle.updateRSETHPrice();

    // 4. Withdrawal manager is now paused
    assertTrue(withdrawalManager.paused());

    // 5. User cannot complete their already-unlocked withdrawal (rsETH already burned)
    vm.prank(user);
    vm.expectRevert("Pausable: paused");
    withdrawalManager.completeWithdrawal(asset, "");
}
```

    function _getTotalEthInProtocol() private view returns (uint256 totalETHInProtocol) {
        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);
        address[] memory supportedAssets = lrtConfig.getSupportedAssetList();
        uint256 supportedAssetCount = supportedAssets.length;

            unchecked {
                ++assetIdx;
            }
        }
