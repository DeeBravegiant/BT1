## Title
Fee Charged on Loss Recovery Due to Incorrect TVL Basis â€” (`contracts/LRTOracle.sol`)

## Summary
`LRTOracle._updateRsETHPrice()` computes `previousTVL` using the last stored `rsETHPrice`, which is written down after any loss event. When the protocol subsequently recovers, the entire recovery delta is treated as new profit and subjected to the protocol fee, even though no genuine gain has occurred. The contract already tracks `highestRsethPrice` as a one-way ratchet but never uses it as the fee basis.

## Finding Description
At line 234, `previousTVL` is computed as `rsethSupply.mulWad(rsETHPrice)`, where `rsETHPrice` is the last stored price â€” which is written down to a depressed value after a loss. At line 244â€“246, any `totalETHInProtocol > previousTVL` triggers fee collection:

```solidity
// L233-247

if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
```

After a loss cycle: `rsETHPrice` is written down (e.g., 1.00 â†’ 0.95 ETH). On recovery back to 1.00 ETH, `previousTVL` is 95 ETH (not 100 ETH), so `rewardAmount = 5 ETH` is treated as profit. The fee is minted as rsETH to the treasury (line 306), permanently diluting all existing holders. The variable `highestRsethPrice` (lines 252â€“296) is a correct all-time-high ratchet but is only used for pause/threshold checks â€” never for fee gating.

`updateRSETHPrice()` is a public, permissionless function (line 87), callable by any address when the protocol is not paused.

## Impact Explanation
**High â€” Theft of unclaimed yield.** rsETH holders lose a portion of their yield every time the protocol recovers from any downward price movement. The fee is minted as rsETH to the treasury (line 306), permanently diluting the share of every existing holder. The magnitude scales with `protocolFeeInBPS` and the depth/frequency of loss-recovery cycles; repeated oscillations compound the dilution over time. This matches the allowed impact: *High â€” Theft of unclaimed yield*.

## Likelihood Explanation
**Medium.** LST price ratios (stETH/ETH, rETH/ETH) fluctuate routinely, and EigenLayer slashing is an explicit protocol risk. Any price update following a prior write-down â€” even a minor one â€” triggers the incorrect fee. Because `updateRSETHPrice()` is public and permissionless (line 87), any external caller can crystallise the fee immediately after a recovery, requiring no special privileges or coordination.

## Recommendation
Replace the fee basis with `highestRsethPrice` so fees are only charged on genuine all-time-high gains:

```solidity
// Use peak price as basis, not last stored price
uint256 previousTVL = rsethSupply.mulWad(highestRsethPrice);
```

This ensures `rewardAmount` is only positive when `totalETHInProtocol` exceeds the prior peak TVL, not merely when it exceeds a depressed post-loss TVL.
