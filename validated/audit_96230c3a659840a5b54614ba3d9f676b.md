### Title
Gateway Admission Uses Stale Previous-Block L2 Gas Price Instead of Next-Block Price, Causing Incorrect Transaction Admission/Rejection — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidator::validate_resource_bounds()` gates mempool admission on the **previous committed block's** L2 gas price rather than the **next block's** L2 gas price. Because the EIP-1559 fee market adjusts the price every block, the reference value used for the threshold check is structurally stale. This causes the gateway to admit transactions that will fail at execution time (when the next block's price is higher) and to reject valid transactions (when the next block's price is lower). The code itself carries an open TODO acknowledging the wrong reference is used.

### Finding Description

In `validate_resource_bounds()`, the gateway reads the L2 gas price from `gateway_fixed_block_state_reader.get_block_info()`:

```rust

    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient` is constructed with `latest_block_number` at the time of the call and caches the result in a `OnceCell<BlockInfo>`: [2](#0-1) [3](#0-2) 

The factory always snapshots the latest committed block number at instantiation time: [4](#0-3) 

So `previous_block_l2_gas_price` is always the price of the **last finalized block**, not the price that will govern the **next block** being built.

The next block's L2 gas price is computed by the EIP-1559 mechanism in `calculate_next_base_gas_price()`: [5](#0-4) 

The maximum per-block price change is `price / gas_price_max_change_denominator`. The orchestrator computes and stores this as `next_l2_gas_price` in `FeeMarketInfo` and in `ProposalFin`: [6](#0-5) 

The gateway has no access to this value and instead uses the stale committed price.

The threshold check in `validate_tx_l2_gas_price_within_threshold` computes:

```
threshold = (min_gas_price_percentage / 100) * previous_block_l2_gas_price
```

With the default `min_gas_price_percentage = 100`, the threshold equals `previous_block_l2_gas_price` exactly. [7](#0-6) [8](#0-7) 

### Impact Explanation

**Incorrect admission (invalid tx accepted):** When the network is congested, the next block's price is higher than the previous block's price. A transaction with `max_price_per_unit = previous_block_price` passes the gateway threshold check but will fail at execution time because `max_price_per_unit < next_block_gas_price`. The blockifier's post-execution fee check (`check_can_pay_fee`) will find the actual fee `actual_gas × next_block_price` exceeds the account's balance (which was only verified to cover `max_amount × previous_block_price`). The transaction is included in the block as a revert and the user is charged a fee for a failed transaction — a direct economic loss.

**Incorrect rejection (valid tx rejected):** When congestion is low, the next block's price is lower than the previous block's price. A transaction with `max_price_per_unit` between `next_block_price` and `previous_block_price` is rejected by the gateway even though it would succeed in the next block. This is a liveness failure for users.

Both outcomes fall under: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block. Any block with gas usage above or below the target produces a price that differs from the previous block's price. Under the default `min_gas_price_percentage = 100`, the discrepancy is present on every non-stable block. No special attacker capability is required; any user submitting a transaction at the boundary of the threshold triggers the incorrect decision.

### Recommendation

Replace `previous_block_l2_gas_price` with the **next block's** L2 gas price, computed by applying `calculate_next_l2_gas_price_for_fin` to the latest committed block's price and gas usage. The TODO comment in the code already identifies this as the correct fix:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
```

The orchestrator already exposes `next_l2_gas_price` in `FeeMarketInfo` and in `ProposalFin::fin_payload`. The gateway should read this value (e.g., from the state sync layer or from a dedicated endpoint) and use it as the reference for the threshold check.

### Proof of Concept

1. Previous committed block's L2 gas price: `P = 30_000_000_000` FRI.
2. Previous block was at full capacity → EIP-1559 computes next block price: `P' = P + P/gas_price_max_change_denominator ≈ P * 1.021`.
3. User submits an `AllResources` invoke transaction with `l2_gas.max_price_per_unit = P` and `l2_gas.max_amount = A`.
4. **Gateway check** (`validate_resource_bounds`): `P >= 100% × P` → **passes**.
5. Transaction enters the mempool and is pulled into the next block being built.
6. **Blockifier pre-validation** (`verify_can_pay_committed_bounds`): account balance checked against `A × P` → passes (account holds exactly `A × P`).
7. **Blockifier execution**: block gas price is `P'`. Actual gas used = `G ≤ A`.
8. **Blockifier post-execution** (`check_can_pay_fee`): actual fee = `G × P' > G × P`. Account balance `A × P` may be insufficient to cover `G × P'`.
9. Transaction reverts with `InsufficientFeeTokenBalance`; user is charged a fee for a failed transaction.

The reverse scenario (price falling) causes the gateway to reject a transaction with `max_price_per_unit = P'` (which is valid for the next block) because `P' < P = threshold`.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}

pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {
        StatefulTransactionValidatorConfig {
            validate_resource_bounds: true,
            max_allowed_nonce_gap: 200,
            reject_future_declare_txs: true,
            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
            versioned_constants_overrides: None,
        }
    }
}
