### Title
Weight Proof Sub-Epoch Sampling Skips Boundary-Coincident Epochs — (`File: chia/full_node/weight_proof.py`)

### Summary
`_sample_sub_epoch` uses a double-strict inequality (`start_of_epoch_weight < weight < end_of_epoch_weight`) to decide whether a sub-epoch must be included in a weight proof. When a sampled weight equals either boundary exactly, the sub-epoch is silently excluded from the required set. Because both the prover and the verifier call the same function, a malicious peer can craft a weight proof whose summaries steer the shared RNG so that one or more sampled weights land exactly on sub-epoch weight boundaries, removing those sub-epochs from the validation requirement and reducing the cryptographic coverage of the proof.

### Finding Description

`_sample_sub_epoch` in `chia/full_node/weight_proof.py` decides whether a sub-epoch falls within a sampled weight range:

```python
if start_of_epoch_weight < weight < end_of_epoch_weight:
    choose = True
    break
``` [1](#0-0) 

Both boundary values are excluded. When `weight == start_of_epoch_weight` or `weight == end_of_epoch_weight`, `choose` stays `False` and the sub-epoch is not added to `sampled_sub_epochs`.

`validate_sub_epoch_sampling` then checks that every sub-epoch in `sampled_sub_epochs` is present in the proof:

```python
for idx in range(1, len(sub_epoch_weight_list)):
    if _sample_sub_epoch(sub_epoch_weight_list[idx - 1], sub_epoch_weight_list[idx], weight_to_check):
        sampled_sub_epochs[idx - 1] = True
``` [2](#0-1) 

A sub-epoch that is never added to `sampled_sub_epochs` is never required to be present. The prover can omit its challenge segments entirely and validation still passes.

The RNG seed is `summaries[-2].get_hash()`, derived from the summaries embedded in the weight proof itself:

```python
seed = summaries[-2].get_hash()
rng = random.Random(seed)
``` [3](#0-2) 

The sampled weights are computed as `uint128(q * float(total_weight))` — a float truncated to integer. Because sub-epoch weights are also integers, a crafted proof can arrange for a truncated float to equal a sub-epoch boundary exactly.

### Impact Explanation

A syncing node accepts a weight proof from any peer without prior authentication. If a malicious peer crafts a weight proof whose summaries cause the RNG to emit weights that coincide with sub-epoch boundaries, those sub-epochs are silently dropped from the required validation set. The node accepts the proof as valid while having verified fewer sub-epochs than the protocol intends, weakening the probabilistic guarantee that the claimed chain is the heaviest honest chain. This is a forged weight proof trust path: an unprivileged peer can cause a syncing node to accept a chain with reduced cryptographic coverage.

### Likelihood Explanation

The attacker controls the summaries embedded in the weight proof (they must chain correctly via `std_hash`, but the attacker constructs the entire proof). The sampled weights are `uint128(q * float(total_weight))`. Because `float` arithmetic truncates, and sub-epoch weights are integers, the attacker can search for summary configurations where the truncated float equals a boundary. The search space is bounded by the number of sub-epochs and the precision of float arithmetic, making this computationally feasible for a motivated attacker. The attack requires no keys, no admin access, and no on-chain funds.

### Recommendation

Change the inner comparison in `_sample_sub_epoch` to include both boundary values:

```python
# Before
if start_of_epoch_weight < weight < end_of_epoch_weight:

# After
if start_of_epoch_weight <= weight <= end_of_epoch_weight:
``` [4](#0-3) 

This ensures that a sampled weight that falls exactly on a sub-epoch boundary still causes that sub-epoch to be required in the proof, preserving the intended probabilistic security coverage.

### Proof of Concept

1. Attacker constructs a weight proof with crafted `sub_epoch_data` entries whose cumulative weights are chosen so that `sub_epoch_weight_list[k]` (a boundary value) equals `uint128(q * float(total_weight))` for some RNG output `q`.
2. The attacker omits the challenge segments for sub-epoch `k-1` from `sub_epoch_segments`.
3. The syncing node calls `validate_sub_epoch_sampling`. `_sample_sub_epoch(sub_epoch_weight_list[k-1], sub_epoch_weight_list[k], weight_to_check)` returns `False` because `weight == end_of_epoch_weight` is excluded by the strict inequality.
4. Sub-epoch `k-1` is never added to `sampled_sub_epochs`, so its absence from the proof is never checked.
5. `validate_sub_epoch_sampling` returns `True`; the proof passes with one fewer verified sub-epoch than intended. [5](#0-4) [6](#0-5)

def _sample_sub_epoch(
    start_of_epoch_weight: uint128,
    end_of_epoch_weight: uint128,
    weight_to_check: list[uint128] | None,
) -> bool:
    """
    weight_to_check: list[uint128] is expected to be sorted
    """
    if weight_to_check is None:
        return True
    if weight_to_check[-1] < start_of_epoch_weight:
        return False
    if weight_to_check[0] > end_of_epoch_weight:
        return False
    choose = False
    for weight in weight_to_check:
        if weight > end_of_epoch_weight:
            return False
        if start_of_epoch_weight < weight < end_of_epoch_weight:
            log.debug(f"start weight: {start_of_epoch_weight}")
            log.debug(f"weight to check {weight}")
            log.debug(f"end weight: {end_of_epoch_weight}")
            choose = True
            break

    return choose
```

def validate_sub_epoch_sampling(
    rng: random.Random, sub_epoch_weight_list: list[uint128], weight_proof: WeightProof
) -> bool:
    tip = weight_proof.recent_chain_data[-1]
    weight_to_check = _get_weights_for_sampling(rng, tip.weight, weight_proof.recent_chain_data)
    sampled_sub_epochs: dict[int, bool] = {}
    for idx in range(1, len(sub_epoch_weight_list)):
        if _sample_sub_epoch(sub_epoch_weight_list[idx - 1], sub_epoch_weight_list[idx], weight_to_check):
            sampled_sub_epochs[idx - 1] = True
            if len(sampled_sub_epochs) == WeightProofHandler.MAX_SAMPLES:
                break
    curr_sub_epoch_n = -1
    for sub_epoch_segment in weight_proof.sub_epoch_segments:
        if curr_sub_epoch_n < sub_epoch_segment.sub_epoch_n:
            if sub_epoch_segment.sub_epoch_n in sampled_sub_epochs:
                del sampled_sub_epochs[sub_epoch_segment.sub_epoch_n]
        curr_sub_epoch_n = sub_epoch_segment.sub_epoch_n
    if len(sampled_sub_epochs) > 0:
        return False
    return True
```
