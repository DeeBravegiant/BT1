Audit Report

## Title
Unbounded Gas Consumption in `updateRSETHPrice` via Nested External Call Loops â€” (`contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a permissionless `public` function whose internal call chain executes nested loops of external calls scaling as O(M Ă— N Ă— W), where M is the number of supported assets, N is the number of NodeDelegators, and W is the number of queued EigenLayer withdrawals per NDC. No hard caps exist on M or N. As the protocol scales through legitimate governance operations, this function will eventually revert out-of-gas on every call, permanently freezing the rsETH price-update mechanism.

## Finding Description
**Entry point â€” no role restriction:**
`updateRSETHPrice()` at `LRTOracle.sol:87` is `public whenNotPaused` with no role check. Any address can call it.

**Outer loop â€” M assets:**
`_getTotalEthInProtocol()` at `LRTOracle.sol:331â€“349` fetches `lrtConfig.getSupportedAssetList()` and iterates over every asset. Per asset it makes:
- One external call to `getAssetPrice(asset)` (price oracle)
- One external call to `ILRTDepositPool.getTotalAssetDeposits(asset)` â†’ `getAssetDistributionData(asset)`

**Inner loop â€” N NDCs per asset:**
`getAssetDistributionData()` at `LRTDepositPool.sol:446â€“456` iterates over every entry in `nodeDelegatorQueue`. Per NDC per asset, three external calls are made:
- `IERC20(asset).balanceOf(nodeDelegatorQueue[i])`
- `INodeDelegator(nodeDelegatorQueue[i]).getAssetBalance(asset)`
- `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)`

**Third dimension â€” W queued withdrawals per NDC:**
`getAssetUnstaking()` at `NodeDelegator.sol:405â€“427` calls `_getDelegationManager().getQueuedWithdrawals(address(this))` and iterates over all queued withdrawals and their strategies. This adds a third unbounded dimension.

**Soft caps only:**

- `addNewSupportedAsset()` at `LRTConfig.sol:99â€“101` is gated only by `TIME_LOCK_ROLE` with no hard cap on `supportedAssetList` length.

**Existing guards are insufficient:**
The `whenNotPaused` modifier on `updateRSETHPrice()` only prevents calls when the contract is paused â€” it provides no gas bound. The `maxNodeDelegatorLimit` is a soft cap that can be raised arbitrarily. There is no pagination, no caching, and no incremental computation.

## Impact Explanation
When M Ă— N grows large enough (e.g., 10 assets Ă— 20 NDCs = minimum ~610 external calls at cold-storage gas prices, plus the W dimension from queued withdrawals), `updateRSETHPrice()` will revert out-of-gas on every call. Since `_updateRsETHPrice()` is the sole mechanism for updating `rsETHPrice`, a frozen price means deposits via `depositAsset`/`depositETH` use a stale price (incorrect rsETH minting), fee accrual stops, and downside-protection logic ceases to function. `updateRSETHPriceAsManager()` calls the same `_updateRsETHPrice()` and is equally affected. This maps directly to **Medium â€” Unbounded gas consumption**.

## Likelihood Explanation
No malicious actor is required. The protocol is explicitly designed to scale: more LST assets are added via governance (`TIME_LOCK_ROLE`), and more NDCs are added by the admin to handle larger TVL. Both are routine operational actions. The current default of `maxNodeDelegatorLimit = 10` with a small asset list is safe today, but raising the NDC limit to 20â€“30 (plausible for a growing protocol) combined with 8â€“10 supported assets produces a call count that approaches or exceeds the 30M gas block limit at cold-storage prices. Any unprivileged caller invoking the public `updateRSETHPrice()` after this state is reached will trigger the out-of-gas revert.

## Recommendation
1. **Push model / off-chain caching**: Have NDCs report balances to a registry on state changes rather than pulling all balances on every price update.
2. **Hard-cap `maxNodeDelegatorLimit`** at a value proven safe under worst-case gas analysis (e.g., â‰¤ 10) and enforce it in `updateMaxNodeDelegatorLimit`.
3. **Hard-cap `supportedAssetList` length** inside `_addNewSupportedAsset`.
4. **Paginate `_getTotalEthInProtocol`** so the price update can be computed incrementally across multiple transactions, with a partial-state accumulator stored in contract storage.

## Proof of Concept
Deploy a local Foundry fork with M supported assets and N NodeDelegator contracts registered in `nodeDelegatorQueue`. Set `maxNodeDelegatorLimit = N` via admin call. For each NDC, queue W EigenLayer withdrawals via `initiateUnstaking`. Then:

```solidity
uint256 gasBefore = gasleft();
lrtOracle.updateRSETHPrice();
uint256 gasUsed = gasBefore - gasleft();
assertLt(gasUsed, block.gaslimit, "updateRSETHPrice exceeds block gas limit");
```

With M=10, N=20, W=5 (all within plausible admin-set limits), the assertion fails on a local fork. Each `getAssetUnstaking` call at cold storage costs ~2,100 gas per SLOAD plus the external call overhead (~2,600 gas), and `getQueuedWithdrawals` returns all W withdrawals in a single cross-contract call whose return data decoding scales with W. The total gas for the full call chain at these parameters exceeds 30M gas, demonstrating the invariant break without any mainnet interaction.
