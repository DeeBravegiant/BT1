Audit Report

## Title
Unrestricted `updateRSETHPrice()` Allows Any Caller to Trigger Protocol Fee Minting and Dilute rsETH Holders — (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no role restriction, while its sibling `updateRSETHPriceAsManager()` is correctly gated by `onlyLRTManager`. Any unprivileged address can invoke `_updateRsETHPrice()` at will, triggering rsETH fee minting to the treasury and diluting all existing rsETH holders' unclaimed yield. The daily fee cap (`maxFeeMintAmountPerDay`) limits per-period damage but does not prevent systematic daily extraction by any external actor.

## Finding Description

`LRTOracle.sol` exposes two entry points into `_updateRsETHPrice()`:

```solidity
// Line 87 — no role check, callable by anyone
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}

// Line 94 — correctly restricted
function updateRSETHPriceAsManager() external onlyLRTManager {
    _updateRsETHPrice();
}
``` [1](#0-0) 

Inside `_updateRsETHPrice()`, whenever `totalETHInProtocol > previousTVL` and the protocol is not paused, a fee is computed and minted to the treasury:

```solidity
// Lines 244–246
if (!protocolPaused && totalETHInProtocol > previousTVL) {
    uint256 rewardAmount = totalETHInProtocol - previousTVL;
    protocolFeeInETH = (rewardAmount * lrtConfig.protocolFeeInBPS()) / 10_000;
}
``` [2](#0-1) 

```solidity
// Lines 301–307
uint256 rsethAmountToMintAsProtocolFee = protocolFeeInETH.divWad(newRsETHPrice);
_checkAndUpdateDailyFeeMintLimit(rsethAmountToMintAsProtocolFee);
if (rsethAmountToMintAsProtocolFee > 0) {
    address treasury = lrtConfig.getContract(LRTConstants.PROTOCOL_TREASURY);
    IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee);
``` [3](#0-2) 

The `_checkAndUpdateDailyFeeMintLimit` function resets the period counter each day and enforces `maxFeeMintAmountPerDay`, but does not restrict who may call the outer function. Any external actor can call `updateRSETHPrice()` at the start of each new daily period to extract the full `maxFeeMintAmountPerDay` in rsETH fees to the treasury, every day, without any authorization. [4](#0-3) 

Additionally, if on-chain conditions produce a price drop exceeding `pricePercentageLimit`, any caller can trigger a full protocol pause:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [5](#0-4) 

Note that for price *increases* above the threshold, a non-manager caller is explicitly reverted (`PriceAboveDailyThreshold`), confirming the protocol applies caller-based access control selectively — but omits it on the public entry point entirely. [6](#0-5) 

## Impact Explanation

**Primary — High: Theft of unclaimed yield.**
Staking and EigenLayer rewards accrue as an increase in `totalETHInProtocol`. Until `updateRSETHPrice()` is called, this yield belongs to rsETH holders as price appreciation. Any caller can invoke `updateRSETHPrice()` to force fee extraction: up to `maxFeeMintAmountPerDay` rsETH is minted to the treasury each day, permanently diluting holders. An attacker can call this at the start of every daily period to ensure the maximum daily fee is extracted every day, systematically maximizing the rate of yield extraction from rsETH holders without any authorization. The minted rsETH reduces each holder's proportional claim on the underlying ETH.

**Secondary — Medium: Temporary freezing of funds.**
If on-chain conditions (e.g., a slashing event or oracle dip) produce a price drop exceeding `pricePercentageLimit`, any caller can trigger a full pause of `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` by calling `updateRSETHPrice()`. This freezes all deposits and withdrawals until an admin calls `unpause()`.

## Likelihood Explanation

The entry point is `public` with no authentication. Any EOA or contract can call it at zero cost beyond gas. The fee-minting path is triggered whenever rewards have accrued since the last call, which is a routine on-chain condition after any validator reward sweep or EigenLayer strategy yield increase. Likelihood is **High** for the yield theft path and **Medium** for the pause path (requires a qualifying price drop condition to exist on-chain).

## Recommendation

Apply the same `onlyLRTManager` (or a dedicated keeper role) access control used on `updateRSETHPriceAsManager()` to `updateRSETHPrice()`. If permissionless price updates are desired for liveness, separate the fee-minting logic from the price-read path: restrict fee extraction to an authorized caller while allowing the price snapshot to remain publicly readable via a view function.

## Proof of Concept

1. Rewards accrue in the protocol (e.g., EigenLayer strategy yields increase `totalETHInProtocol` above `rsethSupply * rsETHPrice`).
2. Attacker (any EOA) calls `LRTOracle.updateRSETHPrice()` at the start of a new daily period.
3. `_updateRsETHPrice()` computes `rewardAmount = totalETHInProtocol - previousTVL > 0`.
4. `protocolFeeInETH = rewardAmount * protocolFeeInBPS / 10_000` is computed.
5. `_checkAndUpdateDailyFeeMintLimit` passes (new period, counter reset to 0).
6. `rsethAmountToMintAsProtocolFee = protocolFeeInETH / newRsETHPrice` rsETH is minted to the treasury.
7. All existing rsETH holders are diluted; their share of the underlying ETH decreases.
8. Attacker repeats step 2 at the start of each new daily period to extract the maximum `maxFeeMintAmountPerDay` fee every day without any authorization.

**Foundry test plan:**
- Fork mainnet or deploy a local fixture with `LRTOracle`, `LRTDepositPool`, `RSETH`, and `LRTConfig`.
- Simulate reward accrual by increasing the mock strategy balance so `_getTotalEthInProtocol()` returns a value above `rsethSupply * rsETHPrice`.
- Call `updateRSETHPrice()` from an unprivileged address (`vm.prank(attacker)`).
- Assert that `RSETH.balanceOf(treasury)` increased and that existing holders' proportional ETH claim decreased.
- Advance time by 1 day (`vm.warp(block.timestamp + 1 days)`), simulate further reward accrual, and repeat to demonstrate systematic daily extraction up to `maxFeeMintAmountPerDay`.
