Audit Report

## Title
Stale `rsETHPrice` Silently Propagated as Fresh by `RSETHPriceFeed` When Protocol Is Paused ‚Äî (File: `contracts/oracles/RSETHPriceFeed.sol`)

## Summary
`RSETHPriceFeed.latestRoundData()` computes rsETH/USD by multiplying a live Chainlink ETH/USD answer by `LRTOracle.rsETHPrice()`, but returns only the ETH/USD `updatedAt` timestamp ‚Äî never the rsETH price's own staleness. When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it calls `_pause()` and returns early without writing the new lower price to `rsETHPrice`. Because `updateRSETHPrice()` is gated by `whenNotPaused`, the stored value is frozen at the pre-drop (inflated) level. Any external lending protocol consuming `RSETHPriceFeed` receives an inflated rsETH/USD price paired with a fresh `updatedAt`, enabling over-collateralized borrowing and direct theft of lender funds.

## Finding Description

**Root cause 1 ‚Äî `RSETHPriceFeed` exposes only the ETH/USD staleness timestamp:**

`contracts/oracles/RSETHPriceFeed.sol` lines 68‚Äì69:
```solidity
(roundId, answer, startedAt, updatedAt, answeredInRound) = ETH_TO_USD.latestRoundData();
answer = int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18;
```
`updatedAt` is taken verbatim from the ETH/USD Chainlink feed (refreshed every few minutes). The rsETH/ETH component ‚Äî `RS_ETH_ORACLE.rsETHPrice()` ‚Äî is a stored state variable in `LRTOracle` whose own update timestamp is never surfaced or checked.

**Root cause 2 ‚Äî `updateRSETHPrice()` is blocked when the oracle is paused:**

`contracts/LRTOracle.sol` lines 87‚Äì89:
```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```
The `whenNotPaused` modifier (line 47‚Äì50) reverts with `ContractPaused` when `paused == true`.

**Root cause 3 ‚Äî Downside protection pauses the oracle and returns without updating `rsETHPrice`:**

`contracts/LRTOracle.sol` lines 277‚Äì282:
```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;   // rsETHPrice is NOT updated ‚Äî stays at pre-drop value
}
```
The actual write `rsETHPrice = newRsETHPrice` (line 313) is never reached. The stored value is permanently frozen at the inflated pre-drop level until an admin calls `unpause()`.

**Note on partial mitigation:** `updateRSETHPriceAsManager()` (lines 94‚Äì96) allows a manager to call `_updateRsETHPrice()` while paused, bypassing `whenNotPaused`. However, this is a privileged function and does not close the exploitation window ‚Äî it only allows an admin to remediate after the fact.

**Exploit sequence:**
1. A significant price drop (e.g., EigenLayer slashing) causes `newRsETHPrice` to fall more than `pricePercentageLimit` below `highestRsethPrice`.
2. `_updateRsETHPrice()` calls `_pause()` and returns. `rsETHPrice` remains at, e.g., `1.05e18` (pre-drop). True price is `0.95e18`.
3. `updateRSETHPrice()` reverts for all unprivileged callers.
4. Attacker acquires rsETH at the true market price (e.g., via DEX at `0.95 ETH`).
5. Attacker calls `RSETHPriceFeed.latestRoundData()` ‚Äî receives `updatedAt` from ETH/USD (e.g., 2 minutes ago, passes any staleness check) and `answer = 1.05e18 * ETH_USD / 1e18` (inflated by ~10%).
6. Attacker deposits rsETH into an external lending protocol (e.g., Aave/Morpho) using `RSETHPriceFeed` as the collateral oracle.
7. Lending protocol values 1000 rsETH at 1050 ETH (true value: 950 ETH) and allows borrowing up to ~1050 ETH.
8. Attacker borrows maximum, defaults. Lending protocol holds 950 ETH of collateral against 1050 ETH of debt ‚Äî a 100 ETH loss per 1000 rsETH deposited.

## Impact Explanation
**Critical ‚Äî Direct theft of user funds.** Lenders in any external protocol integrated with `RSETHPriceFeed` can have their deposited assets stolen. The attacker's profit equals the spread between the inflated oracle price and the true market price, multiplied by the rsETH collateral deposited. The loss is borne by the lending protocol's liquidity providers. This matches the allowed impact: *"Direct theft of any user funds, whether at-rest or in-motion."*

## Likelihood Explanation
**Medium-High.** The trigger condition ‚Äî a price drop exceeding `pricePercentageLimit` ‚Äî is an explicitly designed circuit-breaker expected to fire during adverse market events (EigenLayer slashing, LST depeg). The `pricePercentageLimit` is configurable; a conservative setting (e.g., 1‚Äì2%) can be triggered by routine volatility. The exploitation window lasts from the moment of the pause until an admin manually calls `unpause()` or `updateRSETHPriceAsManager()`, which may take hours. `RSETHPriceFeed` is purpose-built as a Chainlink-compatible feed for external integrations, making it highly likely to be consumed by lending protocols. The attacker requires no special privileges ‚Äî only the ability to hold rsETH and interact with an external lending protocol.

## Recommendation
1. **Surface the rsETH price update timestamp.** Store a `rsETHPriceUpdatedAt` timestamp in `LRTOracle` whenever `rsETHPrice` is written (line 313). In `RSETHPriceFeed.latestRoundData()`, return `min(ethUpdatedAt, rsETHPriceUpdatedAt)` as `updatedAt`.
2. **Write the new (lower) price before pausing.** In `_updateRsETHPrice()`, assign `rsETHPrice = newRsETHPrice` before calling `_pause()` and `return`, so the stored value reflects reality even while the protocol is paused.
3. **Add a staleness guard in `RSETHPriceFeed`.** Revert if `rsETHPriceUpdatedAt` is older than a configurable threshold (e.g., 24 hours).
