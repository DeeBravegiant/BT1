### Title
Native Fee ETH Permanently Locked in `OmniBridge` — No Withdrawal Mechanism - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

---

### Summary

`OmniBridge.sol` accepts ETH as `nativeFee` on every `initTransfer()` call and also exposes an open `receive() external payable` fallback, but the contract contains no function to withdraw or distribute the accumulated ETH. Every `nativeFee` payment made by any user is permanently locked in the contract.

---

### Finding Description

In `OmniBridge.initTransfer()`, the caller sends `msg.value` covering three components: the bridged `amount` (for native-ETH transfers), the `nativeFee`, and the Wormhole message fee (`extensionValue`).

For ERC-20 transfers (`tokenAddress != address(0)`):

```
extensionValue = msg.value - nativeFee;   // forwarded to Wormhole
// nativeFee remains in the contract
```

For native-ETH transfers (`tokenAddress == address(0)`):

```
extensionValue = msg.value - amount - nativeFee;  // forwarded to Wormhole
// amount is held for finTransfer release
// nativeFee remains in the contract
``` [1](#0-0) 

Only `extensionValue` is forwarded to Wormhole via `initTransferExtension`. The `nativeFee` portion is never forwarded, never emitted to a fee recipient, and never returned. [2](#0-1) 

Additionally, the contract declares an unconditional ETH receiver:

```solidity
receive() external payable {}
``` [3](#0-2) 

A complete audit of all public and admin functions in `OmniBridge.sol` reveals **no `withdrawETH`, `withdrawFees`, or any other function** that can move accumulated ETH out of the contract. The full function list — `addCustomToken`, `removeCustomToken`, `deployToken`, `setMetadata`, `logMetadata`, `finTransfer`, `initTransfer`, `initTransfer1155`, `pause`, `pauseAll`, `upgradeToken`, `setNearBridgeDerivedAddress` — none of them drain the `nativeFee` balance. [4](#0-3) 

`finTransfer` does release ETH to recipients when `tokenAddress == address(0)`, but only the `payload.amount` — it has no path to recover `nativeFee`. [5](#0-4) 

---

### Impact Explanation

**Critical — Irreversible fund lock of user-paid `nativeFee` ETH.**

Every `initTransfer()` call with `nativeFee > 0` permanently deposits ETH into the contract with no recovery path. Over time, the cumulative `nativeFee` balance grows monotonically and is unclaimable by users, relayers, or the protocol. This matches the allowed impact: *"Irreversible fund lock… permanently unclaimable user or protocol value in bridge… fee… flows."*

---

### Likelihood Explanation

**High.** `nativeFee` is a standard parameter of the normal `initTransfer()` user flow. Any user bridging tokens and paying a non-zero `nativeFee` (e.g., to cover NEAR-side gas) triggers the lock. No special attacker role or privileged access is required — the entry point is fully public and exercised in routine protocol operation.

---

### Recommendation

1. **Add an admin-controlled withdrawal function**, e.g.:
   ```solidity
   function withdrawETH(address payable recipient, uint256 amount)
       external onlyRole(DEFAULT_ADMIN_ROLE) {
       (bool ok,) = recipient.call{value: amount}("");
       require(ok);
   }
   ```
2. **Alternatively**, route `nativeFee` directly to a designated fee-recipient address inside `initTransfer()` rather than retaining it in the contract, mirroring the NEAR-side `claim_fee` design.
3. **Document** the intended lifecycle of `nativeFee` ETH on the EVM side so the accounting is auditable.

---

### Proof of Concept

1. User calls `initTransfer(usdcAddress, 1000e6, 0, 1e17, "near:alice.near", "")` with `msg.value = 1e17 + wormholeFee`.
2. Inside `initTransfer`: `extensionValue = msg.value - 1e17 = wormholeFee`; `1e17` wei (`nativeFee`) remains in the contract.
3. `initTransferExtension` forwards only `extensionValue` to Wormhole.
4. No function in `OmniBridge.sol` can move the `1e17` wei out of the contract.
5. After N users each pay `nativeFee`, `N × nativeFee` wei is permanently locked.
6. The open `receive()` fallback additionally allows anyone to send ETH directly, compounding the locked balance.
