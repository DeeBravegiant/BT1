### Title
Unauthenticated P2P Peer Can Poison Stored Gas Prices via Unvalidated `SignedBlockHeader` — (`crates/apollo_p2p_sync/src/client/header.rs`)

---

### Summary

`parse_data_for_block` accepts a `SignedBlockHeader` from any unauthenticated p2p peer and passes it directly to `write_to_storage`, which persists all gas price fields (`l1_gas_price`, `l1_data_gas_price`, `l2_gas_price` — both `price_in_fri` and `price_in_wei`) without any validation. No cryptographic signature verification is performed anywhere in the p2p sync header path. Even if it were, the `BlockSignature` only covers `poseidon_hash(block_hash, state_diff_commitment)` and does not commit to gas prices. A malicious peer can therefore store arbitrary gas prices for any synced block.

---

### Finding Description

**Entrypoint:** Any unauthenticated peer on the `/starknet/headers/0.1.0-rc.0` p2p protocol.

**Path:**

1. `parse_data_for_block` receives a `SignedBlockHeader` from the network. [1](#0-0) 

The only checks performed are:
- Block number ordering (lines 104–113)
- Signature vector length == 1 (lines 115–119)

No cryptographic verification of the signature is performed. `verify_block_signature` is defined in `starknet_api` but is never called in the p2p sync path. [2](#0-1) 

2. The accepted `SignedBlockHeader` is written directly to storage via `append_header`, persisting all gas price fields verbatim. [3](#0-2) 

3. The protobuf converter populates `l1_gas_price`, `l1_data_gas_price`, and `l2_gas_price` (both `price_in_fri` and `price_in_wei`) directly from the wire message with no bounds or consistency checks. [4](#0-3) 

4. Even if signature verification were added, the `BlockSignature` signs only `poseidon_hash(block_hash, state_diff_commitment)` — gas prices are not covered. [5](#0-4) 

5. Gas prices **are** part of the block hash via `gas_prices_to_hash`, which hashes all six price fields (wei and fri for l1, l1_data, l2) for version ≥ 0.13.4. [6](#0-5) 

6. The batcher's `get_block_info` reads gas prices directly from the stored header to build `BlockInfo`, which feeds into `PartialBlockHashComponents::new`. [7](#0-6) [8](#0-7) 

---

### Impact Explanation

**Direct (High — RPC):** Any node syncing via p2p with a malicious peer will store wrong gas prices for historical blocks. All RPC responses for those blocks (`starknet_getBlockWithTxHashes`, `starknet_estimateFee` for historical context, `starknet_traceTransaction`) will return attacker-controlled gas prices, constituting authoritative-looking wrong values.

**Indirect (Critical — Consensus/Commitment):** The consensus orchestrator reads the previous block's gas prices from storage via `previous_proposal_init_from_block_header` as a fallback when the L1 gas price provider is unavailable. [9](#0-8) [10](#0-9) 

If a sequencer node syncs historical blocks via p2p and the provider fails, it will propose a block with attacker-controlled gas prices. Since gas prices are committed into the block hash via `calculate_block_hash`, the resulting `PartialBlockHash` will diverge from honest sequencers, breaking consensus.

---

### Likelihood Explanation

- The p2p sync protocol is unauthenticated; any peer can connect and respond to header queries.
- The attack requires no special privileges — only the ability to respond to a `BlockHeadersRequest` on the p2p network.
- The RPC impact path is unconditional (no provider failure required).
- The consensus divergence path requires the L1 gas price provider to fail, which is a realistic operational condition (network outage, rate limiting, etc.).

---

### Recommendation

1. **Cryptographically verify the `BlockSignature`** against the known sequencer public key before accepting any `SignedBlockHeader`. Reject headers with invalid signatures.
2. **Recompute the block hash** from the received header fields and compare it against the `block_hash` field in the message. Since gas prices are committed into the block hash, any tampered gas price will produce a mismatch.
3. As a defense-in-depth measure, cross-check gas prices against a trusted source (e.g., L1 data or a quorum of peers) before storing.

---

### Proof of Concept

```
1. Attacker connects to a syncing node's p2p endpoint on /starknet/headers/0.1.0-rc.0.
2. Node sends a BlockHeadersRequest for block N.
3. Attacker responds with a SignedBlockHeader for block N where:
     l1_gas_price_fri = 1   (canonical value: 1000)
     l1_data_gas_price_fri = 1
     l2_gas_price_fri = 1
   The signatures field contains exactly one (arbitrary) ConsensusSignature
   (length check passes; no cryptographic verification is performed).
4. parse_data_for_block accepts the header (block number matches, signatures.len() == 1).
5. write_to_storage calls append_header, persisting price_in_fri = 1 for all three fields.
6. Assert: storage_reader.get_block_header(N).l1_gas_price.price_in_fri == GasPrice(1).
7. Assert: calculate_block_hash(PartialBlockHashComponents { l1_gas_price: {fri:1,...}, ... })
           != calculate_block_hash(PartialBlockHashComponents { l1_gas_price: {fri:1000,...}, ... })
   (gas_prices_to_hash produces a different Poseidon hash, so the final block hash differs).
```

        block_number: BlockNumber,
        _storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            // TODO(noamsp): investigate and remove this timeout.
            let maybe_signed_header =
                timeout(Duration::from_secs(15), signed_headers_response_manager.next())
                    .await
                    .ok()
                    .flatten()
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
            let Some(signed_block_header) = maybe_signed_header?.0 else {
                return Ok(None);
            };
            // TODO(shahak): Check that parent_hash is the same as the previous block's hash
            // and handle reverts.
            if block_number
                != signed_block_header.block_header.block_header_without_hash.block_number
            {
                return Err(ParseDataError::BadPeer(BadPeerError::HeadersUnordered {
                    expected_block_number: block_number,
                    actual_block_number: signed_block_header
                        .block_header
                        .block_header_without_hash
                        .block_number,
                }));
            }
            if signed_block_header.signatures.len() != ALLOWED_SIGNATURES_LENGTH {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongSignaturesLength {
                    signatures: signed_block_header.signatures,
                }));
            }
            Ok(Some(signed_block_header))
        }
        .boxed()
    }
