Audit Report

## Title
Beacon Chain ETH Unstaking Shares Returned as Wei Without Gwei-to-Wei Conversion, Deflating rsETH Price — (`contracts/NodeDelegator.sol`)

## Summary
In `NodeDelegator.getAssetUnstaking()`, queued withdrawal shares for the beacon chain ETH strategy are returned as-is without conversion, while LST strategies correctly call `strategy.sharesToUnderlyingView()`. Because EigenLayer's beacon chain ETH strategy denominates shares in Gwei (enforced by `SharesNotMultipleOfGwei` and `recordBeaconChainETHBalanceUpdate`), the returned value is ~1e9× smaller than the actual Wei amount. This understates total protocol ETH, deflates `rsETHPrice`, and causes new depositors to receive excess rsETH at the expense of existing holders' yield.

## Finding Description
The root cause is at `NodeDelegator.sol` lines 421–424:

```solidity
uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
amount += strategyAsset == LRTConstants.ETH_TOKEN
    ? sharesToUnstake                                    // Gwei shares returned as Wei — no conversion
    : strategy.sharesToUnderlyingView(sharesToUnstake);  // correct for LSTs
``` [1](#0-0) 

EigenLayer's `IEigenPodManager` enforces that beacon chain ETH shares are multiples of Gwei (`SharesNotMultipleOfGwei` error; `recordBeaconChainETHBalanceUpdate` reverts if not a whole Gwei amount), meaning 32 ETH = 32e9 shares, not 32e18. [2](#0-1) [3](#0-2) 

The corrupted value propagates upward:

1. `getETHDistributionData()` accumulates `ethUnstakingFromEigenLayer` from `getAssetUnstaking(ETH_TOKEN)`: [4](#0-3) 

2. `getTotalAssetDeposits()` sums all ETH components including `assetUnstakingFromEigenLayer`: [5](#0-4) 

3. `_getTotalEthInProtocol()` uses `getTotalAssetDeposits` per asset: [6](#0-5) 

4. `rsETHPrice` is computed by dividing total ETH by rsETH supply: [7](#0-6) 

5. `getRsETHAmountToMint` divides by `rsETHPrice`, so a deflated price mints excess rsETH: [8](#0-7) 

No existing guard prevents this: the branch is intentional code, not an oversight in a check.

## Impact Explanation
**High — Theft of unclaimed yield.** When ETH is queued for unstaking (between `initiateUnstaking` and `completeUnstaking`), `getTotalAssetDeposits(ETH_TOKEN)` is understated by approximately `actual_ETH_unstaking × (1 − 1/1e9) ≈ actual_ETH_unstaking`. This deflates `rsETHPrice`. Every subsequent depositor mints more rsETH than their deposit warrants, diluting the proportional claim of existing rsETH holders on the protocol's ETH — a direct, quantifiable theft of accrued yield from existing holders.

## Likelihood Explanation
Triggered whenever the operator calls `initiateUnstaking` for the beacon chain ETH strategy, which is a routine, expected operational action (e.g., validator exits). No attacker capability is required beyond being a normal depositor after the operator's routine action. Any unprivileged depositor calling `depositETH` during the unstaking window benefits at the expense of existing holders. The condition is repeatable and persistent until `completeUnstaking` is called.

## Recommendation
Remove the ETH-specific branch and apply `sharesToUnderlyingView` uniformly:

```diff
- amount += strategyAsset == LRTConstants.ETH_TOKEN
-     ? sharesToUnstake
-     : strategy.sharesToUnderlyingView(sharesToUnstake);
+ amount += strategy.sharesToUnderlyingView(sharesToUnstake);
```

This mirrors the correct handling already applied to LST strategies and converts Gwei shares to Wei for the beacon chain ETH strategy.

## Proof of Concept
1. Operator calls `initiateUnstaking` queuing 32e9 beacon chain ETH shares (representing 32 ETH = 32e18 Wei).
2. Call `NodeDelegator.getAssetUnstaking(ETH_TOKEN)` → returns `32e9` (Gwei) instead of `32e18` (Wei).
3. `LRTDepositPool.getTotalAssetDeposits(ETH_TOKEN)` is understated by `32e18 − 32e9 ≈ 32e18`.
4. `LRTOracle._getTotalEthInProtocol()` is understated by the same amount.
5. `_updateRsETHPrice()` computes `rsETHPrice = (totalETH_understated) / rsethSupply` — deflated price.
6. A depositor calling `depositETH(32 ETH)` receives `rsethAmountToMint = (32e18 × assetPrice) / rsETHPrice_deflated` — excess rsETH minted, diluting all existing holders.

Foundry fork test: deploy against a mainnet fork, have the operator call `initiateUnstaking` with beacon chain ETH shares, then assert `getAssetUnstaking(ETH_TOKEN)` returns a value in Wei (≥ 1e18 per ETH) rather than Gwei (≤ 1e9 per ETH). The assertion will fail, confirming the bug.
