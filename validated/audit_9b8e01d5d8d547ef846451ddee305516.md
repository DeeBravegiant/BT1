The bounds check at line 57 of `access_structure_adapter.h` contains a signed-integer underflow that makes it fail to catch the exact scenario described.

### Title
Signed-integer underflow in `child_indices` bounds guard allows out-of-bounds child-index injection into access-structure tree walker — (`src/cbmpc/c_api/access_structure_adapter.h`)

---

### Summary

The C-to-C++ conversion function `to_cpp_access_structure` in `src/cbmpc/c_api/access_structure_adapter.h` contains a bounds check that silently passes when a node's `child_indices_count` exceeds the global `child_indices_count`. The subtraction `in->child_indices_count - n.child_indices_count` wraps to a negative `int32_t`, and since `child_indices_offset` is guaranteed non-negative by an earlier check, the comparison `offset > negative_value` is always `false`, so `E_BADARG` is never returned. The loop at lines 93–94 then reads past the end of the declared `child_indices` array, consuming attacker-controlled memory as child node indices.

---

### Finding Description

The guard is at: [1](#0-0) 

```cpp
if (n.child_indices_offset < 0 || n.child_indices_count < 0) return E_BADARG;
if (n.child_indices_count > 0) {
    if (n.child_indices_offset > in->child_indices_count - n.child_indices_count) return E_BADARG;
}
```

All three fields are `int32_t`. When `n.child_indices_count` (e.g. 5) exceeds `in->child_indices_count` (e.g. 2), the subtraction `2 - 5 = -3` is a valid, negative `int32_t`. The prior check at line 55 guarantees `n.child_indices_offset >= 0`, so `0 > -3` evaluates to `false` and the guard does not fire.

Execution then falls through to the child-index loop: [2](#0-1) 

```cpp
const int32_t off = n.child_indices_offset;
for (int32_t i = 0; i < n.child_indices_count; i++) {
    const int32_t child_idx = in->child_indices[static_cast<size_t>(off + i)];
```

With `off=0` and `n.child_indices_count=5` but only 2 elements declared globally, indices 2, 3, 4 are read out-of-bounds. Because the caller constructs the `cbmpc_access_structure_t` and its backing arrays, they control the memory immediately following the declared `child_indices` array. The values read at those positions are used directly as node indices passed to the recursive `build()` call: [3](#0-2) 

The recursive `build()` only validates that each `child_idx` is in `[0, nodes_count)`: [4](#0-3) 

Since the attacker also controls `nodes` and `nodes_count`, they can craft a `nodes` array where those injected indices are valid, causing the tree walker to build a `coinbase::api::access_structure_t` with extra children that were never declared in the global `child_indices_count`.

---

### Impact Explanation

The resulting `access_structure_t` (C++ tree) is passed downstream to `to_internal_access_structure` in `src/cbmpc/api/access_structure_util.h`, which calls `validate_access_structure_node` and then `build_internal_ac_node`: [5](#0-4) 

By the time those functions run, the C-layer struct has already been converted to a well-formed C++ tree — the injected extra children appear as legitimate `access_structure_t::children` entries. The downstream validator operates on the C++ tree and has no way to detect that the children originated from out-of-bounds reads. The result is that a DKG, refresh, or signing protocol driven by this access structure accepts a quorum tree whose shape was not declared by the caller, satisfying the "accepted invalid cryptographic output" criterion.

---

### Likelihood Explanation

The entrypoint is the public C API (`cbmpc_access_structure_t`), reachable by any unprivileged caller. The attacker only needs to:
1. Declare `child_indices_count = N` (small, e.g. 2) globally.
2. Allocate a backing array larger than `N` and place chosen node indices at positions `N` and beyond.
3. Set a node's `child_indices_count` to a value larger than `N` (e.g. 5) with `child_indices_offset = 0`.

No threshold collusion, no memory corruption beyond the caller's own allocation, and no privileged access is required.

---

### Recommendation

Replace the subtraction-based check with one that cannot underflow. The minimal fix is to add an explicit pre-check before the subtraction:

```cpp
if (n.child_indices_count > 0) {
    // Guard against underflow: check count fits first.
    if (n.child_indices_count > in->child_indices_count) return E_BADARG;
    if (n.child_indices_offset > in->child_indices_count - n.child_indices_count) return E_BADARG;
}
```

Or equivalently, use 64-bit arithmetic to avoid the signed underflow entirely:

```cpp
if ((int64_t)n.child_indices_offset + n.child_indices_count > in->child_indices_count) return E_BADARG;
``` [6](#0-5)

### Proof of Concept

```c
// child_indices backing array has 5 slots; only 2 are declared globally.
int32_t child_indices_backing[5] = {1, 2, 3, 4, 5};  // indices 2-4 are "hidden"

// Three nodes: root (AND), and two leaves.
cbmpc_access_structure_node_t nodes[3] = {
    // Node 0: AND node, claims 5 children via child_indices_count=5,
    // but global child_indices_count=2 — triggers the bypass.
    { CBMPC_ACCESS_STRUCTURE_NODE_AND, NULL, 0, /*offset=*/0, /*count=*/5 },
    { CBMPC_ACCESS_STRUCTURE_NODE_LEAF, "alice", 0, 0, 0 },
    { CBMPC_ACCESS_STRUCTURE_NODE_LEAF, "bob",   0, 0, 0 },
};

cbmpc_access_structure_t ac = {
    .nodes              = nodes,
    .nodes_count        = 3,
    .child_indices      = child_indices_backing,
    .child_indices_count = 2,   // declares only 2, but node 0 asks for 5
    .root_index         = 0,
};

// Expected: non-zero error (E_BADARG).
// Actual:   CBMPC_SUCCESS — the guard at line 57 passes because 0 > (2-5) is false.
cbmpc_error_t rc = /* any DKG/refresh AC entry point */(&ac, ...);
assert(rc != CBMPC_SUCCESS);  // FAILS — vulnerability confirmed
```

  // Clear any existing tree.
  delete out.root;

  if (!curve.valid()) return coinbase::error(E_BADARG, "access_structure: invalid curve");

  // Basic shape validation (independent of job).
  error_t rv = validate_access_structure_node(in);

  // Validate that leaf set matches job.party_names exactly.
  std::set<std::string> leaf_names;
  rv = collect_leaf_names(in, leaf_names);
  if (rv) return rv;

  std::set<std::string> party_set;
  for (const auto& name_view : party_names) party_set.insert(std::string(name_view));

  if (leaf_names != party_set)
    return coinbase::error(E_BADARG, "access_structure: leaf names must match job.party_names exactly");

  // Build internal node tree with generated internal node names.
  std::unordered_set<std::string> used;
  used.reserve(leaf_names.size() * 2 + 8);
  used.insert(std::string());  // root name

  coinbase::crypto::ss::node_t* root = nullptr;
  rv = build_internal_ac_node(in, /*depth=*/0, /*is_root=*/true, used, counter, root);
  if (rv) return rv;

  out.curve = curve;
  out.root = root;

  rv = out.validate_tree();
