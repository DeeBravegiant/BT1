### Title
Consensus Votes Accepted Without Signature Verification, Enabling Forged-Voter Quorum Manipulation - (File: `crates/apollo_consensus/src/single_height_consensus.rs`, `crates/apollo_consensus/src/manager.rs`)

### Summary

The Apollo consensus layer accepts incoming `Vote` messages (prevotes and precommits) from the network and counts them toward quorum without ever verifying the cryptographic signature against the claimed `voter` address. Any network peer can forge votes from any validator identity, potentially manufacturing a false quorum for an attacker-chosen block commitment.

### Finding Description

The `Vote` struct carries both a `voter: ContractAddress` (the claimed validator identity) and a `signature: RawSignature` field. [1](#0-0) 

When the state machine creates its own votes, the signature is explicitly left empty with a `TODO` acknowledging the gap: [2](#0-1) 

When votes arrive from the network, `manager.rs::handle_vote` parses the protobuf message and forwards it directly to `shc.handle_vote(message)` with no signature check: [3](#0-2) 

`SingleHeightConsensus::handle_vote` checks only for height mismatch, duplicate/conflict status, and round caching — never the cryptographic signature: [4](#0-3) 

The `StateMachine` comment states "SHC handles: authentication, replays, and conflicts," but the actual `handle_vote` path in `SingleHeightConsensus` performs no authentication of the `signature` field against the `voter` address. [5](#0-4) 

The infrastructure for signature verification exists (`verify_precommit_vote_signature` in `apollo_signature_manager`) but is never called on the inbound vote path: [6](#0-5) 

A second, related `TODO` in the proposal validation path also defers signature verification: [7](#0-6) 

### Impact Explanation

An unprivileged attacker who can connect to the consensus P2P network can:

1. Observe the current validator set (public information from the committee).
2. Craft `Vote` messages with `voter` set to any legitimate validator's `ContractAddress` and an arbitrary `proposal_commitment`.
3. Broadcast these forged votes; each is accepted and counted at full validator weight toward quorum.
4. By forging votes from validators whose combined weight meets the quorum threshold, force `upon_decision` to fire for an attacker-chosen `ProposalCommitment`, causing `DecisionReached` to be emitted for a block the honest network never agreed on.

The decided block commitment flows into `decision_reached` → batcher → state sync → storage, resulting in a wrong state root, wrong receipts, wrong events, and wrong L1 messages being committed to the chain.

### Likelihood Explanation

The P2P consensus network is reachable by any node that can establish a libp2p connection. Validator addresses are deterministic and publicly derivable from the committee. The attack requires no privileged access, no key material, and no prior state — only the ability to send well-formed protobuf `Vote` messages with a spoofed `voter` field. The `RawSignature::default()` (all-zero) value passes all current checks because no check exists.

### Recommendation

Before counting any inbound vote toward quorum, verify its `signature` field against the `voter`'s registered public key using the existing `verify_precommit_vote_signature` / `verify_identity` infrastructure. Specifically:

- In `SingleHeightConsensus::handle_vote`, look up the voter's public key from the committee and call `verify_precommit_vote_signature` (or an equivalent prevote verifier). Reject votes with invalid or missing signatures before passing them to the state machine.
- Remove the `TODO(Asmaa): sign the vote` placeholder and implement actual ECDSA signing of outbound votes in `make_self_vote`.
- Address the `TODO(matan): Switch to signature validation` in `validate_proposal` to authenticate `ProposalFin` by signature rather than hash equality alone.

### Proof of Concept

```rust
// Attacker connects to the consensus P2P network and sends:
let forged_vote = Vote {
    vote_type: VoteType::Precommit,
    height: current_height,
    round: current_round,
    // Attacker-chosen block commitment
    proposal_commitment: Some(ProposalCommitment(attacker_chosen_felt)),
    // Claimed identity of a high-weight legitimate validator
    voter: high_weight_validator_address,
    // Empty/default signature — never checked
    signature: RawSignature::default(),
};
// Broadcast forged_vote for each validator until quorum weight is reached.
// No key material required. The state machine counts each vote at full weight.
```

The `upon_decision` rule fires once the forged precommit weight meets the quorum threshold, emitting `SMRequest::DecisionReached` for the attacker-chosen commitment, which propagates through `decision_reached` into the batcher and storage layer.

                {
                    error!("Unable to send continue_propagation. {:?}", metadata);
                }
                (message, metadata)
            }
            (Err(e), metadata) => {
                // Failed to parse consensus message. Report the peer and drop the vote.
                self.report_peer(broadcast_channels, &metadata);
                warn!(

                return Ok(VecDeque::new());
            }

        // TODO(matan): We need to figure out an actual caching strategy under 2 constraints:
        // 1. Malicious - must be capped so a malicious peer can't DoS us.
        // 2. Parallel proposals - we may send/receive a proposal for (H+1, 0).
        match message.height.cmp(&height) {
            std::cmp::Ordering::Greater => {
                if self.cache.should_cache_vote(&height, 0, &message) {
                    trace!("Cache message for a future height. {:?}", message);
                    let duplicate_report = self.cache.cache_future_vote(message);
                    if let Err(duplicate_report) = duplicate_report {

                    }

                Ok(VecDeque::new())

                trace!("Drop message from past height. {:?}", message);
                Ok(VecDeque::new())
            }
            std::cmp::Ordering::Equal => match shc {
                Some(shc) => {
                    if self.cache.should_cache_vote(&height, shc.current_round(), &message) {
                        Ok(shc.handle_vote(message))
                    } else {
                        Ok(VecDeque::new())
                    }
                }
                None => {
                    trace!("Drop message from just completed height. {:?}", message);
                    Ok(VecDeque::new())
                }
            },
        }

        debug!("Received {init:?}");
        let height = self.state_machine.height();
        if init.height != height {
            warn!("Invalid proposal height: expected {:?}, got {:?}", height, init.height);
            return VecDeque::new();
        }
        // TODO(guyn): replace this with assert_eq, but also need to fix simulation_test.
        let Ok(proposer_id) = self.committee.get_proposer(height, init.round) else {
            return VecDeque::new();
        };
        if init.proposer != proposer_id {
            warn!("Invalid proposer: expected {:?}, got {:?}", proposer_id, init.proposer);
            return VecDeque::new();
        }
        // Avoid duplicate validations:
        // - If SM already has an entry for this round, a (re)proposal was already recorded.
        // - If we already started validating this round, ignore repeats.
        if self.state_machine.has_proposal_for_round(init.round)
            || self.pending_validation_rounds.contains(&init.round)
        {
            warn!("Round {} already handled a proposal, ignoring", init.round);
            return VecDeque::new();
        }
        let timeout = self.timeouts.get_proposal_timeout(init.round);
        info!(
            "Accepting {init:?}. node_round: {}, timeout: {timeout:?}",

        // parallel (e.g., due to repeats or spam).
