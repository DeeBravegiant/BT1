Audit Report

## Title
Zero-Rate Propagation via Unguarded `updateRate()` / `lzReceive()` Causes Division-by-Zero DoS in Pool Deposits — (`contracts/cross-chain/MultiChainRateProvider.sol`, `contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary

`MultiChainRateProvider.updateRate()` is permissionless and performs no zero-check on the rate fetched from `getLatestRate()` before encoding and broadcasting it via LayerZero. `CrossChainRateReceiver.lzReceive()` performs no zero-check before writing the decoded value to `rate`. If a zero rate propagates to the receiver, every call to `RSETHPool.viewSwapRsETHAmountAndFee` and `RSETHPoolNoWrapper.viewSwapRsETHAmountAndFee` panics with a division-by-zero, bricking all pool deposits until a valid non-zero rate update arrives.

## Finding Description

**Step 1 — Permissionless, unguarded `updateRate()`**

`MultiChainRateProvider.updateRate()` carries only `nonReentrant` — no access control. Any EOA can call it at any time. [1](#0-0) 

The function fetches `latestRate = getLatestRate()` and immediately encodes it with no zero-check before dispatching via LayerZero.

**Step 2 — `getLatestRate()` can return 0**

`RSETHMultiChainRateProvider.getLatestRate()` delegates directly to `ILRTOracle(rsETHPriceOracle).rsETHPrice()` with no floor or sanity check. [2](#0-1) 

`LRTOracle.rsETHPrice` is a stored `uint256` state variable initialized to 0 by default. [3](#0-2) 

It returns 0 before the first `updateRSETHPrice()` call, or if underlying price feeds return stale/zero data causing the oracle's arithmetic to produce zero.

**Step 3 — `lzReceive()` stores zero unconditionally**

`CrossChainRateReceiver.lzReceive()` correctly gates on `msg.sender == layerZeroEndpoint` and `srcAddress == rateProvider`, but performs no zero-check before writing `rate = _rate`. [4](#0-3) 

**Step 4 — Division-by-zero in both pool contracts**

`RSETHPool.viewSwapRsETHAmountAndFee` divides by `rsETHToETHrate` (sourced from `getRate()` → `CrossChainRateReceiver.rate`) with no zero-guard: [5](#0-4) 

`RSETHPoolNoWrapper.viewSwapRsETHAmountAndFee` has the identical pattern: [6](#0-5) 

Both `deposit()` functions call `viewSwapRsETHAmountAndFee` unconditionally, so every deposit reverts while `rate == 0`. [7](#0-6) [8](#0-7) 

## Impact Explanation

All ETH and token deposits into `RSETHPool` and `RSETHPoolNoWrapper` revert with a Solidity panic (division by zero) for the entire window between the zero-rate `lzReceive` and the next valid non-zero rate update. No funds are lost, but the pool fails to deliver its promised exchange service. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

## Likelihood Explanation

`updateRate()` is permissionless, so any caller can trigger it. The oracle returning 0 is realistic in at least one concrete scenario: `LRTOracle.rsETHPrice` is a stored state variable initialized to 0 — if `updateRate()` is called before `updateRSETHPrice()` has ever been invoked on the oracle, the zero value is encoded and dispatched. Additionally, transient oracle failures (all underlying price feeds returning stale/zero data) can produce a zero result without oracle operator compromise. The DoS window persists until the next successful non-zero `updateRate()` propagates through LayerZero.

## Recommendation

1. In `MultiChainRateProvider.updateRate()`: add `require(latestRate > 0, "rate cannot be zero")` before encoding and sending.
2. In `CrossChainRateReceiver.lzReceive()`: add `require(_rate > 0, "rate cannot be zero")` before writing to `rate`.
3. Optionally add a staleness/sanity bound (e.g., rate must be within ±X% of the previous rate) to guard against extreme oracle deviations.

## Proof of Concept

```solidity
// Foundry test outline:
// 1. Deploy LRTOracle (rsETHPrice == 0 before any updateRSETHPrice() call)
// 2. Deploy RSETHMultiChainRateProvider pointing to the oracle
// 3. Deploy mock LayerZero endpoint
// 4. Deploy CrossChainRateReceiver configured with the mock endpoint and provider
// 5. Call provider.updateRate{value: fee}()
//    → encodes 0, sends via mock LZ endpoint
// 6. Mock LZ endpoint calls receiver.lzReceive(srcChainId, srcAddress, 0, abi.encode(uint256(0)))
//    → rate is set to 0
// 7. Assert receiver.getRate() == 0
// 8. Call RSETHPool.viewSwapRsETHAmountAndFee(1 ether)
//    → reverts with Panic(0x12) division-by-zero
// 9. Call RSETHPool.deposit{value: 1 ether}("ref")
//    → reverts, confirming full deposit DoS

// Alternatively, using a mock oracle:
contract MockLRTOracle {
    function rsETHPrice() external pure returns (uint256) { return 0; }
}
// Deploy provider with MockLRTOracle, trigger updateRate, simulate lzReceive,
// confirm pool deposit reverts with division-by-zero panic.
```
