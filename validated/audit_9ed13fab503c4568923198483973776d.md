### Title
Attacker-Controlled `EvmFinality::Latest` Bypasses BNB Finality Guarantee, Enabling Attestation of Reorg-Vulnerable Transactions — (`crates/foreign-chain-inspector/src/evm/inspector.rs`)

---

### Summary

An unprivileged caller can submit a `verify_foreign_transaction` request for BNB with `EvmFinality::Latest`. No guard at the contract layer, the node dispatch layer, or the inspector layer rejects this variant for BNB. The result is that `verify_finality_level` queries `eth_getBlockByNumber('latest')` instead of `'finalized'`, the check passes as soon as the latest head is at or past the receipt block, and the MPC network produces a signed `ForeignTxSignPayloadV1` attesting to a transaction that has not reached BNB's finality threshold and can be reorganized away.

---

### Finding Description

**Entrypoint — contract layer**

`verify_foreign_transaction` in `crates/contract/src/lib.rs` accepts any `VerifyForeignTransactionRequestArgs`. The only guard is a chain-whitelist check:

```rust
if !supported_chains.contains(&requested_chain) { env::panic_str(...) }
``` [1](#0-0) 

There is no check on the `finality` field inside the request. The contract ABI exposes `EvmFinality` as an enum with three valid string values — `"Latest"`, `"Safe"`, `"Finalized"` — all accepted: [2](#0-1) 

**Node dispatch layer — no BNB-specific finality restriction**

In `crates/node/src/providers/verify_foreign_tx/sign.rs`, the BNB branch converts the caller-supplied finality with no restriction:

```rust
let finality: EthereumFinality = request.finality.clone().try_into()?;
``` [3](#0-2) 

**Conversion layer — `Latest` passes through**

`TryFrom<dtos::EvmFinality> for EthereumFinality` maps `Latest → Latest` without error: [4](#0-3) 

**Inspector layer — `Latest` maps to `eth_getBlockByNumber('latest')`**

`verify_finality_level` unconditionally maps `EthereumFinality::Latest` to `FinalityTag::Latest`:

```rust
EthereumFinality::Latest => FinalityTag::Latest,
``` [5](#0-4) 

The RPC call therefore fetches the **latest** (non-finalized) head. The only check is `head.number >= receipt_block_number`. For a transaction in the most recently mined block, this passes immediately.

---

### Impact Explanation

BNB Smart Chain uses Parlia PoSA consensus. The `latest` tag returns the most recently produced block, which has not undergone BNB's finality process (~15 blocks for fast finality). Reorgs on BSC are documented and occur in practice.

When the check passes with `Latest`, the MPC network signs a `ForeignTxSignPayloadV1` whose `request` field encodes `EvmFinality::Latest` and whose `values` field contains the block hash and logs of a non-finalized block: [6](#0-5) 

A downstream bridge contract that trusts this signed attestation will execute an inbound transfer. If the BNB block is subsequently reorganized away, the bridge has credited funds for a transaction that no longer exists on BNB — a classic double-spend / invalid bridge execution condition matching the High-impact scope.

---

### Likelihood Explanation

- The attacker only needs to submit a well-formed `EvmRpcRequest` with `finality: EvmFinality::Latest` — no special privilege, no threshold collusion.
- BNB reorgs are rare but real; the attacker can time the request to a block that is at the tip of a competing fork, or simply rely on natural short reorgs.
- The `verify_block_is_canonical` check (comparing block hash at the receipt height) does not help: it only confirms the block is on the *current* canonical chain at the moment of the RPC call, not that it is finalized. [7](#0-6) 

---

### Recommendation

Reject `EvmFinality::Latest` (and optionally `Safe`) for BNB at the node dispatch layer, before calling the inspector. The simplest fix is to add a per-chain finality allowlist in `sign.rs`:

```rust
dtos::ForeignChainRpcRequest::Bnb(request) => {
    let finality: EthereumFinality = request.finality.clone().try_into()?;
    if finality != EthereumFinality::Finalized {
        bail!("BNB requires EthereumFinality::Finalized");
    }
    // ...
}
```

Alternatively, introduce a `BnbFinality` newtype that only exposes `Finalized`, preventing the invalid variant from ever reaching the inspector.

---

### Proof of Concept

Build a deterministic unit test with a mock RPC client:

1. Mock `eth_getTransactionReceipt` → receipt at block 90, status success.
2. Mock `eth_getBlockByNumber('latest')` → head at block 90 (`head.number >= receipt_block_number` passes).
3. Mock `eth_getBlockByNumber(90)` → same block hash (canonical check passes).
4. Call `BnbInspector::extract(tx_id, EthereumFinality::Latest, extractors)`.
5. Assert `Ok(values)` is returned — a signed payload is produced for a non-finalized transaction.
6. Repeat with `EthereumFinality::Finalized` and mock `eth_getBlockByNumber('finalized')` → head at block 75 (`head.number < 90`) → assert `Err(NotFinalized)`.

This confirms the finality guarantee differs and that `Latest` produces an attestation for a block that has not reached BNB's finality threshold. [8](#0-7) [3](#0-2)

    async fn verify_finality_level(
        &self,
        receipt_block_number: U64,
        finality: EthereumFinality,
    ) -> Result<(), ForeignChainInspectionError> {
        let finality_tag = match finality {
            EthereumFinality::Finalized => FinalityTag::Finalized,
            EthereumFinality::Safe => FinalityTag::Safe,
            EthereumFinality::Latest => FinalityTag::Latest,
        };
        let args = GetBlockByNumberArgs::new(
            BlockNumberOrTag::Tag(finality_tag),
            ReturnFullTransactionObjects::from(false),
        );
        let head: GetBlockByNumberResponse = self
            .client
            .request(GET_BLOCK_BY_NUMBER_METHOD, &args)
            .await?;

        if head.number < receipt_block_number {
            return Err(ForeignChainInspectionError::NotFinalized);
        }
        Ok(())
    }

    async fn verify_block_is_canonical(
        &self,
        receipt_block_number: U64,
        receipt_block_hash: H256,
    ) -> Result<(), ForeignChainInspectionError> {
        let args = GetBlockByNumberArgs::new(
            BlockNumberOrTag::Number(receipt_block_number),
            ReturnFullTransactionObjects::from(false),
        );
        let canonical: GetBlockByNumberResponse = self
            .client
            .request(GET_BLOCK_BY_NUMBER_METHOD, &args)
            .await?;

        let hash_matches = canonical.hash == receipt_block_hash;
        let height_matches = canonical.number == receipt_block_number;
        if !hash_matches || !height_matches {
            return Err(ForeignChainInspectionError::NonCanonicalBlock {
                block_number: receipt_block_number.as_u64(),
                receipt_hash: receipt_block_hash.into(),
                canonical_hash: canonical.hash.into(),
            });
        }
        Ok(())
    }
