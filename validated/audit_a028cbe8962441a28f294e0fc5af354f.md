### Title
Block-Level Reference Script Size Limit Bypassed via Sub-Transaction Reference Scripts in Dijkstra Era — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/Bbody.hs`)

---

### Summary

In the Dijkstra era, `dijkstraBbodyTransition` delegates the block-level reference script size check to `Conway.validateBodyRefScriptsSizeTooBig`, which internally calls `totalRefScriptSizeInBlock`. That function iterates over top-level transactions only and calls `txNonDistinctRefScriptsSize` per tx — a function that reads only the top-level tx's own `inputsTxBodyL` and `referenceInputsTxBodyL`. Sub-transaction reference inputs are never visited. An attacker can therefore craft top-level transactions whose sub-transactions collectively reference arbitrarily large reference scripts from the pre-existing UTxO, bypassing `ppMaxRefScriptSizePerBlockG` entirely.

---

### Finding Description

**Block-level check — does NOT count sub-tx ref scripts:**

`dijkstraBbodyTransition` at line 363 calls:

```haskell
Conway.validateBodyRefScriptsSizeTooBig @era pp blockBody (ls ^. utxoL)
``` [1](#0-0) 

`validateBodyRefScriptsSizeTooBig` computes the total via `totalRefScriptSizeInBlock`:

```haskell
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | pvMajor protVer <= natVersion @10 = ...
  | otherwise = snd $ F.foldl' accum (utxo, 0) txs
  where
    accum (!accUtxo, !accSum) tx = ... accSum + txNonDistinctRefScriptsSize (UTxO accUtxo) tx
``` [2](#0-1) 

`txNonDistinctRefScriptsSize` only reads the top-level tx's own inputs:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

`txSeqBlockBodyL` for Dijkstra yields only `StrictSeq (Tx TopTx era)` — sub-transactions are embedded inside each top-level tx body and are never iterated: [4](#0-3) 

**Per-tx check — correctly counts sub-tx ref scripts:**

`validateAllRefScriptSize` in `dijkstraLedgerTransition` uses `batchNonDistinctRefScriptsSize`:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum (foldMap' (Sum . txNonDistinctRefScriptsSize utxo)
                       (tx ^. bodyTxL . subTransactionsTxBodyL))
``` [5](#0-4) 

This enforces `ppMaxRefScriptSizePerTxG` correctly, but only the per-tx limit — not the per-block limit. [6](#0-5) 

**The gap:** `validateBodyRefScriptsSizeTooBig` (block-level) uses `txNonDistinctRefScriptsSize` (top-level inputs only), while `validateAllRefScriptSize` (per-tx) uses `batchNonDistinctRefScriptsSize` (top-level + all sub-tx inputs). There is no Dijkstra-specific override of `validateBodyRefScriptsSizeTooBig` or `totalRefScriptSizeInBlock`.

---

### Impact Explanation

An attacker submits a block containing N top-level transactions, each carrying K sub-transactions that reference large scripts from the pre-existing UTxO. The top-level tx body itself carries zero reference inputs. The block-level check computes a total of 0 bytes (no top-level ref inputs), while the actual reference script bytes loaded during sub-tx validation is N × K × scriptSize. The `ppMaxRefScriptSizePerBlockG` limit is completely ineffective for sub-transaction reference scripts.

This matches the **Medium** allowed impact: attacker-controlled transactions exceed intended validation limits (the block-level reference script byte cap).

---

### Likelihood Explanation

- Dijkstra era is not yet deployed on mainnet, but the code is production-targeted and the bug is present in the current implementation.
- No privileged access is required. Any transaction author can embed sub-transactions in a top-level tx.
- The attack is deterministic and locally reproducible: craft top-level txs with empty `referenceInputsTxBodyL` but sub-txs referencing large UTxO-resident scripts; submit in a block; observe that `validateBodyRefScriptsSizeTooBig` passes despite the aggregate sub-tx ref script size exceeding `ppMaxRefScriptSizePerBlockG`.

---

### Recommendation

Replace the call to `Conway.validateBodyRefScriptsSizeTooBig` in `dijkstraBbodyTransition` with a Dijkstra-specific variant that uses `batchNonDistinctRefScriptsSize` (or an equivalent block-level accumulator) instead of `txNonDistinctRefScriptsSize` when summing reference script sizes across all top-level transactions and their sub-transactions.

---

### Proof of Concept

1. Deploy Dijkstra era with default protocol parameters (`ppMaxRefScriptSizePerBlockG = B`, `ppMaxRefScriptSizePerTxG = T`, where `T < B`).
2. Pre-populate the UTxO with reference script outputs totalling `S` bytes, where `S > B`.
3. Construct a block with one top-level tx that has:
   - Zero entries in `referenceInputsTxBodyL` and `inputsTxBodyL` (top-level).
   - `ceil(S / T)` sub-transactions, each referencing a disjoint subset of the pre-existing reference script UTxO entries, with each sub-tx's batch size ≤ T.
4. Submit the block.
5. **Expected (correct) behaviour:** `BodyRefScriptsSizeTooBig` failure because aggregate sub-tx ref script bytes > B.
6. **Actual behaviour:** Block accepted. `totalRefScriptSizeInBlock` returns 0 (no top-level ref inputs), so the check passes. Sub-tx ref scripts are only validated against the per-tx limit inside `validateAllRefScriptSize`, which also passes because each sub-tx batch ≤ T.

totalRefScriptSizeInBlock ::
  (AlonzoEraTx era, BabbageEraTxBody era) => ProtVer -> StrictSeq (Tx TopTx era) -> UTxO era -> Int
totalRefScriptSizeInBlock protVer txs (UTxO utxo)
  | pvMajor protVer <= natVersion @10 =
      getSum $ foldMap' (Monoid.Sum . txNonDistinctRefScriptsSize (UTxO utxo)) txs

validateAllRefScriptSize ::
  ( EraTx era
  , DijkstraEraTxBody era
  ) =>
  PParams era ->
  UTxO era ->
  Tx TopTx era ->
  Test (DijkstraLedgerPredFailure era)
validateAllRefScriptSize pp utxo tx =
  let totalRefScriptSize = batchNonDistinctRefScriptsSize utxo tx
      maxRefScriptSizePerTx = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerTxG
   in failureUnless (totalRefScriptSize <= maxRefScriptSizePerTx) $
        DijkstraTxRefScriptsSizeTooBig
          Mismatch
            { mismatchSupplied = totalRefScriptSize
            , mismatchExpected = maxRefScriptSizePerTx
            }
