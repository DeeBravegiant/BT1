### Title
Unprivileged `TransferToGasKey` Action Allows Any Caller to Lock Victim's Gas Key Deletion - (`File: runtime/runtime/src/actions.rs`)

### Summary

The `check_actor_permissions` function in nearcore's runtime omits an actor-identity check for the `TransferToGasKey` action, allowing any unprivileged account or contract to force tokens into another account's gas key balance. When the gas key balance is pushed above `GasKeyInfo::MAX_BALANCE_TO_BURN`, the victim's `DeleteKey` and `DeleteAccount` actions both fail with `GasKeyBalanceTooHigh`, permanently blocking key/account deletion until the victim manually withdraws the excess balance — which the attacker can continuously counteract.

### Finding Description

`check_actor_permissions` in `runtime/runtime/src/actions.rs` enforces `actor_id == account_id` for sensitive actions (`DeployContract`, `Stake`, `AddKey`, `DeleteKey`, `WithdrawFromGasKey`, etc.), but explicitly exempts `TransferToGasKey`:

```rust
Action::CreateAccount(_)
| Action::FunctionCall(_)
| Action::Transfer(_)
| Action::TransferToGasKey(_) => (),   // ← no actor check
``` [1](#0-0) 

This means any account can submit a transaction with `receiver_id = victim` and `Action::TransferToGasKey { public_key: victim_gas_key_pk, deposit: amount }`, or any deployed contract can call the `promise_batch_action_transfer_to_gas_key` host function targeting the victim's account. [2](#0-1) 

When executed, `action_transfer_to_gas_key` unconditionally adds `action.deposit` to the gas key's stored balance:

```rust
gas_key_info.balance = gas_key_info.balance.checked_add(action.deposit)...?;
set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);
``` [3](#0-2) 

`action_delete_key` and `action_delete_account` both check the gas key balance against `GasKeyInfo::MAX_BALANCE_TO_BURN` (1 NEAR) and return `GasKeyBalanceTooHigh` if exceeded:

```rust
let gas_key_balance_to_burn = compute_gas_key_balance_sum(state_update, account_id)?;
if gas_key_balance_to_burn > GasKeyInfo::MAX_BALANCE_TO_BURN {
    result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh { ... }.into());
    return Ok(());
}
``` [4](#0-3) 

The test `test_delete_gas_key_balance_too_high` confirms that depositing 1 NEAR + 1 yoctoNEAR into a gas key causes `DeleteKey` to fail and the key to remain in state: [5](#0-4) 

### Impact Explanation

An attacker who knows a victim's gas key public key (observable on-chain via `view_access_keys`) can:

1. Send a signed transaction from their own account to `receiver_id = victim` with `TransferToGasKey { public_key: victim_gas_key, deposit: 1_NEAR + 1 }`.
2. The victim's gas key balance is pushed above `MAX_BALANCE_TO_BURN`.
3. `DeleteKey` on that gas key now fails with `GasKeyBalanceTooHigh` — the key cannot be removed.
4. `DeleteAccount` also fails because `compute_gas_key_balance_sum` exceeds the limit — the account cannot be deleted.
5. The victim can call `WithdrawFromGasKey` (which has no balance cap check) to reduce the balance, but the attacker can immediately re-fund it, sustaining the lock indefinitely at a cost of ~1 NEAR per cycle.

The corrupted protocol value is `gas_key_info.balance` in the trie state, which causes `DeleteKey` and `DeleteAccount` receipts to produce `GasKeyBalanceTooHigh` action errors, permanently altering the outcome of those receipts and the resulting state root. [6](#0-5) 

### Likelihood Explanation

- Gas key public keys are publicly visible via RPC (`view_access_keys`).
- The attack requires only a standard signed transaction — no special privileges.
- Cost to the attacker is ~1 NEAR per lock cycle; the victim must spend gas to withdraw and the attacker can immediately re-lock.
- The `promise_batch_action_transfer_to_gas_key` host function is exposed to all deployed contracts, widening the attack surface. [7](#0-6) 

### Recommendation

Add `TransferToGasKey` to the actor-permission guard in `check_actor_permissions`, requiring `actor_id == account_id` before the action is applied:

```rust
Action::DeployContract(_)
| Action::Stake(_)
| Action::AddKey(_)
| Action::DeleteKey(_)
| Action::DeployGlobalContract(_)
| Action::UseGlobalContract(_)
| Action::WithdrawFromGasKey(_)
| Action::TransferToGasKey(_) => {   // ← add this
    if actor_id != account_id {
        return Err(ActionErrorKind::ActorNoPermission { ... }.into());
    }
}
``` [8](#0-7) 

If the intent is to allow third-party funding of gas keys (e.g., relayers), a separate, bounded host function with explicit opt-in by the account owner should be designed instead of reusing the unrestricted `TransferToGasKey` action path.

### Proof of Concept

**Attacker-controlled entry path** (direct transaction, no contract needed):

```
1. Victim creates account `victim.near` with a gas key (public key `GK_PK`).
2. Attacker submits a signed transaction:
     signer_id:   attacker.near
     receiver_id: victim.near
     actions:     [TransferToGasKey { public_key: GK_PK, deposit: 1_001_000_000_000_000_000_000_001 }]
3. Runtime executes action_transfer_to_gas_key on victim.near:
     gas_key_info.balance += deposit  →  balance > MAX_BALANCE_TO_BURN (1 NEAR)
4. Victim attempts DeleteKey { public_key: GK_PK }:
     → GasKeyBalanceTooHigh error; key remains in trie state.
5. Victim attempts DeleteAccount { beneficiary_id: ... }:
     → GasKeyBalanceTooHigh error; account cannot be deleted.
6. Victim calls WithdrawFromGasKey to reduce balance;
   attacker immediately re-sends TransferToGasKey to restore it above the limit.
```

The `promise_batch_action_transfer_to_gas_key` host function (confirmed callable from any contract via the test contract in `runtime/near-test-contracts/test-contract-rs/src/lib.rs`) provides an additional on-chain entry path requiring no direct transaction signing by the attacker. [9](#0-8)

pub(crate) fn check_actor_permissions(
    action: &Action,
    account: &Option<Account>,
    actor_id: &AccountId,
    account_id: &AccountId,
) -> Result<(), ActionError> {
    match action {
        Action::DeployContract(_)
        | Action::Stake(_)
        | Action::AddKey(_)
        | Action::DeleteKey(_)
        | Action::DeployGlobalContract(_)
        | Action::UseGlobalContract(_)
        | Action::WithdrawFromGasKey(_) => {
            if actor_id != account_id {
                return Err(ActionErrorKind::ActorNoPermission {
                    account_id: account_id.clone(),
                    actor_id: actor_id.clone(),
                }
                .into());
            }
        }
        Action::DeleteAccount(_) => {
            if actor_id != account_id {
                return Err(ActionErrorKind::ActorNoPermission {
                    account_id: account_id.clone(),
                    actor_id: actor_id.clone(),
                }
                .into());
            }
            let account = account.as_ref().unwrap();
            if !account.locked().is_zero() {
                return Err(ActionErrorKind::DeleteAccountStaking {
                    account_id: account_id.clone(),
                }
                .into());
            }
        }
        Action::CreateAccount(_)
        | Action::FunctionCall(_)
        | Action::Transfer(_)
        | Action::TransferToGasKey(_) => (),
        Action::Delegate(_) | Action::DelegateV2(_) => (),
        Action::DeterministicStateInit(_) => (),
    };
    Ok(())

            public_key_len as usize,

            burn_base,

            use_byte,

pub(crate) fn action_transfer_to_gas_key(
    state_update: &mut TrieUpdate,
    result: &mut ActionResult,
    account_id: &AccountId,
    action: &TransferToGasKeyAction,
) -> Result<(), RuntimeError> {
    let Some(mut access_key) = get_access_key(state_update, account_id, &action.public_key)? else {
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };
    let Some(gas_key_info) = access_key.gas_key_info_mut() else {
        // Key exists but is not a gas key
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };

    gas_key_info.balance = gas_key_info.balance.checked_add(action.deposit).ok_or_else(|| {
        RuntimeError::StorageError(StorageError::StorageInconsistentState(
            "gas key balance integer overflow".to_string(),
        ))
    })?;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);
    Ok(())
}
