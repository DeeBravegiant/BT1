### Title
Gas Key Balance Permanently Locked When `GasKeyFunctionCall` Key Is the Only Signing Key and Balance Exceeds `MAX_BALANCE_TO_BURN` - (`runtime/runtime/src/access_keys.rs`, `runtime/runtime/src/actions.rs`)

### Summary

When a gas key's balance exceeds `GasKeyInfo::MAX_BALANCE_TO_BURN` (1 NEAR), both `DeleteKey` and `DeleteAccount` are unconditionally blocked. The only intended recovery path is `WithdrawFromGasKey`. However, a `GasKeyFunctionCall` access key cannot sign a `WithdrawFromGasKey` transaction. If an account's only remaining key is a `GasKeyFunctionCall` key, the gas key balance above 1 NEAR is permanently locked with no protocol-level recovery path — not even for the account owner.

### Finding Description

`GasKeyInfo::MAX_BALANCE_TO_BURN` is set to exactly 1 NEAR: [1](#0-0) 

`delete_gas_key` in `runtime/runtime/src/access_keys.rs` unconditionally rejects deletion when the balance exceeds this threshold: [2](#0-1) 

`action_delete_account` in `runtime/runtime/src/actions.rs` applies the same hard block on the aggregate gas key balance: [3](#0-2) 

The only recovery path is `action_withdraw_from_gas_key`, which moves balance from the gas key back to the account: [4](#0-3) 

However, `WithdrawFromGasKey` is not a `FunctionCall` action. The `verify_and_charge_gas_key_tx_ephemeral` (and `verify_and_charge_tx_ephemeral`) path enforces that a `GasKeyFunctionCall` key can only sign transactions containing a single `FunctionCall` action to the permitted receiver/method: [5](#0-4) 

`AccessKeyPermission::GasKeyFunctionCall` returns `Some` from `function_call_permission()`, so the restriction applies: [6](#0-5) 

A `GasKeyFunctionCall` key attempting to sign a `WithdrawFromGasKey` transaction will fail with `RequiresFullAccess` or `MethodNameMismatch`. Therefore, if an account's only remaining key is a `GasKeyFunctionCall` key and its gas key balance exceeds 1 NEAR:

- `DeleteKey` → `GasKeyBalanceTooHigh` (blocked)
- `DeleteAccount` → `GasKeyBalanceTooHigh` (blocked)
- `WithdrawFromGasKey` → cannot be signed by `GasKeyFunctionCall` key (blocked)

The gas key balance is permanently inaccessible. There is no protocol-level override or admin escape hatch.

### Impact Explanation

The corrupted protocol value is the gas key balance stored in `GasKeyInfo.balance` within the account's trie state. This balance is real NEAR tokens that were transferred into the gas key via `TransferToGasKey`. Once locked, these tokens cannot be withdrawn, burned (deletion is blocked), or transferred. The account itself cannot be deleted. The funds are permanently lost from the user's perspective, with no recovery path at the protocol level.

### Likelihood Explanation

The scenario requires:
1. An account funds a `GasKeyFunctionCall` gas key with more than 1 NEAR.
2. The account subsequently deletes all full-access and `GasKeyFullAccess` keys, leaving only the `GasKeyFunctionCall` key.

This can occur through user error (e.g., a user migrating to gas keys and deleting their full-access key prematurely), or through a contract that calls `TransferToGasKey` on behalf of the user, pushing the balance above the threshold after the user has already removed their full-access key. The likelihood is low but the impact is total and irreversible.

### Recommendation

The protocol should ensure that `WithdrawFromGasKey` is always accessible regardless of the signing key type, OR that `DeleteKey` for a gas key with balance > `MAX_BALANCE_TO_BURN` automatically returns the balance to the account rather than blocking the operation entirely. A simpler fix is to allow `GasKeyFunctionCall` keys to sign `WithdrawFromGasKey` actions targeting their own gas key, since this is a self-recovery operation that does not expand the key's effective permissions.

### Proof of Concept

1. Account `alice` has a full-access key `K_full` and a `GasKeyFunctionCall` key `K_gas` restricted to calling `contract.near::some_method`.
2. Alice sends `TransferToGasKey { public_key: K_gas, deposit: 2_000_000_000_000_000_000_000_000 }` (2 NEAR) signed by `K_full`. Gas key balance is now 2 NEAR > `MAX_BALANCE_TO_BURN`.
3. Alice sends `DeleteKey { public_key: K_full }` signed by `K_full`. Now only `K_gas` remains.
4. Alice attempts `DeleteKey { public_key: K_gas }` signed by `K_gas` → runtime returns `GasKeyBalanceTooHigh`. [7](#0-6) 

5. Alice attempts `DeleteAccount { beneficiary_id: ... }` signed by `K_gas` → runtime returns `GasKeyBalanceTooHigh`.
6. Alice attempts `WithdrawFromGasKey { public_key: K_gas, amount: 1_500_000_000_000_000_000_000_000 }` signed by `K_gas` → runtime returns `RequiresFullAccess` (or `MethodNameMismatch`) because `K_gas` is a `GasKeyFunctionCall` key and `WithdrawFromGasKey` is not a `FunctionCall` action. [8](#0-7) 

The 2 NEAR in the gas key balance is permanently locked. The account cannot be deleted and the balance cannot be recovered.

pub(crate) fn action_withdraw_from_gas_key(
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    action: &WithdrawFromGasKeyAction,
) -> Result<(), RuntimeError> {
    let Some(mut access_key) = get_access_key(state_update, account_id, &action.public_key)? else {
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };
    let Some(gas_key_info) = access_key.gas_key_info_mut() else {
        // Key exists but is not a gas key
        result.result = Err(ActionErrorKind::GasKeyDoesNotExist {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
        }
        .into());
        return Ok(());
    };

    let Some(updated_balance) = gas_key_info.balance.checked_sub(action.amount) else {
        result.result = Err(ActionErrorKind::InsufficientGasKeyBalance {
            account_id: account_id.clone(),
            public_key: Box::new(action.public_key.clone()),
            balance: gas_key_info.balance,
            required: action.amount,
        }
        .into());
        return Ok(());
    };
    gas_key_info.balance = updated_balance;
    set_access_key(state_update, account_id.clone(), action.public_key.clone(), &access_key);

    let new_account_balance = account.amount().checked_add(action.amount).ok_or_else(|| {
        RuntimeError::StorageError(StorageError::StorageInconsistentState(
            "Account balance integer overflow".to_string(),
        ))
    })?;
    account.set_amount(new_account_balance);
    Ok(())
