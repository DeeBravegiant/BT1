### Title
Data Layer Mirror Coins Are Spendable by Anyone Due to Signature-Free `P2_PARENT.curry(1)` Puzzle — (`File: chia/wallet/db_wallet/db_wallet_puzzles.py`)

---

### Summary

The Data Layer mirror puzzle is constructed as `P2_PARENT.curry(Program.to(1))`. Because the curried inner puzzle is `1` (the CLVM identity/anyone-can-spend program), any unprivileged attacker who knows a mirror coin's parent coin details — all of which are public on-chain — can craft a valid spend bundle that redirects the mirror coin's locked XCH to an arbitrary address. The only ownership check exists in the wallet Python layer (`DataLayerWallet.delete_mirror`), which is trivially bypassed by submitting a raw spend bundle directly.

---

### Finding Description

**Mirror puzzle construction:**

`create_mirror_puzzle()` returns `P2_PARENT.curry(Program.to(1))`. [1](#0-0) 

`P2_PARENT` is the "pay-to-parent" puzzle. It verifies that the spender provides the correct parent coin lineage (`parent_coin.parent_coin_info`, `parent_inner_puzzle`, `parent_coin.amount`) and then runs the **curried inner puzzle** against `inner_sol` to produce conditions. The curried inner puzzle here is `Program.to(1)` — the CLVM identity program — which simply returns its argument (`inner_sol`) as conditions. No `AGG_SIG_ME` or `AGG_SIG_UNSAFE` condition is emitted by the puzzle itself.

**Solution structure used when spending a mirror coin:**

```python
Program.to([
    parent_coin.parent_coin_info,
    parent_inner_puzzle,
    parent_coin.amount,
    inner_sol,
])
``` [2](#0-1) 

All four fields are public on-chain data. An attacker can read `parent_coin.parent_coin_info`, `parent_inner_puzzle` (from the parent's spend), and `parent_coin.amount` from any full node, then supply an `inner_sol` that creates a `CREATE_COIN` condition sending the mirror coin's XCH to the attacker's puzzle hash.

**Wallet-level guard is bypassable:**

The only ownership enforcement is a Python-level check inside `DataLayerWallet.delete_mirror`:

```python
inner_puzzle_derivation = await self.wallet_state_manager.puzzle_store \
    .get_derivation_record_for_puzzle_hash(parent_coin.puzzle_hash)
if inner_puzzle_derivation is None:
    raise ValueError(f"DL Wallet does not have permission to delete mirror with ID {mirror_id}")
``` [3](#0-2) 

This guard is never evaluated when an attacker submits a raw `SpendBundle` directly to the full node via `push_tx` or the peer protocol. The CLVM puzzle itself imposes no signature requirement.

---

### Impact Explanation

Mirror coins hold real XCH (the `amount` parameter supplied by the creator): [4](#0-3) 

An attacker can drain the XCH from **any** mirror coin on-chain — regardless of who created it — by submitting a spend bundle with `inner_sol = [[51, attacker_puzhash, mirror_coin.amount]]`. This constitutes unauthorized spend of XCH from singleton-adjacent Data Layer coins, matching the Critical impact category: *unauthorized spend of XCH affecting Data Layer-linked coins*.

---

### Likelihood Explanation

- Mirror coins and their parent coin details are fully public on-chain.
- No private key, admin access, or special privilege is required.
- The attacker only needs to query a full node for the mirror coin's parent spend, then submit a crafted `SpendBundle`.
- Any mirror coin with non-trivial XCH locked is a target.

---

### Recommendation

Replace the inner puzzle `Program.to(1)` with a puzzle that enforces a signature from the parent coin's owner. Specifically, `create_mirror_puzzle` should curry a standard `p2_delegated_puzzle_or_hidden_puzzle` (or equivalent) keyed to the creator's public key, so that spending requires a valid `AGG_SIG_ME` condition. Alternatively, the mirror puzzle should assert a coin announcement from the parent coin's spend, ensuring only the parent's owner can authorize the mirror spend. [5](#0-4)

### Proof of Concept

1. Alice creates a mirror coin with `amount = 1_000_000_000_000` mojos (1 XCH). The mirror coin's puzzle hash is `MIRROR_PUZZLE_HASH = create_mirror_puzzle().get_tree_hash()`.
2. Attacker queries the full node for the mirror coin's parent coin spend (public). Extracts `parent_coin.parent_coin_info`, `parent_inner_puzzle`, `parent_coin.amount`.
3. Attacker constructs:
   ```python
   inner_sol = Program.to([[51, attacker_puzhash, mirror_coin.amount]])
   solution = Program.to([
       parent_coin.parent_coin_info,
       parent_inner_puzzle,
       parent_coin.amount,
       inner_sol,
   ])
   coin_spend = make_spend(mirror_coin, create_mirror_puzzle(), solution)
   spend_bundle = SpendBundle([coin_spend], G2Element())  # no signature needed
   ```
4. Attacker submits `spend_bundle` via `push_tx` to any full node.
5. The spend is accepted (no `AGG_SIG` condition required by the puzzle). Alice's 1 XCH is transferred to the attacker. [1](#0-0) [6](#0-5)
