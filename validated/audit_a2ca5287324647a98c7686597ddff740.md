Audit Report

## Title
Chainlink Price Staleness Indicators Ignored in `ChainlinkPriceOracle.getAssetPrice()` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all validity fields, accepting any price including stale ones without any staleness check. A stale low price propagates through `LRTOracle._updateRsETHPrice()` and can trigger the protocol's downside-protection pause, temporarily freezing all user deposits and withdrawals. The same codebase applies correct staleness checks in `ChainlinkOracleForRSETHPoolCollateral`, confirming the omission is unintentional.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` at [1](#0-0)  discards all five return values except `answer`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same repo correctly validates `answeredInRound < roundID` (stale round) and `timestamp == 0` (incomplete round): [2](#0-1) 

The stale price propagates through the following confirmed call chain:

1. `LRTOracle.getAssetPrice(asset)` delegates to `ChainlinkPriceOracle.getAssetPrice()`. [3](#0-2) 
2. `_getTotalEthInProtocol()` calls `getAssetPrice()` for every supported asset. [4](#0-3) 
3. `_updateRsETHPrice()` computes `newRsETHPrice` from the stale total ETH. [5](#0-4) 
4. If `newRsETHPrice` drops beyond `pricePercentageLimit`, both `lrtDepositPool` and `withdrawalManager` are paused. [6](#0-5) 
5. `updateRSETHPrice()` is `public` with no access control, callable by any address. [7](#0-6) 

The SECURITY.md exclusion for "Incorrect data supplied by third-party oracles" does not apply here: the oracle is supplying correct data including staleness metadata; the bug is the protocol's own code ignoring that metadata. [8](#0-7) 

## Impact Explanation
**Medium — Temporary freezing of funds.** When a Chainlink feed goes stale with a price sufficiently below the last recorded `highestRsethPrice`, any unprivileged caller invoking `updateRSETHPrice()` causes `_updateRsETHPrice()` to compute a depressed `newRsETHPrice`. If the drop exceeds `pricePercentageLimit`, `lrtDepositPool.pause()` and `withdrawalManager.pause()` are called, freezing all user deposits and withdrawals until an admin manually unpauses. [9](#0-8) 

**Low — Contract fails to deliver promised returns.** A stale high price causes `_getTotalEthInProtocol()` to overestimate TVL, setting `rsETHPrice` too high. Depositors then receive fewer rsETH tokens than entitled via `getRsETHAmountToMint()`. [10](#0-9) 

## Likelihood Explanation
Chainlink feeds can go stale during Ethereum network congestion, L2 sequencer downtime, or oracle node failures — historically observed events. The trigger function `updateRSETHPrice()` is public with no access control, so any external actor can lock in a stale price the moment it is live. No privileged access, no victim mistake, and no unrealistic assumption is required. [7](#0-6) 

## Recommendation
Apply the same staleness checks already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Optionally add a configurable `heartbeat` threshold check (`block.timestamp - updatedAt > heartbeat`) for feeds with known update frequencies. [11](#0-10) 

## Proof of Concept
1. Deploy a mock Chainlink aggregator that returns a price 10% below the true market price with a stale `updatedAt` and `answeredInRound < roundId`.
2. Register it as the price feed for a supported LST asset via `ChainlinkPriceOracle.updatePriceFeedFor()`.
3. Set `pricePercentageLimit` to `5e16` (5%) on `LRTOracle`.
4. Call `LRTOracle.updateRSETHPrice()` from an unprivileged EOA.
5. Observe: `_getTotalEthInProtocol()` returns a value ~10% below the true TVL; `newRsETHPrice` is ~10% below `highestRsethPrice`; `diff > pricePercentageLimit.mulWad(highestRsethPrice)` evaluates to true; `lrtDepositPool.pause()` and `withdrawalManager.pause()` are called; all deposits and withdrawals are frozen.
6. Confirm no admin action was required and the caller was unprivileged. [6](#0-5)
