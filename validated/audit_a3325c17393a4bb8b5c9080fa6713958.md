Audit Report

## Title
Stale `CrossChainRateReceiver` Rate in `AGETHPoolV3` Enables Over-Minting of agETH â€” (`contracts/agETH/AGETHPoolV3.sol`)

## Summary

`CrossChainRateReceiver.getRate()` returns the last stored `rate` with no staleness enforcement, despite storing `lastUpdated`. Because agETH is yield-bearing, its true ETH rate monotonically increases; any staleness window makes the denominator in `AGETHPoolV3.viewSwapAgETHAmountAndFee()` artificially low, minting more agETH than the deposited collateral warrants. Unlike `RSETHPoolV3`, `AGETHPoolV3` has no `limitDailyMint` modifier and no pause mechanism, leaving the over-minting unbounded.

## Finding Description

**Root cause:** `CrossChainRateReceiver.getRate()` unconditionally returns `rate` without checking `block.timestamp - lastUpdated` against any maximum age: [1](#0-0) 

`rate` is only updated when a LayerZero message arrives via `lzReceive()`: [2](#0-1) 

`AGETHPoolV3.getRate()` delegates directly to this oracle with no additional staleness guard: [3](#0-2) 

The stale rate is then used as the denominator in `viewSwapAgETHAmountAndFee()`: [4](#0-3) 

The public `deposit(address, uint256, string)` entrypoint has no `limitDailyMint` modifier and no `whenNotPaused` guard: [5](#0-4) 

By contrast, `RSETHPoolV3.deposit()` applies both `whenNotPaused` and `limitDailyMint`: [6](#0-5) 

The asymmetric-staleness scenario is directly reachable: the collateral token oracle (`ChainlinkOracleForRSETHPoolCollateral`) has a round-based staleness check, while the agETH oracle has none: [7](#0-6) 

**Exploit flow:**
1. Oracle staleness occurs naturally (LayerZero message delay, fee exhaustion, or infrequent `updateRate()` calls on the provider side).
2. Attacker calls `AGETHPoolV3.deposit(token, largeAmount, referralId)`.
3. `viewSwapAgETHAmountAndFee` computes `agETHAmount = amountAfterFee * tokenToETHRate / staleRate`, where `staleRate < trueRate`, yielding excess agETH.
4. Attacker bridges excess agETH to L1 and redeems for ETH.
5. Protocol has issued agETH unbacked by collateral.

## Impact Explanation

**Critical â€” Protocol insolvency.** The protocol mints agETH in excess of the deposited collateral's true ETH value. The attacker redeems the excess agETH on L1 for real ETH, leaving the protocol with a collateral shortfall. With no daily mint cap on `AGETHPoolV3`, a single large deposit can extract the full excess in one transaction. At 5% APY and 24-hour staleness, a 1,000 ETH deposit yields ~0.137 ETH of unbacked issuance; scaled to protocol-level deposits, this constitutes direct, concrete insolvency.

## Likelihood Explanation

Cross-chain oracle staleness is a routine, non-adversarial condition requiring no privileged access, no front-running, and no external protocol compromise. The yield-bearing nature of agETH guarantees the stale rate is always lower than the true rate, making the direction of the exploit deterministic. The attack is repeatable across any staleness window and requires only a standard ERC-20 deposit call from any unprivileged address.

## Recommendation

1. **Enforce a staleness threshold in `CrossChainRateReceiver.getRate()`:**
   ```solidity
   uint256 public constant MAX_STALENESS = 24 hours;
   function getRate() external view returns (uint256) {
       require(block.timestamp - lastUpdated <= MAX_STALENESS, "Rate stale");
       return rate;
   }
   ```
2. **Add a defence-in-depth staleness check in `AGETHPoolV3.viewSwapAgETHAmountAndFee()`** by reading `AGETHRateReceiver.lastUpdated` directly before using the rate.
3. **Add a `limitDailyMint` modifier and a `pause` mechanism to `AGETHPoolV3`** analogous to those present in `RSETHPoolV3`, to bound worst-case exposure during any staleness window.

## Proof of Concept

**Differential model test (no fork required):**

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

contract PoC {
    /// @notice Demonstrates excess agETH minted when oracle is stale.
    /// @param depositAmount  Collateral amount (18-decimal)
    /// @param tokenRate      Token/ETH rate (e.g. 1e18 for ETH)
    /// @param staleRate      agETH/ETH rate from N hours ago
    /// @param freshRate      Current true agETH/ETH rate (freshRate > staleRate always)
    function testOverMint(
        uint256 depositAmount,
        uint256 tokenRate,
        uint256 staleRate,
        uint256 freshRate
    ) external pure returns (uint256 excess) {
        require(freshRate > staleRate, "agETH is yield-bearing: fresh > stale");
        uint256 mintedStale = depositAmount * tokenRate / staleRate;
        uint256 mintedFresh = depositAmount * tokenRate / freshRate;
        excess = mintedStale - mintedFresh;
        assert(excess > 0);
    }
}
```
