### Title
Off-by-one inclusive slice in `valid_entries` allows expired MPC image hash to remain accepted — (`crates/contract/src/tee/proposal.rs`)

---

### Summary

`AllowedDockerImageHashes::valid_entries` uses `rposition` to find the rightmost expired entry and then returns `get(cutoff_index..)`, an **inclusive** slice. This means the expired entry at `cutoff_index` is always included in the returned valid set. During the window between when M1's grace period expires and when M2's grace period expires, M1 is still accepted for new attestation submissions.

> **Note on the question's scenario**: Step (3) states "time advances to T2 + grace_period + 1 (M1 expired, M2 still valid)." At that exact timestamp M2's deadline (`T2 + grace_period`) is also in the past, so M2 is expired too. The correct window for the bug is `T1 + grace_period < current_time < T2 + grace_period`. The step (4) logic is correct for that window; only the timestamp label in step (3) is off. The underlying bug is real.

---

### Finding Description

`valid_entries` is defined as:

```rust
fn valid_entries(&self, tee_upgrade_deadline_duration: Duration) -> Vec<AllowedMpcDockerImage> {
    let current_time = Timestamp::now();
    let cutoff_index = self
        .allowed_tee_proposals
        .iter()
        .rposition(|allowed_docker_image| {
            ...
            grace_period_deadline < current_time   // true  ⟺  entry is expired
        })
        .unwrap_or(0);

    self.allowed_tee_proposals
        .get(cutoff_index..)   // ← inclusive: expired entry at cutoff_index is returned
        .unwrap_or(&[])
        .to_vec()
}
``` [1](#0-0) 

The list is sorted oldest-first: `[M1(T1), M2(T2)]` with `T1 < T2`.

**In the window `T1 + grace_period < now < T2 + grace_period`:**

| Entry | Deadline | `grace_period_deadline < now` | `rposition` match? |
|-------|----------|-------------------------------|---------------------|
| M1 (index 0) | T1 + GP | YES (expired) | YES |
| M2 (index 1) | T2 + GP | NO (still valid) | NO |

`rposition` returns `Some(0)` → `cutoff_index = 0` → `get(0..)` = `[M1, M2]`.

M1 is expired but is still included. A node running M1 can call `add_participant` → `verify_locally` with `mpc_docker_image_hash = M1`, and the check passes because M1 is in the returned slice. [2](#0-1) 

The existing tests do **not** cover this window. `test_clean_expired` and `only_latest_hash_after_grace_period` both advance time to `T2 + grace_period + 1`, at which point M2 is also expired, so `rposition` finds M2 (index 1) and `get(1..)` correctly excludes M1. The intermediate window is never tested. [3](#0-2) [4](#0-3) 

---

### Impact Explanation

`add_participant` feeds the result of `get_allowed_mpc_docker_image_hashes` directly into `verify_locally`: [2](#0-1) 

A node still running M1 (an image whose grace period has passed) can submit a fresh attestation and have it stored in `stored_attestations`. That stored attestation is then used by `reverify_participants` / `is_caller_an_attested_participant` to authorize the node for protocol operations including signing. The invariant "nodes running expired MPC code must not obtain fresh attestations" is broken for the entire duration `(T2 − T1)` — the gap between when M1 and M2 were voted in.

---

### Likelihood Explanation

The window is open for exactly `T2 − T1` seconds after M1's grace period expires. In practice, image hash rotations happen infrequently (weeks apart), so the window can be days to weeks long. Any node operator who has not yet upgraded from M1 to M2 can exploit this passively — no special tooling is required beyond running the old image and calling `submit_participant_info` as normal.

---

### Recommendation

Distinguish the "no expired entry" case from the "found an expired entry" case, and use an exclusive upper bound:

```rust
match self.allowed_tee_proposals.iter().rposition(|e| {
    e.added.checked_add(tee_upgrade_deadline_duration)
        .map_or(true, |deadline| deadline < current_time)
}) {
    None => self.allowed_tee_proposals.to_vec(),   // nothing expired → return all
    Some(cutoff) => {
        // keep at least the newest entry as disaster-recovery fallback
        let start = if cutoff + 1 < self.allowed_tee_proposals.len() {
            cutoff + 1
        } else {
            cutoff   // all expired: return only the newest
        };
        self.allowed_tee_proposals.get(start..).unwrap_or(&[]).to_vec()
    }
}
```

    fn valid_entries(&self, tee_upgrade_deadline_duration: Duration) -> Vec<AllowedMpcDockerImage> {
        let current_time = Timestamp::now();
        // get the index of the most recently enforced docker image
        let cutoff_index = self
            .allowed_tee_proposals
            .iter()
            .rposition(|allowed_docker_image| {
                let Some(grace_period_deadline) = allowed_docker_image
                    .added
                    .checked_add(tee_upgrade_deadline_duration)
                else {
                    log!("Error: timestamp overflowed when calculating grace_period_deadline.");
                    return true;
                };
                // if the grace period for this docker hash is in the past, then older hashes are no longer accepted
                grace_period_deadline < current_time
            })
            .unwrap_or(0);

        self.allowed_tee_proposals
            .get(cutoff_index..)
            .unwrap_or(&[])
            .to_vec()
    }
