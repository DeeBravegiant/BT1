Audit Report

## Title
Missing `limitDailyMint` Modifier on `RSETHPoolNoWrapper` Deposit Functions Allows Single-Transaction Pool Drain - (File: contracts/pools/RSETHPoolNoWrapper.sol)

## Summary
`RSETHPoolNoWrapper` omits the `limitDailyMint` safety modifier from both `deposit` overloads. Every other pool variant in the protocol enforces a per-day distribution cap on rsETH. Because `RSETHPoolNoWrapper` holds a pre-minted rsETH balance and transfers it directly rather than minting wrapper tokens, a single depositor with sufficient ETH or LST can exhaust the entire rsETH reserve atomically, leaving the pool unable to serve any subsequent depositor until an operator manually refills it.

## Finding Description
`RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, `RSETHPoolV3WithNativeChainBridge`, and `RSETHPoolV2ExternalBridge` all apply `limitDailyMint` to their deposit entry points. The modifier tracks cumulative rsETH distributed within the current day and reverts if the daily ceiling would be exceeded: [1](#0-0) 

`RSETHPoolNoWrapper.deposit(string)` and `deposit(address,uint256,string)` carry only `nonReentrant` and `whenNotPaused`: [2](#0-1) 

Unlike the wrapper-based pools that call `wrsETH.mint(...)` (unbounded supply), `RSETHPoolNoWrapper` holds a finite pre-minted rsETH balance and calls `rsETH.safeTransfer(msg.sender, rsETHAmount)`. Once that balance reaches zero, every subsequent `safeTransfer` reverts with an ERC-20 insufficient-balance error. There is no on-chain mechanism to cap how much rsETH a single call can extract, so the entire reserve is reachable in one transaction. [3](#0-2) [4](#0-3) 

Contrast with `RSETHPoolV3.deposit` which mints on demand and is still rate-limited: [5](#0-4) 

And `RSETHPoolV3ExternalBridge.deposit` which also enforces the cap: [6](#0-5) 

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

The attacker pays oracle-fair ETH (or LST) for every rsETH received; no funds are stolen and no accounting is corrupted. However, after the drain the pool holds zero rsETH and every subsequent `deposit` call reverts until an operator refills the contract. The pool ceases to deliver its core service — exchanging ETH/LST for rsETH — to all other users for an indefinite period. This matches the allowed Low impact: *contract fails to deliver promised returns, but doesn't lose value*.

## Likelihood Explanation
No special role, flash loan, oracle manipulation, or governance capture is required. The only prerequisite is holding enough ETH or a supported LST to match the pool's rsETH reserve at the current oracle rate — a realistic condition for a pool designed to hold meaningful liquidity. The attack is repeatable each time the pool is refilled.

## Recommendation
Add the `limitDailyMint` modifier (or an equivalent daily-distribution cap) to both `deposit` overloads in `RSETHPoolNoWrapper`, consistent with every other pool variant. The modifier should track distributed rsETH against a configurable daily ceiling and reset each day relative to `startTimestamp`, exactly as implemented in `RSETHPoolV3.limitDailyMint`. [1](#0-0) 

## Proof of Concept
1. `RSETHPoolNoWrapper` is deployed on Arbitrum/Unichain holding `N` rsETH tokens.
2. Attacker reads `N = rsETH.balanceOf(pool)` and the current rate `r = getRate()`.
3. Attacker calls `deposit("")` with `msg.value = N * r / 1e18` (adjusted for `feeBps`).
4. `viewSwapRsETHAmountAndFee` returns `rsETHAmount ≈ N`; `rsETH.safeTransfer(attacker, N)` succeeds.
5. Pool rsETH balance is now 0. All subsequent `deposit` calls revert on `safeTransfer` until the pool is refilled by an operator. [7](#0-6)

    modifier limitDailyMint(uint256 amount, address token) {
        if (block.timestamp < startTimestamp) {
            revert MintBeforeStartTimestamp();
        }

        uint256 rsETHAmount;

        // Calculate the amount of rsETH that will be minted
        if (token == ETH_IDENTIFIER) {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount);
        } else {
            (rsETHAmount,) = viewSwapRsETHAmountAndFee(amount, token);
        }

        uint256 currentDay = getCurrentDay();

        // If the current day is greater than the last mint day, reset the daily mint amount
        if (currentDay > lastMintDay) {
            lastMintDay = currentDay;
            dailyMintAmount = 0;
        }

        // Check if the daily mint amount plus the amount to mint is greater than the daily mint limit
        if (dailyMintAmount + rsETHAmount > dailyMintLimit) {
            revert DailyMintLimitExceeded();
        }

        dailyMintAmount += rsETHAmount;
        _;
    }

    function viewSwapRsETHAmountAndFee(uint256 amount) public view returns (uint256 rsETHAmount, uint256 fee) {
        fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
    }
