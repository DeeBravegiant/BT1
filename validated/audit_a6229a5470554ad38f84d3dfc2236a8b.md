### Title
Missing Access Control on `clean_invalid_attestations` Allows Any Caller to Prune Participant Attestations - (File: crates/contract/src/lib.rs)

### Summary

`MpcContract::clean_invalid_attestations()` is explicitly documented as "Callable by anyone" and carries no access-control guard. Every other post-resharing cleanup method in the same contract is either marked `#[private]` (self-call only) or checks that the caller is a current participant. The missing restriction lets any unprivileged NEAR account trigger attestation pruning at an adversarially chosen moment, which can remove attestations for current participants whose TEE quotes have just expired but have not yet been renewed, causing `verify_tee()` to see fewer valid participants than the threshold and halting all signing operations or triggering an unwanted resharing.

### Finding Description

`clean_invalid_attestations` mutates `tee_state.stored_attestations` by removing entries that fail `reverify_participants`. The function accepts a caller-controlled `max_scan: u32` parameter with no upper bound, so a single call can sweep the entire attestation map.

`clean_invalid_attestations` is the only mutable cleanup endpoint with no restriction at all.

The underlying `TeeState::clean_invalid_attestations` removes every entry whose `reverify_participants` result is not `TeeQuoteStatus::Valid`: [4](#0-3) 

### Impact Explanation

MPC nodes must renew their TEE attestations periodically (every ~7 days per the design doc). There is a window between the moment an attestation expires and the moment the node calls `submit_participant_info` to renew it. During this window the attestation is technically invalid and will be removed by `clean_invalid_attestations`.

An attacker who calls `clean_invalid_attestations(max_scan: u32::MAX)` immediately after one or more participants' attestations expire (but before renewal) removes those entries from `stored_attestations`. The next call to `verify_tee()` then sees fewer valid participants:

- If the surviving count drops below the governance threshold, `accept_requests` is set to `false`, halting all signature requests and CKD requests for every user of the MPC network.
- If the surviving count is still above the threshold but below the full participant set, an unwanted resharing is triggered, disrupting the key-event lifecycle. [5](#0-4) 

This breaks the request-lifecycle and participant-state invariants without requiring any privileged access or network-level DoS.

### Likelihood Explanation

Attestations expire on a known, predictable schedule (the `tee_upgrade_deadline_duration_seconds` config value is public). An attacker can monitor on-chain attestation timestamps via `get_attestation` / `get_tee_accounts`, compute the exact expiry block, and submit the call in the same block the attestation becomes invalid. The attack requires no special role, no collusion, and no leaked key — only a NEAR account and knowledge of the public contract state.

### Recommendation

Apply the same access-control pattern used by `remove_non_participant_update_votes`: restrict `clean_invalid_attestations` to self-calls (the post-resharing promise) and direct calls from current participants. Alternatively, mark it `#[private]` so it can only be invoked as a self-call promise, consistent with all other cleanup endpoints.

```rust
// Authorize the caller: allow self-calls and direct calls from a current participant.
let caller = env::predecessor_account_id();
let is_self_call = caller == env::current_account_id();
if !is_self_call && !participants.is_participant_given_account_id(&caller) {
    return Err(InvalidState::NotParticipant { account_id: caller }.into());
}
```

        // Caller must be a participant (node or operator).
        self.voter_or_panic();

            return Err(InvalidState::ProtocolStateNotRunning.into());
        };
        let current_params = running_state.parameters.clone();

        let tee_upgrade_deadline_duration =
            Duration::from_secs(self.config.tee_upgrade_deadline_duration_seconds);

        match self.tee_state.reverify_and_cleanup_participants(
            current_params.participants(),
            tee_upgrade_deadline_duration,
        ) {
            TeeValidationResult::Full => {
                self.accept_requests = true;
                log!("All participants have an accepted Tee status");
                Ok(true)
            }
            TeeValidationResult::Partial {
                participants_with_valid_attestation,
            } => {
                let remaining = participants_with_valid_attestation.len();
                // Defense in depth: the surviving participant set must keep the full
                // threshold relation intact — the GovernanceThreshold must still sit
                // within its bounds for the smaller set (in particular it must not
                // exceed the remaining participant count or the upper cap) and must
                // remain at least every domain's ReconstructionThreshold (the kickout
                // keeps the existing per-domain thresholds). Otherwise we refuse and
                // wait for manual intervention.
                let max_reconstruction_threshold =
                    max_reconstruction_threshold(running_state.domains.domains());

                        remaining,

                    self.accept_requests = false;
                    return Ok(false);

    pub fn clean_invalid_attestations(
        &mut self,
        tee_upgrade_deadline_duration: Duration,
        max_scan: usize,
    ) -> u32 {
        let has_invalid_attestation = |node_id: &NodeId| {
            !matches!(
                self.reverify_participants(node_id, tee_upgrade_deadline_duration),
                TeeQuoteStatus::Valid
            )
        };

        // Materialize candidates before any mutation to avoid iterator invalidation.
        let invalid_tls_keys: Vec<Ed25519PublicKey> = self
            .stored_attestations
            .iter()
            .take(max_scan)
            .filter(|(_, node_attestation)| has_invalid_attestation(&node_attestation.node_id))
            .map(|(tls_pk, _)| tls_pk.clone())
            .collect();

        let removed = u32::try_from(invalid_tls_keys.len())
            .expect("u32 should always be convertible from usize on wasm32");

        for tls_pk in invalid_tls_keys {
            self.stored_attestations.remove(&tls_pk);
        }
        removed

    /// whose TLS key matches an attested node belonging to the caller account.
    ///
    /// Handles multiple participants per account and supports legacy mock nodes.
    pub(crate) fn is_caller_an_attested_participant(
        &self,
        participants: &Participants,
    ) -> Result<(), AttestationCheckError> {
        let signer_account_pk = env::signer_account_pk();
        let signer_id = env::signer_account_id();

        let info = participants
            .info(&signer_id)
            .ok_or(AttestationCheckError::CallerNotParticipant)?;

        let attestation = self
            .stored_attestations
            .get(&info.tls_public_key)
            .ok_or(AttestationCheckError::AttestationNotFound)?;

        if attestation.node_id.account_id != signer_id {
            return Err(AttestationCheckError::AttestationOwnerMismatch);
        }

        // Stored account keys are Ed25519 by construction; a non-Ed25519
        // signer necessarily mismatches.
        let signer_ed25519 = Ed25519PublicKey::try_from(&signer_account_pk)
            .map_err(|_| AttestationCheckError::AttestationKeyMismatch)?;
        if attestation.node_id.account_public_key != signer_ed25519 {
            return Err(AttestationCheckError::AttestationKeyMismatch);
        }

        Ok(())
    }
