Audit Report

## Title
`getMinAmount` Integer Truncation Causes Unexpected Revert for Small Amounts — (`contracts/bridges/TACWETHBridge.sol`)

## Summary
`getMinAmount` computes `minAmount` via integer division and reverts with `InvalidMinAmount` if the result truncates to zero. Because `bridgeTokenToL1` and `getNativeFee` both call `getMinAmount` internally, any `amount` satisfying `amount * (BASIS_POINTS_DIVISOR - slippageTolerance) < BASIS_POINTS_DIVISOR` causes both functions to revert. The contract's only documented rejection condition for positive amounts is `ZeroAmount` (`amount == 0`), so this silent dead zone violates the implied contract interface.

## Finding Description
`getMinAmount` at line 174 computes:

```solidity
uint256 minAmount = amount * (BASIS_POINTS_DIVISOR - slippageTolerance) / BASIS_POINTS_DIVISOR;
if (minAmount == 0) {
    revert InvalidMinAmount();
}
``` [1](#0-0) 

Solidity integer division truncates toward zero. For any `amount` where `amount * (BASIS_POINTS_DIVISOR - slippageTolerance) < BASIS_POINTS_DIVISOR`, the result is `0` and the function reverts. With the default operational `slippageTolerance` of even 1 bp, `amount = 1 wei` produces `1 * 9999 / 10000 = 0`.

`getNativeFee` calls `getMinAmount` at line 153, propagating the revert to the view quoting function: [2](#0-1) 

`bridgeTokenToL1` calls `getNativeFee` at line 108, propagating the revert to the state-changing bridge entry point: [3](#0-2) 

The only guard before `getMinAmount` is called is `amount == 0` (line 103/145), which explicitly promises that any positive amount is valid. The `InvalidMinAmount` revert is never surfaced to callers as an expected outcome for positive amounts. [4](#0-3) 

Note: The sub-case of an admin setting `slippageTolerance = 10_000` via `setSlippageTolerance` is excluded as it requires privileged access. The valid, unprivileged trigger is the small-amount truncation under any non-zero operational slippage.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

Any user attempting to bridge a dust amount (e.g., `1 wei`) receives an opaque `InvalidMinAmount` revert rather than the expected `ZeroAmount` revert or a successful bridge. No funds are lost since the revert occurs before any token transfer. The contract fails to deliver its promised bridging service for the affected amount range, and `getNativeFee` (a view function used by UIs/integrators to pre-check feasibility) also reverts unexpectedly, breaking quoting flows.

## Likelihood Explanation
Triggerable by any unprivileged caller of `bridgeTokenToL1` or `getNativeFee` with a sufficiently small positive `amount`. With `slippageTolerance = 1` (0.01%), `amount = 1 wei` always fails. The condition is deterministic and reproducible with no special setup. Likelihood is low in practice since bridging 1 wei is economically irrational, but the interface contract is still violated.

## Recommendation
Change the `slippageTolerance` guard from `>` to `>=` to reject the degenerate 100% case:

```solidity
if (newSlippageTolerance >= BASIS_POINTS_DIVISOR) revert InvalidSlippageTolerance();
``` [5](#0-4) 

In `getMinAmount`, floor the result to 1 instead of reverting, or add a minimum-amount pre-check in `bridgeTokenToL1`/`getNativeFee` that returns a descriptive error before calling `getMinAmount`:

```solidity
uint256 minAmount = amount * (BASIS_POINTS_DIVISOR - slippageTolerance) / BASIS_POINTS_DIVISOR;
if (minAmount == 0) minAmount = 1; // floor to avoid spurious revert on dust amounts
``` [1](#0-0) 

## Proof of Concept
```solidity
// forge test --match-test testGetMinAmountRevertsForDustAmount -vvv
function testGetMinAmountRevertsForDustAmount() public {
    // slippageTolerance = 500 (5%), amount = 1 wei
    // minAmount = 1 * (10000 - 500) / 10000 = 9500 / 10000 = 0 → InvalidMinAmount
    vm.expectRevert(TACWETHBridge.InvalidMinAmount.selector);
    bridge.getMinAmount(1);
}

function testGetNativeFeeRevertsForDustAmount() public {
    // getNativeFee calls getMinAmount → same revert propagates
    vm.expectRevert(TACWETHBridge.InvalidMinAmount.selector);
    bridge.getNativeFee(1, address(0xBEEF));
}

function testBridgeTokenToL1RevertsForDustAmount() public {
    // bridgeTokenToL1 calls getNativeFee → same revert propagates
    vm.expectRevert(TACWETHBridge.InvalidMinAmount.selector);
    bridge.bridgeTokenToL1{value: 0}(address(0xBEEF), 1);
}
```
