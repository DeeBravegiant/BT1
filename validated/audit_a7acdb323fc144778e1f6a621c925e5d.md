Audit Report

## Title
`emergencyWithdrawFromAave` Fails to Reset `isAaveIntegrationEnabled`, Temporarily Freezing ETH Withdrawals - (File: contracts/LRTWithdrawalManager.sol)

## Summary

`emergencyWithdrawFromAave` withdraws all ETH from Aave back to the contract but never sets `isAaveIntegrationEnabled = false`. After the call, the contract holds a stale `isAaveIntegrationEnabled = true` flag with zero Aave balance. Once the returned ETH is consumed by the first batch of `completeWithdrawal` calls, every subsequent ETH withdrawal reverts with `InsufficientAaveBalance`, temporarily freezing user funds until an admin separately calls `setAaveIntegrationEnabled(false)`.

## Finding Description

`emergencyWithdrawFromAave` at L551–563 performs the withdrawal but omits the state reset:

```solidity
function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
    if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();
    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    if (aaveBalance == 0) revert InsufficientAaveBalance();
    _collectInterestToTreasury();
    uint256 withdrawnAmount = _withdrawFromAave(amount);
    emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
    // isAaveIntegrationEnabled is never set to false
}
```

`_withdrawFromAave` at L918 decrements `totalETHDepositedToAave` to zero, but `isAaveIntegrationEnabled` remains `true`.

`_processWithdrawalCompletion` at L720–731 then reads the stale flag:

```solidity
if (isAaveIntegrationEnabled && asset == LRTConstants.ETH_TOKEN) {
    uint256 contractBalance = address(this).balance;
    if (contractBalance < request.expectedAssetAmount) {
        uint256 amountNeeded = request.expectedAssetAmount - contractBalance;
        _withdrawFromAave(amountNeeded);   // ← reverts
    }
}
```

`_withdrawFromAave` at L908–909 unconditionally reverts when the Aave balance is zero:

```solidity
uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
if (aaveBalance == 0) revert InsufficientAaveBalance();
```

By contrast, `setAaveIntegrationEnabled(false)` at L486–504 correctly withdraws funds, revokes the aWETH approval, and sets `isAaveIntegrationEnabled = false`. `emergencyWithdrawFromAave` skips all of this.

## Impact Explanation

**Medium — Temporary freezing of user ETH withdrawal funds.**

After the emergency withdrawal, the ETH returned to the contract is finite. Once it is consumed by the first batch of `completeWithdrawal` calls, every subsequent ETH withdrawal reverts. Users whose rsETH was already burned at `initiateWithdrawal` time cannot claim their ETH until an admin manually calls `setAaveIntegrationEnabled(false)`. The freeze is temporary (recoverable by admin action) but real and affects all pending ETH withdrawals.

## Likelihood Explanation

`emergencyWithdrawFromAave` is an explicitly designed operational path for crisis scenarios (Aave exploit, liquidity crunch). The PAUSER calling it is a legitimate, intended action — not an attack. No malicious actor is required. The freeze manifests automatically for every ETH `completeWithdrawal` call once the contract's native balance falls below a request's `expectedAssetAmount`. The only recovery is a separate admin transaction.

## Recommendation

Mirror what `setAaveIntegrationEnabled(false)` already does inside `emergencyWithdrawFromAave`:

```solidity
function emergencyWithdrawFromAave(uint256 amount) external nonReentrant onlyRole(LRTConstants.PAUSER_ROLE) {
    if (!isAaveIntegrationEnabled) revert AaveIntegrationNotEnabled();
    uint256 aaveBalance = aaveAWETH.balanceOf(address(this));
    if (aaveBalance == 0) revert InsufficientAaveBalance();
    _collectInterestToTreasury();
    uint256 withdrawnAmount = _withdrawFromAave(amount);
+   isAaveIntegrationEnabled = false;
+   _revokeApprovalToAaveWETHGateway();
+   emit AaveIntegrationEnabled(false);
    emit EmergencyWithdrawFromAave(withdrawnAmount, address(this));
}
```

## Proof of Concept

1. Aave integration is enabled; operator calls `unlockQueue` for ETH → `depositToAaveExternal` deposits 100 ETH; `totalETHDepositedToAave = 100`, `isAaveIntegrationEnabled = true`.
2. PAUSER calls `emergencyWithdrawFromAave(type(uint256).max)` → 100 ETH returns to contract, `totalETHDepositedToAave = 0`, **`isAaveIntegrationEnabled` remains `true`**.
3. User A calls `completeWithdrawal(ETH_TOKEN, ...)` for 60 ETH → `contractBalance (100) >= 60` → succeeds; contract holds 40 ETH.
4. User B calls `completeWithdrawal(ETH_TOKEN, ...)` for 50 ETH → `contractBalance (40) < 50` → enters Aave branch at L720 → calls `_withdrawFromAave(10)` → `aaveAWETH.balanceOf(address(this)) == 0` → **reverts `InsufficientAaveBalance`**.
5. User B and all subsequent ETH withdrawers are frozen until admin calls `setAaveIntegrationEnabled(false)`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
