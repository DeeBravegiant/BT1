Audit Report

## Title
Unbounded Nested-Loop Gas Consumption in `updateRSETHPrice()` Triggered by Any Caller - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function whose execution cost scales as `O(assets × NDCs × queuedWithdrawals × strategies)` due to nested loops across supported assets, NodeDelegators, and EigenLayer queued withdrawals. As the protocol scales under normal operation, this multiplicative gas growth can cause the function to revert out-of-gas, freezing the rsETH price. The same loop is also traversed on every user deposit, which would simultaneously freeze deposits.

## Finding Description
`updateRSETHPrice()` is declared `public whenNotPaused` with no role restriction, allowing any EOA or contract to invoke it. [1](#0-0) 

It calls `_updateRsETHPrice()` → `_getTotalEthInProtocol()`, which iterates over every entry in `supportedAssetList` and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` for each. [2](#0-1) 

`getTotalAssetDeposits` delegates to `getAssetDistributionData`, which loops over every NDC in `nodeDelegatorQueue` and calls `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)` for each NDC. For ETH, the same pattern occurs via `getETHDistributionData`. [3](#0-2) [4](#0-3) 

`getAssetUnstaking` makes an external call to EigenLayer's `DelegationManager.getQueuedWithdrawals(address(this))` — once per (asset, NDC) pair — and then iterates over every withdrawal and every strategy within each withdrawal. [5](#0-4) 

The critical inefficiency is that `getQueuedWithdrawals` is called `assets × NDCs` times, fetching the same NDC withdrawal list redundantly for each asset. With 5 assets and 10 NDCs, this is 50 external calls to EigenLayer, each returning up to `maxUncompletedWithdrawalCount` withdrawals.

The same `getTotalAssetDeposits` path is triggered on every user deposit via `_beforeDeposit` → `_checkIfDepositAmountExceedesCurrentLimit`. [6](#0-5) 

There is no explicit cap on `supportedAssetList.length`; assets are added via `TIME_LOCK_ROLE` with no upper bound enforced. [7](#0-6) 

`maxNodeDelegatorLimit` is initialized to 10 and is admin-settable upward; `maxUncompletedWithdrawalCount` is also admin-settable. Neither parameter individually is unreasonable, but their product with the asset count grows multiplicatively. [8](#0-7) 

## Impact Explanation
**Medium — Unbounded gas consumption.** Under realistic mainnet growth (e.g., 5 assets × 10 NDCs × 20 queued withdrawals × 3 strategies = 3,000 inner iterations plus 50 external EigenLayer calls), the gas cost of `updateRSETHPrice()` can approach or exceed the Ethereum block gas limit, causing permanent reversion. This freezes the rsETH price. Simultaneously, `depositETH` and `depositAsset` traverse the same loop per deposited asset across all NDCs, also reverting and temporarily freezing all user deposits.

## Likelihood Explanation
No privileged action by an attacker is required. The function fails naturally as the protocol grows to a sufficient scale — adding more LSTs, more NodeDelegators, and accumulating more EigenLayer queued withdrawals during normal undelegation and rebalancing operations. The parameters that bound the loops (`maxNodeDelegatorLimit`, `maxUncompletedWithdrawalCount`, `supportedAssetList.length`) are each individually reasonable but their product is not constrained. Any EOA can trigger the failure simply by calling `updateRSETHPrice()` after the protocol has scaled.

## Recommendation
- Restructure `getAssetUnstaking` calls so that `getQueuedWithdrawals` is fetched once per NDC and the result is filtered across all assets in a single pass, eliminating the `assets × NDCs` redundant external calls.
- Maintain an incremental `assetUnstaking` accumulator updated on `initiateUnstaking` / `completeUnstaking` / `undelegate` rather than recomputing from EigenLayer state on every read.
- Introduce a hard cap on `supportedAssetList.length` analogous to `maxNodeDelegatorLimit`.
- Add a gas-budget guard or paginated view to `_getTotalEthInProtocol` so a single call cannot exhaust the block gas limit.

## Proof of Concept
1. Deploy with 5 supported assets, 10 NDCs (`maxNodeDelegatorLimit = 10`), and `maxUncompletedWithdrawalCount = 20`.
2. Operator calls `initiateUnstaking` on each NDC until each has 20 queued EigenLayer withdrawals with 3 strategies each.
3. Any EOA calls `LRTOracle.updateRSETHPrice()`.
4. Execution: `updateRSETHPrice` → `_updateRsETHPrice` → `_getTotalEthInProtocol` → 5 × `getTotalAssetDeposits` → 5 × `getAssetDistributionData` → 5 × 10 NDC iterations → 50 calls to `getQueuedWithdrawals`, each returning 20 withdrawals × 3 strategies = 3,000 inner loop iterations plus 50 external calls.
5. At ~5,000 gas per storage read and external call overhead, total gas exceeds several million, approaching or surpassing the block gas limit, causing the transaction to revert out-of-gas.
6. With `updateRSETHPrice` reverting, rsETH price is frozen. Simultaneously, any call to `depositETH` or `depositAsset` traverses the same NDC × withdrawal loop for the deposited asset, also reverting and blocking all new deposits.

```text
        }
```

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
