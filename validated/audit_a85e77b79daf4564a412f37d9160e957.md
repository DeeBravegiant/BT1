### Title
Dijkstra Era Minimum Fee Underestimated: Sub-Transaction Reference Script Size Excluded from Batch Fee Calculation - (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

In the Dijkstra era, the minimum-fee validation for a top-level transaction uses `getConwayMinFeeTxUtxo`, which internally calls `txNonDistinctRefScriptsSize` — a function that only measures reference scripts belonging to the **top-level** transaction. The correct function for a Dijkstra batch is `batchNonDistinctRefScriptsSize`, which also accumulates reference-script sizes across all **sub-transactions**. Because the wrong (smaller) size is supplied to the tiered exponential fee formula, the computed minimum fee is systematically lower than the true cost of executing the batch, allowing an unprivileged sender to submit a Dijkstra transaction whose fee is below the protocol-intended minimum.

---

### Finding Description

**Root cause — wrong variable in fee calculation**

The `EraUTxO DijkstraEra` instance explicitly sets:

```haskell
getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` passes the reference-script size to the fee formula as:

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

`txNonDistinctRefScriptsSize` only inspects the top-level transaction's own `inputsTxBodyL` and `referenceInputsTxBodyL`; it never traverses `subTransactionsTxBodyL`:

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

**The correct batch-aware function already exists but is never used in fee validation**

The Dijkstra UTxO module exports `batchNonDistinctRefScriptsSize`, which correctly sums reference-script sizes across the top-level transaction **and** all sub-transactions:

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

This function is never wired into the fee-validation path.

**The fee check in the Dijkstra UTXO rule uses the underestimated minimum**

```haskell
{- minfee pp txTop utxo₀ ≤ txfee txb -}
runTest $ Shelley.validateFeeTooSmallUTxO pp tx originalUtxo
```
