### Title
Dijkstra Batch Minimum Fee Underpayment Due to Sub-Transaction Reference Scripts Being Excluded from Fee Calculation - (File: `eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

---

### Summary

The Dijkstra era's `EraUTxO` instance sets `getMinFeeTxUtxo = getConwayMinFeeTxUtxo`, which only counts reference scripts from the top-level transaction's inputs when computing the minimum fee. However, Dijkstra introduces sub-transactions that can themselves carry reference inputs pointing to UTxOs with reference scripts. A dedicated function `batchNonDistinctRefScriptsSize` exists to aggregate reference script sizes across the entire batch (top-level + all sub-transactions), but it is never wired into the fee validation path. As a result, any Dijkstra transaction whose sub-transactions use reference scripts will have its minimum fee computed without accounting for those scripts' sizes, allowing systematic underpayment of the reference-script fee component.

---

### Finding Description

**Root cause — `getMinFeeTxUtxo` delegates to the Conway implementation:**

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, line 141
instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getConwayMinFeeTxUtxo
``` [1](#0-0) 

`getConwayMinFeeTxUtxo` computes the reference-script size using only the top-level transaction's inputs:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs, lines 174-175
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
``` [2](#0-1) 

`txNonDistinctRefScriptsSize` only unions the top-level transaction's `referenceInputs` and `inputs`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/UTxO.hs, lines 183-187
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
``` [3](#0-2) 

**The correct batch-aware function exists but is unused in fee validation:**

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs, lines 264-277
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum
      ( foldMap'
          (Sum . txNonDistinctRefScriptsSize utxo)
          (tx ^. bodyTxL . subTransactionsTxBodyL)
      )
``` [4](#0-3) 

`batchNonDistinctRefScriptsSize` correctly aggregates reference script sizes from both the top-level transaction and every sub-transaction, but it is never called from `getMinFeeTxUtxo` or from any fee-validation rule in the Dijkstra UTXO rule.

**Fee validation path uses `getMinFeeTxUtxo`:**

The inherited `feesOK` check (from Alonzo/Babbage) calls `getMinFeeTxUtxo pp tx u`:

```haskell
-- eras/alonzo/impl/src/Cardano/Ledger/Alonzo/Rules/Utxo.hs, lines 292-298
feesOK pp tx u@(UTxO utxo) =
  let ...
      minFee = getMinFeeTxUtxo pp tx u
   in sequenceA_
        [ failureUnless
            (minFee <= theFee)
            (FeeTooSmallUTxO Mismatch {mismatchSupplied = theFee, mismatchExpected = minFee})
``` [5](#0-4) 

For Dijkstra, this resolves to `getConwayMinFeeTxUtxo`, which passes only `txNonDistinctRefScriptsSize` (top-level only) to `getConwayMinFeeTx`:

```haskell
-- eras/conway/impl/src/Cardano/Ledger/Conway/Tx.hs, lines 103-112
getConwayMinFeeTx pp tx refScriptsSize =
  alonzoMinFeeTx pp tx <+> refScriptsFee
  where
    refScriptCostPerByte = unboundRational (pp ^. ppMinFeeRefScriptCostPerByteL)
    refScriptsFee =
      tierRefScriptFee
        (unboundRational $ pp ^. ppRefScriptCostMultiplierG)
        (fromIntegral @Word32 @Int . unNonZero $ pp ^. ppRefScriptCostStrideG)
        refScriptCostPerByte
        refScriptsSize
``` [6](#0-5) 

The `refScriptsSize` passed in is zero for any reference scripts that appear only in sub-transaction inputs, so `tierRefScriptFee` charges nothing for them.

---

### Impact Explanation

**Medium.** An unprivileged transaction sender can craft a Dijkstra top-level transaction whose sub-transactions carry reference inputs pointing to UTxOs with large reference scripts. The ledger's minimum-fee check will compute `refScriptsFee` using only the top-level transaction's reference script sizes, ignoring all sub-transaction reference scripts. The sender therefore pays less than the protocol-mandated minimum fee for the actual resource consumption of the batch. This directly modifies fees outside design parameters, matching the allowed Medium impact: *"Attacker-controlled transactions… modify fees… outside design parameters."*

---

### Likelihood Explanation

**High.** The misconfiguration is structural: `getMinFeeTxUtxo = getConwayMinFeeTxUtxo` is unconditional. Every Dijkstra transaction that includes sub-transactions with reference inputs to UTxOs containing reference scripts will trigger the underpayment. No special conditions or timing are required; any transaction sender can exploit this on every submission.

---

### Recommendation

Replace the `getMinFeeTxUtxo` binding in the Dijkstra `EraUTxO` instance with a Dijkstra-specific implementation that uses `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
-- eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs
instance EraUTxO DijkstraEra where
  ...
  getMinFeeTxUtxo = getDijkstraMinFeeTxUtxo

getDijkstraMinFeeTxUtxo :: (EraTx era, DijkstraEraTxBody era) => PParams era -> Tx l era -> UTxO era -> Coin
getDijkstraMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ batchNonDistinctRefScriptsSize utxo tx
```

This ensures the tiered reference-script fee (`tierRefScriptFee`) is applied to the total reference script size across the entire batch, consistent with the intent expressed by `batchNonDistinctRefScriptsSize`.

---

### Proof of Concept

1. Deploy a UTxO output containing a large Plutus reference script (e.g., 50 KiB).
2. Construct a Dijkstra top-level transaction with an empty `referenceInputs` and `inputs` set (no reference scripts at the top level).
3. Add a sub-transaction whose `referenceInputs` includes the UTxO from step 1.
4. Set the top-level fee to the value returned by `getMinFeeTxUtxo` (which will be computed without the 50 KiB sub-transaction reference script).
5. Submit the transaction. The ledger accepts it because `feesOK` calls `getMinFeeTxUtxo`, which returns a fee that excludes the sub-transaction reference script cost.
6. The correct fee (using `batchNonDistinctRefScriptsSize`) would be higher by `tierRefScriptFee multiplier stride costPerByte 51200` (approximately `15 * 25600 + 15 * 1.2 * 25600 = 844800` lovelace at default parameters), which the attacker avoids paying entirely.
