### Title
Byzantine Attested Participant Can Inject Malformed `big_r` Prefix Into Yield-Resume Buffer via `contract::respond` — (`crates/near-mpc-signature-verifier/src/lib.rs`, `crates/near-mpc-crypto-types/src/conversions/near.rs`, `crates/contract/src/lib.rs`)

### Summary

`verify_ecdsa_signature` extracts only bytes `[1..33]` of `big_r.affine_point` (the x-coordinate) and never reads `affine_point[0]` (the SEC1 compressed-point prefix). A single Byzantine attested participant can therefore submit a `K256Signature` with an arbitrary prefix byte — including `0x00` — pass signature verification, have the full malformed struct serialized into the yield-resume buffer, and cause every waiting caller to receive a `K256Signature` whose `big_r` cannot be decompressed by `TryFrom<&K256AffinePoint> for k256::AffinePoint`.

### Finding Description

**Step 1 — Prefix byte is silently discarded during verification.**

`to_ecrecover_bytes` builds the 64-byte input to `ecrecover` as:

```
bytes[..32]  = self.big_r.affine_point[1..]   // x-coordinate only
bytes[32..]  = self.s.scalar
``` [1](#0-0) 

`affine_point[0]` is never read. `verify_ecdsa_signature` passes this 64-byte slice directly to `near_sdk::env::ecrecover`, which only needs `r` (x-coordinate), `s`, and `recovery_id`. [2](#0-1) 

**Step 2 — `contract::respond` serializes the full struct including the invalid prefix.**

After `verify_ecdsa_signature` returns `Ok`, the entire `response` — including the attacker-controlled `big_r.affine_point[0]` — is serialized with `serde_json::to_vec` and written into the yield-resume buffer via `resolve_yields_for`. [3](#0-2) 

**Step 3 — `return_signature_and_clean_state_on_success` returns the malformed struct verbatim.**

The yield-resume callback deserializes the stored bytes back into a `dtos::SignatureResponse` and returns it directly to the original caller with no further validation. [4](#0-3) 

**Step 4 — Caller's decompression fails.**

Any caller that attempts to convert `big_r` back to a `k256::AffinePoint` via the standard conversion path calls `k256::AffinePoint::from_bytes`, which requires the prefix byte to be `0x02` or `0x03`. With `0x00` it returns `None` → `CryptoConversionError::InvalidPoint`. [5](#0-4) 

The honest construction path (`K256Signature::from_ecdsa_recoverable`) always writes `0x02` or `0x03` into `affine_point[0]`, so the contract implicitly relies on this invariant holding for every stored response — but it is never enforced on the incoming `respond` call. [6](#0-5) 

### Impact Explanation

The yield-resume buffer stores a `K256Signature` whose `big_r.affine_point[0]` is `0x00`. Every caller waiting on that yield receives this malformed struct. Callers that rely on decompressing `big_r` (e.g., to reconstruct the full R point for EIP-2098 compact encoding, for Schnorr adaptor protocols, or for any downstream library that calls `TryFrom<&K256AffinePoint>`) receive an unrecoverable error. The contract's response-integrity invariant — that the yield-resume buffer only ever contains well-formed, fully decompressible `K256Signature` values — is broken by a single Byzantine participant without any threshold collusion.

### Likelihood Explanation

Requires one attested participant (TEE attestation is a real barrier, but it is the normal operating state of every MPC node). The attacker only needs to participate honestly in the threshold signing round to obtain valid `r` and `s` values, then flip `affine_point[0]` before calling `respond`. No additional collusion, network access, or operator misconfiguration is needed. The attack is deterministic and repeatable.

### Recommendation

In `contract::respond`, before calling `verify_ecdsa_signature`, validate that `signature_response.big_r.affine_point[0]` is `0x02` or `0x03` and return `RespondError::InvalidSignature` otherwise. Alternatively, add this check inside `verify_ecdsa_signature` itself, or perform a full round-trip decompression (`TryFrom<&K256AffinePoint> for k256::AffinePoint`) as part of the verification step so that any invalid prefix is caught before the response is committed to the yield buffer.
