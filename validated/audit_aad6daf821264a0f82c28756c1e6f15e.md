Audit Report

## Title
Missing Staleness Check on Cross-Chain Rate Enables Over-Minting of wrsETH — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver` stores `lastUpdated` when a LayerZero message arrives but `getRate()` returns `rate` unconditionally with no freshness enforcement. `RSETHPoolV3.deposit()` consumes this rate to compute mint amounts, so any period during which the LayerZero relayer fails to deliver updates causes every depositor to receive more `wrsETH` than their ETH contribution warrants, permanently diluting the yield accrued by existing holders.

## Finding Description
`CrossChainRateReceiver` records both the rate and its timestamp on every `lzReceive` call:

```solidity
// CrossChainRateReceiver.sol L95-97
rate = _rate;
lastUpdated = block.timestamp;
```

However, `getRate()` returns the stored value with no staleness guard:

```solidity
// CrossChainRateReceiver.sol L103-105
function getRate() external view returns (uint256) {
    return rate;
}
```

`RSETHPoolV3.getRate()` delegates directly to this oracle (L235-237), and `viewSwapRsETHAmountAndFee` uses it to compute the mint amount (L304-307):

```solidity
uint256 rsETHToETHrate = getRate();
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
```

`deposit()` mints the result without any additional rate-freshness check (L258-262). The variable `lastUpdated` is written but never read anywhere in the contract or its consumers. There is no circuit-breaker that pauses deposits when the rate ages beyond a threshold.

## Impact Explanation
rsETH is a yield-bearing token whose ETH price increases monotonically. A stale rate is always lower than the true current rate. Because mint amount = `amountAfterFee * 1e18 / staleRate`, a lower stale rate produces a larger mint amount. Every depositor during the stale window receives excess `wrsETH` without a corresponding increase in backing rsETH, permanently diluting the yield that existing holders have accrued but not yet claimed. This constitutes **theft of unclaimed yield — High**.

## Likelihood Explanation
The condition requires only that the LayerZero relayer cease delivering messages, which can occur due to gas funding lapse, relayer misconfiguration, or sustained destination-chain congestion — all ordinary operational failures requiring no attacker capability. No privileged action is needed to trigger the condition. Any unprivileged depositor calling `deposit()` during the stale window exploits the mispricing. The `paused` mechanism exists but requires a `PAUSER_ROLE` holder to act manually; there is no automatic circuit-breaker. The daily mint limit (`dailyMintLimit`) caps total damage per day but does not prevent the per-deposit over-minting.

## Recommendation
Add a configurable `maxStaleness` parameter and revert in `getRate()` if the rate is too old:

```solidity
uint256 public maxStaleness; // e.g. 24 hours

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= maxStaleness, "Rate is stale");
    return rate;
}
```

`maxStaleness` should be set by the owner and should be shorter than the expected relayer update interval. This ensures deposits revert rather than proceed with a mispriced rate when the relayer is inactive.

## Proof of Concept
```solidity
// Foundry fork test on any L2 where RSETHPoolV3 is deployed
function testStaleRateOverMint() public {
    // 1. Record initial state
    uint256 R0 = receiver.rate();
    uint256 T0 = receiver.lastUpdated();

    // 2. Advance 30 days without any lzReceive call
    vm.warp(block.timestamp + 30 days);

    // 3. Confirm rate is stale and getRate() does not revert
    assertEq(receiver.lastUpdated(), T0);
    assertEq(receiver.getRate(), R0); // no revert

    // 4. Fetch true current price from L1 oracle (via fork or mock)
    uint256 trueRate = lrtOracle.rsETHPrice();
    assertGt(trueRate, R0); // price has grown

    // 5. Compute expected vs actual mint
    uint256 depositAmt = 1 ether;
    uint256 mintedActual   = depositAmt * 1e18 / R0;       // what pool mints
    uint256 mintedExpected = depositAmt * 1e18 / trueRate; // what it should mint
    assertGt(mintedActual, mintedExpected); // over-minting confirmed

    // 6. Execute deposit and verify balance
    vm.deal(attacker, depositAmt);
    vm.prank(attacker);
    pool.deposit{value: depositAmt}("");
    assertEq(wrsETH.balanceOf(attacker), mintedActual);
}
