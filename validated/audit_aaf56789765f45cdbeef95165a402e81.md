### Title
Missing Declaration-at-State-Number Check for Cairo 0 Classes in `ExecutionStateReader::get_compiled_class` Causes Wrong RPC Fee Estimation and Simulation Results - (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` applies an asymmetric guard: Cairo 1 classes fetched from the class manager are gated by `is_contract_class_declared` against the queried `state_number`, but Cairo 0 (deprecated) classes are returned unconditionally. A developer TODO comment in the code explicitly acknowledges this gap. Because the blockifier's `try_declare` function calls `state.get_compiled_class` to decide whether a class is already declared, any Cairo 0 class that exists in the class manager but was declared only in a later block will cause `starknet_estimateFee` / `starknet_simulateTransactions` to return a wrong authoritative-looking error ("class already declared") for historical-block queries.

---

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `class_manager_handle` is `Some`, the code fetches the class from the class manager and then branches on its type:

```rust
// crates/apollo_rpc_execution/src/state_reader.rs  lines 121-141
return match contract_class {
    ContractClass::V1(casm_contract_class) => {
        let is_declared = is_contract_class_declared(   // ← guard present for V1
            &self.storage_reader.begin_ro_txn()...?,
            &class_hash,
            self.state_number,
        )...?;
        if is_declared {
            Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
        } else {
            Err(StateError::UndeclaredClassHash(class_hash))
        }
    }
    // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
    ContractClass::V0(deprecated_contract_class) => {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))  // ← NO guard
    }
};
``` [1](#0-0) 

The class manager is a global store populated whenever a class is declared (regardless of block number). If class H was declared in block N+1, the class manager holds H. When the RPC executes a transaction at `state_number` N (right after block N), the `ExecutionStateReader` will return H as a valid `RunnableCompiledClass::V0` without verifying that H is actually declared at state N.

The blockifier's `try_declare` function uses exactly this call to decide whether a class is already declared:

```rust
// crates/blockifier/src/transaction/transactions.rs  lines 393-407
match state.get_compiled_class(class_hash) {
    Err(StateError::UndeclaredClassHash(_)) => { /* declare it */ Ok(()) }
    Err(error) => Err(error)?,
    Ok(_) => {
        // Class is already declared, cannot redeclare.
        Err(TransactionExecutionError::DeclareTransactionError { class_hash })
    }
}
``` [2](#0-1) 

Because the Cairo 0 path skips the state-number guard, `try_declare` receives `Ok(_)` and concludes the class is already declared, even though it is not declared at the queried block.

The RPC `estimate_fee` and `simulate_transactions` paths both create an `ExecutionStateReader` with `class_manager_handle = Some(...)` and pass it to `execute_transactions`: [3](#0-2) 

The contrast with `SyncStateReader` (used in the gateway stateful validator) is instructive: it always calls `is_class_declared_at` before returning any class, for both Cairo 0 and Cairo 1: [4](#0-3) 

The test suite for `SyncStateReader` explicitly covers the `class_not_declared_but_in_class_manager` case and expects `UndeclaredClassHash`: [5](#0-4) 

No equivalent test exists for `ExecutionStateReader` with a Cairo 0 class that is in the class manager but not declared at the queried state number.

---

### Impact Explanation

**Impact: High** — RPC execution, fee estimation, and simulation return an authoritative-looking wrong value.

Concrete corrupted value: `starknet_estimateFee` / `starknet_simulateTransactions` for a Cairo 0 declare transaction at block N returns `DeclareTransactionError { class_hash: H }` ("class already declared"), when the correct answer is that H is not declared at block N and the transaction would succeed.

---

### Likelihood Explanation

**Likelihood: Medium.**

Trigger conditions:
1. A Cairo 0 class H is declared in block N+1 (so H is in the class manager).
2. Any user calls `starknet_estimateFee` or `starknet_simulateTransactions` for a declare transaction for H at block N (any block before H's declaration block).

Both conditions are reachable by any unprivileged user without special access. Historical fee estimation and simulation are standard RPC operations. The class manager is populated as part of normal block processing, so condition 1 is satisfied automatically once any Cairo 0 class is declared.

---

### Recommendation

Apply the same `is_contract_class_declared` guard to the `ContractClass::V0` branch that is already applied to `ContractClass::V1`. The TODO comment acknowledges this is needed once `get_class_definition_block_number` is fixed; the fix for deprecated classes should use `get_deprecated_class_definition_block_number` (already available in storage, as used by `is_class_declared_at` in `StateSync`):

```rust
ContractClass::V0(deprecated_contract_class) => {
    let is_declared = /* check get_deprecated_class_definition_block_number
                         against self.state_number */;
    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
``` [6](#0-5) 

---

### Proof of Concept

1. Deploy a Cairo 0 contract class H in block N+1 (any normal deprecated declare transaction).
2. Call `starknet_estimateFee` with a `DeclareV0`/`DeclareV1` transaction for class H, specifying `block_id = N` (any block before N+1).
3. The RPC creates `ExecutionStateReader { state_number: right_after(N), class_manager_handle: Some(...) }`.
4. Execution calls `try_declare(H, ...)` → `state.get_compiled_class(H)`.
5. `ExecutionStateReader::get_compiled_class` finds H in the class manager (declared in block N+1), hits the `ContractClass::V0` branch, skips `is_contract_class_declared`, and returns `Ok(RunnableCompiledClass::V0(...))`.
6. `try_declare` receives `Ok(_)` and returns `Err(DeclareTransactionError { class_hash: H })`.
7. The RPC returns an error claiming H is already declared at block N — which is false.

The correct behavior (as implemented for Cairo 1 and in `SyncStateReader` for both class types) would return `Err(StateError::UndeclaredClassHash(H))` at step 5, causing `try_declare` to proceed with the declaration and the fee estimation to succeed.

        let is_class_declared = self
            .runtime
            .block_on(self.state_sync_client.is_class_declared_at(self.block_number, class_hash))
            .map_err(|e| StateError::StateReadError(e.to_string()))?;

                     it was declared"
                )
            });

        Ok(contract_class)
