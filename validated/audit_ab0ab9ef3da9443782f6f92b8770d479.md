### Title
Unauthenticated Remote Batcher Server Allows Pre-Registration of `proposal_id` to DoS Proposal Validation - (`File: crates/apollo_batcher/src/batcher.rs`)

### Summary

The `Batcher`'s `RemoteComponentServer` binds to `0.0.0.0` with no authentication layer. Any network-reachable process can call `validate_block` (or `start_height`) with a valid `proposal_id` and block metadata before the legitimate consensus orchestrator does. Because `set_active_proposal` enforces a uniqueness invariant on `proposal_id`, the legitimate call arrives to find the slot already occupied and returns `ProposalAlreadyExists` / `AnotherProposalInProgress`, causing `initiate_validation` to fail and the validator to drop the proposal entirely.

### Finding Description

**Root cause — no caller authentication on the remote batcher server.**

`RemoteComponentServer` is the HTTP/2 transport used in distributed deployments. Its configuration struct contains no TLS, no token, and no IP-allowlist field: [1](#0-0) 

The production distributed config binds the batcher to every interface: [2](#0-1) 

Every `BatcherRequest` variant — including `ValidateBlock` and `StartHeight` — is dispatched without any caller check: [3](#0-2) 

**State-marking invariant that can be pre-empted.**

`validate_block` calls `spawn_proposal`, which calls `set_active_proposal`. That function enforces two uniqueness guards: [4](#0-3) 

If an attacker pre-calls `validate_block` with the same `proposal_id` the orchestrator will use, the orchestrator's subsequent call hits either `ProposalAlreadyExists` (if the attacker's task already completed) or `AnotherProposalInProgress` (if it is still running).

**Attack path.**

1. Attacker observes (or predicts) the current storage height `N` — public chain state.
2. Attacker calls `start_height({height: N})` on the unauthenticated batcher port. Succeeds because `active_height` is `None` at that moment.
3. Attacker calls `validate_block({proposal_id: ProposalId(K), block_number: N, ...})`. `proposal_id` is a simple counter starting at 0; the attacker tries values 0, 1, 2, … or observes prior traffic.
4. Legitimate consensus orchestrator calls `start_height({height: N})` → `HeightInProgress` error, **or** calls `validate_block({proposal_id: ProposalId(K), ...})` → `ProposalAlreadyExists` / `AnotherProposalInProgress`.

**Propagation to consensus.**

`initiate_validation` maps any batcher error to `ValidateProposalError::Batcher`: [5](#0-4) 

`validate_proposal` returns this error, the spawned task logs `PROPOSAL_FAILED`, and the validator never sends a precommit vote for the proposal: [6](#0-5) 

### Impact Explanation

A network-reachable attacker can prevent any validator node from completing proposal validation for any round at any height. Repeated pre-registration across multiple validators prevents the network from reaching the 2/3+ precommit quorum required for a decision, stalling block production indefinitely. No funds are at risk, but liveness is fully broken.

### Likelihood Explanation

In a distributed Kubernetes deployment the batcher pod is reachable by any other pod in the cluster (absent explicit `NetworkPolicy` rules). An attacker who compromises any co-located workload, or who exploits a misconfigured ingress, gains direct access. The `proposal_id` counter starts at 0 and increments by 1, making it trivially predictable. The block height is public chain state. No secret material is required.

### Recommendation

Add mutual-TLS or a shared-secret token to `RemoteServerConfig` and verify it in `RemoteComponentServer`'s request handler before dispatching any `BatcherRequest`. As a defence-in-depth measure, `validate_block` and `start_height` should also verify that the caller identity matches the single trusted consensus orchestrator peer.

### Proof of Concept

```
# Attacker (any pod in the cluster) — batcher port is BATCHER_IP:PORT

# Step 1: learn current storage height N from public RPC
HEIGHT=N

# Step 2: pre-register the height

# Step 3: pre-register proposal_id=0 (first proposal at this height)

  "proposal_id": 0,
  "block_info": {"block_number": $HEIGHT, ...},
  "deadline": "<future timestamp>",
  "retrospective_block_hash": null

# Result: legitimate orchestrator's StartHeight → HeightInProgress

# validate_proposal returns ValidateProposalError::Batcher

    #[instrument(skip(self), err)]
    pub async fn start_height(&mut self, input: StartHeightInput) -> BatcherResult<()> {
        if self.active_height == Some(input.height) {
            return Err(BatcherError::HeightInProgress);
        }

        let storage_height = self.get_height_from_storage()?;
        if storage_height != input.height {
            return Err(BatcherError::StorageHeightMarkerMismatch {
                marker_height: storage_height,
                requested_height: input.height,
            });
        }

        self.abort_active_height().await;

        info!("Starting to work on height {}.", input.height);
        self.active_height = Some(input.height);

        Ok(())
    }

    async fn set_active_proposal(&mut self, proposal_id: ProposalId) -> BatcherResult<()> {
        if self.executed_proposals.lock().await.contains_key(&proposal_id) {
            return Err(BatcherError::ProposalAlreadyExists { proposal_id });
        }

        let mut active_proposal = self.active_proposal.lock().await;
        if let Some(active_proposal_id) = *active_proposal {
            return Err(BatcherError::AnotherProposalInProgress {
                active_proposal_id,
                new_proposal_id: proposal_id,
            });
        }

        debug!("Set proposal {} as the one being generated.", proposal_id);
        *active_proposal = Some(proposal_id);
        Ok(())
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}
