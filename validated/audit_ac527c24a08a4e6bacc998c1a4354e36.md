### Title
Unchecked Caller-Supplied `payload_hash` in `respond_verify_foreign_tx` Enables Cross-Request Signature Replay by a Single Byzantine Node - (File: `crates/contract/src/lib.rs`)

---

### Summary

`respond_verify_foreign_tx` verifies that the submitted ECDSA signature is valid over the caller-supplied `response.payload_hash`, but never verifies that `payload_hash` actually corresponds to the pending request. A single malicious attested MPC node can replay a valid threshold signature obtained from any prior legitimate signing operation to resolve a completely different pending foreign-tx request with a forged payload hash, causing the original caller to receive a response that falsely attests to a foreign transaction that was never verified.

---

### Finding Description

In `respond_verify_foreign_tx`, the signature check is:

```rust
let payload_hash: [u8; 32] = response.payload_hash.0;

// Check the signature is correct against the root public key
near_mpc_signature_verifier::verify_ecdsa_signature(
    signature_response,
    &payload_hash,
    &secp_pk,
)
.is_ok()
```

`payload_hash` is taken verbatim from the caller-supplied `response` argument. The contract never independently recomputes `SHA256(borsh(ForeignTxSignPayload { request, values }))` for the pending request and never asserts that `response.payload_hash` equals that expected value. [1](#0-0) 

Contrast this with the regular `respond` path, where the payload hash is taken from the **stored request** (submitted by the user on-chain), not from the response:

```rust
let payload_hash = request.payload.as_ecdsa().expect("Payload is not ECDSA");
near_mpc_signature_verifier::verify_ecdsa_signature(
    signature_response,
    payload_hash,
    &expected_public_key,
)
``` [2](#0-1) 

On the node side, `build_signature_request` correctly computes `payload_hash = compute_msg_hash(ForeignTxSignPayload)` and uses it as the ECDSA payload with a zero tweak (root key, no derivation):

```rust
Ok(SignatureRequest {
    payload: Payload::Ecdsa(payload_bytes),
    tweak: Tweak::new([0u8; 32]),
    ...
})
``` [3](#0-2) 

The contract's invariant check — "the MPC network signed the correct foreign-chain observation" — is therefore only half-enforced: it confirms the signature is cryptographically valid over *some* hash, but not that the hash encodes the correct `(request, extracted_values)` pair for the pending request. [4](#0-3) 

---

### Impact Explanation

A single Byzantine attested MPC node that has participated in any prior legitimate `verify_foreign_transaction` signing round for the same domain possesses a valid threshold signature `sig_A` over `hash_A = SHA256(borsh(ForeignTxSignPayload{T1, values_A}))`.

The attacker calls:

```
respond_verify_foreign_tx(
    request = <pending request B for transaction T2>,
    response = { payload_hash: hash_A, signature: sig_A }
)
```

The contract checks `ECDSA.verify(sig_A, hash_A, root_pk) == true` — which passes — and resolves request B. The original caller of `verify_foreign_transaction(T2)` receives `VerifyForeignTransactionResponse { payload_hash: hash_A, signature: sig_A }`.

Because `payload_hash` encodes `extracted_values` that the caller never observes independently, the caller cannot distinguish this from a legitimate response. A bridge contract that uses this response to authorize an inbound transfer (the primary use case per the design document) would execute based on a transaction that was never actually verified by the MPC network.

This matches: **High — forged foreign-chain verification that causes invalid bridge execution or double-spend conditions.** [5](#0-4) 

---

### Likelihood Explanation

- The attacker is a **single attested MPC participant** — strictly below the signing threshold. No collusion is required.
- The attacker only needs to have participated in **one prior legitimate signing round** for the same domain to obtain a reusable `(hash_A, sig_A)` pair.
- The call to `respond_verify_foreign_tx` is gated only by `assert_caller_is_signer()` and `assert_caller_is_attested_participant_and_protocol_active()` — both satisfied by any single active participant.
- The attack is silent: the contract emits no event distinguishing a replayed response from a legitimate one. [6](#0-5) 

---

### Recommendation

The contract must independently verify that `response.payload_hash` encodes the correct request. Since `extracted_values` are not known at request submission time, the simplest fix is to require the responder to include the full `ForeignTxSignPayload` in the response, then have the contract:

1. Recompute `expected_hash = SHA256(borsh(ForeignTxSignPayload))`.
2. Assert `expected_hash == response.payload_hash`.
3. Assert that `payload.request == pending_request.request` (the embedded request matches the on-chain pending request).

This mirrors how `respond` derives the expected payload from the stored `SignatureRequest` rather than trusting the caller.

---

### Proof of Concept

1. **Setup**: Attacker is an attested participant in a domain D MPC network (threshold = t, attacker is 1 of n nodes, n ≥ t+1).
2. **Obtain reusable signature**: A legitimate `verify_foreign_transaction` request for transaction T1 is processed. The attacker participates in the threshold signing protocol and records `hash_A = SHA256(borsh(ForeignTxSignPayload{T1, values_A}))` and the resulting threshold signature `sig_A`.
3. **Target request**: A bridge contract submits `verify_foreign_transaction(T2)` for a high-value inbound transfer. The pending request B is now in `pending_verify_foreign_tx_requests`.
4. **Replay**: Attacker calls:
   ```
   respond_verify_foreign_tx(
       request = B,   // the pending request for T2
       response = VerifyForeignTransactionResponse {
           payload_hash: hash_A,   // hash of T1's payload
           signature: sig_A,       // valid threshold sig over hash_A
       }
   )
   ```
5. **Contract accepts**: `ECDSA.verify(sig_A, hash_A, root_pk_D) == true` → `signature_is_valid = true` → `resolve_yields_for` resolves request B.
6. **Bridge executes**: The bridge contract receives `{payload_hash: hash_A, signature: sig_A}`, verifies the signature (valid), and releases funds — for a transaction T2 that was never verified by the MPC network. [7](#0-6) [3](#0-2)

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

fn build_signature_request(
    request: &VerifyForeignTxRequest,
    foreign_tx_payload: &dtos::ForeignTxSignPayload,
) -> anyhow::Result<SignatureRequest> {
    let payload_hash: [u8; ECDSA_PAYLOAD_SIZE_BYTES] =
        foreign_tx_payload.compute_msg_hash()?.into();
    let payload_bytes: BoundedVec<u8, ECDSA_PAYLOAD_SIZE_BYTES, ECDSA_PAYLOAD_SIZE_BYTES> =
        payload_hash.into();

    Ok(SignatureRequest {
        id: request.id,
        receipt_id: request.receipt_id,
        payload: Payload::Ecdsa(payload_bytes),
        tweak: Tweak::new([0u8; 32]),
        entropy: request.entropy,
        timestamp_nanosec: request.timestamp_nanosec,
        domain: request.domain_id,
    })
}

impl ForeignTxSignPayload {
    pub fn compute_msg_hash(&self) -> std::io::Result<Hash256> {
        let mut hasher = sha2::Sha256::new();
        borsh::BorshSerialize::serialize(self, &mut hasher)?;
        Ok(Hash256(hasher.finalize().into()))
    }
