Audit Report

## Title
Withdrawal Completion Blocked When Protocol Is Paused, Temporarily Freezing User rsETH - (File: contracts/LRTWithdrawalManager.sol)

## Summary
`initiateWithdrawal()` irrevocably transfers a user's rsETH into `LRTWithdrawalManager` at call time. Every subsequent recovery path — `completeWithdrawal()`, `completeWithdrawalForUser()`, and `instantWithdrawal()` — is gated by `whenNotPaused`. The contract can be paused by any unprivileged caller via the public `updateRSETHPrice()` function whenever the computed rsETH price falls more than `pricePercentageLimit` below `highestRsethPrice`, causing all in-flight withdrawal requests to be frozen for the duration of the pause.

## Finding Description
`initiateWithdrawal()` calls `IERC20(lrtConfig.rsETH()).safeTransferFrom(msg.sender, address(this), rsETHUnstaked)` (line 166), permanently removing rsETH from the user's wallet and placing it under contract custody. After this point the user has no rsETH and no alternative recovery mechanism.

The only paths to recover value are:
- `completeWithdrawal()` — `whenNotPaused` (line 183)
- `completeWithdrawalForUser()` — `whenNotPaused` (line 199)
- `instantWithdrawal()` — `whenNotPaused` (line 219)
- `unlockQueue()` — `whenNotPaused` (line 279), so even the operator unlock step is blocked

The auto-pause is triggered inside `_updateRsETHPrice()` (lines 277–281 of `LRTOracle.sol`):
```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```
`updateRSETHPrice()` is declared `public` with no access control beyond `whenNotPaused` on the oracle itself (line 87 of `LRTOracle.sol`), so any EOA or contract can call it. If the current rsETH price is more than `pricePercentageLimit` below `highestRsethPrice`, the call pauses `LRTWithdrawalManager` unconditionally. Once paused, every withdrawal completion reverts, and only an admin calling `unpause()` can restore access.

## Impact Explanation
Users who have called `initiateWithdrawal()` have irrevocably surrendered their rsETH to the contract. While the contract is paused they cannot execute `completeWithdrawal()`, `completeWithdrawalForUser()`, or `instantWithdrawal()` to recover any value. Their funds are frozen for the entire duration of the pause. This is a concrete **temporary freezing of funds**, matching the Medium allowed impact.

## Likelihood Explanation
The trigger is a public, permissionless function (`updateRSETHPrice()`). Any unprivileged caller can invoke it at any time. A price drop beyond `pricePercentageLimit` — caused by EigenLayer validator slashing, a large redemption reducing TVL, or any other event that lowers the protocol's total ETH — is sufficient to trigger the auto-pause. The manual pause path via `PAUSER_ROLE` is an additional vector. Both paths are realistic during normal protocol operation (maintenance windows, security incidents, market stress). The condition is repeatable and does not require victim mistakes or privileged collusion.

## Recommendation
Remove `whenNotPaused` from `completeWithdrawal()`, `completeWithdrawalForUser()`, and optionally `instantWithdrawal()`, since these functions only release assets already held in custody — they do not accept new deposits or introduce new protocol risk. Alternatively, introduce a separate, narrower pause flag (e.g., `depositsAndInitiationsPaused`) that blocks `initiateWithdrawal()` and new deposits but leaves completion of already-queued withdrawals unaffected. This preserves the downside-protection intent of the oracle auto-pause while ensuring users can always recover committed funds.

## Proof of Concept
1. Alice calls `initiateWithdrawal(asset, rsETHAmount, referralId)`. Her rsETH is transferred to `LRTWithdrawalManager` (line 166). A `WithdrawalRequest` is recorded.
2. Any external caller (Bob) calls `LRTOracle.updateRSETHPrice()`. The computed rsETH price is more than `pricePercentageLimit` below `highestRsethPrice`. `_updateRsETHPrice()` calls `withdrawalManager.pause()` (line 279 of `LRTOracle.sol`).
3. Alice calls `completeWithdrawal(asset, referralId)`. The call reverts at the `whenNotPaused` modifier (line 183 of `LRTWithdrawalManager.sol`).
4. Alice tries `instantWithdrawal()` — reverts at `whenNotPaused` (line 219).
5. An operator tries `completeWithdrawalForUser()` — reverts at `whenNotPaused` (line 199).
6. Alice's rsETH remains locked in `LRTWithdrawalManager` until an admin calls `unpause()`.

**Foundry test sketch:**
