### Title
Missing Vote Signature Verification Allows Unauthenticated Gossipsub Peer to Forge Validator Votes and Corrupt Consensus — (`crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`handle_vote` in `SingleHeightConsensus` contains an explicit `// TODO(Asmaa): verify the signature` comment and performs **zero cryptographic verification** before counting a received `Vote` toward quorum. Any unauthenticated gossipsub peer that knows a validator's `ContractAddress` (public on-chain information) can broadcast `Vote` messages with that address as `voter` and an arbitrary `signature` field. The vote passes all existing guards and is counted toward the prevote/precommit quorum, enabling a single attacker to forge a full quorum from N distinct validator addresses.

---

### Finding Description

**Entry point — gossipsub Vote message:**

A `Vote` is deserialized from protobuf with no signature check at the wire layer. [1](#0-0) 

The `signature` field is accepted as-is (defaulting to empty if absent): [2](#0-1) 

**Vote handler — signature verification explicitly absent:**

`handle_vote` performs only three checks before accepting the vote:
1. Height equality
2. Committee membership (`vote.voter` is a known validator address)
3. Duplicate/conflict detection keyed by `(round, voter)`

The signature is never verified: [3](#0-2) 

The TODO is explicit: [4](#0-3) 

**Self-votes are also unsigned:**

The node's own votes are created with `RawSignature::default()`, confirming the signing infrastructure is not yet wired: [5](#0-4) 

**Verification functions exist but are never called in the vote path:**

`verify_precommit_vote_signature` is implemented in `signature_manager` but is not invoked anywhere in the consensus vote handling pipeline: [6](#0-5) 

**Quorum counting proceeds on forged votes:**

Once a vote passes the committee-membership check, it is inserted into `prevotes`/`precommits` maps and its weight is accumulated toward the quorum threshold: [7](#0-6) 

A decision is emitted when precommit weight exceeds the quorum threshold and a matching proposal is stored: [8](#0-7) 

---

### Impact Explanation

An unauthenticated gossipsub peer who knows the validator set (public on-chain data) can:

1. **Forge a prevote quorum** — send `Vote{vote_type: Prevote, voter: <validator_i>, proposal_commitment: <attacker_chosen_hash>, signature: <garbage>}` for each of N validators. The node locks on the attacker-chosen `proposal_commitment` and broadcasts a precommit for it.
2. **Forge a precommit quorum** — repeat for `Precommit` votes. Combined with a matching proposal injected via the proposal stream (a separate but equally unauthenticated path), the node emits `DecisionReached` for an arbitrary `proposal_commitment`, corrupting the decided block hash for height H.

Even without a matching proposal, forging a prevote quorum causes the node to lock and precommit for an unvalidated value, breaking consensus liveness and potentially safety across the network.

---

### Likelihood Explanation

- Validator `ContractAddress` values are public (on-chain staking registry).
- Gossipsub peers are unauthenticated at the vote-content level; any peer can send any `Vote` message.
- The exploit requires only crafting valid protobuf bytes with a known address and any signature bytes — no cryptographic secret is needed.
- The TODO comment confirms this is a known, unimplemented control, not a design choice.

---

### Recommendation

1. Implement signature verification inside `handle_vote` before the committee-membership check. Reject any vote whose `signature` does not verify against the `voter`'s registered public key for the serialized vote payload.
2. Wire `verify_precommit_vote_signature` (already implemented in `apollo_signature_manager`) into the vote handling path, and add an equivalent for prevotes.
3. Ensure `make_self_vote` signs the vote before broadcasting, so the signing/verification round-trip can be tested end-to-end.
4. Add a protocol-state test that injects `Vote` messages with valid `voter` addresses but invalid signatures and asserts no quorum is formed and no `DecisionReached` is emitted.

---

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
