### Title
Missing Vote Signature Verification Allows Any Peer to Impersonate Validators and Corrupt Consensus — (`crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` contains an explicit `// TODO(Asmaa): verify the signature` comment and performs **no cryptographic verification** of the `Vote.signature` field. Combined with the protobuf converter silently defaulting a missing `signature` to `RawSignature::default()` (empty bytes), any unauthenticated gossipsub peer can craft a `Vote` claiming to be any committee validator and have it counted toward quorum.

---

### Finding Description

**Step 1 — Protobuf deserialization silently accepts empty signatures.**

In `crates/apollo_protobuf/src/converters/consensus.rs` lines 86–88, when `protobuf::Vote.signature` is `None`, the converter substitutes `RawSignature::default()` (an empty byte vector) without returning an error:

```rust
// Convert Hashes to RawSignature (default to empty if None)
let signature =
    value.signature.map(|hashes| hashes.try_into()).transpose()?.unwrap_or_default();
``` [1](#0-0) 

**Step 2 — Gossipsub transport authenticates the libp2p peer, not the consensus voter.**

`MixedBehaviour` configures gossipsub with `MessageAuthenticity::Signed(keypair)`. This signs the gossipsub *envelope* with the libp2p node key, proving which peer sent the message. It does **not** bind the `voter: ContractAddress` field inside the `Vote` protobuf to any key. [2](#0-1) 

**Step 3 — `handle_gossipsub_behaviour_event` forwards raw bytes without inspection.**

The network manager simply routes the raw bytes to `broadcasted_messages_senders` with no content-level validation. [3](#0-2) 

**Step 4 — `SingleHeightConsensus::handle_vote` explicitly skips signature verification.**

The only guards are: correct height, `vote.voter` present in the committee list (by `ContractAddress` value alone), and duplicate/conflict detection. The signature is never checked — the TODO comment confirms this is a known gap:

```rust
pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
    // TODO(Asmaa): verify the signature
    ...
    if !self.committee.members().iter().any(|s| s.address == vote.voter) {
        debug!("Ignoring vote from non validator: vote={:?}", vote);
        return VecDeque::new();
    }
    // No signature check before accepting
    self.state_machine.handle_event(sm_vote)
}
``` [4](#0-3) 

**Step 5 — `verify_precommit_vote_signature` exists but is never called in the consensus path.**

The function is implemented in `crates/apollo_signature_manager/src/signature_manager.rs` but a `grep` across all of `crates/apollo_consensus/` returns zero matches for any signature verification call. [5](#0-4) 

**Step 6 — Even self-votes use empty signatures**, confirming the entire vote-signing subsystem is unimplemented:

```rust
// TODO(Asmaa): sign the vote
signature: RawSignature::default(),
``` [6](#0-5) 

---

### Impact Explanation

An attacker who can connect as any libp2p peer (no validator key required) can:

1. Enumerate the current committee's `ContractAddress` values (published in committee data).
2. Serialize a `Vote` protobuf with `voter = <target_validator_address>`, `signature = None` (or any bytes), and the correct `height`/`round`/`proposal_commitment`.
3. Publish it over gossipsub signed with their own libp2p key (satisfying `MessageAuthenticity::Signed`).
4. The receiving node's `handle_vote` accepts it, inserts it into the state machine's vote map, and counts it toward quorum.

With `f+1` injected votes (Byzantine quorum threshold), an attacker can force a quorum for an arbitrary `ProposalCommitment`, causing `DecisionReached` to fire for a block the honest validators did not actually agree on. This corrupts the decided block and all downstream state derived from it.

---

### Likelihood Explanation

- The attack requires only a standard libp2p connection to the gossipsub mesh — no validator key, no privileged access.
- The committee addresses are not secret.
- The duplicate-vote guard only blocks a second vote for the same `(round, voter)` pair; a first injected vote for any silent validator passes immediately.
- The TODO comment confirms the gap is known and unresolved.

---

### Recommendation

1. **Implement signature verification in `SingleHeightConsensus::handle_vote`** before the vote is passed to the state machine. Call `verify_precommit_vote_signature` (or the equivalent prevote verifier) using the voter's public key from the committee record.
2. **Reject votes with empty/default signatures** at the protobuf conversion boundary, or at the earliest point in `handle_vote`, rather than silently accepting them.
3. **Sign self-votes** in `StateMachine::make_self_vote` using the node's `SignatureManager` before broadcasting.
4. Consider adding a gossipsub message validation callback that rejects messages whose `Vote.voter` does not match the originating libp2p peer's authenticated consensus identity.

    fn handle_gossipsub_behaviour_event(
        &mut self,
        event: gossipsub_impl::ExternalEvent,
    ) -> Result<(), NetworkError> {
        let gossipsub_impl::ExternalEvent::Received { originated_peer_id, message, topic_hash } =
            event;

        let message_size = message.len();
        self.update_broadcast_metric(&topic_hash, |broadcast_metrics| {
            broadcast_metrics.received_broadcast_message_metrics.record_message(message_size);
        });

        trace!("Received broadcast message with topic hash: {topic_hash:?}");
        let broadcasted_message_metadata = BroadcastedMessageMetadata {
            originator_id: OpaquePeerId::private_new(originated_peer_id),
            encoded_message_length: message_size,
        };
        let Some(sender) = self.broadcasted_messages_senders.get_mut(&topic_hash) else {
            panic!(
                "Received a message from a topic we're not subscribed to with hash {topic_hash:?}"
            );
        };
        let send_result = sender.try_send((message, broadcasted_message_metadata));
        if let Err(e) = send_result {
            if e.is_disconnected() {
                return Err(NetworkError::BroadcastChannelsDropped { topic_hash });
            } else if e.is_full() {
                warn!(
                    "Receiver buffer is full. Dropping broadcasted message for topic with hash: \
                     {topic_hash:?}."
                );
            }
        }
        Ok(())
    }

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
