Audit Report

## Title
Pusher Delegation Signature Replay in `allowPushers` Enables Forced Re-Delegation After Revocation, Causing Oracle Stale-Price DoS on Pool Swaps — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

## Summary

`CompressedOracleV1.allowPushers` constructs its EIP-191 signed digest from `(chainid, address(this), deadline, pusher, msg.sender)` with no nonce and no used-signature tracking. A creator who retains a pusher's signed consent can replay the identical signature to re-establish `namespaceRemapping[pusher] = creator` immediately after the pusher calls `revokePusher()`, as long as `block.timestamp <= deadline`. This permanently redirects the pusher's price updates away from their own namespace until the deadline expires, starving any `PriceProvider`/pool depending on that namespace and causing every `swap()` call to revert with `BidIsZero`.

## Finding Description

`allowPushers` at lines 204–209 of `CompressedOracle.sol` verifies the pusher's EIP-191 signature and writes `namespaceRemapping[pusher] = msg.sender` with no nonce and no record that the signature was consumed:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;   // overwrite, not increment
```

`revokePusher()` at lines 238–243 sets `namespaceRemapping[msg.sender] = address(0)`, restoring the pusher's own namespace. Because the signed tuple is identical to the one already verified, the creator can immediately resubmit the same `(deadline, pusher, sig)` calldata to `allowPushers`, which passes signature verification again and overwrites the mapping back to `creator`. This cycle repeats indefinitely until `block.timestamp > deadline`.

The `fallback` push path at lines 315–316 reads `namespaceRemapping[msg.sender]` on every call:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

While the delegation is active, every push the pusher makes lands in the creator's namespace, not the pusher's own namespace, so the pusher's namespace `refTime` never advances.

The code's own NatDoc comment (lines 186–191) acknowledges the replay concern but incorrectly claims the deadline alone is sufficient protection — the deadline only prevents replay *after* it expires, not within the window.

## Impact Explanation

The price chain is `MetricOmmPool.swap()` → `_getBidAndAskPriceX64()` → `PriceProvider.getBidAndAskPrice()` → `CompressedOracleV1.price(feedId, pool)`, which reads the pusher's namespace. When that namespace receives no updates, `refTime` becomes stale. `PriceProvider._getBidAndAskPrice()` (lines 197–199) returns `(0, type(uint128).max)`:

```solidity
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
    return (0, type(uint128).max);
}
```

`MetricOmmPool._getBidAndAskPriceX64()` (lines 806–808) then reverts with `BidIsZero`:

```solidity
if (bid == 0) revert BidIsZero();
```

Every `swap()` call reverts for the entire remaining deadline window — broken core pool functionality / unusable swap flow, which is within the allowed impact gate.

## Likelihood Explanation

- Any pusher who previously signed a delegation with a future deadline is at risk.
- The creator needs only to retain the original calldata and resubmit it after `revokePusher()`.
- `allowPushers` is permissionless; no privileged role is required.
- Business relationships that end before the deadline expires are a realistic trigger.
- The attack is repeatable on every `revokePusher()` call until the deadline passes.

## Recommendation

Add a per-pusher nonce included in the signed digest, invalidating any prior signature on use:

```solidity
mapping(address => uint256) public pusherNonces;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonces[pusher]++
    ))
);
```

Alternatively, record used signature hashes in a `mapping(bytes32 => bool) public usedSignatures` and revert on reuse.

## Proof of Concept

1. **Setup**: Pusher A operates a namespace; a pool's `PriceProvider` points to `feedIdOf(pusherA, slot, pos)`.
2. **Delegation**: Pusher A signs `keccak256(abi.encode(chainid, oracle, deadline=T, pusherA, creatorB))` and hands the signature to Creator B.
3. **First call**: Creator B calls `allowPushers(T, [pusherA], [sig])` → `namespaceRemapping[pusherA] = creatorB`. Pusher A's pushes land in Creator B's namespace.
4. **Revocation**: Pusher A calls `revokePusher()` → `namespaceRemapping[pusherA] = address(0)`. Pusher A's pushes return to their own namespace.
5. **Replay** (while `block.timestamp <= T`): Creator B calls `allowPushers(T, [pusherA], [sig])` with the identical signature → `namespaceRemapping[pusherA] = creatorB` again. Steps 4–5 repeat indefinitely.
6. **Oracle starvation**: Pusher A's namespace receives no updates; `refTime` goes stale.
7. **Pool DoS**: `PriceProvider._getBidAndAskPrice()` returns `(0, type(uint128).max)`; `MetricOmmPool.swap()` reverts with `BidIsZero` on every call until deadline T expires.
