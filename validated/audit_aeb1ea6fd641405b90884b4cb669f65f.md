Audit Report

## Title
Fee Baseline Uses `rsETHPrice` Instead of `highestRsethPrice`, Charging Fees on Slashing Recovery — (File: `contracts/LRTOracle.sol`)

## Summary
`_updateRsETHPrice()` computes `previousTVL` using `rsETHPrice` (the last stored price) rather than `highestRsethPrice` (the all-time high). When a slashing event causes a price drop that falls within `pricePercentageLimit` (so the protocol is not paused and `rsETHPrice` is updated to the lower value), any subsequent price recovery is treated as new yield and subjected to protocol fees — even though the price has not yet returned to its previous peak. rsETH holders who already absorbed the slashing loss are further penalized by having their recovery gains partially extracted as protocol fees.

## Finding Description
In `_updateRsETHPrice()` at line 234, `previousTVL` is computed as:

```solidity
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);
```

The fee is then charged at lines 244–247 whenever `totalETHInProtocol > previousTVL`.

The downside protection at lines 270–282 only pauses the protocol when the price drop **exceeds** `pricePercentageLimit`. When the drop is **within** the limit (or `pricePercentageLimit == 0`), the function does not return early, and `rsETHPrice` is updated to the new lower value at line 313. `highestRsethPrice` is never lowered — it only increases (line 294).

This creates the following exploit path:

1. `rsETHPrice = highestRsethPrice = P_high` (e.g., 1.1e18).
2. Slashing event: `totalETHInProtocol` drops; `updateRSETHPrice()` is called; `newRsETHPrice = P_low` (e.g., 1.0e18). The drop is within `pricePercentageLimit`, so no pause occurs. `rsETHPrice` is stored as `P_low`; `highestRsethPrice` remains `P_high`.
3. Yield accrues; `totalETHInProtocol` rises to `rsethSupply × P_mid` where `P_low < P_mid < P_high` (e.g., 1.05e18).
4. Any caller invokes `updateRSETHPrice()` (public, no access control):
   - `previousTVL = rsethSupply × P_low`
   - `rewardAmount = rsethSupply × (P_mid − P_low)` — treated as new yield
   - `protocolFeeInETH = rewardAmount × feeBPS / 10_000`
   - Fee rsETH is minted to treasury at line 306.
5. With `highestRsethPrice = P_high` as the correct baseline: `previousTVL = rsethSupply × P_high > totalETHInProtocol`, so `rewardAmount = 0` and no fee should be taken.

`highestRsethPrice` is already tracked and updated correctly (lines 293–296) but is never consulted when deciding whether a fee is owed — an inconsistency that is the root cause.

## Impact Explanation
rsETH holders who suffered a slashing loss have their recovery gains partially extracted as protocol fees. With a 10% fee rate: price drops 1.1 → 1.0 (slashing), recovers to 1.05; `rewardAmount = rsethSupply × 0.05 ETH`; `protocolFeeInETH = rsethSupply × 0.005 ETH` minted to treasury. Holders who already lost 0.1 ETH/rsETH to slashing now lose an additional fee on their 0.05 ETH/rsETH recovery and can never fully recover their slashing loss through normal yield accrual.

**Impact: High — Theft of unclaimed yield.**

## Likelihood Explanation
EigenLayer slashing is a realistic and expected event for a restaking protocol. The protocol already tracks `highestRsethPrice` specifically to handle price-drop scenarios, confirming the scenario is anticipated. The bug is triggered by any slashing event where the price drop falls within `pricePercentageLimit` (or when `pricePercentageLimit == 0`, which disables the guard entirely). `updateRSETHPrice()` is public and callable by any address with no privileged access required. No special conditions beyond a slashing event followed by partial recovery are needed.

**Likelihood: Medium.**

## Recommendation
Replace `rsETHPrice` with `highestRsethPrice` as the fee baseline so that fees are only charged on genuine new yield above the previous peak:

```solidity
// Before (buggy):
uint256 previousTVL = rsethSupply.mulWad(rsETHPrice);

// After (fixed):
uint256 previousTVL = rsethSupply.mulWad(highestRsethPrice);
```

Since `highestRsethPrice >= rsETHPrice` is invariant after initialization (it is only ever increased at line 294), this change ensures `rewardAmount > 0` only when the protocol has genuinely surpassed its previous all-time high TVL.

## Proof of Concept
Minimal Foundry test sequence (fork or local):

1. Deploy protocol; set `rsETHPrice = highestRsethPrice = 1.1e18`; set `pricePercentageLimit = 0` (or any value ≥ the intended drop percentage).
2. Simulate slashing: reduce `totalETHInProtocol` so that `newRsETHPrice = 1.0e18`. Call `updateRSETHPrice()`. Assert `rsETHPrice == 1.0e18` and `highestRsethPrice == 1.1e18`.
3. Simulate yield accrual: increase `totalETHInProtocol` to `rsethSupply × 1.05e18`. Call `updateRSETHPrice()` from an unprivileged address.
4. Assert that `protocolFeeInETH > 0` and that treasury rsETH balance increased — confirming fees were charged on recovery.
5. Replay step 3 with the fix (`previousTVL = rsethSupply.mulWad(highestRsethPrice)`): assert `protocolFeeInETH == 0` because `totalETHInProtocol < previousTVL`.
