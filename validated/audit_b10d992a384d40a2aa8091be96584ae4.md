### Title
Permissionless Absorb Spend Resets `ASSERT_HEIGHT_RELATIVE` Countdown on LEAVING_POOL Singleton, Permanently Blocking Pool Exit - (File: `chia/pools/pool_puzzles.py`)

### Summary
The pool singleton's absorb spend path requires **no signature** and can be submitted by any unprivileged party for any pool singleton, including one in `LEAVING_POOL` (waiting room) state. Because an absorb spend spends the current singleton coin and recreates it at the current block height, it resets the `ASSERT_HEIGHT_RELATIVE` countdown that guards the pool-exit travel spend. An attacker who monitors the chain can repeatedly trigger absorb spends on a victim's LEAVING_POOL singleton whenever a farming reward coin appears at the victim's `p2_singleton_puzzle_hash`, permanently preventing the victim from completing their pool exit.

### Finding Description

The `create_absorb_spend` function in `chia/pools/pool_puzzles.py` constructs a spend bundle that requires an **empty BLS signature** (`G2Element()`): [1](#0-0) 

For a singleton in `LEAVING_POOL` (waiting room) state, the absorb inner solution is: [2](#0-1) 

This is confirmed by the lifecycle test, which submits the absorb with `G2Element()` (no signature) while the singleton is in the waiting room: [3](#0-2) 

The waiting room inner puzzle (`POOL_WAITINGROOM_INNERPUZ`) enforces `ASSERT_HEIGHT_RELATIVE relative_lock_height` only on the **escape/travel** spend path (spend_type=1), not on the absorb path (spend_type=0). Because a singleton always spends its current coin and recreates a new one, the absorb spend at block H' creates a new singleton coin whose creation height is H'. Any subsequent escape spend must satisfy `ASSERT_HEIGHT_RELATIVE relative_lock_height` measured from H', effectively resetting the countdown.

The `relative_lock_height` is enforced by the waiting room puzzle: [4](#0-3) 

And the escape attempt fails with `ASSERT_HEIGHT_RELATIVE_FAILED` if the countdown has not elapsed: [5](#0-4) 

The maximum allowed `relative_lock_height` is 1000 blocks: [6](#0-5) 

### Impact Explanation

**High** — An unprivileged attacker can permanently prevent a farmer from completing their pool exit (transitioning from `LEAVING_POOL` back to `SELF_POOLING`). The attacker needs only to:

1. Observe the victim's singleton in `LEAVING_POOL` state on-chain (all singleton state is public).
2. Watch for farming reward coins appearing at the victim's `p2_singleton_puzzle_hash`.
3. Submit a valid absorb spend bundle (no signature required, zero fee) for each reward coin.

Each absorb resets the `ASSERT_HEIGHT_RELATIVE` countdown. As long as the victim continues farming (producing reward coins at their `p2_singleton_puzzle_hash`), the attacker can sustain the DoS indefinitely at negligible cost. The victim's funds are not stolen, but their ability to exit the pool — a protected singleton state transition — is permanently blocked. This matches the allowed High impact: *"Permanent or long-lived inability for honest farmers to process valid pool actions"* and *"Corruption of pool membership state with direct security impact"*.

### Likelihood Explanation

The attack is cheap (zero-fee spend bundles), requires no privileged access, and all necessary information (singleton state, `p2_singleton_puzzle_hash`, reward coin IDs) is publicly visible on-chain. Any farmer actively trying to leave a pool is a viable target. The only prerequisite is that the victim continues to farm blocks, which is the normal operating condition.

### Recommendation

The absorb spend path in the waiting room (`POOL_WAITINGROOM_INNERPUZ`) should be modified so that an absorb spend does **not** spend and recreate the singleton coin, or alternatively, the `ASSERT_HEIGHT_RELATIVE` countdown should be anchored to the block height at which the singleton first entered the `LEAVING_POOL` state (stored in the puzzle or solution), rather than to the most recent coin creation height. A simpler mitigation is to disallow absorb spends entirely while the singleton is in `LEAVING_POOL` state, since the pool is no longer entitled to those rewards once the farmer has declared intent to leave.

### Proof of Concept

```
1. Alice's pool singleton enters LEAVING_POOL state at block H.
   relative_lock_height = 1000 (maximum allowed).
   Alice must wait until block H + 1000 to complete her exit.

2. At block H + 500, Alice farms a block.
   A reward coin appears at her p2_singleton_puzzle_hash.

3. Bob (attacker) calls create_absorb_spend() with Alice's singleton state,
   submits SpendBundle(coin_spends, G2Element()) — no signature, zero fee.
   This is accepted by the mempool and included in block H + 501.

4. Alice's singleton coin is now recreated at block H + 501.
   Her ASSERT_HEIGHT_RELATIVE countdown resets: she must now wait until
   block H + 501 + 1000 = H + 1501.

5. Bob repeats step 3 every time Alice farms a reward coin.
   Alice can never reach the required relative height to escape.
```

The absorb-while-in-waiting-room path is exercised without any signature in the existing test suite: [3](#0-2)

def create_absorb_spend(
    last_coin_spend: CoinSpend,
    current_state: PoolState,
    launcher_coin: Coin,
    height: uint32,
    genesis_challenge: bytes32,
    delay_time: uint64,
    delay_ph: bytes32,
) -> list[CoinSpend]:
    inner_puzzle: Program = pool_state_to_inner_puzzle(
        current_state, launcher_coin.name(), genesis_challenge, delay_time, delay_ph
    )
    reward_amount: uint64 = calculate_pool_reward(height)
    if is_pool_member_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol: Program = Program.to([reward_amount, height])
    elif is_pool_waitingroom_inner_puzzle(inner_puzzle):
        # inner sol is (spend_type, destination_puzhash, pool_reward_amount, pool_reward_height, extra_data)
        inner_sol = Program.to([0, reward_amount, height])
    else:
        raise ValueError
    # full sol = (parent_info, my_amount, inner_solution)
    coin: Coin | None = get_most_recent_singleton_coin_from_coin_spend(last_coin_spend)
    assert coin is not None

    if coin.parent_coin_info == launcher_coin.name():
        parent_info: Program = Program.to([launcher_coin.parent_coin_info, launcher_coin.amount])
    else:
        p = Program.from_bytes(bytes(last_coin_spend.puzzle_reveal))
        last_coin_spend_inner_puzzle: Program | None = get_inner_puzzle_from_puzzle(p)
        assert last_coin_spend_inner_puzzle is not None
        parent_info = Program.to(
            [
                last_coin_spend.coin.parent_coin_info,
                last_coin_spend_inner_puzzle.get_tree_hash(),
                last_coin_spend.coin.amount,
            ]
        )
    full_solution: SerializedProgram = SerializedProgram.to([parent_info, last_coin_spend.coin.amount, inner_sol])
    full_puzzle: SerializedProgram = create_full_puzzle(inner_puzzle, launcher_coin.name()).to_serialized()
    assert coin.puzzle_hash == full_puzzle.get_tree_hash()

    reward_parent: bytes32 = pool_parent_id(height, genesis_challenge)
    p2_singleton_puzzle = create_p2_singleton_puzzle(
        SINGLETON_MOD_HASH, launcher_coin.name(), delay_time, delay_ph
    ).to_serialized()
    reward_coin: Coin = Coin(reward_parent, p2_singleton_puzzle.get_tree_hash(), reward_amount)
    p2_singleton_solution = SerializedProgram.to([inner_puzzle.get_tree_hash(), reward_coin.name()])
    assert p2_singleton_puzzle.get_tree_hash() == reward_coin.puzzle_hash
    assert full_puzzle.get_tree_hash() == coin.puzzle_hash
    assert get_inner_puzzle_from_puzzle(Program.from_bytes(bytes(full_puzzle))) is not None

    coin_spends = [
        CoinSpend(coin, full_puzzle, full_solution),
        CoinSpend(reward_coin, p2_singleton_puzzle, p2_singleton_solution),
    ]
    return coin_spends
