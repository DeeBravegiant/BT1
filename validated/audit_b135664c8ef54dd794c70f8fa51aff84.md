### Title
Cross-Network Replay of `DelegateAction` Signatures Due to Missing Chain/Network Domain Separation - (File: core/primitives/src/action/delegate.rs)

### Summary
`DelegateAction` (meta transaction) signatures do not include any chain or network identifier in the signed payload. A `SignedDelegateAction` created and broadcast on testnet can be replayed verbatim on mainnet by any observer acting as a relayer, executing the user's intended testnet actions against their real mainnet account.

### Finding Description
`DelegateAction::get_nep461_hash()` computes the signing hash as:

```rust
let signable = SignableMessage::new(&self, SignableMessageType::DelegateAction);
let bytes = borsh::to_vec(&signable).expect("Failed to deserialize");
hash(&bytes)
```

`SignableMessage` prepends only a constant NEP discriminant (`2^30 + 366`) to the borsh-serialized action body. The `DelegateAction` struct itself contains `sender_id`, `receiver_id`, `actions`, `nonce`, `max_block_height`, and `public_key` — but **no `block_hash` and no chain/network identifier**. [1](#0-0) [2](#0-1) 

This contrasts with regular `SignedTransaction`, which includes a `block_hash` that is chain-specific (a block hash from mainnet does not exist on testnet, so the transaction fails the validity-period check). `DelegateAction` has no such anchor. Its only time-bound is `max_block_height`, which is a raw block height integer shared across all NEAR networks — mainnet and testnet have been running since 2020 and are at comparable heights, so a `max_block_height` valid on testnet is also valid on mainnet. [3](#0-2) 

The same issue applies to `DelegateActionV2` / `VersionedDelegateActionPayload::get_nep461_hash()`: [4](#0-3) 

### Impact Explanation
An attacker who observes a `SignedDelegateAction` on testnet (visible in any testnet block) can:
1. Extract the `SignedDelegateAction` bytes.
2. Wrap them in a new mainnet `SignedTransaction` (the attacker acts as the relayer — no special privilege required).
3. Submit to mainnet via public RPC.

On mainnet, `apply_delegate_action` verifies the signature against `get_nep461_hash()`, which produces the **identical hash** as on testnet (same discriminant, same payload, no chain ID). If the user's mainnet access key nonce is below the delegate action's nonce and `max_block_height` is still valid, the action executes. The corrupted protocol values are: the user's mainnet **account balance** (if the inner action is a transfer), **access key state** (if the inner action adds/removes keys), or **contract storage** (if the inner action is a function call). [5](#0-4) 

### Likelihood Explanation
- NEAR account IDs are identical across mainnet and testnet; users routinely operate the same named account on both networks.
- Developers and power users frequently reuse the same ED25519 key pair on testnet and mainnet.
- Testnet `DelegateAction` transactions are publicly visible on-chain; any observer can extract the `SignedDelegateAction`.
- Mainnet and testnet block heights are in the same order of magnitude, so a `max_block_height` set to `current_testnet_height + 100` is typically also valid on mainnet.
- The attacker needs no validator, relayer-operator, or admin privilege — only the ability to submit a transaction via public RPC.

### Recommendation
Include a chain/network identifier in the `DelegateAction` signed payload. The simplest approach is to add a `chain_id` field (matching the genesis `chain_id` string, e.g., `"mainnet"`, `"testnet"`) to `DelegateAction` and `DelegateActionV2`, and incorporate it into `get_nep461_hash()`. Alternatively, encode the chain ID into the `MessageDiscriminant` or into `SignableMessage` so that signatures produced on one network are cryptographically invalid on any other. [6](#0-5) 

### Proof of Concept
1. Alice signs a `DelegateAction` on testnet to transfer 10 NEAR to Bob:
   ```
   DelegateAction { sender_id: "alice.near", receiver_id: "alice.near",
     actions: [Transfer(10 NEAR)], nonce: 5, max_block_height: 150_000_000,
     public_key: <alice_key> }
   ```
   Hash = `SHA256(discriminant_366 || borsh(action))` — identical on any NEAR network.

2. Attacker observes the `SignedDelegateAction` on testnet (public block data).

3. Attacker constructs a mainnet `SignedTransaction`:
   ```
   signer_id: attacker.near, receiver_id: alice.near,
   actions: [Delegate(signed_delegate_action_from_testnet)]
   block_hash: <recent mainnet block hash>
   ```

4. Attacker submits via mainnet RPC. `apply_delegate_action` calls `signed.verify()` → `get_nep461_hash()` → same hash → signature valid. Nonce check passes (alice.near mainnet nonce < 5). `max_block_height` check passes (mainnet height < 150_000_000). Alice's mainnet account loses 10 NEAR. [5](#0-4) [7](#0-6)

}

impl SignedDelegateAction {
    pub fn verify(&self) -> bool {
        let delegate_action = &self.delegate_action;
        let hash = delegate_action.get_nep461_hash();
        let public_key = &delegate_action.public_key;

        self.signature.verify(hash.as_ref(), public_key)
    }

    pub fn sign(singer: &Signer, delegate_action: DelegateAction) -> Self {
        let signature = singer.sign(delegate_action.get_nep461_hash().as_bytes());
        Self { delegate_action, signature }
    }

#[derive(BorshSerialize)]
pub struct SignableMessage<'a, T> {
    pub discriminant: MessageDiscriminant,
    pub msg: &'a T,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash)]
#[non_exhaustive]
pub enum SignableMessageType {
    /// A delegate action, intended for a relayer to included it in an action list of a transaction.
    DelegateAction,
    /// A delegate action with gas key support, intended for a relayer to include it in an action
    /// list of a transaction.
    DelegateActionV2,
}

#[derive(thiserror::Error, Debug)]
#[non_exhaustive]
pub enum ReadDiscriminantError {
    #[error("does not fit any known categories")]
    UnknownMessageType,
    #[error("NEP {0} does not have a known on-chain use")]
    UnknownOnChainNep(u32),
    #[error("NEP {0} does not have a known off-chain use")]
    UnknownOffChainNep(u32),
    #[error("discriminant is in the range for transactions")]
    TransactionFound,
}

#[derive(thiserror::Error, Debug)]
#[non_exhaustive]
pub enum CreateDiscriminantError {
    #[error("nep number {0} is too big")]
    NepTooLarge(u32),
}

impl<'a, T: BorshSerialize> SignableMessage<'a, T> {
    pub fn new(msg: &'a T, ty: SignableMessageType) -> Self {
        let discriminant = ty.into();
        Self { discriminant, msg }
    }

    pub fn sign(&self, signer: &Signer) -> Signature {
        let bytes = borsh::to_vec(&self).expect("Failed to deserialize");
        let hash = hash(&bytes);
        signer.sign(hash.as_bytes())
    }
