Audit Report

## Title
Unprivileged Caller Can Trigger Full Protocol Pause via Public `updateRSETHPrice()` When rsETH Price Drops Below Threshold — (`contracts/LRTOracle.sol`)

## Summary
`LRTOracle.updateRSETHPrice()` is a `public` function guarded only by `whenNotPaused`, with no role check. Its internal logic unconditionally pauses `LRTDepositPool`, `LRTWithdrawalManager`, and the oracle itself whenever the computed rsETH price falls below `highestRsethPrice` by more than `pricePercentageLimit`. Because `SwETHPriceOracle.getAssetPrice()` passes `ISwETH.getRate()` through verbatim with no bounds or staleness check, any transient dip in the swETH exchange rate is sufficient to trigger the auto-pause branch, which any unprivileged address can exploit by calling the public function at the right block.

## Finding Description

**No access control on `updateRSETHPrice()`:**

The function is declared `public` with only `whenNotPaused`:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
``` [1](#0-0) 

Any EOA or contract can call this at any time.

**`SwETHPriceOracle` passes rate through without validation:**

```solidity
return ISwETH(swETHAddress).getRate();
``` [2](#0-1) 

No floor, no staleness check, no deviation bound. A transient downward rate update from the Swell protocol directly reduces the computed `newRsETHPrice`.

**Auto-pause branch in `_updateRsETHPrice()`:**

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [3](#0-2) 

When `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit × highestRsethPrice`, all three contracts are paused unconditionally.

**`highestRsethPrice` is monotonically increasing:**

```solidity
if (newRsETHPrice > highestRsethPrice) {
    highestRsethPrice = newRsETHPrice;
}
``` [4](#0-3) 

It is never decremented, so any transient dip below the all-time high satisfies `newRsETHPrice < highestRsethPrice`.

**`LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` require `PAUSER_ROLE`:**

```solidity
function pause() external onlyRole(LRTConstants.PAUSER_ROLE) {
    _pause();
}
``` [5](#0-4) [6](#0-5) 

For the auto-pause calls at lines 278–279 of `LRTOracle.sol` to succeed, `LRTOracle` must hold `PAUSER_ROLE` on both contracts. This is a required deployment configuration for the downside-protection feature to function — the code is explicitly designed this way.

**Unpause requires separate admin calls on all three contracts:**

```solidity
function unpause() external whenPaused onlyLRTAdmin { ... }  // LRTOracle
function unpause() external onlyLRTAdmin { _unpause(); }     // LRTDepositPool
function unpause() external onlyLRTAdmin { _unpause(); }     // LRTWithdrawalManager
``` [7](#0-6) [8](#0-7) [9](#0-8) 

**Exploit path:**
1. `highestRsethPrice` is set to some value H from prior price updates.
2. The swETH rate dips (e.g., due to a rebase, slashing, or periodic update) such that the newly computed `newRsETHPrice` < H and the difference exceeds `pricePercentageLimit × H`.
3. An unprivileged attacker calls `oracle.updateRSETHPrice()`.
4. The auto-pause branch fires: `lrtDepositPool.pause()`, `withdrawalManager.pause()`, `_pause()` are all called.
5. All user deposits (`depositETH`, `depositAsset`) and withdrawals (`initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal`) are blocked until an admin manually unpauses all three contracts.

**Existing checks are insufficient:** The `whenNotPaused` modifier only prevents calling the function when already paused — it provides no protection against the first trigger. The `pricePercentageLimit > 0` guard only means the feature must be configured; once set to any nonzero value (e.g., 1% = `1e16`), a modest rate dip suffices.

## Impact Explanation

All user deposits and withdrawals are gated by `whenNotPaused` on `LRTDepositPool` and `LRTWithdrawalManager`. Once the auto-pause fires, every user interaction is blocked until an admin manually unpauses all three contracts. This constitutes **temporary freezing of funds** — a Medium-severity impact within the allowed scope.

## Likelihood Explanation

The swETH rate (`ISwETH.getRate()`) is updated periodically by the Swell protocol. A rate update that corrects downward (e.g., after a slashing event, rebase, or accounting adjustment) is a realistic, non-adversarial on-chain condition. Because `highestRsethPrice` is the all-time high and `pricePercentageLimit` can be set to a small value, even a modest transient dip satisfies the trigger. The attacker needs no capital, no privilege, and no external compromise — only the ability to monitor the swETH rate and call a public function at the right block. The attack is repeatable after each admin unpause.

## Recommendation

1. **Restrict `updateRSETHPrice()` to authorized callers** (e.g., add `onlyLRTManager` or a dedicated keeper role), matching the pattern already used by `updateRSETHPriceAsManager()`.
2. **Decouple the auto-pause from the public price-update path.** The auto-pause should only be triggerable by a privileged role, not as a side-effect of a permissionless call.
3. **Add a sanity bound to `SwETHPriceOracle.getAssetPrice()`** — reject rates that deviate more than a configurable percentage from the last known value, or use a time-weighted average.

## Proof of Concept

```solidity
// Fork test against mainnet fork with swETH as a supported asset
// and pricePercentageLimit set (e.g., 1e16 = 1%)
contract AuditPoC is Test {
    LRTOracle oracle = LRTOracle(<oracle_proxy>);
    LRTDepositPool pool = LRTDepositPool(<pool_proxy>);
    LRTWithdrawalManager wm = LRTWithdrawalManager(<wm_proxy>);

    function testUnprivilegedPause() public {
        // 1. Record current highestRsethPrice (set from prior price updates)
        // 2. Use vm.store to set swETH rate so that newRsETHPrice < highestRsethPrice
        //    by more than pricePercentageLimit * highestRsethPrice
        // 3. Call as unprivileged address
        assertFalse(pool.paused());
        assertFalse(wm.paused());

        vm.prank(address(1)); // no role
        oracle.updateRSETHPrice(); // public, no role check

        assertTrue(pool.paused());   // LRTDepositPool paused
        assertTrue(wm.paused());     // LRTWithdrawalManager paused
        assertTrue(oracle.paused()); // LRTOracle paused
    }
}
```

The call at step 3 reaches the auto-pause branch at `LRTOracle.sol` lines 277–281 because: (a) `updateRSETHPrice()` has no role guard, (b) `SwETHPriceOracle` passes the rate verbatim, and (c) `highestRsethPrice` is the all-time high. All three `paused()` assertions pass, and no admin action was required to trigger the freeze.
