### Title
Single Attested Participant Bypasses Threshold for `AppPublicKey` CKD Requests — (`crates/contract/src/lib.rs`, line 676)

---

### Summary

The `respond_ckd` function in `crates/contract/src/lib.rs` performs zero cryptographic verification for the `AppPublicKey` variant of a CKD request. A single attested participant can call `respond_ckd` with an arbitrary `(big_y, big_c)` pair and the contract will accept and deliver it to the requester, bypassing the threshold-authorization requirement entirely.

---

### Finding Description

In `respond_ckd`, the match on `request.app_public_key` has two arms: [1](#0-0) 

The `AppPublicKeyPV` arm calls `ckd_output_check`, which performs a full BLS12-381 pairing check:

```
e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)
``` [2](#0-1) 

The `AppPublicKey` arm is an empty block — **no verification whatsoever**.

The caller authorization check `assert_caller_is_attested_participant_and_protocol_active` only verifies that the signer is a registered participant with a valid attestation. It enforces no threshold — a single participant satisfies it: [3](#0-2) 

After the (absent) check, `resolve_yields_for` immediately resumes all pending yields with the attacker-supplied bytes: [4](#0-3) 

There is no accumulation of responses, no quorum, no threshold gate — one call from one attested participant resolves the request.

---

### Impact Explanation

The `AppPublicKey` CKD protocol is designed so the app holds a private scalar `a` and submits `app_pk = a·G1`. The MPC network computes `big_y = y·G1` and `big_c = hash(pk, app_id)·msk + a·big_y`, so the app recovers `derived_secret = big_c - a·big_y = hash(pk, app_id)·msk`. This is the confidential derived key.

A Byzantine participant sets `big_y = G1_identity` and `big_c = C_attacker`. The app computes:

```
derived_secret = C_attacker - a · identity = C_attacker
```

The derived secret is fully attacker-controlled. This constitutes **unauthorized confidential key derivation output without the required participant authorization** — a Critical-scope impact under the program rules.

---

### Likelihood Explanation

- The attacker must be a registered, attested participant (below-threshold Byzantine node).
- Pending `AppPublicKey` CKD requests are observable on-chain via the `pending_ckd_requests` map.
- The attacker races the legitimate MPC response. Since the attacker calls a single NEAR transaction while the honest nodes must complete a multi-round off-chain protocol, the attacker has a structural timing advantage.
- The `AppPublicKey` variant is the default/legacy form (it is the plain-string deserialization path and is used in tests and the example CLI). [5](#0-4) 

---

### Recommendation

For `AppPublicKey`, the contract cannot run the full pairing check (no G2 key is available). Two mitigations are possible:

1. **Deprecate `AppPublicKey` entirely** and require all callers to use `AppPublicKeyPV`, which is fully verifiable on-chain.
2. **Require threshold-many identical responses** before resolving the yield, analogous to how `vote_pk` / `vote_reshared` accumulate votes. The contract would store a map from `(CKDRequest, hash(response))` to a set of responding participants and only call `resolve_yields_for` once a threshold is reached.

Option 1 is strongly preferred because it eliminates the unverifiable code path entirely and is consistent with the `AppPublicKeyPV` design.

---

pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}

```rust
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
