### Title
Witness-Count Truncation via `zip` Allows Unauthorized UTxO Spending — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

---

### Summary

`updateUTxOTxWitness` uses Haskell's `zip` to pair input addresses with witnesses. `zip` silently truncates to the shorter list. A transaction with N inputs but only M < N witnesses causes the last N−M inputs to be spent with **zero witness verification**, allowing an attacker to steal UTxO outputs they do not own.

---

### Finding Description

The vulnerable line is in `updateUTxOTxWitness`:

```haskell
mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))   -- ← truncates to min(N, M)
  `wrapError` UTxOValidationTxValidationError
``` [1](#0-0) 

`addresses` is built from `NE.toList $ txInputs tx` — length N (one per input). `witness` is `taWitness ta` — a `Vector TxInWitness` of length M. When M < N, `zip` produces only M pairs; the last N−M inputs are never passed to `validateWitness`. [2](#0-1) 

No other guard compensates:

- **`validateTxAux`** checks size, fee, and value conservation — not witness count vs. input count. [3](#0-2) 
- **`validateTx`** checks attributes, network magic, and UTxO membership — not witness count. [4](#0-3) 
- **The CBOR decoder for `ATxAux`** decodes a 2-element list `(tx, witness)` with no constraint that `length witness == length (txInputs tx)`. [5](#0-4) 
- **`updateUTxOTx`** (the final UTxO state update) applies the spend unconditionally, ignoring witnesses entirely. [6](#0-5) 

The `whenTxValidation` guard does **not** protect against this — it only controls whether the witness-check block runs at all. In production, `applyTxAux` is called with the caller-supplied `validationMode`, and `applyMempoolPayload` passes through whatever mode the node uses. Under `BlockValidation` / `TxValidation` mode the block executes, but the `zip` truncation still silently skips uncovered inputs. [7](#0-6) 

---

### Impact Explanation

**Critical — Direct loss of ADA through an invalid ledger state transition.**

An attacker can spend a UTxO output they do not own by crafting a transaction with 2 inputs (one theirs, one the victim's) and only 1 witness (for their own input). The victim's input is consumed and its ADA redirected to attacker-controlled outputs, with no cryptographic authorization ever checked.

---

### Likelihood Explanation

The attack requires no privilege, no key leakage, no governance majority, and no external dependency. Any node accepting mempool transactions is a valid entry point. The attacker only needs to know a target UTxO outpoint (publicly visible on-chain). The crafted `ATxAux` is valid CBOR and passes all non-witness checks (fee, size, value conservation) as long as the attacker funds the fee from their own input.

---

### Recommendation

Replace the `zip`-based pairing with an explicit length-equality check before validation:

```haskell
let witnessCount = V.length witness
    inputCount   = length addresses
when (witnessCount /= inputCount) $
  throwError $ UTxOValidationTxValidationError
             $ TxValidationWitnessCountMismatch inputCount witnessCount

mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))
  `wrapError` UTxOValidationTxValidationError
```

A new `TxValidationWitnessCountMismatch` constructor should be added to `TxValidationError`. This ensures a transaction is rejected whenever the number of witnesses does not exactly equal the number of inputs.

---

### Proof of Concept

```
Given:
  utxo = { txIn_A → (addrA, 10 ADA),   -- attacker owns addrA
            txIn_V → (addrV, 100 ADA) } -- victim owns addrV

Craft ATxAux:
  tx.inputs  = [txIn_A, txIn_V]          -- N = 2
  tx.outputs = [(addrAttacker, 109 ADA)] -- steal victim's funds minus fee
  witness    = [VKWitness vkA sigA]      -- M = 1, only covers txIn_A

Call: updateUTxOTxWitness env utxo craftedTxAux

Execution:
  addresses = [addrA, addrV]             -- length 2
  zip addresses [VKWitness vkA sigA]
    = [(addrA, VKWitness vkA sigA)]      -- length 1, addrV dropped
  validateWitness called once (addrA) → passes
  validateTxAux: fee = 10+100-109 = 1 ADA ≥ minFee → passes
  updateUTxOTx: removes txIn_A and txIn_V, adds addrAttacker output

Result: 100 ADA stolen from addrV with no witness check on txIn_V.
```

validateTxAux ::
  MonadError TxValidationError m =>
  Environment ->
  UTxO ->
  ATxAux ByteString ->

validateTxAux env utxo (ATxAux (Annotated tx _) _ txBytes) = do
  -- Check that the size of the transaction is less than the maximum
  txSize
    <= maxTxSize
    `orThrowError` TxValidationTxTooLarge txSize maxTxSize

  -- Calculate the minimum fee from the 'TxFeePolicy'
  minFee <-
    if isRedeemUTxO inputUTxO
      then pure $ mkKnownLovelace @0
      else calculateMinimumFee feePolicy

  -- Calculate the balance of the output 'UTxO'
  balanceOut <-
    balance (txOutputUTxO tx)
      `wrapError` TxValidationLovelaceError "Output Balance"

  -- Calculate the balance of the restricted input 'UTxO'
  balanceIn <-
    balance inputUTxO
      `wrapError` TxValidationLovelaceError "Input Balance"

  -- Calculate the 'fee' as the difference of the balances
  fee <-
    subLovelace balanceIn balanceOut
      `wrapError` TxValidationLovelaceError "Fee"

  -- Check that the fee is greater than the minimum
  (minFee <= fee) `orThrowError` TxValidationFeeTooSmall tx minFee fee
  where
    Environment {protocolParameters} = env

    maxTxSize = ppMaxTxSize protocolParameters
    feePolicy = ppTxFeePolicy protocolParameters

    txSize :: Natural
    txSize = fromIntegral $ BS.length txBytes

    inputUTxO = S.fromList (NE.toList (txInputs tx)) <| utxo

    calculateMinimumFee ::
      MonadError TxValidationError m => TxFeePolicy -> m Lovelace
    calculateMinimumFee = \case
      TxFeePolicyTxSizeLinear txSizeLinear ->
        calculateTxSizeLinear txSizeLinear txSize
          `wrapError` TxValidationLovelaceError "Minimum Fee"

validateTx env utxo (Annotated tx _) = do
  -- Check that the transaction attributes are less than the max size
  unknownAttributesLength (txAttributes tx)
    < 128
    `orThrowError` TxValidationUnknownAttributes

  -- Check that outputs have valid NetworkMagic
  let nm = makeNetworkMagic protocolMagic
  txOutputs tx `forM_` validateTxOutNM nm

  -- Check that every input is in the domain of 'utxo'

    Environment {protocolMagic, utxoConfiguration} = env

updateUTxOTx env utxo aTx@(Annotated tx _) = do
  unlessNoTxValidation (validateTx env utxo aTx)
    `wrapErrorWithValidationMode` UTxOValidationTxValidationError

  UTxO.union (S.fromList (NE.toList (txInputs tx)) </| utxo) (txOutputUTxO tx)

updateUTxOTxWitness env utxo ta = do
  whenTxValidation $ do
    -- Get the signing addresses for each transaction input from the 'UTxO'
    addresses <-
      mapM (`UTxO.lookupAddress` utxo) (NE.toList $ txInputs tx)
        `wrapError` UTxOValidationUTxOError

    -- Validate witnesses and their signing addresses
    mapM_
      (uncurry $ validateWitness pmi sigData)
      (zip addresses (V.toList witness))
      `wrapError` UTxOValidationTxValidationError

    -- Validate the tx including witnesses
    validateTxAux env utxo ta
      `wrapError` UTxOValidationTxValidationError

  -- Update 'UTxO' ignoring witnesses
  updateUTxOTx env utxo aTx
  where
    Environment {protocolMagic} = env
    pmi = getAProtocolMagicId protocolMagic

    aTx@(Annotated tx _) = aTaTx ta
