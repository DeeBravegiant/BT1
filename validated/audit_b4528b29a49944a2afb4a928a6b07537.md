Audit Report

## Title
Stale Inflated `rsETHPrice` Permanently Broadcast to L2 Receivers After Oracle Auto-Pause — (`contracts/cross-chain/RSETHMultiChainRateProvider.sol`)

## Summary

When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it calls `_pause()` and returns early at line 281, leaving `rsETHPrice` frozen at the pre-drop (inflated) value. Because `MultiChainRateProvider.updateRate()` has no access control, any unprivileged caller can broadcast this stale inflated rate to all registered L2 `RSETHRateReceiver` contracts. L2 wrsETH/rsETH pool users cannot claim correct yield as the rate they see is higher than the actual protocol backing, and the discrepancy persists indefinitely while the oracle remains paused.

## Finding Description

**Step 1 — Auto-pause freezes `rsETHPrice` at inflated value.**

In `_updateRsETHPrice()`, when the price drop exceeds the configured threshold, the function calls `_pause()` and immediately returns: [1](#0-0) 

This `return` at line 281 exits before reaching the assignment `rsETHPrice = newRsETHPrice` at line 313: [2](#0-1) 

`rsETHPrice` is therefore frozen at the last known (higher) value.

**Step 2 — `whenNotPaused` blocks all future public price updates.**

`updateRSETHPrice()` is gated by `whenNotPaused`: [3](#0-2) 

The only bypass is `updateRSETHPriceAsManager()`, which requires `MANAGER` role: [4](#0-3) 

Critically, even a manager calling `updateRSETHPriceAsManager()` while TVL remains depressed will re-enter `_updateRsETHPrice()`, hit the same `isPriceDecreaseOffLimit` branch, call `_pause()` (which is a no-op since already paused), and `return` early again — still without updating `rsETHPrice`. The freeze is therefore indefinite as long as TVL stays below the threshold.

**Step 3 — `getLatestRate()` reads the stale storage variable directly.**

`RSETHMultiChainRateProvider.getLatestRate()` reads `rsETHPrice` from storage with no freshness or pause check: [5](#0-4) 

**Step 4 — `updateRate()` is permissionless.**

`MultiChainRateProvider.updateRate()` has no role check — any address can call it: [6](#0-5) 

It calls `getLatestRate()`, stores the stale result in `rate`, and broadcasts it via LayerZero to every registered `RSETHRateReceiver`. There is no guard preventing broadcast of a stale rate.

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield.**

L2 `RSETHRateReceiver` contracts receive and store the inflated stale rate. Any L2 yield-bearing pool (wrsETH/rsETH) that uses this rate for yield accrual computes yield against an inflated exchange rate that no longer reflects actual protocol backing. The discrepancy between the stale rate and the true rate represents yield that L2 users cannot correctly claim. The freeze persists until an admin manually unpauses the oracle AND either TVL recovers above the threshold or `pricePercentageLimit` is set to zero — both requiring privileged action with no on-chain enforcement deadline.

## Likelihood Explanation

- `pricePercentageLimit` is an admin-configured parameter explicitly designed to be set (e.g., 1% = `1e16`); its presence implies it is expected to be non-zero in production.
- A 2%+ TVL drop is realistic: EigenLayer slashing events, a depegged LST, or a large coordinated withdrawal can all cause this.
- `updateRSETHPrice()` is public — the auto-pause can be triggered by any caller, including a bot monitoring for TVL drops.
- `updateRate()` requires only ETH for LayerZero fees — any actor can call it to broadcast the stale rate.
- No privileged access is required at any step of the exploit.

## Recommendation

1. **Add a pause guard in `updateRate()`**: check `ILRTOracle(rsETHPriceOracle).paused()` and revert if true, preventing broadcast of a rate that cannot be refreshed.
2. **Alternatively**, add a `lastUpdated` timestamp to `LRTOracle` and reject broadcasts in `updateRate()` if the rate is older than a configurable threshold (e.g., 24 hours).
3. **Or**, restrict `updateRate()` to an authorized role (operator/manager) so that stale-rate broadcasts require deliberate action.

## Proof of Concept

```solidity
// Fork test outline (Foundry)
function test_staleCrossChainRateAfterAutoPause() public {
    // 1. Set pricePercentageLimit to 1% (1e16)
    vm.prank(admin);
    lrtOracle.setPricePercentageLimit(1e16);

    // 2. Record current rsETHPrice (pre-drop)
    uint256 preDropPrice = lrtOracle.rsETHPrice();

    // 3. Simulate 2% TVL drop by mocking asset oracle to return lower price
    mockAssetOracle.setPrice(currentPrice * 98 / 100);

    // 4. Anyone calls updateRSETHPrice() — triggers auto-pause, returns early
    lrtOracle.updateRSETHPrice();

    // 5. Assert oracle is paused and rsETHPrice is unchanged (frozen at inflated value)
    assertTrue(lrtOracle.paused());
    assertEq(lrtOracle.rsETHPrice(), preDropPrice);

    // 6. Unprivileged caller broadcasts stale rate to L2
    vm.deal(attacker, 1 ether);
    vm.prank(attacker);
    rateProvider.updateRate{value: 0.1 ether}();

    // 7. Assert broadcast rate equals stale pre-drop price
    assertEq(rateProvider.rate(), preDropPrice);

    // 8. Warp 7 days — rate is still stale, L2 still sees inflated rate
    vm.warp(block.timestamp + 7 days);
    assertEq(lrtOracle.rsETHPrice(), preDropPrice);
    assertEq(rateProvider.rate(), preDropPrice);

    // 9. Even manager calling updateRSETHPriceAsManager() re-triggers pause and returns early
    vm.prank(manager);
    lrtOracle.updateRSETHPriceAsManager(); // hits isPriceDecreaseOffLimit, _pause() no-op, returns
    assertEq(lrtOracle.rsETHPrice(), preDropPrice); // still frozen
}
```

        uint256 latestRate = getLatestRate();

        rate = latestRate;

        lastUpdated = block.timestamp;

        bytes memory _payload = abi.encode(latestRate);

        uint256 rateReceiversLength = rateReceivers.length;

        for (uint256 i; i < rateReceiversLength;) {
            uint16 dstChainId = uint16(rateReceivers[i]._chainId);

            bytes memory remoteAndLocalAddresses = abi.encodePacked(rateReceivers[i]._contract, address(this));

            (uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
                .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

            ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
                dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
            );

            unchecked {
                ++i;
            }
        }

        emit RateUpdated(rate);
    }
