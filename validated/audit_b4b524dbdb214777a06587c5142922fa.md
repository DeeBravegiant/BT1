### Title
Gateway Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Systematic Admission/Execution Mismatch — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful resource-bounds check reads the **current block's execution price** (`l2_gas_price`) from the previous block header to gate incoming transactions, but the next block will actually execute at the EIP-1559-adjusted **`next_l2_gas_price`** stored in the same header. Because the EIP-1559 price ticks automatically every block based on gas usage, the admission threshold is systematically wrong: transactions admitted during a rising-price period will fail execution, and transactions rejected during a falling-price period would have succeeded.

---

### Finding Description

`BlockHeaderWithoutHash` stores two distinct L2 gas price fields:

- `l2_gas_price` — the price at which block N was executed.
- `next_l2_gas_price` — the EIP-1559-adjusted price at which block N+1 will be executed. [1](#0-0) 

`update_state_sync_with_new_block` writes both fields correctly when a block is committed:

```rust
l2_gas_price: cende_block_info.gas_prices.l2_gas_price_per_token(), // block N's price
next_l2_gas_price: self.l2_gas_price,                               // block N+1's price
``` [2](#0-1) 

`next_l2_gas_price` is computed via the EIP-1559 formula in `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price`, which adjusts the price up or down by up to `1/gas_price_max_change_denominator` per block depending on whether gas usage exceeded the target: [3](#0-2) 

The gateway's `validate_resource_bounds` then reads the **wrong field** — `l2_gas_price` (block N's execution price) — instead of `next_l2_gas_price` (block N+1's execution price). A TODO comment in the production code explicitly acknowledges this gap:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;   // ← reads l2_gas_price, not next_l2_gas_price
``` [4](#0-3) 

`GatewayFixedBlockSyncStateClient.get_block_info()` maps `block_header.l2_gas_price.price_in_fri` into the returned `BlockInfo`, confirming it never surfaces `next_l2_gas_price`: [5](#0-4) 

The threshold applied is:

```rust
let threshold = (gas_price_threshold_multiplier * previous_block_l2_gas_price.get().0).to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...) }
``` [6](#0-5) 

With `min_gas_price_percentage = 100` (the production default), the threshold equals `l2_gas_price` of block N exactly — not `next_l2_gas_price` of block N+1. [7](#0-6) 

---

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions or rejects valid transactions before sequencing.**

**Rising-price scenario (gas usage > `gas_target`):**
`next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit = l2_gas_price` passes the gateway check but will fail blockifier pre-validation in block N+1 with `InsufficientResourceBounds` because `max_price_per_unit < next_l2_gas_price`. The mempool fills with transactions that are guaranteed to fail execution.

**Falling-price scenario (gas usage < `gas_target`):**
`next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit` between `next_l2_gas_price` and `l2_gas_price` is rejected by the gateway even though it would execute successfully in block N+1. Valid user transactions are incorrectly dropped.

The magnitude of the discrepancy grows with gas usage deviation from target. At maximum block fill (`max_block_size`), the price can increase by up to `1/gas_price_max_change_denominator` per block — a non-trivial delta at production gas prices. [8](#0-7) 

---

### Likelihood Explanation

- The condition is triggered every block where gas usage deviates from `gas_target` (60% of `max_block_size`), which is the normal operating state under any real load.
- No privileged access is required; any user submitting a transaction at the boundary price experiences this.
- The TODO comment confirms the developers are aware the wrong field is being read, meaning the bug is present in the current production code path.
- `validate_resource_bounds` is gated by `self.config.validate_resource_bounds`, which is enabled in production (`pointer_target: "validate_resource_bounds"`).

---

### Recommendation

Replace the `l2_gas_price` read in `validate_resource_bounds` with `next_l2_gas_price` from the block header. Since `GatewayFixedBlockSyncStateClient.get_block_info()` does not currently expose `next_l2_gas_price`, either:

1. Add a `get_next_l2_gas_price()` method to `GatewayFixedBlockStateReader` that reads `block_header_without_hash.next_l2_gas_price` directly, or
2. Extend the returned `BlockInfo` (or a wrapper) to carry `next_l2_gas_price` alongside the existing gas price fields.

The admission threshold should then be:

```rust
let next_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // reads next_l2_gas_price
    .await?;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    next_block_l2_gas_price,
)?;
```

### Proof of Concept

**Setup:** Production config with `validate_resource_bounds = true`, `min_gas_price_percentage = 100`. Block N has `l2_gas_price = 10_000_000_000 fri` and is filled to `max_block_size` (high usage).

**Step 1 — EIP-1559 tick:** `calculate_next_base_gas_price` computes `next_l2_gas_price` for block N+1. With `gas_price_max_change_denominator = 8` (example), the price increases by `10_000_000_000 / 8 = 1_250_000_000`, giving `next_l2_gas_price = 11_250_000_000 fri`. [8](#0-7) 

**Step 2 — Gateway admission:** User submits a transaction with `max_price_per_unit = 10_000_000_000`. The gateway reads `l2_gas_price = 10_000_000_000` from block N's header, computes `threshold = 10_000_000_000`, and admits the transaction (`10_000_000_000 >= 10_000_000_000`). [6](#0-5) 

**Step 3 — Block N+1 execution:** The batcher builds block N+1 with `l2_gas_price = 11_250_000_000` (from `next_l2_gas_price`). The blockifier's pre-validation rejects the transaction: `max_price_per_unit (10_000_000_000) < block_l2_gas_price (11_250_000_000)` → `InsufficientResourceBounds`.

**Result:** The transaction passed gateway admission but fails execution. The mempool accepted an invalid transaction. Under sustained high load, every transaction priced at the previous block's `l2_gas_price` boundary will be admitted and then fail — a systematic admission/execution mismatch that grows with each consecutive high-usage block.

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
