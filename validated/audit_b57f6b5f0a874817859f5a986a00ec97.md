Audit Report

## Title
Single Reverting Asset Price Oracle Permanently Blocks All rsETH Price Updates — (File: `contracts/LRTOracle.sol`)

## Summary

`LRTOracle._getTotalEthInProtocol()` iterates over every supported asset and calls `getAssetPrice(asset)` inside the loop with no error handling. If any one asset's Chainlink oracle reverts — a documented behavior under sequencer downtime, deprecated aggregators, or stale-price circuit breakers — the entire `updateRSETHPrice()` call reverts. Protocol fee minting and the rsETH exchange rate update are both blocked until an admin manually replaces the broken oracle.

## Finding Description

`_getTotalEthInProtocol()` (private, called by every price-update path) iterates over all supported assets and calls `getAssetPrice(asset)` at line 339 with no `try/catch`, no `continue`, and no skip logic:

```solidity
// LRTOracle.sol L336–348
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    uint256 assetER = getAssetPrice(asset);   // ← hard revert propagates up
    uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
    totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    unchecked { ++assetIdx; }
}
```

`getAssetPrice` (L156–158) delegates to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)`. The concrete implementation is `ChainlinkPriceOracle.getAssetPrice()` (L49–55), which calls `priceFeed.latestRoundData()` at line 52 — a call that Chainlink documents as reverting when the aggregator is deprecated, when an L2 sequencer is offline, or when a circuit breaker fires. Additionally, the `onlySupportedOracle` modifier (L40–45) itself reverts if `assetPriceOracle[asset] == address(0)`.

A single revert anywhere in the loop propagates through `_getTotalEthInProtocol()` → `_updateRsETHPrice()` → `updateRSETHPrice()` (public, L87) or `updateRSETHPriceAsManager()` (L94). Both entry points are blocked.

## Impact Explanation

**Medium — Permanent freezing of unclaimed yield.** Protocol fees are minted inside `_updateRsETHPrice()` at lines 299–308 via `IRSETH.mint(treasury, rsethAmountToMintAsProtocolFee)`. If `_getTotalEthInProtocol()` reverts, this mint never executes. Yield accrues in the underlying LST positions but is never captured as rsETH for the treasury for the entire duration the oracle is broken.

**Low — Contract fails to deliver promised returns.** `rsETHPrice` (L313) is never updated, so all depositors and withdrawers transact at a stale exchange rate for as long as the oracle is unavailable.

## Likelihood Explanation

No attacker action is required. The protocol supports multiple LST assets (stETH, ETHx, etc.), each backed by a Chainlink feed. Chainlink `latestRoundData()` is well-documented to revert under real-world infrastructure conditions: L2 sequencer restarts, deprecated aggregator contracts, and stale-price guards. With N supported assets the probability that at least one feed is temporarily unavailable grows with N. `updateRSETHPrice()` is a public function callable by any address, so any user can observe the failure. The condition is self-resolving only after an admin calls `updatePriceOracleFor`, which requires privileged access and manual intervention.

## Recommendation

Wrap the `getAssetPrice` call in a `try/catch` so a single failing oracle does not abort the entire aggregation:

```solidity
for (uint16 assetIdx; assetIdx < supportedAssetCount;) {
    address asset = supportedAssets[assetIdx];
    try this.getAssetPrice(asset) returns (uint256 assetER) {
        uint256 totalAssetAmt = ILRTDepositPool(lrtDepositPoolAddr).getTotalAssetDeposits(asset);
        totalETHInProtocol += totalAssetAmt.mulWad(assetER);
    } catch {
        emit AssetPriceOracleFailed(asset); // alert monitoring
        // optionally revert only if the asset has non-negligible TVL
    }
    unchecked { ++assetIdx; }
}
```

Alternatively, maintain a last-known-good price per asset and fall back to it on revert, with a staleness bound.

## Proof of Concept

1. Protocol has two supported assets: `stETH` (oracle healthy) and `ETHx` (Chainlink aggregator deprecated or L2 sequencer offline → `latestRoundData()` reverts).
2. Any caller invokes `updateRSETHPrice()` (public, no access control).
3. `_getTotalEthInProtocol()` processes `stETH` successfully, then calls `getAssetPrice(ETHx)`.
4. `ChainlinkPriceOracle.getAssetPrice(ETHx)` calls `priceFeed.latestRoundData()` which reverts.
5. The revert propagates: `_getTotalEthInProtocol` → `_updateRsETHPrice` → `updateRSETHPrice` — entire tx reverts.
6. `rsETHPrice` is never updated (L313 never reached). Fee mint at L304–307 never executes. Price-drop auto-pause at L278–281 also cannot fire.
7. Condition persists until admin calls `updatePriceOracleFor(ETHx, newOracle)`.

**Foundry fork test plan**: fork mainnet/L2, set the ETHx Chainlink aggregator to a mock that reverts on `latestRoundData`, call `lrtOracle.updateRSETHPrice()`, assert the call reverts and `rsETHPrice` is unchanged, assert treasury rsETH balance is unchanged despite TVL growth.
