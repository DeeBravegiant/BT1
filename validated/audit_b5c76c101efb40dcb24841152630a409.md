### Title
Unvalidated `ProposalInit.builder` allows proposer to inject arbitrary sequencer address into block hash and execution context - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

`ProposalInit.builder` is accepted verbatim from the network without any check in `is_proposal_init_valid`. This field is mapped to `sequencer_address` in `BlockInfo` by `convert_to_sn_api_block_info`, flows into `PartialBlockHashComponents.sequencer`, and is committed to the block hash. Every other security-sensitive field in `ProposalInit` (`height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, `fee_proposal_fri`) is validated; `builder` is the sole unvalidated field that reaches the block hash and the execution context.

### Finding Description

**`is_proposal_init_valid` does not check `builder`**

The full validation function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, all four L1 gas prices, and `fee_proposal_fri`. `builder` is absent from every check: [4](#0-3) 

The `proposer` field is validated at the consensus-manager layer against the committee:

```rust

`builder` has no equivalent check anywhere in the proposal-validation path.

**Both the proposer and the validator call `convert_to_sn_api_block_info`**

On the proposer side (`initiate_build`):

...

On the validator side (`initiate_validation`):

```rust

    ...
    block_info: convert_to_sn_api_block_info(init)?,

Because the validator uses the proposer-supplied `init.builder` without checking it, the batcher executes the block with whatever `sequencer_address` the proposer chose, and the resulting `PartialBlockHash` (and therefore `ProposalCommitment`) encodes that address.

**The wrong address persists through `decision_reached`**

`finalize_decision` also calls `convert_to_sn_api_block_info(init)` with the stored `init`:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs:552

``` [8](#0-7) 

So the injected `builder` address propagates into state-sync and the cende blob as well.

### Impact Explanation

A designated proposer for round R at height H can set `ProposalInit.builder` to any address X:

1. **Wrong sequencer address in the block hash.** `PartialBlockHashComponents.sequencer` = X → `PartialBlockHash` ≠ the hash that would have been produced with the legitimate builder address → the committed `ProposalCommitment` is wrong.
2. **Wrong execution context.** The blockifier executes every transaction in the block with `block_info.sequencer_address = X`. Any contract that calls the `get_sequencer_address` syscall receives X, not the real sequencer. This corrupts execution results, events, and receipts for any contract that branches on the sequencer address.
3. **Wrong fee collection.** Transaction fees are credited to the sequencer address. With X injected, fees flow to an attacker-controlled address instead of the legitimate sequencer.

All validators independently re-execute with the same proposer-supplied `init.builder`, so they all converge on the same wrong commitment and vote for it — the block is finalized with the corrupted state.

### Likelihood Explanation

The attack requires the adversary to be the designated proposer for at least one round (determined by the committee). In a permissioned or semi-permissioned validator set this is a realistic condition. The cost is zero beyond normal block-proposal participation; no additional resources or external access are needed.

### Recommendation

Add a check for `builder` inside `is_proposal_init_valid` (or in `ProposalInitValidation`) against the locally-configured expected builder address, mirroring the existing check for `proposer` at the manager layer:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder),
    ));
}
```

`ProposalInitValidation` should carry `expected_builder: ContractAddress` populated from `self.config.static_config.builder_address` at the call site in `SequencerConsensusContext::validate_proposal`. [9](#0-8) [10](#0-9) 

### Proof of Concept

1. Adversary controls a validator that is the designated proposer for height H, round R.
2. In `initiate_build`, instead of using `args.builder_address` (the legitimate sequencer), the adversary sets `builder: ContractAddress::from(0xdeadbeef_u64)`.
3. The adversary computes the correct `fin.proposal_commitment` for this `builder` value (the batcher will produce it honestly given the injected `block_info`).
4. The proposal is broadcast. Validators receive `ProposalInit` with `builder = 0xdeadbeef`.
5. `is_proposal_init_valid` passes — `builder` is never checked.
6. `initiate_validation` calls `convert_to_sn_api_block_info(init)` → `sequencer_address = 0xdeadbeef`.
7. The batcher executes all transactions with `sequencer_address = 0xdeadbeef`; any `get_sequencer_address` syscall returns `0xdeadbeef`; fees are credited to `0xdeadbeef`.
8. `PartialBlockHashComponents.sequencer = 0xdeadbeef` → `PartialBlockHash` encodes the wrong address.
9. All validators compute the same wrong `ProposalCommitment` and vote for it.
10. The block is finalized with the wrong sequencer address committed on-chain.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        proposal_init_validation.previous_proposal_init.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: fri={l1_gas_prices_fri:?}, wei={l1_gas_prices_wei:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_wei = l1_gas_prices_wei.l1_gas_price;
    let l1_data_gas_price_wei = l1_gas_prices_wei.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;
    let l1_gas_price_wei_proposed = init_proposed.l1_gas_price_wei;
    let l1_data_gas_price_wei_proposed = init_proposed.l1_data_gas_price_wei;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        )
        && within_margin(l1_gas_price_wei_proposed, l1_gas_price_wei, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_wei_proposed,
            l1_data_gas_price_wei,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, expected \
                 L1 gas price WEI={l1_gas_price_wei}, proposed={l1_gas_price_wei_proposed}, \
                 expected L1 data gas price WEI={l1_data_gas_price_wei}, \
                 proposed={l1_data_gas_price_wei_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }

    // fee_proposal is required iff Starknet version >= V0_14_3.
    let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
    match (init_proposed.fee_proposal_fri, fee_proposal_required) {
        (Some(_), false) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal must be absent before V0_14_3, got Some at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        (None, true) => {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "fee_proposal is required at V0_14_3+, got None at version {}",
                    init_proposed.starknet_version
                ),
            ));
        }
        _ => {}
    }

    // Validate fee_proposal is within the configured margin of fee_actual.
    // During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
    if let (Some(fee_actual), Some(fee_proposal)) =
        (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
    {
        let (lower_bound, upper_bound) = fee_proposal_bounds(
            fee_actual,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
            return Err(ValidateProposalError::InvalidProposalInit(
                init_proposed.clone(),
                proposal_init_validation.clone(),
                format!(
                    "Fee proposal out of bounds: fee_actual={}, fee_proposal={}, allowed \
                     range=[{lower_bound}, {upper_bound}]",
                    fee_actual.0, fee_proposal.0
                ),
            ));
        }
    }

    Ok(())
