### Title
Keyshare Private Material Exposed via Debug Logging in Migration Service Client — (File: `crates/node/src/migration_service/web/client.rs`)

### Summary
The migration service client unconditionally logs the full decrypted keyshare vector — including every node's raw private signing share — at `tracing::debug` level after receiving it from the peer. Because `Keyshare` and `KeyshareData` derive `Debug` without redaction, any log sink reachable by the host process (stdout, syslog, GCP Cloud Logging, Datadog, etc.) receives the plaintext private-share scalar whenever debug logging is active. In a TEE deployment the host is an explicit adversary; the log channel is the exact analog of the clipboard in the reference report.
