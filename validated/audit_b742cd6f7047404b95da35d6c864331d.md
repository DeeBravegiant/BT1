### Title
`paid_fee_on_l1` Hardcoded to `Fee(1)` in Consensus Path Bypasses L1 Handler Fee Adequacy Check — (`File: crates/apollo_transaction_converter/src/transaction_converter.rs`)

---

### Summary

`TransactionConverter::convert_consensus_l1_handler_to_internal_l1_handler` unconditionally injects `paid_fee_on_l1 = Fee(1)` for every L1 handler transaction arriving through the consensus path. The blockifier's only fee guard for L1 handlers checks `paid_fee != Fee(0)`. Because `Fee(1) != Fee(0)` is always true, the guard is permanently bypassed: no L1 handler transaction processed through consensus is ever checked for fee adequacy, and the `paid_fee_on_l1` value written to central storage is always the synthetic `Fee(1)` rather than the actual amount paid on L1.

---

### Finding Description

**Step 1 – `paid_fee_on_l1` is stripped at the consensus boundary.**

`ConsensusTransaction::L1Handler` carries only `transaction::L1HandlerTransaction` (the raw type), which has no `paid_fee_on_l1` field. [1](#0-0) 

When the proposer serialises an L1 handler for broadcast, the actual fee paid on L1 is silently dropped.

**Step 2 – The validator re-injects a synthetic `Fee(1)`.**

`convert_consensus_tx_to_internal_consensus_tx` calls `convert_consensus_l1_handler_to_internal_l1_handler`, which hardcodes `paid_fee_on_l1 = Fee(1)` with an acknowledged TODO: [2](#0-1) 

**Step 3 – The blockifier's only fee guard checks `paid_fee != Fee(0)`.**

After execution succeeds, `execute_raw` reads `self.paid_fee_on_l1` and rejects only the zero case: [3](#0-2) 

Because the converter always supplies `Fee(1)`, this guard is permanently satisfied regardless of what was actually paid on L1.

**Step 4 – Wrong value propagates to central storage.**

`CentralL1HandlerTransaction` serialises `paid_fee_on_l1` from the executable transaction: [4](#0-3) 

Every L1 handler transaction committed to the Cende blob will carry `paid_fee_on_l1 = 0x1` regardless of the actual L1 fee, as confirmed by the fixture: [5](#0-4) 

**The same pattern exists in the RPC re-execution path:** [6](#0-5) 

---

### Impact Explanation

The invariant "an L1 handler transaction must have paid a non-zero fee on L1 before the sequencer executes it" is broken for every transaction that arrives through the consensus path. The fee adequacy check is permanently bypassed: `Fee(1)` is always non-zero, so the guard at line 106 of `l1_handler_transaction.rs` never fires. Additionally, the `paid_fee_on_l1` value written to the authoritative Cende blob is always the synthetic `Fee(1)`, not the actual amount paid on L1, producing incorrect receipt/fee data in the committed block record.

This maps to: **Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact** and **RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value** (the Cende blob is the authoritative record consumed by downstream verifiers).

---

### Likelihood Explanation

This is triggered unconditionally for every L1 handler transaction processed through the consensus path — there is no attacker-controlled input required. The proposer strips `paid_fee_on_l1` when building `ConsensusTransaction::L1Handler`, and the validator always re-injects `Fee(1)`. The code path is exercised in every block that contains an L1 handler transaction.

---

### Recommendation

1. Carry `paid_fee_on_l1` through the consensus wire format. Add the field to the protobuf `L1HandlerV0` message and propagate it through `ConsensusTransaction::L1Handler`.
2. Remove the hardcoded `Fee(1)` in `convert_consensus_l1_handler_to_internal_l1_handler` and use the value received from the proposer.
3. Strengthen the fee check in `execute_raw` from `paid_fee != Fee(0)` to `paid_fee >= receipt.fee` (the actual execution cost), consistent with the error message that already reports the required fee.
4. Apply the same fix to `stored_txn_to_executable_txn` in the RPC path, reading the stored `paid_fee_on_l1` from storage rather than hardcoding `Fee(1)`.

---

### Proof of Concept

```
1. Proposer builds a block containing an L1 handler transaction with paid_fee_on_l1 = Fee(0)
   (e.g., sourced from a buggy or malicious L1 contract that emits a message without fee).

2. Proposer calls convert_internal_consensus_tx_to_consensus_tx:
     InternalConsensusTransaction::L1Handler(tx) =>
         Ok(ConsensusTransaction::L1Handler(tx.tx))   // paid_fee_on_l1 is DROPPED

3. Validator receives ConsensusTransaction::L1Handler and calls
   convert_consensus_tx_to_internal_consensus_tx →
   convert_consensus_l1_handler_to_internal_l1_handler:
     Fee(1)   // hardcoded, TODO(Gilad)

4. Blockifier executes the transaction. After execution:
     let paid_fee = self.paid_fee_on_l1;  // = Fee(1)
     if paid_fee == Fee(0) { ... }        // FALSE — guard never fires

5. l1_handler_tx_execution_info sets receipt.fee = Fee(0) and returns Ok.
   The transaction is committed with paid_fee_on_l1 = 0x1 in the Cende blob,
   regardless of what was actually paid on L1.
``` [7](#0-6) [8](#0-7)

        &self,
        tx: ConsensusTransaction,
    ) -> TransactionConverterResult<(InternalConsensusTransaction, Option<VerifyAndStoreProofTask>)>
    {
        match tx {
            ConsensusTransaction::RpcTransaction(tx) => {
                let (internal_tx, proof_data) = self.convert_rpc_tx_to_internal(tx).await?;
                let task = proof_data.map(|(proof_facts, proof)| {
                    self.spawn_verify_and_store_proof(proof_facts, proof)
                });
                Ok((InternalConsensusTransaction::RpcTransaction(internal_tx), task))
            }
            ConsensusTransaction::L1Handler(tx) => {
                let internal_tx = self.convert_consensus_l1_handler_to_internal_l1_handler(tx)?;
                Ok((InternalConsensusTransaction::L1Handler(internal_tx), None))
            }
        }
