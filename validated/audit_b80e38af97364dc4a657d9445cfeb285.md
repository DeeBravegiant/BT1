Audit Report

## Title
Nested Unbounded Loops in `getAssetUnstaking()` Called Per-Asset Per-NDC Cause `updateRSETHPrice()` to Exceed Block Gas Limit - (File: contracts/NodeDelegator.sol, contracts/LRTDepositPool.sol, contracts/LRTOracle.sol)

## Summary
`updateRSETHPrice()` in `LRTOracle` is a public function that triggers a deeply nested call chain: for each supported asset, for each NDC, `getAssetUnstaking()` is called, which itself calls `getQueuedWithdrawals()` on EigenLayer and iterates over all queued withdrawals and their strategies. As the protocol scales through normal admin and operator actions, the cumulative gas cost of this chain can exceed the 30M block gas limit, permanently preventing oracle price updates and blocking all protocol fee minting.

## Finding Description

The call chain is confirmed in the code:

**Layer 1** — `_getTotalEthInProtocol()` iterates over all supported assets and calls `getTotalAssetDeposits(asset)` for each: [1](#0-0) 

**Layer 2** — `getAssetDistributionData()` iterates over all NDCs and calls `getAssetUnstaking(asset)` for each NDC: [2](#0-1) 

**Layer 3** — `getAssetUnstaking()` calls `getQueuedWithdrawals()` (an unbounded external STATICCALL to EigenLayer) and then iterates over every queued withdrawal and every strategy within each withdrawal: [3](#0-2) 

The critical inefficiency is that `getQueuedWithdrawals(address(this))` is called once per `(asset, NDC)` pair, not once per NDC. With N supported assets and M NDCs, this results in N×M calls to `getQueuedWithdrawals`, each returning the full set of queued withdrawals for that NDC. The total external call count is `O(assets × NDCs × queuedWithdrawals × strategiesPerWithdrawal)`.

The public entry point has no privilege requirement beyond the contract not being paused: [4](#0-3) 

`updateRSETHPriceAsManager()` calls the same `_updateRsETHPrice()` internal function and is equally affected once the gas threshold is crossed: [5](#0-4) 

The `maxUncompletedWithdrawalCount` guard in `LRTUnstakingVault` limits queued withdrawals per NDC but does not prevent the multiplicative scaling across assets — `getAssetUnstaking` is called once per `(NDC, asset)` pair, so the bound on withdrawals per NDC is multiplied by the number of supported assets.

## Impact Explanation

**Unbounded gas consumption (Medium):** The total gas cost of `updateRSETHPrice()` scales as `O(assets × NDCs × withdrawals × strategies)`. With `maxNodeDelegatorLimit = 10`, 5 supported assets, and a modest number of queued EigenLayer withdrawals, the 30M block gas limit is reachable through normal protocol operation. Once crossed, `updateRSETHPrice()` and `updateRSETHPriceAsManager()` both revert on every call.

**Permanent freezing of unclaimed yield (Medium):** Protocol fee minting via `IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee)` occurs exclusively inside `_updateRsETHPrice()`. If this function becomes uncallable, all accrued protocol fee yield is permanently frozen — it can never be minted to the treasury. [6](#0-5) 

## Likelihood Explanation

No adversarial action is required. The condition is triggered by ordinary protocol scaling:
- Admin adds supported LST assets (increases N)
- Admin raises `maxNodeDelegatorLimit` and adds NDCs (increases M)
- Operators call `initiateUnstaking()` or `undelegate()` during normal restaking operations (increases queued withdrawals per NDC)

All three of these are expected, documented protocol operations. The `maxUncompletedWithdrawalCount` cap does not prevent the issue because the cap applies per NDC, but `getAssetUnstaking` is called N times per NDC (once per asset), multiplying the EigenLayer call count by the number of supported assets.

## Recommendation

1. **Fetch `getQueuedWithdrawals` once per NDC, not once per `(NDC, asset)` pair.** Refactor `_getTotalEthInProtocol()` or `getAssetDistributionData()` to retrieve all queued withdrawals for each NDC a single time and compute balances for all assets in a single pass over the result.
2. **Cache unstaking amounts lazily.** Store a per-`(NDC, asset)` `assetUnstaking` value updated when withdrawals are queued (`initiateUnstaking`, `undelegate`) or completed (`completeUnstaking`), and read the cached value in `getAssetUnstaking()` instead of querying EigenLayer live.
3. **Document and enforce a hard cap** on `assets × NDCs × maxUncompletedWithdrawalCount × strategiesPerWithdrawal` relative to the block gas limit, and enforce it in `addNodeDelegatorContractToQueue` and asset addition.
4. **Paginate `updateRSETHPrice`** if a fully on-chain O(1) solution is not feasible.

## Proof of Concept

Assume the following realistic protocol state:
- 5 supported assets (ETH, stETH, ETHx, rETH, sfrxETH)
- 10 NDCs (`maxNodeDelegatorLimit = 10`, all slots filled)
- 20 queued EigenLayer withdrawals per NDC (within `maxUncompletedWithdrawalCount`)
- 2 strategies per withdrawal

External calls in one `updateRSETHPrice()` invocation:
- `getQueuedWithdrawals`: 5 assets × 10 NDCs = **50 external calls** (each reads unbounded EigenLayer storage)
- `strategy.sharesToUnderlyingView`: 50 × 20 withdrawals × 2 strategies = **2,000 external calls**

At a conservative ~5,000 gas per cold external call plus EigenLayer internal storage reads, this exceeds 30M gas. A Foundry fork test against Ethereum mainnet can reproduce this by deploying the protocol with the above parameters, queuing 20 withdrawals per NDC via `initiateUnstaking`, and calling `updateRSETHPrice()` with `gas: 30_000_000` — the call will revert with out-of-gas, demonstrating permanent oracle freeze and fee minting blockage.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
