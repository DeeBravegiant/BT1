Audit Report

## Title
`highestRsethPrice` Never Resets Downward Enables Permanent Re-Pause Griefing After Slashing — (`contracts/LRTOracle.sol`)

## Summary
After a legitimate price drop exceeding `pricePercentageLimit` (e.g., from a slashing event), `highestRsethPrice` remains at its historical peak with no admin-accessible reset path. Because `updateRSETHPrice()` is unrestricted and the downside-protection check always compares the current price against the all-time high, any public caller can re-trigger an auto-pause immediately after each admin unpause, indefinitely freezing deposits and withdrawals.

## Finding Description

`_updateRsETHPrice()` contains the downside-protection block at lines 270–282: [1](#0-0) 

When `isPriceDecreaseOffLimit` is true, the function pauses `lrtDepositPool`, `withdrawalManager`, and `LRTOracle` itself, then returns early — without updating `highestRsethPrice`.

`highestRsethPrice` only ever increases: [2](#0-1) 

The only downward reset is when `rsethSupply == 0`: [3](#0-2) 

This is not a realistic post-slashing state. There is no admin setter for `highestRsethPrice`.

`unpause()` does not reset `highestRsethPrice`: [4](#0-3) 

`updateRSETHPrice()` is callable by any EOA or contract: [5](#0-4) 

`updateRSETHPriceAsManager()` (no `whenNotPaused`) also calls `_updateRsETHPrice()`: [6](#0-5) 

When called while paused, it reaches the same `isPriceDecreaseOffLimit` branch, calls `_pause()` (which is a no-op since already paused at line 319–320), and returns early — so the manager also cannot advance past the check to update `highestRsethPrice`. [7](#0-6) 

**Concrete griefing loop:**
1. `highestRsethPrice = 1.2e18`, `pricePercentageLimit = 1e16` (1%), real price stabilizes at `1.0e18` after slashing.
2. Admin unpauses all three contracts.
3. Attacker calls `updateRSETHPrice()`. `diff = 0.2e18 > pricePercentageLimit.mulWad(1.2e18) = 1.2e16` → `isPriceDecreaseOffLimit = true` → protocol auto-pauses again.
4. Repeat from step 2 indefinitely.

## Impact Explanation

Every call to `updateRSETHPrice()` after a legitimate price drop exceeding `pricePercentageLimit` re-pauses `LRTDepositPool`, `WithdrawalManager`, and `LRTOracle`. Deposits and withdrawals are frozen for all users. The admin's only programmatic workaround is `setPricePercentageLimit(0)`, which disables the downside-protection feature entirely — a forced trade-off between resuming operations and maintaining the safety mechanism. This constitutes **Medium — Temporary freezing of funds**: funds are not lost, but user access to deposits and withdrawals is indefinitely blocked as long as the griefing continues.

## Likelihood Explanation

The entry point is fully public with no role restriction. The precondition (a slashing event dropping price by more than `pricePercentageLimit` from the historical peak) is a realistic on-chain event for an LST protocol. Once the condition is met, it is permanently satisfied until `pricePercentageLimit` is zeroed out. Any caller — including an automated bot — can exploit this with a single zero-cost transaction after each admin unpause.

## Recommendation

Add an admin function to reset `highestRsethPrice` to the current price after a confirmed legitimate price drop:

```solidity
function resetHighestRsethPrice() external onlyLRTAdmin {
    highestRsethPrice = rsETHPrice;
    emit HighestRsethPriceReset(rsETHPrice);
}
```

Alternatively, reset `highestRsethPrice` inside `unpause()` so the reference point is always anchored to the price at the time of resumption, preventing the condition from being immediately re-triggered.

## Proof of Concept

```solidity
// Preconditions:
// highestRsethPrice = 1.2e18 (set historically before slashing)
// pricePercentageLimit = 1e16 (1%)
// After slashing, oracle returns price = 1.0e18
// Protocol was auto-paused; admin calls unpause() on all three contracts

for (uint i = 0; i < 3; i++) {
    assert(!oracle.paused());
    assert(!depositPool.paused());
    assert(!withdrawalManager.paused());

    // Any public caller re-pauses: diff=0.2e18 > 1.2e16
    oracle.updateRSETHPrice();

    assert(oracle.paused());
    assert(depositPool.paused());
    assert(withdrawalManager.paused());

    // Admin unpauses for next iteration
    oracle.unpause();
    depositPool.unpause();
    withdrawalManager.unpause();
}
// highestRsethPrice unchanged at 1.2e18; loop repeatable indefinitely
```
