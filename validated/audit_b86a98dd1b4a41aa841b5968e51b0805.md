### Title
Unvalidated `ProposalInit.builder` Field Allows Any Proposer to Inject Arbitrary Sequencer Address into Block Hash and Execution Context — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates many `ProposalInit` fields but never validates the `builder` field. `builder` is used verbatim as `sequencer_address` in block execution and is baked into the block hash. Any proposer in the consensus rotation can set an arbitrary `builder` address; validators accept it, execute the block with the wrong sequencer address, redirect all transaction fees, and commit a corrupted block hash. The `ProposalFin` commitment check does not prevent this because both sides derive the commitment from the same attacker-supplied `builder`.

---

### Finding Description

**Root cause — missing field in `is_proposal_init_valid`**

`is_proposal_init_valid` (lines 252–419 of `validate_proposal.rs`) validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, `timestamp`, all four L1 gas-price fields, and `fee_proposal_fri`. The `builder` field is never compared against any locally-trusted reference value. [1](#0-0) 

`ProposalInitValidation` — the struct that carries the validator's own trusted parameters — has no `builder_address` member, so there is no reference value to check against. [2](#0-1) 

**How `builder` reaches the block hash and execution context**

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`: [3](#0-2) 

`PartialBlockHashComponents::new` stores that address as `sequencer`: [4](#0-3) 

`calculate_block_hash` chains `sequencer` into the Poseidon hash: [5](#0-4) 

**Why the `ProposalFin` commitment check does not guard against this**

After the batcher finishes, `validate_proposal` compares `built_block` (computed by the batcher using the attacker-supplied `builder`) against `received_fin.proposal_commitment` (also computed by the proposer using the same `builder`). Both sides agree, so the check passes: [6](#0-5) 

**End-to-end exploit path**

1. Attacker is a validator in the consensus rotation and wins the right to propose block H.
2. Attacker constructs `ProposalInit` with `builder = attacker_address`.
3. Validators call `is_proposal_init_valid` → `builder` is never checked → passes.
4. `initiate_validation` passes `block_info` with `sequencer_address = attacker_address` to the batcher.
5. All transactions in the block are executed with `sequencer_address = attacker_address`; fees are paid to `attacker_address`.
6. `PartialBlockHashComponents` and `calculate_block_hash` bake `attacker_address` into the block hash.
7. The proposer's `ProposalFin` commitment was computed with the same `attacker_address`, so the mismatch check passes.
8. The block is committed with a wrong sequencer address and a corrupted block hash.

---

### Impact Explanation

**Critical.** Three distinct invariants are broken simultaneously:

| Invariant | Broken value |
|---|---|
| Correct fee/balance accounting | All transaction fees in the block are redirected to `attacker_address` |
| Correct blockifier/syscall result | `get_sequencer_address` syscall returns `attacker_address` for every transaction in the block |
| Correct block hash / state commitment | `sequencer` field in `PartialBlockHashComponents` is `attacker_address`, corrupting the block hash anchored to L1 |

This matches the allowed impact categories: "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" and "Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input."

---

### Likelihood Explanation

Any validator that wins a proposal slot can trigger this with zero additional privilege. The attack requires only the ability to craft a `ProposalInit` message, which every proposer already does. No cryptographic break, no social engineering, and no special knowledge beyond the normal proposer role is needed.

---

### Recommendation

1. Add `builder_address: ContractAddress` to `ProposalInitValidation`, populated from `ContextStaticConfig.builder_address` at the call site in `SequencerConsensusContext::validate_proposal`. [7](#0-6) 

2. In `is_proposal_init_valid`, reject any `init_proposed.builder` that does not equal `proposal_init_validation.builder_address`, analogous to the existing `height` / `l1_da_mode` / `l2_gas_price_fri` check: [8](#0-7) 

3. Add a unit test in `validate_proposal_test.rs` that sets `init.builder` to a wrong address and asserts `InvalidProposalInit` is returned.

---

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
