Audit Report

## Title
Unguarded `receiveFrom*` Functions Allow Unprivileged ETH Injection, Triggering Spurious Fee rsETH Minting Against Non-Genuine Rewards — (`contracts/LRTDepositPool.sol`)

## Summary

`receiveFromRewardReceiver()`, `receiveFromLRTConverter()`, and `receiveFromNodeDelegator()` in `LRTDepositPool` carry no caller access control, allowing any external account to inject arbitrary ETH. Because `getETHDistributionData()` reads `address(this).balance` directly and `_updateRsETHPrice()` treats any TVL increase as genuine yield, the injected ETH causes fee rsETH to be minted to the treasury against non-genuine rewards, reducing the yield delivered to existing rsETH holders.

## Finding Description

**Step 1 — No access control on receive functions.**
All three functions are bare `external payable` with no modifier, no role check, and no caller validation. [1](#0-0) 

**Step 2 — `getETHDistributionData` reads raw contract balance.**
`ethLyingInDepositPool = address(this).balance` immediately reflects any injected ETH. [2](#0-1) 

**Step 3 — `updateRSETHPrice()` is publicly callable.**
No role restriction; any account can trigger the price update after injecting ETH. [3](#0-2) 

**Step 4 — Fee is computed on the injected ETH as genuine reward.**
`rewardAmount = totalETHInProtocol - previousTVL` includes the injected ETH; `protocolFeeInETH` is a percentage of that amount. [4](#0-3) 

**Step 5 — Fee rsETH is minted to the treasury.**
The treasury receives rsETH minted against the externally injected ETH. [5](#0-4) 

**Partial mitigations that do not prevent the attack:**
- `pricePercentageLimit` reverts only if the price increase exceeds the configured threshold; small injections within the limit succeed. [6](#0-5) 
- `maxFeeMintAmountPerDay` caps total daily fee minting but does not prevent the spurious fee from being taken within that cap. [7](#0-6) 

## Impact Explanation

Existing rsETH holders receive less yield than the protocol promises. When fee rsETH is minted to the treasury against externally injected ETH, the new rsETH price is computed as `(totalETHInProtocol - protocolFeeInETH) / rsethSupply`. Holders' share of the injected ETH is reduced by the fee percentage, even though the injected ETH was not a protocol-earned reward. The protocol does not lose value (the injected ETH remains in the pool), but holders receive less yield than they are entitled to. This matches **Low — Contract fails to deliver promised returns, but doesn't lose value**.

## Likelihood Explanation

The attack requires only a public call with ETH attached — no role, no approval, no special state. The economic cost to the attacker is the injected ETH itself (which is donated to the pool). A griever willing to spend ETH can repeatedly trigger this within the `pricePercentageLimit` and `maxFeeMintAmountPerDay` bounds. Likelihood is **Medium** given the direct, permissionless path.

## Recommendation

Add caller validation to all three receive functions. Only the designated `RewardReceiver`, `LRTConverter`, and registered `NodeDelegator` contracts should be permitted to call them:

```solidity
function receiveFromRewardReceiver() external payable {
    if (msg.sender != lrtConfig.getContract(LRTConstants.LRT_REWARD_RECEIVER))
        revert InvalidCaller();
}

function receiveFromLRTConverter() external payable {
    if (msg.sender != lrtConfig.getContract(LRTConstants.LRT_CONVERTER))
        revert InvalidCaller();
}

function receiveFromNodeDelegator() external payable {
    if (isNodeDelegator[msg.sender] == 0)
        revert InvalidCaller();
}
```

## Proof of Concept

```solidity
// Foundry test (local fork or mock setup)
function test_spuriousFeeMintViaUnguardedReceive() public {
    uint256 treasuryBalanceBefore = rsETH.balanceOf(treasury);
    uint256 priceBefore = lrtOracle.rsETHPrice();

    // Attacker injects ETH via the unguarded function (no role required)
    vm.deal(attacker, 1 ether);
    vm.prank(attacker);
    lrtDepositPool.receiveFromRewardReceiver{value: 1 ether}();

    // Trigger price update (public, no role required)
    lrtOracle.updateRSETHPrice();

    // Treasury gained fee rsETH minted against injected ETH
    assertGt(rsETH.balanceOf(treasury), treasuryBalanceBefore);
    // Price increased due to injected ETH counted as TVL
    assertGt(lrtOracle.rsETHPrice(), priceBefore);
    // Existing holders received less yield than entitled to
}
```
