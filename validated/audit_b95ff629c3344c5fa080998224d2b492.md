Audit Report

## Title
Permissionless `createPool()` Forwards Unbound `salt` to CREATE2, Enabling Front-Run Pool Hijack - (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary
`createPool()` is permissionless and forwards `params.salt` verbatim to `MetricOmmPoolDeployer.deploy()`, which uses it as the raw CREATE2 salt with no binding to `msg.sender`. An attacker who observes a pending `createPool()` in the mempool can front-run it with the same `salt`, a malicious `priceProvider` that satisfies `_validatePriceProvider`, and `admin = attacker`. The attacker's pool occupies the deterministic address permanently; the victim's transaction reverts on the CREATE2 collision, and all off-chain infrastructure pre-computing that address routes swaps to the attacker's manipulated oracle.

## Finding Description
`PoolParameters.salt` is a raw `bytes32` with no caller binding: [1](#0-0) 

`createPool()` carries no access control and forwards the salt verbatim: [2](#0-1) [3](#0-2) 

The deployer uses it directly as the CREATE2 salt: [4](#0-3) 

`_validatePriceProvider` only checks that `token0()`/`token1()` match — it does not inspect bid/ask behavior — so a malicious oracle that returns correct token addresses but manipulated prices passes validation: [5](#0-4) 

After deployment the attacker's pool is registered in all factory mappings (`poolAdmin`, `poolFeeConfig`, `idxToPool`, `poolToIdx`), so `isPool()` returns `true` for the hijacked address: [6](#0-5) 

**Exploit flow:**
1. Attacker deploys `MaliciousPriceProvider(token0, token1)` — returns correct token addresses, returns attacker-chosen bid/ask.
2. Attacker observes victim's pending `createPool(salt=S, admin=victim, priceProvider=legit)`.
3. Attacker submits `createPool(salt=S, admin=attacker, priceProvider=malicious)` with higher gas.
4. Attacker's transaction lands first; CREATE2 address is occupied.
5. Victim's transaction reverts (CREATE2 collision).
6. Attacker's pool passes `isPool()` and all factory lookups; attacker holds `poolAdmin` role.
7. Off-chain routers/quoters that pre-computed the address via CREATE2 route swaps to the attacker's pool, which executes at manipulated bid/ask prices.

## Impact Explanation
- **Bad-price execution**: The attacker's `priceProvider` returns arbitrary bid/ask values. Every swap routed to the hijacked address executes at attacker-chosen prices, causing traders to receive less than the oracle/bin curve permits or the pool to receive less than owed input — a direct loss of user principal.
- **Admin-boundary break**: The attacker becomes `poolAdmin` of the pool at the expected address through an unprivileged front-run, bypassing the intended admin assignment. They can set fees to the maximum cap and redirect `adminFeeDestination` to themselves.

## Likelihood Explanation
`createPool()` is permissionless on any chain with a public mempool (Ethereum mainnet, Base). The attacker only needs to pre-deploy one malicious `PriceProvider` contract. The attack is repeatable: every new `createPool()` attempt by the legitimate creator with a new salt can be front-run again. No special privileges are required.

## Recommendation
Bind the salt to `msg.sender` inside `createPool()` before forwarding to the deployer:

```solidity
bytes32 boundSalt = keccak256(abi.encode(msg.sender, params.salt));
pool = MetricOmmPoolDeployer(poolDeployer).deploy(
    MetricOmmPoolDeployer.DeployParams({
        salt: boundSalt,
        ...
    })
);
```

This ensures two different callers using the same `params.salt` produce different CREATE2 addresses, eliminating the front-running surface while preserving deterministic address prediction for the legitimate creator.

## Proof of Concept

```solidity
contract MaliciousPriceProvider {
    address public token0;
    address public token1;
    constructor(address t0, address t1) { token0 = t0; token1 = t1; }
    function getBidAndAsk() external pure returns (uint128, uint128) {
        return (1, type(uint128).max); // extreme spread
    }
}

function test_frontrun_createPool_salt_hijack() public {
    bytes32 targetSalt = keccak256("VICTIM_POOL_SALT");

    PoolParameters memory victimParams = _defaultPoolParams();
    victimParams.salt = targetSalt;
    victimParams.admin = victim;

    MaliciousPriceProvider malOracle = new MaliciousPriceProvider(
        victimParams.token0, victimParams.token1
    );

    PoolParameters memory attackerParams = victimParams;
    attackerParams.admin = attacker;
    attackerParams.adminFeeDestination = attacker;
    attackerParams.priceProvider = address(malOracle);

    vm.prank(attacker);
    address hijackedPool = factory.createPool(attackerParams);

    vm.prank(victim);
    vm.expectRevert(); // CREATE2 collision
    factory.createPool(victimParams);

    assertEq(factory.poolAdmin(hijackedPool), attacker);
    assertTrue(factory.isPool(hijackedPool));
}
```
