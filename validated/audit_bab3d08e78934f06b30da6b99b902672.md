### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Inject Arbitrary `sequencer_address` into Block Hash and Execution Context — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates most fields of `ProposalInit` but never checks `builder`. The `builder` field is mapped directly to `sequencer_address` in `BlockInfo` by `convert_to_sn_api_block_info`, flows into `PartialBlockHashComponents::sequencer`, and is hashed into the canonical block hash via `calculate_block_hash`. Any contract that calls `get_block_info()` during execution also reads this attacker-controlled value. A legitimate-but-malicious proposer can set `builder` to any address, and every validator will silently accept and execute the block with that address as the sequencer.

---

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field, described as "Address of the one who builds/sequences the block." [1](#0-0) 

`is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices (within margin), `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri`. It never checks `builder`. [2](#0-1) 

`ProposalInitValidation` — the struct that carries the validator's local reference values — has no `builder` field at all, so there is no local expectation to compare against. [3](#0-2) 

`initiate_validation` calls `convert_to_sn_api_block_info(init)` and passes the result directly to the batcher as `ValidateBlockInput.block_info`. [4](#0-3) 

`convert_to_sn_api_block_info` maps `init.builder` → `sequencer_address` in `starknet_api::block::BlockInfo`: [5](#0-4) 

`PartialBlockHashComponents::new` maps `block_info.sequencer_address` → `sequencer`: [6](#0-5) 

`calculate_block_hash` chains `sequencer.0` into the Poseidon hash at position 4: [7](#0-6) 

The consensus manager checks `init.proposer` against the committee but never checks `init.builder`: [8](#0-7) 

---

### Impact Explanation

**Wrong execution result (Critical):** Every transaction executed during validation runs with the attacker-chosen `sequencer_address` as the block's sequencer. Contracts that call `get_block_info()` (e.g., fee-collection contracts, sequencer-gated access-control logic, or any contract that branches on `sequencer_address`) receive the injected value. This is a wrong syscall result from the blockifier for accepted input.

**Wrong block hash (Critical):** The committed `PartialBlockHash` — and therefore the final block hash — includes the attacker-chosen sequencer address. All downstream consumers of the block hash (L1 anchoring, proof verification, state sync) operate on a hash derived from an unchecked, proposer-supplied field.

---

### Likelihood Explanation

The attacker must be a legitimate, staked committee member who wins the proposer slot for a round. No external or unprivileged trigger is required beyond that. The `builder` field is freely settable in `ProposalInit` with no wire-level constraint, and the validator has no local reference value to compare it against. The attack is silent — no error is logged, no vote is withheld.

---

### Recommendation

Add `builder` to `ProposalInitValidation` and check it in `is_proposal_init_valid`, the same way `l1_da_mode` and `l2_gas_price_fri` are checked:

```rust
// In ProposalInitValidation:
pub builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder, init_proposed.builder
        ),
    ));
}
```

The validator's `builder` reference value should be sourced from the node's own configured `builder_address`, the same source used by `initiate_build`. [9](#0-8) 

---

### Proof of Concept

1. A staked validator wins the proposer slot for height `H`, round `R`.
2. It constructs `ProposalInit` with `builder = ContractAddress(0xdeadbeef)` (any arbitrary address).
3. It broadcasts this `ProposalInit` to all peers.
4. Each peer's `validate_proposal` calls `is_proposal_init_valid`, which checks `height`, `l1_da_mode`, `l2_gas_price_fri`, gas prices, `starknet_version`, `version_constant_commitment`, `timestamp`, and `fee_proposal_fri` — but **not** `builder`. Validation passes.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: 0xdeadbeef, ... }`.
6. The batcher executes all transactions with `sequencer_address = 0xdeadbeef`. Any contract calling `get_block_info().sequencer_address` receives `0xdeadbeef`.
7. `BlockExecutionArtifacts::new` calls `PartialBlockHashComponents::new(&block_info, ...)`, embedding `sequencer = 0xdeadbeef`.
8. `PartialBlockHash::from_partial_block_hash_components` hashes this into the `ProposalCommitment`.
9. Consensus reaches decision; the block is committed with `sequencer_address = 0xdeadbeef` in both the execution context and the canonical block hash. [10](#0-9)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
