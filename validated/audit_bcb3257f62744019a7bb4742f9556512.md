Audit Report

## Title
Unbounded Gas Consumption in `getAssetUnstaking()` Nested Loop Causes Temporary Freezing of Deposits and Withdrawals — (File: `contracts/NodeDelegator.sol`)

## Summary

`NodeDelegator.getAssetUnstaking()` iterates over all EigenLayer queued withdrawals and their strategies via a live `getQueuedWithdrawals()` call. This function is invoked for every NDC on every deposit, withdrawal initiation, and price update, creating O(NDCs × queued_withdrawals × strategies) gas complexity. As queued withdrawals accumulate — including through EigenLayer operator-forced undelegations that bypass the protocol's soft cap — the gas cost grows unboundedly, causing `depositETH()`, `depositAsset()`, and `initiateWithdrawal()` to revert with out-of-gas, temporarily freezing user funds until the EigenLayer withdrawal delay expires.

## Finding Description

**Root cause:** `getAssetUnstaking()` in `contracts/NodeDelegator.sol` (L405–427) performs a live enumeration of all EigenLayer queued withdrawals with a nested loop over strategies, making external calls to `strategy.underlyingToken()` and `strategy.sharesToUnderlyingView()` for each entry.

**Call path to deposit:**
`depositETH()` / `depositAsset()` (L87, L111) → `_beforeDeposit()` (L648) → `_checkIfDepositAmountExceedesCurrentLimit()` (L676) → `getTotalAssetDeposits()` (L385) → `getAssetDistributionData()` / `getETHDistributionData()` (L426, L467) → `getAssetUnstaking()` called once per NDC (L451, L488).

**Call path to withdrawal:**
`initiateWithdrawal()` (L170) → `getAvailableAssetAmount()` (L599) → `getTotalAssetDeposits()` → same chain.

**Call path to price update:**
`updateRSETHPrice()` → `_getTotalEthInProtocol()` (L331) → `getTotalAssetDeposits()` per supported asset → same chain.

**Why the mitigation fails:** `LRTUnstakingVault.setMaxUncompletedWithdrawalCount()` caps the protocol's own counter at 80 (L153), with the comment explicitly acknowledging forced undelegations as a known bypass: *"Need buffer for theoretical operator forced undelegations (ndc count * asset count = 15)"* (L152). `increaseUncompletedWithdrawalCount()` (L184) is only called by NodeDelegator during protocol-initiated unstaking — EigenLayer operator-forced undelegations create queued withdrawals directly in EigenLayer without incrementing this counter. The actual count iterated by `getAssetUnstaking()` is fetched live from `getQueuedWithdrawals()` and is not bounded by the protocol's soft cap. The reconciliation function `setUncompletedWithdrawalCount()` (L164) is admin-only and not automatic.

## Impact Explanation

**Medium — Temporary freezing of funds.** When the total queued EigenLayer withdrawals across all NDCs grows large enough to exhaust the block gas limit inside `getAssetUnstaking()`'s nested loop, `depositETH()`, `depositAsset()`, and `initiateWithdrawal()` all revert. Users cannot deposit or withdraw for the duration of EigenLayer's withdrawal delay (minimum 7 days), constituting a temporary freeze of user funds. `updateRSETHPrice()` also becomes uncallable, staling the rsETH price. The protocol's own comment acknowledges that beyond 120 total uncompleted withdrawals, `updateRSETHPrice()` fails. The freeze is temporary (not permanent) because withdrawals will eventually complete after the delay, reducing the queued count.

## Likelihood Explanation

EigenLayer operators can force-undelegate NDCs at any time — this is a designed EigenLayer feature, not an attack or external compromise. The protocol explicitly acknowledges this risk in its own code comments. With `maxNodeDelegatorLimit = 10` NDCs and multiple strategies per NDC, a coordinated or incidental wave of forced undelegations can push the queued withdrawal count past the gas-safe threshold without any action from the protocol. No privileged access is required on the protocol side; the trigger is an external EigenLayer operator action.

## Recommendation

1. **Decouple `getAssetUnstaking()` from the hot path:** Maintain a running tally of unstaking amounts updated at queue/complete time (in `NodeDelegator.unstakeAsset()` and `completeUnstaking()`) rather than recomputing by iterating live EigenLayer state on every user interaction.
2. **Cache and bound the loop:** If live enumeration is required, enforce a hard cap on the number of queued withdrawals iterated (not just a soft protocol counter), reverting or skipping beyond the cap.
3. **Automate reconciliation:** Make `setUncompletedWithdrawalCount()` callable permissionlessly or trigger it automatically on forced undelegation detection, so the protocol's accounting stays accurate without admin intervention.

## Proof of Concept

1. Deploy with 10 NDCs (`maxNodeDelegatorLimit = 10`), each delegated to an EigenLayer operator with 3 supported strategies.
2. EigenLayer operators force-undelegate all 10 NDCs, creating e.g. 8 queued withdrawals per NDC × 3 strategies = 240 strategy iterations in `getAssetUnstaking()`, called 10 times per `getTotalAssetDeposits()` invocation = 2,400 external calls.
3. Call `depositETH{value: 1 ether}(0, "")`. The call chain reaches `getETHDistributionData()` → 10× `getAssetUnstaking(ETH_TOKEN)` → 10× (loop over 8 withdrawals × 3 strategies), each iteration making external calls to EigenLayer strategy contracts.
4. Observe the transaction reverts with out-of-gas before `_mintRsETH()` is reached.
5. Repeat for `initiateWithdrawal()` — same revert.
6. All deposits and withdrawals remain blocked until EigenLayer's withdrawal delay expires and `completeUnstaking()` is called for each queued withdrawal — a minimum 7-day window during which user funds are frozen.

**Foundry fork test plan:** Fork mainnet at a block where EigenLayer is live. Deploy/configure the protocol with 10 NDCs. Use `vm.prank` to simulate EigenLayer operator force-undelegation calls (`DelegationManager.undelegate()`) for each NDC, creating queued withdrawals. Then call `depositETH` and assert the transaction reverts with out-of-gas, or measure gas consumption scaling with `vm.expectRevert` / gas snapshots.
