Audit Report

## Title
Proportional Output Rescaling Lacks Zero-Guard, Allowing Trader to Pay Full Input and Receive Zero Output — (`metric-core/contracts/libraries/SwapMath.sol`)

## Summary

In `buyToken0InBinSpecifiedIn` and `buyToken1InBinSpecifiedIn`, when the analytical target position overshoots the remaining input budget, the code proportionally rescales `out0Scaled` (or `out1Scaled`) using floor division with no zero-guard. When the pre-rescale output is small (e.g., 1 scaled unit) and `state.amountSpecifiedRemainingScaled < totalIn1Scaled`, the rescaled output rounds to exactly 0. The code then consumes the full remaining input from swap state, so the trader pays real tokens and receives nothing — a direct swap conservation failure.

## Finding Description

In `buyToken0InBinSpecifiedIn`, the rescaling block at lines 624–634 is:

```solidity
if (totalIn1Scaled > state.amountSpecifiedRemainingScaled) {
    uint256 delta = targetPos - currBinPos;
    uint256 scaledDelta = Math.ceilDiv(delta * state.amountSpecifiedRemainingScaled, totalIn1Scaled);
    if (scaledDelta == 0) scaledDelta = 1;          // ← guard exists for position
    targetPos = currBinPos + scaledDelta;

    // ← NO guard for out0Scaled
    out0Scaled = (out0Scaled * state.amountSpecifiedRemainingScaled) / totalIn1Scaled;
    totalIn1Scaled = state.amountSpecifiedRemainingScaled;
}
``` [1](#0-0) 

`scaledDelta` is correctly guarded against zero, but `out0Scaled` is not. If `out0Scaled == 1` (achievable with a bin balance of 1 scaled unit) and `state.amountSpecifiedRemainingScaled < totalIn1Scaled`, then:

```
out0Scaled = (1 * state.amountSpecifiedRemainingScaled) / totalIn1Scaled = 0
```

The subsequent accounting block unconditionally uses these values:

```solidity
binState.token0BalanceScaled -= out0Scaled.toUint104();          // subtracts 0
binState.token1BalanceScaled += totalIn1Scaled - protocolFeeAmountScaled;  // gains full input
state.amountSpecifiedRemainingScaled -= totalIn1Scaled;          // full input consumed
state.amountCalculatedScaled += out0Scaled;                      // adds 0
``` [2](#0-1) 

The early-exit guard at lines 550–552 only rejects inputs too small to afford the *starting price*, not inputs that would produce zero output after proportional rescaling: [3](#0-2) 

The identical pattern exists in `buyToken1InBinSpecifiedIn` for `out1Scaled`: [4](#0-3) 

## Impact Explanation

A trader executing an exact-input swap pays their full specified input amount in token1 (or token0) and receives 0 token0 (or token1) in return. The input tokens are permanently transferred into the pool and credited to the bin's balance. This is a direct, irreversible loss of user principal and a swap conservation failure: the pool receives the owed input but the trader does not receive the owed output.

## Likelihood Explanation

The condition requires: (1) a bin with a very small `token0BalanceScaled` (e.g., 1 scaled unit), which occurs naturally as liquidity is consumed across bins; (2) a swap input that passes the starting-price early-exit check but is less than the cost of 1 scaled output unit; (3) no effective slippage protection (`minAmountOut = 0` or a check that accepts 0, which is common for integrators). Tokens with high price ratios (e.g., WBTC/USDC) or low decimals increase probability. The condition is reachable by any unprivileged caller via the public `swap` function.

## Recommendation

Add a zero-guard for `out0Scaled` immediately after the proportional rescaling in `buyToken0InBinSpecifiedIn`, mirroring the existing `scaledDelta` guard:

```solidity
out0Scaled = (out0Scaled * state.amountSpecifiedRemainingScaled) / totalIn1Scaled;
if (out0Scaled == 0) {
    return (currBinPos, 0, 0, 0, 0);
}
totalIn1Scaled = state.amountSpecifiedRemainingScaled;
```

Apply the same fix to `out1Scaled` in `buyToken1InBinSpecifiedIn`.

## Proof of Concept

**Setup:**
- `binState.token0BalanceScaled = 1` (minimal bin balance)
- `currBinPos = 0`, `lowerPriceX64 = 2^64` (price = 1.0), `upperPriceX64 = 2^64 + 2^64/100` (1% spread)
- `currBinBuyFeeX64 = 0`, `spreadFeeE6 = 0`

**Execution:**
1. `startingPriceX64 = 2^64`. Early-exit check: `(amountIn << 64) >= 2^64` passes for `amountIn >= 1`.
2. Analytical target position computes `out0Scaled = 1` (full bin), `totalIn1Scaled = avgPrice ≈ 1 * avgPrice`.
3. Trader sends `amountIn` in `[1, totalIn1Scaled - 1]`, entering the rescaling branch.
4. `scaledDelta` guard fires, `targetPos` updated correctly.
5. `out0Scaled = (1 * amountIn) / totalIn1Scaled = 0` (floor division, since `amountIn < totalIn1Scaled`).
6. `totalIn1Scaled = amountIn`, `state.amountCalculatedScaled += 0`.
7. Trader pays `amountIn` scaled units of token1, receives 0 token0.
