Audit Report

## Title
`rsETHPrice` and `highestRsethPrice` Unconditionally Reset to `1 ether` When `rsethSupply == 0`, Enabling First-Depositor Theft of Orphaned Protocol Rewards - (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle._updateRsETHPrice()` unconditionally overwrites both `rsETHPrice` and `highestRsethPrice` to `1 ether` whenever `rsethSupply == 0`, discarding all previously accumulated price history. Because `updateRSETHPrice()` is a public function callable by any unprivileged address, an attacker can trigger this reset the moment the last rsETH is burned, then deposit at the artificially deflated price to capture all EigenLayer staking rewards that remain in `NodeDelegator` contracts but have not yet been swept to the unstaking vault.

## Finding Description

**Root cause — unconditional price reset:**

In `contracts/LRTOracle.sol` at lines 218–222:

```solidity
if (rsethSupply == 0) {
    rsETHPrice = 1 ether;
    highestRsethPrice = 1 ether;
    return;
}
```

Both state variables are overwritten with no regard for the previous price. The public entry point at lines 87–89 imposes only a `whenNotPaused` guard:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

**How `rsethSupply` reaches zero:**

rsETH is burned via two paths in `contracts/LRTWithdrawalManager.sol`:
- `instantWithdrawal` (line 229): `IRSETH(lrtConfig.rsETH()).burnFrom(address(msg.sender), rsETHUnstaked)` — immediate burn, gated only by `isInstantWithdrawalEnabled[asset]`.
- `unlockQueue` (line 305): `IRSETH(lrtConfig.rsETH()).burnFrom(address(this), rsETHBurned)` — burns rsETH held by the manager after the delay period.

Both paths are reachable through the normal withdrawal lifecycle. A coordinated mass-exit or protocol migration can drive `rsethSupply` to zero.

**Why residual ETH remains after supply reaches zero:**

`_getTotalEthInProtocol()` (lines 331–349 of `LRTOracle.sol`) calls `ILRTDepositPool.getTotalAssetDeposits(asset)`, which in turn calls `getETHDistributionData()` (lines 467–500 of `LRTDepositPool.sol`). That function sums `getEffectivePodShares()` across all `NodeDelegator` contracts — i.e., EigenLayer pod shares including continuously accruing staking rewards. These rewards are **not** automatically swept to the unstaking vault when rsETH is burned. `unlockQueue` pays users based on `unstakingVault.balanceOf(asset)` (line 849 of `LRTWithdrawalManager.sol`), not on the full EigenLayer balance. Any rewards that accrue after the last `unlockQueue` call, or that were never swept, remain in the `NodeDelegator` and are still counted by `_getTotalEthInProtocol()`.

**Exploit flow:**

1. Attacker calls `updateRSETHPrice()` once `rsethSupply == 0`. `rsETHPrice → 1 ether`, `highestRsethPrice → 1 ether`.
2. Attacker calls `LRTDepositPool.depositETH{value: 1 ether}(0, "")`. `getRsETHAmountToMint` (line 520 of `LRTDepositPool.sol`) computes `(1e18 × 1e18) / 1e18 = 1 rsETH`. Attacker receives 1 rsETH.
3. Attacker calls `updateRSETHPrice()` again. Now `rsethSupply = 1`, `totalETHInProtocol = 1 ETH (deposit) + R ETH (orphaned EigenLayer rewards)`. `rsETHPrice = (1 + R) × 1e18`.
4. Attacker withdraws 1 rsETH and receives `(1 + R)` ETH, stealing `R` ETH of orphaned staking rewards.

**`pricePercentageLimit` does not block the attack:**

`pricePercentageLimit` is a storage variable with no explicit initialization, so it defaults to `0`. The price-increase guard at lines 256–257 is `pricePercentageLimit > 0 && ...`, which evaluates to `false` when the limit is zero, meaning the price jump from `1 ether` to `(1 + R) ether` is not blocked for an unprivileged caller.

**Secondary impact — weakened circuit breaker:**

Resetting `highestRsethPrice` from, e.g., `1.05 ether` to `1 ether` means a subsequent price drop to `0.95 ether` (a ~9.5% drop from the true peak) no longer triggers the auto-pause at lines 270–282, permanently weakening downside protection for all subsequent depositors.

## Impact Explanation

**High — Theft of unclaimed yield.** The attacker captures the entire balance of EigenLayer staking rewards that have accrued inside `NodeDelegator` contracts but have not been swept to the unstaking vault. The magnitude equals all rewards accumulated since the last operator sweep. Additionally, `highestRsethPrice` is permanently reset, weakening the price-deviation circuit-breaker for the new deposit epoch.

## Likelihood Explanation

`rsethSupply` reaching zero is a realistic edge case during a protocol migration, a coordinated mass-exit, or when `instantWithdrawal` is enabled and all holders exit. No privileged access is required to trigger the reset — any EOA can call `updateRSETHPrice()` the moment supply hits zero. `pricePercentageLimit` defaults to `0`, so the price-increase guard is disabled unless the admin has explicitly configured it, making the single-block capture path viable without any manager intervention.

## Recommendation

Preserve the last known price when `rsethSupply` drops to zero rather than resetting it:

```solidity
if (rsethSupply == 0) {
    // Do not reset rsETHPrice or highestRsethPrice.
    // The stored price remains valid for the next depositor.
    return;
}
```

If a deliberate reset to `1 ether` is required (e.g., after a full protocol migration), it should be gated behind an explicit admin-only function rather than triggered automatically by a public call.

## Proof of Concept

**Foundry fork test outline:**

1. Fork mainnet with a live KelpDAO deployment where `rsETHPrice > 1 ether` and EigenLayer pod shares `R > 0` exist in a `NodeDelegator`.
2. Burn all outstanding rsETH by simulating the full withdrawal lifecycle (`initiateWithdrawal` → `unlockQueue` → `completeWithdrawal`) for all holders, or by enabling and using `instantWithdrawal`. Verify `IRSETH.totalSupply() == 0`.
3. Assert `LRTDepositPool.getTotalAssetDeposits(ETH_TOKEN) > 0` (orphaned rewards still present via `getEffectivePodShares()`).
4. Call `LRTOracle.updateRSETHPrice()` from an attacker EOA. Assert `rsETHPrice == 1 ether`.
5. Call `LRTDepositPool.depositETH{value: 1 ether}(0, "")` from the attacker. Assert attacker receives exactly `1 rsETH`.
6. Call `LRTOracle.updateRSETHPrice()` again. Assert `rsETHPrice == (1 ether + R)`.
7. Initiate and complete withdrawal of `1 rsETH`. Assert attacker receives `(1 + R)` ETH, confirming theft of `R` ETH of orphaned staking rewards.
