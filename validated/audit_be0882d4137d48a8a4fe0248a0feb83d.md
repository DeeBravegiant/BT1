### Title
Missing CKD Output Verification for Legacy `AppPublicKey` Requests in `respond_ckd` — (`crates/contract/src/lib.rs`)

---

### Summary

`respond_ckd` applies a cryptographic BLS pairing check (`ckd_output_check`) only to the `AppPublicKeyPV` (publicly verifiable) variant of CKD requests. The legacy `AppPublicKey` variant receives **no output verification at all**. A single Byzantine attested participant (strictly below the signing threshold) can call `respond_ckd` with an arbitrary, forged `CKDResponse` for any pending legacy-format request, and the contract will accept and deliver it to the waiting user.

---

### Finding Description

In `respond_ckd`, the contract branches on the request's `app_public_key` variant:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}   // ← no check whatsoever
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` enforces the BLS pairing equation `e(big_c, g2) = e(big_y, app_pk2) · e(hash_point, public_key)`, which cryptographically binds the response to the MPC network's master key and the user's app identity: [2](#0-1) 

For `AppPublicKey` (the legacy, privately verifiable format), the arm is an empty block — the contract performs **zero cryptographic validation** on `big_y` and `big_c` before resolving the yield and returning the response to the caller. [3](#0-2) 

By contrast, `respond` (for ECDSA/EdDSA signatures) always verifies the signature against the derived public key before resolving: [4](#0-3) 

The inconsistency is a direct structural analog to the Footium EIP2981 issue: `AppPublicKeyPV` implements the output check; `AppPublicKey` does not.

---

### Impact Explanation

A single Byzantine attested participant can:

1. Observe any pending `AppPublicKey` CKD request on-chain (stored in `pending_ckd_requests`).
2. Construct an arbitrary `CKDResponse` with random G1 points for `big_y` and `big_c`.
3. Call `respond_ckd` with that forged response before the honest MPC nodes submit theirs.
4. The contract accepts it (no check), resolves the yield, and delivers the garbage/attacker-chosen ciphertext to the user's TEE application.

The user's TEE app decrypts a meaningless or attacker-influenced key instead of the legitimate MPC-derived secret. Because the yield is already resolved, the user cannot retry the same request — they must submit a new one, which the attacker can race again. This breaks the production safety invariant that every CKD response delivered by the contract is a valid output of the threshold BLS protocol.

---

### Likelihood Explanation

The `AppPublicKey` (legacy) format is still accepted by the contract and documented as the default single-G1-point format. Any user or developer contract that submits a CKD request using the plain string or `{"AppPublicKey": ...}` encoding is vulnerable. A single Byzantine attested participant — one who has passed TEE attestation but acts maliciously — can exploit this without any threshold collusion, simply by racing the honest nodes. [5](#0-4) 

---

### Recommendation

Apply the same `ckd_output_check` to `AppPublicKey` requests. For the privately verifiable variant, the check can be adapted: since `pk2` is not available, the contract should at minimum verify that `big_c = hash_app_id_with_pk(pk, app_id) * msk + pk1 * y` using the available G1 public key and the network's BLS master key. Alternatively, deprecate the `AppPublicKey` variant and require all new requests to use `AppPublicKeyPV`, which is already publicly verifiable on-chain.

---

### Proof of Concept

```
1. Alice submits request_app_private_key with AppPublicKey (legacy G1 point A).
   → pending_ckd_requests[CKDRequest{app_public_key: AppPublicKey(A), ...}] = [yield_id]

2. Byzantine participant Eve (attested, in active participant set) calls:
   respond_ckd(
     request = CKDRequest{app_public_key: AppPublicKey(A), app_id, domain_id},
     response = CKDResponse{big_y: random_G1, big_c: random_G1}
   )

3. Contract executes respond_ckd:
   - assert_caller_is_attested_participant_and_protocol_active() → passes (Eve is attested)
   - match AppPublicKey(_) => {}  ← no check
   - resolve_yields_for(..., serde_json::to_vec(&response)) → yield resolved

4. Alice's TEE app receives CKDResponse{big_y: random_G1, big_c: random_G1}.
   Decryption yields garbage; the legitimate MPC response is discarded.
``` [6](#0-5)

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

/// Check that `e(big_c, g2) = e(big_y, app_pk2) . e(hash_point, public_key)`.
///
/// Point validation is fully delegated to the host, as in
/// [`app_public_key_check`].
pub(crate) fn ckd_output_check(
    app_id: &dtos::CkdAppId,
    output: &CKDResponse,
    app_public_key: &dtos::CKDAppPublicKeyPV,
    public_key: &dtos::Bls12381G2PublicKey,
) -> bool {
    let big_c = env::bls12381_p1_decompress(&output.big_c);
    let big_y = env::bls12381_p1_decompress(&output.big_y);
    let pk2 = env::bls12381_p2_decompress(&app_public_key.pk2);
    let pk = env::bls12381_p2_decompress(public_key);
    let hash_point = hash_app_id_with_pk(public_key.as_slice(), app_id.as_ref());

    let pairing_input = [
        big_c.as_slice(),
        MINUS_G2_GENERATOR_UNCOMPRESSED.as_slice(),
        big_y.as_slice(),
        pk2.as_slice(),
        hash_point.as_slice(),
        pk.as_slice(),
    ]
    .concat();
    env::bls12381_pairing_check(&pairing_input)
}
