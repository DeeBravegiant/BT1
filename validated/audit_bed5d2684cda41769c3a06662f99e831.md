Audit Report

## Title
Depositor Can Mint Zero rsETH Shares Due to Integer Truncation in `getRsETHAmountToMint` - (File: contracts/LRTDepositPool.sol)

## Summary
`getRsETHAmountToMint` computes the rsETH mint amount via integer division that truncates to zero for dust-level deposits when `rsETHPrice > amount * assetPrice`. Because `_beforeDeposit` only guards against `rsethAmountToMint < minRSETHAmountExpected`, a caller who passes `minRSETHAmountExpected = 0` bypasses all output checks, causing assets to be absorbed into protocol TVL while the depositor receives 0 rsETH.

## Finding Description
In `getRsETHAmountToMint` (L520):

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

This is plain Solidity integer division. When `amount * assetPrice < rsETHPrice`, the result is `0`.

`_beforeDeposit` (L657–669) has two guards:

1. `if (depositAmount == 0 || depositAmount < minAmountToDeposit)` — `minAmountToDeposit` is declared at L30 and never set in `initialize()`, so it defaults to `0`. Any non-zero `depositAmount` passes.
2. `if (rsethAmountToMint < minRSETHAmountExpected)` — when the caller passes `minRSETHAmountExpected = 0`, the condition `0 < 0` is `false`, so no revert occurs.

There is no `require(rsethAmountToMint > 0)` anywhere in the deposit path. `_mintRsETH(0)` (L686–690) calls `IRSETH(rsethToken).mint(msg.sender, 0)`, minting nothing. For ETH deposits, the ETH is already in the contract via `msg.value` before `_beforeDeposit` is called (L87). For LST deposits, `safeTransferFrom` executes at L114 after `_beforeDeposit` returns. In both cases the assets are transferred in and counted in `getTotalAssetDeposits`, increasing rsETH price for all existing holders, while the depositor receives nothing.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

A depositor who sends a dust-level amount with `minRSETHAmountExpected = 0` suffers a complete loss of the deposited amount while receiving 0 rsETH. The protocol TVL increases, benefiting all existing rsETH holders. The protocol itself does not lose value; the loss is borne entirely by the depositor.

## Likelihood Explanation
The condition `amount * assetPrice < rsETHPrice` is satisfied for very small (sub-gwei) deposit amounts under normal conditions where rsETHPrice ≈ 1e18. The risk grows as rsETH appreciates. Any caller who omits a meaningful slippage guard (passes `minRSETHAmountExpected = 0`) is exposed. This is a realistic mistake for integrators or unsophisticated users. `minAmountToDeposit` defaults to `0` and requires admin action to raise, so the default deployment provides no protection.

## Recommendation
Add an explicit zero-output guard in `_beforeDeposit` after computing `rsethAmountToMint`:

```solidity
rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);

if (rsethAmountToMint == 0) {
    revert ZeroRsETHMinted();
}

if (rsethAmountToMint < minRSETHAmountExpected) {
    revert MinimumAmountToReceiveNotMet();
}
```

This ensures no deposit can succeed with a zero output regardless of the caller-supplied slippage parameter.

## Proof of Concept

**Preconditions:**
- rsETH price = `1.1e18` (10% appreciation above peg)
- ETH asset price = `1e18`
- `minAmountToDeposit = 0` (default, unset by admin)

**Steps:**
1. Call `depositETH{value: 1}(0, "")` — sends 1 wei ETH with `minRSETHAmountExpected = 0`.
2. `_beforeDeposit(ETH_TOKEN, 1, 0)` is entered:
   - `1 == 0` → false; `1 < 0` → false. Passes L657.
   - `getRsETHAmountToMint(ETH_TOKEN, 1)` → `(1 * 1e18) / 1.1e18 = 0` (integer truncation). Returns `0`.
   - `0 < 0` → false. Passes L667.
3. `_mintRsETH(0)` is called → `IRSETH.mint(msg.sender, 0)` → 0 rsETH minted.
4. The 1 wei ETH is held by `LRTDepositPool`, counted in `getTotalAssetDeposits`, and increases rsETH price for all existing holders.
5. The depositor holds 0 rsETH and has permanently lost 1 wei.

**Foundry test sketch:**
```solidity
function test_zeroMintDustDeposit() public {
    // Setup: rsETHPrice = 1.1e18, ETH assetPrice = 1e18
    vm.prank(user);
    depositPool.depositETH{value: 1}(0, "");
    assertEq(rsETH.balanceOf(user), 0); // user receives nothing
    assertGt(address(depositPool).balance, 0); // ETH absorbed
}
```
