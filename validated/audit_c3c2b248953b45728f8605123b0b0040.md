Audit Report

## Title
Stale Inflated `rsETHPrice` Broadcast to Destination Chains After Price-Drop Early Return — (`contracts/cross-chain/RSETHMultiChainRateProvider.sol`)

## Summary
When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it pauses the protocol and returns early, leaving `rsETHPrice` at its old, higher value. Because `MultiChainRateProvider.updateRate()` is permissionless and `RSETHMultiChainRateProvider.getLatestRate()` reads `rsETHPrice` with no pause check, any caller can immediately broadcast the stale inflated rate to all destination chains via LayerZero, causing users on those chains to receive fewer rsETH tokens than they are owed.

## Finding Description
**Root cause — stale storage after early return.**
In `LRTOracle._updateRsETHPrice()`, when `isPriceDecreaseOffLimit` is true (lines 277–281), the function pauses the protocol and returns immediately. The assignment `rsETHPrice = newRsETHPrice` at line 313 is never executed, so `rsETHPrice` retains the pre-drop value.

**No access control on `updateRate()`.**
`MultiChainRateProvider.updateRate()` (line 108) is declared `external payable nonReentrant` with no role check and no check that `LRTOracle` is paused. Any EOA or contract can call it at any time.

**`getLatestRate()` reads stale storage.**
`RSETHMultiChainRateProvider.getLatestRate()` (lines 26–28) calls `ILRTOracle(rsETHPriceOracle).rsETHPrice()`, which is a plain public state-variable getter. It does not inspect the `paused` flag on `LRTOracle` and returns the old, inflated value.

**Inflated rate broadcast.**
`updateRate()` encodes the stale rate and sends it via `ILayerZeroEndpoint.send()` to every registered `RateReceiver` (lines 115–136). The window persists for the entire paused period — until an admin manually unpauses and re-calls `updateRSETHPrice()`.

## Impact Explanation
Destination-chain pools receive an inflated rsETH/ETH rate. Users depositing ETH on those chains are quoted a higher price per rsETH than the actual collateral value warrants, so they receive fewer rsETH tokens than they are owed. The contract does not revert and does not directly lose custodied funds, but it fails to deliver the exchange rate it promises. **Impact: Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
- `updateRate()` is fully permissionless; no privilege or front-running is required.
- The early-return path is a normal operational event triggered by any sufficiently large price drop.
- The stale rate can be broadcast repeatedly by any address throughout the entire paused period.
- No victim mistake is required; any user depositing on a destination chain during the paused window is affected.

## Recommendation
Add a pause check inside `RSETHMultiChainRateProvider.getLatestRate()`:

```solidity
function getLatestRate() public view override returns (uint256) {
    require(!ILRTOracle(rsETHPriceOracle).paused(), "Oracle paused");
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

Alternatively, restrict `updateRate()` to a privileged role (e.g., `onlyOwner`) so it cannot be called by arbitrary actors during a pause event.

## Proof of Concept
```
Setup:
  - LRTOracle.rsETHPrice        = 1.1e18
  - LRTOracle.highestRsethPrice = 1.1e18
  - LRTOracle.pricePercentageLimit = 5e16  (5%)
  - Underlying collateral drops → newRsETHPrice = 1.0e18
    (diff = 0.1e18 > 5% of 1.1e18 = 0.055e18 → isPriceDecreaseOffLimit = true)

Trigger:
  1. Anyone calls LRTOracle.updateRSETHPrice()
     → _updateRsETHPrice() hits early-return at line 281
     → protocol paused; rsETHPrice still = 1.1e18

  2. Anyone calls RSETHMultiChainRateProvider.updateRate()
     → getLatestRate() returns 1.1e18 (stale)
     → LayerZero message sent to all destination chains with rate = 1.1e18

Assert:
  - Destination chain pools price rsETH at 1.1e18
  - True current value is 1.0e18
  - User depositing 1 ETH receives 1/1.1 ≈ 0.909 rsETH instead of 1.0 rsETH
  - Step 2 can be repeated by any address for the entire paused period
```
