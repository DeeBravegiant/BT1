### Title
Deterministic RNG Seed in Weight Proof Segment Selection Enables Targeted PoSpace Forgery - (File: chia/full_node/weight_proof.py)

### Summary
The weight proof validation uses `random.Random` seeded with `summaries[-2].get_hash()`, a value fully determined by the weight proof data the attacker themselves supply. An attacker constructing a fraudulent weight proof knows this seed before building the proof, allowing them to pre-compute exactly which segment per sub-epoch will have its proof-of-space (PoSpace) validated. They can then include valid PoSpace only for those pre-selected segments and fraudulent PoSpace for all others, reducing the farming power required to forge a weight proof that passes validation.

### Finding Description

In `validate_weight_proof_inner` and `validate_weight_proof_single_proc`, the RNG seed is derived from the second-to-last sub-epoch summary hash:

```python
seed = summaries[-2].get_hash()
rng = random.Random(seed)
``` [1](#0-0) [2](#0-1) 

The `summaries` list is reconstructed entirely from `weight_proof.sub_epochs`, data supplied by the peer sending the proof. While the final summary hash is verified against `recent_chain_data`, the second-to-last summary (used as the seed) is only implicitly committed through the chained hash structure — and the attacker, who constructs the entire fraudulent chain, knows all summary hashes before transmitting the proof.

This same `rng` is then passed into `_validate_sub_epoch_segments`, where it selects which segment per sub-epoch receives PoSpace validation:

```python
sampled_seg_index = rng.choice(range(len(segments)))
``` [3](#0-2) 

The `sampled_seg_index == idx` boolean is forwarded to `_validate_segment` to gate whether PoSpace is checked for that segment:

```python
for idx, segment in enumerate(segments):
    valid_segment, ip_iters, slot_iters, slots, vdf_list = _validate_segment(
        constants, segment, curr_ssi, prev_ssi, curr_difficulty,
        prev_ses, idx == 0,
        sampled_seg_index == idx,   # PoSpace only validated here
        height,
    )
``` [4](#0-3) 

For every segment where `sampled_seg_index != idx`, PoSpace is **not** validated. VDF proofs are still checked for all segments, but the proof-of-space — the component that proves the farmer actually held disk space — is skipped.

The same `rng` object is also reused after `validate_sub_epoch_sampling` for the segment-level selection, meaning the state of the PRNG at the point of `rng.choice(...)` is fully deterministic from the seed: [5](#0-4) 

### Impact Explanation

An unprivileged attacker who wants to trick a light-client wallet into syncing to a fraudulent chain can:

1. Construct a fraudulent chain with valid VDF proofs (requires computation, not privileged access).
2. Compute `summaries[-2].get_hash()` from their own chain data — this is fully known before sending the proof.
3. Simulate `random.Random(seed).choice(range(len(segments)))` offline for each sub-epoch to learn exactly which segment index will be selected.
4. Include a genuine PoSpace only for the selected segment per sub-epoch; populate all other segments with fabricated PoSpace data.
5. Send the crafted `WeightProof` to a syncing light client.

The validator accepts the proof because VDF proofs are valid and the one sampled segment per sub-epoch passes PoSpace validation. The attacker's effective farming-power requirement is reduced by a factor proportional to the number of segments per sub-epoch (bounded by `_max_sub_epoch_segments`), compared to what would be required if the selection were unpredictable. [6](#0-5) 

The result is **forged weight proof trust**: a light-client wallet (via `WalletWeightProofHandler.validate_weight_proof`) accepts a fraudulent chain as the canonical heaviest chain, enabling double-spend or transaction history rewriting against that client. [7](#0-6) 

### Likelihood Explanation

The attacker must:
- Compute valid VDF proofs for the fraudulent chain (expensive but feasible with dedicated hardware).
- Hold at least one valid plot that wins the challenge for the one selected segment per sub-epoch (much less farming power than winning all segments).
- Know the seed — which they do trivially, since it is derived from their own chain data.

No privileged access, leaked keys, or cryptographic breaks are required. The attack is reachable by any peer that can send a `WeightProof` message to a syncing node or wallet.

### Recommendation

The segment-selection seed must not be derivable by the party constructing the proof before the proof is committed. Options include:

1. **Verifier-supplied nonce**: The syncing node sends a random challenge nonce; the prover includes it in the proof; the seed becomes `H(summaries[-2] || nonce)`. The prover cannot pre-compute the seed without first committing to the proof structure.
2. **Fiat-Shamir over the full proof**: Seed the RNG with a hash of the entire serialized `WeightProof` (excluding the segment data), so the seed is fixed only after all segment data is committed.
3. **Interactive protocol**: Use a commit-reveal scheme where the verifier reveals the seed only after the prover has committed to the segment data.

rng = random.Random(seed)

# Simulate validate_sub_epoch_sampling consuming rng state
# (attacker replays the same calls the validator will make)

# Now predict sampled_seg_index for each sub-epoch
for sub_epoch_n, segments in segments_by_sub_epoch.items():
    sampled = rng.choice(range(len(segments)))
    # Only segment `sampled` needs real PoSpace; all others can be garbage
    for idx in range(len(segments)):
        if idx != sampled:
            segments[idx] = forge_pospace(segments[idx])  # fabricated PoSpace

# Send crafted weight_proof — validator accepts it
```

The attacker replays the exact PRNG call sequence the validator will execute (seeded identically), predicts `sampled_seg_index` for every sub-epoch, and fabricates PoSpace in all non-selected segments. The validator's `_validate_segment(..., sampled_seg_index == idx, ...)` call skips PoSpace for those segments, and the proof passes. [8](#0-7)

def _max_sub_epoch_segments(constants: ConsensusConstants) -> int:
    # SUB_EPOCH_BLOCKS // MIN_BLOCKS_PER_CHALLENGE_BLOCK challenge slots per
    # sub-epoch, plus one because the SES requires both the sub-epoch block
    # threshold AND deficit == 0, which may not coincide.  A challenge slot
    # can contain more than MIN_BLOCKS_PER_CHALLENGE_BLOCK blocks.
    max_blocks = constants.SUB_EPOCH_BLOCKS + constants.MIN_BLOCKS_PER_CHALLENGE_BLOCK - 1
    return (max_blocks - 1) // constants.MIN_BLOCKS_PER_CHALLENGE_BLOCK + 1
```

def _validate_sub_epoch_segments(
    constants: ConsensusConstants,
    rng: random.Random,
    weight_proof_bytes: bytes,
    summaries_bytes: list[bytes],
    height: uint32,
    validate_from: int = 0,
) -> list[tuple[VDFProof, ClassgroupElement, VDFInfo]] | None:
    summaries = summaries_from_bytes(summaries_bytes)
    sub_epoch_segments: SubEpochSegments = SubEpochSegments.from_bytes(weight_proof_bytes)
    rc_sub_slot_hash = constants.GENESIS_CHALLENGE
    total_blocks, total_ip_iters = 0, 0
    total_slot_iters, total_slots = 0, 0
    total_ip_iters = 0
    prev_ses: SubEpochSummary | None = None
    segments_by_sub_epoch = map_segments_by_sub_epoch(sub_epoch_segments.challenge_segments)
    curr_ssi = constants.SUB_SLOT_ITERS_STARTING
    vdfs_to_validate = []
    max_segments = _max_sub_epoch_segments(constants)
    for sub_epoch_n, segments in segments_by_sub_epoch.items():
        if len(segments) > max_segments:
            log.error(f"sub_epoch {sub_epoch_n} has {len(segments)} segments, maximum allowed is {max_segments}")
            return None
        prev_ssi = curr_ssi
        curr_difficulty, curr_ssi = _get_curr_diff_ssi(constants, sub_epoch_n, summaries)
        log.debug(f"validate sub epoch {sub_epoch_n}")
        # recreate RewardChainSubSlot for next ses rc_hash
        sampled_seg_index = rng.choice(range(len(segments)))
        if sub_epoch_n > 0:
            rc_sub_slot = __get_rc_sub_slot(constants, segments[0], summaries, curr_ssi)
            if rc_sub_slot is None:
                log.error(f"failed to reconstruct rc sub slot for sub_epoch {sub_epoch_n}")
                return None
            prev_ses = summaries[sub_epoch_n - 1]
            rc_sub_slot_hash = rc_sub_slot.get_hash()
        if not summaries[sub_epoch_n].reward_chain_hash == rc_sub_slot_hash:
            log.error(f"failed reward_chain_hash validation sub_epoch {sub_epoch_n}")
            return None

        # skip validation up to fork height
        if sub_epoch_n < validate_from:
            continue

        for idx, segment in enumerate(segments):
            valid_segment, ip_iters, slot_iters, slots, vdf_list = _validate_segment(
                constants,
                segment,
                curr_ssi,
                prev_ssi,
                curr_difficulty,
                prev_ses,
                idx == 0,
                sampled_seg_index == idx,
                height,
            )
            vdfs_to_validate.extend(vdf_list)
            if not valid_segment:
                log.error(f"failed to validate sub_epoch {segment.sub_epoch_n} segment {idx} slots")
                return None
