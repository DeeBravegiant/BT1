### Title
Missing Preservation-of-Value Check for Sub-Transactions Allows Unconstrained Value Creation — (`File: eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/Rules/SubUtxo.hs`)

---

### Summary

The Dijkstra era introduces nested ("sub") transactions. The `SUBUTXO` transition rule that validates each sub-transaction deliberately omits the preservation-of-value (`consumed == produced`) check that every other UTxO rule enforces. The top-level `UTXO` rule does call `validateValueNotConservedUTxO`, but it binds `consumed = conwayConsumed` — a Conway-era function that has no knowledge of sub-transactions and therefore does not aggregate sub-transaction inputs, withdrawals, or minting into the balance equation. The Dijkstra-specific aggregating function `getConsumedDijkstraValue` is wired to the separate `getConsumedValue` method, which is never invoked by the preservation-of-value check. The result is that a sub-transaction's value flows are never verified to balance, allowing an unprivileged transaction author to create ADA or native assets from nothing.

---

### Finding Description

**Root cause — `SUBUTXO` rule omits the balance check entirely**

`dijkstraSubUtxoTransition` validates validity intervals, input existence, output sizes, and network IDs, but contains no call to `validateValueNotConservedUTxO`: [1](#0-0) 

The conversion helper that maps top-level UTxO failures to sub-UTxO failures explicitly marks `ValueNotConservedUTxO` as `error "Impossible"`, confirming the omission is structural, not accidental: [2](#0-1) 

**Root cause — top-level check uses `conwayConsumed`, which ignores sub-transactions**

The top-level `dijkstraUtxoTransition` does call `validateValueNotConservedUTxO`: [3](#0-2) 

`validateValueNotConservedUTxO` resolves `consumed` through the `EraUTxO` class method. For `DijkstraEra`, that binding is: [4](#0-3) 

`consumed = conwayConsumed` is the Conway-era implementation. Conway predates sub-transactions entirely; `conwayConsumed` only sums the top-level transaction body's inputs, withdrawals, refunds, and minting. It has no path to `subTransactionsTxBodyL`.

**The Dijkstra-specific aggregating function is wired to a different method**

`getConsumedDijkstraValue` correctly folds sub-transaction consumed values: [5](#0-4) 

But it is bound to `getConsumedValue`, not to `consumed`. `validateValueNotConservedUTxO` calls `consumed`, so `getConsumedDijkstraValue` is never reached during the balance check.

Similarly, `dijkstraProducedValue` aggregates sub-transaction produced values: [6](#0-5) 

but it is bound to `getProducedValue`, not to the `produced` method used by `validateValueNotConservedUTxO`.

The net effect is that both sides of the balance equation in `validateValueNotConservedUTxO` are evaluated using Conway-era functions that see only the top-level transaction body. Sub-transaction inputs are consumed from the UTxO (via `updateUTxOStateNoFees`) and sub-transaction outputs are added to the UTxO, but the difference between those two quantities is never checked. [7](#0-6) 

---

### Impact Explanation

**Critical — Direct creation of ADA or native assets through an invalid ledger state transition.**

A sub-transaction author can set sub-transaction outputs to any value exceeding the sub-transaction's inputs. Because `updateUTxOStateNoFees` unconditionally removes the sub-transaction's inputs from the UTxO and adds its outputs: [8](#0-7) 

the UTxO gains the difference as newly created value. The top-level balance check does not see this difference because it uses `conwayConsumed` (top-level inputs only) on both sides. Honest nodes applying the same ledger rules will all accept the block, so the inflated UTxO becomes permanent ledger state. Native-asset minting policies are also bypassed for sub-transactions because the `SUBUTXO` rule does not enforce that the `mint` field is covered by a valid policy script — the minting policy check lives in `UTXOW`/`SUBUTXOW`, but the balance equation that would catch uncovered minting is absent.

---

### Likelihood Explanation

Any unprivileged transaction author can construct a Dijkstra-era transaction containing a sub-transaction. No special key, governance role, or consensus threshold is required. The attacker controls the sub-transaction body entirely, including its `outputs` and `mint` fields. The attack is deterministic and requires only the ability to submit a transaction to the network.

---

### Recommendation

1. **Add `validateValueNotConservedUTxO` to `dijkstraSubUtxoTransition`** using the sub-transaction's own inputs against `originalUtxo` and the sub-transaction's own outputs/deposits/minting. This mirrors how every prior era's UTxO rule enforces balance.

2. **Override `consumed` in the `EraUTxO DijkstraEra` instance** to use `getConsumedDijkstraValue` (or an equivalent that aggregates sub-transaction values) so that the top-level balance check also covers the aggregate batch. The current split between `consumed = conwayConsumed` and `getConsumedValue = getConsumedDijkstraValue` leaves the preservation-of-value check blind to sub-transaction value flows.

3. **Remove the `error "Impossible"` guard** on `ValueNotConservedUTxO` in `dijkstraUtxoToDijkstraSubUtxoPredFailure` once the check is added, so that sub-transaction balance failures are properly surfaced.

---

### Proof of Concept

1. Construct a Dijkstra-era top-level transaction `txTop` with a single sub-transaction `txSub`.
2. `txSub` spends one UTxO entry worth 1 ADA and declares a single output worth 1 000 000 ADA (or mints 1 000 000 units of a native asset without a corresponding policy script in the `mint` field).
3. `txTop` itself is balanced normally (its own inputs equal its own outputs plus fee).
4. Submit `txTop` to the network.
5. `dijkstraSubUtxoTransition` validates `txSub`: inputs exist in the UTxO ✓, outputs are not too small ✓, outputs are not too big ✓, network IDs match ✓ — no balance check is performed.
6. `updateUTxOStateNoFees` removes the 1 ADA input and adds the 1 000 000 ADA output.
7. `dijkstraUtxoTransition` calls `validateValueNotConservedUTxO` on `txTop`'s body using `conwayConsumed`, which sees only `txTop`'s own inputs and outputs — the sub-transaction's imbalance is invisible.
8. The block is accepted by all honest nodes; 999 999 ADA has been created from nothing.

```haskell
getConsumedDijkstraValue ::
  forall era l.
  ( DijkstraEraTxBody era
  , EraUTxO era
  , Value era ~ MaryValue
  , STxLevel l era ~ STxBothLevels l era
  ) =>
  PParams era ->
  (Credential Staking -> Maybe Coin) ->
  (Credential DRepRole -> Maybe Coin) ->
  UTxO era ->
  TxBody l era ->
  Value era
getConsumedDijkstraValue pp lookupStakingDeposit lookupDRepDeposit utxo txBody =
  withBothTxLevels
    txBody
    ( \topTxBody ->
        txBodyConsumedValue topTxBody <> subTransactionsConsumedValue topTxBody
    )
    txBodyConsumedValue
  where
    txBodyConsumedValue :: forall m. TxBody m era -> Value era
    txBodyConsumedValue = getConsumedMaryValue pp lookupStakingDeposit lookupDRepDeposit utxo
    subTransactionsConsumedValue topTxBody =
      foldMap'
        (getConsumedValue pp lookupStakingDeposit lookupDRepDeposit utxo . view bodyTxL)
        (topTxBody ^. subTransactionsTxBodyL)
```

```haskell
-- | Ensure that value consumed and produced matches up exactly
--
-- > consumed pp utxo txb = produced pp poolParams txb
validateValueNotConservedUTxO ::
  (EraUTxO era, EraCertState era) =>
  PParams era ->
  UTxO era ->
  CertState era ->
  TxBody TopTx era ->
  Test (ShelleyUtxoPredFailure era)
validateValueNotConservedUTxO pp utxo certState txBody =
  failureUnless (consumedValue == producedValue) $
    ValueNotConservedUTxO Mismatch {mismatchSupplied = consumedValue, mismatchExpected = producedValue}
  where
    consumedValue = consumed pp certState utxo txBody
    producedValue = produced pp certState txBody
```

```haskell
updateUTxOStateNoFees pp utxos txBody certState govState depositChangeEvent txUtxODiffEvent = do
  let UTxOState {utxosUtxo, utxosDeposited, utxosFees, utxosDonation} = utxos
      UTxO utxo = utxosUtxo
      !utxoAdd = txouts txBody -- These will be inserted into the UTxO
      {- utxoDel  = txins txb ◁ utxo -}
      !(utxoWithout, utxoDel) = extractKeys utxo (txBody ^. inputsTxBodyL)
      {- newUTxO = (txins txb ⋪ utxo) ∪ outs txb -}
      newUTxO = utxoWithout `Map.union` unUTxO utxoAdd
      deletedUTxO = UTxO utxoDel
      totalRefunds = certsTotalRefundsTxBody pp certState txBody
      totalDeposits = certsTotalDepositsTxBody pp certState txBody
      depositChange = totalDeposits <-> totalRefunds
  depositChangeEvent depositChange
  txUtxODiffEvent deletedUTxO utxoAdd
  pure $!
    UTxOState
      { utxosUtxo = UTxO newUTxO
      , utxosDeposited = utxosDeposited <> depositChange
      , utxosFees = utxosFees
      , utxosGovState = govState
      , utxosInstantStake =
          deleteInstantStake deletedUTxO (addInstantStake utxoAdd (utxos ^. instantStakeL))
      , utxosDonation = utxosDonation
      }
```
