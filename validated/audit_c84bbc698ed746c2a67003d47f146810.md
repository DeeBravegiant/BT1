### Title
Witness Count Not Validated Against Input Count — `zip` Truncation Allows Unwitnessed Inputs to Pass — (`eras/byron/ledger/impl/src/Cardano/Chain/UTxO/Validation.hs`)

---

### Summary

`updateUTxOTxWitness` uses Haskell's `zip` to pair input addresses with witnesses. `zip` silently truncates to the shorter list. There is no check anywhere that `length witness == length (txInputs tx)`. An attacker can submit a transaction with N inputs but fewer than N witnesses; the unwitnessed inputs are silently accepted.

---

### Finding Description

In `updateUTxOTxWitness`: [1](#0-0) 

```haskell
mapM_
  (uncurry $ validateWitness pmi sigData)
  (zip addresses (V.toList witness))
```

`addresses` has exactly `|txInputs tx|` elements. `witness` is an attacker-supplied `Vector TxInWitness` with no enforced length. [2](#0-1) 

Haskell's `zip` stops at the shorter list. If the attacker provides `k < N` witnesses, only the first `k` inputs are validated. The remaining `N - k` inputs are silently accepted with no witness check.

No other function enforces the invariant. `validateTx` checks inputs exist in the UTxO and network magic. `validateTxAux` checks fee, size, and balance. Neither checks `length witness == length inputs`. [3](#0-2) [4](#0-3) 

---

### Impact Explanation

**Critical — direct theft of ADA through an invalid ledger state transition.**

Concrete exploit:
- Tx has 2 inputs: `input_1` from `addr_A` (attacker-controlled), `input_2` from `addr_B` (victim).
- Attacker provides `witness = [witness_A]` (length 1, valid only for `addr_A`).
- `zip [addr_A, addr_B] [witness_A]` → `[(addr_A, witness_A)]`.
- `validateWitness` is called exactly once, for `addr_A`. It passes.
- `addr_B`'s input is never validated. The UTxO update proceeds, consuming both inputs and crediting the attacker's outputs.
- The attacker spends `addr_B`'s funds with no valid witness.

Regarding the specific same-address scenario in the question (two inputs from `addr_A`, one witness duplicated): the `zip` truncation does allow 1 witness to cover N inputs from the same address, violating the protocol invariant that N witnesses are required for N inputs. However, since the attacker already owns `addr_A`, this sub-case does not enable theft on its own. The critical theft path is the cross-address case above, enabled by the same root cause.

---

### Likelihood Explanation

The exploit is fully unprivileged. The attacker only needs to:
1. Know a victim UTxO entry (`addr_B`, value).
2. Control one UTxO entry at any address (`addr_A`).
3. Construct a valid `ATxAux` with 2 inputs and a 1-element witness vector.

This is constructable locally, requires no governance access, no key leakage, and no network majority.

---

### Recommendation

Add an explicit length guard before the `zip` in `updateUTxOTxWitness`:

```haskell
let nInputs   = length addresses
    nWitnesses = V.length witness
nInputs == nWitnesses
  `orThrowError` UTxOValidationTxValidationError
      (TxValidationWitnessWrongKey ... )  -- or a dedicated error constructor
```

Alternatively, use `zipExact` / `zipWithM` that fails on length mismatch, or assert `V.length witness == NE.length (txInputs tx)` before the `zip`. [5](#0-4) 

---

### Proof of Concept

```
1. Pick victim UTxO: (txid_B, ix_B) → addr_B, 100 ADA
2. Pick attacker UTxO: (txid_A, ix_A) → addr_A, 1 ADA
3. Build Tx:
     inputs  = [(txid_A, ix_A), (txid_B, ix_B)]
     outputs = [TxOut attacker_addr 100_ADA]   -- fee aside
4. Compute sigData = hash(Tx)
5. Build witness vector (length 1):
     witness = [VKWitness vk_A (sign sk_A sigData)]
6. Submit ATxAux{tx, witness} under TxValidation mode.
7. zip [addr_A, addr_B] [witness_A]
     → only (addr_A, witness_A) validated → passes
     → addr_B input never checked → passes
8. UTxO updated: both inputs consumed, attacker receives 100 ADA.
```

updateUTxOTxWitness env utxo ta = do
  whenTxValidation $ do
    -- Get the signing addresses for each transaction input from the 'UTxO'
    addresses <-
      mapM (`UTxO.lookupAddress` utxo) (NE.toList $ txInputs tx)
        `wrapError` UTxOValidationUTxOError

    -- Validate witnesses and their signing addresses
    mapM_
      (uncurry $ validateWitness pmi sigData)
      (zip addresses (V.toList witness))
      `wrapError` UTxOValidationTxValidationError

    -- Validate the tx including witnesses
    validateTxAux env utxo ta
      `wrapError` UTxOValidationTxValidationError

  -- Update 'UTxO' ignoring witnesses
  updateUTxOTx env utxo aTx
  where
    Environment {protocolMagic} = env
    pmi = getAProtocolMagicId protocolMagic

    aTx@(Annotated tx _) = aTaTx ta
    witness = taWitness ta
    sigData = recoverSigData aTx
