### Title
Missing Zero-Address Validation on `ProposalInit.builder` Allows Malicious Proposer to Corrupt Sequencer Address, Block Hash, and Fee Accounting — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` never checks that `ProposalInit.builder` is a valid (non-zero) contract address. Because `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, a malicious proposer can broadcast `builder = 0x0`, which passes every validator check, causes all block fees to be transferred to address `0x0`, and embeds a zero sequencer address into the block hash.

---

### Finding Description

`ProposalInit` carries a `builder: ContractAddress` field — the address of the entity that sequences the block. [1](#0-0) 

During protobuf deserialization, the field is converted via `TryFrom<StarkHash> for ContractAddress`, which delegates to `PatriciaKey::try_from`. That conversion only rejects values ≥ `CONTRACT_ADDRESS_DOMAIN_SIZE` (2²⁵¹); `Felt::ZERO` is well within range and is accepted without error. [2](#0-1) 

`ContractAddress::validate()` — the method that enforces the lower bound `> 0x1` — is **never called** on `builder` during proposal validation. [3](#0-2) 

`is_proposal_init_valid` checks timestamp, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, L1 gas prices, and `fee_proposal`, but performs **no check on `init.builder`**: [4](#0-3) 

After validation passes, `convert_to_sn_api_block_info` assigns `init.builder` directly to `sequencer_address`: [5](#0-4) 

`sequencer_address` is then used in three critical downstream paths:

1. **Fee transfer** — every transaction's fee is sent to `sequencer_address` as the ERC-20 recipient: [6](#0-5) 

2. **Block hash** — `sequencer_address` is chained into the Poseidon block hash: [7](#0-6) [8](#0-7) 

3. **`is_sequencer_the_sender` guard** — the zero address changes the outcome of the concurrency fee-transfer path: [9](#0-8) 

---

### Impact Explanation

**Critical — Incorrect fee/balance accounting with economic impact; wrong block hash committed.**

When `builder = 0x0` is accepted:

- All transaction fees for the block are transferred to address `0x0` via the ERC-20 `transfer` call. The legitimate sequencer receives nothing; the fees are effectively burned.
- The block hash is computed with `sequencer = 0x0` instead of the real sequencer address, producing a permanently wrong on-chain commitment.
- The state diff records fee-token balance updates at address `0x0`, corrupting the canonical state.

---

### Likelihood Explanation

**Medium.** The attacker must be an elected consensus proposer for a round (i.e., a validator in the active set). Once elected, no additional privilege is required — the malicious `ProposalInit` is broadcast over the normal P2P channel, passes all existing `is_proposal_init_valid` checks, and is accepted by every honest validator node. The attack is repeatable every time the malicious validator is elected proposer.

---

### Recommendation

Add an explicit address-validity check for `init.builder` (and `init.proposer`) inside `is_proposal_init_valid`, mirroring the check already applied to gateway transaction sender addresses:

```rust
// In is_proposal_init_valid, after the existing checks:
init_proposed.builder.validate().map_err(|_| {
    ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("builder address is invalid: {:?}", init_proposed.builder),
    )
})?;
init_proposed.proposer.validate().map_err(|_| {
    ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!("proposer address is invalid: {:?}", init_proposed.proposer),
    )
})?;
```

`ContractAddress::validate()` already enforces the range `(0x1, L2_ADDRESS_UPPER_BOUND)`, which excludes both the zero address and the reserved block-hash-table address. [3](#0-2) 

---

### Proof of Concept

1. A malicious validator is elected proposer for round R at height H.
2. It constructs a `ProposalInit` with all fields valid except `builder = ContractAddress::default()` (`0x0`).
3. It broadcasts the proposal over the P2P stream.
4. Each honest validator calls `is_proposal_init_valid` — all checks pass (timestamp, gas prices, version, etc.); `builder` is never checked. [10](#0-9) 
5. `initiate_validation` calls `convert_to_sn_api_block_info`, producing `BlockInfo { sequencer_address: 0x0, … }`. [11](#0-10) 
6. The batcher executes all transactions with `sequencer_address = 0x0`; every fee transfer sends tokens to `0x0`. [6](#0-5) 
7. `PartialBlockHashComponents::new` chains `0x0` as the sequencer into the block hash. [7](#0-6) 
8. Consensus reaches decision; the block is committed with a wrong hash and all fees burned to `0x0`.

```rust
#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
```
