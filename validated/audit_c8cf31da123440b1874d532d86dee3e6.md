Audit Report

## Title
Uninitialized `rsETHPrice` in `LRTOracle` Causes Division-by-Zero on All Deposit Paths — (File: `contracts/LRTDepositPool.sol`)

## Summary
`LRTOracle.initialize()` sets only `lrtConfig` and never initializes `rsETHPrice`, leaving it at the Solidity default of `0`. `LRTDepositPool.getRsETHAmountToMint` unconditionally divides by `lrtOracle.rsETHPrice()`, so every call to `depositETH` and `depositAsset` reverts with an arithmetic panic (`0x12`) until `updateRSETHPrice()` is explicitly called post-deployment. No funds are lost, but the protocol's primary entry point is completely non-functional during this window.

## Finding Description
`LRTOracle.initialize()` (L64–68) only assigns `lrtConfig` and emits an event; `rsETHPrice` is never written, so it remains `0`. The only code path that sets `rsETHPrice` is `_updateRsETHPrice()` (L214–316), which is invoked by the public `updateRSETHPrice()` (L87–89) or the manager-gated `updateRSETHPriceAsManager()` (L94–96). When `rsethSupply == 0` (the state at fresh deployment), `_updateRsETHPrice()` sets `rsETHPrice = 1 ether` and returns early (L218–222), but this function is never called inside `initialize()`.

`LRTDepositPool.getRsETHAmountToMint` (L506–521) computes:
```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```
with no guard against `rsETHPrice == 0`. Both `depositETH` (L76–93) and `depositAsset` (L99–118) call `_beforeDeposit` (L648–670), which calls `getRsETHAmountToMint` (L665). Solidity 0.8.x raises an arithmetic panic revert (`0x12`) on integer division by zero, so every deposit attempt fails until `updateRSETHPrice()` is called externally.

No existing check in `_beforeDeposit` or `getRsETHAmountToMint` guards against a zero `rsETHPrice`.

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

All user deposits via `depositETH` and `depositAsset` are completely blocked from the moment the contract is deployed until `updateRSETHPrice()` is called. No user funds are at risk of theft or permanent loss; the protocol's core deposit functionality is simply non-operational during this window.

## Likelihood Explanation
The condition is triggered on every fresh deployment or proxy upgrade that resets `rsETHPrice` to `0`. Because `updateRSETHPrice()` is `public` with no role restriction (only `whenNotPaused`), any caller — including the depositor themselves — can unblock the protocol by calling it first. The window is therefore bounded by the time between deployment and the first call to `updateRSETHPrice()`, but it is structurally guaranteed to exist on every deployment.

## Recommendation
At the end of `LRTOracle.initialize()`, call `_updateRsETHPrice()` (or directly assign `rsETHPrice = 1 ether` when `rsethSupply == 0`) to ensure `rsETHPrice` is valid before any dependent function can be called. This eliminates the uninitialized-state window entirely:

```solidity
function initialize(address lrtConfigAddr) external initializer {
    UtilLib.checkNonZeroAddress(lrtConfigAddr);
    lrtConfig = ILRTConfig(lrtConfigAddr);
    emit UpdatedLRTConfig(lrtConfigAddr);
    _updateRsETHPrice(); // ensures rsETHPrice = 1 ether at genesis
}
```

## Proof of Concept
1. Deploy `LRTOracle` proxy and call `initialize(lrtConfigAddr)`. `rsETHPrice` is `0`.
2. Deploy `LRTDepositPool` proxy and call `initialize(lrtConfigAddr)`.
3. Any user calls `depositETH{value: 1 ether}(0, "")`.
4. Execution reaches `_beforeDeposit` → `getRsETHAmountToMint(ETH_TOKEN, 1 ether)`.
5. `lrtOracle.rsETHPrice()` returns `0`.
6. Solidity executes `(1e18 * assetPrice) / 0` → arithmetic panic revert `0x12`.
7. All deposits fail identically until `updateRSETHPrice()` is called externally.

**Foundry test sketch:**
```solidity
function test_depositRevertsBeforeUpdateRSETHPrice() public {
    // deploy proxies, initialize both contracts, do NOT call updateRSETHPrice()
    assertEq(lrtOracle.rsETHPrice(), 0);
    vm.expectRevert(); // arithmetic panic 0x12
    lrtDepositPool.depositETH{value: 1 ether}(0, "");
    // call updateRSETHPrice() and confirm deposit succeeds
    lrtOracle.updateRSETHPrice();
    assertEq(lrtOracle.rsETHPrice(), 1 ether);
    lrtDepositPool.depositETH{value: 1 ether}(0, ""); // succeeds
}
```
