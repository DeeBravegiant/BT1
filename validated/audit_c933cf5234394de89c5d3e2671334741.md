### Title
Proposer-supplied `builder` address in `ProposalInit` is not validated by the consensus validator, allowing arbitrary `sequencer_address` in committed blocks — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The `builder` field of `ProposalInit` is proposer-controlled and flows directly into `sequencer_address` inside the block's `BlockInfo` without any bounds or identity check in `is_proposal_init_valid`. A malicious consensus proposer can set `builder` to any arbitrary `ContractAddress`. Because both the proposer's and every validator's batcher execute transactions under the same `block_info` (derived from the received `ProposalInit`), all nodes converge on the same `partial_block_hash` and vote the block through. The committed block therefore carries a fabricated `sequencer_address` in its header and execution context.

---

### Finding Description

`is_proposal_init_valid` in `crates/apollo_consensus_orchestrator/src/validate_proposal.rs` enforces the following fields of `ProposalInit`:

| Field | Check |
|---|---|
| `height` | exact match |
| `l1_da_mode` | exact match |
| `l2_gas_price_fri` | exact match |
| `starknet_version` | exact match |
| `version_constant_commitment` | exact match (sentinel) |
| `timestamp` | window bounds |
| `l1_gas_price_fri/wei`, `l1_data_gas_price_fri/wei` | within-margin |
| `fee_proposal_fri` | within `fee_actual` bounds (when window is full) | [1](#0-0) 

The `builder` field is **absent from every check**. It is never compared against any locally-derived expected value, and `ProposalInitValidation` contains no `builder` member: [2](#0-1) 

After `is_proposal_init_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` verbatim to `sequencer_address`: [3](#0-2) 

The resulting `BlockInfo` is passed to the batcher's `validate_block` call, which uses it as the authoritative block context for all transaction execution: [4](#0-3) 

Because every validator receives the same `ProposalInit.builder` and feeds it into their own batcher, all nodes execute transactions under the same (fabricated) `sequencer_address`. They therefore compute identical `partial_block_hash` values and reach consensus on the corrupted block.

The same `builder` value is also forwarded to `finalize_decision` → `update_state_sync_with_new_block` and `prepare_blob_for_next_height`, so the wrong `sequencer_address` is persisted in state-sync storage and the centralized cende blob: [5](#0-4) 

---

### Impact Explanation

**Critical — Wrong execution result from syscall/execution logic for accepted input.**

Any contract that calls the `get_sequencer_address` / `get_execution_info` syscall during the affected block receives the fabricated address instead of the real sequencer address. This corrupts:

1. **Execution results and receipts** — contracts that branch on `get_sequencer_address` (e.g., fee-token contracts, access-control logic, or any contract that whitelists the sequencer) produce wrong state transitions, wrong events, and wrong return values.
2. **Block header stored in state** — `sequencer_address` in `BlockHeaderWithoutHash` is wrong for the committed block, which propagates to every downstream consumer (RPC `starknet_getBlockWithTxs`, state-sync peers, provers).
3. **Cende blob / L1 anchoring** — the blob sent to the centralized recorder carries the wrong `sequencer_address`, corrupting the off-chain record used for proof generation. [6](#0-5) 

---

### Likelihood Explanation

Any node that wins a proposer slot in the BFT rotation can trigger this. No special privilege beyond being a consensus participant is required — the same role that the M-03 analogue (a MergingPool reward winner) occupies in the original report. The attack requires no off-chain coordination and leaves no detectable trace in the `ProposalCommitment` (since `builder` is not hashed into it).

---

### Recommendation

1. Add `expected_builder: ContractAddress` to `ProposalInitValidation` and populate it from the node's own configuration (the same source used in `initiate_build`).
2. In `is_proposal_init_valid`, assert `init_proposed.builder == proposal_init_validation.expected_builder`, mirroring the existing exact-match checks for `height`, `l1_da_mode`, and `l2_gas_price_fri`.
3. If the builder address is legitimately variable (e.g., delegated building), maintain a validator-side allowlist and check membership. [7](#0-6) 

---

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
