Audit Report

## Title
Unbounded Nested Loop in `getAssetUnstaking()` Scales Gas Cost of All User-Facing Deposit and Price-Update Paths - (File: contracts/NodeDelegator.sol)

## Summary
`NodeDelegator.getAssetUnstaking()` iterates over all EigenLayer-queued withdrawals and all strategies per withdrawal via a nested loop, making external calls in each iteration. This function is invoked for every NDC in `LRTDepositPool.getAssetDistributionData()` and `getETHDistributionData()`, which are themselves called for every supported asset in `LRTOracle._getTotalEthInProtocol()`. As the protocol scales through legitimate operator activity, the multiplicative gas cost of `depositETH()`, `depositAsset()`, and `updateRSETHPrice()` can exceed the block gas limit, temporarily freezing deposits and price updates.

## Finding Description

**Root cause:** `NodeDelegator.getAssetUnstaking()` fetches the full list of queued withdrawals from EigenLayer's `DelegationManager` and iterates over every withdrawal and every strategy within each withdrawal, making external calls (`strategy.underlyingToken()`, `strategy.sharesToUnderlyingView()`) inside the inner loop: [1](#0-0) 

This function is called once per NDC inside `LRTDepositPool.getAssetDistributionData()`: [2](#0-1) 

And again once per NDC inside `getETHDistributionData()`: [3](#0-2) 

`LRTOracle._getTotalEthInProtocol()` calls `getTotalAssetDeposits(asset)` — which calls `getAssetDistributionData()` — for **every supported asset**: [4](#0-3) 

`getTotalAssetDeposits()` is the aggregator that calls `getAssetDistributionData()`: [5](#0-4) 

**Public entry points that trigger the full chain:**

- `LRTOracle.updateRSETHPrice()` — callable by anyone with no access control beyond `whenNotPaused`: [6](#0-5) 

- `LRTDepositPool.depositETH()` and `depositAsset()` — callable by any depositor. Both call `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()`: [7](#0-6) 

**Why existing guards are insufficient:**

`maxNodeDelegatorLimit` is initialized to 10 but is admin-updatable with no upper bound: [8](#0-7) [9](#0-8) 

There is no cap on the number of queued withdrawals returned by EigenLayer's `DelegationManager.getQueuedWithdrawals()`. The number of strategies per withdrawal is also unbounded at the contract level. No caching or lazy-accounting mechanism exists; the full loop executes on every read.

**Gas cost formula:** For `updateRSETHPrice()`: `(supported assets) × (NDC count) × (queued withdrawals per NDC) × (strategies per withdrawal)` external calls. For `depositETH()`/`depositAsset()`: `(NDC count) × (queued withdrawals per NDC) × (strategies per withdrawal)` external calls per deposit, for the single deposited asset.

## Impact Explanation

When the product of the multipliers grows large enough, every call to `depositETH()`, `depositAsset()`, and `updateRSETHPrice()` reverts with out-of-gas. This temporarily freezes all new deposits and prevents rsETH price updates. A stale rsETH price in turn blocks the withdrawal unlock flow that depends on a current `rsETHPrice`. This matches the allowed impact **Medium — Temporary Freezing of Funds**.

## Likelihood Explanation

No adversarial action is required. The protocol is designed to scale: governance adds more supported assets, admins add more NDCs to distribute EigenLayer delegation, and operators routinely queue withdrawals as part of normal restaking operations. All three multipliers grow through legitimate protocol usage. The condition arises organically as the protocol matures. Likelihood is **Medium** — it requires the protocol to reach a certain scale, but there is no on-chain mechanism preventing it.

## Recommendation

1. **Lazy accounting:** Track `assetUnstaking` in a storage variable updated on `initiateUnstaking()` and `completeUnstaking()` rather than recomputing it by iterating EigenLayer's queue on every read.
2. **Single-pass per NDC:** Restructure `getAssetDistributionData()` so that `getAssetUnstaking()` is called once per NDC across all assets (returning a map), rather than once per `(NDC × asset)` pair.
3. **Enforce explicit caps:** Validate `maxNodeDelegatorLimit` and `maxUncompletedWithdrawalCount` against a gas budget at configuration time, and enforce an on-chain upper bound in `updateMaxNodeDelegatorLimit()`.

## Proof of Concept

1. Admin adds 5 supported assets to `LRTConfig`.
2. Admin adds 10 NDCs to `LRTDepositPool` (`maxNodeDelegatorLimit = 10`).
3. Operator calls `initiateUnstaking()` on each NDC until each NDC accumulates `W` queued withdrawals, each containing `K` strategies (e.g., W=20, K=5).
4. Any user calls `updateRSETHPrice()`.
5. The call chain `updateRSETHPrice → _updateRsETHPrice → _getTotalEthInProtocol → getTotalAssetDeposits (×5 assets) → getAssetDistributionData → getAssetUnstaking` executes the nested loop `5 × 10 × 20 × 5 = 5,000` iterations, each making 1–2 external calls to EigenLayer strategy contracts.
6. At sufficient scale, the transaction reverts with out-of-gas; no price update can be made.
7. Any user calling `depositETH(1 ether, "")` triggers `_beforeDeposit → _checkIfDepositAmountExceedesCurrentLimit → getTotalAssetDeposits(ETH) → getETHDistributionData → getAssetUnstaking (×10 NDCs × W × K)`, also reverting with out-of-gas.
8. All deposits and price updates are frozen until the number of queued withdrawals naturally decreases (withdrawals complete), with no protocol-level remedy available.

```text
    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
```
