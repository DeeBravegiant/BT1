### Title
Unbounded `fee_proposal_fri` During Startup Window Permanently Inflates L2 Gas Price - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

During the first `fee_proposal_window_size` blocks after V0_14_3 activation (or network startup), the `fee_proposal_fri` bounds check inside `is_proposal_init_valid` is unconditionally skipped because `fee_actual` is `None`. A malicious proposer can set `fee_proposal_fri` to an arbitrarily large value. Those values are recorded verbatim into `fee_proposals_window`, which is the sole input to `compute_fee_actual`. Once the window fills, `fee_actual` is anchored to the inflated median, and every subsequent block's `l2_gas_price` is derived from that inflated baseline. Because the per-block margin is small (parts-per-thousand), the fee cannot be corrected quickly, causing all users to pay inflated L2 gas fees for hundreds or thousands of blocks.

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` enforces that a proposer's `fee_proposal_fri` lies within a geometric band around `fee_actual`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

The guard fires only when **both** `fee_actual` and `fee_proposal` are `Some`. `fee_actual` is computed by `compute_fee_actual`, which returns `None` whenever the sliding window has fewer than `window_size` entries or any entry is `None`:

During the startup window, every validator computes `fee_actual = None` and passes it into `ProposalInitValidation.fee_actual`:

```rust
fee_actual: compute_fee_actual(
    &self.fee_proposals_window,
    init.height,
    VersionedConstants::latest_constants().fee_proposal_window_size,

``` [3](#0-2) 

Because `fee_actual` is `None`, the `if let` pattern in `is_proposal_init_valid` never matches, and any `fee_proposal_fri` value — including `u128::MAX` — passes validation.

The accepted value is then stored permanently in `fee_proposals_window` at `finalize_decision`:

```rust
self.record_fee_proposal(height, init.fee_proposal_fri);
``` [4](#0-3) 

Once the window fills, `fee_actual` becomes the median of the inflated values. `proposal_commitment_from` binds `fee_proposal_fri` into the `ProposalCommitment`:

```rust
pub(crate) fn proposal_commitment_from(
    partial: PartialBlockHash,
    fee_proposal: Option<GasPrice>,
) -> ProposalCommitment {
    let Some(fee_proposal) = fee_proposal else {
        return ProposalCommitment(partial.0);
    };
    ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
}
``` [5](#0-4) 

The inflated `fee_actual` then feeds `calculate_next_l2_gas_price_for_fin`, which sets `l2_gas_price` for the next block. Because `l2_gas_price_fri` is validated with an exact equality check:

```rust
&& init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri
``` [6](#0-5) 

all honest validators independently compute the same inflated `l2_gas_price` and accept proposals that carry it, locking in the inflated fee level.

### Impact Explanation

A malicious proposer who controls any block during the first `window_size` blocks can set `fee_proposal_fri = u128::MAX`. After the window fills, `fee_actual` is anchored to a very high value. Because the per-block margin is only a few parts-per-thousand, correcting the fee back to a normal level requires hundreds to thousands of blocks. During this period all users pay inflated L2 gas fees; at the extreme (`u128::MAX`), no transaction can afford the fee, causing a complete economic denial-of-service. This matches the impact category: **Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact**.

### Likelihood Explanation

The attack window is the first `fee_proposal_window_size` blocks after V0_14_3 activates (or after any network restart that clears the window). Any validator who is scheduled as proposer during that window can execute the attack without any special privilege beyond the normal proposer role assigned by the Tendermint round-robin. The attack is silent — the inflated `fee_proposal_fri` passes all existing validation checks — and its effect is durable.

### Recommendation

Enforce an absolute upper bound on `fee_proposal_fri` even when `fee_actual` is `None`. During the startup window, clamp `fee_proposal_fri` to a configurable maximum (e.g., `max_l2_gas_price` from `VersionedConstants` or a multiple of the current `l2_gas_price`). This prevents a proposer from seeding the window with extreme values before the median-based guard becomes active.

```rust
// Proposed addition before the existing fee_actual guard:
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    if fee_proposal.0 > MAX_ALLOWED_FEE_PROPOSAL {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
```

Alternatively, seed the window with the genesis `l2_gas_price` for all heights below `window_size` so that `fee_actual` is always `Some` and the existing bounds check is always active.

### Proof of Concept

1. Network launches at height 0 with V0_14_3 (`fee_proposal_window_size = W`).
2. Attacker is scheduled as proposer for block 0 (or any block `< W`).
3. Attacker sets `fee_proposal_fri = u128::MAX` in `ProposalInit`.
4. `is_proposal_init_valid` is called; `proposal_init_validation.fee_actual = None` because `compute_fee_actual` returns `None` for an empty window. The `if let (Some(fee_actual), Some(fee_proposal))` guard does not fire. Proposal is accepted.
5. `finalize_decision` calls `record_fee_proposal(height, Some(u128::MAX))`, inserting the inflated value into `fee_proposals_window`.
6. Steps 2–5 repeat for each block the attacker proposes within `[0, W)`.
7. At height `W`, `compute_fee_actual` returns `Some(median)` where the median is dominated by `u128::MAX` entries.
8. `calculate_next_l2_gas_price_for_fin` sets `l2_gas_price` to a value derived from this inflated `fee_actual`.
9. All subsequent `ProposalInit` messages carry the inflated `l2_gas_price_fri`; validators accept them because they independently compute the same inflated value.
10. All user transactions that specify resource bounds below the inflated `l2_gas_price` are rejected, causing a sustained economic denial-of-service.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
