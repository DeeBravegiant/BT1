Audit Report

## Title
`RSETHPriceFeed.latestRoundData()` Returns Stale `rsETHPrice` With Fresh ETH/USD `updatedAt` During Oracle Pause - (File: `contracts/oracles/RSETHPriceFeed.sol`)

## Summary
`RSETHPriceFeed.latestRoundData()` computes the rsETH/USD price by multiplying the stored `rsETHPrice` from `LRTOracle` by the live ETH/USD Chainlink price, but returns `updatedAt` sourced solely from the ETH/USD feed. When `LRTOracle` auto-pauses on a downward price deviation, `rsETHPrice` is frozen while `updatedAt` continues advancing with the ETH/USD heartbeat, causing the feed to silently serve stale composite prices that pass standard Chainlink staleness checks.

## Finding Description
`RSETHPriceFeed.latestRoundData()` (L63–70) forwards all five return values from `ETH_TO_USD.latestRoundData()` and then overwrites only `answer` with `int256(RS_ETH_ORACLE.rsETHPrice()) * answer / 1e18`. The `updatedAt` field is never touched and reflects only when the ETH/USD Chainlink feed last updated.

`RS_ETH_ORACLE.rsETHPrice()` reads `LRTOracle.rsETHPrice`, a plain `uint256` storage variable (L28) that is only written inside `_updateRsETHPrice()` (L313). That function is reached exclusively through `updateRSETHPrice()` (public, `whenNotPaused`, L87) or `updateRSETHPriceAsManager()` (manager-only, L94).

Inside `_updateRsETHPrice()`, when the computed new price drops more than `pricePercentageLimit` below `highestRsethPrice`, the function calls `_pause()` and returns immediately (L277–281) — **before** writing `rsETHPrice = newRsETHPrice` (L313). From that point:
- `paused = true` causes every subsequent `updateRSETHPrice()` call to revert at the `whenNotPaused` modifier.
- `rsETHPrice` is frozen at its pre-pause value.
- The ETH/USD Chainlink feed continues updating on its normal heartbeat.
- Every call to `latestRoundData()` returns a fresh `updatedAt` (seconds old) paired with a stale `rsETHPrice` (potentially hours or days old).

Any consumer that checks `block.timestamp - updatedAt < threshold` will pass the staleness guard and consume an incorrect rsETH/USD price. The same structural defect exists in `getRoundData()` (L53–61), which additionally applies the **current** `rsETHPrice` to a **historical** ETH/USD round.

## Impact Explanation
`RSETHPriceFeed` is a Chainlink `AggregatorV3Interface`-compatible contract whose sole purpose is to deliver an accurate, timely rsETH/USD price with correct staleness metadata. During any oracle pause — which is triggered automatically by on-chain price movement — the contract returns a `updatedAt` timestamp that is current while the `answer` contains a stale rsETH component. The contract fails to deliver its promised return without any on-chain signal to consumers. This matches the allowed Low impact: **Contract fails to deliver promised returns, but doesn't lose value** (value loss, if any, occurs in external integrators, not within this contract).

## Likelihood Explanation
The pause trigger is fully permissionless: `updateRSETHPrice()` is a public function callable by any EOA or contract. No special role is required to initiate the call. If market conditions cause the computed rsETH price to drop beyond `pricePercentageLimit` relative to `highestRsethPrice`, the pause fires automatically. This is a designed safety mechanism, not an exotic edge case. Once paused, the stale-timestamp condition persists until an admin manually calls `unpause()`, which has no time bound. The condition is therefore repeatable whenever market volatility exceeds the configured threshold.

## Recommendation
Introduce a `rsETHPriceUpdatedAt` storage variable in `LRTOracle` that is written to `block.timestamp` each time `rsETHPrice` is successfully updated (at L313 of `_updateRsETHPrice()`). Expose it via the `IRSETHOracle` interface. In `RSETHPriceFeed.latestRoundData()`, replace the forwarded `updatedAt` with `min(ethUpdatedAt, rsETHPriceUpdatedAt)`. This ensures that any consumer performing a standard Chainlink staleness check correctly detects when either price component is stale.

## Proof of Concept
1. Deploy `LRTOracle` with a non-zero `pricePercentageLimit` (e.g., 1%).
2. Call `updateRSETHPrice()` once to establish `rsETHPrice` and `highestRsethPrice`.
3. Manipulate underlying asset prices (or mock the price fetcher) so that the next computed rsETH price is more than 1% below `highestRsethPrice`.
4. Call `updateRSETHPrice()` — `_updateRsETHPrice()` detects `isPriceDecreaseOffLimit = true`, calls `_pause()`, and returns without updating `rsETHPrice`. `paused = true`.
5. Advance block time by several hours (simulating ETH/USD heartbeat updates).
6. Call `RSETHPriceFeed.latestRoundData()`:
   - `updatedAt` = current ETH/USD feed timestamp (seconds ago — passes any reasonable staleness check).
   - `answer` = frozen pre-pause `rsETHPrice` × current ETH/USD price (rsETH component hours stale).
7. Assert `block.timestamp - updatedAt < 3600` passes (staleness check succeeds) while `rsETHPrice` has not changed since step 2.
