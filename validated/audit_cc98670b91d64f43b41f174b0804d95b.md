### Title
Single Byzantine Participant Can Permanently Block Key Generation or Resharing via `vote_abort_key_event_instance` - (File: `crates/contract/src/state/key_event.rs`)

### Summary

The `vote_abort` function in `KeyEvent` — exposed publicly as `vote_abort_key_event_instance` — allows **any single attested participant** to immediately abort the current key generation or resharing instance without threshold consensus. This is the direct analog of the Salty H-05 pattern: a state-manipulation call that a sub-threshold actor can use to perpetually block a critical protocol operation.

### Finding Description

`KeyEvent::vote_abort` in `crates/contract/src/state/key_event.rs` unconditionally sets `self.instance = None` upon a single valid participant call:

```rust
pub fn vote_abort(&mut self, key_event_id: KeyEventId) -> Result<(), Error> {
    let candidate = self.verify_vote(&key_event_id)?;
    if self.instance.as_ref().unwrap().completed.contains(&candidate) {
        return Err(VoteError::VoteAlreadySubmitted.into());
    }
    self.instance = None;   // ← single-participant abort, no threshold
    Ok(())
}
``` [1](#0-0) 

The only guard is that the caller must be an attested participant who has not yet cast a `vote_success` for this attempt. The public entry point `vote_abort_key_event_instance` enforces `assert_caller_is_attested_participant_and_protocol_active()`, which confirms the caller is a registered, TEE-attested participant — but imposes no threshold requirement on the abort itself. [2](#0-1) 

The attack loop:

1. Contract enters `Initializing` (key generation) or `Resharing` state.
2. The leader calls `start_keygen_instance` / `start_reshare_instance`, creating a `KeyEventInstance` with `expires_on = block_height + 1 + timeout_blocks`.
3. A Byzantine participant immediately calls `vote_abort_key_event_instance` with the current `KeyEventId` (public on-chain state). `instance` is set to `None`.
4. The leader must call `start` again (incrementing `next_attempt_id`). The Byzantine participant aborts again.
5. This repeats indefinitely — the honest majority can never accumulate the required votes because the instance is destroyed before they can all vote. [3](#0-2) 

The `start` function itself is correctly leader-gated and checks for an active instance, so the leader cannot race the abort. The Byzantine participant only needs to submit one transaction per attempt, which is far cheaper than the leader's coordination overhead. [4](#0-3) 

### Impact Explanation

**For first-time key generation (`Initializing` state):** The system has no usable keys. If key generation is permanently blocked, the MPC network can never produce threshold signatures. Any funds on foreign chains that were intended to be controlled by derived MPC keys are permanently inaccessible. This matches the "permanent freezing of funds controlled by the MPC network" criterion.

**For resharing (`Resharing` state):** Existing keys remain usable (the contract falls back to the previous keyset for signing), but the participant set can never be updated. A Byzantine outgoing participant can block their own removal indefinitely, or a Byzantine incoming participant can block the resharing that would give them a keyshare — preventing governance from ever completing.

The `vote_cancel_keygen` / `vote_cancel_resharing` escape hatches require threshold votes from honest participants, but the Byzantine participant can simply abort every new attempt started after the cancel, creating an indefinite loop. [5](#0-4) 

### Likelihood Explanation

- Requires only **one** attested participant (strictly below threshold).
- The `KeyEventId` needed to call `vote_abort_key_event_instance` is fully public via the `state()` view method.
- The cost is a single NEAR transaction per attempt — negligible.
- The attack is sustainable indefinitely with no economic barrier.
- A participant being removed in a resharing has a direct financial incentive to block the resharing.

### Recommendation

Require threshold consensus to abort a key event instance, mirroring the threshold requirement already applied to `vote_cancel_keygen` and `vote_cancel_resharing`:

```rust
// In KeyEvent::vote_abort (or a new vote_abort accumulator):
self.abort_votes.insert(candidate);
if self.abort_votes.len() >= self.parameters.threshold().value() as usize {
    self.instance = None;
    self.abort_votes.clear();
}
```

Alternatively, remove `vote_abort_key_event_instance` entirely and rely solely on the existing block-height timeout (`expires_on`) to clear stalled instances, since honest nodes already send `vote_abort_key_event_instance` only on genuine local failures. [6](#0-5) 

### Proof of Concept

```
1. Contract is in Initializing state (first key generation, no existing keys).
2. Leader (lowest participant ID) calls start_keygen_instance({epoch_id: E, domain_id: D, attempt_id: 0}).
   → instance = Some(KeyEventInstance { attempt_id: 0, expires_on: H+101, ... })
3. Byzantine participant P calls vote_abort_key_event_instance({epoch_id: E, domain_id: D, attempt_id: 0}).
   → verify_vote passes (P is a participant, hasn't voted success yet)
   → instance = None
4. Leader calls start_keygen_instance({..., attempt_id: 1}).
5. Byzantine participant calls vote_abort_key_event_instance({..., attempt_id: 1}).
   → instance = None again
6. Steps 4–5 repeat indefinitely.
Result: Contract is permanently stuck in Initializing state. No threshold signatures can ever be produced.
```

    /// The leader is always the participant with the lowest participant ID.
    pub fn start(&mut self, key_event_id: KeyEventId, timeout_blocks: u64) -> Result<(), Error> {
        self.cleanup_if_timed_out();
        if self.instance.is_some() {
            return Err(KeyEventError::ActiveKeyEvent.into());
        }
        let expected_key_event_id =
            KeyEventId::new(self.epoch_id, self.domain.id, self.next_attempt_id);
        if key_event_id != expected_key_event_id {
            return Err(KeyEventError::KeyEventIdMismatch.into());
        }
        self.verify_leader()?;
        self.instance = Some(KeyEventInstance::new(self.next_attempt_id, timeout_blocks));
        self.next_attempt_id = self.next_attempt_id.next();
        Ok(())
    }
