Audit Report

## Title
Missing `msg.sender == account` Validation Allows Anyone to Force-Claim on Behalf of Any User - (File: contracts/utils/MerkleDistributor/MerkleDistributor.sol)

## Summary

`MerkleDistributor.claim()` accepts a caller-supplied `account` parameter with no check that `msg.sender == account`. Any external caller can supply a victim's public merkle proof and force-trigger their claim at the current `feeInBPS` rate, permanently redirecting the fee portion to `protocolTreasury`. The sibling contract `KernelMerkleDistributor` explicitly guards against this in `_processClaim()`, confirming the protocol's own recognition that the check is required.

## Finding Description

`MerkleDistributor.claim()` performs merkle proof verification against the supplied `account` but never validates that the caller is that account:

```solidity
// contracts/utils/MerkleDistributor/MerkleDistributor.sol L97-121
function claim(
    uint256 index,
    address account,       // caller-supplied, never validated against msg.sender
    uint256 cumulativeAmount,
    bytes32[] calldata merkleProof
) external override whenNotPaused {
    ...
    bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
    if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
        revert InvalidMerkleProof();
    }
``` [1](#0-0) 

After proof verification, the fee is deducted and tokens are transferred:

```solidity
// L137-144
uint256 fee = (claimableAmount * feeInBPS) / 10_000;
uint256 amountToSend = claimableAmount - fee;
IERC20(token).safeTransfer(account, amountToSend);
IERC20(token).safeTransfer(protocolTreasury, fee);
``` [2](#0-1) 

The merkle proof for any `account` is derivable from the publicly posted off-chain merkle tree. An attacker can call `claim(index, victimAddress, cumulativeAmount, victimProof)` and the call succeeds unconditionally. The victim's `userClaims[account].lastClaimedIndex` is updated, consuming the claim slot.

By contrast, `KernelMerkleDistributor._processClaim()` explicitly enforces:

```solidity
// contracts/KERNEL/KernelMerkleDistributor.sol L311-313
if (account != msg.sender) {
    revert Unauthorized();
}
``` [3](#0-2) 

This guard is absent entirely from `MerkleDistributor`. The `MAX_FEE_IN_BPS` is 1000 (10%), meaning up to 10% of a user's claimable tokens can be permanently redirected to `protocolTreasury` by a force-claim. [4](#0-3) 

## Impact Explanation

**Low. Contract fails to deliver promised returns, but doesn't lose value.**

The concrete worst case: `feeInBPS` is non-zero (up to 10%) and the owner has announced or is about to execute a fee reduction via `setFeeInBPS`. An attacker front-runs the fee reduction by force-claiming for all users at the current higher fee. The fee portion (`claimableAmount * feeInBPS / 10_000`) is permanently sent to `protocolTreasury` rather than remaining claimable by the user after the fee drops. The user receives fewer tokens than they would have received by waiting. Their claim slot is consumed; they cannot reclaim the lost fee portion until the next merkle root update with a higher `cumulativeAmount`. [5](#0-4) 

## Likelihood Explanation

High. The attack requires no special privileges, no funds, and no private information. The merkle proof for any `account` is derivable from the publicly posted merkle tree. Any external caller can execute this against any user at any time the contract is unpaused. The only precondition for financial harm beyond griefing is a non-zero `feeInBPS`, which is a configurable and expected operational state.

## Recommendation

Add the same guard already present in `KernelMerkleDistributor._processClaim()` to `MerkleDistributor.claim()`:

```solidity
if (account != msg.sender) revert Unauthorized();
```

This requires adding `Unauthorized` to the `IMerkleDistributor` interface error list. The same fix should be applied to `MerkleBlastPointsDistributor.claim()` for consistency, though its impact there is limited to event-based griefing since no tokens are transferred. [6](#0-5) 

## Proof of Concept

1. Alice holds a valid merkle proof for `(index=5, account=alice, cumulativeAmount=1000e18)`. Current `feeInBPS = 1000` (10%). Owner submits a transaction to call `setFeeInBPS(0)`.
2. Bob (attacker) reads Alice's proof from the off-chain merkle tree and front-runs the fee reduction:
   ```solidity
   merkleDistributor.claim(5, alice, 1000e18, aliceProof);
   ```
3. The call succeeds with no revert. Alice receives `900e18` tokens; `100e18` goes to `protocolTreasury`.
4. `userClaims[alice].lastClaimedIndex` is set to `5` and `cumulativeAmount` to `1000e18`. Alice's claim slot is consumed.
5. The owner's `setFeeInBPS(0)` executes. Alice cannot reclaim the `100e18` lost to the fee — she would have received the full `1000e18` had she been allowed to claim after the fee dropped.

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeBlastPointAmount,
        uint256 cumulativeBlastGoldAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node =
            keccak256(abi.encodePacked(index, account, cumulativeBlastPointAmount, cumulativeBlastGoldAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }

        // Calculate the claimable amount
        uint256 claimableBlastPoints = cumulativeBlastPointAmount - userClaims[account].cumulativeBlastPointAmount;
        uint256 claimableBlastGold = cumulativeBlastGoldAmount - userClaims[account].cumulativeBlastGoldAmount;

        // Ensure there is something to claim
        if (claimableBlastPoints == 0 && claimableBlastGold == 0) {
            revert NoPointsToClaim();
        }

        // Update user claim info
        userClaims[account].lastClaimedIndex = index;
        userClaims[account].cumulativeBlastPointAmount = cumulativeBlastPointAmount;
        userClaims[account].cumulativeBlastGoldAmount = cumulativeBlastGoldAmount;

        emit Claimed(index, account, claimableBlastPoints, claimableBlastGold);
    }
