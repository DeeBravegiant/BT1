### Title
Missing `state_diff_commitment` Verification in P2P Sync Allows Malicious Peer to Permanently Corrupt `declared_classes_block` Table — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The p2p sync client assembles a `ThinStateDiff` from peer-supplied chunks and validates only the item count (`state_diff_length`) against the stored block header. It never computes or compares the Poseidon `state_diff_commitment` hash. A malicious peer can therefore inject an arbitrary `DeclaredClass` entry into any block's state diff (provided the total item count matches). Because `append_state_diff` inserts into `declared_classes_block` only when the key is absent, the forged block number is written permanently. When the canonical declaration for the same class hash arrives at a later block, the idempotent guard silently skips the update, leaving the wrong block number in the table forever. Every subsequent call to `is_declared` and `get_class_definition_block_number` for that class hash returns the attacker-chosen block number.

---

### Finding Description

**Missing commitment verification in `parse_data_for_block`**

`StateDiffStreamBuilder::parse_data_for_block` reads `target_state_diff_len` from the stored block header and rejects chunks whose cumulative length does not match. It never reads `block_header.state_diff_commitment` and never calls `calculate_state_diff_hash`. [1](#0-0) 

After assembling the diff, `write_to_storage` calls `append_state_diff` directly with no commitment check. [2](#0-1) 

**Idempotent insert in `append_state_diff`**

For every `(class_hash, _)` in `thin_state_diff.class_hash_to_compiled_class_hash`, the storage layer inserts into `declared_classes_block` only when the key is absent. A key written by a forged diff at block N is never overwritten when the canonical diff at block M > N is processed. [3](#0-2) 

**Downstream consumers read the corrupted value**

`get_class_definition_block_number` returns the raw value from `declared_classes_block_table` with no further validation. [4](#0-3) 

`is_declared` in `ApolloReader` (used by the blockifier during execution) compares this value against `state_number.0`. If the forged block N < canonical block M, the class appears declared at every state number in `[N, M)` when it should not be. [5](#0-4) 

`get_class_definition_at` uses the same table for its block-number guard, so it too returns the class body at state numbers `[N, M)` once the class body is written at block M. [6](#0-5) 

The RPC `get_class` and `get_compiled_class` handlers both gate access on `get_class_definition_block_number`, so they serve the class at the wrong historical state. [7](#0-6) [8](#0-7) 

---

### Impact Explanation

After the canonical class body is written at block M, any execution or RPC query that resolves the class at a `StateNumber` in `[N, M)` will:

- `is_declared` → `true` (should be `false`): blockifier allows a `declare` transaction to be treated as a re-declaration or allows a `deploy_account`/`invoke` to use the class before it was legitimately on-chain.
- `get_class_definition_at` → returns the Sierra class body (should return `None`): RPC `starknet_getClass` serves the class at a block where it did not exist.
- `get_compiled_class` → returns CASM (should return `CLASS_HASH_NOT_FOUND`): RPC `starknet_getCompiledCasm` serves compiled code at the wrong historical point.

The corruption is permanent for the lifetime of the node's database because no re-sync of already-processed blocks occurs and no revert is triggered for canonical blocks.

---

### Likelihood Explanation

The attacker must be a p2p peer that the victim node connects to. P2P peers are explicitly low-trust in the sync model. The attacker must craft a diff whose total item count equals the header's `state_diff_length` (trivially achievable by substituting one legitimate item for a forged `DeclaredClass` chunk). The attacker must target a class hash that will be canonically declared at a future block; this is observable from the mempool or L1 `declare` transactions before they are finalized. No privileged access is required.

---

### Recommendation

After assembling the full `ThinStateDiff` from peer chunks, compute its Poseidon hash and compare it against the `state_diff_commitment` stored in the block header before calling `write_to_storage`. The function `calculate_state_diff_hash` already exists for this purpose. [9](#0-8) 

The check should be inserted at the end of `parse_data_for_block`, after `validate_deprecated_declared_classes_non_conflicting`, analogous to the existing verification in `apollo_committer`. [10](#0-9) 

---

### Proof of Concept

```
1. Victim node starts p2p sync. Block 0 header is stored with
   state_diff_length = 1 and state_diff_commitment = H(canonical_diff_0).

2. Attacker peer responds to the state diff query for block 0 with a single
   DeclaredClass chunk { class_hash: X, compiled_class_hash: fake_casm }.
   Length = 1 == target_state_diff_len → passes validation.
   declared_classes_block[X] = BlockNumber(0) is written.

3. Victim node later syncs block 5. The canonical diff for block 5 contains
   DeclaredClass { class_hash: X, compiled_class_hash: real_casm }.
   append_state_diff checks: declared_classes_block.get(X) = Some(0) → not_declared = false → skips insert.
   declared_classes_block[X] remains BlockNumber(0).

4. After block 5's class body is written:
   get_class_definition_block_number(X) == Some(BlockNumber(0))  // wrong, should be Some(5)
   is_declared(X) at StateNumber(3) == true                      // wrong, should be false
   get_class_definition_at(StateNumber(3), X) == Some(sierra)    // wrong, should be None
```

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
