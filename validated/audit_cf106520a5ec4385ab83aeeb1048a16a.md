Audit Report

## Title
Chainlink Price Staleness Indicators Ignored in `ChainlinkPriceOracle.getAssetPrice()` - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `updatedAt` and `answeredInRound`, accepting stale prices without any validity check. A stale price that falls below `highestRsethPrice` by more than `pricePercentageLimit` will cause `_updateRsETHPrice()` to pause `LRTDepositPool` and `LRTWithdrawalManager`, temporarily freezing all user funds. The same codebase applies correct staleness checks in `ChainlinkOracleForRSETHPoolCollateral`, confirming the omission is unintentional.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` fetches the price as:

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values (`roundId`, `answer`, `startedAt`, `updatedAt`, `answeredInRound`) are available per the interface declared at lines 14–17, but only `answer` is consumed. [1](#0-0) 

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` in the same repository validates both `answeredInRound < roundID` (stale round) and `timestamp == 0` (incomplete round): [2](#0-1) 

The stale price propagates through the following call chain:

1. `LRTOracle.updateRSETHPrice()` (public, no access control) → `_updateRsETHPrice()` [3](#0-2) 
2. `_updateRsETHPrice()` → `_getTotalEthInProtocol()` → `getAssetPrice(asset)` → `ChainlinkPriceOracle.getAssetPrice()` [4](#0-3) 
3. `newRsETHPrice` is computed from the stale total ETH value [5](#0-4) 
4. If `newRsETHPrice` drops more than `pricePercentageLimit` below `highestRsethPrice`, both `lrtDepositPool` and `withdrawalManager` are paused [6](#0-5) 

No existing guard in `ChainlinkPriceOracle` or `LRTOracle` checks the age or round completeness of the returned price before it is used in TVL computation.

## Impact Explanation
**Medium — Temporary freezing of funds.** When a Chainlink feed goes stale and the last reported price is sufficiently below the true market price, any unprivileged caller invoking `updateRSETHPrice()` causes `LRTDepositPool.pause()` and `LRTWithdrawalManager.pause()` to execute, freezing all deposits and withdrawals until an admin manually unpauses. This matches the allowed impact "Temporary freezing of funds."

Secondary **Low — Contract fails to deliver promised returns**: a stale price above the true market price inflates `rsETHPrice`, causing depositors to receive fewer rsETH tokens than entitled and withdrawers to receive excess assets.

## Likelihood Explanation
Chainlink feeds are known to go stale during Ethereum network congestion, L2 sequencer downtime, or oracle node failures — historically observed events. `updateRSETHPrice()` is a public function with no access control, so any external account can call it at the exact moment a stale price is live, locking in the bad price before the oracle recovers. The attacker needs no funds, no special role, and no victim cooperation. The condition is repeatable whenever a feed lags.

## Recommendation
Apply the same staleness checks already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0) revert IncompleteRound();
    if (price <= 0) revert InvalidPrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Optionally add a configurable `heartbeat` threshold check (`block.timestamp - updatedAt > heartbeat`) for feeds with known update frequencies.

## Proof of Concept
1. Deploy a mock Chainlink aggregator that returns a price 10% below the true market price with a stale `updatedAt` (e.g., 25 hours ago) and `answeredInRound == roundId - 1`.
2. Register the mock feed in `ChainlinkPriceOracle` for a supported LST asset.
3. Call `LRTOracle.updateRSETHPrice()` from any EOA.
4. `_getTotalEthInProtocol()` uses the stale 10%-low price → `newRsETHPrice` is ~10% below `highestRsethPrice`.
5. If `pricePercentageLimit` is set to 5% (5e16), the condition `diff > pricePercentageLimit.mulWad(highestRsethPrice)` evaluates to true.
6. `lrtDepositPool.pause()` and `withdrawalManager.pause()` are called — all deposits and withdrawals are frozen.
7. Confirm: `lrtDepositPool.paused() == true`, `withdrawalManager.paused() == true`.

Foundry fork test: fork mainnet, replace the Chainlink feed address stored in `assetPriceFeed` with a mock returning a stale answer, call `updateRSETHPrice()`, and assert both contracts are paused.
