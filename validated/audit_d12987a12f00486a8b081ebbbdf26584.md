### Title
Unprivileged `TransferToGasKey` Can Grief Gas Key and Account Deletion - (File: `runtime/runtime/src/access_keys.rs`)

### Summary
Any unprivileged user can send a `TransferToGasKey` action to any account, funding a gas key they do not own. Both `delete_gas_key` and `action_delete_account` gate deletion on a hard balance threshold (`GasKeyInfo::MAX_BALANCE_TO_BURN = 1 NEAR`). An attacker who funds a victim's gas key above this threshold causes the victim's `DeleteKey` and `DeleteAccount` receipts to fail with `GasKeyBalanceTooHigh`, mirroring the GEB pattern where an external actor manipulates a balance check to block a critical protocol operation.

### Finding Description

`delete_gas_key` in `runtime/runtime/src/access_keys.rs` enforces:

```rust
if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
    result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh { ... }.into());
    return Ok(());
}
``` [1](#0-0) 

`action_delete_account` in `runtime/runtime/src/actions.rs` enforces the same check over the aggregate of all gas keys:

```rust
let gas_key_balance_to_burn = compute_gas_key_balance_sum(state_update, account_id)?;
if gas_key_balance_to_burn > GasKeyInfo::MAX_BALANCE_TO_BURN {
    result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh { ... }.into());
    return Ok(());
}
``` [2](#0-1) 

The threshold is exactly 1 NEAR: [3](#0-2) 

`action_transfer_to_gas_key` is dispatched with `account_id` set to the **receipt receiver**, not the transaction signer. There is no ownership check — any sender can target any account: [4](#0-3) 

The `TransferToGasKeyAction` struct carries only a public key and a deposit amount; no authorization field exists: [5](#0-4) 

The host function `promise_batch_action_transfer_to_gas_key` also allows contracts to fund gas keys on arbitrary accounts: [6](#0-5) 

### Impact Explanation

An attacker who sends a `TransferToGasKey` receipt to a victim's account with `deposit > 1 NEAR` causes every subsequent `DeleteKey` (for that gas key) and `DeleteAccount` receipt to fail with `GasKeyBalanceTooHigh`. The corrupted protocol value is the **action execution outcome** (`ExecutionStatus::Failure`) and the **account state** (gas key and account cannot be removed). The victim's balance is not stolen, but their ability to reclaim storage and close their account is blocked.

The victim can work around the attack by combining `WithdrawFromGasKey` and `DeleteKey`/`DeleteAccount` in a single transaction (atomic receipt), which drains the balance before the threshold check runs. However, users who submit a bare `DeleteKey` or `DeleteAccount` without the preceding withdrawal will receive an unexpected failure, and the attacker can continuously re-fund the key to maintain the block.

### Likelihood Explanation

The attack costs the attacker slightly more than 1 NEAR per targeted gas key (the funds are burned on eventual deletion, so the attacker loses them). The entry path is a standard signed transaction submitted via public RPC — no privileged access is required. The victim's `DeleteKey` and `DeleteAccount` transactions fail silently with an action error, which may not be immediately understood as an external attack.

### Recommendation

Before enforcing the `MAX_BALANCE_TO_BURN` threshold in `delete_gas_key` and `action_delete_account`, automatically transfer the gas key balance back to the account (analogous to the GEB fix of calling `settleDebt` before the balance check). This eliminates the externally manipulable precondition. Alternatively, document that `WithdrawFromGasKey` must precede `DeleteKey`/`DeleteAccount` in the same receipt when a gas key carries a non-trivial balance, and enforce this at the protocol level rather than relying on user awareness.

### Proof of Concept

1. Victim `alice.near` owns a gas key `GK` with balance 0.
2. Attacker `bob.near` submits:
   ```
   SignedTransaction {
     signer_id: "bob.near",
     receiver_id: "alice.near",
     actions: [TransferToGasKeyAction { public_key: GK, deposit: 1_000_000_000_000_000_000_000_001 }]
   }
   ```
3. `action_transfer_to_gas_key` runs on `alice.near`'s state; `GK.balance` becomes `> 1 NEAR`.
4. Alice submits `DeleteKey { public_key: GK }`.
5. `delete_gas_key` checks `gas_key_info.balance > MAX_BALANCE_TO_BURN` → true → returns `GasKeyBalanceTooHigh`. The key is not deleted.
6. Same outcome for `DeleteAccount` if the aggregate gas key balance exceeds 1 NEAR. [7](#0-6) [2](#0-1)

fn delete_gas_key(
    config: &RuntimeConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
) -> Result<(), RuntimeError> {
    if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
        result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh {
            account_id: account_id.clone(),
            public_key: Some(Box::new(public_key.clone())),
            balance: gas_key_info.balance,
        }
        .into());
        return Ok(());
    }
    result.tokens_burnt =
        result.tokens_burnt.checked_add(gas_key_info.balance).ok_or(IntegerOverflowError)?;
    let num_nonces = gas_key_info.num_nonces as usize;
    for i in 0..gas_key_info.num_nonces {
        remove_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i);
    }
    let nonce_key_len = gas_key_nonce_key_len(account_id, &public_key.into());
    let nonce_remove_compute = storage_removes_compute(
        &config.wasm_config.ext_costs,
        num_nonces,
        nonce_key_len * num_nonces,
        AccessKey::NONCE_VALUE_LEN * num_nonces,
    );
    result.compute_usage = safe_add_compute(result.compute_usage, nonce_remove_compute)?;
    remove_access_key(state_update, account_id.clone(), public_key.clone());
    account.set_storage_usage(account.storage_usage().saturating_sub(gas_key_storage_cost(
        &config.fees,
        public_key,
        access_key,
        gas_key_info.num_nonces,
    )));
    Ok(())
}

pub fn promise_batch_action_transfer_to_gas_key(
    ctx: &mut Ctx,
    memory: &mut [u8],
    promise_idx: u64,
    public_key_len: u64,
    public_key_ptr: u64,
    amount_ptr: u64,
) -> Result<()> {
    ctx.result_state.gas_counter.pay_base(base)?;
    if ctx.context.is_view() {
        return Err(HostError::ProhibitedInView {
            method_name: "promise_batch_action_transfer_to_gas_key".to_string(),
        }
        .into());
    }
    let public_key = get_public_key(
        &mut ctx.result_state.gas_counter,
        memory,
        &ctx.registers,
        public_key_ptr,
        public_key_len,
    )?;
    let amount =
        Balance::from_yoctonear(get_u128(&mut ctx.result_state.gas_counter, memory, amount_ptr)?);
    let (receipt_idx, sir) = promise_idx_to_receipt_idx_with_sir(ctx, promise_idx)?;
    let receiver_id = ctx.ext.get_receipt_receiver(receipt_idx);
    let send = gas_key_transfer_send_fee(&ctx.fees_config, sir, public_key_len as usize);
    let exec =
        gas_key_transfer_exec_fee(&ctx.fees_config, receiver_id.len(), public_key_len as usize);
    let burn_base = send.base;
    let use_base = burn_base.gas.checked_add(exec.base.gas).ok_or(HostError::IntegerOverflow)?;
    ctx.result_state.gas_counter.pay_action_accumulated(
        burn_base,
        use_base,
        ActionCosts::gas_key_transfer_base,
    )?;
    let burn_byte = send.per_byte;
    let use_byte =
        burn_byte.gas.checked_add(exec.per_byte.gas).ok_or(HostError::IntegerOverflow)?;
    ctx.result_state.gas_counter.pay_action_accumulated(
        burn_byte,
        use_byte,
        ActionCosts::gas_key_byte,
    )?;
    ctx.result_state.deduct_balance(amount)?;
    ctx.ext.append_action_transfer_to_gas_key(receipt_idx, public_key.decode()?, amount);
    Ok(())
