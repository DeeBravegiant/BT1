Audit Report

## Title
Unguarded Near-Zero Chainlink Price Commits Corrupted `rsETHPrice` When `pricePercentageLimit == 0`, Enabling Mint-at-Near-Zero-Price Attack — (`contracts/oracles/ChainlinkPriceOracle.sol`, `contracts/LRTOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` has no runtime minimum price floor. Because `pricePercentageLimit` defaults to `0` and is never set in `LRTOracle.initialize()`, the downside circuit-breaker in `_updateRsETHPrice()` is permanently disabled: a near-zero Chainlink answer causes `rsETHPrice` to be committed as near-zero with no pause triggered. Once the feed recovers, any depositor calling `depositAsset()` receives a massively inflated rsETH mint, diluting and effectively stealing from all existing holders.

## Finding Description

**Root cause 1 — No runtime price floor in `ChainlinkPriceOracle.getAssetPrice()`:**

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L52-54
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

No staleness check, no negative-price guard, no minimum floor. `price = 1` with `decimals = 8` returns `1e10` (0.00000001 ETH) and passes silently. The one-time sanity check in `updatePriceOracleForValidated()` (lines 102–107) only runs at oracle-setup time and does not protect against runtime price drift.

**Root cause 2 — `pricePercentageLimit` defaults to `0`, permanently disabling the downside circuit-breaker:**

`LRTOracle.initialize()` (lines 64–68) never sets `pricePercentageLimit`. It defaults to `0`. In `_updateRsETHPrice()`:

```solidity
// contracts/LRTOracle.sol L273-274
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
```

When `pricePercentageLimit == 0`, `isPriceDecreaseOffLimit` is always `false` regardless of how catastrophic the price drop is. Execution falls through unconditionally to:

```solidity
// contracts/LRTOracle.sol L313
rsETHPrice = newRsETHPrice;   // near-zero price committed
```

No pause is triggered, no early return, and the corrupted price is stored.

**Root cause 3 — `updateRSETHPrice()` is public and permissionless:**

```solidity
// contracts/LRTOracle.sol L87
function updateRSETHPrice() public whenNotPaused {
```

Any external caller can invoke this during a bad-price window.

**Exploit flow:**

1. A Chainlink LST/ETH feed transiently returns a near-zero answer (e.g., LST depegging event, feed misconfiguration — historically documented for stETH, rETH).
2. Attacker calls `updateRSETHPrice()`. `_getTotalEthInProtocol()` multiplies asset balances by the near-zero price, yielding a near-zero `totalETHInProtocol`. `newRsETHPrice` is computed as near-zero. Since `pricePercentageLimit == 0`, `isPriceDecreaseOffLimit` is `false`, no pause fires, and `rsETHPrice = newRsETHPrice` (near-zero) is committed.
3. Feed recovers to normal.
4. Attacker calls `depositAsset()`. `getRsETHAmountToMint()` computes:
   ```solidity
   rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
   // = (1e18 * 1e18) / 1e10 = 1e26 rsETH per 1 LST deposited
   ```
   The numerator uses the live (recovered) oracle price; the denominator uses the stale near-zero stored `rsETHPrice`.
5. Attacker holds an overwhelming share of rsETH supply and redeems it for the protocol's real collateral.

**Why existing guards fail:**

- `updatePriceOracleForValidated()`: one-time setup check only, not a runtime guard.
- `isPriceDecreaseOffLimit`: permanently `false` when `pricePercentageLimit == 0` (the default, unset state).
- `RSETH.checkDailyMintLimit`: caps daily mints, but `maxMintAmountPerDay` must be set large for normal protocol operation; it limits but does not prevent the attack.

## Impact Explanation

**Critical — Direct theft of user funds / Protocol insolvency.**

An attacker depositing 1 LST (1e18 wei) when `rsETHPrice = 1e10` and `getAssetPrice = 1e18` receives `1e26` rsETH against a total existing supply of ~1e24. The attacker holds the overwhelming majority of rsETH supply and can redeem it for the protocol's real collateral, constituting direct theft from all existing holders and rendering the protocol insolvent.

## Likelihood Explanation

- `pricePercentageLimit == 0` is the **default state** after `initialize()`. No admin action is needed to be in the vulnerable state; an admin action (`setPricePercentageLimit`) is required to escape it.
- `updateRSETHPrice()` is **public and permissionless** — any external caller can invoke it during a bad-price window.
- LST depegging events (stETH May 2022, rETH) and Chainlink feed misconfiguration are historically documented real-world occurrences; no oracle operator compromise is required.
- The attacker does not need to control the feed; they only need to observe the bad-price window and act opportunistically.

## Recommendation

1. **Add a runtime minimum price floor in `ChainlinkPriceOracle.getAssetPrice()`**: revert if `price <= 0` or if the normalized price falls below a configurable minimum (e.g., `1e16`).
2. **Enforce `pricePercentageLimit > 0` at initialization**, or add a hard-coded minimum price drop guard in `_updateRsETHPrice()` that does not depend on `pricePercentageLimit` being set (e.g., revert or pause if `newRsETHPrice < rsETHPrice / 2`).
3. **Add a minimum `newRsETHPrice` sanity check** before committing it (e.g., revert or pause if `newRsETHPrice < 1e16`).

## Proof of Concept

```solidity
// Foundry fork/unit test — unmodified production contracts
// Preconditions:
//   pricePercentageLimit == 0 (default, never set in initialize())
//   maxMintAmountPerDay set to a large value (required for normal operation)
//   1000 stETH deposited by existing users, rsETHPrice ~ 1e18

// 1. Set mock Chainlink feed to return answer = 1 (decimals = 8)
mockFeed.setAnswer(1);

// 2. Anyone calls the public updateRSETHPrice()
//    _getTotalEthInProtocol(): 1000e18 * 1e10 / 1e18 = 1e13
//    newRsETHPrice = 1e13 * 1e18 / 1000e18 = 1e10
//    pricePercentageLimit == 0 → isPriceDecreaseOffLimit = false → no pause
//    rsETHPrice = 1e10  ← committed
lrtOracle.updateRSETHPrice();

// 3. Feed recovers to normal
mockFeed.setAnswer(1e8); // 1.0 ETH per stETH (8 decimals)

// 4. Attacker deposits 1 stETH (1e18 wei)
//    getAssetPrice(stETH) = 1e18 (live, recovered)
//    rsETHPrice = 1e10 (stale, near-zero)
//    rsethAmountToMint = (1e18 * 1e18) / 1e10 = 1e26 rsETH
stETH.approve(address(depositPool), 1e18);
depositPool.depositAsset(stETH, 1e18, 0, "");

// 5. Assert attacker holds disproportionate rsETH
assertGt(rsETH.balanceOf(attacker), rsETH.totalSupply() / 2);
```
