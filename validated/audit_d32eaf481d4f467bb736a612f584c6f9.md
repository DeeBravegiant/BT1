Audit Report

## Title
Missing Arbitrum L2 Sequencer Uptime Check Allows Stale Chainlink Price Exploitation in L2 Pool Deposits - (File: contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol)

## Summary
`ChainlinkOracleForRSETHPoolCollateral.getRate()` fetches collateral token/ETH rates from Chainlink without verifying the Arbitrum sequencer is online. When the sequencer is down or in its post-restart grace period, Chainlink L2 feeds return the last known (stale) price while all existing guards pass. An unprivileged attacker can exploit this window via the public `deposit(address,uint256,string)` function to receive more wrsETH than their deposited collateral is worth, extracting value from the pool's collateral backing.

## Finding Description
`ChainlinkOracleForRSETHPoolCollateral.getRate()` (lines 26–37) calls `AggregatorV3Interface(oracle).latestRoundData()` and applies three guards:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

None of these guards detect sequencer downtime. When the Arbitrum sequencer is offline, Chainlink feeds freeze at the last published round: no new round starts, so `answeredInRound == roundID`; the `timestamp` is non-zero (last completed round); and `ethPrice` is positive. All three guards pass while the price is stale.

A grep across all contracts confirms no sequencer uptime feed is consulted anywhere. The exploit path is:

1. `RSETHPoolV3.deposit(token, amount, referralId)` (line 271) — public, no role required.
2. Calls `viewSwapRsETHAmountAndFee(amount, token)` (line 286).
3. Which calls `IOracle(supportedTokenOracle[token]).getRate()` (line 331) — resolves to `ChainlinkOracleForRSETHPoolCollateral.getRate()`.
4. Returns the stale inflated price; `rsETHAmount = amountAfterFee * tokenToETHRate / rsETHToETHrate` (line 334) mints excess wrsETH.

## Impact Explanation
**High — Theft of unclaimed yield / protocol value extraction.** The attacker mints wrsETH at a stale inflated collateral rate, receiving more wrsETH than the deposited collateral backs. This dilutes the collateral backing for all existing wrsETH holders, constituting direct extraction of yield/value from the pool. The magnitude is bounded by the `dailyMintLimit` modifier but is repeatable across sequencer restart windows.

## Likelihood Explanation
**Low-Medium.** Arbitrum sequencer outages are rare but documented. The attack requires: (1) sequencer downtime coinciding with adverse price movement, (2) the attacker acting within the post-restart window before Chainlink publishes a new round (typically minutes). The attack is permissionless — any address can call `deposit` — requires no special role, no front-running of victims, and no governance capture.

## Recommendation
Add a sequencer uptime check at the top of `ChainlinkOracleForRSETHPoolCollateral.getRate()` using Chainlink's Arbitrum Sequencer Uptime Feed (`0xFdB631F5EE196F0ed6FAa767959853A9F217697D`):

```solidity
(, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
if (answer != 0) revert SequencerDown();
if (block.timestamp - startedAt < GRACE_PERIOD) revert GracePeriodNotOver();
```

Store `sequencerUptimeFeed` as an immutable set in the constructor, and define `GRACE_PERIOD` as 3600 seconds per Chainlink's recommendation.

## Proof of Concept
**Foundry fork test plan (Arbitrum mainnet fork):**

1. Fork Arbitrum at a block where wstETH/ETH = 1.10e18.
2. Mock the Chainlink wstETH/ETH aggregator (`0x...`) to simulate sequencer downtime: freeze `latestRoundData()` returning `(N, 1.10e18, ..., T_old, N)` — `answeredInRound == roundID`, `timestamp != 0`.
3. Simulate price drop to 1.05e18 (update mock to return 1.05e18 only on a new round, which hasn't started yet).
4. Call `RSETHPoolV3.deposit(wstETH, 100e18, "")` from an unprivileged address.
5. Assert `wrsETH.balanceOf(attacker)` equals `100e18 * 1.10e18 / rsETHToETHrate` (stale rate) rather than `100e18 * 1.05e18 / rsETHToETHrate` (fair rate).
6. Confirm ~4.8% excess wrsETH minted, representing extraction from the pool's collateral backing.
