### Title
Attacker-Controlled CRCAT Spend Permanently Converts Victim's CATWallet to CRCATWallet with Attacker-Chosen `authorized_providers` — (`chia/wallet/wallet_state_manager.py`)

### Summary

`WalletStateManager.handle_cat` unconditionally calls `CRCATWallet.convert_to_cr` with `authorized_providers` and `proofs_checker` extracted directly from an on-chain CRCAT spend, with no check that those values were chosen by the wallet owner. Any party that holds CRCAT coins for the same TAIL as a victim's `CATWallet` can permanently mutate the victim's wallet type and authorization policy by broadcasting a crafted CRCAT spend bundle.

### Finding Description

The attack entry point is `WalletStateManager.handle_cat` in `chia/wallet/wallet_state_manager.py`.

**Step 1 — Wallet receives the attacker's coin via hint subscription.**

The victim's wallet subscribes to its own puzzle hashes. The full node delivers coin state updates for any coin whose spend memo (hint) matches a subscribed puzzle hash. The attacker sets the hint in their CRCAT spend to `victim_ph` — a puzzle hash the victim has published on-chain — so the victim's wallet receives the coin state update for the attacker's output CRCAT coin.

**Step 2 — `handle_cat` hint guard is bypassed.** [1](#0-0) 

The hint is attacker-supplied. Because `victim_ph` is in the victim's puzzle store, `derivation_record` is non-`None` and execution continues.

**Step 3 — CRCAT inner-puzzle-hash guard is bypassed.** [2](#0-1) 

`crcat.inner_puzzle_hash` is the inner puzzle hash of the **output** coin, taken from the CREATE_COIN condition emitted by the attacker's inner puzzle. The attacker sets this to exactly `construct_pending_approval_state(victim_ph, amount).get_tree_hash()`, satisfying the second branch of the `or` and bypassing the `return None`.

**Step 4 — No existing CRCAT wallet is found, but a CATWallet is.** [3](#0-2) 

Because the victim only has a `CATWallet` (not a `CRCATWallet`) for this TAIL, the CRCAT-wallet loop at line 1292 finds nothing. The CAT-wallet loop at line 1299 finds the victim's wallet and calls `convert_to_cr` with `crcat.authorized_providers` and `crcat.proofs_checker` — both fully attacker-controlled values extracted from the on-chain spend.

**Step 5 — `convert_to_cr` permanently overwrites the wallet record.** [4](#0-3) 

`convert_to_cr` writes a new `CRCATInfo` with the attacker's providers to the user store and replaces the in-memory wallet object. There is no rollback path.

**Root cause:** `handle_cat` treats `authorized_providers` and `proofs_checker` as trusted data derived from the on-chain puzzle reveal, but these fields are chosen by whoever minted or is spending the CRCAT — not by the victim.

### Impact Explanation

- The victim's `WalletType` is permanently changed from `CAT` to `CRCAT`.
- The victim's wallet now requires a VC issued by the attacker's `authorized_providers` for every future CAT spend.
- The victim's existing standard CAT coins (coin type `CoinType.CAT`) are no longer spendable through the wallet: `coin_record_to_crcat` only accepts `CoinType.CRCAT` / `CoinType.CRCAT_PENDING`. [5](#0-4) 

- The victim cannot create a new `CATWallet` for the same TAIL through normal wallet flows because the converted `CRCATWallet` now occupies that slot.
- Net effect: victim's CAT balance is frozen; all future spends are gated on attacker-controlled VC authorization.

### Likelihood Explanation

The attacker needs:
1. CRCAT coins for the same TAIL as the victim's `CATWallet` (obtainable if the TAIL allows open minting, or by receiving CRCAT coins from any third party).
2. One of the victim's puzzle hashes (public on-chain from any prior XCH or CAT transaction).

No keys, admin access, or cryptographic breaks are required. The spend bundle is valid and passes consensus validation. The attack is a single confirmed block event.

### Recommendation

Before calling `convert_to_cr`, verify that the `authorized_providers` and `proofs_checker` in the on-chain CRCAT match a user-configured policy (e.g., stored in a config or prompted via RPC). At minimum, require explicit user confirmation before mutating wallet type based on an incoming on-chain event. The conversion should never be triggered automatically from an arbitrary third-party spend.

### Proof of Concept

```
1. Victim wallet has CATWallet for TAIL X.
2. Attacker mints CRCAT coins for TAIL X with:
     authorized_providers = [attacker_did]
     proofs_checker = ProofsChecker(["attacker_flag"])
3. Attacker spends their CRCAT, inner puzzle outputs:
     CREATE_COIN(
       construct_pending_approval_state(victim_ph, amount).get_tree_hash(),
       amount,
       [victim_ph]   # hint
     )
4. Block confirms. Victim wallet receives coin state update (hint = victim_ph).
5. handle_cat is called:
   - hint guard passes (victim_ph in puzzle store)
   - inner_puzzle_hash guard passes (matches pending_approval_state)
   - no CRCAT wallet found for TAIL X
   - CATWallet found → convert_to_cr(cat_wallet, [attacker_did], ProofsChecker(["attacker_flag"]))
6. Assert: victim wallet type == WalletType.CRCAT
7. Assert: victim wallet info.authorized_providers == [attacker_did]
8. Victim's existing CAT coins are unspendable via wallet.
```

    @classmethod
    async def convert_to_cr(
        cls,
        cat_wallet: CATWallet,
        authorized_providers: list[bytes32],
        proofs_checker: ProofsChecker,
    ) -> None:
        replace_self = cls()
        replace_self.standard_wallet = cat_wallet.standard_wallet
        replace_self.log = logging.getLogger(cat_wallet.get_name())
        replace_self.log.info(f"Converting CAT wallet {cat_wallet.id()} to CR-CAT wallet")
        replace_self.wallet_state_manager = cat_wallet.wallet_state_manager
        replace_self.info = cls.wallet_info_type(
            cat_wallet.cat_info.limitations_program_hash, None, authorized_providers, proofs_checker
        )
        await cat_wallet.wallet_state_manager.user_store.update_wallet(
            WalletInfo(
                cat_wallet.id(), cat_wallet.get_name(), uint8(WalletType.CRCAT.value), bytes(replace_self.info).hex()
            )
        )
        updated_wallet_info = await cat_wallet.wallet_state_manager.user_store.get_wallet_by_id(cat_wallet.id())
        assert updated_wallet_info is not None
        replace_self.wallet_info = updated_wallet_info

        cat_wallet.wallet_state_manager.wallets[cat_wallet.id()] = replace_self

    def coin_record_to_crcat(self, coin_record: WalletCoinRecord) -> CRCAT:
        if coin_record.coin_type not in {CoinType.CRCAT, CoinType.CRCAT_PENDING}:  # pragma: no cover
            raise ValueError(f"Attempting to spend a non-CRCAT coin: {coin_record.coin.name().hex()}")
        if coin_record.metadata is None:  # pragma: no cover
            raise ValueError(f"Attempting to spend a CRCAT coin without metadata: {coin_record.coin.name().hex()}")
        try:
            metadata: CRCATMetadata = CRCATWallet.get_metadata_from_record(coin_record)
            crcat: CRCAT = CRCAT(
                coin_record.coin,
                self.info.limitations_program_hash,
                metadata.lineage_proof,
                self.info.authorized_providers,
                self.info.proofs_checker.as_program(),
                (
                    construct_pending_approval_state(
                        metadata.inner_puzzle_hash, uint64(coin_record.coin.amount)
                    ).get_tree_hash()
                    if coin_record.coin_type == CoinType.CRCAT_PENDING
                    else metadata.inner_puzzle_hash
                ),
            )
            return crcat
        except Exception as e:  # pragma: no cover
            raise ValueError(f"Error parsing CRCAT metadata: {e}")
