### Title
Fee-on-Transfer Token Overcredits Recipient on NEAR, Undercollateralizing the EVM Bridge Vault - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary

`OmniBridge.initTransfer()` records the caller-supplied `amount` in the `InitTransfer` event after calling `safeTransferFrom`, without accounting for transfer fees deducted by fee-on-transfer ERC20 tokens. The NEAR bridge reads that event amount verbatim and credits the recipient with the full `amount`, while the EVM vault only holds `amount − fee`. Every such transfer permanently undercollateralizes the bridge by the fee, eventually making some redemptions unclaimable.

### Finding Description

In `OmniBridge.initTransfer()`, the branch that handles plain (non-bridge, non-custom-minter) ERC20 tokens calls:

```solidity
IERC20(tokenAddress).safeTransferFrom(
    msg.sender,
    address(this),
    amount          // requested amount, not actual received amount
);
``` [1](#0-0) 

Immediately after, the function emits:

```solidity
emit BridgeTypes.InitTransfer(
    msg.sender, tokenAddress, currentOriginNonce,
    amount,   // ← still the caller-supplied value, not what the vault received
    fee, nativeFee, recipient, message
);
``` [2](#0-1) 

For a fee-on-transfer token the vault receives `amount − transferFee`, but the event records `amount`. The CLAUDE.md security invariant states explicitly: *"The NEAR side reads this event … it is the only data the NEAR side sees."* [3](#0-2) 

On the NEAR side, `fin_transfer_callback` constructs the `TransferMessage` directly from the event field:

```rust
amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
``` [4](#0-3) 

This amount is then minted or unlocked to the recipient without any cross-check against what the EVM vault actually holds. The same `amount` value is also propagated through `OmniBridgeWormhole.initTransferExtension()`:

```solidity
Borsh.encodeUint128(amount),   // caller-supplied, not vault-received
``` [5](#0-4) 

### Impact Explanation

Each `initTransfer` with a fee-on-transfer token creates a deficit of `transferFee` tokens in the EVM vault. The NEAR side mints/credits `amount` wrapped tokens backed by only `amount − transferFee` locked tokens. When recipients bridge back, the EVM vault must release `amount` but holds less. The deficit accumulates across transfers; the last redeemers find the vault insolvent and their redemptions permanently unclaimable. This breaks the core backing guarantee documented in the security invariants.

This maps to: **Critical — Irreversible fund lock / permanently unclaimable user value in bridge flows**, and **High — balance-accounting divergence that breaks backing guarantees**.

### Likelihood Explanation

Any unprivileged user can call `initTransfer` with any ERC20 token. Fee-on-transfer tokens (e.g. STA, PAXG, tokens with configurable fees) are real, deployed assets. No special setup, leaked key, or colluding party is required. The vulnerability is triggered by normal protocol use with a supported token class.

### Recommendation

Measure the vault's actual received amount by comparing balances before and after `safeTransferFrom`, and use the measured amount in the event:

```solidity
} else {
    uint256 balanceBefore = IERC20(tokenAddress).balanceOf(address(this));
    IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
    uint256 received = IERC20(tokenAddress).balanceOf(address(this)) - balanceBefore;
    require(received > 0, "zero received");
    amount = uint128(received);   // shadow with actual received amount
}
```

The corrected `amount` must then flow into both the `InitTransfer` event and the Wormhole payload so the NEAR side credits only what the vault actually holds.

### Proof of Concept

1. Deploy or use any fee-on-transfer ERC20 (e.g. 1% fee) that is registered as a plain (non-bridge, non-custom-minter) token in `OmniBridge`.
2. Call `initTransfer(tokenAddress, 1000, 0, 0, "recipient.near", "")`.
3. `safeTransferFrom` moves 1000 tokens from the caller; the token contract deducts 1% fee, so the vault receives 990.
4. The `InitTransfer` event records `amount = 1000`.
5. The NEAR relayer submits the event proof; `fin_transfer_callback` reads `init_transfer.amount = 1000` and mints 1000 wrapped tokens to `recipient.near`.
6. The EVM vault holds only 990 tokens but has issued a claim for 1000.
7. Repeat N times; the vault deficit grows by 10 per iteration.
8. When the 100th user attempts to bridge back 1000 tokens, the vault cannot satisfy the redemption — funds are permanently unclaimable. [6](#0-5) [7](#0-6)

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

        initTransferExtension(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue
        );

        emit BridgeTypes.InitTransfer(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message
        );

    pub fn fin_transfer_callback(
        &mut self,
        #[serializer(borsh)] storage_deposit_actions: &Vec<StorageDepositAction>,
        #[serializer(borsh)] predecessor_account_id: AccountId,
    ) -> PromiseOrValue<Nonce> {
        let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
        };
        require!(
            self.factories
                .get(&init_transfer.emitter_address.get_chain())
                == Some(init_transfer.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        let decimals = self
            .token_decimals
            .get(&init_transfer.token)
            .near_expect(BridgeError::TokenDecimalsNotFound);

        let destination_nonce =
            self.get_next_destination_nonce(init_transfer.recipient.get_chain());
        let transfer_message = TransferMessage {
            origin_nonce: init_transfer.origin_nonce,
            token: init_transfer.token,
            amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
            recipient: init_transfer.recipient,
            fee: Self::denormalize_fee(&init_transfer.fee, decimals),
            sender: init_transfer.sender,
            msg: init_transfer.msg,
            destination_nonce,
            origin_transfer_id: None,
        };

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
            self.process_fin_transfer_to_near(
                recipient,
                &predecessor_account_id,
                transfer_message,
                storage_deposit_actions,
            )
            .into()
        } else {
            self.process_fin_transfer_to_other_chain(predecessor_account_id, transfer_message);
            PromiseOrValue::Value(destination_nonce)
        }
    }
