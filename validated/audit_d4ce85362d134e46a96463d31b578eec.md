Audit Report

## Title
Unchecked Zero Return from `getAssetPrice()` Causes Silent Zero-Mint on Deposit — (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` casts the `int256` price from Chainlink's `latestRoundData()` directly to `uint256` with no `price > 0` guard. If the feed returns `0`, the zero silently propagates through `LRTDepositPool.getRsETHAmountToMint()`, producing `rsethAmountToMint = 0`. A depositor who passes `minRSETHAmountExpected = 0` will have their LST transferred to the protocol while receiving zero rsETH, with no on-chain path to recover the funds.

## Finding Description

**Root cause — `ChainlinkPriceOracle.getAssetPrice()`:**

```solidity
// contracts/oracles/ChainlinkPriceOracle.sol L49-55
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

No check that `price > 0`. If `price == 0`, the function returns `0`. [1](#0-0) 

**Zero propagation — `getRsETHAmountToMint()`:**

```solidity
// contracts/LRTDepositPool.sol L519-521
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

When `getAssetPrice(asset)` returns `0`, `rsethAmountToMint = 0` regardless of `amount`. [2](#0-1) 

**Slippage guard bypass — `_beforeDeposit()`:**

```solidity
// contracts/LRTDepositPool.sol L665-669
rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);
if (rsethAmountToMint < minRSETHAmountExpected) {
    revert MinimumAmountToReceiveNotMet();
}
```

When `minRSETHAmountExpected = 0`, the condition `0 < 0` is false — no revert. [3](#0-2) 

**Asset transfer with zero mint — `depositAsset()`:**

```solidity
// contracts/LRTDepositPool.sol L113-116
IERC20(asset).safeTransferFrom(msg.sender, address(this), depositAmount);
_mintRsETH(rsethAmountToMint); // rsethAmountToMint == 0
```

The user's LST is transferred to the pool, then `IRSETH.mint(user, 0)` is called. [4](#0-3) 

**`RSETH.mint(to, 0)` does not revert:**

The `checkDailyMintLimit(0)` modifier evaluates `currentPeriodMintedAmount + 0 > maxMintAmountPerDay`, which is false under normal conditions. OpenZeppelin's `_mint(to, 0)` also does not revert. The call succeeds and the user's rsETH balance remains zero. [5](#0-4) [6](#0-5) 

**Registration-time sanity check is insufficient:**

`updatePriceOracleForValidated()` validates `price` is between `1e16` and `1e19` only at oracle registration time. The price can move to `0` after registration (feed deprecation, circuit-breaker, migration window), and no runtime guard exists. [7](#0-6) 

**Secondary impact — `_getTotalEthInProtocol()`:**

A zero asset price also silently drops that asset's entire TVL contribution from the rsETH price calculation, potentially triggering the downside-protection pause. [8](#0-7) 

## Impact Explanation

**Critical — Permanent freezing of user funds.**

The depositor's LST is held by `LRTDepositPool` with no on-chain record linking it to the depositor. Because `LRTWithdrawalManager.initiateWithdrawal` requires the caller to hold and burn rsETH, a user who received zero rsETH has no protocol-sanctioned path to recover their deposit. The funds are permanently locked from the user's perspective.

## Likelihood Explanation

**Low-Medium.** Two independent conditions must coincide:

1. A supported Chainlink feed returns `price = 0` — possible during feed deprecation, a circuit-breaker event, or a feed migration window. Chainlink feeds have historically returned `0` or stale data during such transitions.
2. The depositor passes `minRSETHAmountExpected = 0` — a realistic default for integrators, bots, or users unfamiliar with the slippage parameter.

Neither condition requires privileged access or attacker-controlled state. The victim is an unprivileged depositor; the trigger is an external feed anomaly combined with a missing on-chain guard.

## Recommendation

Add a `price > 0` guard in `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Additionally:
- Add a staleness check (`updatedAt + heartbeat > block.timestamp`) and a round-completeness check (`answeredInRound >= roundId`).
- Add a defense-in-depth non-zero guard in `getRsETHAmountToMint()`: `if (rsethAmountToMint == 0) revert ZeroMintAmount()`.

## Proof of Concept

1. Chainlink feed for asset `X` returns `price = 0` (feed deprecated or circuit-breaker active).
2. User calls:
   ```solidity
   LRTDepositPool.depositAsset(X, 1e18, 0, "");
   ```
3. `_beforeDeposit` → `getRsETHAmountToMint(X, 1e18)`.
4. `lrtOracle.getAssetPrice(X)` → `ChainlinkPriceOracle.getAssetPrice(X)` → returns `0`.
5. `rsethAmountToMint = (1e18 * 0) / rsETHPrice = 0`.
6. `0 < 0` is false → no revert.
7. `IERC20(X).safeTransferFrom(user, depositPool, 1e18)` — user's 1 LST transferred.
8. `IRSETH.mint(user, 0)` — succeeds, user receives 0 rsETH.
9. User holds 0 rsETH; `LRTWithdrawalManager.initiateWithdrawal` requires rsETH balance → user cannot withdraw. Funds permanently locked.

**Foundry fork test plan:**
- Fork mainnet, mock the Chainlink aggregator for a supported asset to return `answer = 0`.
- Call `depositAsset` with `minRSETHAmountExpected = 0`.
- Assert: depositor's LST balance decreased by `depositAmount`, depositor's rsETH balance is `0`, `LRTDepositPool` LST balance increased by `depositAmount`.
