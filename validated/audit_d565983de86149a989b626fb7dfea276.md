Audit Report

## Title
Missing Chainlink `latestRoundData()` Staleness and Validity Checks in `ChainlinkPriceOracle.getAssetPrice()` — (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards all validation fields, accepting stale, zero, or incomplete prices without revert. This unvalidated price propagates into `LRTOracle._updateRsETHPrice()`, where a stale inflated price causes phantom TVL growth, triggering excess protocol-fee rsETH minting to the treasury and permanently diluting existing rsETH holders' yield. A zero price triggers the downside-protection pause, temporarily freezing all deposits and withdrawals. Both paths are reachable by any unprivileged caller.

## Finding Description

`contracts/oracles/ChainlinkPriceOracle.sol` line 52 reads:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

All five return values (`roundId`, `answer`, `startedAt`, `updatedAt`, `answeredInRound`) are available but only `answer` is consumed. There is no check that `updatedAt` is recent, `answeredInRound >= roundId`, or `price > 0`.

The same codebase already applies all three guards in `contracts/pools/oracle/ChainlinkOracleForRSETHPoolCollateral.sol` lines 30–32:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
```

The unvalidated price propagates through two critical paths:

**Path 1 — Fee minting (yield theft):**
`LRTOracle._getTotalEthInProtocol()` (line 339) calls `getAssetPrice(asset)` for every supported LST. The stale inflated value inflates `totalETHInProtocol`. At line 244, `totalETHInProtocol > previousTVL` evaluates true on phantom TVL growth, computing `rewardAmount` and `protocolFeeInETH` on the phantom gain. At lines 301–307, `IRSETH.mint(treasury, rsethAmountToMintAsProtocolFee)` executes, minting excess rsETH to the treasury and permanently diluting all existing rsETH holders. `updateRSETHPrice()` (line 87) is `public` with no caller restriction beyond `whenNotPaused`, so any unprivileged address can trigger this.

**Path 2 — Protocol pause (fund freeze):**
If a feed returns 0, `_getTotalEthInProtocol()` undercounts TVL, `newRsETHPrice` drops, and if the drop exceeds `pricePercentageLimit`, lines 277–281 pause `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`, freezing all user deposits and withdrawals until an admin manually unpauses.

**Path 3 — Per-deposit rsETH minting:**
`LRTDepositPool.getRsETHAmountToMint()` line 520 computes `rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice()`. A stale asset price directly inflates or deflates rsETH issued per deposit.

The `pricePercentageLimit` upside check (lines 252–266) only reverts for non-manager callers when the price increase exceeds the configured threshold; a moderately stale price (e.g., 2% inflated within a 5% limit) passes through entirely. The `maxFeeMintAmountPerDay` cap bounds but does not eliminate the yield theft per day.

## Impact Explanation

**High — Theft of unclaimed yield:** When a Chainlink LST/ETH feed goes stale at an inflated value, `_updateRsETHPrice()` computes a phantom TVL increase, mints excess fee rsETH to the treasury proportional to the phantom gain, and permanently dilutes all existing rsETH holders. The dilution is irreversible once the mint executes. This matches the allowed impact: *Theft of unclaimed yield*.

**Medium — Temporary freezing of funds:** A feed returning 0 or a severely depressed stale value causes the downside-protection logic to auto-pause `LRTDepositPool` and `LRTWithdrawalManager`, freezing all user deposits and withdrawals until admin intervention. This matches: *Temporary freezing of funds*.

## Likelihood Explanation

Chainlink feeds go stale during network congestion, sequencer outages, or feed deprecation — documented, recurring events. No privileged access, oracle operator compromise, or victim mistake is required. `updateRSETHPrice()` is callable by any unprivileged external address. The protocol already treats this risk as real enough to guard against it in `ChainlinkOracleForRSETHPoolCollateral`, yet the same guard is absent in `ChainlinkPriceOracle`, which is the oracle used for the core L1 deposit and price-update paths. The condition is repeatable on every `updateRSETHPrice()` call while the feed remains stale.

## Recommendation

Apply the same validation pattern already present in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`, adding a configurable `MAX_STALENESS_PERIOD`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);

    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();

    if (answeredInRound < roundId) revert StalePrice();
    if (updatedAt == 0)            revert IncompleteRound();
    if (price <= 0)                revert InvalidPrice();
    if (block.timestamp - updatedAt > MAX_STALENESS_PERIOD) revert StalePrice();

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

## Proof of Concept

1. A Chainlink stETH/ETH feed goes stale (`updatedAt` is 25 hours old, last reported price 1.06e18 while true rate has dropped to 1.04e18).
2. Any unprivileged caller invokes `LRTOracle.updateRSETHPrice()`.
3. `_getTotalEthInProtocol()` → `LRTOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` → returns 1.06e18 with no revert.
4. `totalETHInProtocol` is overcounted; `totalETHInProtocol > previousTVL` is true.
5. `rewardAmount = totalETHInProtocol - previousTVL` is computed on the phantom gain; `protocolFeeInETH` is derived from it.
6. `rsethAmountToMintAsProtocolFee` passes `_checkAndUpdateDailyFeeMintLimit` (within daily cap for a moderate stale delta).
7. `IRSETH(rsETHTokenAddress).mint(treasury, rsethAmountToMintAsProtocolFee)` executes — excess rsETH is permanently minted to the treasury, diluting all existing rsETH holders.

**Foundry fork test plan:**
- Fork mainnet at a block where a supported LST feed's `updatedAt` is > `MAX_STALENESS_PERIOD` ago (or mock `latestRoundData` to return a stale timestamp and inflated price).
- Record `rsETH.balanceOf(treasury)` and `rsETH.totalSupply()` before.
- Call `lrtOracle.updateRSETHPrice()` from an unprivileged address.
- Assert `rsETH.balanceOf(treasury)` increased and `rsETH.totalSupply()` increased by more than legitimate yield warrants.
- Assert existing holder's proportional share of underlying ETH decreased.
