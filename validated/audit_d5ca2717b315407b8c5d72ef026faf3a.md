### Title
`ProposalInit.builder` Is Never Validated by Consensus Validators, Allowing a Malicious Proposer to Redirect All Block Fees to an Arbitrary Address - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

### Summary

`ProposalInit` carries a `builder` field (the sequencer/block-builder address that receives fee transfers). The validator's `is_proposal_init_valid` function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, `starknet_version`, `version_constant_commitment`, and `fee_proposal_fri` — but never checks `builder`. A legitimate-but-malicious proposer can therefore set `builder` to any arbitrary address. Because `builder` is forwarded verbatim to the batcher as `sequencer_address` via `convert_to_sn_api_block_info`, every fee transfer in the block is sent to the attacker-controlled address, and the wrong `sequencer_address` is committed to the chain state.

### Finding Description

`ProposalInit` is defined with a `builder` field: [1](#0-0) 

When a proposer builds a block, `builder` is set from `args.builder_address`: [2](#0-1) 

The resulting `ProposalInit` is broadcast over the network. On the validator side, `is_proposal_init_valid` is called before the proposal is accepted: [3](#0-2) 

The function checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, and all four L1 gas prices. It does **not** check `init_proposed.builder` against any locally-trusted reference value. After passing this check, the `init` (including the unchecked `builder`) is forwarded to the batcher: [4](#0-3) 

The `block_info` produced by `convert_to_sn_api_block_info(init)` maps `builder` to `sequencer_address`. The blockifier then uses `sequencer_address` as the recipient of every fee transfer in the block: [5](#0-4) 

The `proposer` identity is checked at the consensus-manager level: [6](#0-5) 

But `builder` — a distinct, economically significant field — is never checked anywhere in the validation path.

### Impact Explanation

A malicious but legitimately-scheduled proposer (one whose address is returned by `get_proposer_for_height` for the current height/round) can set `ProposalInit.builder` to any address they control. Validators will accept the proposal because `builder` is not part of `is_proposal_init_valid`. The batcher executes the block with the attacker-supplied `sequencer_address`, causing:

1. **All fee transfers in the block are sent to the attacker's address** — every `execute_fee_transfer` call uses `block_context.block_info.sequencer_address` as the recipient.
2. **Wrong `sequencer_address` is committed to the canonical chain state** — any contract that reads `get_execution_info().block_info.sequencer_address` (e.g., for access-control checks) will observe the attacker's address.
3. **Wrong block header is finalized** — `finalize_decision` calls `convert_to_sn_api_block_info(init)` and passes the result to state sync and the cende pipeline, permanently recording the wrong sequencer address.

This matches the allowed impact: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact* and *Wrong state … committed*.

### Likelihood Explanation

The attack requires the adversary to be a scheduled proposer for at least one height/round, which is a normal role in a decentralized validator set. No special privilege beyond being a validator is needed. The attack is silent (no on-chain error, no log warning) and takes effect in a single block.

### Recommendation

Add a check inside `is_proposal_init_valid` that rejects any proposal whose `builder` does not match the locally-configured `builder_address`. The validator already stores `builder_address` in `SequencerConsensusContext`; it should be included in `ProposalInitValidation` and compared:

```rust
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.expected_builder, init_proposed.builder
        ),
    ));
}
```

### Proof of Concept

1. Attacker is a legitimate validator scheduled to propose at `height=H, round=R`.
2. Attacker constructs `ProposalInit` with all fields set to valid values **except** `builder`, which is set to `attacker_address`.
3. Attacker broadcasts the proposal stream.
4. Each validator calls `is_proposal_init_valid` — all checks pass because `builder` is not checked.
5. `initiate_validation` is called with `block_info` containing `sequencer_address = attacker_address`.
6. The batcher executes all transactions; every `execute_fee_transfer` sends fees to `attacker_address`.
7. `validate_proposal` returns `Ok(built_block)` and the commitment is sent to consensus.
8. Consensus reaches decision; `finalize_decision` commits the block with `sequencer_address = attacker_address` to the chain. [7](#0-6) [8](#0-7) [9](#0-8)

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }

async fn initiate_validation(
    batcher: Arc<dyn BatcherClient>,
    state_sync_client: SharedStateSyncClient,
    init: &ProposalInit,
    proposal_id: ProposalId,
    timeout_plus_margin: Duration,
    clock: &dyn Clock,
    compare_retrospective_block_hash: bool,
) -> ValidateProposalResult<()> {
    let chrono_timeout = chrono::Duration::from_std(timeout_plus_margin)
        .expect("Can't convert timeout to chrono::Duration");

    let input = ValidateBlockInput {
        proposal_id,
        deadline: clock.now() + chrono_timeout,
        retrospective_block_hash: retrospective_block_hash(
            batcher.clone(),
            state_sync_client,
            init,
            compare_retrospective_block_hash,
        )
        .await
        .map_err(ValidateProposalError::from)?,
        block_info: convert_to_sn_api_block_info(init)?,
    };
    debug!("Initiating validate proposal: input={input:?}");
    batcher.validate_block(input.clone()).await.map_err(|err| {
        ValidateProposalError::Batcher(
            format!("Failed to initiate validate proposal {input:?}."),
            err,
        )
    })?;
    Ok(())
}
