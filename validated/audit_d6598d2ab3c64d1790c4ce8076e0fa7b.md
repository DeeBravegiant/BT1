### Title
Timelord `_check_for_new_ip` hardcodes `uint32(0)` as `prev_transaction_block_height`, permanently preventing V2-proof blocks from being infused after `HARD_FORK2_HEIGHT` — (`File: chia/timelord/timelord.py`)

### Summary
`_check_for_new_ip` passes `uint32(0)` as `prev_transaction_block_height` to `iters_from_block`, while `_can_infuse_unfinished_block` correctly passes `self.last_state.get_last_tx_height()`. After `HARD_FORK2_HEIGHT`, any V2-proof block is accepted and scheduled by `_can_infuse_unfinished_block` but unconditionally rejected inside `_check_for_new_ip`, so the timelord never emits a `NewInfusionPointVDF` for it. The block is silently dropped at the infusion stage.

### Finding Description

`_can_infuse_unfinished_block` calls `iters_from_block` with the correct `prev_transaction_block_height`: [1](#0-0) 

`_check_for_new_ip` calls the same function with a hardcoded `uint32(0)`: [2](#0-1) 

Both calls flow through `iters_from_block` → `validate_pospace_and_get_required_iters` → `verify_and_get_quality_string`: [3](#0-2) 

Inside `verify_and_get_quality_string`, V2 proofs are unconditionally rejected when `prev_transaction_block_height < HARD_FORK2_HEIGHT`: [4](#0-3) 

Because `uint32(0) < HARD_FORK2_HEIGHT` is always true, `verify_and_get_quality_string` returns `None` for every V2 proof in `_check_for_new_ip`. This causes `validate_pospace_and_get_required_iters` to return `None`, which causes `iters_from_block` to hit `assert required_iters is not None` and raise `AssertionError`. The exception is caught by the inner `except` block at line 596–598, and the block is skipped with `continue`. [5](#0-4) 

After the loop, if **all** pending unfinished blocks are V2 proofs, `ip_iters` remains `None` and `assert ip_iters is not None` at line 602 raises an unhandled `AssertionError`, crashing `_check_for_new_ip`. [6](#0-5) 

### Impact Explanation

After `HARD_FORK2_HEIGHT`, every V2-proof unfinished block forwarded by the full node to the timelord is:
1. Accepted and scheduled by `_can_infuse_unfinished_block` (correct height used).
2. Silently dropped in `_check_for_new_ip` (height 0 used → V2 rejected).
3. Never completed into a `NewInfusionPointVDF`.

As V2 plots become the dominant plot type post-fork, an increasing fraction of winning proofs are V2. When all pending blocks are V2 proofs, the `assert ip_iters is not None` crash also fires. This constitutes a **permanent inability for honest farmers using V2 plots to have their blocks infused**, matching the High impact criterion: *"Permanent or long-lived inability for honest nodes, wallets, farmers, harvesters, or timelords to process valid blocks … under normal network assumptions."*

### Likelihood Explanation

The trigger is any legitimate V2-plot farmer winning a block after `HARD_FORK2_HEIGHT`. No adversarial intent is required; normal farming activity is sufficient. The condition is deterministic and reproducible.

### Recommendation

Replace `uint32(0)` with `self.last_state.get_last_tx_height()` in `_check_for_new_ip`, mirroring the call in `_can_infuse_unfinished_block`:

```python
# line 594 — fix:
self.last_state.get_last_tx_height(),   # was: uint32(0)
```

### Proof of Concept

1. Set up a timelord with `last_state` where `last_tx_block_block_height >= HARD_FORK2_HEIGHT`.
2. Construct a `NewUnfinishedBlockTimelord` whose `reward_chain_block.proof_of_space` is a valid V2 proof (`strength_v2 is not None`).
3. Call `_can_infuse_unfinished_block(block)` → returns a nonzero `new_block_iters`; block is appended to `self.unfinished_blocks` and the iteration is added to `iters_to_submit`.

5. Observe: `iters_from_block` raises `AssertionError` (caught internally), `block` remains `None`, no `NewInfusionPointVDF` is emitted. If this is the only block, `assert ip_iters is not None` also fires.
6. Assert that no infusion point message was sent to the full node — confirming the invariant violation.
