Audit Report

## Title
Uncheckpointed EigenPod ETH Excluded from `getEffectivePodShares()`, Enabling Theft of Unclaimed Yield — (`contracts/NodeDelegator.sol`, `contracts/LRTDepositPool.sol`, `contracts/LRTOracle.sol`)

## Summary
`NodeDelegator.getEffectivePodShares()` returns only `stakedButUnverifiedNativeETH + withdrawableShare`, omitting any ETH sitting in the EigenPod contract that has not yet been credited via a completed checkpoint. Because `updateRSETHPrice()` is a permissionless public function, an attacker can observe execution-layer reward ETH arriving at the EigenPod, call `updateRSETHPrice()` to lock in the understated price, then deposit to mint more rsETH than their proportional share of real assets, capturing yield earned by existing holders.

## Finding Description

**Root cause — `getEffectivePodShares()` (NodeDelegator.sol L556–562):**
The function returns exactly two terms:
- `stakedButUnverifiedNativeETH` — incremented only by `stake32Eth()`, decremented by `verifyWithdrawalCredentials()`.
- `withdrawableShare` — the DelegationManager's view of withdrawable shares for `beaconChainETHStrategy`, updated only after `verifyWithdrawalCredentials` or after a checkpoint is finalized.

ETH that arrives at the EigenPod address via execution-layer rewards or partial beacon-chain withdrawals (`NonBeaconChainETHReceived`) sits in the EigenPod's raw balance and is not credited as EigenLayer shares until `startCheckpoint` + `verifyCheckpointProofs` completes. Neither term captures it.

**Propagation path:**
1. `getAssetDistributionData(ETH_TOKEN)` (LRTDepositPool.sol L440–441) delegates to `getETHDistributionData()`.
2. `getETHDistributionData()` (LRTDepositPool.sol L487) calls `getEffectivePodShares()` for each NDC — the uncheckpointed pod ETH is absent.
3. `getTotalAssetDeposits(ETH_TOKEN)` (LRTDepositPool.sol L385–396) sums the distribution data.
4. `_getTotalEthInProtocol()` (LRTOracle.sol L341) calls `getTotalAssetDeposits` for every supported asset, producing an understated TVL.
5. `updateRSETHPrice()` (LRTOracle.sol L87–88) is `public whenNotPaused` — no role restriction — so any caller can store the understated price.
6. `getRsETHAmountToMint` (LRTDepositPool.sol L520) divides by the stored `rsETHPrice`; a lower price mints more rsETH per ETH deposited.

**Why existing checks are insufficient:**
There is no guard preventing `updateRSETHPrice()` from being called while uncheckpointed ETH exists in the pod. The `whenNotPaused` modifier only blocks calls when the contract is paused; it does not enforce checkpoint state. The comment on `getETHDistributionData()` ("rewards are not accounted here") acknowledges the gap but provides no on-chain enforcement.

## Impact Explanation
**High — Theft of unclaimed yield.**
The uncheckpointed ETH in the EigenPod is accrued execution-layer staking yield belonging to existing rsETH holders. By depositing while the price is understated, an attacker mints rsETH at a below-fair-value rate. When the checkpoint is later finalized and TVL jumps, the attacker's rsETH is worth more than they paid for it, at the direct expense of prior holders whose per-token backing is diluted. This is a concrete, quantifiable transfer of yield from existing holders to the attacker, matching the "Theft of unclaimed yield" impact class.

## Likelihood Explanation
- Execution-layer rewards flow to EigenPods continuously on mainnet; the window between receipt and checkpoint finalization is routine and can span hours to days.
- `updateRSETHPrice()` is permissionless — an attacker needs no privileged role.
- The ETH arrival is observable on-chain (EigenPod `NonBeaconChainETHReceived` event or balance change), so the attacker can time the call precisely.
- No governance capture, oracle compromise, or victim mistake is required. The only prerequisite is an active EigenPod receiving rewards, which is the normal operating state.

## Recommendation
**Option A (preferred):** In `getEffectivePodShares()`, add `address(eigenPod).balance` to the returned value so that ETH sitting in the pod but not yet checkpointed is still counted as protocol-owned ETH:
```solidity
return stakedButUnverifiedNativeETH + withdrawableShare + address(eigenPod).balance;
```

**Option B:** Gate `updateRSETHPrice()` so it can only execute when no checkpoint is in progress and no uncheckpointed pod ETH exists (e.g., require `eigenPod.currentCheckpointTimestamp() == 0` and `address(eigenPod).balance == 0`). This prevents price updates during the vulnerable window but does not fix the underlying accounting gap.

## Proof of Concept
```solidity
// Fork test (mainnet or Holesky fork with an active NDC/EigenPod)
function test_uncheckpointedPodEthInflatesMint() public {
    // Precondition: NDC has EigenPod with 1 verified validator.
    // rsETHPrice = 1e18, totalSupply = 100e18 rsETH, TVL = 100 ETH.

    // 1. Simulate 1 ETH of execution-layer rewards arriving at the EigenPod.
    vm.deal(address(eigenPod), 1 ether);
    // getEffectivePodShares() still returns 32e18 (withdrawableShare), ignoring pod balance.

    // 2. Attacker calls permissionless updateRSETHPrice().
    lrtOracle.updateRSETHPrice();
    // Price computed on 100 ETH TVL (pod ETH excluded) → rsETHPrice = 1e18.

    // 3. Attacker deposits 1 ETH.
    uint256 minted = depositPool.getRsETHAmountToMint(ETH_TOKEN, 1 ether);
    // minted = 1e18 * 1e18 / 1e18 = 1e18 rsETH
    // Fair share = 1/101 * 101 ≈ 0.9901e18 rsETH
    assertGt(minted, 0.99 ether); // attacker receives ~1% excess

    // 4. Operator finalizes checkpoint → EigenLayer credits 1 ETH as shares.
    finalizeCheckpoint();
    lrtOracle.updateRSETHPrice();
    // TVL = 102 ETH, supply = 101 rsETH → price = 102/101 ≈ 1.0099e18
    // Attacker's 1 rsETH worth ~1.0099 ETH; they paid 1 ETH → captured ~0.0099 ETH of yield.
    // Original 100 rsETH holders: expected 101/100 = 1.01 ETH each, got 102/101 ≈ 1.0099 ETH each.
}
```
The `getEffectivePodShares()` call in step 2 returns only `stakedButUnverifiedNativeETH + withdrawableShare`, confirming the 1 ETH pod balance is excluded and the price is understated.
