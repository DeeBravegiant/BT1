Audit Report

## Title
Missing Staleness Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables Excess rsETH Minting at Stale Prices - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` calls `latestRoundData()` but discards `roundId`, `updatedAt`, and `answeredInRound`, accepting stale or invalid prices without reverting. This price feeds directly into `LRTDepositPool.getRsETHAmountToMint()`, allowing any depositor to mint rsETH at an inflated stale rate when a Chainlink LST/ETH feed lags behind a real depeg event, diluting all existing rsETH holders.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` reads only the `price` field:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

No check is performed for `answeredInRound < roundId` (stale round), `updatedAt == 0` (incomplete round), or `price <= 0` (invalid price). By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` — used for L2 pool collateral pricing — enforces all three:

```solidity
if (answeredInRound < roundID) revert StalePrice();
if (timestamp == 0) revert IncompleteRound();
if (ethPrice <= 0) revert InvalidPrice();
``` [2](#0-1) 

The stale price propagates through `LRTOracle.getAssetPrice()` into `LRTDepositPool.getRsETHAmountToMint()`:

```solidity
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
``` [3](#0-2) 

This value is used directly in `depositAsset()`, which is callable by any unprivileged user with no access control beyond `whenNotPaused` and `onlySupportedERC20Token`. [4](#0-3) 

## Impact Explanation
When a Chainlink LST/ETH feed is stale at a price higher than the true current rate (e.g., an LST has depegged but the feed has not updated within its heartbeat window), a depositor receives rsETH computed at the inflated stale rate. The excess minted rsETH is backed by less real value than it represents, directly diluting the yield and principal of all existing rsETH holders. This constitutes **theft of unclaimed yield** (High severity per the allowed impact scope). Repeated exploitation during a prolonged stale period can compound into protocol insolvency.

## Likelihood Explanation
Chainlink LST/ETH feeds operate on heartbeat intervals (commonly 24 hours) and deviation thresholds. During extreme market events or L1 congestion, feeds can remain stale for hours while the underlying asset depegs. The entry path (`depositAsset()`) requires no special permissions — any external account can call it at any time. The precondition (stale feed + depegged LST) is realistic and has occurred historically with LSTs. Likelihood is **Medium**.

## Recommendation
Apply the same validation pattern used in `ChainlinkOracleForRSETHPoolCollateral` to `ChainlinkPriceOracle.getAssetPrice()`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
        priceFeed.latestRoundData();
    require(answeredInRound >= roundId, "Stale price");
    require(updatedAt != 0, "Incomplete round");
    require(price > 0, "Invalid price");
    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, add a configurable `maxStaleness` duration check: `require(block.timestamp - updatedAt <= maxStaleness, "Price too old")`.

## Proof of Concept
1. Assume the stETH/ETH Chainlink feed last updated 25 hours ago at `1.05e18`. The true current rate has dropped to `0.95e18` due to a depeg, but the feed has not updated.
2. Attacker calls `LRTDepositPool.depositAsset(stETH, 1000e18, 0, "")`.
3. `_beforeDeposit` → `getRsETHAmountToMint` → `lrtOracle.getAssetPrice(stETH)` → `ChainlinkPriceOracle.getAssetPrice(stETH)` returns `1.05e18`.
4. `rsethAmountToMint = (1000e18 * 1.05e18) / rsETHPrice` — computed at the stale inflated rate.
5. Attacker receives ~10.5% more rsETH than the deposited stETH is actually worth, diluting all existing rsETH holders.

**Foundry fork test plan:** Fork mainnet at a block where a targeted LST/ETH feed is within its heartbeat but the LST has depegged. Deploy a mock `AggregatorV3Interface` returning a stale high price with `answeredInRound < roundId`. Register it via `ChainlinkPriceOracle.updatePriceFeedFor`. Call `depositAsset` and assert that `rsethAmountToMint` exceeds the fair-value equivalent, confirming the dilution.
