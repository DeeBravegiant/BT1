Audit Report

## Title
Excess ETH Permanently Locked in `updateRate()` Due to Missing Post-Loop Refund - (File: contracts/cross-chain/MultiChainRateProvider.sol)

## Summary
`MultiChainRateProvider.updateRate()` is a public payable function that forwards exactly the on-chain `estimatedFee` to each LayerZero `send()` call per receiver, but never returns the difference between `msg.value` and the sum of per-receiver fees to the caller. The contract contains no ETH recovery mechanism, so any surplus ETH sent by the caller is permanently frozen.

## Finding Description
In `MultiChainRateProvider.updateRate()` (L108–137), the loop queries `estimateFees()` for each receiver and forwards exactly that amount to `ILayerZeroEndpoint.send()`:

```solidity
(uint256 estimatedFee,) = ILayerZeroEndpoint(layerZeroEndpoint)
    .estimateFees(dstChainId, address(this), _payload, false, bytes(""));

ILayerZeroEndpoint(layerZeroEndpoint).send{ value: estimatedFee }(
    dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
);
```

The `payable(msg.sender)` argument is LayerZero's internal refund address for its own unused gas — it does not cause LayerZero to return the contract's unspent `msg.value` to the caller. After the loop completes, no code refunds `msg.value − Σ(estimatedFee_i)` to the caller. The contract is `abstract` and `Ownable` but defines no `receive()`, `fallback()`, or owner-callable ETH sweep function anywhere in its body (L1–182).

This contrasts directly with `CrossChainRateProvider.updateRate()` (L96–98), which passes `msg.value` in full to the single `send()` call, allowing LayerZero to refund any excess:

```solidity
ILayerZeroEndpoint(layerZeroEndpoint).send{ value: msg.value }(
    dstChainId, remoteAndLocalAddresses, _payload, payable(msg.sender), address(0x0), bytes("")
);
```

`MultiChainRateProvider` cannot use this pattern because it must split value across multiple receivers, making the missing post-loop refund a structural omission.

## Impact Explanation
Any ETH sent above the exact sum of per-receiver estimated fees is permanently frozen in the contract. There is no owner sweep, no upgrade path, and no drain function. This constitutes **permanent freezing of funds**, which is a Critical impact under the allowed scope.

## Likelihood Explanation
`updateRate()` has no access control — any external account may call it. Callers routinely add a safety buffer on top of `estimateTotalFee()` to avoid reverts from fee fluctuation between the off-chain estimate and on-chain execution. Every such call leaves a small residual permanently locked. The gap accumulates across all callers over time. No special privileges or unusual conditions are required.

## Recommendation
After the loop, refund any remaining contract balance to `msg.sender`:

```solidity
uint256 remaining = address(this).balance;
if (remaining > 0) {
    (bool ok,) = payable(msg.sender).call{value: remaining}("");
    require(ok, "refund failed");
}
```

Alternatively, pre-compute the total fee, require `msg.value` to equal it exactly, and revert otherwise.

## Proof of Concept
1. Two rate receivers are configured; `estimateTotalFee()` returns `0.01 ETH`.
2. Caller invokes `updateRate{value: 0.02 ETH}()` (standard buffer practice).
3. The loop sends `estimatedFee_0` and `estimatedFee_1` (summing to `0.01 ETH`) to LayerZero.
4. After the loop, `address(this).balance == 0.01 ETH`. No refund is issued.
5. The `0.01 ETH` surplus is permanently locked; no function in the contract can recover it.

**Foundry test sketch:**
```solidity
function test_excessEthLocked() public {
    // deploy MultiChainRateProvider concrete impl with 2 receivers
    // mock LayerZero endpoint to return estimatedFee = 0.005 ether each
    uint256 callerBalanceBefore = caller.balance;
    vm.prank(caller);
    provider.updateRate{value: 0.02 ether}();
    // assert contract holds 0.01 ether
    assertEq(address(provider).balance, 0.01 ether);
    // assert caller lost 0.02 ether, not 0.01 ether
    assertEq(callerBalanceBefore - caller.balance, 0.02 ether);
}
```
