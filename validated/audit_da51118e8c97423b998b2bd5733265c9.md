Audit Report

## Title
Missing Chainlink `latestRoundData()` Return Value Validation Allows Stale/Zero Price to Corrupt `rsETHPrice` — (`contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice` captures only `price` from `latestRoundData()`, discarding `updatedAt`, `answeredInRound`, and `roundId` with no validity checks. A stale or zero-returning feed is silently accepted, the corrupted price flows through `LRTOracle._getTotalEthInProtocol()` into `_updateRsETHPrice()`, and the resulting near-zero `rsETHPrice` is stored on-chain. Because `updateRSETHPrice()` is public and `pricePercentageLimit` defaults to `0` (disabling the circuit breaker), any caller can trigger the corrupt update and immediately deposit to mint rsETH at an astronomically inflated rate.

## Finding Description

**Root cause — `ChainlinkPriceOracle.sol` lines 52–54:**

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

`roundId`, `startedAt`, `updatedAt`, and `answeredInRound` are all discarded. No check is made that `price > 0`, `updatedAt > 0`, `updatedAt >= block.timestamp - staleness`, or `answeredInRound >= roundId`.

**Propagation path:**

1. `_getTotalEthInProtocol()` (LRTOracle.sol L336–343) calls `getAssetPrice(asset)` for every supported asset and multiplies by total deposits. If any asset's feed returns `price = 0`, that asset contributes `0` to `totalETHInProtocol`, collapsing the total.
2. `_updateRsETHPrice()` (L250) computes `newRsETHPrice = totalETHInProtocol.divWad(rsethSupply)` — near-zero when the total collapses.
3. The circuit breaker at L256–274 is gated on `pricePercentageLimit > 0`. Since `pricePercentageLimit` is a `uint256` state variable (L29) that is zero-initialized and must be explicitly set by admin via `setPricePercentageLimit`, the downside-pause and upside-revert branches are both dead by default.
4. `updateRSETHPrice()` (L87) is `public whenNotPaused` — any EOA can call it.
5. `getRsETHAmountToMint()` (LRTDepositPool.sol L519–520) computes `amount * getAssetPrice(asset) / lrtOracle.rsETHPrice()`. With `rsETHPrice` near-zero, the denominator collapses and the minted amount is orders of magnitude above fair value.

**Partial mitigation that does not close the gap:**

`updatePriceOracleForValidated` (LRTOracle.sol L101–108) validates that the oracle price is between `1e16` and `1e19` at registration time. This does not protect against a feed that was valid at registration and later goes stale or returns zero. Additionally, the unvalidated `updatePriceOracleFor` (L113) is also callable by admin and bypasses this check entirely.

## Impact Explanation

**Critical — Protocol insolvency.**

When a supported asset's Chainlink feed returns `price = 0` (feed deprecated, circuit-breaker triggered on the aggregator, or round not yet completed):

- `getAssetPrice` returns `0` for that asset.
- `_getTotalEthInProtocol()` undervalues the protocol TVL.
- `newRsETHPrice` collapses proportionally.
- Any depositor calling `depositAsset` / `depositETH` immediately after `updateRSETHPrice()` receives `amount * getAssetPrice(asset) / rsETHPrice` rsETH — a value far exceeding their deposit's fair worth.
- The excess rsETH represents claims on real underlying assets, draining the protocol and diluting all existing rsETH holders to near-worthlessness.

This is direct theft of existing depositors' funds and constitutes protocol insolvency, matching the Critical impact class.

## Likelihood Explanation

- Chainlink feeds can return `price = 0` or a stale price during aggregator deprecation, circuit-breaker events, or L2 sequencer downtime — none of which require any privileged action or external compromise.
- `updateRSETHPrice()` is permissionless; the attacker calls it themselves at the moment the feed enters the bad state.
- `pricePercentageLimit` is `0` by default, so no automatic circuit-breaker fires unless the admin has explicitly configured it post-deployment.
- The attack is repeatable: each time the feed enters a bad state, the window is open until the admin pauses or corrects the oracle.

## Recommendation

Add the following guards inside `ChainlinkPriceOracle.getAssetPrice`:

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound) =
    priceFeed.latestRoundData();

require(price > 0,                                          "Chainlink: non-positive price");
require(updatedAt > 0,                                      "Chainlink: round not complete");
require(answeredInRound >= roundId,                         "Chainlink: stale round");
require(block.timestamp - updatedAt <= STALENESS_THRESHOLD, "Chainlink: stale price");
```

Additionally, set `pricePercentageLimit` to a non-zero value (e.g., `5e16` for 5%) during deployment/initialisation so the circuit-breaker in `_updateRsETHPrice` is always active.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;
import "forge-std/Test.sol";

// Simulates a Chainlink feed that has gone stale / returns price = 0
contract StaleFeed {
    function decimals() external pure returns (uint8) { return 8; }
    function latestRoundData() external pure returns (
        uint80, int256, uint256, uint256, uint80
    ) {
        // price = 0, updatedAt = 0 — uninitialized / deprecated round
        return (1, 0, 0, 0, 1);
    }
}

contract PoCTest is Test {
    function test_staleFeedCorruptsRsETHPrice() public {
        // 1. Fork mainnet / deploy protocol locally.
        // 2. Deploy StaleFeed; register it via ChainlinkPriceOracle.updatePriceFeedFor
        //    (LRTManager role) for one supported LST asset.
        // 3. Confirm pricePercentageLimit == 0 (default).
        // 4. Call lrtOracle.updateRSETHPrice() as any EOA — succeeds without revert.
        // 5. Assert lrtOracle.rsETHPrice() << 1 ether (collapsed).
        // 6. Deposit a small amount of a *healthy* asset.
        // 7. Assert rsETH minted >> fair value (denominator near-zero).
    }
}
```

The mock feed returns `price = 0`; `getAssetPrice` returns `0` without reverting; `_getTotalEthInProtocol` undervalues TVL; `rsETHPrice` is written to a near-zero value; subsequent deposits mint rsETH at a rate orders of magnitude above fair value.
