### Title
Consensus Vote Signature Never Verified — Any Peer Can Forge Votes for Arbitrary Validators - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

---

### Summary

`SingleHeightConsensus::handle_vote` accepts any network-delivered `Vote` and counts it as a legitimate vote from the address in `vote.voter`, without verifying that the sender cryptographically controls that address. A single malicious peer can forge prevotes and precommits attributed to any committee member, manufacture a fake 2/3 quorum, and force an unauthorized consensus decision on a block that honest validators never voted for.

---

### Finding Description

`handle_vote` in `single_height_consensus.rs` performs two checks before accepting a vote:

1. Height equality (`vote.height != height`)
2. Committee membership (`self.committee.members().iter().any(|s| s.address == vote.voter)`)

It explicitly skips cryptographic verification with a `// TODO(Asmaa): verify the signature` comment: [1](#0-0) 

The `Vote` struct carries a `signature: RawSignature` field that is structurally present but never checked on the receive path: [2](#0-1) 

On the send side, `make_self_vote` also defers signing with `// TODO(Asmaa): sign the vote`, emitting `RawSignature::default()`: [3](#0-2) 

The codebase itself acknowledges the consequence in `cache_future_vote`:

> "Since vote signatures are not yet verified, a peer can forge votes with arbitrary voter addresses; without this cap that would grow `future_votes` without bound and exhaust memory." [4](#0-3) 

A `verify_precommit_vote_signature` function already exists in `apollo_signature_manager` but is never called in the vote-handling path: [5](#0-4) 

The full vote-acceptance path is: network broadcast → `MultiHeightManager::handle_vote` → `SingleHeightConsensus::handle_vote` → `StateMachine::handle_prevote` / `handle_precommit` → quorum accounting → `DecisionReached`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

A malicious peer that is connected to the consensus broadcast channel can:

1. Craft `Vote { vote_type: Precommit, height: H, round: R, proposal_commitment: Some(attacker_block_id), voter: validator_N_address, signature: RawSignature::default() }` for each committee member.
2. Broadcast all forged votes. Each passes the committee-membership check and the duplicate check.
3. Once forged votes accumulate to ≥ 2/3 of total weight (plus the virtual-proposer requirement), `upon_decision` fires and `DecisionReached` is emitted.
4. `decision_reached` is called on the context, committing the attacker's block to the chain. [8](#0-7) [9](#0-8) 

The committed block is one that honest validators never voted for. This corrupts the canonical state root, all state transitions, receipts, events, and L1 messages for that height and every subsequent height.

**Impact category**: Critical — wrong state, receipt, event, and storage values resulting from a block committed via unauthorized consensus decision.

---

### Likelihood Explanation

The attacker must be a connected peer on the consensus broadcast topic. The consensus network uses libp2p and is not fully permissioned at the transport layer; the only guard is network-level peer authentication (challenge/identity), not per-message vote authentication. Once a peer is connected, it can send arbitrarily crafted `Vote` protobuf messages. The exploit requires no privileged access beyond network connectivity to the consensus gossip channel.

---

### Recommendation

Implement vote signature verification inside `handle_vote` before the committee-membership check. The infrastructure already exists:

- `verify_precommit_vote_signature(block_hash, signature, public_key)` in `apollo_signature_manager`
- The committee already maps each validator address to a weight; extend it to also carry the validator's public key
- Verify `vote.signature` against the vote content (height, round, proposal_commitment, voter) using the public key associated with `vote.voter`
- Reject and `report_peer` on any vote whose signature does not verify

Also complete the signing side: replace `RawSignature::default()` in `make_self_vote` with an actual ECDSA signature over the vote content. [10](#0-9) 

---

### Proof of Concept

```
// Attacker is a libp2p peer connected to the consensus broadcast topic.
// Observes current height H, round R, and the proposer's block commitment C.

for validator_address in committee.members() {
    let forged_vote = Vote {
        vote_type: VoteType::Precommit,
        height: H,
        round: R,
        proposal_commitment: Some(attacker_chosen_commitment),
        voter: validator_address,          // impersonating this validator
        signature: RawSignature::default(), // no valid signature needed
    };
    broadcast_channel.send(forged_vote);
    // handle_vote: height check passes, committee check passes,
    //              signature check: SKIPPED (TODO comment)
    // vote is inserted into precommits map with full validator weight
}
// After enough forged precommits accumulate to ≥ 2/3 weight:
// upon_decision fires → DecisionReached(attacker_chosen_commitment)
// decision_reached commits the attacker's block to the chain
```

This is the direct Sequencer analog of the Solidity `voteExtraordinary(tokenHolders[i], proposalId)` loop: in both cases the `voter`/`account_` field is trusted without proof of authorization, allowing a single caller to manufacture votes on behalf of the entire eligible set.
