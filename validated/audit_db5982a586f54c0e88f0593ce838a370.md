Audit Report

## Title
Staked Principal Temporarily Frozen When `maxNumberOfWithdrawalsPerUser` Is Uninitialized - (File: contracts/KERNEL/KernelDepositPool.sol)

## Summary
`KernelDepositPool.initialize()` never sets `maxNumberOfWithdrawalsPerUser`, leaving it at the Solidity default of `0`. Any user who calls `stake()` before the admin calls `setMaxNumberOfWithdrawalsPerUser()` will find their principal permanently unwithdrawable until the admin acts, because `initiateWithdrawal()` unconditionally reverts when the limit is zero. There is no on-chain guard preventing deposits in this misconfigured state.

## Finding Description
`initialize()` sets only `kernelToken`, `rewardsToken`, and the admin role — `maxNumberOfWithdrawalsPerUser` is never assigned and defaults to `0`. [1](#0-0) 

`stake()` accepts deposits with no check on `maxNumberOfWithdrawalsPerUser`: [2](#0-1) 

`initiateWithdrawal()` contains the guard:
```solidity
if (userWithdrawalIds[msg.sender].length >= maxNumberOfWithdrawalsPerUser) revert WithdrawalLimitReached();
``` [3](#0-2) 

When `maxNumberOfWithdrawalsPerUser == 0`, a fresh staker has `userWithdrawalIds[msg.sender].length == 0`. The condition `0 >= 0` evaluates to `true`, so every call to `initiateWithdrawal()` reverts. `claimWithdrawal()` requires a withdrawal ID that can only be created by a successful `initiateWithdrawal()`, so there is no alternative on-chain recovery path. [4](#0-3) 

`setMaxNumberOfWithdrawalsPerUser()` explicitly rejects `0`, so the admin cannot "reset" the value — they must set a positive integer to unblock withdrawals: [5](#0-4) 

## Impact Explanation
Any user who stakes before the admin configures `maxNumberOfWithdrawalsPerUser` has their KERNEL tokens locked with no on-chain self-service recovery path. This is a **temporary freezing of funds** (Medium), matching the allowed impact scope. The freeze is not permanent because the admin can unblock it, but the user has no recourse on their own and cannot predict when the admin will act.

## Likelihood Explanation
`stake()` is public and unrestricted. `initialize()` does not enforce any ordering between deployment and configuration of `maxNumberOfWithdrawalsPerUser`. There is no on-chain guard, no event, and no revert warning a user that the contract is not yet fully configured. Any user who reads the ABI and calls `stake()` immediately after deployment — before the admin completes setup — will be affected. This is a realistic race condition during the deployment window.

## Recommendation
Initialize `maxNumberOfWithdrawalsPerUser` to a safe non-zero default inside `initialize()`:

```solidity
maxNumberOfWithdrawalsPerUser = MAX_WITHDRAWALS_PER_USER; // 100
```

Or add a guard to `stake()` that reverts when `maxNumberOfWithdrawalsPerUser == 0`:

```solidity
if (maxNumberOfWithdrawalsPerUser == 0) revert WithdrawalLimitNotConfigured();
```

## Proof of Concept
1. Admin deploys and calls `KernelDepositPool.initialize(admin, kernelToken, rewardToken)`. `maxNumberOfWithdrawalsPerUser` is `0`.
2. Alice calls `stake(1000e18)`. Succeeds — `balanceOf[Alice] = 1000e18`.
3. Alice calls `initiateWithdrawal(1000e18)`.
   - Check: `userWithdrawalIds[Alice].length >= maxNumberOfWithdrawalsPerUser` → `0 >= 0` → `true` → reverts `WithdrawalLimitReached`.
4. Alice has no other on-chain path to recover her principal.
5. Alice's 1000 KERNEL remain frozen until the admin calls `setMaxNumberOfWithdrawalsPerUser(N)` for some `N > 0`.

**Foundry test plan:**
```solidity
function test_stakeBeforeMaxWithdrawalsSet() public {
    // Deploy and initialize without calling setMaxNumberOfWithdrawalsPerUser
    vm.prank(alice);
    kernelDepositPool.stake(1000e18);
    assertEq(kernelDepositPool.balanceOf(alice), 1000e18);

    vm.prank(alice);
    vm.expectRevert(KernelDepositPool.WithdrawalLimitReached.selector);
    kernelDepositPool.initiateWithdrawal(1000e18);
}
```

        Withdrawal storage withdrawal = withdrawals[_withdrawalId];

        if (withdrawal.user == address(0)) {
            revert WithdrawalDoesNotExist();
        }

        if (withdrawal.user != msg.sender) {
            revert NotYourWithdrawal();
        }

        if (block.timestamp < withdrawal.unlockTime) {
            revert WithdrawalNotReady();
        }

        if (withdrawal.claimed) {
            revert WithdrawalAlreadyClaimed();
        }

        withdrawal.claimed = true;

        // Remove the withdrawal ID from the user's list of withdrawal IDs
        uint256[] storage userWithdrawalIdsArray = userWithdrawalIds[msg.sender];
        for (uint256 i = 0; i < userWithdrawalIdsArray.length; ++i) {
            if (userWithdrawalIdsArray[i] == _withdrawalId) {
                userWithdrawalIdsArray[i] = userWithdrawalIdsArray[userWithdrawalIdsArray.length - 1];
                userWithdrawalIdsArray.pop();
                break;
            }
        }

        // Transfer the KERNEL tokens from contract to user
        kernelToken.safeTransfer(msg.sender, withdrawal.amount);

        emit WithdrawalClaimed(msg.sender, withdrawal.amount, _withdrawalId);
    }
