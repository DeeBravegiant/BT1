### Title
Unbounded Native-Stack Recursion in `post_order_traverse_updated_nodes` During Memtrie Chunk Application - (File: core/store/src/trie/mem/memtrie_update.rs)

### Summary
`MemTrieUpdate::post_order_traverse_updated_nodes` performs a fully recursive post-order traversal of updated trie nodes using the native Rust call stack with no depth guard. An unprivileged user who deploys a contract and stores keys that form a deep chain in the Patricia Merkle Trie can force this recursion to a depth proportional to the trie depth, potentially overflowing the native stack during chunk application and crashing every validator node that processes the block.

### Finding Description
`post_order_traverse_updated_nodes` is a static recursive function called from `to_memtrie_changes_internal`, which is called from `to_trie_changes` and `to_memtrie_changes_only` at the end of every memtrie update:

```rust
fn post_order_traverse_updated_nodes(
    node_id: UpdatedNodeId,
    updated_nodes: &Vec<Option<UpdatedMemTrieNodeWithSize>>,
    ordered_nodes: &mut Vec<UpdatedNodeId>,
) {
    let node = updated_nodes[node_id].as_ref().unwrap();
    match &node.node {
        UpdatedMemTrieNode::Branch { children, .. } => {
            for child in children.iter() {
                if let Some(OldOrUpdatedNodeId::Updated(child_node_id)) = child {
                    Self::post_order_traverse_updated_nodes(   // ← recursive, no depth limit
                        *child_node_id, updated_nodes, ordered_nodes,
                    );
                }
            }
        }
        UpdatedMemTrieNode::Extension { child, .. } => {
            if let OldOrUpdatedNodeId::Updated(child_node_id) = child {
                Self::post_order_traverse_updated_nodes(       // ← recursive, no depth limit
                    *child_node_id, updated_nodes, ordered_nodes,
                );
            }
        }
        _ => {}
    }
    ordered_nodes.push(node_id);
}
```

The recursion depth equals the number of `Updated` nodes on the path from the trie root to the modified leaf. When a key is inserted into an existing trie of depth D, all D nodes on the path are converted from `Old` to `Updated`, so the recursion reaches depth D. There is no heap-based worklist, no depth counter, and no early termination. [1](#0-0) 

`to_memtrie_changes_internal` calls this function unconditionally before returning `MemTrieChanges`: [2](#0-1) 

`to_trie_changes` (the production path used during chunk application) calls `to_memtrie_changes_internal`: [3](#0-2) 

### Impact Explanation
The corrupted protocol value is the **finality head**: if the native stack overflows during `apply_chunk` on a block that contains the triggering transaction, every validator node running with memtries loaded crashes with SIGSEGV. On restart each node re-processes the same block, crashes again, and enters a permanent crash loop. The chain head cannot advance; no new state root, outcome root, or receipt is produced. This constitutes a chain halt triggered by a single user-submitted transaction.

### Likelihood Explanation
An attacker must build a trie of depth D sufficient to overflow the native stack (typically 8 MB; at ~300–500 bytes per frame, D ≈ 16,000–26,000 nodes). The worst-case key structure is a chain where each key is a byte-prefix of the next (keys of length 1, 2, 3, … N bytes). Total storage staked grows as O(N²), making very large depths economically expensive. However, the vulnerability requires no special privilege: any account can deploy a contract and call `storage_write` across many transactions to accumulate the required trie depth. The absence of any depth guard in the code means the overflow is deterministic once the threshold is reached.

### Recommendation
Replace the recursive traversal in `post_order_traverse_updated_nodes` with an explicit heap-allocated worklist (e.g., a `Vec` used as a stack), mirroring the iterative approach already used in `traverse_all_nodes` in `core/store/src/trie/state_parts.rs`: [4](#0-3) 

That function already performs the same kind of trie traversal iteratively using an explicit `Vec` stack, demonstrating the correct pattern. Apply the same transformation to `post_order_traverse_updated_nodes`.

### Proof of Concept
1. Deploy a contract on NEAR mainnet/testnet.
2. Over many transactions, call `storage_write` with keys `[0x00]`, `[0x00, 0x00]`, `[0x00, 0x00, 0x00]`, … up to length N, where each key is a byte-prefix of the next. This builds a Patricia Merkle Trie chain of depth ≈ N nodes.
3. Submit one final transaction that writes or deletes any key in this contract. During chunk application, `Runtime::apply` → `Trie::update_with_memtrie` → `MemTrieUpdate::to_trie_changes` → `to_memtrie_changes_internal` → `post_order_traverse_updated_nodes` recurses to depth N.
4. When N exceeds the native stack capacity, every validator node with memtries loaded crashes with a stack overflow, halting the chain.

        MEMTRIE_NUM_NODES_CREATED_FROM_UPDATES
            .with_label_values(&[&self.shard_uid])
            .inc_by(self.updated_nodes.len() as u64);
        let mut ordered_nodes = Vec::new();
        Self::post_order_traverse_updated_nodes(0, &self.updated_nodes, &mut ordered_nodes);

        let hashes_and_serialized_nodes =
            self.compute_hashes_and_serialized_nodes(&ordered_nodes, &self.updated_nodes);

        let node_ids_with_hashes = hashes_and_serialized_nodes
            .iter()
            .map(|(node_id, hash, _)| (*node_id, *hash))
            .collect();
        (
            MemTrieChanges { node_ids_with_hashes, updated_nodes: self.updated_nodes },
            hashes_and_serialized_nodes
                .into_iter()
                .map(|(_, hash, serialized)| (hash, serialized))
                .collect(),
        )
    }

        let old_root =
            self.root.map(|root| root.as_ptr(self.memory).view().node_hash()).unwrap_or_default();
        let mut refcount_changes = self
            .nodes_tracker
            .take()
            .expect("Cannot to_trie_changes for memtrie changes only")
            .finalize();
        let (memtrie_changes, hashes_and_serialized) = self.to_memtrie_changes_internal();

        // We've accounted for the dereferenced nodes, as well as value addition/subtractions.
        // The only thing left is to increment refcount for all new nodes.
        for (node_hash, node_serialized) in hashes_and_serialized {
            refcount_changes.add(node_hash, node_serialized, 1);
        }
        let (insertions, deletions) = refcount_changes.into_changes();

        TrieChanges {
            old_root,
            new_root: memtrie_changes
                .node_ids_with_hashes
                .last()
                .map(|(_, hash)| *hash)
                .unwrap_or_default(),
            insertions,
            deletions,
            memtrie_changes: Some(memtrie_changes),
            children_memtrie_changes: Default::default(),
        }
    }

        fn traverse_all_nodes<F: FnMut(&CryptoHash) -> Result<(), StorageError>>(
            &self,
            mut on_enter: F,
        ) -> Result<(), StorageError> {
            if self.root == Trie::EMPTY_ROOT {
                return Ok(());
            }
            let mut stack: Vec<(CryptoHash, TrieStorageNodeWithSize, CrumbStatus)> = Vec::new();
            let root_node = self.retrieve_storage_node(&self.root)?;
            stack.push((self.root, root_node, CrumbStatus::Entering));
            while let Some((hash, node, position)) = stack.pop() {
                if let CrumbStatus::Entering = position {
                    on_enter(&hash)?;
                }
                let mut on_enter_value = |value: &ValueHandle| {
                    if let ValueHandle::HashAndSize(value) = value {
                        on_enter(&value.hash)
                    } else {
                        unreachable!("only possible while mutating")
                    }
                };
                match &node.node {
                    TrieStorageNode::Empty => {
                        continue;
                    }
                    TrieStorageNode::Leaf { value, .. } => {
                        on_enter_value(value)?;
                        continue;
                    }
                    TrieStorageNode::Branch { children, value } => match position {
                        CrumbStatus::Entering => {
                            if let Some(value) = value {
                                on_enter_value(value)?;
                            }
                            stack.push((hash, node, CrumbStatus::AtChild(0)));
                            continue;
                        }
                        CrumbStatus::AtChild(mut i) => loop {
                            if (i as usize) >= NUM_CHILDREN {
                                stack.push((hash, node, CrumbStatus::Exiting));
                                break;
                            }
                            if let Some(child_hash) = children[i as usize] {
                                let child = self.retrieve_storage_node(&child_hash)?;
                                stack.push((child_hash, node, CrumbStatus::AtChild(i + 1)));
                                stack.push((child_hash, child, CrumbStatus::Entering));
                                break;
                            }
                            i += 1;
                        },
                        CrumbStatus::Exiting => {
                            continue;
                        }
                        CrumbStatus::At => {
                            continue;
                        }
                    },
                    TrieStorageNode::Extension { child, .. } => {
                        if let CrumbStatus::Entering = position {
                            let child_hash = *child;
                            let child = self.retrieve_storage_node(&child_hash)?;
                            stack.push((child_hash, node, CrumbStatus::Exiting));
                            stack.push((child_hash, child, CrumbStatus::Entering));
                        }
                    }
                }
            }
            Ok(())
        }
