### Title
Stale DRep Expiry Data in Ratification Vote Count Allows Governance Actions to Pass with Insufficient Votes - (File: eras/conway/impl/src/Cardano/Ledger/Conway/Rules/Ratify.hs)

---

### Summary

The `dRepAcceptedRatio` function in the Conway `RATIFY` rule checks DRep expiry using only the stored `drepExpiry` field from the `RatifyEnv` snapshot, without accounting for accumulated dormant epochs (`numDormantEpochs`). This is structurally inconsistent with the correct expiry predicate used elsewhere in the codebase, which adds `numDormantEpochs` to the stored value. Because `RatifyEnv` carries no `numDormantEpochs` field, the ratification rule cannot perform the correct check. When dormant epochs have accumulated, DReps that are actually still active are silently excluded from the vote-count denominator, artificially inflating the acceptance ratio and allowing governance actions to ratify with fewer votes than the protocol requires.

---

### Finding Description

**Root cause — `dRepAcceptedRatio` in `Ratify.hs`**

The function iterates over the DRep stake distribution and skips any DRep whose stored expiry is strictly less than `reCurrentEpoch`:

```haskell
Just drepState
  | reCurrentEpoch > drepExpiry drepState -> (yes, tot) -- drep is expired, so we don't consider it
``` [1](#0-0) 

The `RatifyEnv` record that carries `reCurrentEpoch` and `reDRepState` contains no `numDormantEpochs` field:

```haskell
data RatifyEnv era = RatifyEnv
  { reInstantStake :: InstantStake era
  , reStakePoolDistr :: PoolDistr
  , reDRepDistr :: Map DRep (CompactForm Coin)
  , reDRepState :: Map (Credential DRepRole) DRepState
  , reCurrentEpoch :: EpochNo
  , reCommitteeState :: CommitteeState era
  , reAccounts :: Accounts era
  , reStakePools :: Map (KeyHash StakePool) StakePoolState
  }
``` [2](#0-1) 

**Correct expiry predicate — `isDRepExpired` in the testlib**

The correct check, used in tests to verify protocol behaviour, adds `numDormantEpochs` to the stored value before comparing:

```haskell
pure $
  binOpEpochNo (+) (vState ^. vsNumDormantEpochsL) (drep' ^. drepExpiryL)
    < currentEpoch
``` [3](#0-2) 

**How dormant epochs accumulate**

`numDormantEpochs` is incremented at every epoch boundary that passes without any governance proposals. The stored `drepExpiry` is only updated (to absorb the dormant count) when a new proposal is submitted via `updateDormantDRepExpiry`:

```haskell
updateDormantDRepExpiry currentEpoch vState =
  if numDormantEpochs == EpochNo 0
    then vState
    else
      vState
        & vsNumDormantEpochsL .~ EpochNo 0
        & vsDRepsL %~ Map.map updateExpiry
``` [4](#0-3) 

The `DRepPulser` snapshot that feeds `RATIFY` is taken at the epoch boundary, before any proposals in the new epoch are processed. If `numDormantEpochs > 0` at snapshot time, the stored `drepExpiry` values in the snapshot are stale — they do not reflect the dormant-epoch extension — and the `RATIFY` rule has no mechanism to compensate.

**Concrete discrepancy**

Let `E` = stored `drepExpiry`, `D` = `numDormantEpochs` at snapshot time, `C` = `reCurrentEpoch`.

| Check | Expires when |
|---|---|
| `dRepAcceptedRatio` (production) | `C > E` |
| `isDRepExpired` (correct) | `C > E + D` |

When `D ≥ 1`, the production check expires DReps up to `D` epochs earlier than the correct check. A DRep with stored expiry `E = C − 1` and `D = 1` has actual expiry `C`, so it is still active — but `dRepAcceptedRatio` treats it as expired and removes its stake from the denominator.

**Effect on the acceptance ratio**

The DRep acceptance ratio is `yes / (yes + no)`, where non-voting active DReps count as "no". Removing active non-voting DReps from the denominator raises the ratio, potentially pushing it above the required threshold without the corresponding yes-votes. [5](#0-4)

dRepAcceptedRatio RatifyEnv {reDRepDistr, reDRepState, reCurrentEpoch} gasDRepVotes govAction =
  toInteger yesStake %? toInteger totalExcludingAbstainStake
  where
    accumStake (!yes, !tot) drep (CompactCoin stake) =
      case drep of
        DRepCredential cred ->
          case Map.lookup cred reDRepState of
            Nothing -> (yes, tot) -- drep is not registered, so we don't consider it
            Just drepState
              | reCurrentEpoch > drepExpiry drepState -> (yes, tot) -- drep is expired, so we don't consider it
              | otherwise ->
                  case Map.lookup cred gasDRepVotes of
                    -- drep hasn't voted for this action, so we don't count
                    -- the vote but we consider it in the denominator:
                    Nothing -> (yes, tot + stake)
                    Just VoteYes -> (yes + stake, tot + stake)
                    Just Abstain -> (yes, tot)
                    Just VoteNo -> (yes, tot + stake)
        DRepAlwaysNoConfidence ->
          case govAction of
            NoConfidence _ -> (yes + stake, tot + stake)
            _ -> (yes, tot + stake)
        DRepAlwaysAbstain -> (yes, tot)
    (yesStake, totalExcludingAbstainStake) = Map.foldlWithKey' accumStake (0, 0) reDRepDistr

updateDormantDRepExpiry currentEpoch vState =
  if numDormantEpochs == EpochNo 0
