Audit Report

## Title
Permissionless `updateRSETHPrice()` + Staleness-Free `EthXPriceOracle` Enables Temporary Protocol-Wide Freeze via ETHx Rate Drop — (`contracts/oracles/EthXPriceOracle.sol` / `contracts/LRTOracle.sol`)

## Summary
`EthXPriceOracle.getAssetPrice()` blindly forwards Stader's `getExchangeRate()` with no staleness check, no deviation bounds, and no circuit-breaker. Because `LRTOracle.updateRSETHPrice()` is a permissionless `public` function, any external caller can invoke it at the moment the ETHx exchange rate has dropped (e.g., due to a validator slash), satisfying the `isPriceDecreaseOffLimit` condition and atomically pausing `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`. Recovery requires manual `onlyLRTAdmin` intervention with no automatic or permissionless unpause path.

## Finding Description
**Step 1 — Oracle has no staleness or bounds check.**
`EthXPriceOracle.getAssetPrice()` is a one-liner passthrough to `IETHXStakePoolsManager.getExchangeRate()`. There is no `updatedAt` timestamp comparison, no minimum/maximum rate bounds, and no consecutive-anomaly requirement before the value is accepted as valid.

**Step 2 — Raw rate flows directly into TVL calculation.**
`LRTOracle._getTotalEthInProtocol()` calls `getAssetPrice(asset)` for every supported asset including ETHx, multiplying by total deposits. A drop in the ETHx exchange rate directly reduces `totalETHInProtocol`, which reduces `newRsETHPrice`.

**Step 3 — Single-call downside protection triggers a protocol-wide pause.**
Inside `_updateRsETHPrice()`, if `newRsETHPrice < highestRsethPrice` and `diff > pricePercentageLimit.mulWad(highestRsethPrice)`, the function calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` on `LRTOracle` itself, then returns immediately. This is a single atomic transaction.

**Step 4 — The trigger function is permissionless.**
`updateRSETHPrice()` is declared `public whenNotPaused` with no role restriction. Any EOA or contract can call it at any time.

**Step 5 — No permissionless recovery path exists.**
All three `unpause()` functions are gated by `onlyLRTAdmin`. There is no time-based auto-unpause, no permissionless recovery once the price recovers, and no mechanism to distinguish a transient rate dip from a genuine protocol loss.

**Exploit flow:**
1. Stader experiences a validator slash event; `getExchangeRate()` returns a value below the previous peak by more than `pricePercentageLimit`.
2. Attacker (any EOA) calls `lrtOracle.updateRSETHPrice()`.
3. `_updateRsETHPrice()` computes `newRsETHPrice` using the depressed ETHx rate, finds `isPriceDecreaseOffLimit == true`, and pauses all three contracts.
4. All deposits (`depositETH`, `depositAsset`) and all withdrawals (`initiateWithdrawal`, `completeWithdrawal`, `instantWithdrawal`) revert with `Paused`.
5. Protocol remains frozen until an admin manually calls `unpause()` on each contract.

## Impact Explanation
**Medium — Temporary freezing of funds.** All user-facing deposit and withdrawal functions across `LRTDepositPool` and `LRTWithdrawalManager` are blocked for the duration of the pause. Users with pending withdrawals cannot complete them; new depositors cannot enter. The freeze is not permanent in the absolute sense — a live admin can call `unpause()` — but if the admin is a multisig with a timelock or is slow to respond, users are locked out for an extended and unpredictable period. This matches the allowed impact class "Medium: Temporary freezing of funds."

## Likelihood Explanation
**Medium.** Validator slashing on Stader is a realistic, historically observed event. The ETHx exchange rate is monotonically increasing under normal conditions but decreases when validators are slashed. `pricePercentageLimit` is a live admin-configurable parameter; if set to a tight value (e.g., 0.5–1%), even a modest slash suffices to trigger the condition. The trigger call is permissionless, so any on-chain observer of a Stader slash event can immediately call `updateRSETHPrice()` to lock the protocol. The attack requires no capital, no special privileges, and is repeatable after each unpause if the ETHx rate remains depressed.

## Recommendation
1. **Add a staleness check** in `EthXPriceOracle.getAssetPrice()`: compare `block.timestamp` against the last update timestamp from Stader's oracle and revert if the data is stale beyond a configurable threshold.
2. **Add a minimum consecutive-anomaly requirement** before triggering the pause (e.g., require two successive out-of-bounds readings within a short window), preventing a single transient dip from locking the protocol.
3. **Add a permissionless unpause path** conditioned on the price having recovered above the pause threshold, so a transient anomaly does not require manual admin intervention and cannot be weaponized for indefinite griefing.
4. **Consider restricting `updateRSETHPrice()`** to a keeper role or adding a minimum call interval to reduce the attack surface for timing-based exploitation.

function test_transientETHxDropPausesProtocol() public {
    // 1. Set pricePercentageLimit to 1% (1e16)
    vm.prank(admin);
    lrtOracle.setPricePercentageLimit(1e16);

    // 2. Establish highestRsethPrice at normal rate
    lrtOracle.updateRSETHPrice();
    uint256 peak = lrtOracle.highestRsethPrice();
    assertGt(peak, 0);

    // 3. Mock getExchangeRate() to return a value 2% below current
    //    (simulating a slash event reducing total ETH in Stader)
    uint256 slashedRate = IETHXStakePoolsManager(staderProxy).getExchangeRate() * 98 / 100;
    vm.mockCall(
        staderProxy,
        abi.encodeWithSelector(IETHXStakePoolsManager.getExchangeRate.selector),
        abi.encode(slashedRate)
    );

    // 4. Any EOA calls updateRSETHPrice() — no role required
    vm.prank(address(0xdead));
    lrtOracle.updateRSETHPrice();

    // 5. All three contracts are now paused

    vm.prank(address(0xdead));
    vm.expectRevert();
    lrtOracle.unpause();

    vm.prank(user);
    vm.expectRevert();
    lrtDepositPool.depositETH{value: 1 ether}(0, "");
}
