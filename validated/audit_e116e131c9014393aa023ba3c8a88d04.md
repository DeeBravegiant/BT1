Audit Report

## Title
Unprivileged Caller Can Trigger Protocol-Wide Auto-Pause via Public `updateRSETHPrice()` — (`contracts/LRTOracle.sol`)

## Summary

`LRTOracle.updateRSETHPrice()` is callable by any address with no role restriction. When the computed `newRsETHPrice` falls more than `pricePercentageLimit` below `highestRsethPrice`, the downside-protection branch in `_updateRsETHPrice()` unconditionally pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. Because no privileged role is required to invoke this path, any external caller can force a protocol-wide pause whenever the price condition is satisfied, freezing all deposits and withdrawals until an admin manually unpauses each contract.

## Finding Description

**Entry point — no access control:**

`updateRSETHPrice()` carries only the `whenNotPaused` modifier; there is no role check. [1](#0-0) 

The privileged counterpart `updateRSETHPriceAsManager()` is gated with `onlyLRTManager`, confirming the protocol recognizes that some price-update paths require privilege — but the public path is left unrestricted. [2](#0-1) 

**Auto-pause branch — no `msg.sender` check:**

Inside `_updateRsETHPrice()`, if `newRsETHPrice < highestRsethPrice` and the difference exceeds `pricePercentageLimit.mulWad(highestRsethPrice)`, the function pauses all three contracts and returns early. There is no check on who the caller is. [3](#0-2) 

**Oracle — no staleness or bounds guard:**

`RETHPriceOracle.getAssetPrice()` blindly forwards the live Rocket Pool exchange rate with zero validation, meaning any real on-chain rate drop is immediately reflected in the price computation. [4](#0-3) 

**`_pause()` is unconditional:**

The internal `_pause()` helper simply sets `paused = true`; it has no caller guard of its own. [5](#0-4) 

**Exploit flow:**

1. `highestRsethPrice` is set to the all-time high rsETH price (updated on every successful price increase).
2. A real on-chain event causes `newRsETHPrice` to fall more than `pricePercentageLimit` below `highestRsethPrice` — e.g., a Rocket Pool slashing event reducing `rETH.getExchangeRate()`, or a price drop in any other supported LST asset.
3. An unprivileged EOA calls `updateRSETHPrice()`.
4. `_updateRsETHPrice()` computes the new price, enters the downside-protection branch, and calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()`.
5. All deposits and withdrawals are frozen. Only an admin can unpause each contract individually.

Without the public entry point, a manager could choose to delay the price update until the rate recovers, avoiding an unnecessary pause. The public function removes that discretion entirely.

## Impact Explanation

All deposits (`LRTDepositPool`) and withdrawals (`LRTWithdrawalManager`) are frozen until an admin manually unpauses each contract. This is **temporary freezing of funds**, a valid Medium impact per the allowed scope. The freeze is not permanent because an admin can unpause, but it requires no capital, no special role, and no front-running from the attacker — a single transaction suffices.

## Likelihood Explanation

- `updateRSETHPrice()` is public; any EOA or contract can call it at any time the contract is not already paused.
- The price condition requires a real on-chain drop in the rsETH price relative to `highestRsethPrice`. This is realistic during Rocket Pool slashing events or during price drops in other supported LST assets.
- `pricePercentageLimit` is admin-configurable; at 1 % (`1e16`), even modest slashing or LST price volatility can satisfy the condition.
- No capital, no special role, and no victim interaction is required — a single `updateRSETHPrice()` call from any address triggers the pause.
- The attack is repeatable: after an admin unpauses, if the price condition still holds, any caller can immediately re-trigger the pause.

## Recommendation

1. **Restrict `updateRSETHPrice()` to privileged callers** — gate it with `onlyLRTManager` or a dedicated `PRICE_UPDATER_ROLE`, consistent with `updateRSETHPriceAsManager()`. If a public keeper path is needed, add a time-lock or cooldown so the auto-pause branch is only reachable by privileged callers.
2. **Add staleness and bounds validation to `RETHPriceOracle`** — reject prices that deviate more than a configurable band from the last accepted value, or that are older than a maximum age.
3. **Separate the auto-pause trigger from the public price-update path** — the downside-protection pause should only be reachable by privileged callers or a dedicated keeper with a time-lock, not by arbitrary public callers.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.27;

import "forge-std/Test.sol";

contract ExploitTest is Test {
    ILRTOracle oracle = ILRTOracle(<deployed_oracle>);
    ILRTDepositPool pool = ILRTDepositPool(<deployed_pool>);
    ILRTWithdrawalManager wm = ILRTWithdrawalManager(<deployed_wm>);
    IrETH reth = IrETH(<reth_address>);

    function testUnprivilegedPause() external {
        // 1. Admin sets pricePercentageLimit to 1% (1e16)
        vm.prank(admin);
        oracle.setPricePercentageLimit(1e16);

        // 2. Record current highestRsethPrice (set from prior updates)
        uint256 peak = oracle.highestRsethPrice();

        // 3. Mock rETH.getExchangeRate() to return a value 2% below
        //    the value that produced the current peak rsETH price
        uint256 dippedRate = reth.getExchangeRate() * 98 / 100;
        vm.mockCall(
            address(reth),
            abi.encodeWithSelector(IrETH.getExchangeRate.selector),
            abi.encode(dippedRate)
        );

        // 4. Unprivileged caller triggers the auto-pause
        vm.prank(address(1));
        oracle.updateRSETHPrice();

        // 5. Assert all three contracts are paused
        assertTrue(pool.paused(),   "DepositPool not paused");
        assertTrue(wm.paused(),     "WithdrawalManager not paused");
        assertTrue(oracle.paused(), "LRTOracle not paused");
    }
}
```

The test requires no privileged role. It passes on unmodified production code whenever the mocked (or real) rate causes `newRsETHPrice` to fall more than `pricePercentageLimit` below `highestRsethPrice`. On a mainnet fork, step 3 can be replaced with a real Rocket Pool slashing simulation or a real LST price drop, removing the need for `vm.mockCall` entirely.
