Audit Report

## Title
Unchecked `int256`â†’`uint256` Cast of Chainlink Price Silently Accepts Non-Positive Values â€” (`File: contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` casts the raw `int256 answer` from Chainlink's `latestRoundData()` directly to `uint256` with no sign check. A zero price silently propagates into `LRTOracle._updateRsETHPrice()`, collapsing computed TVL and triggering the protocol's downside-protection pause, freezing all deposits and withdrawals. A negative price causes an overflow revert on every oracle call, achieving the same freeze via a different path. The sister oracle in the same repo already applies the correct guard, confirming the omission is an oversight.

## Finding Description
`ChainlinkPriceOracle.getAssetPrice()` at lines 49â€“55 reads the Chainlink answer and immediately casts it without any sign validation:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

In Solidity 0.8, explicit type casts (`uint256(int256)`) do **not** revert on negative values â€” they wrap via two's complement. Only subsequent arithmetic operations are checked. Two failure modes follow:

**Case A â€” `price < 0`:** `uint256(negative_int256)` wraps to a value near `2^256`. The subsequent `* 1e18` multiplication overflows and reverts under Solidity 0.8 checked arithmetic. Every call routing through this oracle reverts, freezing deposits and price updates.

**Case B â€” `price == 0`:** `uint256(0) * 1e18 / decimals = 0`. The function silently returns `0`. This zero is consumed by `_getTotalEthInProtocol()` in `LRTOracle`, which sums `totalAssetAmt.mulWad(assetER)` for each supported asset. [2](#0-1) 

With a zero asset price, the affected asset contributes nothing to `totalETHInProtocol`, collapsing `newRsETHPrice`. The downside-protection logic then evaluates:

```solidity
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
``` [3](#0-2) 

`updateRSETHPrice()` is `public` with no role restriction â€” any unprivileged caller can trigger this path. [4](#0-3) 

The correct guard is already present in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
``` [5](#0-4) 

`ChainlinkPriceOracle` is the oracle registered for LST assets (e.g., stETH) via `assetPriceOracle` in `LRTOracle`. [6](#0-5) 

## Impact Explanation
**Temporary freezing of funds (Medium).** In Case B, a zero Chainlink price causes `LRTDepositPool` and `LRTWithdrawalManager` to be paused automatically, freezing all user deposits and withdrawals until an admin manually unpauses after the oracle recovers. In Case A, the overflow revert freezes deposits and price updates for the duration of the anomalous round. Neither path results in permanent loss of funds, placing this squarely in the **Medium â€” Temporary freezing of funds** category.

## Likelihood Explanation
Chainlink LST/ETH feeds (e.g., stETH/ETH) are mature and rarely return non-positive values under normal conditions. However, circuit breakers can clamp prices at configured minimums (including zero) during extreme market stress or feed misconfiguration. No privileged role is required to trigger the impact â€” any public caller of `updateRSETHPrice()` (e.g., a keeper bot) initiates the path. The omission is confirmed by the presence of the identical guard in `ChainlinkOracleForRSETHPoolCollateral`, indicating the team is aware of the risk class.

## Recommendation
Add a non-positive price guard in `ChainlinkPriceOracle.getAssetPrice()`, mirroring the pattern in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
function getAssetPrice(address asset) external view onlySupportedAsset(asset) returns (uint256) {
    AggregatorV3Interface priceFeed = AggregatorV3Interface(assetPriceFeed[asset]);
    (, int256 price,,,) = priceFeed.latestRoundData();

    if (price <= 0) revert InvalidPrice();  // add this guard

    return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
}
```

Additionally, consider adding staleness checks (`answeredInRound < roundId`, `timestamp == 0`) consistent with `ChainlinkOracleForRSETHPoolCollateral.getRate()`. [7](#0-6) 

## Proof of Concept
1. Chainlink's stETH/ETH feed enters a circuit-breaker state and returns `answer = 0` for one round.
2. Any unprivileged caller invokes `LRTOracle.updateRSETHPrice()`.
3. `_updateRsETHPrice()` â†’ `_getTotalEthInProtocol()` â†’ `getAssetPrice(stETH)` â†’ `ChainlinkPriceOracle.getAssetPrice(stETH)` â†’ returns `0`.
4. stETH contributes `0 * balance / 1e18 = 0` to `totalETHInProtocol`, collapsing computed TVL.
5. `newRsETHPrice` drops far below `highestRsethPrice`; with `pricePercentageLimit > 0`, `isPriceDecreaseOffLimit` is `true`.
6. `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` are called automatically.
7. All user deposits and withdrawals are frozen until an admin unpauses â€” a temporary freeze of funds.
