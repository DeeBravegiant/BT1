Audit Report

## Title
Stale Cross-Chain Rate in `AGETHRateReceiver` Enables Over-Minting of agETH, Diluting Existing Holders' Yield — (`contracts/cross-chain/CrossChainRateReceiver.sol`, `contracts/agETH/AGETHPoolV3.sol`)

## Summary

`CrossChainRateReceiver.getRate()` returns the last LayerZero-pushed rate with no staleness check against `lastUpdated`. `AGETHPoolV3` uses this rate as the sole denominator when computing agETH mint amounts. When the agETH/ETH rate appreciates on mainnet but the receiver is not updated, new depositors receive excess agETH, permanently diluting the yield accrued by existing holders.

## Finding Description

`CrossChainRateReceiver` records `lastUpdated` on every `lzReceive` call but never reads it in `getRate()`:

```solidity
// CrossChainRateReceiver.sol L95-97
rate = _rate;
lastUpdated = block.timestamp;   // written, never read by getRate()

// CrossChainRateReceiver.sol L103-105
function getRate() external view returns (uint256) {
    return rate;                  // no staleness guard
}
```

`AGETHRateReceiver` adds no override or freshness logic — it is a pure thin wrapper with only a constructor.

`AGETHPoolV3.deposit()` calls `viewSwapAgETHAmountAndFee()`, which divides by the oracle rate unconditionally:

```solidity
// AGETHPoolV3.sol L160-168
uint256 agETHToETHrate = getRate();           // stale rate accepted silently
agETHAmount = amountAfterFee * 1e18 / agETHToETHrate;
```

`getRate()` in the pool is a direct pass-through to `agETHOracle` (the `AGETHRateReceiver`). There is no staleness check at any layer of the call stack. The minted agETH is immediately transferred to the depositor, making the dilution irreversible.

## Impact Explanation

**High — Theft of unclaimed yield.** When the agETH/ETH rate appreciates (e.g., 1.00 → 1.05 over weeks), the stale denominator causes `agETHAmount = amountAfterFee * 1e18 / 1.00e18` instead of the correct `/ 1.05e18`. Each depositor receives ~5% excess agETH. This excess supply is not backed by ETH, so it permanently reduces the per-token ETH claim of every existing holder — the yield they have already accrued is transferred to new depositors. The dilution is irreversible once `agETH.mint()` executes.

## Likelihood Explanation

No attacker action is required. The condition is passive: any operational gap (LayerZero downtime, insufficient fee balance, operator lapse) leaves the rate stale. agETH is a continuously appreciating yield-bearing token, so any multi-week gap produces a measurable discrepancy. The `deposit()` function is fully public and callable by any address with no role restriction.

## Recommendation

Add a staleness threshold to `CrossChainRateReceiver.getRate()`:

```solidity
uint256 public constant MAX_RATE_AGE = 1 days;

function getRate() external view returns (uint256) {
    require(block.timestamp - lastUpdated <= MAX_RATE_AGE, "Rate is stale");
    return rate;
}
```

Alternatively, enforce the check inside `AGETHPoolV3.viewSwapAgETHAmountAndFee()` by reading `lastUpdated` directly from the oracle interface and reverting if the age exceeds the threshold.

## Proof of Concept

```solidity
// Fork destination chain at a block where lastUpdated is current.
// Do NOT call updateRate() for 30 days.
// Mainnet rate has appreciated from 1.00e18 to 1.05e18.
// AGETHRateReceiver still holds rate = 1.00e18.

vm.warp(block.timestamp + 30 days);

uint256 depositAmount = 1 ether;
(uint256 agETHAmount, ) = agETHPoolV3.viewSwapAgETHAmountAndFee(depositAmount);

// Stale rate 1.00e18  → agETHAmount ≈ 1.00e18 (minus fee)
// Correct rate 1.05e18 → agETHAmount ≈ 0.952e18 (minus fee)
// ~5% excess minted; existing holders' yield permanently diluted.

assertGt(agETHAmount, correctAmount * 105 / 100);
assertGt(block.timestamp - agETHRateReceiver.lastUpdated(), 30 days);
```

Root cause confirmed at:
- `CrossChainRateReceiver.getRate()` L103–105 — no staleness check
- `AGETHPoolV3.viewSwapAgETHAmountAndFee()` L160–168 — unconditional rate consumption
- `AGETHPoolV3.deposit()` L121–125 — mints and transfers immediately, making dilution irreversible
