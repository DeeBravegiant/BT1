Audit Report

## Title
Downside Protection in `_updateRsETHPrice()` Compares Against All-Time High, Causing Temporary Freeze of User Withdrawals on Legitimate EigenLayer Slashing - (File: contracts/LRTOracle.sol)

## Summary

`LRTOracle._updateRsETHPrice()` auto-pauses the entire protocol whenever the new rsETH price drops more than `pricePercentageLimit` below `highestRsethPrice` (the all-time high). Because the threshold is anchored to the ATH and never resets downward, a legitimate EigenLayer slashing event that causes even a modest percentage drop can exceed the threshold and trigger a protocol-wide pause. Any public caller of `updateRSETHPrice()` can fire this path immediately after slashing is reflected on-chain, freezing rsETH already locked in `LRTWithdrawalManager` until admin manually unpauses.

## Finding Description

`_updateRsETHPrice()` computes the pause condition as:

```
diff = highestRsethPrice - newRsETHPrice
isPriceDecreaseOffLimit = pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice)
``` [1](#0-0) 

`highestRsethPrice` is only ever updated upward and is never reset: [2](#0-1) 

This means the effective pause threshold is `newRsETHPrice < highestRsethPrice × (1 − pricePercentageLimit)`. If the ATH equals the current price (the common steady-state), a slashing event causing a drop larger than `pricePercentageLimit` from the current price immediately triggers the pause. `updateRSETHPrice()` is unrestricted: [3](#0-2) 

When the pause fires, `LRTWithdrawalManager` is paused. Both `completeWithdrawal` and `unlockQueue` carry `whenNotPaused`: [4](#0-3) [5](#0-4) 

Users who called `initiateWithdrawal` already transferred their rsETH into the contract: [6](#0-5) 

Their rsETH is locked with no trustless recovery path. The oracle itself is also paused, blocking any further `updateRSETHPrice()` calls until admin intervention: [7](#0-6) 

Unpausing requires `onlyLRTAdmin` on both `LRTWithdrawalManager` and `LRTOracle`: [8](#0-7) [9](#0-8) 

## Impact Explanation

**Temporary freezing of funds (Medium).** Users who have initiated withdrawals — transferring rsETH to `LRTWithdrawalManager` — cannot complete them while the protocol is paused. The rsETH is held in the contract with no trustless exit. Recovery requires admin (`onlyLRTAdmin`) to call `unpause()` on `LRTWithdrawalManager` and `LRTOracle`. If governance processes are slow or the admin key is unavailable, the freeze is extended.

## Likelihood Explanation

EigenLayer operator slashing is an explicitly documented and expected risk of restaking, confirmed by the presence of `SlashingLib.sol` in this repository. When slashing reduces the ETH value of assets held by `NodeDelegator`, `_getTotalEthInProtocol()` returns a lower value, lowering `newRsETHPrice`. Since `updateRSETHPrice()` is public, any user, keeper, or bot can trigger this path immediately after slashing is reflected on-chain. No special privileges are required. The scenario is repeatable on every slashing event.

## Recommendation

1. **Compare against the previous price, not the all-time high.** Use `previousPrice` (already computed at line 228) as the reference for the downside threshold instead of `highestRsethPrice`. This prevents a single large historical appreciation from making the protocol permanently sensitive to small absolute drops.
2. **Use a separate, larger downside threshold.** Introduce a dedicated `maxDownsidePercentage` calibrated to realistic slashing magnitudes, distinct from the upside `pricePercentageLimit`.
3. **Emit and alert rather than auto-pause.** Emit an event and let governance decide whether to pause, rather than auto-pausing on every price update that crosses the threshold.

## Proof of Concept

1. Protocol launches; rsETH price grows from `1.0e18` to `1.5e18`. `highestRsethPrice = 1.5e18`.
2. Admin sets `pricePercentageLimit = 1e16` (1%).
3. Pause threshold: `1.5e18 × 1% = 0.015e18`. Any price below `1.485e18` triggers the pause.
4. User calls `initiateWithdrawal`, transferring rsETH to `LRTWithdrawalManager`.
5. An EigenLayer operator is slashed; restaked asset value drops 1.5%. New rsETH price = `1.4775e18`. Diff = `0.0225e18 > 0.015e18`.
6. Any address calls `updateRSETHPrice()`.
7. `_updateRsETHPrice()` detects `isPriceDecreaseOffLimit = true`, pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`.
8. User cannot call `completeWithdrawal` (reverts on `whenNotPaused`). Operator cannot call `unlockQueue` (reverts on `whenNotPaused`).
9. rsETH remains locked in `LRTWithdrawalManager` until admin calls `unpause()` on both contracts.
