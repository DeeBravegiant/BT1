### Title
Deposits for Non-Winning `propose_update` Proposals Are Permanently Locked When Any Update Executes - (File: crates/contract/src/update.rs)

### Summary
When `do_update` executes a winning contract/config proposal, it unconditionally clears **all** pending proposals and their associated storage-staking deposits. Because `UpdateEntry` stores no proposer identity and no refund path exists, the NEAR tokens locked by every non-winning proposer are permanently trapped in the contract.

### Finding Description
`propose_update` in `crates/contract/src/lib.rs` requires each proposer to attach a deposit sized to cover the storage cost of their proposal (contract WASM bytes + overhead):

```rust
let required = ProposedUpdates::required_deposit(&update);
// exact required amount is kept; only the excess is refunded
``` [1](#0-0) 

The deposit is computed from the raw WASM size plus a 128-account vote-overhead estimate:

```rust
fn bytes_used(update: &Update) -> u128 {
    let mut bytes_used = std::mem::size_of::<UpdateEntry>() as u128;
    bytes_used += 128 * std::mem::size_of::<AccountId>() as u128;
    match update {
        Update::Contract(code) => { bytes_used += code.len() as u128; }
        ...
    }
    bytes_used
}
``` [2](#0-1) 

When threshold votes are reached, `do_update` removes the winning entry and then calls `.clear()` on the entire entries map, erasing every other pending proposal:

```rust
pub fn do_update(&mut self, id: &UpdateId, gas: Gas) -> Option<Promise> {
    let entry = self.entries.remove(id)?;
    // Clear all entries as they might be no longer valid
    self.entries.clear();
    self.vote_by_participant.clear();
    ...
}
``` [3](#0-2) 

`UpdateEntry` contains no `proposer: AccountId` field, so there is no record of who paid for each cleared proposal and therefore no way to issue refunds: [4](#0-3) 

The contract has no separate "cancel proposal and refund" endpoint. The only method that touches proposals after creation is `vote_update` → `do_update`, which never refunds.

### Impact Explanation
A typical NEAR contract WASM is 1–5 MB. At NEAR's storage cost of ~1 NEAR per 10 KB, a single contract-upgrade proposal deposit is **100–500 NEAR**. When multiple participants each propose a different upgrade (a normal governance scenario), only one wins; every other proposer's deposit is silently absorbed into the contract balance with no recovery path. This breaks the production accounting invariant that storage-staking deposits must be returned when the storage they cover is freed.

### Likelihood Explanation
The scenario is routine: the README and tests explicitly show multiple simultaneous proposals (`test_propose_update_contract_many`). Every successful contract upgrade that races against at least one other proposal triggers the loss. No adversarial action is required — normal governance operation is sufficient. [5](#0-4) 

### Recommendation
- Store the proposer `AccountId` inside `UpdateEntry` at proposal time.
- In `do_update`, before calling `self.entries.clear()`, iterate over all remaining entries and schedule `Promise::new(entry.proposer).transfer(refund_amount)` for each, where `refund_amount = env::storage_byte_cost() * entry.bytes_used`.
- Alternatively, expose a `cancel_update(id)` endpoint that lets a proposer withdraw their own proposal and receive a refund, and call it implicitly for all non-winning entries inside `do_update`.

### Proof of Concept
1. Participant A calls `propose_update` with a 1 MB WASM, attaching ~100 NEAR deposit. Proposal ID = 0.
2. Participant B calls `propose_update` with a different 1 MB WASM, attaching ~100 NEAR deposit. Proposal ID = 1.
3. Threshold participants vote for proposal 1. `vote_update` calls `do_update(&1, gas)`.
4. `do_update` removes entry 1, then calls `self.entries.clear()` — entry 0 (and its 100 NEAR deposit) is erased with no refund to Participant A.
5. Participant A's 100 NEAR is now permanently locked in the MPC contract balance with no mechanism to recover it. [6](#0-5) [7](#0-6)

    #[payable]
    #[handle_result]
    pub fn propose_update(
        &mut self,
        #[serializer(borsh)] args: ProposeUpdateArgs,
    ) -> Result<UpdateId, Error> {
        // Only voters can propose updates:
        let proposer = self.voter_or_panic();
        let update: Update = args.try_into()?;

        let attached = env::attached_deposit();
        let required = ProposedUpdates::required_deposit(&update);
        if attached < required {
            return Err(InvalidParameters::InsufficientDeposit {
                attached: attached.as_yoctonear(),
                required: required.as_yoctonear(),
            }
            .into());
        }

        let id = self.proposed_updates.propose(update);

        log!(
            "propose_update: signer={}, id={:?}",
            env::signer_account_id(),
            id,
        );

        // Refund the difference if the proposer attached more than required.
        if let Some(diff) = attached.checked_sub(required)
            && diff > NearToken::from_yoctonear(0)
        {
            Promise::new(proposer).transfer(diff).detach();
        }

        Ok(id)
    }

    pub fn do_update(&mut self, id: &UpdateId, gas: Gas) -> Option<Promise> {
        let entry = self.entries.remove(id)?;

        // Clear all entries as they might be no longer valid
        self.entries.clear();
        self.vote_by_participant.clear();

        let mut promise = Promise::new(env::current_account_id());
        match entry.update {
            Update::Contract(code) => {
                // deploy contract then do a `migrate` call to migrate state.
                promise = promise.deploy_contract(code).function_call(
                    method_names::MIGRATE,
                    Vec::new(),
                    NearToken::from_near(0),
                    gas,
                );
            }
            Update::Config(config) => {
                // If we vote for a new config, we should use
                // the value `contract_upgrade_deposit_tera_gas` from the config
                // as the new gas value
                let new_config_gas_value = Gas::from_tgas(config.contract_upgrade_deposit_tera_gas);
                promise = promise.function_call(
                    method_names::UPDATE_CONFIG,
                    serde_json::to_vec(&(&config,)).unwrap(),
                    NearToken::from_near(0),
                    new_config_gas_value,
                );
            }
        }
        Some(promise)
    }

fn bytes_used(update: &Update) -> u128 {
    let mut bytes_used = std::mem::size_of::<UpdateEntry>() as u128;

    // Assume a high max of 128 participant votes per update entry.
    bytes_used += 128 * std::mem::size_of::<AccountId>() as u128;

    match update {
        Update::Contract(code) => {
            bytes_used += code.len() as u128;
        }
        Update::Config(config) => {
            let bytes = serde_json::to_vec(&config).unwrap();
            bytes_used += bytes.len() as u128;
        }
    }

    bytes_used
}
