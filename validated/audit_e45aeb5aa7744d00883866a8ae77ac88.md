### Title
Sub-Transaction Reference Script Sizes Excluded from Batch Fee Calculation — (`eras/dijkstra/impl/src/Cardano/Ledger/Dijkstra/UTxO.hs`)

### Summary

The Dijkstra era's minimum-fee check for batch transactions uses `getConwayMinFeeTxUtxo`, which only measures reference-script sizes from the **top-level** transaction's inputs. Sub-transaction reference inputs are aggregated by `batchNonDistinctRefScriptsSize` for the size-limit check but are **never** fed into the tiered fee formula. An unprivileged submitter can therefore pay a fee that covers only the top-level reference-script deserialization overhead while forcing nodes to deserialize arbitrarily large sub-transaction reference scripts at no extra cost.

---

### Finding Description

**`getConwayMinFeeTxUtxo` — only top-level inputs** [1](#0-0) 

```haskell
getConwayMinFeeTxUtxo pparams tx utxo =
  getMinFeeTx pparams tx $ txNonDistinctRefScriptsSize utxo tx
```

`txNonDistinctRefScriptsSize` unions only `referenceInputsTxBodyL` and `inputsTxBodyL` of the **single** `tx` argument: [2](#0-1) 

```haskell
txNonDistinctRefScriptsSize utxo tx = getSum $ foldMap (Sum . originalBytesSize . snd) refScripts
  where
    inputs = (tx ^. bodyTxL . referenceInputsTxBodyL) `Set.union` (tx ^. bodyTxL . inputsTxBodyL)
    refScripts = getReferenceScriptsNonDistinct utxo inputs
```

Sub-transactions are never visited.

**Dijkstra wires `getConwayMinFeeTxUtxo` as its `getMinFeeTxUtxo`** [3](#0-2) 

**Fee check in `dijkstraUtxoTransition` calls `Shelley.validateFeeTooSmallUTxO`** [4](#0-3) 

`validateFeeTooSmallUTxO` calls `getMinFeeTxUtxo` internally: [5](#0-4) 

So the minimum fee is computed with `txNonDistinctRefScriptsSize` applied to the top-level tx only — sub-tx reference inputs are invisible to it.

**`batchNonDistinctRefScriptsSize` does aggregate sub-tx sizes — but only for the size cap** [6](#0-5) 

```haskell
batchNonDistinctRefScriptsSize utxo tx =
  txNonDistinctRefScriptsSize utxo tx
    + getSum (foldMap' (Sum . txNonDistinctRefScriptsSize utxo)
                       (tx ^. bodyTxL . subTransactionsTxBodyL))
```

This is used only in `validateAllRefScriptSize`, which enforces `ppMaxRefScriptSizePerTxG`: [7](#0-6) 

The size cap prevents unbounded scripts, but it does **not** feed the aggregated size back into the tiered fee formula. The fee check and the size check use different size computations.

---

### Impact Explanation

The tiered ref-script fee was introduced specifically to price the deserialization overhead of large Plutus scripts (see ADR-009). The same overhead exists for sub-transaction reference scripts in a Dijkstra batch, but the fee formula ignores it. An attacker can fill sub-transactions with reference inputs pointing to large scripts (up to `ppMaxRefScriptSizePerTxG` in aggregate) and pay only the fee for the top-level tx's reference scripts. This underprices batch transactions with large sub-tx reference scripts, recreating the exact DDoS vector the tiered fee was designed to close.

Impact: **Medium** — attacker-controlled transactions modify fees outside design parameters.

---

### Likelihood Explanation

The exploit requires no privilege: any user can submit a batch transaction. The discrepancy is structural — `getMinFeeTxUtxo = getConwayMinFeeTxUtxo` is set at the `EraUTxO DijkstraEra` instance level and `txNonDistinctRefScriptsSize` has no awareness of sub-transactions. The attack is locally testable by constructing a batch tx whose sub-txs reference large scripts and comparing `getConwayMinFeeTxUtxo` against `batchNonDistinctRefScriptsSize`-based fee.

---

### Recommendation

Override `getMinFeeTxUtxo` in the `EraUTxO DijkstraEra` instance to use `batchNonDistinctRefScriptsSize` instead of `txNonDistinctRefScriptsSize`:

```haskell
getMinFeeTxUtxo pp tx utxo =
  getMinFeeTx pp tx (batchNonDistinctRefScriptsSize utxo tx)
```

This mirrors the pattern already used in `validateAllRefScriptSize` and ensures the tiered fee formula receives the full batch reference-script size.

---

### Proof of Concept

1. Deploy a UTxO entry containing a large Plutus script (e.g., 50 KiB) as a reference script.
2. Construct a Dijkstra batch transaction whose **sub-transaction** has `referenceInputsTxBodyL` pointing to that UTxO entry; the top-level tx has no reference inputs.
3. Compute `getConwayMinFeeTxUtxo pp batchTx utxo` — the result includes zero tiered ref-script fee.
4. Compute `getMinFeeTx pp batchTx (batchNonDistinctRefScriptsSize utxo batchTx)` — the result includes the tiered fee for the 50 KiB script.
5. Submit the batch tx with the fee from step 3. `validateFeeTooSmallUTxO` passes; `validateAllRefScriptSize` also passes (50 KiB < `ppMaxRefScriptSizePerTxG`). The transaction is accepted with an underpaid fee.

validateFeeTooSmallUTxO ::
  EraUTxO era =>

  Tx TopTx era ->

validateFeeTooSmallUTxO pp tx utxo =
  failureUnless (minFee <= txFee) $
    FeeTooSmallUTxO

validateAllRefScriptSize ::
  ( EraTx era
  , DijkstraEraTxBody era
  ) =>
  PParams era ->
  UTxO era ->
  Tx TopTx era ->
  Test (DijkstraLedgerPredFailure era)
validateAllRefScriptSize pp utxo tx =
  let totalRefScriptSize = batchNonDistinctRefScriptsSize utxo tx
      maxRefScriptSizePerTx = fromIntegral @Word32 @Int $ pp ^. ppMaxRefScriptSizePerTxG
   in failureUnless (totalRefScriptSize <= maxRefScriptSizePerTx) $
        DijkstraTxRefScriptsSizeTooBig
          Mismatch
            { mismatchSupplied = totalRefScriptSize
            , mismatchExpected = maxRefScriptSizePerTx
            }
