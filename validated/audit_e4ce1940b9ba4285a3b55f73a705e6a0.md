Audit Report

## Title
Fee Truncation to Zero Allows Fee-Free Deposits for Low-Decimal Tokens - (`contracts/pools/RSETHPoolV3.sol`, `contracts/agETH/AGETHPoolV3.sol`, `contracts/pools/RSETHPool.sol`, `contracts/pools/RSETHPoolV3ExternalBridge.sol`, `contracts/pools/RSETHPoolNoWrapper.sol`)

## Summary
All pool contracts compute protocol fees as `amount * feeBps / 10_000` using Solidity integer division, which truncates toward zero. For tokens with low decimal precision (e.g., EURS at 2 decimals), deposits below the truncation threshold produce a fee of exactly `0`, allowing depositors to receive the full rsETH/agETH mint while the protocol collects no fee revenue. This is exploitable by any unprivileged depositor once a low-decimal token is added as a supported token.

## Finding Description
The fee calculation is identical across all affected contracts:

```solidity
fee = amount * feeBps / 10_000;
uint256 amountAfterFee = amount - fee;
```

For `fee > 0` to hold, `amount * feeBps >= 10_000` must be satisfied. With `feeBps = 5` (0.05%) and a 2-decimal token like EURS:

- Threshold: `amount < 10_000 / 5 = 2_000` units → `< 20 EURS`
- `1_999 * 5 / 10_000 = 9_995 / 10_000 = 0` (truncated)

The deposit function in `RSETHPoolV3.sol` accumulates `feeEarnedInToken[token] += fee`, so a zero fee means no protocol revenue is recorded for that deposit. The `addSupportedToken` function accepts any ERC20 token with a valid oracle and imposes no decimal-precision constraint, making low-decimal token support a realistic operational scenario. There are no minimum deposit checks, no minimum fee checks, and no guards preventing repeated sub-threshold deposits. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

## Impact Explanation
**High — Theft of unclaimed yield (protocol fees).** Protocol fees are the treasury's yield from pool operations. When `fee` truncates to `0`, the protocol receives no revenue from the deposit while the depositor receives the full rsETH/agETH mint equivalent of their entire deposit. The depositor effectively receives a benefit (fee-free swap) that the protocol is contractually entitled to collect. This can be repeated indefinitely for every deposit below the truncation threshold, systematically eliminating all fee revenue for that token. The depositor receives more value than intended at the direct expense of the protocol's fee accrual.

## Likelihood Explanation
The precondition is that a low-decimal token must be added via `addSupportedToken` (a TIMELOCK_ROLE action). This is a realistic operational decision: EURS (2 decimals), USDC (6 decimals), and USDT (6 decimals) are common bridged stablecoins on L2 chains where these pools are deployed. Once such a token is added, any unprivileged depositor can exploit the truncation without any special privileges by simply calling `deposit(token, amount, referralId)` with `amount` below the threshold. The attack requires no flash loans, no oracle manipulation, and no coordination. **Likelihood: Medium.**

## Recommendation
Add a minimum fee enforcement after computing the fee, or use ceiling division:

```solidity
// Option 1: Revert if fee rounds to zero (when a fee is configured)
fee = amount * feeBps / 10_000;
require(fee > 0 || feeBps == 0, "Fee truncated to zero");

// Option 2: Ceiling division to always round up
fee = (amount * feeBps + 9_999) / 10_000;
```

Alternatively, enforce a per-token minimum deposit amount in `addSupportedToken` or `deposit` that guarantees `amount * feeBps >= 10_000` for any configured `feeBps`.

## Proof of Concept
Using `AGETHPoolV3` with EURS (2 decimals) added as a supported token and `feeBps = 5`:

1. Attacker calls `deposit(EURS, 1_999, "ref")` (≈ 19.99 EURS).
2. `viewSwapAgETHAmountAndFee(1_999, EURS)` computes:
   - `fee = 1_999 * 5 / 10_000 = 9_995 / 10_000 = 0`
   - `amountAfterFee = 1_999 - 0 = 1_999`
3. `feeEarnedInToken[EURS] += 0` — no fee recorded.
4. Attacker receives agETH for the full 1,999 EURS with zero fee paid.
5. Repeating across many deposits, the protocol collects zero fees for all EURS deposits below 2,000 units.
