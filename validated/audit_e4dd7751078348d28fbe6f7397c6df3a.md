Audit Report

## Title
Missing `msg.sender == account` Validation Enables Forced Fee Extraction on Victim Claims — (`contracts/utils/MerkleDistributor/MerkleDistributor.sol`)

## Summary

`MerkleDistributor.claim` accepts a caller-supplied `account` parameter and verifies a merkle proof for it, but never checks that `msg.sender == account`. When `feeInBPS > 0`, any external caller who possesses a valid merkle proof for a victim can trigger the claim, causing a permanent, irrecoverable fee deduction from the victim's allocation that is sent to `protocolTreasury`. The sibling contract `KernelMerkleDistributor` explicitly guards against this with an `Unauthorized` revert, confirming the protocol is aware of the pattern but failed to apply it consistently.

## Finding Description

`MerkleDistributor.claim` (L97–147) accepts `account` as a caller-supplied parameter and verifies the merkle proof against `(index, account, cumulativeAmount)`, but contains no identity check: [1](#0-0) 

After proof verification, the fee is computed and permanently deducted: [2](#0-1) 

The fee cap is 10%: [3](#0-2) 

By contrast, `KernelMerkleDistributor._processClaim` enforces caller identity before any state changes: [4](#0-3) 

**Exploit path:**
1. Owner sets `feeInBPS = 1000` (10%) and publishes a merkle root.
2. Off-chain distribution publishes Alice's proof: `(index=1, account=Alice, cumulativeAmount=1000e18, proof=[...])`.
3. Attacker (Bob) calls `merkleDistributor.claim(1, Alice, 1000e18, aliceProof)`.
4. Proof verification passes — it is valid for Alice.
5. `fee = 100e18` is sent to `protocolTreasury`; `900e18` is sent to Alice.
6. `userClaims[Alice].lastClaimedIndex` is updated; Alice can never reclaim the 100e18 fee.
7. Bob spent only gas. Alice permanently lost 10% of her allocation.

No existing check prevents this. `whenNotPaused` only blocks when the contract is paused. The `isClaimed` check only prevents double-claiming. The merkle proof check validates the data but not the caller.

## Impact Explanation

**High — Theft of unclaimed yield.**

Every user whose merkle proof is known (all users, since proofs are distributed publicly for self-claiming) can be force-claimed at any time. At the maximum configured fee of 10%, up to 10% of every user's token allocation is permanently redirected to `protocolTreasury` without the user's consent. The victim's `userClaims` record is updated, making the loss irreversible.

## Likelihood Explanation

**High.** Merkle proofs for distributor contracts are always published off-chain — otherwise users cannot claim. No special privilege, role, flash loan, or capital is required. Any EOA can execute the attack for any victim in a single transaction, at any time after the merkle root is set and before the victim self-claims.

## Recommendation

Add a caller-identity check at the top of `claim`, mirroring the pattern already used in `KernelMerkleDistributor._processClaim`:

```solidity
if (account != msg.sender) {
    revert Unauthorized();
}
```

This requires adding `Unauthorized` to `IMerkleDistributor`'s error declarations (it is currently absent from `MerkleDistributor.sol`'s interface) and inserting the check before the merkle proof verification in `claim`. [5](#0-4) 

## Proof of Concept

**Foundry fork test outline:**

```solidity
function test_forcedClaimWithFee() public {
    // Setup: owner sets feeInBPS = 1000, sets merkle root
    vm.prank(owner);
    distributor.setFeeInBPS(1000);
    vm.prank(owner);
    distributor.setMerkleRoot(merkleRoot);

    uint256 aliceBalanceBefore = token.balanceOf(alice);
    uint256 treasuryBalanceBefore = token.balanceOf(protocolTreasury);

    // Attacker (bob) calls claim on behalf of alice using her public proof
    vm.prank(bob);
    distributor.claim(aliceIndex, alice, aliceCumulativeAmount, aliceProof);

    // Alice receives only 90% of her allocation
    assertEq(token.balanceOf(alice) - aliceBalanceBefore, aliceCumulativeAmount * 9000 / 10_000);
    // Treasury receives 10% fee — alice never consented
    assertEq(token.balanceOf(protocolTreasury) - treasuryBalanceBefore, aliceCumulativeAmount * 1000 / 10_000);
    // Alice cannot reclaim — already marked claimed
    vm.expectRevert(IMerkleDistributor.AlreadyClaimed.selector);
    vm.prank(alice);
    distributor.claim(aliceIndex, alice, aliceCumulativeAmount, aliceProof);
}
```

    function claim(
        uint256 index,
        address account,
        uint256 cumulativeAmount,
        bytes32[] calldata merkleProof
    )
        external
        override
        whenNotPaused
    {
        if (currentMerkleRoot == bytes32(0)) {
            revert ZeroValueProvided();
        }

        if (index == 0 || index > currentIndex) {
            revert InvalidIndex();
        }

        if (isClaimed(index, account)) {
            revert AlreadyClaimed();
        }

        // Verify the merkle proof.
        bytes32 node = keccak256(abi.encodePacked(index, account, cumulativeAmount));
        if (!MerkleProofUpgradeable.verify(merkleProof, currentMerkleRoot, node)) {
            revert InvalidMerkleProof();
        }
