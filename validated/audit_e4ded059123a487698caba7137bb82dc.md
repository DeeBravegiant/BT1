Audit Report

## Title
Missing L2 Sequencer Uptime Check Allows Stale-Price Swaps After Sequencer Recovery - (File: `smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

## Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are explicitly designed for L2 chains (evidenced by the `FUTURE_TOLERANCE` immutable for sequencer clock skew) but contain no sequencer uptime check. After an L2 sequencer outage, the pre-outage oracle price remains frozen in the oracle contract; upon recovery, `_isStale` passes if the outage duration is less than `MAX_TIME_DELTA` (up to 7 days), allowing `MetricOmmPool.swap` to execute at a price that no longer reflects current market conditions, causing direct LP principal loss.

## Finding Description

Both contracts implement `_isStale` that compares oracle `refTime` against `block.timestamp`:

```solidity
// PriceProviderL2.sol L135-150, ProtectedPriceProviderL2.sol L138-153
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

`MAX_TIME_DELTA` is bounded only by `(0, 7 days]`: [1](#0-0) 

Neither contract contains any reference to a sequencer uptime feed in production code — the `sequencerUptimeFeed` / `GRACE_PERIOD` pattern exists only in the compiled ABI of `ChainlinkVerifierL2` inside `registry.json`, confirming team awareness but no integration in the price providers.

**Exploit path:**

1. Sequencer goes down at `T=0`; oracle `refTime = T=0`, price = `$P`.
2. Real market price moves to `$P'` off-chain during outage (duration `D < MAX_TIME_DELTA`).
3. Sequencer recovers at `T=D`; `block.timestamp = D`, oracle `refTime = 0`.
4. `_isStale(0, D, MAX_TIME_DELTA, FUTURE_TOL)` → `D - 0 ≤ MAX_TIME_DELTA` → **not stale**.
5. `_getBidAndAskPrice()` returns bid/ask based on pre-outage `$P`. [2](#0-1) 
6. Pool swap settles at stale price: [3](#0-2) 

No additional guard exists between `getBidAndAskPrice()` and `_executeSwap`. The `FeedStalled` revert only triggers on the `(0, type(uint128).max)` sentinel, which is never returned when the outage is shorter than `MAX_TIME_DELTA`: [4](#0-3) [5](#0-4) 

## Impact Explanation

An unprivileged attacker can call `MetricOmmPool.swap` immediately after sequencer recovery, buying tokens at the pre-outage ask price when the real market price has moved adversarially. The pool's LP reserves absorb the difference. This is a direct loss of LP principal matching the allowed impact gate: **bad-price execution — stale bid/ask quote reaches a pool swap**.

## Likelihood Explanation

L2 sequencer outages are documented historical events on Arbitrum, Optimism, and Base. No privileged access is required — `swap` is public. The exploitation window equals the outage duration. For Chainlink feeds with heartbeat of 1 h or 24 h, operators will set `MAX_TIME_DELTA` to match the heartbeat, making any outage shorter than that window exploitable. The attack is executable by any EOA the moment the sequencer recovers.

## Recommendation

Add a sequencer uptime check at the top of `_getBidAndAskPrice()` in both `PriceProviderL2` and `ProtectedPriceProviderL2`, following the pattern already present in the registry's `ChainlinkVerifierL2`:

```solidity
AggregatorV3Interface public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600;

function _checkSequencer() internal view {
    if (address(sequencerUptimeFeed) == address(0)) return;
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    if (answer != 0 || block.timestamp - startedAt < GRACE_PERIOD)
        revert SequencerDown();
}
```

Call `_checkSequencer()` before the oracle read in `_getBidAndAskPrice()`. If the sequencer is down or within the grace period, revert with `FeedStalled` (or a dedicated `SequencerDown` error), blocking all swaps until prices are fresh.

## Proof of Concept

```
Setup:
  - Deploy PriceProviderL2 on Arbitrum fork with MAX_TIME_DELTA = 3600 (1 hour)
  - Pool: ETH/USDC, oracle price = $3000

Attack:
  T=0:    Sequencer goes down. Last oracle refTime = 0, price = $3000.
  T=0..D: Real ETH price drops to $2800 off-chain. No L2 txs possible.
  T=D:    Sequencer recovers (D = 1800s < MAX_TIME_DELTA = 3600s).
          _isStale(0, 1800, 3600, FUTURE_TOL) → 1800 ≤ 3600 → NOT stale.
          getBidAndAskPrice() returns bid/ask based on $3000.
  T=D+1:  Attacker calls pool.swap(zeroForOne=true) — buys ETH at $3000 ask.
          Real market price is $2800. Attacker sells on CEX for $2800.
          Pool LPs absorb $200/ETH loss.

Foundry fork test: fork Arbitrum at a block after a historical sequencer recovery,
deploy PriceProviderL2 with MAX_TIME_DELTA = 3600, warp block.timestamp to D,
call getBidAndAskPrice() — assert it returns non-stale price despite outage.
```
