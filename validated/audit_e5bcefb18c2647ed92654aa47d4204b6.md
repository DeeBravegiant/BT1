### Title
Malicious Harvester Can Unconditionally Redirect Farmer Block Reward via Unenforced `farmer_reward_address_override` — (File: `chia/farmer/farmer_api.py`)

### Summary

CHIP-22 introduced `farmer_reward_address_override` to allow harvesters to take a fee from the farmer's 1/8 block reward under a quality-based convention. However, the farmer node accepts any harvester-supplied override address without enforcing the fee quality threshold — the only response to a threshold violation is a log warning. A malicious harvester can redirect the entire farmer reward to an arbitrary address on every block it wins, with no enforcement mechanism to stop it.

### Finding Description

In `chia/protocols/harvester_protocol.py`, the `NewProofOfSpace` message carries an optional `farmer_reward_address_override: bytes32 | None` field that any connected harvester can set to any arbitrary puzzle hash. [1](#0-0) 

In `chia/farmer/farmer_api.py`, `_process_respond_signatures` unconditionally replaces the farmer's configured reward address with whatever the harvester supplied: [2](#0-1) 

The resulting `farmer_reward_address` is passed directly into `DeclareProofOfSpace`, which the full node uses to construct the block's `farmer_reward_puzzle_hash` in the foliage. There is no consensus-layer check on this field for non-genesis blocks. [3](#0-2) 

The only guard is `notify_farmer_reward_taken_by_harvester_as_fee`, called at line 128–129 of `farmer_api.py` when the override is present. This function checks the fee quality convention but **only emits log warnings** — it never rejects the override or prevents the block from being built with the attacker's address: [4](#0-3) 

A harvester that sets `fee_info=None` triggers the warning path at line 929–934 but the override is still honored. A harvester that sets `fee_info.applied_fee_threshold` above the actual fee quality also triggers a warning but the override is still honored. [5](#0-4) 

### Impact Explanation

The farmer's 1/8 block reward (e.g., 250,000,000,000 mojos per block at current halving) is paid to whatever puzzle hash appears in `foliage_block_data.farmer_reward_puzzle_hash`. A malicious harvester can set this to its own address on every block it finds proofs for, permanently diverting all farmer rewards to itself. This is an unauthorized XCH payout redirection affecting real on-chain coin creation.

### Likelihood Explanation

Any harvester operator can exploit this. Farmers routinely connect to third-party harvesters (remote harvesting is a standard Chia deployment pattern). The harvester only needs to set one field in the `NewProofOfSpace` or `RespondSignatures` message. No key material, admin access, or cryptographic break is required. The farmer has no way to detect or prevent this beyond reading log warnings.

### Recommendation

The farmer should enforce the CHIP-22 fee quality convention rather than merely logging violations. Specifically:

1. If `farmer_reward_address_override` is set but `fee_info` is `None`, **reject** the proof (do not build the block with the override).
2. If `fee_info` is present but `fee_quality > applied_fee_threshold`, **reject** the proof.
3. Optionally, cap the override to a farmer-configured maximum fee rate so the farmer retains control over how much of their reward a harvester may claim.

### Proof of Concept

1. Attacker operates a harvester and connects it to a victim farmer.
2. When the harvester finds a valid proof of space, it sends `NewProofOfSpace` with `farmer_reward_address_override = attacker_puzzle_hash` and `fee_info = None`.
3. `FarmerAPI.new_proof_of_space` calls `notify_farmer_reward_taken_by_harvester_as_fee`, which logs a warning but does not reject.
4. `_process_respond_signatures` sets `farmer_reward_address = attacker_puzzle_hash` at line 918.
5. `DeclareProofOfSpace` is sent to the full node with the attacker's address as `farmer_puzzle_hash`.
6. The full node builds the block; the farmer reward coin is created at the attacker's puzzle hash and is immediately spendable by the attacker. [6](#0-5) [7](#0-6) [8](#0-7)

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )
