Audit Report

## Title
Zero-Value `pricePercentageLimit` Silently Disables Both Price-Protection Guards in `_updateRsETHPrice` â€” (File: `contracts/LRTOracle.sol`)

## Summary
`LRTOracle._updateRsETHPrice()` gates both the upside-revert and the downside auto-pause on `pricePercentageLimit > 0`. Because `pricePercentageLimit` is zero by default (Solidity storage default) and `setPricePercentageLimit` accepts zero without restriction, both safety checks are permanently disabled from deployment until an admin explicitly sets a non-zero value. Any unprivileged caller can invoke the public `updateRSETHPrice()` and commit an arbitrarily large price drop without triggering the auto-pause of `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle`.

## Finding Description
In `_updateRsETHPrice()` (lines 256â€“257 and 273â€“274 of `contracts/LRTOracle.sol`):

```solidity
bool isPriceIncreaseOffLimit =
    pricePercentageLimit > 0 && priceDifference > pricePercentageLimit.mulWad(highestRsethPrice);
// ...
bool isPriceDecreaseOffLimit =
    pricePercentageLimit > 0 && diff > pricePercentageLimit.mulWad(highestRsethPrice);
```

When `pricePercentageLimit == 0`, both expressions short-circuit to `false` regardless of actual price movement. The upside branch (lines 260â€“266) that reverts for non-manager callers is never reached, and the downside branch (lines 277â€“282) that calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()` is never reached.

`pricePercentageLimit` is declared as a plain `uint256` storage variable (line 29), so it initializes to zero. `setPricePercentageLimit` (lines 125â€“128) accepts any value including zero with no lower-bound guard. The exploit path requires no privileged role: `updateRSETHPrice()` (line 87) is `public` with only a `whenNotPaused` modifier.

Concrete trigger sequence:
1. `pricePercentageLimit` is at its default of `0` (or reset to `0` by admin).
2. A slashing event or accounting error causes `_getTotalEthInProtocol()` to return a value significantly below `rsethSupply * rsETHPrice`.
3. Any EOA calls `updateRSETHPrice()`.
4. `isPriceDecreaseOffLimit` evaluates to `false`; the auto-pause block is skipped.
5. `rsETHPrice` is updated to the depressed value; `LRTDepositPool` and `LRTWithdrawalManager` remain unpaused.

## Impact Explanation
The auto-pause circuit-breaker is a documented safety guarantee of the protocol. When `pricePercentageLimit == 0`, this guarantee is entirely absent: deposits and withdrawals continue at the post-slash price with no halting mechanism. This maps to **Low â€” contract fails to deliver promised safety returns (auto-pause mechanism) without direct loss of principal**.

## Likelihood Explanation
The condition is present from the moment of deployment and persists until an admin explicitly calls `setPricePercentageLimit` with a non-zero value. `updateRSETHPrice()` is permissionless. Any window between deployment and configuration, or any future admin reset to zero, exposes the protocol. No special attacker capability is required beyond calling a public function.

## Recommendation
Replace the `> 0` sentinel with an explicit boolean disable flag, or treat `pricePercentageLimit == 0` as "always enforce" and introduce a separate `priceProtectionEnabled` flag. Additionally, add a lower-bound guard in `setPricePercentageLimit` to reject zero if the intent is to always enforce a non-zero tolerance:

```solidity
function setPricePercentageLimit(uint256 _pricePercentageLimit) external onlyLRTAdmin {
    if (_pricePercentageLimit == 0) revert InvalidPricePercentageLimit();
    pricePercentageLimit = _pricePercentageLimit;
    emit PricePercentageLimitUpdate(_pricePercentageLimit);
}
```

And in `initialize`, set a safe default (e.g., `pricePercentageLimit = 5e16` for 5%).

## Proof of Concept
Foundry fork test outline:
1. Deploy `LRTOracle` without calling `setPricePercentageLimit` (leave at default `0`).
2. Seed `rsethSupply > 0` and set `rsETHPrice` / `highestRsethPrice` to `1 ether`.
3. Manipulate the mock `NodeDelegator` / `LRTDepositPool` so `_getTotalEthInProtocol()` returns `0.5 ether * rsethSupply` (50% drop).
4. Call `updateRSETHPrice()` from an unprivileged EOA.
5. Assert: `rsETHPrice == 0.5 ether`, `lrtDepositPool.paused() == false`, `withdrawalManager.paused() == false`.
