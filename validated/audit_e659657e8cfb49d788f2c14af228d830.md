Audit Report

## Title
`protocolUnpausePool` Sets Admin-Paused (Level 1) Instead of Active (Level 0), Permanently Blocking Swaps — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary
`protocolUnpausePool` calls `setPause(1)` (admin-paused) instead of `setPause(0)` (active). After a protocol pause/unpause cycle on any active pool, `pauseLevel` is left at 1, causing every subsequent swap to revert with `PoolPaused`. The protocol owner cannot restore the pool to active state unilaterally; only the pool admin can, via `unpausePool`.

## Finding Description
The pause system has three levels: 0 = active, 1 = admin-paused, 2 = protocol-paused.

`protocolPausePool` accepts source levels 0 and 1, setting the pool to level 2:
```solidity
// L392-396
function protocolPausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 0 && cur != 1) revert InvalidPauseTransition(cur, 2);
    IMetricOmmPoolFactoryActions(pool).setPause(2);
}
```

`protocolUnpausePool` transitions from level 2 to level **1**, not level 0:
```solidity
// L399-403
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 1);
    IMetricOmmPoolFactoryActions(pool).setPause(1); // BUG: should be 0
}
```

`_checkNotPaused` rejects any non-zero level:
```solidity
// L643-644
function _checkNotPaused() internal view {
    if (pauseLevel != 0) revert PoolPaused();
}
```

The only path to level 0 is `unpausePool`, which is restricted to the pool admin:
```solidity
// L467-471
function unpausePool(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 1) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0);
}
```

A pool that was active (level 0) before a protocol pause is driven through `0 → 2 → 1` and left in admin-paused state it was never in before. The protocol owner has no further lever.

## Impact Explanation
After a protocol pause/unpause cycle on any live pool, all swaps revert with `PoolPaused` because `pauseLevel == 1 != 0`. LP fee accrual and swap volume are lost for the entire duration. If the pool admin key is lost or unresponsive, the pool is permanently stuck. This is broken core pool functionality causing unusable swap flows, matching the allowed impact gate.

## Likelihood Explanation
Triggered on every protocol pause/unpause cycle applied to a pool at level 0 — the normal operational state for any live pool. No special attacker setup is required; the protocol owner exercising their documented authority is sufficient. The condition is repeatable and deterministic.

## Recommendation
Change `protocolUnpausePool` to call `setPause(0)` and update the transition guard to target 0:
```solidity
function protocolUnpausePool(address pool) external override nonReentrant onlyOwner {
    (uint8 cur,,,,,) = PoolStateLibrary._slot0(pool);
    if (cur != 2) revert InvalidPauseTransition(cur, 0);
    IMetricOmmPoolFactoryActions(pool).setPause(0); // restore to active
}
```
Alternatively, store the pre-pause level in factory state when `protocolPausePool` is called and restore it on unpause.

## Proof of Concept
1. Pool deployed and active: `pauseLevel = 0`.
2. Protocol owner calls `protocolPausePool(pool)` — allowed (`cur == 0`). Pool: `pauseLevel = 2`.
3. Protocol owner calls `protocolUnpausePool(pool)` — sets `pauseLevel = 1`.
4. Any user calls `swap(...)` → reverts `PoolPaused` (`pauseLevel != 0`).
5. Protocol owner has no further function to call; only pool admin can call `unpausePool` (1 → 0).
6. If `poolAdmin[pool]` is unresponsive, pool is permanently paused despite the protocol owner having "unpaused" it.
