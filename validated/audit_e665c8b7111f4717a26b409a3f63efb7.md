### Title
Cross-Chain Replay via Untracked Completed Foreign-Transaction Verifications — (File: `crates/contract/src/lib.rs`)

### Summary

The `verify_foreign_transaction` endpoint never records which foreign-chain transactions have already been verified and signed. After a verification completes and its pending-request entry is removed, the identical foreign transaction can be re-submitted an unlimited number of times. Each re-submission causes MPC nodes to produce a fresh, cryptographically valid ECDSA signature over the same payload hash, giving the caller an unbounded supply of distinct valid attestations for a single on-chain event.

### Finding Description

`verify_foreign_transaction` stores each pending request in `pending_verify_foreign_tx_requests`, a `LookupMap<VerifyForeignTransactionRequest, Vec<YieldIndex>>`. [1](#0-0) 

When `respond_verify_foreign_tx` is called, `resolve_yields_for` drains and **removes** the entry from the map: [2](#0-1) 

After removal, no record of the completed verification is persisted anywhere in contract state. The contract has no "completed verifications" set analogous to the consumed-nonce or consumed-signId pattern. Consequently, the same `VerifyForeignTransactionRequest` value (same `ForeignChainRpcRequest`, `domain_id`, `payload_version`) can be re-submitted immediately after the first response is delivered. [3](#0-2) 

The `VerifyForeignTransactionRequest` key does not include the caller's account ID: [4](#0-3) 

So any caller (or the same caller) can re-submit the same foreign transaction after the first round completes. Each round causes MPC nodes to consume a presignature and produce a new ECDSA signature. Because ECDSA signing is randomized, each round yields a **distinct** valid signature `Sₙ` over the same deterministic `payload_hash H`: [5](#0-4) 

The `respond_verify_foreign_tx` function verifies the signature against the root public key but performs no check that this foreign transaction was not already verified in a prior round: [6](#0-5) 

### Impact Explanation

The `verify_foreign_transaction` feature is the trust anchor for bridge inbound flows (e.g., Omnibridge: foreign chain → NEAR). A bridge contract is expected to call `verify_foreign_transaction`, receive the MPC-signed attestation `(payload_hash, signature)`, and act on it (e.g., mint tokens, release funds). If the MPC contract guarantees that each foreign transaction produces exactly one attestation, the bridge contract only needs to track used `payload_hash` values. But because the MPC contract allows unlimited re-verification, an attacker obtains multiple distinct valid signatures `S1, S2, …, Sₙ` over the same `payload_hash H`. A bridge contract that checks only "is this signature valid over this payload?" — without also tracking used payload hashes — will accept each `Sₙ` as a fresh, independent attestation, enabling double-spend or unlimited minting from a single foreign-chain deposit. Even a bridge that does track `payload_hash` values must do so correctly and completely; the MPC contract's failure to enforce one-time verification shifts the entire replay-prevention burden onto every downstream consumer, violating the security model stated in the design documentation. [7](#0-6) 

**Impact class**: High — Cross-chain replay enabling invalid bridge execution or double-spend conditions.

### Likelihood Explanation

The attack requires no privileged access. Any NEAR account can call `verify_foreign_transaction` with 1 yoctonear deposit. After the first verification completes (pending entry removed), the attacker immediately re-submits the same request. The only cost is 1 yoctonear per round plus gas. The attack is fully reachable from an unprivileged external caller and requires no threshold collusion or key compromise.

### Recommendation

After `respond_verify_foreign_tx` successfully resolves yields, persist the `VerifyForeignTransactionRequest` (or a compact hash of it) in a durable "completed verifications" set in contract storage. At the start of `verify_foreign_transaction`, reject any request whose key is already present in that set with a clear error (e.g., `ForeignTransactionAlreadyVerified`). This mirrors the fix applied in the referenced `claimtoken.sol` report: consuming the identifier after first use to prevent re-submission.

### Proof of Concept

1. Attacker calls `verify_foreign_transaction({ request: Bitcoin(tx_id=[0xAA;32], ...), domain_id: 0, payload_version: V1 })` with 1 yoctonear. Contract stores the request in `pending_verify_foreign_tx_requests`.
2. MPC nodes verify the Bitcoin transaction, compute `payload_hash H`, produce signature `S1`, call `respond_verify_foreign_tx`. Contract calls `resolve_yields_for`, which removes the entry from `pending_verify_foreign_tx_requests` and resumes the attacker's yield with `(H, S1)`.
3. Attacker immediately calls `verify_foreign_transaction` again with the identical arguments. Because `pending_verify_foreign_tx_requests` no longer contains the key (it was removed in step 2), the contract accepts the request and queues a new yield.
4. MPC nodes process the request again, produce a new signature `S2 ≠ S1` over the same `H` (ECDSA is randomized). Attacker receives `(H, S2)`.
5. Attacker now holds two distinct valid MPC signatures `S1` and `S2` over the same foreign-transaction payload hash `H`. Steps 3–4 can be repeated to accumulate `S3, S4, …`.
6. Attacker submits each `(H, Sₙ)` to a bridge contract that checks only signature validity, triggering the bridge action (e.g., token mint) once per signature. [8](#0-7) [9](#0-8)

    pub fn verify_foreign_transaction(&mut self, request: VerifyForeignTransactionRequestArgs) {
        log!(
            "verify_foreign_transaction: predecessor={:?}, request={:?}",
            env::predecessor_account_id(),
            request
        );

        self.check_request_preconditions(
            request.domain_id,
            DomainPurpose::ForeignTx,
            Gas::from_tgas(self.config.sign_call_gas_attachment_requirement_tera_gas),
            MINIMUM_SIGN_REQUEST_DEPOSIT,
        );

        let requested_chain = request.request.chain();
        let supported_chains = self.get_supported_foreign_chains();
        if !supported_chains.contains(&requested_chain) {
            env::panic_str(
                &InvalidParameters::ForeignChainNotSupported {
                    requested: requested_chain,
                }
                .to_string(),
            );
        }

        let callback_gas = Gas::from_tgas(
            self.config
                .return_signature_and_clean_state_on_success_call_tera_gas,
        );

        let request = args_into_verify_foreign_tx_request(request);
        let callback_args = serde_json::to_vec(&(&request,)).unwrap();
        self.enqueue_yield_request(
            method_names::RETURN_VERIFY_FOREIGN_TX_AND_CLEAN_STATE_ON_SUCCESS,
            callback_args,
            callback_gas,
            move |this, id| this.add_verify_foreign_tx_request(request, id),
        );
    }

    pub fn respond_verify_foreign_tx(
        &mut self,
        request: VerifyForeignTransactionRequest,
        response: VerifyForeignTransactionResponse,
    ) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();

        log!(
            "respond_verify_foreign_tx: signer={}, request={:?}",
            &signer,
            &request
        );

        self.assert_caller_is_attested_participant_and_protocol_active();

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        let domain = request.domain_id;
        let public_key = self.public_key_extended(domain.0.into())?;

        let signature_is_valid = match (&response.signature, public_key) {
            (
                dtos::SignatureResponse::Secp256k1(signature_response),
                PublicKeyExtended::Secp256k1 { near_public_key },
            ) => {
                let secp_pk = dtos::Secp256k1PublicKey::try_from(&near_public_key)
                    .expect("Secp256k1 variant always has a secp256k1 key");

                let payload_hash: [u8; 32] = response.payload_hash.0;

                // Check the signature is correct against the root public key
                near_mpc_signature_verifier::verify_ecdsa_signature(
                    signature_response,
                    &payload_hash,
                    &secp_pk,
                )
                .is_ok()
            }
            (signature_response, public_key_requested) => {
                return Err(RespondError::SignatureSchemeMismatch {
                    mpc_scheme: Box::new(signature_response.clone()),
                    user_scheme: Box::new(public_key_requested),
                }
                .into());
            }
        };

        if !signature_is_valid {
            return Err(RespondError::InvalidSignature.into());
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_verify_foreign_tx_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

/// Resume every yield queued for `request` with `response_bytes`, draining the
/// fan-out map in one pass. Returns `Err(RequestNotFound)` if the map held no entry.
///
/// Resuming a yield that has already timed out is a no-op at the SDK level.
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}

fn build_signature_request(
    request: &VerifyForeignTxRequest,
    foreign_tx_payload: &dtos::ForeignTxSignPayload,
) -> anyhow::Result<SignatureRequest> {
    let payload_hash: [u8; ECDSA_PAYLOAD_SIZE_BYTES] =
        foreign_tx_payload.compute_msg_hash()?.into();
    let payload_bytes: BoundedVec<u8, ECDSA_PAYLOAD_SIZE_BYTES, ECDSA_PAYLOAD_SIZE_BYTES> =
        payload_hash.into();

    Ok(SignatureRequest {
        id: request.id,
        receipt_id: request.receipt_id,
        payload: Payload::Ecdsa(payload_bytes),
        tweak: Tweak::new([0u8; 32]),
        entropy: request.entropy,
        timestamp_nanosec: request.timestamp_nanosec,
        domain: request.domain_id,
    })
}
