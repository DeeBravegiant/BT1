### Title
Zero ERC1155 Address Accepted in `logMetadata1155` / `initTransfer1155`, Enabling Unbacked Token Minting on NEAR — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary

`logMetadata1155` and `initTransfer1155` in `OmniBridge.sol` accept `tokenAddress = address(0)` without any zero-address guard. Because calling a void interface function on `address(0)` in the EVM returns `(true, "")` without reverting, an unprivileged attacker can emit a legitimate `InitTransfer` event without depositing any real ERC1155 tokens, then redeem that event on NEAR to mint unbacked wrapped tokens.

### Finding Description

`logMetadata1155` is a fully public, permissionless function: [1](#0-0) 

It accepts any `tokenAddress`, including `address(0)`, and stores it directly into `multiTokens[deterministicToken].tokenAddress` with no zero-address check. Because the sentinel value for "slot unset" is `address(0)`, the slot is written as `{tokenAddress: address(0), tokenId: tokenId}` — permanently stuck in the "unset" state from the guard's perspective, but with `tokenId` populated.

`initTransfer1155` is equally permissionless: [2](#0-1) 

It calls `IERC1155(tokenAddress).safeTransferFrom(msg.sender, address(this), tokenId, amount, "")`. `IERC1155.safeTransferFrom` is a `void` function (no return value). In the EVM, a call to `address(0)` — which has no code — returns `(true, "")`. Solidity does not insert an `extcodesize` check for raw interface calls, so the call silently succeeds without transferring any tokens.

`deriveDeterministicAddress` produces a non-zero address from `address(0)` + `tokenId`: [3](#0-2) 

So `deterministicToken` is a valid-looking address, and the emitted `InitTransfer` event is indistinguishable from a legitimate one.

### Impact Explanation

An attacker can mint an arbitrary quantity of unbacked wrapped tokens on NEAR:

1. Call `logMetadata1155(address(0), tokenId)` → emits `LogMetadata(deterministicToken, "0x000...000", "", 0)` and (in `OmniBridgeWormhole`) publishes a Wormhole message.
2. Submit the Wormhole VAA to NEAR's `deploy_token` → NEAR deploys a wrapped token for `deterministicToken`.
3. Call `initTransfer1155(address(0), tokenId, N, 0, 0, "attacker.near", "")` → `IERC1155(address(0)).safeTransferFrom(...)` silently succeeds; `InitTransfer` event is emitted with `amount = N`.
4. Submit proof of the `InitTransfer` event to NEAR's `fin_transfer` → NEAR mints `N` wrapped tokens to `attacker.near`.

No real ERC1155 tokens are ever deposited. The attacker receives `N` unbacked wrapped tokens on NEAR, breaking the 1:1 backing guarantee.

This matches the **Critical** impact category: unauthorized creation of wrapped bridge assets through a verification failure.

### Likelihood Explanation

The attack is fully permissionless — no role, key, or colluding party is required. Both `logMetadata1155` and `initTransfer1155` are `external payable` with no access control. The only cost is the Wormhole message fee. The EVM behavior (call to `address(0)` returns success) is deterministic and well-known.

### Recommendation

Add a zero-address guard at the top of both functions:

```solidity
// In logMetadata1155 and initTransfer1155:
require(tokenAddress != address(0), "ERR_ZERO_TOKEN_ADDRESS");
```

Additionally, the `multiTokens` sentinel check (`if (multiToken.tokenAddress == address(0))`) should be replaced with an explicit boolean `isSet` flag so that a zero `tokenAddress` cannot be confused with an uninitialized slot.

### Proof of Concept

```solidity
// 1. Register zero ERC1155 address (permissionless)
bridge.logMetadata1155{value: wormholeFee}(address(0), 42);
// → emits LogMetadata(deterministicToken, "0x000...000", "", 0)
// → Wormhole VAA published

// 2. On NEAR: submit VAA to deploy_token → NEAR deploys wrapped token for deterministicToken

// 3. Mint unbacked tokens (permissionless)
bridge.initTransfer1155{value: wormholeFee}(
    address(0),   // tokenAddress — no code, call returns (true,"")
    42,           // tokenId
    1_000_000,    // amount — no real tokens transferred
    0,            // fee
    0,            // nativeFee
    "attacker.near",
    ""
);
// → IERC1155(address(0)).safeTransferFrom(...) silently succeeds
// → InitTransfer event emitted with amount = 1_000_000

// 4. On NEAR: submit InitTransfer proof to fin_transfer
// → NEAR mints 1_000_000 unbacked wrapped tokens to attacker.near
```

    function logMetadata1155(
        address tokenAddress,
        uint256 tokenId
    ) external payable {
        address deterministicToken = deriveDeterministicAddress(
            tokenAddress,
            tokenId
        );

        MultiTokenInfo storage multiToken = multiTokens[deterministicToken];

        if (multiToken.tokenAddress == address(0)) {
            multiToken.tokenAddress = tokenAddress;
            multiToken.tokenId = tokenId;
        } else {
            if (
                multiToken.tokenAddress != tokenAddress ||
                multiToken.tokenId != tokenId
            ) {
                revert ERC1155MappingMismatch();
            }
        }

        logMetadataExtension(
            deterministicToken,
            Strings.toHexString(tokenAddress),
            "",
            0
        );

        emit BridgeTypes.LogMetadata(
            deterministicToken,
            Strings.toHexString(tokenAddress),
            "",
            0
        );
    }

    function deriveDeterministicAddress(
        address tokenAddress,
        uint256 tokenId
    ) public pure returns (address) {
        return
            address(
                bytes20(keccak256(abi.encodePacked(tokenAddress, tokenId)))
            );
    }
