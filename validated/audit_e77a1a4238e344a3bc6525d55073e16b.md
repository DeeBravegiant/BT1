### Title
Missing State Diff Commitment Verification in P2P Sync Allows Malicious Peer to Substitute `compiled_class_hash` — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`parse_data_for_block` in the p2p sync client assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages and writes it directly to storage. The only integrity check performed is that the total entry count matches `state_diff_length` from the stored block header. The `state_diff_commitment` (a Poseidon hash stored in the same header that covers `class_hash_to_compiled_class_hash`) is **never computed or compared** against the assembled diff. A malicious peer can therefore substitute any `compiled_class_hash` value for any `class_hash` in a `DeclaredClass` chunk, and the node will silently persist the forged mapping.

---

### Finding Description

**Entrypoint:** `StateDiffStreamBuilder::parse_data_for_block`

The function reads `target_state_diff_len` from the stored header and loops until `current_state_diff_len == target_state_diff_len`, calling `unite_state_diffs` on each received chunk: [1](#0-0) 

Inside `unite_state_diffs`, a `DeclaredClass` chunk is inserted with no content validation: [2](#0-1) 

After the loop, the only additional check is for duplicate deprecated classes: [3](#0-2) 

There is no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment`. The assembled diff is then written unconditionally: [4](#0-3) 

**The commitment infrastructure exists but is not wired into the p2p sync path.** `calculate_state_diff_hash` covers `class_hash_to_compiled_class_hash`: [5](#0-4) 

The `apollo_committer` has a `verify_state_diff_hash` guard, but it is a separate component not invoked here: [6](#0-5) 

The central sync path even has an explicit TODO acknowledging the same missing check: [7](#0-6) 

---

### Impact Explanation

After the forged state diff is committed, `get_compiled_class_hash` reads `class_hash_to_compiled_class_hash` from the stored state diff and returns the attacker-controlled value: [8](#0-7) 

Any contract deployed with class `H` will subsequently have its CASM artifact resolved via the attacker's `compiled_class_hash`. This satisfies the Critical impact category: **wrong compiled class / CASM artifact selected for execution**.

---

### Likelihood Explanation

The attacker only needs to be an unauthenticated p2p peer. The length check (`state_diff_length` from the header) is trivially satisfied by sending exactly one `DeclaredClass` chunk for the target class hash. No cryptographic material is required. The peer is only reported (`report_peer()`) on a `BadPeerError`, which is not triggered by a well-formed but content-forged chunk.

---

### Recommendation

In `parse_data_for_block`, after the assembly loop, compute `calculate_state_diff_hash(&result)` and compare it against `header.state_diff_commitment`. Reject with `BadPeerError` on mismatch. This is the same pattern already implemented (but not used here) in `apollo_committer`: [9](#0-8) 

---

### Proof of Concept

1. Sync a block header for block N that declares class `H` with canonical `compiled_class_hash = C`. The header's `state_diff_commitment` covers `(H → C)`.
2. As the p2p peer responding to the state diff query, send a single `StateDiffChunk::DeclaredClass { class_hash: H, compiled_class_hash: ATTACKER_HASH }`. The length is 1, matching `state_diff_length = 1` in the header.
3. `parse_data_for_block` accepts the chunk (length matches, no duplicate), calls `write_to_storage` → `append_state_diff`.
4. Read back `get_state_diff(N).class_hash_to_compiled_class_hash[H]` — it equals `ATTACKER_HASH`, not `C`.
5. Any subsequent `get_compiled_class_hash(H)` call returns `ATTACKER_HASH`, causing execution to select the wrong CASM artifact for all contracts of class `H`.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
