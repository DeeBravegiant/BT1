Audit Report

## Title
Unbounded Nested Loops in `getAssetDistributionData` and `_getTotalEthInProtocol` Cause OOG, Temporarily Freezing Deposits and Price Updates - (File: contracts/LRTDepositPool.sol, contracts/LRTOracle.sol)

## Summary
`getAssetDistributionData()` and `getETHDistributionData()` loop over every NDC and call `INodeDelegator.getAssetUnstaking()` on each, which itself iterates over all queued EigenLayer withdrawals and their strategies via `DelegationManager.getQueuedWithdrawals()`. As queued withdrawals accumulate through normal operator `initiateUnstaking()` calls, the combined loop depth grows without bound, causing `depositETH()`/`depositAsset()` and the public `updateRSETHPrice()` to revert with out-of-gas, temporarily freezing user deposits and staling the rsETH price.

## Finding Description
**Confirmed call chain — deposit path:**

`depositETH()` / `depositAsset()` → `_beforeDeposit()` → `_checkIfDepositAmountExceedesCurrentLimit()` → `getTotalAssetDeposits()` → `getAssetDistributionData()`. [1](#0-0) 

Inside `getAssetDistributionData()`, a loop over `nodeDelegatorQueue` calls `getAssetUnstaking()` on every NDC: [2](#0-1) 

`getAssetUnstaking()` fetches all pending EigenLayer withdrawals live and iterates over them with a nested strategy loop: [3](#0-2) 

The `queuedWithdrawals` array returned by `DelegationManager.getQueuedWithdrawals()` is unbounded — it grows with every `initiateUnstaking()` call and shrinks only when `completeUnstaking()` is called.

**Confirmed call chain — price update path:**

`updateRSETHPrice()` is public with only `whenNotPaused`: [4](#0-3) 

It calls `_getTotalEthInProtocol()`, which loops over every supported asset and calls `getTotalAssetDeposits()` for each, triggering the full NDC × queued-withdrawal nested loop per asset: [5](#0-4) 

This produces a four-level loop: `supportedAssets.length × nodeDelegatorQueue.length × queuedWithdrawals.length × strategies.length`, with external calls at the innermost level (`strategy.underlyingToken()`, `strategy.sharesToUnderlyingView()`).

**Why existing guards are insufficient:**

`maxNodeDelegatorLimit` (initialized to 10) bounds the outer NDC loop but does nothing to bound the inner `queuedWithdrawals` loop, which is controlled entirely by EigenLayer state accumulated through routine operator operations. [6](#0-5) 

## Impact Explanation
**Medium — Unbounded gas consumption / Temporary freezing of funds.**

When the nested loop exceeds the block gas limit (~30M gas on mainnet), every call to `depositETH()` and `depositAsset()` reverts, preventing users from depositing. Simultaneously, `updateRSETHPrice()` reverts, leaving `rsETHPrice` stale and causing mispricing of new deposits and withdrawals. Both impacts are temporary (until queued withdrawals are completed) but can persist for extended periods during high unstaking activity.

## Likelihood Explanation
The precondition — accumulated queued withdrawals — arises from normal operator operations (`initiateUnstaking()` is a routine function). With `maxNodeDelegatorLimit = 10` NDCs and a realistic accumulation of 20–30 queued withdrawals per NDC (each with 1–2 strategies), the gas cost of the nested loop across 3 supported assets in `_getTotalEthInProtocol()` becomes prohibitive. No attacker action is required; the state degrades organically. Any unprivileged user calling `depositETH()` or `updateRSETHPrice()` triggers the revert once the threshold is crossed.

## Recommendation
1. **Paginate `getAssetDistributionData`**: Accept `from`/`to` index parameters for the NDC loop so callers can split the computation across multiple transactions.
2. **Cache `assetUnstaking` off-chain**: Replace the live `getQueuedWithdrawals()` scan with an operator-updated cached value stored on-chain; expose the live computation only in a dedicated view function not called from state-changing paths.
3. **Decouple price update from full TVL scan**: Store per-asset TVL snapshots updated incrementally rather than recomputing the full sum on every `updateRSETHPrice()` call.

## Proof of Concept
1. Deploy with 3 supported assets and 5 NDCs (`nodeDelegatorQueue.length = 5`).
2. Operator calls `initiateUnstaking()` repeatedly; each NDC accumulates 30 queued withdrawals with 2 strategies each.
3. Call `depositETH{value: 1 ether}(0, "")` from an unprivileged EOA.
4. Execution: `_beforeDeposit` → `getTotalAssetDeposits(ETH)` → `getETHDistributionData()` → 5 NDC iterations → each calls `getAssetUnstaking(ETH)` → fetches 30 withdrawals × 2 strategies = 60 inner iterations per NDC, each with external EigenLayer storage reads → 300 total inner iterations.
5. For `updateRSETHPrice()`: outer loop runs for 3 assets, each triggering the same 300-iteration inner loop → 900 total inner iterations plus 15 `getAssetBalance()` external calls.
6. Foundry fork test: set up the above state on a mainnet fork, call both functions, and assert revert with `OutOfGas` or measure gas consumption exceeding 30M.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }

        address lrtDepositPoolAddr = lrtConfig.getContract(LRTConstants.LRT_DEPOSIT_POOL);

            unchecked {
                ++assetIdx;
            }
        }
    }
