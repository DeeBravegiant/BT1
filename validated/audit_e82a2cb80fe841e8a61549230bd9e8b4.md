### Title
PVE Access-Structure `combine_ac` Skips Ciphertext Proof Verification, Enabling Key Substitution - (File: `src/cbmpc/api/pve_batch_ac.cpp`)

---

### Summary

The public API function `coinbase::api::pve::combine_ac` unconditionally calls `ec_pve_ac_t::aggregate_to_restore_row` with `skip_verify=true`, bypassing the cut-and-choose proof that binds the ciphertext to the correct public keys. An attacker who can substitute the ciphertext blob (e.g., a compromised backup store) can supply a crafted ciphertext that encodes a key they already know; the quorum members will honestly decrypt it, and the recovering party will accept the attacker's key as their own without any error.

---

### Finding Description

`combine_ac` is the public API entry point for the multi-party PVE recovery path. Its signature is:

```cpp
error_t combine_ac(const base_pke_i& base_pke, curve_id curve,
                   const access_structure_t& ac, mem_t ciphertext,
                   int attempt_index, mem_t label,
                   const leaf_shares_t& quorum_shares,
                   std::vector<buf_t>& out_xs);
``` [1](#0-0) 

Notably, it accepts no `expected_Qs` / public-key argument. Internally it calls:

```cpp
rv = pve_ct.aggregate_to_restore_row(bridge, ac_internal, attempt_index, label, quorum_bn, xs_bn,
                                     /*skip_verify=*/true, pk_ptrs);
``` [2](#0-1) 

Inside `aggregate_to_restore_row`, the `skip_verify` flag gates the entire cut-and-choose proof check:

```cpp
if (!skip_verify) {
    if (all_ac_pks.empty())
        return coinbase::error(E_BADARG, "all_ac_pks is required when skip_verify is false");
    if (rv = verify(base_pke, ac, all_ac_pks, Q, label)) return rv;
}
``` [3](#0-2) 

The `verify` function re-derives the challenge bits `b` from the full commitment transcript `(Q, label, c0, c1, quorum_c0, quorum_c1, X0, X1)` and checks them against the stored `b`, confirming the ciphertext was honestly formed for the correct public keys `Q`:

```cpp
buf128_t b_tag;
b_tag = crypto::ro::hash_string(Q, label, c0, c1, quorum_c0, quorum_c1, X0, X1).bitlen(SEC_P_COM);
if (b_tag != b) return coinbase::error(E_CRYPTO);
``` [4](#0-3) 

When `skip_verify=true`, this entire check is omitted. The only remaining integrity check is:

```cpp
if (x[j] * G != Q[j]) return coinbase::error(E_CRYPTO);
``` [5](#0-4) 

But `Q[j]` is read directly from the **unverified ciphertext blob** itself:

```cpp
void convert(coinbase::converter_t& converter) {
    converter.convert(Q, L, b);
    ...
}
``` [6](#0-5) 

So the check `x[j] * G == Q[j]` only verifies internal self-consistency of the attacker-supplied blob, not that `Q[j]` matches the legitimate key.

The same pattern exists in the single-recipient batch path:

```cpp
rv = pve_ct.decrypt(bridge, pve_keyref(dk_mem), pve_keyref(ek_mem), label, icurve,
                    xs_bn, /*skip_verify=*/true);
``` [7](#0-6) 

---

### Impact Explanation

The PVE scheme is used for key backup and recovery. The "publicly verifiable" property is precisely the guarantee that the ciphertext commits to the correct key material. Skipping `verify` in the combine step removes this guarantee entirely. A party recovering their key share from a tampered ciphertext will silently accept a wrong scalar as their own private key share, with no error returned. Any subsequent signing or derivation operation will produce output under the attacker's key rather than the legitimate one.

---

### Likelihood Explanation

The ciphertext blob is an opaque serialized buffer passed across a trust boundary (backup service, cloud storage, HSM escrow). The `combine_ac` API accepts it as untrusted input but performs no proof verification. The separate `verify_ac` function exists but is not called inside `combine_ac`, and the API signature gives no indication that the caller must invoke it first. Any integrator following the natural call sequence (`partial_decrypt_ac_attempt` → `combine_ac`) will skip verification entirely. [8](#0-7) 

---

### Recommendation

1. **Remove `skip_verify=true` from `combine_ac`**: require the caller to supply the expected public keys and pass them through to `aggregate_to_restore_row` with `skip_verify=false`.
2. **Alternatively**, perform the `verify` call inside `combine_ac` before calling `aggregate_to_restore_row`, using the `Q` values extracted from the ciphertext cross-checked against caller-supplied expected public keys.
3. **For `decrypt_batch`**: apply the same fix — remove `skip_verify=true` and require the caller to supply the encryption key for verification.

---

### Proof of Concept

**Setup**: Access structure `THRESHOLD(2-of-3)` over leaves `{p1, p2, p3}`. Legitimate key `x_real` with public key `Q_real = x_real * G`. Attacker knows key `x_evil` with `Q_evil = x_evil * G`.

**Step 1 — Attacker crafts a ciphertext**: The attacker calls `encrypt_ac` with `x_evil` and the same access structure and label, producing `ct_evil`. This is a valid, honestly-formed PVE ciphertext for `x_evil`.

**Step 2 — Attacker substitutes the ciphertext**: The attacker replaces the legitimate `ct_real` in backup storage with `ct_evil`.

**Step 3 — Quorum members partially decrypt**: Each quorum member calls `partial_decrypt_ac_attempt(curve, ac, ct_evil, attempt_index, leaf_name, dk, label, share)`. This function does not verify the ciphertext; it simply decrypts the per-leaf KEM ciphertext inside `ct_evil` and returns a share of the masking key `K_evil`. [9](#0-8) 

**Step 4 — Recovering party calls `combine_ac`**: The recovering party calls `combine_ac(curve, ac, ct_evil, attempt_index, label, quorum_shares, xs_out)`. Because `skip_verify=true`, no proof check is performed. The function reconstructs `K_evil`, decrypts the AES-GCM layer, and recovers `x_evil`. The check `x_evil * G == Q_evil` passes because `Q_evil` is embedded in `ct_evil`. [10](#0-9) 

**Result**: `xs_out` contains `x_evil` — the attacker's key — with `SUCCESS` returned. The recovering party installs `x_evil` as their key share. The attacker can now produce valid signatures for any message using `x_evil`, while the legitimate key `x_real` is permanently lost.

error_t combine_ac(const base_pke_i& base_pke, curve_id curve, const access_structure_t& ac, mem_t ciphertext,
                   int attempt_index, mem_t label, const leaf_shares_t& quorum_shares, std::vector<buf_t>& out_xs) {
  if (const error_t rv = coinbase::api::detail::validate_mem_arg_max_size(
          ciphertext, "ciphertext", coinbase::api::detail::MAX_CIPHERTEXT_BLOB_SIZE))
    return rv;
  if (const error_t rv = coinbase::api::detail::validate_mem_arg(label, "label")) return rv;
  error_t rv = UNINITIALIZED_ERROR;

  const auto icurve = to_internal_curve(curve);
  if (!icurve.valid()) return coinbase::error(E_BADARG, "unsupported curve");

  coinbase::crypto::ss::ac_owned_t ac_internal;
  std::set<std::string> leaf_names;
  if (rv = to_internal_ac_and_leaves(ac, icurve, ac_internal, leaf_names)) return rv;
  if (rv = validate_quorum_shares_subset(leaf_names, quorum_shares, "quorum_shares")) return rv;

  pve_ac_ciphertext_blob_v1_t blob;
  if (rv = parse_ac_ciphertext(ciphertext, blob)) return rv;

  coinbase::mpc::ec_pve_ac_t pve_ct;
  if (rv = coinbase::convert(pve_ct, blob.ct)) return rv;

  for (const auto& q : pve_ct.get_Q()) {
    if (q.get_curve() != icurve) return coinbase::error(E_BADARG, "ciphertext curve mismatch");
  }

  const int expected_share_size = icurve.order().get_bin_size();
  for (const auto& [name_view, share_bytes] : quorum_shares) {
    if (share_bytes.size != expected_share_size) return coinbase::error(E_BADARG, "quorum_shares: invalid share size");
  }

  std::map<std::string, coinbase::crypto::bn_t> quorum_bn;
  for (const auto& [name_view, share_bytes] : quorum_shares) {
    quorum_bn.emplace(std::string(name_view), coinbase::crypto::bn_t::from_bin(share_bytes));
  }

  detail::base_pke_bridge_t bridge(base_pke);
  coinbase::mpc::ec_pve_ac_t::pks_t pk_ptrs;

  std::vector<coinbase::crypto::bn_t> xs_bn;
  rv = pve_ct.aggregate_to_restore_row(bridge, ac_internal, attempt_index, label, quorum_bn, xs_bn,
                                       /*skip_verify=*/true, pk_ptrs);
  if (rv) {
    out_xs.clear();
    return rv;
  }

  std::vector<buf_t> out_local;
  out_local.resize(xs_bn.size());
  const int out_len = icurve.order().get_bin_size();
  for (size_t i = 0; i < xs_bn.size(); i++) out_local[i] = xs_bn[i].to_bin(out_len);
  out_xs = std::move(out_local);
  return SUCCESS;
