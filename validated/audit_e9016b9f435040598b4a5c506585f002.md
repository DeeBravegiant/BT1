Audit Report

## Title
Stale Inflated `rsETHPrice` Broadcast to Destination Chains After Price-Drop Early Return — (`contracts/cross-chain/RSETHMultiChainRateProvider.sol`)

## Summary
When `LRTOracle._updateRsETHPrice()` detects a price drop exceeding `pricePercentageLimit`, it pauses the protocol and returns early without writing the new lower price to `rsETHPrice`. Because `MultiChainRateProvider.updateRate()` is permissionless and `RSETHMultiChainRateProvider.getLatestRate()` reads `rsETHPrice` directly from storage with no pause check, any caller can immediately broadcast the stale, inflated pre-drop rate to all destination chains, causing users on those chains to receive fewer rsETH tokens than they are owed.

## Finding Description
**Root cause — early return skips the price write:**
In `LRTOracle._updateRsETHPrice()`, when `isPriceDecreaseOffLimit` is true, the function pauses the protocol and returns at line 281 without ever reaching the assignment `rsETHPrice = newRsETHPrice` at line 313. The storage variable retains the old, higher value. [1](#0-0) [2](#0-1) 

**No access control on `updateRate()`:**
`MultiChainRateProvider.updateRate()` is `external payable` with only a `nonReentrant` guard — any EOA or contract can call it at any time, including while `LRTOracle` is paused. [3](#0-2) 

**`getLatestRate()` reads stale storage with no pause check:**
`RSETHMultiChainRateProvider.getLatestRate()` calls `ILRTOracle(rsETHPriceOracle).rsETHPrice()`, which is a plain public state variable getter defined in `ILRTOracle`. It performs no check on whether `LRTOracle` is paused and returns the stale, inflated value. [4](#0-3) [5](#0-4) 

**Stale rate broadcast via LayerZero:**
`updateRate()` encodes the stale rate and sends it via LayerZero to every registered `RateReceiver` on destination chains. [6](#0-5) 

**Existing guards are insufficient:**
`updateRSETHPrice()` is protected by `whenNotPaused`, so re-triggering the oracle update is blocked after the pause. However, this guard does not extend to `updateRate()` in the cross-chain provider, leaving the broadcast path fully open. [7](#0-6) 

## Impact Explanation
Destination-chain pools receive an inflated rsETH/ETH rate. Users depositing ETH on those chains are quoted a higher price per rsETH than the actual current collateral value warrants, so they receive fewer rsETH tokens than they are owed. The contract does not revert and no funds are directly stolen, but it fails to deliver the promised exchange rate. This matches the allowed impact: **Low — Contract fails to deliver promised returns, but doesn't lose value.**

## Likelihood Explanation
- `updateRate()` is fully permissionless; any address can call it.
- The early-return/pause path is a normal operational event triggered by any sufficiently large price drop.
- No front-running, privileged access, or victim mistake is required.
- The stale rate can be broadcast repeatedly by anyone for the entire duration of the paused period, until an admin manually unpauses and calls `updateRSETHPriceAsManager()`.

## Recommendation
Add a pause check in `RSETHMultiChainRateProvider.getLatestRate()`:

```solidity
function getLatestRate() public view override returns (uint256) {
    ILRTOracle oracle = ILRTOracle(rsETHPriceOracle);
    require(!ILRTOracle_Pausable(rsETHPriceOracle).paused(), "Oracle paused");
    return oracle.rsETHPrice();
}
```

Alternatively, restrict `updateRate()` in `MultiChainRateProvider` to a privileged role (e.g., `onlyOwner`) so it cannot be called by arbitrary actors during a pause event.

## Proof of Concept
```
Setup:
  - LRTOracle.rsETHPrice        = 1.1e18
  - LRTOracle.highestRsethPrice = 1.1e18
  - LRTOracle.pricePercentageLimit = 5e16  (5%)
  - Underlying collateral drops → newRsETHPrice = 1.0e18
    (drop = 0.1e18 > 5% of 1.1e18 = 0.055e18 → isPriceDecreaseOffLimit = true)

Step 1: Anyone calls LRTOracle.updateRSETHPrice()
  → _updateRsETHPrice() hits early-return at line 281
  → LRTOracle.paused = true
  → rsETHPrice still = 1.1e18 (line 313 never reached)

Step 2: Anyone calls RSETHMultiChainRateProvider.updateRate() (with ETH for LZ fees)
  → getLatestRate() returns ILRTOracle.rsETHPrice() = 1.1e18 (stale)
  → LayerZero message sent to all destination chains with rate = 1.1e18

Assert:
  - Destination chain pools price rsETH at 1.1e18
  - True current value is 1.0e18
  - User depositing 1 ETH receives 1/1.1 ≈ 0.909 rsETH instead of 1.0 rsETH
  - Step 2 can be repeated by any address for the entire paused period
```

Foundry fork test plan: deploy `LRTOracle` + `RSETHMultiChainRateProvider` on a mainnet fork, manipulate underlying asset prices to trigger the `isPriceDecreaseOffLimit` path, call `updateRSETHPrice()` as an unprivileged address, assert `LRTOracle.paused == true` and `rsETHPrice == 1.1e18`, then call `updateRate()` as a separate unprivileged address and assert the emitted `RateUpdated` event carries the stale `1.1e18` value.
