### Title
Single Byzantine Participant Can Deliver Fabricated CKD Output for `AppPublicKey` Variant by Racing the Honest Network - (File: crates/contract/src/lib.rs)

### Summary
`respond_ckd` in `mpc-contract` performs a cryptographic pairing check (`ckd_output_check`) only for the `AppPublicKeyPV` variant of a CKD request. For the `AppPublicKey` variant, the match arm is empty ‚Äî no verification of the response is performed. A single Byzantine attested participant can observe a pending `AppPublicKey` CKD request on-chain, immediately call `respond_ckd` with an arbitrary `CKDResponse`, and have it accepted before the honest threshold network completes its protocol. The first accepted call drains the entire yield queue, so the honest network's subsequent response is rejected as `RequestNotFound`. The user receives a fabricated derived key.

### Finding Description

In `respond_ckd`, the response verification branch is:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ‚Üê no check
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
``` [1](#0-0) 

For `AppPublicKeyPV`, `ckd_output_check` verifies the pairing equation `e(big_c, G2) = e(big_y, app_pk2) ¬∑ e(H(pk, app_id), pk)`, which proves the response was produced using the MPC network's master secret key: [2](#0-1) 

For `AppPublicKey`, no equivalent check exists. The caller only needs to pass `assert_caller_is_attested_participant_and_protocol_active()`: [3](#0-2) 

Once `respond_ckd` returns `Ok`, `resolve_yields_for` immediately drains the entire fan-out queue with the fabricated bytes: [4](#0-3) 

The `AppPublicKey` variant intentionally omits the G2 component (`pk2`) that `ckd_output_check` requires, making on-chain verification impossible without protocol changes. However, the contract still allows a single participant to resolve the request with any payload, bypassing the threshold requirement entirely.

The `SignatureRequest` analog (`respond`) does not have this problem because ECDSA signatures are verifiable on-chain against the derived public key: [5](#0-4) 

The race condition is structurally asymmetric: the honest network must complete a multi-round threshold protocol before submitting; the Byzantine participant can submit immediately after seeing the request indexed on-chain, with no cryptographic work required.

### Impact Explanation

A single Byzantine attested participant (strictly below the signing threshold) can deliver an arbitrary `CKDResponse` `(big_y, big_c)` to any user who submitted `request_app_private_key` with the `AppPublicKey` variant. The user computes `sig = C + (‚àía) ¬∑ Y` from the fabricated values and derives a key that has no relationship to the MPC network's master secret. This is unauthorized confidential key derivation output without the required participant authorization ‚Äî matching the Critical impact tier.

### Likelihood Explanation

Every attested participant in the active set has the capability to execute this attack. The attacker:
1. Monitors the NEAR indexer for `CKDRequestFromChain` events with `AppPublicKey` variant
2. Constructs any `CKDResponse` (e.g., `big_y = G1`, `big_c = G1`)
3. Calls `respond_ckd` immediately ‚Äî no threshold protocol needed

The honest network must complete a multi-round BLS12-381 protocol before it can respond. The Byzantine participant wins the race with near certainty. No collusion above one participant is required.

### Recommendation

1. **Short-term:** Reject `AppPublicKey` CKD requests at the contract level until a response-integrity mechanism is in place, or restrict `respond_ckd` to a designated coordinator account whose identity is committed to in the request.
2. **Long-term:** Require callers to use `AppPublicKeyPV` (which supplies `pk2` enabling the on-chain pairing check), or introduce an off-chain threshold-agreement proof that the contract can verify before resolving the yield.

### Proof of Concept

1. Alice calls `request_app_private_key` with `CKDAppPublicKey::AppPublicKey(alice_g1_pk)` and pays the deposit. The request is queued in `pending_ckd_requests`.
2. Byzantine participant Eve (a single attested node) observes the indexed `CKDRequestFromChain` event.
3. Eve immediately calls `respond_ckd(ckd_request, CKDResponse { big_y: G1_GENERATOR, big_c: G1_GENERATOR })`.
4. The contract passes `assert_caller_is_attested_participant_and_protocol_active()`, enters the `AppPublicKey` match arm (empty body), and calls `resolve_yields_for` ‚Äî draining Alice's yield with the fabricated bytes.
5. Alice's promise resolves with `big_y = G1`, `big_c = G1`. She computes `sig = G1 + (‚àía) ¬∑ G1 = (1 ‚àí a) ¬∑ G1`, which is not `msk ¬∑ H(pk, app_id)`.
6. The honest network later calls `respond_ckd` with the correct response; the contract returns `Err(RequestNotFound)` because the queue was already drained. [6](#0-5) [7](#0-6)

    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }

/// Resume every yield queued for `request` with `response_bytes`, draining the
/// fan-out map in one pass. Returns `Err(RequestNotFound)` if the map held no entry.
///
/// Resuming a yield that has already timed out is a no-op at the SDK level.
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
