### Title
Unguarded `assert sub_slot is not None` in `__get_cc_sub_slot` Crashes Weight Proof Validation via Crafted Segment — (`chia/full_node/weight_proof.py`)

---

### Summary

`__get_cc_sub_slot` iterates `reversed(range(idx))` to find a preceding slot-end entry. When `idx == 0` the range is empty, the loop body never executes, `sub_slot` stays `None`, and `assert sub_slot is not None` raises an unhandled `AssertionError`. The guard in `__validate_pospace` that is supposed to protect this case only covers the exact triple `(first_in_sub_epoch=True AND sub_epoch_n == 0 AND idx == 0)`. Any segment with `sub_epoch_n > 0` and a challenge block at sub-slot index 0 bypasses the guard and crashes the validator.

---

def __get_cc_sub_slot(sub_slots: list[SubSlotData], idx: int, ses: SubEpochSummary | None) -> ChallengeChainSubSlot:

---

### Impact Explanation

An unprivileged remote peer sends a crafted `WeightProof` during sync. The `AssertionError` propagates unhandled through `_validate_sub_epoch_segments` → `validate_weight_proof_inner` → `validate_weight_proof`, crashing the sync task. Depending on how the full node handles the exception at the call site in `full_node.py`, this can leave the node in an inconsistent sync state or crash the process entirely — a chain halt for the targeted node.

---

### Likelihood Explanation

Weight proofs are accepted from any peer during sync with no prior authentication. The preliminary checks (`_validate_sub_epoch_summaries`, `validate_sub_epoch_sampling`) operate on attacker-controlled fields (sub-epoch data, recent chain data) and can be satisfied by crafting a self-consistent but fake weight proof. The crash is reached before any cryptographic VDF validation occurs. With only one segment in the target sub-epoch, `rng.choice(range(1)) == 0` always, guaranteeing the malicious segment is sampled.

---

### Recommendation

Replace both `assert` statements in `__get_cc_sub_slot` with explicit error-returning logic, and update the caller to handle `None`:

```python
def __get_cc_sub_slot(
    sub_slots: list[SubSlotData], idx: int, ses: SubEpochSummary | None
) -> ChallengeChainSubSlot | None:
    sub_slot: SubSlotData | None = None
    for i in reversed(range(idx)):
        sub_slot = sub_slots[i]
        if sub_slot.cc_slot_end_info is not None:
            break
    if sub_slot is None or sub_slot.cc_slot_end_info is None:
        log.error("malformed segment: no slot-end entry before challenge block at index %d", idx)
        return None
    ...
```
