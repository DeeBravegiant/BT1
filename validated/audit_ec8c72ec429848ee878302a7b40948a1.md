Audit Report

## Title
Unauthenticated `updateRSETHPrice()` Allows Any Caller to Trigger Protocol-Wide Pause — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is declared `public whenNotPaused` with no role check, allowing any external account to invoke it. When the computed rsETH price has fallen below `highestRsethPrice` by more than `pricePercentageLimit`, the internal `_updateRsETHPrice()` unconditionally pauses `LRTDepositPool`, `LRTWithdrawalManager`, and `LRTOracle` itself. An unprivileged attacker can exploit this to force a protocol-wide pause, temporarily freezing all user deposits and withdrawals.

## Finding Description
`updateRSETHPrice()` at line 87 carries only the `whenNotPaused` modifier — no role check:

```solidity
function updateRSETHPrice() public whenNotPaused {
    _updateRsETHPrice();
}
```

Inside `_updateRsETHPrice()`, the downside-protection branch at lines 270–282 executes unconditionally for any caller:

```solidity
if (isPriceDecreaseOffLimit) {
    if (!lrtDepositPool.paused()) lrtDepositPool.pause();
    if (!withdrawalManager.paused()) withdrawalManager.pause();
    _pause();
    return;
}
```

This is in direct contrast to the upside-protection branch (lines 260–266), which explicitly checks `msg.sender` for the `MANAGER` role before reverting:

```solidity
if (!IAccessControl(address(lrtConfig)).hasRole(LRTConstants.MANAGER, msg.sender)) {
    revert PriceAboveDailyThreshold();
}
```

No equivalent guard exists on the downside path. The privileged counterpart `updateRSETHPriceAsManager()` (line 94, `onlyLRTManager`) exists precisely so managers can update price in situations where the public function would pause — but the public function itself has no protection against triggering the pause. Since `LRTOracle` must hold `PAUSER_ROLE` on `LRTDepositPool` and `LRTWithdrawalManager` for the intended downside-protection mechanism to function at all, the oracle's calls to `lrtDepositPool.pause()` and `withdrawalManager.pause()` will succeed when invoked by any caller through the public path.

## Impact Explanation
When the attacker's call triggers the pause:
- `LRTDepositPool` is paused → no new deposits of ETH or LSTs.
- `LRTWithdrawalManager` is paused → no withdrawal completions or instant withdrawals.
- `LRTOracle` itself is paused → no further price updates via the public path.

All user funds in flight (pending withdrawals, queued deposits) are frozen until an admin with `onlyLRTAdmin` manually calls `unpause()` on each contract. This constitutes **temporary freezing of funds**, a Medium-severity impact within the allowed scope.

## Likelihood Explanation
The price condition (`newRsETHPrice < highestRsethPrice` by more than `pricePercentageLimit`) can arise from natural LST oracle price dips (stETH, ETHx, etc.) reducing `totalETHInProtocol`. An attacker monitoring on-chain oracle values can time the call to the exact block where the condition is satisfied. No special privileges, tokens, or capital are required — only a standard transaction submission. The attack is repeatable: after the admin unpauses, the attacker can trigger it again if the price condition recurs.

## Recommendation
Restrict `updateRSETHPrice()` to a permissioned role (e.g., `onlyLRTOperator` or `onlyLRTManager`), mirroring the pattern already used by `updateRSETHPriceAsManager()`. Alternatively, split the public path so it can only update price when no pause threshold is crossed, and reserve the pause-triggering logic exclusively for the manager path. The current design conflates "anyone can refresh the price" with "anyone can pause the protocol."

## Proof of Concept
1. Monitor on-chain LST oracle prices until the computed `newRsETHPrice` would fall below `highestRsethPrice * (1 - pricePercentageLimit)`.
2. Call `LRTOracle.updateRSETHPrice()` from any EOA (no role required).
3. `_updateRsETHPrice()` computes `newRsETHPrice`, enters the `isPriceDecreaseOffLimit` branch, and calls `lrtDepositPool.pause()`, `withdrawalManager.pause()`, and `_pause()`.
4. All deposits and withdrawals are frozen. The admin must manually call `unpause()` on each of the three contracts via `onlyLRTAdmin`-gated functions.

**Foundry fork test outline:**
```solidity
function testAnyCallerCanPauseProtocol() public {
    // Fork mainnet, set up oracle with pricePercentageLimit > 0
    // Manipulate LST oracle price downward past threshold
    vm.prank(attacker); // unprivileged EOA
    lrtOracle.updateRSETHPrice();
    assertTrue(lrtDepositPool.paused());
    assertTrue(withdrawalManager.paused());
    assertTrue(lrtOracle.paused());
}
```
