### Title
Unconstrained `fee_proposal_fri` During Bootstrap Window Poisons Future `fee_actual` Computation — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid`, the `fee_proposal_fri` bounds check is guarded by a conditional that fires only when `fee_actual` is `Some`. During the first `window_size` blocks (bootstrap / near-genesis), `fee_actual` is always `None`, so the bounds check is silently skipped. A malicious proposer selected during this window can set `fee_proposal_fri` to any value — including `u128::MAX` or `0` — and every honest validator will accept the proposal. The extreme value is then committed into the block and recorded in `fee_proposals_window`. Once the window fills, `compute_fee_actual` returns a poisoned median that permanently distorts the `l2_gas_price` for all subsequent blocks.

---

### Finding Description

**Root cause — conditional skip of fee bounds enforcement:**

In `is_proposal_init_valid`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
```

When `proposal_init_validation.fee_actual` is `None`, the entire `if let` body is skipped. `fee_actual` is `None` whenever `compute_fee_actual` returns `None`, which happens for every height below `window_size` (the bootstrap window). [1](#0-0) 

**Why `fee_actual` is `None` during bootstrap:**

`compute_fee_actual` performs a `checked_sub(window_size)` on the current height. If the height is below `window_size`, it returns `None` immediately. It also returns `None` if any entry in the window is missing or recorded as `None`. [2](#0-1) 

**The proposal still proceeds and the value is committed:**

Even though the bounds check is skipped, the proposal is accepted and `fee_proposal_fri` is bound into the block commitment hash via `proposal_commitment_from(partial_block_hash, fee_proposal_fri)` — `Poseidon(partial.0, fee_proposal_fri)` for V0_14_3+ blocks. [3](#0-2) 

**The poisoned value is recorded at decision time:**

In `finalize_decision`, `record_fee_proposal(height, init.fee_proposal_fri)` inserts the proposer-supplied value into `fee_proposals_window` unconditionally. [4](#0-3) 

**The poisoned window feeds future gas price computation:**

`calculate_next_l2_gas_price` calls `compute_fee_actual` and passes the result to `calculate_next_l2_gas_price_for_fin`, which uses `fee_actual` as the reference for the EIP-1559-style base-fee adjustment. A poisoned `fee_actual` (e.g., `u128::MAX`) forces all future `l2_gas_price` values toward the extreme. [5](#0-4) 

**Validators also skip the check:**

The validator path constructs `ProposalInitValidation` with `fee_actual: compute_fee_actual(...)` and passes it to `is_proposal_init_valid`. During bootstrap, `fee_actual` is `None` on the validator side too, so honest validators accept the extreme `fee_proposal_fri` without objection. [6](#0-5) 

**Exact parallel to the StRSR bug:**

| StRSR | Sequencer |
|---|---|
| `if (!main.pausedOrFrozen()) _payoutRewards()` | `if let (Some(fee_actual), Some(fee_proposal)) = ...` |
| Staking allowed while paused | Proposal accepted with any `fee_proposal_fri` during bootstrap |
| New staker steals yield from existing stakers | Malicious proposer poisons `fee_actual` for all future blocks |

---

### Impact Explanation

**Impact: Critical — Incorrect fee/gas accounting with economic impact.**

A malicious proposer during the bootstrap window (first `window_size` blocks) can set `fee_proposal_fri = u128::MAX`. After `window_size` blocks, `compute_fee_actual` returns a median dominated by this extreme value. `calculate_next_l2_gas_price_for_fin` then computes an extreme `l2_gas_price`, making all subsequent transactions prohibitively expensive (or, with `fee_proposal_fri = 0`, free). The `fee_proposal_bounds` check for future blocks enforces that each new proposal stays within a multiplicative margin of this poisoned `fee_actual`, so the distortion persists and compounds across hundreds of blocks until the window naturally rotates out the poisoned entries.

The corrupted value is `fee_actual` (the median stored in `fee_proposals_window`), which directly determines the `l2_gas_price` embedded in every subsequent `ProposalInit` and enforced by every validator.

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

The attack requires a committee member to be selected as proposer during the bootstrap window (the first `window_size` blocks of the network's life). `window_size` is a protocol constant (`fee_proposal_window_size` in `VersionedConstants`). The bootstrap window is a one-time event at genesis. A single malicious committee member who happens to be selected as proposer during any of these early blocks can execute the attack. Because honest validators do not enforce bounds during bootstrap, no 2/3 quorum can block the poisoned proposal.

---

### Recommendation

Add a fallback bounds check for the bootstrap period. When `fee_actual` is `None`, enforce that `fee_proposal_fri` lies within a reasonable absolute range anchored to the configured minimum gas price (e.g., `[min_gas_price / K, min_gas_price * K]` for some constant `K`). This prevents extreme values from entering `fee_proposals_window` even before the sliding window is fully populated.

Alternatively, mirror the StRSR mitigation: before the first block is proposed, pre-populate `fee_proposals_window` with the configured `min_gas_price` for all `window_size` bootstrap heights so that `fee_actual` is never `None` and bounds are always enforced.

---

### Proof of Concept

1. Network launches at genesis. `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< window_size`.
2. Malicious committee member is selected as proposer for block `N` where `N < window_size`.
3. Proposer constructs `ProposalInit` with `fee_proposal_fri = Some(GasPrice(u128::MAX))`.
4. `is_proposal_init_valid` is called on every validator. The check at line 398 evaluates `(None, Some(u128::MAX))` — the `if let` does not match, bounds check is skipped, `Ok(())` is returned.
5. All honest validators accept the proposal. Block `N` is committed. `record_fee_proposal(N, Some(u128::MAX))` is called.
6. Repeat for as many bootstrap blocks as the attacker controls.
7. At height `window_size`, `compute_fee_actual` returns `Some(median)` where the median is dominated by `u128::MAX` entries.
8. `calculate_next_l2_gas_price_for_fin` receives this poisoned `fee_actual` and computes an extreme `l2_gas_price`.
9. All future `ProposalInit` messages must carry `l2_gas_price_fri` near `u128::MAX` (enforced by the `l2_gas_price_fri` equality check at line 312–314 of `is_proposal_init_valid`), making every transaction on the network prohibitively expensive. [1](#0-0) [2](#0-1) [7](#0-6) [4](#0-3) [6](#0-5)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
