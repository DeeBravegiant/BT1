Audit Report

## Title
Nested Unbounded Loops in `updateRSETHPrice()` Enable Gas-Limit DoS of Price Oracle and Permanent Freezing of Unclaimed Yield — (File: contracts/LRTOracle.sol)

## Summary
`LRTOracle.updateRSETHPrice()` is a public, permissionless function whose gas cost scales as O(assets × NDCs × queued_withdrawals × strategies_per_withdrawal) due to a deeply nested call chain. Each inner iteration involves external calls, including redundant `getQueuedWithdrawals()` fetches once per NDC per asset. As the protocol scales through normal admin and operator actions, the cumulative gas cost can exceed the Ethereum block gas limit (~30M gas), permanently preventing price updates and freezing protocol fee minting.

## Finding Description
The call chain is confirmed in the codebase:

`updateRSETHPrice()` (public, `whenNotPaused` only) → `_updateRsETHPrice()` → `_getTotalEthInProtocol()` loops over every supported asset and calls `ILRTDepositPool.getTotalAssetDeposits(asset)` per asset.

`getTotalAssetDeposits` calls `getAssetDistributionData`, which loops over every entry in `nodeDelegatorQueue` and calls `INodeDelegator(nodeDelegatorQueue[i]).getAssetUnstaking(asset)` per NDC per asset.

`getAssetUnstaking` issues a full external call to `DelegationManager.getQueuedWithdrawals(address(this))` — loading all queued withdrawal data — and then iterates over every withdrawal and every strategy within each withdrawal, making additional external calls to `strategy.sharesToUnderlyingView()`.

The critical inefficiency: `getQueuedWithdrawals` is called once per NDC **per asset**, meaning the same data is fetched N×M times redundantly (N = supported assets, M = NDCs). With 5 assets and 10 NDCs, this is 50 redundant full external data loads before any inner loop work.

No gas cap or budget check exists anywhere in this call chain. The `maxNodeDelegatorLimit` starts at 10 but is admin-adjustable upward. The `maxUncompletedWithdrawalCount` is also admin-settable. Both grow through normal protocol operation.

## Impact Explanation
**Medium. Unbounded gas consumption** — the function has no gas cap and its cost grows multiplicatively with protocol scale. At sufficient scale, `updateRSETHPrice()` becomes permanently uncallable.

**Medium. Permanent freezing of unclaimed yield** — protocol fee minting (lines 299–308 of `LRTOracle.sol`) is gated inside `_updateRsETHPrice()`. If the function cannot execute due to gas exhaustion, no protocol fees are ever minted again, permanently freezing yield owed to the treasury.

The downside protection circuit breaker (lines 270–282) is also gated inside `_updateRsETHPrice()`, meaning a stale price disables the protocol's safety mechanism.

## Likelihood Explanation
`updateRSETHPrice()` is callable by any unprivileged external account — it has no role check, only `whenNotPaused`. The gas exhaustion condition arises through normal protocol scaling: admin adding NDCs (via `addNodeDelegatorContractToQueue`), admin adding supported assets, and operators initiating withdrawals via `initiateUnstaking`. None of these require malicious intent. With 5 assets × 10 NDCs × 10 queued withdrawals × 2 strategies = 1,000 inner loop iterations plus 50 redundant `getQueuedWithdrawals` external calls, the gas cost approaches or exceeds the block gas limit before accounting for storage reads and outer oracle calls. Increasing any parameter linearly increases gas with no on-chain ceiling.

## Recommendation
1. Cache `getQueuedWithdrawals` results per NDC and reuse them across all asset queries rather than calling it once per NDC per asset. A single pass over all NDCs fetching all withdrawal data once would reduce the external call count from N×M to M.
2. Introduce a dedicated aggregator view that computes total ETH in a single pass, avoiding redundant external calls.
3. Add an explicit gas budget check or enforce a hard cap on the number of NDCs and queued withdrawals iterated in a single `updateRSETHPrice()` call.
4. Consider computing TVL incrementally via per-asset snapshots updated lazily, separating the price update from the full TVL computation.

    function getAssetUnstaking(address asset) external view returns (uint256 amount) {
        (IDelegationManager.Withdrawal[] memory queuedWithdrawals, uint256[][] memory withdrawalShares) =
            _getDelegationManager().getQueuedWithdrawals(address(this));

        for (uint256 withdrawalIndex = 0; withdrawalIndex < queuedWithdrawals.length; withdrawalIndex++) {
            IDelegationManager.Withdrawal memory withdrawal = queuedWithdrawals[withdrawalIndex];

            for (uint256 strategyIndex = 0; strategyIndex < withdrawal.strategies.length; strategyIndex++) {
                IStrategy strategy = withdrawal.strategies[strategyIndex];

                address strategyAsset = address(strategy) == address(lrtConfig.beaconChainETHStrategy())
                    ? LRTConstants.ETH_TOKEN
                    : address(strategy.underlyingToken());

                if (strategyAsset != asset) continue;

                uint256 sharesToUnstake = withdrawalShares[withdrawalIndex][strategyIndex];
                amount += strategyAsset == LRTConstants.ETH_TOKEN
                    ? sharesToUnstake
                    : strategy.sharesToUnderlyingView(sharesToUnstake);
            }
        }
    }
