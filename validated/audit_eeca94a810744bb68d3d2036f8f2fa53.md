Audit Report

## Title
Missing Staleness Check in `ChainlinkPriceOracle.getAssetPrice` Enables Over-Redemption via `instantWithdrawal` — (File: `contracts/oracles/ChainlinkPriceOracle.sol`)

## Summary

`ChainlinkPriceOracle.getAssetPrice` discards the `updatedAt` return value from `latestRoundData`, performing no staleness check. `instantWithdrawal` uses this potentially stale price as the denominator in `getExpectedAssetAmount` with no price-bounds guard, unlike `unlockQueue` which enforces caller-supplied min/max bounds via `_validatePrices`. A stale, deflated asset price inflates `assetAmountUnlocked`, allowing an attacker to burn a small amount of rsETH and redeem a disproportionately large amount of yield-bearing assets from `LRTUnstakingVault`.

## Finding Description

**Root cause 1 — no staleness check in `ChainlinkPriceOracle.getAssetPrice`:**

At `contracts/oracles/ChainlinkPriceOracle.sol` line 52, only the `answer` field is consumed from `latestRoundData`; `updatedAt` and `answeredInRound` are silently dropped with no `block.timestamp - updatedAt > STALENESS_THRESHOLD` guard:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
``` [1](#0-0) 

**Root cause 2 — `instantWithdrawal` has no price-bounds guard:**

`unlockQueue` calls `_validatePrices` with caller-supplied `minimumAssetPrice`/`maximumAssetPrice` bounds before processing: [2](#0-1) 

`instantWithdrawal` calls `getExpectedAssetAmount` directly at line 228 with no equivalent validation: [3](#0-2) 

**The vulnerable formula:**

`getExpectedAssetAmount` computes:
```
underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset)
``` [4](#0-3) 

`lrtOracle.rsETHPrice()` is a **stored** value updated by `updateRSETHPrice()`. `lrtOracle.getAssetPrice(asset)` delegates to `ChainlinkPriceOracle.getAssetPrice` which reads **live** from Chainlink without any staleness check. A stale, deflated denominator inflates `underlyingToReceive`. [5](#0-4) 

**Existing cap is insufficient:**

`instantWithdrawal` checks that `assetAmountUnlocked` does not exceed `getAssetsAvailableForInstantWithdrawal`, which returns `vaultBalance - queuedWithdrawalsBuffer`. This caps the absolute drain per call but does not prevent the attacker from receiving more assets than the fair ETH value of the rsETH burned. [6](#0-5) 

## Impact Explanation

**High — Theft of unclaimed yield.**

The attacker burns rsETH worth `X` ETH at the fair price but redeems assets worth `X * (fairPrice / stalePrice)` ETH. The surplus is extracted directly from `LRTUnstakingVault`, draining yield-bearing assets (stETH, ETHx, etc.) that belong to other protocol participants awaiting withdrawal. The `instantWithdrawalFee` (0–10%) reduces but does not eliminate profit when price deviation exceeds the fee. [7](#0-6) 

## Likelihood Explanation

**Medium.** Chainlink LST/ETH feeds (stETH/ETH, ETHx/ETH) have 24-hour heartbeat intervals and 0.5–1% deviation thresholds. A feed can go stale without oracle operator compromise during network congestion or when price movement is insufficient to trigger a deviation update. An attacker monitoring on-chain `updatedAt` values can detect staleness and act within the same block. No privileged access is required — `instantWithdrawal` is callable by any rsETH holder. [8](#0-7) 

## Recommendation

1. **Add a staleness check in `ChainlinkPriceOracle.getAssetPrice`:**

```solidity
(uint80 roundId, int256 price,, uint256 updatedAt, uint80 answeredInRound)
    = priceFeed.latestRoundData();
require(answeredInRound >= roundId, "Stale round");
require(block.timestamp - updatedAt <= STALENESS_THRESHOLD, "Stale price");
require(price > 0, "Non-positive price");
``` [1](#0-0) 

2. **Add price-bounds validation to `instantWithdrawal`**, mirroring the `_validatePrices` pattern already used in `unlockQueue`. Accept caller-supplied or governance-configured min/max bounds for both `rsETHPrice` and `assetPrice` and revert if either is outside the window. [9](#0-8) 

## Proof of Concept

Foundry fork-test outline (mainnet fork):

```solidity
// 1. Deploy a MockAggregator returning a price 1% below real stETH/ETH rate
//    with updatedAt = block.timestamp - 25 hours (past the 24h heartbeat).
MockAggregator staleFeed = new MockAggregator(
    realPrice * 99 / 100,       // deflated by 1%
    block.timestamp - 25 hours
);

// 2. As LRTManager, point the stETH price feed to the mock.
chainlinkOracle.updatePriceFeedFor(stETH, address(staleFeed));

// 3. Attacker holds rsETHUnstaked worth fairValue ETH at the real price.
uint256 rsETHUnstaked = 1 ether;
uint256 fairAssetAmount = rsETHUnstaked * rsETHPrice / realAssetPrice;

// 4. Call instantWithdrawal — no staleness revert, no price-bounds revert.
withdrawalManager.instantWithdrawal(stETH, rsETHUnstaked, "");

// 5. Assert attacker received more than fair value.
uint256 received = stETH.balanceOf(attacker);
// received = rsETHUnstaked * rsETHPrice / (realPrice * 0.99)
//           = fairAssetAmount / 0.99 ≈ fairAssetAmount * 1.0101
assert(received > fairAssetAmount);
// Profit ≈ 1% of fairAssetAmount, minus instantWithdrawalFee.
// If fee = 0, any staleness is profitable.
```

The assertion passes on unmodified production code because `ChainlinkPriceOracle.getAssetPrice` never checks `updatedAt` and `instantWithdrawal` never validates the price against any bounds. [10](#0-9) [11](#0-10)
