### Title
Unconstrained `fee_proposal_fri` During Bootstrap Window Corrupts Fee Market and `next_l2_gas_price` — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

In `is_proposal_init_valid`, the `fee_proposal_fri` field of `ProposalInit` is required to be present for Starknet V0_14_3+ blocks, but its value is **only** validated against `fee_actual` bounds when `fee_actual` is `Some`. During the bootstrap window — the first `fee_proposal_window_size` blocks after V0_14_3 activation, when `fee_actual` is `None` — no bounds are enforced at all. A malicious proposer can set `fee_proposal_fri` to any arbitrary value (e.g., `u128::MAX`). Validators accept it, the value is committed into the block hash and stored in `fee_proposals_window`, and it then corrupts future `fee_actual` computations and `next_l2_gas_price` for all subsequent blocks.

---

### Finding Description

In `is_proposal_init_valid` in `validate_proposal.rs`, the bounds check on `fee_proposal_fri` is gated on `fee_actual` being `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(ValidateProposalError::InvalidProposalInit(...));
    }
}
``` [1](#0-0) 

When `fee_actual` is `None` (bootstrap window), the `if let` pattern simply does not match and the entire bounds check is silently skipped. The only check that still runs is the presence check (requiring `fee_proposal_fri` to be `Some` for V0_14_3+): [2](#0-1) 

The `fee_actual` field in `ProposalInitValidation` is populated by `compute_fee_actual`, which returns `None` whenever the sliding window does not yet have `window_size` complete entries: [3](#0-2) 

Both the `validate_proposal` path (current round) and the queued-proposal path (future round dequeued on `set_height_and_round`) construct `ProposalInitValidation` with `fee_actual: compute_fee_actual(...)`, so both paths are equally affected: [4](#0-3) [5](#0-4) 

The accepted `fee_proposal_fri` is then bound into the `ProposalCommitment` via `proposal_commitment_from`: [6](#0-5) 

It is also stored in `fee_proposals_window` (via `record_fee_proposal` called at `decision_reached`) and later used as the median input to `compute_fee_actual` for future blocks. That `fee_actual` is then passed as the `effective_min` floor to `calculate_next_l2_gas_price_for_fin`: [7](#0-6) 

If `fee_actual = u128::MAX`, then `effective_min = u128::MAX`, and `next_l2_gas_price` is set to `u128::MAX`, making every transaction in the following block fail with an insufficient-fee error.

---

### Impact Explanation

**Critical — Incorrect fee/gas accounting with economic impact.**

A malicious proposer who controls more than `window_size / 2` consecutive proposal slots during the bootstrap window can set `fee_proposal_fri = u128::MAX` for each slot. Validators accept every such proposal (no bounds check). Once the window fills, `fee_actual = median = u128::MAX`, `next_l2_gas_price = u128::MAX`, and all user transactions in the immediately following block are rejected for insufficient fee. The corrupted value persists in the window for `window_size` more blocks, causing a sustained denial-of-service on the fee market.

The corrupted `fee_proposal_fri` is also committed into the `ProposalCommitment` hash (`Poseidon(partial_block_hash, fee_proposal_fri)`), so the wrong value is permanently anchored in the chain's commitment history.

---

### Likelihood Explanation

**Medium.** The bootstrap window is a one-time event at V0_14_3 activation. In a Tendermint-style rotation, a single validator is the proposer for roughly `1/N` of slots. Controlling `window_size/2 + 1` consecutive slots requires either significant stake, collusion among validators, or a network with few validators. However, the window is finite and predictable, making it a high-value target for a coordinated attack at a known upgrade boundary.

---

### Recommendation

During the bootstrap window, validate `fee_proposal_fri` against `l2_gas_price` (the proposer's own fallback value) with the same multiplicative margin used in normal operation. Concretely, in `is_proposal_init_valid`, when `fee_actual` is `None` but `fee_proposal_fri` is `Some`, check that `fee_proposal_fri` is within `fee_proposal_margin_ppt` of `proposal_init_validation.l2_gas_price_fri` (the validator's own current L2 gas price). This mirrors the existing `within_margin` pattern already used for L1 gas prices and prevents arbitrary values from entering the window.

---

### Proof of Concept

1. V0_14_3 activates at block `N`. `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< N + window_size`.
2. Malicious proposer controls proposal slots `N` through `N + window_size/2`. For each slot, it sets `fee_proposal_fri = u128::MAX` in `ProposalInit`.
3. `is_proposal_init_valid` runs. The presence check passes (`Some(_)` with `fee_proposal_required = true`). The bounds check is skipped because `proposal_init_validation.fee_actual = None`. Proposal is accepted.
4. After `decision_reached`, `fee_proposals_window[N..N+window_size/2+1]` all contain `Some(u128::MAX)`.
5. At block `N + window_size`, `compute_fee_actual` returns `median = u128::MAX`.
6. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX`; `next_l2_gas_price = u128::MAX`.
7. Block `N + window_size + 1`'s gas price is `u128::MAX`. Every transaction submitted with any finite `max_price_per_unit` is rejected. The block is empty. The corrupted value persists for `window_size` more blocks.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
