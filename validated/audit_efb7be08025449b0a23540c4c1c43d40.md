Audit Report

## Title
Missing Zero-Amount Guard After `rsETHAmount` Computation in `deposit()` Allows ETH Acceptance With Zero wrsETH Minted — (`contracts/pools/RSETHPoolV2NBA.sol`)

## Summary

`RSETHPoolV2NBA.deposit()` checks that `msg.value > 0` but does not check that the computed `rsETHAmount > 0` before calling `wrsETH.mint()`. Because `viewSwapRsETHAmountAndFee` uses integer division, any deposit where `amountAfterFee * 1e18 < rsETHToETHrate` truncates to `rsETHAmount = 0`. OpenZeppelin's `_mint` does not revert on a zero amount, so the transaction succeeds: the depositor's ETH is permanently accepted by the contract and 0 wrsETH is returned, with no user-facing recovery path.

## Finding Description

In `RSETHPoolV2NBA.viewSwapRsETHAmountAndFee`:

```solidity
rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
``` [1](#0-0) 

This is integer division. `rsETHToETHrate` is the rsETH/ETH exchange rate, which starts at `1e18` and grows monotonically as yield accrues. Once any yield has accrued, `rsETHToETHrate > 1e18`. At that point, a deposit of 1 wei with `feeBps = 0` produces:

```
amountAfterFee = 1
rsETHAmount    = 1 * 1e18 / rsETHToETHrate = 0   (integer truncation)
```

`deposit()` then proceeds unconditionally to `wrsETH.mint(msg.sender, 0)`: [2](#0-1) 

`RsETHTokenWrapper.mint()` delegates directly to OZ `_mint`, which does not revert on a zero amount: [3](#0-2) 

The call succeeds. `feeEarnedInETH` is unchanged (fee = 0 on a 1-wei deposit), and the ETH sits in `address(this).balance - feeEarnedInETH`. The depositor receives 0 wrsETH and has no function to reclaim their ETH.

The sibling contracts `RSETHPoolV2ExternalBridge`, `RSETHPoolV3`, `RSETHPoolV3ExternalBridge`, and `RSETHPoolV3WithNativeChainBridge` all contain `if (rsETHAmount == 0) revert InvalidAmount()` after computing the swap amount. `RSETHPoolV2NBA` is missing this guard.

## Impact Explanation

**Medium — Temporary freezing of funds.**

A depositor who sends any amount where `amountAfterFee * 1e18 < rsETHToETHrate` will have their ETH permanently accepted by the contract and receive 0 wrsETH. The depositor has no user-facing function to recover the ETH. The ETH is not permanently lost to the protocol — `BRIDGER_ROLE` can drain it via `moveAssetsForBridging()` — but the depositor has no recovery path, constituting temporary freezing of user funds. [4](#0-3) 

## Likelihood Explanation

The trigger condition (`rsETHToETHrate > 1e18`) is the normal operating state of rsETH after any yield accrual. No oracle manipulation, admin compromise, or special permissions are required. Any unprivileged external user who sends a dust deposit (e.g., 1 wei) — whether by accident, a buggy frontend, or deliberate griefing — will hit this path. The condition is always satisfied in production.

## Recommendation

Add a zero-amount check after computing `rsETHAmount` in `deposit()`, consistent with the pattern already used in `RSETHPoolV2ExternalBridge` and the V3 contracts:

```solidity
(uint256 rsETHAmount, uint256 fee) = viewSwapRsETHAmountAndFee(amount);

if (rsETHAmount == 0) revert InvalidAmount();   // guard against rounding to zero
```

## Proof of Concept

```solidity
// Preconditions:
//   rsETHToETHrate = 1.05e18  (5% yield accrued — normal production state)
//   feeBps = 0
//   Pool is unpaused, wrsETH minter role granted to pool

function testZeroMintOnDustDeposit() public {
    oracle.setRate(1.05e18);

    uint256 balanceBefore = wrsETH.balanceOf(alice);
    uint256 poolEthBefore = address(pool).balance;

    vm.prank(alice);
    pool.deposit{value: 1}("ref");   // 1 wei — succeeds with no revert

    // Invariant violated:
    assertEq(wrsETH.balanceOf(alice), balanceBefore);    // 0 wrsETH minted
    assertEq(address(pool).balance, poolEthBefore + 1);  // ETH accepted
    // Alice has no function to reclaim her 1 wei
}
```

    function viewSwapRsETHAmountAndFee(uint256 amount) public view returns (uint256 rsETHAmount, uint256 fee) {
        fee = amount * feeBps / 10_000;
        uint256 amountAfterFee = amount - fee;

        // rate of rsETH in ETH
        uint256 rsETHToETHrate = getRate();

        // Calculate the final rsETH amount
        rsETHAmount = amountAfterFee * 1e18 / rsETHToETHrate;
    }
