### Title
Single Participant Can Reset All Accumulated Key Event Votes via `vote_abort` — (File: `crates/contract/src/state/key_event.rs`)

### Summary
Any single participant who has not yet cast a success vote can call `vote_abort` and immediately nullify the entire active key-event instance, discarding all accumulated success votes from every other participant. This is the direct analog of the VaderBond vesting-period reset: a single action by one party wipes accumulated progress for all parties, forcing the entire round to restart.

### Finding Description
`KeyEvent::vote_abort` sets `self.instance = None` unconditionally after a single participant invokes it:

```rust
pub fn vote_abort(&mut self, key_event_id: KeyEventId) -> Result<(), Error> {
    let candidate = self.verify_vote(&key_event_id)?;
    if self
        .instance
        .as_ref()
        .unwrap()
        .completed
        .contains(&candidate)
    {
        return Err(VoteError::VoteAlreadySubmitted.into());
    }
    self.instance = None;   // ← entire instance wiped by one participant
    Ok(())
}
