Audit Report

## Title
Donation to EigenLayer Strategy Inflates `sharesToUnderlyingView`, Triggering Illegitimate Protocol-Fee rsETH Minting — (`contracts/NodeDelegatorHelper.sol`, `contracts/LRTOracle.sol`)

## Summary

Any unprivileged actor can transfer LST tokens directly to an EigenLayer `StrategyBase` contract, inflating `sharesToUnderlyingView` without issuing new shares. When `updateRSETHPrice()` is subsequently called (a public, permissionless function), the protocol interprets the inflated TVL delta as genuine staking yield, computes a `protocolFeeInETH` on it, and mints excess rsETH to the treasury. Existing rsETH holders bear the dilution: their proportional claim on protocol TVL is reduced by exactly the fee amount, constituting theft of unclaimed yield.

## Finding Description

**Root cause:** `NodeDelegatorHelper.getAssetBalance` converts the NDC's withdrawable share count to an underlying token amount via `IStrategy(strategy).sharesToUnderlyingView(withdrawableShare)`. EigenLayer's canonical `StrategyBase` implements this as:

```
sharesToUnderlyingView(shares) = shares * underlyingToken.balanceOf(address(this)) / totalShares
```

A direct `token.transfer(strategyAddress, amount)` increases `balanceOf(strategy)` without touching `totalShares`, inflating the per-share value proportionally.

**Exploit flow:**

1. Attacker calls `token.transfer(strategyAddress, donationAmount)`. No EigenLayer deposit occurs; `totalShares` is unchanged.

2. `NodeDelegatorHelper.getAssetBalance` (L31–39) fetches the NDC's unchanged withdrawable share count via `DelegationManager.getWithdrawableShares`, then calls `sharesToUnderlyingView` — which now returns a larger value due to the inflated `balanceOf`.

3. `LRTDepositPool.getAssetDistributionData` (L447–451) sums `getAssetBalance` across all NDCs, producing an inflated `assetStakedInEigenLayer`.

4. `LRTOracle._getTotalEthInProtocol` (L331–349) multiplies the inflated asset amount by the asset price oracle, producing an inflated `totalETHInProtocol`.

5. `LRTOracle._updateRsETHPrice` (L243–247) computes `rewardAmount = totalETHInProtocol - previousTVL` and `protocolFeeInETH = rewardAmount * protocolFeeInBPS / 10_000` on the artificial delta.

6. `LRTOracle._updateRsETHPrice` (L299–308) mints `protocolFeeInETH / newRsETHPrice` rsETH to the treasury.

7. `updateRSETHPrice()` (L87–89) is `public whenNotPaused` — callable by anyone.

**Why existing guards are insufficient:**

- `pricePercentageLimit` (L256–257): Only blocks calls where the price increase exceeds the configured threshold. If `pricePercentageLimit == 0` (default, unset), there is no check at all. Even when set, the attacker calibrates the donation to stay just below the threshold and repeats daily.

- `maxFeeMintAmountPerDay` (L205–206): When set to a non-zero value, the attacker calibrates the donation to stay under the cap. The cap resets every 24 hours, making the attack repeatable indefinitely. (Note: if `maxFeeMintAmountPerDay == 0`, the check `0 + feeAmount > 0` reverts for any non-zero fee, so the attack requires this value to be set to a positive amount — which is the intended operational state.)

## Impact Explanation

**High — Theft of unclaimed yield.** The donated tokens remain in the strategy (TVL is genuinely higher), but the protocol incorrectly treats the donation as staking yield and extracts a protocol fee from it. The fee is minted as rsETH to the treasury, diluting existing rsETH holders' proportional claim on TVL by exactly the fee amount. The yield that should accrue entirely to rsETH holders is partially diverted to the treasury. A motivated actor who controls or benefits from the treasury (or holds a short position on rsETH) has direct economic incentive to execute this attack repeatedly.

## Likelihood Explanation

**High.** The attack requires no role, signature, or governance action — only the ability to transfer tokens and call a public function. The attacker sacrifices the donated tokens but profits if they benefit from the treasury. The attack is repeatable every 24 hours (daily cap reset) and can be calibrated to stay below both the `pricePercentageLimit` threshold and the `maxFeeMintAmountPerDay` cap. The only precondition is that `maxFeeMintAmountPerDay > 0`, which is the intended operational configuration.

## Recommendation

1. **Snapshot-based accounting:** Track the NDC's deposited share count and the strategy's `totalShares`/`totalUnderlying` at deposit time. Compute yield only from changes in the strategy's internal accounting, not from `balanceOf`.

2. **Use deposit-share accounting instead of `sharesToUnderlyingView`:** Derive the underlying value from the NDC's recorded deposit shares and the strategy's `totalShares` at a trusted checkpoint, ignoring any balance increase not attributable to the strategy's own deposit logic.

3. **Enforce a minimum time between price updates:** Require a minimum elapsed time (e.g., 1 hour) between successive `updateRSETHPrice` calls to limit attack frequency.

4. **Require `pricePercentageLimit > 0`:** Enforce a non-zero limit in the initializer/reinitializer so the threshold guard is always active.

## Proof of Concept

**Foundry fork test plan (mainnet fork):**

```solidity
// 1. Fork mainnet at a block where NDC has stETH shares in EigenLayer stETH strategy
// 2. Record: treasuryRsETHBefore, rsETHPriceBefore, totalSupplyBefore
// 3. Transfer donationAmount of stETH directly to the EigenLayer stETH strategy address
//    (no StrategyManager.depositIntoStrategy call — totalShares unchanged)
// 4. Call LRTOracle.updateRSETHPrice() from an unprivileged EOA
// 5. Assert:
//    - rsETH.balanceOf(treasury) > treasuryRsETHBefore
//    - The minted fee amount ≈ donationAmount * assetPrice * protocolFeeInBPS / 10_000 / newRsETHPrice
//    - rsETH.totalSupply() increased by the fee amount
//    - rsETHPrice increased by less than it would have without the fee (holders diluted)
// 6. Repeat step 3–5 the next day to demonstrate repeatability
```

The test passes on an unmodified fork: `treasuryAfter > treasuryBefore` with `feeRseth` proportional to `donationAmount * protocolFeeInBPS / 10_000`, proving excess protocol-fee rsETH is minted against a donation rather than genuine staking yield.
