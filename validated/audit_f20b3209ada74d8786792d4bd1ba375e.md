Audit Report

## Title
Stale `rsETHPrice` Enables Inflated Asset Payouts in `instantWithdrawal()` — (File: contracts/LRTWithdrawalManager.sol)

## Summary
`instantWithdrawal()` computes the asset payout using `lrtOracle.rsETHPrice()`, a stored value that is not updated atomically before the withdrawal executes. When an underlying asset's oracle price drops, `getAssetPrice(asset)` reflects the new lower price immediately while `rsETHPrice` remains at the old higher value, causing the formula to yield more assets than the rsETH burned is actually worth. Unlike the queued withdrawal path, no `min()` cap is applied, so the full inflated amount is redeemed from `LRTUnstakingVault` and transferred to the attacker.

## Finding Description
`instantWithdrawal()` calls `getExpectedAssetAmount(asset, rsETHUnstaked)` at line 228 to determine the payout: [1](#0-0) 

`getExpectedAssetAmount()` computes the payout as: [2](#0-1) 

`lrtOracle.rsETHPrice()` is a stored state variable updated only when `_updateRsETHPrice()` is explicitly called: [3](#0-2) [4](#0-3) 

`updateRSETHPrice()` is permissionless but is never called atomically inside `instantWithdrawal()`: [5](#0-4) 

When an underlying asset price drops (e.g., stETH/ETH decreases), `lrtOracle.getAssetPrice(asset)` reflects the new lower price immediately, but `rsETHPrice` remains at the old higher value. The formula then yields `rsETHUnstaked × staleHighRsETHPrice / newLowAssetPrice`, which is larger than the true redemption value.

The queued withdrawal path applies a protective `min(expectedAssetAmount, currentReturn)` cap at unlock time: [6](#0-5) 

`instantWithdrawal()` has no such cap — it redeems exactly `assetAmountUnlocked` from the vault and transfers it to the user: [7](#0-6) 

The downside protection in `_updateRsETHPrice()` only pauses the protocol when the price decrease exceeds `pricePercentageLimit`: [8](#0-7) 

For small price drops within the limit, the protocol is not paused, and the staleness window remains exploitable.

## Impact Explanation
**High — Theft of unclaimed yield / direct asset theft from `LRTUnstakingVault`.** The attacker burns rsETH at a stale (inflated) price and receives more underlying assets than their rsETH is worth. The vault is drained by the excess amount on every call. This matches the allowed impact: **High — Theft of unclaimed yield**.

## Likelihood Explanation
**Medium.** The required conditions are: (1) a price movement in an underlying asset (routine — stETH, ETHx fluctuate continuously); (2) a staleness window before `updateRSETHPrice()` is called (always exists since it is not called on every block); (3) `isInstantWithdrawalEnabled[asset]` being `true` and sufficient vault balance. No privileged access, oracle compromise, or governance capture is required. The attack is repeatable across multiple assets and calls until instant-withdrawal liquidity is exhausted.

## Recommendation
Call `_updateRsETHPrice()` (or an equivalent internal update) atomically at the start of `instantWithdrawal()` before computing `assetAmountUnlocked`. Alternatively, apply the same `min(expectedAmount, currentReturn)` cap used in `_calculatePayoutAmount()` to the instant withdrawal path, or compute the payout directly from live TVL rather than the cached `rsETHPrice`.

## Proof of Concept
1. **T0**: stETH oracle price = 1.05 ETH, `rsETHPrice` = 1.05 ETH (recently updated; assume 1:1 rsETH:stETH for simplicity).
2. **T1**: stETH oracle price drops to 1.00 ETH (live oracle updates immediately). `rsETHPrice` remains 1.05 ETH (stale). The drop is within `pricePercentageLimit`, so no pause is triggered.
3. Attacker calls `instantWithdrawal(stETH, 100e18, "")` before `updateRSETHPrice()` is called.
4. `assetAmountUnlocked = 100e18 × 1.05e18 / 1.00e18 = 105e18 stETH`
5. True redemption value: `100e18 × 1.00e18 / 1.00e18 = 100e18 stETH`
6. Attacker receives **5 stETH more than deserved**, draining the vault by that excess.
7. Attacker repeats across multiple assets and calls until the vault's instant-withdrawal liquidity is exhausted.

**Foundry fork test plan**: Fork mainnet, set `rsETHPrice` to a stale high value via `vm.store`, lower the stETH price oracle return value, call `instantWithdrawal()` as an unprivileged address, and assert that the received amount exceeds `rsETHUnstaked × liveAssetPrice / liveRsETHPrice`.
