### Title
Unvalidated `ProposalInit.builder` Field Allows Proposer to Commit Arbitrary Sequencer Address into Block Hash and Fee Collection — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_proposal_init_valid` validates every security-sensitive field of `ProposalInit` except `builder`. The `builder` field is the sequencer address that flows directly into `BlockInfo.sequencer_address`, then into `PartialBlockHashComponents.sequencer`, and finally into `calculate_block_hash`. A Byzantine proposer can set `builder` to any arbitrary address; the validator accepts the proposal, executes the block with the wrong sequencer address, and both sides converge on a `ProposalCommitment` derived from a corrupted block hash. The committed block header, the L1-anchored block hash, and all fee transfers are permanently wrong.

---

### Finding Description

`ProposalInit` carries a `builder` field (the sequencer/builder address). The proposer sets it freely in `initiate_build`:

```rust
// crates/apollo_consensus_orchestrator/src/build_proposal.rs:169-188
let init = ProposalInit {
    builder: args.builder_address,   // proposer-supplied, from config
    ...
};
```

On the validator side, `is_proposal_init_valid` checks `timestamp`, `starknet_version`, `version_constant_commitment`, `height`, `l1_da_mode`, `l2_gas_price_fri`, all four L1 gas prices, and `fee_proposal_fri`. It never checks `builder`:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs:252-419
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    ...
) -> ValidateProposalResult<()> {
    // timestamp, starknet_version, version_constant_commitment,
    // height/l1_da_mode/l2_gas_price_fri, L1 prices, fee_proposal — all checked.
    // builder: never referenced.
    Ok(())
}
```

`ProposalInitValidation` itself has no `builder_address` field, so there is no reference value to compare against:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs:75-85
pub(crate) struct ProposalInitValidation {
    pub height: BlockNumber,
    pub block_timestamp_window_seconds: u64,
    pub previous_proposal_init: Option<PreviousProposalInitInfo>,
    pub l1_da_mode: L1DataAvailabilityMode,
    pub l2_gas_price_fri: GasPrice,
    pub starknet_version: StarknetVersion,
    pub fee_actual: Option<GasPrice>,
    // no builder_address
}
```

After `is_proposal_init_valid` passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which maps `init.builder` directly to `sequencer_address`:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs:329-332
Ok(starknet_api::block::BlockInfo {
    sequencer_address: init.builder,   // ← arbitrary proposer-supplied value
    ...
})
```

`PartialBlockHashComponents::new` then copies that address into the `sequencer` slot:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs:231
sequencer: SequencerContractAddress(block_info.sequencer_address),
```

And `calculate_block_hash` chains it into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs:258
.chain(&partial_block_hash_components.sequencer.0)
```

Because both the proposer and the validator derive `PartialBlockHash` from the same `ProposalInit.builder`, they converge on the same (corrupted) `ProposalCommitment`. Consensus proceeds, the block is finalized, and the wrong sequencer address is permanently embedded in the block hash and the block header stored by state sync.

---

### Impact Explanation

**Wrong block hash / state commitment (Critical).** The sequencer address is a direct input to `calculate_block_hash`. A forged `builder` produces a `PartialBlockHash` — and therefore a `ProposalCommitment` and a final `BlockHash` — that encodes the attacker-chosen address. This hash is committed to L1 and used as the `previous_block_hash` for all subsequent blocks, permanently corrupting the chain's hash chain.

**Wrong fee collection (Critical, economic).** `BlockInfo.sequencer_address` is the address to which transaction fees are transferred during execution. Setting `builder` to an attacker-controlled address redirects all fees for the block to that address.

**Wrong `get_sequencer_address()` syscall result (Critical).** Any contract that calls the `get_sequencer_address` syscall during execution of transactions in this block receives the forged address, producing wrong execution results and wrong receipts/events.

**Wrong block header in state sync (High).** `update_state_sync_with_new_block` stores `SequencerContractAddress(init.builder)` in `BlockHeaderWithoutHash.sequencer`, so the corrupted address propagates to all downstream consumers of the block header.

---

### Likelihood Explanation

The proposer role rotates among validators in BFT consensus. Any validator that becomes proposer can craft a `ProposalInit` with an arbitrary `builder` value. No special privilege beyond being the current-round proposer is required. The validator's `is_proposal_init_valid` path has no guard whatsoever on this field, so the attack succeeds silently on every honest validator node.

---

### Recommendation

Add `builder_address: ContractAddress` to `ProposalInitValidation` (populated from the node's own config, the same source used by `ProposalBuildArguments.builder_address`) and add an equality check inside `is_proposal_init_valid`:

```rust
if init_proposed.builder != proposal_init_validation.builder_address {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected={:?}, proposed={:?}",
            proposal_init_validation.builder_address, init_proposed.builder
        ),
    ));
}
```

This mirrors the pattern already used for `height`, `l1_da_mode`, `l2_gas_price_fri`, and `starknet_version`.

---

### Proof of Concept

1. A Byzantine node wins the proposer slot for height H.
2. It constructs `ProposalInit { builder: attacker_address, ... }` with all other fields valid.
3. It streams the proposal to validators.
4. Each validator calls `is_proposal_init_valid` — passes, because `builder` is never checked.
5. `initiate_validation` → `convert_to_sn_api_block_info` → `BlockInfo { sequencer_address: attacker_address }`.
6. The batcher executes the block; all fee transfers go to `attacker_address`; `get_sequencer_address()` returns `attacker_address`.
7. `BlockExecutionArtifacts::new` → `PartialBlockHashComponents::new` → `sequencer: attacker_address` → `PartialBlockHash` encodes `attacker_address`.
8. `ProposalCommitment` matches on both sides (both used the same `builder`); consensus decides.
9. `finalize_decision` → `update_state_sync_with_new_block` stores `attacker_address` as the sequencer in the block header.
10. The final `BlockHash` (used as parent hash for block H+1 and committed to L1) permanently encodes `attacker_address` in the sequencer slot.

**Corrupted values**: `BlockHash` at height H, `BlockHeaderWithoutHash.sequencer` at height H, all fee balances for block H, all `get_sequencer_address()` return values for block H.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
