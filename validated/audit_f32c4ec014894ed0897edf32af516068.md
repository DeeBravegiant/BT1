### Title
Unvalidated `ProposalInit.builder` Field Used as `sequencer_address` in Block Execution, Allowing Malicious Proposer to Redirect Fees and Corrupt Syscall Results — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`ProposalInit` carries two distinct address fields: `proposer` (the consensus proposer, validated against the committee) and `builder` (the block-builder/sequencer address, **never validated**). `convert_to_sn_api_block_info` maps `init.builder` directly to `BlockInfo.sequencer_address`, which drives fee transfers and the `get_sequencer_address` syscall. A malicious proposer can set `builder` to any arbitrary address; the validator accepts the proposal without checking this field, causing the committed block to carry the wrong sequencer address.

---

### Finding Description

`ProposalInit` has two address fields:

```
proposer  – "Address of the one who proposed the block in consensus."
builder   – "Address of the one who builds/sequences the block."
``` [1](#0-0) 

`convert_to_sn_api_block_info` converts a `ProposalInit` into a `starknet_api::block::BlockInfo` and assigns `init.builder` to `sequencer_address`:

```rust
Ok(starknet_api::block::BlockInfo {
    block_number: init.height,
    block_timestamp: BlockTimestamp(init.timestamp),
    sequencer_address: init.builder,   // ← proposer-supplied, never validated
    ...
})
``` [2](#0-1) 

This `BlockInfo` is passed to the batcher for both the build path (`initiate_build`) and the validate path (`initiate_validation`): [3](#0-2) [4](#0-3) 

`is_proposal_init_valid` validates `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `starknet_version`, `version_constant_commitment`, L1 gas prices, and `fee_proposal_fri`. It does **not** validate `builder`: [5](#0-4) 

The consensus manager validates `init.proposer` against the committee-determined proposer for the round, but `init.builder` is never checked: [6](#0-5) 

The proposer sets `builder` from its own static config (`builder_address`), with a TODO noting it should eventually come from the committee: [7](#0-6) 

---

### Impact Explanation

`BlockInfo.sequencer_address` is the address to which transaction fees are transferred and the value returned by the `get_sequencer_address` syscall. It is also an input to `PartialBlockHashComponents`, which feeds the block hash.

A malicious proposer sets `init.builder` to an arbitrary address (e.g., their own address, a burn address, or zero). The validator accepts the proposal without checking this field. The committed block then:

1. **Transfers all transaction fees to the wrong address** — direct economic theft or destruction of fees (Critical: economic impact).
2. **Returns the wrong value from `get_sequencer_address`** — any contract that gates logic on the sequencer address (access control, fee exemptions, etc.) behaves incorrectly (Critical: wrong syscall/execution result).
3. **Produces a block hash that encodes the wrong sequencer address** — the committed state root and block hash are permanently wrong.

---

### Likelihood Explanation

Exploiting this requires the attacker to be the elected proposer for a consensus round. In a staked, permissioned validator set this is a privileged position; however, any validator can become a proposer in rotation. The attack requires no special off-chain capability beyond being a consensus participant and setting a single field in the outbound `ProposalInit` message.

---

### Recommendation

Add validation of `init.builder` inside `is_proposal_init_valid`. The validator should compare `init.builder` against a locally-known expected value — either the proposer's own registered builder address (from the committee/staking contract) or, until that data is available, the validator's own configured `builder_address`. Reject any proposal whose `builder` field does not match the expected value, mirroring the existing check for `proposer`. [8](#0-7) 

---

### Proof of Concept

1. Attacker is the elected proposer for height H, round R.
2. In `initiate_build`, the attacker's node sets `builder_address` (static config) to `ATTACKER_ADDR` before building the proposal.
3. The resulting `ProposalInit` carries `builder = ATTACKER_ADDR`.
4. Validators receive the proposal; `is_proposal_init_valid` passes (no check on `builder`).
5. `initiate_validation` calls `convert_to_sn_api_block_info`, producing `BlockInfo { sequencer_address: ATTACKER_ADDR, ... }`.
6. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDR`; all fees are transferred to `ATTACKER_ADDR`; `get_sequencer_address` returns `ATTACKER_ADDR`.
7. Consensus reaches decision; the block is committed with the wrong sequencer address and wrong fee state.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
    }
