Audit Report

## Title
Unguarded Zero-Rate Broadcast via `updateRate()` Temporarily Freezes L2 Pool Deposits — (`contracts/cross-chain/CrossChainRateProvider.sol`)

## Summary
`CrossChainRateProvider.updateRate()` carries no access control and no zero-value guard. Before `LRTOracle.updateRSETHPrice()` is ever called, `rsETHPrice` holds its default value of `0`. Any caller can invoke `updateRate()` in that window, broadcasting `abi.encode(0)` via LayerZero. `CrossChainRateReceiver.lzReceive()` stores the value unconditionally, setting `rate = 0` on L2. Every subsequent `deposit()` call on the downstream pools then reverts with a division-by-zero panic until a corrective rate is broadcast.

## Finding Description

**No access control on `updateRate()`:**
`CrossChainRateProvider.updateRate()` is `external payable nonReentrant` with no ownership or role check. [1](#0-0) 
Any EOA can call it at any time, paying only the LayerZero messaging fee.

**Oracle starts at zero:**
`LRTOracle.rsETHPrice` is a plain `uint256` storage variable; its default value is `0`. [2](#0-1) 
It is only written inside `_updateRsETHPrice()`, reachable via `updateRSETHPrice()` or `updateRSETHPriceAsManager()`. [3](#0-2) 
Before either is called, `rsETHPrice == 0`.

**Rate source reads the uninitialized value:**
`RSETHRateProvider.getLatestRate()` returns `ILRTOracle(rsETHPriceOracle).rsETHPrice()` directly. [4](#0-3) 

**No zero-check before broadcast:**
`updateRate()` encodes and sends whatever `getLatestRate()` returns, including `0`, with no validation. [5](#0-4) 

**Receiver stores zero unconditionally:**
`CrossChainRateReceiver.lzReceive()` decodes the payload and writes it to `rate` with no zero-check. [6](#0-5) 

**Division by zero in deposit path:**
`RSETHPoolV2NBA.viewSwapRsETHAmountAndFee()` divides by `rsETHToETHrate` (sourced from `getRate()` → `rate`). [7](#0-6) 
`RSETHPoolV3ExternalBridge.viewSwapRsETHAmountAndFee()` does the same. [8](#0-7) 
Both `deposit()` functions call `viewSwapRsETHAmountAndFee()` unconditionally, so every deposit reverts with `Panic(0x12)`. [9](#0-8) [10](#0-9) 

## Impact Explanation

**Medium. Temporary freezing of funds.**

All L2 pool deposits revert with a division-by-zero panic for the duration of the window in which `rate == 0`. Recovery requires no privileged action: anyone can call `LRTOracle.updateRSETHPrice()` (public, no role restriction) to set a valid non-zero price, then call `updateRate()` again to broadcast the corrected rate. Once LayerZero delivers the new message, `lzReceive()` overwrites `rate` and deposits resume. No user funds are permanently locked; the impact is a temporary denial of the deposit function.

## Likelihood Explanation

The precondition — `rsETHPrice == 0` — exists during the deployment window between contract deployment and the first `updateRSETHPrice()` call. An attacker monitoring the mempool or deployment transactions can call `updateRate()` in that window at the cost of only the LayerZero messaging fee. The window is narrow in practice but non-zero and exploitable on any chain where the provider is deployed before the oracle is primed. The attack is permissionless and repeatable on any new deployment.

## Recommendation

1. **Add a zero-check in `updateRate()`** before broadcasting:
   ```solidity
   uint256 latestRate = getLatestRate();
   require(latestRate != 0, "Rate must be non-zero");
   ``` [11](#0-10) 

2. **Add a zero-check in `lzReceive()`** as defense-in-depth:
   ```solidity
   uint256 _rate = abi.decode(_payload, (uint256));
   require(_rate != 0, "Rate must be non-zero");
   ``` [6](#0-5) 

3. **Add access control to `updateRate()`** (e.g., `onlyOwner` or a keeper role) to prevent arbitrary callers from triggering rate updates at inopportune times. [1](#0-0) 

## Proof of Concept

```solidity
// Foundry fork test
// 1. Deploy LRTOracle — rsETHPrice == 0, updateRSETHPrice() never called
// 2. Deploy RSETHRateProvider pointing to LRTOracle, with LZ endpoint + rateReceiver configured
// 3. Deploy CrossChainRateReceiver (RSETHRateReceiver) on L2 pointing to the provider
// 4. Deploy RSETHPoolV2NBA with rsETHOracle = RSETHRateReceiver

// Attack (any EOA):
rateProvider.updateRate{value: lzFee}();
// LZ delivers abi.encode(0) → lzReceive() → rateReceiver.rate == 0

// Assert state:
assertEq(rateReceiver.rate(), 0);

// Victim deposit reverts:
vm.expectRevert(stdError.divisionError);
pool.deposit{value: 1 ether}("ref");
// Panic(0x12): division by zero in viewSwapRsETHAmountAndFee

// Recovery (no privilege required):
lrtOracle.updateRSETHPrice();           // sets rsETHPrice = 1e18 (if supply == 0) or computed price
rateProvider.updateRate{value: lzFee}(); // broadcasts corrected rate
// After LZ delivery: rateReceiver.rate > 0, deposits resume
```
