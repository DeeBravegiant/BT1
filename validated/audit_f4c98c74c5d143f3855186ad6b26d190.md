Audit Report

## Title
Silent Zero-Mint on Small Deposits Permanently Locks User Funds - (File: contracts/LRTDepositPool.sol)

## Summary
`LRTDepositPool.depositETH()` and `depositAsset()` accept user funds and silently mint zero rsETH when the deposit amount is too small for integer division in `getRsETHAmountToMint()` to produce a non-zero result. Because `minAmountToDeposit` defaults to zero and the only slippage guard is bypassable by passing `minRSETHAmountExpected = 0`, the user's ETH or LST is permanently locked in the pool with no recovery path.

## Finding Description

`getRsETHAmountToMint()` computes:

```solidity
// LRTDepositPool.sol L520
rsethAmountToMint = (amount * lrtOracle.getAssetPrice(asset)) / lrtOracle.rsETHPrice();
```

When `amount * assetPrice < rsETHPrice`, Solidity integer division truncates to **0**. [1](#0-0) 

`_beforeDeposit()` has two guards, both of which fail to catch this:

1. **Amount check** (L657): `depositAmount == 0 || depositAmount < minAmountToDeposit`. Since `minAmountToDeposit` is never set in `initialize()`, it defaults to `0`. Any non-zero `depositAmount` passes. [2](#0-1) [3](#0-2) 

2. **Slippage check** (L667): `rsethAmountToMint < minRSETHAmountExpected`. When the caller passes `minRSETHAmountExpected = 0`, the condition `0 < 0` is false and no revert occurs. [4](#0-3) 

Execution then reaches `_mintRsETH(0)`, which calls `RSETH.mint(msg.sender, 0)`. [5](#0-4) 

The `checkDailyMintLimit(0)` modifier in `RSETH` does not revert (adding 0 to `currentPeriodMintedAmount` cannot exceed `maxMintAmountPerDay`), and OpenZeppelin's `_mint(to, 0)` is a no-op. [6](#0-5) [7](#0-6) 

For `depositETH`, the ETH is already in the contract via `msg.value` before `_beforeDeposit` is called. For `depositAsset`, `safeTransferFrom` executes at L114 after `_beforeDeposit` returns successfully — in both cases the user's assets are transferred in and 0 rsETH is minted. [8](#0-7) [9](#0-8) 

There is no user-facing withdrawal function to recover deposited assets without holding rsETH. The funds are permanently locked. [10](#0-9) 

## Impact Explanation

The depositor's ETH or LST is permanently transferred into `LRTDepositPool` with zero rsETH minted in return and no recovery path. This constitutes **Critical: Permanent freezing of funds** (and equivalently, direct permanent loss of user funds). The locked assets marginally inflate the rsETH price, benefiting existing holders at the victim's expense. [11](#0-10) 

## Likelihood Explanation

Any unprivileged external caller can trigger this by calling `depositETH{value: N}(0, "")` or `depositAsset(asset, N, 0, "")` with a small `N`. No special role or privilege is required. As rsETH appreciates over time, the minimum deposit required to mint 1 rsETH grows, widening the vulnerable range. Naive integrators, bots, and users who omit the slippage parameter are realistic victims. The condition is repeatable and worsens as the protocol matures. [12](#0-11) 

## Recommendation

Add an explicit zero-shares guard in `_beforeDeposit()` immediately after computing `rsethAmountToMint`:

```solidity
rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);
if (rsethAmountToMint == 0) revert ZeroRsETHMinted();
```

Additionally, enforce a non-zero `minAmountToDeposit` in `initialize()` calibrated to always produce at least 1 rsETH at the current price, and keep it updated as rsETH appreciates. [13](#0-12) 

## Proof of Concept

1. rsETH price appreciates to `1.5e18` (1.5 ETH per rsETH) via `LRTOracle.rsETHPrice()`.
2. User calls `LRTDepositPool.depositETH{value: 1}(0, "")`.
3. `_beforeDeposit(ETH_TOKEN, 1, 0)` is entered.
4. `depositAmount == 0` → false; `1 < 0` (uint256) → false. Check passes.
5. `getRsETHAmountToMint(ETH_TOKEN, 1)` → `(1 * 1e18) / 1.5e18` → `0`.
6. `0 < 0` → false. `MinimumAmountToReceiveNotMet` is not thrown.
7. `_mintRsETH(0)` → `RSETH.mint(msg.sender, 0)` → `checkDailyMintLimit(0)` passes → `_mint(msg.sender, 0)` → no-op.
8. User's 1 wei ETH is permanently locked in `LRTDepositPool`; user holds 0 rsETH.

        private

    {
        if (depositAmount == 0 || depositAmount < minAmountToDeposit) {
            revert InvalidAmountToDeposit();
        }

        if (_checkIfDepositAmountExceedesCurrentLimit(asset, depositAmount)) {
            revert MaximumDepositLimitReached();
        }

        rsethAmountToMint = getRsETHAmountToMint(asset, depositAmount);

        if (rsethAmountToMint < minRSETHAmountExpected) {
            revert MinimumAmountToReceiveNotMet();
        }
    }
