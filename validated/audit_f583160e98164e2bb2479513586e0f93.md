### Title
Unguarded Empty-List Index Access in `_validate_summaries_weight` Crashes Wallet Weight-Proof Validation — (File: `chia/full_node/weight_proof.py`)

### Summary
`_validate_summaries_weight` unconditionally accesses `summaries[-1]` without checking whether `summaries` is empty. The wallet-side entry point `WalletWeightProofHandler.validate_weight_proof` lacks the empty-`sub_epochs` guard that the full-node path has, so a malicious peer can send a crafted `WeightProof` with `sub_epochs=[]` to trigger an unhandled `IndexError` that crashes the wallet's weight-proof validation and blocks sync.

### Finding Description
`_validate_summaries_weight` in `chia/full_node/weight_proof.py` begins by indexing the last element of `summaries`:

```python
def _validate_summaries_weight(...) -> bool:
    num_over = summaries[-1].num_blocks_overflow   # line 929 – IndexError if empty
    ses_end_height = (len(summaries) - 1) * constants.SUB_EPOCH_BLOCKS + num_over - 1
``` [1](#0-0) 

`summaries` is produced by `_map_sub_epoch_summaries`, which iterates over `weight_proof.sub_epochs`. When that list is empty the loop body never executes and `summaries` is returned as `[]`. [2](#0-1) 

`_validate_sub_epoch_summaries` calls `_validate_summaries_weight` with the (potentially empty) result without any length guard: [3](#0-2) 

The full-node `WeightProofHandler.validate_weight_proof` guards against this before calling `_validate_sub_epoch_summaries`:

```python
if len(weight_proof.sub_epochs) == 0:
    return False, uint32(0), []
``` [4](#0-3) 

`WalletWeightProofHandler.validate_weight_proof` has **no such guard**:

```python
summaries, sub_epoch_weight_list = _validate_sub_epoch_summaries(self._constants, weight_proof)
await asyncio.sleep(0)
if summaries is None or sub_epoch_weight_list is None:
    raise ValueError("weight proof failed sub epoch data validation")
``` [5](#0-4) 

The `IndexError` raised inside `_validate_summaries_weight` is not a `None` return, so the `if summaries is None` check never fires; the exception propagates uncaught through `validate_weight_proof` and into the wallet sync path.

### Impact Explanation
`WalletWeightProofHandler.validate_weight_proof` is called from `WalletNode.fetch_and_update_weight_proof`: [6](#0-5) 

An unhandled `IndexError` here aborts the wallet's sync update. If the attacker is the peer the wallet is syncing from, the wallet cannot advance its chain tip, constituting a long-lived inability to process sync updates — matching the **High** impact category.

### Likelihood Explanation
Any unprivileged peer that the wallet connects to can advertise a peak and respond to `RequestProofOfWeight` with a crafted `WeightProof`. The wallet performs only shallow checks on `recent_chain_data[-1]` (height, weight, header hash) before passing the proof to `validate_weight_proof`: [7](#0-6) 

No cryptographic material or privileged access is required to craft a `WeightProof` with `sub_epochs=[]` and a `recent_chain_data` whose last entry matches the advertised peak.

### Recommendation
Add the same guard that the full-node path uses at the top of `WalletWeightProofHandler.validate_weight_proof`:

```python
if len(weight_proof.sub_epochs) == 0:
    raise ValueError("weight proof has no sub epochs")
```

Alternatively, add a length check inside `_validate_summaries_weight` before accessing `summaries[-1]`:

```python
if not summaries:
    return False
``` [8](#0-7) 

### Proof of Concept
1. Run a malicious full-node peer that responds to `RequestProofOfWeight` with a crafted `WeightProof`:
   - `sub_epochs = []`
   - `sub_epoch_segments = []`
   - `recent_chain_data = [<HeaderBlock matching the advertised peak>]`
2. Advertise a peak to a wallet node and wait for it to request a weight proof.
3. Send the crafted proof. `WalletWeightProofHandler.validate_weight_proof` calls `_validate_sub_epoch_summaries`, which calls `_validate_summaries_weight(constants, 0, [], weight_proof)`, which raises `IndexError: list index out of range` at `summaries[-1]` (line 929).
4. The exception propagates through `fetch_and_update_weight_proof`, aborting the wallet's sync update. [9](#0-8) [10](#0-9)

def _validate_summaries_weight(
    constants: ConsensusConstants,
    sub_epoch_data_weight: uint128,
    summaries: list[SubEpochSummary],
    weight_proof: WeightProof,
) -> bool:
    num_over = summaries[-1].num_blocks_overflow
    ses_end_height = (len(summaries) - 1) * constants.SUB_EPOCH_BLOCKS + num_over - 1
    curr: HeaderBlock | None = None
    for block in weight_proof.recent_chain_data:
        if block.reward_chain_block.height == ses_end_height:
            curr = block
    if curr is None:
        return False

    return curr.reward_chain_block.weight == sub_epoch_data_weight
```
