Audit Report

## Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Additional Fees — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary
`setPoolAdminFees` enforces the factory's `maxAdminSpreadFeeE6` cap before writing admin spread fees to the pool, but `setPoolBinAdditionalFees` forwards `addFeeBuyE6`/`addFeeSellE6` directly to the pool with no cap validation. Because per-bin fees are added directly to `baseFeeX64` in swap execution, a pool admin can charge traders above the factory-enforced ceiling — including when `maxAdminSpreadFeeE6 = 0` — constituting a direct admin-boundary break.

## Finding Description
**Guarded path — global admin spread fee:**

`setPoolAdminFees` enforces the cap before writing: [1](#0-0) 

**Unguarded path — per-bin additional fees:**

`setPoolBinAdditionalFees` passes values directly to the pool with no cap check: [2](#0-1) 

`setBinAdditionalFees` on the pool only validates the bin index range, not the fee magnitude: [3](#0-2) 

**How per-bin fees reach traders in swap execution:**

The per-bin fee is added directly to `baseFeeX64` to compute effective buy/sell prices: [4](#0-3) 

This same pattern is applied in `_getInitialStateForSwap`, which loads `binState = _binStates[curBinIdxCache]` (including `addFeeBuyE6`/`addFeeSellE6`) for every swap iteration, confirming the per-bin fees affect actual swap execution, not just the price quoter. [5](#0-4) 

A `uint16` field allows values up to 65,535 E6 ≈ 6.55% additional fee per bin. The factory's `HARD_MAX_SPREAD_FEE_E6 = 200_000` and the owner-configurable `maxAdminSpreadFeeE6` are both completely bypassed for this code path. [6](#0-5) 

## Impact Explanation
The factory owner uses `maxAdminSpreadFeeE6` to guarantee traders that no pool admin can charge more than a defined spread. A pool admin can silently call `setPoolBinAdditionalFees` with `addFeeBuyE6 = 65535` on any active bin, adding ~6.55% to the effective ask price (or reducing the effective bid price by ~6.55%) beyond the capped global fee. If the factory owner sets `maxAdminSpreadFeeE6 = 0` to prohibit admin spread fees entirely, the pool admin can still extract up to ~6.55% per bin through this uncapped channel. The excess accrues as LP spread revenue in the pool's bin balances. This is a direct admin-boundary break: pool admin exceeds the factory-enforced fee cap, causing traders to receive worse execution than the factory's published caps imply.

## Likelihood Explanation
The pool admin is a semi-trusted role with direct, permissioned access to `setPoolBinAdditionalFees`. No special conditions, flash loans, or oracle manipulation are required. The call is a single transaction executable immediately after pool deployment. Any pool admin who wishes to extract additional value from traders can do so at any time. [7](#0-6) 

## Recommendation
Add a cap check in `setPoolBinAdditionalFees` mirroring the check in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce a combined cap: `globalAdminSpreadFee + addFeeBuyE6 ≤ maxAdminSpreadFeeE6`.

## Proof of Concept
1. Factory owner calls `setFeeCaps(..., newMaxAdminSpreadFeeE6 = 0, ...)` — intending to prohibit all admin spread fees.
2. Pool admin calls `setPoolAdminFees(pool, 0, 0)` — passes the cap check, no global fee set.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — **no cap check fires**.
4. `_binStates[0].addFeeBuyE6 = 65535` and `addFeeSellE6 = 65535` are written to storage.
5. On the next swap through bin 0, `buyFeeX64 = baseFeeX64 + Math.mulDiv(65535, ONE_X64, 1e6)`, inflating the ask price by ~6.55% above the oracle mid-price plus base spread.
6. Traders pay ~6.55% more than the factory's zero-admin-fee cap permits; the excess accrues as LP spread revenue in the pool's bin balances.
