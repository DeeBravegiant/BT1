Audit Report

## Title
Missing Zero-Rate Guard in `lzReceive` Allows `rate = 0` to Be Stored, Breaking Cross-Chain Pool Operations — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver.lzReceive` performs three access-control checks but applies no sanity check on the decoded rate value, unconditionally storing any `uint256` including zero. Because `MultiChainRateProvider.updateRate()` is permissionless and `LRTOracle.rsETHPrice` defaults to `0` before `updateRSETHPrice()` is ever called, an unprivileged caller can trigger a cross-chain push of `rate = 0`, causing all downstream pool operations that consume `getRate()` to revert until a corrective LZ message is delivered.

## Finding Description
`CrossChainRateReceiver.lzReceive` (lines 93–97) decodes and stores the rate with no validity check:

```solidity
uint256 _rate = abi.decode(_payload, (uint256));
rate = _rate;          // no require(_rate > 0)
lastUpdated = block.timestamp;
```

`RSETHMultiChainRateProvider.getLatestRate()` (lines 26–28) reads `rsETHPrice` directly from `LRTOracle`:

```solidity
function getLatestRate() public view override returns (uint256) {
    return ILRTOracle(rsETHPriceOracle).rsETHPrice();
}
```

`LRTOracle.rsETHPrice` is a plain `uint256` storage variable (line 28) that is default-initialized to `0`. Before `updateRSETHPrice()` is ever called, `rsETHPrice == 0`. `MultiChainRateProvider.updateRate()` (line 108) is fully permissionless (`external payable nonReentrant`, no role check), so any caller can invoke it during the deployment window to broadcast `rate = 0` cross-chain.

A second path exists: in `_updateRsETHPrice`, if `totalETHInProtocol == 0` while `rsethSupply > 0`, `newRsETHPrice` computes to `0` (line 250). The downside-protection guard (lines 273–274) only fires when `pricePercentageLimit > 0`, which defaults to `0` (never set in `initialize`), so execution falls through to `rsETHPrice = newRsETHPrice = 0` at line 313.

Once `rate = 0` is stored in the receiver, `RSETHPoolV3ExternalBridge.viewSwapAssetToPremintedRsETH` (lines 523–524) explicitly reverts:

```solidity
uint256 rsETHToETHrate = getRate();
if (rsETHToETHrate == 0) revert UnsupportedOracle();
```

Any other pool path that divides by `getRate()` would revert with a division-by-zero. All cross-chain pool operations are broken until a corrective `lzReceive` with a valid rate is delivered.

## Impact Explanation
**Low. Contract fails to deliver promised returns, but doesn't lose value.**

Cross-chain deposits and swaps revert for all users on the destination chain. No funds are lost because the revert prevents any state change, but the contract fails to deliver its promised exchange-rate service until a corrective LZ message is sent.

## Likelihood Explanation
**Low-to-Medium.** Scenario A (pre-initialization) is a deployment-window race condition requiring no privileges — `updateRate()` is permissionless and callable by anyone before `updateRSETHPrice()` has ever been invoked. Scenario B requires `pricePercentageLimit` to remain at its default of `0` and all protocol TVL to reach zero while rsETH supply is non-zero. Neither scenario requires any privileged compromise.

## Recommendation
Add a zero-rate guard in `lzReceive` before storing the decoded value:

```solidity
uint256 _rate = abi.decode(_payload, (uint256));
require(_rate > 0, "Rate must be > 0");
rate = _rate;
```

Additionally, add a symmetric guard in `MultiChainRateProvider.updateRate()` to prevent broadcasting a zero rate in the first place:

```solidity
uint256 latestRate = getLatestRate();
require(latestRate > 0, "Rate must be > 0");
```

## Proof of Concept
Deploy `RSETHRateReceiver` with a mock LZ endpoint. Before any `updateRSETHPrice()` call (so `rsETHPrice == 0`), call `updateRate()` on the provider — it reads `rsETHPrice = 0` and sends the LZ message. The endpoint delivers `lzReceive` with `payload = abi.encode(0)`. The receiver stores `rate = 0`. Any subsequent call to `getRate()` returns `0`, causing `RSETHPoolV3ExternalBridge.viewSwapAssetToPremintedRsETH` to revert with `UnsupportedOracle()`.

```solidity
function test_lzReceive_accepts_zero_rate() public {
    bytes memory payload = abi.encode(uint256(0));
    vm.prank(address(endpoint));
    receiver.lzReceive(srcChainId, abi.encodePacked(rateProvider), 0, payload);
    assertEq(receiver.rate(), 0); // passes — invariant broken
}
```

Run with: `forge test --match-test test_lzReceive_accepts_zero_rate -vvv`
