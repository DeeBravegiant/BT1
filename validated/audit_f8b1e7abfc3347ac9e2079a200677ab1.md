### Title
Unvalidated `builder` Field in `ProposalInit` Allows Fee Redirection to Arbitrary Address — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

The `builder` field of `ProposalInit` is accepted verbatim from the network during proposal validation. It is never checked against any locally-trusted value. Because `builder` is forwarded directly to the blockifier as `sequencer_address` — the address that collects transaction fees — a malicious proposer can silently redirect all block fees to an arbitrary address, and every validator node will accept the block.

### Finding Description

`is_proposal_init_valid` in `validate_proposal.rs` enforces the following `ProposalInit` fields:

| Field | Validated |
|---|---|
| `height` | ✓ |
| `l1_da_mode` | ✓ |
| `l2_gas_price_fri` | ✓ |
| `l1_gas_price_fri/wei` | ✓ (within-margin) |
| `l1_data_gas_price_fri/wei` | ✓ (within-margin) |
| `starknet_version` | ✓ |
| `version_constant_commitment` | ✓ |
| `fee_proposal_fri` | ✓ |
| `timestamp` | ✓ |
| **`builder`** | **✗ — never checked** | [1](#0-0) 

The `builder` field is then consumed unconditionally by `convert_to_sn_api_block_info`, which maps it to `sequencer_address` in the `BlockInfo` struct that is handed to the batcher: [2](#0-1) 

The batcher executes every transaction in the block using this `BlockInfo`. In Starknet, `sequencer_address` is the recipient of all transaction fees. The proposer also computes `proposal_commitment` using the same `builder`-derived `sequencer_address`, so the `ProposalFinMismatch` guard at line 244 passes cleanly — both sides hash the same (attacker-chosen) address: [3](#0-2) 

The `ProposalInitValidation` struct, which carries all locally-trusted reference values, has no `builder` slot, confirming the omission is structural: [4](#0-3) 

When the proposer builds its own block, it sets `builder` from a local config field `builder_address`: [5](#0-4) 

Validators have this same config but never compare it against the incoming `init.builder`.

### Impact Explanation

Every validator node executes the block with the attacker-supplied `sequencer_address`. All transaction fees for the entire block are credited to that address. The committed block header records the attacker's address as the sequencer, making the fee theft permanent and on-chain. This matches the **Critical** impact class: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.*

### Likelihood Explanation

The attack requires the adversary to be the elected proposer for at least one round. In a permissioned or semi-permissioned validator set this is a scheduled, predictable event. No special network position, Sybil capability, or cryptographic break is needed — the attacker simply sets one field in the `ProposalInit` they author. Likelihood: **Medium**.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it inside `is_proposal_init_valid`:

```rust
// In ProposalInitValidation:
pub expected_builder: ContractAddress,

// In is_proposal_init_valid:
if init_proposed.builder != proposal_init_validation.expected_builder {
    return Err(ValidateProposalError::InvalidProposalInit(
        init_proposed.clone(),
        proposal_init_validation.clone(),
        format!(
            "builder mismatch: expected {:?}, proposed {:?}",
            proposal_init_validation.expected_builder,
            init_proposed.builder
        ),
    ));
}
```

`expected_builder` should be populated from the node's own `builder_address` configuration, the same value used in `initiate_build`.

### Proof of Concept

1. Attacker controls validator `V` and waits for (or engineers) a round where `V` is the elected proposer.
2. `V` constructs `ProposalInit` with `builder = ATTACKER_ADDR` (any address it controls).
3. `V` broadcasts the proposal stream. Every peer calls `validate_proposal` → `is_proposal_init_valid`.
4. `is_proposal_init_valid` checks height, gas prices, version, etc. — **`builder` is never read**.
5. `initiate_validation` calls `convert_to_sn_api_block_info(init)`, producing `BlockInfo { sequencer_address: ATTACKER_ADDR, … }`.
6. The batcher executes all transactions; fees accumulate to `ATTACKER_ADDR`.
7. `finish_proposal` returns a `partial_block_hash` computed over `sequencer_address = ATTACKER_ADDR`.
8. `V` sent `ProposalFin.proposal_commitment` computed with the same address → `built_block == received_fin.proposal_commitment` → no mismatch error.
9. Consensus reaches decision; the block is committed with `ATTACKER_ADDR` as the fee recipient. [6](#0-5) [7](#0-6) [8](#0-7)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let proposal_init_info = PreviousProposalInitInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
