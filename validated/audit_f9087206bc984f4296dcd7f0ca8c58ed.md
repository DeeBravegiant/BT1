Audit Report

## Title
Stale Rate Served Indefinitely by `getRate()` Enables Mispriced rsETH Minting — (`contracts/cross-chain/CrossChainRateReceiver.sol`)

## Summary
`CrossChainRateReceiver` tracks `lastUpdated` on every `lzReceive` call but `getRate()` returns `rate` unconditionally without consulting it. If the LayerZero bridge stops delivering updates, all downstream L2 pool contracts continue minting rsETH at an arbitrarily old rate, allowing new depositors to receive more rsETH than they are entitled to at the expense of existing holders' accrued yield.

## Finding Description
`CrossChainRateReceiver` stores `rate` and `lastUpdated` as state variables: [1](#0-0) 

`lzReceive` updates both on every successful bridge message: [2](#0-1) 

`getRate()` returns only `rate`, never consulting `lastUpdated`: [3](#0-2) 

Every pool contract delegates its own `getRate()` directly to `IOracle(rsETHOracle).getRate()` with no additional freshness guard. For example, `RSETHPoolV2NBA`: [4](#0-3) 

And `RSETHPoolV3`: [5](#0-4) 

The minting formula in every pool divides by the stale rate: [6](#0-5) 

rsETH is a yield-bearing token whose rate monotonically increases over time. If the bridge halts and the stored rate is stale (lower than the true current rate), the denominator is smaller than it should be, so depositors receive more rsETH than they are entitled to. The surplus rsETH represents a claim on ETH that was not deposited, diluting all existing rsETH holders' accrued yield.

By contrast, `ChainlinkOracleForRSETHPoolCollateral` explicitly checks for staleness before returning a price: [7](#0-6) 

`CrossChainRateReceiver` has no equivalent guard despite tracking `lastUpdated`.

## Impact Explanation
**High — Theft of unclaimed yield.** rsETH accrues staking yield continuously, reflected as a monotonically increasing rsETH/ETH rate. When the bridge is down and the stored rate is stale (lower than the true rate), new depositors mint rsETH at a favorable rate. The extra rsETH they receive represents a proportional claim on the underlying ETH pool that dilutes existing holders' accrued staking yield. The longer the bridge is down and the more deposits occur, the larger the aggregate dilution. The `RSETHRateReceiver` is deployed on Arbitrum, Optimism, Base, Scroll, Blast, Mode, Linea, zkSync, Zircuit, and other chains, making the attack surface broad.

## Likelihood Explanation
LayerZero bridge liveness depends on off-chain relayers and gas funding. The `updateRate()` call on the provider side requires the caller to supply ETH for cross-chain gas; if no one calls it or the relayer stalls, the receiver silently serves a stale rate. No on-chain circuit breaker exists to detect or halt this condition. Any unprivileged depositor can exploit this by simply calling `deposit()` on any pool while the bridge is down — no special access or setup is required.

## Recommendation
Add a configurable `maxStaleness` threshold and revert in `getRate()` if the rate is too old, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
uint256 public maxStaleness; // e.g. 86400 (24 hours)

error StaleRate();

function getRate() external view returns (uint256) {
    if (block.timestamp - lastUpdated > maxStaleness) revert StaleRate();
    return rate;
}
```

## Proof of Concept
```solidity
// Fork test on any L2 where RSETHRateReceiver is deployed as rsETHOracle
function test_staleRateMispricing() external {
    // 1. Simulate one valid lzReceive delivering rate R at T0
    vm.prank(layerZeroEndpoint);
    receiver.lzReceive(srcChainId, abi.encodePacked(rateProvider, address(0)), 0,
        abi.encode(1.05e18)); // rate = 1.05 ETH per rsETH

    assertEq(receiver.getRate(), 1.05e18);

    // 2. Warp 48 hours — bridge delivers no updates
    vm.warp(block.timestamp + 48 hours);

    // 3. getRate() still returns the stale rate — no revert
    assertEq(receiver.getRate(), 1.05e18); // true rate would be ~1.0506e18

    // 4. Pool mints rsETH at stale rate
    // deposit 1 ETH → rsETHAmount = 1e18 * 1e18 / 1.05e18 ≈ 0.952381e18
    // at true rate 1.0506e18 → should be ≈ 0.951837e18
    // depositor receives ~0.000544e18 extra rsETH per ETH deposited
    // at scale (e.g. 10,000 ETH deposited during outage) → ~5.44 ETH stolen from existing holders
}
```
