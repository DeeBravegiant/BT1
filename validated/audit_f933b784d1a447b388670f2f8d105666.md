### Title
Malicious Harvester Can Unconditionally Redirect Farmer XCH Block Rewards via `RespondSignatures.farmer_reward_address_override` - (File: chia/farmer/farmer_api.py)

### Summary
A malicious third-party harvester can redirect the farmer's XCH block reward to an arbitrary attacker-controlled address by injecting `farmer_reward_address_override` in the `RespondSignatures` protocol message. The farmer's `_process_respond_signatures()` function accepts this field unconditionally, with no fee-quality check and no cross-validation against the value (if any) declared in the earlier `NewProofOfSpace` message.

### Finding Description

CHIP-22 introduces a fee convention that allows third-party harvesters to claim the farmer's block reward as a service fee, but only when the proof quality meets a harvester-declared threshold. The advisory check is implemented in `notify_farmer_reward_taken_by_harvester_as_fee()` and is called in `new_proof_of_space()` when `NewProofOfSpace.farmer_reward_address_override` is non-`None`.

`RespondSignatures` carries its own independent `farmer_reward_address_override` field: [1](#0-0) 

In `_process_respond_signatures()`, the farmer unconditionally replaces its configured `farmer_target` with whatever address the harvester supplies in this field: [2](#0-1) 

This overridden address is then placed directly into the `DeclareProofOfSpace` message sent to the full node: [3](#0-2) 

The full node uses `farmer_ph` (derived from `DeclareProofOfSpace.farmer_puzzle_hash`) when constructing the foliage, which permanently determines where the farmer reward coin is created: [4](#0-3) 

The fee-quality check that exists in `new_proof_of_space()` is (a) advisory-only — it only logs warnings and never blocks the flow — and (b) only examines `NewProofOfSpace.farmer_reward_address_override`, a completely separate field from `RespondSignatures.farmer_reward_address_override`: [5](#0-4) [6](#0-5) 

A malicious harvester therefore has two independent bypass paths:

1. **Direct bypass via `RespondSignatures`**: Send `NewProofOfSpace` with `farmer_reward_address_override=None` (no advisory check triggered), then respond to `RequestSignatures` with `RespondSignatures.farmer_reward_address_override=<attacker_address>`. The farmer accepts it with zero validation.

2. **Advisory-only bypass via `NewProofOfSpace`**: Even if `NewProofOfSpace.farmer_reward_address_override` is set with a failing fee quality, `notify_farmer_reward_taken_by_harvester_as_fee()` only emits a log warning and returns `None`; the block-making flow continues and the override is still used.

### Impact Explanation

Every time the malicious harvester's plot wins a block, the farmer's entire reward (1/8 of the block subsidy plus all transaction fees, currently ~0.25 XCH per block) is sent to the attacker's puzzle hash instead of the farmer's configured `farmer_target`. The farmer receives nothing. This is an unconditional, on-chain, irreversible diversion of XCH.

This matches the Critical impact category: **reward diversion affecting XCH**.

### Likelihood Explanation

Any third-party harvester service (e.g., GPU-accelerated harvesters, DrPlotter-style services) connected to the farmer can exploit this. The farmer has no in-process mechanism to detect the substitution before the block is submitted. The harvester is already a trusted network peer by design, so no additional privilege escalation is required.

### Recommendation

In `_process_respond_signatures()`, when `response.farmer_reward_address_override` is non-`None`:

1. Retrieve the corresponding `NewProofOfSpace` entry from `self.farmer.proofs_of_space` and check whether its `farmer_reward_address_override` matches the one in `RespondSignatures`. If they differ, reject the override and use `self.farmer.farmer_target`.
2. Perform the same fee-quality check as `notify_farmer_reward_taken_by_harvester_as_fee()` and **enforce** it (return `None` / drop the block candidate) rather than merely logging a warning.
3. Consider making the fee-quality check in `new_proof_of_space()` a hard gate rather than advisory-only.

    def notify_farmer_reward_taken_by_harvester_as_fee(
        self, sp: farmer_protocol.NewSignagePoint, proof_of_space: harvester_protocol.NewProofOfSpace
    ) -> None:
        """
        Apply a fee quality convention (see CHIP-22: https://github.com/Chia-Network/chips/pull/88)
        given the proof and signage point. This will be tested against the fee threshold reported
        by the harvester (if any), and logged.
        """
        assert proof_of_space.farmer_reward_address_override is not None

        challenge_str = str(sp.challenge_hash)

        ph_prefix = self.config["network_overrides"]["config"][self.config["selected_network"]]["address_prefix"]
        farmer_reward_puzzle_hash = encode_puzzle_hash(proof_of_space.farmer_reward_address_override, ph_prefix)

        self.log.info(
            f"Farmer reward for challenge '{challenge_str}' "
            + f"taken by harvester for reward address '{farmer_reward_puzzle_hash}'"
        )

        fee_quality = calculate_harvester_fee_quality(proof_of_space.proof.proof, sp.challenge_hash)
        fee_quality_rate = float(fee_quality) / float(0xFFFFFFFF) * 100.0

        if proof_of_space.fee_info is not None:
            fee_threshold = proof_of_space.fee_info.applied_fee_threshold
            fee_threshold_rate = float(fee_threshold) / float(0xFFFFFFFF) * 100.0

            if fee_quality <= fee_threshold:
                self.log.info(
                    f"Fee threshold passed for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
            else:
                self.log.warning(
                    f"Invalid fee threshold for challenge '{challenge_str}': "
                    + f"{fee_quality_rate:.3f}%/{fee_threshold_rate:.3f}% ({fee_quality}/{fee_threshold})"
                )
                self.log.warning(
                    "Harvester illegitimately took a fee reward that "
                    + "did not belong to it or it incorrectly applied the fee convention."
                )
        else:
            self.log.warning(
                "Harvester illegitimately took reward by failing to provide its fee rate "
                + f"for challenge '{challenge_str}'. "
                + f"Fee quality was {fee_quality_rate:.3f}% ({fee_quality} or 0x{fee_quality:08x})"
            )
