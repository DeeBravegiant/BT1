Audit Report

## Title
Missing Non-Positive Price Guard in `getAssetPrice` Enables Deposit DoS via Overflow - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary
`ChainlinkPriceOracle.getAssetPrice()` casts the `int256 answer` from Chainlink's `latestRoundData()` directly to `uint256` with no sign check. If the feed returns a negative value, the cast produces a near-`2^256` value and the subsequent multiplication overflows under Solidity 0.8 checked arithmetic, reverting the call. This revert propagates through `LRTOracle.getAssetPrice()` → `LRTDepositPool.getRsETHAmountToMint()` → `depositAsset()`/`depositETH()`, causing a temporary freeze of all deposit functionality until the feed recovers.

## Finding Description
In `contracts/oracles/ChainlinkPriceOracle.sol` at lines 52–54, the price is read and immediately cast without validation:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
``` [1](#0-0) 

There is no guard of the form `if (price <= 0) revert ...` before the cast. If `price` is any negative `int256` value (e.g., `-1`), `uint256(-1)` evaluates to `2^256 - 1`. The multiplication `(2^256 - 1) * 1e18` overflows and reverts under Solidity 0.8's checked arithmetic.

The call chain that reaches this code from a public entry point is:

1. `LRTDepositPool.depositAsset()` / `depositETH()` → `_beforeDeposit()` → `getRsETHAmountToMint()` [2](#0-1) 

2. `getRsETHAmountToMint()` calls `lrtOracle.getAssetPrice(asset)` [3](#0-2) 

3. `LRTOracle.getAssetPrice()` delegates to `IPriceFetcher(assetPriceOracle[asset]).getAssetPrice(asset)`, which resolves to `ChainlinkPriceOracle.getAssetPrice()` [4](#0-3) 

The sister contract `ChainlinkOracleForRSETHPoolCollateral` already performs the correct check before casting:

```solidity
if (ethPrice <= 0) revert InvalidPrice();
uint256 normalizedPrice = uint256(ethPrice) * 1e18 / ...
``` [5](#0-4) 

`ChainlinkPriceOracle` has no equivalent guard.

## Impact Explanation
When any supported LST asset's Chainlink feed returns a negative price, `ChainlinkPriceOracle.getAssetPrice()` reverts due to arithmetic overflow. This revert propagates through the entire deposit call stack, causing `depositAsset()` and `depositETH()` to revert for all users. Additionally, `LRTOracle.updateRSETHPrice()` (which calls `_getTotalEthInProtocol()` → `getAssetPrice()` for every supported asset) also reverts, preventing rsETH price updates. This constitutes a **temporary freezing of funds** (Medium impact) — all deposit functionality is bricked until the affected Chainlink feed returns a positive value. [6](#0-5) 

## Likelihood Explanation
Chainlink aggregators enforce min/max circuit breakers. When an asset's price falls below the configured minimum, the aggregator returns the minimum boundary value rather than the true market price. For some feeds this boundary is `0`; for others it can be a small positive integer. A return of exactly `0` does not cause overflow (it returns `0` from `getAssetPrice`, which causes `getRsETHAmountToMint` to return `0` and silently mint nothing or revert on the `minRSETHAmountExpected` check), but any negative sentinel value causes the overflow revert described above. This condition requires no attacker — it occurs organically during extreme market stress, precisely when protocol availability is most critical. No privileged action is required to trigger it.

## Recommendation
Add a non-positive price check before the cast, mirroring the pattern already used in `ChainlinkOracleForRSETHPoolCollateral`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

Additionally, consider adding staleness checks (`updatedAt`, `answeredInRound`) as `ChainlinkOracleForRSETHPoolCollateral` does at lines 30–31. [7](#0-6) 

## Proof of Concept
1. Deploy or fork with a supported LST asset (e.g., stETH) whose `assetPriceFeed` is set to a mock Chainlink aggregator.
2. Configure the mock to return `price = -1` from `latestRoundData()`.
3. Call `LRTDepositPool.depositAsset(stETH, amount, 0, "")` as any unprivileged user.
4. Execution reaches `ChainlinkPriceOracle.getAssetPrice(stETH)`: `uint256(-1) * 1e18` overflows → revert.
5. All subsequent deposit calls revert identically until the mock/feed returns a positive value.

Foundry test sketch:
```solidity
function test_negativePrice_depositDoS() public {
    mockFeed.setPrice(-1);
    vm.expectRevert(); // arithmetic overflow
    depositPool.depositAsset(stETH, 1 ether, 0, "");
}
```
