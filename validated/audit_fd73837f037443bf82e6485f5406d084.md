Audit Report

## Title
Missing `price > 0` Validation in `ChainlinkPriceOracle.getAssetPrice()` Enables Withdrawal DoS - (File: contracts/oracles/ChainlinkPriceOracle.sol)

## Summary

`ChainlinkPriceOracle.getAssetPrice()` casts the raw `int256` Chainlink answer to `uint256` without verifying `price > 0`. A zero return from the feed propagates directly into `LRTWithdrawalManager.getExpectedAssetAmount()`, causing a division-by-zero revert that blocks all new withdrawal initiations and instant withdrawals for the affected LST. The same codebase applies the correct guard in `ChainlinkOracleForRSETHPoolCollateral.getRate()`, confirming this is a code-level oversight.

**Note on the insolvency claim:** The report's negative-price insolvency path is not exploitable. For any `price < 0`, the two's-complement reinterpretation yields a value near `type(uint256).max`. The subsequent `uint256(price) * 1e18` multiplication in `getAssetPrice()` overflows and reverts under Solidity 0.8's checked arithmetic. The only concrete, reachable impact is the zero-price DoS.

## Finding Description

`ChainlinkPriceOracle.getAssetPrice()` at line 52–54 performs no sign or zero check:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

When `price == 0`, `getAssetPrice()` returns `0`. This zero propagates into two public entry points:

1. `LRTWithdrawalManager.getExpectedAssetAmount()` (line 593): `underlyingToReceive = amount * lrtOracle.rsETHPrice() / lrtOracle.getAssetPrice(asset)` — division by zero, reverts.
2. `LRTWithdrawalManager.initiateWithdrawal()` (line 168) calls `getExpectedAssetAmount()` directly, so every new withdrawal request for the affected asset reverts.
3. `LRTWithdrawalManager.instantWithdrawal()` (line 228) also calls `getExpectedAssetAmount()`, blocking instant withdrawals.

`LRTDepositPool.getRsETHAmountToMint()` (line 520) computes `(amount * 0) / rsETHPrice = 0`, which is caught by the `minRSETHAmountExpected` slippage guard in `_beforeDeposit()` (line 667) for any caller who sets a non-zero minimum — so the deposit path has a partial mitigation.

By contrast, `ChainlinkOracleForRSETHPoolCollateral.getRate()` (line 32) explicitly guards: `if (ethPrice <= 0) revert InvalidPrice();` before the same cast-and-scale pattern.

## Impact Explanation

**Temporary or permanent freezing of funds — Critical/Medium.**

When the Chainlink LST/ETH feed for any supported asset (stETH, ETHx, sfrxETH) returns `0`, all calls to `initiateWithdrawal` and `instantWithdrawal` for that asset revert. Users holding rsETH who wish to redeem for that LST cannot queue new requests. Existing already-queued requests (which store `expectedAssetAmount` at initiation time) can still be completed via `completeWithdrawal`, so funds already in the queue are not frozen. However, any rsETH holder who has not yet initiated a withdrawal is locked out for the duration of the zero-price condition. If the feed is permanently deprecated without replacement, this becomes a permanent freeze.

## Likelihood Explanation

Chainlink feeds can return `0` in documented edge cases: a newly deployed feed with no completed round, a deprecated feed, or a circuit-breaker activation that sets the answer to `0`. These are not attacker-controlled events, but they are within the known operational envelope of Chainlink infrastructure. The protocol has no control over when a feed enters this state, and the missing guard is entirely within the protocol's control to fix — as evidenced by the identical guard already present in `ChainlinkOracleForRSETHPoolCollateral`.

## Recommendation

Mirror the guard already present in `ChainlinkOracleForRSETHPoolCollateral.getRate()`:

```solidity
(, int256 price,,,) = priceFeed.latestRoundData();
if (price <= 0) revert InvalidPrice();
return uint256(price) * 1e18 / 10 ** uint256(priceFeed.decimals());
```

## Proof of Concept

1. Deploy a mock `AggregatorV3Interface` whose `latestRoundData()` returns `(1, 0, block.timestamp, block.timestamp, 1)` (price = 0).
2. Register it as the price feed for stETH via `ChainlinkPriceOracle.updatePriceFeedFor(stETH, mockFeed)`.
3. Call `LRTWithdrawalManager.initiateWithdrawal(stETH, amount, "")` from any rsETH holder.
4. Execution reaches `getExpectedAssetAmount(stETH, amount)` → `amount * rsETHPrice / 0` → arithmetic panic revert.
5. All subsequent `initiateWithdrawal` and `instantWithdrawal` calls for stETH revert until the feed is replaced.

Foundry fork test outline:
```solidity
function test_zeroPriceDoS() public {
    MockChainlinkFeed zeroFeed = new MockChainlinkFeed(0, 8);
    vm.prank(lrtManager);
    chainlinkOracle.updatePriceFeedFor(stETH, address(zeroFeed));

    vm.prank(rsETHHolder);
    vm.expectRevert(); // division by zero
    withdrawalManager.initiateWithdrawal(stETH, 1 ether, "");
}
