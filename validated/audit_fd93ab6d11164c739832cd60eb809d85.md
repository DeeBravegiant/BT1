### Title
Unvalidated `builder` Field in `ProposalInit` Allows Proposer to Inject Arbitrary Sequencer Address into Block Hash and Execution Context — (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`ProposalInit.builder` is accepted verbatim from the network and passed directly into the block execution context as `sequencer_address`, into `PartialBlockHashComponents.sequencer`, and into the committed block header — but `is_proposal_init_valid` never checks it against any locally-known expected value. A malicious-but-legitimately-selected proposer can set `builder` to any arbitrary address, causing every validator to execute the block with the wrong sequencer address, commit a wrong block hash, and store a wrong block header.

### Finding Description

`ProposalInit` carries a `builder` field (the "address of the one who builds/sequences the block"): [1](#0-0) 

During `build_proposal`, the proposer sets this field from its own local config: [2](#0-1) 

The comment at line 806 even acknowledges the field is not yet sourced from the committee: [3](#0-2) 

During `validate_proposal`, `is_proposal_init_valid` checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `starknet_version`, `version_constant_commitment`, all four L1 gas price fields, and `fee_proposal_fri`. It does **not** check `builder`: [4](#0-3) 

The unchecked `builder` value is then passed directly into `convert_to_sn_api_block_info` as `sequencer_address`: [5](#0-4) 

This `BlockInfo` is forwarded to the batcher via `validate_block`, so every transaction in the block executes with the proposer-supplied `sequencer_address`. The same value is captured in `PartialBlockHashComponents`: [6](#0-5) 

`sequencer` is chained into `calculate_block_hash`: [7](#0-6) 

And the same `init.builder` is written into the committed block header via `update_state_sync_with_new_block`: [8](#0-7) 

Because the validator's batcher re-executes with the proposer-supplied `builder`, the `ProposalCommitment` (partial block hash) computed by both sides matches, so the `ProposalFinMismatch` guard does not fire: [9](#0-8) 

Consensus reaches agreement on a block whose sequencer address, block hash, and execution context are all controlled by the proposer.

### Impact Explanation

**Wrong execution result (High):** Contracts calling `get_sequencer_address()` in execute mode receive the attacker-controlled address. Any contract that uses the sequencer address for access control, fee routing, or conditional logic will behave incorrectly for the entire block.

**Wrong block hash committed (High):** `PartialBlockHashComponents.sequencer` is part of the Poseidon hash chain. A manipulated `builder` produces a different `PartialBlockHash` and therefore a different `ProposalCommitment` and final `BlockHash` than would be produced with the correct sequencer address.

**Wrong block header stored (High):** The `sequencer` field of the committed `BlockHeaderWithoutHash` reflects the attacker-supplied address, corrupting the authoritative RPC view (`starknet_getBlockWithTxHashes`, etc.).

### Likelihood Explanation

The attack requires the adversary to be the legitimately selected proposer for a round (i.e., a Byzantine validator node). In a Tendermint network with `f < n/3` Byzantine validators, this is within the standard threat model. No additional privilege is required beyond being selected as proposer. The `proposer` identity is checked at the consensus layer, but `builder` is not checked anywhere in the validation path.

### Recommendation

Add `builder` to `ProposalInitValidation` and enforce it in `is_proposal_init_valid`, comparing `init_proposed.builder` against the locally-configured `builder_address` (or, once available, the committee-derived value referenced in the TODO). This mirrors how `l2_gas_price_fri`, `l1_da_mode`, and `starknet_version` are already validated against locally-known expected values.

### Proof of Concept

1. A malicious validator node is selected as proposer for round `r` at height `h`.
2. It constructs `ProposalInit { builder: ContractAddress(0xdeadbeef), ... }` with all other fields valid.
3. It streams this `ProposalInit` followed by valid transaction batches and a `ProposalFin` whose `proposal_commitment` is computed using `0xdeadbeef` as the sequencer address.
4. Each validator node calls `is_proposal_init_valid` — the `builder` field is not checked, so validation passes.
5. Each validator calls `batcher.validate_block(ValidateBlockInput { block_info: convert_to_sn_api_block_info(&init), ... })`, executing all transactions with `sequencer_address = 0xdeadbeef`.
6. The batcher returns a `ProposalCommitment` derived from `PartialBlockHashComponents { sequencer: 0xdeadbeef, ... }`.
7. This matches `ProposalFin.proposal_commitment`, so `ProposalFinMismatch` is not triggered.
8. Consensus votes `Precommit` on the commitment; `decision_reached` commits the block.
9. The committed block header has `sequencer = 0xdeadbeef`, the block hash encodes `0xdeadbeef`, and any contract that called `get_sequencer_address()` during execution received `0xdeadbeef`.

#[instrument(level = "warn", skip_all, fields(?proposal_init_validation, ?init_proposed))]
async fn is_proposal_init_valid(
    proposal_init_validation: &ProposalInitValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        proposal_init_validation.previous_proposal_init.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + proposal_init_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now,
                proposal_init_validation.block_timestamp_window_seconds,
                init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.starknet_version != proposal_init_validation.starknet_version {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "starknet_version mismatch: expected={:?}, proposed={:?}",
                proposal_init_validation.starknet_version, init_proposed.starknet_version
            ),
        ));
    }
    // `version_constant_commitment` is proposer-supplied (network-derived). It is not yet a real
    // commitment (see `expected_version_constant_commitment`): the only valid value is the
    // sentinel, so reject anything else. Enforcing the same value the proposer emits keeps the two
    // sides in lockstep, so a real value cannot ship on one side without the other.
    let expected_commitment = expected_version_constant_commitment();
    if init_proposed.version_constant_commitment != expected_commitment {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!(
                "version_constant_commitment mismatch: expected={expected_commitment:?}, \
                 proposed={:?}",
                init_proposed.version_constant_commitment
            ),
        ));
    }
    if !(init_proposed.height == proposal_init_validation.height
        && init_proposed.l1_da_mode == proposal_init_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == proposal_init_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            "ProposalInit validation failed".to_string(),
        ));
