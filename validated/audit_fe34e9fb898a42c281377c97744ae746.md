### Title
Untrusted Peer Can Inject Arbitrary `launcher_id` into DataLayer Mirror Store via Unvalidated Coin Spend Solution — (`chia/data_layer/data_layer_wallet.py`)

### Summary

`DataLayerWallet.coin_added` fetches the parent coin's spend from the peer to extract a `launcher_id` via `get_mirror_info`. The `fetch_coin_spend` helper validates the puzzle hash against the coin's on-chain `puzzle_hash`, but **does not validate the solution**. An untrusted peer can return the correct puzzle (passing the hash check) paired with a crafted solution that encodes an attacker-chosen `launcher_id` in the `CREATE_COIN` memos. If that `launcher_id` is one the victim is already tracking, the mirror record is written to `dl_store` with the injected value.

### Impact Explanation

The attacker can associate any real mirror coin (with `puzzle_hash == MIRROR_PUZZLE_HASH`) with an arbitrary `launcher_id` that the victim is tracking. Consequences:

- The victim's `dl_store` mirror table is corrupted: a coin the attacker chose is permanently linked to a launcher the victim owns/tracks.
- `get_mirrors_for_launcher` returns the injected record, causing the DataLayer service to advertise or attempt to sync from incorrect mirror URLs.
- The wallet subscribes to the injected coin via `add_interested_coin_ids`, wasting resources and potentially masking the real mirror's lifecycle events.
- When the injected mirror coin is spent, `singleton_removed` deletes the mirror record, silently removing what the wallet believes is a legitimate mirror for the victim's launcher.

This is squarely within **High: Corruption of Data Layer root/store state with direct security impact**. [6](#0-5) 

### Likelihood Explanation

**Preconditions (all realistic):**

1. Victim has a `DataLayerWallet` tracking ≥1 launcher (standard for any DL user).
2. A real mirror coin exists on-chain with `puzzle_hash == MIRROR_PUZZLE_HASH` — trivially created by anyone sending mojos to that fixed hash.
3. The victim's wallet receives the coin state for that mirror coin. The wallet state manager explicitly routes any coin with `puzzle_hash == MIRROR_PUZZLE_HASH` to the `DataLayerWallet` (line 1837), and the wallet subscribes to the mirror puzzle hash when it creates its own mirrors.
4. The attacker controls one peer the victim connects to — a standard untrusted-peer threat model.

The attacker needs to know one `launcher_id` the victim tracks; these are public on-chain values. [7](#0-6) 

### Recommendation

In `fetch_coin_spend`, after receiving the solution, verify it against the on-chain removals root (the same merkle-proof path already used for coin existence). Alternatively, in `coin_added`, cross-check the extracted `launcher_id` against the actual on-chain memos committed in the block (e.g., via the hint index or by re-deriving the expected memos from the confirmed transaction). At minimum, the solution should be validated the same way coin states are validated — against a block header the wallet has already verified. [2](#0-1)

async def fetch_coin_spend(height: uint32, coin: Coin, peer: WSChiaConnection) -> CoinSpend:
    solution_response = await peer.call_api(
        FullNodeAPI.request_puzzle_solution, RequestPuzzleSolution(coin.name(), height)
    )
    if solution_response is None or not isinstance(solution_response, RespondPuzzleSolution):
        raise PeerRequestException(f"Was not able to obtain solution {solution_response}")
    coin_id = coin.name()
    if solution_response.response.puzzle.get_tree_hash() != coin.puzzle_hash:
        raise PeerRequestException(f"Peer returned wrong puzzle hash for coin {coin_id}")
    if solution_response.response.coin_name != coin_id:
        raise PeerRequestException(f"Peer returned wrong coin name in puzzle solution for coin {coin_id}")

    return make_spend(
        coin,
        solution_response.response.puzzle,
        solution_response.response.solution,
    )

def get_mirror_info(
    parent_puzzle: Program | SerializedProgram, parent_solution: Program | SerializedProgram
) -> tuple[bytes32, list[bytes]]:
    assert type(parent_puzzle) is type(parent_solution)
    _, conditions = run_with_cost(parent_puzzle, INFINITE_COST, parent_solution)
    for condition in conditions.as_iter():
        if (
            condition.first().as_python() == ConditionOpcode.CREATE_COIN
            and condition.at("rf").as_python() == create_mirror_puzzle().get_tree_hash()
        ):
            memos: list[bytes] = condition.at("rrrf").as_python()
            launcher_id = bytes32(memos[0])
            return launcher_id, [url for url in memos[1:]]
    raise ValueError("The provided puzzle and solution do not create a mirror coin")

    async def is_launcher_tracked(self, launcher_id: bytes32) -> bool:
        async with self.db_wrapper.reader_no_transaction() as conn:
            cursor = await conn.execute("SELECT COUNT(*) from singleton_records WHERE launcher_id=?", (launcher_id,))
            row = await cursor.fetchone()
            await cursor.close()
        if row is not None:
            count: int = row[0]
            return count > 0
        else:
            return False
