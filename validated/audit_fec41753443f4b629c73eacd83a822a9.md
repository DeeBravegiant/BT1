Audit Report

## Title
Dust Staker Earns Zero Rewards Due to Integer Truncation in `earned()` — (`contracts/KERNEL/KernelDepositPool.sol`)

## Summary
The `earned()` function performs integer division by `DECIMAL_PRECISION` (1e18) after multiplying `balanceOf[_account]` by the per-token reward delta. For stakers whose balance is sufficiently small relative to `totalKernelStaked`, this division truncates to zero on every update cycle. The `updateReward` modifier advances `userRewardPerTokenPaid[_account]` to the current checkpoint even when `earned()` returns zero, permanently discarding the truncated micro-reward with no recovery path.

## Finding Description
`rewardPerToken()` accumulates a global index scaled up by `DECIMAL_PRECISION`:

```solidity
// KernelDepositPool.sol:412-413
rewardPerTokenStored + (rewardRate * (lastTimeRewardApplicable() - updatedAt) * DECIMAL_PRECISION)
    / totalKernelStaked;
```

`earned()` reverses the scaling via integer division:

```solidity
// KernelDepositPool.sol:422-423
return (balanceOf[_account] * (rewardPerToken() - userRewardPerTokenPaid[_account]) / DECIMAL_PRECISION)
    + rewards[_account];
```

The effective earned amount simplifies to:

```
balanceOf * rewardRate * elapsed / totalKernelStaked
```

When this product is less than 1, Solidity truncates to zero. The `updateReward` modifier then stores this zero and advances the checkpoint:

```solidity
// KernelDepositPool.sol:237-238
rewards[_account] = earned(_account);           // stores 0
userRewardPerTokenPaid[_account] = rewardPerTokenStored;  // checkpoint advanced past truncated delta
```

Every subsequent call to `stake`, `withdraw`, or `getReward` repeats this cycle: the truncated delta is discarded and the checkpoint is moved forward, making the loss permanent and cumulative. There is no `sweep`, `recover`, or admin rescue function in the contract.

**Concrete example:**
- `totalKernelStaked = 1e24` (1 million KERNEL, 18 decimals)
- `rewardRate = 1e18` (1 reward token/second)
- `elapsed = 604,800 s` (7 days)
- `balanceOf = 1 wei`
- `earned = 1 × 1e18 × 604800 / 1e24 = 0.6048 → truncates to 0`

## Impact Explanation
**Low — Contract fails to deliver promised returns, but doesn't lose value.**

A dust staker receives zero reward tokens despite staking for a full reward period, violating the invariant that every staker accrues nonzero rewards proportional to their stake over any sufficiently long period. The absolute value of the stranded rewards per dust staker is sub-wei of the reward token (less than 1 wei in the example above), so there is no material financial loss. The contract does not lose value in any economically meaningful sense, but the behavioral promise of proportional reward accrual is broken for dust-level positions.

## Likelihood Explanation
Any unprivileged user who calls `stake()` with a balance satisfying `balanceOf * rewardRate * elapsed < totalKernelStaked` is affected. The condition is fully permissionless and requires no special role or coordination. With large pools and moderate reward rates, the affected threshold can be non-trivial in wei terms (e.g., up to ~1 wei KERNEL in the example). The path is repeatable across any reward period.

## Recommendation
Enforce a minimum stake amount that guarantees at least 1 wei of reward is earned per period, i.e., require `balanceOf * rewardRate * duration >= totalKernelStaked` at stake time. Alternatively, accumulate truncated remainders at the `rewardPerToken` level and carry them forward rather than discarding them. A simpler mitigation is to add an admin `recoverExcessRewards()` function so stranded dust can be reclaimed and redistributed to active stakers.

## Proof of Concept
```solidity
function testDustStakerEarnsZero() public {
    // 1 million KERNEL already staked by whale
    uint256 whaleStake = 1_000_000e18;
    kernelToken.mint(whale, whaleStake);
    vm.prank(whale); pool.stake(whaleStake);

    // Admin starts reward period: 1 reward token/second for 7 days
    rewardsToken.mint(admin, 604_800e18);
    vm.prank(admin); pool.setRewardsDuration(7 days);
    vm.prank(admin); pool.notifyRewardAmount(604_800e18);

    // Dust staker stakes 1 wei
    kernelToken.mint(dustStaker, 1);
    vm.prank(dustStaker); pool.stake(1);

    // Fast-forward full reward period
    vm.warp(block.timestamp + 7 days);

    // Claim rewards
    vm.prank(dustStaker); pool.getReward();

    // earned = 1 * 1e18 * 604800 / 1e24 = 0.6048 → 0
    assertEq(rewardsToken.balanceOf(dustStaker), 0);
    assertEq(pool.rewards(dustStaker), 0);
}
```
