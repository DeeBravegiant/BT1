### Title
Missing CKD Output Validation for `AppPublicKey` Variant Allows Single Malicious Participant to Forge Key Derivation Response - (File: `crates/contract/src/lib.rs`)

---

### Summary

`respond_ckd` in `MpcContract` validates the CKD response only when the request carries the `AppPublicKeyPV` variant. When the request uses the `AppPublicKey` variant, the validation branch is a no-op. A single attested participant — strictly below the signing threshold — can therefore call `respond_ckd` with an arbitrary `CKDResponse`, and the contract will accept it, resolving the user's yield with a forged key derivation output.

---

### Finding Description

In `respond_ckd`, after verifying the caller is an attested participant, the function branches on the request's `app_public_key` field:

```rust
match &request.app_public_key {
    dtos::CKDAppPublicKey::AppPublicKey(_) => {}          // ← no validation
    dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
        if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
            env::panic_str("CKD output check failed");
        }
    }
}
```

For `AppPublicKeyPV`, `ckd_output_check` verifies the response against the domain's BLS12-381 root public key. For `AppPublicKey`, the branch is empty. The function then unconditionally calls `resolve_yields_for`, which removes the pending request from the map and resumes every queued yield with the supplied — entirely unvalidated — response bytes. [1](#0-0) 

This is structurally analogous to the WithdrawProxy M-16 finding. Just as `s.finalAuctionEnd == 0` before any auction starts, allowing the `onlyWhenNoActiveAuction` guard to pass and enabling premature redemption, the `AppPublicKey` branch here is an empty guard — the intended safety check is simply absent, and the first caller to submit any response wins. In both cases the contract's own lifecycle/validation logic contains a reachable code path where the check is silently skipped, and an adversary exploits that gap to corrupt the output received by the victim. [2](#0-1) 

The `resolve_yields_for` helper removes the request on first call, so all subsequent honest-node responses return `RequestNotFound` and are discarded. [3](#0-2) 

---

### Impact Explanation

A single attested participant can:

1. Monitor the chain for a `request_app_private_key` call that uses `AppPublicKey`.
2. Immediately call `respond_ckd` with an arbitrary `CKDResponse` of their choosing.
3. Because `resolve_yields_for` removes the request on first call, all subsequent honest-node responses return `RequestNotFound`.
4. The user's yield is resumed with the forged key derivation output.

The attacker can supply a key they control, effectively substituting their own derived key for the user's. This constitutes **unauthorized confidential key derivation output without the required threshold participant authorization** — matching the Critical impact tier: *"Unauthorized transaction execution, threshold signature issuance, or confidential key derivation output without the required participant authorization."*

---

### Likelihood Explanation

- Requires only **one** attested participant to be malicious — well below the signing threshold.
- The attack window is the block interval between the user's `request_app_private_key` transaction landing and the honest nodes' `respond_ckd` transactions landing — a realistic race condition on a live network.
- No special privileges, leaked keys, or physical TEE attacks are needed beyond holding a valid attestation.
- `AppPublicKey` is a documented, production-facing variant of `CKDAppPublicKey`, so real user traffic will exercise this path.

---

### Recommendation

Add the same `ckd_output_check` validation for the `AppPublicKey` variant, or — if the derived key is intentionally public and no secrecy guarantee is claimed — at minimum verify that the response is cryptographically consistent with the domain's root public key and the request's derivation path, preventing arbitrary forgery. The empty match arm should never be the final word on whether a response is accepted.

---

### Proof of Concept

1. Alice calls `request_app_private_key` with `CKDRequestArgs { app_public_key: AppPublicKey(alice_pk), domain_id, derivation_path }`.
2. The contract stores the pending CKD request and yields.
3. Mallory (a single attested participant) observes the transaction and immediately calls `respond_ckd(request, forged_response)` where `forged_response` contains a key Mallory controls.
4. `respond_ckd` passes all checks: Mallory is an attested participant, the protocol is running, and the `AppPublicKey` branch is a no-op. [4](#0-3) 
5. `resolve_yields_for` removes the request and resumes Alice's yield with `forged_response`. [5](#0-4) 
6. Honest nodes' subsequent `respond_ckd` calls return `RequestNotFound` because the request was already removed. [6](#0-5) 
7. Alice receives Mallory's forged key derivation output instead of the correct MPC-derived key.

```rust
    #[handle_result]
    pub fn respond_ckd(&mut self, request: CKDRequest, response: CKDResponse) -> Result<(), Error> {
        let signer = Self::assert_caller_is_signer();
        log!("respond_ckd: signer={}, request={:?}", &signer, &request);

        if !self.protocol_state.is_running_or_resharing() {
            return Err(InvalidState::ProtocolStateNotRunning.into());
        }

        if !self.accept_requests {
            return Err(TeeError::TeeValidationFailed.into());
        }

        self.assert_caller_is_attested_participant_and_protocol_active();

        let PublicKeyExtended::Bls12381 {
            public_key: dtos::PublicKey::Bls12381(public_key),
        } = self.public_key_extended(request.domain_id)?
        else {
            env::panic_str("Domain is not compatible with CKD (expected Bls12381 curve)");
        };

        match &request.app_public_key {
            dtos::CKDAppPublicKey::AppPublicKey(_) => {}
            dtos::CKDAppPublicKey::AppPublicKeyPV(app_pk) => {
                if !ckd_output_check(&request.app_id, &response, app_pk, &public_key) {
                    env::panic_str("CKD output check failed");
                }
            }
        }

        pending_requests::resolve_yields_for(
            &mut self.pending_ckd_requests,
            &request,
            serde_json::to_vec(&response).unwrap(),
        )
    }
```

```rust
pub(crate) fn resolve_yields_for<K>(
    requests: &mut LookupMap<K, Vec<YieldIndex>>,
    request: &K,
    response_bytes: Vec<u8>,
) -> Result<(), Error>
where
    K: BorshSerialize + BorshDeserialize + Clone + Ord,
{
    let resumed = requests
        .remove(request)
        .unwrap_or_default()
        .into_iter()
        .map(|YieldIndex { data_id }| {
            env::promise_yield_resume(&data_id, response_bytes.clone());
        })
        .count();

    if resumed > 0 {
        Ok(())
    } else {
        Err(InvalidParameters::RequestNotFound.into())
    }
}
```
